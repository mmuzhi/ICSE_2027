=== SOURCE CODE (java) ===
package org.example;

public class NumberConverter {

    public static String decimalToBinary(int decimalNum) {
        return Integer.toBinaryString(decimalNum);
    }

    public static int binaryToDecimal(String binaryNum) {
        return Integer.parseInt(binaryNum, 2);
    }

    public static String decimalToOctal(int decimalNum) {
        return Integer.toOctalString(decimalNum);
    }

    public static int octalToDecimal(String octalNum) {
        return Integer.parseInt(octalNum, 8);
    }

    public static String decimalToHex(int decimalNum) {
        return Integer.toHexString(decimalNum);
    }

    public static int hexToDecimal(String hexNum) {
        return Integer.parseInt(hexNum, 16);
    }
}

The user wants to translate a Java class `NumberConverter` into Python.
The Java class has static methods for converting between decimal, binary, octal, and hex.
I need to keep the behavior identical, including edge cases and exceptions.

Let's look at the Java methods:
1. `decimalToBinary(int decimalNum)`
   Java: `Integer.toBinaryString(decimalNum)`
   Python equivalent: `bin(decimalNum)[2:]` or `format(decimalNum, 'b')`
   Wait, Java's `Integer.toBinaryString` handles negative numbers by returning the binary representation of the two's complement. For example, `Integer.toBinaryString(-1)` returns `"11111111111111111111111111111111"`.
   Python's `bin(-1)` returns `'-0b1'`.
   To match Java's behavior for negative numbers:
   If `decimalNum >= 0`, `format(decimalNum, 'b')` or `bin(decimalNum)[2:]`.
   If `decimalNum < 0`, `format(decimalNum & 0xFFFFFFFF, 'b')` or `bin(decimalNum & 0xFFFFFFFF)[2:]`.
   Wait, `Integer.toBinaryString(-1)` is 32 bits. `Integer.toBinaryString(-1)` returns `"11111111111111111111111111111111"`.
   What about `Integer.toOctalString(-1)`? It returns `"37777777777"`.
   What about `Integer.toHexString(-1)`? It returns `"ffffffff"`.
   Let's verify:
   Java `Integer.toBinaryString(-5)` -> `"11111111111111111111111111111011"`
   Python: `format(-5 & 0xFFFFFFFF, 'b')` -> `"11111111111111111111111111111011"`
   Java `Integer.toOctalString(-5)` -> `"37777777773"`
   Python: `format(-5 & 0xFFFFFFFF, 'o')` -> `"37777777773"`
   Java `Integer.toHexString(-5)` -> `"fffffffb"`
   Python: `format(-5 & 0xFFFFFFFF, 'x')` -> `"fffffffb"`

   Wait, what if `decimalNum` is positive?
   Java `Integer.toBinaryString(5)` -> `"101"`
   Python `format(5 & 0xFFFFFFFF, 'b')` -> `"101"` (since leading zeros are not printed for positive numbers, except for 0 which is "0").
   Wait, `format(0, 'b')` -> `"0"`.

import re

class NumberConverter:

    @staticmethod
    def decimalToBinary(decimalNum):
        return format(decimalNum & 0xFFFFFFFF, 'b')

    @staticmethod
    def binaryToDecimal(binaryNum):
        try:
            if not re.fullmatch(r'[+-]?[01]+', binaryNum):
                raise ValueError(f"For input string: \"{binaryNum}\"")
        except TypeError:
            raise ValueError("For input string: \"null\"")
        val = int(binaryNum, 2)
        if val > 2147483647 or val < -2147483648:
            raise ValueError(f"For input string: \"{binaryNum}\"")
        return val

    @staticmethod
    def decimalToOctal(decimalNum):
        return format(decimalNum & 0xFFFFFFFF, 'o')

    def octalToDecimal(octalNum):
