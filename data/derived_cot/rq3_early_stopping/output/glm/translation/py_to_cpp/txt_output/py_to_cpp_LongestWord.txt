import string


class LongestWord:

    def __init__(self):
        self.word_list = []

    def add_word(self, word):
        self.word_list.append(word)

    def find_longest_word(self, sentence):
        longest_word = ""

        std::string cleaned;

                cleaned += c;
            }
        }
        
        // sentence = re.split(' ', sentence)
        std::vector<std::string> words;
        std::string current_word;

Is there any edge case where `std::tolower` behaves differently from Python's `str.lower()`?
For ASCII characters, `std::tolower` perfectly matches `str.lower()`.
For UTF-8, Python 3's `str.lower()` handles multi-byte characters properly, while C++'s `std::tolower` only processes one byte at a time. Given the context of typical programming problems and the fact that `string.punctuation` is ASCII-only, it's safe to assume ASCII input. If we needed full Unicode support, we'd need a library like ICU, but that's overkill and not standard C++. The current solution is the best standard C++ equivalent.

One small detail: `word.length() > longest_word.length()`.
In Python, `len(word) > len(longest_word)`.
`std::string::length()` returns `size_t`, which is unsigned. `>` comparison works correctly.
If `word` is `""` and `longest_word` is `""`, `0 > 0` is false. Matches.
