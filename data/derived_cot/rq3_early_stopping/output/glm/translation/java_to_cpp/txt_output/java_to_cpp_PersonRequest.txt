### Key translation decisions:

| Java | C++ | Rationale |
|------|-----|-----------|
| `String` (nullable reference) | `std::optional<std::string>` | Java `String` can be `null`; `std::optional` is the C++ equivalent for nullable value semantics. |
| `null` / `return null` | `std::nullopt` | Direct equivalent for "no value". |
| `name == null` | `!name.has_value()` | Null-check idiom for `std::optional`. |
| `name.isEmpty()` | `name->empty()` | Arrow operator to access the contained value. |
| `sex.equals("Man")` | `*sex == "Man"` | Dereference the optional and use `operator==`. |
| `phoneNumber.matches("\\d{11}")` | `std::all_of(..., ::isdigit)` | Since length is already verified as 11, checking all chars are digits is equivalent. `unsigned char` cast avoids UB with `std::isdigit`. |
| `private` validate methods | `static` private methods | They don't access instance state; making them `static` is idiomatic without changing behavior. |
| Getters return `String` | Getters return `const std::optional<std::string>&` | Reference return avoids unnecessary copies, mirroring Java's reference return semantics. |
