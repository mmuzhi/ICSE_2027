### Key translation decisions:

| Java | C++ | Rationale |
|---|---|---|
| `Object` parameter with `instanceof` | `std::variant<Tuple, std::vector<Tuple>>` | Closest safe equivalent to Java's runtime type checking; enforces valid types at compile time |
| `List<Integer>` | `std::vector<int>` | Idiomatic C++ for a dynamically-sized list of ints |
| `IllegalArgumentException` | `std::invalid_argument` | Direct C++ equivalent |
| `ArrayList<Double>` | `std::vector<double>` | Same as above |
| `.stream().mapToDouble(...).average().orElse(0.0)` | `std::accumulate` + division + empty check | Replicates the "average or 0.0 if empty" semantics |
| Inner `public` class `Tuple` | Nested `public` class `Tuple` | Preserves the same scoping and access pattern |
| `subList.get(i)` | `subList[i]` | Idiomatic C++ vector element access |
| Loop variable `int i` | `std::size_t i` | Correct type for vector indices in C++; arithmetic with `1.0`/`1.0 / (i+1)` still produces `double` results |
