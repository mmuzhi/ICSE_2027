=== SOURCE CODE (java) ===
package org.example;

import java.util.HashMap;
import java.util.Map;

public class BookManagement {
    private Map<String, Integer> inventory;

    public BookManagement() {
        this.inventory = new HashMap<>();
    }

    public void addBook(String title, int quantity) {
        if (inventory.containsKey(title)) {
            inventory.put(title, inventory.get(title) + quantity);
        } else {
            inventory.put(title, quantity);
        }
    }

    public void removeBook(String title, int quantity) throws Exception {
        if (!inventory.containsKey(title) || inventory.get(title) < quantity) {
            throw new Exception("Invalid operation");
        }
        int newQuantity = inventory.get(title) - quantity;
        if (newQuantity == 0) {
            inventory.remove(title);
        } else {
            inventory.put(title, newQuantity);
        }
    }

    public Map<String, Integer> viewInventory() {
        return new HashMap<>(inventory);
    }

    public int viewBookQuantity(String title) {
        return inventory.getOrDefault(title, 0);
    }
}

<think>
The user wants me to translate a Java class to C++ while keeping behavior identical.

Let me analyze the Java code:
1. It's a `BookManagement` class with a `Map<String, Integer>` called `inventory`.
2. `addBook(String title, int quantity)`: If the title exists, add quantity; otherwise, put the new entry.
3. `removeBook(String title, int quantity) throws Exception`: If the title doesn't exist or quantity is less than requested, throw an Exception. Otherwise, subtract quantity. If new quantity is 0, remove the entry; otherwise, update it.
4. `viewInventory()`: Returns a copy of the inventory map.
5. `viewBookQuantity(String title)`: Returns the quantity if the title exists, otherwise 0.

C++ Translation details:
- `Map<String, Integer>` -> `std::unordered_map<std::string, int>`
- `addBook` can just use `inventory[title] += quantity;` since `unordered_map` default constructs the value (0 for int) if the key doesn't exist. Wait, the Java code does `inventory.put(title, inventory.get(title) + quantity)`, which is exactly what `+=` does in C++. So `inventory[title] += quantity;` is perfectly identical.
- `removeBook` throws an `Exception` in Java. In C++, we can throw `std::runtime_error` or `std::invalid_argument` or a custom exception. `std::runtime_error` is a good fit.
- `viewInventory` returns a copy of the map. In C++, returning `std::unordered_map<std::string, int>` by value will make a copy (or move, but functionally identical to returning a copy).
- `viewBookQuantity` uses `getOrDefault`. In C++, we can use `auto it = inventory.find(title); if (it != inventory.end()) return it->second; return 0;` or simply `if (inventory.count(title)) return inventory[title]; return 0;` or `inventory.contains(title)` in C++20. Wait, `inventory.find` is the most idiomatic and safe way without modifying the map (unlike `operator[]` which inserts if not found).

Let's write out the C++ class.

```cpp
#include <string>
#include <unordered_map>
#include <stdexcept>

class BookManagement {
private:
    std::unordered_map<std::string, int> inventory;

public:
    BookManagement() = default;

    void addBook(const std::string& title, int quantity) {
        inventory[title] += quantity;
    }

    void removeBook(const std::string& title, int quantity) {
        auto it = inventory.find(title);
        if (it == inventory.end() || it->second < quantity) {
            throw std::runtime_error("Invalid operation");
        }
        int newQuantity = it->second - quantity;
        if (newQuantity == 0) {
            inventory.erase(it);
        } else {
            it->second = newQuantity;
        }
    }

    std::unordered_map<std::string, int> viewInventory() const {
        return inventory;
    }

    int viewBookQuantity(const std::string& title) const {
        auto it = inventory.find(title);
        if (it != inventory.end()) {
            return it->second;
        }
        return 0;
    }
};
```

Wait, are there any edge cases?
- `quantity` being negative? The Java code doesn't check for it. The C++ code doesn't check for it. Behavior is identical.
- `title` being an empty string? Both handle it the same way.
- Map ordering? Java `HashMap` has no ordering guarantees, C++ `std::unordered_map` has no ordering guarantees. So they are identical in this aspect. If I had used `std::map`, it would be sorted, which changes behavior (e.g. if someone iterates over the returned inventory, though not explicitly done here). `unordered_map` is the correct choice.

void removeBook(const std::string& title, int quantity) {
    if (inventory.count(title) == 0 || inventory[title] < quantity) {
