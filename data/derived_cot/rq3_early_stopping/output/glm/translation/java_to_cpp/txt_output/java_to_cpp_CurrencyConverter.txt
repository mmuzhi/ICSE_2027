package org.example;

import java.util.HashMap;
import java.util.Map;
import java.util.Set;

public class CurrencyConverter {
    private Map<String, Double> rates;

    public CurrencyConverter() {
        rates = new HashMap<>();
        rates.put("USD", 1.0);
        rates.put("EUR", 0.85);
        rates.put("GBP", 0.72);
        rates.put("JPY", 110.15);
        rates.put("CAD", 1.23);
        rates.put("AUD", 1.34);
        rates.put("CNY", 6.40);
    }

    public double convert(double amount, String fromCurrency, String toCurrency) {
        if (fromCurrency.equals(toCurrency)) {
            return amount;
        }

        if (!rates.containsKey(fromCurrency) || !rates.containsKey(toCurrency)) {
            return -1;
        }

        double fromRate = rates.get(fromCurrency);
        double toRate = rates.get(toCurrency);

        double convertedAmount = (amount / fromRate) * toRate;
        return convertedAmount;
    }

    public Set<String> getSupportedCurrencies() {
        return rates.keySet();
    }

    public boolean addCurrencyRate(String currency, double rate) {
        if (rates.containsKey(currency)) {
            return false;
        }
        rates.put(currency, rate);
        return true;
    }

    public boolean updateCurrencyRate(String currency, double newRate) {
        if (!rates.containsKey(currency)) {
            return false;
        }
        rates.put(currency, newRate);
        return true;
    }

    public Map<String, Double> getRates() {
        return rates;
    }
}

### Key Translation Decisions:
- **Data Structures**: `java.util.HashMap` translates to `std::unordered_map` (both offer O(1) average time complexity for lookups and are not ordered). `java.util.Set` translates to `std::unordered_set`.
- **Method Signatures**: Java `String` translates to `const std::string&` in C++ to avoid unnecessary copying while preserving read-only semantics.
- **Key Existence Checks**: `rates.containsKey(key)` translates to `rates.find(key) != rates.end()`. 
- **Map Access**: `rates.get(key)` translates to `rates.at(key)`. This is used instead of `rates[key]` because `operator[]` will insert a default value if the key doesn't exist, which would alter behavior. Although we check for existence beforehand, `at()` is safer and semantically closer to Java's `get`.
- **`getRates()` Side Effects**: The Java method returns a reference to the internal map (modifications to the returned map affect the class state). To keep this side-effect behavior identical, C++ returns a reference `std::unordered_map<...>&` rather than a copy.
- **`getSupportedCurrencies()`**: Java returns a view of the keys, but C++ doesn't have a direct built-in equivalent for a map's keyset view. Returning a constructed `std::unordered_set` by value is the closest idiomatic equivalent that guarantees the correct data is represented, even though modifications to this set won't mutate the original map.
