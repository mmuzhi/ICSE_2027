=== SOURCE CODE (java) ===
package org.example;

import java.util.HashMap;
import java.util.Map;

public class BookManagement {
    private Map<String, Integer> inventory;

    public BookManagement() {
        this.inventory = new HashMap<>();
    }

    public void addBook(String title, int quantity) {
        if (inventory.containsKey(title)) {
            inventory.put(title, inventory.get(title) + quantity);
        } else {
            inventory.put(title, quantity);
        }
    }

    public void removeBook(String title, int quantity) throws Exception {
        if (!inventory.containsKey(title) || inventory.get(title) < quantity) {
            throw new Exception("Invalid operation");
        }
        int newQuantity = inventory.get(title) - quantity;
        if (newQuantity == 0) {
            inventory.remove(title);
        } else {
            inventory.put(title, newQuantity);
        }
    }

    public Map<String, Integer> viewInventory() {
        return new HashMap<>(inventory);
    }

    public int viewBookQuantity(String title) {
        return inventory.getOrDefault(title, 0);
    }
}

The key aspects to preserve include the inventory management logic, exception handling, and the immutability of the returned inventory map.

### Approach
1. **Data Structure**: Use a dictionary in Python to represent the inventory, similar to the HashMap in Java.
2. **Method Translation**: Translate each Java method into a Python method, ensuring the same logic and behavior:
   - **addBook**: Update the quantity of a book or add it if it doesn't exist.
   - **removeBook**: Check if the requested quantity is available, throw an exception if not, and update the inventory accordingly.
   - **viewInventory**: Return a new dictionary copy to prevent modification of the internal state.
   - **viewBookQuantity**: Return the quantity of a book, defaulting to 0 if the book is not found.
3. **Exception Handling**: Use Python's `ValueError` to mimic the Java `Exception` thrown in the `removeBook` method.
4. **Immutability**: Return a new dictionary in `viewInventory` to ensure the internal state is not modified externally.

### Solution Code
```python
class BookManagement:
    def __init__(self):
        self.inventory = {}

    def add_book(self, title: str, quantity: int) -> None:
        if title in self.inventory:
            self.inventory[title] += quantity
        else:
            self.inventory[title] = quantity

    def remove_book(self, title: str, quantity: int) -> None:
        current_quantity = self.inventory.get(title, 0)
        if current_quantity < quantity:
            raise ValueError("Invalid operation")
        new_quantity = current_quantity - quantity
        if new_quantity == 0:
            del self.inventory[title]
        else:
            self.inventory[title] = new_quantity

    def view_inventory(self) -> dict:
        return self.inventory.copy()

    def view_book_quantity(self, title: str) -> int:
        return self.inventory.get(title, 0)
```

### Explanation
- **Initialization**: The `__init__` method initializes an empty dictionary for the inventory.
- **add_book**: Checks if the book exists in the inventory. If it does, it increments the quantity; otherwise, it adds the book with the given quantity.
- **remove_book**: Checks if the requested quantity is available. If not, it raises a `ValueError`. Otherwise, it updates the inventory, removing the book if the quantity drops to zero.
- **view_inventory**: Returns a copy of the inventory dictionary to prevent external modification.
- **view_book_quantity**: Returns the quantity of the specified book, defaulting to 0 if the book is not found.

This translation ensures that all edge cases and behaviors from the Java code are preserved in the Python implementation, adhering to the problem constraints and requirements.
