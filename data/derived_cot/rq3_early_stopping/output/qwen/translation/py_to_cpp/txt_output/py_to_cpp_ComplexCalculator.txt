To solve this problem, we need to translate a Python class that performs basic arithmetic operations on complex numbers into an equivalent C++ class. The solution must maintain the same behavior, including handling complex numbers correctly and throwing exceptions when division by zero occurs.

### Approach
1. **Complex Number Handling**: Use the C++ Standard Library's `std::complex` template to handle complex numbers, ensuring arithmetic operations are performed correctly.
2. **Exception Handling**: Implement division by zero by throwing a `std::runtime_error` exception, mirroring Python's behavior of raising a `ZeroDivisionError`.
3. **Static Methods**: Convert each arithmetic operation (addition, subtraction, multiplication, division) into static methods within the C++ class, maintaining the same parameter and return types as the Python counterpart.
4. **Edge Cases**: Ensure that division by zero is handled by checking the denominator before performing the division, providing an exception in such cases.

### Solution Code
```cpp
#include <complex>
#include <stdexcept>

class ComplexCalculator {
public:
    static std::complex<double> add(const std::complex<double>& c1, const std::complex<double>& c2) {
        return c1 + c2;
    }

    static std::complex<double> subtract(const std::complex<double>& c1, const std::complex<double>& c2) {
        return c1 - c2;
    }

    static std::complex<double> multiply(const std::complex<double>& c1, const std::complex<double>& c2) {
        return c1 * c2;
    }

    static std::complex<double> divide(const std::complex<double>& c1, const std::complex<double>& c2) {
        double denominator = c2.real() * c2.real() + c2.imag() * c2.imag();
        if (denominator == 0) {
            throw std::runtime_error("Division by zero");
        }
        double real = (c1.real() * c2.real() + c1.imag() * c2.imag()) / denominator;
        double imag = (c1.imag() * c2.real() - c1.real() * c2.imag()) / denominator;
        return std::complex<double>(real, imag);
    }
};
```
