To solve this problem, we need to partition a given string into the fewest number of substrings such that each letter appears in exactly one substring. The solution involves identifying the last occurrence of each character in the string and then traversing the string to determine the optimal partition points.

### Approach
1. **Identify Last Occurrences**: First, we create a dictionary to record the last occurrence index of each character in the string. This helps in knowing the farthest point each character appears.
2. **Traverse the String**: Using two pointers, `start` and `i`, we traverse the string. For each character at position `i`, we update the maximum last occurrence (`max_last`) encountered so far. If `i` reaches `max_last`, it means all characters from `start` to `i` can form a valid partition. We then record the length of this partition and reset `start` to the next position after `i`.
3. **Collect Results**: The lengths of these partitions are collected into a list and returned as the result.

### Solution Code
```python
class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        last_occurrence = {}
        for i in range(len(s)-1, -1, -1):
            if s[i] not in last_occurrence:
                last_occurrence[s[i]] = i
        
        i = 0
        start = 0
        max_last = 0
        ans = []
        while i < len(s):
            max_last = max(max_last, last_occurrence[s[i]])
            if i == max_last:
                ans.append(i - start + 1)
                start = i + 1
            i += 1
        
        return ans
```

### Explanation
1. **Last Occurrence Dictionary**: The dictionary `last_occurrence` is built by iterating from the end of the string to the beginning. This ensures that each entry in the dictionary holds the highest index where the character appears.
2. **Traversal and Partitioning**: The main loop traverses the string from left to right. For each character, it updates `max_last` to the highest last occurrence encountered. If the current index `i` matches `max_last`, it means all characters from the start of the current partition to `i` can be safely partitioned. The length of this partition is recorded, and the start is reset to the next position.
3. **Efficiency**: The algorithm efficiently processes each character exactly once, making it linear in time complexity, O(n), where n is the length of the string. The space complexity is O(k), where k is the number of distinct characters in the string, due to the dictionary storing last occurrences.
