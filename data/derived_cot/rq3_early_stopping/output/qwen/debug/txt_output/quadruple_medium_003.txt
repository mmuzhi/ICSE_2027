To solve this problem, we need to perform a level-order traversal (BFS) of a binary tree and return the nodes level by level from bottom to top. The initial code provided has several issues, including incorrect handling of `None` children, a syntax error in the reversal function, and improper return type when the root is `None`.

### Approach
1. **Level-Order Traversal (BFS)**: We use a queue to traverse the tree level by level. For each node, we process its value and enqueue its children (if they exist) to ensure only valid nodes are processed.
2. **Reversing Levels**: After collecting all levels from top to bottom, we reverse the list of levels to present them from bottom to top.
3. **Edge Case Handling**: If the root is `None`, we return an empty list instead of `None` to adhere to the expected return type.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        q, ans = [root], []
        while q:
            n = len(q)
            level = []
            for i in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        ans.reverse()
        return ans
```

### Explanation
1. **Initialization**: The function starts by checking if the root is `None`. If so, it returns an empty list.
2. **BFS Traversal**: A queue is initialized with the root node. For each level, the number of nodes is determined by the current queue length. Each node's value is added to the current level list, and its children (if they exist) are enqueued.
3. **Level Collection**: After processing all nodes in the current level, the level list is appended to the result list `ans`.
4. **Reversal**: The collected levels (from top to bottom) are reversed to present the levels from bottom to top before returning the result.

This approach efficiently handles the level-order traversal and ensures the result is formatted correctly from bottom to top while avoiding any issues with `None` children or incorrect syntax.
