The solution involves checking each possible integer from `n-1` down to `0` to see if it and its complement (i.e., `n - i`) meet the criteria.

### Approach
1. **Problem Analysis**: The task is to find two integers that add up to `n` without containing the digit '0'. The solution must efficiently check each possible integer starting from the largest (i.e., `n-1`) down to `0` to find the first valid pair.
2. **Intuition**: By iterating from `n-1` down to `0`, we can find the largest possible integer `a` such that both `a` and `n - a` do not contain the digit '0'. This ensures we return the first valid pair encountered during the iteration.
3. **Algorithm Selection**: The algorithm involves a simple loop that checks each integer in descending order. For each integer `i`, it verifies that both `i` and `n - i` do not contain the digit '0'. If such a pair is found, it is returned immediately.
4. **Complexity Analysis**: The time complexity is O(n) in the worst case, where `n` is the given integer. This is because we might need to check all integers from `n-1` down to `0` in the worst-case scenario. However, in practice, the solution is efficient as it typically finds a valid pair much sooner.
