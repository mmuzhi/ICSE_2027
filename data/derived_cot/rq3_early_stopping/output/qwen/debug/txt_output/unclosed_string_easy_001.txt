To solve this problem, we need to increment a number represented as a list of digits by 1 and return the result as a list of digits. The key insight is that the digits are given in a standard positional notation (most significant digit first), and we need to handle cases where incrementing the least significant digit might cause a cascade of carries, potentially increasing the number of digits.

### Approach
1. **Convert Digits to String**: The first step is to convert the list of digits into a string. This allows us to easily manipulate the number as a whole.
2. **Increment the Number**: Convert the string back into an integer, increment it by 1, and then convert it back to a string. This handles all cases, including those where incrementing the least significant digit causes a carry-over (e.g., [9] becomes [1, 0]).
3. **Convert Back to List of Digits**: Finally, convert the resulting string back into a list of integers, which is the required output format.

This approach efficiently handles all edge cases, such as numbers ending with one or more 9s, by leveraging Python's ability to handle arbitrarily large integers.

### Solution Code
```python
class Solution:
    def plusOne(self, digits: List[int]) -> List[int]:
        s = ''.join(map(str, digits))
        i = int(s) + 1
        li = list(map(int, str(i)))
        return li
```

### Explanation
1. **String Conversion**: The list of digits is converted into a string by joining each digit as a string. For example, [1, 2, 3] becomes "123".
2. **Integer Increment**: The string is converted to an integer, incremented by 1, and then converted back to a string. For example, "123" becomes 124, which is then converted to "124".
3. **List Conversion**: The resulting string is split into individual characters, each converted back to an integer, forming the output list. For example, "124" becomes [1, 2, 4].

This method ensures that all cases, including those with trailing 9s, are handled correctly by leveraging Python's built-in functions for string and integer manipulation. The solution is concise and efficient, with a time complexity of O(n), where n is the number of digits, due to the operations involved in converting between strings and integers.
