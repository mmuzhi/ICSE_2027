=== RAW ANSWER ===

To solve this problem, we need to find the value of the bottom-leftmost node in a binary tree. The bottom-leftmost node is defined as the node with the maximum depth and the smallest index (leftmost) in that depth. 

### Approach
The approach involves performing a breadth-first search (BFS) traversal of the tree. During the traversal, we keep track of the depth of each node. For each node encountered, if its depth is greater than the maximum depth recorded so far, we update the maximum depth and record the node's value as the potential answer. This ensures that by the end of the traversal, we have the leftmost node of the deepest level in the tree.

### Solution Code
```python
from collections import deque
from typing import Optional

# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def findBottomLeftValue(self, root: Optional[TreeNode]) -> int:
        if not root:
            return None
        queue = deque()
        queue.append((root, 0))
        max_depth = -1
        result = None
        
        while queue:
            node, depth = queue.popleft()
            if depth > max_depth:
                max_depth = depth
                result = node.val
            if node.left:
                queue.append((node.left, depth + 1))
            if node.right:
                queue.append((node.right, depth + 1))
                
        return result
```

### Explanation
1. **Initialization**: We start by checking if the root node is `None`. If it is, we return `None` since the tree is empty.
2. **BFS Setup**: We initialize a queue to perform BFS. The queue stores tuples of `(node, depth)`, where `node` is the current tree node and `depth` is its depth in the tree.
3. **Traversal**: We process each node in the queue by popping it from the front. For each node, if its depth is greater than the current maximum depth, we update the maximum depth and record its value as the result.
4. **Child Processing**: We enqueue the left and right children of the current node with their respective depths (current depth + 1).
5. **Result**: After processing all nodes, the result variable holds the value of the bottom-leftmost node, which is then returned.

This approach efficiently traverses the tree in BFS order, ensuring that we correctly identify the deepest and leftmost node by leveraging depth tracking during traversal. The time complexity is O(n), where n is the number of nodes in the tree, as each node is processed exactly once. The space complexity is also O(n) due to the queue used in BFS.
