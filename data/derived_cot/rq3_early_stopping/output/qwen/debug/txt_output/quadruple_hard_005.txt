=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays without merging them into a single array, which ensures optimal performance with a time complexity of O(m + n), where m and n are the lengths of the two arrays. The median is the middle value in an ordered list of numbers. If the combined length of the two arrays is even, the median is the average of the two middle numbers.

### Approach
1. **Initialization**: Start with two pointers, `p1` and `p2`, initialized to 0. These pointers will traverse the two arrays.
2. **Merging Process**: Compare elements from both arrays using the pointers. The smaller element is added to a result array, and the corresponding pointer is incremented.
3. **Remaining Elements**: After one array is exhausted, append the remaining elements from the other array to the result.
4. **Median Calculation**: Depending on whether the total number of elements is odd or even, compute the median by either taking the middle element directly or the average of the two middle elements.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        p1 = 0
        p2 = 0
        merged = []
        
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                merged.append(nums1[p1])
                p1 += 1
            else:
                merged.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            merged.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            merged.append(nums2[p2])
            p2 += 1
        
        total_length = len(merged)
        if total_length % 2 == 1:
            return float(merged[total_length // 2])
        else:
            return (merged[total_length // 2 - 1] + merged[total_length // 2]) / 2.0
```

### Explanation
1. **Initialization**: The pointers `p1` and `p2` start at the beginning of each array.
2. **Merging Process**: The while loop continues as long as both pointers are within their respective array bounds. The smaller element between `nums1[p1]` and `nums2[p2]` is added to the merged list, and the corresponding pointer is incremented.
3. **Remaining Elements**: Once one array is fully traversed, the remaining elements from the other array are added to the merged list.
4. **Median Calculation**: The median is determined by checking if the total length of the merged list is odd or even. For an odd length, the middle element is the median. For an even length, the average of the two middle elements is the median. The result is returned as a float to handle both cases appropriately.
