The player who cannot make a move loses. The solution involves sorting both players' stone piles in descending order and then simulating the game by alternately taking the largest remaining pile from each player's collection.

### Approach
1. **Sort the Piles**: Both Alice's and Bob's piles are sorted in descending order to prioritize taking the largest piles first, which is the optimal strategy.
2. **Simulate the Game**: Players take turns starting with Alice. On each turn, the current player takes the largest remaining pile from their own collection. The game continues until all piles are exhausted.
3. **Determine the Winner**: After processing all piles, compare the total stones taken by each player. The player with more stones wins; if they are equal, the game ends in a draw.

### Solution Code
```python
class Solution:
    def stoneGameVI(self, a: List[int], b: List[int]) -> int:
        a.sort(reverse=True)
        b.sort(reverse=True)
        alice = 0
        bob = 0
        n = len(a)
        for i in range(n):
            if i % 2 == 0:
                alice += a[i]
            else:
                bob += b[i]
        if alice > bob:
            return 1
        elif alice < bob:
            return -1
        else:
            return 0
```
