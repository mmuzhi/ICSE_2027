To solve this problem, we need to perform a level-order traversal (BFS) of a binary tree and return the nodes level by level from bottom to top. The initial code had two main issues: it returned `None` for an empty tree instead of an empty list, and it incorrectly appended `None` children to the queue, leading to errors during processing.

### Approach
1. **Check for Empty Tree**: If the root is `None`, return an empty list `[]` as specified.
2. **BFS Traversal**: Use a queue to traverse the tree level by level. For each node, append its value to the current level's list and its non-`None` children to the queue.
3. **Reverse Levels**: After collecting all levels from top to bottom, reverse the list of levels to present them from bottom to top.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root: 
            return []
        q, ans = [root], []
        while q:
            n, l = len(q), []
            for i in range(n):
                curr = q.pop(0)
                l.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(l)
        return ans[::-1]
```

### Explanation
1. **Empty Tree Handling**: The function first checks if the root is `None` and returns an empty list if true.
2. **BFS Initialization**: The queue starts with the root node. The `ans` list will store each level's node values.
3. **Level Processing**: For each level, the code processes all nodes in the queue. For each node, its value is added to the current level's list (`l`), and its non-`None` children are added to the queue for the next level.
4. **Level Reversal**: After processing all levels, the `ans` list, which contains levels from top to bottom, is reversed using slicing (`ans[::-1]`) to present the levels from bottom to top.

This approach efficiently handles the BFS traversal and ensures the levels are returned in the correct order while avoiding errors from processing `None` children.
