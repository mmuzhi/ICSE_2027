To solve this problem, we need to determine if a given string containing parentheses `(`, `)`, and wildcards `*` is valid. A valid string must ensure that every opening parenthesis `(` is properly closed by a corresponding closing parenthesis `)`, and wildcards `*` can be used as either `(`, `)`, or an empty character.

### Approach
The approach involves using two variables, `leftMin` and `leftMax`, to track the minimum and maximum number of open parentheses at any point while traversing the string. The key idea is to simulate the balance of parentheses, treating wildcards as potential opening or closing parentheses. 

1. **Initialization**: Start with `leftMin` and `leftMax` set to 0.
2. **Traversal**: For each character in the string:
   - If the character is `(`, increment both `leftMin` and `leftMax` by 1.
   - If the character is `)`, decrement both `leftMin` and `leftMax` by 1. If `leftMin` becomes negative, set it to 0.
   - If the character is `*`, decrement `leftMin` by 1 (treating it as a closing parenthesis) and increment `leftMax` by 1 (treating it as an opening parenthesis). If `leftMin` becomes negative, set it to 0.
3. **Check Validity**: If at any point `leftMax` becomes negative, the string is invalid. After processing all characters, if `leftMin` is 0, the string is valid; otherwise, it is invalid.

This approach efficiently checks the validity by dynamically adjusting the balance of parentheses, considering wildcards as flexible entities that can either increase or decrease the balance.

### Solution Code
```python
class Solution:
    def checkValidString(self, s):
        leftMin = 0
        leftMax = 0
        for c in s:
            if c == '(':
                leftMin += 1
                leftMax += 1
            elif c == ')':
                leftMin = max(leftMin - 1, 0)
                leftMax -= 1
            else:  # c == '*'
                leftMin = max(leftMin - 1, 0)
                leftMax += 1
            if leftMax < 0:
                return False
        return leftMin == 0
```

### Explanation
- **Initialization**: `leftMin` and `leftMax` start at 0, representing the minimum and maximum number of open parentheses.
- **Traversal**:
  - For `(`, both `leftMin` and `leftMax` increase by 1, indicating an opening parenthesis.
  - For `)`, both `leftMin` and `leftMax` decrease by 1. `leftMin` is set to 0 if it becomes negative to avoid negative counts.
  - For `*`, `leftMin` is decremented (treating it as a closing parenthesis) and `leftMax` is incremented (treating it as an opening parenthesis). Again, `leftMin` is set to 0 if negative.
- **Check Validity**: If `leftMax` ever becomes negative during traversal, the string is immediately deemed invalid. After processing all characters, if `leftMin` is 0, all opening parentheses have been properly closed, making the string valid; otherwise, it is invalid.

This method efficiently checks the validity of the string by dynamically adjusting the balance of parentheses, leveraging wildcards' flexibility while ensuring correctness.
