=== RAW ANSWER ===

To solve this problem, we need to find the bitwise complement of a given integer. The bitwise complement of a number is obtained by flipping all its bits. The challenge lies in correctly determining the number of bits used in the binary representation of the number and then flipping each bit to get the complement.

### Approach
1. **Handle the Special Case for Zero**: If the input number is zero, its complement should be 1 (since zero in binary is represented as a single bit, and flipping it gives 1).
2. **Binary Conversion**: For non-zero numbers, convert the number to its binary string representation, excluding the '0b' prefix.
3. **Bit Flipping**: Iterate through each bit in the binary string and flip it (0 becomes 1 and 1 becomes 0).
4. **Convert Back to Integer**: The resulting flipped binary string is then converted back to an integer.

This approach ensures that the complement is computed correctly for all non-negative integers by considering the minimal number of bits required to represent the number in binary.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        if num == 0:
            return 1
        bin_str = bin(num)[2:]
        flipped_str = ''.join('1' if bit == '0' else '0' for bit in bin_str)
        return int(flipped_str, 2)
```

### Explanation
1. **Special Case Handling**: The code first checks if the input number is zero. If it is, the complement is 1, as flipping a single bit (0) results in 1.
2. **Binary Conversion**: The number is converted to a binary string using Python's built-in `bin()` function, and the '0b' prefix is removed by slicing the string from index 2 onwards.
3. **Bit Flipping**: Each bit in the binary string is flipped. For example, '0' becomes '1' and vice versa. This is done using a simple generator expression that iterates over each character in the binary string.
4. **Conversion to Integer**: The flipped binary string is converted back to an integer using the `int()` function with base 2, effectively giving the bitwise complement of the original number.

This approach efficiently handles all non-negative integers and correctly computes their bitwise complements by leveraging Python's built-in functions for binary conversion and string manipulation.
