To solve this problem, we need to generate all possible valid IP addresses from a given string. An IP address consists of four numbers separated by dots, each ranging from 0 to 255. The solution involves using a backtracking approach to explore all possible ways to split the string into four parts, ensuring each part is a valid number.

### Approach
1. **Backtracking**: The solution uses a recursive backtracking method to explore all possible splits of the string into four parts. The algorithm tries to form each part by considering substrings starting from the current position, ensuring the substring is a valid number (0-255) and does not have leading zeros unless the number itself is zero.
2. **Base Case**: When the entire string is processed and four parts are formed, the resulting IP address is added to the result list.
3. **Pruning**: The algorithm prunes invalid paths early by checking the length of the current parts and the position in the string. If more than four parts are formed or the string is exhausted without forming four parts, the recursion stops.

### Solution Code
```python
class Solution():
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        n = len(s)
        
        def backtrack(i):
            if i == n and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i >= n:
                return
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 3 and i + j < n:
                num = s[i:i+j+1]
                if int(num) < 256:
                    cur.append(num)
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

1. **Initialization**: The `res` list stores all valid IP addresses, and `cur` list keeps track of the current parts being formed.
2. **Backtracking Function**: The `backtrack` function is defined to explore all valid splits starting from index `i`.
3. **Base Case Handling**: If the entire string is processed (`i == n`) and exactly four parts are formed (`len(cur) == 4`), the current parts are joined into a string and added to `res`.
4. **Invalid Path Pruning**: If more than four parts are formed or the string is exhausted without forming four parts, the function returns immediately.
5. **Zero Handling**: If the current character is '0', it is added as a part, and the function proceeds to the next character. After the recursive call, the part is removed to backtrack.
6. **Non-Zero Handling**: For non-zero characters, the function tries all possible substrings (up to 3 characters) starting from the current position. Each valid substring (converted to an integer and checked against 255) is added as a part, and the function proceeds recursively.
