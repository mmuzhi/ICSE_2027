To solve this problem, we need to compute the sum of absolute differences for each element in an array. The sum for each element should include the absolute differences between the element and every other element in the array, including itself (which results in zero and does not affect the sum).

### Approach
1. **Problem Analysis**: The task is to compute for each element in the array the sum of absolute differences between that element and every other element in the array. A naive approach would involve iterating over each element and computing the sum directly, leading to an O(n^2) time complexity, which is inefficient for large arrays.

2. **Intuition**: To optimize, we can leverage sorting and prefix sums. By sorting the array, we can efficiently compute the sum of elements less than or equal to the current element and the sum of elements greater than the current element. This allows us to break down the problem into two parts: the left part (elements ≤ current element) and the right part (elements > current element).

3. **Algorithm Selection**: 
   - **Sorting**: First, we sort the array to facilitate the computation of prefix sums.
   - **Prefix Sums**: We compute prefix sums of the sorted array to quickly access the sum of the first `k` elements.
   - **Binary Search**: For each element in the original array, we use binary search (via `bisect_right`) to find the count of elements less than or equal to it in the sorted array. This count helps in determining the sum of these elements using the prefix sums.

4. **Complexity Analysis**: 
   - **Time Complexity**: Sorting the array takes O(n log n) time. Computing prefix sums is O(n). For each element, binary search takes O(log n) time, leading to an overall time complexity of O(n log n).
   - **Space Complexity**: We use O(n) space to store the sorted array and prefix sums.

### Explanation
1. **Sorting and Prefix Sums**: The array is sorted to enable efficient computation of the sum of elements less than or equal to any given element. The prefix sums array allows quick access to the sum of the first `k` elements in the sorted array.
2. **Binary Search**: For each element in the original array, we determine how many elements are less than or equal to it using binary search (`bisect_right`). This count helps in splitting the array into two parts.
3. **Sum Calculation**: The sum of absolute differences is computed by:
   - **Left Part (Elements ≤ current element)**: The sum is calculated as `(current element * count) - (sum of those elements)`.
   - **Right Part (Elements > current element)**: The sum is calculated as `(sum of all elements) - (sum of left part) - (current element * remaining elements)`.
4. **Result Construction**: The results for each element are collected into a list and returned.

This approach efficiently computes the required sums by leveraging sorting and binary search, ensuring optimal performance even for large input sizes.
