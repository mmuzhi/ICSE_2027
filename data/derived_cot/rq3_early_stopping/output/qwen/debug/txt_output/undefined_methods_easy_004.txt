=== RAW ANSWER ===

To solve this problem, we need to determine the minimum number of operations required to make all elements in a given array zero. The operations allowed are incrementing or decrementing elements by one. The key observation here is that the minimum operations required is equal to the number of unique non-zero elements in the array. Each unique non-zero element requires at least one operation to turn it into zero.

### Approach
1. **Problem Analysis**: The problem requires us to find the minimum operations needed to turn all elements of the array into zeros. Each operation can increment or decrement an element by one. The solution involves counting the number of unique non-zero elements in the array, as each such element will require at least one operation to be converted to zero.
2. **Intuition**: The minimum operations needed is directly related to the number of unique non-zero elements. For example, if the array has three unique non-zero elements, the minimum operations required will be three, as each element needs to be processed once.
3. **Algorithm Selection**: We iterate through the array, collecting all unique non-zero elements into a set. The size of this set gives the number of unique non-zero elements, which is the answer.
4. **Complexity Analysis**: The algorithm involves a single pass through the array to build the set of unique non-zero elements, resulting in a time complexity of O(n), where n is the number of elements in the array. The space complexity is O(k), where k is the number of unique non-zero elements, due to the storage required for the set.

### Solution Code
```python
class Solution:
    def minimumOperations(self, nums: List[int]) -> int:
        uniq_non_zero = set()
        for num in nums:
            if num != 0:
                uniq_non_zero.add(num)
        return len(uniq_non_zero)
```

### Explanation
1. **Initialization**: We start by initializing an empty set `uniq_non_zero` to store unique non-zero elements from the array.
2. **Iteration and Set Population**: For each element in the array, if the element is not zero, we add it to the set. Using a set ensures that only unique elements are stored.
3. **Result Calculation**: The size of the set `uniq_non_zero` gives the number of unique non-zero elements. This size is returned as the result, representing the minimum operations needed to turn all array elements into zeros.

This approach efficiently counts the unique non-zero elements, leveraging set properties to ensure uniqueness and simplicity. The solution is optimal with linear time complexity and minimal space usage.
