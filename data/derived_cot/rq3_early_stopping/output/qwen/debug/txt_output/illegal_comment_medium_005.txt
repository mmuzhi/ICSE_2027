To solve this problem, we need to find the k-th smallest positive factor of a given integer `n`. The solution involves iterating through all possible divisors of `n` from 1 to `n` and counting each divisor until we reach the k-th one. 

### Approach
1. **Problem Analysis**: The task is to determine the k-th smallest positive factor of `n`. Factors of `n` are numbers that divide `n` without leaving a remainder. For example, the factors of 12 are 1, 2, 3, 4, 6, and 12.
2. **Intuition**: By iterating from 1 to `n`, we can check each number to see if it is a factor of `n`. Each time a factor is found, we increment a counter. Once the counter reaches `k`, we return the current factor. If no k-th factor is found (which can happen if `k` is larger than the number of factors), we return -1.
3. **Algorithm Selection**: A straightforward approach is used where we loop through each integer from 1 to `n` and check divisibility. This approach is efficient for the given constraints as it directly checks each potential factor in ascending order.
4. **Complexity Analysis**: The time complexity is O(n) in the worst case, where `n` is the input number. This occurs when `n` is a prime number, as we must check all numbers from 1 to `n` to find all factors. However, in practice, the loop can be optimized by checking up to the square root of `n` and considering pairs of factors, but for simplicity and clarity, the direct approach is used here.

### Solution Code
```python
class Solution:
    def kthFactor(self, n: int, k: int) -> int:
        count = 0
        for i in range(1, n + 1):
            if n % i == 0:
                count += 1
                if count == k:
                    return i
        return -1
```

### Explanation
1. **Initialization**: We initialize a counter `count` to zero to keep track of the number of factors found.
2. **Loop Through Numbers**: We iterate from 1 to `n` (inclusive). For each number `i`, we check if `i` is a factor of `n` by verifying if `n % i == 0`.
3. **Count Factor**: If `i` is a factor, we increment the counter `count` and check if it matches `k`. If it does, we immediately return `i` as the k-th factor.
4. **Return -1 if Not Found**: If the loop completes without finding the k-th factor, we return -1, indicating that there are fewer than `k` factors for `n`.

This approach efficiently checks each potential factor in ascending order and returns the k-th one if found, or -1 otherwise. The solution is straightforward and leverages a simple loop to ensure correctness.
