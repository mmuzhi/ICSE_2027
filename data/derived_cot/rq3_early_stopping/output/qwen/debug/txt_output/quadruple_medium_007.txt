=== RAW ANSWER ===

To solve this problem, we need to perform a level-order traversal of a binary tree and return the nodes' values level by level from bottom to top. The initial code attempts to do this but contains several bugs. The key issues are:

1. Incorrect calculation of the number of nodes in the current level.
2. Handling of an empty tree root by returning `None` instead of an empty list.
3. The reversal function is correctly defined but not used appropriately in the context of the entire solution.

### Approach
1. **Level-Order Traversal (BFS)**: We use a queue to traverse the tree level by level. Starting from the root, we process each node, collect its value, and enqueue its left and right children.
2. **Reversing the Levels**: After collecting all levels from top to bottom, we reverse the list of levels to present them from bottom to top.
3. **Edge Case Handling**: If the tree is empty (root is `None`), we return an empty list.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans) - 1
            while l < r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans
        
        if not root:
            return []
        q = [root]
        ans = []
        while q:
            n = len(q)
            l = []
            for i in range(n):
                curr = q.pop(0)
                l.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(l)
        return checkReverse(ans)
```

### Explanation
1. **Level-Order Traversal**: The code uses a queue to traverse each level of the tree. For each node dequeued, its value is added to the current level's list, and its children are enqueued if they exist.
2. **Reversing Levels**: The `checkReverse` function reverses the list of levels in-place by swapping elements from the start and end of the list until the middle is reached.
3. **Edge Case Handling**: If the root is `None`, the function returns an empty list immediately, ensuring correct behavior for an empty tree.
4. **Efficiency**: The algorithm efficiently processes each node exactly once, resulting in a time complexity of O(n), where n is the number of nodes in the tree. The space complexity is also O(n) due to the storage required for the queue and the result list.
