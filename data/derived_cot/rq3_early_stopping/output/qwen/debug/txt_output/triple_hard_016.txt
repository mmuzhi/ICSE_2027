=== RAW ANSWER ===

To solve this problem, we need to find all possible configurations where `n` queens can be placed on an `n x n` chessboard such that no two queens attack each other. The solution involves using a backtracking approach to explore all valid configurations while ensuring that queens do not share the same row, column, or diagonal.

### Approach
1. **Backtracking with Pruning**: The solution uses backtracking to place queens column by column. For each column, it tries to place a queen in each row, ensuring that the chosen position does not conflict with previously placed queens.
2. **Tracking Conflicts**: Three arrays are used to track conflicts:
   - `low` array tracks which rows are already occupied.
   - `upper` array tracks which anti-diagonals (from top-right to bottom-left) are occupied.
   - `lower` array tracks which diagonals (from top-left to bottom-right) are occupied.
3. **Base Case**: When all `n` queens are placed (i.e., `col == n`), the current configuration is added to the result.
4. **Recursive Backtracking**: For each valid position in the current column, the solution places a queen, marks the conflicting positions, and recursively processes the next column. After returning from the recursion, it unmarks the positions to backtrack.

### Solution Code
```python
class Solution:
    def totalNQueens(self, n: int) -> int:
        def addans(board, ans):
            temp = []
            for i in range(n):
                for j in range(n):
                    if board[i][j] == "Q":
                        temp.append(j + 1)
            ans.append(temp)
        
        def solve(col, board, low, upper, lower, ans):
            if col == n:
                addans(board, ans)
                return
            for row in range(n):
                if low[row] == 0 and upper[n - 1 + col - row] == 0 and lower[row + col] == 0:
                    board[row][col] = "Q"
                    low[row] = 1
                    upper[n - 1 + col - row] = 1
                    lower[row + col] = 1
                    solve(col + 1, board, low, upper, lower, ans)
                    board[row][col] = "."
                    low[row] = 0
                    upper[n - 1 + col - row] = 0
                    lower[row + col] = 0
        
        ans = []
        board = [["."] * n for _ in range(n)]
        low = [0] * n
        upper = [0] * (2 * n - 1)
        lower = [0] * (2 * n - 1)
        solve(0, board, low, upper, lower, ans)
        return len(ans)
```

### Explanation
1. **Initialization**: The chessboard is initialized with `.` (empty cells), and the tracking arrays (`low`, `upper`, `lower`) are set to zero.
2. **Backtracking Function (`solve`)**:
   - **Base Case**: If all queens are placed (`col == n`), the current board configuration is added to the result.
   - **Recursive Case**: For each row in the current column, check if placing a queen is valid (no conflicts in row, anti-diagonal, or diagonal). If valid, place the queen, mark the conflicting positions, and proceed to the next column. After returning, unmark the positions to explore other configurations.
3. **Result Construction**: The `addans` function converts the current board configuration into the required format (list of column indices for each row) and adds it to the result list.
4. **Efficiency**: The backtracking approach efficiently prunes invalid configurations early, ensuring optimal performance. The time complexity is O(n!) in the worst case, but practical performance is better due to early termination of invalid paths.
