### Approach
1. **Problem Analysis**: The goal is to minimize the absolute difference between the total cost (base cost + sum of selected toppings) and the target. Each topping can be used up to twice, so we double the list of topping costs to simulate this constraint.
2. **Dynamic Programming with Memoization**: We use a recursive function `fn(i, x)` where `i` is the current index in the doubled topping list, and `x` is the residual amount needed to reach the target from the base cost. The function returns the sum of toppings chosen from index `i` onwards that is closest to `x`.
3. **Base Cases**: If `x` becomes negative, it means adding more toppings would only increase the total cost beyond the target, so we return 0. If we've processed all toppings, we also return 0.
4. **Recursive Case**: For each topping, we decide whether to include it or not. The function compares the results of both choices and selects the one with the smallest absolute difference from `x`, breaking ties by choosing the smaller value.

### Solution Code
```python
class Solution:
    def closestCost(self, baseCosts: List[int], toppingCosts: List[int], target: int) -> int:
        toppingCosts *= 2
        
        @cache
        def fn(i, x):
            if x < 0 or i == len(toppingCosts):
                return 0
            take = toppingCosts[i] + fn(i+1, x - toppingCosts[i])
            skip = fn(i+1, x)
            take_val = take
            skip_val = skip
            if abs(take_val - x) < abs(skip_val - x) or (abs(take_val - x) == abs(skip_val - x) and take_val < skip_val):
                return take_val
            else:
                return skip_val
        
        ans = float('inf')
        for bc in baseCosts:
            res = fn(0, target - bc)
            total = bc + res
            if abs(total - target) < abs(ans - target) or (abs(total - target) == abs(ans - target) and total < ans):
                ans = total
        return ans
```

2. **Memoization**: The `@cache` decorator is used to memoize the results of the recursive function `fn(i, x)`, avoiding redundant calculations.
3. **Recursive Function**: The function `fn(i, x)` computes the optimal sum of toppings from index `i` onwards to minimize the absolute difference from `x`. It considers both including and excluding the current topping.
4. **Base Cases**: If `x` is negative or all toppings are processed, the function returns 0, indicating no more toppings should be added.
5. **Main Loop**: For each base cost, the function calculates the closest total cost by combining the base cost with the optimal topping sum. The result is updated to ensure it is the closest to the target, with ties broken by choosing the smaller total cost.

This approach efficiently explores all possible topping combinations using dynamic programming and memoization, ensuring optimal performance even for larger input sizes.

class Solution:
    def closestCost(self, baseCosts: List[int], toppingCosts: List[int], target: int) -> int:
        toppingCosts *= 2
        
        @cache
        def fn(i, x):
            if x < 0 or i == len(toppingCosts):
                return 0
            take = toppingCosts[i] + fn(i+1, x - toppingCosts[i])
            skip = fn(i+1, x)
            take_val = take
            skip_val = skip
            if abs(take_val - x) < abs(skip_val - x) or (abs(take_val - x) == abs(skip_val - x) and take_val < skip_val):
                return take_val
            else:
                return skip_val
        
        ans = float('inf')
        for bc in baseCosts:
            res = fn(0, target - bc)
            total = bc + res
            if abs(total - target) < abs(ans - target) or (abs(total - target) == abs(ans - target) and total < ans):
                ans = total
        return ans
