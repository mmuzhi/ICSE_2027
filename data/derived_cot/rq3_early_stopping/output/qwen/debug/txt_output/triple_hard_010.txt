### Approach
1. **Initial Checks**: If `s2` contains any character not present in `s1`, it's impossible to form any repetition of `s2`, so we return 0.
2. **Filtering Characters**: We filter `s1` to include only characters present in `s2` to optimize the search process.
3. **Cycle Detection**: We simulate the process of forming `s2` repeated sequences by tracking the start and end positions in `s1`. This helps in detecting cycles in the formation process, allowing us to compute the maximum repetitions efficiently without processing the entire concatenated string.
4. **Cycle Utilization**: Once a cycle is detected, we use the cycle's properties to compute the remaining repetitions needed and adjust the result accordingly.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def getMaxRepetitions(self, s1: str, n1: int, s2: str, n2: int) -> int:
        if not s2:
            return 0
        
        rec, track = [0], defaultdict(int)
        ct = start = ptr = 0
        
        if not set(s2).issubset(set(s1)):
            return 0
        
        s1 = ''.join(char for char in s1 if char in set(s2))
        
        while True:
            for char in s2:
                ptr = s1.find(char, start)
                if ptr == -1:
                    ct += 1
                    ptr = s1.find(char)
                start = ptr + 1
            rec.append(ct + 1)
            
            if rec[-1] >= n1:
                return (len(rec) - 2) // n2
                
            if ptr not in track:
                track[ptr] = len(rec) - 1
            else:
                break
        
        cycleStart = rec[track[ptr]]
        cycle1 = ct + 1 - cycleStart
        cycle2 = len(rec) - 1 - track[ptr]
        rest = n1 - cycleStart
        
        rem = cycleStart + rest % cycle1
        
        ptr2 = 0
        while rec[ptr2] <= rem:
            ptr2 += 1
            
        return (cycle2 * (rest // cycle1) + ptr2 - 1) // n2
```

### Explanation
1. **Initial Checks**: The code first checks if `s2` is empty or if it contains characters not present in `s1`. If either condition is true, it returns 0 immediately.
2. **Filtering Characters**: The string `s1` is filtered to retain only characters present in `s2`, which simplifies the search process for forming `s2` repeated sequences.
3. **Cycle Detection**: The loop attempts to form `s2` by sequentially finding each character in the filtered `s1`. The `rec` list records the cumulative count of `s1` copies used to form each `s2`. If the cumulative count exceeds `n1`, the result is derived from the recorded positions.
4. **Cycle Utilization**: Upon detecting a cycle in the formation process, the code calculates the number of full cycles and remaining characters needed. The result is computed using the cycle's properties and the remaining characters, ensuring efficient calculation without explicit string concatenation.

This approach efficiently handles large values of `n1` by leveraging cycle detection and properties of string repetition, ensuring optimal performance.
