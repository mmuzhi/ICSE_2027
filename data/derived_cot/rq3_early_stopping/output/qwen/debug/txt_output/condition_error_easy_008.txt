To solve this problem, we need to find the next greater element for each element in `nums1` from the array `nums2`. The next greater element for an element `x` in `nums1` is the first element in `nums2` that is greater than `x` and appears after the element `x` in `nums2`. If no such element exists, we return -1 for that element in `nums1`.

### Approach
1. **Problem Analysis**: For each element in `nums1`, we need to find the next element in `nums2` that is greater than it. The challenge is to efficiently locate this element while handling cases where the element might not be found (i.e., return -1).

2. **Intuition**: The straightforward approach involves iterating through `nums2` for each element in `nums1` to find the next greater element. For each element in `nums1`, we first locate its position in `nums2` and then search subsequent elements until we find a greater one or exhaust all elements.

3. **Algorithm Selection**: 
   - For each element in `nums1`, find its index in `nums2`.
   - Starting from the next index of this found position in `nums2`, search for the first element that is greater than the current element from `nums1`.
   - If found, append this element to the result list; otherwise, append -1.

4. **Complexity Analysis**: The algorithm involves nested loops. The outer loop runs for each element in `nums1`, and the inner loop can run up to the length of `nums2` in the worst case. Thus, the time complexity is O(n*m), where n is the length of `nums1` and m is the length of `nums2`. This is efficient for small input sizes but may not be optimal for very large arrays.

### Solution Code
```python
class Solution:
    def nextGreaterElement(self, nums1: List[int], nums2: List[int]) -> List[int]:
        out = []
        for num in nums1:
            idx = nums2.index(num)
            j = idx + 1
            found = -1
            while j < len(nums2):
                if num < nums2[j]:
                    found = nums2[j]
                    break
                j += 1
            out.append(found)
        return out
```

3. **Search for Next Greater Element**: Starting from the index right after the found position in `nums2`, we check each subsequent element. If we find an element greater than `num`, we record it and break out of the loop. If no such element is found, we set `found` to -1.
4. **Append Result**: After determining the next greater element (or -1 if not found), we append it to the result list `out`.
5. **Return Result**: Finally, we return the result list `out` containing the next greater elements for all elements in `nums1`.

This approach efficiently handles the problem by leveraging simple iteration and conditional checks, ensuring correctness while maintaining clarity.
