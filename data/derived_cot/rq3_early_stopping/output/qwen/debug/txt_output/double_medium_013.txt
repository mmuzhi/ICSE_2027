To solve this problem, we need to find the K-th bit of the N-th term in the Thue-Morse sequence. The Thue-Morse sequence is constructed recursively by appending the bitwise complement of the previous term. The challenge is to efficiently determine the K-th bit without generating the entire sequence up to N.

### Approach
1. **Base Case Handling**: The first bit (K=1) of any term in the Thue-Morse sequence is always '0' if we are in the original sequence (R=True), and '1' if we are in the complement sequence (R=False).
2. **Mid Calculation**: For any term N, the sequence length is \(2^N\). The midpoint of this sequence is at position \(2^{N-1}\).
3. **Recursive Handling**:
   - If K is less than the midpoint, the K-th bit is the same as the K-th bit in the (N-1)-th term with the same R value.
   - If K equals the midpoint, the K-th bit is the same as the midpoint bit of the (N-1)-th term but with R flipped.
   - If K is greater than the midpoint, the K-th bit is the complement of the bit at position \(2 \times \text{mid} - K + 1\) in the (N-1)-th term with R flipped.

### Solution Code

class Solution:
    def findKthBit(self, N, K, R=True):
        if K == 1:
            if R:
                return '0'
            else:
                return '1'

        if K < mid:

### Explanation
1. **Base Case**: When K is 1, the bit is '0' if R is True (original sequence) or '1' if R is False (complement sequence).
2. **Mid Calculation**: The midpoint `mid` is calculated as \(2^{N-1}\), which splits the sequence into two halves.
3. **Recursive Cases**:
   - **K < mid**: The K-th bit is found in the first half of the sequence, so we recursively call the function with N-1, K, and the same R.
   - **K == mid**: The K-th bit is the midpoint of the first half, so we recursively call with N-1, mid, and flipped R.
   - **K > mid**: The K-th bit is in the second half, which is the complement of the first half. We find the corresponding bit in the first half by calculating `2*mid - K + 1`, flip R, and then return the complement of the bit obtained.

This approach efficiently narrows down the search to the relevant part of the sequence without generating the entire sequence, leveraging recursion and bitwise properties of the Thue-Morse sequence.
