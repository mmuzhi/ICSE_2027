To solve this problem, we need to move all zeros in an array to the end while maintaining the relative order of the non-zero elements. The solution involves using a two-pointer technique to efficiently traverse and modify the array in-place.

### Approach
1. **Initialization**: Start with two pointers, `slow` and `fast`, both set to the beginning of the array.
2. **Traversal**: Use the `fast` pointer to traverse through each element in the array.
3. **Swapping Condition**: Whenever the `fast` pointer encounters a non-zero element and the `slow` pointer is at a zero, swap these elements. This ensures that non-zero elements are moved towards the front.
4. **Increment Slow**: After swapping or if the `fast` pointer encounters a non-zero element, increment the `slow` pointer. This pointer marks the position where the next non-zero element should be placed.

This approach ensures that all zeros are moved to the end of the array while keeping the non-zero elements in their original order. The algorithm operates in O(n) time complexity, where n is the number of elements in the array, and uses O(1) extra space.

### Solution Code
```python
class Solution:
    def moveZeroes(self, nums: list) -> None:
        slow = 0
        for fast in range(len(nums)):
            if nums[fast] != 0 and nums[slow] == 0:
                nums[slow], nums[fast] = nums[fast], nums[slow]
            if nums[fast] != 0:
                slow += 1
```

### Explanation
- **Initialization**: The `slow` pointer starts at 0, indicating the position where the next non-zero element should be placed.
- **Traversal**: The `fast` pointer iterates through each element in the array. For each element:
  - If the `fast` pointer finds a non-zero element and the `slow` pointer is at a zero, the elements at `slow` and `fast` are swapped. This moves the non-zero element to the `slow` position.
  - If the `fast` pointer encounters a non-zero element (whether swapped or not), the `slow` pointer is incremented to the next position.
- **Result**: After processing all elements, all zeros are shifted to the end of the array, while non-zero elements remain in their original order.

This approach efficiently handles the problem with a single pass through the array, making it optimal for performance.
