=== RAW ANSWER ===

To solve this problem, we need to detect if a given singly-linked list contains a cycle. The approach involves using the two-pointer technique, which efficiently checks for cycles in linear time with constant space complexity.

### Approach
The two-pointer technique involves using two pointers that move through the list at different speeds. One pointer moves one step at a time (slow pointer), and the other moves two steps at a time (fast pointer). If there is a cycle in the list, the fast pointer will eventually meet the slow pointer within the cycle. If there is no cycle, the fast pointer will reach the end of the list.

1. **Initialization**: Start both pointers (slow and fast) at the head of the list.
2. **Traversal**: While the fast pointer and its next node exist, move the slow pointer one step and the fast pointer two steps.
3. **Cycle Detection**: If at any point the slow pointer and the fast pointer point to the same node, a cycle exists, and we return `True`.
4. **Termination**: If the fast pointer reaches the end of the list (i.e., `fast` or `fast.next` is `None`), return `False` as no cycle is found.

### Solution Code
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        slow = head
        fast = head
        while fast and fast.next:
            slow = slow.next
            fast = fast.next.next
            if slow == fast:
                return True
        return False
```

### Explanation
- **Initialization**: Both `slow` and `fast` start at the head of the list.
- **Loop Condition**: The loop continues as long as `fast` and `fast.next` are not `None`, ensuring the fast pointer can move two steps.
- **Pointer Movement**: In each iteration, the slow pointer moves one step (`slow = slow.next`), and the fast pointer moves two steps (`fast = fast.next.next`).
- **Cycle Check**: If `slow` and `fast` point to the same node during any iteration, the function returns `True`, indicating a cycle.
- **Termination**: If the loop exits without detecting a cycle, the function returns `False`.

This approach efficiently checks for cycles in O(n) time complexity, where n is the number of nodes in the list, and O(1) space complexity, making it optimal for the problem constraints.
