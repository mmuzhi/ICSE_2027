### Approach
1. **Problem Analysis**: The task is to count the number of elements in the array that are smaller than each element. A naive approach would involve comparing each element with every other element, resulting in a time complexity of O(n^2). However, we can optimize this by using a frequency array.
2. **Intuition**: By first counting the occurrences of each number in the array, we can then for each number, sum the frequencies of all numbers that are strictly smaller. This approach leverages the fact that we can precompute the counts and then use them to answer each query in constant time.
3. **Algorithm Selection**: We use a frequency array (or list) to count occurrences of each number. Since the numbers can range from -100 to 100 (as per common constraints), we shift the numbers by 100 to handle negative values, allowing us to use a fixed-size array of size 201 (from 0 to 200).
