To solve this problem, we need to find the maximum gap between any two consecutive elements in a sorted list of numbers. The initial code had two main issues: an undefined variable and incorrect logic for finding the maximum gap instead of the minimum. 

### Approach
1. **Sorting the Array**: The first step is to sort the input array in ascending order. This allows us to easily compute the differences between consecutive elements.
2. **Edge Case Handling**: If the array has fewer than two elements, the maximum gap is zero since there are no pairs to compare.
3. **Finding Maximum Gap**: Iterate through the sorted array, compute the difference between each consecutive pair, and keep track of the largest difference encountered.

### Solution Code
```python
class Solution:
    def maximumGap(self, nums: list) -> int:
        nums = sorted(nums)
        if len(nums) < 2:
            return 0
        max_gap = 0
        for i in range(len(nums) - 1):
            gap = nums[i+1] - nums[i]
            if gap > max_gap:
                max_gap = gap
        return max_gap
```

### Explanation
1. **Sorting**: The input array is sorted to ensure that consecutive elements are in ascending order, which simplifies the calculation of gaps.
2. **Edge Case Handling**: If the array contains fewer than two elements, there are no gaps to consider, so the function returns 0.
3. **Iterating Through Pairs**: For each element in the sorted array (except the last one), the difference between the current element and the next element is calculated. This difference is compared with the current maximum gap, updating the maximum gap if a larger difference is found.
4. **Result**: The function returns the largest gap found during the iteration, which is the solution to the problem.

This approach efficiently computes the maximum gap by leveraging sorting and a single pass through the array, ensuring optimal performance with a time complexity of O(n log n) due to the sorting step.
