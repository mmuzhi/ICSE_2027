To solve this problem, we need to generate all valid IP addresses from a given string by splitting it into four parts. Each part must be a number between 0 and 255, and parts cannot have leading zeros unless the part is exactly "0".

### Approach
1. **Backtracking**: We use a backtracking approach to explore all possible ways to split the string into four parts. The algorithm recursively tries to form valid parts by considering substrings of lengths 1 to 3 (since each part can be up to 3 digits).
2. **Base Case**: When the entire string is processed (i.e., the current index reaches the end of the string and exactly four parts are formed), we join the parts with dots and add the resulting IP address to the result list.
3. **Handling Leading Zeros**: If a part starts with '0', it can only be a single '0' (not '00' or '000'). This is handled separately to avoid invalid parts.
4. **Validation**: Each substring is converted to an integer to check if it is within the range [0, 255]. If it is, it is considered a valid part.

### Solution Code
```python
from typing import List

class Solution():
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i >= len(s):
                return
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 3 and i + j < len(s):
                num = int(s[i:i+j+1])
                if num < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

1. **Initialization**: The `res` list stores all valid IP addresses, and `cur` is a temporary list to hold the current parts being formed.
2. **Backtracking Function**: The `backtrack` function is defined inside `restoreIpAddresses` to keep track of the current index `i` and the current parts `cur`.
3. **Base Case**: If `i` reaches the end of the string and exactly four parts are formed, the parts are joined into a string and added to `res`.
4. **Invalid Cases**: If more than four parts are formed or `i` exceeds the string length, the function returns without further processing.
5. **Zero Handling**: If the current character is '0', it is added as a single part, and the function proceeds to the next index.
6. **Non-Zero Handling**: For non-zero characters, the function checks all possible substrings of lengths 1 to 3. Each valid substring (numeric value < 256) is added to `cur`, and the function proceeds recursively. After the recursive call, the part is removed from `cur` to backtrack and explore other possibilities.
7. **Result**: After exploring all possibilities starting from index 0, the function returns the list of all valid IP addresses.

This approach efficiently explores all valid combinations using backtracking, ensuring correctness by validating each part during the process. The complexity is exponential in the worst case, but it is manageable for strings up to 12 characters (the maximum length for a valid IP address string).
