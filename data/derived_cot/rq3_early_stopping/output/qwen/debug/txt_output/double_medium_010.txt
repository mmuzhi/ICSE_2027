To solve this problem, we need to compute the minimum edit distance between two strings, `word1` and `word2`. The edit distance is defined as the minimum number of operations required to transform one string into another. The operations allowed are insertion, deletion, or substitution of a single character.

### Approach
The approach involves using dynamic programming to build a 2D table `dp` where `dp[i][j]` represents the minimum edit distance between the first `i` characters of `word1` and the first `j` characters of `word2`. The table is filled based on the following recurrence relations:

1. **Base Cases**:
   - If one of the strings is empty, the edit distance is the length of the other string (all insertions or deletions).
   - `dp[i][0] = i` (convert empty string to first `i` characters of `word1` requires `i` deletions).
   - `dp[0][j] = j` (convert empty string to first `j` characters of `word2` requires `j` insertions).

2. **Recursive Case**:
   - If the characters at positions `i-1` and `j-1` in `word1` and `word2` respectively are the same, no operation is needed, so `dp[i][j] = dp[i-1][j-1]`.
   - If the characters differ, the minimum edit distance is derived from three operations:
     - **Deletion**: Remove the character from `word1` (cost `dp[i-1][j] + 1`).
     - **Insertion**: Insert the character from `word2` into `word1` (cost `dp[i][j-1] + 1`).
     - **Substitution**: Replace the character in `word1` with the corresponding character in `word2` (cost `dp[i-1][j-1] + 1`).
   - The minimum of these three values plus one gives the edit distance for `dp[i][j]`.

### Solution Code
```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m = len(word1)
        n = len(word2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        
        for i in range(m + 1):
            dp[i][0] = i
        for j in range(n + 1):
            dp[0][j] = j
        
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    dp[i][j] = dp[i-1][j-1]
                else:
                    dp[i][j] = min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) + 1
        
        return dp[m][n]
```

### Explanation
- **Initialization**: The `dp` table is initialized with dimensions `(m+1) x (n+1)` where `m` and `n` are the lengths of `word1` and `word2` respectively.
- **Base Cases**: The first row and column of the `dp` table are filled with the lengths of the respective strings, representing the number of operations needed if one string is empty.
- **Filling the DP Table**: For each character in `word1` and `word2`, if the characters match, the value is taken from the diagonal cell (no operation needed). If they don't match, the value is the minimum of the three possible operations (deletion, insertion, substitution) plus one.
- **Result**: The value at `dp[m][n]` gives the minimum edit distance between the entire strings `word1` and `word2`.

This approach efficiently computes the edit distance using dynamic programming, leveraging previously computed values to avoid redundant calculations, resulting in an optimal solution with a time complexity of O(m*n) and space complexity of O(m*n).
