To solve this problem, we need to find a peak element in an integer array. A peak element is defined as an element that is strictly greater than its neighbors. The solution involves using a binary search approach to efficiently locate a peak element in logarithmic time.

### Approach
1. **Problem Analysis**: The problem requires finding an index of a peak element in an array where each element is greater than its immediate neighbors. The array can have multiple peaks, and any one of them is acceptable.
2. **Intuition**: The binary search approach is suitable here because the array can be visualized as having a "valley" in the middle, and the peak must lie in one of the two halves. By comparing the middle element with its neighbors, we can determine whether the peak lies to the left or right of the middle element.
3. **Algorithm Selection**: We use binary search between the start (`low`) and end (`high`) of the array. For each midpoint (`mid`), we check if it is a peak by verifying if it is greater than its left and right neighbors (if they exist). If it is a peak, we return its index. If not, we adjust the search range based on whether the middle element is less than its right neighbor (indicating the peak is on the right) or greater than its right neighbor (indicating the peak is on the left).
4. **Complexity Analysis**: The binary search runs in O(log n) time, where n is the number of elements in the array. This is efficient and optimal for the problem constraints.

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        if n == 1:
            return 0
        
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] > nums[mid-1]) and (mid == n-1 or nums[mid] > nums[mid+1]):
                return mid
            elif nums[mid] < nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

1. **Initialization**: The function starts by determining the length of the array (`n`). If the array has only one element, that element is trivially a peak, so we return its index (0).
2. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to `n-1`.
3. **Midpoint Calculation**: In each iteration, the midpoint (`mid`) is calculated as the average of `low` and `high`.
4. **Peak Check**: The code checks if the midpoint is a peak by verifying if it is greater than its left neighbor (if it exists) and its right neighbor (if it exists). If both conditions are met, the midpoint is returned as the peak index.
5. **Adjusting Search Range**: If the midpoint is not a peak, the search range is adjusted. If the midpoint is less than its right neighbor, the peak must be on the right side, so `low` is set to `mid + 1`. Otherwise, the peak is on the left side, and `high` is set to `mid - 1`.
6. **Termination**: The loop continues until a peak is found or the search range is exhausted. The problem guarantees at least one peak, so the loop will always find a valid index.
