To solve this problem, we need to generate the first `numRows` rows of Pascal's triangle. Pascal's triangle is a triangular array of binomial coefficients where each number is the sum of the two numbers directly above it. The first row is [1], the second row is [1, 1], the third row is [1, 2, 1], and so on.

### Approach
1. **Base Case Handling**: If `numRows` is 1, return a list containing the first row [1].
2. **Recursive Generation**: For `numRows` greater than 1, recursively generate the first `numRows - 1` rows.
3. **Building the Current Row**: The current row is constructed by starting with 1, then for each adjacent pair in the previous row, summing them, and finally appending another 1 at the end.
4. **Appending the Row**: The newly constructed row is appended to the list of previously generated rows, which is then returned.

The key insight is that each row in Pascal's triangle can be derived from the previous row by summing adjacent elements. This approach efficiently builds each row based on the row above it, leveraging recursion to handle the generation of each subsequent row.
