To solve this problem, we need to find all possible configurations where `n` queens can be placed on an `n x n` chessboard such that no two queens attack each other. The solution involves using a backtracking approach to explore all possible positions for the queens while ensuring that no two queens share the same row, column, or diagonal.

### Approach
1. **Backtracking with Pruning**: The solution uses backtracking to place queens column by column. For each column, it tries placing a queen in every available row, ensuring that the placement does not conflict with previously placed queens.
2. **Tracking Availability**: Three arrays are used to keep track of available rows and diagonals:
   - `low` array tracks the availability of rows (0 indicates available, 1 indicates occupied).
   - `upper` array tracks the availability of the anti-diagonals (calculated as `col - row + (n-1)`).
   - `lower` array tracks the availability of the positive diagonals (calculated as `row + col`).
3. **Base Case**: When all columns are filled (i.e., `col == n`), the current configuration is added to the list of solutions.
4. **Recursive Backtracking**: After placing a queen in a valid position, the function recursively tries to place queens in subsequent columns. If no valid placement is found, it backtracks by resetting the availability markers.

### Solution Code
```python
class Solution:
    def totalNQueens(self, n: int) -> int:
        def addans(board, ans):
            temp = []
            for i in range(n):
                for j in range(n):
                    if board[i][j] == "Q":
                        temp.append(j + 1)
            ans.append(temp)
        
        def solve(col, board, low, upper, lower, ans):
            if col == n:
                addans(board, ans)
                return
            for row in range(n):
                if low[row] == 0 and upper[n - 1 + col - row] == 0 and lower[row + col] == 0:
                    board[row][col] = "Q"
                    low[row] = 1
                    upper[n - 1 + col - row] = 1
                    lower[row + col] = 1
                    solve(col + 1, board, low, upper, lower, ans)
                    low[row] = 0
                    upper[n - 1 + col - row] = 0
                    lower[row + col] = 0
        
        ans = []
        board = [["0"] * n for _ in range(n)]
        low = [0] * n
        upper = [0] * (2 * n - 1)
        lower = [0] * (2 * n - 1)
        solve(0, board, low, upper, lower, ans, n)
        return len(ans)
```

### Explanation
1. **Initialization**: The `addans` function collects the positions of queens in the current configuration and appends them to the result list. The `solve` function is the core backtracking function that tries to place queens column by column.
2. **Base Case Handling**: When all columns (`col == n`) are processed, the current board configuration is added to the result.
3. **Placement Check**: For each row in the current column, the function checks if the row and both diagonals are available (marked by 0 in the respective arrays). If available, it places a queen and marks the row and diagonals as occupied.
4. **Recursion and Backtracking**: The function recursively processes the next column. If no valid placement is found, it backtracks by resetting the availability markers for the current column's row and diagonals.
5. **Result Compilation**: The initial call to `solve` starts the process from the first column (index 0). The final result is the count of all valid configurations stored in `ans`.

This approach efficiently explores all possible configurations while pruning invalid paths early, ensuring optimal performance for the given problem constraints.
