To solve this problem, we need to count the number of reachable nodes from node 0 in an undirected graph, avoiding any nodes specified in the `restricted` list. The solution involves performing a Breadth-First Search (BFS) starting from node 0, ensuring that we only traverse nodes not present in the `restricted` list.

### Approach
1. **Graph Representation**: Represent the graph using an adjacency list where each node points to its connected neighbors.
2. **BFS Initialization**: Start BFS from node 0. Before beginning the BFS, mark all nodes in the `restricted` list as visited to exclude them from traversal.
3. **BFS Execution**: Process each node in the queue by checking if it has been visited. If not, mark it as visited, increment the count of reachable nodes, and enqueue its unvisited neighbors.
4. **Edge Case Handling**: Ensure node 0 is included in the adjacency list even if it has no edges, to handle cases where node 0 is isolated.

### Solution Code
```python
from typing import List
from collections import deque, defaultdict

class Solution:
    def reachableNodes(self, n: int, edges: List[List[int]], restricted: List[int]) -> int:
        adj_list = defaultdict(list)
        for x, y in edges:
            adj_list[x].append(y)
            adj_list[y].append(x)
        
        if 0 not in adj_list:
            adj_list[0] = []
        
        que = deque()
        que.append(0)
        result = 0
        visited = set()
        for node in restricted:
            visited.add(node)
        
        while que:
            cur = que.popleft()
            if cur in visited:
                continue
            visited.add(cur)
            result += 1
            for neighbor in adj_list[cur]:
                que.append(neighbor)
        
        return result
```

3. **BFS Execution**: The BFS processes each node by checking if it's visited. Unvisited nodes are marked as visited, their count is incremented, and their neighbors are enqueued for further processing.
4. **Edge Case Handling**: The adjacency list for node 0 is initialized to an empty list if it wasn't present in the input edges, ensuring node 0 is considered in the BFS even if isolated.

This approach efficiently counts all reachable nodes from node 0 while avoiding restricted nodes, leveraging BFS for systematic traversal and a visited set to prevent revisiting nodes.
