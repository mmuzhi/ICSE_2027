The solution involves iterating through each number from 1 to n, checking divisibility conditions, and appending the appropriate string to the result list.

### Approach
1. **Problem Analysis**: The task requires generating a sequence where numbers divisible by 3 are replaced with "Fizz", numbers divisible by 5 with "Buzz", and numbers divisible by both 3 and 5 with "FizzBuzz". All other numbers should be converted to their string representation.
2. **Intuition**: By iterating from 1 to n, we can check each number's divisibility. The order of checks is important: first check for divisibility by both 3 and 5, then by 3, then by 5, and finally default to the number itself.
3. **Algorithm Selection**: A simple loop from 1 to n with conditional checks for each number's divisibility properties. This approach ensures each number is processed in constant time, leading to an efficient solution.
4. **Complexity Analysis**: The algorithm runs in O(n) time, as it processes each number exactly once. The space complexity is O(n) to store the result list.

### Solution Code
```python
class Solution:
    def fizzBuzz(self, n: int) -> List[str]:
        arr = []
        for i in range(1, n+1):
            if i % 3 == 0 and i % 5 == 0:
                arr.append("FizzBuzz")
            elif i % 3 == 0:
                arr.append("Fizz")
            elif i % 5 == 0:
                arr.append("Buzz")
            else:
                arr.append(str(i))
        return arr
```

### Explanation
1. **Initialization**: An empty list `arr` is initialized to store the result strings.
2. **Loop Through Numbers**: The loop runs from 1 to n (inclusive). For each number `i`, the following checks are performed:
   - If `i` is divisible by both 3 and 5, "FizzBuzz" is appended to the list.
   - If `i` is divisible by 3 but not 5, "Fizz" is appended.
   - If `i` is divisible by 5 but not 3, "Buzz" is appended.
   - If none of the above conditions are met, the number is converted to a string and appended.
3. **Return Result**: The list `arr` is returned, containing the processed strings for all numbers from 1 to n.

This approach efficiently handles each number in linear time, ensuring correctness and optimal performance.
