### Approach
1. **Problem Analysis**: The problem requires summing the minimum values of all possible contiguous subarrays in an array. A brute-force approach would involve checking every possible subarray, which is inefficient for large arrays. Instead, we use a more efficient method leveraging a monotonic stack to find the boundaries where each element is the minimum.

2. **Intuition**: For each element in the array, the number of subarrays where it is the minimum can be determined by finding the nearest greater elements to its left and right. The left boundary is the first element to the left that is greater than the current element, and the right boundary is the first element to the right that is greater than the current element. The number of subarrays where the current element is the minimum is then given by the product of the distances to these boundaries.

3. **Algorithm Selection**: We use a monotonic stack to compute the left and right boundaries for each element. The left boundary is found by iterating from left to right, and the right boundary by iterating from right to left. For each element, the left boundary is the index of the previous greater element, and the right boundary is the index of the next greater element.

4. **Complexity Analysis**: The algorithm processes each element twice (once for left boundaries and once for right boundaries), and each element is pushed and popped from the stack at most once. Thus, the time complexity is O(n), where n is the number of elements in the array. The space complexity is O(n) due to the storage required for the left and right boundary arrays.

class Solution:
    def sumSubarrayMins(self, arr: List[int]) -> int:
        n = len(arr)
        left = [-1] * n
        right = [n] * n
        stack = []
        for i in range(n):

                stack.pop()
