To solve this problem, we need to find the maximum depth of a binary tree. The maximum depth is defined as the number of nodes along the longest path from the root node down to the farthest leaf node. 

### Approach
The approach involves using an iterative Depth-First Search (DFS) approach with a stack to traverse the tree. Here's the step-by-step breakdown:

1. **Initialization**: Start by initializing a stack with the root node and a depth of 1 (if the root exists). If the root is None, return 0 immediately.
2. **Traversal**: Use a while loop to process each node in the stack. For each node, check if it is not None. If it is not None, update the maximum depth encountered so far.
3. **Child Processing**: For each non-None node, push its left and right children onto the stack with an incremented depth, provided the children exist (i.e., they are not None).
4. **Result**: After processing all nodes, the maximum depth found during the traversal is returned.

This approach ensures that we explore each path from the root to the leaves, keeping track of the deepest path encountered.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        if not root:
            return 0
        
        stack = [[root, 1]]
        res = 0

        while stack:
            node, depth = stack.pop()
            if node:
                res = max(res, depth)
                if node.left:
                    stack.append([node.left, depth + 1])
                if node.right:
                    stack.append([node.right, depth + 1])
            
        return res
```

### Explanation
1. **Initialization**: The stack starts with the root node and an initial depth of 1. If the root is None, the function returns 0 immediately since the tree depth is 0.
2. **Traversal Loop**: The loop continues as long as there are nodes in the stack. Each iteration pops the top node and its depth from the stack.
3. **Depth Update**: If the popped node is not None, the current depth is compared with the maximum depth found so far, updating the result if necessary.
4. **Child Processing**: For each non-None node, its left and right children (if they exist) are pushed onto the stack with an incremented depth. This ensures that each path from the root to the leaves is explored.
5. **Result Return**: After processing all nodes, the maximum depth encountered during the traversal is returned.

This approach efficiently explores all paths in the tree using DFS, ensuring the correct maximum depth is found while avoiding unnecessary processing of None children.
