To solve this problem, we need to compute the minimum edit distance between two strings, `word1` and `word2`. The edit distance is defined as the minimum number of operations required to transform one string into another. The allowed operations are inserting a character, deleting a character, or replacing a character, each costing one operation.

### Approach
The approach involves using dynamic programming to build a table `dp` where `dp[i][j]` represents the minimum edit distance between the first `i` characters of `word1` and the first `j` characters of `word2`. The table is filled based on the following recurrence relations:

1. **Base Cases**:
   - If `word1` is empty, the edit distance is the length of `word2` (all insertions).
   - If `word2` is empty, the edit distance is the length of `word1` (all deletions).

2. **Recursive Case**:
   - If the characters at positions `i-1` and `j-1` match, the edit distance is the same as the distance between the substrings before these characters (`dp[i-1][j-1]`).
   - If the characters do not match, the edit distance is the minimum of:
     - The distance obtained by deleting the character from `word1` (`dp[i-1][j] + 1`).
     - The distance obtained by inserting the character from `word2` into `word1` (`dp[i][j-1] + 1`).
     - The distance obtained by replacing the character in `word1` with the character in `word2` (`dp[i-1][j-1] + 1`).

### Solution Code
```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m = len(word1)
        n = len(word2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        for i in range(m + 1):
            dp[i][0] = i
        for j in range(n + 1):
            dp[0][j] = j
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    dp[i][j] = dp[i-1][j-1]
                else:
                    dp[i][j] = min(dp[i-1][j-1] + 1, dp[i-1][j] + 1, dp[i][j-1] + 1)
        return dp[m][n]
```

### Explanation
1. **Initialization**: The DP table `dp` is initialized with dimensions `(m+1) x (n+1)`, where `m` and `n` are the lengths of `word1` and `word2`, respectively.
2. **Base Cases**: The first row and column of the table are filled with the lengths of `word2` and `word1`, respectively, representing the cost of converting an empty string to the other string.
3. **Filling the DP Table**: For each character in `word1` and `word2`, the table is filled by comparing the characters. If they match, the value is taken from the diagonal cell. If they don't match, the value is the minimum of the three possible operations (delete, insert, replace).
4. **Result**: The value at `dp[m][n]` gives the minimum edit distance between the entire strings `word1` and `word2`.

This approach efficiently computes the edit distance using dynamic programming, leveraging previously computed values to avoid redundant calculations and ensuring optimal performance.
