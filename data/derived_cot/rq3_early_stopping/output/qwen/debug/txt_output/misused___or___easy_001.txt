To solve this problem, we need to find common restaurants between two lists and return those restaurants that have the smallest sum of their indices in both lists. The solution involves efficiently checking for common elements and calculating their index sums to determine the smallest sum.

### Approach
1. **Dictionary Creation**: Create a dictionary from the second list (`list2`) where the keys are the restaurant names and the values are their respective indices. This allows for O(1) lookups to check if a restaurant exists in `list2`.
2. **Index Sum Calculation**: Iterate through the first list (`list1`) and for each restaurant present in `list2`, compute the sum of its indices in both lists. Store these sums along with the restaurant names in a list.
3. **Sorting and Selection**: Sort the list of tuples (sum, restaurant) by the sum. The first element(s) in this sorted list will have the smallest sum. Collect all restaurant names that have this smallest sum and return them.

### Solution Code
```python
class Solution:
    def findRestaurant(self, list1: List[str], list2: List[str]) -> List[str]:
        d2 = {}
        for i in range(len(list2)):
            d2[list2[i]] = i
        
        l = []
        for i in range(len(list1)):
            if list1[i] in d2:
                l.append([i + d2[list1[i]], list1[i]])
        
        if not l:
            return []
        
        l.sort()
        l1 = [l[0][1]]
        for i in range(1, len(l)):
            if l[i][0] == l[i-1][0]:
                l1.append(l[i][1])
            else:
                break
        
        return l1
```

### Explanation
1. **Dictionary Creation**: The dictionary `d2` is built from `list2` to map each restaurant name to its index. This step ensures quick lookups to verify if a restaurant from `list1` exists in `list2`.
2. **Index Sum Calculation**: For each restaurant in `list1`, if it exists in `list2`, the sum of its indices in both lists is calculated and stored along with the restaurant name in a list `l`.
3. **Sorting and Selection**: The list `l` is sorted lexicographically, which first sorts by the sum (the first element of each tuple) and then by the restaurant name (the second element). The first element(s) in this sorted list will have the smallest sum. The solution then collects all restaurant names that share this smallest sum and returns them in a list.

This approach efficiently narrows down the common restaurants to those with the smallest index sums, leveraging dictionary lookups and sorting for optimal performance.
