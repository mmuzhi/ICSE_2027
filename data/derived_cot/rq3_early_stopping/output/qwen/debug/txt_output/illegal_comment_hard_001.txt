### Approach
The approach involves using dynamic programming to efficiently count these slices. The key insight is recognizing that for any element at index `i`, the number of arithmetic slices ending at `i` depends on the difference between `nums[i]` and `nums[i-1]` compared to the difference between `nums[i-1]` and `nums[i-2]`. 

1. **Dynamic Programming Array**: We use a dynamic programming array `dp` where `dp[i]` represents the number of arithmetic slices ending at index `i`.
2. **Base Case**: For the first two elements, there are no slices possible, so `dp[0]` and `dp[1]` are set to 0.
3. **Iterate Through Array**: For each index `i` starting from 2, check if the difference between `nums[i]` and `nums[i-1]` is the same as the difference between `nums[i-1]` and `nums[i-2]`. If they are the same, then `dp[i]` is incremented by `dp[i-1] + 1`; otherwise, `dp[i]` is set to 0.
4. **Summing Up**: The total number of arithmetic slices is the sum of all values in the `dp` array.

### Solution Code
```python
class Solution:
    def numberOfArithmeticSlices(self, nums: List[int]) -> int:
        n = len(nums)
        if n < 3:
            return 0
        
        dp = [0] * n
        total = 0
        
        for i in range(2, n):
            if nums[i] - nums[i-1] == nums[i-1] - nums[i-2]:
                dp[i] = dp[i-1] + 1
            else:
                dp[i] = 0
            total += dp[i]
        
        return total
```

### Explanation
- **Initialization**: The `dp` array is initialized to store the count of arithmetic slices ending at each index. The `total` variable accumulates the sum of all valid slices.
- **Base Case Handling**: If the array has fewer than 3 elements, it's impossible to form any arithmetic slices, so we return 0.
- **Dynamic Programming Update**: For each index `i` starting from 2, we check if the current element and the previous two elements form an arithmetic sequence. If they do, the number of new slices ending at `i` is `dp[i-1] + 1` (where `dp[i-1]` accounts for slices ending at `i-1` and the +1 accounts for the new slice starting at `i-2` and ending at `i`). If not, `dp[i]` is set to 0.
- **Result Calculation**: The `total` is updated by adding `dp[i]` for each valid slice found during the iteration.

This approach efficiently counts all arithmetic slices in linear time with constant space complexity, making it optimal for the given problem.
