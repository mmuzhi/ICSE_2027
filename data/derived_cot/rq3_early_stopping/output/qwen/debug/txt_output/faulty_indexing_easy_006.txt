=== RAW ANSWER ===

To solve this problem, we need to determine the maximum depth of a binary tree. The maximum depth is defined as the number of nodes along the longest path from the root node down to the farthest leaf node. The solution involves traversing the tree and keeping track of the maximum depth encountered during the traversal.

### Approach
The approach to solve this problem involves using a depth-first search (DFS) strategy. We use a stack to keep track of nodes and their corresponding depths. The stack is initialized with the root node and its depth (1). We then repeatedly pop the top element of the stack, update the maximum depth if the current node's depth is greater, and push its left and right children onto the stack with an incremented depth. This ensures that we explore all paths from the root to the leaves, updating the maximum depth whenever a deeper node is encountered.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        if not root:
            return 0
        stack = [[root, 1]]
        res = 0
        
        while stack:
            node, depth = stack.pop()
            if depth > res:
                res = depth
            if node.left:
                stack.append([node.left, depth + 1])
            if node.right:
                stack.append([node.right, depth + 1])
                
        return res
```

### Explanation
1. **Initialization**: The stack is initialized with the root node and its depth (1). If the root is `None`, the function returns 0 immediately, as an empty tree has a depth of 0.
2. **Traversal**: The loop continues as long as there are nodes in the stack. For each node popped from the stack, its depth is compared with the current maximum depth (`res`), updating `res` if the current node's depth is greater.
3. **Child Processing**: For each node, its left and right children (if they exist) are pushed onto the stack with their respective depths incremented by 1.
4. **Result**: After processing all nodes, the maximum depth found (`res`) is returned.

This approach efficiently explores all paths from the root to the leaves using a stack-based DFS, ensuring that the maximum depth is correctly identified. The time complexity is O(n), where n is the number of nodes in the tree, as each node is processed exactly once. The space complexity is also O(n) due to the stack usage, which in the worst case (a skewed tree) could store all nodes.
