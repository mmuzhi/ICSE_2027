To solve this problem, we need to find the kth largest level sum in a binary tree. The level sum is the sum of all node values at a particular level of the tree. The approach involves traversing the tree level by level, calculating the sum of each level, and then determining the kth largest sum among these level sums.

### Approach
1. **Level Order Traversal (BFS)**: We use a queue to traverse the tree level by level. Starting from the root node, we process each node in the current level, summing their values, and then enqueue their children for the next level.
2. **Store Level Sums**: For each level processed, we store the sum of node values in a list.
3. **Sort and Select**: After processing all levels, we sort the list of level sums in ascending order. The kth largest sum is then found by accessing the element at position `-k` (since Python lists are 0-indexed and negative indices count from the end).

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
from collections import deque

class Solution:
    def kthLargestLevelSum(self, root: Optional[TreeNode], k: int) -> int:
        if not root:
            return -1
        
        dq = deque([root])
        level_sums = []
        
        while dq:
            level_size = len(dq)
            current_sum = 0
            for _ in range(level_size):
                node = dq.popleft()
                current_sum += node.val
                if node.left:
                    dq.append(node.left)
                if node.right:
                    dq.append(node.right)
            level_sums.append(current_sum)
        
        level_sums.sort()
        if len(level_sums) < k:
            return -1
        return level_sums[-k]
```

### Explanation
1. **Initialization**: We start by checking if the root node is empty. If it is, we return -1 immediately.
2. **Queue Setup**: We initialize a queue with the root node to begin the level-order traversal.
3. **Processing Levels**: For each level, we process all nodes in the current level. The sum of node values for the current level is calculated, and the children of each node are added to the queue for the next level.
4. **Storing Sums**: The sum of each level is stored in a list called `level_sums`.
5. **Finding the kth Largest Sum**: After processing all levels, we sort the `level_sums` list in ascending order. The kth largest sum is then accessed using negative indexing (`-k`), which gives the element at position `len(level_sums) - k` from the end of the sorted list. If there are fewer than `k` levels, we return -1.

This approach efficiently processes each level, calculates the required sums, and retrieves the desired kth largest sum with optimal sorting and indexing.
