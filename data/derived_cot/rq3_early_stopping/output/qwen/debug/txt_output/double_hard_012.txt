import bisect
from typing import List

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        win = []
        ans = []
        for i, x in enumerate(nums):
            if i >= k:
                # Remove the element that leaves
                pos = bisect.bisect_left(win, nums[i-k])
                win.pop(pos)
            bisect.insort(win, x)
            if len(win) == k:
                if k % 2 == 1:
                    ans.append(win[k//2])
                else:
                    ans.append((win[k//2-1] + win[k//2]) / 2)

To solve this problem, we need to efficiently compute the median of every sliding window in a given array of integers. The sliding window moves one element at a time, and the window size is fixed to `k`. The challenge is to compute the median for each window efficiently, especially since the window slides continuously and elements are added and removed frequently.

### Approach
The approach involves maintaining a sorted list of the current window elements. For each new element entering the window, we insert it into the sorted list while maintaining the order. When the window slides, the oldest element is removed from the sorted list. The median is then calculated based on the current window's elements. 

1. **Maintaining a Sorted Window**: We use a list to keep track of the elements in the current window, sorted in ascending order. This allows us to efficiently access the median elements.
2. **Insertion and Removal**: For each new element, we insert it into the sorted list using binary search (via `bisect.insort`). When the window slides, we remove the oldest element using binary search to find its position (`bisect.bisect_left`) and then remove it.
3. **Median Calculation**: Depending on whether the window size `k` is odd or even, the median is either the middle element of the sorted list or the average of the two middle elements.

This approach ensures that each insertion and removal operation is handled in O(n) time due to the list operations, where `n` is the size of the window. While this is not the most efficient approach for very large windows, it is straightforward and works well for moderate-sized inputs.

### Solution Code
```python
import bisect
from typing import List

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        win = []
        ans = []
        for i, x in enumerate(nums):
            if i >= k:
                pos = bisect.bisect_left(win, nums[i-k])
                win.pop(pos)
            bisect.insort(win, x)
            if len(win) == k:
                if k % 2 == 1:
                    ans.append(win[k//2])
                else:
                    ans.append((win[k//2-1] + win[k//2]) / 2)
        return ans
```

### Explanation
1. **Initialization**: We initialize an empty list `win` to keep track of the current window elements and an empty list `ans` to store the median values.
2. **Iterating through the Array**: For each element in the array:
   - **Removal**: If the current index is greater than or equal to `k`, the oldest element (at index `i - k`) is removed from the window. The position of this element is found using `bisect.bisect_left`, and it is removed from the sorted list.
   - **Insertion**: The current element is inserted into the sorted list using `bisect.insort`, which maintains the sorted order.
   - **Median Calculation**: Once the window size reaches `k`, the median is calculated. For an odd `k`, the median is the middle element of the sorted list. For an even `k`, the median is the average of the two middle elements.
3. **Output**: The list `ans` contains the median values for each window, which is returned as the result.

This approach efficiently handles the sliding window median calculation by leveraging the sorted list and binary search operations, ensuring correctness while maintaining simplicity.
