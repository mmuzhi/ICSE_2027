=== RAW ANSWER ===

To solve this problem, we need to determine if all occurrences of numbers in the input array are unique. Specifically, we need to check if the frequency of each distinct number in the array is unique. 

### Approach
1. **Count Frequencies**: Use the `Counter` class from the `collections` module to count the occurrences of each number in the input array.
2. **Check Uniqueness of Frequencies**: Convert the counts obtained from the `Counter` into a list and then check if the length of this list is equal to the length of the set created from this list. If they are equal, it means all frequencies are unique; otherwise, there are duplicate frequencies.

### Solution Code
```python
from collections import Counter

class Solution:
    def uniqueOccurrences(self, arr: List[int]) -> bool:
        d = Counter(arr)
        counts = list(d.values())
        return len(counts) == len(set(counts))
```

### Explanation
1. **Counting Frequencies**: The `Counter` class efficiently counts the occurrences of each element in the input array. For example, if the input array is `[1, 2, 2, 3, 3, 3]`, the `Counter` will produce `{1: 1, 2: 2, 3: 3}`.
2. **Checking Uniqueness**: The values from the `Counter` (i.e., the counts of each element) are converted into a list. The set of these values removes any duplicates. If the length of the original list and the set are the same, it means all counts are unique. For instance, in the example above, the list `[1, 2, 3]` has no duplicates, so the function returns `True`. In contrast, for an input like `[1, 1, 2, 2]`, the counts are `[2, 2]`, which has duplicates, so the function returns `False`.

This approach efficiently checks for unique frequencies using basic operations on counts, ensuring optimal performance with a time complexity of O(n), where n is the number of elements in the input array.
