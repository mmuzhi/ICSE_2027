The key insight is that the last element of the postorder array will always be the root of the current subtree. We then use the inorder traversal to determine the left and right subtrees of the root. The left subtree consists of elements before the root in the inorder array, and the right subtree consists of elements after the root.

### Approach
1. **Mapping Inorder Indices**: Create a dictionary to map each value in the inorder array to its index for quick lookup.
2. **Recursive Tree Construction**: Use a helper function that takes the current subtree boundaries in both inorder and postorder arrays. The function:
   - **Base Case**: If the current subtree boundaries are invalid (left > right), return None.
   - **Root Selection**: The root of the current subtree is the last element in the current postorder segment.
   - **Recursive Calls**: The left and right subtrees are constructed by recursively calling the helper function with the appropriate segments of the inorder and postorder arrays.
3. **Initial Call**: Start the recursion with the entire inorder and postorder arrays.
