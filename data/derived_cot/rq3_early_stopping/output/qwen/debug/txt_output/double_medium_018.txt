=== RAW ANSWER ===

To solve this problem, we need to determine if it's possible to finish all courses given certain prerequisites. The solution involves checking for cycles in the prerequisite graph. If there's a cycle, it means some courses cannot be taken in any order, making it impossible to finish all courses.

### Approach
1. **Graph Construction**: Represent each course and its prerequisites as a directed graph using a dictionary where each key is a course, and the value is a list of its prerequisites.
2. **Cycle Detection**: Use Depth-First Search (DFS) to detect cycles in the graph. During DFS, maintain two sets: one to track visited nodes and another to track nodes currently in the recursion stack.
3. **DFS Execution**: For each course, if it hasn't been visited, initiate DFS. During DFS, if a prerequisite is found in the current recursion stack, a cycle exists, and the function returns false. If no cycles are found during DFS for all courses, return true.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        pre = defaultdict(list)
        for a, b in prerequisites:
            pre[a].append(b)
        
        visited = set()
        stack = set()
        
        def dfs(course):
            visited.add(course)
            stack.add(course)
            for p in pre[course]:
                if p not in visited:
                    if not dfs(p):
                        return False
                elif p in stack:
                    return False
            stack.remove(course)
            return True
        
        for course in range(numCourses):
            if course not in visited:
                if not dfs(course):
                    return False
        return True
```

### Explanation
1. **Graph Construction**: The graph is built using a defaultdict where each course points to its list of prerequisites. This helps in traversing the graph efficiently.
2. **Cycle Detection**: The DFS function checks for cycles by marking nodes as visited and maintaining a stack of nodes in the current recursion path. If a node being processed is found in the stack, it indicates a cycle.
3. **DFS Execution**: The main loop iterates over all courses. For each unvisited course, it starts DFS. If DFS finds a cycle, it immediately returns false. If all courses are processed without detecting any cycles, the function returns true, indicating all courses can be finished. This approach efficiently checks for cycles in the graph, ensuring correctness with a time complexity of O(N + E), where N is the number of courses and E is the number of prerequisites.
