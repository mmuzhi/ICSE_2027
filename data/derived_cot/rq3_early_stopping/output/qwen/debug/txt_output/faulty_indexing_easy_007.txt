=== RAW ANSWER ===

To solve this problem, we need to implement a binary search algorithm that correctly finds the index of a target value in a sorted array. The initial code had a bug because it used a linear search to check if the target was present, which is inefficient and incorrect for a binary search. Additionally, the binary search part did not handle the case where the target was not found in the array.

### Approach
1. **Binary Search Initialization**: We start with two pointers, `left` and `right`, initialized to 0 and the length of the array, respectively. These pointers define the current search interval.
2. **Binary Search Loop**: While the `left` pointer is less than the `right` pointer, we calculate the middle index `mid`. If the element at `mid` is less than the target, we adjust the `left` pointer to `mid + 1` to search the right half. Otherwise, we adjust the `right` pointer to `mid` to search the left half.
3. **Check for Target**: After the loop, the `left` pointer will be at the insertion point. We check if this position is within the array bounds and if the element at `left` matches the target. If so, return the index; otherwise, return -1.

### Solution Code
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums)
        
        while left < right:
            mid = (left + right) // 2
            if nums[mid] < target:
                left = mid + 1
            else:
                right = mid
        
        if left < len(nums) and nums[left] == target:
            return left
        else:
            return -1
```

### Explanation
1. **Initialization**: The `left` pointer starts at 0, and the `right` pointer starts at the length of the array. This setup allows us to handle the entire array as the initial search space.
2. **Loop Execution**: The loop continues as long as `left` is less than `right`. The middle index `mid` is calculated, and the element at `mid` is compared to the target. If the element is less than the target, the search continues in the right half by moving `left` to `mid + 1`. Otherwise, the search continues in the left half by moving `right` to `mid`.
3. **Result Check**: After exiting the loop, `left` points to the first occurrence of the target or the insertion point. If `left` is within the array bounds and the element at `left` matches the target, the index `left` is returned. Otherwise, -1 is returned indicating the target is not present in the array.

This approach efficiently narrows down the search space using binary search and correctly handles both found and not-found cases, ensuring optimal performance with a time complexity of O(log n).
