To solve this problem, we need to determine if it's possible to finish all courses given the prerequisites. The solution involves checking for cycles in the directed graph formed by the course dependencies. If there is a cycle, it's impossible to finish all courses; otherwise, it is possible.

### Approach
1. **Graph Construction**: Represent the courses and their prerequisites as a directed graph where each node is a course, and edges point from a course to its prerequisites.
2. **Cycle Detection**: Use Depth-First Search (DFS) to detect cycles in the graph. During the DFS traversal, if we encounter a node that is already in the current recursion stack, a cycle exists.
3. **State Tracking**: Maintain two sets during DFS: one to track nodes in the current recursion stack (`rec_stack`) and another to track nodes that have been fully processed (`visited`).

### Solution Code
```python
class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        pre = {i: [] for i in range(numCourses)}
        for a, b in prerequisites:
            pre[b].append(a)
        
        visited = set()
        rec_stack = set()
        
        def dfs(course):
            if course in rec_stack:
                return False
            if course in visited:
                return True
            
            rec_stack.add(course)
            for p in pre[course]:
                if not dfs(p):
                    return False
            rec_stack.remove(course)
            visited.add(course)
            return True
        
        for course in range(numCourses):
            if not dfs(course):
                return False
        return True
```

### Explanation
1. **Graph Construction**: The graph is built using a dictionary where each key is a course, and the value is a list of its prerequisites. This ensures that each course is included in the graph, even if it has no prerequisites.
2. **Cycle Detection**: The DFS function checks for cycles by maintaining a set (`rec_stack`) of nodes currently in the recursion stack. If a node is encountered again in the same stack, a cycle exists.
3. **State Tracking**: The `visited` set keeps track of nodes that have been processed without cycles. If a node is already in `visited`, its subgraph is assumed to be acyclic.
4. **Iterative Check**: The main loop calls DFS for each course. If any course returns `False` (indicating a cycle), the function immediately returns `False`. If all courses are processed without cycles, the function returns `True`.

This approach efficiently checks for cycles in the graph using DFS with state tracking, ensuring correctness while maintaining clarity and simplicity.
