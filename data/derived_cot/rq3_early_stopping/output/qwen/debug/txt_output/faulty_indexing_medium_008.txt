### Approach
1. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to the last index of the array.
2. **Edge Case Handling**: If the array has only one element, that element is trivially a peak.
3. **Binary Search Execution**: 
   - **Mid Calculation**: Compute the mid index during each iteration.
   - **Peak Check**: Determine if the mid element is a peak by checking if it is greater than or equal to its left and right neighbors (handling boundary cases where the mid element is at the start or end of the array).

class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        if n == 1:
            return 0

### Explanation

2. **Edge Case Handling**: If the array length is 1, the only element is the peak.
3. **Mid Calculation**: The mid index is calculated as the average of `low` and `high`.
4. **Peak Check**: The mid element is checked to see if it is a peak by verifying it is greater than or equal to its left and right neighbors. Boundary conditions (first or last element) are handled separately.
5. **Direction Decision**: If the mid element is not a peak, the search continues in the right half if the mid element is less than its right neighbor; otherwise, the search continues in the left half.
6. **Termination**: The loop terminates when a peak is found or the search space is exhausted, returning -1 if no peak is found (though the problem guarantees at least one peak).

This approach efficiently narrows down the search space using binary search, ensuring optimal performance with a time complexity of O(log n).
