To solve this problem, we need to efficiently compute the number of seats booked for each day from 1 to n based on a list of flight bookings. Each booking specifies a start day, end day, and the number of seats to be booked for those days. The challenge is to process these bookings in an optimal way without resorting to a brute-force approach, which would be too slow for large inputs.

### Approach
The solution leverages the difference array technique (also known as the prefix sum array technique) to efficiently compute the number of seats booked for each day. The key idea is to use a difference array to mark the start and end of each booking's effect. Here's how it works:

1. **Initialize a Difference Array**: Create an array `arr` of size `n+1` initialized to zero. This array will help in marking the changes in the number of seats booked.
2. **Mark Bookings in Difference Array**: For each booking `[start, end, seats]`:
   - Increment `arr[start-1]` by `seats` (indicating the start of the booking).
   - Decrement `arr[end]` by `seats` (indicating the end of the booking's effect).
3. **Compute Prefix Sum**: The prefix sum of the difference array gives the cumulative number of seats booked for each day. This is because the difference array marks the points where the number of seats changes, and the prefix sum reconstructs the actual values.

This approach ensures that each booking is processed in constant time, and the overall time complexity is O(n + m), where m is the number of bookings, making it very efficient.

```python
class Solution:
    def corpFlightBookings(self, bookings: List[List[int]], n: int) -> List[int]:
        arr = [0] * (n + 1)
        for lv, ar, seats in bookings:
            arr[lv - 1] += seats
            arr[ar] -= seats
        return list(accumulate(arr[:n]))
```

### Explanation
1. **Initialization**: The array `arr` is initialized to store the difference in the number of seats booked. The size `n+1` is chosen to handle all days from 1 to n without index issues.
2. **Processing Bookings**: For each booking, the start day (`lv`) is adjusted to zero-based indexing (`lv-1`), and the seats are added to this position. The end day (`ar`) is marked by subtracting the seats at position `ar` (which is one past the last day the booking affects).
3. **Prefix Sum Calculation**: The `accumulate` function computes the prefix sum of the first `n` elements of the difference array. This prefix sum gives the total seats booked for each day from 1 to n, as it effectively sums up all the changes marked in the difference array up to that point.

This method efficiently handles overlapping bookings and ensures that the solution is both optimal and easy to understand.
