To solve this problem, we need to compute for each element in the array the sum of absolute differences between that element and every other element in the array. The challenge is to do this efficiently, especially for large arrays, without resorting to a brute-force O(n^2) approach.

### Approach
1. **Sort the Array**: Sorting helps in efficiently calculating the sum of absolute differences by leveraging the properties of sorted elements. 
2. **Prefix Sum Array**: Construct a prefix sum array from the sorted array to quickly access the sum of any subarray.
3. **Binary Search**: For each element in the original array, use binary search (via `bisect_right`) to determine how many elements are less than or equal to it in the sorted array. This gives the count of elements on the left and right of the current element when considering the original array.
4. **Calculate Sums**: For each element, compute the sum of absolute differences for elements to its left and right separately. The left sum is derived from elements less than or equal to the current element, and the right sum from elements greater than the current element. The total sum is the sum of these two parts.

### Solution Code
```python
import bisect

class Solution:
    def getSumAbsoluteDifferences(self, nums: List[int]) -> List[int]:
        n = len(nums)
        sorted_nums = sorted(nums)
        prefix = [0] * n
        if n > 0:
            prefix[0] = sorted_nums[0]
            for i in range(1, n):
                prefix[i] = prefix[i-1] + sorted_nums[i]
        
        res = []
        for x in nums:
            pos = bisect.bisect_right(sorted_nums, x)
            left_sum = x * pos - (prefix[pos-1] if pos > 0 else 0)
            right_sum = (prefix[n-1] - (prefix[pos-1] if pos > 0 else 0)) - x * (n - pos)
            res.append(left_sum + right_sum)
        
        return res
```

### Explanation
1. **Sorting and Prefix Sum**: The array is sorted to facilitate efficient calculation of sums. A prefix sum array is built from this sorted array to allow quick summation of any subarray.
2. **Binary Search**: For each element in the original array, `bisect_right` is used to find the position where the element would be inserted to maintain the sorted order, effectively counting how many elements are less than or equal to it.
3. **Sum Calculation**: The left sum is calculated by multiplying the current element by the count of elements less than or equal to it and subtracting the sum of those elements. The right sum is derived by subtracting the sum of elements greater than the current element from their total and then subtracting the product of the current element and their count.
4. **Efficiency**: The approach efficiently computes the result in O(n log n) time due to sorting and binary search, which is optimal for large input sizes. The space complexity is O(n) due to the storage of the prefix sum array.
