To solve this problem, we need to determine if two strings are almost equivalent based on the frequency of each character in the strings. The strings are considered almost equivalent if, for every character from 'a' to 'z', the absolute difference in their counts between the two strings is at most 3.

### Approach
1. **Frequency Array Initialization**: We initialize an array `A` of size 26 (for each letter in the alphabet) with zeros. This array will store the difference in counts of each character between the two strings.
2. **Count Characters in First String**: For each character in the first string (`word1`), we increment the corresponding index in the array `A` based on the character's position in the alphabet.
3. **Count Characters in Second String**: For each character in the second string (`word2`), we decrement the corresponding index in the array `A`.
4. **Check Conditions**: After processing both strings, we check if all elements in the array `A` satisfy the condition that their absolute value is at most 3. If any element violates this condition, the strings are not almost equivalent.

### Solution Code
```python
class Solution:
    def checkAlmostEquivalent(self, word1: str, word2: str) -> bool:
        A = [0] * 26
        for char in word1:
            A[ord(char) - ord("a")] += 1
        for char in word2:
            A[ord(char) - ord("a")] -= 1
        return all(-3 <= f <= 3 for f in A)
```

### Explanation
1. **Initialization**: The array `A` is initialized to store the difference in character counts between `word1` and `word2`.
2. **Processing `word1`**: For each character in `word1`, we convert the character to its corresponding index (0 for 'a', 1 for 'b', etc.) and increment the value at that index in `A`.
3. **Processing `word2`**: For each character in `word2`, we convert the character to its index and decrement the value at that index in `A`.
4. **Checking Conditions**: The `all` function checks if every element in `A` is within the range [-3, 3]. If all elements meet this condition, the function returns `True`, indicating the strings are almost equivalent; otherwise, it returns `False`.

This approach efficiently checks the almost equivalence by leveraging a frequency array and ensures optimal performance with a linear pass through each string.
