### Approach
1. **Initial Condition Check**: The initial condition `if target not in nums` is not necessary for binary search in a sorted array. Binary search inherently checks for the presence of the target during the search process, so this check can be removed for efficiency.
2. **Binary Search Logic**: The binary search logic in the provided code was incorrect. Specifically:
   - The initial `right` pointer was set to `len(nums)`, which is one past the last index. This was adjusted to `len(nums) - 1` to correctly define the search space.
   - The condition `if nums[mid] < target` correctly moves the `left` pointer to `mid + 1`, but the `else` branch should move the `right` pointer to `mid - 1` to narrow the search space properly.
   - The loop now correctly checks if the target is found at `nums[mid]` and returns the index immediately. If the target is not found, the loop exits and returns `-1`.

### Solution Code
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        
        while left <= right:
            mid = (left + right) // 2
            if nums[mid] == target:
                return mid
            elif nums[mid] < target:
                left = mid + 1
            else:
                right = mid - 1
        
        return -1
```

### Explanation
1. **Initialization**: The `left` pointer starts at the beginning of the array (index 0), and the `right` pointer starts at the last valid index (`len(nums) - 1`).
2. **Loop Execution**: The loop continues as long as `left` is less than or equal to `right`, ensuring the search space is valid.
3. **Mid Calculation**: The middle index `mid` is calculated as the average of `left` and `right`.
4. **Comparison**:
   - If `nums[mid]` equals the target, the index `mid` is returned immediately.
   - If `nums[mid]` is less than the target, the `left` pointer is moved to `mid + 1` to search the right half.
   - If `nums[mid]` is greater than the target, the `right` pointer is moved to `mid - 1` to search the left half.
5. **Termination**: If the loop exits without finding the target, `-1` is returned, indicating the target is not present in the array.

This approach ensures an efficient O(log n) time complexity, typical of binary search, making it suitable for large datasets.
