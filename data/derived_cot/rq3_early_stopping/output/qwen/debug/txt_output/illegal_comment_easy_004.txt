To solve this problem, we need to determine for each number in a given list how many numbers are smaller than it. The solution involves counting the occurrences of each number and then using these counts to efficiently compute the required values for each element.

### Approach
1. **Frequency Array Initialization**: We initialize an array `count` of size 101 (assuming numbers are between 0 and 100) to keep track of the frequency of each number in the input list.
2. **Count Frequencies**: We iterate through the input list and increment the corresponding index in the `count` array for each number encountered.
3. **Compute Result**: For each number in the input list, we compute the sum of frequencies of all numbers strictly smaller than it by summing the elements in the `count` array from index 0 to `num-1`. This sum gives the count of numbers smaller than the current number.

### Solution Code
```python
class Solution:
    def smallerNumbersThanCurrent(self, nums: List[int]) -> List[int]:
        count = [0] * 101
        res = []
        for num in nums:
            count[num] += 1
        for num in nums:
            res.append(sum(count[:num]))
        return res
```

### Explanation
1. **Frequency Array**: The `count` array is initialized to 101 zeros to handle numbers in the range 0 to 100. Each index in this array corresponds to a number, and the value at each index represents how many times that number appears in the input list.
2. **Counting Frequencies**: By iterating through the input list, we populate the `count` array. For each number encountered, we increment the corresponding index in the `count` array.
3. **Calculating Result**: For each number in the input list, we sum the values in the `count` array from index 0 to `num-1`. This sum represents the total number of elements in the input list that are smaller than the current number. The result for each number is then appended to the result list.

This approach efficiently leverages the frequency array to avoid nested loops, thus optimizing the solution to run in O(n) time for counting frequencies and O(n) time for computing the result, leading to an overall O(n) time complexity. The space complexity is O(1) due to the fixed-size frequency array.
