The solution involves efficiently counting the occurrences of `s2` in the concatenated string without explicitly constructing it, leveraging cycle detection to optimize performance for large inputs.

### Approach
1. **Initial Checks**: If `s2` is empty or contains characters not present in `s1`, return 0 since `s2` cannot be formed.
2. **Filter `s1`**: Remove characters from `s1` that are not in `s2` to simplify the search.
3. **Cycle Detection**: Use a dictionary to track the starting positions of `s2` formation to detect cycles in the search process. This helps in reducing redundant computations for large `n1`.
4. **Simulation**: For each character in `s2`, find its occurrence in `s1`. If not found in the current copy, increment the copy count and reset the search to the beginning of `s1`.
5. **Cycle Utilization**: Once a cycle is detected, calculate the number of `s2` substrings formed in the cycle and use this to determine the total occurrences efficiently without processing each copy individually.
