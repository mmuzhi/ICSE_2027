### Approach
1. **Identify Fixed Numbers**: A fixed number is one that appears on both sides of the same card. These numbers cannot be used without potentially causing duplicates if they appear on other cards.
2. **Check for Smallest Available Number**: Iterate through all numbers present on either side of any card. The smallest number not found in the fixed numbers set is the answer, as it can be used without causing duplicates.

### Solution Code
```python
class Solution:
    def flipgame(self, fronts: List[int], backs: List[int]) -> int:
        bad = set(i for i, j in zip(fronts, backs) if i == j)
        all_nums = set(fronts + backs)
        for num in sorted(all_nums):
            if num not in bad:
                return num
        return 0
```

### Explanation
1. **Fixed Numbers Identification**: The set `bad` is created by iterating through each card and adding numbers that appear on both sides (i.e., `fronts[i] == backs[i]`).
2. **Available Numbers Check**: The set `all_nums` contains all unique numbers from both `fronts` and `backs`. By sorting this set, we can efficiently find the smallest number not in `bad`.
3. **Result Determination**: The loop checks each number in sorted order. The first number not found in `bad` is returned as it can be used without causing duplicates. If no such number exists, the function returns 0.

This approach efficiently narrows down the candidates for the smallest usable number by leveraging set operations and sorted iteration, ensuring optimal performance.
