To solve this problem, we need to find common restaurants between two lists that appear earliest in both lists. The solution involves identifying restaurants present in both lists and then determining which of these have the smallest sum of their indices in the two lists. If multiple restaurants share the same smallest sum, all of them should be included in the result.

### Approach
1. **Create a Dictionary for List2**: We first create a dictionary where each restaurant from the second list (`list2`) is mapped to its index. This allows us to quickly look up the index of any restaurant from the first list (`list1`) in `list2`.
2. **Collect Common Restaurants with Indices**: For each restaurant in `list1`, if it exists in `list2`, we calculate the sum of its indices in both lists and store this sum along with the restaurant name in a list.
3. **Sort and Filter Results**: The collected list is sorted based on the sum of indices. We then iterate through this sorted list to collect all restaurants that have the smallest sum. If multiple restaurants have the same smallest sum, they are all included in the result.

### Solution Code
```python
class Solution:
    def findRestaurant(self, list1: List[str], list2: List[str]) -> List[str]:
        d2 = {}
        for i in range(len(list2)):
            d2[list2[i]] = i
        
        l = []
        for i in range(len(list1)):
            if list1[i] in d2:
                l.append([i + d2[list1[i]], list1[i]])
        
        if not l:
            return []
        
        l.sort(key=lambda x: x[0])
        l1 = []
        l1.append(l[0][1])
        for i in range(1, len(l)):
            if l[i][0] == l[i-1][0]:
                l1.append(l[i][1])
            else:
                break
        return l1
```

### Explanation
1. **Dictionary Creation**: The dictionary `d2` is built from `list2`, where each key is a restaurant name and the value is its index in `list2`. This allows O(1) lookups for restaurant indices in `list2`.
2. **Finding Common Restaurants**: For each restaurant in `list1`, we check if it exists in `d2`. If it does, we compute the sum of its indices in both lists and store this sum along with the restaurant name in a list `l`.
3. **Handling Edge Cases**: If no common restaurants are found (`l` is empty), we return an empty list.
4. **Sorting and Filtering**: The list `l` is sorted by the sum of indices. We then iterate through the sorted list to collect all restaurants that have the smallest sum. The first restaurant in the sorted list is added to the result, and subsequent restaurants with the same sum are added until a different sum is encountered.
5. **Result Compilation**: The result list `l1` contains all restaurants with the smallest sum of indices, which is then returned.

This approach efficiently finds the earliest common restaurants by leveraging dictionary lookups and sorting, ensuring optimal performance and correctness.
