=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays efficiently. The median is the middle value in an ordered list of numbers. If the total number of elements is even, the median is the average of the two middle numbers. 

### Approach
1. **Merge the Arrays**: Since both arrays are sorted, we can merge them into a single sorted array by comparing elements from both arrays and appending the smaller element to the result array. This ensures the merged array remains sorted.
2. **Calculate the Median**: After merging, if the length of the merged array is odd, the median is the middle element. If the length is even, the median is the average of the two middle elements.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        p1 = 0
        p2 = 0
        new = []
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] <= nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
        
        n = len(new)
        if n % 2 == 1:
            return float(new[n//2])
        else:
            return (new[n//2 - 1] + new[n//2]) / 2.0
```

### Explanation
1. **Merging Arrays**: The algorithm uses two pointers to traverse both arrays. The smaller element between the current pointers of the two arrays is appended to the result array, and the corresponding pointer is incremented. This ensures the merged array is sorted.
2. **Handling Remaining Elements**: After one array is exhausted, the remaining elements of the other array are appended to the result array since they are already in order.
3. **Finding the Median**: The median is determined by checking the length of the merged array. If the length is odd, the median is the middle element. If even, it is the average of the two middle elements.

This approach efficiently merges the arrays in linear time, O(n + m), where n and m are the lengths of the two arrays, making it optimal for large input sizes.
