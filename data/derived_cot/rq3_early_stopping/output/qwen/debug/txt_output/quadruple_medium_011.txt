To solve this problem, we need to generate all valid IP addresses from a given string. An IP address consists of four octets, each ranging from 0 to 255. The key challenge is to explore all possible ways to split the string into four parts, ensuring each part is a valid octet without leading zeros (unless the octet is 0 itself).

### Approach
1. **Backtracking**: We use a backtracking approach to explore all possible ways to split the string into four parts. The algorithm starts from the beginning of the string and tries to form valid octets by considering 1 to 3 digits at each step (since octets cannot exceed 3 digits).
2. **Base Case**: When the entire string is processed (i.e., the current index reaches the end of the string) and exactly four octets are formed, the result is added to the list of valid IP addresses.
3. **Termination Conditions**: The recursion terminates if more than four octets are formed or if the current index exceeds the string length.
4. **Handling Leading Zeros**: If the current character is '0', only one digit is considered to avoid leading zeros in multi-digit numbers.
5. **Validation**: For non-zero starting characters, the algorithm checks all possible 1 to 3-digit combinations to ensure they form a number less than 256.

### Solution Code
```python
class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i >= len(s):
                return
            
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            
            j = 0
            while j < 3 and i + j + 1 <= len(s):
                num = int(s[i:i+j+1])
                if num < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Initialization**: The `res` list stores all valid IP addresses, and `cur` list keeps track of the current octets being formed.
2. **Backtracking Function**: The `backtrack` function recursively processes the string starting from index `i`:
   - **Base Case**: If the entire string is processed (`i == len(s)`) and four octets are formed (`len(cur) == 4`), the current octets are joined into a string and added to `res`.
   - **Termination**: If more than four octets are formed or the index `i` exceeds the string length, the function returns without further processing.
   - **Leading Zero Handling**: If the current character is '0', only one digit is considered to avoid invalid octets like "01".
   - **Non-Zero Handling**: For non-zero characters, the function checks all possible 1 to 3-digit combinations. Each combination is converted to an integer and checked to ensure it is less than 256. Valid combinations are added to `cur` and further processed recursively.
3. **Result**: After processing the entire string, the list of valid IP addresses (`res`) is returned.

This approach efficiently explores all valid IP addresses by leveraging backtracking and careful handling of edge cases like leading zeros and octet validation.
