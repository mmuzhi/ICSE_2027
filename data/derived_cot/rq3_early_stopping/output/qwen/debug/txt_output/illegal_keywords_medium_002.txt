### Approach
1. **Problem Analysis**: The problem requires us to process a sorted array and ensure that each element appears at most twice. The challenge is to do this efficiently in-place without using extra space.
2. **Intuition**: Since the array is sorted, all duplicates of a number will be consecutive. We can use a two-pointer technique where one pointer (`j`) tracks the last position of the result, and another pointer (`i`) iterates through the array.
3. **Algorithm Selection**: We iterate through the array starting from the second element. For each element, we check if it is the same as the previous element. If it is, we increment a count; otherwise, we reset the count. If the count does not exceed two, we update the result array at position `j` and increment `j`.
4. **Complexity Analysis**: The algorithm runs in O(n) time, where n is the number of elements in the array, as each element is processed exactly once. The space complexity is O(1) since we only use a few extra variables for tracking.

class Solution:

### Explanation
1. **Initial Check**: If the array has fewer than 2 elements, return its length immediately since no duplicates can be removed.
2. **Two-Pointer Technique**: 
   - `j` starts at 1, indicating the position where the next valid element should be placed.
   - `count` keeps track of how many times the current element has been encountered.
3. **Iteration**: For each element starting from the second one:
   - If the current element is the same as the previous one, increment `count`.
   - If it's different, reset `count` to 1.
   - If `count` is 2 or less, place the current element at position `j` and increment `j`.
4. **Result**: The new length of the array is `j + 1` because `j` is zero-based, and the result array includes elements from index 0 to `j`.
