### Approach
1. **Problem Analysis**: The task involves finding the minimum value in every possible contiguous subarray of the given array and summing these minima. A brute-force approach would involve checking all possible subarrays, which is O(n^2) in time complexity, making it inefficient for large arrays.

2. **Intuition**: The key insight is to recognize that each element in the array contributes to the sum based on how many subarrays it is the minimum. For each element at index `i`, the number of subarrays where it is the minimum can be determined by finding the previous element that is smaller than it (to the left) and the next element that is smaller than it (to the right). 

3. **Algorithm Selection**: We use a monotonic stack to efficiently compute the left and right boundaries for each element. The left boundary is the nearest index to the left where the element is smaller than the current element. The right boundary is the nearest index to the right where the element is smaller than the current element. The number of subarrays where the current element is the minimum is then calculated as `(i - left_bound[i]) * (right_bound[i] - i)`.

4. **Complexity Analysis**: The algorithm processes each element twice (once for the left pass and once for the right pass), resulting in an O(n) time complexity, where n is the number of elements in the array. The space complexity is O(n) due to the stack usage.
