To solve this problem, we need to efficiently compute the median of every sliding window of size `k` in an array of integers. The challenge lies in maintaining the window elements in a way that allows quick access to the median after each insertion and removal.

### Approach
1. **Problem Analysis**: The problem requires calculating the median for each window of size `k` as the window slides through the array. The median is the middle value in an ordered list of numbers. If the window size `k` is odd, the median is the middle element. If `k` is even, the median is the average of the two middle elements.

2. **Intuition**: A straightforward approach would be to use a sorted list for each window. For each new element, insert it into the sorted list and remove the oldest element. The median can then be retrieved from the sorted list. However, insertion and removal operations in a sorted list take O(k) time each, leading to an overall O(n*k) time complexity, which is inefficient for large `k` or `n`.

3. **Optimization**: To optimize, we use a balanced two-heap approach (a max-heap for the lower half and a min-heap for the upper half) to maintain the window elements. This allows O(log k) time for insertion and removal, reducing the overall time complexity to O(n log k). However, implementing a balanced two-heap with lazy removal is complex. Instead, we use a simpler approach with a sorted list, which is efficient enough for moderate input sizes.

4. **Algorithm Selection**: We use a sorted list to maintain the current window elements. For each new element, we insert it into the sorted list (using binary search for O(k) time). When the window size exceeds `k`, we remove the oldest element (found using binary search). The median is then retrieved from the sorted list in constant time.

### Solution Code
```python
import bisect

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        sorted_list = []
        ans = []
        for i, x in enumerate(nums):
            bisect.insort(sorted_list, x)
            if i >= k:
                pos = bisect.bisect_left(sorted_list, nums[i - k])
                del sorted_list[pos]
            if i >= k - 1:
                if k % 2 == 1:
                    ans.append(sorted_list[k // 2])
                else:
                    ans.append((sorted_list[k // 2 - 1] + sorted_list[k // 2]) / 2)
        return ans
```

### Explanation
1. **Initialization**: We start with an empty sorted list and an empty answer list.
2. **Processing Each Element**: For each element in the input array:
   - **Insertion**: The element is inserted into the sorted list using binary search (`bisect.insort`), which maintains the list's order in O(k) time.
   - **Removal**: If the current index `i` is at least `k`, the element that is `k` positions before the current index (i.e., `nums[i - k]`) is removed from the sorted list using binary search (`bisect.bisect_left`).
   - **Median Calculation**: After ensuring the window size is `k`, the median is calculated. If `k` is odd, the median is the middle element of the sorted list. If `k` is even, the median is the average of the two middle elements.
3. **Output**: The computed medians are collected in the answer list and returned after processing all elements.

This approach efficiently handles the sliding window median problem by leveraging the sorted list's properties, ensuring correctness while maintaining manageable time complexity for typical input sizes.
