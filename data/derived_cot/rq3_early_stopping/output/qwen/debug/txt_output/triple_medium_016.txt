The `change` list provides the maximum allowed digit for each position (0-9). The goal is to replace each digit in `num` with the largest possible digit allowed by the corresponding position in `change` if it results in a larger number.

### Approach
1. **Problem Analysis**: The problem requires us to process each digit in the string `num` from left to right. 2. **Intuition**: By processing digits from left to right and replacing them with the largest allowed digit whenever possible, we ensure the resulting number is the maximum possible. This approach leverages the fact that replacing a digit earlier in the number has a greater impact on the overall value.

4. **Complexity Analysis**: The algorithm processes each digit exactly once, making it linear in time complexity, O(n). The space complexity is O(n) due to the conversion of the string to a list for in-place modification.

### Solution Code
```python
class Solution:
    def maximumNumber(self, num: str, change: List[int]) -> str:
        num_list = list(num)
        for i, ch in enumerate(num_list):
            x = int(ch)
            if x < change[x]:
                num_list[i] = str(change[x])
        return ''.join(num_list)
```

1. **Initialization**: The input string `num` is converted into a list of characters for easy manipulation.
2. **Iteration and Replacement**: For each character in the list, the corresponding digit is converted to an integer. If this digit is less than the value specified in the `change` list at the same index, it is replaced with the value from `change`.
3. **Result Construction**: After processing all digits, the modified list is joined back into a string, which is returned as the result.

This approach efficiently ensures that each digit is replaced only if it contributes to a larger number, leveraging the left-to-right processing to maximize the overall value. The solution is both intuitive and optimal for the given problem constraints.
