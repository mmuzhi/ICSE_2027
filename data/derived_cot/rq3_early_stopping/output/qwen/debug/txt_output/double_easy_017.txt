=== RAW ANSWER ===

To solve this problem, we need to determine for each element in an array the count of numbers that are strictly less than the current element. The solution involves efficiently counting these occurrences without resorting to a brute-force approach, which would be computationally expensive.

### Approach
1. **Problem Analysis**: The task is to count how many numbers in the array are smaller than each element. A naive approach would involve comparing each element with every other element, resulting in a time complexity of O(n^2), which is inefficient for large arrays.
2. **Intuition**: By sorting the array, we can use a prefix sum array to store cumulative counts of elements up to each position. This allows us to quickly determine the number of elements less than the current element by leveraging the sorted order.
3. **Algorithm Selection**: 
   - **Sorting**: The array is sorted to establish an order where elements are in ascending sequence.
   - **Prefix Sum Array**: After sorting, we create a prefix sum array where each entry at index `i` contains the count of elements less than or equal to the element at that position. This helps in efficiently determining the count of elements less than the current element by subtracting the prefix sum up to the current element's value from the total elements.
4. **Complexity Analysis**: 
   - **Time Complexity**: Sorting the array takes O(n log n) time, and constructing the prefix sum array takes O(n) time. Thus, the overall time complexity is O(n log n).
   - **Space Complexity**: The space complexity is O(n) due to the storage of the sorted array and the prefix sum array.

### Solution Code
```python
class Solution:
    def smallerNumbersThanCurrent(self, nums: List[int]) -> List[int]:
        sorted_nums = sorted(nums)
        prefix = [0] * len(sorted_nums)
        if len(sorted_nums) > 0:
            prefix[0] = 0
            for i in range(1, len(sorted_nums)):
                prefix[i] = prefix[i-1] + (1 if sorted_nums[i] == sorted_nums[i-1] else 0)
            prefix[-1] = len(sorted_nums)
        result = []
        for num in nums:
            index = 0
            left, right = 0, len(sorted_nums) - 1
            while left <= right:
                mid = (left + right) // 2
                if sorted_nums[mid] < num:
                    index = mid + 1
                    left = mid + 1
                elif sorted_nums[mid] > num:
                    right = mid - 1
                else:
                    left = mid + 1
            count_less = index
            result.append(count_less)
        return result
```

### Explanation
1. **Sorting**: The input array is sorted to facilitate the creation of a prefix sum array that counts occurrences up to each position.
2. **Prefix Sum Array**: The prefix array is built by iterating through the sorted array. Each entry in the prefix array represents the cumulative count of elements up to that position, adjusted for duplicates.
3. **Binary Search**: For each element in the original array, binary search is used on the sorted array to find the position where the element would fit. The number of elements less than the current element is then derived from this position.
4. **Result Construction**: The count of elements less than each original element is collected into a result array, which is returned as the final output.

This approach efficiently leverages sorting and binary search to minimize the time complexity, making it suitable for larger input sizes.
