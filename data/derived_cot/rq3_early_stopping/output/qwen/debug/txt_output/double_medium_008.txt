To solve this problem, we need to determine the number of booked seats for each row in an airplane with `n` rows, given a list of bookings. Each booking specifies a range of rows (from `l` to `r`) and the number of seats to book. The solution involves efficiently calculating the number of seats for each row by using a difference array technique.

### Approach
1. **Difference Array Technique**: We use a difference array to mark the start and end of each booking. This allows us to efficiently compute the prefix sum, which gives the number of seats for each row.
2. **Handling Bookings**: For each booking `[l, r, seats]`, we increment the start of the range `l` by `seats` and decrement the end of the range `r+1` by `seats` (to ensure the effect is only up to `r`).
3. **Prefix Sum Calculation**: After processing all bookings, we compute the prefix sum of the difference array from index 1 to `n` (inclusive) to get the number of seats for each row.

### Solution Code
```python
from typing import List

class Solution:
    def corpFlightBookings(self, bookings: List[List[int]], n: int) -> List[int]:
        arr = [0] * (n + 2)
        for l, r, seats in bookings:
            arr[l] += seats
            arr[r + 1] -= seats
        
        result = []
        current = 0
        for i in range(1, n + 1):
            current += arr[i]
            result.append(current)
        return result
```

### Explanation
1. **Initialization**: We initialize an array `arr` of size `n + 2` with zeros. This extra space ensures we can safely handle the end of the range without index errors.
2. **Processing Bookings**: For each booking `[l, r, seats]`, we add `seats` to the start of the range `l` and subtract `seats` from the position `r + 1`. This marks the beginning and end of the booking's effect on the rows.
3. **Prefix Sum Calculation**: By iterating through the array from index 1 to `n`, we maintain a running sum (`current`) that accumulates the values from the difference array. This running sum gives the number of seats for each row, which is stored in the result list.
4. **Result**: The result list contains the number of booked seats for each row from 1 to `n`, computed efficiently using the difference array and prefix sum technique. This approach ensures optimal performance with a time complexity of O(n + m), where `m` is the number of bookings.
