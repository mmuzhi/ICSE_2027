To solve this problem, we need to compute the sum of the minimum values of all contiguous subarrays in a given array. The challenge is to efficiently calculate this sum without resorting to a brute-force approach, which would be too slow for large arrays.

### Approach
The approach involves using a monotonic increasing stack to keep track of elements and their positions. The key idea is to determine for each element how many subarrays it is the minimum of. For each element, the number of such subarrays is determined by the number of elements to its left that are greater than it and the number of elements to its right that are greater than or equal to it. 

1. **Stack Initialization**: We start by initializing a stack to keep track of indices of elements in the array.
2. **Iterate Through the Array**: For each element in the array, we pop elements from the stack if they are greater than the current element. This helps in identifying the nearest greater elements to the left and right.
3. **Calculate Contributions**: For each popped element, we calculate its contribution to the result by multiplying its value by the number of subarrays where it is the minimum. The number of such subarrays is given by the product of the distances to the previous greater element and the next greater element.
4. **Modulo Operation**: Since the result can be very large, we return it modulo \(10^9 + 7\).

### Solution Code
```python
class Solution:
    def sumSubarrayMins(self, arr: List[int]) -> int:
        stack = []
        res = 0
        arr = [float('-inf')] + arr + [float('-inf')]
        for i, num in enumerate(arr):
            while stack and arr[stack[-1]] > num:
                cur = stack.pop()
                left = cur - (stack[-1] if stack else -1)
                right = i - cur
                res += arr[cur] * left * right
            stack.append(i)
        return res % (10**9 + 7)
```

### Explanation
1. **Stack Initialization**: The stack is used to maintain indices of elements in a way that ensures the elements are in increasing order. This helps in efficiently finding the next greater element for any given element.
2. **Array Augmentation**: The array is augmented with `float('-inf')` at both ends to handle edge cases where the first or last elements might be the minimum in their subarrays.
3. **Processing Each Element**: For each element, we pop elements from the stack that are greater than the current element. This identifies the nearest greater elements to the left and right.
4. **Contribution Calculation**: For each popped element, the number of subarrays where it is the minimum is calculated as the product of the distances to the previous greater element (left) and the next greater element (right). This product is then multiplied by the element's value and added to the result.
5. **Modulo Operation**: The result is returned modulo \(10^9 + 7\) to handle large numbers and prevent overflow.

This approach efficiently computes the sum of minimum subarray values in \(O(n)\) time complexity using a single pass through the array and a stack to keep track of necessary elements. The space complexity is \(O(n)\) due to the stack usage.
