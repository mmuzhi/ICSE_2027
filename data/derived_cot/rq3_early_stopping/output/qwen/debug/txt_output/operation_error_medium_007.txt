To solve this problem, we need to find the minimum eating speed (k) such that Koko can eat all the piles of bananas within `h` hours. The solution involves using binary search to efficiently determine the smallest speed that satisfies the condition.

### Approach
1. **Binary Search Setup**: The binary search is performed between the minimum possible speed (1 banana per hour) and the maximum possible speed (the largest pile of bananas). This range is chosen because Koko can always eat one pile per hour at the maximum speed, and the minimum speed is 1.
2. **Check Function**: For a given speed `k`, the check function calculates the total time required to eat all piles by summing the ceiling of each pile's size divided by `k`. If this total time is less than or equal to `h`, the speed `k` is feasible.
3. **Binary Search Execution**: The binary search adjusts the search range based on whether the current speed is feasible. If the current speed is too slow (total time exceeds `h`), the search moves to higher speeds. Otherwise, it searches for a potentially lower feasible speed.

### Solution Code
```python
class Solution:
    def minEatingSpeed(self, piles: List[int], h: int) -> int:
        def check(x):
            return sum((pile + x - 1) // x for pile in piles) <= h
        
        l = 1
        r = max(piles)
        while l < r:
            mid = (l + r) // 2
            if not check(mid):
                l = mid + 1
            else:
                r = mid
        return l
```

### Explanation
1. **Binary Search Initialization**: The search starts with `l` set to 1 (minimum speed) and `r` set to the largest pile size (maximum speed).
2. **Mid Calculation**: In each iteration, the midpoint `mid` is calculated as the average of `l` and `r`.
3. **Feasibility Check**: The `check` function computes the total time required to eat all piles at speed `mid` using ceiling division (achieved via integer arithmetic `(pile + mid - 1) // mid`). If the total time is within `h` hours, the speed is feasible, and the search continues in the lower half; otherwise, the search moves to higher speeds.
4. **Result**: The loop exits when `l` equals `r`, which is the minimum feasible speed that satisfies the condition.

This approach efficiently narrows down the search space using binary search, ensuring optimal performance even for large input sizes.
