To solve this problem, we need to compute the median of every sliding window in a given list of integers. The sliding window moves one element at a time, and the window size is fixed to `k`. The median is the middle value in the sorted list of the window. If the window size is even, the median is the average of the two middle values.

### Approach
1. **Problem Analysis**: The challenge is to efficiently compute the median for each sliding window. A naive approach would involve sorting the window each time, leading to O(n*k log k) time complexity, which is inefficient for large inputs. Instead, we use a balanced BST (implemented as a sorted list) to maintain the window elements, allowing efficient insertion, deletion, and median retrieval.

2. **Intuition**: By maintaining a sorted list of the current window elements, we can quickly access the median elements. The sorted list allows us to insert and remove elements in O(k) time per operation, which is acceptable since `k` is fixed.

3. **Algorithm Selection**: We use a sorted list to dynamically maintain the window elements. For each new element added to the window, we insert it into the sorted list. If the window size exceeds `k`, we remove the oldest element (the one that is no longer in the window). After each insertion and potential removal, we check if the window size is `k` and compute the median accordingly.

4. **Complexity Analysis**: Each insertion and removal operation in the sorted list takes O(k) time. Since we perform these operations for each of the `n` elements, the overall time complexity is O(n*k). The space complexity is O(k) to store the current window.

### Solution Code
```python
import bisect

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        window = []
        ans = []
        for i, x in enumerate(nums):
            bisect.insort(window, x)
            if i >= k:
                window.remove(nums[i - k + 1])
            if len(window) == k:
                if k % 2 == 1:
                    ans.append(window[k//2])
                else:
                    ans.append((window[k//2 - 1] + window[k//2]) / 2)
        return ans
```

### Explanation
1. **Initialization**: We initialize an empty list `window` to keep track of the current window elements and an empty list `ans` to store the median results.
2. **Iterating through the list**: For each element in the input list `nums`, we insert it into the sorted `window` using binary search (O(k) time per insertion).
3. **Maintaining window size**: If the current index `i` is at least `k`, we remove the element that is no longer in the window (the element at position `i - k + 1` in the original list). This ensures the window always contains the most recent `k` elements.
4. **Computing the median**: After each insertion and removal, if the window size is `k`, we check if `k` is odd or even. For an odd `k`, the median is the middle element in the sorted window. For an even `k`, the median is the average of the two middle elements.
5. **Output**: The results are collected in the `ans` list and returned after processing all elements.
