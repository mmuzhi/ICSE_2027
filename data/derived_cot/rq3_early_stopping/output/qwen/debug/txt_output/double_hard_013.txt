To solve this problem, we need to reverse every k nodes in a linked list. The approach involves traversing the linked list in groups of k nodes, reversing each group, and then moving to the next group. The solution requires careful handling of pointers to ensure that the reversed segments are correctly attached to the previous and next segments of the list.

### Approach
1. **Dummy Node**: We start by creating a dummy node to simplify the reversal process, especially at the beginning of the list.
2. **Group Identification**: For each group of k nodes, we identify the start and end of the group. The start is the first node of the current group, and the end is the k-th node from the start.
3. **Reversal**: We reverse the identified group of k nodes by iterating through each node, updating the pointers to reverse the links.
4. **Pointer Adjustment**: After reversing a group, we adjust the pointers of the previous group and the start of the next group to maintain the correct sequence.
5. **Termination**: The loop terminates when there are fewer than k nodes remaining in the list to form a complete group.

### Solution Code
```python
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        currGroup = dummy

        while True:
            first_node = currGroup.next
            if not first_node:
                break

            kth = self.getKth(first_node, k-1)
            if kth is None:
                break
            nextGroup = kth.next

            prev = None
            curr = first_node
            for _ in range(k):
                nxt = curr.next
                curr.next = prev
                prev = curr
                curr = nxt

            currGroup.next = prev
            currGroup = first_node

        return dummy.next

    def getKth(self, node, k):
        while node and k:
            node = node.next
            k -= 1
        return node
```

### Explanation
1. **Dummy Node**: The dummy node acts as a placeholder at the beginning of the list to simplify the reversal process. This helps in handling the head of the list uniformly.
2. **Group Identification**: The `getKth` function is used to find the k-th node from a given starting node. This helps in identifying the end of the current group of k nodes.
3. **Reversal**: The reversal of the group is done by iterating through each node in the group, updating the `next` pointers to reverse the links. This is done in-place without using extra space.
4. **Pointer Adjustment**: After reversing a group, the previous group's `next` pointer is updated to point to the new head of the reversed group. The current group pointer is then moved to the end of the reversed group to start processing the next group.
5. **Termination**: The loop breaks when there are fewer than k nodes left to form a complete group, ensuring that partial groups are left unchanged.
