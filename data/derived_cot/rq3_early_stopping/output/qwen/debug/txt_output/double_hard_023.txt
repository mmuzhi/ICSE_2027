=== RAW ANSWER ===

To solve this problem, we need to find the number of ways to place N queens on an N×N chessboard such that no two queens attack each other. The solution involves using a backtracking approach to explore all possible configurations of queens on the board while ensuring that no two queens share the same row, column, or diagonal.

### Approach
1. **Backtracking with Pruning**: The algorithm explores all possible positions to place queens one by one. For each position, it checks if placing a queen there would result in any conflict (i.e., another queen in the same row, column, or diagonal).
2. **Tracking Used Columns and Diagonals**: Three arrays are used to keep track of:
   - **Columns**: A boolean array where each index represents a column. If a column is occupied, the corresponding index is marked as used.
   - **Upper Diagonals**: A boolean array where each index represents a diagonal from top-right to bottom-left. The index is calculated as `n-1 + col - row`.
   - **Lower Diagonals**: A boolean array where each index represents a diagonal from bottom-right to top-left. The index is calculated as `row + col`.
3. **Recursive Placement**: The algorithm recursively tries to place queens in subsequent columns, backtracking whenever a placement leads to a conflict. Once all queens are placed successfully, the configuration is recorded.

### Solution Code
```python
class Solution:
    def totalNQueens(self, n: int) -> int:
        def addans(board, ans):
            temp = []
            for row in board:
                for j in range(len(row)):
                    if row[j] == "Q":
                        temp.append(j + 1)
            ans.append(temp)
        
        def solve(col, board, col_used, upper_used, lower_used, ans, n):
            if col == n:
                addans(board, ans)
                return
            for row in range(n):
                if col_used[col] == 0 and upper_used[n - 1 + col - row] == 0 and lower_used[row + col] == 0:
                    board[row][col] = "Q"
                    col_used[col] = 1
                    upper_used[n - 1 + col - row] = 1
                    lower_used[row + col] = 1
                    solve(col + 1, board, col_used, upper_used, lower_used, ans, n)
                    col_used[col] = 0
                    upper_used[n - 1 + col - row] = 0
                    lower_used[row + col] = 0
        
        ans = []
        board = [[0] * n for _ in range(n)]
        col_used = [0] * n
        upper_used = [0] * (2 * n - 1)
        lower_used = [0] * (2 * n - 1)
        solve(0, board, col_used, upper_used, lower_used, ans, n)
        return len(ans)
```

### Explanation
1. **addans Function**: This helper function converts the current board configuration into a list of integers representing the column positions of each queen. It appends this configuration to the results list.
2. **solve Function**: This recursive function attempts to place a queen in the current column (`col`). For each row in the column, it checks if the position is safe (no conflicts). If safe, it places the queen and marks the corresponding column and diagonals as used. It then proceeds to the next column. If all queens are placed (`col == n`), it records the configuration.
3. **Initialization**: The board is initialized to all zeros. Arrays `col_used`, `upper_used`, and `lower_used` are initialized to keep track of used columns and diagonals.
4. **Backtracking**: After exploring all configurations for a given column, the function backtracks by unmarking the used column and diagonals to explore other possibilities.

This approach efficiently explores all valid configurations using backtracking and pruning, ensuring optimal performance for the given problem constraints.
