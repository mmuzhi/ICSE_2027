=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays. The median is the middle value in an ordered list of numbers. If the total number of elements is even, the median is the average of the two middle numbers. 

### Approach
1. **Merge the Arrays**: Combine the two sorted arrays into a single sorted array. This is done by comparing elements from both arrays and appending the smaller element to the result array, incrementing the respective array pointer each time.
2. **Find the Median**: Once the merged array is created, determine the median based on the length of this array. If the length is odd, the median is the middle element. If the length is even, the median is the average of the two middle elements.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        new = []
        p1 = 0
        p2 = 0
        
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
        
        if len(new) % 2 == 0:
            index = len(new) // 2
            median = (new[index] + new[index-1]) / 2
        else:
            median = float(new[len(new)//2])
        return median
```

### Explanation
1. **Merging Arrays**: The code initializes two pointers, `p1` and `p2`, to traverse `nums1` and `nums2` respectively. It compares elements from both arrays and appends the smaller element to the merged array `new`, then moves the respective pointer forward.
2. **Appending Remaining Elements**: After one of the arrays is exhausted, the code appends the remaining elements of the other array to `new`.
3. **Calculating Median**: The median is determined by checking the length of `new`. If the length is even, the median is the average of the two middle elements. If the length is odd, the median is the middle element. The result is returned as a float to handle both integer and floating-point results appropriately.

This approach efficiently merges the two sorted arrays and computes the median in linear time, O(n + m), where n and m are the lengths of the two arrays, making it optimal for large input sizes.
