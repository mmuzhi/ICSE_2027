To solve this problem, we need to find the longest word from a given list of candidates that can be formed by deleting some characters from a given string without changing the order of the remaining characters. This is essentially checking if each candidate word is a subsequence of the given string.

### Approach
1. **Problem Analysis**: The task is to determine the longest word in the list `d` that is a subsequence of the string `s`. If multiple words of the same maximum length exist, the lexicographically smallest one should be chosen.
2. **Intuition**: For each candidate word in the list, we need to verify if it appears in the string `s` in the same order, though not necessarily consecutively. This is the definition of a subsequence.
3. **Algorithm Selection**: The solution involves iterating over each candidate word and checking if it is a subsequence of `s` using a two-pointer technique. One pointer traverses the string `s`, and the other traverses the candidate word. If all characters of the candidate are found in order within `s`, it is a valid subsequence.
4. **Optimization**: The algorithm efficiently checks each candidate in linear time relative to the length of `s` and the candidate word, making it suitable for the problem constraints.

### Solution Code
```python
class Solution:
    def findLongestWord(self, s: str, d: List[str]) -> str:
        res = ''
        for cand in d:
            if self.find(s, cand) and (len(cand) < len(res) or (len(cand) == len(res) and cand < res)):
                res = cand
        return res
    
    def find(self, s, d):
        i, j = 0, 0
        while i < len(s) and j < len(d):
            if s[i] == d[j]:
                j += 1
            i += 1
        return j == len(d)
```

### Explanation
1. **Initialization**: The `findLongestWord` method initializes the result `res` as an empty string.
2. **Iteration and Check**: For each candidate word in the list `d`, it checks if the word is a subsequence of `s` using the helper method `find`. If it is, and if the candidate is either longer than the current result or of the same length but lexicographically smaller, the result is updated.
3. **Subsequence Check**: The `find` method uses two pointers to traverse `s` and the candidate word. The pointer for `s` moves forward in each iteration, while the pointer for the candidate word advances only when a matching character is found. If the entire candidate word is matched, the method returns `True`, indicating it is a subsequence.
4. **Result Compilation**: After processing all candidates, the method returns the longest valid subsequence, or an empty string if no valid subsequence exists.

This approach efficiently checks each candidate word in linear time, ensuring optimal performance for the given problem constraints.
