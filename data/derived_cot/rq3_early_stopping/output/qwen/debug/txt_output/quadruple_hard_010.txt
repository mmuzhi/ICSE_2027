To solve this problem, we need to compute the median of each sliding window of size `k` in an array of integers `nums`. The median is the middle value in an ordered list of numbers. If the window size `k` is odd, the median is the middle element. If `k` is even, the median is the average of the two middle elements.

### Approach
1. **Balanced BST Maintenance**: We use a balanced binary search tree (BST) to maintain the sliding window elements. The BST supports efficient insertion, deletion, and querying of the k-th smallest element.
2. **Sliding Window Processing**: For each element in the array:
   - **Insertion**: Insert the current element into the BST.
   - **Removal**: If the BST size exceeds `k`, remove the element that is no longer in the current window (i.e., the element that was `k` positions before the current element).
   - **Median Calculation**: Once the BST size equals `k`, compute the median. For odd `k`, the median is the middle element. For even `k`, the median is the average of the two middle elements.

### Solution Code
```python
class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        tree = None
        ans = []
        for i, x in enumerate(nums):
            tree = insert(tree, x)
            if size(tree) > k:
                tree = remove(tree, nums[i - k + 1])

                else:

        return ans
```

### Explanation
1. **Insertion and Removal**: The BST dynamically maintains the elements of the current window. Insertion adds the current element, and removal removes the element that slides out of the window as the window moves.
2. **Median Calculation**: After each insertion and potential removal, if the window size is `k`, the median is calculated. For odd `k`, the median is the element at position `k//2 + 1` in the BST. For even `k`, it's the average of elements at positions `k//2` and `k//2 + 1`.
3. **Efficiency**: The BST operations (insertion, deletion, and querying the k-th element) are O(log n) on average, making the overall complexity O(n log n) for processing all elements, which is efficient for large input sizes.

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        tree = None
        ans = []
        for i, x in enumerate(nums):
            tree = insert(tree, x)
            if size(tree) > k:
                tree = remove(tree, nums[i - k + 1])
            if size(tree) == k:
                if k % 2 == 1:
                    ans.append(get(tree, k // 2 + 1))
                else:
                    ans.append((get(tree, k // 2) + get(tree, k // 2 + 1)) / 2)
        return ans
