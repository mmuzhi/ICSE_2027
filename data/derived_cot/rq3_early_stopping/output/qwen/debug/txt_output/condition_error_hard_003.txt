To solve this problem, we need to find the minimum number of moves required to transform the given initial board configuration into the target configuration [[1,2,3],[4,5,0]] using a sliding puzzle mechanism. The blank space (0) can be moved to adjacent tiles (up, down, left, right), and each move counts as one step.

### Approach
1. **Breadth-First Search (BFS)**: We use BFS to explore all possible states of the board level by level. This ensures that the first time we reach the target state, it is with the minimum number of moves.
2. **State Representation**: Each state of the board is represented as a 2x3 grid. To efficiently check visited states, we convert each board state into a tuple of tuples and store these tuples in a set.
3. **Neighbor Generation**: For each board state, we find the position of the blank space (0) and generate all possible neighboring states by moving the blank space to adjacent positions (up, down, left, right). Each neighbor is a new board state created by swapping the blank space with the adjacent tile.
4. **Termination Condition**: The BFS terminates when the target state [[1,2,3],[4,5,0]] is encountered, returning the number of moves taken to reach it. If the target state is unreachable (though theoretically all states are solvable for a 2x3 puzzle), we return -1.

### Solution Code
```python
from collections import deque
from typing import List

class Solution:
    def slidingPuzzle(self, board: List[List[int]]) -> int:
        def get_neighbors(board):
            neighbors = []
            r, c = 0, 0
            for i in range(2):
                for j in range(3):
                    if board[i][j] == 0:
                        r, c = i, j
            for di, dj in [(0, 1), (1, 0), (0, -1), (-1, 0)]:
                ni, nj = r + di, c + dj
                if 0 <= ni < 2 and 0 <= nj < 3:
                    new_board = [row[:] for row in board]
                    new_board[r][c], new_board[ni][nj] = new_board[ni][nj], new_board[r][c]
                    neighbors.append(new_board)
            return neighbors

        queue = deque()
        queue.append((board, 0))
        seen = set()
        seen.add(tuple(tuple(row) for row in board))

        while queue:
            current, moves = queue.popleft()
            if current == [[1,2,3],[4,5,0]]:
                return moves
            for neighbor in get_neighbors(current):
                if tuple(tuple(row) for row in neighbor) not in seen:
                    seen.add(tuple(tuple(row) for row in neighbor))
                    queue.append((neighbor, moves + 1))
        return -1
```

### Explanation
1. **BFS Initialization**: The BFS starts with the initial board state and zero moves. A queue is used to manage the states to be explored, and a set keeps track of visited states to avoid cycles.
2. **Neighbor Generation**: For each state dequeued, the position of the blank space (0) is found. For each adjacent cell (up, down, left, right), a new state is created by swapping the blank space with the adjacent tile. This new state is added to the queue if it hasn't been visited before.
3. **Termination Check**: Each time a state is dequeued, it is checked against the target state [[1,2,3],[4,5,0]]. If matched, the current move count is returned immediately, ensuring the minimum moves are found.
4. **Efficiency**: The BFS efficiently explores all possible states level by level, guaranteeing the shortest path (minimum moves) to the target state. The use of a set for visited states ensures each state is processed only once, optimizing the search.
