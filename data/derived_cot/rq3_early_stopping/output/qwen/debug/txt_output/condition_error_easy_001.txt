=== RAW ANSWER ===

To solve this problem, we need to find the third maximum number in a given list of integers. If there are fewer than three distinct numbers, we should return the maximum number. The solution involves removing duplicates, sorting the distinct numbers in descending order, and then selecting the third element if it exists.

### Approach
1. **Remove Duplicates**: Convert the input list into a set to eliminate duplicate values.
2. **Sort Descending**: Sort the distinct numbers in descending order to easily access the largest, second largest, and third largest numbers.
3. **Check Length**: If the sorted list has at least three elements, return the third element (which is the third largest). Otherwise, return the first element (the maximum) since there are fewer than three distinct numbers.

### Solution Code
```python
class Solution:
    def thirdMax(self, nums: List[int]) -> int:
        distinct_nums = sorted(set(nums), reverse=True)
        if len(distinct_nums) < 3:
            return distinct_nums[0]
        else:
            return distinct_nums[2]
```

### Explanation
1. **Remove Duplicates**: Using `set(nums)` ensures we only consider unique values in the list.
2. **Sort Descending**: Sorting the distinct numbers in descending order (`reverse=True`) allows us to directly access the largest numbers by their indices.
3. **Check Length**: By checking if the length of the sorted distinct numbers is at least 3, we determine whether a third maximum exists. If it does, we return the third element (`distinct_nums[2]`). If not, we return the largest number (`distinct_nums[0]`), as required by the problem statement.

This approach efficiently handles both cases where there are three or more distinct numbers and where there are fewer than three distinct numbers, ensuring correctness with minimal computational overhead.
