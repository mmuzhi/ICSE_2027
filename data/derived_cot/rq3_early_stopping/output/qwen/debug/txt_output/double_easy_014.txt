=== RAW ANSWER ===

To solve this problem, we need to find the complement of a given integer. The complement is obtained by flipping all the bits in the binary representation of the number. The solution involves converting the number into its binary string representation, flipping each bit, and then converting the resulting string back into an integer.

### Approach
1. **Binary Conversion**: The given number is processed bit by bit from the least significant bit (LSB) to the most significant bit (MSB).
2. **Bit Flipping**: For each bit in the binary representation, if the bit is 1, we append 0 to the complement string, and if the bit is 0, we append 1.
3. **String Reversal**: Since we process the bits from LSB to MSB, the resulting string is in reverse order. We reverse this string to get the correct binary representation of the complement.
4. **Conversion to Integer**: The reversed binary string is then converted back to an integer using base 2.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        com = ''
        while num > 0:
            if num % 2 == 1:
                com += '0'
            else:
                com += '1'
            num = num // 2
        return int(com[::-1], 2) if com else 0
```

### Explanation
1. **Binary Processing**: The loop continues as long as the number is greater than 0. In each iteration, the least significant bit of the number is checked. If it is 1, a '0' is added to the complement string; if it is 0, a '1' is added.
2. **Division**: The number is divided by 2 (using integer division) to shift the bits right, effectively removing the least significant bit processed in each iteration.
3. **String Reversal**: After processing all bits, the complement string is reversed because the bits were processed from LSB to MSB, resulting in the string being in reverse order.
4. **Conversion**: The reversed string is converted back to an integer using base 2. If the original number was 0, the complement string would be empty, so we handle this edge case by returning 0.

This approach efficiently computes the complement by leveraging bit manipulation and string operations, ensuring correctness for all non-negative integers.
