=== RAW ANSWER ===

To solve this problem, we need to find the minimum cost to reach the bottom-right corner of a grid from the top-left corner, where moving through obstacles (cells with value 0) requires paying a cost of 1 to remove them. The solution involves using Dijkstra's algorithm to explore the grid efficiently, prioritizing paths with the least cost.

### Approach
1. **Problem Analysis**: The grid consists of cells that are either walls (0) or flowers (1). Moving through a flower costs nothing, but moving through a wall requires paying 1 to remove it. The goal is to reach the bottom-right corner with the minimum total cost.
2. **Intuition**: Dijkstra's algorithm is suitable here because it efficiently finds the shortest path in a graph with non-negative edge weights. Each cell in the grid can be considered a node, and edges represent moves to adjacent cells. The cost to enter a cell is 0 if it's a flower and 1 if it's a wall (only the first time it's encountered).
3. **Algorithm Selection**: We use a priority queue (min-heap) to explore cells in order of increasing cost. For each cell, we check its neighbors and update their costs if a cheaper path is found. We maintain a `dp` array to store the minimum cost to reach each cell and a `payed` array to track which walls have been removed to avoid paying multiple times for the same wall.
4. **Complexity Analysis**: The algorithm runs in O(m*n log(m*n)) time due to the priority queue operations, where m and n are the grid dimensions. The space complexity is O(m*n) for storing the `dp` and `payed` arrays.

### Solution Code
```python
import heapq
from typing import List

class Solution:
    def minCost(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        dp = [[float('inf')] * n for _ in range(m)]
        payed = [[False] * n for _ in range(m)]
        heap = []
        
        if grid[0][0] == 0:
            dp[0][0] = 1
            payed[0][0] = True
            heapq.heappush(heap, (1, 0, 0))
        else:
            dp[0][0] = 0
            heapq.heappush(heap, (0, 0, 0))
        
        while heap:
            cost, x, y = heapq.heappop(heap)
            if x == m-1 and y == n-1:
                return cost
            if cost != dp[x][y]:
                continue
            for dx, dy in [(0, 1), (0, -1), (1, 0), (-1, 0)]:
                nx, ny = x + dx, y + dy
                if 0 <= nx < m and 0 <= ny < n:
                    if grid[nx][ny] == 0 and not payed[nx][ny]:
                        new_cost = cost + 1
                        payed[nx][ny] = True
                    else:
                        new_cost = cost
                    if new_cost < dp[nx][ny]:
                        dp[nx][ny] = new_cost
                        heapq.heappush(heap, (new_cost, nx, ny))
        return dp[m-1][n-1]
```

### Explanation
1. **Initialization**: We start by initializing the `dp` array with infinity to represent initially unreachable cells and the `payed` array to keep track of walls that have been removed.
2. **Priority Queue**: The priority queue starts with the initial cell (0, 0). If the starting cell is a wall, the initial cost is 1; otherwise, it's 0.
3. **Processing Cells**: For each cell processed, if it's the target cell (bottom-right corner), we return the current cost. Otherwise, we explore all four adjacent cells.
4. **Cost Calculation**: For each adjacent cell, if it's a wall and hasn't been removed, the cost increases by 1, and the wall is marked as removed. If it's a flower or already removed, the cost remains the same.
5. **Updating Costs**: If the calculated cost for a neighbor is lower than the previously recorded cost, we update the `dp` array and push the neighbor into the priority queue for further exploration.
6. **Termination**: The algorithm terminates when the target cell is reached, returning the minimum cost stored in `dp[m-1][n-1]`.

This approach ensures that we efficiently explore the grid using Dijkstra's algorithm, prioritizing paths with the least cost, and correctly handling the removal of walls only once.
