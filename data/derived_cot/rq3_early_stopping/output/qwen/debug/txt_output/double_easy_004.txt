To solve this problem, we need to find the longest consecutive sequence in an array where each number is exactly one more than the previous number. The solution involves using a frequency map to count occurrences of each number and then checking for consecutive pairs to determine the longest sequence.

### Approach
1. **Frequency Map Construction**: First, we construct a frequency map where each key is a unique number from the input array, and the value is the count of how many times that number appears in the array.
2. **Finding Consecutive Pairs**: For each number in the frequency map, we check if the next consecutive number (current number + 1) exists in the map. If it does, we calculate the sum of the frequencies of the current number and the next consecutive number.
3. **Tracking Maximum Length**: We keep track of the maximum sum encountered during the iteration, which represents the longest consecutive sequence of numbers differing by 1.

### Solution Code
```python
class Solution:
    def findLHS(self, nums: List[int]) -> int:
        count_map = {}
        for num in nums:
            count_map[num] = count_map.get(num, 0) + 1
        result = 0
        for num in count_map:
            if num + 1 in count_map:
                result = max(result, count_map[num] + count_map[num + 1])
        return result
```

### Explanation
1. **Frequency Map Construction**: The loop iterates through each number in the input array, updating the frequency map to keep track of how many times each number appears.
2. **Finding Consecutive Pairs**: For each number in the frequency map, the code checks if the next consecutive number (current number + 1) exists. If it does, the sum of the frequencies of the current number and the next consecutive number is computed.
3. **Tracking Maximum Length**: The maximum value encountered during these checks is stored in the `result` variable, which is returned as the length of the longest consecutive sequence of numbers differing by 1.

This approach efficiently leverages the frequency map to avoid redundant checks and ensures optimal performance with a time complexity of O(n), where n is the number of elements in the input array.
