=== PROMPT ===
You will be given a competitive programming problem.
You have an empty bag.
You are given Q queries, which must be processed in order.
There are three types of queries.

- 1 x : Put one ball with the integer x written on it into the bag.
- 2 x : Remove one ball with the integer x written on it from the bag and discard it. It is guaranteed that the bag has a ball with the integer x written on it when this query is given.
- 3 : Print the number of different integers written on the balls in the bag.

Input

The input is given from Standard Input in the following format:
Q
\text{query}_1
\text{query}_2
\vdots
\text{query}_Q

The i-th query \text{query}_i is given in one of the following three formats:
1 x

2 x

3

Output

If there are K queries of the third type, print K lines.
The i-th line (1 \leq i \leq K) should contain the answer to the i-th query of the third type.

Constraints


- 1 \leq Q \leq 2 \times 10^{5}
- 1 \leq x \leq 10^{6}
- When a query of the second type is given, the bag has a ball with the integer x written on it.
- There is at least one query of the third type.
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to efficiently process a series of queries on a bag of balls, where each query either adds a ball, removes a ball, or asks for the number of distinct integers currently in the bag. The challenge lies in handling these operations in constant time per query to ensure the solution is efficient even for large input sizes.

### Approach
1. **Problem Analysis**: The problem involves three types of queries: adding a ball with a specific integer, removing a ball with a specific integer, and querying the count of distinct integers in the bag. The key observation is that we only need to track the presence (or absence) of each integer in the bag, not their counts, since the query asks for distinct integers.
2. **Intuition**: Using a set to keep track of distinct integers allows us to efficiently add and remove elements (O(1) average time complexity). However, we also need to handle multiple occurrences of the same integer. To manage this, we use a dictionary to count occurrences of each integer. When an integer's count drops to zero, we remove it from the set; when it increases from zero to one, we add it to the set.
3. **Algorithm Selection**: 
   - **Dictionary (counts)**: Tracks the number of occurrences of each integer in the bag.
   - **Set (distinct)**: Maintains the set of integers currently present in the bag (count > 0).
   - For each query:
     - **Add (1 x)**: Increment the count of `x` in the dictionary. If the count becomes 1, add `x` to the set.
     - **Remove (2 x)**: Decrement the count of `x` in the dictionary. If the count drops to zero, remove `x` from the set.
     - **Query (3)**: Output the size of the set, which gives the number of distinct integers.
4. **Complexity Analysis**: Each operation (add, remove, query) is O(1) on average due to the use of a dictionary and a set. With Q queries, the total time complexity is O(Q), which is optimal for the given constraints (Q ≤ 2×10^5).

### Solution Code
```python
import sys

def main():
    q = int(sys.stdin.readline())
    counts = {}
    distinct = set()
    for _ in range(q):
        data_line = sys.stdin.readline().split()
        if not data_line:
            continue
        t = int(data_line[0])
        if t == 1:
            x = int(data_line[1])
            counts[x] = counts.get(x, 0) + 1
            if counts[x] == 1:
                distinct.add(x)
        elif t == 2:
            x = int(data_line[1])
            counts[x] = counts.get(x, 0) - 1
            if counts[x] == 0:
                distinct.remove(x)
        else:
            print(len(distinct))

if __name__ == '__main__':
    main()
```

### Explanation
- **Initialization**: The program starts by reading the number of queries, `q`. It initializes a dictionary `counts` to keep track of how many times each integer appears in the bag and a set `distinct` to store integers currently present (count > 0).
- **Processing Queries**: For each query:
  - **Add (1 x)**: The integer `x` is added to the dictionary. If its count transitions from 0 to 1, it is added to the set.
  - **Remove (2 x)**: The integer `x` is removed from the dictionary. If its count drops to zero, it is removed from the set.
  - **Query (3)**: The size of the set `distinct` is printed, representing the number of distinct integers in the bag.
- **Efficiency**: The use of a dictionary and set ensures that each operation (addition, removal, query) is handled in constant time on average, making the solution efficient for large input sizes. The program reads input line by line for optimal performance.
