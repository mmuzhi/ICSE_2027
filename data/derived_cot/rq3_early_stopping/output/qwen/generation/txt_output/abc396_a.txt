=== ANSWER ===

To solve this problem, we need to determine if there exists any sequence of three or more identical elements consecutively in a given integer sequence. The solution involves checking each possible triplet of consecutive elements in the sequence to see if all three elements are the same.

### Approach
1. **Problem Analysis**: The problem requires checking for any three consecutive identical elements in the sequence. The sequence length is at least 3, so we can directly iterate through the sequence starting from the first element up to the third last element.
2. **Intuition**: For each element at position `i`, we check if it is equal to the next two elements (i.e., `A[i] == A[i+1] == A[i+2]`). If such a triplet is found, we immediately return "Yes". If no such triplet is found after checking all possible triplets, we return "No".
3. **Algorithm Selection**: The straightforward approach is to use a loop that runs from the first element to the third last element, checking each triplet of consecutive elements. This approach is efficient with a time complexity of O(N), where N is the length of the sequence, as each element is checked exactly once.
4. **Optimization**: The solution is already optimal for the given constraints (N ≤ 100) and does not require any further optimization.

### Solution Code
```python
def main():
    n = int(input().strip())
    arr = list(map(int, input().split()))
    
    for i in range(n - 2):
        if arr[i] == arr[i+1] == arr[i+2]:
            print("Yes")
            return
    print("No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The first line of input is read as an integer `n`, representing the length of the sequence. The second line is read as a list of integers.
2. **Checking Triplets**: The loop runs from the first element to the third last element (i.e., `range(n - 2)`). For each element at position `i`, it checks if the elements at positions `i`, `i+1`, and `i+2` are all the same.
3. **Early Termination**: If a triplet of identical elements is found, the program prints "Yes" and exits immediately. If no such triplet is found after checking all possible triplets, it prints "No".

This approach efficiently checks for the presence of three consecutive identical elements in linear time, making it suitable for the given problem constraints.
