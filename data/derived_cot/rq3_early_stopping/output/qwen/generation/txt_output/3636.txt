=== ANSWER ===

To solve this problem, we need to determine if a given string of digits is balanced. A string is considered balanced if the sum of the digits at even indices (0-indexed) is equal to the sum of the digits at odd indices. 

### Approach
1. **Problem Analysis**: The task involves checking the sums of digits at even and odd positions in the string. The solution requires iterating through the string once, calculating the sums for even and odd indices separately, and then comparing these sums.
2. **Intuition**: By traversing the string, we can easily separate digits into two groups based on their indices (even or odd). The sums of these groups are then compared to determine if the string is balanced.
3. **Algorithm Selection**: The algorithm involves a single pass through the string. For each character at an even index, add its digit value to the even sum; for each character at an odd index, add its digit value to the odd sum. After processing all characters, compare the two sums.
4. **Complexity Analysis**: The algorithm runs in O(n) time, where n is the length of the string, since each character is processed exactly once. The space complexity is O(1) as only two additional variables (for the sums) are used.

### Solution Code
```python
class Solution:
    def isBalanced(self, num: str) -> bool:
        even_sum = 0
        odd_sum = 0
        
        for i, char in enumerate(num):
            if i % 2 == 0:
                even_sum += int(char)
            else:
                odd_sum += int(char)
                
        return even_sum == odd_sum
```

### Explanation
1. **Initialization**: Two variables, `even_sum` and `odd_sum`, are initialized to zero. These will hold the cumulative sums of digits at even and odd indices, respectively.
2. **Iteration**: The loop processes each character in the string using `enumerate` to get both the index and the character. For each character at an even index (i.e., index 0, 2, 4, ...), its digit value is added to `even_sum`. For characters at odd indices (i.e., index 1, 3, 5, ...), their digit values are added to `odd_sum`.
3. **Comparison**: After processing all characters, the function checks if `even_sum` equals `odd_sum`. If they are equal, the function returns `True`; otherwise, it returns `False`.

This approach efficiently checks the balance condition by leveraging a single pass through the string, ensuring optimal performance and correctness.
