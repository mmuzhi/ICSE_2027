To solve this problem, we need to format a given real number such that it meets specific conditions regarding its decimal representation. The conditions are:
1. The decimal part must not have trailing zeros.
2. There must not be an unnecessary trailing decimal point.

### Approach
1. **Input Handling**: Read the input string which represents the real number.
2. **Check for Decimal Point**: If the input does not contain a decimal point, output the number as is.
3. **Split Integer and Fractional Parts**: If a decimal point is present, split the string into the integer part and the fractional part.
4. **Handle Empty Integer Part**: If the integer part is empty (which can happen if the input starts with a decimal point), set it to '0'.
5. **Remove Trailing Zeros**: Strip any trailing zeros from the fractional part.
6. **Check for Empty Fractional Part**: If the fractional part becomes empty after removing trailing zeros, output only the integer part. Otherwise, output the integer part followed by a decimal point and the stripped fractional part.

### Solution Code
```python
s = input().strip()

if '.' not in s:
    print(s)
else:
    parts = s.split('.')
    int_part = parts[0]
    if int_part == '':
        int_part = '0'
    frac_part = parts[1].rstrip('0')
    if frac_part == '':
        print(int_part)
    else:
        print(int_part + '.' + frac_part)
```

### Explanation
1. **Input Handling**: The input is read as a string and stripped of any leading or trailing whitespace.
2. **Decimal Check**: If the input string does not contain a decimal point, it is printed directly since no formatting is needed.
3. **Splitting Parts**: The string is split into two parts at the decimal point. The part before the decimal is the integer part, and the part after is the fractional part.
4. **Empty Integer Part**: If the integer part is empty (e.g., input is ".001"), it is set to '0' to handle cases where the number is less than 1.
5. **Trailing Zeros Removal**: The fractional part is processed to remove any trailing zeros. For example, "340" becomes "34".
6. **Output Decision**: If the fractional part becomes empty after removing trailing zeros (e.g., "000" becomes ""), only the integer part is printed. Otherwise, the integer part and the stripped fractional part are combined with a decimal point for output.

This approach ensures that the output adheres to the specified conditions by efficiently handling both the integer and fractional parts of the input number.
