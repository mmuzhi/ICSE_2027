You will be given a competitive programming problem.
You are given an N \times N grid S and an M \times M grid T. The cell at the i-th row from the top and the j-th column from the left is denoted by (i,j).
The colors of the cells in S and T are represented by N^2 characters S_{i,j} (1\leq i,j\leq N) and M^2 characters T_{i,j} (1\leq i,j\leq M), respectively. In grid S, cell (i,j) is white if S_{i,j} is ., and black if S_{i,j} is #. The same applies for grid T.
Find T within S. More precisely, output integers a and b (1 \leq a,b \leq N-M+1) that satisfy the following condition:

- S_{a+i-1,b+j-1} = T_{i,j} for every i,j (1\leq i,j \leq M).

Input

The input is given from Standard Input in the following format:
N M
S_{1,1}S_{1,2}\dots S_{1,N}

T_{1,1}T_{1,2}\dots T_{1,M}
T_{2,1}T_{2,2}\dots T_{2,M}
\vdots
T_{M,1}T_{M,2}\dots T_{M,M}

Output

Print a and b in this order, separated by a space on one line.

Constraints


- 1 \leq M \leq N \leq 50
- N and M are integers.
- Each of S_{i,j} and T_{i,j} is . or #.
- There is exactly one pair (a,b) satisfying the condition.

=== ANSWER ===

To solve this problem, we need to find a subgrid within a larger grid that exactly matches a given smaller grid. The solution involves checking all possible positions in the larger grid where the smaller grid could fit and verifying if the subgrid at each position matches the given grid.

### Approach
1. **Problem Analysis**: The problem requires us to search for a given MxM grid (T) within an NxN grid (S). The solution must output the top-left coordinates (a, b) of the subgrid in S that matches T exactly. The constraints ensure there is exactly one such pair (a, b).

2. **Intuition**: The solution involves iterating over all possible starting positions (i, j) in S where a subgrid of size MxM can fit. For each starting position, we compare each cell of the subgrid with the corresponding cell in T. If all cells match, we return the 1-indexed coordinates (i+1, j+1).

3. **Algorithm Selection**: The algorithm checks each possible top-left corner of the subgrid in S. For each corner, it verifies the entire MxM subgrid against T. The search space is limited to positions where the subgrid does not exceed the bounds of S, i.e., i ranges from 0 to N-M and j ranges from 0 to N-M (0-indexed).

4. **Complexity Analysis**: The algorithm has a time complexity of O((N-M+1)^2 * M^2), which is feasible given the constraints (N and M are at most 50). The space complexity is O(N^2 + M^2) to store the grids S and T.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    n, m = map(int, data[0].split())
    s = []
    for i in range(1, 1 + n):
        s.append(data[i].strip())
    t = []
    for i in range(1 + n, 1 + n + m):
        t.append(data[i].strip())
    
    for i in range(n - m + 1):
        for j in range(n - m + 1):
            match = True
            for k in range(m):
                for l in range(m):
                    if s[i + k][j + l] != t[k][l]:
                        match = False
                        break
                if not match:
                    break
            if match:
                print(f"{i + 1} {j + 1}")
                return

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input is read from standard input. The first line contains the grid sizes N and M. The next N lines define the grid S, and the following M lines define the grid T.

2. **Grid Storage**: The grids S and T are stored as lists of strings for easy access to individual cells.

3. **Subgrid Matching**: The algorithm iterates over each possible starting position (i, j) in S where a subgrid of size MxM can be placed. For each position, it checks if the subgrid matches T by comparing each corresponding cell.

4. **Output**: Once a matching subgrid is found, the algorithm prints the 1-indexed coordinates (i+1, j+1) and exits, as the problem guarantees exactly one solution.

This approach efficiently checks all possible positions for the subgrid match, ensuring correctness while adhering to the problem constraints.
