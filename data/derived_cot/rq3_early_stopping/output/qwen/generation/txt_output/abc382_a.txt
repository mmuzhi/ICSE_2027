You will be given a competitive programming problem.
There are N boxes arranged in a row, and some of these boxes contain cookies.
The state of these boxes is represented by a string S of length N.
Specifically, the i-th box (1\leq i \leq N) from the left contains one cookie if the i-th character of S is @, and is empty if it is ..
Over the next D days, Takahashi will choose and eat one cookie per day from among the cookies in these boxes.
Determine how many of the N boxes will be empty after D days have passed.
(It can be proved that this value does not depend on which cookies Takahashi chooses each day.)
It is guaranteed that S contains at least D occurrences of @.

Input

The input is given from Standard Input in the following format:
N D
S

Output

Print the number of boxes that will be empty after D days have passed among the N boxes.

Constraints


- 1 \leq D \leq N \leq 100
- N and D are integers.
- S is a string of length N consisting of @ and ..
- S contains at least D occurrences of @.

Sample Input 1

5 2
.@@.@

Sample Output 1

4

For example, Takahashi might act as follows:

- Day 1: There are cookies in the 2nd, 3rd, and 5th boxes from the left. He chooses the cookie in the 2nd box to eat.
- Day 2: There are cookies in the 3rd and 5th boxes. He chooses the cookie in the 5th box to eat.
- After two days have passed, only the 3rd box from the left contains a cookie. Therefore, four out of the five boxes are empty.

Even though Takahashi might choose differently on each day than in this example, there will still be four empty boxes after two days.
Therefore, the answer is 4.

Sample Input 2

3 3
@@@

Sample Output 2

3

Sample Input 3

10 4
@@@.@@.@@.

Sample Output 3

7

def main():
    import sys
    data = sys.stdin.read().splitlines()
    n, d = map(int, data[0].split())
    s = data[1].strip()
    m = s.count('@')
    empty_boxes = (n - m) + d
    print(empty_boxes)

if __name__ == "__main__":
    main()

To solve this problem, we need to determine the number of boxes that will be empty after Takahashi eats one cookie per day for D days. The key observation here is that the final state of the boxes does not depend on the order in which Takahashi eats the cookies. 

### Approach
1. **Problem Analysis**: The problem involves N boxes arranged in a row, some of which contain cookies (represented by '@') and others are empty (represented by '.'). Takahashi eats exactly D cookies over D days. The goal is to find how many boxes will be empty after these D days.
2. **Insight**: The total number of empty boxes after D days consists of two parts:
   - **Originally Empty Boxes**: These are the boxes that were empty from the start (i.e., '.'). There are (N - M) such boxes, where M is the total number of cookies initially present.
   - **Boxes with Eaten Cookies**: These are the boxes that originally had a cookie but had their cookie eaten. Since Takahashi eats D cookies, there are D such boxes.
3. **Algorithm Selection**: The solution involves counting the initial number of cookies (M) in the string S. The number of empty boxes is then calculated as (N - M) + D.
4. **Complexity Analysis**: The algorithm involves a single pass through the string to count the '@' characters, which is O(N) time complexity, where N is the length of the string. This is efficient given the constraints (N ≤ 100).

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    n, d = map(int, data[0].split())
    s = data[1].strip()
    m = s.count('@')
    empty_boxes = (n - m) + d
    print(empty_boxes)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the number of boxes (N), the number of days (D), and the string (S) representing the initial state of the boxes.
2. **Counting Cookies**: The variable `m` is the count of '@' characters in the string S, representing the initial number of cookies.
3. **Calculating Empty Boxes**: The total empty boxes after D days is computed as the sum of originally empty boxes (N - M) and the boxes from which cookies were eaten (D).
4. **Output Result**: The result is printed, which is the total number of empty boxes after D days.

This approach efficiently leverages the problem's constraints and guarantees to provide the correct result by focusing on the initial state and the number of cookies eaten.
