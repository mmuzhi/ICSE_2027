You will be given a competitive programming problem.
Takahashi decided to hold a programming contest.
The contest consists of five problems: A, B, C, D, E, with scores a, b, c, d, e, respectively.
There are 31 participants, and all of them solved at least one problem.
More specifically, for every non-empty subsequence (not necessarily contiguous) of the string ABCDE, there is a participant named after that subsequence who solved the problems corresponding to the letters in their name and did not solve the other problems.
For example, participant A solved only problem A, and participant BCE solved problems B, C, and E.
Print the names of the participants in order of their obtained scores, from the largest to the smallest.
The score obtained by a participant is the sum of the scores of the problems they solved.
If two participants obtained the same score, print the one whose name is lexicographically smaller first.
What does "lexicographically smaller" mean?
In short, "lexicographically smaller" refers to the order in which words would appear in a dictionary.
More precisely, for distinct strings S,T consisting of uppercase English letters, S is lexicographically smaller than T if either of the following conditions holds:

- The length |S| of S is less than the length of T, and the first |S| characters of T match S.
- There exists an integer 1\leq i\leq\min\{ |S|,|T|\} that satisfy both of the following two conditions:
- For every integer j with 1\leq j\lt i, the j-th character of S equals the j-th character of T.
- The i-th character of S is alphabetically smaller than the i-th character of T.


For example, if S= AB and T= ABC, the first condition holds, so S is lexicographically smaller than T.
If S= ABD and T= ACD, the second condition holds for i=2, so S is lexicographically smaller than T.

Output

If two subsequences have the same score, they should be sorted lexicographically in ascending order.

### Approach
1. **Generate Subsequences**: For each bitmask from 1 to 31 (since there are 5 problems, resulting in 2^5 - 1 = 31 non-empty subsequences), we determine which problems are included by checking each bit in the bitmask. Each bit corresponds to a problem (A for bit 0, B for bit 1, etc.).
2. **Calculate Scores**: For each subsequence identified by the bitmask, compute the total score by summing the scores of the included problems.
3. **Sort Participants**: Sort the participants first by their total score in descending order. If two participants have the same score, sort them lexicographically in ascending order.
4. **Output Results**: Print the sorted list of participant names, each on a new line.

### Solution Code
```python
a, b, c, d, e = map(int, input().split())
scores = [a, b, c, d, e]
letters = "ABCDE"
participants = []

for bitmask in range(1, 32):
    name = ""
    total = 0
    for i in range(5):
        if bitmask & (1 << i):
            name += letters[i]
            total += scores[i]
    participants.append((total, name))

participants.sort(key=lambda x: (-x[0], x[1]))

for p in participants:
    print(p[1])
```

### Explanation
1. **Reading Input**: The input consists of five integers representing the scores for problems A, B, C, D, and E.
2. **Bitmask Generation**: For each bitmask from 1 to 31, we check each bit to determine which problems are included in the subsequence. For example, a bitmask of 1 (binary 00001) includes only problem A.
3. **Score Calculation**: For each included problem in the subsequence, the corresponding score is added to the total score for that subsequence.
4. **Sorting**: The participants are sorted primarily by their total score in descending order. If scores are equal, the subsequences are sorted lexicographically (e.g., "AB" comes before "AC").
5. **Output**: The sorted list of participant names is printed, each on a new line, ensuring the highest scores are listed first, with lexicographical order handling ties.

This approach efficiently generates all possible non-empty subsequences, calculates their scores, and sorts them according to the specified criteria, providing the desired output in a clear and concise manner.
