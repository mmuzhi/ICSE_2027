def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    parts = data[0].split()
    N = int(parts[0])
    D = int(parts[1])
    S = data[1].strip()
    
    # Extract indices of '@'
    indices = [i for i, char in enumerate(S) if char == '@']
    # Sort in descending order and take the first D
    indices.sort(reverse=True)
    remove_set = set(indices[:D])
    
    result_chars = []
    for i, char in enumerate(S):
        if char == '@' and i in remove_set:
            result_chars.append('.')
        else:
            result_chars.append(char)
            
    print(''.join(result_chars))

if __name__ == "__main__":
    main()

To solve this problem, we need to determine which cookies in a row of boxes remain after Takahashi eats one cookie per day for D days, always choosing the rightmost available cookie each day. The solution involves identifying the positions of the cookies and then determining which of these positions will be left after D days of removal.

### Approach
1. **Problem Analysis**: The key observation is that Takahashi always picks the rightmost cookie available each day. This means the cookies are removed in descending order of their indices. Therefore, the D rightmost cookies in the original string will be removed after D days.
2. **Intuition**: By collecting all the indices of the cookies in the string and sorting them in descending order, we can determine the D largest indices (rightmost cookies) that will be removed. The remaining cookies are those not in this set of indices.
3. **Algorithm Selection**: 
   - **Step 1**: Read the input values N (number of boxes) and D (number of days), and the string S representing the initial state of the boxes.
   - **Step 2**: Extract all indices where the character is '@' (indicating a cookie) and sort them in descending order.
   - **Step 3**: The first D indices from this sorted list are the positions of the cookies to be removed.
   - **Step 4**: Construct the result string by checking each box: if it originally had a cookie and its index is in the set of removed indices, mark it as '.'; otherwise, retain the original character ('@' or '.').
4. **Complexity Analysis**: The algorithm involves a linear scan of the string to collect indices (O(N)), sorting these indices (O(N log N) in the worst case), and another linear scan to construct the result (O(N)). Given the constraints (N ≤ 100), this approach is efficient.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    parts = data[0].split()
    N = int(parts[0])
    D = int(parts[1])
    S = data[1].strip()
    
    indices = [i for i, char in enumerate(S) if char == '@']
    indices.sort(reverse=True)
    remove_set = set(indices[:D])
    
    result_chars = []
    for i, char in enumerate(S):
        if char == '@' and i in remove_set:
            result_chars.append('.')
        else:
            result_chars.append(char)
            
    print(''.join(result_chars))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the number of boxes (N), the number of days (D), and the string (S) representing the initial state of the boxes.
2. **Collecting Indices**: The indices of all '@' characters in the string S are collected into a list. These indices represent the positions of cookies.
3. **Sorting Indices**: The collected indices are sorted in descending order to prioritize the rightmost cookies.
4. **Determining Removed Indices**: The first D indices from the sorted list are the positions of the cookies to be removed.
5. **Constructing Result**: For each box in the original string, if it originally contained a cookie and its index is in the set of removed indices, it is marked as '.'; otherwise, the original character is retained.
6. **Output**: The resulting string is printed, showing which cookies remain after D days.

This approach efficiently determines the state of each box after D days by leveraging the observation that cookies are removed from right to left, ensuring optimal performance even for the upper constraint limits.
