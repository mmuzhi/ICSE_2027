To solve this problem, we need to count the number of triples (i, j, k) in a given string S such that the characters at positions i, j, and k are 'A', 'B', and 'C' respectively, and the positions are equally spaced (i.e., j - i = k - j). 

### Approach
1. **Problem Analysis**: The problem requires finding all valid triples (i, j, k) where the characters at these positions are 'A', 'B', and 'C' in that order, and the positions are equally spaced. The key insight is that for each possible gap size `d`, we can check all starting positions `i` such that `i`, `i + d`, and `i + 2*d` are within the bounds of the string and the characters at these positions meet the required conditions.

2. **Algorithm Selection**: The solution involves iterating over all possible gap sizes `d` from 1 to half the length of the string. For each gap size `d`, we then iterate over all possible starting positions `i` such that `i + 2*d` does not exceed the string length. For each valid `i`, we check if the characters at positions `i`, `i + d`, and `i + 2*d` are 'A', 'B', and 'C' respectively. If they are, we increment our count.

3. **Complexity Analysis**: The algorithm has a time complexity of O(n^2) where `n` is the length of the string. This is because for each gap size `d` (which can be up to `n/2`), we check up to `n` starting positions. Given the constraint that the string length is at most 100, this approach is efficient and feasible.

### Solution Code
```python
S = input().strip()
n = len(S)
count = 0
for d in range(1, (n // 2) + 1):
    for i in range(0, n - 2*d + 1):
        if S[i] == 'A' and S[i+d] == 'B' and S[i+2*d] == 'C':
            count += 1
print(count)
```

### Explanation
1. **Input Handling**: The input string `S` is read and stripped of any leading or trailing whitespace.
2. **Initialization**: The length of the string `n` is determined, and a counter `count` is initialized to zero to keep track of valid triples.
3. **Gap Iteration**: The outer loop iterates over all possible gap sizes `d` from 1 to `n // 2`. This ensures that `i + 2*d` does not exceed the string length.
4. **Position Check**: For each gap size `d`, the inner loop checks all starting positions `i` from 0 to `n - 2*d`. For each `i`, it verifies if the characters at positions `i`, `i + d`, and `i + 2*d` are 'A', 'B', and 'C' respectively. If they are, the counter `count` is incremented.
5. **Output**: Finally, the total count of valid triples is printed.

This approach efficiently checks all possible triples by leveraging nested loops to explore every valid combination of starting positions and gap sizes, ensuring correctness while maintaining optimal performance for the given constraints.
