To solve this problem, we need to determine the total number of turns required to reduce all enemies' health to zero or less, given a specific attack pattern. The attack pattern alternates between dealing 1 damage per turn and dealing 3 damage every third turn. The challenge is to efficiently compute the number of turns without simulating each turn individually, especially since the health values can be very large (up to 10^9) and there can be up to 200,000 enemies.

### Approach
1. **Problem Analysis**: The key observation is that the damage dealt per turn follows a periodic pattern: 1, 1, 3, 1, 1, 3, etc. This pattern repeats every three turns, contributing a total of 5 damage per cycle (3 turns). For each enemy, we need to find the minimal number of turns required to reduce its health to zero or less, considering the cumulative effect of the periodic damage pattern starting from the current turn.

2. **Binary Search for Efficiency**: For each enemy, we use binary search to determine the minimal number of turns (k) required. The binary search checks if the cumulative damage from the current turn to the current turn plus k-1 turns meets or exceeds the enemy's health. The damage calculation involves counting the number of multiples of 3 in the range [current turn, current turn + k - 1], which is efficiently computed using integer division.

3. **Complexity Analysis**: The binary search for each enemy runs in O(log h) time, where h is the enemy's health. Given that there are up to 200,000 enemies and each health value can be up to 10^9, the total operations are around 200,000 * 30 (since log2(10^9) is approximately 30), which is feasible within time constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    H = list(map(int, data[1:1+n]))
    
    total_turns = 0
    for h in H:
        s = total_turns + 1
        low, high = 0, h
        while low < high:
            mid = (low + high) // 2
            damage = mid + 2 * ((s + mid - 1) // 3 - (s - 1) // 3)
            if damage >= h:
                high = mid
            else:
                low = mid + 1
        total_turns += low
        
    print(total_turns)

if __name__ == "__main__":
    main()
```

2. **Initialization**: `total_turns` keeps track of the cumulative turns taken to defeat all enemies.
3. **Binary Search for Each Enemy**: For each enemy, the binary search determines the minimal number of turns (k) needed to reduce its health to zero or less. The search space is between 0 and the enemy's health (h), as the worst-case scenario is dealing 1 damage per turn.
4. **Damage Calculation**: The damage for k turns is calculated by considering the periodic pattern. The formula `mid + 2 * ((s + mid - 1) // 3 - (s - 1) // 3)` computes the total damage by counting the number of multiples of 3 in the range [s, s + mid - 1] and adjusting the damage accordingly.
5. **Output**: After processing all enemies, the total number of turns is printed.
