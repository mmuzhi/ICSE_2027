=== ANSWER ===

To solve this problem, we need to calculate the minimal total traveled distance of a finger moving across a keyboard arranged in a specific permutation of the alphabet. The keyboard is represented by a string where each character's position is its index in the string (1-indexed). The finger starts at the position of 'A' and must press each subsequent letter from 'B' to 'Z' in order, moving directly to the next letter's position each time. The goal is to find the sum of the absolute distances between consecutive positions in this sequence.

### Approach
1. **Mapping Letters to Positions**: First, we create a dictionary where each letter from 'A' to 'Z' is mapped to its corresponding position in the given string. The position is determined by the 1-based index of the letter in the string.
2. **Sequence of Positions**: Using the mapping, we generate a list of positions corresponding to the sequence of letters from 'A' to 'Z'.
3. **Calculate Total Distance**: We then compute the total distance by summing the absolute differences between each consecutive pair of positions in the sequence. This gives the minimal total traveled distance as the finger moves from 'A' to 'Z'.

### Solution Code
```python
def main():
    S = input().strip()
    mapping = {}
    for idx, char in enumerate(S):
        mapping[char] = idx + 1
        
    letters = [chr(ord('A') + i) for i in range(26)]
    positions = [mapping[letter] for letter in letters]
    
    total_distance = 0
    for i in range(1, 26):
        total_distance += abs(positions[i] - positions[i-1])
        
    print(total_distance)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Mapping Creation**: The code starts by reading the input string and creating a dictionary where each character in the string is mapped to its 1-based index. This allows quick lookup of any letter's position.
2. **Position Sequence Generation**: The sequence of letters from 'A' to 'Z' is generated using ASCII values. For each letter in this sequence, the corresponding position from the dictionary is retrieved.
3. **Distance Calculation**: The total distance is computed by iterating through the sequence of positions and summing the absolute differences between each consecutive pair of positions. This sum represents the minimal distance the finger travels while pressing all the keys in order from 'A' to 'Z'.

This approach efficiently leverages dictionary lookups and arithmetic operations to solve the problem in linear time relative to the number of letters, making it optimal for the given constraints.
