The hash value of a character is the index of that character in the English alphabet (e.g., 'a' → 0, 'b' → 1, ..., 'z' → 25).

Return result.

The only substring: "mxz", 12 + 23 + 25 = 60, 60 % 26 = 8, result[0] = 'i'.

=== ANSWER ===

To solve this problem, we need to hash a given string into a new string by processing each substring of a specified length, calculating a hash value for each substring, and then constructing the result string from these hash values.

### Approach
1. **Problem Analysis**: The problem requires dividing the input string into substrings of length `k`, then for each substring, compute the sum of the ASCII values of its characters (where 'a' is 0, 'b' is 1, etc.), take the modulo 26 of this sum, and map the result back to a lowercase letter. The resulting letters form the output string.
2. **Intuition**: By breaking the string into equal parts of size `k`, we can process each part independently. For each part, summing the character positions (0-based) in the alphabet and then taking modulo 26 ensures the result stays within the bounds of a single lowercase letter.
3. **Algorithm Selection**: The algorithm involves iterating over the string in steps of `k`, processing each substring of length `k`, computing the required hash value, and appending the corresponding character to the result string.
4. **Complexity Analysis**: The algorithm processes each character in the string exactly once, leading to a time complexity of O(n), where n is the length of the string. The space complexity is O(n/k) for storing the result, which is optimal for the given problem constraints.

### Solution Code
```python
class Solution:
    def stringHash(self, s: str, k: int) -> str:
        n = len(s)
        result = []
        for i in range(0, n, k):
            total = 0
            for char in s[i:i+k]:
                total += ord(char) - ord('a')
            hashed_char = total % 26
            result.append(chr(hashed_char + ord('a')))
        return ''.join(result)
```

### Explanation
1. **Initialization**: The function starts by determining the length of the input string `s` and initializes an empty list `result` to store the hashed characters.
2. **Processing Substrings**: The outer loop iterates over the string in steps of `k`, processing each substring of length `k` starting from index `i`.
3. **Summing Character Values**: For each character in the current substring, its position in the alphabet (0 for 'a', 1 for 'b', etc.) is calculated by subtracting the ASCII value of 'a' from the character's ASCII value. These values are summed up for the entire substring.
4. **Modulo Operation and Character Mapping**: The sum is taken modulo 26 to ensure it falls within the range 0-25. This value is then converted back to a lowercase letter by adding the ASCII value of 'a' and converting to a character.
5. **Result Construction**: The resulting characters from each substring are joined into a single string, which is returned as the final hashed string.

This approach efficiently processes each substring, computes the required hash values, and constructs the result string in linear time, adhering to the problem constraints and requirements.
