=== ANSWER ===

To solve this problem, we need to determine the maximum number of containers that can be loaded onto an n x n cargo deck without exceeding the ship's weight capacity. Each container has a fixed weight, and the total weight of all containers must not surpass the given maximum weight capacity.

### Approach
1. **Problem Analysis**: The problem involves calculating the maximum number of containers that can be placed on an n x n deck, where each container weighs exactly `w`. The solution must consider both the physical constraints of the deck (it can hold at most `n * n` containers) and the weight constraint (the total weight must not exceed `maxWeight`).
2. **Key Insight**: The maximum number of containers is constrained by two factors:
   - The total number of cells on the deck, which is `n * n`.
   - The weight capacity divided by the weight per container, which is `maxWeight // w` (using integer division to ensure we do not exceed the weight limit).
3. **Algorithm Selection**: The solution involves computing both constraints and selecting the smaller value between them. This ensures that we do not load more containers than the deck can physically hold or more than the ship's weight capacity allows.

### Solution Code
```python
class Solution:
    def maxContainers(self, n: int, w: int, maxWeight: int) -> int:
        return min(n * n, maxWeight // w)
```

### Explanation
1. **Problem Analysis**: The deck has `n * n` cells, each capable of holding one container. The total weight of all containers must not exceed `maxWeight`.
2. **Key Insight**: The solution hinges on two simple calculations:
   - The maximum containers based on deck space: `n * n`.
   - The maximum containers based on weight capacity: `maxWeight // w`.
3. **Algorithm Selection**: By taking the minimum of these two values, we ensure that the solution adheres to both constraints. For example, if the deck can hold 4 containers but the weight capacity only allows 3, the answer is 3.
4. **Complexity**: The approach involves basic arithmetic operations (multiplication and division) which run in constant time, O(1), making it highly efficient even for the upper constraint limits.
