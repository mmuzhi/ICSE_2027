package org.example;

import java.util.HashMap;
import java.util.Map;

public class BookManagement {
    private Map<String, Integer> inventory;

    public BookManagement() {
        this.inventory = new HashMap<>();
    }

    public void addBook(String title, int quantity) {
        if (inventory.containsKey(title)) {
            inventory.put(title, inventory.get(title) + quantity);
        } else {
            inventory.put(title, quantity);
        }
    }

    public void removeBook(String title, int quantity) throws Exception {
        if (!inventory.containsKey(title) || inventory.get(title) < quantity) {
            throw new Exception("Invalid operation");
        }
        int newQuantity = inventory.get(title) - quantity;
        if (newQuantity == 0) {
            inventory.remove(title);
        } else {
            inventory.put(title, newQuantity);
        }
    }

    public Map<String, Integer> viewInventory() {
        return new HashMap<>(inventory);
    }

    public int viewBookQuantity(String title) {
        return inventory.getOrDefault(title, 0);
    }
}

=== RAW ANSWER ===

To translate the provided Java code into C++ while maintaining identical behavior, we need to address several key aspects: data structures, exception handling, and method signatures. Here's the step-by-step solution:

### Approach
1. **Data Structure**: Use `std::map` instead of `HashMap` since `std::map` provides ordered operations and is idiomatic C++ for key-value pairs.
2. **Exception Handling**: In C++, exceptions are typically `std::exception` derived. We use `std::runtime_error` for clarity and idiomatic C++.
3. **Method Signatures**: Ensure all methods have the same behavior, including throwing exceptions when invalid operations occur.
4. **Copy Semantics**: Return a copy of the map in `viewInventory` to prevent external modification of internal state.

### Solution Code
```cpp
#include <map>
#include <string>
#include <stdexcept>

class BookManagement {
private:
    std::map<std::string, int> inventory;

public:
    BookManagement() = default;

    void addBook(const std::string& title, int quantity) {
        auto it = inventory.find(title);
        if (it != inventory.end()) {
            it->second += quantity;
        } else {
            inventory.insert({title, quantity});
        }
    }

    void removeBook(const std::string& title, int quantity) {
        auto it = inventory.find(title);
        if (it == inventory.end() || it->second < quantity) {
            throw std::runtime_error("Invalid operation");
        }
        it->second -= quantity;
        if (it->second == 0) {
            inventory.erase(it);
        }
    }

    std::map<std::string, int> viewInventory() const {
        return inventory;
    }

    int viewBookQuantity(const std::string& title) const {
        auto it = inventory.find(title);
        if (it != inventory.end()) {
            return it->second;
        }
        return 0;
    }
};
```

### Explanation
1. **Data Structure**: `std::map` is used for its ordered nature and ease of lookup, insertion, and deletion, mirroring the Java `HashMap` behavior but with ordered operations.
2. **Exception Handling**: The `removeBook` method throws a `std::runtime_error` when invalid operations occur, maintaining the same exception message as the Java counterpart.
3. **Method Implementations**:
   - `addBook`: Checks if the book exists and updates the quantity; otherwise, inserts a new entry.
   - `removeBook`: Throws an exception if the book is not found or insufficient quantity exists. Adjusts the quantity and removes the entry if it reaches zero.
   - `viewInventory`: Returns a copy of the internal map to ensure encapsulation.
   - `viewBookQuantity`: Returns the quantity of the book or zero if not found, matching the Java behavior.

This approach ensures the C++ class behaves identically to the Java code, adhering to idiomatic C++ practices where appropriate.
