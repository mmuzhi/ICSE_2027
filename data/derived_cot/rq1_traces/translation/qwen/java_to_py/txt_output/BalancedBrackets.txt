package org.example;

import java.util.Stack;

public class BalancedBrackets {
    private Stack<Character> stack;
    private String leftBrackets;
    private String rightBrackets;
    private String expr;

    public BalancedBrackets(String expr) {
        this.stack = new Stack<>();
        this.leftBrackets = "({[";
        this.rightBrackets = ")}]";
        this.expr = expr;
    }

    public void clearExpr() {
        StringBuilder cleanedExpr = new StringBuilder();
        for (char c : expr.toCharArray()) {
            if (leftBrackets.indexOf(c) != -1 || rightBrackets.indexOf(c) != -1) {
                cleanedExpr.append(c);
            }
        }
        this.expr = cleanedExpr.toString();
    }

    public boolean checkBalancedBrackets() {
        clearExpr();
        for (char Brkt : expr.toCharArray()) {
            if (leftBrackets.indexOf(Brkt) != -1) {
                stack.push(Brkt);
            } else {
                if (stack.isEmpty()) {
                    return false;
                }
                char currentBrkt = stack.pop();
                if (currentBrkt == '(' && Brkt != ')') {
                    return false;
                }
                if (currentBrkt == '{' && Brkt != '}') {
                    return false;
                }
                if (currentBrkt == '[' && Brkt != ']') {
                    return false;
                }
            }
        }
        return stack.isEmpty();
    }

    public String getExpr() {
        return expr;
    }

    public static void main(String[] args) {
        BalancedBrackets b = new BalancedBrackets("a(b)c");
        System.out.println(b.checkBalancedBrackets());
    }
}

=== RAW ANSWER ===

To solve this problem, we need to translate a Java program that checks for balanced brackets in an expression into an equivalent Python program. The solution must maintain the same behavior, including handling edge cases and exceptions correctly.

### Approach
The Java program uses a stack to keep track of opening brackets and ensures that each closing bracket matches the most recent unmatched opening bracket. The approach involves:
1. **Cleaning the Expression**: Remove all characters that are not brackets.
2. **Checking Balance**: Use a stack to track opening brackets. For each closing bracket encountered, check if it matches the top of the stack. If not, the expression is unbalanced. After processing all brackets, the stack must be empty for the expression to be balanced.

In Python, we use a list as a stack due to its efficient append and pop operations. The solution mirrors the Java logic closely, ensuring identical behavior by:
- Using a list for stack operations.
- Cleaning the expression by filtering out non-bracket characters.
- Checking bracket matches using a dictionary for quick lookups.

### Solution Code
```python
class BalancedBrackets:
    def __init__(self, expr):
        self.stack = []
        self.left_brackets = "({["
        self.right_brackets = ")}]"
        self.expr = expr

    def clean_expr(self):
        cleaned_chars = []
        for char in self.expr:
            if char in self.left_brackets or char in self.right_brackets:
                cleaned_chars.append(char)
        self.expr = ''.join(cleaned_chars)

    def check_balanced_brackets(self):
        self.clean_expr()
        matching_brackets = {')': '(', ']': '[', '}': '{'}
        for char in self.expr:
            if char in self.left_brackets:
                self.stack.append(char)
            else:
                if not self.stack:
                    return False
                top = self.stack.pop()
                if matching_brackets[char] != top:
                    return False
        return len(self.stack) == 0

    def get_expr(self):
        return self.expr

if __name__ == "__main__":
    b = BalancedBrackets("a(b)c")
    print(b.check_balanced_brackets())
```

### Explanation
1. **Initialization**: The `BalancedBrackets` class initializes with an expression and sets up the stack and bracket sets.
2. **Cleaning Expression**: The `clean_expr` method filters out any non-bracket characters from the expression.
3. **Checking Balance**: The `check_balanced_brackets` method processes each character in the cleaned expression:
   - **Opening Brackets**: Pushed onto the stack.
   - **Closing Brackets**: Popped from the stack and checked against the corresponding opening bracket. If the stack is empty or the brackets don't match, the expression is unbalanced.
4. **Edge Cases**: The method handles cases where closing brackets appear without matching opening brackets or vice versa.
5. **Output**: The `get_expr` method allows retrieval of the processed expression for further use.

This approach ensures that the Python solution behaves identically to the Java original, maintaining all edge cases and exceptions.
