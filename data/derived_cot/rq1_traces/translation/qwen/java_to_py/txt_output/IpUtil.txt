package org.example;

import java.net.InetAddress;
import java.net.UnknownHostException;

public class IpUtil {

    public static boolean isValidIpv4(String ipAddress) {
        try {
            InetAddress.getByName(ipAddress);
            return ipAddress.contains(".") && !ipAddress.contains(":");
        } catch (UnknownHostException e) {
            return false;
        }
    }

    public static boolean isValidIpv6(String ipAddress) {
        try {
            InetAddress.getByName(ipAddress);
            return ipAddress.contains(":");
        } catch (UnknownHostException e) {
            return false;
        }
    }

    public static String getHostname(String ipAddress) {
        try {
            InetAddress inetAddress = InetAddress.getByName(ipAddress);

            if ("0.0.0.0".equals(ipAddress)) {
                return InetAddress.getLocalHost().getHostName();
            }

            String hostname = inetAddress.getCanonicalHostName();
            if (hostname.equals(ipAddress)) {
                return null;
            }
            return hostname;
        } catch (UnknownHostException e) {
            return null;
        }
    }
}

=== RAW ANSWER ===

To solve this problem, we need to translate Java methods for validating IPv4 and IPv6 addresses and retrieving hostnames into equivalent Python methods. The goal is to maintain the same behavior, including input/outputs, side effects, edge cases, and exception handling.

### Approach
1. **Validation Methods**:
   - **IPv4 Validation**: The method checks if the input string contains at least one dot and no colons. It then attempts to resolve the address as an IPv4 address. If successful, it returns true; otherwise, false.
   - **IPv6 Validation**: The method checks if the input string contains at least one colon. It then attempts to resolve the address as an IPv6 address. If successful, it returns true; otherwise, false.

2. **Hostname Retrieval**:
   - **Special Case Handling**: If the input is "0.0.0.0", the method retrieves the local host's canonical name.
   - **General Case**: For other inputs, it resolves the address to an IP, then retrieves the hostname. If the hostname matches the input address, it returns null; otherwise, it returns the hostname. Any exceptions during resolution result in returning null.

### Solution Code
```python
import socket

def is_valid_ipv4(ip_address):
    try:
        socket.getaddrinfo(ip_address, None, socket.AF_INET)
        if '.' in ip_address and ':' not in ip_address:
            return True
        else:
            return False
    except Exception:
        return False

def is_valid_ipv6(ip_address):
    try:
        socket.getaddrinfo(ip_address, None, socket.AF_INET6)
        if ':' in ip_address:
            return True
        else:
            return False
    except Exception:
        return False

def get_hostname(ip_address):
    if ip_address == "0.0.0.0":
        try:
            local_host = socket.getlocalhost()
            return local_host.getfqdn()
        except Exception:
            return None
    else:
        try:
            # Resolve the input to an address
            addr = socket.getaddrinfo(ip_address, None)[0][4][0]
            # Get the hostname for the resolved address
            hostname = socket.gethostbyaddr(addr)[0]
        except (socket.herror, socket.gaierror) as e:
            hostname = ip_address
        if hostname == ip_address:
            return None
        else:
            return hostname
```

- **IPv4 Validation**: The method uses `socket.getaddrinfo` with `AF_INET` to check if the address is valid IPv4. It also checks for the presence of dots and absence of colons in the string.

- **Hostname Retrieval**: The method first handles the special case "0.0.0.0" by returning the local host's name. For other addresses, it resolves the address to an IP, then retrieves the hostname.
