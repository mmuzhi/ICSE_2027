public class NumberConverter {

    public static String decimalToBinary(int decimalNum) {
        return Integer.toBinaryString(decimalNum);
    }

    public static int binaryToDecimal(String binaryNum) {
        return Integer.parseInt(binaryNum, 2);
    }

    public static String decimalToOctal(int decimalNum) {
        return Integer.toOctalString(decimalNum);
    }

    public static int octalToDecimal(String octalNum) {
        return Integer.parseInt(octalNum, 8);
    }

    public static String decimalToHex(int decimalNum) {
        return Integer.toHexString(decimalNum);
    }

    public static int hexToDecimal(String hexNum) {
        return Integer.parseInt(hexNum, 16);
    }
}

=== RAW ANSWER ===

To solve this problem, we need to translate a Java class that converts numbers between decimal, binary, octal, and hexadecimal formats into an equivalent Python class. The goal is to maintain the same behavior, inputs, outputs, side effects, edge cases, and exception handling as the original Java code.

### Approach
1. **Method Translation**: Each method from the Java class is translated into a static method in Python using the `@staticmethod` decorator.
2. **String Conversion**: For converting integers to strings (binary, octal, hex), Python's built-in functions `bin()`, `oct()`, and `hex()` are used. These functions return strings prefixed with '0b', '0o', or '0x' respectively, so we slice off the prefix.
3. **String Conversion to Integers**: For converting strings to integers (binary, octal, hex), Python's `int()` function is used with the appropriate base (2, 8, or 16). This function handles both signed and unsigned numbers correctly.
4. **Exception Handling**: Python's `ValueError` is used for invalid number formats, which corresponds to Java's `NumberFormatException`. Python's `TypeError` is used for invalid input types, similar to Java's `NullPointerException` when null is passed.

### Solution Code
```python
class NumberConverter:

    @staticmethod
    def decimal_to_binary(decimal_num):
        return bin(decimal_num)[2:]

    @staticmethod
    def binary_to_decimal(binary_num):
        return int(binary_num, 2)

    @staticmethod
    def decimal_to_octal(decimal_num):
        return oct(decimal_num)[1:]

    @staticmethod
    def octal_to_decimal(octal_num):
        return int(octal_num, 8)

    @staticmethod
    def decimal_to_hex(decimal_num):
        return hex(decimal_num)[2:]

    @staticmethod
    def hex_to_decimal(hex_num):
        return int(hex_num, 16)
```

### Explanation
1. **decimal_to_binary**: Converts an integer to its binary string representation by removing the '0b' prefix from the result of `bin()`.
2. **binary_to_decimal**: Converts a binary string to an integer using `int()` with base 2.
3. **decimal_to_octal**: Converts an integer to its octal string representation by removing the '0o' prefix from the result of `oct()`.
4. **octal_to_decimal**: Converts an octal string to an integer using `int()` with base 8.
5. **decimal_to_hex**: Converts an integer to its hexadecimal string representation by removing the '0x' prefix from the result of `hex()`.
6. **hex_to_decimal**: Converts a hexadecimal string to an integer using `int()` with base 16.

Each method preserves the behavior of its Java counterpart by handling both valid and invalid inputs appropriately, ensuring the same exception types and error conditions are maintained.
