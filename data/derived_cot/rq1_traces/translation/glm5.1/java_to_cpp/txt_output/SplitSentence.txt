=== SOURCE CODE (java) ===
package org.example;

import java.util.ArrayList;
import java.util.List;
import java.util.regex.Matcher;
import java.util.regex.Pattern;

public class SplitSentence {

    public List<String> splitSentences(String sentencesString) {
        List<String> sentences = new ArrayList<>();
        Pattern pattern = Pattern.compile("(?<!\\w\\.\\w.)(?<![A-Z][a-z]\\.)(?<=\\.|\\?)\\s");
        Matcher matcher = pattern.matcher(sentencesString);
        int lastEnd = 0;
        while (matcher.find()) {
            sentences.add(sentencesString.substring(lastEnd, matcher.end() - 1));
            lastEnd = matcher.end();
        }
        if (lastEnd < sentencesString.length()) {
            sentences.add(sentencesString.substring(lastEnd));
        }
        return sentences;
    }

    public int countWords(String sentence) {
        String cleanedSentence = sentence.replaceAll("[^a-zA-Z\\s]", "");
        String[] words = cleanedSentence.split("\\s+");
        return words.length;
    }

    public int processTextFile(String sentencesString) {
        List<String> sentences = splitSentences(sentencesString);
        int maxCount = 0;
        for (String sentence : sentences) {
            int count = countWords(sentence);
            if (count > maxCount) {
                maxCount = count;
            }
        }
        return maxCount;
    }
}

### Key Translation Details:
1. **Regex Lookbehinds Workaround:** Standard C++ `<regex>` (ECMAScript flavor) does not support lookbehinds (`(?<=...)` and `(?<!...)`). To maintain identical behavior without relying on Boost, the regex pattern `(?<!\w\.\w.)(?<![A-Z][a-z]\.)(?<=\.|\?)\s` was manually translated into explicit index checks mirroring the exact conditions of the lookbehinds.
2. **Java `split("\\s+")` Quirks:** Java's `String.split()` has a well-known edge case where splitting an empty string returns an array of length 1 containing the empty string itself (i.e., `"".split("\\s+").length == 1`). The C++ translation explicitly checks for an empty `cleanedSentence` and returns `1` to faithfully replicate this. For non-empty strings, `std::istringstream >> word` naturally handles the splitting and trimming of whitespace identically to Java's regex split.
3. **Locale-Agnostic Character Checks:** Java regex character classes like `\w`, `[A-Z]`, and `\s` operate strictly on ASCII/Unicode code points. C++ standard library functions like `std::isalpha` or `std::isspace` are locale-dependent. To guarantee identical behavior across all environments, custom static helpers (`is_w`, `is_upper_az`, `is_s`, etc.) were implemented using direct ASCII range checks matching Java's definitions.
