        if (key == null) {
            return null;
        }
        return _toCamelCase(key);
    }

    static String _toCamelCase(String key) {
        String[] parts = key.split("_");
        StringBuilder camelCaseString = new StringBuilder(parts[0]);
        for (int i = 1; i < parts.length; i++) {
            camelCaseString.append(parts[i].substring(0, 1).toUpperCase()).append(parts[i].substring(1).toLowerCase());
        }
        return camelCaseString.toString();
    }

    public Set<String> keySet() {
        return data.keySet();
    }

    public int size() {
        return data.size();

### Key translation decisions:

| Java | C++ | Rationale |
|------|-----|-----------|
| `LinkedHashMap<String, Object>` | `vector<string>` + `unordered_map<string, any>` | Vector preserves insertion order; map provides O(1) lookup — together they replicate `LinkedHashMap` semantics |
| `Object` (nullable) | `std::any*` (nullptr = not found) | Closest to Java's nullable reference return from `get()` |
| `String.split("_")` | Manual split + trailing-empty removal | Java's `split` discards trailing empty strings by default; the C++ split replicates this |
| `substring(0,1).toUpperCase()` | `toupper(parts[i][0])` | Direct equivalent; throws `out_of_range` on empty segment to match Java's `IndexOutOfBoundsException` |
| `Set<String> keySet()` | `vector<string>` | Returns a snapshot copy in insertion order; a true view would require a custom wrapper class |
| `null` key check in `_convertKey` | Omitted | C++ `std::string` cannot be null; the check is unnecessary |
| Package-private methods | `private` | C++ has no package-private visibility; `private` is the closest appropriate choice |
