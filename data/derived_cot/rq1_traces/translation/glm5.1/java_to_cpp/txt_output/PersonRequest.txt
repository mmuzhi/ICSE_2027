### Key translation decisions:

1. **`null` → `std::nullopt`**: Java's nullable `String` is represented as `std::optional<std::string>`. This preserves the three-state semantics (has-value/empty-string/no-value) that the Java code relies on in its validation logic.

2. **`String.matches("\\d{11}")` → `std::regex_match(..., std::regex("\\d{11}"))`**: Both match against the entire string. `std::regex_match` only returns `true` if the entire input matches, just like Java's `matches()`.

3. **Constructor parameters as `std::optional<std::string>`**: This preserves the ability to pass "null" (as `std::nullopt`) to the constructor, matching the Java API where any parameter can be `null`.

4. **Validation methods are `static`**: They don't access instance state, so making them `static` is idiomatic C++ with no behavioral change.

5. **Getters return `const std::optional<std::string>&`**: Avoids unnecessary copies while maintaining read-only access, consistent with Java's reference-returning getters.

6. **`package org.example` → `namespace org::example`**: Direct structural equivalent.
