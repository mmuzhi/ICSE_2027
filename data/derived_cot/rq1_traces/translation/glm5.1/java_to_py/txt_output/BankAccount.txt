=== SOURCE CODE (java) ===
package org.example;

public class BankAccount {
    private int balance;

    public BankAccount(int balance) {
        this.balance = balance;
    }

    public BankAccount() {
        this(0);
    }

    public int deposit(int amount) {
        if (amount < 0) {
            throw new IllegalArgumentException("Invalid amount");
        }
        this.balance += amount;
        return this.balance;
    }

    public int withdraw(int amount) {
        if (amount < 0) {
            throw new IllegalArgumentException("Invalid amount");
        }
        if (amount > this.balance) {
            throw new IllegalArgumentException("Insufficient balance.");
        }
        this.balance -= amount;
        return this.balance;
    }

    public int viewBalance() {
        return this.balance;
    }

    public void transfer(BankAccount otherAccount, int amount) {
        this.withdraw(amount);
        otherAccount.deposit(amount);
    }

    public static void main(String[] args) {
        BankAccount account1 = new BankAccount();
        BankAccount account2 = new BankAccount();
        account1.deposit(1000);
        account1.transfer(account2, 300);
        System.out.println("account1.balance = " + account1.viewBalance());
        System.out.println("account2.balance = " + account2.viewBalance());
    }
}

=== RAW ANSWER ===
```python
class BankAccount:
    def __init__(self, balance=0):
        # Using _balance to indicate it's intended to be private, matching the Java code's intent
        self._balance = balance

    def deposit(self, amount):
        if amount < 0:
            raise ValueError("Invalid amount")
        self._balance += amount
        return self._balance

    def withdraw(self, amount):
        if amount < 0:
            raise ValueError("Invalid amount")
        if amount > self._balance:
            raise ValueError("Insufficient balance.")
        self._balance -= amount
        return self._balance

    def view_balance(self):
        return self._balance

    def transfer(self, other_account, amount):
        self.withdraw(amount)
        other_account.deposit(amount)

if __name__ == "__main__":
    account1 = BankAccount()
    account2 = BankAccount()
    account1.deposit(1000)
    account1.transfer(account2, 300)
    print(f"account1.balance = {account1.view_balance()}")
    print(f"account2.balance = {account2.view_balance()}")
```

### Key Translation Decisions:
- **Constructors:** Java's constructor overloading (`BankAccount()` and `BankAccount(int balance)`) is idiomatically replaced in Python using a default argument `balance=0` in the `__init__` method.
- **Privacy:** Java's `private` modifier doesn't have a direct equivalent in Python. The single underscore prefix `_balance` is used as the idiomatic Python convention to indicate that the attribute is intended for internal use only.
- **Exceptions:** Java's `IllegalArgumentException` is translated to Python's built-in `ValueError`, which serves the exact same purpose of signaling inappropriate argument values.
- **Method Names:** Converted from Java's `camelCase` to Python's idiomatic `snake_case` (e.g., `viewBalance` -> `view_balance`, `otherAccount` -> `other_account`).
- **Execution Block:** The `main` method is translated to an `if __name__ == "__main__":` block, which is the Python standard for executable scripts. String concatenation in `System.out.println` is replaced with an f-string.
