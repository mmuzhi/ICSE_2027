        return tm;

        long d = a / b;
        long r = a % b;
        if (r != 0 && ((a < 0) ^ (b < 0))) d--;

            abs_result = (int_part % 2 == 0) ? int_part : int_part + 1;

Key translation decisions:

1. **`std::tm` as the datetime type** — closest C++ equivalent to Python's `datetime.datetime`, used for both the `datetime` member and return type of `string_to_datetime`.

2. **`sscanf` for parsing** — Python's `strptime` with `%m`/`%d`/etc. accepts single-digit values (e.g., `"7"` matches `%m`). `sscanf` with `%d` handles both `"7"` and `"07"`, matching this leniency.

3. **`timedelta.seconds` replication** — Python's `.seconds` attribute returns only the within-day component (0–86399), *not* total seconds. This is a known gotcha. The C++ code replicates this via `floor_div` to compute days, then `total_seconds - days * 86400`.

4. **Python's `round()` (banker's rounding)** — Python rounds half-to-even (`round(2.5) == 2`), unlike C++'s `std::round` which rounds half-away-from-zero. A custom `py_round` function replicates this exactly.

5. **`floor_div`** — C++ integer division truncates toward zero; Python's `//` floors. The helper corrects this for negative dividends.

6. **Public `datetime` member** — matches Python's accessible `self.datetime`.
