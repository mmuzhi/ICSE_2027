### Key translation notes:

1. **`zip` behavior**: Python's `zip(x_interp, y_interp)` iterates up to the shorter list. In C++, this is replicated with `std::min(x_interp.size(), y_interp.size())` and indexed iteration.

2. **`range(len(x) - 1)`**: Translated as `size_t i = 0; i + 1 < x.size(); ++i`. Using `i + 1 < x.size()` avoids unsigned underflow that `i < x.size() - 1` would cause when `x` is empty (since `x.size() - 1` would wrap around for `size_t(0)`).

3. **Chained comparison `x[i] <= xi <= x[i+1]`**: Python's chained comparison is split into `x[i] <= xi && xi <= x[i + 1]` in C++.

4. **`break` semantics**: The nested `break` statements in the inner loops replicate Python's behavior — the inner `break` exits the `j`-loop, then the outer `break` exits the `i`-loop, moving to the next interpolation point.

5. **No-match behavior**: If an interpolation point doesn't fall within any interval, it is silently skipped (no value appended), matching the Python behavior.
