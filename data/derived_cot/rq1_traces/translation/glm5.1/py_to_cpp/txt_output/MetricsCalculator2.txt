### Key design decisions:

1. **Overloads instead of runtime type dispatch**: The Python code uses `type()` checks to handle both single tuples and lists of tuples in one method. C++ uses function overloading — `mrr(const DataPoint&)` for a single entry and `mrr(const std::vector<DataPoint>&)` for a list — achieving identical behavior with compile-time type safety (invalid types are rejected at compile time rather than throwing a runtime exception).

2. **Type aliases**: `DataPoint = std::pair<std::vector<int>, int>` mirrors the Python tuple `([0,1,...], int)`, and `ResultType = std::pair<double, std::vector<double>>` mirrors the Python return of `(float, list[float])`.

3. **MRR computation**: Matches the Python logic exactly — computes `sub_list[i] / (i+1)` for each position and returns the first positive value (equivalent to `sub_list * ranking_array` then finding the first `> 0` element).

4. **MAP computation**: Builds `right_ranking_list` identically (running count of 1s for correct answers, 0 for wrong answers), then sums `right_ranking_list[i] / (i+1)` and divides by `total_num`.

5. **Edge cases preserved**: Empty list returns `{0.0, {0.0}}`, `total_num == 0` returns `{0.0, {0.0}}`, and no correct answer in the list yields `mr = 0.0` / `ap = 0.0`.
