class MovieTicketDB:
    def __init__(self, db_name):
        self.connection = sqlite3.connect(db_name)
        self.cursor = self.connection.cursor()
        self.create_table()

    def create_table(self):
        self.cursor.execute('''

                movie_name TEXT,
                theater_name TEXT,
                seat_number TEXT,
                customer_name TEXT
            )

        self.cursor.execute('''
            SELECT * FROM tickets WHERE customer_name = ?

        return tickets

### Key translation decisions:

| Python | C++ |
|---|---|
| `sqlite3.connect()` / `.cursor()` | `sqlite3_open()` — C API doesn't separate cursor from connection; all operations go through the `sqlite3*` handle |
| `cursor.execute()` with `?` params | `sqlite3_prepare_v2()` + `sqlite3_bind_*()` — parameterized queries done in two steps |
| `connection.commit()` | Implicit in C API — each `sqlite3_step()` that modifies auto-commits unless you explicitly begin a transaction |
| `cursor.fetchall()` → `list[tuple]` | `std::vector<std::tuple<int, std::string, std::string, std::string, std::string>>` — preserves the row structure |
| Implicit connection cleanup | Destructor calls `sqlite3_close()` (RAII); copy is deleted since `sqlite3*` isn't copyable |
| Python exceptions from sqlite3 | `std::runtime_error` thrown on any SQLite error, matching the fail-loud behavior |
