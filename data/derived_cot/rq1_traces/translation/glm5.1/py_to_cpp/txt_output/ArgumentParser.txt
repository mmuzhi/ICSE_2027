### Key design decisions:

1. **`std::any` for argument values** — Python's dict can hold any type (`str`, `int`, `bool`, etc.). `std::any` is the C++ equivalent for type-erased storage. Users retrieve values via `std::any_cast<T>`.

2. **`std::function<std::any(const std::string&)>` for type converters** — Python uses first-class types (`int`, `str`) as callables. C++ has no equivalent, so `std::function` wraps conversion logic. For example, an `int` type would be passed as `[](const std::string& v) -> std::any { return std::stoi(v); }`.

3. **Default `arg_type`** — Python's default is `str`, which returns the string unchanged. The C++ default (`nullptr`) is replaced internally with a lambda that wraps the string in `std::any`, matching `str()` behavior.

4. **`std::optional<std::any>` for `get_argument`** — Matches Python's `dict.get()` returning `None` for missing keys.

5. **Return type of `parse_arguments`** — `std::pair<bool, std::optional<std::unordered_set<std::string>>>` matches Python's `(True, None)` / `(False, missing_args)`.

6. **`catch (...)` in `_convert_type`** — Catches all exceptions (analogous to Python catching `ValueError` and `KeyError`), returning the original value on failure.
