### Key design decisions:

- **`Dict` type alias**: `std::map<std::string, std::variant<int, std::string>>` mirrors Python's `dict` with mixed value types, supporting empty dicts `{}` and keyed access.

- **Polymorphic returns via `std::variant`**: Python methods return different types (lists, dicts, bools, strings, ints). Each C++ method uses a `std::variant` over all possible return types:
  - `add_white_list` / `del_white_list` → `variant<vector<int>, bool>`
  - `recv` → `variant<int, bool, string>` (−1 for invalid, `false` for unauthorized, content string on success)
  - `send` → `variant<monostate, string>` (`monostate` = Python's `None`)
  - `show` → `variant<Dict, bool>`

- **Struct assignment**: `receive_struct` and `send_struct` are cleared and repopulated (rather than assigned via initializer lists) to avoid ambiguity with nested `std::variant` brace initialization.

- **Validation**: Key-existence checks in `recv`/`send` mirror Python's `"addr" not in info` checks. The `isinstance(info, dict)` check is implicitly satisfied by C++'s static typing. Type mismatches in variant access (e.g., `"addr"` holding a string) will throw `std::bad_variant_access`, analogous to Python's implicit `TypeError`.
