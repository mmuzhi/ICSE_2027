#include <vector>

class AvgPartition {
public:
    AvgPartition(const std::vector<int>& lst, int limit);
    std::pair<int, int> set_num() const;
    std::vector<int> get(int index) const;

private:
    std::vector<int> lst;
    int limit;
};
AvgPartition::AvgPartition(const std::vector<int>& lst, int limit)
    : lst(lst), limit(limit) {}

std::pair<int, int> AvgPartition::set_num() const {
    int size = lst.size() / limit;
    int remainder = lst.size() % limit;
    return { size, remainder };
}

std::vector<int> AvgPartition::get(int index) const {
    auto [size, remainder] = set_num();
    int start = index * size + std::min(index, remainder);
    int end = start + size;
    if (index + 1 <= remainder) {
        end += 1;
    }
    return std::vector<int>(lst.begin() + start, lst.begin() + end);
}

=== RAW ANSWER ===

To solve this problem, we need to translate a given C++ class into Python while maintaining identical behavior. The C++ class partitions a list into several sublists based on a given limit, distributing any extra elements as evenly as possible across the partitions.

### Approach
1. **Initialization**: The class constructor stores the input list and the partition limit.
2. **Calculate Partition Parameters**: The method `set_num` computes two values:
   - `size`: The minimum number of elements each partition will have.
   - `remainder`: The number of partitions that will have one extra element.
3. **Retrieve Partition**: The method `get` retrieves a specific partition by index:
   - **Start Index Calculation**: The start index is computed as `index * size + min(index, remainder)`.
   - **End Index Calculation**: The end index is initially `start + size`. If the current partition index is less than `remainder`, the end index is incremented by 1 to include an extra element.
   - **Return Partition**: The partition is returned as a sublist from the start index to the end index.

### Solution Code
```python
class AvgPartition:
    def __init__(self, lst, limit):
        self.lst = lst
        self.limit = limit
    
    def set_num(self):
        n = len(self.lst)
        size = n // self.limit
        remainder = n % self.limit
        return size, remainder
    
    def get(self, index):
        size, remainder = self.set_num()
        start = index * size + min(index, remainder)
        end = start + size
        if index < remainder:
            end += 1
        return self.lst[start:end]
```

### Explanation
- **Initialization**: The `__init__` method stores the input list (`lst`) and the partition limit (`limit`).
- **Partition Parameters**: The `set_num` method calculates the base size of each partition (`size`) and the number of partitions that will have an extra element (`remainder`).
- **Partition Retrieval**: The `get` method:
  - Computes the start index by considering both the base size and any extra elements allocated to partitions with indices less than `remainder`.
  - Adjusts the end index to include an extra element if the current partition index is within the first `remainder` partitions.
  - Returns the sublist from the computed start index to the end index.

This approach ensures that the partitions are as evenly distributed as possible, with any extra elements added to the first few partitions. The Python implementation mirrors the C++ behavior precisely, including handling edge cases like an empty list or zero limit (which would cause a division by zero error).
