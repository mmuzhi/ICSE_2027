=== SOURCE CODE (cpp) ===

#include <algorithm>

#include <stack>
#include <string>

class BalancedBrackets {
public:
    explicit BalancedBrackets(const std::string& expr);
    bool check_balanced_brackets();

    void clearExpr();

    std::stack<char> stack;
    const std::string leftBrackets = "({[";
    const std::string rightBrackets = ")}]";
    std::string expr;
};

BalancedBrackets::BalancedBrackets(const std::string& expr) : expr(expr) {}

void BalancedBrackets::clearExpr() {
    expr.erase(std::remove_if(expr.begin(), expr.end(), [this](char c) {
        return leftBrackets.find(c) == std::string::npos && rightBrackets.find(c) == std::string::npos;
        }), expr.end());
}

bool BalancedBrackets::check_balanced_brackets() {
    clearExpr();
    for (char Brkt : expr) {
        if (leftBrackets.find(Brkt) != std::string::npos) {
            stack.push(Brkt);
        }
        else {
            if (stack.empty()) return false;
            char Current_Brkt = stack.top();
            stack.pop();
            if (Current_Brkt == '(' && Brkt != ')') return false;
            if (Current_Brkt == '{' && Brkt != '}') return false;
            if (Current_Brkt == '[' && Brkt != ']') return false;
        }
    }
    return stack.empty();
}

=== RAW ANSWER ===

To solve this problem, we need to determine if a given string has balanced brackets. The solution involves checking that every opening bracket has a corresponding closing bracket in the correct order. The approach involves processing the string to remove any non-bracket characters and then using a stack to validate the balance of brackets.

### Approach
1. **Initialization**: The class `BalancedBrackets` is initialized with the input string. The class maintains a stack to keep track of opening brackets and defines sets of valid opening and closing brackets.
2. **Cleaning the Expression**: The `clearExpr` method processes the input string to remove any characters that are not valid brackets (either opening or closing brackets). This is done using a list comprehension to filter out non-bracket characters.
3. **Checking Balance**: The `check_balanced_brackets` method processes each character in the cleaned string:
   - If the character is an opening bracket, it is pushed onto the stack.
   - If it is a closing bracket, the method checks if the stack is not empty. If the stack is empty, the brackets are unbalanced.
   - The top character from the stack is popped, and the method checks if the closing bracket matches the corresponding opening bracket using a predefined mapping (`{ '(': ')', '{': '}', '[': ']' }`). If not, the brackets are unbalanced.
4. **Final Check**: After processing all characters, if the stack is empty, all opening brackets have been matched with corresponding closing brackets, indicating balanced brackets. Otherwise, the brackets are unbalanced.

### Solution Code
```python
class BalancedBrackets:
    def __init__(self, expr):
        self.stack = []
        self.leftBrackets = "({["
        self.rightBrackets = ")}]"
        self.expr = expr

    def clearExpr(self):
        self.expr = ''.join([c for c in self.expr if c in self.leftBrackets or c in self.rightBrackets])

    def check_balanced_brackets(self):
        self.clearExpr()
        mapping = {'(': ')', '{': '}', '[': ']'}
        for char in self.expr:
            if char in self.leftBrackets:
                self.stack.append(char)
            else:
                if not self.stack:
                    return False
                top_char = self.stack.pop()
                if mapping[top_char] != char:
                    return False
        return len(self.stack) == 0
```

2. **Cleaning the Expression**: The `clearExpr` method modifies the input string by retaining only those characters that are valid brackets (either opening or closing). This is achieved using a list comprehension that filters characters present in `leftBrackets` or `rightBrackets`, then joins them into a new string.
3. **Checking Balance**: The `check_balanced_brackets` method processes each character in the cleaned string:
   - **Opening Brackets**: When an opening bracket is encountered, it is pushed onto the stack.
   - **Closing Brackets**: For each closing bracket, the method checks if the stack is empty (indicating unbalanced brackets). If not, the top character is popped from the stack. The method then verifies if the closing bracket matches the expected one using a predefined mapping. If not, the brackets are unbalanced.
4. **Final Check**: After processing all characters, if the stack is empty, all opening brackets have been matched correctly, and the method returns `True`. Otherwise, it returns `False`, indicating unbalanced brackets.

This approach efficiently checks for balanced brackets by leveraging a stack to track opening brackets and ensuring each closing bracket correctly matches the most recent unmatched opening bracket. The solution also handles non-bracket characters by preprocessing the input string to remove them.
