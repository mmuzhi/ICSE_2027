=== SOURCE CODE (java) ===
package org.example;

public class BankAccount {
    private int balance;

    public BankAccount(int balance) {
        this.balance = balance;
    }

    public BankAccount() {
        this(0);
    }

    public int deposit(int amount) {
        if (amount < 0) {
            throw new IllegalArgumentException("Invalid amount");
        }
        this.balance += amount;
        return this.balance;
    }

    public int withdraw(int amount) {
        if (amount < 0) {
            throw new IllegalArgumentException("Invalid amount");
        }
        if (amount > this.balance) {
            throw new IllegalArgumentException("Insufficient balance.");
        }
        this.balance -= amount;
        return this.balance;
    }

    public int viewBalance() {
        return this.balance;
    }

    public void transfer(BankAccount otherAccount, int amount) {
        this.withdraw(amount);
        otherAccount.deposit(amount);
    }

    public static void main(String[] args) {
        BankAccount account1 = new BankAccount();
        BankAccount account2 = new BankAccount();
        account1.deposit(1000);
        account1.transfer(account2, 300);
        System.out.println("account1.balance = " + account1.viewBalance());
        System.out.println("account2.balance = " + account2.viewBalance());
    }
}

=== RAW ANSWER ===

To solve this problem, we need to translate a Java class representing a bank account into Python while maintaining identical behavior. The Java class includes functionalities for depositing, withdrawing, checking balances, transferring funds between accounts, and handling edge cases such as invalid amounts or insufficient funds.

### Approach
1. **Class Structure**: The Python class `BankAccount` will mirror the Java class structure, with an initializer that sets the balance (defaulting to 0 if not provided).
2. **Deposit Method**: Validates the deposit amount (must be non-negative) before updating the balance.
3. **Withdraw Method**: Validates the withdrawal amount (must be non-negative and not exceed the current balance) before updating the balance.
4. **Transfer Method**: Uses the withdraw and deposit methods to transfer funds between accounts, ensuring atomicity (either both operations succeed or neither does).
5. **Balance Check**: Provides a method to view the current balance.
6. **Exception Handling**: Uses `ValueError` for invalid arguments, equivalent to Java's `IllegalArgumentException`.
7. **Testing**: Includes a main block to test the functionality, similar to the Java main method.

### Solution Code
```python
class BankAccount:
    def __init__(self, balance=0):
        self._balance = balance

    def deposit(self, amount):
        if amount < 0:
            raise ValueError("Invalid amount")
        self._balance += amount
        return self._balance

    def withdraw(self, amount):
        if amount < 0:
            raise ValueError("Invalid amount")
        if amount > self._balance:
            raise ValueError("Insufficient balance.")
        self._balance -= amount
        return self._balance

    def viewBalance(self):
        return self._balance

    def transfer(self, otherAccount, amount):
        self.withdraw(amount)
        otherAccount.deposit(amount)

if __name__ == "__main__":
    account1 = BankAccount()
    account2 = BankAccount()
    account1.deposit(1000)
    account1.transfer(account2, 300)
    print(f"account1.balance = {account1.viewBalance()}")
    print(f"account2.balance = {account2.viewBalance()}")
```

### Explanation
- **Initialization**: The `__init__` method initializes the account balance, defaulting to 0 if no initial balance is provided.
- **Deposit Handling**: The `deposit` method checks if the amount is non-negative before adding it to the balance.
- **Withdrawal Handling**: The `withdraw` method checks for a valid non-negative amount and sufficient funds before deducting the amount from the balance.
- **Transfer Handling**: The `transfer` method uses `withdraw` from the current account and `deposit` to the target account, ensuring the transaction is atomic.
- **Balance Access**: The `viewBalance` method provides read-only access to the current balance.
- **Testing**: The main block creates two accounts, deposits funds into one, transfers funds between them, and prints the resulting balances to verify correctness.

This approach ensures that the Python implementation behaves identically to the original Java code, handling all edge cases and exceptions as specified.
