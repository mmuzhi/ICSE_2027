        if (!(data instanceof List) && !(data instanceof Tuple)) {
            throw new IllegalArgumentException("the input must be a tuple([0,...,1,...],int) or a iteration of list of tuple");
        }

        if (data instanceof Tuple) {
            Tuple tuple = (Tuple) data;
            List<Integer> subList = tuple.getList();
            int totalNum = tuple.getTotalNum();

            if (totalNum == 0) {
                return 0.0;
            }

            double mr = 0.0;
            for (int i = 0; i < subList.size(); i++) {
                if (subList.get(i) == 1) {
                    mr = 1.0 / (i + 1);
                    break;
                }
            }
            return mr;
        }
        else {
            List<Tuple> tupleList = (List<Tuple>) data;
            List<Double> separateResult = new ArrayList<>();

            for (Tuple tuple : tupleList) {
                List<Integer> subList = tuple.getList();
                int totalNum = tuple.getTotalNum();

                if (totalNum == 0) {
                    separateResult.add(0.0);
                } else {
                    double mr = 0.0;
                    for (int i = 0; i < subList.size(); i++) {
                        if (subList.get(i) == 1) {
                            mr = 1.0 / (i + 1);
                            break;
                        }

                    separateResult.add(mr);
                }
            }

        }
    }

The methods accept either a single `Tuple` or a list of `Tuple` objects, compute the respective metrics, and handle edge cases such as zero total numbers appropriately.

### Approach
1. **Inner Class `Tuple`**: We define an inner class `Tuple` within `MetricsCalculator2` to mirror the Java `Tuple` class. This class stores a list of integers and an integer representing the total number of relevant items.
2. **Type Checking**: Both methods (`mrr` and `map`) first check if the input is either a `Tuple` or a list of `Tuple` objects. If not, they raise a `TypeError`.
3. **Handling Single Tuple**: For a single `Tuple`, the methods compute MRR or MAP directly:
   - **MRR**: Finds the first occurrence of 1 in the list and calculates the reciprocal of its rank (1-based index). If no 1 is found, returns 0.0.
   - **MAP**: Computes the average precision by summing the precision at each position where a 1 occurs, then divides by the total number of relevant items.
4. **Handling List of Tuples**: For a list of `Tuple` objects, each tuple is processed to compute individual MRR or MAP values. These values are averaged to produce the final result. If the list is empty, returns 0.0.
5. **Edge Cases**: Both methods return 0.0 if the total number of relevant items (`total_num`) is zero for any tuple.
