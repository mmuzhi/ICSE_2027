package org.example;

import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.Map;

public class BoyerMooreSearch {
    private final String text;
    private final String pattern;
    private final int textLen;
    private final int patLen;

    public BoyerMooreSearch(String text, String pattern) {
        this.text = text;
        this.pattern = pattern;
        this.textLen = text.length();
        this.patLen = pattern.length();
    }

    int matchInPattern(char ch) {
        return pattern.lastIndexOf(ch);
    }

    int mismatchInText(int currentPos) {
        for (int i = patLen - 1; i >= 0; i--) {
            if (pattern.charAt(i) != text.charAt(currentPos + i)) {
                return currentPos + i;
            }
        }
        return -1;
    }

    public List<Integer> badCharacterHeuristic() {
        List<Integer> positions = new ArrayList<>();
        int i = 0;

        Map<Character, Integer> badCharHeuristic = new HashMap<>();
        for (int j = 0; j < patLen; j++) {
            badCharHeuristic.put(pattern.charAt(j), j);
        }

        if(patLen==0)
        {
            for(int j=0;j<=textLen;++j)
            {
                positions.add(j);
            }
            return positions;
        }

        while (i <= textLen - patLen) {
            int mismatchIndex = mismatchInText(i);
            if (mismatchIndex == -1) {
                positions.add(i);
                i += patLen;
            } else {
                char mismatchChar = text.charAt(mismatchIndex);
                int matchIndex = badCharHeuristic.getOrDefault(mismatchChar, -1);
                if (matchIndex >= 0) {
                    i += Math.max(1, mismatchIndex - i - matchIndex);
                }
                else {
                    i += mismatchIndex - i + 1;
                }
            }
        }
        return positions;
    }
}

=== RAW ANSWER ===

To solve this problem, we need to translate a Java implementation of the Boyer-Moore string search algorithm into C++ while maintaining identical behavior. The Boyer-Moore algorithm efficiently searches for occurrences of a pattern within a text by preprocessing the pattern to skip sections of the text that cannot possibly match the pattern, thus reducing the number of comparisons needed.

### Approach
1. **Problem Analysis**: The Java class `BoyerMooreSearch` uses the bad character heuristic of the Boyer-Moore algorithm to find all starting positions of a pattern in a given text. The bad character heuristic leverages the last occurrence of each character in the pattern to determine how far the search window can be shifted when a mismatch occurs.
2. **Key Insights**: 
   - **Preprocessing**: Build a map (`badCharHeuristic`) that records the last occurrence of each character in the pattern.
   - **Edge Handling**: If the pattern is empty, every position in the text is considered a match.
   - **Search Process**: Slide a window over the text. For each window:
     - Check for a match by comparing characters from the end of the pattern.
     - If a mismatch is found, use the bad character heuristic to determine the shift amount.
     - If no mismatch is found, record the starting position and shift the window by the pattern length.
3. **Algorithm Selection**: The bad character heuristic is chosen for its efficiency in skipping sections of the text, making the search process sub-linear in many cases.
4. **Complexity Analysis**: The worst-case time complexity is O(n*m), where `n` is the text length and `m` is the pattern length. However, in practice, the algorithm often performs better due to heuristic-based skips.

3. **mismatchInText**: Checks for a mismatch in the current window starting from the end of the pattern. Returns the position of the first mismatch or -1 if the window matches.
4. **badCharacterHeuristic**:
   - **Edge Case Handling**: If the pattern is empty, all positions in the text are considered matches.
   - **Preprocessing**: Builds a map of each character's last occurrence in the pattern.
   - **Search Loop**: Slides the window over the text:
     - **Full Match**: Records the starting position and shifts the window by the pattern length.
     - **Mismatch Handling**: Uses the bad character heuristic to compute the shift amount:
       - If the mismatched character exists in the pattern, shifts to align its last occurrence with the mismatch in the text.

5. **Efficiency**: The algorithm minimizes comparisons by leveraging the bad character heuristic, making it efficient for large texts with relatively small patterns. The preprocessing step ensures quick lookups during the search phase.
