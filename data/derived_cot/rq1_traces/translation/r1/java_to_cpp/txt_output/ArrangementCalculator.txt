package org.example;

import java.util.ArrayList;
import java.util.List;

public class ArrangementCalculator {
    private List<Object> datas;

    public ArrangementCalculator(List<Object> datas) {
        this.datas = datas;
    }

    public static int count(int n, Integer m) {
        if (m == null || n == m) {
            return factorial(n);
        }
        else {
            return factorial(n) / factorial(n - m);
        }
    }

    public static int countAll(int n) {
        int total = 0;
        for (int i = 1; i <= n; i++) {
            total += count(n, i);
        }
        return total;
    }

    public List<List<Object>> select(Integer m) {
        if (m == null) {
            m = datas.size();
        }
        List<List<Object>> result = new ArrayList<>();
        selectPermutations(new ArrayList<>(), new ArrayList<>(datas), m, result);
        return result;
    }

    public List<List<Object>> selectAll() {
        List<List<Object>> result = new ArrayList<>();
        for (int i = 1; i <= datas.size(); i++) {
            result.addAll(select(i));
        }
        return result;
    }

    private void selectPermutations(List<Object> prefix, List<Object> remaining, int m, List<List<Object>> result) {
        if (prefix.size() == m) {
            result.add(new ArrayList<>(prefix));
            return;
        }
        for (int i = 0; i < remaining.size(); i++) {
            List<Object> newPrefix = new ArrayList<>(prefix);
            newPrefix.add(remaining.get(i));
            List<Object> newRemaining = new ArrayList<>(remaining);
            newRemaining.remove(i);
            selectPermutations(newPrefix, newRemaining, m, result);
        }
    }

    public static int factorial(int n) {
        int result = 1;
        for (int i = 2; i <= n; i++) {
            result *= i;
        }
        return result;
    }
}

The Java class includes methods for counting permutations, generating all possible permutations, and generating permutations of a specific size. 

### Approach
1. **Data Representation**: In Java, the `List<Object>` can hold any type of object. To achieve similar behavior in C++, we use `std::vector<std::shared_ptr<void>>` to store pointers to any type of object. This allows us to handle heterogeneous objects similarly to Java's `Object` type.
2. **Static Methods**: The static methods `count`, `countAll`, and `factorial` are translated directly. The `count` method is overloaded to handle both cases where the size parameter `m` is provided or omitted (using `std::optional` for the omitted case).
3. **Permutation Generation**: The `select` and `selectAll` methods generate permutations by recursively building permutations of the specified size. The recursive helper function `selectPermutations` mirrors the Java implementation, using vectors to manage the current prefix and remaining elements.
4. **Factorial Calculation**: The `factorial` method computes the factorial of a number iteratively, identical to the Java version.

            selectPermutations(newPrefix, newRemaining, m, result);

### Explanation
1. **Data Storage**: The `datas` member is a vector of `shared_ptr<void>`, allowing storage of any object type via pointers.
2. **Static Methods**:
   - `count(int n)` computes the factorial of `n`, representing permutations of all elements.
   - `count(int n, int m)` computes permutations of `n` elements taken `m` at a time, using factorial division.
   - `countAll(int n)` sums permutations of sizes from 1 to `n`.
3. **Permutation Generation**:
   - `select(std::optional<int> m)` generates permutations of size `m` (or all elements if `m` is omitted) using recursion.
   - `selectAll()` generates all permutations of sizes from 1 to the size of `datas`.
4. **Helper Function**: `selectPermutations` recursively builds permutations by adding each remaining element to the current prefix and processing the remaining elements.
5. **Factorial Calculation**: The `factorial` method computes factorials iteratively for integers.

This approach ensures the C++ implementation behaves identically to the original Java code, handling inputs, outputs, edge cases, and side effects consistently.
