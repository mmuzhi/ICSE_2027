package org.example;

import java.util.HashMap;
import java.util.Map;

public class BookManagement {
    private Map<String, Integer> inventory;

    public BookManagement() {
        this.inventory = new HashMap<>();
    }

    public void addBook(String title, int quantity) {
        if (inventory.containsKey(title)) {
            inventory.put(title, inventory.get(title) + quantity);
        } else {
            inventory.put(title, quantity);
        }
    }

    public void removeBook(String title, int quantity) throws Exception {
        if (!inventory.containsKey(title) || inventory.get(title) < quantity) {
            throw new Exception("Invalid operation");
        }
        int newQuantity = inventory.get(title) - quantity;
        if (newQuantity == 0) {
            inventory.remove(title);
        } else {
            inventory.put(title, newQuantity);
        }
    }

    public Map<String, Integer> viewInventory() {
        return new HashMap<>(inventory);
    }

    public int viewBookQuantity(String title) {
        return inventory.getOrDefault(title, 0);
    }
}

=== RAW ANSWER ===

To solve this problem, we need to translate a Java class for book management into C++ while maintaining identical behavior. The Java class uses a `HashMap` to manage book inventory, providing methods to add books, remove books, view inventory, and check the quantity of a specific book. The C++ implementation should mirror this functionality using equivalent C++ constructs.

### Approach
1. **Data Structure Selection**: The Java `HashMap` is replaced with C++ `std::unordered_map` for similar key-value storage with average constant-time complexity for operations.
2. **Method Translation**: Each method in the Java class is translated to C++:
   - **Constructor**: Initializes the `std::unordered_map`.
   - **addBook**: Adds a book to the inventory or updates its quantity if it already exists.
   - **removeBook**: Removes a specified quantity of a book, throwing an exception if the operation is invalid (book not found or insufficient quantity).
   - **viewInventory**: Returns a copy of the current inventory.
   - **viewBookQuantity**: Returns the quantity of a specified book, or 0 if the book is not found.
3. **Exception Handling**: The Java `Exception` is replaced with C++ `std::runtime_error` to handle invalid removal operations.
4. **Memory Management**: The C++ implementation ensures that operations like removing a book when quantity reaches zero correctly erase the entry from the map.

### Solution Code
```cpp
#include <unordered_map>
#include <stdexcept>
#include <string>

class BookManagement {
private:
    std::unordered_map<std::string, int> inventory;

public:
    BookManagement() = default;

    void addBook(const std::string& title, int quantity) {
        auto it = inventory.find(title);
        if (it != inventory.end()) {
            it->second += quantity;
        } else {
            inventory[title] = quantity;
        }
    }

    void removeBook(const std::string& title, int quantity) {
        auto it = inventory.find(title);
        if (it == inventory.end() || it->second < quantity) {
            throw std::runtime_error("Invalid operation");
        }
        it->second -= quantity;
        if (it->second == 0) {
            inventory.erase(it);
        }
    }

    std::unordered_map<std::string, int> viewInventory() const {
        return inventory;
    }

    int viewBookQuantity(const std::string& title) const {
        auto it = inventory.find(title);
        if (it != inventory.end()) {
            return it->second;
        }
        return 0;
    }
};
```

### Explanation
- **Constructor**: Initializes an empty `std::unordered_map` to store book titles and their quantities.
- **addBook**: Checks if the book exists in the inventory. If it does, the quantity is updated; otherwise, a new entry is added.
- **removeBook**: Checks if the book exists and has sufficient quantity. If not, it throws a `std::runtime_error`. Otherwise, it reduces the quantity and removes the book if the quantity becomes zero.
- **viewInventory**: Returns a copy of the inventory map to prevent external modifications to the original inventory.
- **viewBookQuantity**: Returns the quantity of the specified book if it exists; otherwise, returns 0.

This C++ implementation faithfully replicates the behavior of the original Java code, including input/output handling, side effects, edge cases, and exception conditions. The use of `std::unordered_map` ensures efficient operations similar to Java's `HashMap`.
