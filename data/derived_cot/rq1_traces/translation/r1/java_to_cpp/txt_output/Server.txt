=== RAW ANSWER ===

To solve this problem, we need to translate Java code into C++ while maintaining identical behavior, including inputs/outputs, side effects, edge cases, and exceptions. The Java code involves managing a whitelist of addresses and handling send/receive operations with structured data. The key challenge is replicating Java's dynamic typing (using `Object`) in C++, which we achieve using `std::any` from C++17.

### Approach
1. **Data Structures**: Replace Java's `ArrayList` with `std::vector` for the whitelist and `HashMap` with `std::unordered_map` for the send and receive structures. The maps use `std::string` keys and `std::any` values to handle dynamic types (int and string).
2. **Methods**:
   - **Whitelist Management**: Methods `addWhiteList` and `delWhiteList` return pointers to the internal vector (or `nullptr` for failure), allowing direct modification as in Java.
   - **Message Handling**: 
     - `recv` checks for valid input and whitelist presence, returning `std::any` containing -1 (error), false (not whitelisted), or the received content.
     - `send` validates input and returns `nullptr` for success or an error message string.
   - **State Inspection**: `show` returns pointers to internal maps based on the requested type (send/receive) or `nullptr` for invalid types.
3. **Exception Handling**: Similar to Java, type mismatches during `std::any_cast` operations throw exceptions (translated from Java's `ClassCastException` to C++'s `std::bad_any_cast`).

- **Message Reception**: `recv` validates input parameters and checks the whitelist. It returns an `std::any` containing an error code (-1), a boolean (false if not whitelisted), or the received content (as a string).

- **State Inspection**: `show` returns a pointer to the requested internal map (send or receive) or `nullptr` for invalid types, allowing direct manipulation of the map contents as in Java.
- **Dynamic Typing**: `std::any` is used to handle dynamic types (int, bool, string) similar to Java's `Object`, with `std::any_cast` providing type-safe access (throwing `std::bad_any_cast` on type mismatches).
