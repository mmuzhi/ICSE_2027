   - **Left Brackets**: Push onto the stack.
   - **Right Brackets**: Pop the top of the stack. If the stack is empty, throw an exception (matching Python's behavior). Otherwise, check if the popped left bracket matches the current right bracket.
4. **Final Check**: After processing all characters, if the stack is not empty, return false; otherwise, return true.

### Solution Code
```cpp
#include <vector>
#include <string>
#include <stdexcept>

class BalancedBrackets {
public:
    std::vector<char> stack;
    std::string left_brackets;
    std::string right_brackets;
    std::string expr;

    BalancedBrackets(std::string expr) : expr(expr) {
        left_brackets = "({[";
        right_brackets = ")}]";
    }

    void clear_expr() {
        std::string cleaned;
        for (char c : expr) {
            if (left_brackets.find(c) != std::string::npos || 
                right_brackets.find(c) != std::string::npos) {
                cleaned += c;
            }
        }
        expr = cleaned;
    }

    bool check_balanced_brackets() {
        clear_expr();

        for (char Brkt : expr) {
            if (left_brackets.find(Brkt) != std::string::npos) {
                stack.push_back(Brkt);
            } else {
                if (stack.empty()) {
                    throw std::out_of_range("pop from empty stack");
                }
                char Current_Brkt = stack.back();
                stack.pop_back();

                if (Current_Brkt == '(') {
                    if (Brkt != ')') {
                        return false;
                    }
                }
                if (Current_Brkt == '{') {
                    if (Brkt != '}') {
                        return false;
                    }
                }
                if (Current_Brkt == '[') {
                    if (Brkt != ']') {
                        return false;
                    }
                }
            }
        }

        if (!stack.empty()) {
            return false;
        }
        return true;
    }
};
```

### Explanation
1. **Initialization**: The constructor initializes the expression and sets of left and right brackets.
2. **Cleaning Expression**: The `clear_expr` method filters out non-bracket characters, leaving only brackets in the expression.
3. **Processing Brackets**: 
   - **Left Brackets**: Each left bracket encountered is pushed onto the stack.
   - **Right Brackets**: For each right bracket, the top of the stack is popped. If the stack is empty, an exception is thrown. The popped left bracket is checked against the current right bracket for a match; mismatches return false immediately.
4. **Final Check**: After processing all characters, if the stack is empty, all brackets are balanced (returns true); otherwise, unbalanced (returns false).
