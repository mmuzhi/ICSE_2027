=== RAW ANSWER ===

To solve this problem, we need to maximize a given number by strategically changing its digits based on a provided list. The goal is to replace digits with larger digits from left to right, ensuring that once we start modifying the number, we stop if we encounter a digit that cannot be increased further.

### Approach
1. **Problem Analysis**: The task involves converting each digit of the number to a potentially larger digit from a predefined list. The conversion should be done greedily from left to right, meaning we prioritize higher digits on the left side. Once we start modifying the number, we stop if we encounter a digit that cannot be increased further (i.e., it is already greater than its corresponding value in the change list).

2. **Key Insight**: For each digit, check if it can be replaced by a larger digit from the change list. If so, replace it and continue processing subsequent digits. If a digit is already larger than its corresponding change value and we have started modifying the number, we stop further processing to ensure the number is maximized correctly.

3. **Algorithm Selection**: We traverse the digits of the number from left to right. For each digit, if it can be increased (digit < change[digit]), we replace it and mark that we are in the modification phase. If we encounter a digit that is already larger than its change value and we are in the modification phase, we stop processing further digits.

4. **Complexity Analysis**: The algorithm processes each digit exactly once, resulting in a linear time complexity O(n), where n is the number of digits in the input string. The space complexity is O(n) to store the digits as a list for processing.

### Solution Code
```python
class Solution:
    def maximumNumber(self, num: str, change: List[int]) -> str:
        num_list = list(num)
        on = False
        for i, ch in enumerate(num_list):
            x = int(ch)
            if x < change[x]:
                on = True
                num_list[i] = str(change[x])
            elif x > change[x] and on:
                break
        return "".join(num_list)
```

### Explanation
1. **Initialization**: Convert the input string into a list of characters for easy manipulation.
2. **Tracking Modification State**: The `on` flag is initialized to `False` to indicate whether we have started modifying the number.
3. **Digit Processing**: For each digit in the number:
   - Convert the character to an integer.
   - If the digit can be increased (digit < change[digit]), replace it with the corresponding value from the change list and set `on` to `True`.
   - If the digit is already larger than its change value and we are in the modification phase (`on` is `True`), break out of the loop to stop further processing.
4. **Result Construction**: Join the modified list back into a string and return it.

This approach efficiently maximizes the number by leveraging a greedy strategy, ensuring optimal performance and correctness.
