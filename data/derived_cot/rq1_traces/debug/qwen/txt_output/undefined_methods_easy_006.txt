To solve this problem, we need to compute the nth Fibonacci number efficiently using memoization to avoid recalculating the same values repeatedly. The initial code provided has two main issues: the recursive function was defined without the `self` parameter, and the memoization array was not properly initialized before recursive calls. 

### Approach
1. **Memoization Setup**: We use a list `Fib` to store computed Fibonacci numbers up to `n`. This list is initialized with `-1` to indicate uncomputed values.
2. **Base Cases**: The base cases for `n = 0` and `n = 1` are directly handled by returning `0` and `1` respectively.
3. **Recursive Calculation**: For `n >= 2`, the method checks if the value is already computed (not `-1`). If not, it recursively computes the value using the previous two Fibonacci numbers and stores it in the `Fib` list.
4. **Closure**: The inner function `fib` uses the `Fib` array from the outer method's scope, leveraging Python's closure to access the array without passing it explicitly.

### Solution Code
```python
class Solution:
    def fib(self, n: int) -> int:
        if n == 0:
            return 0
        if n == 1:
            return 1
        Fib = [-1] * (n + 1)
        Fib[0] = 0
        Fib[1] = 1
        
        def fib(n):
            if Fib[n] != -1:
                return Fib[n]
            Fib[n] = fib(n - 1) + fib(n - 2)
            return Fib[n]
        
        return fib(n)
```

### Explanation
1. **Initialization**: The `Fib` array is initialized to `-1` to mark all positions as uncomputed. The first two positions are set to `0` and `1` for the base cases.
2. **Recursive Function**: The inner function `fib` checks if the value at position `n` is already computed. If not, it computes the value by summing the previous two values, stores it in `Fib[n]`, and returns it.
3. **Efficiency**: By storing computed values, the solution avoids redundant calculations, reducing the time complexity from exponential O(2^n) to linear O(n) with space complexity O(n) due to the memoization array.

This approach ensures that each Fibonacci number is computed only once, making the solution efficient and correct for all valid inputs.
