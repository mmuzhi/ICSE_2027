=== RAW ANSWER ===

To solve this problem, we need to answer multiple queries efficiently, where each query asks for the maximum sum of elements from two arrays (`nums1` and `nums2`) such that the elements satisfy certain conditions. Specifically, for a query `[x, y]`, we need to find the maximum value of `a_i + b_i` where `a_i <= x` and `b_i >= y` for some index `i`.

### Approach
1. **Sorting and Discretization**: 
   - First, we sort the points from `nums1` and `nums2` by their `nums1` values in ascending order. This helps in efficiently processing queries by leveraging binary search.
   - We then discretize the `nums2` values (and query `y` values) to reduce the range of values we need to handle, making the subsequent range queries more efficient.

2. **Offline Query Processing**:
   - We sort the queries based on their `x` values in ascending order. This allows us to process each query in sequence, adding relevant points to our data structure as we go.

3. **Segment Tree for Range Maximum Queries**:
   - We use a segment tree (specifically, a max-heap segment tree) to keep track of the maximum `a_i + b_i` values for each discretized `b_i` value. This tree supports efficient range maximum queries, which are crucial for answering each query in logarithmic time relative to the number of distinct `b_i` values.

4. **Processing Queries**:
   - For each query, we add all points from the sorted list that have `a_i <= x` (the query's `x` value) into the segment tree. We then query the tree for the maximum `a_i + b_i` value among points with `b_i >= y`.

### Solution Code
```python
import math
from typing import List
import bisect

class Solution:
    def maximumSumQueries(self, nums1: List[int], nums2: List[int], queries: List[List[int]]) -> List[int]:
        n = len(nums1)
        points = []
        for i in range(n):
            points.append((nums1[i], nums2[i]))
        points.sort(key=lambda x: x[0])
        
        all_bs = []
        for a_val, b_val in points:
            all_bs.append(b_val)
        for query in queries:
            x, y = query
            all_bs.append(y)
        all_bs.sort()
        size_b = len(all_bs)
        def discretize(val):
            return bisect.bisect_left(all_bs, val)
        
        size = 1
        while size < size_b:
            size *= 2
        seg = [-10**18] * (2 * size)
        
        def update(pos, value):
            pos += size
            seg[pos] = max(seg[pos], value)
            while pos > 1:
                pos //= 2
                seg[pos] = max(seg[2*pos], seg[2*pos+1])
                
        def query(l, r):
            if l > r:
                return -10**18
            l += size
            r += size
            res = -10**18
            while l <= r:
                if l % 2 == 1:
                    res = max(res, seg[l])
                    l += 1
                if r % 2 == 0:
                    res = max(res, seg[r])
                    r -= 1
                l //= 2
                r //= 2
            return res
        
        sorted_queries = []
        for idx, (x, y) in enumerate(queries):
            sorted_queries.append((x, y, idx))
        sorted_queries.sort(key=lambda x: x[0])
        
        j = 0
        ans = [-1] * len(queries)
        for x, y, idx in sorted_queries:
            while j < n and points[j][0] <= x:
                b_val = points[j][1]
                s_val = points[j][0] + points[j][1]
                b_idx = discretize(b_val)
                update(b_idx, s_val)
                j += 1
            y_disc = discretize(y)
            if y_disc >= size_b:
                ans[idx] = -1
            else:
                res_val = query(y_disc, size_b - 1)
                if res_val < 0:
                    ans[idx] = -1
                else:
                    ans[idx] = res_val
        return ans
```

### Explanation
1. **Sorting and Discretization**: The points from `nums1` and `nums2` are sorted by their `nums1` values. The `nums2` values and query `y` values are discretized to a smaller range for efficient processing.
2. **Segment Tree Construction**: A segment tree is built to handle range maximum queries. The tree is updated with the maximum `a_i + b_i` values for each discretized `b_i` value.
3. **Query Processing**: Queries are processed in ascending order of `x`. For each query, relevant points (those with `a_i <= x`) are added to the segment tree. The tree is then queried for the maximum `a_i + b_i` value among points with `b_i >= y`.
4. **Efficiency**: The approach efficiently handles the constraints by leveraging sorting, discretization, and a segment tree, ensuring that each query is processed in logarithmic time relative to the number of distinct `b_i` values. This makes the solution scalable for large input sizes.
