=== RAW ANSWER ===

To solve this problem, we need to find all ancestors for each node in a directed graph. An ancestor of a node is any node that can reach it, excluding the node itself. The solution involves using a topological sorting approach (Kahn's algorithm) to process nodes in an order where each node's ancestors are known before processing its neighbors.

### Approach
1. **Graph Construction**: Build an adjacency list representation of the graph and compute the in-degree for each node.
2. **Topological Sorting**: Use a queue to process nodes with zero in-degree first. For each node processed, update the ancestor sets of its neighbors by adding the current node and all its ancestors.
3. **BFS Propagation**: Decrease the in-degree of each neighbor and push it into the queue if its in-degree becomes zero. This ensures that all ancestors of a node are processed before the node itself.
4. **Result Compilation**: After processing all nodes, convert each node's ancestor set into a sorted list to form the final result.

### Solution Code
```python
from collections import deque

class Solution:
    def getAncestors(self, n: int, edges: List[List[int]]) -> List[List[int]]:
        graph = [[] for _ in range(n)]
        indegrees = [0] * n
        
        for edge in edges:
            u, v = edge
            graph[u].append(v)
            indegrees[v] += 1
        
        ans = [set() for _ in range(n)]
        q = deque()
        
        for i in range(n):
            if indegrees[i] == 0:
                q.append(i)
                
        while q:
            cur = q.popleft()
            for v in graph[cur]:
                ans[v].add(cur)
                ans[v].update(ans[cur])
                indegrees[v] -= 1
                if indegrees[v] == 0:
                    q.append(v)
                    
        result = []
        for s in ans:
            result.append(sorted(list(s)))
            
        return result
```

### Explanation
1. **Graph Construction**: The graph is built using an adjacency list where each node points to its neighbors. The in-degree array keeps track of the number of incoming edges for each node.
2. **Topological Sorting**: Nodes with zero in-degree are initial sources. These nodes are processed first since they have no ancestors. For each node processed, its neighbors' ancestor sets are updated to include the current node and all its ancestors.
3. **BFS Propagation**: The in-degree of each neighbor is decremented. If a neighbor's in-degree reaches zero, it is added to the queue, ensuring all its ancestors are processed before it.
4. **Result Compilation**: Each node's ancestor set is converted from a set to a sorted list to provide the final result, ensuring the ancestors are listed in ascending order.

This approach efficiently computes all ancestors for each node by leveraging topological sorting and BFS, ensuring correctness and optimal performance.
