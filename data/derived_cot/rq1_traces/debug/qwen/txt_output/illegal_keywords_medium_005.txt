=== RAW ANSWER ===

To solve this problem, we need to find the minimum difference between any two elements in a sorted array. The approach involves sorting the array and then iterating through the sorted elements to compute the absolute difference between adjacent elements. The smallest of these differences is the answer.

### Approach
1. **Sort the Array**: Sorting helps in bringing the closest elements together, making it easier to find the minimum difference between adjacent elements.
2. **Iterate Through Adjacent Elements**: After sorting, the minimum difference between any two elements will be the smallest difference between any pair of adjacent elements in the sorted array.
3. **Track Minimum Difference**: Initialize a variable to keep track of the smallest difference encountered during the iteration. Update this variable whenever a smaller difference is found.

### Solution Code
```python
from typing import List

class Solution:
    def findValueOfPartition(self, nums: List[int]) -> int:
        if len(nums) == 0:
            return 0
        nums.sort()
        min_diff = float('inf')
        for i in range(1, len(nums)):
            min_diff = min(min_diff, abs(nums[i] - nums[i-1]))
        return min_diff if min_diff != float('inf') else 0
```

### Explanation
1. **Import List**: The `List` type is imported from the `typing` module to correctly annotate the input parameter as a list of integers.
2. **Handle Empty Array**: If the input array is empty, the function returns 0 as there are no elements to compare.
3. **Sort the Array**: The array is sorted to ensure that the closest elements are adjacent, allowing us to find the minimum difference efficiently.
4. **Find Minimum Difference**: The loop iterates through the sorted array, comparing each element with its predecessor. The smallest absolute difference encountered during this process is stored in `min_diff`.
5. **Return Result**: If the array has only one element, the loop doesn't execute, and the function returns 0. Otherwise, it returns the smallest difference found.

This approach efficiently leverages sorting to minimize the number of comparisons needed, ensuring optimal performance with a time complexity of O(n log n) due to the sorting step, followed by a linear scan O(n) for finding the minimum difference.
