=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays without merging them into a single array, which would take O(n) time complexity. The median is the middle value in an ordered list of numbers. If the combined length of the two arrays is even, the median is the average of the two middle numbers.

### Approach
1. **Two Pointers Technique**: We use two pointers, one for each array, to traverse both arrays simultaneously.
2. **Merging Process**: Compare elements from both arrays and merge them into a new array in sorted order. This involves:
   - Comparing the current elements of both arrays and appending the smaller one to the merged array.
   - Moving the pointer of the array from which the element was taken to the next position.
   - Appending any remaining elements from either array after one of the pointers reaches the end of its array.
3. **Finding the Median**: After merging, the median can be determined based on the length of the merged array:
   - If the length is odd, the median is the middle element.
   - If the length is even, the median is the average of the two middle elements.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        p1 = 0
        p2 = 0
        merged = []
        
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                merged.append(nums1[p1])
                p1 += 1
            else:
                merged.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            merged.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            merged.append(nums2[p2])
            p2 += 1
        
        n = len(merged)
        if n % 2 == 1:
            return float(merged[n//2])
        else:
            return (merged[n//2 - 1] + merged[n//2]) / 2.0
```

### Explanation
1. **Initialization**: We start with two pointers, `p1` and `p2`, initialized to 0, representing the start of each array. A merged list is initialized to store the combined elements.
2. **Merging Arrays**: The while loop continues as long as both pointers are within their respective array bounds. In each iteration, the smaller of the two current elements is added to the merged list, and the corresponding pointer is incremented.
3. **Remaining Elements**: After one of the pointers reaches the end of its array, any remaining elements from the other array are appended to the merged list.
4. **Median Calculation**: The median is found by checking the length of the merged list. If the length is odd, the median is the middle element. If even, it is the average of the two middle elements. The result is returned as a float to handle both cases appropriately.

This approach efficiently merges the arrays in linear time and space, ensuring optimal performance with O(n) time complexity, where n is the total number of elements in both arrays.
