The solution involves using dynamic programming with memoization to efficiently explore all possible ways to make cuts while ensuring the constraints are met.

### Approach

2. **Base Case Handling**: If the total number of apples in the entire pizza is less than `k`, it's impossible to create `k` pieces each with at least one apple, so we return 0.
3. **Dynamic Programming with Memoization**: We define a recursive function `dp(i, j, k)` that calculates the number of ways to cut the subpizza starting from position `(i, j)` to the bottom-right corner `(n-1, m-1)` into `k` pieces. The function checks for horizontal and vertical cuts, ensuring each resulting piece has at least one apple and the remaining subpizza can be cut into `k-1` pieces.
4. **Memoization**: To optimize the recursive calls, we use memoization to store already computed states, avoiding redundant calculations and improving efficiency.

                right = get_apples(i, y+1, n-1, m-1)
