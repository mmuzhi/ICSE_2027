To solve this problem, we need to find the value of the bottom-leftmost node in a binary tree. The bottom-leftmost node is defined as the leftmost node at the maximum depth (deepest level) of the tree. 

### Approach
The approach involves performing a breadth-first search (BFS) traversal of the tree. During the traversal, we keep track of the depth of each node. Whenever we encounter a node at a greater depth than the current maximum depth, we update the maximum depth and record the node's value as the potential answer. This ensures that by the end of the traversal, we have the leftmost node of the deepest level.

1. **Initialization**: Start by checking if the root is empty, in which case return None.
2. **BFS Traversal**: Use a queue to traverse the tree level by level. Each element in the queue is a tuple containing the current node and its depth.
3. **Depth Tracking**: For each node dequeued, check if its depth is greater than the previously recorded maximum depth. If so, update the maximum depth and set the answer to the node's value.
4. **Child Processing**: Enqueue the left and right children of the current node with their respective depths (current depth + 1).
5. **Result**: After processing all nodes, the answer will be the value of the leftmost node at the deepest level.

### Solution Code
```python
import collections

class Solution:
    def findBottomLeftValue(self, root: Optional[TreeNode]) -> int:
        if not root:
            return None
        queue = collections.deque([(root, 0)])
        max_depth = -1
        ans = None
        while queue:
            node, depth = queue.popleft()
            if depth > max_depth:
                max_depth = depth
                ans = node.val
            if node.left:
                queue.append((node.left, depth + 1))
            if node.right:
                queue.append((node.right, depth + 1))
        return ans
```

1. **Initialization**: The function starts by checking if the input root is None, returning None if true.

3. **BFS Loop**: The loop processes each node in the queue. For each node, it checks if the node's depth is greater than the current maximum depth. If so, it updates the maximum depth and sets the answer to the node's value.
4. **Child Enqueue**: The left and right children of the current node are enqueued with their depths incremented by 1.
5. **Result Extraction**: After processing all nodes, the answer variable holds the value of the leftmost node at the deepest level, which is returned as the result.

This approach efficiently tracks the deepest level and the leftmost node during BFS traversal, ensuring optimal performance and correctness.
