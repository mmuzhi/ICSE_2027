To solve this problem, we need to find the closest cost to a target value by combining base costs with toppings, where each topping can be used at most twice. The solution involves dynamic programming with memoization to explore all possible combinations of toppings efficiently.

### Approach
1. **Problem Analysis**: The goal is to determine the minimum absolute difference between a total cost (base cost plus any combination of toppings) and the target value. Each topping can be used up to twice, so we duplicate the list of topping costs to simulate this constraint.
2. **Dynamic Programming with Memoization**: We use a recursive function with memoization to explore all possible sums of toppings starting from each index. The function checks if skipping or taking the current topping (up to two times) yields a better result based on the absolute difference from the target value.
3. **Base Case Handling**: If we run out of toppings or the remaining target value becomes negative, we return 0 as no further toppings can be added.
4. **Combining Results**: For each base cost, we compute the closest total cost by combining it with the best topping combination found by the recursive function. The overall answer is the minimum total cost across all base costs, considering the smallest absolute difference from the target.

class Solution:
    def closestCost(self, baseCosts: List[int], toppingCosts: List[int], target: int) -> int:
        toppingCosts *= 2
        
        from functools import cache
        
        @cache
        def fn(i, x):
            if i == len(toppingCosts) or x < 0:
                return 0
            option1 = fn(i+1, x)
            option2 = toppingCosts[i] + fn(i+1, x - toppingCosts[i])
            return min(option1, option2, key=lambda y: (abs(y - x), y))
        
        ans = float('inf')
        for bc in baseCosts:
            total = bc + fn(0, target - bc)
            ans = min(ans, total, key=lambda x: (abs(x - target), x))
        return ans
