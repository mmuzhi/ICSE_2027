To solve this problem, we need to find the complement of a given non-negative integer. The complement is obtained by flipping all the bits in the binary representation of the number. The solution involves converting the number into its binary string representation, reversing it to handle the least significant bit first, flipping each bit, and then converting the resulting string back to an integer.

### Approach
1. **Handle the Zero Case**: If the input number is zero, its complement is also zero.
2. **Build the Binary String**: For non-zero numbers, process each bit of the number by repeatedly dividing by 2. If the remainder is 1, append '1'; otherwise, append '0'. This builds the binary string in reverse order (least significant bit first).
3. **Reverse the String**: Reverse the binary string to get the usual order (most significant bit first).
4. **Flip the Bits**: Iterate through each bit in the reversed string and flip '0' to '1' and '1' to '0'.
5. **Convert to Integer**: Convert the flipped binary string back to an integer.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        if num == 0:
            return 0
        com = ''
        while num > 0:
            if num % 2 == 1:
                com += '1'
            else:
                com += '0'
            num = num // 2
        com = com[::-1]
        flipped_com = ''.join('0' if bit == '1' else '1' for bit in com)
        return int(flipped_com, 2)
```

### Explanation
1. **Zero Handling**: The function immediately returns 0 if the input number is zero, as the complement of zero is zero.
2. **Binary String Construction**: The loop processes each bit of the number by checking the remainder when divided by 2. If the remainder is 1, it appends '1'; otherwise, it appends '0'. This constructs the binary string in reverse order.
3. **String Reversal**: The binary string is reversed to convert it from least significant bit first to most significant bit first, which is the standard binary representation.
4. **Bit Flipping**: Each bit in the reversed string is flipped (0 becomes 1 and 1 becomes 0).
5. **Integer Conversion**: The flipped binary string is converted back to an integer using base 2, which gives the desired complement of the original number.

This approach efficiently handles the complement calculation by leveraging bit manipulation and string operations, ensuring correctness for all non-negative input values.
