To solve this problem, we need to count the number of strictly increasing or decreasing sequences of digits (from 0 to 9) of length `n` (where `n` is the length of the input string `s` plus one) that satisfy the given pattern in `s`. Each character in `s` dictates whether the next digit in the sequence should be strictly increasing ('I') or strictly decreasing ('D') compared to the previous digit.

### Approach
1. **Dynamic Programming with Memoization**: We use a depth-first search (DFS) approach with memoization to efficiently count the number of valid sequences. The state in our DFS is defined by the current index in the string `s` and the last digit used in the sequence.
2. **State Transition**: For each character in `s`, if it is 'I', the next digit must be greater than the last digit. If it is 'D', the next digit must be less than the last digit. We explore all possible digits that satisfy the condition and sum the counts from these explorations.
3. **Modulo Operation**: Since the result can be very large, we use modulo \(10^9 + 7\) to keep the numbers manageable during the computation.

### Solution Code
```python
from functools import lru_cache

class Solution:
    def numPermsDISequence(self, s: str) -> int:
        n = len(s) + 1
        if n == 1:
            return 10
        
        @lru_cache(maxsize=None)
        def dfs(i, last_digit):
            if i == n:
                return 1
            total = 0
            if s[i-1] == 'D':
                for next_digit in range(0, last_digit + 1):
                    total = (total + dfs(i+1, next_digit)) % (10**9+7)
            else:
                for next_digit in range(last_digit + 1, 10):
                    total = (total + dfs(i+1, next_digit)) % (10**9+7)
            return total
        
        return dfs(1, 9) % (10**9+7)
```

1. **Base Case Handling**: If the sequence length `n` is 1, there are 10 possible digits (0 through 9), so we return 10 directly.
2. **DFS with Memoization**: The `dfs` function is defined with memoization to store intermediate results. It takes the current index `i` and the last digit used `last_digit`.
3. **Transition Logic**: For each position `i` (adjusted for zero-based indexing in `s`), if the character is 'D', the next digit must be between 0 and `last_digit`. If the character is 'I', the next digit must be between `last_digit + 1` and 9.
4. **Modulo Operation**: The results are continuously taken modulo \(10^9 + 7\) to handle large numbers and prevent overflow.
