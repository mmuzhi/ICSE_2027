```python
            if abs(total - target) < abs(ans - target):
                ans = total
            elif abs(total - target) == abs(ans - target) and total < ans:
                ans = total
        return ans
```

### Explanation
1. **Doubling Toppings**: The list of topping costs is doubled to simulate the two uses allowed per topping. This transforms the problem into selecting a subset of the new list without repetition.
2. **Memoization**: The `fn` function uses memoization to store results of subproblems (i, x), where `i` is the current index in the doubled list and `x` is the remaining target value. This avoids redundant calculations and significantly improves efficiency.
3. **Recursive Exploration**: The function explores two options at each step: skipping the current topping or using it once. The result is the combination that minimizes the absolute difference between the sum of selected toppings and the remaining target.
4. **Base Case Handling**: If the remaining target `x` becomes negative or all toppings are considered, the function returns 0, indicating no further toppings can be added.
5. **Result Calculation**: For each base cost, the function calculates the minimal total cost (base + toppings) by calling `fn(0, target - bc)`. The result is the minimal absolute difference from the target across all base costs, with ties broken by selecting the smaller total cost. 

This approach efficiently narrows down the solution space using dynamic programming and memoization, ensuring optimal performance even for larger input sizes.
