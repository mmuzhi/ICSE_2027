=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays. The median is the middle value in an ordered list of numbers. If the combined length of the arrays is even, the median is the average of the two middle numbers. 

### Approach
The approach involves merging the two sorted arrays into a single sorted array and then determining the median based on the length of this merged array. Here's a step-by-step breakdown:

1. **Merge the Arrays**: Use two pointers to traverse both arrays simultaneously, comparing elements and appending the smaller one to a new merged array. This ensures the merged array remains sorted.
2. **Handle Remaining Elements**: After one array is exhausted, append the remaining elements of the other array to the merged array.
3. **Calculate the Median**: If the merged array has an even number of elements, compute the average of the two middle elements. If the count is odd, the median is the middle element.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        p1 = 0
        p2 = 0
        new = []
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
        
        n = len(new)
        if n % 2 == 0:
            index = n // 2
            median = (new[index - 1] + new[index]) / 2.0
        else:
            median = new[n // 2]
        return median
```

### Explanation
1. **Merging Arrays**: The code uses two pointers (`p1` and `p2`) to traverse `nums1` and `nums2`. The smaller element between the current pointers is appended to the new array, and the corresponding pointer is incremented. This continues until one of the arrays is exhausted.
2. **Appending Remaining Elements**: Once one array is fully traversed, the remaining elements of the other array are appended to the new array in their original order, maintaining the sorted property.
3. **Finding the Median**: The median is determined by checking if the length of the merged array is even or odd. For an even length, the average of the two middle elements is taken. For an odd length, the middle element is selected directly.

This approach efficiently merges the arrays in linear time, O(m + n), where m and n are the lengths of the input arrays, ensuring optimal performance.
