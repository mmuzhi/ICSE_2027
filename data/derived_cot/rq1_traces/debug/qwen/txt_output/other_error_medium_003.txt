To solve this problem, we need to determine the maximum consecutive integer starting from 1 that can be formed using the given coins, where each coin can be used only once. The solution involves a greedy approach where we sort the coins and iterate through them, ensuring that we can form all consecutive integers up to a certain point.

### Approach
1. **Sort the Coins**: Sorting helps in systematically checking each coin in ascending order, which is crucial for the greedy approach.
2. **Initialize Current Maximum**: Start with a current maximum value (initially 0) representing the highest consecutive integer that can be formed so far.
3. **Iterate Through Coins**: For each coin, check if it can be used to extend the current maximum consecutive sequence. Specifically, if the coin value is greater than the current maximum plus one, it means we cannot form the next consecutive integer, and we should break the loop. Otherwise, add the coin value to the current maximum.
4. **Return Result**: The result is the current maximum plus one, which represents the next integer in the consecutive sequence that cannot be formed.

This approach efficiently checks the possibility of forming consecutive integers by leveraging the sorted order of coins and the greedy property that smaller coins are more beneficial for forming consecutive sequences.

### Solution Code
```python
class Solution:
    def getMaximumConsecutive(self, coins: List[int]) -> int:
        coins_sorted = sorted(coins)
        ans = 0
        for coin in coins_sorted:
            if coin > ans + 1:
                break
            ans += coin
        return ans + 1
```

### Explanation
1. **Sorting the Coins**: The coins are sorted to ensure we process them in ascending order, which allows us to use smaller coins first for forming consecutive integers.
2. **Initialization**: The variable `ans` is initialized to 0, representing that initially, no consecutive integers can be formed.
3. **Iteration and Condition Check**: For each coin, we check if it is greater than `ans + 1`. If it is, we break the loop because we cannot form the next consecutive integer. Otherwise, we add the coin's value to `ans`, extending the range of consecutive integers we can form.
4. **Result Calculation**: The final result is `ans + 1`, which is the next integer after the last consecutive integer we could form. For example, if we can form integers up to 10, the result will be 11, indicating that 11 is the next integer that cannot be formed.

This approach efficiently determines the maximum consecutive integers that can be formed by leveraging sorting and a greedy strategy, ensuring optimal performance with a time complexity of O(n log n) due to the sorting step.
