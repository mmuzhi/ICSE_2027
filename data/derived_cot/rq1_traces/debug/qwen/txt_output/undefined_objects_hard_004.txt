To solve this problem, we need to find the minimum number of moves required to transform the given board configuration into the target configuration `[[1, 2, 3], [4, 5, 0]]` using a sliding puzzle mechanism. The puzzle allows moving the empty space (0) in four directions: up, down, left, or right. The solution involves using a Breadth-First Search (BFS) algorithm to explore all possible states level by level, ensuring the shortest path is found first.

### Approach
1. **Problem Analysis**: The sliding puzzle problem is a classic BFS problem where each state is represented by the current configuration of the board. The goal is to transition from the initial state to the target state by moving the empty space (0) in valid directions.
2. **BFS Initialization**: Start from the initial board state and explore all possible neighboring states (generated by moving the empty space in four directions) level by level. This ensures the first time we reach the target state, it is via the shortest path.
3. **State Tracking**: Use a queue to manage the BFS process and a set to keep track of visited states to avoid cycles and redundant processing.
4. **Neighbor Generation**: For each state, locate the empty space (0) and generate all valid neighboring states by moving the empty space in the four possible directions (up, down, left, right) if the move is within the board boundaries.

### Solution Code
```python
from collections import deque
from typing import List

class Solution:
    def slidingPuzzle(self, board: List[List[int]]) -> int:
        def get_neighbors(board):
            neighbors = []
            r, c = 0, 0
            for i in range(2):
                for j in range(3):
                    if board[i][j] == 0:
                        r, c = i, j
            for di, dj in [(0, 1), (1, 0), (0, -1), (-1, 0)]:
                ni, nj = r + di, c + dj
                if 0 <= ni < 2 and 0 <= nj < 3:
                    new_board = [row[:] for row in board]
                    new_board[r][c], new_board[ni][nj] = new_board[ni][nj], 0
                    neighbors.append(new_board)
            return neighbors

        queue = deque()
        queue.append((board, 0))
        seen = set()
        seen.add(tuple(tuple(row) for row in board))

        while queue:
            board, moves = queue.popleft()
            if board == [[1, 2, 3], [4, 5, 0]]:
                return moves
            for neighbor in get_neighbors(board):
                if tuple(tuple(row) for row in neighbor) not in seen:
                    seen.add(tuple(tuple(row) for row in neighbor))
                    queue.append((neighbor, moves + 1))
        return -1
```

### Explanation
1. **get_neighbors Function**: This helper function generates all valid neighboring states by moving the empty space (0) in four possible directions. It first locates the empty space and then checks each direction to see if the move is within bounds. For each valid move, it creates a new board configuration by swapping the empty space with the adjacent tile.
2. **BFS Initialization**: The BFS starts from the initial board state, stored in a queue along with the number of moves taken to reach it (0 initially). A set is used to keep track of visited states to prevent revisiting and cycles.
3. **BFS Execution**: For each state dequeued, the code checks if it matches the target state. If it does, the number of moves taken is returned immediately. Otherwise, all valid neighboring states are generated, checked for prior visitation, and enqueued with an incremented move count.
4. **Termination**: If the queue is exhausted without finding the target state, the function returns -1, indicating the target state is unreachable from the initial state. The BFS guarantees the shortest path is found first due to its level-order traversal nature.
