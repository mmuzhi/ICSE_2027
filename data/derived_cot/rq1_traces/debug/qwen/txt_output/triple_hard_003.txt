To solve this problem, we need to count the number of arithmetic slices in an array. An arithmetic slice is a contiguous subarray where the difference between consecutive elements is constant. 

### Approach
The approach involves using dynamic programming to efficiently count these slices. The key insight is that for any element at index `i`, the number of arithmetic slices ending at `i` can be derived from the slices ending at the previous element. Specifically, if the difference between the current element and the previous element is the same as the difference between the previous element and the element before that, then the number of new slices ending at `i` is incremented by the number of slices ending at `i-1` plus one. 

1. **Dynamic Programming Array Initialization**: We initialize a dynamic programming array `dp` where `dp[i]` will store the number of arithmetic slices ending at index `i`.
2. **Base Case Handling**: For the first two elements, there are no slices of length at least 2, so we skip them.
3. **Iterate Through the Array**: For each index `i` starting from 2, check if the difference between `nums[i]` and `nums[i-1]` is the same as the difference between `nums[i-1]` and `nums[i-2]`. If true, then `dp[i]` is set to `dp[i-1] + 1`; otherwise, it is set to 0.
4. **Summing Up Slices**: The total number of arithmetic slices is the sum of all values in the `dp` array.

### Solution Code
```python
class Solution:
    def numberOfArithmeticSlices(self, nums: List[int]) -> int:
        n = len(nums)
        if n < 3:
            return 0
        dp = [0] * n
        dp[2] = 1 if nums[2] - nums[1] == nums[1] - nums[0] else 0
        total = dp[2]
        for i in range(3, n):
            if nums[i] - nums[i-1] == nums[i-1] - nums[i-2]:
                dp[i] = dp[i-1] + 1
            else:
                dp[i] = 0
            total += dp[i]
        return total
```

### Explanation
1. **Initialization**: The `dp` array is initialized with zeros. The first two elements are skipped since they cannot form slices of length at least 2.
2. **Base Case**: The third element (`dp[2]`) is set to 1 if the first three elements form an arithmetic slice; otherwise, it is 0.
3. **Iteration**: For each subsequent element, the algorithm checks if the current element continues the arithmetic sequence from the previous elements. If it does, the number of new slices ending at this element is incremented by the number of slices ending at the previous element plus one. If not, it resets to zero.
4. **Summing**: The total count is accumulated by summing up all values in the `dp` array, which gives the total number of arithmetic slices in the array.

This approach efficiently counts all arithmetic slices in linear time, making it optimal for large input arrays.
