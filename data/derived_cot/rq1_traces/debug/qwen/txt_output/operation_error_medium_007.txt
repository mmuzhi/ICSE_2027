To solve this problem, we need to determine the minimum eating speed required for a bee to finish all piles of berries within a given number of hours. The solution involves using binary search to efficiently find the optimal eating speed.

### Approach
1. **Binary Search Setup**: The solution leverages binary search over the possible eating speeds, ranging from 1 to the maximum pile size. This approach efficiently narrows down the search space to find the minimal speed that meets the requirement.
2. **Feasibility Check**: For each candidate speed during the binary search, we check if the total time required to eat all piles at that speed is within the given hours. The total time is calculated by summing the ceiling of each pile divided by the speed, which gives the hours needed per pile.
3. **Binary Search Execution**: The binary search adjusts the search range based on whether the current speed is feasible. If the current speed is too slow (i.e., the total time exceeds the available hours), we increase the lower bound. Otherwise, we decrease the upper bound to find a potentially faster speed.

### Solution Code
```python
class Solution:
    def minEatingSpeed(self, piles: List[int], h: int) -> int:
        def check(x):
            total = 0
            for ele in piles:
                total += (ele + x - 1) // x
            return total <= h

        l = 1
        r = max(piles)
        while l < r:
            mid = (l + r) // 2
            if not check(mid):
                l = mid + 1
            else:
                r = mid
        return l
```

### Explanation
1. **Binary Search Initialization**: The binary search starts with `l` set to 1 (minimum possible eating speed) and `r` set to the maximum pile size (maximum possible eating speed).
2. **Midpoint Calculation**: In each iteration, the midpoint `mid` is calculated as the average of `l` and `r`.
3. **Feasibility Check**: The `check` function calculates the total time required to eat all piles at speed `mid`. For each pile, it computes the ceiling of the division by `mid` using integer arithmetic to avoid floating-point operations.
4. **Adjusting Search Range**: If the total time exceeds `h`, the search continues in the upper half by setting `l` to `mid + 1`. Otherwise, the search continues in the lower half by setting `r` to `mid`.
5. **Result**: The loop exits when `l` equals `r`, which is the minimal eating speed that allows the bee to finish all piles within `h` hours.

This approach efficiently narrows down the solution using binary search, ensuring optimal performance even for large input sizes.
