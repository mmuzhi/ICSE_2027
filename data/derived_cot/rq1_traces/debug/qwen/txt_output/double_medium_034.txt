=== RAW ANSWER ===

To solve this problem, we need to generate a list of numbers from 1 to n in lexicographical order. Lexicographical order for numbers is determined by comparing their string representations. For example, the number 10 comes before 2 because the string "10" is lexicographically smaller than "2".

### Approach
The approach involves the following steps:
1. **Convert Numbers to Strings**: Convert each number from 1 to n into its string representation. This allows us to compare the numbers lexicographically.
2. **Sort the Strings**: Sort the list of strings lexicographically. This ensures that numbers with more digits come after those with fewer digits if they start with the same digits.
3. **Convert Back to Integers**: Convert the sorted strings back into integers to form the final result list.

This approach efficiently leverages Python's built-in sorting on strings, which naturally handles lexicographical order. The time complexity is O(n log n) due to the sorting step, which is optimal for this problem given the constraints.

### Solution Code
```python
class Solution:
    def lexicalOrder(self, n: int) -> List[int]:
        lst = [str(i) for i in range(1, n+1)]
        lst.sort()
        return [int(x) for x in lst]
```

### Explanation
1. **String Conversion**: The list comprehension `[str(i) for i in range(1, n+1)]` converts each number from 1 to n into a string. This allows us to use lexicographical comparison.
2. **Sorting**: The `sort()` method sorts the strings in lexicographical order. For example, "1" comes before "10" because "1" is a prefix of "10" and shorter strings come first in lexicographical order.
3. **Integer Conversion**: The sorted strings are then converted back to integers using another list comprehension, resulting in the desired lexicographical order of numbers.

This method efficiently handles the problem by utilizing Python's string operations and sorting capabilities, ensuring correctness and simplicity.
