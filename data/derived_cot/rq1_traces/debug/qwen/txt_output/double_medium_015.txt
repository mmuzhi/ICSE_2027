To solve this problem, we need to compute the sum of absolute differences for each element in an array with every other element. The solution involves efficiently calculating these sums using prefix sums to avoid a brute-force approach, which would be too slow for large arrays.

### Approach
1. **Problem Analysis**: The task is to compute for each element in the array the sum of absolute differences between that element and all other elements. A brute-force approach would involve comparing each element with every other element, resulting in an O(n^2) time complexity, which is inefficient for large arrays.

2. **Intuition**: By leveraging prefix sums, we can break down the problem into two parts for each element: the sum of absolute differences with elements to its left and the sum with elements to its right. For a sorted array (in non-decreasing order), the left part will always be less than or equal to the current element, and the right part will always be greater than or equal to the current element. This allows us to use the prefix sums to compute these sums efficiently.

3. **Algorithm Selection**: 
   - **Prefix Sum Array**: Compute a prefix sum array where each element at index `i` is the sum of all elements before `i` in the original array.
   - **Left and Right Sums**: For each element at index `i`, the left sum is calculated as `nums[i] * i - prefix[i]`, and the right sum is calculated as `right_sum - nums[i] * (n - i - 1)`, where `right_sum` is the sum of elements to the right of `i`.

4. **Complexity Analysis**: The algorithm runs in O(n) time for both the prefix sum calculation and the sum of absolute differences computation, making it efficient with O(n) space complexity.

class Solution:
    def getSumAbsoluteDifferences(self, nums: List[int]) -> List[int]:
        n = len(nums)
        total = sum(nums)
        if n == 0:
            return []

            if i < n-1:
                right_sum = total - prefix[i] - nums[i]

            else:
                right_abs = 0
            result.append(left_abs + right_abs)
        return result

### Explanation
1. **Prefix Sum Calculation**: The prefix array is constructed such that each element at index `i` contains the sum of all elements from the start of the array up to, but not including, the element at `i`. This allows quick access to the sum of any subarray starting from the beginning.

2. **Left and Right Sum Calculation**: For each element at index `i`, the left sum is derived from the prefix sum. The right sum is calculated by subtracting the sum of elements up to `i` (including `i`) from the total sum of the array. 

3. **Combining Results**: The total sum of absolute differences for each element is the sum of the left and right sums. This approach efficiently computes the result in linear time by leveraging the properties of prefix sums and the sorted nature of the array.

This solution assumes the input array is sorted in non-decreasing order. If the array is not sorted, the results may not be accurate, as the left and right sums rely on the order of elements.
