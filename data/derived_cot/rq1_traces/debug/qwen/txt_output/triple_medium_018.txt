To solve this problem, we need to perform a level-order traversal of a binary tree and return the nodes' values in a bottom-up manner. The initial code provided has several bugs that need to be addressed, including incorrect variable usage and improper handling of child nodes.

### Approach
1. **Level-Order Traversal (BFS)**: We use a queue to traverse the tree level by level. Starting from the root node, we process each node, adding its children to the queue in left-to-right order.
2. **Handling Empty Tree**: If the tree is empty (root is None), we return an empty list.
3. **Reversing Levels**: After collecting all levels from top to bottom, we reverse the list of levels to present them from bottom to top.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans)
            while l <= r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root:
            return []
        q, ans = [root], []
        while q:
            n = len(q)
            level_vals = []
            for i in range(n):
                curr = q.pop(0)
                level_vals.append(curr.val)
                if curr.left is not None:
                    q.append(curr.left)
                if curr.right is not None:
                    q.append(curr.right)
            ans.append(level_vals)
        return checkReverse(ans)
```

### Explanation
1. **Initialization**: The function starts by defining a helper function `checkReverse` to reverse the list of levels.
2. **Empty Tree Check**: If the root is None, the function returns an empty list.
3. **Queue Processing**: Using a queue, we process each node level by level. For each node, we append its value to the current level's list and its children (if they exist) to the queue.
4. **Level Collection**: After processing all nodes at the current level, the level's values are added to the result list.
5. **Reversing Levels**: The collected levels (from top to bottom) are reversed using `checkReverse` to present the result from bottom to top.

This approach ensures that we efficiently traverse the tree level by level and return the nodes' values in the desired bottom-up order.
