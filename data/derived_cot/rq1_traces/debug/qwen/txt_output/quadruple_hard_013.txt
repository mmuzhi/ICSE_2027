=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays efficiently. The median is the middle value in an ordered list of numbers. If the combined length of the two arrays is even, the median is the average of the two middle numbers; otherwise, it is the middle number.

### Approach
1. **Merge the Arrays**: We merge the two sorted arrays into a single sorted array by comparing elements from both arrays and appending the smaller element to the result array. This ensures that the merged array is sorted.
2. **Handle Edge Cases**: If both input arrays are empty, the result should be 0.0 since the median of an empty set is undefined.
3. **Calculate the Median**: After merging, if the length of the merged array is even, the median is the average of the two middle elements. If the length is odd, the median is the middle element.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        p1 = 0
        p2 = 0
        new = []
        
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
        
        n = len(new)
        if n == 0:
            return 0.0
        
        if n % 2 == 0:
            index = n // 2
            median = (new[index] + new[index-1]) / 2
        else:
            median = float(new[n//2])
        
        return median
```

### Explanation
1. **Merging Arrays**: The code starts by initializing two pointers, `p1` and `p2`, to the start of each array. It then iterates through both arrays, comparing elements and appending the smaller one to the result array `new`. This ensures the merged array is sorted.
2. **Appending Remaining Elements**: After one of the arrays is exhausted, the remaining elements of the other array are appended to `new`.
3. **Edge Case Handling**: If the merged array is empty (both input arrays were empty), the function returns 0.0.
4. **Median Calculation**: The median is calculated based on the length of the merged array. For an even length, it averages the two middle elements; for an odd length, it takes the middle element. The result is returned as a float.

This approach efficiently merges the arrays in linear time and calculates the median in constant time, making it suitable for large input sizes.
