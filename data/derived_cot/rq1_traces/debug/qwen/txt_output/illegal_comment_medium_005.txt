=== RAW ANSWER ===

To solve this problem, we need to find the k-th smallest positive factor of a given integer `n`. The solution involves iterating through all integers from 1 to `n`, checking if each integer is a factor of `n`, and counting these factors until we reach the k-th factor.

### Approach
1. **Initialization**: Start by initializing a counter `j` to zero. This counter will keep track of the number of factors found.
2. **Iteration**: Loop through all integers from 1 to `n` (inclusive). For each integer `i`:
   - **Check Divisibility**: If `n` is divisible by `i`, increment the counter `j`.
   - **Check for k-th Factor**: If the counter `j` reaches `k`, immediately return `i` as it is the k-th factor.
3. **Completion**: If the loop completes without finding `k` factors, return -1 indicating that there are fewer than `k` factors.

This approach efficiently checks each integer in ascending order, ensuring the k-th factor is found in its correct position. The loop breaks early once the k-th factor is identified, optimizing performance for large `n` and `k`.

### Solution Code
```python
class Solution:
    def kthFactor(self, n: int, k: int) -> int:
        j = 0
        for i in range(1, n + 1):
            if n % i == 0:
                j += 1
                if j == k:
                    return i
        return -1
```

### Explanation
- **Initialization**: The counter `j` starts at 0 to count the number of factors found.
- **Loop Through Integers**: The loop runs from 1 to `n`, checking each integer `i` to see if it divides `n` without a remainder.
- **Factor Counting**: For each divisor `i`, the counter `j` is incremented. If `j` reaches `k`, the function returns `i` immediately, as it is the k-th factor.
- **Early Termination**: The loop exits as soon as the k-th factor is found, ensuring efficiency. If the loop completes without finding `k` factors, the function returns -1.

This approach efficiently finds the k-th factor by leveraging simple iteration and early termination, making it both correct and optimal for the given problem constraints.
