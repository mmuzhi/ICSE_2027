=== RAW ANSWER ===

To solve this problem, we need to determine for each element in an array how many numbers are strictly smaller than it. The solution involves counting the frequency of each number in the array and then using these counts to efficiently compute the required values for each element.

### Approach
1. **Frequency Counting**: We first count the occurrences of each number in the array. This is done using an array `count` of size 101 (indices 0 to 100), where each index represents a number, and the value at each index represents the count of that number in the input array.
2. **Cumulative Sum Calculation**: For each number in the input array, the number of elements smaller than it is the sum of the counts of all numbers from 0 up to (but not including) the current number. This sum is computed by summing the elements of the `count` array from index 0 to `num-1`.

### Solution Code
```python
class Solution:
    def smallerNumbersThanCurrent(self, nums: List[int]) -> List[int]:
        count = [0] * 101
        res = []
        for num in nums:
            count[num] += 1
        for num in nums:
            res.append(sum(count[:num]))
        return res
```

### Explanation
1. **Frequency Counting**: The loop iterates through each number in the input array `nums` and increments the corresponding index in the `count` array. For example, if the number 3 is encountered, the value at `count[3]` is incremented by 1.
2. **Cumulative Sum Calculation**: For each number in `nums`, the solution calculates the sum of all counts from index 0 to `num-1` in the `count` array. This sum represents the number of elements in `nums` that are strictly smaller than the current number. The result for each number is appended to the list `res`.
3. **Return Result**: The list `res` is returned, containing the count of numbers smaller than each element in the input array for every position.

This approach efficiently leverages the frequency array to avoid recalculating the count of smaller numbers for each element, thus optimizing both time and space complexity. The time complexity is O(n + 101), where n is the number of elements in the input array, and the space complexity is O(1) due to the fixed-size `count` array.
