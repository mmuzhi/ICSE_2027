=== RAW ANSWER ===

To solve this problem, we need to compute the sum of the minimum values of all possible subarrays in a given array. The challenge lies in efficiently calculating this sum without resorting to a brute-force approach, which would be computationally expensive.

### Approach
The efficient approach involves using a monotonic stack to determine the boundaries for each element where it is the minimum in its subarrays. Here's the step-by-step breakdown:

1. **Left Boundaries Calculation**: For each element in the array, find the nearest previous element that is smaller than the current element. This is done using a stack where elements are popped until a smaller element is found.
2. **Right Boundaries Calculation**: Similarly, for each element, find the nearest next element that is smaller than the current element by traversing the array from right to left and using a stack.
3. **Sum Calculation**: For each element, multiply its value by the number of subarrays where it is the minimum. The number of such subarrays is determined by the distances to the left and right boundaries. The total sum is then computed modulo \(10^9 + 7\).

This approach ensures that each element's contribution is calculated in linear time, making the solution efficient with a time complexity of \(O(n)\) and space complexity of \(O(n)\).

1. **Left Boundaries Calculation**: The stack keeps track of indices of elements in non-decreasing order. 2. **Right Boundaries Calculation**: The array is traversed from right to left. The stack helps find the next element that is smaller than the current element, marking the right boundary. This defines the end point where the current element remains the minimum.
3. **Sum Calculation**: For each element, the number of subarrays where it is the minimum is the product of the distances to its left and right boundaries. Multiplying this by the element's value and summing all contributions gives the total sum, which is returned modulo \(10^9 + 7\).
