=== RAW ANSWER ===

To solve this problem, we need to determine the number of seats booked for each flight from 1 to n based on a list of bookings. Each booking specifies a range of flights (from flight l to flight r) and the number of seats to book. The challenge is to efficiently compute the total seats booked for each flight using a difference array technique.

### Approach
1. **Difference Array Technique**: This technique is used to efficiently apply range updates. We initialize an array `arr` of size `n+1` (to handle 1-based indexing) with zeros.
2. **Marking Bookings**: For each booking `[l, r, seats]`, we:
   - Add `seats` at index `l-1` (since flights are 1-indexed, we convert to 0-indexed).
   - Subtract `seats` at index `r` (to cancel the effect after the last flight in the range).
3. **Prefix Sum Calculation**: After processing all bookings, the prefix sum of the array (from index 0 to n-1) gives the total seats booked for each flight. This is because the difference array technique ensures that the prefix sum at any index `i` (0-indexed) represents the cumulative seats booked up to flight `i+1`.

### Solution Code
```python
class Solution:
    def corpFlightBookings(self, bookings: List[List[int]], n: int) -> List[int]:
        arr = [0] * (n + 1)
        for lv, ar, seats in bookings:
            arr[lv - 1] += seats
            arr[ar] -= seats
        
        res = []
        current = 0
        for i in range(n):
            current += arr[i]
            res.append(current)
        return res
```

### Explanation
1. **Initialization**: We start by creating an array `arr` of size `n+1` initialized to zero. This array will help us mark the start and end of each booking range.
2. **Processing Bookings**: For each booking `[l, r, seats]`:
   - We add `seats` to `arr[l-1]` to indicate the start of the booking.
   - We subtract `seats` from `arr[r]` to indicate the end of the booking (the effect stops after flight `r`).
3. **Prefix Sum Calculation**: By iterating through the first `n` elements of `arr` and maintaining a running sum (`current`), we compute the total seats booked for each flight. The result list `res` contains the cumulative sum after each iteration, which directly gives the number of seats booked for each flight from 1 to n.

This approach efficiently handles range updates and queries in linear time, making it optimal for large input sizes.
