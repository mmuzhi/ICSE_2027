To solve this problem, we need to determine if a given string containing '(', ')', and '*' characters is valid. The string is considered valid if all opening brackets '(' are properly closed by corresponding closing brackets ')', and any '*' can be treated as either '(', ')' or an empty string to help balance the parentheses.

### Approach
The approach involves using two counters, `low` and `high`, to keep track of the minimum and maximum number of unmatched opening brackets at each point while traversing the string. The `low` counter ensures that there are enough closing brackets to match the opening brackets, and the `high` counter prevents the number of opening brackets from exceeding the available closing brackets. 

1. **Initialization**: Start with `low` and `high` both set to 0.
2. **Traversal**: For each character in the string:
   - If the character is '(', increment both `low` and `high` by 1.
   - If the character is '*', increment `high` by 1 and decrement `low` by 1 if `low` is greater than 0 (treat '*' as a closing bracket). This step accounts for the flexibility of '*' to act as either an opening or closing bracket.
   - If the character is ')', decrement both `low` and `high` by 1. If `high` becomes negative at any point, the string is invalid.
3. **Validation**: After processing all characters, if `low` is 0, the string is valid; otherwise, it is invalid.

### Solution Code
```python
class Solution:
    def checkValidString(self, s):
        low = 0
        high = 0
        for c in s:
            if c == '(':
                low += 1
                high += 1
            elif c == '*':
                high += 1
                if low > 0:
                    low -= 1
                else:
                    low = 0
            else:  # c == ')'
                low = max(0, low - 1)
                high -= 1
                if high < 0:
                    return False
        return low == 0
```

### Explanation
- **Initialization**: The variables `low` and `high` are initialized to 0 to start tracking the number of unmatched opening brackets.
- **Traversal**:
  - For '(', both `low` and `high` are incremented, indicating a new opening bracket.
  - For '*', `high` is incremented (as it can act as an opening bracket), and `low` is decremented if it is positive (as it can act as a closing bracket). This step ensures that '*' is considered for both possibilities.
  - For ')', both `low` and `high` are decremented. If `high` becomes negative, it means there are more closing brackets than opening brackets, making the string invalid.
- **Validation**: After processing all characters, if `low` is 0, all opening brackets have been properly closed, and the string is valid; otherwise, it is invalid.

This approach efficiently checks the validity of the string by dynamically adjusting the counts of unmatched opening brackets and ensuring that the closing brackets are properly matched. The use of two counters allows the solution to handle the wildcard '*' flexibly while maintaining correctness.

    def checkValidString(self, s):
        low = 0
        high = 0
        for c in s:
            if c == '(':
                low += 1
                high += 1
            elif c == '*':
                high += 1
                if low > 0:
                    low -= 1
                else:
                    low = 0
            else:  # c == ')'
                low = max(0, low - 1)
                high -= 1
                if high < 0:
                    return False
        return low == 0
