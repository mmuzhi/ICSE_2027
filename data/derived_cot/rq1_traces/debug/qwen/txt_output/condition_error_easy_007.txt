To solve this problem, we need to generate the first `numRows` rows of Pascal's triangle. The initial code provided contains a bug in its base case handling and recursion logic. Specifically, when `numRows` is 0, it incorrectly returns a list containing a single row `[1]` instead of an empty list. Additionally, the recursive approach builds an extra row when `numRows` is 1, resulting in two rows instead of one.

### Approach
1. **Base Case Handling**: 
   - If `numRows` is 0, return an empty list since there are no rows to generate.
   - If `numRows` is 1, return a list containing a single row `[1]`.

2. **Recursive Step**:
   - For `numRows` greater than 1, recursively generate the first `numRows - 1` rows.
   - The last row of the recursively generated list is used to construct the next row by summing adjacent elements.
   - The newly constructed row is appended to the list of rows, and the updated list is returned.

This approach ensures that the correct number of rows is generated, handling edge cases such as 0 and 1 rows appropriately.

### Solution Code
```python
class Solution:
    def generate(self, numRows: int) -> List[List[int]]:
        if numRows == 0:
            return []
        if numRows == 1:
            return [[1]]
        prev = self.generate(numRows - 1)
        fin = prev[-1]
        now = [1]
        for i in range(len(fin)-1):
            now.append(fin[i] + fin[i+1])
        now.append(1)
        prev.append(now)
        return prev
```

### Explanation
1. **Base Cases**:
   - **numRows = 0**: Returns an empty list since no rows are needed.
   - **numRows = 1**: Returns a list containing the single row `[1]`.

2. **Recursive Case**:
   - The function calls itself with `numRows - 1` to generate the first `numRows - 1` rows.
   - The last row from the generated list is used to create the next row by summing adjacent elements (e.g., `[1, 1]` becomes `[1, 2, 1]`).
   - The newly created row is appended to the list of rows, and the updated list is returned.

This approach efficiently builds Pascal's triangle by leveraging recursion to handle each row in sequence, ensuring correctness for all valid inputs.

class Solution:
    def generate(self, numRows: int) -> List[List[int]]:
        if numRows == 0:
            return []
        if numRows == 1:
            return [[1]]
        prev = self.generate(numRows - 1)
        fin = prev[-1]
        now = [1]
        for i in range(len(fin)-1):
            now.append(fin[i] + fin[i+1])
        now.append(1)
        prev.append(now)
        return prev
