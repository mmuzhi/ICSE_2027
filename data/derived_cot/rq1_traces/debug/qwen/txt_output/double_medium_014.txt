=== RAW ANSWER ===

To solve this problem, we need to determine if it's possible to finish all courses given the prerequisites. The solution involves checking if the directed graph formed by the prerequisites contains any cycles. If there is a cycle, it's impossible to finish all courses; otherwise, it is possible.

### Approach
1. **Graph Construction**: Represent the courses and their prerequisites as a directed graph using a dictionary where each course maps to its list of prerequisites.
2. **Cycle Detection**: Use Depth-First Search (DFS) to detect cycles in the graph. During the DFS traversal:
   - If a node is encountered that is already in the current recursion stack, a cycle is detected.
   - If a node has already been visited (and processed), it is skipped.
   - Nodes are marked as visited after their DFS traversal is complete.
3. **Iterate Over All Courses**: For each course, if it hasn't been visited, perform DFS starting from that course. If any DFS call detects a cycle, return False immediately. If no cycles are found after all courses are processed, return True.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        pre = defaultdict(list)
        for a, b in prerequisites:
            pre[b].append(a)
        
        visited = set()
        rec_stack = set()
        
        def dfs(course):
            if course in rec_stack:
                return False
            if course in visited:
                return True
            
            visited.add(course)
            rec_stack.add(course)
            for p in pre[course]:
                if not dfs(p):
                    return False
            rec_stack.remove(course)
            return True
        
        for course in range(numCourses):
            if course not in visited:
                if not dfs(course):
                    return False
        return True
```

### Explanation
1. **Graph Construction**: The graph is built using a defaultdict where each course points to its prerequisites. For example, if the input is `[[0, 1]]`, then course 1 requires course 0, so `pre[1] = [0]`.
2. **Cycle Detection**: The DFS function checks for cycles by maintaining two sets: `visited` (nodes that have been processed) and `rec_stack` (nodes in the current DFS path). If a node is found in `rec_stack` during traversal, a cycle exists.
3. **Iterating Courses**: The main loop ensures each course is processed only once. If any course's DFS call returns False (indicating a cycle), the function immediately returns False. If all courses are processed without detecting any cycles, the function returns True.

This approach efficiently checks for cycles in the graph using DFS, ensuring that all courses can be completed if no cycles are present. The time complexity is O(N + E), where N is the number of courses and E is the number of prerequisites, as each node and edge is processed once.
