3. **Algorithm Selection**: 
   - **Line Equation**: For two points \((x1, y1)\) and \((x2, y2)\), the line equation is derived from the two-point form. Special cases for vertical and horizontal lines are handled separately.
   - **Normalization**: The coefficients \(A\), \(B\), and \(C\) are normalized by their greatest common divisor (GCD) and adjusted to ensure a consistent representation.
   - **Counting Points**: For each unique line, iterate through all points to count how many lie on the line.
4. **Complexity Analysis**: The algorithm checks all pairs of points (\(O(n^2)\) and, for each pair, counts points on the line (\(O(n)\)), resulting in an overall time complexity of \(O(n^3)\). This is feasible for small \(n\) (up to 100) as specified in similar LeetCode problems.

```python
import math
from typing import List

class Solution:
    def maxPoints(self, points: List[List[int]]) -> int:
        n = len(points)
        if n <= 1:
            return n
        
        seen = set()
        max_count = 0
        
        for i in range(n):
            x1, y1 = points[i]
            for j in range(i+1, n):
                x2, y2 = points[j]
                if points[i] == points[j]:
                    continue
                
                if x1 == x2:
                    A = 1
                    B = 0
                    C = -x1
                else:
                    A = y2 - y1
                    B = x1 - x2
                    C = x2 * y1 - x1 * y2
                
                nums = []
                if A != 0:
                    nums.append(abs(A))
                if B != 0:
                    nums.append(abs(B))
                if C != 0:
                    nums.append(abs(C))
                
                if not nums:
                    g = 1
                else:
                    g = nums[0]
                    for num in nums[1:]:
                        g = math.gcd(g, num)
                
                A_norm = A // g
                B_norm = B // g
                C_norm = C // g
                
                if A_norm < 0:
                    A_norm, B_norm, C_norm = -A_norm, -B_norm, -C_norm
                elif A_norm == 0:
                    if B_norm < 0:
                        B_norm, C_norm = -B_norm, -C_norm
                    elif B_norm == 0:
                        if C_norm < 0:
                            C_norm = -C_norm
                key = (A_norm, B_norm, C_norm)
                
                if key in seen:
                    continue
                seen.add(key)
                
                count = 0
                for k in range(n):
                    x, y = points[k]
                    if A_norm * x + B_norm * y + C_norm == 0:
                        count += 1
                if count > max_count:
                    max_count = count
        
        return max_count
```

### Explanation
1. **Initialization**: The function starts by checking if there are 0 or 1 points, returning immediately in such cases.
2. **Line Equation Calculation**: For each pair of distinct points, the line equation \(Ax + By + C = 0\) is computed. Special cases for vertical lines (infinite slope) are handled separately.
3. **Normalization**: The coefficients \(A\), \(B\), and \(C\) are normalized by their GCD to ensure a consistent line representation. The sign is adjusted so the first non-zero coefficient is positive.
4. **Point Counting**: For each unique line, the code counts all points that satisfy the line equation. The maximum count across all lines is returned.
5. **Efficiency**: By normalizing line equations and using a set to track unique lines, the solution efficiently checks all possible lines and counts collinear points without redundancy. The algorithm handles duplicates and all edge cases, including vertical and horizontal lines.
