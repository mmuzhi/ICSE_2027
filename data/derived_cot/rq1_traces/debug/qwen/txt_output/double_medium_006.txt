2. **Satisfies Trip Check**: For each midpoint time during the binary search, we check if the total number of trips that can be completed within that time is at least `totalTrips`. This is done by summing `mid // t` for each trip time `t` in the list. If the sum meets or exceeds `totalTrips`, the midpoint time is a candidate for the minimal time.
3. **Binary Search Adjustment**: Depending on whether the midpoint time satisfies the condition, we adjust the binary search bounds. If it does, we search for a smaller time; otherwise, we search for a larger time.

### Solution Code
```python
class Solution:
    def satisfiesTrip(self, mid, time, totalTrips):
        trip = 0
        for t in time:
            trip += mid // t
        if trip >= totalTrips:
            return True
        return False

    def minimumTime(self, time: List[int], totalTrips: int) -> int:
        time.sort(reverse=True)
        minimum = min(time)
        left = minimum
        right = minimum * totalTrips
        while left < right:
            mid = (left + right) // 2
            if self.satisfiesTrip(mid, time, totalTrips):
                right = mid
            else:
                left = mid + 1
        return left
```

### Explanation
1. **Binary Search Initialization**: The binary search starts with `left` set to the smallest trip time and `right` set to the product of the smallest trip time and the total number of trips. This defines the range where the minimal time must lie.
2. **Midpoint Calculation**: During each iteration, the midpoint `mid` is calculated as the average of `left` and `right`.
3. **Trip Completion Check**: The `satisfiesTrip` function checks if the midpoint time allows completing at least `totalTrips` trips by summing the integer division of `mid` by each trip time. If the sum meets or exceeds `totalTrips`, the search continues in the lower half; otherwise, it continues in the upper half.
4. **Result Determination**: The loop exits when `left` equals `right`, which is the minimal time required to complete the specified number of trips. This value is returned as the result.

This approach efficiently narrows down the search space using binary search, ensuring optimal performance even for large input sizes.
