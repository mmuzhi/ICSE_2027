=== RAW ANSWER ===

To solve this problem, we need to determine the minimum number of operations required to make all elements in an array non-zero. Each operation allows us to increment an element by 1, and we can perform this operation any number of times on any element. The goal is to ensure every element in the array is at least 1.

### Approach
1. **Problem Analysis**: The problem requires transforming all elements of the array to be non-zero (i.e., at least 1) using the minimum number of operations. Each operation increments an element by 1. The key observation here is that for each element in the array, if it is less than 1, we need to increment it by the difference between 1 and its current value. If the element is already 1 or greater, no operations are needed for that element.
2. **Algorithm Selection**: The solution involves iterating through each element in the array. For each element, if it is less than 1, we calculate the number of increments required to make it at least 1. The sum of these increments across all elements gives the total minimum operations needed.
3. **Complexity Analysis**: The algorithm processes each element exactly once, resulting in a linear time complexity of O(n), where n is the number of elements in the array. This is optimal for the given problem as each element must be checked at least once.

### Solution Code
```python
class Solution:
    def minimumOperations(self, nums: List[int]) -> int:
        operations = 0
        for num in nums:
            if num < 1:
                operations += (1 - num)
        return operations
```

### Explanation
- **Initialization**: We start by initializing a variable `operations` to 0, which will keep track of the total number of increments needed.
- **Iteration**: For each element in the array, we check if it is less than 1. If it is, we calculate how many increments are needed to make it at least 1 (which is `1 - num`) and add this value to `operations`.
- **Result**: After processing all elements, `operations` contains the total number of increments required to make all elements non-zero, which is returned as the result.

This approach efficiently computes the minimum operations by focusing on each element's deficit from 1, ensuring optimal performance with linear time complexity.
