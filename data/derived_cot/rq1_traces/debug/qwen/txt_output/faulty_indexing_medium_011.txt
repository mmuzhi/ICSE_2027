To solve this problem, we need to identify invalid transactions based on two conditions: the transaction amount exceeds 1000, or there exists another transaction by the same person within 60 minutes but in a different city. 

### Approach
1. **Problem Analysis**: The task involves processing a list of transactions and marking each transaction as invalid if it meets either of the two conditions. The key challenge is efficiently checking for transactions by the same person within a 60-minute window but in a different city.
2. **Intuition**: By building a dictionary that maps each person's name to a list of their transaction times and cities, we can efficiently check for invalid transactions. For each transaction, we first check if the amount exceeds 1000. If not, we then check all previous transactions of the same person to see if any occur within 60 minutes in a different city.
3. **Algorithm Selection**: We use a dictionary to store transactions grouped by person. For each transaction, we check all other transactions of the same person (excluding the current one) to see if any meet the time and city conditions.
4. **Complexity Analysis**: The algorithm involves building a dictionary in O(n) time, where n is the number of transactions. For each transaction, we potentially check all previous transactions of the same person, leading to an O(n^2) worst-case time complexity. However, given the constraints, this approach is efficient enough.

### Solution Code
```python
class Solution:
    def invalidTransactions(self, transactions: List[str]) -> List[str]:
        invalid = []
        txn_dict = {}
        for trn in transactions:
            parts = trn.split(',')
            name = parts[0]
            time = parts[1]
            amount = parts[2]
            city = parts[3]
            if name not in txn_dict:
                txn_dict[name] = []
            txn_dict[name].append((time, city))
        
        for i, trn in enumerate(transactions):
            parts = trn.split(',')
            name = parts[0]
            time_str = parts[1]
            amount = parts[2]
            city = parts[3]
            if int(amount) > 1000:
                invalid.append(trn)
            else:
                if name in txn_dict:
                    other_transactions = txn_dict[name].copy()
                    try:
                        other_transactions.remove((time_str, city))
                    except ValueError:
                        pass
                    current_time = int(time_str)
                    for t, c in other_transactions:
                        t_int = int(t)
                        if c != city and abs(current_time - t_int) <= 60:
                            invalid.append(trn)
                            break
        return invalid
```

### Explanation
1. **Dictionary Construction**: We first build a dictionary (`txn_dict`) where each key is a person's name, and the value is a list of tuples containing the transaction time and city for all transactions by that person.
2. **Transaction Processing**: For each transaction, we split the string into its components (name, time, amount, city). If the amount exceeds 1000, the transaction is immediately marked as invalid.
3. **Same Person Check**: For transactions with amounts ≤ 1000, we check all other transactions by the same person. We create a copy of the person's transaction list, remove the current transaction, and then check each remaining transaction to see if it occurs within 60 minutes and in a different city. If such a transaction is found, the current transaction is marked as invalid.
4. **Result Compilation**: The list of invalid transactions is compiled and returned after processing all transactions.

This approach efficiently checks both conditions for invalid transactions by leveraging dictionary lookups and careful exclusion of the current transaction during the same-person check.
