To solve this problem, we need to find a peak element in an array where a peak element is defined as an element that is strictly greater than its neighbors. The solution involves using a binary search approach to efficiently locate the peak element without checking every element individually.

### Approach
1. **Problem Analysis**: The problem requires finding an index in the array where the element is greater than both its left and right neighbors (if they exist). The array can have duplicates, but the peak must be strictly greater than its neighbors.
2. **Intuition**: The solution leverages binary search to narrow down the search space. By comparing the middle element with its neighbors, we can determine whether the peak lies to the left or right of the middle element. If the middle element is greater than its neighbors, it is a peak. Otherwise, the peak must be in the direction where the neighboring element is greater.
3. **Algorithm Selection**: Binary search is chosen because it efficiently reduces the search space by half in each iteration, leading to a logarithmic time complexity.
4. **Complexity Analysis**: The binary search approach ensures that the solution runs in O(log n) time, where n is the number of elements in the array. This is optimal for large arrays as it avoids a linear scan.

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 1:
            return 0
        low, high = 0, n - 1
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] > nums[mid - 1]) and (mid == n - 1 or nums[mid] > nums[mid + 1]):
                return mid
            elif nums[mid] < nums[mid + 1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

### Explanation
1. **Initial Check**: If the array has only one element, that element is trivially a peak, so we return its index (0).
2. **Binary Search Setup**: Initialize `low` to 0 and `high` to the last index of the array.
3. **Binary Search Loop**: 
   - **Mid Calculation**: Compute the middle index `mid` of the current search range.
   - **Peak Check**: If the middle element is greater than its left and right neighbors (if they exist), return its index.
   - **Direction Adjustment**: If the middle element is less than its right neighbor, the peak must be to the right, so adjust `low` to `mid + 1`. Otherwise, adjust `high` to `mid - 1`.
4. **Termination**: The loop terminates when `low` exceeds `high`, indicating no peak was found (though the problem guarantees at least one peak exists).

This approach efficiently narrows down the search space using binary search, ensuring optimal performance with a logarithmic time complexity.

        low, high = 0, n - 1
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] > nums[mid - 1]) and (mid == n - 1 or nums[mid] > nums[mid + 1]):
                return mid
            elif nums[mid] < nums[mid + 1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
