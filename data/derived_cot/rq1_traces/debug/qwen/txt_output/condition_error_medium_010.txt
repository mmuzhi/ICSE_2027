=== RAW ANSWER ===

To solve this problem, we need to find the k-th smallest factor of a given integer `n`. The factors of a number are the integers that divide it completely without leaving a remainder. The solution involves iterating through all possible integers from 1 to `n` and checking if each integer is a factor of `n`. The k-th factor encountered during this iteration is the answer. If there are fewer than `k` factors, we return -1.

### Approach
1. **Initialization**: Start with a counter `j` initialized to 0 to keep track of the number of factors found.
2. **Iteration**: Loop through each integer `i` from 1 to `n`:
   - **Check for Factors**: For each `i`, check if `n` is divisible by `i` (i.e., `n % i == 0`). If it is, increment the counter `j` and check if `j` equals `k`.
   - **Early Termination**: If `j` equals `k` at any point, immediately return `i` as it is the k-th factor.
3. **Completion**: If the loop completes without finding `k` factors, return -1.

This approach efficiently checks each integer in ascending order, ensuring the first `k` factors are found in order. The loop breaks early if the k-th factor is found, optimizing performance for cases where `k` is small.

### Solution Code
```python
class Solution:
    def kthFactor(self, n: int, k: int) -> int:
        j = 0
        for i in range(1, n + 1):
            if n % i == 0:
                j += 1
                if j == k:
                    return i
        return -1
```

### Explanation
- **Initialization**: The variable `j` starts at 0 to count how many factors have been found.
- **Loop Through Integers**: The loop runs from 1 to `n`. For each integer `i`:
  - **Divisibility Check**: If `n` is divisible by `i`, `j` is incremented by 1.
  - **Check for k-th Factor**: If `j` reaches `k`, the current integer `i` is returned immediately as it is the k-th factor.
- **Return -1 if Not Found**: If the loop completes without finding `k` factors, the function returns -1, indicating the k-th factor does not exist.

This approach efficiently finds the k-th smallest factor by leveraging early termination and straightforward iteration, ensuring optimal performance and correctness.
