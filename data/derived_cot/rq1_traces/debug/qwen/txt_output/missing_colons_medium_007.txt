=== RAW ANSWER ===

To solve this problem, we need to perform a level-order traversal (BFS) of a binary tree and then return the nodes level by level from bottom to top. The initial code attempts to do this but contains a few bugs that need to be fixed.

### Approach
1. **Level-Order Traversal (BFS)**: The code uses a queue to traverse the tree level by level. For each level, it processes all nodes in the queue, collects their values, and appends their children to the queue for the next level.
2. **Reversing the Result**: After collecting all levels from top to bottom, the code reverses the list of levels to present them from bottom to top. The reversal function uses a two-pointer technique to swap elements in-place.
3. **Edge Cases**: The code needs to handle cases where the tree is empty (root is None) by returning an empty list.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans)-1
            while l <= r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root:
            return []
        q, ans = [root], []
        while q:
            n, l = len(q), []
            for i in range(n):
                curr = q.pop(0)
                l.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(l)
        return checkReverse(ans)
```

### Explanation
1. **Level-Order Traversal**: The BFS starts from the root node. For each node, its value is added to the current level's list, and its children are added to the queue for processing in the next level.
2. **Reversing Levels**: The `checkReverse` function reverses the list of levels in-place using a two-pointer approach. The left pointer starts at the beginning of the list, and the right pointer starts at the end. They move towards each other, swapping elements until they meet.
3. **Edge Handling**: If the tree is empty (root is None), the function returns an empty list immediately. Otherwise, it proceeds with the BFS and reversal steps.

This approach ensures that the tree is traversed level by level, and the levels are presented in reverse order from bottom to top, providing the correct output as required.
