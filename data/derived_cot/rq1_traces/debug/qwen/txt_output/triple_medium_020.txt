To solve this problem, we need to reconstruct the original array from a list of adjacent pairs. The original array is a permutation of integers from 1 to n, and the given adjacent pairs represent consecutive elements in the array. The solution involves building a graph from these pairs and then traversing the graph to reconstruct the array.

### Approach
1. **Graph Construction**: We first build an undirected graph where each node represents an element in the array, and edges represent the adjacent pairs. This helps in identifying the connections between elements.
2. **Identify Endpoints**: In a linear array (permutation), there are exactly two nodes with degree 1 (endpoints). These endpoints are the starting and ending points of the array.
3. **Depth-First Search (DFS)**: Using DFS, we traverse the graph starting from one of the endpoints. During traversal, we keep track of visited nodes to avoid revisiting and ensure we traverse the entire chain of nodes. The DFS ensures we visit each node exactly once, reconstructing the array in the process.

### Solution Code
```python
class Solution:
    def restoreArray(self, adjacentPairs: List[List[int]]) -> List[int]:
        if not adjacentPairs:
            return [1]
        
        graph = {}
        for u, v in adjacentPairs:
            graph.setdefault(u, []).append(v)
            graph.setdefault(v, []).append(u)
        
        degrees = {node: len(graph[node]) for node in graph}
        endpoints = [node for node in graph if degrees[node] == 1]
        
        if len(endpoints) != 2:
            start = next(iter(graph.keys()))
        else:
            start = endpoints[0]
        
        seen = {start}
        stack = [start]
        ans = []
        prev = None
        while stack:
            node = stack.pop()
            ans.append(node)
            for neighbor in graph[node]:
                if neighbor == prev:
                    continue
                if neighbor not in seen:
                    seen.add(neighbor)
                    prev = node
                    stack.append(neighbor)
        
        return ans
```

### Explanation
1. **Graph Construction**: The graph is built by iterating through each adjacent pair and adding edges in both directions (since the graph is undirected).
2. **Endpoint Identification**: Nodes with degree 1 are identified as endpoints. These are the starting and ending points of the original array.
3. **DFS Traversal**: Starting from one endpoint, we perform DFS to traverse the graph. For each node visited, we add it to the result list and explore its unvisited neighbors. The traversal ensures we move from one end of the array to the other, reconstructing the original sequence.
4. **Edge Cases**: If the input list of adjacent pairs is empty, the array consists of a single element, which is `[1]` as per the problem's permutation constraint. If there aren't exactly two endpoints (though the problem assumes a linear array), we default to starting from any node in the graph.

This approach efficiently reconstructs the original array by leveraging graph traversal, ensuring correctness and handling all specified edge cases.
