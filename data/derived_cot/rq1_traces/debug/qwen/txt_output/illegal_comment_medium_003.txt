=== RAW ANSWER ===

To solve this problem, we need to convert a sorted singly-linked list into a balanced binary search tree (BST). The key insight here is that the middle element of the sorted list should be the root of the BST, with the left half of the list forming the left subtree and the right half forming the right subtree. This approach ensures that the resulting BST is balanced, which optimizes search, insertion, and deletion operations.

### Approach
1. **Extract Elements from Linked List**: Traverse the sorted linked list and collect all its elements into a list. This step ensures we have all the elements in a convenient, indexable format.
2. **Recursive Tree Construction**: Use a helper function that constructs the BST recursively. The helper function takes the left and right indices of the current subarray (from the extracted list) and constructs the BST by:
   - Finding the middle element of the current subarray to serve as the root.
   - Recursively building the left subtree from the left half of the subarray.
   - Recursively building the right subtree from the right half of the subarray.

### Solution Code
```python
from typing import Optional

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def sortedListToBST(self, head: Optional[ListNode]) -> Optional[TreeNode]:
        nums = []
        curr = head
        while curr:
            nums.append(curr.val)
            curr = curr.next
        
        def helper(l, r):
            if l > r:
                return None
            mid = (l + r) // 2
            root = TreeNode(nums[mid])
            root.left = helper(l, mid-1)
            root.right = helper(mid+1, r)
            return root
        
        return helper(0, len(nums)-1)
```

### Explanation
1. **Extracting Elements**: The code first traverses the linked list starting from the head node, appending each node's value to a list. This results in a list containing all elements of the sorted linked list.
2. **Helper Function**: The helper function is defined to construct the BST recursively. It takes the left and right indices of the current subarray (initially the entire list). The middle element of this subarray is chosen as the root of the current subtree. The left subtree is built from the elements before the middle element, and the right subtree from the elements after.
3. **Base Case**: If the left index exceeds the right index, the helper function returns `None`, indicating no more nodes to process.
4. **Initial Call**: The helper function is initially called with the entire range of the list (from index 0 to the last index), which starts the recursive construction of the BST.

This approach efficiently balances the BST by always selecting the middle element as the root, ensuring the tree remains balanced and operations are optimized. The time complexity is O(n log n) due to the recursive division of the list, and the space complexity is O(n) for storing the list and the recursion stack.
