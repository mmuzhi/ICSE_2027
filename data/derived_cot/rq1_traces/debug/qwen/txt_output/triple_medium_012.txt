### Approach
1. **Problem Analysis**: The problem requires finding the maximum difference between any two consecutive elements in a sorted array. The initial code attempted to find the minimum gap but incorrectly used absolute values and undefined variables.
2. **Intuition**: By sorting the array, we ensure that the largest gap between any two consecutive elements will be the solution. We then iterate through the sorted array to compute the differences between consecutive elements and keep track of the maximum difference encountered.
3. **Algorithm Selection**: The algorithm involves sorting the array and then traversing it once to compute the differences between consecutive elements, updating the maximum gap found during the traversal.
4. **Complexity Analysis**: Sorting the array takes O(n log n) time, and the subsequent traversal is O(n). Thus, the overall time complexity is dominated by the sorting step, O(n log n). The space complexity is O(1) if we sort in-place, or O(n) if we use additional space for sorting.

### Solution Code
```python
class Solution:
    def maximumGap(self, nums: list) -> int:
        if len(nums) < 2:
            return 0
        nums = sorted(nums)
        max_gap = 0
        for i in range(1, len(nums)):
            gap = nums[i] - nums[i-1]
            if gap > max_gap:
                max_gap = gap
        return max_gap
```

### Explanation
1. **Initial Check**: If the array has fewer than two elements, the maximum gap is 0 since no pairs exist.
2. **Sorting**: The array is sorted to ensure consecutive elements are in ascending order, simplifying the gap calculation.
3. **Gap Calculation**: The loop starts from the second element and computes the difference between each element and its predecessor. The maximum gap encountered during this traversal is stored and returned.
4. **Efficiency**: The solution efficiently computes the maximum gap by leveraging sorting and a single pass through the array, ensuring optimal performance.
