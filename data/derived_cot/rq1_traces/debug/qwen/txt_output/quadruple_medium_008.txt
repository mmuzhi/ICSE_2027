To solve this problem, we need to find the closest meeting node in a directed graph where two nodes (node1 and node2) start traversing the graph. The goal is to determine the node that both traversals can reach with the smallest maximum distance from either traversal. If no such node exists, we return -1.

### Approach
1. **Problem Analysis**: The problem involves traversing a directed graph from two starting nodes (node1 and node2) and finding a node that is reachable by both traversals. The solution requires calculating the shortest distances from each starting node to all other nodes and then determining the node with the smallest maximum distance from either traversal.

2. **Intuition**: We use Breadth-First Search (BFS) to compute the shortest distances from each starting node to all other nodes. BFS is suitable here because it explores nodes level by level, ensuring the shortest path is found first.

3. **Algorithm Selection**: 
   - **BFS for node1**: Compute the shortest distances from node1 to all other nodes.
   - **BFS for node2**: Compute the shortest distances from node2 to all other nodes.
   - **Combine Results**: For each node, if it is reachable from both starting nodes, compute the maximum of the two distances. Track the node with the smallest maximum distance, and in case of ties, the smallest node index.

4. **Complexity Analysis**: The BFS operations run in O(N) time for each starting node, where N is the number of nodes. Since we perform BFS twice, the total time complexity is O(N). The space complexity is O(N) to store the distances for both BFS traversals.

### Solution Code
```python
from collections import deque
from typing import List

class Solution:
    def closestMeetingNode(self, edges: List[int], node1: int, node2: int) -> int:
        n = len(edges)
        INF = float('inf')
        d1 = [INF] * n
        d2 = [INF] * n
        
        # BFS for node1
        q = deque()
        q.append(node1)
        d1[node1] = 0
        while q:
            cur = q.popleft()
            nxt = edges[cur]
            if nxt == -1:
                continue
            if nxt < 0 or nxt >= n:
                continue
            if d1[nxt] == INF:
                d1[nxt] = d1[cur] + 1
                q.append(nxt)
        
        # BFS for node2
        q = deque()
        q.append(node2)
        d2[node2] = 0
        while q:
            cur = q.popleft()
            nxt = edges[cur]
            if nxt == -1:
                continue
            if nxt < 0 or nxt >= n:
                continue
            if d2[nxt] == INF:
                d2[nxt] = d2[cur] + 1
                q.append(nxt)
        
        ans = -1
        mmin = INF
        for i in range(n):
            if d1[i] != INF and d2[i] != INF:
                m = max(d1[i], d2[i])
                if m < mmin:
                    mmin = m
                    ans = i
                elif m == mmin:
                    if i < ans:
                        ans = i
        
        return ans
```

### Explanation
1. **Initialization**: We initialize two distance arrays, `d1` and `d2`, with infinity to represent unreachable nodes. The size of these arrays is equal to the number of nodes in the graph.
2. **BFS for node1**: Starting from `node1`, we traverse the graph using BFS, updating the shortest distances in `d1` for each reachable node.
3. **BFS for node2**: Similarly, we perform BFS starting from `node2`, updating distances in `d2`.
4. **Finding the Meeting Node**: After computing distances from both starting nodes, we iterate through all nodes. For each node, if it is reachable from both starting nodes, we compute the maximum of the two distances. We keep track of the smallest such maximum distance and the corresponding node index.
5. **Result**: The node with the smallest maximum distance is returned. If no such node exists, we return -1.

This approach efficiently computes the required meeting node by leveraging BFS to explore the graph and combining the results from both traversals.
