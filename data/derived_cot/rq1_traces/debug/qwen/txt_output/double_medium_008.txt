=== RAW ANSWER ===

To solve this problem, we need to determine the number of bookings for each day from 1 to n based on given flight bookings. Each booking specifies a range of days (from day l to day r) and the number of seats booked. The solution involves efficiently calculating the cumulative bookings for each day using a difference array (prefix sum) technique.

### Approach
1. **Difference Array Technique**: We use a difference array to mark the start and end of each booking's impact. For each booking [l, r, seats], we add `seats` to the start day `l` and subtract `seats` from the day `r+1` (if it exists). This marks the beginning and end of the booking's effect.
2. **Prefix Sum Calculation**: After processing all bookings, we compute the prefix sum of the difference array. The prefix sum at any day `i` gives the total bookings for that day, as it accumulates all the changes marked by the difference array from the start up to day `i`.
3. **Efficiency**: This approach efficiently handles the problem in O(n + m) time, where n is the number of days and m is the number of bookings. The space complexity is O(n) due to the difference array used.

### Solution Code
```python
from typing import List
from itertools import accumulate

class Solution:
    def corpFlightBookings(self, bookings: List[List[int]], n: int) -> List[int]:
        arr = [0] * (n + 2)
        for booking in bookings:
            l, r, seats = booking
            arr[l] += seats
            if r + 1 < len(arr):
                arr[r + 1] -= seats
        result = list(accumulate(arr))
        return result[1:1 + n]
```

### Explanation
1. **Initialization**: We initialize an array `arr` of size `n + 2` (to accommodate all days from 1 to n and the day `n+1` for boundary handling).
2. **Processing Bookings**: For each booking, we update the start day `l` by adding the number of seats and the day `r+1` by subtracting the number of seats (if `r+1` is within bounds). This marks the start and end of the booking's impact.
3. **Prefix Sum Calculation**: Using the `accumulate` function, we compute the prefix sum of the difference array. The prefix sum at any index `i` gives the total bookings up to day `i`.
4. **Result Extraction**: The result is derived from the prefix sum array, starting from index 1 to index `n` (inclusive), which corresponds to the bookings for days 1 to n.

This approach efficiently handles the booking ranges and computes the daily bookings using simple array operations, ensuring optimal performance and correctness.
