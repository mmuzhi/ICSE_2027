To solve this problem, we need to determine if a given string containing parentheses and asterisks is valid. The string is considered valid if it can be balanced such that all opening parentheses are properly closed, and asterisks can be treated as either opening parentheses, closing parentheses, or empty characters.

### Approach
The approach involves using a two-pointer technique to track the minimum and maximum number of valid parentheses configurations as we iterate through the string. The idea is to maintain two counters, `low` and `high`, where:
- `low` tracks the minimum number of valid configurations remaining.
- `high` tracks the maximum number of valid configurations remaining.

For each character in the string:
- If the character is '(', both `low` and `high` are incremented.
- If the character is ')', both `low` and `high` are decremented. If `high` becomes negative, the string is invalid.
- If the character is '*', `low` is decremented (but not below zero) and `high` is incremented. If `high` becomes negative, the string is invalid.

After processing all characters, if `low` is zero, the string is valid; otherwise, it is invalid.

### Solution Code
```python
class Solution:
    def checkValidString(self, s):
        low = 0
        high = 0
        for c in s:
            if c == '(':
                low += 1
                high += 1
            elif c == ')':
                low = max(low - 1, 0)
                high -= 1
                if high < 0:
                    return False
            else:  # c == '*'
                low = max(low - 1, 0)
                high += 1
                if high < 0:
                    return False
        return low == 0
```

### Explanation
1. **Initialization**: We start with `low` and `high` set to zero, representing the minimum and maximum number of valid configurations.
2. **Iteration through the string**: For each character:
   - **'('**: Both `low` and `high` are incremented because an opening parenthesis can increase the number of valid configurations.
   - **')'**: Both `low` and `high` are decremented. If `high` becomes negative, it means there are more closing parentheses than valid configurations, making the string invalid.
   - **'*'**: `low` is decremented (but not below zero) and `high` is incremented because an asterisk can act as a closing parenthesis, reducing the number of valid configurations.
3. **Final Check**: After processing all characters, if `low` is zero, it means there are no unmatched opening parentheses, and the string is valid; otherwise, it is invalid.

This approach efficiently checks the validity of the string by dynamically adjusting the range of possible valid configurations, ensuring optimal performance with a time complexity of O(n), where n is the length of the string.
