=== RAW ANSWER ===

To solve this problem, we need to find a peak element in an integer array. A peak element is defined as an element that is greater than its neighbors. The solution involves efficiently searching for such an element using a binary search approach, which reduces the time complexity compared to a linear scan.

### Approach
1. **Binary Search Setup**: We use binary search to efficiently narrow down the search space. The search space is between the start (`low`) and end (`high`) of the array.
2. **Peak Identification**: For each midpoint (`mid`) during the binary search, we check if it is a peak by comparing it with its neighbors. If it is, we return the index.
3. **Direction Adjustment**: If the current element is not a peak, we adjust the search direction based on the comparison with the next element. If the current element is less than the next, the peak must be on the right side; otherwise, it must be on the left side.

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        
        if n == 1:
            return 0
        
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] >= nums[mid-1]) and (mid == n-1 or nums[mid] >= nums[mid+1]):
                return mid
            elif nums[mid] < nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

### Explanation
1. **Binary Search Initialization**: The binary search starts with `low` set to 0 and `high` set to `n-1` (the last index of the array).
2. **Edge Case Handling**: If the array has only one element, that element is trivially a peak, so we return its index (0).
3. **Binary Search Loop**: The loop continues while `low` is less than or equal to `high`. The midpoint `mid` is calculated in each iteration.
4. **Peak Check**: For the midpoint, we check if it is a peak by verifying if it is greater than or equal to its neighbors (handling edge cases where `mid` is the first or last element).
5. **Adjust Search Direction**: If the midpoint is not a peak, we move the search direction based on whether the midpoint is less than its next element (indicating the peak is to the right) or greater than its next element (indicating the peak is to the left).
6. **Termination**: The loop terminates when a peak is found, returning its index, or when the search space is exhausted, returning -1 (though the problem guarantees a peak exists, this handles edge cases gracefully).

This approach efficiently locates a peak element in O(log n) time complexity, leveraging binary search to avoid a linear scan.
