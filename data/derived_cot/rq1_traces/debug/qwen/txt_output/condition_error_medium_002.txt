=== RAW ANSWER ===

To solve this problem, we need to restore all possible valid IP addresses from a given string. A valid IP address consists of four parts, each ranging from 0 to 255, separated by dots. The solution involves generating all possible combinations of four parts from the string and checking if they form a valid IP address.

### Approach
1. **Backtracking**: The approach uses backtracking to explore all possible ways to split the string into four parts. Each part must be a number between 0 and 255, and parts cannot have leading zeros unless the part is exactly "0".
2. **Base Case**: When the entire string is processed (i.e., we have four parts and the index reaches the end of the string), we record the IP address.
3. **Recursive Case**: For each position in the string, we try to form parts of length 1 to 3 digits (if the part starts with '0', it can only be one digit). Each valid part is added to the current list of parts, and the function recurses to the next position. After the recursive call, the part is removed from the list to backtrack and explore other possibilities.

### Solution Code
```python
class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        n = len(s)
        
        def backtrack(i):
            if i == n and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i > n:
                return
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 3 and i + j < n:
                if int(s[i:i+j+1]) < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Initialization**: The solution initializes an empty list `res` to store the resulting IP addresses and an empty list `cur` to keep track of the current parts being formed.
2. **Backtracking Function**: The `backtrack` function is defined to explore all valid splits:
   - **Base Case**: If the current index `i` reaches the end of the string and exactly four parts have been formed, the parts are joined into a string and added to `res`.
   - **Invalid Cases**: If more than four parts are formed or the index exceeds the string length, the function returns.
   - **Zero Handling**: If the current character is '0', it is added as a part (since leading zeros are not allowed), and the function recurses. After the recursive call, the part is removed to backtrack.
   - **Non-zero Handling**: For non-zero characters, the function tries parts of lengths 1 to 3 digits. Each part is checked to ensure it is a number less than 256. Valid parts are added to `cur`, and the function recurses. After the recursive call, the part is removed to explore other possibilities.
3. **Result Compilation**: After the backtracking completes, the function returns the list of all valid IP addresses found.

This approach efficiently explores all possible valid IP addresses by leveraging backtracking and pruning invalid paths early, ensuring optimal performance for the given problem constraints.
