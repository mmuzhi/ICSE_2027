To solve this problem, we need to compute the minimum edit distance (Levenshtein distance) between two strings, `word1` and `word2`. The edit distance is defined as the minimum number of operations required to transform one string into another, where each operation can be an insertion, deletion, or substitution of a single character.

### Approach
1. **Dynamic Programming (DP) Setup**: We use a DP table `dp` where `dp[i][j]` represents the minimum edit distance between the first `i` characters of `word1` and the first `j` characters of `word2`.
2. **Base Cases**: 
   - If `word1` is empty, the edit distance is the length of `word2` (all insertions).
   - If `word2` is empty, the edit distance is the length of `word1` (all deletions).
3. **Filling the DP Table**:
   - If the characters at positions `i-1` and `j-1` in `word1` and `word2` match, the edit distance remains the same as `dp[i-1][j-1]`.
   - If the characters do not match, the edit distance is the minimum of:
     - `dp[i-1][j] + 1` (deletion of `word1[i-1]`)
     - `dp[i][j-1] + 1` (insertion of `word2[j-1]` into `word1`)
     - `dp[i-1][j-1] + 1` (substitution of `word1[i-1]` with `word2[j-1]`)

### Solution Code
```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m = len(word1)
        n = len(word2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        for i in range(m + 1):
            dp[i][0] = i
        for j in range(n + 1):
            dp[0][j] = j
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    dp[i][j] = dp[i-1][j-1]
                else:
                    dp[i][j] = min(dp[i-1][j] + 1, dp[i][j-1] + 1, dp[i-1][j-1] + 1)
        return dp[-1][-1]
```

### Explanation
- **Initialization**: The DP table `dp` is initialized with dimensions `(m+1) x (n+1)` to accommodate all possible substrings of `word1` and `word2`.
- **Base Cases**: The first row and column of the DP table are filled with the lengths of the respective strings, representing the number of operations needed to convert an empty string to the full string.
- **DP Table Filling**: For each character in `word1` and `word2`, if the characters match, the value is taken from the diagonal cell (`dp[i-1][j-1]`). If they don't match, the value is computed as the minimum of the three possible operations (insertion, deletion, substitution) plus one.
- **Result**: The bottom-right cell of the DP table (`dp[m][n]`) contains the minimum edit distance between the entire strings `word1` and `word2`.

This approach efficiently computes the edit distance using dynamic programming, leveraging overlapping subproblems and optimal substructure properties to minimize redundant calculations.
