=== RAW ANSWER ===

To solve this problem, we need to rearrange the elements of an array based on a given pivot value. The goal is to place all elements less than the pivot first, followed by elements equal to the pivot, and then elements greater than the pivot, all while maintaining their original order within each category.

### Approach
1. **Problem Analysis**: The task is to partition the array into three parts:
   - Elements less than the pivot, maintaining their original order.
   - Elements equal to the pivot, maintaining their original order.
   - Elements greater than the pivot, maintaining their original order.
2. **Intuition**: By iterating through the array once, we can categorize each element into one of the three lists based on its value relative to the pivot. This ensures that we process each element in its original order and efficiently build the desired partitions.
3. **Algorithm Selection**: We use a single pass through the array with conditional checks to determine which list each element belongs to. This approach is efficient with a time complexity of O(n), where n is the number of elements in the array.
4. **Complexity Analysis**: The algorithm runs in linear time, O(n), as each element is processed exactly once. The space complexity is O(n) to store the three partitions.

### Solution Code
```python
class Solution:
    def pivotArray(self, nums: List[int], pivot: int) -> List[int]:
        less = []
        equal = []
        greater = []
        for num in nums:
            if num < pivot:
                less.append(num)
            elif num == pivot:
                equal.append(num)
            else:
                greater.append(num)
        return less + equal + greater
```

### Explanation
1. **Initialization**: We initialize three empty lists: `less`, `equal`, and `greater`.
2. **Iteration**: For each element in the input array:
   - If the element is less than the pivot, it is added to the `less` list.
   - If the element equals the pivot, it is added to the `equal` list.
   - If the element is greater than the pivot, it is added to the `greater` list.
3. **Combining Results**: The final result is obtained by concatenating the `less`, `equal`, and `greater` lists in that order, ensuring all elements are in their correct positions relative to the pivot while preserving their original order within each category.

This approach efficiently categorizes each element in a single pass, ensuring optimal performance and correctness.
