2. **Brute Force Simulation**: For each character in the concatenated string (conceptually), simulate the matching process for `s2`. This involves:
   - Tracking the current position in `s1` and `s2`.
   - For each character in `s2`, find its next occurrence in `s1` starting from the current position.
   - If the character is found, move to the next position in `s2` and reset the position in `s1` if `s2` is fully matched.
3. **Cycle Detection**: Although the brute force approach is used here, cycle detection can be implemented for efficiency with large `n1`. However, for clarity and correctness, the brute force method is chosen.

### Solution Code
```python
class Solution:
    def getMaxRepetitions(self, s1: str, n1: int, s2: str, n2: int) -> int:
        if not set(s2).issubset(set(s1)):
            return 0
        
        total_length = n1 * len(s1)
        if total_length == 0:
            return 0
        
        count = 0
        j = 0
        for i in range(total_length):
            current_char = s1[i % len(s1)]
            if current_char == s2[j]:
                j += 1
                if j == len(s2):
                    count += 1
                    j = 0
        return count // n2
```

### Explanation
1. **Initial Check**: The code first checks if all characters in `s2` are present in `s1`. If not, it returns 0 immediately.
2. **Total Length Calculation**: The total length of the concatenated string is calculated as `n1 * len(s1)`.
3. **Brute Force Simulation**: The code iterates over each character in the conceptual concatenated string. For each character, it checks if it matches the current character in `s2`. If it does, it moves to the next character in `s2`. When `s2` is fully matched, the count is incremented, and `s2` is reset to start from the beginning.
4. **Result Calculation**: The total count of `s2` found is divided by `n2` to determine how many times `s2` can be repeated in the concatenated string, ensuring the result is an integer division.

This approach efficiently counts the occurrences without constructing the entire concatenated string, making it suitable for large values of `n1`. However, for very large `n1`, a cycle detection approach would be more efficient, but the brute force method is used here for clarity and correctness.
