=== RAW ANSWER ===

To solve this problem, we need to count the number of elements in an array that are strictly less than each element in the array. The solution should work for any integer inputs, not just those within a specific range.

### Approach
1. **Frequency Dictionary**: First, we create a dictionary to count the occurrences of each number in the input array. This helps us efficiently determine how many numbers are less than a given number by leveraging the counts of smaller numbers.
2. **Sorted Distinct Numbers**: We then extract the distinct numbers from the dictionary and sort them. This sorted list allows us to efficiently find all numbers less than any given number by breaking early during iteration.
3. **Counting Smaller Numbers**: For each number in the input array, we iterate through the sorted distinct numbers, summing the counts of all numbers that are strictly less than the current number. The sorted order ensures we can break early once we encounter a number greater than or equal to the current number, optimizing the process.

### Solution Code
```python
class Solution:
    def smallerNumbersThanCurrent(self, nums: List[int]) -> List[int]:
        count_dict = {}
        for num in nums:
            count_dict[num] = count_dict.get(num, 0) + 1
        
        distinct_nums = sorted(count_dict.keys())
        
        res = []
        for num in nums:
            total = 0
            for d in distinct_nums:
                if d < num:
                    total += count_dict[d]
                else:
                    break
            res.append(total)
        
        return res
```

### Explanation
1. **Frequency Dictionary**: The dictionary `count_dict` is populated by iterating through the input array `nums`, counting how many times each number appears. This allows us to quickly access the count of any number during our comparisons.
2. **Sorted Distinct Numbers**: The distinct numbers from the dictionary are sorted to facilitate efficient searching. This sorted list helps in breaking the iteration early once a number greater than or equal to the current number is found.
3. **Counting Smaller Numbers**: For each number in `nums`, we iterate through the sorted distinct numbers. For each number in this list that is less than the current number, we add its count to a running total. Once a number greater than or equal to the current number is encountered, we break out of the loop, as all subsequent numbers will also be greater. The total count is then added to the result list.

This approach efficiently handles any integer inputs by leveraging a frequency dictionary and sorted distinct numbers, ensuring optimal performance even for larger arrays within the problem constraints.

class Solution:
    def smallerNumbersThanCurrent(self, nums: List[int]) -> List[int]:
        count_dict = {}
        for num in nums:
            count_dict[num] = count_dict.get(num, 0) + 1
        
        distinct_nums = sorted(count_dict.keys())
        
        res = []
        for num in nums:
            total = 0
            for d in distinct_nums:
                if d < num:
                    total += count_dict[d]
                else:
                    break
            res.append(total)
        
        return res
