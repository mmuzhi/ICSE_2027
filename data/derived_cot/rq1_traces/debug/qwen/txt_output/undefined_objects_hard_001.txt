To solve this problem, we need to reverse every k consecutive nodes in a linked list and return the modified list. The solution involves traversing the linked list in groups of k nodes, reversing each group, and then linking the reversed groups together.

### Approach
1. **Dummy Node**: Use a dummy node to simplify the reversal process, especially handling the head of the list.
2. **Group Identification**: For each group of k nodes, identify the k-th node using a helper function `getKth`.
3. **Reversal**: Reverse the identified group of k nodes. This involves traversing the group and reversing the links between nodes.
4. **Linking Groups**: After reversing a group, update the pointers to link the reversed group correctly to the previous part of the list and move to the next group.

### Solution Code
```python
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        currGroup = dummy
        
        while True:
            kth = self.getKth(currGroup, k)
            if not kth:
                break
            nextGroup = kth.next
            
            prev_node = None
            curr = currGroup.next
            for _ in range(k):
                next_node = curr.next
                curr.next = prev_node
                prev_node = curr
                curr = next_node
            
            currGroup.next = prev_node
            currGroup = nextGroup
        
        return dummy.next

    def getKth(self, node, k):
        while node and k:
            node = node.next
            k -= 1
        return node
```

### Explanation
1. **Dummy Node**: The dummy node simplifies edge cases, especially when reversing the head of the list. It acts as a placeholder to start the reversal process.
2. **Group Identification**: The `getKth` function moves k nodes from the starting node, returning the k-th node if it exists. This helps in identifying the end of each group to be reversed.
3. **Reversal Process**: For each group of k nodes, the reversal is done by iterating k times. In each iteration, the next node's pointer is reversed, effectively reversing the links. This process continues until k nodes are reversed.
4. **Linking Groups**: After reversing a group, the previous group's pointer is updated to point to the start of the reversed group (now the k-th node becomes the new head of the reversed segment). The current group pointer is then moved to the node after the reversed group to process the next segment.

This approach efficiently handles the reversal in groups of k nodes, ensuring the solution is both correct and optimal.
