=== RAW ANSWER ===

To solve this problem, we need to perform a level-order traversal of a binary tree and return the nodes level by level from bottom to top. The initial code attempts to do this but contains several bugs, including an incorrect reversal function and improper handling of an empty tree.

### Approach
1. **Level-Order Traversal (BFS)**: We use a queue to traverse the tree level by level. Starting with the root node, we process each node in the current level, add its children to the queue, and record their values in a list representing the current level.
2. **Reversing Levels**: After collecting all levels from top to bottom, we reverse the list of levels to present them from bottom to top.
3. **Edge Case Handling**: If the tree is empty (root is None), we return an empty list.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        q, ans = [root], []
        while q:
            n = len(q)
            level = []
            for i in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        return ans[::-1]
```

### Explanation
1. **Initialization**: We start by checking if the tree is empty. If it is, we return an empty list.
2. **Queue Processing**: We initialize a queue with the root node. For each level, we process all nodes in the queue, recording their values and adding their children to the queue.
3. **Level Collection**: Each level's node values are collected into a list, which is then appended to the results list.
4. **Reversal**: After processing all levels, we reverse the results list to present the levels from bottom to top and return it.

This approach efficiently handles the level-order traversal and ensures the result is in the desired order by leveraging a simple reversal at the end. The time complexity is O(N) where N is the number of nodes in the tree, as each node is processed exactly once. The space complexity is also O(N) due to the storage required for the queue and the results list.
