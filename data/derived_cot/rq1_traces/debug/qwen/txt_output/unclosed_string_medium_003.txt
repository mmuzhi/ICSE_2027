2. **Base Case**: When the entire string is processed (i.e., four parts are formed), the current combination of parts is joined into a string and added to the result list.
3. **Pruning**: The recursion stops if more than four parts are formed or the end of the string is reached without forming exactly four parts.

### Solution Code
```python
class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i >= len(s):
                return
            if s[i] == '0':
                cur.append(s[i])
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 4 and i + j < len(s):
                if int(s[i:i+j+1]) < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Initialization**: The `res` list stores all valid IP addresses, and `cur` is a temporary list to hold the current combination of parts during backtracking.
2. **Backtracking Function**: The `backtrack` function is defined to recursively explore all valid splits:
   - **Base Case**: If the current index `i` reaches the end of the string and exactly four parts are formed, the parts are joined into a string and added to `res`.
   - **Invalid Cases**: If more than four parts are formed or the index exceeds the string length, the function returns without further processing.
   - **Zero Handling**: If the current character is '0', it is added as a single part, and the function proceeds to the next character.
   - **Non-Zero Handling**: For non-zero characters, the function checks all possible 1 to 3-digit parts (ensuring no leading zeros in multi-digit parts) and proceeds recursively.
3. **Result Compilation**: After exploring all possibilities starting from index 0, the function returns the list of all valid IP addresses found.

This approach efficiently explores all valid IP address combinations by leveraging backtracking and pruning invalid paths early, ensuring optimal performance for the given constraints.
