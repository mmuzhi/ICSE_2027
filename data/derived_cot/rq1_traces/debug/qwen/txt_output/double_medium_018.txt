To solve this problem, we need to determine if it's possible to finish all given courses without any cycles in the prerequisites. The solution involves checking for cycles in a directed graph where each node represents a course, and edges represent prerequisite relationships.

### Approach
1. **Graph Construction**: We first construct a graph where each course points to its prerequisites. This graph is built using a defaultdict where each course (as a key) maps to a list of its prerequisites.
2. **Cycle Detection**: We use a Depth-First Search (DFS) approach to detect cycles. The DFS function checks if a course can be finished by verifying that all its prerequisites can be completed. During the DFS traversal:
   - If a course has no prerequisites, it can be finished immediately.
   - If a course is encountered again during the traversal (indicating a cycle), it cannot be finished.
   - The DFS marks a course as being processed (in the `taken` set) before exploring its prerequisites and removes it once all prerequisites are processed.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        pre = defaultdict(list)
        for a, b in prerequisites:
            pre[b].append(a)
        
        taken = set()

        def dfs(course):
            if not pre[course]:
                return True
            if course in taken:
                return False
            taken.add(course)
            for p in pre[course]:
                if not dfs(p):
                    return False
            taken.remove(course)
            return True
        
        for course in range(numCourses):
            if not dfs(course):
                return False
        return True
```

### Explanation
1. **Graph Construction**: The graph is built by iterating over each prerequisite pair in the input. For each pair `[a, b]`, we add `a` to the list of prerequisites for `b`. This ensures that the graph correctly represents the dependencies between courses.
2. **Cycle Detection with DFS**: The DFS function checks if a course can be finished by ensuring all its prerequisites are met. If a course is found in the `taken` set during traversal, it indicates a cycle, making it impossible to finish the course. The `taken` set is used to keep track of courses currently being processed to detect cycles.
3. **Iterative Check**: The main function iterates over each course and calls the DFS function. If any course cannot be finished (due to a cycle), the function returns `False`. If all courses can be finished, it returns `True`.

This approach efficiently checks for cycles in the prerequisite graph, ensuring that all courses can be completed in some order without any circular dependencies.
