=== RAW ANSWER ===

To solve this problem, we need to determine if it's possible to finish all courses given certain prerequisites. The solution involves checking for cycles in the directed graph formed by the prerequisites, as the presence of a cycle indicates that some courses cannot be completed.

### Approach
1. **Graph Construction**: Represent each course and its prerequisites as a directed graph. Each course points to its prerequisites.
2. **Cycle Detection**: Use Depth-First Search (DFS) to detect cycles in the graph. If a cycle is found, it's impossible to finish all courses, and we return `False`.
3. **Topological Sort**: If no cycles are found, the courses can be topologically sorted, and we return `True`.

The key insight is that a cycle in the graph means there's a dependency loop, making it impossible to schedule all courses. The DFS approach efficiently checks for cycles by marking courses as visited and checking for back edges during traversal.

### Solution Code
```python
class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        pre = defaultdict(list)
        for course, p in prerequisites:
            pre[course].append(p)
        
        taken = set()

        def dfs(course):
            if not pre[course]:
                return True
            if course in taken:
                return False
            taken.add(course)
            for p in pre[course]:
                if not dfs(p):
                    return False
            return True
        
        for course in range(numCourses):
            if course not in taken:
                if not dfs(course):
                    return False
        return True
```

### Explanation
1. **Graph Construction**: The graph is built using a defaultdict where each course maps to its list of prerequisites. This helps in navigating the dependencies during DFS.
2. **DFS Cycle Detection**: The DFS function checks if a course can be finished by ensuring all its prerequisites are satisfied. If a course is revisited during the traversal (indicating a cycle), the function returns `False`.
3. **Cycle Handling**: The `taken` set keeps track of courses that have been processed. If a course is already in the set, it means we've encountered it in the current or previous DFS path, indicating a cycle.
4. **Iterative Check**: The main loop iterates over all courses. If any course cannot be finished (due to a cycle or unresolved prerequisites), the function returns `False`. If all courses are processed without cycles, it returns `True`.

This approach efficiently checks for cycles and ensures all courses can be completed in a valid order if no cycles are found.
