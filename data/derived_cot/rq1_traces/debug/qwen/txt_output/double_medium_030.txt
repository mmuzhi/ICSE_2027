To solve this problem, we need to sort an array of 0s, 1s, and 2s in-place with a time complexity of O(n) and a space complexity of O(1). The solution involves using a three-pointer technique to partition the array into three sections: 0s, 1s, and 2s.

### Approach
1. **Three-Pointer Technique**: We use three pointers, `low`, `mid`, and `high`, to divide the array into three parts:
   - **0s**: All elements from the start up to `low-1`.
   - **1s**: Elements from `low` to `mid-1`.
   - **2s**: Elements from `mid` to `high`.

2. **Initialization**: Initialize `low` and `mid` to 0, and `high` to the last index of the array.

3. **Loop**: Iterate through the array using the `mid` pointer:
   - If the element at `mid` is 0, swap it with the element at `low` and increment both `low` and `mid`.
   - If the element at `mid` is 1, simply increment `mid`.
   - If the element at `mid` is 2, swap it with the element at `high` and decrement `high`.

This approach ensures that all 0s are moved to the beginning, all 2s to the end, and all 1s remain in the middle, thus sorting the array in-place.

### Solution Code
```python
class Solution:
    def sortColors(self, nums: List[int]) -> None:
        low = 0
        mid = 0
        high = len(nums) - 1
        while mid <= high:
            if nums[mid] == 0:
                nums[low], nums[mid] = nums[mid], nums[low]
                low += 1
                mid += 1
            elif nums[mid] == 1:
                mid += 1
            else:  # nums[mid] == 2
                nums[mid], nums[high] = nums[high], nums[mid]
                high -= 1
```

### Explanation
- **Initialization**: The `low` and `mid` pointers start at the beginning of the array, while `high` starts at the end.
- **Loop**: The loop continues as long as `mid` is less than or equal to `high`.
  - **0 Handling**: When encountering a 0, it is swapped with the element at `low`, and both `low` and `mid` are incremented.
  - **1 Handling**: When encountering a 1, only `mid` is incremented, as 1s should be in the middle.
  - **2 Handling**: When encountering a 2, it is swapped with the element at `high`, and `high` is decremented.
- **Termination**: The loop exits when `mid` exceeds `high`, ensuring all elements are sorted in-place.

This approach efficiently partitions the array into three sections with a single pass, making it optimal for the given problem.
