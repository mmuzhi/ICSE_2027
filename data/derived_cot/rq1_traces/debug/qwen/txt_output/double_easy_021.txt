=== RAW ANSWER ===

To solve this problem, we need to determine the maximum depth of a binary tree. The maximum depth is defined as the number of nodes along the longest path from the root node down to the deepest node in the tree. The solution involves traversing the tree while keeping track of the depth at each node and updating the maximum depth encountered during the traversal.

### Approach
The approach uses an iterative Depth-First Search (DFS) approach with a stack to keep track of nodes and their respective depths. The steps are as follows:
1. **Initialization**: Start by initializing a stack with the root node and its depth (1). If the root is None, the maximum depth is 0.
2. **Traversal**: While the stack is not empty, pop the top element (node and depth) from the stack.
3. **Update Maximum Depth**: If the node is not None, update the maximum depth if the current node's depth is greater than the previously recorded maximum.
4. **Push Children**: Push the left and right children of the current node onto the stack with their respective depths (current depth + 1).
5. **Termination**: After processing all nodes, return the recorded maximum depth.

This approach efficiently explores all paths from the root to the deepest nodes, ensuring the maximum depth is correctly identified.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxDepth(self, root: Optional[TreeNode]) -> int:
        stack = [[root, 1]]
        res = 0

        while stack:
            node, depth = stack.pop()
            if node:
                res = max(res, depth)
                stack.append([node.left, depth + 1])
                stack.append([node.right, depth + 1])
        
        return res
```

### Explanation
- **Initialization**: The stack starts with the root node and depth 1. The variable `res` is initialized to 0 to keep track of the maximum depth found.
- **Traversal Loop**: The loop continues as long as there are nodes in the stack. Each iteration processes the top node of the stack.
- **Depth Update**: If the node is not None, its depth is compared with the current maximum (`res`), updating `res` if necessary.
- **Child Processing**: The left and right children of the current node are pushed onto the stack with their depths incremented by 1. This ensures that all paths from the root are explored.
- **Result**: After processing all nodes, the maximum depth (`res`) is returned. If the root is None, the loop doesn't execute, and the function returns 0, which is correct for an empty tree.

This approach efficiently explores the tree using DFS, ensuring all paths are considered while maintaining a clear and concise solution.
