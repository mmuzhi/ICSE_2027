=== RAW ANSWER ===

To solve this problem, we need to find the longest consecutive sequence of integers in an unsorted array. A consecutive sequence is defined as a sequence of integers where each subsequent integer is exactly one more than the previous one (e.g., [1, 2, 3]). The solution involves efficiently checking for these sequences and determining the longest one.

### Approach
1. **Problem Analysis**: The task is to find the longest sequence of consecutive integers in an array. The sequence must be contiguous in value, not necessarily in the array's order. For example, in the array [1, 2, 3, 5], the longest consecutive sequence is [1, 2, 3] with a length of 3.

2. **Intuition**: The key insight is that for each number in the array, if the number minus one is not present in the array, it marks the start of a potential consecutive sequence. We then extend this sequence by checking for each subsequent number (current number + 1) in the array until we can no longer find the next consecutive number.

3. **Algorithm Selection**: 
   - **Set for Lookup**: Using a set allows O(1) average-time complexity for checking the existence of numbers.
   - **Iterate and Extend**: For each number in the set, if the previous number (num - 1) is not found, we start a sequence from this number and extend it by checking each subsequent number (num + 1, num + 2, etc.) until the next number is not found in the set.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n), where n is the number of unique elements in the array. Each element is processed once, and each check for the next number is O(1).
   - **Space Complexity**: O(n), due to the storage required for the set.

### Solution Code
```python
class Solution:
    def findLHS(self, nums: List[int]) -> int:
        num_set = set(nums)
        longest = 0
        for num in num_set:
            if num - 1 not in num_set:
                current = num
                while current + 1 in num_set:
                    current += 1
                longest = max(longest, current - num + 1)
        return longest
```

### Explanation
1. **Initialization**: Convert the input array into a set for O(1) lookups. Initialize `longest` to keep track of the maximum consecutive sequence length found.

2. **Iteration**: For each number in the set, check if the previous number (num - 1) is not present in the set. If true, this number is the start of a consecutive sequence.

3. **Sequence Extension**: Starting from this number, keep incrementing the current number and check if the next number (current + 1) exists in the set. Continue this process until the next number is not found.

4. **Update Length**: Calculate the length of the consecutive sequence from the start number to the last found number (current) and update `longest` if this sequence is longer than the previously found sequences.

5. **Result**: After processing all numbers, return the length of the longest consecutive sequence found. If no such sequence exists (other than single elements), return 0.

This approach efficiently checks all possible consecutive sequences by leveraging set operations and linear traversal, ensuring optimal performance.
