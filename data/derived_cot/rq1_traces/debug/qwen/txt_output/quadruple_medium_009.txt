=== RAW ANSWER ===

To solve this problem, we need to restore all possible valid IP addresses from a given string. An IP address consists of four parts separated by dots, where each part is a number between 0 and 255. The solution involves using a backtracking approach to explore all possible ways to split the string into four parts, ensuring each part meets the validity criteria.

### Approach
1. **Backtracking with Base Case**: The solution uses a recursive backtracking method to explore all possible splits of the string into four parts. The base case checks if the entire string has been processed and exactly four parts have been formed.
2. **Handling Leading Zeros**: If a part starts with '0', it can only be a single digit (0). Any subsequent digits would form an invalid number (e.g., "01" is invalid).
3. **Non-Zero Parts**: For parts that do not start with '0', the algorithm considers parts of up to three digits (since a part cannot exceed three digits in a valid IP address). Each part must be less than 256.
4. **Early Termination**: The algorithm terminates early if more than four parts are formed or if the current index exceeds the string length, optimizing the backtracking process.

### Solution Code
```python
class Solution():
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i > len(s):
                return
            if s[i] == '0':
                cur.append(s[i])
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 3 and i + j < len(s):
                if int(s[i:i+j+1]) < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Initialization**: The `res` list stores all valid IP addresses, and `cur` is a temporary list to hold the current parts being formed.
2. **Backtracking Function**: The `backtrack` function recursively processes the string starting from index `i`:
   - **Base Case**: If the end of the string is reached and exactly four parts are formed, the current parts are joined into a string and added to `res`.
   - **Invalid Cases**: If more than four parts are formed or the index exceeds the string length, the function returns.
   - **Zero Handling**: If the current character is '0', it is added as a part, and the function proceeds to the next character.
   - **Non-Zero Handling**: For non-zero characters, the function checks all possible parts of up to three digits, ensuring they are less than 256. Each valid part is added, and the function proceeds recursively.
3. **Result Compilation**: After the backtracking completes, the `res` list contains all valid IP addresses, which is returned as the result.

This approach efficiently explores all valid splits using backtracking, ensuring correctness and optimal performance by pruning invalid paths early.
