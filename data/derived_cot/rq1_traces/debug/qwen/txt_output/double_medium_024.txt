To solve this problem, we need to determine if there exists a path from the top-left corner (0,0) to the bottom-right corner (m-1, n-1) of a given binary grid, moving only right and down, such that all cells along the path are reachable from the destination when moving only left and up. 

### Approach
1. **Reverse BFS (Breadth-First Search)**: Start from the destination cell (m-1, n-1) and perform a BFS moving only left and up. Mark all reachable cells with the value 2. This step identifies all cells that can lead to the destination when moving in reverse directions.
2. **Forward BFS**: Starting from the source cell (0,0), perform a BFS moving only right and down. The BFS can only traverse cells marked as 2 from the reverse BFS step. This ensures that the path from the source to the destination exists and is composed of cells that can reach the destination in reverse.

### Solution Code
```python
from collections import deque

class Solution:
    def isPossibleToCutPath(self, grid: List[List[int]]) -> bool:
        m, n = len(grid), len(grid[0])
        dirs_rev = [(-1, 0), (0, -1)]
        dq_rev = deque()
        dq_rev.append((m-1, n-1))
        grid[m-1][n-1] = 2
        while dq_rev:
            i, j = dq_rev.popleft()
            for di, dj in dirs_rev:
                ni, nj = i + di, j + dj
                if 0 <= ni < m and 0 <= nj < n and grid[ni][nj] == 1:
                    grid[ni][nj] = 2
                    dq_rev.append((ni, nj))
        
        dq = deque()
        if grid[0][0] == 2:
            dq.append((0, 0))
            grid[0][0] = 0
        while dq:
            i, j = dq.popleft()
            if i == m-1 and j == n-1:
                return True
            for di, dj in [(0, 1), (1, 0)]:
                ni, nj = i + di, j + dj
                if 0 <= ni < m and 0 <= nj < n and grid[ni][nj] == 2:
                    dq.append((ni, nj))
                    grid[ni][nj] = 0
        return False
```

### Explanation
1. **Reverse BFS**: The algorithm starts from the bottom-right cell (m-1, n-1) and explores all cells reachable by moving left and up. Each visited cell is marked with 2. This step ensures that only cells contributing to a valid path from the source to the destination are considered.
2. **Forward BFS**: Starting from the top-left cell (0,0), the algorithm checks if the starting cell is marked as 2 (indicating it can reach the destination in reverse). If so, it proceeds to explore right and down directions, only traversing cells marked as 2. Each visited cell is marked as 0 to prevent revisiting. If the destination cell (m-1, n-1) is reached during this BFS, the function returns True; otherwise, it returns False after exhausting all possibilities.

This approach efficiently checks for the existence of a valid path by leveraging reverse BFS to mark potential path cells and forward BFS to verify the path's existence, ensuring optimal performance and correctness.
