To solve this problem, we need to partition a given string into the fewest number of substrings such that each letter appears in only one substring. The solution involves identifying the last occurrence of each character in the string and then using a greedy approach to determine the optimal partitions.

### Approach
1. **Identify Last Occurrences**: First, we create a dictionary to store the last occurrence index of each character in the string. This helps in knowing the farthest position each character appears.
2. **Greedy Partitioning**: Starting from the beginning of the string, we determine the end of the current partition by checking the last occurrence of each character encountered within the current partition. If any character's last occurrence is beyond the current partition's end, we extend the partition to include that position.
3. **Collect Results**: Once the end of a partition is determined, we record its length and move the start index to the next position after the end of the current partition.

This approach ensures that each partition contains all occurrences of its characters, thus satisfying the condition that each letter appears in only one substring.

### Solution Code
```python
class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        L = len(s)
        last = {char: idx for idx, char in enumerate(s)}
        i, ans = 0, []
        while i < L:
            end = last[s[i]]
            j = i + 1
            while j < end:
                if last[s[j]] > end:
                    end = last[s[j]]
                j += 1
            ans.append(end - i + 1)
            i = end + 1
        return ans
```

### Explanation
1. **Dictionary for Last Occurrences**: The dictionary `last` is built by iterating through the string once, mapping each character to its highest index (last occurrence).
2. **Partitioning Loop**: The loop starts at the beginning of the string (`i = 0`). For each character at position `i`, the initial partition end is set to the last occurrence of that character. The inner loop then checks each subsequent character within the current partition to see if its last occurrence extends beyond the current end. If it does, the end is updated.
3. **Recording Partitions**: Once the end of the current partition is confirmed, its length is recorded in the result list. The start index is then moved to the position right after the current partition's end.
4. **Termination**: The loop continues until the entire string is processed, ensuring all characters are included in their respective partitions.

This method efficiently partitions the string by leveraging the last occurrence information, ensuring optimal and correct partitions with minimal computational overhead.
