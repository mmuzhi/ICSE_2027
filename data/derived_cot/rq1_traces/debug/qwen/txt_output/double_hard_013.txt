            class Solution:
                def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
                    if not head or k == 1:
                        return head

                    # Find the kth node
                    def getKth(node, k):
                        while node and k:
                            node = node.next
                            k -= 1
                        return node

                    prev = None
                    curr = head
                    while curr:
                        # Find the kth node starting from curr
                        kth = getKth(curr, k)
                        if not kth:
                            break

                        nextGroup = kth.next

                        # Reverse the segment from curr to kth
                        prev_node = None
                        curr_node = curr
                        while curr_node != kth:

To solve this problem, we need to reverse every k nodes in a given linked list. The approach involves traversing the linked list, reversing each group of k nodes, and then linking the reversed groups together. The key challenge is to correctly identify the start and end of each group and to perform the reversal while maintaining the integrity of the linked list structure.

### Approach
1. **Dummy Node**: Use a dummy node to simplify edge cases, especially when the entire list or part of it needs to be reversed.
2. **Group Identification**: For each group of k nodes, identify the k-th node from the current starting position using a helper function.
3. **Reversal**: Reverse the identified group of k nodes. This involves iterating through the nodes, reversing their pointers, and keeping track of the new head of the reversed segment.
4. **Linking Groups**: After reversing a group, link the previous group's tail to the new head of the reversed segment. Then, move the pointer to the end of the reversed segment to start processing the next group.

class Solution:

1. **Dummy Node**: The dummy node acts as a placeholder at the beginning of the list, simplifying the reversal process and handling edge cases where the head of the list might be reversed.
2. **Group Identification**: The `getKth` function is used to find the k-th node from the current starting node, which marks the end of the current group to be reversed.
3. **Reversal**: For each group of k nodes, the code iterates through the nodes, reversing their pointers. This is done by maintaining a `prev_node` (initially `None`) and `curr_node` (the current node being processed), and swapping the `next` pointers.
4. **Linking Groups**: After reversing a group, the `prev` pointer is updated to point to the new head of the reversed segment (`prev_node`). The `prev` pointer is then moved to the end of the reversed segment (`kth`) to start processing the next group.
5. **Termination**: The loop terminates when there are fewer than k nodes remaining in the list, ensuring only complete groups of k nodes are reversed.
