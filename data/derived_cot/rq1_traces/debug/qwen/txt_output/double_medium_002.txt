To solve this problem, we need to find a peak element in an array. A peak element is defined as an element that is greater than or equal to its neighbors. The array can have multiple peaks, and we can return any of them. The solution uses a binary search approach to efficiently locate a peak element.

### Approach
1. **Binary Search Setup**: We initialize two pointers, `low` and `high`, to the start and end of the array, respectively.
2. **Edge Case Handling**: If the array has only one element, that element is trivially a peak.
3. **Binary Search Loop**: While `low` is less than or equal to `high`, we compute the middle index `mid`.
4. **Peak Check**: For the middle element at `mid`, check if it is a peak by verifying if it is greater than or equal to its left neighbor (if it exists) and its right neighbor (if it exists).
5. **Adjust Search Range**: If the middle element is not a peak, determine the direction of the search:
   - If the middle element is less than its right neighbor, the peak must be on the right side, so we move `low` to `mid + 1`.
   - Otherwise, the peak must be on the left side, so we move `high` to `mid - 1`.

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        
        if n == 1:
            return 0
        
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] >= nums[mid-1]) and (mid == n-1 or nums[mid] >= nums[mid+1]):
                return mid
            elif nums[mid] <= nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

### Explanation
- **Binary Search Setup**: The binary search is initialized with `low` at the start (0) and `high` at the end (n-1) of the array.
- **Edge Case Handling**: If the array length is 1, the single element is the peak.
- **Binary Search Loop**: The loop continues as long as `low` is less than or equal to `high`. The middle index `mid` is calculated in each iteration.
- **Peak Check**: The middle element is checked to see if it is a peak by comparing it with its left and right neighbors. If it is a peak, the index is returned immediately.
- **Adjust Search Range**: If the middle element is not a peak, the search is narrowed down:
  - If the middle element is less than its right neighbor, the peak must be on the right side, so `low` is set to `mid + 1`.
  - If the middle element is greater than its right neighbor, the peak must be on the left side, so `high` is set to `mid - 1`.
This approach efficiently narrows down the search space by leveraging binary search, ensuring optimal performance with a time complexity of O(log n).
