To solve this problem, we need to increment a number represented as a list of digits by one and return the result as a list of digits. The solution involves converting the list of digits into a string, then into an integer, incrementing the integer by one, and finally converting the result back into a list of digits.

### Approach
1. **Convert List to String**: The given list of digits is converted into a string by joining each digit as a string. For example, the list `[1, 2, 3]` becomes the string `"123"`.
2. **Convert String to Integer and Increment**: The string is then converted into an integer, incremented by one, and stored. For example, the integer `123` becomes `124`.
3. **Convert Integer Back to String and List**: The incremented integer is converted back into a string, and each character of the string is converted into an integer to form the final list. For example, the string `"124"` becomes the list `[1, 2, 4]`.

This approach efficiently handles the increment operation by leveraging string manipulation and integer conversion, ensuring correctness even for numbers ending with trailing 9s (e.g., `[9, 9, 9]` becomes `[1, 0, 0, 0]`).

### Solution Code
```python
class Solution:
    def plusOne(self, digits: List[int]) -> List[int]:
        s = ''.join(map(str, digits))
        i = int(s) + 1
        li = list(map(int, str(i)))
        return li
```

### Explanation
1. **String Conversion**: The list of digits is converted into a string by joining each element with `map(str, digits)`, ensuring each digit is treated as a string. For example, `[1, 2, 3]` becomes `"123"`.
2. **Integer Increment**: The string is converted to an integer using `int(s)`, incremented by one, and stored in the variable `i`. For example, `"123"` becomes `124`.
3. **Back to List**: The incremented integer is converted back into a string, and each character of the string is converted into an integer using `map(int, str(i))`, resulting in a list of digits. For example, `124` becomes `[1, 2, 4]`.

This method efficiently handles the increment operation and ensures the result is returned as a list of digits, even for edge cases like all 9s.
