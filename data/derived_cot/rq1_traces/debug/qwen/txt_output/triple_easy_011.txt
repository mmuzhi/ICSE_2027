=== RAW ANSWER ===

To solve this problem, we need to find the complement of a given integer by flipping all its bits in its binary representation. The complement should be calculated based on the minimal binary representation of the number, which means we consider the binary digits from the least significant bit (LSB) to the most significant bit (MSB). 

### Approach
1. **Special Case Handling for Zero**: If the input number is zero, its complement is 1, as flipping the single bit (0) results in 1.
2. **Bit Processing**: For non-zero numbers, process each bit from the LSB to the MSB. For each bit, if it is 1, append '0' to the complement string; if it is 0, append '1'.
3. **String Reversal**: The bits are processed from LSB to MSB, so the resulting string needs to be reversed to form the correct binary representation of the complement.
4. **Conversion to Integer**: Convert the reversed string from binary to an integer to get the final result.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        if num == 0:
            return 1
        com = ''
        while num > 0:
            if num % 2 == 1:
                com += '0'
            else:
                com += '1'
            num = num // 2
        return int(com[::-1], 2)
```

### Explanation
1. **Special Case Handling**: The code first checks if the input number is zero. If so, it directly returns 1, as the complement of zero (in a single-bit representation) is 1.
2. **Bit Processing Loop**: The loop continues as long as the number is greater than zero. In each iteration, the least significant bit is checked. If it is 1, '0' is appended to the complement string; if it is 0, '1' is appended. The number is then divided by 2 to shift the bits right, effectively moving to the next bit.
3. **String Reversal**: After processing all bits, the complement string is reversed because the bits were processed from LSB to MSB. Reversing ensures the string is in the correct order for binary interpretation.
4. **Binary Conversion**: The reversed string is converted from binary to an integer, which gives the complement of the original number. For example, the binary string "010" (which is 2 in decimal) is the complement of the number 5 (binary "101").

This approach efficiently computes the complement by leveraging bit manipulation and string operations, ensuring correctness for all non-negative integers.
