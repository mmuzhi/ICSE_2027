The solution involves using a trie data structure to efficiently check for the presence of forbidden substrings within any part of the current substring being considered.

### Approach
1. **Trie Construction**: Build a trie from all forbidden substrings. Each node in the trie represents a character, and an "end" marker indicates the completion of a forbidden substring.
2. **Forbidden Substring Check**: For each substring of the word, starting from the end and moving backwards, check if it contains any forbidden substring. The check involves traversing the trie for every possible starting point within the substring.
3. **Adjusting Substring Boundaries**: If a forbidden substring is found within the current substring, adjust the end boundary (j) to exclude the forbidden substring. Specifically, move the start of the substring to just after the beginning of the forbidden substring.
4. **Tracking Maximum Length**: Keep track of the maximum length of valid substrings encountered during the iteration.
