To solve this problem, we need to determine for each element in an array how many numbers are strictly smaller than it. The initial approach using a fixed-size array of 101 elements was faulty because it could not handle numbers outside the range [0, 100]. 

### Approach
1. **Frequency Dictionary**: First, we count the frequency of each number in the input array using a dictionary. This allows us to handle any integer values, not just those in the range [0, 100].
2. **Sorted Keys**: We then sort the keys (unique numbers) from the dictionary to process them in ascending order.
3. **Prefix Sum Array**: We build a prefix sum array where each element at index `i` contains the sum of frequencies of all numbers less than or equal to the corresponding key. This helps in quickly calculating the count of numbers smaller than any given number.
4. **Binary Search for Efficiency**: For each number in the input array, we use binary search to find the largest number in the sorted keys that is less than the current number. The prefix sum up to this number gives the count of numbers smaller than the current number.

### Solution Code
```python
from bisect import bisect_left
from collections import defaultdict

class Solution:
    def smallerNumbersThanCurrent(self, nums: List[int]) -> List[int]:
        freq = defaultdict(int)
        for num in nums:
            freq[num] += 1
        
        keys = sorted(freq.keys())
        prefix = []
        for key in keys:
            if prefix:
                prefix.append(prefix[-1] + freq[key])
            else:
                prefix.append(freq[key])
        
        res = []
        for num in nums:
            idx = bisect_left(keys, num)
            if idx == 0:
                res.append(0)
            else:
                res.append(prefix[idx-1])
        return res
```

1. **Frequency Dictionary**: The dictionary `freq` stores how many times each number appears in the input array. This helps in efficiently calculating the count of numbers smaller than any given number.
2. **Sorted Keys**: The keys from the dictionary are sorted to process them in ascending order. This allows us to use binary search to quickly locate the position of any number.
3. **Prefix Sum Array**: The `prefix` array is built by iterating through the sorted keys. Each entry in `prefix` at index `i` is the sum of frequencies of all numbers up to the `i-th` key. This enables us to compute the count of numbers smaller than a given number in constant time.
4. **Binary Search**: For each number in the input array, we use `bisect_left` to find the position where the number would fit in the sorted keys. The count of numbers smaller than the current number is then the prefix sum up to the key just before this position. If the number is the smallest in the array, the count is zero.
