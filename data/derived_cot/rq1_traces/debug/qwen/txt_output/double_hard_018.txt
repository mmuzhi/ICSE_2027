To solve this problem, we need to find the longest cycle in a directed graph. The solution involves traversing the graph using Depth-First Search (DFS) to detect cycles and determine their lengths. The key insight is to track visited nodes and their depths during traversal to identify cycles efficiently.

### Approach
1. **Graph Traversal**: We use DFS to explore each node in the graph. For each unvisited node, we initiate a DFS traversal.
2. **Cycle Detection**: During traversal, if we encounter a node that has already been visited in the current DFS path, a cycle is detected. The length of the cycle is calculated as the difference in depths of the current node and the encountered node plus one.
3. **Tracking Visited Nodes**: We maintain a visited array to keep track of globally visited nodes and a separate set for nodes in the current DFS path to detect cycles.
4. **Depth Tracking**: For each node in the current DFS path, we record its depth (number of nodes visited from the start) to compute the cycle length when a node is revisited.

### Solution Code
```python
class Solution:
    def longestCycle(self, edges: List[int]) -> int:
        n = len(edges)
        visited = [False] * n
        mx = -1
        
        for i in range(n):
            if not visited[i]:
                stack = [i]
                visited[i] = True
                depth = [-1] * n
                depth[i] = 0
                current_set = {i}
                while stack:
                    x = stack.pop()
                    y = edges[x]
                    if y == -1:
                        continue
                    if not visited[y]:
                        visited[y] = True
                        depth[y] = depth[x] + 1
                        stack.append(y)
                        current_set.add(y)
                    else:
                        if y in current_set:
                            cycle_length = depth[x] - depth[y] + 1
                            if cycle_length > mx:
                                mx = cycle_length
                for node in current_set:
                    depth[node] = -1
        return mx
```

### Explanation
1. **Initialization**: We start by initializing a visited array to keep track of nodes that have been processed globally and a variable `mx` to store the maximum cycle length found.
2. **DFS Traversal**: For each unvisited node, we start a DFS. During traversal, we mark nodes as visited and record their depths.
3. **Cycle Detection**: If during traversal we encounter a node that is already in the current DFS path (`current_set`), a cycle is detected. The cycle length is calculated using the depths of the current node and the encountered node.
4. **Resetting Depths**: After completing the DFS for a component, we reset the depths of nodes in that component to ensure correct tracking in subsequent traversals.
5. **Result**: The maximum cycle length encountered during the traversal is returned. If no cycles are found, the function returns -1.

This approach efficiently detects cycles in a directed graph using DFS and tracks node depths to determine cycle lengths, ensuring optimal performance and correctness.
