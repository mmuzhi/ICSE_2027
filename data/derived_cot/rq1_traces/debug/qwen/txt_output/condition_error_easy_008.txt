The next greater element for an element `x` in `nums1` is the first element in `nums2` (to the right of `x`'s first occurrence) that is greater than `x`. If no such element exists, we return -1 for that element in `nums1`.

### Approach
1. **Problem Analysis**: The task involves iterating over each element in `nums1` and finding its next greater element in `nums2`. The challenge is to efficiently locate the first occurrence of each element in `nums2` and then scan to the right for a greater element.
2. **Intuition**: For each element in `nums1`, we first locate its first occurrence in `nums2`. If the element is not found, the result for that element is -1. Otherwise, we scan the subsequent elements in `nums2` starting from the element's position in `nums2` to find the next greater element.
3. **Algorithm Selection**: The solution involves a nested loop where the outer loop processes each element in `nums1`, and the inner loop scans `nums2` from the element's position to the end. This approach ensures we find the next greater element efficiently for each element in `nums1`.
4. **Complexity Analysis**: The algorithm has a time complexity of O(n*m), where `n` is the length of `nums1` and `m` is the length of `nums2`. This is because for each element in `nums1`, we may scan up to `m` elements in `nums2`. The space complexity is O(1) as we only use a few extra variables.

### Solution Code
```python
class Solution:
    def nextGreaterElement(self, nums1: List[int], nums2: List[int]) -> List[int]:
        out = []
        for x in nums1:
            try:
                idx = nums2.index(x)
            except ValueError:
                out.append(-1)
                continue
            found = False
            for j in range(idx + 1, len(nums2)):
                if nums2[j] > x:
                    out.append(nums2[j])
                    found = True
                    break
            if not found:
                out.append(-1)
        return out
```

### Explanation
1. **Initialization**: We start with an empty list `out` to store the results.
2. **Processing Each Element in `nums1`**: For each element `x` in `nums1`, we attempt to find its first occurrence in `nums2` using `nums2.index(x)`. If `x` is not found, we append -1 to `out` and move to the next element in `nums1`.
3. **Finding the Next Greater Element**: If `x` is found in `nums2`, we then scan the elements in `nums2` starting from the next index of `x`'s position. The first element greater than `x` encountered during this scan is appended to `out`. If no such element is found, -1 is appended to `out`.
4. **Result Compilation**: After processing all elements in `nums1`, the compiled results are returned in `out`.

This approach efficiently addresses the problem by leveraging direct index lookup and linear scans, ensuring correctness while maintaining clarity.
