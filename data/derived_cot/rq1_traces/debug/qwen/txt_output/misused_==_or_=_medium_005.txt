=== RAW ANSWER ===

To solve this problem, we need to determine if a given singly-linked list is a sub-path of a binary tree. A sub-path means that the linked list can be found as a contiguous downward path in the binary tree, starting from any node in the tree.

### Approach
1. **Base Cases Handling**: 
   - If the linked list is empty (i.e., `head` is `None`), it is trivially a sub-path, so return `True`.
   - If the tree node is `None` but the linked list is not, it means we haven't matched the entire list, so return `False`.

2. **Matching Current Node**:
   - If the current node's value in the tree matches the current node's value in the linked list, we need to check if the remaining part of the linked list (i.e., `head.next`) can be found in either the left or right subtree of the current tree node.
   - If the remaining part of the linked list is empty (i.e., `head.next` is `None`), it means the entire list has been matched, so return `True`.

3. **Recursive Check**:
   - If the current node's value does not match, recursively check the left and right subtrees of the current node for the entire linked list.

### Solution Code
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def isSubPath(self, head: Optional[ListNode], root: Optional[TreeNode]) -> bool:
        if head is None:
            return True
            
        if root is None:
            return False
            
        if head.val == root.val:
            if head.next is None:
                return True
            return self.isSubPath(head.next, root.left) or self.isSubPath(head.next, root.right)
            
        return self.isSubPath(head, root.left) or self.isSubPath(head, root.right)
```

### Explanation
- **Base Cases**: The function first checks if the linked list is empty (`head is None`), in which case it returns `True` since an empty list is trivially a sub-path. If the tree node is `None` but the linked list is not, it returns `False` because we cannot match the remaining list nodes.
- **Current Node Matching**: If the current tree node's value matches the current linked list node's value, the function checks if the remaining linked list nodes can be found in either the left or right subtree of the current tree node. If the remaining list is empty (`head.next is None`), it means the entire list has been matched, so it returns `True`.
- **Recursive Check**: If the current node's value does not match, the function recursively checks both the left and right subtrees for the entire linked list. This ensures that the linked list is checked in all possible downward paths of the binary tree.

This approach efficiently checks all possible paths in the binary tree for the given linked list by leveraging recursion and careful handling of base cases and node matching.
