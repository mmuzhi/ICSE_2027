The initial approach in the provided code is flawed, so we need to implement a correct method that efficiently counts the numbers in each digit-length block and jumps to the appropriate block until we find the k-th number.

### Approach
1. **Digit-Length Block Processing**: The numbers from 1 to n can be divided into blocks based on their digit lengths. For example, numbers with 1 digit (1-9), 2 digits (10-99), etc.
2. **Counting Numbers in Each Block**: For each block, calculate the count of numbers that fall within the range from the start of the block to the minimum of n or the end of the block (which is `10^d - 1` for a block of d digits).
3. **Jump to the Correct Block**: Subtract the count of numbers in each block from k until k falls within the current block. The k-th number in the block is then found by adding (k-1) to the start of the block.

### Solution Code
```python
class Solution:
    def findKthNumber(self, n: int, k: int) -> int:
        d = 1
        start = 1
        while start <= n:
            end = min(n, 10**d - 1)
            cnt = end - start + 1
            if k <= cnt:
                return start + k - 1
            else:
                k -= cnt
                d += 1
                start = 10**(d-1)
```

### Explanation
1. **Initialization**: Start with the smallest digit length (d = 1) and the first number in the block (start = 1).
2. **Block Processing**: For each digit length d, calculate the end of the block as the minimum of n and `10^d - 1`. The count of numbers in this block is `end - start + 1`.
3. **Check k**: If k is within the current block's count, return the k-th number by adding (k-1) to the start of the block. Otherwise, subtract the block's count from k and move to the next digit length by multiplying the start by 10 (or setting it to `10^(d-1)` for the new digit length).
4. **Termination**: The loop terminates when the k-th number is found within a block, ensuring efficiency by processing entire blocks at a time rather than individual numbers.

This approach efficiently narrows down the search to the correct block and then locates the exact number within that block, making it optimal for large values of n and k.
