The key insight is that the last element of the postorder traversal will always be the root of the current subtree. We then use the inorder traversal to split the subtree into left and right parts. The left part of the inorder traversal corresponds to the left subtree, and the right part corresponds to the right subtree. The postorder traversal is then split accordingly to match these parts.

### Approach
1. **Mapping Inorder Values**: Create a dictionary to map each value in the inorder traversal to its index for quick lookup.
2. **Recursive Tree Construction**: Use a helper function to recursively build the tree:
   - **Base Case**: If the current segment of the inorder traversal is empty (i.e., left index exceeds right index), return `None`.
   - **Root Selection**: The root of the current subtree is the last element of the current segment in the postorder traversal.
   - **Splitting Segments**: Using the root's position in the inorder traversal, split the inorder segment into left and right parts. The corresponding segments in the postorder traversal are adjusted to match these splits.
   - **Recursive Calls**: Recursively build the left and right subtrees using the appropriate segments of the inorder and postorder traversals.

# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def buildTree(self, inorder: List[int], postorder: List[int]) -> Optional[TreeNode]:
        inorder_map = {val: idx for idx, val in enumerate(inorder)}
        n = len(inorder)
