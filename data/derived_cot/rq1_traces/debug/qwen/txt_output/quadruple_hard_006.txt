To solve this problem, we need to find the longest substring of a given string that does not contain any forbidden substrings. The solution involves using a trie data structure to efficiently check for forbidden substrings and a two-pointer technique to determine the longest valid substring.

### Approach
1. **Trie Construction**: Build a trie from all forbidden substrings. Each node in the trie will mark the end of a forbidden substring.
2. **Helper Function**: Create a helper function `isForbidden` that checks if a given substring contains any forbidden substring by traversing the trie. If during traversal, an "end" marker is encountered, it returns the length of the forbidden substring found.
3. **Two-Pointer Technique**: Iterate over the string from the end to the beginning, adjusting the right pointer to exclude any forbidden substrings found. The length of the longest valid substring is tracked and updated during the iteration.

### Solution Code
```python
class Solution:
    def longestValidSubstring(self, word: str, forbidden: List[str]) -> int:
        if not word:
            return 0
        
        trie = {}
        for f in forbidden:
            t = trie
            for c in f:
                if c not in t:
                    t[c] = {}
                t = t[c]
            t["end"] = True
        
        def isForbidden(s):
            t = trie
            for idx, c in enumerate(s):
                if c not in t:
                    return False
                t = t[c]
                if "end" in t:
                    return idx + 1
            return False
        
        n = len(word)
        j = n + 1
        res = 0
        for i in range(n - 1, -1, -1):
            truc = isForbidden(word[i:j])
            if truc:
                j = i + truc
            res = max(res, j - i)
        return res
```

### Explanation
1. **Trie Construction**: The trie is built by iterating over each forbidden substring. For each character in a forbidden substring, a corresponding node is created in the trie. An "end" marker is added at the final node to indicate the completion of a forbidden substring.
2. **Helper Function**: The `isForbidden` function checks if a substring contains any forbidden substring by traversing the trie. If a character is not found in the current trie node, the function returns `False`. If an "end" marker is encountered during traversal, it returns the length of the forbidden substring found.
3. **Two-Pointer Technique**: The main loop starts from the end of the string and moves leftwards. For each position, it checks the substring from the current position to the right pointer using `isForbidden`. If a forbidden substring is found, the right pointer is adjusted to exclude the forbidden substring. The maximum length of valid substrings encountered during the iteration is stored and returned as the result.

This approach efficiently checks for forbidden substrings using a trie and dynamically adjusts the substring boundaries to find the longest valid substring, ensuring optimal performance.
