The function starts from the first topping in the doubled list and recursively considers both including and excluding the current topping. The base case handles scenarios where the remaining target value is negative or all toppings have been processed.

4. **Optimization**: Memoization (`@cache`) is used to store intermediate results and avoid redundant calculations, significantly improving the efficiency of the solution. The recursion explores all possible sums of toppings that can be added to a base cost to get as close as possible to the target.

5. **Complexity Analysis**: The time complexity is determined by the number of unique states in the memoization cache, which is O(n * x), where n is the number of toppings (doubled) and x is the target value. The space complexity is also O(n * x) due to the memoization cache.

from functools import cache

class Solution:
    def closestCost(self, baseCosts: List[int], toppingCosts: List[int], target: int) -> int:
