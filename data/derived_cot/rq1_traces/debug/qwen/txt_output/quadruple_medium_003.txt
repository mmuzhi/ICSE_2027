To solve this problem, we need to perform a level-order traversal (BFS) of a binary tree and return the nodes' values from bottom to top. The initial code had several issues, including incorrect handling of `None` children, improper list reversal, and inefficient queue operations. 

### Approach
1. **Check for Empty Tree**: If the root is `None`, return an empty list.
2. **Breadth-First Search (BFS)**: Use a queue to traverse the tree level by level. For each node, append its value to the current level's list and its children (if they exist) to the queue.
3. **Reverse the Result**: After collecting all levels from top to bottom, reverse the list of levels to get the bottom-up order.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        q, ans = [root], []
        while q:
            n = len(q)
            level = []
            for i in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        return ans[::-1]
```

### Explanation
1. **Initial Check**: The function first checks if the root is `None` and returns an empty list if true.
2. **Queue Initialization**: A queue is initialized with the root node to start the BFS.
3. **Level Processing**: For each node in the queue, its value is added to the current level's list. The node's left and right children (if they exist) are added to the queue for processing in subsequent levels.
4. **Result Construction**: After processing all nodes at the current level, the level's values are added to the result list. This process continues until the queue is empty.
5. **Reverse Result**: The final result is obtained by reversing the list of levels, which transforms the top-down traversal into a bottom-up traversal.

This approach efficiently handles the tree traversal and ensures the result is in the correct order while avoiding common pitfalls like processing `None` children or incorrect list reversal.
