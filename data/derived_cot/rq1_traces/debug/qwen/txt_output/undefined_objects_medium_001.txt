=== RAW ANSWER ===

To solve this problem, we need to count the number of reachable nodes starting from node 0 in a graph, while avoiding any nodes specified in the restricted list. The solution involves performing a Breadth-First Search (BFS) starting from node 0, ensuring that we do not visit any restricted nodes during the traversal.

### Approach
1. **Graph Construction**: Build an adjacency list to represent the graph from the given edges.
2. **Restricted Nodes Handling**: Mark all nodes in the restricted list as visited before starting the BFS to ensure they are not processed.
3. **BFS Initialization**: Start BFS from node 0. If node 0 is restricted, return 0 immediately as no nodes can be reached.
4. **BFS Execution**: For each node dequeued, increment the count of reachable nodes and enqueue all its unvisited neighbors. Skip nodes that are already visited or restricted.
5. **Result Calculation**: The BFS traversal ensures that all reachable nodes are counted, and the result is the total count of such nodes.

### Solution Code
```python
from collections import deque, defaultdict

class Solution:
    def reachableNodes(self, n: int, edges: List[List[int]], restricted: List[int]) -> int:
        adj_list = defaultdict(list)
        for u, v in edges:
            adj_list[u].append(v)
            adj_list[v].append(u)
        
        visited = set(restricted)
        if 0 in visited:
            return 0
        
        que = deque([0])
        visited.add(0)
        count = 1
        
        while que:
            cur = que.popleft()
            for neighbor in adj_list[cur]:
                if neighbor not in visited:
                    visited.add(neighbor)
                    count += 1
                    que.append(neighbor)
        
        return count
```

### Explanation
1. **Graph Construction**: The adjacency list is built using a defaultdict where each node points to its connected neighbors via the given edges.
2. **Restricted Nodes Handling**: All nodes in the restricted list are marked as visited initially. If node 0 is restricted, the function returns 0 immediately since it's the starting point and cannot be visited.
3. **BFS Initialization**: The BFS starts from node 0, which is added to the queue and marked as visited. The count is initialized to 1 to account for node 0.
4. **BFS Execution**: For each node processed, its neighbors are checked. If a neighbor hasn't been visited, it is added to the queue, marked as visited, and the count is incremented.
5. **Result Calculation**: The BFS ensures all reachable nodes are counted, and the final count is returned as the result.

This approach efficiently explores all reachable nodes while avoiding restricted nodes, leveraging BFS for optimal traversal.
