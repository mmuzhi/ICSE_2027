To solve this problem, we need to find the maximum integer `k` in the given array such that both `k` and `-k` are present in the array. The solution involves sorting the array in descending order and then checking each element to see if its negative counterpart exists in the array. The first such element encountered during the iteration will be the maximum `k` due to the descending order of the sorted array.

### Approach
1. **Sort the Array**: The array is sorted in descending order to prioritize checking larger elements first. This ensures that the first valid `k` found during the iteration will be the maximum possible.
2. **Create a Set**: A set is used for O(1) lookups to check if the negative of the current element exists in the array.
3. **Iterate and Check**: For each element in the sorted array, check if its negative exists in the set. The first element that satisfies this condition is returned immediately as it is the largest possible `k`.
4. **Return Result**: If no such element is found after the iteration, return -1.

### Solution Code
```python
class Solution:
    def findMaxK(self, nums: List[int]) -> int:
        nums = sorted(nums, reverse=True)
        s = set(nums)
        for num in nums:
            if (-num) in s:
                return num
        return -1
```

### Explanation
1. **Sorting**: The array is sorted in descending order to ensure that the largest elements are checked first. This optimization helps in finding the maximum `k` efficiently.
2. **Set for Lookup**: By converting the sorted array into a set, we enable constant-time lookups to check for the presence of the negative counterpart of each element.
3. **Iteration and Check**: The loop iterates through each element in the sorted array. For each element, it checks if the negative of the element exists in the set. If found, the element is returned immediately because it is the largest valid `k` due to the descending order of the array.
4. **Result Handling**: If the loop completes without finding any valid `k`, the function returns -1, indicating no such element exists in the array.

This approach efficiently combines sorting and set operations to solve the problem in O(n log n) time complexity due to the sorting step, with O(n) space complexity for the set. The solution ensures optimal performance by leveraging early termination upon finding the first valid `k`.
