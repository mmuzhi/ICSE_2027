### Explanation
1. **Initialization**: The grid dimensions are determined, and a directions tuple is defined to map cell values to movement directions. A visited matrix tracks visited cells to prevent revisiting.
2. **BFS Setup**: The BFS starts from the top-left cell (0,0), marked as visited, and added to the queue with an initial cost of 1.
3. **Processing**: For each cell dequeued, if it is the bottom-right cell, the current cost is returned. Otherwise, the next cell is determined by the current cell's value. If the next cell is within bounds, not visited, and not an obstacle, it is marked as visited, the cost is incremented, and it is enqueued.
4. **Termination**: The loop terminates when the bottom-right cell is reached, returning the accumulated cost. If the destination is unreachable, the function returns the cost up to the point of exhaustion, though the problem assumes a path exists.

This approach efficiently navigates the grid using BFS, ensuring the minimal number of cells are visited to reach the destination, adhering to the deterministic movement rules.
