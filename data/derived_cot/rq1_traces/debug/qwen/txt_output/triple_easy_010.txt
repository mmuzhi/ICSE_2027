=== RAW ANSWER ===

To solve this problem, we need to find the bitwise complement of a given positive integer. The bitwise complement involves flipping all the bits of the number's binary representation. The challenge is to correctly compute this complement considering the number's binary representation and handle edge cases such as zero.

### Approach
1. **Problem Analysis**: The task is to flip each bit of the given number's binary representation. For example, the binary representation of 5 (101) becomes 010 (which is 2 in decimal). The complement should be computed in the same bit length as the original number.
2. **Intuition**: The solution involves processing each bit of the number from least significant to most significant, flipping each bit (changing 1 to 0 and 0 to 1), and then combining these flipped bits into the correct order (most significant to least significant) to form the complement number.
3. **Algorithm Selection**: 
   - **Bit Processing**: Iterate through each bit of the number by repeatedly dividing the number by 2. For each bit, append the flipped bit (0 if the bit was 1, and 1 if the bit was 0) to a string.
   - **String Handling**: After processing all bits, reverse the string to convert it from least significant bit first to most significant bit first. This reversed string represents the binary complement.
   - **Edge Case Handling**: If the input number is zero, the complement is 1 (since flipping the single bit 0 gives 1).

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        if num == 0:
            return 1
        com = ''
        while num:
            if num % 2 == 1:
                com += '0'
            else:
                com += '1'
            num //= 2
        return int(com[::-1], 2)
```

### Explanation
1. **Edge Case Handling**: If the input number is zero, the function immediately returns 1, as the complement of 0 (considered as a 1-bit number) is 1.
2. **Bit Processing Loop**: The loop processes each bit of the number by checking the least significant bit (using modulo 2). If the bit is 1, it appends '0' to the string `com`; otherwise, it appends '1'. The number is then divided by 2 to shift the bits right, continuing until the number becomes zero.
3. **String Reversal and Conversion**: After processing all bits, the string `com` contains the flipped bits in reverse order (least significant to most significant). Reversing this string (`com[::-1]`) converts it to the correct binary representation of the complement. The reversed string is then converted to an integer using base 2.

This approach efficiently computes the bitwise complement by leveraging bit manipulation and string handling, ensuring correctness for all positive integers and handling the zero case appropriately.
