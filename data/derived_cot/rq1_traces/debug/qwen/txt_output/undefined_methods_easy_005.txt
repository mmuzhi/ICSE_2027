To solve this problem, we need to determine if an array can be split into three non-empty contiguous parts such that the sum of each part is equal. The solution involves checking if the total sum of the array is divisible by 3 and then verifying the existence of three parts with the required sum.

### Approach
1. **Check Total Sum Divisibility**: The first step is to check if the total sum of the array is divisible by 3. If not, it's impossible to split the array into three parts with equal sums, so we return False.
2. **Calculate Each Part's Sum**: If the total sum is divisible by 3, each part should sum to one-third of the total sum.
3. **Traverse the Array**: Traverse the array while maintaining a running sum. Whenever the running sum equals the target sum for each part, reset the running sum and increment the count of parts found.
4. **Check for Third Part**: After finding two parts, ensure that the remaining elements form a non-empty third part with the required sum. Since the total sum is divisible by 3, the third part will automatically have the correct sum if the first two parts do.

### Solution Code
```python
class Solution:
    def canThreePartsEqualSum(self, arr: List[int]) -> bool:
        total = sum(arr)
        if total % 3 != 0:
            return False
        each_sum = total // 3
        count = 0
        current_sum = 0
        for i in range(len(arr)):
            current_sum += arr[i]
            if current_sum == each_sum:
                current_sum = 0
                count += 1
                if count == 2:
                    if i + 1 < len(arr):
                        return True
                    else:
                        return False
        return False
```

### Explanation
1. **Total Sum Check**: The code first checks if the total sum of the array is divisible by 3. If not, it returns False immediately.
2. **Target Sum Calculation**: If the total sum is divisible by 3, the target sum for each part (`each_sum`) is calculated as one-third of the total sum.
3. **Traversal and Sum Tracking**: The code traverses the array, accumulating the sum of elements. Whenever the accumulated sum matches `each_sum`, it resets the accumulator and increments the part count.
4. **Third Part Validation**: After identifying two parts, the code checks if there are remaining elements to form a third part. If the third part is non-empty, it returns True; otherwise, it returns False.
5. **Efficiency**: The solution efficiently checks the required conditions in a single pass through the array, making it optimal with a time complexity of O(n), where n is the number of elements in the array.
