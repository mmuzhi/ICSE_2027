=== RAW ANSWER ===

To solve this problem, we need to determine the node with the highest score in a directed graph where each node has exactly one outgoing edge. The score of a node is defined as the sum of the indices of all edges that point to it. If multiple nodes share the highest score, we return the smallest node index.

### Approach
1. **Initialization**: We start by initializing a score array `sc` of size `n` (number of nodes) with all zeros. This array will keep track of the cumulative scores for each node.
2. **Iterate through edges**: For each node `i` (from 0 to n-1), we add `i` to the score of the node specified by `edges[i]`.
3. **Track maximum score and smallest node index**: While iterating, we keep track of the maximum score encountered (`mx`) and the corresponding node index (`ans`). If a higher score is found, we update both `mx` and `ans`. If a score equals `mx`, we update `ans` to the smaller node index between the current `ans` and the current node index.
4. **Return result**: After processing all edges, the variable `ans` holds the node with the highest score, or the smallest node index in case of a tie.

### Solution Code
```python
class Solution:
    def edgeScore(self, edges: List[int]) -> int:
        n = len(edges)
        sc = [0] * n
        mx = 0
        ans = 0
        for i in range(n):
            sc[edges[i]] += i
            if sc[edges[i]] > mx:
                mx = sc[edges[i]]
                ans = edges[i]
            elif sc[edges[i]] == mx:
                if ans > edges[i]:
                    ans = edges[i]
        return ans
```

### Explanation
- **Initialization**: The score array `sc` is initialized to zero for all nodes. This array will accumulate the sum of indices for each node's incoming edges.
- **Iteration**: For each node `i`, the index `i` is added to the score of the node specified by `edges[i]`. This step effectively computes the score for each node by summing all the indices of edges pointing to it.
- **Tracking Maximum and Minimum Node Index**: During the iteration, if a node's score exceeds the current maximum (`mx`), both `mx` and `ans` are updated. If a node's score matches `mx`, `ans` is updated to the smaller of the current `ans` and the current node index, ensuring the smallest node index is retained in case of ties.
- **Result**: After processing all edges, `ans` contains the node with the highest score. If multiple nodes have the same highest score, `ans` will be the smallest node index among them due to the tie-breaking condition during the iteration.
