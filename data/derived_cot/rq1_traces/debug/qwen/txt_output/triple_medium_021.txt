To solve this problem, we need to find the K-th bit of the N-th term in the Thue-Morse sequence. The Thue-Morse sequence is constructed by starting with "0" and repeatedly appending the bitwise complement of the current sequence. The challenge is to efficiently determine the K-th bit without generating the entire sequence up to N.

### Approach
1. **Base Case Handling**: When N is 1, the sequence is "0" for K=1 and "1" for K=2, depending on the boolean R. The boolean R determines whether the first bit is '0' (if R is True) or '1' (if R is False).
2. **Recursive Breakdown**: For N > 1, the sequence length is 2^N. The first half of the sequence is the (N-1)-th term, and the second half is the bitwise complement of the first half. 
3. **Mid Calculation**: Calculate the midpoint (mid) of the current sequence length, which is 2^(N-1).
4. **Position Handling**: 
   - If K is in the first half (K ≤ mid), recursively find the K-th bit in the (N-1)-th term with the same R.
   - If K is in the second half (K > mid), recursively find the (K - mid)-th bit in the (N-1)-th term and return its complement.

### Solution Code
```python
class Solution:
    def findKthBit(self, N, K, R=True):
        if N == 1:
            if K == 1:
                return '0' if R else '1'
            elif K == 2:
                return '1' if R else '0'
        mid = 1 << (N-1)
        if K <= mid:
            return self.findKthBit(N-1, K, R)
        else:
            bit = self.findKthBit(N-1, K - mid, R)
            return '1' if bit == '0' else '0'
```

### Explanation
- **Base Case Handling**: The function first checks if N is 1. If so, it returns '0' or '1' based on the boolean R for K=1, and the complement for K=2.
- **Recursive Breakdown**: For N > 1, the function calculates the midpoint (mid) of the sequence length (2^N). This midpoint divides the sequence into two halves.
- **Position Handling**: 
  - If K is in the first half, the function recursively calls itself with N-1 and the same K and R.
  - If K is in the second half, the function recursively finds the corresponding bit in the first half (by adjusting K to K - mid) and returns its complement (flipped bit).
This approach efficiently narrows down the problem size by half each recursive call, leveraging the properties of the Thue-Morse sequence to avoid unnecessary computations.
