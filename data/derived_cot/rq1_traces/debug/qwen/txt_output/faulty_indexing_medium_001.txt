To solve this problem, we need to perform a level-order traversal (BFS) of a binary tree and return the nodes level by level from bottom to top. The initial code attempted to do this but had a bug in the reversal function used to reverse the levels. The key issue was that the reversal function incorrectly reversed the list by using an out-of-bounds index and an incorrect loop condition.

### Approach
1. **Level-Order Traversal (BFS)**: We start by traversing the tree level by level using a queue. Each level's nodes are processed from left to right, and their values are stored in a temporary list.
2. **Reversing Levels**: After collecting all levels from top to bottom, we reverse the list of levels to present them from bottom to top. The reversal function was modified to correctly reverse the list by:
   - Using the correct indices (0 to len(ans)-1).
   - Changing the loop condition to `l < r` to ensure proper swapping until the middle of the list is reached.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans) - 1
            while l < r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root: 
            return None
        q, ans = [root], []
        while q:
            n = len(q)
            l = []
            for i in range(n):
                curr = q.pop(0)
                l.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(l)
        return checkReverse(ans)
```

### Explanation
1. **BFS Traversal**: The algorithm uses a queue to process each node level by level. For each node, its value is added to the current level's list, and its children (if any) are added to the queue for further processing.
2. **Reversing Levels**: After collecting all levels in the order from top to bottom, the `checkReverse` function reverses the list of levels. This function uses two pointers, `l` (start) and `r` (end), to swap elements until they meet in the middle, effectively reversing the list in-place.
3. **Edge Handling**: If the tree is empty (root is None), the function returns None immediately, as there are no levels to process.

This approach ensures that the levels are correctly reversed from top to bottom to bottom to top, providing the desired level-order bottom-up traversal.

# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans) - 1
            while l < r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root: 
            return None
        q, ans = [root], []
        while q:
            n = len(q)
            l = []
            for i in range(n):
                curr = q.pop(0)
                l.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(l)
        return checkReverse(ans)
