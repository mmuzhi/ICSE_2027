To solve this problem, we need to simulate a series of operations on a stack of scores, where each operation modifies the stack in a specific way. The goal is to compute the total sum of all scores after processing all operations.

### Approach
1. **Problem Analysis**: The problem involves processing a list of operations that can either push new scores onto a stack, modify existing scores, or remove scores. The operations include:
   - 'D': Double the last score and push it onto the stack.
   - 'C': Remove the last score from the stack.
   - '+': Push the sum of the last two scores onto the stack.
   - A numerical string: Push the numerical value onto the stack.

2. **Intuition**: We can use a stack data structure to keep track of the scores. For each operation, we perform the corresponding action:
   - For 'D', we double the top element of the stack and push it back.
   - For 'C', we simply pop the top element.
   - For '+', we pop the top two elements, sum them, and push the result back.
   - For numerical strings, we convert them to integers and push them onto the stack.

3. **Algorithm Selection**: A stack is the most appropriate data structure here because it allows efficient access to the top elements and supports push/pop operations. We process each operation in sequence, modifying the stack as needed.

4. **Complexity Analysis**: Each operation is processed in constant time, O(1), and the total number of operations is O(n), where n is the length of the input list. Thus, the overall time complexity is O(n). The space complexity is O(n) in the worst case, where all operations push new scores onto the stack.

### Solution Code
```python
class Solution:
    def calPoints(self, operations: List[str]) -> int:
        stack = []
        for op in operations:
            if op == 'D':
                stack.append(2 * stack[-1])
            elif op == 'C':
                stack.pop()
            elif op == '+':
                stack.append(stack[-1] + stack[-2])
            else:
                stack.append(int(op))
        return sum(stack)
```

### Explanation
1. **Initialization**: We start with an empty stack to keep track of the scores.
2. **Processing Operations**:
   - **'D' Operation**: We double the top element of the stack and push the result back onto the stack.
   - **'C' Operation**: We remove the top element from the stack.
   - **'+' Operation**: We sum the top two elements of the stack and push the result back onto the stack.
   - **Numerical String**: We convert the string to an integer and push it onto the stack.
3. **Result Calculation**: After processing all operations, the sum of all elements in the stack gives the total score.

This approach efficiently handles each operation in constant time, ensuring optimal performance and correctness.
