=== RAW ANSWER ===

To solve this problem, we need to find a peak element in an integer array. A peak element is an element that is greater than or equal to its neighbors. The solution involves using a binary search approach to efficiently locate the peak element.

### Approach
1. **Binary Search Setup**: We start by initializing two pointers, `low` and `high`, to the start and end of the array, respectively.
2. **Mid Calculation**: In each iteration, we calculate the middle index `mid` to divide the array into two halves.
3. **Peak Check**: We check if the middle element is a peak by verifying if it is greater than or equal to its neighbors. If `mid` is the first or last element, we only compare with the available neighbor.
4. **Adjust Search Range**: If the middle element is not a peak, we adjust the search range:
   - If the middle element is less than its next element, the peak must be in the right half, so we move `low` to `mid + 1`.
   - Otherwise, the peak must be in the left half, so we move `high` to `mid - 1`.
5. **Termination**: The loop continues until a peak element is found. The problem guarantees at least one peak, so the loop will terminate successfully.

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        while low <= high:
            mid = (low + high) // 2
            is_peak = (mid == 0 or nums[mid] >= nums[mid-1]) and (mid == n-1 or nums[mid] >= nums[mid+1])
            if is_peak:
                return mid
            elif nums[mid] <= nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

### Explanation
1. **Binary Search Initialization**: The variables `low` and `high` are initialized to the start and end indices of the array.
2. **Mid Calculation**: The middle index `mid` is calculated as the average of `low` and `high`.
3. **Peak Check**: The condition checks if the middle element is a peak by ensuring it is greater than or equal to its neighbors. If `mid` is the first element, only the right neighbor is checked; if it's the last element, only the left neighbor is checked.
4. **Adjusting Search Range**: If the middle element is not a peak, the search range is adjusted based on the comparison between the middle element and its next element. If the middle element is less than the next, the peak lies in the right half; otherwise, it lies in the left half.
5. **Termination**: The loop exits when a peak is found, returning the index. The problem guarantees a peak, so the loop will always find one. If no peak is found (though unlikely given the problem constraints), the function returns -1 as a fallback.
