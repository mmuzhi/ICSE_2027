Directly constructing the concatenated string is infeasible for large `n1`, so we use the Knuth-Morris-Pratt (KMP) algorithm's prefix function to efficiently track the occurrences of `s2` as we iterate through `s1` repeated `n1` times.

2. **KMP Algorithm**: The KMP algorithm helps in finding patterns efficiently by utilizing a prefix function (`pi`) that stores the length of the longest prefix of `s2` that is also a suffix for each substring of `s2`. This allows us to skip unnecessary comparisons during the search.

3. **Simulation**: For each character in `s1` repeated `n1` times, we update the current state in `s2` using the prefix function. If the state reaches the length of `s2`, it means `s2` has been found, and we increment the count. The state is reset to 0 after each complete match.

4. **Complexity**: The algorithm efficiently processes each character in `s1` for `n1` times, leveraging the KMP prefix function to avoid redundant checks. The time complexity is O(n1 * len(s1)), which is optimal for the given constraints.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def getMaxRepetitions(self, s1: str, n1: int, s2: str, n2: int) -> int:
        if not s2:
            return 0
        # Build the prefix array for s2
        pi = [0] * len(s2)
        j = 0
        for i in range(1, len(s2)):
            while j > 0 and s2[i] != s2[j]:
                j = pi[j-1]
            if s2[i] == s2[j]:
                j += 1
            else:
                j = 0
            pi[i] = j

        count = 0
        state = 0
        for _ in range(n1):
            for char in s1:
                while state > 0 and char != s2[state]:
                    state = pi[state-1]
                if char == s2[state]:
                    state += 1
                else:
                    state = 0
                if state == len(s2):
                    count += 1
                    state = 0
        return count
```

### Explanation
1. **Prefix Function Construction**: The `pi` array is built for `s2` to store the length of the longest prefix which is also a suffix for each substring ending at position `i` in `s2`. This helps in efficiently transitioning between states during the search.

2. **Iterating Through Characters**: For each character in `s1` repeated `n1` times, the current state in `s2` is updated. If the character matches the current state, the state advances; otherwise, it uses the prefix function to find the next valid state.

3. **Counting Occurrences**: Whenever the state reaches the length of `s2`, it means `s2` has been found as a substring, and the count is incremented. The state is reset to 0 after each complete match to allow overlapping matches.

4. **Result**: The total count of `s2` occurrences in the concatenated string of `n1` copies of `s1` is returned. The solution efficiently handles large inputs by leveraging the KMP algorithm's properties.
