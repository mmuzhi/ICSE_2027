=== RAW ANSWER ===

To solve this problem, we need to implement a binary search algorithm that efficiently finds the index of a target value in a sorted array. The solution should return the index if the target is found, and -1 if it is not present in the array.

### Approach
The approach involves using a standard binary search algorithm to locate the target value in the sorted array. The binary search works by repeatedly dividing the search interval in half. If the target value is found, its index is returned. If the search interval is exhausted without finding the target, the function returns -1.

1. **Initialization**: Start with two pointers, `left` set to 0 and `right` set to the length of the array.
2. **Binary Search Loop**: While `left` is less than `right`, compute the middle index `mid`. If the value at `mid` is less than the target, adjust `left` to `mid + 1`. Otherwise, adjust `right` to `mid`.
3. **Termination Check**: After the loop, check if `left` is within bounds and if the value at `left` equals the target. If both conditions are met, return `left`; otherwise, return -1.

This approach ensures that the solution is efficient with a time complexity of O(log n), where n is the number of elements in the array.

### Solution Code
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums)
        while left < right:
            mid = (left + right) // 2
            if nums[mid] < target:
                left = mid + 1
            else:
                right = mid
        
        if left == len(nums) or nums[left] != target:
            return -1
        return left
```

### Explanation
1. **Initialization**: The `left` pointer starts at the beginning of the array (index 0), and the `right` pointer starts just beyond the last element (index `len(nums)`).
2. **Binary Search Loop**: The loop continues while `left` is less than `right`. The middle index `mid` is calculated. If the value at `mid` is less than the target, the search continues in the right half by setting `left` to `mid + 1`. Otherwise, the search continues in the left half by setting `right` to `mid`.
3. **Termination Check**: After the loop, `left` points to the first element that is not less than the target. If this element equals the target, its index is returned. Otherwise, -1 is returned indicating the target is not present in the array.

This method efficiently narrows down the search space by half in each iteration, making it optimal for large sorted arrays.
