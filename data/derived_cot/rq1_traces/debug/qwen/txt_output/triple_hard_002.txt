To solve this problem, we need to determine the maximum number of components into which a given tree can be split such that each component has the same sum of node values. The solution involves checking all possible divisors of the total sum of the tree's node values to see if the tree can be partitioned into components with each component's sum equal to a candidate value.

### Approach
1. **Tree Construction**: First, we construct an adjacency list representation of the tree from the given edges.
2. **Total Sum Calculation**: Calculate the total sum of all node values in the tree.
3. **Divisor Check**: Iterate over all possible divisors of the total sum from 1 to half of the total sum. For each divisor `cand`, check if the total sum is divisible by `cand`.
4. **DFS for Component Check**: For each valid divisor `cand`, perform a depth-first search (DFS) to verify if the tree can be split into components each with sum `cand`. The DFS function accumulates the sum of each subtree and checks if it equals `cand`. If it does, the function returns 0; otherwise, it returns the subtree's sum.
5. **Result Calculation**: If the DFS confirms that the tree can be split into components of sum `cand`, the maximum number of components is `total // cand`. The loop checks from the largest possible divisor to the smallest, ensuring the first valid divisor found gives the maximum number of components.

### Solution Code
```python
class Solution:
    def componentValue(self, nums: List[int], edges: List[List[int]]) -> int:
        n = len(nums)
        tree = [[] for _ in range(n)]
        for u, v in edges:
            tree[u].append(v)
            tree[v].append(u)
        
        total = sum(nums)

                continue

                        continue
                    s += dfs(v, u)
                return 0 if s == cand else s
            if dfs(0, -1) == 0:
                return total // cand - 1
        return 0
```
