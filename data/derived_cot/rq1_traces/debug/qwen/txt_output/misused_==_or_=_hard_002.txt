The length of the cycle is calculated as the difference between the current step count and the step count when the node was first visited.

### Approach
1. **Problem Analysis**: The problem requires detecting cycles in a directed graph where each node has exactly one outgoing edge. The goal is to find the longest cycle in terms of the number of edges.
2. **Intuition**: By traversing the graph from each node and marking visited nodes with their first visit time, we can detect cycles when a node is revisited. The cycle length is determined by the difference in visit times.
3. **Algorithm Selection**: We use a modified Depth-First Search (DFS) approach where each node is processed once per traversal. For each node, we maintain a visited array that records the starting node and the time of first visit. If a node is revisited during the same traversal, a cycle is detected, and its length is calculated.
4. **Complexity Analysis**: The algorithm processes each node once, and for each node, it may traverse up to all other nodes in the worst case. Thus, the time complexity is O(n^2), where n is the number of nodes. The space complexity is O(n) for storing the visited array.

### Solution Code
```python
class Solution:
    def longestCycle(self, edges: List[int]) -> int:
        n = len(edges)
        ans = -1
        for i in range(n):
            v = [0] * n
            t = 1
            c = i
            while c >= 0:
                if c >= n:
                    break
                if v[c] != 0:
                    if v[c][0] == i:
                        ans = max(ans, t - v[c][1])
                    break
                else:
                    v[c] = [i, t]
                    t += 1
                    c = edges[c]
        return ans
```
