=== RAW ANSWER ===

To solve this problem, we need to perform a level-order traversal (BFS) of a binary tree and return the nodes' values in a bottom-up manner. The solution involves traversing the tree level by level from left to right and then reversing the order of the levels to achieve the bottom-up traversal.

### Approach
1. **Level-Order Traversal (BFS)**: We start by traversing the tree level by level using a queue. Each node is processed in the order they appear in the queue. For each node, we add its value to the current level's list and enqueue its left and right children (if they exist).
2. **Reverse the Levels**: After collecting all levels from top to bottom, we reverse the list of levels to present them from bottom to top. This reversal is done by a helper function that swaps elements from the start and end of the list until the middle is reached.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans)-1
            while l <= r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root:
            return []
        q = [root]
        ans = []
        while q:
            n = len(q)
            level = []
            for i in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        return checkReverse(ans)
```

### Explanation
1. **Level-Order Traversal**: The BFS starts with the root node. For each node, its value is added to the current level's list, and its children are added to the queue for processing in subsequent levels. This ensures that nodes are processed level by level from left to right.
2. **Reversing Levels**: After all levels are processed, the list of levels is reversed. The helper function `checkReverse` swaps elements from the start and end of the list, moving towards the center, effectively reversing the order of the levels. This results in the levels being ordered from bottom to top, which is the desired output.

This approach efficiently handles the traversal and reversal, ensuring the correct order of nodes in the bottom-up level-order traversal.
