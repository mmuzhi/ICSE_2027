=== RAW ANSWER ===

To solve this problem, we need to generate all possible valid IP addresses from a given string of digits. An IP address consists of four parts, each ranging from 0 to 255. Each part must not have leading zeros unless the part itself is zero.

### Approach
The approach involves using a backtracking method to explore all possible ways to split the string into four parts. The key steps are:
1. **Base Case**: If we reach the end of the string and have exactly four parts, we join these parts into a valid IP address and add it to the result list.
2. **Invalid Cases**: If the number of parts exceeds four or we go beyond the string length, we backtrack.
3. **Leading Zero Handling**: If the current character is '0', we can only take one digit (since "01" is invalid), so we immediately proceed with the next character.
4. **Non-Leading Zero Handling**: For other digits, we try taking 1, 2, or 3 digits (if they form a number ≤ 255) and proceed recursively.

### Solution Code
```python
class Solution():
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i > len(s):
                return
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 3 and i + j < len(s):
                num = int(s[i:i+j+1])
                if num < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i+j+1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Base Case Handling**: The function checks if the current index `i` has reached the end of the string and if exactly four parts have been formed. If so, it joins these parts into a string and adds it to the result list.
2. **Invalid Cases**: The function checks if the number of parts exceeds four or if the index `i` is beyond the string length, in which case it backtracks.
3. **Leading Zero Handling**: If the current character is '0', it is added as a single part, and the function proceeds to the next character. This avoids invalid parts like "01".
4. **Non-Leading Zero Handling**: For other digits, the function tries taking 1, 2, or 3 digits (up to 3 digits to stay within the 0-255 range). Each valid number is added to the current parts, and the function proceeds recursively. The loop ensures we only consider up to 3 digits.

This approach efficiently explores all possible valid IP addresses by leveraging backtracking and careful handling of edge cases like leading zeros. The solution ensures all generated IP addresses are valid according to the specified constraints.
