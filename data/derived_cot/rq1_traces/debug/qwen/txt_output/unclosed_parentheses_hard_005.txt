To solve this problem, we need to find the longest cycle in a directed graph where each node has at most one outgoing edge. The solution involves traversing the graph using Depth-First Search (DFS) to detect cycles and determine their lengths.

### Approach
1. **Graph Traversal**: We traverse each node in the graph. For each unvisited node, we start a DFS to explore all reachable nodes.
2. **Cycle Detection**: During traversal, we keep track of visited nodes and the depth at which each node was first encountered. If we encounter a node that has already been visited in the current traversal, a cycle is detected.
3. **Cycle Length Calculation**: The length of the detected cycle is calculated by subtracting the depth at which the node was first encountered from the current depth. This gives the number of edges in the cycle.
4. **Tracking Maximum Cycle**: We keep track of the maximum cycle length encountered during the traversal. If no cycles are found, we return -1.

### Solution Code
```python
class Solution:
    def longestCycle(self, edges: List[int]) -> int:
        n = len(edges)
        bl = [0] * n
        mp = defaultdict(int)
        mx = -1
        for i in range(n):
            if bl[i] == 0:
                x = i
                l = 0
                st = set()
                while x != -1 and bl[x] == 0:
                    bl[x] = 1
                    mp[x] = l
                    l += 1
                    st.add(x)
                    x = edges[x]
                if x != -1 and x in st:
                    mx = max(mx, l - mp[x])
        return mx
```

1. **Initialization**: We initialize a visited array `bl` to keep track of visited nodes, a dictionary `mp` to store the depth of each node during traversal, and a variable `mx` to store the maximum cycle length found.
2. **Traversal**: For each unvisited node, we start a DFS. During traversal, we mark nodes as visited and record their depths. Nodes are added to a set `st` to track nodes visited in the current traversal.
3. **Cycle Detection**: If during traversal we encounter a node that is already in the current set `st`, a cycle is detected. The cycle length is calculated as the difference between the current depth and the depth of the first occurrence of the node.
4. **Result**: After processing all nodes, if no cycles were found, `mx` remains -1. Otherwise, it holds the length of the longest cycle in the graph.

This approach efficiently detects cycles and calculates their lengths by leveraging DFS and tracking visited nodes and their depths, ensuring optimal performance for the given problem constraints.
