### Explanation
1. **Neighbor Generation**: The `get_neighbors` function generates all possible next states by moving the blank space (0) to adjacent positions (up, down, left, right). Each new state is created by swapping the blank space with an adjacent number.
2. **BFS Initialization**: The BFS starts from the initial board configuration, stored in a queue along with the number of moves taken to reach it.
3. **State Tracking**: A set is used to keep track of visited states to prevent revisiting and cycles.
4. **Target Check**: For each state dequeued, the code checks if it matches the target configuration. If found, the number of moves is returned immediately.
5. **Termination**: If the queue is exhausted without finding the target state, the function returns -1, indicating the target is unreachable from the initial state.

This approach efficiently explores all possible states level by level, ensuring the shortest path to the target configuration is found if it exists.
