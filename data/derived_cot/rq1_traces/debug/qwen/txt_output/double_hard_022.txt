To solve this problem, we need to find the longest substring within a given word that does not contain any forbidden substrings. The solution involves using a trie data structure to efficiently check for forbidden substrings and a sliding window approach to determine the longest valid substring.

### Approach
1. **Trie Construction**: Build a trie from the list of forbidden substrings. Each node in the trie represents a character in a forbidden substring, and an "end" marker is added at the end of each forbidden substring to denote its completion.
2. **Sliding Window**: Use a sliding window approach starting from the end of the word. For each position `i` in the word, check the substring from `i` to `j` (initially the end of the word) to see if it contains any forbidden substring.
3. **Forbidden Substring Check**: For each substring, use the trie to check if it contains any forbidden substring starting from `i`. If a forbidden substring is found, adjust the window `j` to exclude it.
4. **Max Length Tracking**: Track the maximum length of valid substrings encountered during the sliding window process.

    def longestValidSubstring(self, word: str, forbidden: List[str]) -> int:
        if not word:

### Explanation
1. **Trie Construction**: The trie is built by iterating over each forbidden substring. For each character in a forbidden substring, a new node is added to the trie. An "end" marker is added at the end of each forbidden substring to indicate its completion.
2. **Sliding Window**: The algorithm starts from the end of the word (`j = len(word)`) and moves the start of the substring (`i`) from the end to the beginning. For each position `i`, the substring `word[i:j]` is checked for forbidden substrings.
3. **Forbidden Substring Check**: The `isForbidden` function checks if the substring starting at `i` contains any forbidden substring by traversing the trie. If a forbidden substring is found, it returns its length; otherwise, it returns 0.
4. **Window Adjustment**: If a forbidden substring is found, the window `j` is adjusted to `i + truc - 1`, where `truc` is the length of the forbidden substring. This ensures that the next check starts before the forbidden substring.
5. **Max Length Tracking**: The maximum length of valid substrings (`j - i`) is tracked and returned as the result.

This approach efficiently checks for forbidden substrings using a trie and adjusts the sliding window to find the longest valid substring, ensuring optimal performance.
