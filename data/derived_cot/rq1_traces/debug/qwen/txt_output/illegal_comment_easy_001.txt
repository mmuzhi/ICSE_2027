=== RAW ANSWER ===

To solve this problem, we need to determine if an array can be split into three contiguous non-empty parts such that the sum of each part is exactly one-third of the total sum of the array. The solution involves checking the divisibility of the total sum by 3 and then verifying if the array can be partitioned into three parts with the required sum.

### Approach
1. **Check Divisibility**: First, we check if the total sum of the array is divisible by 3. If not, it's impossible to split the array into three parts with equal sums, so we return false.
2. **Track Sums and Counts**: We traverse the array while maintaining a running sum (`sumi`) and a count of parts found (`count`). For each element, we add it to the running sum. If the running sum equals one-third of the total sum, we reset the running sum to zero and increment the part count.
3. **Check Completion**: If at any point the count of parts reaches 2, it means the third part will automatically have the correct sum (since the total sum is divisible by 3), so we return true. If the entire array is processed without finding three parts, we return false.

### Solution Code
```python
class Solution:
    def canThreePartsEqualSum(self, arr: List[int]) -> bool:
        total = sum(arr)
        each_sum = total // 3
        if total % 3 != 0:
            return False
        sumi = 0
        count = 0
        for x in arr:
            if count == 2:
                return True
            sumi += x
            if sumi == each_sum:
                sumi = 0
                count += 1
        return False
```

### Explanation
1. **Divisibility Check**: The code first checks if the total sum of the array is divisible by 3. If not, it immediately returns false because equal partition is impossible.
2. **Traversal and Sum Tracking**: The code then iterates through each element in the array, accumulating the sum in `sumi`. If `sumi` equals one-third of the total sum (`each_sum`), it resets `sumi` to zero and increments the part count (`count`).
3. **Early Termination**: If during the traversal the count of parts reaches 2, the code returns true because the third part will inherently have the correct sum. If the loop completes without finding three parts, the function returns false.

This approach efficiently checks for the possibility of splitting the array into three parts with equal sums by leveraging the divisibility check and a single pass through the array, ensuring optimal performance.
