=== RAW ANSWER ===

To solve this problem, we need to find the kth largest element in an unsorted array efficiently. The initial approach of sorting the entire array and then accessing the kth largest element is correct but not the most efficient for large arrays. Instead, we can use a min-heap to keep track of the k largest elements encountered during a single pass through the array.

### Approach
1. **Problem Analysis**: The task is to find the kth largest element in an unsorted array. A naive approach would involve sorting the array and then picking the (n - k)th element, but this approach has a time complexity of O(n log n), which can be inefficient for large arrays.
2. **Intuition**: By maintaining a min-heap of size k, we can efficiently keep track of the k largest elements. As we iterate through the array, each element is pushed into the heap. If the heap size exceeds k, the smallest element in the heap is popped, ensuring that the heap always contains the k largest elements seen so far.
3. **Algorithm Selection**: The min-heap approach allows us to achieve an average time complexity of O(n log k), which is more efficient than sorting for large arrays. The `heapq` module in Python provides the necessary functions to implement this approach efficiently.
4. **Optimization**: Using the `heapq.nlargest` function simplifies the code and leverages an optimized implementation for finding the k largest elements, further enhancing efficiency.

### Solution Code
```python
import heapq

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        return heapq.nlargest(k, nums)[-1]
```

1. **Import heapq**: The `heapq` module is imported to utilize its heap functions.
2. **Function Definition**: The `findKthLargest` method is defined with parameters `self`, `nums` (the input array), and `k` (the position of the largest element to find).
3. **Heapq.nlargest**: The `heapq.nlargest` function is used to retrieve the k largest elements from the array. The result is a list of these elements, sorted in descending order.
4. **Accessing the kth Largest Element**: The last element of the list returned by `heapq.nlargest` is the kth largest element, which is then returned as the result.

This approach efficiently finds the kth largest element with a time complexity of O(n log k), making it suitable for large input arrays. The solution is both concise and leverages Python's built-in optimizations for better performance.
