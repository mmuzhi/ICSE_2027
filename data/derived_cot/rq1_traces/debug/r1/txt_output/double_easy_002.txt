### Approach
1. **Identify Even and Odd Digits**: Split the digits of the input number into two lists: one for even digits and one for odd digits.
2. **Max-Heap Simulation**: Convert these lists into max-heaps using negative values (since Python's `heapq` module provides a min-heap by default).

### Solution Code
```python
import heapq

class Solution:
    def largestInteger(self, num: int) -> int:
        evenlist = []
        oddlist = []
        nums = [int(x) for x in str(num)]
        for i in nums:
            if i % 2 == 0:
                evenlist.append(i)
            else:
                oddlist.append(i)
        even = [-x for x in evenlist]
        odd = [-x for x in oddlist]
        heapq.heapify(even)
        heapq.heapify(odd)
        result = []
        for ele in nums:
            if ele in evenlist:
                result += [-heapq.heappop(even)]
            else:
                result += [-heapq.heappop(odd)]
        return int(''.join(map(str, result)))
```

### Explanation
1. **Splitting Digits**: The input number is converted into a list of its digits. These digits are then split into two lists based on their parity (even or odd).
2. **Max-Heap Creation**: By negating the values of the digits, we can use Python's `heapq` to simulate a max-heap. This allows us to efficiently retrieve the largest remaining digit when needed.
3. **Reconstructing the Number**: For each digit in the original number, we check its parity and replace it with the largest remaining digit of the same parity using the respective max-heap. This ensures that each digit is replaced in the order they appeared but with the largest possible values available.
4. **Result Construction**: The processed digits are combined into a string and converted back to an integer to form the final result.

This approach efficiently uses heaps to maintain the order of digits while ensuring the largest possible values are used, resulting in the correct solution.
