To solve this problem, we need to reverse nodes in a linked list in groups of size `k`. If the number of nodes remaining is less than `k`, they should remain in their original order. The solution involves iterating through the linked list, reversing each group of `k` nodes, and properly linking the reversed groups together.

### Approach
1. **Dummy Node**: Use a dummy node to simplify edge cases, such as handling the head of the list.
2. **Group Identification**: For each group, determine the k-th node using a helper function `getKth`. If the k-th node is not found, break out of the loop as there are not enough nodes to form a group.
3. **Reversing Nodes**: Reverse the nodes in the current group. This involves adjusting the pointers such that each node in the group points to the previous node in the reversed order.
4. **Linking Groups**: After reversing a group, link the previous group's end to the new head of the reversed group and update the current group's end to the next group's start.

### Solution Code
```python
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        curr_group = dummy  # Tracks the end of the previous reversed group

        while True:
            kth = self.getKth(curr_group, k)
            if not kth:
                break
            next_group = kth.next

            # Reverse the current group
            prev, curr = next_group, curr_group.next
            while curr != next_group:
                temp = curr.next
                curr.next = prev
                prev = curr
                curr = temp

            # Connect the previous group to the reversed group and move to the next
            tmp = curr_group.next
            curr_group.next = kth  # kth is now the head of the reversed group
            curr_group = tmp  # Move to the end of the current reversed group

        return dummy.next

    def getKth(self, node: ListNode, k: int) -> Optional[ListNode]:
        while node and k > 0:
            node = node.next
            k -= 1
        return node if k == 0 else None
```

### Explanation
1. **Dummy Node Initialization**: The dummy node is initialized to handle edge cases and simplify the linking process.
2. **Finding the k-th Node**: The `getKth` function moves `k` steps from the given node to find the end of the current group. If it cannot find the k-th node, it returns `None`.
3. **Reversing the Group**: The current group is reversed by iterating through each node and adjusting the pointers. The reversal starts from the first node of the group and continues until the end of the group.
4. **Linking Groups**: After reversing, the previous group's end is linked to the new head of the reversed group. The current group pointer is then updated to the end of the reversed group to process the next group.

This approach ensures that each group of `k` nodes is reversed efficiently and linked correctly, maintaining the overall structure of the linked list.
