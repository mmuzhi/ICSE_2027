To solve this problem, we need to determine the maximum number of times we can obtain string `s2` from the string `s1` repeated `n1` times, divided by `n2`. The solution involves detecting cycles in the repetition pattern of `s1` and `s2` to optimize the calculation.

### Approach
1. **Check Feasibility**: First, check if all characters of `s2` are present in `s1`. If not, return 0 immediately.
2. **Filter `s1`**: Filter `s1` to include only characters present in `s2` to optimize the search process.
3. **Cycle Detection**: Track the repetition pattern of `s2` within `s1` to detect cycles. This helps in avoiding redundant calculations by leveraging the repeating pattern.
4. **Cycle Calculation**: Once a cycle is detected, compute the number of `s2` repetitions that can be formed within the remaining `s1` repetitions after accounting for the cycle.
5. **Handle Remaining Repetitions**: Calculate the maximum number of `s2` repetitions that can be formed from the remaining part of `s1` after processing all complete cycles.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def getMaxRepetitions(self, s1: str, n1: int, s2: str, n2: int) -> int:
        rec, track = [0], defaultdict(int)
        ct = start = 0

        if not set(s2).issubset(set(s1)):
            return 0

        s1 = ''.join(char for char in s1 if char in set(s2))
        
        while True:
            for char in s2:
                ptr = s1.find(char, start)
                if ptr == -1:
                    ct += 1
                    ptr = s1.find(char)
                start = ptr + 1
            rec.append(ct + 1)

            if rec[-1] > n1:
                return (len(rec) - 2) // n2

            if ptr not in track:
                track[ptr] = len(rec) - 1
            else:
                break
        
        cycle_start_idx = track[ptr]
        cycle_start = rec[cycle_start_idx]
        cycle1 = (ct + 1) - cycle_start
        cycle2 = len(rec) - 1 - cycle_start_idx
        rest = n1 - cycle_start
        rem = cycle_start + rest % cycle1

        ptr2 = 0
        while ptr2 < len(rec) and rec[ptr2] <= rem:
            ptr2 += 1

        total = cycle2 * (rest // cycle1) + (ptr2 - 1 - cycle_start_idx)
        return total // n2
```
