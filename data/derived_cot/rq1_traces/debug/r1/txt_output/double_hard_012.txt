=== RAW ANSWER ===

To solve this problem, we need to find the median of a sliding window of size `k` moving through a list of integers. The median should be efficiently calculated as the window slides from the start to the end of the list.

### Approach
The approach involves using a data structure that maintains elements in sorted order, allowing efficient insertion, deletion, and access by index. The `SortedList` from the `sortedcontainers` library is well-suited for this purpose as it provides O(log n) time complexity for these operations. Here's the detailed approach:

1. **Data Structure**: Use a `SortedList` to maintain the elements of the current window in sorted order.
2. **Insertion**: For each new element, add it to the `SortedList`.
3. **Removal**: If the size of the list exceeds `k`, remove the element that is leaving the window (the element added `k` steps before the current element).
4. **Median Calculation**: Once the window size reaches `k`, calculate the median. For an odd-sized window, the median is the middle element. For an even-sized window, it is the average of the two middle elements.

### Solution Code
```python
from sortedcontainers import SortedList
from typing import List

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        tree = SortedList()
        ans = []
        for i, x in enumerate(nums):
            tree.add(x)
            if len(tree) > k:
                tree.remove(nums[i - k])
            if len(tree) == k:
                if k % 2 == 1:
                    median = tree[k // 2]
                else:
                    median = (tree[k // 2 - 1] + tree[k // 2]) / 2
                ans.append(float(median))
        return ans
```

### Explanation
- **Data Structure**: The `SortedList` maintains elements in sorted order, allowing efficient median calculation.
- **Insertion and Removal**: Each element is added to the list, and when the window size exceeds `k`, the oldest element (the one that is no longer in the window) is removed. This ensures the list always contains the current window's elements.
- **Median Calculation**: 
  - For an odd-sized window (`k` is odd), the median is the middle element, accessed directly by index `k // 2`.
  - For an even-sized window (`k` is even), the median is the average of the two middle elements, accessed by indices `k // 2 - 1` and `k // 2`.

This approach efficiently handles dynamic insertion and removal of elements while maintaining sorted order, ensuring the median can be calculated in O(log k) time for each window, leading to an overall time complexity of O(n log k), where `n` is the number of elements in the input list.
