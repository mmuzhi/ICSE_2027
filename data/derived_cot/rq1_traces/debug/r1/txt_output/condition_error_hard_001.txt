=== RAW ANSWER ===

To solve this problem, we need to find all pairs of distinct indices (i, j) such that the concatenation of words[i] and words[j] forms a palindrome. The solution involves using a hash map to efficiently check for reversed words and considering different cases where the concatenation of words can form a palindrome.

### Approach
1. **Hash Map for Reversed Words**: Create a hash map where the keys are the reversed versions of the words and the values are their original indices. This helps in quickly checking if the reverse of any word exists in the list.
2. **Check for Direct Pairs**: For each word, check if its reverse exists in the hash map. If it does, add the pair (i, j) where j is the index of the reversed word.
3. **Handle Empty Strings**: If a word is a palindrome and there's an empty string in the list, both (i, empty) and (empty, i) pairs are valid.
4. **Split Words into Prefix and Suffix**: For each word, split it into all possible prefix and suffix combinations. Check if the suffix exists in the hash map and the prefix is a palindrome, or if the prefix exists in the hash map and the suffix is a palindrome.

### Solution Code
```python
class Solution:
    def palindromePairs(self, words: List[str]) -> List[List[int]]:
        backward, res = {}, set()
        for i, word in enumerate(words):
            backward[word[::-1]] = i

        for i, word in enumerate(words):
            # Case 1: word matches a reversed word (different index)
            if word in backward and backward[word] != i:
                res.add((i, backward[word]))
            
            # Case 2: word is a palindrome and there's an empty string
            if word != "" and "" in backward and word == word[::-1]:
                res.add((i, backward[""]))
                res.add((backward[""], i))
            
            # Case 3: split into prefix and suffix
            for j in range(len(word)):
                # Check suffix can form a pair with reversed suffix and prefix is a palindrome
                if word[j:] in backward and word[:j] == word[:j][::-1]:
                    res.add((backward[word[j:]], i))
                # Check prefix can form a pair with reversed prefix and suffix is a palindrome
                if word[:j] in backward and word[j:] == word[j:][::-1]:
                    res.add((i, backward[word[:j]]))
        
        return [list(pair) for pair in res]
```

### Explanation
1. **Hash Map Initialization**: The hash map `backward` is populated with reversed words as keys and their original indices as values.
2. **Direct Pairs Check**: For each word, if its reverse exists in the hash map and is not the same word, the pair is added to the result set.
3. **Empty String Handling**: If a word is a palindrome and there's an empty string, both possible pairs involving the empty string are added.
4. **Prefix and Suffix Splits**: For each possible split of the word into prefix and suffix, check if the suffix (or prefix) exists in the hash map and if the corresponding prefix (or suffix) is a palindrome. This ensures that the concatenation of the words forms a palindrome.

This approach efficiently checks all possible pairs using a hash map and handles edge cases such as empty strings and palindromic splits, ensuring all valid pairs are found without duplicates.
