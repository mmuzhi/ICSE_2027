=== RAW ANSWER ===
To solve this problem, we need to determine if it's possible to finish all courses given the prerequisites, which is essentially a problem of detecting cycles in a directed graph. The courses form nodes in the graph, and the prerequisites form directed edges from a course to its prerequisite. If there's a cycle in this graph, it's impossible to complete all courses; otherwise, it is possible.

### Approach
1. **Graph Representation**: Represent the courses and their prerequisites using an adjacency list. Each course maps to a list of its prerequisites.
2. **Cycle Detection**: Use Depth-First Search (DFS) to detect cycles in the directed graph. During DFS traversal, maintain a visited array to track the state of each course:
   - **0 (Unvisited)**: The course has not been processed yet.
   - **1 (Visiting)**: The course is currently in the recursion stack of DFS, meaning it's being processed.
   - **2 (Visited)**: The course and all its prerequisites have been processed without any cycles.
3. **DFS Traversal**: For each unvisited course, start DFS. If during DFS we encounter a course that is currently being visited (state 1), it indicates a cycle, and we return `False`. If all prerequisites for a course are processed successfully, mark it as visited (state 2) and proceed.
4. **Check All Courses**: Iterate through all courses. If any course leads to a cycle during DFS, return `False`; otherwise, after processing all courses, return `True`.

### Solution Code
```python
from collections import defaultdict
from typing import List

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        pre = defaultdict(list)
        for course, p in prerequisites:
            pre[course].append(p)
        
        visited = [0] * numCourses

        def dfs(course):
            if visited[course] == 1:
                return False
            if visited[course] == 2:
                return True
            visited[course] = 1
            for p in pre[course]:
                if not dfs(p):
                    return False
            visited[course] = 2
            return True
        
        for course in range(numCourses):
            if visited[course] == 0:
                if not dfs(course):
                    return False
        return True
```

### Explanation
1. **Graph Representation**: The `pre` dictionary is built where each key is a course, and the value is a list of its prerequisites. This adjacency list represents the directed graph.
2. **Visited Tracking**: The `visited` array keeps track of the state of each course during DFS. Initially, all courses are unvisited (state 0).
3. **DFS Function**: The `dfs` function checks the state of the current course:
   - If it's currently being visited (state 1), a cycle is detected, and it returns `False`.
   - If it's already visited (state 2), it returns `True` as this course has been processed.
   - Otherwise, it marks the course as visiting (state 1), processes all its prerequisites recursively, and if no cycles are found, marks the course as visited (state 2) and returns `True`.
4. **Main Loop**: The main loop iterates over each course. If any course is unvisited, it initiates DFS. If DFS detects a cycle, the function returns `False` immediately. If all courses are processed without cycles, it returns `True`.

This approach efficiently checks for cycles in the course prerequisite graph using DFS with three states, ensuring optimal performance and correctness.
