=== RAW ANSWER ===

To solve this problem, we need to find the minimum difference between the highest and lowest elements of any k consecutive elements in a sorted array. The optimal approach involves using a sliding window technique after sorting the array to efficiently compute the minimum difference.

### Approach
1. **Sort the Array**: Sorting the array ensures that the elements are in ascending order, which allows us to use a sliding window approach to check consecutive elements efficiently.
2. **Sliding Window Technique**: By maintaining a window of size k, we can slide the window from the start to the end of the array. For each window, we calculate the difference between the maximum and minimum elements (which are the first and last elements of the window due to the sorted order).
3. **Edge Cases Handling**: The problem constraints ensure that k is a valid value (1 ≤ k ≤ length of the array), so we don't need to handle invalid k values separately.

### Solution Code
```python
from typing import List

class Solution:
    def minimumDifference(self, nums: List[int], k: int) -> int:
        nums.sort()
        l, r = 0, k - 1
        res = float("inf")
        while r < len(nums):
            res = min(res, nums[r] - nums[l])
            r += 1
            l += 1
        return res
```

### Explanation
1. **Sorting the Array**: The array is sorted to ensure that the elements are in ascending order. This allows us to consider only consecutive elements for the minimum difference calculation.
2. **Initializing Pointers**: We initialize two pointers, `l` (left) starting at 0 and `r` (right) starting at `k-1`. This defines the initial window of size k.
3. **Sliding the Window**: We iterate through the array by moving both pointers to the right one step at a time. For each window, we compute the difference between the elements at the right and left pointers (which are the maximum and minimum elements of the current window, respectively).
4. **Updating the Result**: We keep track of the minimum difference encountered during the iteration and return it as the result.

This approach efficiently computes the minimum difference using a sliding window technique on a sorted array, ensuring a time complexity of O(n log n) due to sorting, followed by a linear scan O(n) where n is the length of the array.
