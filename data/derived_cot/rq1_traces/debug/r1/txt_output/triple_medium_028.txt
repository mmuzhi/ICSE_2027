=== RAW ANSWER ===

To solve this problem, we need to find the maximum gap between consecutive elements in a sorted list of integers. The approach involves sorting the list and then iterating through the sorted list to compute the maximum difference between consecutive elements.

### Approach
1. **Check for Edge Cases**: If the list has fewer than 2 elements, the maximum gap is 0.
2. **Sort the List**: Sorting the list ensures that consecutive elements are in order, allowing us to compute the gaps between them efficiently.
3. **Compute Maximum Gap**: Iterate through the sorted list and compute the absolute difference between each pair of consecutive elements. Track the maximum gap encountered during this iteration.

### Solution Code
```python
class Solution:
    def maximumGap(self, nums: list) -> int:
        if len(nums) < 2:
            return 0
        nums = sorted(nums)
        max_gap = 0
        for i in range(len(nums) - 1):
            current_gap = abs(nums[i] - nums[i+1])
            if current_gap > max_gap:
                max_gap = current_gap
        return max_gap
```

### Explanation
1. **Edge Case Handling**: The function first checks if the list has fewer than 2 elements. If so, it immediately returns 0 since no gaps can exist.
2. **Sorting**: The list is sorted to arrange elements in ascending order, which is necessary to compute consecutive gaps.
3. **Iterating and Computing Gaps**: By iterating through the sorted list, we compute the absolute difference between each consecutive pair of elements. The maximum of these differences is tracked and returned as the result.

This approach ensures that we efficiently find the maximum gap with a time complexity dominated by the sorting step, which is \(O(n \log n)\), where \(n\) is the number of elements in the list. The subsequent iteration through the list runs in \(O(n)\) time.
