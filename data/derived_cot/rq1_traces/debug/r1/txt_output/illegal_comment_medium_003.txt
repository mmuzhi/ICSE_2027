=== RAW ANSWER ===
To solve this problem, we need to convert a sorted singly linked list into a height-balanced binary search tree (BST). The key insight here is that the middle element of the sorted list should be the root of the BST, and the left and right subtrees should be constructed recursively from the elements before and after the middle element, respectively.

### Approach
1. **Convert Linked List to Array**: Traverse the linked list and store its elements in an array. This allows for efficient random access to the elements, which is necessary for finding the middle elements during BST construction.
2. **Recursive BST Construction**: Use a helper function to construct the BST. The helper function takes the left and right indices of the current subarray. The middle element of the subarray is chosen as the root of the current subtree. The left subtree is constructed from the elements to the left of the middle element, and the right subtree is constructed from the elements to the right of the middle element.
3. **Base Case Handling**: The recursion terminates when the left index exceeds the right index, indicating that no elements are left to process, and `None` is returned.

### Solution Code
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def sortedListToBST(self, head: Optional[ListNode]) -> Optional[TreeNode]:
        nums = []
        curr = head
        while curr:
            nums.append(curr.val)
            curr = curr.next
        
        def helper(l, r):
            if l > r:
                return None
            mid = (l + r) // 2
            root = TreeNode(nums[mid])
            root.left = helper(l, mid - 1)
            root.right = helper(mid + 1, r)
            return root
        
        return helper(0, len(nums) - 1)
```

### Explanation
1. **Linked List to Array Conversion**: The code traverses the linked list starting from the head node, appending each node's value to an array. This step ensures that we have random access to the elements, which simplifies the process of finding the middle element during BST construction.
2. **Recursive BST Construction**: The helper function `helper(l, r)` constructs the BST by selecting the middle element of the current subarray (defined by indices `l` and `r`) as the root of the subtree. The left and right subtrees are then constructed recursively from the subarrays to the left and right of the middle element.
3. **Base Case**: When the left index `l` exceeds the right index `r`, the function returns `None`, indicating that the current subtree is empty.
4. **Efficiency**: The conversion from linked list to array takes O(n) time, where n is the number of elements in the list. The recursive construction of the BST also takes O(n) time, as each element is processed exactly once. The overall time complexity is O(n), and the space complexity is O(n) due to the storage of the array.
