Our goal is to find the split point that minimizes this maximum value.

### Approach
1. **Prefix Sums**: Compute prefix sums for both rows of the grid. These prefix sums help in efficiently calculating the sum of any segment of the rows.
2. **Iterate Over Split Points**: For each possible split point (column where the first robot moves down), compute the maximum points the second robot can collect from either the left or right part of the grid.
3. **Calculate Options**: For each split point, calculate two options:
   - **Option 1**: Sum of the first row's left part and the second row's right part.
   - **Option 2**: Sum of the second row's left part and the first row's right part.
4. **Track Minimum Maximum**: Track the minimum value of the maximum points the second robot can collect across all possible split points.

### Solution Code
```python
from typing import List

class Solution:
    def gridGame(self, grid: List[List[int]]) -> int:
        n = len(grid[0])
        prefix0 = [0] * (n + 1)
        prefix1 = [0] * (n + 1)
        
        for i in range(n):
            prefix0[i+1] = prefix0[i] + grid[0][i]
            prefix1[i+1] = prefix1[i] + grid[1][i]
        
        sum0 = prefix0[n]
        sum1 = prefix1[n]
        
        result = float('inf')
        
        for i in range(n):
            option1 = prefix0[i] + (sum1 - prefix1[i])
            option2 = prefix1[i+1] + (sum0 - prefix0[i+1])
            current_max = max(option1, option2)
            result = min(result, current_max)
        
        return result
```

### Explanation
1. **Prefix Sums Calculation**: We compute prefix sums for both rows to quickly get the sum of any subarray. This helps in efficiently calculating the sums required for each split point.
2. **Iterate Over Split Points**: For each column `i`, we consider it as the split point where the first robot moves down from the first row to the second row.
3. **Option Calculation**:
   - **Option 1**: The sum of the first row's elements from the start to just before the split point (`prefix0[i]`) plus the sum of the second row's elements from the split point to the end (`sum1 - prefix1[i]`).
   - **Option 2**: The sum of the second row's elements from the start to the split point (`prefix1[i+1]`) plus the sum of the first row's elements from just after the split point to the end (`sum0 - prefix0[i+1]`).
4. **Track Minimum Maximum**: For each split point, we compute the maximum of the two options and keep track of the minimum value of these maxima across all split points.

This approach ensures that we efficiently compute the optimal path for the first robot, minimizing the maximum points the second robot can collect.
