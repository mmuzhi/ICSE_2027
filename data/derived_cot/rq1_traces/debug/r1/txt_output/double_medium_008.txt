=== RAW ANSWER ===
To solve this problem, we need to efficiently compute the total number of seats booked for each flight from 1 to n based on a list of bookings. Each booking specifies a range of flights [first, last] and the number of seats booked for each flight in that range. 

### Approach
The approach involves using a difference array technique to mark the start and end of each booking range. Here's the step-by-step breakdown:

1. **Initialization**: Create an array `arr` of size `n+1` initialized to zero. This array will help in marking the start and end of each booking range.
2. **Processing Bookings**: For each booking [first, last, seats]:
   - **Mark Start**: Increment the value at `arr[first-1]` by the number of seats. This marks the start of the booking range.
   - **Mark End**: Decrement the value at `arr[last]` by the number of seats. This marks the position right after the end of the booking range, ensuring that the seats are only added within the specified range.
3. **Prefix Sum Calculation**: Compute the prefix sum of the `arr` array. The prefix sum up to each index gives the total seats booked for the corresponding flight (since flight `i` corresponds to index `i-1` in the array).
4. **Result Extraction**: The result is the first `n` elements of the prefix sum array, representing the total seats for flights 1 to n.

### Solution Code
```python
from typing import List
from itertools import accumulate

class Solution:
    def corpFlightBookings(self, bookings: List[List[int]], n: int) -> List[int]:
        arr = [0] * (n + 1)
        for first, last, seats in bookings:
            arr[first - 1] += seats
            if last < n + 1:
                arr[last] -= seats
        prefix = list(accumulate(arr))
        return prefix[:n]
```

### Explanation
- **Initialization**: The array `arr` of size `n+1` is initialized to zero. This array will store the differences that mark the start and end of each booking range.
- **Processing Bookings**: For each booking, we add the seats at the starting flight index (adjusted to zero-based) and subtract the same number of seats at the index right after the ending flight. This ensures that when we compute the prefix sum, the seats are correctly allocated to all flights within the range.
- **Prefix Sum Calculation**: The prefix sum of the `arr` array is computed. This step aggregates the differences, resulting in the total seats booked for each flight up to that point.
- **Result Extraction**: The first `n` elements of the prefix sum array are returned, as they correspond to the total seats for flights 1 through n. The last element of the prefix sum array (index `n`) is not included because it represents the cumulative sum beyond the last flight, which is irrelevant to our solution.

This approach efficiently handles the range updates using the difference array technique, reducing the problem to a simple prefix sum computation with an optimal time complexity of O(n + m), where m is the number of bookings.
