=== RAW ANSWER ===
To solve this problem, we need to remove duplicates from a sorted array such that each element appears at most twice. The solution must be in-place, meaning we should not use extra space beyond modifying the input array. The function should return the new length of the array after duplicates have been removed.

### Approach
1. **Initial Check**: If the array length is less than 3, return the length immediately since no duplicates beyond two can exist in such cases.
2. **Two Pointers Technique**: Use two pointers, `p1` and `p2`, starting at indices 1 and 2 respectively. The pointer `p1` tracks the position where the next valid element should be placed, and `p2` scans through the array.
3. **Check for Third Duplicate**: For each element at `p2`, check if it is the same as the elements at `p1` and `p1-1`. If so, it means adding this element would result in three consecutive duplicates, so we skip it by incrementing `p2`.
4. **Place Valid Elements**: If the element at `p2` is not a third duplicate, increment `p1`, copy the element from `p2` to `p1`, and then increment both pointers.
5. **Return Result**: The new length of the array after processing is `p1 + 1` because `p1` is a zero-based index.

### Solution Code
```python
class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        if len(nums) < 3:
            return len(nums)
        
        p1, p2 = 1, 2
        
        while p2 < len(nums):
            if nums[p2] == nums[p1] and nums[p2] == nums[p1-1]:
                p2 += 1
            else:
                p1 += 1
                nums[p1] = nums[p2]
                p2 += 1
        
        return p1 + 1
```

### Explanation
- **Initial Check**: The first condition handles arrays of length 0, 1, or 2, which inherently have no more than two duplicates, so their lengths are returned as-is.
- **Two Pointers**: The pointers `p1` and `p2` start at positions 1 and 2. The `p1` pointer marks the end of the valid part of the array (elements processed so far with at most two duplicates), while `p2` traverses the rest of the array.
- **Skip Third Duplicates**: If the element at `p2` is the same as the elements at `p1` and `p1-1`, it forms a third duplicate, so `p2` is incremented to skip this element.
- **Place Valid Elements**: When the element at `p2` is not a third duplicate, it is copied to the position after `p1` (i.e., `p1 + 1`), and both pointers are incremented.
- **Result**: The new length is `p1 + 1` because `p1` is the last valid index, and the number of valid elements is one more than this index (since indices are zero-based).

This approach efficiently processes the array in a single pass, ensuring optimal time complexity of O(n) where n is the length of the array, and space complexity of O(1) as no additional space is used.
