=== RAW ANSWER ===

To solve this problem, we need to find the complement of a given positive integer. The complement of a number is obtained by flipping each bit in its binary representation. For example, the complement of 5 (which is 101 in binary) is 010, which is 2 in decimal.

### Approach
1. **Binary Conversion and Flipping**: Convert the given number to its binary form bit by bit. While converting, flip each bit (0 becomes 1 and 1 becomes 0).
2. **String Construction**: Construct a string of these flipped bits. Since we process the number from the least significant bit (LSB) to the most significant bit (MSB), the resulting string will be in reverse order.
3. **Reversing the String**: Reverse the constructed string to get the correct order of the flipped bits.
4. **Conversion to Integer**: Convert the reversed string from binary to an integer.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        com = ''
        while num > 0:
            if num % 2 == 1:
                com += '0'
            else:
                com += '1'
            num = num // 2
        return int(com[::-1], 2) if com else 0
```

### Explanation
1. **Binary Conversion and Flipping**: The loop processes each bit of the number by repeatedly dividing the number by 2. For each bit, if it is 1, we append '0' to the result string, and if it is 0, we append '1'.
2. **String Construction**: The result string is built in reverse order because we start from the LSB. For example, processing 5 (binary 101) results in the string '010' (LSB to MSB).
3. **Reversing the String**: The string is reversed to correct the order from MSB to LSB. For the example above, reversing '010' gives '010', which is the correct binary representation of the complement.
4. **Conversion to Integer**: The reversed string is converted back to an integer using Python's `int` function with base 2. If the input number is 0, the result is 0 as there are no bits to process.

This approach efficiently handles the conversion and flipping of bits, ensuring the correct result is obtained in linear time relative to the number of bits in the input number.
