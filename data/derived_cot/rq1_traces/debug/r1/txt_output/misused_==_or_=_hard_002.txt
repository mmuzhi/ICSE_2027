=== RAW ANSWER ===

To solve this problem, we need to find the longest cycle in a directed graph where each node has at most one outgoing edge. The approach involves traversing each node and using a time-based tracking mechanism to detect cycles efficiently.

### Approach
1. **Cycle Detection with Time Tracking**: For each node, we start a traversal and mark each visited node with the current time step. If we encounter a node that has already been visited in the current traversal, we calculate the cycle length using the difference in time steps.
2. **Visited Array**: We maintain an array `v` where each entry keeps track of the starting node of the traversal and the time step at which the node was visited. This helps in identifying cycles and avoiding reprocessing nodes from previous traversals.
3. **Early Termination**: If a node is encountered that was visited in a previous traversal, the loop breaks early to avoid unnecessary processing.

### Solution Code
```python
from typing import List

class Solution:
    def longestCycle(self, edges: List[int]) -> int:
        visited = [0] * len(edges)
        max_cycle = -1
        for node in range(len(edges)):
            time = 1
            current = node
            while current >= 0:
                if visited[current] != 0:
                    if visited[current][0] == node:
                        max_cycle = max(max_cycle, time - visited[current][1])
                    break
                else:
                    visited[current] = [node, time]
                    time += 1
                    current = edges[current]
        return max_cycle
```

### Explanation
1. **Initialization**: The `visited` array is initialized to keep track of each node's traversal start and time step. `max_cycle` is initialized to -1 to handle cases with no cycles.
2. **Traversal Loop**: For each node, we start a traversal and reset the time step. The `current` variable tracks the current node in the traversal.
3. **Cycle Check**: During traversal, if a node is revisited within the same traversal (same start node), the cycle length is calculated and `max_cycle` is updated. If the node was visited in a different traversal, the loop breaks early.
4. **Time Tracking**: Each node visited in the current traversal is marked with the start node and the current time step, which helps in detecting cycles and their lengths.

This approach efficiently detects cycles using time-based tracking and ensures each node is processed only once per traversal, leading to an optimal solution.
