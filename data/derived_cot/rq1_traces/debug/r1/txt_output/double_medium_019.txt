=== RAW ANSWER ===
To solve this problem, we need to efficiently compute the number of seat bookings for each flight based on a list of booking records. The booking records specify a range of flights (from `first` to `last` inclusive) and the number of seats booked for that range. The challenge is to handle these ranges optimally without resorting to a brute-force approach, which would be inefficient for large inputs.

### Approach
1. **Difference Array Technique**: The key insight is to use the difference array (or prefix sum array) technique. This method allows us to mark the start and end of each booking range efficiently. For each booking `[first, last, seats]`, we:
   - Add `seats` at the start index (`first - 1`).
   - Subtract `seats` at the index immediately after the end (`last`), indicating that subsequent flights beyond `last` should not include these seats.
2. **Prefix Sum Calculation**: After processing all bookings, we compute the prefix sum of the difference array. The prefix sum at each index gives the total seats booked for the corresponding flight (flight `i` corresponds to index `i-1` in the array).

### Solution Code
```python
class Solution:
    def corpFlightBookings(self, bookings: List[List[int]], n: int) -> List[int]:
        arr = [0] * (n + 2)
        for first, last, seats in bookings:
            arr[first - 1] += seats
            arr[last] -= seats
        res = []
        total = 0
        for i in range(n):
            total += arr[i]
            res.append(total)
        return res
```

### Explanation
1. **Initialization**: We initialize an array `arr` of size `n + 2` with zeros. The extra two positions (indices `n` and `n+1`) help manage the subtraction step without index-out-of-bounds errors.
2. **Processing Bookings**: For each booking `[first, last, seats]`:
   - **Addition at Start**: We add `seats` to `arr[first - 1]` because this marks the start of the booking range (flight `first`).
   - **Subtraction at End**: We subtract `seats` from `arr[last]` to indicate that flights beyond `last` should not include these seats. This subtraction affects flights starting from `last + 1`.
3. **Prefix Sum Calculation**: We iterate through the array, accumulating the values. The accumulated value at each index `i` (from `0` to `n-1`) represents the total seats booked for flight `i + 1`.
4. **Result Construction**: The results for each flight are stored in a list `res` and returned.

This approach efficiently handles the bookings in linear time relative to the number of bookings and flights, making it optimal for large inputs. The difference array technique ensures that each booking is processed in constant time, and the prefix sum calculation is linear in the number of flights.
