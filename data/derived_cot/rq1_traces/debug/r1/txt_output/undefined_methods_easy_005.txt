=== RAW ANSWER ===

To solve this problem, we need to determine if an array can be split into three non-empty parts such that each part has the same sum. The solution involves checking if the total sum of the array is divisible by three and then verifying if we can partition the array into three segments with the required sum.

### Approach
1. **Check Divisibility**: First, check if the total sum of the array is divisible by three. If not, it's impossible to split the array into three equal parts, so we return false immediately.
2. **Accumulate and Reset Sum**: Traverse the array while accumulating the sum. Each time the accumulated sum matches the target sum (which is one-third of the total sum), reset the accumulated sum and increment a counter.
3. **Check Remaining Elements**: After processing all elements, ensure that we have found exactly two valid segments and that the remaining elements sum up to the target sum. This ensures that the third segment is non-empty and valid.

### Explanation
1. **Divisibility Check**: The total sum of the array is checked for divisibility by three. If not divisible, return false.
2. **Accumulate and Reset**: As we iterate through the array, we accumulate the sum. Each time the accumulated sum matches the target (one-third of the total sum), we reset the accumulator and increment the count. This helps in identifying the first two valid segments.
3. **Final Check**: After processing all elements, we check if exactly two valid segments were found and if the remaining elements sum up to the target. This ensures the third segment is valid and non-empty.

This approach efficiently checks the necessary conditions in a single pass through the array, ensuring optimal performance with a time complexity of O(n), where n is the length of the array.
