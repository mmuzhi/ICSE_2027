=== RAW ANSWER ===

To solve this problem, we need to convert a given integer to an Excel column title. Excel columns are labeled in a way similar to a base-26 number system but with a twist: there is no zero. Instead, the columns start from 'A' (1) up to 'Z' (26), then continue with 'AA' (27), 'AB' (28), and so on.

### Approach
The key insight is to recognize that this problem is akin to converting a number to a different base, but with adjustments since there's no zero in this system. The approach involves repeatedly dividing the number by 26 and adjusting the remainder to map correctly to the letters 'A'-'Z'. Here's the step-by-step approach:

1. **Adjust the Number**: Since there's no zero, we subtract 1 from the number each time before processing to align the remainder correctly with the letters.
2. **Compute Remainder**: Use modulo 26 to find the current character.
3. **Convert to Character**: Convert the remainder to the corresponding character using ASCII values.
4. **Update the Number**: Divide the number by 26 to process the next higher place value.
5. **Build the Result**: Append each character to the result string and reverse it at the end to get the correct order.

### Solution Code
```python
class Solution:
    def convertToTitle(self, n: int) -> str:
        ans = ""
        while n > 0:
            n -= 1
            ans += chr(ord('A') + n % 26)
            n = n // 26
        return ans[::-1]
```

### Explanation
1. **Adjust the Number**: By decrementing `n` by 1 before each iteration, we adjust the number to a zero-based index, which simplifies the modulo operation.
2. **Compute Remainder**: The remainder after dividing by 26 gives the position of the current character in the alphabet.
3. **Convert to Character**: Using `chr(ord('A') + remainder)` converts the zero-based index to the corresponding uppercase letter.
4. **Update the Number**: Integer division by 26 moves to the next higher place value in the base-26 system.
5. **Build the Result**: Characters are appended in reverse order (least significant digit first), so the final result is reversed to produce the correct Excel column title.

This approach efficiently handles all cases, including edge cases like `n = 26` (which maps to 'Z') and larger numbers like `n = 702` (which maps to 'ZZ'). The algorithm runs in O(log n) time complexity due to the repeated division by 26, making it both time and space efficient.
