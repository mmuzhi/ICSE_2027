=== RAW ANSWER ===
To solve this problem, we need to find the length of the longest harmonious subsequence in an array. A harmonious subsequence is defined as a subsequence where the difference between the maximum and minimum elements is exactly 1. 

### Approach
1. **Frequency Count**: We first create a frequency map (dictionary) where each key is a number from the input array, and the value is the count of occurrences of that number in the array.
2. **Check Consecutive Numbers**: For each number in the frequency map, we check if there exists a consecutive number (i.e., `num + 1`). If such a number exists, the length of the harmonious subsequence formed by these two numbers is the sum of their counts.
3. **Update Result**: We keep track of the maximum length encountered during the iteration over the frequency map.

### Solution Code
```python
class Solution:
    def findLHS(self, nums: List[int]) -> int:
        result = 0
        count_map = {}
        for num in nums:
            if num not in count_map:
                count_map[num] = 1
            else:
                count_map[num] += 1
        for num, count in count_map.items():
            if num + 1 in count_map:
                result = max(count + count_map[num + 1], result)
        return result
```

### Explanation
1. **Frequency Map Construction**: We iterate through each number in the input array, updating the frequency map. If the number is not present in the map, we initialize its count to 1; otherwise, we increment its count.
2. **Finding Harmonious Subsequence**: For each number in the frequency map, we check if the next consecutive number (i.e., `num + 1`) exists in the map. If it does, the sum of the counts of these two numbers gives the length of a potential harmonious subsequence.
3. **Result Update**: We compare this sum with the current maximum length (`result`) and update `result` if the new sum is larger. After processing all numbers, `result` holds the length of the longest harmonious subsequence, or 0 if none exists.

This approach efficiently checks all possible pairs of consecutive numbers using the frequency map, ensuring optimal performance with a time complexity of O(n), where n is the number of elements in the input array. The space complexity is also O(n) due to the storage requirements for the frequency map.
