=== RAW ANSWER ===

To solve this problem, we need to find the kth largest level sum in a binary tree. The approach involves using a breadth-first search (BFS) to traverse the tree level by level, compute the sum of each level, and then determine the kth largest sum from these computed values.

### Approach
1. **Breadth-First Search (BFS)**: We use BFS to traverse the tree level by level. This allows us to compute the sum of values at each level efficiently.
2. **Handling Edge Cases**: If the tree is empty (i.e., the root is `None`), we immediately return -1 as there are no levels to consider.
3. **Sum Calculation**: For each level, we compute the sum of the node values and store these sums in an array.
4. **Sorting and Selection**: After collecting all level sums, we sort them in descending order. The kth largest element is then found at the (k-1)th index of the sorted array. If there are fewer than k levels, we return -1.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
import collections

class Solution:
    def kthLargestLevelSum(self, root: Optional[TreeNode], k: int) -> int:
        if not root:
            return -1
        dq = collections.deque([root])
        level_sums = []
        while dq:
            level_sum = 0
            level_length = len(dq)
            for _ in range(level_length):
                node = dq.popleft()
                level_sum += node.val
                if node.left:
                    dq.append(node.left)
                if node.right:
                    dq.append(node.right)
            level_sums.append(level_sum)
        level_sums.sort(reverse=True)
        return level_sums[k-1] if k <= len(level_sums) else -1
```

### Explanation
1. **Edge Case Handling**: The code first checks if the root is `None`. If so, it returns -1 immediately since there are no levels to process.
2. **BFS Traversal**: Using a deque, we traverse the tree level by level. For each level, we compute the sum of node values and store this sum in the `level_sums` list.
3. **Sum Calculation**: For each node in the current level, we add its value to `level_sum` and enqueue its children (if they exist) for processing in the next level.
4. **Sorting and Selection**: After processing all levels, the sums are sorted in descending order. The kth largest sum is then accessed at index `k-1` if there are at least k levels; otherwise, -1 is returned.

This approach ensures that we efficiently compute the required sums and correctly determine the kth largest value using sorting and index access.
