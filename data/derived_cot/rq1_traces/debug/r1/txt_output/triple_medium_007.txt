=== RAW ANSWER ===

To solve this problem, we need to perform a level order traversal of a binary tree from the bottom up. This means that we start from the leaf nodes and move up to the root, collecting values level by level.

### Approach
1. **Breadth-First Search (BFS)**: We use BFS to traverse the tree level by level. This is done using a queue to keep track of nodes at each level.
2. **Collect Levels**: As we traverse each level, we collect the values of the nodes at that level into a list.
3. **Reverse the Result**: After collecting all levels from top to bottom, we reverse the list of levels to get the bottom-up order.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        q, ans = [root], []
        while q:
            n = len(q)
            level = []
            for _ in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        ans.reverse()
        return ans
```

### Explanation
1. **Check for Empty Tree**: If the root is `None`, we return an empty list immediately.
2. **Initialize Queue and Result List**: We start with a queue containing the root node and an empty list to store the result.
3. **Process Each Level**: For each level, we determine the number of nodes (`n`) at the current level using the length of the queue. We then process each node in this level by dequeuing them, collecting their values, and enqueuing their children (if they exist).
4. **Collect Level Values**: The values of the nodes at each level are collected into a temporary list which is then appended to the result list.
5. **Reverse the Result**: After processing all levels, we reverse the result list to achieve the bottom-up order.

This approach efficiently traverses the tree using BFS, collects all levels, and then reverses the result to meet the problem's requirements. The time complexity is O(n), where n is the number of nodes in the tree, and the space complexity is O(n) due to the queue and result list storage.
