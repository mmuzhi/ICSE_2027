=== RAW ANSWER ===
To solve this problem, we need to compute the sum of the minimums of every subarray in a given array. The challenge lies in efficiently calculating this sum without resorting to a brute-force approach, which would be computationally expensive for large arrays.

### Approach
1. **Problem Analysis**: The problem requires summing the minimum elements of all possible subarrays of the given array. A naive approach would involve generating all subarrays and finding the minimum for each, leading to an O(n^3) time complexity, which is infeasible for large arrays (n ≤ 3 * 10^4).

2. **Intuition**: For each element in the array, determine how many subarrays have that element as the minimum. This can be done by finding the previous smaller element to the left and the next smaller element to the right of the current element. The number of such subarrays is given by the product of the distances from the current element to these boundaries.

3. **Algorithm Selection**: We use a monotonic stack to efficiently track the boundaries. The stack helps in maintaining indices of elements in a non-decreasing order. For each element, we pop elements from the stack that are larger than the current element to compute their contribution to the sum. The left boundary for the popped element is the next element in the stack (or the start of the array if the stack is empty), and the right boundary is the current index. After processing all elements, any remaining elements in the stack are processed with the right boundary as the end of the array.

4. **Complexity Analysis**: Each element is pushed and popped from the stack at most once, leading to an O(n) time complexity. The space complexity is O(n) due to the stack storage.

```python
        while stack:
            cur = stack.pop()
            left_bound = stack[-1] if stack else -1
            right_bound = n
            res = (res + arr[cur] * (cur - left_bound) * (right_bound - cur)) % mod
        
        return res
```

### Explanation
1. **Initialization**: We initialize a stack to keep track of indices, a result variable to store the sum, and the length of the array.
2. **Processing Elements**: For each element in the array:
   - While the stack is not empty and the current element is smaller than the element at the top of the stack, we pop the top element. The left boundary for this popped element is the new top of the stack (or the start of the array if the stack is empty), and the right boundary is the current index. We then update the result by adding the contribution of the popped element (its value multiplied by the number of subarrays where it is the minimum).
   - Push the current index onto the stack.
3. **Remaining Elements**: After processing all elements, any remaining indices in the stack are processed similarly, with the right boundary set to the end of the array.
4. **Modulo Operation**: The result is taken modulo \(10^9 + 7\) to handle large numbers as specified.

This approach efficiently computes the sum by leveraging the properties of a monotonic stack to determine the contribution of each element in linear time.
