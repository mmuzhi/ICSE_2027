To solve this problem, we need to compute the maximum depth of an n-ary tree using a breadth-first search (BFS) approach. The depth of a tree is defined as the number of edges from the root node to the farthest leaf node. The BFS approach processes the tree level by level, incrementing the depth counter each time we move to the next level.

### Approach
1. **Initialization**: Start by checking if the root node is `None`. If so, return 0 because an empty tree has a depth of 0.
2. **BFS Setup**: Initialize a queue (`nodes_queue`) with the root node and an empty queue (`next_nodes_queue`) to hold nodes of the next level. Initialize `depth` to 0.
3. **Level Processing**: While the current queue is not empty, dequeue each node from the front of the queue. For each dequeued node, enqueue all its children into `next_nodes_queue` if they exist.
4. **Level Completion**: After processing all nodes in the current queue, swap `nodes_queue` and `next_nodes_queue` to process the next level. Increment `depth` to indicate completion of the current level.
5. **Termination**: The loop terminates when there are no more nodes to process. The `depth` variable holds the maximum depth of the tree, which is then returned.

### Solution Code
```python
"""
```

1. **Initial Check**: The method first checks if the root is `None`. If true, it returns 0 immediately.
2. **BFS Initialization**: The BFS starts with the root node in `nodes_queue`. The `depth` is initialized to 0.
3. **Processing Nodes**: For each node dequeued from `nodes_queue`, all its children are enqueued into `next_nodes_queue` if they exist.

5. **Termination**: The loop continues until there are no more nodes to process. The final `depth` value, representing the maximum depth of the tree, is returned.
