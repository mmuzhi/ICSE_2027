To solve this problem, we need to reverse nodes in a linked list in groups of size k. If the remaining nodes are fewer than k, they should remain as they are. The approach involves iterating through the list, reversing each group of k nodes, and properly linking the reversed groups together.

### Approach
1. **Dummy Node Initialization**: Use a dummy node to simplify edge cases, such as handling the head of the list.
2. **Finding k-th Node**: For each group, determine the k-th node to identify the end of the current group.
3. **Reversing the Group**: Reverse the nodes in the current group using a standard reversal technique, adjusting pointers to maintain the list structure.
4. **Linking Groups**: After reversing a group, link the previous group's end to the new head of the reversed group and update pointers for the next iteration.

### Solution Code
```python
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        curr_group = dummy

        while True:
            kth = self.getKth(curr_group, k)
            if not kth:
                break
            next_group = kth.next

            # Reverse the current group
            prev, curr = next_group, curr_group.next
            while curr != next_group:
                tmp = curr.next
                curr.next = prev
                prev = curr
                curr = tmp

            # Update pointers to connect the reversed group
            tmp = curr_group.next
            curr_group.next = kth  # kth is the new head after reversal
            curr_group = tmp  # Move to the end of the current reversed group

        return dummy.next

    def getKth(self, node, k):
        while node and k > 0:
            node = node.next
            k -= 1
        return node
```

### Explanation
1. **Dummy Node**: The dummy node helps handle edge cases by providing a consistent starting point.
2. **Finding k-th Node**: The `getKth` function moves k steps from a given node to find the end of the current group. If there are fewer than k nodes remaining, it returns `None`, indicating no more groups to reverse.
3. **Reversing the Group**: Starting from the first node of the current group, we reverse the links of each node until we reach the end of the group. This involves adjusting the `next` pointers to point to the previous node, effectively reversing the group.
4. **Linking Groups**: After reversing a group, the previous group's end is linked to the new head of the reversed group. The current group pointer is then updated to the end of the reversed group to process the next group.

This approach ensures that each group of k nodes is reversed efficiently, and the overall list structure is maintained correctly. The time complexity is O(n), where n is the number of nodes, as each node is processed exactly twice (once for traversal and once for reversal).
