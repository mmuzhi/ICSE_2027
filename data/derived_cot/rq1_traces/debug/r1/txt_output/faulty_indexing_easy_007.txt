=== RAW ANSWER ===
To solve this problem, we need to implement a binary search algorithm to find the index of a target value in a sorted array. The initial code had a few bugs: it inefficiently checked if the target was in the array using a linear scan, and it returned the element at the found index instead of the index itself. 

### Approach
1. **Binary Search Setup**: Initialize `left` to 0 and `right` to the length of the array. These pointers define the search range [left, right).
2. **Binary Search Execution**: While `left` is less than `right`, compute the middle index `mid` of the current range. If the element at `mid` is less than the target, adjust `left` to `mid + 1` to search the right half. Otherwise, adjust `right` to `mid` to search the left half.
3. **Post-Search Check**: After the loop, check if `left` is within the array bounds and if the element at `left` matches the target. If so, return `left`; otherwise, return -1 indicating the target was not found.

This approach efficiently narrows down the search range by half each iteration, leading to an O(log n) time complexity, which is optimal for binary search.

### Solution Code
```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums)
        while left < right:
            mid = (left + right) // 2
            if nums[mid] < target:
                left = mid + 1
            else:
                right = mid
        if left < len(nums) and nums[left] == target:
            return left
        else:
            return -1
```

### Explanation
- **Initialization**: The search starts with `left` at 0 and `right` at the array length, defining the initial range [0, n).
- **Loop Execution**: The loop continues as long as `left` is less than `right`. The middle index `mid` is calculated, and the array element at `mid` is compared to the target. If it is less than the target, the search continues in the right half by updating `left` to `mid + 1`. Otherwise, the search continues in the left half by updating `right` to `mid`.
- **Post-Loop Check**: After exiting the loop, `left` points to the first element greater than or equal to the target. If this element matches the target, `left` is returned; otherwise, -1 is returned, indicating the target was not found in the array.

This approach efficiently locates the target (if present) in logarithmic time, adhering to the principles of binary search.
