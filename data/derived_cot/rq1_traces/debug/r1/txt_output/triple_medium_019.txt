=== RAW ANSWER ===

To solve this problem, we need to compute the sum of the minimum values of all possible subarrays of a given array. The approach involves using a monotonic stack to efficiently determine the contribution of each element as the minimum in various subarrays.

### Approach
1. **Monotonic Stack**: We use a stack to maintain indices of elements in increasing order. This helps in efficiently finding the previous and next smaller elements for each element in the array.
2. **Contribution Calculation**: For each element, when it is popped from the stack, we calculate its contribution as the minimum in subarrays. This is determined by the distance to the previous smaller element (left boundary) and the next smaller element (right boundary).
3. **Handling Remaining Elements**: After processing all elements, any remaining elements in the stack are processed to account for subarrays where these elements are the minimum up to the end of the array.

### Solution Code
```python
class Solution:
    def sumSubarrayMins(self, arr: List[int]) -> int:
        stack = []
        res = 0
        
        for i, num in enumerate(arr):
            while stack and arr[stack[-1]] > num:
                cur = stack.pop()
                left = stack[-1] if stack else -1
                res += arr[cur] * (i - cur) * (cur - left)
            stack.append(i)
        
        # Process remaining elements in the stack
        n = len(arr)
        while stack:
            cur = stack.pop()
            left = stack[-1] if stack else -1
            res += arr[cur] * (n - cur) * (cur - left)
        
        return res % (10**9 + 7)
```

### Explanation
1. **Monotonic Stack**: The stack is used to keep track of indices in a way that ensures the elements are in increasing order. This allows us to efficiently find the previous and next smaller elements.
2. **Contribution Calculation**: When an element is popped from the stack (indicating a next smaller element has been found), we calculate the number of subarrays where the popped element is the minimum. This is done using the distances to the previous smaller element (left boundary) and the next smaller element (right boundary).
3. **Handling Remaining Elements**: After processing all elements, any elements left in the stack are processed to account for subarrays extending to the end of the array, as these elements do not have a next smaller element within the array bounds.

This approach efficiently computes the sum in O(n) time complexity, where n is the length of the array, by leveraging the properties of the monotonic stack to avoid redundant calculations.
