=== RAW ANSWER ===
To solve this problem, we need to find the maximum gap between successive elements in a sorted array. The maximum gap is the largest difference between any two consecutive elements when the array is sorted in ascending order. 

### Approach
1. **Sort the Array**: First, we sort the array to ensure that all elements are in ascending order. This allows us to easily compute the differences between consecutive elements.
2. **Edge Case Handling**: If the array has fewer than two elements, the maximum gap is zero because there are no two elements to compare.
3. **Iterate Through Sorted Array**: For each pair of consecutive elements in the sorted array, compute the absolute difference between them.
4. **Track Maximum Gap**: Maintain a variable to keep track of the maximum gap encountered during the iteration. Update this variable whenever a larger gap is found.

### Solution Code
```python
class Solution:
    def maximumGap(self, nums: List[int]) -> int:
        if len(nums) < 2:
            return 0
        nums = sorted(nums)
        max_gap = 0
        for i in range(len(nums)-1):
            gap = abs(nums[i] - nums[i+1])
            if gap > max_gap:
                max_gap = gap
        return max_gap
```

### Explanation
1. **Sorting the Array**: The array is sorted to arrange elements in ascending order, which simplifies the process of finding consecutive differences.
2. **Edge Case Check**: If the array contains fewer than two elements, the function immediately returns 0 since no gaps can exist.
3. **Finding Maximum Gap**: The function initializes `max_gap` to 0. It then iterates through each consecutive pair of elements in the sorted array, calculates their absolute difference, and updates `max_gap` if the current difference is larger than the previously recorded maximum.
4. **Result**: After processing all consecutive pairs, the function returns the largest gap found.

This approach efficiently computes the maximum gap by leveraging sorting and a single pass through the sorted array, ensuring optimal performance with a time complexity dominated by the sorting step, typically O(n log n).
