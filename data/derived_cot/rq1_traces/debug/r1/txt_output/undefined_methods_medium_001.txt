=== RAW ANSWER ===

To solve this problem, we need to compute the sum of absolute differences for each element in a sorted array. The sum of absolute differences for an element at index `i` is the sum of the differences between that element and all elements before it, plus the sum of the differences between that element and all elements after it. 

### Approach
The key insight here is to leverage prefix sums to efficiently compute these differences. By maintaining a running sum of elements before the current index (`s1`) and a running sum of elements from the current index to the end (`s2`), we can compute the required sums in linear time.

1. **Initialization**: Compute the total sum of the array and initialize two running sums, `s1` (sum of elements before the current index) and `s2` (sum of elements from the current index to the end).
2. **Iterate through the array**: For each element at index `i`, compute the sum of absolute differences using the running sums:
   - **Left part**: The sum of differences between the current element and all elements before it.
   - **Right part**: The sum of differences between the current element and all elements after it.
3. **Update running sums**: Adjust `s1` and `s2` as we move to the next element.

### Solution Code
```python
from typing import List

class Solution:
    def getSumAbsoluteDifferences(self, nums: List[int]) -> List[int]:
        n = len(nums)
        total = sum(nums)
        s1 = 0
        s2 = total
        arr = []
        for i in range(n):
            left = nums[i] * i - s1
            right = (s2 - nums[i]) - nums[i] * (n - i - 1)
            arr.append(left + right)
            s1 += nums[i]
            s2 = total - s1
        return arr
```

### Explanation
1. **Initialization**: `total` is the sum of all elements in the array. `s1` starts at 0 (sum of elements before the first element), and `s2` starts at `total` (sum of all elements initially).
2. **Loop through each element**:
   - **Left part**: For each element at index `i`, the sum of differences with elements before it is calculated as `nums[i] * i - s1`. Here, `nums[i] * i` is the theoretical sum if all previous elements were `nums[i]`, and `s1` is the actual sum of previous elements.
   - **Right part**: The sum of differences with elements after it is calculated as `(s2 - nums[i]) - nums[i] * (n - i - 1)`. Here, `s2 - nums[i]` is the sum of elements after `i`, and `nums[i] * (n - i - 1)` is the theoretical sum if all subsequent elements were `nums[i]`.
3. **Update running sums**: After processing each element, `s1` is incremented by the current element, and `s2` is updated to `total - s1` to reflect the sum of elements from the next index to the end.

This approach efficiently computes the required sums in linear time, making it optimal for large arrays.
