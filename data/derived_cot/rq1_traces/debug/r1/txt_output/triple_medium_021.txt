To solve this problem, we need to determine the Kth bit in the sequence generated after N steps. The sequence is built recursively where each step S(n) is formed by concatenating S(n-1), '1', and the reverse of the inverted S(n-1). The challenge is to efficiently find the Kth bit without generating the entire sequence, which can be very large for higher values of N.

### Approach
1. **Recursive Strategy**: The sequence S(n) is built such that the first half is S(n-1), the middle element is always '1', and the second half is the reverse of the inverted S(n-1). This structure allows us to use a recursive approach to determine the Kth bit without constructing the entire sequence.
2. **Midpoint Calculation**: For each step n, the midpoint of the sequence S(n) is at position 2^(n-1). This midpoint is always '1'.
3. **Recursive Cases**:
   - If K is 1, the result is '0' if the current inversion flag (R) is true, otherwise '1'.
   - If K is exactly at the midpoint, the result is '1' if R is true, otherwise '0'.
   - If K is in the first half, recursively check the same position in S(n-1).
   - If K is in the second half, adjust the position to the mirrored position in the first half and invert the flag R.

### Solution Code
```python
class Solution:
    def findKthBit(self, N, K, R=True):
        if K == 1:
            return '0' if R else '1'
        mid = 1 << (N - 1)
        if K == mid:
            return '1' if R else '0'
        elif K < mid:
            return self.findKthBit(N - 1, K, R)
        else:
            return self.findKthBit(N - 1, 2 * mid - K, not R)
```

### Explanation
- **Base Case**: When K is 1, the result is determined by the inversion flag R. If R is true, the bit is '0'; otherwise, it is '1'.
- **Midpoint Check**: If K is at the midpoint (2^(n-1)), the result is '1' if R is true, otherwise '0'.
- **First Half Handling**: If K is in the first half of the sequence, we recursively check the same position in the previous step (S(n-1)).
- **Second Half Handling**: If K is in the second half, we adjust the position to the mirrored position in the first half and invert the flag R to account for the reverse and invert operations in the sequence construction.

This approach efficiently narrows down the position of K by leveraging the recursive structure of the sequence, ensuring we do not generate the entire sequence and thus optimizing both time and space complexity.
