=== RAW ANSWER ===

To solve this problem, we need to determine the number of ways to split a given string `s` into substrings such that each substring is a valid number between 1 and `k` (inclusive). Each valid number must not have leading zeros unless it is exactly "0", but since `k` is at least 1, leading zeros are not allowed.

### Approach
The approach uses dynamic programming with memoization to efficiently count the valid splits. The key steps are:
1. **Recursive Function with Memoization**: We define a recursive function `rec(i)` that returns the number of valid splits starting from index `i` in the string. The results of this function are cached to avoid redundant computations.
2. **Base Cases**: 
   - If the current index `i` reaches the end of the string, return 1 (indicating a valid split).
   - If the current character is '0', return 0 (since numbers with leading zeros are invalid).
3. **Iterate Over Possible Substrings**: For each starting index `i`, iterate over possible ending indices `x` to form substrings. Convert the substring to a number and check if it is within the valid range `[1, k]`.
4. **Early Termination**: If the current substring exceeds `k`, break the loop early to avoid unnecessary checks.

### Solution Code
```python
class Solution:
    def numberOfArrays(self, s: str, k: int) -> int:
        n = len(s)
        mod = 10**9 + 7

        from functools import lru_cache

        @lru_cache(maxsize=None)
        def rec(i=0):
            if i == n:
                return 1
            if s[i] == '0':
                return 0
            res = 0
            curr = ""
            for x in range(i, n):
                curr += s[x]
                num = int(curr)
                if num > k:
                    break
                res = (res + rec(x + 1)) % mod
            return res % mod

        return rec()
```

### Explanation
- **Recursive Function**: The function `rec(i)` computes the number of valid splits starting from index `i`. It uses memoization (`lru_cache`) to store results of previously computed indices to avoid redundant calculations.
- **Base Cases**: When `i` reaches the end of the string (`i == n`), it returns 1. If the current character is '0', it returns 0 immediately as leading zeros are invalid.
- **Substring Formation**: For each starting index `i`, the loop forms substrings by extending the end index `x`. Each substring is converted to an integer and checked against `k`.
- **Early Termination**: If a substring exceeds `k`, the loop breaks early to save computation time.
- **Modulo Operation**: Results are taken modulo `10^9 + 7` to handle large numbers and prevent overflow.

This approach efficiently counts valid splits using dynamic programming and memoization, ensuring optimal performance even for longer strings.
