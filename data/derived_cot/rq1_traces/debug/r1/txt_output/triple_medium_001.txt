=== RAW ANSWER ===

To solve this problem, we need to determine the Kth bit in the sequence generated after N steps. The sequence is built recursively, where each step appends a '1' followed by the reversed and inverted version of the previous step's sequence. The challenge is to efficiently determine the Kth bit without generating the entire sequence, which can be very large for higher values of N.

### Approach
1. **Recursive Structure**: The sequence S(n) is formed as S(n-1) + '1' + reverse(invert(S(n-1))). Here, reverse(invert(S(n-1))) means we first invert each bit of S(n-1) (changing '0' to '1' and vice versa) and then reverse the resulting string.
2. **Divide and Conquer**: Instead of generating the entire sequence, we use a divide and conquer approach to determine the Kth bit by breaking down the problem into smaller subproblems.
3. **Midpoint Calculation**: For each step n, the midpoint of the sequence is at position 2^(n-1). This midpoint is always '1' in the original sequence and '0' if inverted.
4. **Recursive Checks**:
   - If K is 1, return '0' if the current segment is not inverted (R is True), otherwise return '1'.
   - If K is less than the midpoint, recurse on the left part of the sequence (S(n-1)).
   - If K is exactly the midpoint, return '1' if the current segment is not inverted, otherwise return '0'.
   - If K is greater than the midpoint, adjust K to the corresponding position in the reversed and inverted part of the sequence and recurse.

### Solution Code
```python
class Solution:
    def findKthBit(self, N, K, R=True):
        if K == 1:
            return '0' if R else '1'
        mid = 1 << (N - 1)
        if K < mid:
            return self.findKthBit(N - 1, K, R)
        elif K == mid:
            return '1' if R else '0'
        else:
            return self.findKthBit(N - 1, 2 * mid - K, not R)
```

### Explanation
- **Base Case**: When K is 1, the bit is '0' if the current segment is not inverted (R is True) and '1' if it is inverted (R is False).
- **Midpoint Handling**: The midpoint of the sequence at step N is calculated as 2^(N-1). If K is exactly this midpoint, the bit is '1' if not inverted and '0' if inverted.
- **Left Part Handling**: If K is in the left part (less than the midpoint), we recurse on the left part (S(n-1)) without changing the inversion flag.
- **Right Part Handling**: If K is in the right part (greater than the midpoint), we adjust K to the corresponding position in the reversed and inverted part of S(n-1) and toggle the inversion flag (not R).

This approach efficiently narrows down the position of K by leveraging the recursive structure of the sequence, ensuring we do not generate the entire sequence and thus optimizing both time and space complexity.
