=== RAW ANSWER ===
To solve this problem, we need to determine if it's possible to finish all courses given some prerequisite constraints. The problem can be modeled as detecting cycles in a directed graph where each course is a node and each prerequisite constraint is a directed edge from a course to its prerequisite. If there's a cycle in the graph, it's impossible to finish all courses; otherwise, it's possible.

### Approach
1. **Graph Representation**: Represent the courses and their prerequisites using an adjacency list. Each course maps to a list of its prerequisites.
2. **Cycle Detection**: Use Depth-First Search (DFS) to detect cycles in the graph. During DFS traversal, maintain three states for each node:
   - **Unvisited (0)**: The node hasn't been processed yet.
   - **Visiting (1)**: The node is currently in the recursion stack of the DFS traversal.
   - **Visited (2)**: The node and all its prerequisites have been processed without detecting any cycle.
3. **DFS Traversal**: For each unvisited node, start DFS. If during DFS a node is encountered that is already in the visiting state, a cycle is detected, and it's impossible to finish all courses. If no cycles are detected after processing all nodes, it's possible to finish all courses.

### Solution Code
```python
from collections import defaultdict
from typing import List

class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        pre = defaultdict(list)
        for course, p in prerequisites:
            pre[course].append(p)
        
        state = [0] * numCourses
        
        def dfs(course):
            if state[course] == 2:
                return True
            if state[course] == 1:
                return False
            state[course] = 1
            for p in pre[course]:
                if not dfs(p):
                    return False
            state[course] = 2
            return True
        
        for course in range(numCourses):
            if state[course] == 0:
                if not dfs(course):
                    return False
        
        return True
```

### Explanation
1. **Graph Representation**: The adjacency list `pre` is built using a `defaultdict` where each course points to its list of prerequisites. This allows efficient access to the prerequisites during DFS.
2. **State Tracking**: The `state` array keeps track of the processing state of each course. Initially, all courses are unvisited (state 0).
3. **DFS Traversal**: For each course that hasn't been visited, the DFS function is called. During DFS:
   - If the course is already marked as visiting (state 1), a cycle is detected, and the function returns `False`.
   - Otherwise, the course is marked as visiting (state 1), and its prerequisites are processed recursively.
   - If all prerequisites are processed without detecting a cycle, the course is marked as visited (state 2).
4. **Cycle Detection**: If any course during DFS traversal leads to a cycle, the function immediately returns `False`. If all courses are processed without detecting any cycles, the function returns `True`, indicating all courses can be finished.
