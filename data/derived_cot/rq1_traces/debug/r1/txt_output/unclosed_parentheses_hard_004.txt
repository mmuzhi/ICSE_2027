To solve this problem, we need to determine the maximum number of components we can split a tree into such that each component has the same sum. The solution involves checking possible sums (candidates) for each component and verifying if the tree can be divided accordingly using a depth-first search (DFS) approach.

### Approach
1. **Tree Construction**: Convert the given edges into an adjacency list to represent the tree.
2. **DFS Function**: Use a post-order DFS to check if each subtree can be split into components with the desired sum. If a subtree's sum matches the candidate value, it is considered a valid component.
3. **Candidate Check**: Iterate through possible candidate sums from 1 up to half of the total sum of the tree's node values. For each candidate, check if the total sum is divisible by the candidate and if the tree can be split into components with that sum using the DFS function.
4. **Result Calculation**: If a valid candidate is found, return the maximum number of components minus one (since the number of splits is one less than the number of components).

```python
        def fn(u, p):
            """Post-order dfs."""
            ans = nums[u]
            for v in tree[u]: 
                if v != p: 
                    ans += fn(v, u)
            return 0 if ans == cand else ans
        
        total = sum(nums)
        for cand in range(1, total // 2 + 1):
            if total % cand == 0 and fn(0, -1) == 0:
                return total // cand - 1
        return 0
```

### Explanation
1. **Tree Construction**: The tree is built using an adjacency list where each node points to its connected nodes.
2. **DFS Function**: The `fn` function performs a post-order DFS traversal. It calculates the sum of the current node and its descendants. If the sum matches the candidate value, it returns 0 (indicating a valid component), otherwise, it returns the sum.
3. **Candidate Check**: The main loop checks each candidate value from 1 to half of the total sum. For each candidate, it verifies if the total sum is divisible by the candidate and if the DFS function confirms the tree can be split into components of that sum.
4. **Result Calculation**: The maximum number of components is determined by the smallest valid candidate, which gives the highest number of components. The result is the number of components minus one, as the number of splits needed is one less than the number of components.

This approach efficiently checks all possible candidates and uses DFS to validate each candidate, ensuring correctness and optimal performance.
