To solve this problem, we need to determine the maximum profit that can be obtained from multiple transactions in a given list of stock prices, where each transaction consists of buying at a lower price and selling at a higher price. The solution should handle multiple transactions to maximize the profit.

### Approach
The approach used here is based on identifying the optimal points to buy and sell stocks. The key idea is to buy at the lowest possible price (valley) and sell at the highest possible price (peak) that follows. This is done by iterating through the list of prices and making transactions whenever a peak is detected. The algorithm ensures that we accumulate profits from all valid transactions.

### Solution Code
```python
from typing import List

class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        if len(prices) < 2:
            return 0
        if len(prices) == 2:
            output = prices[1] - prices[0]
            return output if output > 0 else 0
        
        i = 0
        j = 1
        stockBuy = prices[i]
        stockSell = prices[j]
        profit = 0
        counter = 0
        
        while counter < len(prices) - 2:
            if stockSell - stockBuy < 0:
                i = counter + 1
                j = i + 1
            else:
                if j + 1 < len(prices) and prices[j+1] > prices[j]:
                    j += 1
                else:
                    profit += stockSell - stockBuy
                    i = j + 1
                    j = i + 1
                    if j >= len(prices):
                        break
            
            stockBuy = prices[i]
            stockSell = prices[j]
            counter += 1
        
        if j < len(prices) and stockSell - stockBuy > 0:
            profit += stockSell - stockBuy
        
        return profit
```

### Explanation
1. **Initial Checks**: The code first checks if the list has fewer than 2 elements, returning 0 as no transactions can be made. For a list of exactly 2 elements, it directly computes the profit.

2. **Initialization**: The variables `i` and `j` are initialized to track the buy and sell indices, starting at 0 and 1 respectively. `stockBuy` and `stockSell` hold the prices at these indices.

3. **Main Loop**: The loop runs from `counter = 0` to `len(prices) - 2` to ensure valid index access. 
   - If selling at the current `j` results in a loss, the buy and sell points are moved to the next possible positions.
   - If selling at the current `j` results in a profit, the code checks if the next day's price is higher. If so, it extends the selling point. Otherwise, it records the profit and resets the buy and sell points to the next valid positions.

4. **Final Check**: After the loop, a final check ensures any remaining valid transaction is recorded.

This approach efficiently tracks the optimal buy and sell points, ensuring maximum profit by accumulating all valid transactions. The solution handles edge cases and ensures no index out-of-bounds errors.
