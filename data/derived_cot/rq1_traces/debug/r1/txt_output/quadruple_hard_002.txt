To solve this problem, we need to determine the maximum number of times we can form the string `s2` repeated `n2` times from the string `s1` repeated `n1` times. The solution involves detecting cycles in the repetition pattern to optimize the calculation.

### Approach
1. **Initial Check**: Verify if all characters of `s2` are present in `s1`. If not, return 0 immediately.
2. **Optimize `s1`**: Filter `s1` to include only characters present in `s2` to streamline processing.
3. **Cycle Detection**: Track the number of times `s1` is cycled through to form `s2` and detect cycles to avoid redundant calculations.
4. **Cycle Calculation**: Once a cycle is detected, compute the number of cycles and the remainder to determine the maximum repetitions efficiently.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def getMaxRepetitions(self, s1: str, n1: int, s2: str, n2: int) -> int:
        rec, track = [0], defaultdict(int)
        ct = start = 0

        if not set(s2).issubset(set(s1)):
            return 0

        s1 = ''.join(char for char in s1 if char in set(s2))

        while True:
            for char in s2:
                ptr = s1.find(char, start)
                if ptr == -1:
                    ct += 1
                    ptr = s1.find(char)
                    if ptr == -1:
                        return 0
                start = ptr + 1
            rec.append(ct + 1)

            if rec[-1] > n1:
                return (len(rec) - 2) // n2

            if ptr in track:
                break
            track[ptr] = len(rec) - 1

        cycle_start_idx = track[ptr]
        cycle_start = rec[cycle_start_idx]
        cycle1 = ct + 1 - cycle_start
        cycle2 = len(rec) - 1 - cycle_start_idx
        rest = n1 - cycle_start

        full_cycles = rest // cycle1
        rem = cycle_start + full_cycles * cycle1

        ptr2 = 0
        while ptr2 < len(rec) and rec[ptr2] <= rem:
            ptr2 += 1

        total_s2 = cycle2 * full_cycles + (ptr2 - 1)
        return total_s2 // n2
```

### Explanation
1. **Initial Check**: We first check if all characters of `s2` are present in `s1`. If not, it's impossible to form `s2` from `s1`, so we return 0.
2. **Optimize `s1`**: We filter `s1` to keep only the characters that are in `s2`, which helps in reducing unnecessary checks.
3. **Cycle Detection**: As we iterate through `s2` to find characters in `s1`, we track the number of times we cycle through `s1` (using `ct`). We store the position of the last character of `s2` in `s1` to detect cycles.
4. **Cycle Calculation**: When a cycle is detected, we compute the start and length of the cycle. Using this information, we calculate how many full cycles fit into the remaining repetitions of `s1` and handle the remainder to find the total number of `s2` repetitions possible. Finally, we divide by `n2` to get the result.

This approach efficiently handles large repetitions by leveraging cycle detection, ensuring the solution is optimal even for large values of `n1` and `n2`.
