To solve this problem, we need to find a peak element in an array. A peak element is an element that is greater than or equal to its neighbors. If the array contains only one element, that element is trivially a peak. For arrays with more than one element, we use binary search to efficiently locate a peak.

### Approach
1. **Problem Analysis**: The task is to find any peak element in an array. A peak element is defined as an element that is greater than or equal to its adjacent elements. For edge elements (first or last), only one neighbor needs to be considered.
2. **Binary Search**: The solution leverages binary search to efficiently find a peak. The algorithm checks the middle element of the current search range:
   - If the middle element is a peak (i.e., it is greater than or equal to its neighbors), return its index.
   - If the middle element is not a peak, move the search towards the side where the adjacent element is larger. This ensures that a peak will be found since the larger adjacent element indicates a potential peak in that direction.
3. **Edge Handling**: Special cases for the first and last elements are handled by checking if the current element is at the boundary, thus avoiding out-of-bounds comparisons.

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        if n == 1:
            return 0
        
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] >= nums[mid-1]) and (mid == n-1 or nums[mid] >= nums[mid+1]):
                return mid
            elif mid < n-1 and nums[mid] <= nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

### Explanation
1. **Initial Check**: If the array has only one element, return 0 as it is trivially a peak.
2. **Binary Search Setup**: Initialize `low` to 0 and `high` to the last index of the array.
3. **Loop Until Peak Found**: While `low` is less than or equal to `high`:
   - Calculate the middle index `mid`.
   - Check if `mid` is a peak by verifying:
     - For the first element (`mid == 0`), it only needs to be greater than or equal to the next element.
     - For the last element (`mid == n-1`), it only needs to be greater than or equal to the previous element.
     - For any other element, it must be greater than or equal to both neighbors.
   - If `mid` is a peak, return `mid`.
   - If the element at `mid` is less than or equal to the next element (and `mid` is not the last element), move the search to the right half (`low = mid + 1`).
   - Otherwise, move the search to the left half (`high = mid - 1`).
4. **Return -1 if No Peak Found**: Although the problem guarantees a peak, this handles any unexpected cases.

This approach efficiently narrows down the search space using binary search, ensuring optimal performance with a time complexity of O(log n).
