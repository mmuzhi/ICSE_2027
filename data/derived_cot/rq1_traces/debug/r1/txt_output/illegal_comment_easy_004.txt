=== RAW ANSWER ===
To solve this problem, we need to determine, for each element in a given list, how many numbers in the list are smaller than the current element. The solution involves efficiently counting these smaller numbers for each element without resorting to a brute-force approach, which would be too slow for large lists.

### Approach
1. **Frequency Counting**: First, we create a frequency array `count` of size 101 (since the numbers in the list range from 0 to 100). This array will store how many times each number appears in the input list.
2. **Prefix Sum Calculation**: Next, we compute a prefix sum array `prefix` where each element at index `i` represents the total count of numbers less than `i` (i.e., the sum of frequencies of all numbers from 0 to `i-1`). This prefix array allows us to quickly look up the count of smaller numbers for any given number.
3. **Result Construction**: Finally, for each number in the original list, we use the prefix array to find the count of smaller numbers and construct the result list accordingly.

This approach ensures that we efficiently compute the required counts with a time complexity of O(n), where n is the number of elements in the input list, making it optimal for large inputs.

### Solution Code
```python
class Solution:
    def smallerNumbersThanCurrent(self, nums: List[int]) -> List[int]:
        count = [0] * 101
        res = []
        for num in nums:
            count[num] += 1 
        prefix = [0] * 101
        cumulative = 0
        for i in range(101):
            prefix[i] = cumulative
            cumulative += count[i]
        for num in nums:
            res.append(prefix[num])
        return res
```

### Explanation
1. **Frequency Array Initialization**: The array `count` is initialized to all zeros. Each index in this array corresponds to a number (0 to 100), and the value at each index represents how many times that number appears in the input list.
2. **Frequency Counting**: We iterate through each number in the input list and increment the corresponding index in the `count` array.
3. **Prefix Sum Array Construction**: The `prefix` array is built by iterating through each possible number (0 to 100). For each number `i`, `prefix[i]` holds the sum of frequencies of all numbers less than `i` (i.e., numbers 0 to `i-1`). This is done using a cumulative sum that accumulates the counts as we iterate.
4. **Result Generation**: For each number in the input list, we use the `prefix` array to directly fetch the count of numbers smaller than the current number and append this count to the result list. This step is efficient because it leverages the precomputed prefix sums, avoiding the need for nested loops or repeated summations.

This method efficiently transforms the problem into a series of linear passes through the data, ensuring optimal performance even for large input sizes.
