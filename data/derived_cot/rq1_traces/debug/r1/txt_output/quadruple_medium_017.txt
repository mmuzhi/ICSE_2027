### Approach
1. **Binary Search**: We use binary search to narrow down the minimum time required. The search space is between the minimum time taken by any bus to complete one trip and the product of the minimum time and the total number of trips (as a worst-case upper bound).
2. **Helper Function**: A helper function `satisfiesTrip` checks if a given time `mid` is sufficient to complete the required number of trips. This function calculates the total number of trips that can be completed in `mid` time by summing up the trips each bus can make and comparing it to the required total trips.
3. **Adjusting Search Space**: Based on the result of the helper function, the binary search adjusts its bounds. If `mid` time is insufficient, the lower bound is increased; if it is sufficient, the upper bound is decreased.

### Solution Code
```python
from typing import List

class Solution:
    def satisfiesTrip(self, mid: int, time: List[int], totalTrips: int) -> bool:
        trip = 0
        for t in time:
            trip += mid // t
        return trip < totalTrips

    def minimumTime(self, time: List[int], totalTrips: int) -> int:
        time.sort(reverse=True)
        minimum = min(time)
        left = minimum
        right = minimum * totalTrips
        while left < right:
            mid = (left + right) // 2
            if self.satisfiesTrip(mid, time, totalTrips):
                left = mid + 1
            else:
                right = mid
        return left
```

### Explanation
1. **Binary Search Initialization**: The binary search starts with `left` set to the minimum time taken by any bus and `right` set to the product of this minimum time and the total number of trips. This ensures we cover all possible values from the best-case to the worst-case scenario.
2. **Helper Function**: The `satisfiesTrip` function calculates the total number of trips that can be completed in `mid` time. If this number is less than the required trips, it returns `True`, indicating that we need a larger time value.
3. **Adjusting Bounds**: During each iteration of the binary search, we check if `mid` time is sufficient. If it is not (`satisfiesTrip` returns `True`), we adjust the lower bound (`left`) to `mid + 1`. If it is sufficient, we adjust the upper bound (`right`) to `mid`, narrowing down the search space until the minimum time is found.

This approach efficiently narrows down the possible minimum time using binary search, ensuring an optimal solution with a time complexity of O(n log n), where n is the number of buses.
