from collections import deque
from typing import List

class Solution:
    def getAncestors(self, n: int, edges: List[List[int]]) -> List[List[int]]:
        graph = [[] for _ in range(n)]
        indegrees = [0] * n

        for edge in edges:
            src, dest = edge[0], edge[1]
            graph[src].append(dest)
            indegrees[dest] += 1

        queue = deque()
        ans = [set() for _ in range(n)]

        for i in range(n):
            if indegrees[i] == 0:
                queue.append(i)

        while queue:
            cur = queue.pop()

            for neighbor in graph[cur]:
                ans[neighbor].add(cur)
                ans[neighbor].update(ans[cur])
                indegrees[neighbor] -= 1
                if indegrees[neighbor] == 0:
                    queue.append(neighbor)

        ans = [sorted(list(s)) for s in ans]
        return ans

To solve this problem, we need to find all ancestors of each node in a directed acyclic graph (DAG). An ancestor of a node is defined as any node from which there is a path (direct or indirect) to the node. The solution involves processing the graph in topological order to efficiently accumulate ancestors for each node.

### Approach
1. **Graph Construction**: We first construct the graph using adjacency lists and compute the indegree of each node. The indegree of a node is the number of edges pointing to it.
2. **Topological Sorting**: We use Kahn's algorithm for topological sorting. Nodes with an indegree of 0 are added to a queue. These nodes have no ancestors and serve as starting points.
3. **Ancestor Accumulation**: For each node processed from the queue, we visit its neighbors. For each neighbor, we add the current node as a direct ancestor and merge all ancestors of the current node into the neighbor's ancestor set. This ensures that all indirect ancestors are also included.
4. **Indegree Update**: After processing a neighbor, we decrement its indegree. If the indegree becomes zero, the neighbor is added to the queue for further processing.
5. **Result Preparation**: Finally, we convert the accumulated ancestor sets for each node into sorted lists and return them as the result.

2. **Initial Queue Setup**: Nodes with no incoming edges (indegree 0) are added to the queue. These nodes have no ancestors.
3. **Topological Processing**: The queue processes nodes in topological order. For each node, its neighbors are visited. The current node is added as a direct ancestor to each neighbor, and all ancestors of the current node are propagated to the neighbor.
4. **Indegree Management**: After processing a neighbor, its indegree is decremented. If the indegree reaches zero, the neighbor is added to the queue, ensuring all ancestors are processed before it.
5. **Result Conversion**: The accumulated sets of ancestors for each node are converted into sorted lists to meet the output requirement.
