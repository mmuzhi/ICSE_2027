=== RAW ANSWER ===

To solve this problem, we need to determine the third maximum number in an array. If the array has fewer than three unique elements, we should return the maximum element instead. The solution must handle both scenarios efficiently.

### Approach
1. **Remove Duplicates**: Convert the list to a set to eliminate duplicate elements.
2. **Check Number of Unique Elements**: Determine if there are at least three unique elements.
3. **Return Third Maximum**: If there are three or more unique elements, sort them and return the third element from the end (which is the third maximum).
4. **Return Maximum**: If there are fewer than three unique elements, return the maximum element.

### Solution Code
```python
class Solution:
    def thirdMax(self, nums: List[int]) -> int:
        unique = set(nums)
        return sorted(unique)[-3] if len(unique) >= 3 else max(unique)
```

### Explanation
1. **Remove Duplicates**: By converting the input list to a set, we ensure all elements are unique.
2. **Check Unique Elements Count**: We check if the number of unique elements is at least three.
3. **Sort and Find Third Maximum**: If there are three or more unique elements, we sort them in ascending order and return the third element from the end (index `[-3]`), which is the third maximum.
4. **Handle Fewer Elements**: If there are fewer than three unique elements, we simply return the maximum element using the `max` function.

This approach efficiently handles both scenarios with a time complexity dominated by the sorting step, which is O(n log n) where n is the number of unique elements. The space complexity is O(n) due to the set and sorted list storage.
