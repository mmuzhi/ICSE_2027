=== RAW ANSWER ===
To solve this problem, we need to count the number of arithmetic subsequences in a given array of integers. An arithmetic subsequence is a sequence of at least three elements where the difference between consecutive elements is the same. The solution involves dynamic programming to efficiently track and count these subsequences.

### Approach
1. **Dynamic Programming Setup**: We use a list of dictionaries, `dp`, where each dictionary at index `i` maps a common difference to the number of arithmetic subsequences ending at `i` with that difference.
2. **Iterate Through Pairs**: For each element `nums[i]` (starting from the second element), we compare it with all previous elements `nums[j]` (where `j < i`). For each pair, we compute the difference `diff = nums[j] - nums[i]`.
3. **Update DP State**: For each pair `(j, i)`, we update `dp[i][diff]` by adding `dp[j][diff] + 1`. The `+1` accounts for the new two-element sequence `(j, i)`, while `dp[j][diff]` represents the count of valid subsequences ending at `j` that can be extended to `i`.
4. **Count Valid Subsequences**: Whenever we extend a subsequence ending at `j` to include `i`, we add `dp[j][diff]` to the total count. This is because extending a valid subsequence (of at least two elements) to include `i` forms a new valid subsequence of at least three elements.
5. **Return Total**: After processing all pairs, the total count of all valid arithmetic subsequences is returned.

### Solution Code
```python
from collections import defaultdict
from typing import List

class Solution:
    def numberOfArithmeticSlices(self, nums: List[int]) -> int:
        total, n = 0, len(nums)
        dp = [defaultdict(int) for _ in nums]
        for i in range(1, n):
            for j in range(i):
                diff = nums[j] - nums[i]
                dp[i][diff] += dp[j][diff] + 1
                total += dp[j][diff]
        return total
```

### Explanation
- **Initialization**: We initialize `total` to zero and create a list `dp` of `defaultdict` objects, one for each element in `nums`. Each dictionary will store the count of subsequences ending at that index with a specific common difference.
- **Processing Pairs**: For each element `nums[i]` (from index 1 to `n-1`), we iterate over all previous elements `nums[j]` (from 0 to `i-1`). For each pair, we compute the difference `diff` between `nums[j]` and `nums[i]`.
- **Updating DP State**: The value `dp[i][diff]` is incremented by `dp[j][diff] + 1`. Here, `dp[j][diff]` is the count of existing subsequences ending at `j` with difference `diff`, and adding 1 accounts for the new two-element sequence `(j, i)`.
- **Counting Valid Subsequences**: The term `dp[j][diff]` is added to `total` because each existing subsequence ending at `j` with difference `diff` can be extended to include `i`, forming a valid arithmetic subsequence of length at least three.
- **Result**: After processing all elements, `total` contains the count of all valid arithmetic subsequences in the array.

This approach efficiently tracks and counts all valid arithmetic subsequences using dynamic programming, leveraging dictionaries to manage different common differences encountered during processing. The complexity is O(n^2), which is feasible for moderate input sizes.
