To solve this problem, we need to find the minimum positive integer speed (in km/h) such that we can travel all the distances in the given list in less than or equal to the specified hour. The key insight is that for the first n-1 segments of the journey, the time taken is the ceiling of the distance divided by the speed, while for the last segment, it's the exact division (without rounding). 

### Approach
1. **Impossibility Check**: If the hour is less than or equal to the number of segments minus one (i.e., `hour <= len(dist) - 1`), it's impossible to complete the journey on time because each of the first n-1 segments requires at least 1 hour, and the last segment requires some positive time. Hence, we return -1 in such cases.
2. **Binary Search Setup**: We initialize the binary search range with `left` as 1 (minimum possible speed) and `right` as 10^7 (a sufficiently large upper bound based on problem constraints).
3. **Binary Search Execution**: For each midpoint speed `mid` in the binary search:
   - **Calculate Total Time**: For the first n-1 segments, compute the time as the ceiling of the distance divided by `mid`. For the last segment, compute the time as the exact division.
   - **Check Feasibility**: If the total time is within the given hour, we adjust the upper bound (`right`) to `mid` to search for a potentially smaller speed. Otherwise, we adjust the lower bound (`left`) to `mid + 1`.
4. **Result**: The loop exits when `left` equals `right`, which is the minimum speed satisfying the time constraint.

### Solution Code
```python
from math import ceil
from typing import List

class Solution:
    def minSpeedOnTime(self, dist: List[int], hour: float) -> int:

        if hour <= n - 1:
```

### Explanation
1. **Impossibility Check**: The condition `hour <= len(dist) - 1` checks if there's enough time to cover the first n-1 segments (each requiring at least 1 hour) and the last segment (requiring some positive time). If not, the journey is impossible, and we return -1.
2. **Binary Search Initialization**: The search range starts from 1 to 10^7, which covers the maximum possible speed needed (as the minimal time for the last segment can be 0.01 hours, and the maximum distance is 10^5, leading to a speed of 10^7).
3. **Midpoint Calculation**: For each midpoint speed `mid`, the total time is computed by summing the ceiling values for the first n-1 segments and the exact division for the last segment.
4. **Feasibility Check**: If the total time is within the given hour, the upper bound is reduced to search for a smaller speed; otherwise, the lower bound is increased.
5. **Result**: The loop exits when the bounds converge, returning the minimum speed that allows the journey to be completed on time. This approach efficiently narrows down the search space using binary search, ensuring optimal performance.
