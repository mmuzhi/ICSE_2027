To solve this problem, we need to find a peak element in an array using an efficient approach. A peak element is an element that is greater than or equal to its neighbors. The binary search algorithm is well-suited for this task due to its logarithmic time complexity.

### Approach
1. **Binary Search Initialization**: Initialize the low and high pointers to the start and end of the array, respectively.
2. **Check Edge Cases**: If the array has only one element, return 0 as the peak element.
3. **Binary Search Loop**: Continuously narrow down the search space by adjusting the low and high pointers based on the comparison of the middle element with its neighbors.
4. **Peak Condition Check**: If the middle element is greater than or equal to its neighbors, it is a peak element, and we return its index.
5. **Adjust Search Space**: If the middle element is less than its right neighbor, move the search space to the right. Otherwise, move it to the left.

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        if n == 1:
            return 0
        
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] >= nums[mid-1]) and (mid == n-1 or nums[mid] >= nums[mid+1]):
                return mid
            elif nums[mid] <= nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

### Explanation
1. **Binary Search Initialization**: The low pointer starts at 0, and the high pointer starts at the last index of the array (n-1).
2. **Edge Case Handling**: If the array has only one element, it is trivially the peak, so return index 0.
3. **Loop Until Peak Found**: The loop continues until the low pointer exceeds the high pointer.
4. **Middle Element Check**: Calculate the middle index. Check if the middle element is a peak by comparing it with its neighbors. If it is, return the middle index.
5. **Adjust Search Space**: If the middle element is less than its right neighbor, move the low pointer to mid + 1 to search the right half. Otherwise, move the high pointer to mid - 1 to search the left half.

This approach efficiently narrows down the potential peak element using binary search, ensuring a time complexity of O(log n) and constant space complexity.
