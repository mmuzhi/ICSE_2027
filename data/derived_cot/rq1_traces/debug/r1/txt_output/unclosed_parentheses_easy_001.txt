To fix this code, we need to address a syntax error and ensure the correct usage of the `heapq` module. The main issues are a missing closing bracket in the list comprehension for the odd numbers and the absence of the `heapq` import. 

### Approach
1. **Identify Even and Odd Digits**: Split the digits of the input number into two lists: one for even digits and one for odd digits.
2. **Max-Heap Simulation**: Convert these lists into max-heaps by negating the values (since Python's `heapq` module provides a min-heap by default).
3. **Reconstruct the Result**: For each digit in the original number, replace it with the largest remaining digit of the same parity (even or odd) using the respective max-heap.

### Solution Code
```python
import heapq

class Solution:
    def largestInteger(self, num: int) -> int:
        evenlist = []
        oddlist = []
        nums = [int(x) for x in str(num)]
        for i in nums:
            if i % 2 == 0:
                evenlist.append(i)
            else:
                oddlist.append(i)
        even = [-x for x in evenlist]
        odd = [-x for x in oddlist]
        heapq.heapify(even)
        heapq.heapify(odd)
        result = []
        for ele in nums:
            if ele in evenlist:
                result += [-heapq.heappop(even)]
            elif ele in oddlist:
                result += [-heapq.heappop(odd)]
        result = [str(x) for x in result]
        return int(''.join(result))
```

### Explanation
1. **Splitting Digits**: The input number is converted into a list of its digits. Each digit is checked for evenness and added to the appropriate list (`evenlist` or `oddlist`).
2. **Heap Conversion**: The even and odd lists are converted into max-heaps by negating their values and using `heapq.heapify`.
3. **Reconstructing the Result**: For each digit in the original number, the code checks if it was originally even or odd. It then pops the largest remaining value from the corresponding heap (using `heappop` on the negated values) to build the result list.
4. **Final Conversion**: The result list is converted to strings and joined into a single integer to produce the final output.

This approach ensures that the largest possible digits are used while maintaining the original parity positions, leading to the correct solution efficiently.
