=== RAW ANSWER ===

To solve this problem, we need to perform a level order traversal of a binary tree starting from the bottom level and moving up to the root. This means that the result should be a list of lists where each sublist represents a tree level, with the sublist for the deepest level appearing first and the root level appearing last.

### Approach
1. **Level Order Traversal**: Use a queue to traverse the tree level by level. This is done using a breadth-first search (BFS) approach where we process each level's nodes and collect their values.
2. **Check for Non-Null Children**: When processing each node, ensure that we only enqueue non-null left and right children to avoid processing `None` values which can lead to errors.
3. **Reverse the Result**: After collecting all levels from top to bottom, reverse the list of levels to get the bottom-up order.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans) - 1
            while l <= r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root:
            return []
        q = [root]
        ans = []
        while q:
            n = len(q)
            level = []
            for _ in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        return checkReverse(ans)
```

### Explanation
1. **Initial Check**: If the root is `None`, return an empty list immediately as there are no nodes to traverse.
2. **Queue Initialization**: Start the BFS with the root node in the queue.
3. **Processing Levels**: For each level, determine the number of nodes (`n`) at the current level. Process each node by dequeuing it, collecting its value, and enqueuing its non-null children.
4. **Collecting Levels**: After processing all nodes at the current level, store the collected values in the result list.
5. **Reversing the Result**: Once all levels are processed, reverse the list of levels to achieve the bottom-up order using a helper function `checkReverse`.

This approach ensures that we efficiently traverse the tree level by level and correctly reverse the order of levels to meet the problem's requirements.
