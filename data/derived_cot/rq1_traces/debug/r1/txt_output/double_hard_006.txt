To solve this problem, we need to determine the maximum number of times the string `s2` can be a subsequence of the string `s1` repeated `n1` times, divided by `n2`. The solution involves detecting cycles in the repetition pattern to optimize the calculation.

### Approach
1. **Initial Checks**: First, check if all characters in `s2` are present in `s1`. If not, return 0 immediately as it's impossible for `s2` to be a subsequence.
2. **Filter `s1`**: Remove any characters from `s1` that are not in `s2` to simplify processing.
3. **Cycle Detection**: Track the number of times `s1` is cycled through while forming `s2` repeatedly. Use a dictionary to detect cycles in the positions where `s2` is found within `s1`.
4. **Cycle Calculation**: Once a cycle is detected, calculate the number of `s1` cycles and `s2` repetitions within the cycle. Use this information to compute the maximum repetitions efficiently.

### Solution Code
```python
class Solution:
    def getMaxRepetitions(self, s1: str, n1: int, s2: str, n2: int) -> int:
        rec, track = [0], {}
        ct = start = 0

        if not set(s2).issubset(set(s1)):
            return 0

        s1 = ''.join(char for char in s1 if char in set(s2))
        
        while True:
            for char in s2:
                ptr = s1.find(char, start)
                if ptr == -1:
                    ct += 1
                    ptr = s1.find(char)
                start = ptr + 1
            rec.append(ct)

            if rec[-1] > n1:
                return (len(rec) - 2) // n2

            if ptr in track:
                break
            track[ptr] = len(rec) - 1
        
        cycle_start = rec[track[ptr]]
        cycle1 = ct - cycle_start
        cycle2 = len(rec) - 1 - track[ptr]
        rest = n1 - cycle_start
        
        rem = cycle_start + rest % cycle1

        ptr2 = 0
        while ptr2 < len(rec) and rec[ptr2] <= rem:
            ptr2 += 1

        return (cycle2 * (rest // cycle1) + (ptr2 - 1)) // n2
```
