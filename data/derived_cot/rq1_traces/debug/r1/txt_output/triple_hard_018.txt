=== RAW ANSWER ===

To solve this problem, we need to compute the median of a sliding window of size `k` as it moves from the left to the right of the input list `nums`. The median is the middle value in an ordered list, and if the window size is even, it is the average of the two middle values.

### Approach
1. **Maintain a Sorted Window**: Use a list to maintain the elements of the current window in sorted order. This allows efficient insertion and deletion operations using binary search.
2. **Insert Elements**: For each new element in the input list, insert it into the sorted position in the window using binary search.
3. **Remove Out-of-Window Elements**: If the window size exceeds `k`, remove the element that is no longer part of the current window. This element is the one that was added `k` steps before the current element.
4. **Compute Median**: Once the window reaches size `k`, compute the median. If the window size is odd, the median is the middle element. If even, it is the average of the two middle elements.

### Solution Code
```python
import bisect
from typing import List

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        window = []
        ans = []
        for i in range(len(nums)):
            bisect.insort(window, nums[i])
            if len(window) > k:
                # Remove the element that's out of the window
                index = bisect.bisect_left(window, nums[i - k])
                window.pop(index)
            if len(window) == k:
                if k % 2 == 1:
                    median = window[k // 2]
                else:
                    median = (window[k // 2 - 1] + window[k // 2]) / 2
                ans.append(median)
        return ans
```

### Explanation
- **Insertion**: Each element is inserted into the sorted list `window` using `bisect.insort`, which maintains the sorted order in O(log n) time.
- **Removal**: When the window size exceeds `k`, the oldest element (the one that is now out of the window) is found using binary search (`bisect.bisect_left`) and removed in O(log n) time.
- **Median Calculation**: The median is calculated based on the sorted window. For an odd-sized window, the middle element is chosen. For an even-sized window, the average of the two middle elements is taken.

This approach ensures that each insertion, deletion, and median calculation is efficient, leading to an overall time complexity of O(n log k), where `n` is the number of elements in the input list.
