=== RAW ANSWER ===
To solve this problem, we need to compute the sum of the minimum values of all possible subarrays of a given array. The challenge lies in efficiently calculating this sum without resorting to a brute-force approach, which would be computationally expensive for large arrays. 

### Approach
The solution leverages a monotonic stack to efficiently track elements and compute the contribution of each element as the minimum in various subarrays. The key insight is that for each element, we determine the range of subarrays where this element is the minimum. This range is defined by the nearest smaller elements to the left and right of the current element. 

1. **Initialization**: We initialize an empty stack to keep track of indices and a variable `res` to accumulate the result.
2. **Processing Elements**: For each element in the array:
   - **Maintain Monotonic Stack**: While the stack is not empty and the current element is smaller than the element at the top of the stack, we pop elements from the stack. For each popped element, we calculate its contribution to the result. The contribution is derived from the number of subarrays where the popped element is the minimum, which is the product of the element's value, the number of elements to the left until the next smaller element, and the number of elements to the right until the next smaller element.
   - **Push Current Index**: After processing all larger elements, the current element's index is pushed onto the stack.
3. **Process Remaining Stack**: After processing all elements, any remaining elements in the stack are processed similarly, considering the end of the array as the right boundary.
4. **Result Calculation**: The accumulated result is returned modulo \(10^9 + 7\) to handle large numbers.

### Solution Code
```python
class Solution:
    def sumSubarrayMins(self, arr: List[int]) -> int:
        stack = []
        res = 0
        n = len(arr)
        
        for i, num in enumerate(arr):
            while stack and arr[stack[-1]] > num:
                cur = stack.pop()
                left_bound = stack[-1] if stack else -1
                res += arr[cur] * (cur - left_bound) * (i - cur)
            stack.append(i)
        
        while stack:
            cur = stack.pop()
            left_bound = stack[-1] if stack else -1
            res += arr[cur] * (cur - left_bound) * (n - cur)
        
        return res % (10**9 + 7)
```

### Explanation
- **Initialization**: The stack is initialized to keep track of indices, and `res` is set to zero to accumulate the result.
- **Processing Elements**: For each element in the array:
  - **Stack Maintenance**: The while loop ensures that the stack remains in increasing order. Elements larger than the current element are popped from the stack.
  - **Contribution Calculation**: For each popped element, the left boundary is the next smaller element to the left (or the start of the array if none exists), and the right boundary is the current index. The product of the element's value and the distances to these boundaries gives the number of subarrays where this element is the minimum.
- **Remaining Stack Processing**: After processing all elements, any remaining elements in the stack are processed with the right boundary set to the end of the array.
- **Result Modulo**: The result is returned modulo \(10^9 + 7\) to ensure it fits within standard integer constraints.

This approach efficiently computes the sum by leveraging the properties of a monotonic stack to avoid redundant calculations, resulting in an optimal solution.
