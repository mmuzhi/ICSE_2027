=== RAW ANSWER ===

To solve this problem, we need to compute the median of a sliding window of size `k` moving through a list of integers. The median should be calculated for each window position and returned as a list of floats.

### Approach
1. **Maintain a Sorted Window**: Use a sorted list to maintain the elements of the current window. This allows efficient insertion and deletion operations while keeping the elements in order.
2. **Insert Elements**: For each new element in the input list, insert it into the sorted window using binary search to maintain order.
3. **Remove Out-of-Window Elements**: When the window size exceeds `k`, remove the element that is no longer part of the current window. This element is identified as the one that was added `k` steps before the current element.
4. **Calculate Median**: Once the window reaches size `k`, compute the median. For an odd-sized window, the median is the middle element. For an even-sized window, it is the average of the two middle elements.

### Solution Code
```python
import bisect
from typing import List

class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        window = []
        ans = []
        for i, x in enumerate(nums):
            bisect.insort(window, x)
            # Remove the element that's out of the window
            if len(window) > k:
                element_to_remove = nums[i - k]
                index = bisect.bisect_left(window, element_to_remove)
                if index < len(window) and window[index] == element_to_remove:
                    del window[index]
            # Check if the window is exactly size k
            if len(window) == k:
                if k % 2 == 1:
                    median = window[k // 2]
                else:
                    median = (window[k // 2 - 1] + window[k // 2]) / 2
                ans.append(median)
        return ans
```

### Explanation
- **Sorted Window**: The window is maintained as a sorted list using `bisect.insort` for efficient insertion.
- **Element Removal**: When the window size exceeds `k`, the oldest element (the one that is now out of the window) is removed using binary search to find its position efficiently.
- **Median Calculation**: The median is calculated based on the sorted window. For odd `k`, the middle element is taken. For even `k`, the average of the two middle elements is computed.
- **Efficiency**: The use of binary search (O(log k)) for insertion and deletion ensures that each element is processed efficiently, leading to an overall time complexity of O(n log k), where `n` is the number of elements in the input list. This approach is optimal for the problem constraints.
