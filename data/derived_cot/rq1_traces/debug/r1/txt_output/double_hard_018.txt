=== RAW ANSWER ===

To solve this problem, we need to find the longest cycle in a directed graph represented by an array where each index points to another node (or -1 if there is no outgoing edge). The solution involves traversing each node and using depth-first search (DFS) to detect cycles and their lengths.

### Approach
1. **Visited Tracking**: Use an array to keep track of visited nodes to ensure each node is processed only once.
2. **Cycle Detection**: For each unvisited node, traverse its path while recording the length of the path. If a node is revisited during the traversal, a cycle is detected.
3. **Cycle Length Calculation**: When a cycle is detected, calculate its length by comparing the current path length with the previously recorded position of the node in the current traversal.

### Solution Code
```python
class Solution:
    def longestCycle(self, edges: List[int]) -> int:
        n = len(edges)
        visited = [0] * n
        max_cycle = -1
        for i in range(n):
            if not visited[i]:
                distance = {}
                current = i
                length = 0
                while current != -1 and not visited[current]:
                    visited[current] = 1
                    distance[current] = length
                    length += 1
                    current = edges[current]
                if current != -1 and current in distance:
                    cycle_length = length - distance[current]
                    max_cycle = max(max_cycle, cycle_length)
        return max_cycle
```

### Explanation
1. **Initialization**: We initialize a `visited` array to keep track of nodes that have been processed.
2. **Traversal Loop**: For each node, if it hasn't been visited, we start a traversal from that node.
3. **Path Recording**: During traversal, we record the distance (path length) from the starting node to each subsequent node using a dictionary.
4. **Cycle Detection**: If we encounter a node that has already been visited in the current traversal (recorded in the dictionary), we calculate the cycle length as the difference between the current path length and the recorded distance of the node.
5. **Cycle Length Update**: We update the maximum cycle length found during the traversal.

This approach ensures that each node is visited exactly once, leading to an efficient solution with a time complexity of O(n), where n is the number of nodes. The space complexity is also O(n) due to the storage used for the visited array and the distance dictionary.
