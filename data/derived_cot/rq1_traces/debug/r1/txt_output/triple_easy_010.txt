=== RAW ANSWER ===

To solve this problem, we need to find the complement of a given integer. The complement of a number is obtained by flipping each bit in its binary representation. For example, the complement of 5 (which is 101 in binary) is 010, which is 2 in decimal.

### Approach
1. **Binary Conversion and Flipping**: Convert the given number to its binary form bit by bit, flipping each bit as we process it. This is done by checking the least significant bit (LSB) of the number in each iteration. If the LSB is 1, we append '0' to our result string, and if it is 0, we append '1'.
2. **Reversing the Result**: Since we process the bits from the least significant to the most significant, the resulting string will be in reverse order. Therefore, we reverse the string before converting it back to an integer.
3. **Handling Edge Cases**: If the input number is 0, its complement should be 1. This case is handled separately as the loop will not execute for 0, leading to an empty result string.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        com = ''
        while num > 0:
            if num % 2 == 1:
                com += '0'
            else:
                com += '1'
            num = num // 2
        if not com:
            return 1
        return int(com[::-1], 2)
```

### Explanation
1. **Binary Conversion and Flipping**: The loop processes each bit of the number starting from the LSB. Each bit is flipped (0 becomes 1 and 1 becomes 0) and appended to the result string `com`.
2. **Reversing the Result**: The result string `com` is built in reverse order (LSB to MSB), so we reverse it to get the correct order (MSB to LSB) before converting it to an integer.
3. **Edge Case Handling**: If the input number is 0, the loop does not execute, and `com` remains empty. We check for this condition and return 1, as the complement of 0 is 1.

This approach efficiently handles all cases, including the edge case of 0, ensuring the correct complement is returned for any non-negative integer input.
