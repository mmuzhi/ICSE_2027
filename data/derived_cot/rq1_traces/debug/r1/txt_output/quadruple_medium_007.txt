=== RAW ANSWER ===

To solve this problem, we need to perform a level order traversal of a binary tree from the bottom up. This means that we start from the leaf nodes and move up to the root, collecting values level by level. The solution involves using a breadth-first search (BFS) approach to traverse the tree level by level and then reversing the result to achieve the bottom-up order.

### Approach
1. **Breadth-First Search (BFS)**: We use a queue to traverse the tree level by level. Starting from the root, we process each level by dequeuing nodes and enqueuing their children.
2. **Check for Valid Children**: When processing each node, we only enqueue its left and right children if they exist to avoid adding `None` values to the queue.
3. **Reverse the Result**: After collecting all levels in a top-down manner, we reverse the list of levels to get the bottom-up order.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans) - 1
            while l <= r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root:
            return []
        q = [root]
        ans = []
        while q:
            n = len(q)
            level = []
            for _ in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        return checkReverse(ans)
```

### Explanation
1. **checkReverse Function**: This helper function reverses the list of levels. It uses two pointers starting from the beginning and end of the list, swapping elements until the entire list is reversed.
2. **Handling Empty Tree**: If the tree is empty (root is `None`), we return an empty list immediately.
3. **BFS Traversal**: Using a queue, we process each level of the tree. For each node in the current level, we collect its value and enqueue its children if they exist.
4. **Collecting Levels**: Each level's values are collected in a sublist, which is then added to the result list.
5. **Reversing the Result**: After collecting all levels in top-down order, we reverse the list to get the bottom-up order required by the problem.

This approach ensures that we efficiently traverse the tree and correctly reverse the levels to meet the problem's requirements.
