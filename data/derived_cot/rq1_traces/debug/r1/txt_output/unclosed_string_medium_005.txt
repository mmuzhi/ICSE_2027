2. **Sentinel Values**: We add sentinel values (negative infinity) at both ends of the array to handle boundary conditions seamlessly. These sentinels ensure that every element in the original array has a previous and next smaller element within the bounds of the modified array.
3. **Contribution Calculation**: For each element, we calculate its contribution as the minimum in subarrays where it is the smallest element. This is done by determining the distance to the previous and next smaller elements using the stack.

### Explanation
1. **Sentinel Values**: The array is modified by adding negative infinity at both ends. This helps in handling edge cases where elements are at the start or end of the original array.
2. **Monotonic Stack**: The stack is used to keep track of indices of elements in a monotonically increasing order. When a smaller element is encountered, we pop elements from the stack and calculate their contributions.
3. **Contribution Calculation**: For each popped element, the contribution is calculated based on the distance to the previous smaller element (left boundary) and the next smaller element (right boundary). This distance determines how many subarrays the current element is the minimum of.
4. **Skipping Sentinels**: Contributions from the sentinel values (added at the start and end of the modified array) are skipped to ensure they do not affect the result.
