### Approach
1. **Monotonic Stack**: We use a stack to keep track of indices of elements in a way that maintains increasing order of their values. This helps in efficiently finding the previous and next smaller elements for each element in the array.
2. **Sentinel Values**: To handle boundary conditions where an element might be the smallest in the entire array, we add sentinel values (negative infinities) at both ends of the array. This simplifies the code by avoiding special cases for the first and last elements.
3. **Contribution Calculation**: For each element popped from the stack, we calculate its contribution to the result based on the distance to its previous and next smaller elements. The contribution is the product of the element's value, the number of subarrays it is the minimum in, and the respective distances.

### Solution Code
```python
class Solution:
    def sumSubarrayMins(self, arr: List[int]) -> int:
        stack = []
        res = 0
        arr = [float('-inf')] + arr + [float('-inf')]
```

### Explanation
1. **Monotonic Stack**: The stack is used to maintain indices of elements in increasing order of their values. When encountering a smaller element, we pop elements from the stack and calculate their contributions.
2. **Sentinel Values**: Adding sentinel values at both ends ensures that every element in the original array has a previous and next smaller element, simplifying the boundary checks.
3. **Contribution Calculation**: For each element popped from the stack (due to encountering a smaller element), the contribution is calculated as the product of the element's value, the distance to the next smaller element (i - cur), and the distance to the previous smaller element (cur - stack[-1]). This gives the number of subarrays where the popped element is the minimum.
