=== RAW ANSWER ===
To solve this problem, we need to rearrange the elements of an array such that all elements less than a given pivot come before all elements equal to the pivot, and all elements greater than the pivot come after the elements equal to the pivot. 

### Approach
1. **Problem Analysis**: The task involves partitioning the array into three segments based on their relationship to the pivot value. The segments should be in the order: elements less than the pivot, elements equal to the pivot, and elements greater than the pivot.
2. **Intuition**: By iterating through each element in the array, we can categorize each element into one of the three segments. We then concatenate these segments in the required order.
3. **Algorithm Selection**: We use three separate lists to hold elements less than, equal to, and greater than the pivot. After processing all elements, we combine these lists in the desired order.
4. **Complexity Analysis**: The algorithm runs in O(n) time, where n is the number of elements in the array, as it processes each element exactly once. The space complexity is O(n) to store the three segments.

### Solution Code
```python
class Solution:
    def pivotArray(self, nums: List[int], pivot: int) -> List[int]:
        less, equal, greater = [], [], []
        for num in nums:
            if num < pivot:
                less.append(num)
            elif num == pivot:
                equal.append(num)
            else:
                greater.append(num)
        return less + equal + greater
```

### Explanation
1. **Initialization**: Three empty lists (`less`, `equal`, and `greater`) are initialized to hold elements based on their comparison with the pivot.
2. **Categorization**: For each element in the input array:
   - If the element is less than the pivot, it is added to the `less` list.
   - If the element is equal to the pivot, it is added to the `equal` list.
   - If the element is greater than the pivot, it is added to the `greater` list.
3. **Combination**: The three lists are concatenated in the order `less + equal + greater` to form the final result, ensuring all elements are rearranged as required.

This approach efficiently partitions the array into the desired segments and combines them in linear time, meeting the problem constraints effectively.
