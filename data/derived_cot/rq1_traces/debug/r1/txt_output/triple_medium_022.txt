### Approach

2. **Recursive Function with Memoization**: Use a recursive function with memoization to explore all possible combinations of toppings, considering each topping up to two times.
3. **Base and Target Adjustment**: For each base cost, adjust the target by subtracting the base cost and find the closest sum of toppings to the adjusted target.

### Solution Code
```python
from typing import List
from functools import cache

class Solution:
    def closestCost(self, baseCosts: List[int], toppingCosts: List[int], target: int) -> int:
        toppingCosts = toppingCosts * 2

        @cache
        def fn(i, x):
            """Return sum of subsequence of toppingCosts[i:] closest to x."""
            if x < 0 or i == len(toppingCosts):
                return 0
            return min(
                fn(i + 1, x),
                toppingCosts[i] + fn(i + 1, x - toppingCosts[i]),
                key=lambda y: (abs(y - x), y)
            )
        
        ans = float('inf')
        for bc in baseCosts:
            current = bc + fn(0, target - bc)
            ans = min(ans, current, key=lambda z: (abs(z - target), z))
        return ans
```

### Explanation
1. **Duplication of Toppings**: The `toppingCosts` list is duplicated to allow each topping to be considered up to two times.
2. **Recursive Function with Memoization**: The `fn` function uses memoization to efficiently explore all combinations of toppings. It recursively decides whether to include the current topping or not, aiming to get the sum closest to the target.
3. **Base Cost Adjustment**: For each base cost, the target is adjusted by subtracting the base cost. The function then finds the best combination of toppings to get as close as possible to this adjusted target.
4. **Optimal Selection**: The minimum cost is determined by comparing the absolute difference from the target and, in case of ties, the smaller value. This ensures the closest possible cost is chosen efficiently.
