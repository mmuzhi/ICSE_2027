Since we do a post-order DFS, we are exploring all possible cuts.
If we can partition it, then there must be a way to cut the edges such that we form `k` components of sum `cand`, and the remaining part (which must be connected to `u`'s parent) has sum `r`.
If this is possible, then our DFS will find it, because it cuts an edge whenever the subtree sum equals `cand`.

If a subtree rooted at `v` (child of `u`) has sum `cand`, we cut the edge `(u, v)`, so `v`'s sum is not passed to `u`.
If `v`'s sum is `< cand`, we don't cut it, and we pass the sum to `u`.
If `v`'s sum is `> cand`, we CANNOT cut it (because we can only cut at `cand`). So we pass the sum `> cand` to `u`.
