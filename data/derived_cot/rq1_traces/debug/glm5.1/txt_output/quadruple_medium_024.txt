Wait, is there any case where `len(dq) == 1` and `dq[0] != (0, 0)` and `dq[0] != (m - 1, n - 1)`, but we CANNOT cut the path?
Suppose we have a grid where all paths go through cell X, so `len(dq) == 1` at some level.

This is exactly the definition of an articulation point (or cut vertex) in a DAG!
Since the graph is a DAG (we only move down and right), any articulation point will appear as a level with only 1 node in the BFS.
Wait, is this true?
In a DAG, if all paths from `s` to `t` go through a node `v`, then `v` is a dominator of `t`.
The dominators of `t` form a tree (the dominator tree).
The immediate dominator of `t` is the closest dominator.
If we do BFS from `s`, the nodes at distance `d` from `s` form a level.
If a level has only 1 node `v`, then `v` is a dominator of all nodes at level `d+1` and beyond.
Wait, is it possible that `v` is a dominator of `t`, but the level has more than 1 node?
Yes, if there are other nodes at the same distance from `s` that are not dominators.
But if a level has only 1 node `v`, then `v` MUST be a dominator of all nodes in subsequent levels.
Because any path to a node at level `d+1` must pass through a node at level `d`. Since `v` is the ONLY node at level `d`, all paths must pass through `v`.
So `v` is a dominator of all subsequent nodes, including `t`.
Thus, flipping `v` will disconnect `s` from `t`.
So `len(dq) == 1` is a SUFFICIENT condition for `v` to be a dominator.
Is it a NECESSARY condition?
No. A dominator might share its level with other nodes.
