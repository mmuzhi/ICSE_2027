=== ANSWER ===
To solve this problem, we need to determine the "power" of every contiguous subarray of size `k` in the given array `nums`. The power of a subarray is defined as its maximum element if all elements in the subarray are consecutive integers and sorted in ascending order; otherwise, the power is -1.

### Approach
1. **Problem Analysis**: The task involves checking each subarray of size `k` to see if it meets two conditions:
   - The elements must be consecutive integers (i.e., each subsequent element is exactly one more than the previous element).
   - The elements must be sorted in ascending order (i.e., the subarray is strictly increasing with each step being 1).
2. **Key Insight**: For a subarray to satisfy both conditions, every adjacent pair of elements in the subarray must satisfy `nums[j] + 1 == nums[j+1]`. This ensures the sequence is strictly increasing and consecutive.
3. **Algorithm Selection**: Iterate over each possible subarray of size `k`. For each subarray, check all adjacent pairs. If all pairs meet the condition, the maximum element (which is the last element of the subarray) is recorded; otherwise, -1 is recorded.
4. **Complexity Analysis**: The solution involves checking each subarray of size `k`, leading to a time complexity of O(n*k), where `n` is the length of `nums`. Given the constraints (n ≤ 500), this approach is efficient.

### Solution Code
```python
class Solution:
    def resultsArray(self, nums: List[int], k: int) -> List[int]:
        n = len(nums)
        res = []
        for i in range(n - k + 1):
            valid = True
            for j in range(i, i + k - 1):
                if nums[j] + 1 != nums[j + 1]:
                    valid = False
                    break
            if valid:
                res.append(nums[i + k - 1])
            else:
                res.append(-1)
        return res
```

### Explanation
1. **Initialization**: The solution initializes an empty list `res` to store the results for each subarray.
2. **Iterate Over Subarrays**: The outer loop runs from the start of the array to `n - k`, ensuring each subarray of size `k` is considered.
3. **Check Validity**: For each subarray, the inner loop checks adjacent elements. If any adjacent pair fails the condition `nums[j] + 1 == nums[j+1]`, the subarray is marked invalid.
4. **Record Result**: If the subarray is valid, the last element (which is the maximum due to the sorted and consecutive nature) is added to `res`; otherwise, -1 is added.
5. **Return Result**: After processing all subarrays, `res` is returned, containing the power values for each subarray.

This approach efficiently checks each subarray's validity and records the result based on the given conditions, ensuring optimal performance within the problem constraints.
