=== ANSWER ===
To solve this problem, we need to maximize the points a tourist can earn over `k` days while traveling between `n` cities. The tourist can either stay in the current city or move to another city each day, earning points based on the respective actions. The solution involves dynamic programming to efficiently compute the maximum points by considering both choices for each day and city.

### Approach
1. **Initialization**: For the first day (day 0), the tourist can start in any city. The points for each city on day 0 are computed as the maximum between staying in that city (earning `stayScore[0][j]`) or moving from any other city (earning `travelScore[prev][j]`).
2. **Dynamic Programming Setup**: For each subsequent day, we use a dynamic programming array `dp` where `dp[j]` represents the maximum points accumulated by ending in city `j` after processing the previous day.
3. **Day Processing**: For each day from 1 to `k-1` and each city `j`, we compute:
   - **Option 1**: Staying in city `j` from the previous day, adding the stay points for the current day.
   - **Option 2**: Moving from any city `prev` to city `j`, adding the travel points from `prev` to `j`.
4. **Update DP Array**: For each city `j`, the new DP value is the maximum of the two options above.
5. **Result Extraction**: After processing all days, the result is the maximum value in the DP array, representing the highest points achievable after `k` days.

### Solution Code
```python
class Solution:
    def maxScore(self, n: int, k: int, stayScore: List[List[int]], travelScore: List[List[int]]) -> int:
        dp = [0] * n
        
        for j in range(n):
            stay_option = stayScore[0][j]
            move_option = max(travelScore[prev][j] for prev in range(n))
            dp[j] = max(stay_option, move_option)
        
        for i in range(1, k):
            new_dp = [0] * n
            for j in range(n):
                stay_here = dp[j] + stayScore[i][j]
                best_move = -10**9
                for prev in range(n):
                    candidate = dp[prev] + travelScore[prev][j]
                    if candidate > best_move:
                        best_move = candidate
                new_dp[j] = max(stay_here, best_move)
            dp = new_dp
        
        return max(dp)
```

### Explanation
1. **Initialization for Day 0**: The initial DP array is populated by considering both choices for each city. The tourist can either stay in the starting city (earning `stayScore[0][j]`) or move from any other city (earning the maximum `travelScore[prev][j]` over all possible starting cities `prev`).
2. **Processing Subsequent Days**: For each subsequent day, the algorithm updates the DP array by evaluating both choices for every city:
   - **Staying**: Adds the stay points for the current day to the points accumulated by staying in the same city from the previous day.
   - **Moving**: Computes the maximum points achievable by moving from any city to the current city, adding the corresponding travel points.
3. **Result**: After processing all days, the maximum value in the DP array gives the highest points the tourist can earn over `k` days, considering all possible paths and choices.

This approach efficiently computes the solution using dynamic programming, with a time complexity of O(k * n^2), which is feasible given the constraints (n and k up to 200).
