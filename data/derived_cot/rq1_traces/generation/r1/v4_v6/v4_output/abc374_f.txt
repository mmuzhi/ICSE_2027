You will be given a competitive programming problem.
KEYENCE is famous for quick delivery.

In this problem, the calendar proceeds as Day 1, Day 2, Day 3, \dots.
There are orders 1,2,\dots,N, and it is known that order i will be placed on Day T_i.
For these orders, shipping is carried out according to the following rules.

- At most K orders can be shipped together.
- Order i can only be shipped on Day T_i or later.
- Once a shipment is made, the next shipment cannot be made until X days later.
- That is, if a shipment is made on Day a, the next shipment can be made on Day a+X.



For each day that passes from order placement to shipping, dissatisfaction accumulates by 1 per day.
That is, if order i is shipped on Day S_i, the dissatisfaction accumulated for that order is (S_i - T_i).
Find the minimum possible total dissatisfaction accumulated over all orders when you optimally schedule the shipping dates.

Input

The input is given from Standard Input in the following format:
N K X
T_1 T_2 \dots T_N

Output

Print the answer as an integer.

Constraints


- All input values are integers.
- 1 \le K \le N \le 100
- 1 \le X \le 10^9
- 1 \le T_1 \le T_2 \le \dots \le T_N \le 10^{12}

Sample Input 1

5 2 3
1 5 6 10 12

Sample Output 1

2

=== ANSWER ===
To solve this problem, we need to minimize the total dissatisfaction accumulated by optimally scheduling the shipping dates for N orders. The dissatisfaction for each order is the difference between its shipping day and its order day. The scheduling must adhere to the given constraints: at most K orders per shipment, each order must be shipped on or after its order day, and there must be a gap of at least X days between consecutive shipments.

### Approach
1. **Problem Analysis**: The problem involves scheduling shipments for orders placed on specific days, with constraints on the number of orders per shipment and the minimum gap between shipments. The goal is to minimize the total dissatisfaction, which is the sum of delays (shipping day minus order day) for all orders.
2. **Key Insight**: The optimal shipping day for a group of orders is the maximum of the latest order day in the group and the previous shipment day plus X. This ensures minimal delay while respecting the constraints.
3. **Dynamic Programming (DP) Setup**: We use a DP approach where `dp[i]` is a dictionary mapping the last shipment day to the minimal dissatisfaction for the first `i` orders. The initial state is `dp[0] = {-X: 0}`, representing no orders processed and the last shipment day set to `-X` to allow the first shipment on day 0 or later.
4. **DP Transition**: For each state `i`, consider all possible group sizes from 1 to K. For each group, compute the new shipment day as the maximum of the previous shipment day plus X and the latest order day in the group. Update the dissatisfaction for the new state `j = i + group_size` by adding the delay for the current group.
5. **Result Extraction**: The answer is the minimal dissatisfaction value in `dp[N]`, representing all orders processed.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    k_val = int(data[1])
    x_val = int(data[2])
    T = list(map(int, data[3:3+n]))
    
    prefix = [0] * (n + 1)
    for i in range(1, n + 1):
        prefix[i] = prefix[i - 1] + T[i - 1]
    
    dp = [dict() for _ in range(n + 1)]
    dp[0][-x_val] = 0
    
    for i in range(n):
        max_group_size = min(k_val, n - i)
        for g in range(1, max_group_size + 1):
            j = i + g
            last_order_in_group = T[i + g - 1]
            group_sum = prefix[j] - prefix[i]
            for s_prev, diss_val in dp[i].items():
                new_s = max(s_prev + x_val, last_order_in_group)
                added_diss = new_s * g - group_sum
                new_diss_val = diss_val + added_diss
                if new_s not in dp[j] or new_diss_val < dp[j][new_s]:
                    dp[j][new_s] = new_diss_val
                    
    ans = min(dp[n].values()) if dp[n] else 0
    print(ans)

if __name__ == "__main__":
    main()
```

3. **DP Processing**: For each order `i`, possible group sizes from 1 to K are considered. The new shipment day for each group is calculated as the maximum of the previous shipment day plus X and the latest order day in the group. The dissatisfaction for the group is computed and added to the total dissatisfaction from previous orders.
4. **Result Extraction**: After processing all orders, the minimal dissatisfaction value from `dp[N]` is printed, representing the optimal solution.
