To solve this problem, we need to find the maximum sum of a contiguous subarray in an array of integers such that the length of the subarray is divisible by a given integer `k`. The solution involves leveraging prefix sums and grouping these sums by their indices modulo `k` to efficiently compute the required maximum sum.

### Approach
1. **Prefix Sum Calculation**: Compute the prefix sums of the given array. The prefix sum array `P` is constructed such that `P[i]` represents the sum of the elements from the start of the array up to index `i-1` (inclusive). This allows the sum of any subarray to be computed in constant time.
2. **Grouping by Residue Modulo `k`**: Group the prefix sums based on their indices modulo `k`. This grouping helps in identifying potential subarrays whose lengths are multiples of `k` because the difference between two indices in the same residue group will be divisible by `k`.
3. **Finding Maximum Sum**: For each residue group, iterate through the prefix sums while maintaining the minimum prefix sum encountered so far. For each subsequent prefix sum in the group, compute the difference between the current prefix sum and the minimum prefix sum encountered so far. This difference represents the sum of a subarray starting just after the minimum prefix index and ending at the current index. Track the maximum such difference across all residue groups to determine the solution.

### Solution Code
```python
class Solution:
    def maxSubarraySum(self, nums: list[int], k: int) -> int:
        n = len(nums)
        P = [0] * (n + 1)
        for i in range(1, n + 1):
            P[i] = P[i - 1] + nums[i - 1]
        
        groups = [[] for _ in range(k)]
        for i in range(n + 1):
            r = i % k
            groups[r].append(P[i])
        
        ans = -10**18
        for r in range(k):
            if len(groups[r]) < 2:
                continue
            min_val = groups[r][0]
            for j in range(1, len(groups[r])):
                candidate = groups[r][j] - min_val
                if candidate > ans:
                    ans = candidate
                if groups[r][j] < min_val:
                    min_val = groups[r][j]
        return ans
```

### Explanation
1. **Prefix Sum Array**: The prefix sum array `P` is initialized with `P[0] = 0` and each subsequent element `P[i]` is the sum of elements from `nums[0]` to `nums[i-1]`. This setup allows the sum of any subarray from index `i` to `j-1` to be calculated as `P[j] - P[i]`.
2. **Grouping Indices**: The indices of the prefix sum array are grouped by their values modulo `k`. This ensures that any two indices in the same group will have a difference that is a multiple of `k`, making the corresponding subarray length divisible by `k`.
3. **Tracking Maximum Sum**: For each residue group, the algorithm iterates through the prefix sums, maintaining the minimum prefix sum encountered so far. For each subsequent prefix sum in the group, the difference between the current prefix sum and the minimum prefix sum is computed. This difference represents the sum of a subarray starting just after the minimum prefix index and ending at the current index. The maximum such difference across all groups is the solution to the problem.

This approach efficiently narrows down the potential subarrays to those whose lengths are divisible by `k` and computes their sums in linear time relative to the input size, making it optimal for large arrays.
