=== PROMPT ===
You will be given a competitive programming problem.
There are N bags, labeled bag 1, bag 2, \ldots, bag N.
Bag i (1 \leq i \leq N) contains A_i stones.
Takahashi can perform the following operation any number of times, possibly zero:

Choose two bags A and B, and move all stones from bag A into bag B.

Find the number of different possible values for the following after repeating the operation.

- B_1 \oplus B_2 \oplus \cdots \oplus B_N, where B_i is the final number of stones in bag i.
Here, \oplus denotes bitwise XOR.


About bitwise XOR
For non-negative integers a and b, the bitwise XOR a \oplus b is defined as follows:


In the binary representation of a \oplus b, the digit in the 2^k place (k \ge 0) is 1 if and only if exactly one of the digits in the 2^k place of a and b is 1; otherwise, it is 0.


For example, 3 \oplus 5 = 6 (in binary, 011 \oplus 101 = 110).
In general, for k non-negative integers x_1, x_2, \ldots, x_k, their bitwise XOR x_1 \oplus x_2 \oplus \cdots \oplus x_k is defined as (\cdots((x_1 \oplus x_2) \oplus x_3) \oplus \cdots) \oplus x_k, which does not depend on the order of x_1, x_2, \ldots, x_k.

It can be proved that under the constraints of this problem, the number of possible values is finite.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the number of different possible values for B_1 \oplus B_2 \oplus \cdots \oplus B_N after repeating the operation.

Constraints


- 2 \leq N \leq 12
- 1 \leq A_i \leq 10^{17}
- All input values are integers.

Sample Input 1

3
2 5 7

Sample Output 1

3

For example, if Takahashi chooses bags 1 and 3 for the operation, then the numbers of stones in bags 1, 2, 3 become 0, 5, 9.
If he stops at this point, the XOR is 0 \oplus 5 \oplus 9 = 12.
The other possible XOR values after repeating the operation are 0 and 14.
Therefore, the possible values are 0, 12, 14; there are three values, so the output is 3.

100000000000000000 100000000000000000

Sample Output 2

Sample Input 3

To solve this problem, we need to determine the number of distinct possible values of the bitwise XOR of the final number of stones in each bag after performing any number of operations where we move all stones from one bag to another. The key insight is that the final configuration of stones in the bags corresponds to partitioning the bags into groups, where each group's stones are combined into a single bag, and the other bags in the group are empty. The bitwise XOR of the final stone counts is then the XOR of the sums of each group.

### Approach
1. **Problem Analysis**: The problem involves moving stones between bags such that each operation transfers all stones from one bag to another. The final state will have some bags containing the combined stones of their group and others empty. The XOR of the final stone counts depends on how we partition the bags into groups and which bag in each group retains the combined stones.
2. **Intuition**: The final XOR value is determined by the sums of the stones in each group, regardless of which bag retains the stones (since the other bags in the group are empty and contribute 0 to the XOR). Therefore, the problem reduces to finding all possible ways to partition the bags into groups and computing the XOR of the sums of these groups.
3. **Algorithm Selection**: We use a depth-first search (DFS) approach to explore all possible partitions of the bags into groups. For each bag, we consider two choices: either start a new group with that bag or add it to an existing group. The DFS recursively builds all possible partitions, and at the end of each partition (when all bags are processed), it computes the XOR of the group sums and records it in a set to ensure distinctness.
4. **Complexity Analysis**: The number of partitions of N bags is given by the Bell number, which for N=12 is approximately 4.2 million. The DFS explores each partition exactly once, and for each partition, the operations (addition, subtraction, and XOR computation) are constant time relative to the number of groups. Thus, the total operations are manageable within the constraints (N ≤ 12).

1. **Reading Input**: The input consists of the number of bags (N) and a list of integers representing the stones in each bag.
2. **DFS Initialization**: We initialize a set `res_set` to store distinct XOR values and a list `groups` to keep track of the current sums of groups during the DFS.
3. **DFS Function**: The `dfs` function processes each bag starting from index 0. For each bag, it explores two options:

4. **Base Case**: When all bags are processed (i == N), the XOR of all group sums is computed and stored in `res_set`.
5. **Backtracking**: After exploring each option, the function backtracks by removing the bag from the new group or reverting the addition to an existing group.
6. **Output**: After processing all partitions, the size of `res_set` (number of distinct XOR values) is printed.
