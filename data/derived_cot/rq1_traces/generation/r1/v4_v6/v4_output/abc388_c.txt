=== PROMPT ===
You will be given a competitive programming problem.
There are N mochi (rice cakes) arranged in ascending order of size.
The size of the i-th mochi (1 \leq i \leq N) is A_i.
Given two mochi A and B, with sizes a and b respectively, you can make one kagamimochi (a stacked rice cake) by placing mochi A on top of mochi B if and only if a is at most half of b.
You choose two mochi out of the N mochi, and place one on top of the other to form one kagamimochi.
Find how many different kinds of kagamimochi can be made.
Two kagamimochi are distinguished if at least one of the mochi is different, even if the sizes of the mochi are the same.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \cdots A_N

Output

Print the number of different kinds of kagamimochi that can be made.

Constraints


- 2 \leq N \leq 5 \times 10^5
- 1 \leq A_i \leq 10^9 \ (1 \leq i \leq N)
- A_i \leq A_{i+1} \ (1 \leq i < N)
- All input values are integers.

Sample Input 1

6
2 3 4 4 7 10

Sample Output 1

8

Note that there are two kinds of kagamimochi where a mochi of size 4 is topped by a mochi of size 2, and two kinds where a mochi of size 10 is topped by a mochi of size 4.

To solve this problem, we need to count the number of distinct pairs of mochi (rice cakes) such that one can be placed on top of the other. The condition for stacking is that the size of the top mochi must be at most half the size of the bottom mochi. Given that the mochi are arranged in ascending order of size, we can efficiently compute the valid pairs using binary search.

### Approach
1. **Problem Analysis**: The problem requires counting all distinct pairs of mochi (i, j) where the size of mochi i (A_i) is at most half the size of mochi j (A_j). The array is sorted in ascending order, which allows us to use binary search for efficient computation.
2. **Intuition**: For each mochi considered as the top (smaller mochi), we can determine how many mochi can serve as the bottom (larger mochi) by finding the first mochi in the sorted array that is at least twice the size of the top mochi. All mochi from that position to the end of the array are valid bottoms.
3. **Algorithm Selection**: 
   - Iterate over each mochi in the array, treating it as the top mochi.
   - For each top mochi of size `x`, use binary search (via `bisect_left`) to find the first position `j` in the array where the mochi size is at least `2*x`.
   - The number of valid bottoms for this top mochi is `N - j`, where `N` is the total number of mochi.
   - Sum these counts for all top mochi to get the total number of valid kagamimochi pairs.
4. **Complexity Analysis**: The algorithm processes each mochi once, performing a binary search each time. The binary search operates in O(log N) time, leading to an overall time complexity of O(N log N), which is efficient for the given constraints (N ≤ 500,000).

### Solution Code
```python
import bisect

def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    total = 0
    for x in A:
        j = bisect.bisect_left(A, 2 * x)
        total += (n - j)
        
    print(total)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first value is the number of mochi `N`, followed by the sizes of the mochi in ascending order.
2. **Processing Each Mochi**: For each mochi (treated as the top mochi), the algorithm calculates the minimum size required for a bottom mochi, which is twice the size of the top mochi.
3. **Binary Search**: Using `bisect_left`, the algorithm finds the first position in the sorted array where the mochi size meets or exceeds this minimum size. This position `j` indicates the start of valid bottom mochi.
4. **Counting Valid Pairs**: The number of valid bottom mochi for the current top is `N - j`, as all mochi from position `j` to the end of the array satisfy the condition.
5. **Output the Result**: The total count of all valid pairs is printed as the result.

This approach efficiently leverages the sorted nature of the input array and binary search to minimize the time complexity, making it suitable for large input sizes.
