=== PROMPT ===
You will be given a competitive programming problem.
There are N cells in a row, numbered 1 to N.
For each 1 \leq i < N, cells i and i+1 are adjacent.
Initially, cell i is painted with color i.
You are given Q queries. Process them in order. Each query is of one of the following two types.

- 1 x c: Repaint the following to color c: all reachable cells reachable from cell x by repeatedly moving to an adjacent cell painted in the same color as the current cell.
- 2 c: Print the number of cells painted with color c.

Input

The input is given from Standard Input in the following format:
N Q
\mathrm{query}_1
\vdots
\mathrm{query}_Q

Each query is given in one of the following two formats:
1 x c

2 c

Output

Let q be the number of queries of the second type. Print q lines.
The i-th line should contain the answer to the i-th such query.

Constraints


- 1 \leq N \leq 5 \times 10^5
- 1 \leq Q \leq 2 \times 10^5
- In queries of the first type, 1 \leq x \leq N.
- In queries of the first and second types, 1 \leq c \leq N.
- There is at least one query of the second type.
- All input values are integers.

Sample Input 1

5 6
1 5 4
1 4 2
2 2
1 3 2
1 2 3
2 3

Sample Output 1

3
4

=== ANSWER ===
To solve this problem, we need to efficiently process a series of queries on a row of cells, where each cell initially has a unique color. The queries involve either repainting a connected component of cells to a new color or querying the number of cells of a specific color. The challenge lies in handling these operations efficiently, especially given the constraints where the number of cells (N) and queries (Q) can be as large as 500,000 and 200,000, respectively.

### Approach
1. **Initialization**: We start by initializing each cell as a separate segment with its own color. We use a Disjoint Set Union (DSU) data structure to manage these segments. The DSU helps in efficiently merging segments and tracking the boundaries (left and right) of each segment.
2. **Processing Queries**:
   - **Repaint Query (1 x c)**: For this query, we first find the segment containing cell `x`. If the segment's current color is already `c`, we skip further processing. Otherwise, we repaint the entire segment to `c`, update the count of cells for the old color and the new color, and then merge adjacent segments that have the same color `c` iteratively.
   - **Count Query (2 c)**: This query simply outputs the current count of cells colored `c`.
3. **Union-Find Operations**: The DSU operations (find and union) are optimized with path compression to ensure near-constant time complexity per operation. The union operation merges adjacent segments by updating the boundaries and the representative of the merged segment.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    q = int(data[1])
    queries = []
    index = 2
    for _ in range(q):
        t = int(data[index])
        index += 1
        if t == 1:
            x = int(data[index])
            c = int(data[index + 1])
            index += 2
            queries.append((1, x, c))
        else:
            c = int(data[index])
            index += 1
            queries.append((2, c))
    
    parent = list(range(n + 1))
    left_bound = [i for i in range(n + 1)]
    right_bound = [i for i in range(n + 1)]
    color = [i for i in range(n + 1)]
    cnt = [0] * (n + 1)
    for i in range(1, n + 1):
        cnt[i] = 1
    
    def find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]
    
    def union(a, b):
        a = find(a)
        b = find(b)
        if a == b:
            return a
        parent[b] = a
        left_bound[a] = min(left_bound[a], left_bound[b])
        right_bound[a] = max(right_bound[a], right_bound[b])
        return a
    
    output = []
    for query in queries:
        if query[0] == 1:
            x, c = query[1], query[2]
            rep = find(x)
            if color[rep] == c:
                continue
            old_color = color[rep]
            size = right_bound[rep] - left_bound[rep] + 1
            cnt[old_color] -= size
            cnt[c] += size
            color[rep] = c
            changed = True
            while changed:
                changed = False
                l = left_bound[rep]
                r = right_bound[rep]
                if l > 1:
                    rep_left = find(l - 1)
                    if rep_left != rep and color[rep_left] == c:
                        rep = union(rep, rep_left)
                        changed = True
                l = left_bound[rep]
                r = right_bound[rep]
                if r < n:
                    rep_right = find(r + 1)
                    if rep_right != rep and color[rep_right] == c:
                        rep = union(rep, rep_right)
                        changed = True
        else:
            c = query[1]
            output.append(str(cnt[c]))
    
    print("\n".join(output))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The DSU structures (`parent`, `left_bound`, `right_bound`, and `color`) are initialized such that each cell is its own segment. The `cnt` array tracks the number of cells for each color.
2. **Processing Queries**:
   - **Repaint Query (1 x c)**: The segment containing cell `x` is found using `find(x)`. If the segment's color is not `c`, it is repainted by updating `cnt` for the old and new colors. Adjacent segments of color `c` are then merged iteratively using `union` operations.
   - **Count Query (2 c)**: The current count of cells colored `c` is retrieved from `cnt[c]` and added to the output.
3. **Union-Find Operations**: The `find` function uses path compression for efficiency. The `union` function merges two segments by updating the parent of one segment and adjusting the boundaries of the resulting segment. This ensures that the DSU operations remain efficient even for large inputs. The iterative merging process after repainting ensures that all adjacent segments of the same color are combined into a single segment.
