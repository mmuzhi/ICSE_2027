You will be given a competitive programming problem.
There is a grid with H rows and W columns. Let (i,j) denote the cell at the i-th row from the top and the j-th column from the left.
If S_{i,j} is #, the cell (i,j) is impassable; if it is ., the cell is passable and contains no house; if it is @, the cell is passable and contains a house.
Initially, Santa Claus is in cell (X,Y). He will act according to the string T as follows.

- Let |T| be the length of the string T. For i=1,2,\ldots,|T|, he moves as follows.
- Let (x,y) be the cell he is currently in.
- If T_i is U and cell (x-1,y) is passable, move to cell (x-1,y).
- If T_i is D and cell (x+1,y) is passable, move to cell (x+1,y).
- If T_i is L and cell (x,y-1) is passable, move to cell (x,y-1).
- If T_i is R and cell (x,y+1) is passable, move to cell (x,y+1).
- Otherwise, stay in cell (x,y).





Find the cell where he is after completing all actions, and the number of distinct houses that he passed through or arrived at during his actions. If the same house is passed multiple times, it is only counted once.

Input

The Input is given from Standard Input in the following format:
H W X Y
S_{1,1}S_{1,2}\ldots S_{1,W}
\dots
S_{H,1}S_{H,2}\ldots S_{H,W}
T

Output

Let (X,Y) be the cell where he is after completing all actions, and C be the number of distinct houses he passed through or arrived at during his actions. Print X,Y,C in this order separated by spaces.

Constraints


- 3 \leq H,W \leq 100
- 1 \leq X \leq H
- 1 \leq Y \leq W
- All given numbers are integers.
- Each S_{i,j} is one of #, ., @.
- S_{i,1} and S_{i,W} are # for every 1 \leq i \leq H.
- S_{1,j} and S_{H,j} are # for every 1 \leq j \leq W.
- S_{X,Y}= .
- T is a string of length at least 1 and at most 10^4, consisting of U, D, L, R.

To solve this problem, we need to simulate Santa Claus's movement on a grid according to a given sequence of moves and track the distinct houses he passes through or arrives at during his journey. The grid consists of passable cells (either empty or containing a house) and impassable cells (walls). Santa starts at a specified position and processes each move in the sequence, moving if the next cell is passable; otherwise, he stays in place. After processing all moves, we determine his final position and the count of distinct houses he visited.

### Approach
1. **Initialization**: Read the grid dimensions (H, W), starting position (X, Y), the grid itself, and the movement sequence (T). Convert the starting position to 0-indexed for easier array access.
2. **Movement Simulation**: For each move in the sequence:
   - Determine the next cell based on the move direction (U, D, L, R).
   - Check if the next cell is within bounds and is not a wall ('#'). If so, move Santa to that cell; otherwise, he remains in the current cell.
   - After moving (or staying), if the current cell contains a house ('@'), add its coordinates to a set to track distinct houses visited.
3. **Result Extraction**: After processing all moves, convert Santa's final position back to 1-indexed and output it along with the count of distinct houses visited.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    H, W, X, Y = map(int, data[0].split())
    grid = []
    for i in range(1, 1 + H):
        grid.append(data[i].strip())
    T = data[1 + H].strip()
    
    x, y = X - 1, Y - 1
    visited_houses = set()
    
    if grid[x][y] == '@':
        visited_houses.add((x, y))
    
    for move in T:
        nx, ny = x, y
        if move == 'U':
            nx = x - 1
        elif move == 'D':
            nx = x + 1
        elif move == 'L':
            ny = y - 1
        elif move == 'R':
            ny = y + 1
        
        if 0 <= nx < H and 0 <= ny < W and grid[nx][ny] != '#':
            x, y = nx, ny
        
        if grid[x][y] == '@':
            visited_houses.add((x, y))
    
    print(f"{x + 1} {y + 1} {len(visited_houses)}")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line contains grid dimensions and starting position, followed by the grid lines, and the movement sequence.
2. **Initial Setup**: The starting position (X, Y) is converted to 0-indexed (x, y) for easier array manipulation. A set `visited_houses` is initialized to store distinct house coordinates.
3. **Processing Moves**: For each move in the sequence T:
   - The next cell (nx, ny) is calculated based on the move direction.
   - If the next cell is within grid bounds and is not a wall, Santa moves to that cell.
   - The current cell (after move or stay) is checked for a house ('@'), and if found, its coordinates are added to the set.
4. **Output**: After processing all moves, Santa's final position is converted back to 1-indexed and printed along with the count of distinct houses visited (size of `visited_houses` set).

This approach efficiently tracks Santa's path and distinct houses visited by leveraging set operations for uniqueness and straightforward grid traversal for movement simulation. The solution handles edge cases such as blocked moves and grid boundaries gracefully.
