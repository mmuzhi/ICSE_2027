                  i += i & -i;
              }

                  i -= i & -i;

=== ANSWER ===
To solve this problem, we need to find the minimum total cost required to sort a permutation in ascending order by performing adjacent swaps. The cost of swapping elements at positions `i` and `i+1` is `i`. The solution involves calculating the cost for each element's movement based on the number of greater elements to its left in the initial permutation.

### Approach
1. **Problem Analysis**: The problem requires sorting a permutation using adjacent swaps where each swap at position `i` costs `i`. The challenge is to minimize the total cost. The key insight is that each element's movement cost can be derived from the number of greater elements to its left. Specifically, for each element `x` at initial position `i`, the cost is calculated as `leftGreater * i - leftGreater * (leftGreater + 1) // 2`, where `leftGreater` is the count of elements greater than `x` to the left of `x` in the initial array.

2. **Fenwick Tree (Binary Indexed Tree)**: To efficiently compute `leftGreater` for each element, we use a Fenwick Tree. This data structure helps in maintaining prefix sums dynamically, allowing us to count the number of elements less than or equal to `x` encountered so far (to the left of the current element).

3. **Algorithm**:
   - Initialize the Fenwick Tree.
   - Iterate over each element in the permutation from left to right.
   - For each element `x` at position `i`, compute `leftGreater` as `(i - 1) - query(x)`, where `query(x)` returns the count of elements less than or equal to `x` seen so far.
   - Calculate the cost for the current element using the formula `leftGreater * i - leftGreater * (leftGreater + 1) // 2`.
   - Update the Fenwick Tree with the current element `x`.
   - Accumulate the cost for all elements to get the total minimum cost.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    a = list(map(int, data[1:1+n]))
    
    fenw = [0] * (n + 1)
    
    def update(index, delta):
        while index <= n:
            fenw[index] += delta
            index += index & -index
            
    def query(index):
        s = 0
        while index > 0:
            s += fenw[index]
            index -= index & -index
        return s

    total_cost = 0
    for idx in range(n):
        x = a[idx]
        pos = idx + 1
        count_left = query(x)
        left_greater = (pos - 1) - count_left
        cost_x = left_greater * pos - left_greater * (left_greater + 1) // 2
        total_cost += cost_x
        update(x, 1)
        
    print(total_cost)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of elements `n` and the permutation array `a`.
2. **Fenwick Tree Setup**: A Fenwick Tree `fenw` is initialized to keep track of elements encountered during the iteration.
3. **Processing Elements**: For each element `x` at position `i` (1-based index):
   - **Querying the Tree**: The number of elements less than or equal to `x` encountered so far (`count_left`) is obtained using the `query` function.
   - **Calculating Left Greater Elements**: The count of elements greater than `x` to the left (`left_greater`) is computed as `(i - 1) - count_left`.
   - **Cost Calculation**: The cost for moving `x` is calculated using the formula `left_greater * i - left_greater * (left_greater + 1) // 2`, which accounts for the positions where swaps occur.
4. **Updating the Tree**: The current element `x` is added to the Fenwick Tree using the `update` function.
5. **Output**: The accumulated total cost is printed as the result.

This approach efficiently computes the minimal cost by leveraging the Fenwick Tree for dynamic range queries, ensuring optimal performance with a time complexity of O(N log N).
