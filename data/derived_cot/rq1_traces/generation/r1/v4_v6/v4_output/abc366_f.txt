Each function is of the form \( f_i(x) = A_i x + B_i \), and the composition of K functions is applied sequentially, with the innermost function applied first.

### Approach
1. **Problem Analysis**: The problem requires selecting K distinct functions from N available functions and determining the optimal order of their composition to maximize the result when applied to the input value 1. The composition of functions is applied from the innermost to the outermost, meaning the first function in the sequence is applied first, followed by the next, and so on.
2. **Key Insight**: The value of the composition depends critically on the order of the functions. For instance, placing a function with a higher multiplicative coefficient (A_i) earlier in the sequence can significantly amplify subsequent functions' contributions.
3. **Algorithm Selection**: Given the constraints (N up to 200,000 and K up to 10), a brute-force approach checking all permutations is infeasible. Instead, we use a hybrid approach:
   - **Exact DFS with Pruning**: If the number of distinct function types (based on A_i and B_i) is small (≤ 50), we perform a depth-first search (DFS) that prunes duplicate function types at each recursion level to avoid redundant calculations.
   - **Heuristic-based DFS**: For larger sets of distinct function types, we use a heuristic where at each recursion level, we only consider the top 50 functions that maximize the current value when applied next. This reduces the branching factor, making the solution feasible for larger inputs.
4. **Complexity Analysis**: 
   - **Exact DFS**: The complexity is \( O(T^K) \), where T is the number of distinct function types. For T ≤ 50 and K ≤ 10, this is manageable.
   - **Heuristic-based DFS**: The complexity is \( O(50^K) \), which is feasible for K ≤ 10 as \( 50^{10} \approx 97.6 \times 10^{12} \). While this is large, practical performance is often better due to pruning and early termination.

import sys

def main():

        b = int(data[index+1])
        index += 2
        functions.append((a, b))
        distinct_types.add((a, b))

    ans = 0
    used = [False] * n

    if T <= 50 or n <= 50:
        def dfs(depth, a, b):
            nonlocal ans
            if depth == k:
                current_val = a + b
                if current_val > ans:
                    ans = current_val
                return
            visited_types = set()
            for i in range(n):
                if not used[i]:
                    typ = functions[i]
                    if typ in visited_types:
                        continue
                    visited_types.add(typ)
                    used[i] = True
                    new_a = a * typ[0]
                    new_b = b + a * typ[1]
                    dfs(depth+1, new_a, new_b)

        dfs(0, 1, 0)

        def dfs_hybrid(depth, a, b):
            nonlocal ans
            if depth == k:

                if current_val > ans:

                return
