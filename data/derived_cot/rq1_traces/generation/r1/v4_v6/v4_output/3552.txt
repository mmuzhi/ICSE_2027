To solve this problem, we need to find the largest integer with `n` digits (as a string) that is both a palindrome and divisible by `k`. The solution involves constructing the largest possible palindrome by focusing on the first half of the digits and ensuring the entire number meets the divisibility condition.

### Approach
1. **Problem Analysis**: The problem requires finding the largest `n`-digit palindrome divisible by `k`. Given the constraints (n up to 10^5 and k between 1 and 9), a brute-force approach is infeasible. Instead, we leverage dynamic programming (DP) to efficiently explore possible digit sequences for the first half of the palindrome.
2. **Key Insight**: A palindrome reads the same forwards and backwards. For even `n`, the first half mirrored forms the second half. For odd `n`, the first half (excluding the middle digit) mirrored forms the second half around a central digit.
3. **Dynamic Programming Setup**: 
   - **Precompute Powers**: Calculate powers of 10 modulo `k` up to `n-1` to handle large exponents efficiently.
   - **Coefficient Calculation**: For each digit in the first half, compute its contribution to the residue (mod `k`). This involves terms from both its position and its mirrored position (if applicable).
   - **Backward DP**: Initialize a DP table where `dp[i][r]` indicates if the digits from position `i` to the end can achieve residue `r`. This is filled by iterating from the end of the first half backwards.
4. **Recovery of Digits**: Using the DP table, greedily select the largest valid digits for each position in the first half, ensuring the entire number meets the divisibility condition.
5. **Construct Palindrome**: Combine the first half with its mirrored version (adjusted for odd `n`) to form the full palindrome.

### Explanation
1. **Precompute Powers**: The array `pow10` stores \(10^i \mod k\) for \(0 \leq i < n\), facilitating efficient residue calculations for large numbers.
2. **Coefficient Calculation**: For each digit in the first half of the palindrome, `coeff[i]` combines contributions from its position and its mirrored position (if applicable), modulo `k`.
3. **Dynamic Programming Table Initialization**: The DP table `dp[i][r]` is initialized to track whether the digits from position `i` to the end can achieve residue `r` modulo `k`. The base case sets `dp[m][0] = True`.
4. **Backward DP Filling**: For each digit position from the end of the first half to the start, and for each possible residue, the DP table is updated by checking if any digit (9 down to 0) can transition to a valid next state.
5. **Digit Recovery**: Starting from residue 0, digits are selected greedily (9 down to 1 for the first digit, 9 down to 0 otherwise) to form the largest valid first half sequence.
6. **Palindrome Construction**: The full palindrome is formed by mirroring the first half (adjusted for odd `n` by excluding the middle digit during mirroring).

This approach efficiently narrows down the search space using dynamic programming and modular arithmetic, ensuring optimal performance even for large `n`.
