To solve this problem, we need to determine whether we can transform string `S` into string `T` using at most one operation. The operations allowed are insertion, deletion, or replacement of a single character. Given that `K` is always 1, we can handle this by checking specific cases based on the lengths of `S` and `T`.

### Approach
1. **Check for Equality**: If `S` is already equal to `T`, no operations are needed, so the answer is "Yes".
2. **Same Length Strings**: If the lengths of `S` and `T` are the same, we can only use a replacement operation. We check if there is exactly one differing character between `S` and `T`. If so, one replacement suffices; otherwise, it's impossible.
3. **Deletion Case**: If `S` is longer than `T` by exactly one character, we check if deleting one character from `S` can result in `T`. This involves finding the first position where `S` and `T` differ and then verifying that the rest of `S` (after skipping the differing character) matches the rest of `T`.
4. **Insertion Case**: If `S` is shorter than `T` by exactly one character, we check if inserting one character into `S` can result in `T`. This involves finding the first position where `S` and `T` differ and then verifying that the rest of `S` matches the rest of `T` (starting from the next character in `T`).
5. **Other Cases**: If the lengths differ by more than one, it's impossible to transform `S` into `T` with a single operation, so the answer is "No".

        print("Yes")

    m = len(T)
    
    if n == m:

        for i in range(n):
            if S[i] != T[i]:
                diff_count += 1
                if diff_count > 1:

        if diff_count == 1:

        else:

    elif n == m + 1:
        i = 0
        while i < m and S[i] == T[i]:
            i += 1
        j = i
        k = i + 1
        while j < m:
            if k >= n or S[k] != T[j]:
                break
            j += 1
            k += 1
        if j == m:

    elif n == m - 1:

### Explanation
1. **Initial Check**: The code first checks if `S` and `T` are identical. If they are, it immediately returns "Yes".
2. **Same Length Handling**: For strings of the same length, the code counts the number of differing characters. If there is exactly one difference, it returns "Yes"; otherwise, "No".
3. **Deletion Handling**: If `S` is longer than `T` by one character, the code finds the first position where they differ. It then checks if the rest of `S` (after skipping the differing character) matches `T` from that position onward. If so, it returns "Yes"; otherwise, "No".
4. **Insertion Handling**: If `S` is shorter than `T` by one character, the code finds the first position where they differ. It then checks if the rest of `S` matches the rest of `T` (starting from the next character in `T`). If so, it returns "Yes"; otherwise, "No".
5. **Other Length Differences**: If the lengths differ by more than one, the code directly returns "No" since a single operation cannot bridge a length difference greater than one.

This approach efficiently checks all possible single-operation transformations in linear time, making it suitable for large input sizes.
