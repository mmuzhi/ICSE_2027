=== ANSWER ===
To solve this problem, we need to determine the number of connected components in an undirected graph constructed from a given 2D array. Each node in the graph represents a row in the array, and an edge exists between two nodes if the number of distinct integers common to both corresponding rows is at least a given integer `k`.

### Approach
1. **Preprocessing**: Convert each row of the input array into a set of distinct integers. This helps in efficiently computing the intersection between any two rows.
2. **Union-Find Data Structure**: Use the Union-Find (Disjoint Set Union, DSU) data structure to manage and merge connected components. This structure efficiently handles the union and find operations with path compression and union by rank optimizations.
3. **Edge Construction**: For each pair of distinct rows, compute the size of the intersection of their sets. If the size is at least `k`, perform a union operation on their corresponding nodes.
4. **Component Counting**: The number of connected components is initially the number of rows. Each successful union operation reduces this count by one, as two distinct components are merged into one.

### Solution Code
```python
class Solution:
    def numberOfComponents(self, properties: List[List[int]], k: int) -> int:
        n = len(properties)
        sets = [set(arr) for arr in properties]
        parent = list(range(n))
        rank = [0] * n
        components = n
        
        def find(x):
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]
        
        for i in range(n):
            for j in range(i+1, n):
                if len(sets[i] & sets[j]) >= k:
                    root_i = find(i)
                    root_j = find(j)
                    if root_i != root_j:
                        if rank[root_i] < rank[root_j]:
                            parent[root_i] = root_j
                        elif rank[root_i] > rank[root_j]:
                            parent[root_j] = root_i
                        else:
                            parent[root_j] = root_i
                            rank[root_i] += 1
                        components -= 1
        return components
```

### Explanation
1. **Preprocessing**: Each row in the input array `properties` is converted into a set of distinct integers. This step ensures that duplicate values within a row do not affect the intersection computation.
2. **Union-Find Initialization**: The `parent` array is initialized such that each node is its own parent initially. The `rank` array is used to keep track of the depth of each node's subtree for union by rank optimization.
3. **Finding and Union Operations**: For each pair of nodes (i, j), the size of the intersection of their sets is checked. If the size meets or exceeds `k`, the `find` operation locates the root of each node. If the roots are different, the `union` operation merges the nodes by updating the parent of one root to point to the other, adjusting ranks to maintain balanced trees.
4. **Component Tracking**: The variable `components` starts at `n` (the number of nodes). Each successful union reduces `components` by one, as two separate components are merged. The final value of `components` gives the number of connected components in the graph.

This approach efficiently constructs the graph and computes connected components using optimized union-find operations, ensuring optimal performance even for the upper constraint limits.
