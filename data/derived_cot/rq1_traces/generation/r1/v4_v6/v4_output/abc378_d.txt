=== PROMPT ===
You will be given a competitive programming problem.
There is a grid of H \times W cells. Let (i, j) denote the cell at the i-th row from the top and the j-th column from the left.
Cell (i, j) is empty if S_{i,j} is ., and blocked if it is #.
Count the number of ways to start from an empty cell and make K moves to adjacent cells (up, down, left, or right), without passing through blocked squares and not visiting the same cell more than once.
Specifically, count the number of sequences of length K+1, ((i_0, j_0), (i_1, j_1), \dots, (i_K, j_K)), satisfying the following.

- 1 \leq i_k \leq H, 1 \leq j_k \leq W, and S_{i_k, j_k} is ., for each 0 \leq k \leq K.
- |i_{k+1} - i_k| + |j_{k+1} - j_k| = 1 for each 0 \leq k \leq K-1.
- (i_k, j_k) \neq (i_l, j_l) for each 0 \leq k < l \leq K.

Input

The input is given from Standard Input in the following format:
H W K
S_{1,1}S_{1,2}\dots S_{1,W}
S_{2,1}S_{2,2}\dots S_{2,W}
\vdots
S_{H,1}S_{H,2}\dots S_{H,W}

Output

Print the answer.

Constraints


- 1 \leq H, W \leq 10
- 1 \leq K \leq 11
- H, W, and K are integers.
- Each S_{i,j} is . or #.
- There is at least one empty cell.

.#

2

To solve this problem, we need to count the number of valid paths in a grid where each path starts from an empty cell (denoted by '.') and consists of exactly K moves (resulting in K+1 distinct cells). The path must adhere to the constraints of moving only to adjacent cells (up, down, left, or right), not passing through blocked cells (denoted by '#'), and not revisiting any cell.

### Approach
1. **Problem Analysis**: The problem involves navigating a grid of size H x W, starting from any empty cell, and making exactly K moves to adjacent cells without revisiting any cell or stepping on blocked cells. The solution requires counting all such valid paths.
2. **Intuition**: Given the constraints (H and W up to 10, K up to 11), a backtracking depth-first search (DFS) approach is feasible. For each empty cell, we initiate a DFS to explore all possible paths of length K+1 (K moves) that start at that cell. The DFS ensures that each path consists of distinct cells and only moves to adjacent, unblocked cells.
3. **Algorithm Selection**: We use a recursive DFS with backtracking. For each starting cell, we maintain a set of visited cells to avoid revisiting. The DFS explores all four possible directions (up, down, left, right) from the current cell. If the current path length reaches K+1 (after K moves), it counts as a valid path.
4. **Complexity Analysis**: In the worst case (all cells are '.'), the number of paths per starting cell is bounded by the number of self-avoiding walks of length K. For K=11, this is approximately 4 * 3^10 = 236,196 per starting cell. With up to 100 starting cells, the total operations are around 23 million, which is manageable for Python given the constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        print(0)
        return
    
    H, W, K = map(int, data[0].split())
    grid = []
    for i in range(1, 1+H):
        grid.append(data[i].strip())
    
    directions = [(0, 1), (0, -1), (1, 0), (-1, 0)]
    
    def dfs(i, j, depth, visited):
        if depth == K:
            return 1
        count = 0
        for dx, dy in directions:
            ni, nj = i + dx, j + dy
            if 0 <= ni < H and 0 <= nj < W:
                if grid[ni][nj] == '.' and (ni, nj) not in visited:
                    visited.add((ni, nj))
                    count += dfs(ni, nj, depth + 1, visited)
                    visited.remove((ni, nj))
        return count
    
    total = 0
    for i in range(H):
        for j in range(W):
            if grid[i][j] == '.':
                visited = set()
                visited.add((i, j))
                total += dfs(i, j, 0, visited)
    print(total)

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read from standard input, where the first line contains H (rows), W (columns), and K (moves). The subsequent lines represent the grid.
2. **DFS Setup**: The DFS function takes the current cell (i, j), the current depth (number of moves made), and a set of visited cells. It returns the count of valid paths starting from (i, j) with the given visited set.
3. **Direction Handling**: For each cell, the DFS explores all four adjacent directions. If the adjacent cell is within bounds, unblocked, and unvisited, it is added to the visited set, and the DFS proceeds recursively.
4. **Base Case**: When the depth reaches K, it signifies a valid path of K+1 cells, and the function returns 1.
5. **Summing Paths**: For each empty cell in the grid, the DFS is initiated, and the total count of valid paths is accumulated and printed.

This approach efficiently explores all possible paths while adhering to the constraints, leveraging backtracking to ensure paths are distinct and valid.
