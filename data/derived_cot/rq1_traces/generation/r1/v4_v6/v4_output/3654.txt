You will be given a competitive programming problem.
You are given an integer array nums and three integers k, op1, and op2.
You can perform the following operations on nums:

Operation 1: Choose an index i and divide nums[i] by 2, rounding up to the nearest whole number. You can perform this operation at most op1 times, and not more than once per index.
Operation 2: Choose an index i and subtract k from nums[i], but only if nums[i] is greater than or equal to k. You can perform this operation at most op2 times, and not more than once per index.

Note: Both operations can be applied to the same index, but at most once each.
Return the minimum possible sum of all elements in nums after performing any number of operations.
 
Example 1:

Input: nums = [2,8,3,19,3], k = 3, op1 = 1, op2 = 1
Output: 23
Explanation:

Apply Operation 2 to nums[1] = 8, making nums[1] = 5.
Apply Operation 1 to nums[3] = 19, making nums[3] = 10.
The resulting array becomes [2, 5, 3, 10, 3], which has the minimum possible sum of 23 after applying the operations.


Example 2:

Apply Operation 2 to nums[2] = 3, making nums[2] = 0.
The resulting array becomes [1, 2, 0], which has the minimum possible sum of 3 after applying the operations.

To solve this problem, we need to minimize the sum of an integer array after performing up to `op1` operations of type 1 and up to `op2` operations of type 2 on its elements. Each operation can be applied at most once per element, and both operations can be applied to the same element, but the order of operations matters for the result.

### Approach
1. **Problem Analysis**: The problem involves performing two types of operations on array elements:
   - **Operation 1**: Divide an element by 2 and round up to the nearest integer (i.e., `ceil(x/2)`).
   - **Operation 2**: Subtract `k` from an element if it is greater than or equal to `k`.
   
   The goal is to minimize the sum of the array after performing these operations within the constraints of `op1` and `op2`.

2. **Key Insights**:
   - **Operation Order**: The order of operations affects the result when both are applied to the same element. For instance, applying Operation 1 followed by Operation 2 might yield a different result than the reverse order. Therefore, for each element eligible for both operations, we compute both possible results and choose the minimum.
   - **Dynamic Programming (DP)**: Given the constraints (array length up to 100 and operations up to 100), a DP approach is feasible. The DP state `dp[a][b]` represents the minimum sum achievable after processing some elements with `a` operations of type 1 and `b` operations of type 2.

3. **Algorithm Selection**:
   - **State Initialization**: Initialize a DP table `dp` where `dp[a][b]` stores the minimum sum after using `a` type 1 and `b` type 2 operations. Start with `dp[0][0] = 0` (no operations used, sum is 0 before processing any elements).
   - **Processing Each Element**: For each element, generate all possible operation outcomes (no operation, Operation 1 only, Operation 2 only if applicable, and both operations if applicable). For both operations, compute the minimum result by considering both orders.
   - **State Update**: For each element, update the DP table by considering each possible operation outcome and updating the state `(a + da, b + db)` where `da` and `db` are the operations used for the current element.

4. **Complexity Analysis**: The algorithm processes each element in the array, and for each element, it updates the DP table of size `(op1 + 1) * (op2 + 1)`. Each update involves checking up to 4 operation outcomes. Hence, the time complexity is O(n * op1 * op2 * 4), which is efficient for the given constraints (n ≤ 100, op1 ≤ 100, op2 ≤ 100).

### Solution Code
```python
class Solution:
    def minArraySum(self, nums: List[int], k: int, op1: int, op2: int) -> int:
        n = len(nums)
        dp = [[10**18] * (op2 + 1) for _ in range(op1 + 1)]
        dp[0][0] = 0
        
        for x in nums:
            options = []
            options.append((x, 0, 0))
            options.append(((x + 1) // 2, 1, 0))
            if k > 0 and x >= k:
                options.append((x - k, 0, 1))
            if k > 0 and x >= k:
                cand1 = 10**18
                if (x + 1) // 2 >= k:
                    cand1 = (x + 1) // 2 - k
                cand2 = (x - k + 1) // 2
                val = min(cand1, cand2)
                options.append((val, 1, 1))
            
            new_dp = [[10**18] * (op2 + 1) for _ in range(op1 + 1)]
            for a in range(op1 + 1):
                for b in range(op2 + 1):
                    if dp[a][b] == 10**18:
                        continue
                    current_sum = dp[a][b]
                    for val, da, db in options:
                        na = a + da
                        nb = b + db
                        if na <= op1 and nb <= op2:
                            if new_dp[na][nb] > current_sum + val:
                                new_dp[na][nb] = current_sum + val
            dp = new_dp
        
        ans = 10**18
        for a in range(op1 + 1):
            for b in range(op2 + 1):
                if dp[a][b] < ans:
                    ans = dp[a][b]
        return ans
```

### Explanation
1. **Initialization**: The DP table `dp` is initialized to a large value (10^18) to represent an initially unreachable state, except `dp[0][0]` which is set to 0 (sum 0 with no operations used).
2. **Processing Elements**: For each element in the array, possible operation outcomes are generated:
   - **No operation**: The element remains unchanged.
   - **Operation 1**: The element is divided by 2 and rounded up.
   - **Operation 2**: If `k > 0` and the element is ≥ `k`, subtract `k` from the element.
   - **Both operations**: If applicable, compute the minimum result from both operation sequences (Operation 1 then 2, or 2 then 1).
3. **DP Update**: For each element, the DP table is updated by considering each operation outcome. The new state `(a + da, b + db)` is updated if the new sum is lower than the current value.
4. **Result Extraction**: After processing all elements, the minimum sum across all valid states `(a, b)` (where `a ≤ op1` and `b ≤ op2`) is returned.

This approach efficiently explores all possible operation combinations within the constraints, ensuring the minimal sum is achieved.
