You are given an integer array nums and two integers, k and m.
Return the maximum sum of k non-overlapping subarrays of nums, where each subarray has a length of at least m.
 
Example 1:

Input: nums = [1,2,-1,3,3,4], k = 2, m = 2
Output: 13
Explanation:
The optimal choice is:

Subarray nums[3..5] with sum 3 + 3 + 4 = 10 (length is 3 >= m).
Subarray nums[0..1] with sum 1 + 2 = 3 (length is 2 >= m).

The total sum is 10 + 3 = 13.

Input: nums = [-10,3,-1,-2], k = 4, m = 1

Explanation:

=== ANSWER ===
To solve this problem, we need to find the maximum sum of `k` non-overlapping subarrays in an integer array `nums`, where each subarray has a length of at least `m`. The solution involves dynamic programming to efficiently compute the optimal subarrays without overlapping while respecting the length constraint.

### Approach
1. **Prefix Sum Calculation**: Compute the prefix sums of the array to facilitate quick sum calculations of any subarray.
2. **Dynamic Programming Initialization**: Initialize two 2D arrays, `dp` and `ep`, where:
   - `dp[j][i]` represents the maximum sum of `j` subarrays from the first `i+1` elements, regardless of whether the last subarray ends at `i`.
   - `ep[j][i]` represents the maximum sum of `j` subarrays from the first `i+1` elements where the last subarray ends at `i`.
3. **Base Case Handling**: For `j=0` (zero subarrays), the sum is zero for all prefixes.
4. **Dynamic Programming Transition**:
   - For each `j` from 1 to `k`, precompute an auxiliary array `arr` where each element `arr[s]` is `dp[j-1][s-1] - P[s]` (with `P[s]` being the prefix sum at `s`). This helps in efficiently calculating the sum when starting a new subarray at `s`.
   - Compute a prefix maximum array `best_arr` from `arr` to quickly access the maximum value in any range.
   - For each position `i` in the array:
     - **Extend the Last Subarray**: If possible, extend the last subarray ending at `i-1` to include `nums[i]`.
     - **Start a New Subarray**: Start a new subarray ending at `i` by considering all valid starting positions `s` (from 0 to `i-m+1`). The sum is derived from `best_arr[s] + P[i+1]`.
   - Update `ep[j][i]` as the maximum of extending or starting a new subarray, and `dp[j][i]` as the maximum of `dp[j][i-1]` and `ep[j][i]`.
5. **Result Extraction**: The answer is found in `dp[k][n-1]`, which holds the maximum sum for `k` subarrays over the entire array.

### Solution Code
```python
class Solution:
    def maxSum(self, nums: List[int], k: int, m: int) -> int:
        n = len(nums)
        P = [0] * (n + 1)
        for i in range(1, n + 1):
            P[i] = P[i - 1] + nums[i - 1]
        
        MIN_VAL = -10**18
        dp = [[MIN_VAL] * n for _ in range(k + 1)]
        ep = [[MIN_VAL] * n for _ in range(k + 1)]
        
        for i in range(n):
            dp[0][i] = 0
        
        for j in range(1, k + 1):
            arr = [MIN_VAL] * (n + 1)
            arr[0] = 0 if j == 1 else MIN_VAL
            for s in range(1, n + 1):
                arr[s] = dp[j - 1][s - 1] - P[s]
            
            best_arr = [MIN_VAL] * (n + 1)
            best_arr[0] = arr[0]
            for s in range(1, n + 1):
                best_arr[s] = max(best_arr[s - 1], arr[s])
            
            for i in range(n):
                candidate1 = MIN_VAL
                if i >= 1 and ep[j][i - 1] != MIN_VAL:
                    candidate1 = ep[j][i - 1] + nums[i]
                
                candidate2 = MIN_VAL
                idx = i - m + 1
                if idx >= 0 and idx <= n:
                    candidate2 = best_arr[idx] + P[i + 1]
                
                ep[j][i] = max(candidate1, candidate2)
                
                if i >= 1:
                    dp[j][i] = max(dp[j][i - 1], ep[j][i])
                else:
                    dp[j][i] = ep[j][i]
        
        return dp[k][n - 1]
```

### Explanation
- **Prefix Sum Array (P)**: This array stores cumulative sums to allow O(1) range sum queries.
- **Dynamic Programming Arrays (dp and ep)**: 
  - `dp[j][i]` tracks the best sum for `j` subarrays up to index `i`, whether or not the last subarray ends at `i`.
  - `ep[j][i]` specifically tracks the best sum when the last subarray ends at `i`.
- **Auxiliary Arrays (arr and best_arr)**: 
  - `arr[s]` helps in calculating the sum when starting a new subarray at `s`.
  - `best_arr` is a prefix maximum array derived from `arr` for efficient maximum value queries.
- **Transition Logic**: For each position `i`, the solution considers extending the last subarray or starting a new one, updating `ep[j][i]` and `dp[j][i]` accordingly.
- **Result**: The value `dp[k][n-1]` gives the maximum sum of `k` valid subarrays over the entire array. This approach efficiently handles the constraints using dynamic programming and prefix sums.
