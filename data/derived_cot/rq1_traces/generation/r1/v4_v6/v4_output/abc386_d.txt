There is an N \times N grid. Takahashi wants to color each cell black or white so that all of the following conditions are satisfied:

- For every row, the following condition holds:

- There exists an integer i\ (0\leq i\leq N) such that the leftmost i cells are colored black, and the rest are colored white.

- For every column, the following condition holds:

- There exists an integer i\ (0\leq i\leq N) such that the topmost i cells are colored black, and the rest are colored white.


Out of these N^2 cells, M of them have already been colored. Among them, the i-th one is at the X_i-th row from the top and the Y_i-th column from the left, and it is colored black if C_i is B and white if C_i is W.
Determine whether he can color the remaining uncolored N^2 - M cells so that all the conditions are satisfied.

Input

The input is given from Standard Input in the following format:
N M
X_1 Y_1 C_1
\vdots
X_M Y_M C_M

Output

If it is possible to satisfy the conditions, print Yes; otherwise, print No.

Constraints


- 1\leq N\leq 10^9
- 1\leq M\leq \min(N^2,2\times 10^5)
- 1\leq X_i,Y_i\leq N
- (X_i,Y_i)\neq (X_j,Y_j)\ (i\neq j)
- C_i is B or W.
- All input numbers are integers.

To solve this problem, we need to determine if it's possible to color an N x N grid such that each row and column satisfies specific conditions. The conditions are:
1. For every row, there exists an integer \(i\) (0 ≤ \(i\) ≤ N) such that the leftmost \(i\) cells are black and the rest are white.
2. For every column, there exists an integer \(i\) (0 ≤ \(i\) ≤ N) such that the topmost \(i\) cells are black and the rest are white.

The grid has some cells pre-colored (either black or white), and we need to decide if the remaining cells can be colored to meet the conditions.

### Approach
1. **Problem Analysis**:
   - The conditions imply that the black cells must form a contiguous block starting from the top-left corner, similar to a staircase pattern. This means for each row, the black cells must be contiguous from the left, and for each column, the black cells must be contiguous from the top.
   - The solution involves determining a non-increasing sequence \(r_1, r_2, \ldots, r_N\) where \(r_i\) represents the last column of black cells in row \(i\). The sequence must be non-increasing because if a row \(i\) has \(r_i\) black cells, the next row \(i+1\) cannot have more black cells than \(r_i\) (due to the column condition).

2. **Key Insights**:
   - For each row, the constraints derived from pre-colored cells must be consistent. Specifically:
     - If a row has both black and white cells, the rightmost black cell must be to the left of the leftmost white cell.
     - For a row with only black cells, the rightmost black cell defines the minimum number of black cells required.
     - For a row with only white cells, the leftmost white cell defines the maximum number of black cells allowed.
   - The sequence \(r_i\) must be non-increasing and satisfy the constraints for each row. The greedy approach processes rows in order, adjusting \(r_i\) to be as large as possible while ensuring it does not exceed the constraints of subsequent rows.

3. **Algorithm Selection**:
   - **Aggregate Constraints**: For each row with pre-colored cells, compute the minimum and maximum allowable values for \(r_i\) (the split point between black and white cells in the row).
   - **Check Row Consistency**: Ensure that for any row, the rightmost pre-colored black cell is to the left of the leftmost pre-colored white cell. If not, the grid cannot be colored.
   - **Greedy Assignment**: Process rows in order (from top to bottom), setting \(r_i\) to the minimum of the previous row's \(r_{i-1}\) and the current row's maximum allowable value. If this value is less than the row's minimum required value, the grid cannot be colored.

4. **Complexity Analysis**:
   - **Time Complexity**: The algorithm processes each of the \(M\) pre-colored cells once to aggregate constraints and then sorts the constraints by row number, which takes \(O(M \log M)\) time. Processing the rows involves a linear pass through the sorted constraints and skipping unconstrained rows, which is \(O(M)\).
   - **Space Complexity**: The algorithm uses \(O(M)\) space to store the constraints for rows with pre-colored cells.
