You will be given a competitive programming problem.
In the nation of AtCoder, there are N cities numbered 1 to N, and M roads numbered 1 to M.
Road i connects cities A_i and B_i bidirectionally and has a length of C_i.
You are given Q queries to process in order. The queries are of the following two types.

- 1 i: Road i becomes closed.
- 2 x y: Print the shortest distance from city x to city y, using only roads that are not closed. If city y cannot be reached from city x, print -1 instead.

It is guaranteed that each test case contains at most 300 queries of the first type.

Input

The input is given from Standard Input in the following format:
N M Q
A_1 B_1 C_1
\vdots
A_M B_M C_M
\mathrm{query}_1
\vdots
\mathrm{query}_Q

Each query is in one of the following two formats:
1 i

2 x y

Output

Process the queries in order.

Constraints


- 2 \leq N \leq 300
- 0 \leq M \leq \frac{N(N-1)}{2}
- 1 \leq A_i < B_i \leq N
- All pairs (A_i, B_i) are distinct.
- 1 \leq C_i \leq 10^9
- 1 \leq Q \leq 2 \times 10^5
- In the queries of the first type, 1 \leq i \leq M.
- The road given in a query of the first type is not already closed at that time.
- The number of queries of the first type is at most 300.
- In the queries of the second type, 1 \leq x < y \leq N.
- All input values are integers.

To solve this problem, we need to efficiently process a series of queries on a graph where roads can be closed, and we need to answer queries about the shortest path between two cities using only the roads that are currently open. Given the constraints, especially the limited number of road closures (up to 300), we can leverage an offline approach that processes the queries in reverse order, adding roads back into the graph and dynamically updating the shortest paths.

### Approach
1. **Initial Setup**: Read the input to construct the graph, including the cities, roads, and queries. Track all roads that will ever be closed during the query processing.
2. **Initial Graph Construction**: Build the initial graph excluding all roads that will be closed at any point. Compute the all-pairs shortest paths (APSP) for this initial graph using the Floyd-Warshall algorithm.
3. **Reverse Query Processing**: Process the queries in reverse order. For each query:
   - **Type 2 (Shortest Path Query)**: Record the current shortest path between the specified cities.
   - **Type 1 (Road Closure)**: Instead of closing the road, we add it back to the graph. Update the shortest paths by considering the newly added road and propagating its effect using the Floyd-Warshall algorithm with the road's endpoints as intermediate nodes.
4. **Output Results**: After processing all queries in reverse, the results for Type 2 queries are collected in reverse order. Reverse these results to match the original query sequence and output them.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    it = iter(data)
    n = int(next(it)); m = int(next(it)); q = int(next(it))
    roads = []
    for _ in range(m):
        a = int(next(it)); b = int(next(it)); c = int(next(it))
        roads.append((a-1, b-1, c))
    
    queries = []
    closed_set = set()
    for _ in range(q):
        t = next(it)
        if t == '1':
            i_road = int(next(it)) - 1
            queries.append((1, i_road))
            closed_set.add(i_road)
        else:
            x = int(next(it)) - 1
            y = int(next(it)) - 1
            queries.append((2, x, y))
    
    INF = 10**18
    dist = [[INF] * n for _ in range(n)]
    for i in range(n):
        dist[i][i] = 0
    
    for idx, (a, b, c) in enumerate(roads):
        if idx not in closed_set:
            if c < dist[a][b]:
                dist[a][b] = c
                dist[b][a] = c
    
    for k in range(n):
        for i in range(n):
            if dist[i][k] == INF:
                continue
            for j in range(n):
                if dist[i][j] > dist[i][k] + dist[k][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    
    results = []
    for i in range(q-1, -1, -1):
        qry = queries[i]
        if qry[0] == 2:
            x, y = qry[1], qry[2]
            results.append(dist[x][y] if dist[x][y] < INF else -1)
        else:
            r_idx = qry[1]
            a, b, c = roads[r_idx]
            if c < dist[a][b]:
                dist[a][b] = c
                dist[b][a] = c
            for k in [a, b]:
                for i in range(n):
                    if dist[i][k] == INF:
                        continue
                    for j in range(n):
                        if dist[i][j] > dist[i][k] + dist[k][j]:
                            dist[i][j] = dist[i][k] + dist[k][j]
    
    results.reverse()
    for res in results:
        print(res)

if __name__ == "__main__":
    main()
```
