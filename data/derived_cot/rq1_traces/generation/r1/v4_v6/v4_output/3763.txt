You are given a 2D integer array squares. Each squares[i] = [x_i, y_i, l_i] represents the coordinates of the bottom-left point and the side length of a square parallel to the x-axis.
Find the minimum y-coordinate value of a horizontal line such that the total area of the squares above the line equals the total area of the squares below the line.
Answers within 10^-5 of the actual answer will be accepted.
Note: Squares may overlap. Overlapping areas should be counted multiple times.
 
Example 1:

Input: squares = [[0,0,1],[2,2,1]]
Output: 1.00000
Explanation:

Any horizontal line between y = 1 and y = 2 will have 1 square unit above it and 1 square unit below it. The lowest option is 1.

Since the areas above and below the line are equal, the output is 7/6 = 1.16667.

squares[i].length == 3

1 <= l_i <= 10^9

To solve this problem, we need to find the minimum y-coordinate of a horizontal line such that the total area of the squares above the line equals the total area of the squares below the line. The solution involves efficiently calculating the areas above and below potential lines and determining the optimal y-coordinate using a sweep line algorithm.

### Approach
1. **Problem Analysis**: Given a list of squares, each defined by its bottom-left corner (x, y) and side length (l), we need to partition the squares with a horizontal line such that the total area below the line equals the total area above the line. The solution must handle overlapping squares, where overlapping areas are counted multiple times.

2. **Key Insight**: The area below the line for each square depends on the line's y-coordinate. For a square [x_i, y_i, l_i]:
   - If the line is below the square (y ≤ y_i), the area below is 0.
   - If the line is above the square (y ≥ y_i + l_i), the entire square area (l_i²) is below the line.
   - If the line intersects the square (y_i < y < y_i + l_i), the area below is (y - y_i) * l_i.

3. **Algorithm Selection**: 
   - **Sweep Line Algorithm**: We process events (start and end of each square) sorted by their y-coordinates. For each interval between consecutive events, the area below the line is a linear function of y, with a constant slope (sum of side lengths of active squares).
   - **Binary Search**: Since the function of area below the line is piecewise linear and non-decreasing, we can determine the exact y-coordinate where the area below equals half the total area by checking each interval between events.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n log n) due to sorting the events (each square contributes two events).
   - **Space Complexity**: O(n) for storing events.

### Solution Code
```python
from itertools import groupby

class Solution:
    def separateSquares(self, squares: List[List[int]]) -> float:
        total_area = 0.0
        events = []
        for sq in squares:
            x, y, l = sq
            total_area += l * l
            events.append((y, l))
            events.append((y + l, -l))
        half = total_area / 2.0
        
        events_sorted = sorted(events, key=lambda x: x[0])
        grouped_events = []
        for key, group in groupby(events_sorted, key=lambda x: x[0]):
            total_delta = sum(delta for _, delta in group)
            grouped_events.append((key, total_delta))
        
        y_prev = 0.0
        F_prev = 0.0
        slope = 0.0
        for y_current, delta in grouped_events:
            F_current = F_prev + slope * (y_current - y_prev)
            if F_prev <= half <= F_current:
                if slope == 0:
                    return y_prev
                else:
                    return y_prev + (half - F_prev) / slope
            slope += delta
            y_prev = y_current
            F_prev = F_current
        
        if abs(F_prev - half) < 1e-5:
            return y_prev
        return y_prev + (half - F_prev) / slope
```

### Explanation
1. **Initialization**: Calculate the total area of all squares and determine half of this total area.
2. **Event Processing**: For each square, generate start (y_i) and end (y_i + l_i) events. These events are sorted and grouped by their y-coordinates to handle multiple events at the same y-coordinate efficiently.
3. **Sweep Line Processing**: Initialize variables to track the current y-coordinate (y_prev), the area below the line (F_prev), and the rate of change (slope). For each event:
   - Compute the area below the line at the current event's y-coordinate (F_current).
   - Check if the half area lies between F_prev and F_current. If so, solve for the exact y-coordinate using linear interpolation.
   - Update the slope and state variables for the next interval.
4. **Final Check**: Handle edge cases where the solution might be at the last event or beyond, ensuring accuracy within the problem's precision constraints.

This approach efficiently narrows down the optimal y-coordinate by leveraging the linear nature of the area function between events, ensuring optimal performance and correctness.
