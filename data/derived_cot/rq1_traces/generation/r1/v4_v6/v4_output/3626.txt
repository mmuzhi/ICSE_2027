=== ANSWER ===
To solve this problem, we need to find the smallest number greater than or equal to a given integer `n` such that the product of its digits is divisible by another given integer `t`. 

### Approach
1. **Problem Analysis**: The task involves checking each number starting from `n` and moving upwards until we find a number whose digit product is divisible by `t`. The digit product of a number is the product of all its digits. If any digit is zero, the product becomes zero, which is divisible by any `t` (since `t` is at least 1).
2. **Intuition**: For each candidate number starting from `n`, compute the product of its digits. If this product is divisible by `t`, return the number immediately. Otherwise, increment the candidate and repeat the process.
3. **Algorithm Selection**: Given the constraints (1 ≤ n ≤ 100), a straightforward iterative approach is efficient. We iterate from `n` upwards, compute the digit product for each number, and check divisibility by `t`.
4. **Optimization**: Early termination occurs when a valid number is found. The loop is guaranteed to terminate because numbers containing a zero digit (like multiples of 10) will always satisfy the condition (product becomes zero).

### Solution Code
```python
class Solution:
    def smallestNumber(self, n: int, t: int) -> int:
        num = n
        while True:
            s = str(num)
            product = 1
            for char in s:
                product *= int(char)
            if product % t == 0:
                return num
            num += 1
```

### Explanation
1. **Initialization**: Start with `num` set to `n`.
2. **Loop**: For each `num`, convert it to a string to process each digit individually.
3. **Product Calculation**: Compute the product of all digits of `num`. If any digit is zero, the product becomes zero.
4. **Divisibility Check**: If the product is divisible by `t`, return `num` immediately. Otherwise, increment `num` and repeat the process.
5. **Termination**: The loop terminates as soon as a valid number is found, which is guaranteed to happen within a reasonable number of steps due to the constraints (e.g., numbers like 100 will have a digit product of zero, which is divisible by any `t`).

This approach efficiently checks each candidate number in ascending order, ensuring the smallest valid number is found with minimal computational overhead.
