To solve this problem, we need to find the minimum total cost of partitioning the given array of numbers into contiguous subarrays. The cost of each subarray depends on the cumulative sum of the elements up to that subarray, the order of the subarray, and the sum of the corresponding cost values. 

### Approach
1. **Problem Analysis**: The problem involves partitioning the array into contiguous subarrays such that the total cost is minimized. The cost of the i-th subarray (from index `l` to `r`) is calculated as `(sum of elements from start to r + k * i) * (sum of costs from l to r)`. The challenge is to efficiently explore all possible partitions and compute the minimal cost.

2. **Dynamic Programming with Convex Hull Trick (CHT)**: 
   - **Prefix Sums**: Precompute prefix sums for both `nums` and `cost` arrays to facilitate quick sum calculations over any subarray.
   - **Initialization**: For the base case (partitioning into a single subarray), the cost is straightforward: `(prefix_sum[i] + k) * prefix_cost[i]` for each index `i`.
   - **Dynamic Programming Transition**: For partitions into `j` subarrays (where `j > 1`), the cost of the last subarray (ending at `i`) is derived from the cost of partitions ending at some index `p` (from `j-2` to `i-1`) into `j-1` subarrays. The transition involves minimizing a linear function of the form `dp[p][j-1] - (prefix_sum[i] + k*j) * prefix_cost[p]`.
   - **Convex Hull Optimization**: To efficiently handle the minimization step in the transition, we use the convex hull trick. This involves maintaining a deque of lines (each representing a candidate partition point `p`) and querying for the minimal value at a specific `x` (which is `prefix_sum[i] + k*j`). The deque is maintained to ensure that only the lines that can be optimal are kept, and lines are added and removed based on their slopes and intercepts.

3. **Complexity Analysis**: The algorithm processes each possible number of subarrays (from 1 to `n`). For each number of subarrays `j`, it processes each index `i` (from `j-1` to `n-1`), and each line insertion and query operation is O(1) amortized due to the convex hull optimization. Thus, the overall complexity is O(n^2), which is efficient for the given constraints (n ≤ 1000).

### Solution Code
```python
import collections

class Solution:
    def minimumCost(self, nums: list, cost: list, k: int) -> int:
        n = len(nums)
        prefix_nums = [0] * n
        prefix_cost = [0] * n
        prefix_nums[0] = nums[0]
        prefix_cost[0] = cost[0]
        for i in range(1, n):
            prefix_nums[i] = prefix_nums[i-1] + nums[i]
            prefix_cost[i] = prefix_cost[i-1] + cost[i]
        
        INF = 10**18
        dp_prev = [INF] * n
        for i in range(n):
            dp_prev[i] = (prefix_nums[i] + k * 1) * prefix_cost[i]
        
        ans = dp_prev[n-1]
        
        for j in range(2, n+1):
            dp_cur = [INF] * n
            dq = collections.deque()
            for i in range(j-1, n):
                m_val = -prefix_cost[i-1]
                c_val = dp_prev[i-1]
                
                while len(dq) >= 2:
                    m0, c0 = dq[-1]
                    m1, c1 = dq[-2]
                    if (c_val - c1) * (m1 - m0) <= (c0 - c1) * (m1 - m_val):
                        dq.pop()
                    else:
                        break
                dq.append((m_val, c_val))
                
                x_val = prefix_nums[i] + k * j
                
                while len(dq) >= 2:
                    m0, c0 = dq[0]
                    m1, c1 = dq[1]
                    if m0 * x_val + c0 >= m1 * x_val + c1:
                        dq.popleft()
                    else:
                        break
                best_val = dq[0][0] * x_val + dq[0][1]
                dp_cur[i] = x_val * prefix_cost[i] + best_val
            
            ans = min(ans, dp_cur[n-1])
            dp_prev = dp_cur
        
        return ans
```

### Explanation
1. **Prefix Sums Calculation**: The prefix sums for both `nums` and `cost` arrays are precomputed to allow efficient range sum queries.
2. **Base Case Initialization**: For partitioning into a single subarray, the cost for each possible ending index is computed directly.
3. **Dynamic Programming with CHT**: For each possible number of subarrays `j` (from 2 to `n`), the algorithm processes each ending index `i` (from `j-1` to `n-1`). The convex hull trick is used to efficiently find the minimal cost transition from partitions ending at previous indices.
4. **Line Management**: The deque maintains lines representing candidate partitions, ensuring only potentially optimal lines are kept. Lines are added to the deque in a way that maintains the convex hull property, and queries for minimal values at specific points are handled efficiently.
5. **Result Calculation**: After processing each number of subarrays, the minimal cost for partitioning the entire array (ending at `n-1`) is updated. The final result is the minimal cost across all possible partitions.
