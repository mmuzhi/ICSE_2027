You are given a string S of length N consisting of 0 and 1.

It is guaranteed that S contains at least K 1-blocks.
Here is a more precise description.

- Let S_{l\ldots r} denote the substring of S from the l-th character through the r-th character.
- We define a substring S_{l\ldots r} of S to be a 1-block if it satisfies all of the following conditions:
- S_l = S_{l+1} = \cdots = S_r =  1
- l = 1 or S_{l-1} =  0
- r = N or S_{r+1} =  0


- 
Suppose that all 1-blocks in S are S_{l_1\ldots r_1}, \ldots, S_{l_m\ldots r_m}, where l_1 < l_2 < \cdots < l_m.
Then, we define the length N string T, obtained by moving the K-th 1-block to immediately after the (K-1)-th 1-block, as follows:

- T_i = S_i for 1 \leq i \leq r_{K-1}
- T_i =  1 for r_{K-1} + 1 \leq i \leq r_{K-1} + (r_K - l_K) + 1
- T_i =  0 for r_{K-1} + (r_K - l_K) + 2 \leq i \leq r_K
- T_i = S_i for r_K + 1 \leq i \leq N

Input

The input is given from Standard Input in the following format:
N K
S

Output

Print the answer.

Constraints


- 1 \leq N \leq 5 \times 10^5
- S is a string of length N consisting of 0 and 1.
- 2 \leq K
- S contains at least K 1-blocks.

### Approach
1. **Identify 1-Blocks**: The first step involves scanning the input string to identify all contiguous segments of '1's (1-blocks) that are surrounded by '0's or the string boundaries. Each 1-block is recorded as a tuple of start and end indices (0-indexed).
2. **Locate Relevant Blocks**: Once all 1-blocks are identified, we locate the (K-1)-th block (the block immediately before the target block) and the K-th block (the target block to be moved).
3. **Compute Segment Lengths**: Calculate the length of the K-th block (L) and the length of the segment between the end of the (K-1)-th block and the end of the K-th block. The zeros length is derived from the difference between the segment length and the block length (L).
4. **Construct Result String**: The result string is constructed by concatenating:
   - The substring from the start of the original string up to the end of the (K-1)-th block.
   - A string of '1's of length L (the moved K-th block).
   - A string of '0's of length (zeros_len) to cover the gap left by moving the block.
   - The substring from the character immediately after the original end of the K-th block to the end of the string.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data: 
        return
    n_k = data[0].split()
    N = int(n_k[0])
    K = int(n_k[1])
    S = data[1].strip()
    
    blocks = []
    i = 0
    while i < N:
        if S[i] == '1':
            start = i
            while i < N and S[i] == '1':
                i += 1
            end = i - 1
            blocks.append((start, end))
        else:
            i += 1
            
    block_prev = blocks[K-2]
    block_move = blocks[K-1]
    L = block_move[1] - block_move[0] + 1
    zeros_len = block_move[1] - block_prev[1] - L
    
    part1 = S[:block_prev[1] + 1]
    part2 = '1' * L
    part3 = '0' * zeros_len
    part4 = S[block_move[1] + 1:]
    
    T = part1 + part2 + part3 + part4
    print(T)

if __name__ == "__main__":
    main()
```
