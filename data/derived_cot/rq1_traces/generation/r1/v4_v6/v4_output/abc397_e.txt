To solve this problem, we need to determine whether a given tree can be decomposed into N disjoint paths, each of length K. The tree has NK vertices, and the decomposition requires partitioning all vertices into N paths such that each path is a sequence of K vertices connected by edges in the tree.

### Approach
1. **Special Case Handling (K = 1)**: If K is 1, each path consists of a single vertex. Since there are exactly N paths (each of one vertex), the answer is always "Yes".
2. **Tree Construction**: For K ≥ 2, we first build the tree from the input edges using an adjacency list.
3. **Postorder Traversal**: We perform a postorder traversal (from leaves to root) to process each node. This allows us to handle child nodes before their parent, facilitating the computation of pending path lengths.
4. **Path Matching**: For each node, we collect the pending path lengths from its children (non-zero values). We then attempt to pair these lengths such that the sum of two pending paths plus the current node equals K (i.e., `x + y + 1 = K`). If such pairs are found, they are removed, and the node is marked as used in a pair.
5. **Pending Path Handling**: After pairing, if there's one remaining pending path, it is extended through the current node (new length = `x + 1`). If this new length equals K, the path is completed (set pending length to 0). If the new length exceeds K, it's impossible, and we return "No".
6. **Root Check**: After processing all nodes, if the root has a pending path length of 0, the decomposition is successful ("Yes"); otherwise, it's ("No").

3. **Tree Construction**: The tree is built using an adjacency list from the input edges.
4. **Postorder Traversal**: Nodes are processed from leaves to root to ensure child nodes are processed before their parents.
5. **Path Matching**: For each node, pending path lengths from children are collected. Pairs of these lengths that sum to `K - 1` (when combined with the current node) are removed. The node is marked as used if any pairs are formed.
6. **Pending Path Extension**: If one pending path remains, it is extended through the current node. If the new length equals K, the path is completed (pending length set to 0). If it exceeds K, the decomposition fails.
7. **Root Check**: After processing all nodes, if the root's pending path length is 0, the decomposition is successful ("Yes"); otherwise, it fails ("No"). This ensures all paths are properly formed without leftover vertices.
