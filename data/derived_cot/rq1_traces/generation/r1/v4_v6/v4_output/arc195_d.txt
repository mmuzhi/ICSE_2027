You will be given a competitive programming problem.
There is a sequence A = (A_1,\ldots,A_N). You can perform the following two types of operations any number of times in any order:

- Let K be the length of A just before the operation. Choose an integer i such that 1 \leq i \leq K-1, and swap the i-th and (i+1)-th elements of A.
- Let K be the length of A just before the operation. Choose an integer i such that 1 \leq i \leq K and all the values from the 1-st through the i-th elements of A are equal, and delete all the elements from the 1-st through the i-th of A.

Find the minimum total number of operations required to make A an empty sequence.
You are given T test cases; solve each of them.

Input

The input is given from Standard Input in the following format:
T
\mathrm{case}_1

Each case is given in the following format:
N

Output

Print the answer for each test case in order, separated by newlines.

Constraints


- 1\leq T\leq 10^5
- 2 \leq N \leq 2 \times 10^5
- 1 \leq A_i \leq N
- The sum of N over all test cases is at most 2\times 10^5.
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to find the minimum number of operations required to completely delete a sequence using two types of operations: swapping adjacent elements and deleting a contiguous prefix of identical elements. The challenge is to minimize the total operations (swaps + deletions) by strategically using swaps to group identical elements at the front, allowing larger deletions in fewer operations.

### Approach
1. **Problem Analysis**: The problem involves processing a sequence where we can swap adjacent elements to group identical elements together at the front. Once grouped, these elements can be deleted in a single operation. The goal is to minimize the total operations (swaps and deletions) required to delete the entire sequence.

2. **Key Insight**: For each element in the sequence, if there exists a subsequent identical element, we can consider swapping it to the front to form a larger contiguous block. The cost of these swaps plus the cost of subsequent deletions can be optimized using dynamic programming.

3. **Dynamic Programming Setup**: We use a dynamic programming array `dp` where `dp[i]` represents the minimum operations needed to delete the first `i` elements of the sequence. Initialize `dp[0]` to 0 (no elements to delete) and the rest to a large value.

4. **Precompute Next Occurrences**: For each element in the sequence, precompute the next occurrence of the same value. This helps in efficiently determining where to swap elements to form contiguous blocks.

5. **DP Transitions**:
   - **Option 1**: Delete the current front element individually. This updates `dp[i+1]` to `min(dp[i+1], dp[i] + 1)`.
   - **Option 2**: If a subsequent identical element exists, bring it to the front (using swaps) and delete the contiguous block. This updates `dp[j+1]` to `min(dp[j+1], dp[i] + (j - i))`, where `j` is the next occurrence of the same element.

6. **Result Extraction**: After processing all elements, `dp[n]` contains the minimum operations needed to delete the entire sequence.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    t = int(data[0])
    index = 1
    results = []
    for _ in range(t):
        n = int(data[index]); index += 1
        a = list(map(int, data[index:index+n]))
        index += n
        
        next_occurrence = [n] * n
        last = [-1] * (n + 1)
        for i in range(n - 1, -1, -1):
            if last[a[i]] != -1:
                next_occurrence[i] = last[a[i]]
            last[a[i]] = i
        
        INF = 10**9
        dp = [INF] * (n + 1)
        dp[0] = 0
        for i in range(n):
            if dp[i] != INF:
                if dp[i + 1] > dp[i] + 1:
                    dp[i + 1] = dp[i] + 1
                j = next_occurrence[i]
                if j < n:
                    if dp[j + 1] > dp[i] + (j - i):
                        dp[j + 1] = dp[i] + (j - i)
        results.append(str(dp[n]))
    print("\n".join(results))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints.
2. **Precomputing Next Occurrences**: For each element in the sequence, we determine the next occurrence of the same value. This helps in identifying potential swaps to group identical elements.
3. **Dynamic Programming Initialization**: The `dp` array is initialized to store the minimum operations needed to delete the first `i` elements, starting with `dp[0] = 0`.
4. **Processing Each Element**: For each element, we consider two options:
   - **Delete Individually**: The cost is simply the previous cost plus one deletion.
   - **Group and Delete**: If a subsequent identical element exists, the cost is the previous cost plus the number of swaps needed to bring it adjacent to the current element, followed by a deletion.
5. **Result Extraction**: After processing all elements, `dp[n]` holds the answer for each test case, which is printed at the end.

This approach efficiently minimizes the operations by leveraging dynamic programming and precomputation of next occurrences, ensuring optimal performance even for large inputs.
