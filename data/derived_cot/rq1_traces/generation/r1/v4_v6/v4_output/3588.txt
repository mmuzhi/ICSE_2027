```python
            a_idx = char_to_idx[a_char]
            for last in range(3):
                for d_index in range(2 * n_val + 1):
                    cnt = dp_prev[last][d_index]
                    if cnt == 0:
                        continue
                    current_d = d_index - n_val
                    for nxt in range(3):
                        if nxt == last:
                            continue
                        d_inc = mat[nxt][a_idx]
                        new_d = current_d + d_inc
                        new_index = new_d + n_val
                        if 0 <= new_index < 2 * n_val + 1:
                            dp_curr[nxt][new_index] = (dp_curr[nxt][new_index] + cnt) % mod
            dp_prev = dp_curr
        
        ans = 0
        for last in range(3):
            for d_index in range(n_val + 1, 2 * n_val + 1):
                ans = (ans + dp_prev[last][d_index]) % mod
        return ans
```

### Explanation
1. **Initialization**: The DP table `dp_prev` is initialized to track counts of sequences for each possible move and point difference. The first round processes Alice's initial move, setting up initial counts for Bob's possible moves.
2. **DP Transition**: For each subsequent round, the DP table is updated (`dp_curr`) by considering all valid next moves (different from the last move) and updating the point difference based on the current round's outcome.
3. **Result Calculation**: After processing all rounds, the result is computed by summing counts from the DP table where the point difference (Bob's points minus Alice's points) is positive. This sum is returned modulo \(10^9 + 7\).

This approach efficiently counts all valid sequences while adhering to the constraints, leveraging dynamic programming to manage state transitions and point differences. The solution handles the problem within feasible computational limits for the given constraints.
