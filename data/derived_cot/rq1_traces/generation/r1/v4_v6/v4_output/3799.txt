=== ANSWER ===
To solve this problem, we need to determine the number of distinct three-digit even numbers that can be formed using digits from a given array. The solution must adhere to the following constraints:
1. The number must be three digits long, meaning it cannot have a leading zero.
2. The number must be even, so the last digit must be 0, 2, 4, 6, or 8.
3. Each digit in the array can be used only once per number, but if a digit appears multiple times in the array, it can be used as many times as it appears (though only once per number).

### Approach
1. **Generate Permutations**: We generate all possible permutations of three digits from the given array. This helps explore every possible combination of three digits.
2. **Filter Valid Numbers**: For each generated permutation:
   - Check if the first digit is not zero to avoid leading zeros.
   - Check if the last digit is even (0, 2, 4, 6, or 8).
3. **Store Distinct Numbers**: Convert valid permutations into three-digit integers and store them in a set to ensure uniqueness.
4. **Count Valid Numbers**: The size of the set at the end gives the count of distinct valid three-digit even numbers.

### Solution Code
```python
from itertools import permutations

class Solution:
    def totalNumbers(self, digits: list[int]) -> int:
        distinct_numbers = set()
        for p in permutations(digits, 3):
            if p[0] == 0:
                continue
            if p[2] % 2 == 0:
                num = p[0] * 100 + p[1] * 10 + p[2]
                distinct_numbers.add(num)
        return len(distinct_numbers)
```

### Explanation
1. **Permutations Generation**: Using `itertools.permutations`, we generate all possible three-digit combinations from the input array. This considers each digit's occurrence frequency, allowing digits appearing multiple times to be used accordingly.
2. **Validation Checks**: For each permutation:
   - The first digit is checked to ensure it is not zero, preventing leading zeros.
   - The last digit is checked for evenness (divisible by 2).
3. **Conversion and Storage**: Valid permutations are converted into three-digit integers (e.g., `(1, 2, 4)` becomes 124) and added to a set to automatically handle duplicates.
4. **Result Calculation**: The number of elements in the set after processing all permutations gives the count of distinct valid three-digit even numbers.

This approach efficiently explores all possible valid combinations while leveraging sets to handle uniqueness, ensuring correctness and optimal performance given the problem constraints.
