To solve this problem, we need to find the minimum number of operations required to follow a sequence of instructions on a ring-shaped structure. The ring consists of N parts, and each instruction specifies which hand (left or right) should move to a specific part. The challenge is to compute the minimal operations while adhering to the constraints that hands cannot occupy the same part at any time.

### Approach
1. **Problem Analysis**: The problem involves moving two hands (left and right) on a circular ring of N parts. Initially, the left hand is at part 1 and the right hand at part 2. Each instruction requires moving one of the hands to a specified part, possibly moving the other hand as well, but ensuring they never occupy the same part. The goal is to minimize the total number of operations (movements) across all instructions.

2. **Key Insight**: For each instruction, the specified hand must reach the target part, while the other hand can be anywhere else. The solution involves dynamic programming (DP) where the state tracks the position of the free hand (the hand not specified in the instruction) after each instruction. 

3. **Dynamic Programming Setup**: 
   - **Initial State**: The initial positions are left hand at 1 and right hand at 2.
   - **DP State**: For each instruction, the DP state is an array where each entry represents the minimal operations required to have the free hand at that position. For example, if the instruction is for the left hand, the DP state tracks the right hand's position.

4. **Transition Between States**: For each instruction, we compute the cost to move from the previous state to the new state. The cost is calculated considering two options:
   - Move the specified hand first to its target, then move the free hand.
   - Move the free hand first, then move the specified hand to its target.
   The minimal cost between these two options is chosen for each possible position of the free hand.

5. **Avoid Function**: This function calculates the minimal moves to move a hand from part `a` to part `c` while avoiding part `x` (the other hand's position). It checks if the minimal path (arc) between `a` and `c` includes `x`. If it does, the hand must take the longer path around the ring.

6. **Complexity**: The approach involves O(N^2) operations per instruction, leading to an overall complexity of O(Q * N^2). Given the constraints (N, Q ≤ 3000), this results in 27 billion operations, which is feasible with optimizations.

def main():
    data = sys.stdin.read().split()
    if not data:

    n = int(data[0])
    q = int(data[1])
    instructions = []
    idx = 2
    for _ in range(q):
        h = data[idx]
        t = int(data[idx + 1])
        instructions.append((h, t))
        idx += 2

    def is_on_minimal_arc(a, c, x, N=n):
        if a == c:
            return False
        if a <= c:
            clockwise_dist = c - a
            counter_dist = a + N - c
        else:
            clockwise_dist = c + N - a
            counter_dist = a - c
        
        if clockwise_dist <= counter_dist:
            if a <= c:
                return a <= x <= c
            else:
                return x >= a or x <= c
        else:
            if a >= c:
                return c <= x <= a
            else:
                return x <= a or x >= c

    def avoid(a, c, x, N=n):
        if a == c:
            return 0
        min_dist = min(abs(a - c), N - abs(a - c))
        if is_on_minimal_arc(a, c, x, N):
            return N - min_dist
        else:
            return min_dist

    INF = 10**18
    dp_prev = None
    hand_prev = None

    for i, (H, T) in enumerate(instructions):

                dp = [INF] * (n + 1)

                    cost = min(

        else:
            if H == 'L':
                new_dp = [INF] * (n + 1)
                for prev_pos in range(1, n + 1):

2. **Dynamic Programming Setup**: The DP array is initialized based on the first instruction. For each subsequent instruction, the DP array is updated by considering all possible positions of the free hand (the hand not specified in the instruction).
3. **State Transition**: For each instruction, the cost to move from the previous state to the new state is computed by considering two options: moving the specified hand first or moving the free hand first. The minimal cost is chosen for each position of the free hand.
4. **Result Calculation**: After processing all instructions, the minimal value in the DP array gives the answer, representing the least number of operations needed to follow all instructions. 

This approach efficiently handles the constraints by leveraging dynamic programming and careful cost calculation for hand movements on the ring.
