=== ANSWER ===
To solve this problem, we need to compute the sum of distinct values in all possible contiguous subarrays of a given sequence of integers. The straightforward approach of enumerating every subarray and counting distinct values would be too slow for large inputs due to its O(N^2) complexity. Instead, we use a more efficient method by leveraging the concept of complementary counting.

### Approach
1. **Problem Analysis**: The problem requires summing the number of distinct values in every contiguous subarray of the given sequence. A direct approach would involve generating all subarrays, which is computationally expensive for large N (up to 200,000). Instead, we can reframe the problem: for each distinct value in the sequence, determine how many subarrays contain at least one occurrence of that value. The sum of these counts across all distinct values gives the desired result.

2. **Insight**: The total number of subarrays is \( \frac{N(N+1)}{2} \). For each distinct value \( v \), the number of subarrays that do not contain \( v \) can be calculated by examining the gaps between consecutive occurrences of \( v \). Specifically, these gaps are segments before the first occurrence, between consecutive occurrences, and after the last occurrence of \( v \). The number of subarrays within each gap is given by the triangular number \( \frac{L(L+1)}{2} \), where \( L \) is the length of the gap.

3. **Algorithm**:
   - **Total Subarrays Calculation**: Compute the total number of subarrays as \( \frac{N(N+1)}{2} \).
   - **Position Tracking**: For each value in the sequence, record the indices where it occurs.
   - **Gap Calculation**: For each value, compute the gaps (segments without the value) and sum the number of subarrays within these gaps.
   - **Complementary Counting**: For each value, subtract the number of subarrays without any occurrence of the value from the total subarrays to get the count of subarrays containing the value. Sum these counts across all values.

4. **Complexity Analysis**: The algorithm processes each element once to build the position dictionary and then processes each occurrence of each value once. The total operations are linear in the size of the input, O(N), making it efficient for the given constraints.

### Solution Code
```python
def main():
    n = int(input().strip())
    A = list(map(int, input().split()))
    
    total_intervals = n * (n + 1) // 2
    pos_dict = {}
    for idx, val in enumerate(A):
        if val not in pos_dict:
            pos_dict[val] = []
        pos_dict[val].append(idx)
    
    ans = 0
    for positions in pos_dict.values():
        gaps_sum = 0
        first_gap = positions[0]
        gaps_sum += first_gap * (first_gap + 1) // 2
        
        for i in range(1, len(positions)):
            gap_len = positions[i] - positions[i-1] - 1
            gaps_sum += gap_len * (gap_len + 1) // 2
        
        last_gap = n - positions[-1] - 1
        gaps_sum += last_gap * (last_gap + 1) // 2
        
        ans += total_intervals - gaps_sum
    
    print(ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Input Handling**: The input consists of an integer \( N \) followed by a sequence of \( N \) integers.
2. **Total Subarrays Calculation**: The total number of contiguous subarrays in an array of length \( N \) is \( \frac{N(N+1)}{2} \).
3. **Position Tracking**: A dictionary maps each distinct value to a list of indices where it appears in the sequence.
4. **Gap Calculation**: For each distinct value, the algorithm calculates the subarrays that do not contain the value by summing subarrays in gaps before the first occurrence, between consecutive occurrences, and after the last occurrence. The number of subarrays in a gap of length \( L \) is \( \frac{L(L+1)}{2} \).
5. **Result Calculation**: For each value, the number of subarrays containing it is the total subarrays minus the subarrays without it. Summing these values across all distinct values gives the final result.
6. **Output**: The computed result is printed.

This approach efficiently computes the solution by leveraging complementary counting and linear processing, making it suitable for large input sizes.
