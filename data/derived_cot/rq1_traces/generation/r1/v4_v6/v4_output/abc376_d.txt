There is a simple directed graph with N vertices numbered from 1 to N and M edges. The i-th edge (1 \leq i \leq M) is a directed edge from vertex a_i to vertex b_i.
Determine whether there exists a cycle that contains vertex 1, and if it exists, find the minimum number of edges among such cycles.

Input

The input is given from Standard Input in the following format:
N M
a_1 b_1
a_2 b_2
\vdots
a_M b_M

Output

If there exists a cycle that contains vertex 1, print the minimum number of edges among such cycles. Otherwise, print -1.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq M \leq \min \left( \frac{N(N-1)}{2},\ 2 \times 10^5 \right)
- 1 \leq a_i \leq N
- 1 \leq b_i \leq N
- a_i \neq b_i
- (a_i, b_i) \neq (a_j, b_j) and (a_i, b_i) \neq (b_j, a_j), if i \neq j.
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to determine whether there exists a cycle in a directed graph that contains vertex 1, and if such a cycle exists, find the minimum number of edges among all such cycles. 

### Approach
1. **Problem Analysis**: The problem involves checking for cycles in a directed graph that include a specific vertex (vertex 1). The key insight is that any cycle containing vertex 1 can be decomposed into two parts: a path from vertex 1 to some vertex \( v \), and a path from \( v \) back to vertex 1. The total number of edges in such a cycle is the sum of the edges in these two paths.
2. **Graph Representation**: We use two adjacency lists to represent the graph: one for the original directed graph (forward graph) and another for the reversed graph (backward graph). The reversed graph helps in efficiently computing the shortest paths from any vertex back to vertex 1.
3. **BFS for Shortest Paths**: We perform a Breadth-First Search (BFS) on the forward graph to compute the shortest distances from vertex 1 to all other vertices. Similarly, we perform BFS on the reversed graph to compute the shortest distances from all vertices back to vertex 1. The reversed graph allows us to treat the problem of finding paths back to vertex 1 as finding paths from vertex 1 in the reversed graph.
4. **Cycle Detection**: For each vertex \( v \) (excluding vertex 1), we check if both the forward distance (from 1 to \( v \)) and the backward distance (from \( v \) to 1) are finite. If they are, the sum of these distances gives the number of edges in a cycle containing vertex 1 and \( v \). The minimum such sum across all vertices \( v \) is our answer. If no such vertex exists, we return -1.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    graph_forward = [[] for _ in range(n+1)]
    graph_backward = [[] for _ in range(n+1)]
    
    index = 2
    for _ in range(m):
        a = int(data[index])
        b = int(data[index+1])
        index += 2
        graph_forward[a].append(b)
        graph_backward[b].append(a)
    
    dist1 = [10**9] * (n+1)
    q = deque()
    dist1[1] = 0
    q.append(1)
    while q:
        cur = q.popleft()
        for nxt in graph_forward[cur]:
            if dist1[nxt] > dist1[cur] + 1:
                dist1[nxt] = dist1[cur] + 1
                q.append(nxt)
    
    dist2 = [10**9] * (n+1)
    q = deque()
    dist2[1] = 0
    q.append(1)
    while q:
        cur = q.popleft()
        for nxt in graph_backward[cur]:
            if dist2[nxt] > dist2[cur] + 1:
                dist2[nxt] = dist2[cur] + 1
                q.append(nxt)
    
    ans = 10**9
    for v in range(2, n+1):
        if dist1[v] != 10**9 and dist2[v] != 10**9:
            ans = min(ans, dist1[v] + dist2[v])
    
    print(-1 if ans == 10**9 else ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are the number of vertices \( N \) and edges \( M \). The subsequent lines represent the directed edges.
2. **Graph Construction**: Two adjacency lists are built: `graph_forward` for the original directed graph and `graph_backward` for the reversed graph (where each edge direction is inverted).
3. **BFS for Forward Distances**: Using BFS starting from vertex 1, we compute the shortest distances (`dist1`) from vertex 1 to all other vertices in the original graph.
4. **BFS for Backward Distances**: Similarly, using BFS on the reversed graph starting from vertex 1, we compute the shortest distances (`dist2`) from all vertices back to vertex 1 in the original graph. This is equivalent to finding the shortest path from each vertex to vertex 1 in the original graph.
5. **Finding Minimum Cycle**: For each vertex \( v \) (from 2 to \( N \)), if both `dist1[v]` (distance from 1 to \( v \)) and `dist2[v]` (distance from \( v \) to 1) are finite, the sum of these distances gives the number of edges in a cycle containing vertex 1 and \( v \). The minimum of these sums across all vertices is the answer. If no such cycle exists, -1 is printed. 

This approach efficiently checks for cycles containing vertex 1 by leveraging BFS on both the original and reversed graphs, ensuring optimal performance even for large graphs.
