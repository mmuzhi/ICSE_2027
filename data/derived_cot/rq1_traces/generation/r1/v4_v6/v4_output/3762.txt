          n = len(points)
          if m < n:
              return 0
          k = m - n  # extra moves, which is extra landings
          low, high = 0, 10**18
          while low < high:
              mid = (low + high + 1) // 2
              total_extra = 0
              for p in points:
                  if mid > p:
                      # extra = ceil(mid / p) - 1
                      extra = (mid + p - 1) // p - 1
                      total_extra += extra
                      if total_extra > k:
                          break
              if total_extra <= k:
                  low = mid
              else:
                  high = mid - 1
          return low

To solve this problem, we need to maximize the minimum value in the `gameScore` array after performing at most `m` moves. The `gameScore` array starts with all zeros, and each move involves moving to an adjacent index (either increasing or decreasing the current index by 1) and adding the points value of that index to the corresponding `gameScore`. The challenge is to distribute the moves such that the minimum value in `gameScore` is as large as possible.

### Approach
1. **Initial Check**: If the number of moves `m` is less than the length of the array `n`, it's impossible to cover all indices at least once, so the minimum value will remain 0. Hence, return 0 in this case.
2. **Binary Search Setup**: We use binary search to determine the maximum possible minimum value (let's call it `x`). The search range is set from 0 to a sufficiently large value (10^18).
3. **Feasibility Check**: For each candidate `x` during the binary search, we calculate the number of extra landings (visits) required for each index to ensure that the product of the number of landings and the points value at that index is at least `x`. Specifically, for each index `i`, the required extra landings are computed as `max(0, ceil(x / points[i]) - 1)`.
4. **Sum Check**: The sum of these extra landings across all indices must not exceed the available extra moves (`k = m - n`). If it does, the candidate `x` is not feasible; otherwise, it is feasible.
5. **Binary Search Execution**: Adjust the binary search bounds based on the feasibility check. If the total extra landings required for `x` is within the available extra moves, we try to find a larger feasible `x`; otherwise, we search for a smaller `x`.

### Solution Code
```python
class Solution:
    def maxScore(self, points: List[int], m: int) -> int:
        n = len(points)
        if m < n:
            return 0
        k = m - n
        low, high = 0, 10**18
        while low < high:
            mid = (low + high + 1) // 2
            total_extra = 0
            for p in points:
                if mid > p:
                    extra = (mid + p - 1) // p - 1
                    total_extra += extra
                    if total_extra > k:
                        break
            if total_extra <= k:
                low = mid
            else:
                high = mid - 1
        return low
```

### Explanation
1. **Initial Check**: The first condition checks if the number of moves `m` is sufficient to cover all indices at least once. If not, the minimum value remains 0.
2. **Binary Search Setup**: The binary search is initialized with `low` as 0 and `high` as a large value (10^18) to cover all possible minimum values.
3. **Feasibility Check**: For each midpoint `mid` in the binary search, the algorithm calculates the number of extra landings needed for each index to ensure that `(1 + extra_landings) * points[i] >= mid`. This is done using the formula `(mid + points[i] - 1) // points[i] - 1`.
4. **Sum Check**: The total extra landings required across all indices is summed up. If this sum is within the available extra moves (`k`), the candidate `mid` is feasible, and the search continues for a higher value; otherwise, the search continues for a lower value.
5. **Result**: The binary search converges to the maximum feasible minimum value, which is returned as the result.

This approach efficiently narrows down the possible minimum values using binary search, ensuring optimal performance even for large input sizes.
