You will be given a competitive programming problem.
You are given a directed graph with N vertices and M edges.
The i-th edge (1 \leq i \leq M) is a directed edge from vertex u _ i to vertex v _ i.
Initially, you are at vertex 1. You want to repeat the following operations until you reach vertex N:

- Perform one of the two operations below:
- Move along a directed edge from your current vertex. This incurs a cost of 1. More precisely, if you are at vertex v, choose a vertex u such that there is a directed edge from v to u, and move to vertex u.
- Reverse the direction of all edges. This incurs a cost of X. More precisely, if and only if there was a directed edge from v to u immediately before this operation, there is a directed edge from u to v immediately after this operation.



It is guaranteed that, for the given graph, you can reach vertex N from vertex 1 by repeating these operations.
Find the minimum total cost required to reach vertex N.

Input

The input is given from Standard Input in the following format:
N M X
u _ 1 v _ 1
u _ 2 v _ 2
\vdots
u _ M v _ M

Output

Print the minimum total cost required to reach vertex N.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq M \leq 2 \times 10^5
- 1 \leq X \leq 10^9
- 1 \leq u _ i \leq N \ (1 \leq i \leq M)
- 1 \leq v _ i \leq N \ (1 \leq i \leq M)
- For the given graph, it is guaranteed that you can reach vertex N from vertex 1 by the operations described.
- All input values are integers.

To solve this problem, we need to find the minimum total cost required to travel from vertex 1 to vertex N in a directed graph, where we can either traverse along an edge (cost 1) or reverse the direction of all edges (cost X). The challenge is to model the problem efficiently, considering the two possible states of the graph (original or reversed) and transitions between them.

### Approach
1. **Problem Analysis**: The problem involves navigating a directed graph where the direction of all edges can be reversed at any vertex, incurring a cost X. Each traversal along an edge costs 1. The goal is to reach vertex N with minimal total cost, starting from vertex 1 in the original graph state.
2. **State Modeling**: We model the problem using two states for each vertex: 
   - **State 0**: The graph is in its original direction.
   - **State 1**: The graph has all edges reversed.
3. **Graph Representation**: We maintain two adjacency lists:
   - `adj0` for the original graph (edges from u to v).
   - `adj1` for the reversed graph (edges from v to u, derived from original edges).
4. **Dijkstra's Algorithm**: We use Dijkstra's algorithm to find the shortest path in this state-augmented graph. Each node in the priority queue is represented as `(cost, vertex, mode)`, where `mode` indicates the current state of the graph (0 or 1).
5. **Transitions**:
   - **Traversal**: In state 0, traverse edges from `adj0`; in state 1, traverse edges from `adj1`.
   - **Reversal**: From any vertex in either state, reverse the graph (toggle state) with cost X.
6. **Termination**: The algorithm terminates when we reach vertex N in either state, and the minimum cost between the two states is the answer.
