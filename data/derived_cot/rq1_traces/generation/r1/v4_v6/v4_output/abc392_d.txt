=== PROMPT ===
You will be given a competitive programming problem.
There are N dice.
The i-th die has K_i faces, with the numbers A_{i,1}, A_{i,2}, \ldots, A_{i,K_i} written on them.
When you roll this die, each face appears with probability \frac{1}{K_i}.
You choose two dice from the N dice and roll them.
Determine the maximum probability that the two dice show the same number, when the dice are chosen optimally.

Input

The input is given from Standard Input in the following format:
N
K_1 A_{1,1} A_{1,2} \dots A_{1,K_1}
\vdots
K_N A_{N,1} A_{N,2} \dots A_{N,K_N}

Output

Print the answer.
Your answer is considered correct if the absolute or relative error from the true solution does not exceed 10^{-8}.

Constraints


- 2 \leq N \leq 100
- 1 \leq K_i
- K_1 + K_2 + \dots + K_N \leq 10^5
- 1 \leq A_{i,j} \leq 10^5
- All input values are integers.

Sample Input 1

3
3 1 2 3
4 1 2 2 1
6 1 2 3 4 5 6

Sample Output 1

0.333333333333333


- When choosing the 1st and 2nd dice, the probability that the outcomes are the same is \frac{1}{3}.
- When choosing the 1st and 3rd dice, the probability is \frac{1}{6}.
- When choosing the 2nd and 3rd dice, the probability is \frac{1}{6}.

Therefore, the maximum probability is \frac{1}{3} = 0.3333333333\ldots.

To solve this problem, we need to determine the maximum probability that two dice show the same number when chosen optimally from a given set of dice. The solution involves efficiently calculating the inner product of frequency vectors for each pair of dice to find the pair with the highest probability of showing the same number.

### Approach
1. **Problem Analysis**: 
   - We are given `N` dice, each with `K_i` faces. Each face has a number between 1 and 100,000.
   - The goal is to choose two dice such that the probability of rolling the same number on both dice is maximized.
   - The probability for a pair of dice (i, j) is calculated as the sum over all numbers of the product of their frequencies in each die, divided by the product of the number of faces in each die (i.e., `(sum_{v} count_i(v) * count_j(v)) / (K_i * K_j)`).

2. **Intuition**:
   - For each number `v` that appears in at least two dice, we can compute the contribution of `v` to the inner product of every pair of dice that contain `v`.
   - By iterating over all numbers from 1 to 100,000, we can accumulate these contributions into a matrix where `mat[i][j]` represents the sum of `count_i(v) * count_j(v)` for all numbers `v` common to dice `i` and `j`.

3. **Algorithm Selection**:
   - **Frequency Per Number**: For each number `v` (1 to 100,000), maintain a list of tuples `(die_index, frequency)` indicating which dice contain `v` and how many times.
   - **Matrix Initialization**: Initialize an `N x N` matrix to zero to store the inner products for each pair of dice.
   - **Matrix Population**: For each number `v`, iterate over all pairs of dice in its list, compute the product of their frequencies, and add this product to the corresponding entries in the matrix.
   - **Probability Calculation**: For each pair of dice `(i, j)`, compute the probability as `mat[i][j] / (K_i * K_j)` and track the maximum probability encountered.

4. **Complexity Analysis**:
   - **Time Complexity**: The dominant operation is processing each number from 1 to 100,000. For each number, if it appears in `m` dice, we process `O(m^2)` pairs. In the worst case, `m` can be 100 (since `N` is 100), leading to 100,000 * 4950 ≈ 49.5 million operations, which is feasible within the constraints.
   - **Space Complexity**: The space required is `O(N^2 + 100,000)` for the matrix and frequency lists, which is manageable given the constraints.

### Solution Code

import sys
from collections import Counter

def main():
    data = sys.stdin.read().splitlines()
    n = int(data[0])
    freq_per_num = [[] for _ in range(100001)]
    sizes = []
    
    for i in range(n):
        parts = data[1+i].split()
        k = int(parts[0])
        sizes.append(k)
        nums = list(map(int, parts[1:1+k]))
        cnt = Counter(nums)
        for num, count_val in cnt.items():
            if 1 <= num <= 100000:
                freq_per_num[num].append((i, count_val))
    
    mat = [[0] * n for _ in range(n)]
    
    for v in range(1, 100001):
        L = freq_per_num[v]
        m = len(L)
        if m < 2:
            continue
        for i_idx in range(m):
            d1, c1 = L[i_idx]
            for j_idx in range(i_idx + 1, m):
                d2, c2 = L[j_idx]
                product = c1 * c2
                mat[d1][d2] += product
                mat[d2][d1] += product
    
    max_prob = 0.0
    for i in range(n):
        for j in range(i + 1, n):
            prob = mat[i][j] / (sizes[i] * sizes[j])
            if prob > max_prob:
                max_prob = prob
    
    print("{:.15f}".format(max_prob))

if __name__ == "__main__":
    main()
