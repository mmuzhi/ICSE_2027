No bridge connects an island to itself, but it is possible for two islands to be directly connected by more than one bridge.
One can travel between any two islands using some bridges.
You are given Q queries, so answer each of them. The i-th query is as follows:

You are given K_i distinct bridges: bridges B_{i,1}, B_{i,2}, \ldots, B_{i,K_i}.
Find the minimum time required to travel from island 1 to island N using each of these bridges at least once.
Only consider the time spent crossing bridges.
You can cross the given bridges in any order and in any direction.

Input

Constraints


- 2 \leq N \leq 400

- 1 \leq Q \leq 3000
- 1 \leq K_i \leq 5

- All input values are integers.

To solve this problem, we need to find the minimum time required to travel from island 1 to island N while using each of the given bridges at least once. The solution involves leveraging graph algorithms and dynamic programming to efficiently handle multiple queries, especially considering that each query can specify up to 5 bridges that must be used.

### Approach
1. **Graph Representation**: Represent the islands and bridges as an undirected graph with weighted edges (bridges). Each bridge connects two islands with a given crossing time.
2. **Preprocessing with Floyd-Warshall**: Since the number of islands (N) is up to 400, we can use the Floyd-Warshall algorithm to compute all-pairs shortest paths. This allows us to quickly determine the shortest path between any two islands without the constraints of the required bridges.
3. **Handling Queries**: For each query, we are given a set of bridges that must be used. The key insight is that the problem reduces to finding a path from island 1 to island N that includes each specified bridge at least once. The path can be decomposed into segments where the required bridges are traversed in some order, and the segments between these bridges are the shortest paths.
4. **Dynamic Programming with State Masking**: For each query, we consider all permutations of the required bridges. For each permutation, we simulate traversing the bridges in order, and the segments between consecutive bridges (or from the start to the first bridge, between bridges, and from the last bridge to the end) are taken as the shortest paths (computed by Floyd-Warshall). We use dynamic programming to track the minimum time for traversing a subset of the required bridges, ending at a particular island (the start or end of a bridge).
5. **Bridge Representation**: Each bridge is represented by its two endpoints and the crossing time. When traversing a bridge, we must consider both directions. Thus, for each bridge, there are two possible directions (from u to v or v to u).

### Solution Code
```python
import sys
import itertools

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    m = int(data[1])
    edges = []
    index = 2
    graph = [[10**18] * (n+1) for _ in range(n+1)]
    for i in range(n+1):
        graph[i][i] = 0
        
    for _ in range(m):
        u = int(data[index])
        v = int(data[index+1])
        t = int(data[index+2])
        index += 3
        edges.append((u, v, t))
        if t < graph[u][v]:
            graph[u][v] = t
            graph[v][u] = t
    
    for k in range(1, n+1):
        for i in range(1, n+1):
            if graph[i][k] == 10**18:
                continue
            for j in range(1, n+1):
                if graph[i][j] > graph[i][k] + graph[k][j]:
                    graph[i][j] = graph[i][k] + graph[k][j]
    
    q = int(data[index])
    index += 1
    out_lines = []
    
    for _ in range(q):
        k = int(data[index])
        index += 1
        bridge_indices = list(map(int, data[index:index+k]))
        index += k
        
        req_edges = []
        for bid in bridge_indices:
            u, v, t = edges[bid-1]
            req_edges.append((u, v, t))
        
        if k == 0:
            out_lines.append(str(graph[1][n]))
            continue
        
        dp = {}
        for mask in range(1 << k):
            dp[mask] = [10**18] * 2
            for each_edge in range(k):
                dp[mask][each_edge] = [10**18] * 2
        
        for i in range(k):
            u, v, t = req_edges[i]
            dp[1 << i][0][0] = graph[1][u] + t
            dp[1 << i][0][1] = graph[1][v] + t
            dp[1 << i][1][0] = graph[1][u]
            dp[1 << i][1][1] = graph[1][v]
        
        for mask in range(1, 1 << k):
            for i in range(k):
                if not (mask & (1 << i)):
                    continue
                for j in range(k):
                    if mask & (1 << j):
                        continue
                    u_i, v_i, t_i = req_edges[i]
                    u_j, v_j, t_j = req_edges[j]
                    for end_i in [u_i, v_i]:
                        for start_j in [u_j, v_j]:
                            cost = graph[end_i][start_j]
                            new_mask = mask | (1 << j)
                            for state in [0, 1]:
                                if state == 0:
                                    base = dp[mask][0][0] if end_i == u_i else dp[mask][0][1]
                                    new_val = base + cost + t_j
                                    if new_val < dp[new_mask][0][0]:
                                        dp[new_mask][0][0] = new_val
                                    if new_val < dp[new_mask][0][1]:
                                        dp[new_mask][0][1] = new_val
                                else:
                                    base = dp[mask][1][0] if end_i == u_i else dp[mask][1][1]
                                    new_val = base + cost
                                    if new_val < dp[new_mask][1][0]:
                                        dp[new_mask][1][0] = new_val
                                    if new_val < dp[new_mask][1][1]:
                                        dp[new_mask][1][1] = new_val
        
        full_mask = (1 << k) - 1
        ans = 10**18
        for i in range(k):
            u, v, t = req_edges[i]
            for end in [u, v]:
                for state in [0, 1]:
                    base = dp[full_mask][state][0] if end == u else dp[full_mask][state][1]
                    if state == 0:
                        total = base + graph[end][n]
                    else:
                        total = base + t + graph[end][n]
                    if total < ans:
                        ans = total
        out_lines.append(str(ans))
    
    print("\n".join(out_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Graph Initialization**: The graph is initialized with large values (10^18) to represent infinity, and the adjacency matrix is built from the input edges.
2. **Floyd-Warshall Algorithm**: This algorithm computes the shortest paths between all pairs of islands, storing the results in a matrix `graph`.
3. **Query Processing**: For each query, the required bridges are identified. If no bridges are specified, the answer is simply the shortest path from island 1 to island N.
4. **Dynamic Programming Setup**: The DP table `dp` is initialized to store the minimum time for each subset of bridges (represented by a bitmask) and the state (whether the last bridge was fully traversed or not).
5. **DP Transitions**: For each subset of bridges, transitions are computed by considering the addition of a new bridge. The DP state keeps track of the current position (either end of a bridge) and whether the bridge has been traversed completely.
6. **Result Calculation**: After processing all subsets, the answer for the query is derived by considering the full set of bridges and adding the shortest path from the last bridge's endpoint to island N.

This approach efficiently handles the constraints by leveraging precomputed shortest paths and dynamic programming to manage the combinatorial aspects of the required bridges.
