=== ANSWER ===
To solve this problem, we need to find the minimum number of operations required to have at least `k` non-overlapping subarrays of size exactly `x` in the given array `nums`, where all elements within each subarray are equal. The operations allowed are increasing or decreasing any element by 1, with each operation counting as one step.

### Approach
1. **Problem Analysis**: The problem involves selecting `k` non-overlapping contiguous subarrays of size `x` from the array `nums`. For each selected subarray, we need to make all elements equal with the minimum number of operations. The cost to make all elements in a subarray equal is the sum of absolute differences between each element and the median of the subarray. However, we can compute this cost efficiently using the formula: `total_sum - sum_of_smallest_m_elements - sum_of_smallest_(x - m)_elements`, where `m = x // 2`.

2. **Sliding Window with Fenwick Trees**: To efficiently compute the cost for each possible subarray of size `x`, we use a sliding window approach combined with Fenwick Trees (Binary Indexed Trees). The Fenwick Trees help maintain the frequencies and sums of elements in the current window, allowing us to quickly compute the sum of the smallest `m` and `x - m` elements.

3. **Dynamic Programming (DP)**: After computing the costs for all possible subarrays, we use dynamic programming to select `k` non-overlapping subarrays with the minimum total cost. The DP state `dp[i][j]` represents the minimum cost to select `j` subarrays starting from index `i` onwards. The recurrence relation is:
   - **Option 1**: Skip the current subarray starting at `i`, i.e., `dp[i][j] = dp[i+1][j]`.
   - **Option 2**: Take the current subarray starting at `i`, i.e., `dp[i][j] = cost[i] + dp[i+x][j-1]`.

```python
import bisect

class Fenw:
    def __init__(self, n):
        self.n = n
        self.tree = [0] * (n + 1)
    
    def update(self, idx, delta):
        while idx <= self.n:
            self.tree[idx] += delta
            idx += idx & -idx
    
    def query(self, idx):
        s = 0
        while idx:
            s += self.tree[idx]
            idx -= idx & -idx
        return s

class Solution:
    def minOperations(self, nums, x, k):
        n = len(nums)
        vals = sorted(set(nums))
        size = len(vals)
        m_val = x // 2
        
        seg_costs = [0] * (n - x + 1)
        
        if n - x + 1 <= 0:
            return 0
        
        freq_tree = Fenw(size)
        sum_tree = Fenw(size)
        total_sum = 0
        
        def add_val(v):
            nonlocal total_sum
            idx = bisect.bisect_left(vals, v)
            freq_tree.update(idx + 1, 1)
            sum_tree.update(idx + 1, v)
            total_sum += v
        
        def remove_val(v):
            nonlocal total_sum
            idx = bisect.bisect_left(vals, v)
            freq_tree.update(idx + 1, -1)
            sum_tree.update(idx + 1, -v)
            total_sum -= v
        
        def query_smallest(t):
            if t == 0:
                return 0
            low, high = 1, size
            while low < high:
                mid = (low + high) // 2
                cnt = freq_tree.query(mid)
                if cnt < t:
                    low = mid + 1
                else:
                    high = mid
            cnt_before = freq_tree.query(low - 1)
            need = t - cnt_before
            s = sum_tree.query(low - 1) + need * vals[low - 1]
            return s
        
        for i in range(x):
            add_val(nums[i])
        seg_costs[0] = total_sum - query_smallest(m_val) - query_smallest(x - m_val)
        
        for i in range(1, n - x + 1):
            remove_val(nums[i - 1])
            add_val(nums[i + x - 1])
            seg_costs[i] = total_sum - query_smallest(m_val) - query_smallest(x - m_val)
        
        INF = 10**18
        dp = [[INF] * (k + 1) for _ in range(n + 1)]
        for i in range(n + 1):
            dp[i][0] = 0
        
        for i in range(n - 1, -1, -1):
            for j in range(1, k + 1):
                dp[i][j] = dp[i + 1][j]
                if i <= n - x:
                    dp[i][j] = min(dp[i][j], seg_costs[i] + dp[i + x][j - 1])
        
        return dp[0][k]
```

### Explanation
1. **Fenwick Tree (Fenw)**: This class is used to maintain frequencies and sums of elements in the current window. It supports point updates and prefix sum queries efficiently.
2. **Sliding Window**: The window slides over the array to consider each possible subarray of size `x`. For each window, we compute the cost to make all elements equal using the formula involving the sum of the smallest `m` and `x - m` elements, where `m = x // 2`.
3. **Dynamic Programming**: The DP array `dp[i][j]` is filled in a bottom-up manner. For each position `i`, we decide whether to skip the current subarray or include it, updating the DP state accordingly.
4. **Result Extraction**: The answer is found in `dp[0][k]`, representing the minimum operations to form `k` non-overlapping subarrays starting from the beginning of the array.

This approach efficiently combines sliding window techniques with Fenwick Trees for dynamic range queries and dynamic programming to solve the problem within feasible time complexity.
