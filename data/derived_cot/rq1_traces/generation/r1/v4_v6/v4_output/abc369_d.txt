- If he lets a monster go, he gains 0 experience points.
- If he defeats a monster with strength X, he gains X experience points.
  If it is an even-numbered defeated monster (2nd, 4th, ...), he gains an additional X experience points.

Find the maximum total experience points he can gain from the N monsters.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the maximum total experience points he can gain from the N monsters as an integer.

Constraints


- 1\leq N\leq 2\times 10^5
- 1\leq A_i\leq 10^9
- All input values are integers.

Sample Input 1

5

28

=== ANSWER ===
To solve this problem, we need to maximize the total experience points Takahashi gains by strategically choosing which monsters to defeat or let go. The experience points are calculated based on the monsters' strengths and their positions in the sequence of defeated monsters. Specifically, defeating a monster grants its base strength as experience points, and every even-indexed defeat (2nd, 4th, etc.) grants an additional bonus equal to the monster's strength.

### Approach
1. **Problem Analysis**: The problem requires selecting a subsequence of monsters to defeat in the order they are encountered. The key challenge is to maximize the total experience points, which includes both the sum of the strengths of all defeated monsters and the sum of the strengths of monsters defeated at even positions in the subsequence.
2. **Dynamic Programming (DP) Setup**: We use a dynamic programming approach with two states:
   - `dp0`: The maximum experience points achievable with an even number of defeats so far (next defeat will be odd-indexed).
   - `dp1`: The maximum experience points achievable with an odd number of defeats so far (next defeat will be even-indexed).
3. **State Transitions**: For each monster, we consider two choices:
   - **Skip the monster**: The states remain unchanged.
   - **Defeat the monster**: 
     - If the current state is `dp0` (even defeats), defeating the monster adds its strength once (since the next defeat is odd-indexed) and transitions to `dp1`.
     - If the current state is `dp1` (odd defeats), defeating the monster adds twice its strength (since the next defeat is even-indexed) and transitions to `dp0`.
4. **Initialization**: Start with `dp0 = 0` (no defeats) and `dp1 = -infinity` (impossible state initially).
5. **Result Calculation**: After processing all monsters, the result is the maximum value between `dp0` and `dp1`.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    dp0 = 0
    dp1 = -10**18
    
    for a in A:
        new_dp0 = max(dp0, dp1 + 2 * a)
        new_dp1 = max(dp1, dp0 + a)
        dp0, dp1 = new_dp0, new_dp1
    
    print(max(dp0, dp1))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first value is the number of monsters `N`, followed by `N` integers representing the monsters' strengths.
2. **Initialization**: `dp0` is initialized to 0 (representing 0 defeats), and `dp1` is set to a very large negative value to indicate an initially impossible state (odd number of defeats with no monsters defeated).
3. **Processing Monsters**: For each monster:
   - **New States Calculation**: 
     - `new_dp0` is updated by taking the maximum of skipping the monster (`dp0`) or defeating it when the previous state was `dp1` (odd defeats), which transitions to an even state by adding twice the monster's strength.
     - `new_dp1` is updated by taking the maximum of skipping the monster (`dp1`) or defeating it when the previous state was `dp0` (even defeats), which transitions to an odd state by adding the monster's strength once.
   - **State Update**: The current states `dp0` and `dp1` are updated to `new_dp0` and `new_dp1` for the next iteration.
4. **Result Output**: After processing all monsters, the result is the maximum value between `dp0` and `dp1`, printed as the maximum experience points achievable.

This approach efficiently computes the optimal solution by leveraging dynamic programming to track the best possible states at each step, ensuring optimal performance even for large input sizes.
