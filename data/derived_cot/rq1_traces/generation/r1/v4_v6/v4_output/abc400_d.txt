To solve this problem, we need to find the minimum number of front kicks Takahashi needs to perform to reach the fish shop from his starting position in a grid composed of roads ('.') and walls ('#'). The key challenge is navigating the grid efficiently by breaking walls with front kicks when necessary, while minimizing the number of such kicks.

### Approach
1. **Problem Analysis**: The grid consists of H rows and W columns. Takahashi starts at (A, B) and aims to reach (C, D). Both the start and end positions are roads. Takahashi can move to adjacent roads (up, down, left, right) without any cost. Alternatively, he can perform a front kick in any of the four directions, which converts walls up to 2 steps away in that direction into roads. Each front kick costs 1 unit.

2. **Intuition**: The problem resembles a shortest path problem where the cost is the number of front kicks. Moving through existing roads is cost-free, while breaking walls incurs a cost. The solution involves using Dijkstra's algorithm where each node (cell) is processed with a priority queue (min-heap) based on the number of kicks used to reach it.

3. **Algorithm Selection**: 
   - **Priority Queue (Min-Heap)**: To efficiently explore cells in order of increasing kick count.
   - **Distance Array**: Tracks the minimum kicks required to reach each cell.
   - **Grid Updates**: When a wall is broken (converted to a road), the grid is updated globally, and adjacent cells are reconsidered for movement.

4. **Key Steps**:
   - **Initialization**: Start at (A-1, B-1) with 0 kicks.
   - **Processing Moves**: For each cell, explore adjacent roads (cost remains the same).
   - **Processing Kicks**: For each direction, break walls up to 2 steps away. If the first step wall is broken, push the adjacent cell into the queue with an incremented kick count.
   - **Termination**: Return the kick count when the target cell (C-1, D-1) is reached.

5. **Complexity Analysis**: The algorithm processes each cell at most once per kick count, leading to O(H*W log(H*W)) time complexity due to the priority queue operations. The space complexity is O(H*W) for storing distances and the grid.

```python
        k, r, c = heapq.heappop(heap)
        if (r, c) == end:
            print(k)
            return
        
        if dist[r][c] < k:
            continue
        
        for dr, dc in dirs:
            nr = r + dr
            nc = c + dc
            if 0 <= nr < H and 0 <= nc < W and grid[nr][nc] == '.':
                if dist[nr][nc] > k:
                    dist[nr][nc] = k
                    heapq.heappush(heap, (k, nr, nc))
        
        for dr, dc in dirs:
            broke_step1 = False
            for step in [1, 2]:
                nr = r + dr * step
                nc = c + dc * step
                if 0 <= nr < H and 0 <= nc < W and grid[nr][nc] == '#':
                    grid[nr][nc] = '.'
                    if step == 1:
                        broke_step1 = True
            
            if broke_step1:
                nxt_r = r + dr
                nxt_c = c + dc
                if dist[nxt_r][nxt_c] > k + 1:
                    dist[nxt_r][nxt_c] = k + 1
                    heapq.heappush(heap, (k + 1, nxt_r, nxt_c))
    
    print(dist[end[0]][end[1]])

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The grid is read from input, and the start and end positions are adjusted to 0-based indexing.
2. **Distance Setup**: A 2D array `dist` is initialized to store the minimum kicks required to reach each cell, starting with `INF` (a large number) except for the start position (0 kicks).
3. **Priority Queue**: A min-heap is used to process cells in order of increasing kick count. The heap starts with the initial position.
4. **Processing Loop**:
   - **Move Handling**: For each cell, adjacent roads are explored. If moving to a road reduces the kick count, it is added to the heap.
   - **Kick Handling**: For each direction, walls up to 2 steps away are broken. If the first-step wall is broken, the adjacent cell is pushed into the heap with an incremented kick count.
5. **Termination**: The loop exits when the target cell is reached, printing the kick count. If the heap is exhausted without reaching the target, the distance to the target is printed (though the problem guarantees reachability). 

This approach efficiently navigates the grid while minimizing front kicks by leveraging Dijkstra's algorithm with dynamic grid updates.
