You will be given a competitive programming problem.
There are N buildings, building 1, building 2, \ldots, building N, arranged in this order in a straight line from west to east. Building 1 is the westernmost, and building N is the easternmost. The height of building i\ (1\leq i\leq N) is H_i.
For a pair of integers (i,j)\ (1\leq i\lt j\leq N), building j can be seen from building i if the following condition is satisfied.

- There is no building taller than building j between buildings i and j. In other words, there is no integer k\ (i\lt k\lt j) such that H_k > H_j.

You are given Q queries. In the i-th query, given a pair of integers (l_i,r_i)\ (l_i\lt r_i), find the number of buildings to the east of building r_i (that is, buildings r_i + 1, r_i + 2, \ldots, N) that can be seen from both buildings l_i and r_i.

Input

The input is given from Standard Input in the following format:
N Q
H_1 H_2 \ldots H_N
l_1 r_1
l_2 r_2
\vdots
l_Q r_Q

Output

Print Q lines. The i-th line (1 \leq i \leq Q) should contain the answer to the i-th query.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq Q \leq 2 \times 10^5
- 1 \leq H_i \leq N
- H_i\neq H_j\ (i\neq j)
- 1 \leq l_i < r_i \leq N
- All input values are integers.

Sample Output 1

      We'll use a Fenwick tree class:

        class Fenw:
            def __init__(self, n):
                self.n = n
                self.fw = [0]*(n+1)
            def update(self, i, delta):   # i is 1-indexed
                while i <= self.n:
                    self.fw[i] += delta
                    i += i & -i
            def query(self, i):   # prefix sum from 1 to i (1-indexed)
                s = 0
                while i:
                    s += self.fw[i]
                    i -= i & -i
                return s

To solve this problem, we need to determine the number of buildings to the east of a given building `r` that can be seen from both buildings `l` and `r` for each query. The key insight is that a building `j` (where `j > r`) can be seen from both `l` and `r` if the immediate taller building to the left of `j` (if any) is located to the left of `l`. 

### Approach
1. **Precompute Next Greater Element to the Left (L)**: For each building `j`, compute the position of the nearest building to the left that is taller than `j`. This helps in determining the visibility condition for `j` from any building to its left.
2. **Offline Query Processing**: 
   - **Sort Queries by `l`**: Process queries in increasing order of `l` to efficiently handle the condition `L[j] < l`.
   - **Fenwick Tree (Binary Indexed Tree)**: Use a Fenwick tree to maintain counts of buildings `j` that satisfy `L[j] < l` as we process each query. For each query, the answer is the count of such buildings `j` where `j > r`.

### Solution Code
```python
import sys

class Fenw:
    def __init__(self, n):
        self.n = n
        self.fw = [0] * (n + 1)
    
    def update(self, i, delta):
        while i <= self.n:
            self.fw[i] += delta
            i += i & -i
            
    def query(self, i):
        s = 0
        while i:
            s += self.fw[i]
            i -= i & -i
        return s

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    N = int(next(it))
    Q = int(next(it))
    H = [int(next(it)) for _ in range(N)]
    
    queries = []
    for _ in range(Q):
        l = int(next(it))
        r = int(next(it))
        queries.append((l, r))
    
    L = [0] * (N + 1)
    stack = []
    for i in range(N):
        while stack and H[stack[-1]] < H[i]:
            stack.pop()
        if stack:
            L[i + 1] = stack[-1] + 1
        else:
            L[i + 1] = 0
        stack.append(i)
    
    points = []
    for j in range(1, N + 1):
        points.append((L[j], j))
    points.sort(key=lambda x: x[0])
    
    sorted_queries = []
    for idx, (l, r) in enumerate(queries):
        sorted_queries.append((l, r, idx))
    sorted_queries.sort(key=lambda x: x[0])
    
    fenw = Fenw(N)
    ptr = 0
    ans = [0] * Q
    for l, r, idx in sorted_queries:
        while ptr < len(points) and points[ptr][0] < l:
            j = points[ptr][1]
            fenw.update(j, 1)
            ptr += 1
        total = fenw.query(N)
        cnt_r = fenw.query(r)
        ans[idx] = total - cnt_r
    
    for a in ans:
        print(a)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of buildings `N`, number of queries `Q`, the heights of the buildings `H`, and the queries themselves.
2. **Computing Next Greater Element to the Left (L)**: Using a stack, we efficiently compute for each building `j` the nearest building to the left that is taller than `j`. This is stored in an array `L` where `L[j]` holds the position of such a building (or 0 if none exists).
3. **Processing Queries Offline**: 
   - **Sorting Queries**: Queries are sorted by their `l` values to process them in increasing order.
   - **Fenwick Tree Operations**: As we process each query, we use a Fenwick tree to dynamically track buildings `j` that satisfy `L[j] < l`. For each query `(l, r)`, the answer is the count of such buildings `j` where `j > r`, obtained by querying the Fenwick tree for the range `(r + 1, N)`.
4. **Output**: The results for each query are printed in the original query order.

This approach efficiently handles the constraints by leveraging offline processing and the Fenwick tree for dynamic range counting, ensuring optimal performance for large inputs.
