=== ANSWER ===
To solve this problem, we need to compute the sum of a geometric series \( X = \sum_{i=0}^{M} N^i \). The challenge lies in efficiently calculating this sum while handling cases where the sum might exceed \( 10^9 \), in which case we should output "inf" instead.

### Approach
1. **Special Case Handling for N = 1**: When \( N = 1 \), the sum simplifies to \( M + 1 \) because each term \( N^i \) is 1. We check if this sum exceeds \( 10^9 \) and output accordingly.
2. **Iterative Sum Calculation for N ≠ 1**: For other values of \( N \), we compute the sum iteratively. Starting with the first term \( N^0 = 1 \), we iteratively add each subsequent term \( N^i \) for \( i \) from 1 to \( M \). After each addition, we check if the cumulative sum exceeds \( 10^9 \). If it does at any point, we immediately output "inf" and terminate. If the entire sum is computed without exceeding \( 10^9 \), we output the sum.

This approach efficiently handles large values of \( N \) (up to \( 10^9 \)) and \( M \) (up to 100) by breaking early when the sum becomes too large, thus avoiding unnecessary computations and potential overflow issues.

### Solution Code
```python
def main():
    MAX_LIMIT = 10**9
    N, M = map(int, input().split())
    
    if N == 1:
        total = M + 1
        print(total if total <= MAX_LIMIT else "inf")
    else:
        total = 0
        term = 1
        for i in range(M + 1):
            total += term
            if total > MAX_LIMIT:
                print("inf")
                return
            if i < M:
                term *= N
        print(total)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input consists of two integers \( N \) and \( M \).
2. **Special Case Handling**: If \( N = 1 \), the sum \( X \) is simply \( M + 1 \). We check if this sum is within the limit \( 10^9 \) and print the result accordingly.
3. **Iterative Sum Calculation**: For \( N \neq 1 \), we initialize `total` to 0 and `term` to 1 (representing \( N^0 \)). For each term from \( i = 0 \) to \( M \):
   - Add the current term to `total`.
   - If at any point `total` exceeds \( 10^9 \), print "inf" and exit.
   - Update `term` to \( N^{i+1} \) for the next iteration, provided we haven't processed all \( M \) terms.
4. **Output Result**: If the loop completes without `total` exceeding \( 10^9 \), print the final sum.

This method ensures efficient computation with early termination for large sums, leveraging the constraints effectively.
