The challenge is to efficiently compute these distances for a large number of points (up to 200,000) while maintaining precision as specified (absolute or relative error ≤ 1e-6).

2. **Intuition**: The solution requires iterating through each point in the sequence, calculating the distance from the current point to the next point, and accumulating these distances. The initial and final moves to and from the origin are handled separately.

3. **Algorithm Selection**: We use the `math.hypot` function from Python's math module to compute the Euclidean distance between two points. This function is chosen for its numerical stability, especially with large coordinates, as it avoids intermediate overflow by scaling the values appropriately during computation.

4. **Complexity Analysis**: The algorithm processes each point exactly once, leading to a time complexity of O(N), where N is the number of points. Each distance computation involves constant time operations, making the solution efficient for the given constraints.

5. **Precision Handling**: The result is formatted to 20 decimal places to meet the output precision requirement, though the inherent precision of double-precision floating-point arithmetic (used by `math.hypot`) ensures that the absolute or relative error remains within the acceptable limit of 1e-6.

### Solution Code

import math

def main():
    n = int(input().strip())
    points = []
    for _ in range(n):
        x, y = map(int, input().split())
        points.append((x, y))
    
    total = 0.0
    if n > 0:
        x, y = points[0]
        total += math.hypot(x, y)

        x1, y1 = points[i-1]
        x2, y2 = points[i]
        dx = x2 - x1
        dy = y2 - y1
        total += math.hypot(dx, dy)

        x, y = points[-1]
