=== ANSWER ===
To solve this problem, we need to determine whether a given permutation of the sequence (1, 2, 3, 4, 5) can be sorted into ascending order by performing exactly one swap of two adjacent elements. 

### Approach
1. **Check if the array is already sorted**: If the input array is already in ascending order (1, 2, 3, 4, 5), then no swaps are needed. However, since the problem requires exactly one swap, we immediately return "No" in this case.
2. **Simulate each possible adjacent swap**: For each adjacent pair in the array (positions 0-1, 1-2, 2-3, 3-4), we perform a swap and check if the resulting array is sorted. 
3. **Check the result after each swap**: After swapping each adjacent pair, we compare the modified array with the sorted sequence (1, 2, 3, 4, 5). If any swap results in the sorted sequence, we return "Yes".
4. **Return "No" if no valid swap is found**: If none of the adjacent swaps results in the sorted sequence, we return "No".

### Solution Code
```python
def main():
    data = list(map(int, input().split()))
    sorted_arr = [1, 2, 3, 4, 5]
    
    if data == sorted_arr:
        print("No")
        return
    
    n = len(data)
    for i in range(n - 1):
        test_arr = data[:]
        test_arr[i], test_arr[i+1] = test_arr[i+1], test_arr[i]
        if test_arr == sorted_arr:
            print("Yes")
            return
    
    print("No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Reading**: The input is read as a list of five integers.
2. **Immediate Check for Sorted Array**: The program first checks if the input array is already sorted. If it is, it prints "No" because zero swaps are performed, which violates the requirement of exactly one swap.
3. **Simulating Adjacent Swaps**: The program iterates over each adjacent pair in the array. For each pair, it creates a copy of the original array, swaps the elements at the current and next positions, and checks if the modified array matches the sorted sequence (1, 2, 3, 4, 5).
4. **Result Determination**: If any swap results in the sorted sequence, the program immediately prints "Yes" and exits. If none of the swaps result in the sorted sequence after checking all possible adjacent pairs, the program prints "No".

This approach efficiently checks all possible single adjacent swaps to determine if the array can be sorted with exactly one operation, ensuring optimal performance given the fixed input size.
