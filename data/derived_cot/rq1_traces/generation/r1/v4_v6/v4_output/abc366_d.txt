You will be given a competitive programming problem.
You are given a positive integer N, and an integer A_{x,y,z} for each triple of integers (x, y, z) such that 1 \leq x, y, z \leq N.
You will be given Q queries in the following format, which must be processed in order.

\displaystyle{\sum_{x=Lx_i}^{Rx_i} \sum_{y=Ly_i}^{Ry_i} \sum_{z=Lz_i}^{Rz_i} A_{x,y,z}}.

\vdots

\vdots
A_{N,N,1} A_{N,N,2} \ldots A_{N,N,N}

Lx_2 Rx_2 Ly_2 Ry_2 Lz_2 Rz_2
\vdots
Lx_Q Rx_Q Ly_Q Ry_Q Lz_Q Rz_Q

Output

Print Q lines.
The i-th line should contain the answer to the i-th query.

Constraints


- 1 \leq N \leq 100
- 1 \leq Q \leq 2 \times 10^{5}
- 0 \leq A_{x,y,z} \leq 999 (1 \leq x, y, z \leq N)
- 1 \leq Lx_i \leq Rx_i \leq N (1 \leq i \leq Q)
- 1 \leq Ly_i \leq Ry_i \leq N (1 \leq i \leq Q)

### Approach
1. **Problem Analysis**: The problem involves a 3D array of size \(N \times N \times N\) and \(Q\) queries. Each query specifies a sub-cube defined by ranges in each dimension. The goal is to compute the sum of elements within the sub-cube for each query efficiently.
2. **Intuition**: The key insight is to precompute a 3D prefix sum array where each element at position \((i, j, k)\) stores the sum of all elements from \((1, 1, 1)\) to \((i, j, k)\). This allows us to answer each query in \(O(1)\) time using inclusion-exclusion principles.
3. **Algorithm Selection**: 
   - **Prefix Sum Array Construction**: We build a 3D prefix sum array `pre` of size \((N+1) \times (N+1) \times (N+1)\). The value at `pre[i][j][k]` is computed by adding the current element and the sums from the adjacent cells, subtracting overlaps.
   - **Query Processing**: For each query, the sum of the sub-cube is derived by combining the values from the prefix sum array using inclusion-exclusion to subtract the parts outside the desired ranges.
4. **Complexity Analysis**: 
   - **Preprocessing**: Constructing the prefix sum array takes \(O(N^3)\) time.
   - **Query Processing**: Each query is answered in \(O(1)\) time, leading to \(O(Q)\) total time for all queries.
   - **Overall Complexity**: The solution efficiently handles the constraints with \(N \leq 100\) and \(Q \leq 2 \times 10^5\).

import sys
