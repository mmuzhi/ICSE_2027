The challenge is to maximize the sum of these heights. The key insight is that to maximize the sum, we should assign the largest possible distinct numbers to the towers while respecting their maximum height constraints.
2. **Intuition**: By sorting the maximum heights in descending order, we prioritize towers with higher constraints first. This allows us to assign the largest possible numbers first, which contributes more to the total sum. For each subsequent tower, we assign the next largest available number that is within its maximum height constraint.
3. **Algorithm Selection**: 
   - **Sorting**: The maximum heights are sorted in descending order to handle towers with higher constraints first.
   - **Greedy Assignment**: We start by assigning the largest number (the first tower's maximum height) to the first tower. For each subsequent tower, we assign the largest available number (current) that is within the tower's maximum height. If no such number is available (current becomes 0), we return -1.
4. **Complexity Analysis**: Sorting the array takes O(n log n) time, where n is the number of towers. The subsequent loop runs in O(n) time. Thus, the overall complexity is O(n log n), which is efficient for the given constraints (n ≤ 10^5).

class Solution:
    def maximumTotalSum(self, maximumHeight: List[int]) -> int:
        maximumHeight.sort(reverse=True)
        current = None

1. **Sorting**: The array `maximumHeight` is sorted in descending order to prioritize towers with higher maximum height constraints. This allows us to assign the largest possible numbers first, maximizing the sum.
2. **Initial Assignment**: The first tower (after sorting) is assigned its maximum height. The variable `current` is set to one less than this height, representing the next largest available number for subsequent towers.
3. **Subsequent Assignments**: For each subsequent tower, we assign the minimum of `current` (the next largest available number) and the tower's maximum height. This ensures we use the largest possible number within the constraint. If `current` becomes zero, it means no positive distinct numbers are left, and we return -1.
4. **Sum Calculation**: The total sum of assigned heights is accumulated and returned if all towers are successfully assigned distinct positive heights.
