You will be given a competitive programming problem.
You are given positive integers N and K.
An integer sequence of length NK where each integer from 1 to N appears exactly K times is called a good integer sequence.
Let S be the number of good integer sequences.
Find the \operatorname{floor}((S+1)/2)-th good integer sequence in lexicographical order.
Here, \operatorname{floor}(x) represents the largest integer not exceeding x.
 What is lexicographical order for sequences?
A sequence S = (S_1,S_2,\ldots,S_{|S|}) is lexicographically smaller than a sequence T = (T_1,T_2,\ldots,T_{|T|}) if either 1. or 2. below holds.
Here, |S| and |T| represent the lengths of S and T, respectively.

-  |S| \lt |T| and (S_1,S_2,\ldots,S_{|S|}) = (T_1,T_2,\ldots,T_{|S|}). 
-  There exists an integer 1 \leq i \leq \min\lbrace |S|, |T| \rbrace such that both of the following hold:

-  (S_1,S_2,\ldots,S_{i-1}) = (T_1,T_2,\ldots,T_{i-1})
-  S_i is (numerically) smaller than T_i.

Input

The input is given from Standard Input in the following format:
N K

Output

Print the desired integer sequence, with elements separated by spaces.

Constraints


- 1 \leq N \leq 500
- 1 \leq K \leq 500
- All input values are integers.

Sample Input 1

2 2

Sample Output 1

1 2 2 1

There are six good integer sequences:

- (1,1,2,2)
- (1,2,1,2)
- (1,2,2,1)
- (2,1,1,2)
- (2,1,2,1)
- (2,2,1,1)

Therefore, the answer is the 3rd sequence in lexicographical order, (1,2,2,1).

To solve this problem, we need to generate the floor((S+1)/2)-th lexicographically smallest sequence of integers where each integer from 1 to N appears exactly K times. The solution involves combinatorial counting and lexicographical sequence generation.

### Approach
1. **Problem Analysis**: The problem requires generating a specific sequence from all possible sequences of length N*K where each integer from 1 to N appears exactly K times. The total number of such sequences, S, is given by the multinomial coefficient: \( S = \frac{(NK)!}{(K!)^N} \). The goal is to find the \(\text{floor}((S+1)/2)\)-th sequence in lexicographical order.

2. **Key Insight**: Instead of enumerating all sequences, which is infeasible for large N and K, we use combinatorial methods to count sequences dynamically. We start by computing the total number of sequences, S, using prime factorization or iterative combinatorial product. However, given constraints (N, K ≤ 500), we compute S as the product of combinatorial coefficients \( \prod_{i=1}^{N} \binom{i*K}{K} \).

3. **Algorithm Selection**: 
   - **Computation of S**: We compute S iteratively by multiplying combinatorial coefficients \( \binom{i*K}{K} \) for each i from 1 to N. This avoids directly computing large factorials.
   - **Sequence Generation**: We generate the desired sequence by, at each step, choosing the smallest integer (in lex order) such that the number of sequences starting with that integer is at least the remaining target index (k0). The count of sequences starting with an integer is derived from the current state using the formula: \( \text{count}_i = \frac{r_i \times \text{current\_count}}{\text{total}} \), where \( r_i \) is the remaining count of integer i, and total is the remaining positions.

4. **Complexity Analysis**: 
   - **Precomputation of S**: The product involves N combinatorial coefficients, each computed in O(K) time, leading to O(N*K) time.
   - **Generation Loop**: For each of the N*K positions, we check up to N integers, leading to O(N^2*K) operations. Each operation involves big integer arithmetic, which can be expensive but is feasible for the given constraints with optimizations.

### Solution Code
```python
import math

def main():
    import sys
    data = sys.stdin.read().split()
    N = int(data[0])
    K = int(data[1])
    
    total_length = N * K
    S_val = 1
    factK = math.factorial(K)
    for i in range(1, N+1):
        n_val = i * K
        if n_val < K:
            comb_val = 0
        else:
            numerator = 1
            for j in range(n_val, n_val - K, -1):
                numerator *= j
            comb_val = numerator // factK
        S_val = S_val * comb_val
    
    k0 = (S_val + 1) // 2
    
    r = [K] * N
    current_count = S_val
    total = N * K
    res = []
    
    for _ in range(total_length):
        for i in range(N):
            if r[i] == 0:
                continue
            count_i = (r[i] * current_count) // total
            if count_i == 0:
                continue
            if k0 <= count_i:
                res.append(i+1)
                r[i] -= 1
                current_count = count_i
                total -= 1
                break
            else:
                k0 -= count_i
    
    print(" ".join(map(str, res)))

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input consists of two integers, N and K, read from standard input.
2. **Computing S (Total Sequences)**: The total number of valid sequences, S, is computed as the product of combinatorial coefficients \( \binom{i*K}{K} \) for i from 1 to N. This avoids direct computation of large factorials.
3. **Target Index Calculation**: The target sequence index, k0, is calculated as \( \text{floor}((S+1)/2) \).
4. **Sequence Generation**: 
   - **Initialization**: We start with all integers 1 to N having K remaining occurrences, and the total remaining positions is N*K.
   - **Iterative Selection**: For each position, we check each integer in lex order. For each integer, we compute how many valid sequences start with it. If this count is at least the remaining target index (k0), we select the integer, update the state, and move to the next position. Otherwise, we subtract the count from k0 and check the next integer.
5. **Output**: The generated sequence is printed as space-separated integers.

This approach efficiently narrows down the sequence by leveraging combinatorial counting and lexicographical ordering without explicitly generating all sequences, making it feasible for large N and K.
