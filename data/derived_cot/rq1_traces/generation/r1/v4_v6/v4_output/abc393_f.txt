Input

The input is given from Standard Input in the following format:
N Q
A_1 A_2 \dots A_N
R_1 X_1
R_2 X_2
\vdots

Output

Constraints


- 1 \leq N,Q \leq 2 \times 10^5
- 1 \leq A_i \leq 10^9
- 1 \leq R_i \leq N
- \min\lbrace A_1, A_2,\dots,A_{R_i} \rbrace\leq X_i\leq 10^9
- All input values are integers.

To solve this problem, we need to efficiently answer multiple queries about the maximum length of a strictly increasing subsequence in the first \( R_i \) elements of a sequence \( A \), where all elements in the subsequence are at most \( X_i \). Given the constraints, a brute-force approach is infeasible, so we use a combination of coordinate compression and a Fenwick tree (or Binary Indexed Tree) for efficient updates and queries.

### Approach
1. **Coordinate Compression**: Since the values in \( A \) and the queries \( X_i \) can be very large (up to \( 10^9 \)), we first compress these values into a smaller range. This involves collecting all unique values from \( A \) and the queries, sorting them, and mapping each value to its index in the sorted list.
2. **Fenwick Tree for Maximum Values**: We use a Fenwick tree that supports point updates and prefix maximum queries. The tree will store the maximum length of any strictly increasing subsequence ending at each compressed value.
3. **Processing Queries Offline**: We sort the queries by \( R_i \) (the number of elements to consider). As we process each element of \( A \) in order, we update the Fenwick tree with the maximum subsequence length ending at the current element. For each query \( (R_i, X_i) \) where \( R_i \) matches the current number of processed elements, we query the Fenwick tree for the maximum subsequence length among values \( \leq X_i \).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    q = int(data[1])
    A = list(map(int, data[2:2+n]))
    queries = []
    idx = 2 + n
    Xs = []
    for i in range(q):
        r = int(data[idx])
        x = int(data[idx+1])
        idx += 2
        queries.append((r, x))
        Xs.append(x)
    
    all_vals = sorted(set(A) | set(Xs))
    comp_dict = {val: i+1 for i, val in enumerate(all_vals)}
    M = len(all_vals)
    
    fenw = [0] * (M + 1)
    
    def fenw_update(i, val):
        while i <= M:
            if val > fenw[i]:
                fenw[i] = val
            i += i & -i
    
    def fenw_query(i):
        res = 0
        while i:
            if fenw[i] > res:
                res = fenw[i]
            i -= i & -i
        return res

    queries_by_r = [[] for _ in range(n + 1)]
    for idx_query, (r, x) in enumerate(queries):
        queries_by_r[r].append((x, idx_query))
    
    ans = [0] * q
    
    for i in range(n):
        v = A[i]
        pos = comp_dict[v]
        prev_max = fenw_query(pos - 1)
        dp = prev_max + 1
        fenw_update(pos, dp)
        
        r_current = i + 1
        for (x, idx_query) in queries_by_r[r_current]:
            posX = comp_dict[x]
            res = fenw_query(posX)
            ans[idx_query] = res
            
    for a in ans:
        print(a)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Coordinate Compression**: The values from the sequence \( A \) and the queries \( X_i \) are combined into a sorted list of unique values. Each value is then mapped to its index in this list, reducing the range of values to the size of the list (which is at most \( 400,000 \) for \( N, Q \leq 200,000 \)).
2. **Fenwick Tree**: The Fenwick tree is initialized to store the maximum length of increasing subsequences ending at each compressed value. The tree supports two operations:
   - **Update (fenw_update)**: Updates the tree at a specific position with a new value, ensuring ancestors are updated if the new value is greater.
   - **Query (fenw_query)**: Retrieves the maximum value in the prefix up to a given index, which corresponds to the maximum subsequence length ending at any value less than or equal to the queried value.
3. **Processing Elements and Queries**: As each element of \( A \) is processed, the Fenwick tree is updated with the maximum subsequence length ending at that element. Queries are processed in order of \( R_i \), leveraging the compressed values to quickly determine the maximum valid subsequence length for elements \( \leq X_i \) in the first \( R_i \) elements.

This approach efficiently handles the constraints by leveraging coordinate compression and the Fenwick tree, ensuring optimal performance for large input sizes.
