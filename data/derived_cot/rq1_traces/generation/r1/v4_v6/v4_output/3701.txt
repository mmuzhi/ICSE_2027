To solve this problem, we need to convert a given string into a "good caption" where every character appears in groups of at least 3 consecutive identical characters. The transformation involves changing characters to adjacent characters in the alphabet (either the previous or next character) with the minimum number of operations. If multiple good captions are possible with the same minimal operations, we return the lexicographically smallest one. If it's impossible to form a good caption, we return an empty string.

### Approach
1. **Problem Analysis**: The problem requires transforming a string such that all characters are grouped in consecutive runs of at least 3 identical characters. Each operation changes a character to an adjacent character in the alphabet (e.g., 'b' can become 'a' or 'c'). The solution must find the minimal number of operations and, among all possible solutions with minimal operations, return the lexicographically smallest string.

2. **Dynamic Programming (DP) Setup**: We use a DP approach where the state `dp[i][c][r]` represents the minimal cost to process the first `i` characters of the string, ending with character `c` (where `c` is an integer representing 'a' to 'z') and a run length `r` (1, 2, or 3, where 3 signifies a run of at least 3 characters). The cost is the sum of absolute differences between the original characters and their target characters.

3. **State Transitions**:
   - **Continue Current Run**: If the current run length is 1 or 2, we must continue with the same character. The run length increases (or stays at 3 if it exceeds 3).
   - **Start New Run**: If the current run length is at least 3, we can start a new run with a different character, resetting the run length to 1.

4. **Lexicographical Order**: When multiple states yield the same cost, we choose the state that results in the lexicographically smallest string. This is handled by storing the actual string for each state during the DP process (for small `n`).

5. **Final State Check**: After processing all characters, valid solutions must end with a run length of 3. We then select the solution with the minimal cost and lexicographically smallest string.

```python
class Solution:
    def minCostGoodCaption(self, caption: str) -> str:
        n = len(caption)
        if n < 3:
            return ""
        
        cap_vals = [ord(ch) - ord('a') for ch in caption]
        
        INF = float('inf')
        dp = [[[ (INF, "") for _ in range(4)] for _ in range(26)] for _ in range(n)]
        
        for c in range(26):
            cost = abs(cap_vals[0] - c)
            s = chr(c + ord('a'))
            if cost < dp[0][c][1][0]:
                dp[0][c][1] = (cost, s)
        
        for i in range(1, n):
            for c in range(26):
                cost_here = abs(cap_vals[i] - c)
                char_here = chr(c + ord('a'))
                
                for r_prev in [1, 2, 3]:
                    for c_prev in range(26):
                        cost_prev, str_prev = dp[i-1][c_prev][r_prev]
                        if cost_prev == INF:
                            continue
                        if r_prev in [1, 2]:
                            if c_prev == c:
                                new_run = r_prev + 1 if r_prev + 1 < 3 else 3
                                new_cost = cost_prev + cost_here
                                new_str = str_prev + char_here
                                if new_cost < dp[i][c][new_run][0] or (new_cost == dp[i][c][new_run][0] and new_str < dp[i][c][new_run][1]):
                                    dp[i][c][new_run] = (new_cost, new_str)
                        else:
                            if c_prev == c:
                                new_cost = cost_prev + cost_here
                                new_str = str_prev + char_here
                                if new_cost < dp[i][c][3][0] or (new_cost == dp[i][c][3][0] and new_str < dp[i][c][3][1]):
                                    dp[i][c][3] = (new_cost, new_str)
                            else:
                                if i == n-1:
                                    continue
                                new_cost = cost_prev + cost_here
                                new_str = str_prev + char_here
                                if new_cost < dp[i][c][1][0] or (new_cost == dp[i][c][1][0] and new_str < dp[i][c][1][1]):
                                    dp[i][c][1] = (new_cost, new_str)
        
        min_cost = INF
        result_str = ""
        for c in range(26):
            cost, s = dp[n-1][c][3]
            if cost < min_cost or (cost == min_cost and s < result_str):
                min_cost = cost
                result_str = s
        
        return result_str if min_cost != INF else ""
```

### Explanation
1. **Initialization**: The DP table `dp` is initialized to store the cost and resulting string for each state `(i, c, r)`, where `i` is the current position, `c` is the current character, and `r` is the run length (1, 2, or 3).
2. **Base Case**: For the first character, we initialize the DP table by considering all possible target characters and their costs.
3. **DP Transitions**: For each subsequent character, we consider continuing the current run (if run length is 1 or 2) or starting a new run (if run length is at least 3). The cost and resulting string are updated accordingly.
4. **Lexicographical Order**: When multiple states have the same cost, the lexicographically smallest string is chosen by comparing strings directly.
5. **Result Extraction**: After processing all characters, we check the final states (ending with run length 3) to find the minimal cost solution. The lexicographically smallest string is returned if a solution exists; otherwise, an empty string is returned.

This approach efficiently computes the minimal operations required to form a good caption while ensuring the solution is lexicographically smallest, leveraging dynamic programming with careful state management.
