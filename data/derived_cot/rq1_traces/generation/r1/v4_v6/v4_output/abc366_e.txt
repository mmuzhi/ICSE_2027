Input

N D

Output

To solve this problem, we need to count the number of integer pairs (x, y) such that the sum of Manhattan distances from (x, y) to all given points is less than or equal to a given non-negative integer D. The Manhattan distance between two points (x, y) and (x_i, y_i) is defined as |x - x_i| + |y - y_i|.

### Approach
1. **Problem Analysis**: The problem requires us to find all integer pairs (x, y) where the sum of Manhattan distances to each given point (x_i, y_i) is ≤ D. The challenge is to efficiently compute this without iterating over all possible (x, y) pairs, which would be computationally infeasible for large coordinate ranges.

2. **Key Insight**: The sum of Manhattan distances can be separated into two independent sums: one for the x-coordinates and one for the y-coordinates. Specifically, the total sum is the sum of |x - x_i| over all i plus the sum of |y - y_i| over all i. This separation allows us to handle the x and y dimensions independently.

3. **Precomputation**:
   - **For y-coordinates**: 
     - Sort the y-coordinates and compute prefix sums.
     - Precompute a function g(y) = sum_i |y - y_i| for all relevant y values. The relevant y values are those in the range [ceil((Sy - D)/N), floor((Sy + D)/N)], where Sy is the sum of all y_i.
     - Store the counts of g(y) values in an array `g_val` and build a prefix sum array `prefix_g` to quickly compute the number of y values where g(y) ≤ any given value.
   - **For x-coordinates**:
     - Similarly, sort the x-coordinates and compute prefix sums.
     - For each x in the range [ceil((Sx - D)/N), floor((Sx + D)/N)], compute f(x) = sum_i |x - x_i|.
     - For each x, if f(x) ≤ D, use the precomputed `prefix_g` to find the number of y values such that g(y) ≤ D - f(x).

4. **Efficiency**: The approach efficiently narrows down the search space for x and y by leveraging the properties of Manhattan distance and precomputed prefix sums. The complexity is dominated by sorting (O(N log N)) and iterating over the middle segments (O(max_x - min_x + max_y - min_y)), which is feasible given the constraints.

```python
            if 0 <= gy <= D_val:
                g_val[gy] += 1
    
    idx = 0
    for y in range(y_min, y_max + 1):
        while idx < n and ys[idx] <= y:
            idx += 1
        left_count = idx
        left_sum = pre_y[idx - 1] if idx > 0 else 0
        right_sum = pre_y[n - 1] - left_sum
        gy = (y * left_count - left_sum) + (right_sum - y * (n - left_count))
        if 0 <= gy <= D_val:
            g_val[gy] += 1
    
    prefix_g = [0] * (D_val + 1)
    if D_val >= 0:
        if D_val >= 0:
            prefix_g[0] = g_val[0]
            for i in range(1, D_val + 1):
                prefix_g[i] = prefix_g[i - 1] + g_val[i]
    
    pre_x = [0] * n
    if n > 0:
        pre_x[0] = xs[0]
        for i in range(1, n):
            pre_x[i] = pre_x[i - 1] + xs[i]
    
    x_min = xs[0]
    x_max = xs[-1]
    total_count = 0
    
    left_start_x = (Sx - D_val + n - 1) // n
    left_end_x = x_min - 1
    if left_start_x <= left_end_x:
        for x in range(left_start_x, left_end_x + 1):
            fx = Sx - n * x
            if fx > D_val:
                continue
            rem = D_val - fx
            if rem < 0:
                continue
            cnt_y = prefix_g[min(rem, D_val)]
            total_count += cnt_y
    
    right_start_x = x_max + 1
    right_end_x = (Sx + D_val) // n
    if right_start_x <= right_end_x:
        for x in range(right_start_x, right_end_x + 1):
            fx = n * x - Sx
            if fx > D_val:
                continue
            rem = D_val - fx
            if rem < 0:
                continue
            cnt_y = prefix_g[min(rem, D_val)]
            total_count += cnt_y
    
    idx = 0
    for x in range(x_min, x_max + 1):
        while idx < n and xs[idx] <= x:
            idx += 1
        left_count = idx
        left_sum = pre_x[idx - 1] if idx > 0 else 0
        right_sum = pre_x[n - 1] - left_sum
        fx = (x * left_count - left_sum) + (right_sum - x * (n - left_count))
        if fx > D_val:
            continue
        rem = D_val - fx
        if rem < 0:
            continue
        cnt_y = prefix_g[min(rem, D_val)]
        total_count += cnt_y
    
    print(total_count)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of points `n`, the distance constraint `D_val`, and lists of x and y coordinates.
2. **Sorting and Prefix Sums**: The x and y coordinates are sorted, and prefix sums are computed to facilitate efficient calculation of Manhattan distances.
3. **Precomputing g(y) Values**: For each relevant y value (within a calculated range), the function g(y) = sum_i |y - y_i| is computed. The counts of these values are stored in `g_val`, and a prefix sum array `prefix_g` is built for quick lookup.
4. **Processing x-coordinates**: For each relevant x value (within a calculated range), the function f(x) = sum_i |x - x_i| is computed. If f(x) ≤ D_val, the number of valid y values (where g(y) ≤ D_val - f(x)) is retrieved from `prefix_g` and added to the total count.
5. **Output**: The total count of valid (x, y) pairs is printed.

This approach efficiently narrows down the search space using mathematical insights and prefix sums, ensuring optimal performance even for large input sizes.
