To solve this problem, we need to compute the sum of scores for all non-empty subsequences of the first `m` elements of a given sequence `A` for each `m` from 1 to `N`. The score of a subsequence is defined as the sum of the greatest common divisors (gcd) of each pair of adjacent elements in the subsequence. 

### Approach
1. **Precomputation**:
   - **Divisors**: Precompute the divisors for all numbers up to the maximum value in the sequence (100,000). This helps in efficiently determining the divisors of each element in the sequence during processing.
   - **Euler's Totient Function (φ)**: Precompute φ for all numbers up to 100,000. This function is used to efficiently compute the sum involving gcd terms using the identity `gcd(x, a) = sum_{d | gcd(x, a)} φ(d)`.
   - **Powers of 2**: Precompute powers of 2 modulo 998244353 up to `N` (500,000). These powers are used to count the number of new subsequences ending with each element when it is added to the sequence.

2. **Dynamic Processing**:
   - **Frequency Array**: Maintain an array `freq` where `freq[d]` represents the sum of counts of subsequences (from the first `m-1` elements) that end with a number divisible by `d`.
   - **Update Mechanism**: For each element `a` in the sequence:
     - **Compute Additional Score**: The additional score contributed by new subsequences ending with `a` is calculated as the sum over all divisors `d` of `a` of `φ(d) * freq[d]`.
     - **Update Total Score**: The total score for the first `m` elements is updated as `F(m) = 2 * F(m-1) + additional_score`, where `F(m-1)` is the total score for the first `m-1` elements.
     - **Update Frequency Array**: For each divisor `d` of `a`, update `freq[d]` by adding `2^(m-1)`, which represents the number of new subsequences ending with `a`.

### Solution Code
```python
import sys

mod = 998244353
max_val = 100000

def precompute_divisors(max_val):
    divisors = [[] for _ in range(max_val + 1)]
    for i in range(1, max_val + 1):
        for j in range(i, max_val + 1, i):
            divisors[j].append(i)
    return divisors

def precompute_phi(max_val):
    phi = list(range(max_val + 1))
    for i in range(2, max_val + 1):
        if phi[i] == i:
            for j in range(i, max_val + 1, i):
                phi[j] -= phi[j] // i
    return phi

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1 + n]))
    
    divisors_list = precompute_divisors(max_val)
    phi_arr = precompute_phi(max_val)
    
    two_power = [1] * (n + 1)
    for i in range(1, n + 1):
        two_power[i] = (two_power[i - 1] * 2) % mod
        
    freq = [0] * (max_val + 1)
    F = 0
    
    for i in range(n):
        a = A[i]
        add_score = 0
        for d in divisors_list[a]:
            add_score = (add_score + phi_arr[d] * freq[d]) % mod
        F = (2 * F + add_score) % mod
        for d in divisors_list[a]:
            freq[d] = (freq[d] + two_power[i]) % mod
        print(F)

if __name__ == "__main__":
    main()
```

   - **Euler's Totient Function**: The `precompute_phi` function calculates φ for all numbers up to 100,000 using a sieve method, which is essential for efficiently computing the sum involving gcd terms.
   - **Powers of 2**: The `two_power` array stores powers of 2 modulo 998244353, used to count the number of new subsequences when an element is added.

2. **Dynamic Processing**:
   - For each element `a` in the sequence, the additional score from new subsequences ending with `a` is computed by summing `φ(d) * freq[d]` for all divisors `d` of `a`. This leverages the identity `gcd(x, a) = sum_{d | gcd(x, a)} φ(d)`.
   - The total score `F` is updated to include the new subsequences' scores using the formula `F = 2 * F + additional_score`, accounting for both existing and new subsequences.
   - The `freq` array is updated for each divisor `d` of `a` by adding `2^(i)`, where `i` is the current element's index, representing the count of new subsequences ending with `a`.

This approach efficiently computes the required scores for each prefix of the sequence by leveraging precomputation and dynamic updates, ensuring optimal performance even for large inputs.
