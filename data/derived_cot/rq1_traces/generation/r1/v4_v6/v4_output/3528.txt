You are given an integer array nums of length n.
Your goal is to start at index 0 and reach index n - 1. You can only jump to indices greater than your current index.
The score for a jump from index i to index j is calculated as (j - i) * nums[i].
Return the maximum possible total score by the time you reach the last index.
 
Example 1:

Input: nums = [1,3,1,5]
Output: 7
Explanation:
First, jump to index 1 and then jump to the last index. The final score is 1 * 1 + 2 * 3 = 7.

Example 2:

Input: nums = [4,3,1,3,2]
Output: 16
Explanation:
Jump directly to the last index. The final score is 4 * 4 = 16.

=== ANSWER ===
To solve this problem, we need to find the maximum total score when jumping from the first index to the last index of an array, where each jump's score is calculated as the product of the distance jumped and the value at the starting index. The challenge is to efficiently compute this maximum score without resorting to a brute-force O(n^2) approach, especially given the constraint that the array length can be as large as 10^5.

### Approach
1. **Problem Analysis**: The problem requires moving from index 0 to index n-1 in an array, making jumps only to higher indices. Each jump from index `i` to index `j` contributes `(j - i) * nums[i]` to the total score. The goal is to maximize the cumulative score upon reaching the last index.

2. **Dynamic Programming with LiChao Tree**: A dynamic programming approach is natural here, where `dp[i]` represents the maximum score to reach index `i`. The recurrence relation is:
   \[
   dp[i] = \max_{j < i} \{ dp[j] + (i - j) \times nums[j] \}
   \]
   This can be rewritten as:
   \[
   dp[i] = \max_{j < i} \{ (dp[j] - j \times nums[j]) + i \times nums[j] \}
   \]
   Here, the term `(dp[j] - j \times nums[j])` is a constant for each `j`, and `i \times nums[j]` is linear in `i`. Thus, for each `j`, we have a line \( f_j(i) = nums[j] \times i + (dp[j] - j \times nums[j]) \). The problem reduces to finding the maximum value among these lines at each `i`.

3. **LiChao Tree Optimization**: To efficiently handle these line queries and updates, we use a LiChao Tree, which allows us to dynamically add lines and query the maximum value at any point in logarithmic time. The tree is built over the domain of indices (0 to n-1), and for each index `i`, we query the maximum value at `i` and then insert the line corresponding to `i` for future queries.

### Solution Code
```python
class Solution:
    def findMaximumScore(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 1:
            return 0
        
        class LiChaoTree:
            def __init__(self, size):
                self.size = size
                self.tree = [None] * (4 * (self.size + 1))
            
            def f(self, line, x):
                a, b = line
                return a * x + b
                
            def update(self, a, b):
                nw = (a, b)
                stack = [(1, 0, self.size)]
                while stack:
                    node, l, r = stack.pop()
                    if l > r:
                        continue
                    mid = (l + r) // 2
                    if self.tree[node] is None:
                        self.tree[node] = nw
                        continue
                    if self.f(nw, mid) > self.f(self.tree[node], mid):
                        self.tree[node], nw = nw, self.tree[node]
                    if l == r:
                        continue
                    left_better = self.f(nw, l) > self.f(self.tree[node], l)
                    right_better = self.f(nw, r) > self.f(self.tree[node], r)
                    if left_better:
                        stack.append((2 * node, l, mid))
                    elif right_better:
                        stack.append((2 * node + 1, mid + 1, r))
            
            def query(self, x):
                node = 1
                l, r = 0, self.size
                res = -10**18
                while l <= r:
                    if self.tree[node] is not None:
                        res = max(res, self.f(self.tree[node], x))
                    if l == r:
                        break
                    mid = (l + r) // 2
                    if x <= mid:
                        node = 2 * node
                        r = mid
                    else:
                        node = 2 * node + 1
                        l = mid + 1
                return res
        
        tree = LiChaoTree(n - 1)
        tree.update(nums[0], 0)
        dp_i = 0
        for i in range(1, n):
            dp_i = tree.query(i)
            if i < n - 1:
                tree.update(nums[i], dp_i - i * nums[i])
        return dp_i
```

### Explanation
1. **Initialization**: For arrays of length 1, the score is 0 since no jumps are needed. For longer arrays, initialize a LiChao Tree covering indices from 0 to n-1.
2. **Line Insertion**: Start by inserting the line corresponding to the first index (0), which is \( f_0(i) = nums[0] \times i \).
3. **Dynamic Programming with Queries**: For each subsequent index `i` (from 1 to n-1):
   - Query the maximum value at `i` from the LiChao Tree, which gives `dp[i]`.
   - If `i` is not the last index, insert a new line \( f_i(x) = nums[i] \times x + (dp[i] - i \times nums[i]) \) into the tree for future queries.
4. **Result**: The value `dp[n-1]` after processing all indices is the maximum score to reach the last index.

This approach efficiently computes the maximum score using dynamic programming optimized with a LiChao Tree, reducing the time complexity from O(n^2) to O(n log n).
