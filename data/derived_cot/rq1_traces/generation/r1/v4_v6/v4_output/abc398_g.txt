What is an odd cycle?

- k is odd.
- v_0=v_k.

Constraints

=== ANSWER ===
To solve this problem, we need to determine the winner of a game played on a bipartite graph (since the graph initially has no odd cycles) where players alternately add edges under specific constraints. The key insight is recognizing that the game is equivalent to the "Shannon switching game" or a variant of the "Colonization game" on bipartite graphs. The solution involves analyzing the graph's structure and computing the Grundy numbers or using combinatorial game theory principles.

### Approach
1. **Graph Representation**: The given graph is bipartite because it has no odd cycles. We can represent it using adjacency lists.
2. **Bipartite Check and Partition**: We perform a BFS or DFS to color the graph and partition it into two sets (say, black and white). This helps in identifying the components and their sizes.
3. **Connected Components**: The graph might be disconnected. Each connected component will be processed separately.
4. **Game Analysis**: The game allows adding edges only between vertices that are in different partitions (black and white) of the same connected component. Adding an edge within the same partition would create an odd cycle, which is not allowed.
5. **Maximum Possible Moves**: For each connected component, the maximum number of edges that can be added is the product of the sizes of the two partitions (black and white) minus the existing edges in that component.
6. **Grundy Numbers and Nim Game**: The entire game can be seen as a Nim game where each connected component is a pile. The number of stones in each pile is the number of edges that can still be added in that component. The XOR (nim-sum) of these piles determines the winner. If the nim-sum is non-zero, the first player (Aoki) can force a win; otherwise, the second player (Takahashi) wins.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        print("Takahashi")
        return
        
    n = int(data[0])
    m = int(data[1])
    graph = [[] for _ in range(n+1)]
    edges = []
    index = 2
    for i in range(m):
        u = int(data[index])
        v = int(data[index+1])
        index += 2
        graph[u].append(v)
        graph[v].append(u)
        edges.append((u, v))
        
    color = [-1] * (n+1)
    components = []
    for i in range(1, n+1):
        if color[i] == -1:
            dq = deque()
            dq.append(i)
            color[i] = 0
            comp = []
            comp.append(i)
            while dq:
                u = dq.popleft()
                for v in graph[u]:
                    if color[v] == -1:
                        color[v] = color[u] ^ 1
                        dq.append(v)
                        comp.append(v)
            components.append(comp)
    
    comp_id = [-1] * (n+1)
    for idx, comp in enumerate(components):
        for node in comp:
            comp_id[node] = idx

    comp_black = [0] * len(components)
    comp_white = [0] * len(components)
    for i in range(1, n+1):
        cid = comp_id[i]
        if color[i] == 0:
            comp_black[cid] += 1
        else:
            comp_white[cid] += 1

    existing_edges_in_comp = [0] * len(components)
    for u, v in edges:
        cid = comp_id[u]
        existing_edges_in_comp[cid] += 1

    nim = 0
    for idx in range(len(components)):
        black = comp_black[idx]
        white = comp_white[idx]
        total_possible_edges = black * white
        remaining = total_possible_edges - existing_edges_in_comp[idx]
        nim ^= remaining

    if nim != 0:
        print("Aoki")
    else:
        print("Takahashi")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of vertices `N`, edges `M`, and the edges themselves.
2. **Graph Construction**: An adjacency list `graph` is built from the edges.
3. **Bipartite Coloring**: Using BFS, each connected component is colored (0 or 1) to partition the graph into two sets (black and white). This ensures that within each component, vertices are divided into two groups.
4. **Component Processing**: For each connected component, the counts of black and white vertices are recorded. The existing edges in each component are also counted.
5. **Nim Game Calculation**: For each component, the number of remaining edges that can be added (without forming an odd cycle) is calculated as `(black * white) - existing_edges`. The XOR (nim-sum) of these values across all components is computed.
6. **Determine Winner**: If the nim-sum is non-zero, the first player (Aoki) wins; otherwise, the second player (Takahashi) wins.

This approach efficiently leverages graph theory and combinatorial game theory to solve the problem within feasible time complexity (O(N + M)).
