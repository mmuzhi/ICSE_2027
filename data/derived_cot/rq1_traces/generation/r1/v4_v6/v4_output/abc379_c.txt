N M

=== ANSWER ===
To solve this problem, we need to determine the minimum number of operations required to distribute stones such that each cell from 1 to N contains exactly one stone. The operations allowed are moving one stone from cell i to cell i+1 if cell i has at least one stone. The solution involves checking feasibility conditions and then calculating the total operations using a mathematical formula.

### Approach
1. **Check Total Stones**: The total number of stones must equal N. If not, it's impossible to achieve the goal, so return -1.
2. **Check Leftmost Cell**: The leftmost cell (cell 1) must have at least one stone initially. If not, it's impossible to fill cell 1, so return -1.
3. **Process Segments**: For each segment between consecutive initial stone piles, ensure there are enough stones to cover the gap between them. The stones available at the start of a segment (including carry-over from previous segments) must be at least the number of cells in the segment. If not, return -1.
4. **Calculate Operations**: The total operations required is the difference between the sum of final positions (1 to N) and the sum of initial positions (each stone's starting cell multiplied by its count).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(-1)
        return
    
    n = int(data[0])
    m = int(data[1])
    X = list(map(int, data[2:2+m]))
    A = list(map(int, data[2+m:2+2*m]))
    
    total_stones = sum(A)
    if total_stones != n:
        print(-1)
        return
        
    stones = list(zip(X, A))
    stones.sort()
    
    if stones[0][0] != 1:
        print(-1)
        return
        
    carry = 0
    for i in range(len(stones)-1):
        x_i, a_i = stones[i]
        x_next = stones[i+1][0]
        avail = a_i + carry
        gap = x_next - x_i
        if avail < gap:
            print(-1)
            return
        carry = avail - gap
        
    total_initial_sum = 0
    for x, a in stones:
        total_initial_sum += x * a
        
    total_final_sum = n * (n + 1) // 2
    ans = total_final_sum - total_initial_sum
    print(ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Input Parsing**: The input is read and parsed into integers N (number of cells), M (number of initial stone piles), X (positions of initial piles), and A (number of stones in each pile).
2. **Total Stones Check**: The sum of all stones in A must equal N; otherwise, it's impossible to distribute one stone per cell, so return -1.
3. **Leftmost Cell Check**: The first pile must be at cell 1; otherwise, cell 1 cannot be filled, so return -1.
4. **Segment Processing**: For each segment between consecutive piles, the available stones (current pile's stones plus carry-over from previous segments) must cover the gap (number of cells between piles). If not, return -1.
5. **Operations Calculation**: The total operations required is computed as the difference between the sum of final positions (1 to N) and the sum of initial positions (each stone's starting cell multiplied by its count). This works because each stone's movement cost is the difference between its final and initial positions.

This approach efficiently checks feasibility and computes the minimal operations using mathematical insights, ensuring optimal performance even for large inputs.
