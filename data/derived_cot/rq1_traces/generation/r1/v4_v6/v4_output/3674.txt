To solve this problem, we need to count the number of subarrays in a given array that can be made non-decreasing by performing at most `k` operations, where each operation increments an element by 1. The key insight is that for each subarray, the minimal number of operations required to make it non-decreasing can be computed using a greedy approach. 

### Approach
1. **Problem Analysis**: The problem requires checking every possible subarray of the given array to determine if it can be made non-decreasing with at most `k` operations. Each operation allows incrementing any element in the subarray by 1. The challenge is to efficiently compute the minimal operations required for each subarray without resorting to a brute-force O(n^2) approach, which would be infeasible for large arrays (n ≤ 10^5).

2. **Key Insight**: For any subarray starting at index `l` and ending at index `r`, the minimal operations required to make it non-decreasing can be computed by traversing the subarray from `l` to `r`. During this traversal:
   - If the current element is less than the previous maximum encountered, the deficit (difference between the previous maximum and the current element) is added to the total operations.
   - If the current element is greater than or equal to the previous maximum, the previous maximum is updated to the current element's value.
   
3. **Optimization**: To avoid O(n^2) complexity, we iterate over each starting index `l` and expand the subarray to the right (increasing `r`). For each expansion, we update the operations needed. If at any point the operations exceed `k`, we conclude that all larger subarrays starting at `l` will also be invalid (since adding more elements can only increase or maintain the operation count). Thus, we break early and count the remaining subarrays starting at `l` as invalid.

4. **Complexity Analysis**: The algorithm runs in O(n^2) in the worst case (e.g., a strictly decreasing array). However, early termination in many cases reduces the average complexity. For the worst-case scenario, optimizations like early termination help mitigate the computational burden.

### Solution Code
```python
class Solution:
    def countNonDecreasingSubarrays(self, nums: List[int], k: int) -> int:
        n = len(nums)
        total_subarrays = n * (n + 1) // 2
        invalid_count = 0
        
        for l in range(n):
            cur_max = nums[l]
            ops = 0
            for r in range(l, n):
                if r > l:
                    if nums[r] < cur_max:
                        ops += cur_max - nums[r]
                    else:
                        cur_max = nums[r]
                    if ops > k:
                        invalid_count += (n - r)
                        break
        return total_subarrays - invalid_count
```

### Explanation
1. **Initialization**: The total number of subarrays in an array of size `n` is `n*(n+1)/2`. We initialize `invalid_count` to zero, which will store the number of subarrays that cannot be made non-decreasing with at most `k` operations.
2. **Iterate Over Subarrays**: For each starting index `l`:
   - Initialize `cur_max` to the value at `l` and `ops` to zero.
   - Expand the subarray to the right (from `l` to `n-1`). For each new element at index `r`:
     - If the element is less than `cur_max`, add the deficit (`cur_max - nums[r]`) to `ops`.
     - If the element is greater than or equal to `cur_max`, update `cur_max` to this element's value.
     - If at any point `ops` exceeds `k`, all larger subarrays starting at `l` will also be invalid. Thus, we increment `invalid_count` by the number of remaining subarrays (from `r` to `n-1`) and break the inner loop.
3. **Result Calculation**: The result is the difference between the total number of subarrays and the count of invalid subarrays.

This approach efficiently checks subarrays while leveraging early termination to optimize performance in many cases, though it remains O(n^2) in the worst case.
