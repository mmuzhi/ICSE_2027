=== ANSWER ===
To solve this problem, we need to determine the number of stones each alien will have after N years, considering the process where each adult alien gives one stone to a newly adult alien if they have at least one stone. The challenge is to efficiently simulate this process for a large number of aliens (up to 500,000) without resorting to a brute-force approach.

### Approach
1. **Problem Analysis**: 
   - Each alien `i` becomes an adult exactly `i` years later.
   - When an alien becomes an adult, every existing adult alien with at least one stone gives one stone to the new adult.
   - The initial number of stones for each alien is given by the array `A`.
   - The goal is to compute the final number of stones each alien has after `N` years.

2. **Key Insight**:
   - The number of stones an alien `i` has right after becoming an adult (`T_i`) is `A_i` plus the number of adults (from aliens 1 to `i-1`) who had at least one stone at the beginning of year `i`.
   - The condition for an adult alien `k` (where `k < i`) to contribute a stone at year `i` is that `T_k + k > i - 1`. This is derived from the observation that the stone count of alien `k` at the beginning of year `i` is `T_k - (i - k - 1)`, which must be positive.
   - After becoming an adult, alien `i` will give one stone in each subsequent year until it runs out of stones. Thus, the final stones for alien `i` is `max(0, T_i - (N - i))`.

3. **Efficient Calculation**:
   - To efficiently compute `T_i` for each alien, we use a Fenwick Tree (Binary Indexed Tree) to maintain the count of aliens `k` (for `k < i`) such that `T_k + k > i - 1`.
   - The Fenwick Tree allows us to perform point updates and range queries in logarithmic time, making the solution feasible for large `N`.

### Solution Code
```python
import sys

MX = 1500000
fenw = [0] * (MX + 1)

def update_fenw(idx, delta):
    while idx <= MX:
        fenw[idx] += delta
        idx += idx & -idx

def query_fenw(idx):
    s = 0
    while idx:
        s += fenw[idx]
        idx -= idx & -idx
    return s

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1 + n]))
    
    res = []
    for i in range(1, n + 1):
        if i == 1:
            count = 0
        else:
            cnt_le = query_fenw(i - 1)
            count = (i - 1) - cnt_le
        T_i = A[i - 1] + count
        F_i = T_i - (n - i)
        if F_i < 0:
            F_i = 0
        D_i = T_i + i
        if D_i <= MX:
            update_fenw(D_i, 1)
        res.append(str(F_i))
    
    print(" ".join(res))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Fenwick Tree Setup**: The Fenwick Tree (`fenw`) is initialized to keep track of the frequencies of values `D_k = T_k + k` for aliens processed so far. This helps in efficiently querying the number of aliens `k` (for `k < i`) that satisfy `D_k > i - 1`.
2. **Processing Each Alien**: For each alien `i` (from 1 to N):
   - **Count Calculation**: If `i > 1`, the number of contributing adults is calculated as `(i - 1) - query_fenw(i - 1)`, where `query_fenw(i - 1)` gives the count of aliens `k` with `D_k <= i - 1`.
   - **T_i Calculation**: `T_i` is set to `A[i - 1] + count`, representing the stones alien `i` has after becoming an adult.
   - **Final Stones Calculation**: The final stones for alien `i` is `max(0, T_i - (N - i))`, accounting for stones given in subsequent years.
   - **Update Fenwick Tree**: `D_i = T_i + i` is updated in the Fenwick Tree for future queries.
3. **Output**: The results for all aliens are printed as a space-separated string.

This approach efficiently handles the large input size by leveraging the Fenwick Tree for logarithmic time updates and queries, ensuring optimal performance.
