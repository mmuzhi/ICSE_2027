=== ANSWER ===
To solve this problem, we need to transform an integer array by first replacing each even number with 0 and each odd number with 1, and then sorting the resulting array in non-decreasing order. The solution involves two straightforward steps: transformation and sorting.

### Approach
1. **Transformation**: Iterate through each element in the input array. For each element, check if it is even or odd. Replace even elements with 0 and odd elements with 1.
2. **Sorting**: After transforming all elements, sort the resulting array. Since the transformed array contains only 0s and 1s, sorting will group all 0s followed by all 1s.

### Solution Code
```python
class Solution:
    def transformArray(self, nums: List[int]) -> List[int]:
        transformed = []
        for num in nums:
            if num % 2 == 0:
                transformed.append(0)
            else:
                transformed.append(1)
        transformed.sort()
        return transformed
```

### Explanation
1. **Transformation Step**: The code processes each element in the input array `nums`. For each element, it checks if the number is even (using `num % 2 == 0`). If true, it appends 0 to the `transformed` list; otherwise, it appends 1.
2. **Sorting Step**: After processing all elements, the `transformed` list is sorted. Sorting ensures that all 0s (from even numbers) come before all 1s (from odd numbers), resulting in a non-decreasing order array.
3. **Result**: The sorted list is returned as the final output.

This approach efficiently combines element-wise transformation and sorting to meet the problem's requirements. The complexity is dominated by the sorting step, which is \(O(n \log n)\) for an array of length \(n\). Given the constraints (array length up to 100), this solution is optimal and straightforward.
