Output: false
Explanation:

Since 4 is not equal to 6, num is not balanced.

The sum of digits at even indices is 2 + 1 + 3 == 6, and the sum of digits at odd indices is 4 + 2 == 6.
Since both are equal the num is balanced.

=== ANSWER ===
To solve this problem, we need to determine if a given string of digits is balanced. A string is considered balanced if the sum of the digits at even indices (0-indexed) equals the sum of the digits at odd indices. 

### Approach
1. **Initialization**: We start by initializing two variables, `even_sum` and `odd_sum`, to zero. These variables will accumulate the sums of digits at even and odd indices, respectively.
2. **Iteration**: We iterate over each character in the input string. For each character, we convert it to an integer.
3. **Index Check**: For each index, if the index is even (i.e., divisible by 2), we add the digit to `even_sum`. Otherwise, we add it to `odd_sum`.
4. **Comparison**: After processing all characters, we compare `even_sum` and `odd_sum`. If they are equal, the string is balanced, and we return `True`; otherwise, we return `False`.

### Solution Code
```python
class Solution:
    def isBalanced(self, num: str) -> bool:
        even_sum = 0
        odd_sum = 0
        for i, char in enumerate(num):
            digit = int(char)
            if i % 2 == 0:
                even_sum += digit
            else:
                odd_sum += digit
        return even_sum == odd_sum
```

### Explanation
- **Initialization**: The variables `even_sum` and `odd_sum` are initialized to zero to start accumulating the sums.
- **Iteration and Index Handling**: Using `enumerate`, we loop through each character in the string. For each character, we convert it to an integer. The index of each character is checked to determine if it is even or odd. If the index is even, the digit is added to `even_sum`; otherwise, it is added to `odd_sum`.
- **Result Comparison**: After processing all digits, we check if `even_sum` equals `odd_sum`. If they are equal, the function returns `True`, indicating the string is balanced; otherwise, it returns `False`.

This approach efficiently checks the balance of the string by processing each digit once, leading to an optimal solution with a time complexity of O(n), where n is the length of the string. The space complexity is O(1) as we only use a constant amount of extra space.
