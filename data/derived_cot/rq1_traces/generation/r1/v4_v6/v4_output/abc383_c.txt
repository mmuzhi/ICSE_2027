You will be given a competitive programming problem.
The AtCoder company office is represented as a grid of H rows and W columns. Let (i, j) denote the cell at the i-th row from the top and j-th column from the left.
The state of each cell is represented by a character S_{i,j}. If S_{i,j} is #, that cell has a wall; if S_{i,j} is ., that cell is a floor; if S_{i,j} is H, that cell has a humidifier placed on a floor cell.
A certain cell is considered humidified if it can be reached from at least one humidifier cell by at most D moves up, down, left, or right without passing through a wall. Note that any cell with a humidifier is always humidified.
Find the number of humidified floor cells.

Input

The input is given from Standard Input in the following format:
H W D
S_{1,1}S_{1,2}\cdotsS_{1,W}
S_{2,1}S_{2,2}\cdotsS_{2,W}
\vdots
S_{H,1}S_{H,2}\cdotsS_{H,W}

Output

Print the answer.

Constraints


- 1 \leq H \leq 1000
- 1 \leq W \leq 1000
- 0 \leq D \leq H\times W
- S_{i,j} is #, ., or H. (1 \leq i \leq H, 1 \leq j \leq W)
- All input numbers are integers.

=== ANSWER ===
To solve this problem, we need to determine the number of humidified floor cells in a grid where a cell is considered humidified if it can be reached from at least one humidifier cell in at most D moves without passing through any walls. The solution involves using a multi-source BFS (Breadth-First Search) to efficiently compute the minimal distance from any humidifier to each floor cell and then count the cells within the distance constraint.

### Approach
1. **Problem Analysis**: The grid consists of cells that can be walls ('#'), floors ('.'), or humidifiers ('H'). A humidified cell is any floor cell (including those with humidifiers) that is reachable from a humidifier within D moves (up, down, left, or right) without passing through walls.
2. **Intuition**: The problem can be modeled as a shortest path problem from multiple sources (humidifiers) to all reachable floor cells. BFS is suitable here because it naturally explores all nodes level by level, which helps in computing the minimal distances efficiently.
3. **Algorithm Selection**: Multi-source BFS is chosen because it starts from all humidifier cells simultaneously and explores neighboring cells in a breadth-first manner. This ensures that each cell's minimal distance from the nearest humidifier is recorded.
4. **Complexity Analysis**: The algorithm processes each cell in the grid once, leading to a time complexity of O(H*W), where H and W are the grid dimensions. This is efficient given the constraints (H and W up to 1000).
5. **Implementation**: 
   - Read input values and grid.
   - Initialize a distance matrix with a large value (indicating unvisited cells).
   - Enqueue all humidifier cells with distance 0.
   - Perform BFS: for each cell dequeued, check its neighbors. If a neighbor is a floor cell and unvisited, update its distance and enqueue it.
   - After BFS, count all floor cells (including humidifiers) with distances ≤ D.

### Solution Code
```python
import collections

def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data: 
        print(0)
        return
        
    H, W, D = map(int, data[0].split())
    grid = []
    for i in range(1, 1+H):
        grid.append(data[i].strip())
    
    dist = [[10**9] * W for _ in range(H)]
    q = collections.deque()
    dirs = [(1,0), (-1,0), (0,1), (0,-1)]
    
    for i in range(H):
        for j in range(W):
            if grid[i][j] == 'H':
                dist[i][j] = 0
                q.append((i, j))
    
    while q:
        i, j = q.popleft()
        for dx, dy in dirs:
            ni, nj = i + dx, j + dy
            if 0 <= ni < H and 0 <= nj < W and grid[ni][nj] != '#' and dist[ni][nj] == 10**9:
                dist[ni][nj] = dist[i][j] + 1
                q.append((ni, nj))
    
    ans = 0
    for i in range(H):
        for j in range(W):
            if grid[i][j] != '#' and dist[i][j] <= D:
                ans += 1
                
    print(ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line contains grid dimensions (H, W) and the maximum allowed distance (D). Subsequent lines represent the grid.
2. **Initialization**: A distance matrix `dist` is initialized to a large value (10^9) to indicate unvisited cells. A queue is initialized to manage BFS traversal.
3. **Multi-source BFS Setup**: All humidifier cells ('H') are identified, their distances set to 0, and they are enqueued.
4. **BFS Execution**: For each cell dequeued, its neighbors are checked. Valid neighbors (within grid bounds, not walls, and unvisited) have their distances updated and are enqueued.
5. **Counting Humidified Cells**: After BFS, all floor cells (including humidifiers) with distances ≤ D are counted and the result is printed.

This approach efficiently computes the minimal distances from any humidifier to all reachable floor cells and counts those within the specified distance constraint, providing an optimal solution to the problem.
