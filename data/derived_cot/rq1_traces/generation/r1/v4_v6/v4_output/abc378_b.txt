=== PROMPT ===
You will be given a competitive programming problem.
In AtCoder City, N types of garbage are collected regularly. The i-th type of garbage (i=1,2,\dots,N) is collected on days when the date modulo q_i equals r_i.
Answer Q queries. In the j-th query (j=1,2,\dots,Q), given that the t_j-th type of garbage is put out on day d_j, answer the next day on which it will be collected.
Here, if the i-th type of garbage is put out on a day when that type of garbage is collected, then the garbage will be collected on the same day.

Input

The input is given from Standard Input in the following format:
N
q_1 r_1
q_2 r_2
\vdots
q_N r_N
Q
t_1 d_1
t_2 d_2
\vdots
t_Q d_Q

Output

Print Q lines. The j-th line (1\leq j \leq Q) should contain the answer to the j-th query.

Constraints


- 1 \leq N \leq 100
- 0 \leq r_i < q_i \leq 10^9
- 1 \leq Q \leq 100
- 1 \leq t_j \leq N
- 1 \leq d_j \leq 10^9
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to determine the next collection day for a specific type of garbage after a given day. The garbage collection days for each type are defined by a modulo condition: for type \(i\), the garbage is collected on days \(d\) where \(d \mod q_i = r_i\). 

### Approach
1. **Problem Analysis**: For each query, we are given a type \(t_j\) and a day \(d_j\). We need to find the smallest day \(x \geq d_j\) such that \(x \mod q_{t_j} = r_{t_j}\).
2. **Key Insight**: The solution involves computing the remainder of \(d_j\) when divided by \(q_{t_j}\) (denoted as \(r_0\)). Based on whether \(r_0\) is less than, equal to, or greater than \(r_{t_j}\), we can determine the next collection day:
   - If \(r_0 \leq r_{t_j}\), the next collection day is \(d_j - r_0 + r_{t_j}\).
   - If \(r_0 > r_{t_j}\), the next collection day is \((d_j // q_{t_j} + 1) \times q_{t_j} + r_{t_j}\).
3. **Algorithm Selection**: The approach efficiently computes the next collection day using arithmetic operations, leveraging the properties of modular arithmetic and integer division.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    garbage = []
    index = 1
    for _ in range(n):
        q = int(data[index])
        r = int(data[index + 1])
        index += 2
        garbage.append((q, r))
    
    q_count = int(data[index])
    index += 1
    queries = []
    for _ in range(q_count):
        t = int(data[index])
        d = int(data[index + 1])
        index += 2
        queries.append((t, d))
    
    results = []
    for t, d in queries:
        q_val, r_val = garbage[t - 1]
        k0 = d // q_val
        r0 = d % q_val
        if r0 <= r_val:
            next_day = k0 * q_val + r_val
        else:
            next_day = (k0 + 1) * q_val + r_val
        results.append(str(next_day))
    
    print("\n".join(results))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first value is \(N\), the number of garbage types. Each subsequent line provides \(q_i\) and \(r_i\) for each type. Following this, \(Q\) (the number of queries) is read, and each query consists of a type \(t_j\) and a day \(d_j\).
2. **Processing Queries**: For each query, the corresponding \(q_{t_j}\) and \(r_{t_j}\) are retrieved. The algorithm then computes \(k_0\) (the quotient of \(d_j\) divided by \(q_{t_j}\)) and \(r_0\) (the remainder).
3. **Determining Next Collection Day**: 
   - If \(r_0 \leq r_{t_j}\), the next collection day is \(k_0 \times q_{t_j} + r_{t_j}\).
   - Otherwise, the next collection day is \((k_0 + 1) \times q_{t_j} + r_{t_j}\).
4. **Output**: The results for all queries are printed in sequence.

This approach efficiently handles each query in constant time, making it optimal for the given constraints. The solution leverages basic arithmetic operations to determine the next valid collection day without any complex computations or data structures.
