You will be given a competitive programming problem.
You are given a simple connected undirected graph with N vertices numbered 1 through N and M edges numbered 1 through M. Edge i connects vertices u_i and v_i, and has a label w_i.
Among all simple paths (paths that do not pass through the same vertex more than once) from vertex 1 to vertex N, find the minimum XOR of the labels of the edges on the path.

Notes on XOR

For non-negative integers A and B, their XOR A \oplus B is defined as follows:


- In the binary representation of A \oplus B, the digit in the place corresponding to 2^k \,(k \ge 0) is 1 if and only if exactly one of the digits in the same place of A and B is 1; otherwise, it is 0.


For example, 3 \oplus 5 = 6 (in binary: 011 \oplus 101 = 110).

In general, the XOR of k integers p_1, \dots, p_k is defined as (\cdots ((p_1 \oplus p_2) \oplus p_3) \oplus \cdots \oplus p_k). It can be proved that it does not depend on the order of p_1, \dots, p_k.

Input

The input is given from Standard Input in the following format:
N M
u_1 v_1 w_1
u_2 v_2 w_2
\vdots
u_M v_M w_M

Output

Print the answer.

Constraints


- 2 \leq N \leq 10
- N-1 \leq M \leq \frac{N(N-1)}{2}
- 1 \leq u_i < v_i \leq N
- 0 \leq w_i < 2^{60}
- The given graph is a simple connected undirected graph.
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to find the minimum XOR of the labels of the edges along any simple path from vertex 1 to vertex N in a given connected undirected graph. A simple path is one that does not revisit any vertex. The XOR operation is commutative and associative, meaning the order of edges in the path does not affect the result. 

### Approach
1. **Problem Analysis**: The problem involves traversing all simple paths from vertex 1 to vertex N in a graph and computing the XOR of the edge labels along each path. The goal is to find the minimum XOR value among all such paths. Given the constraints (N ≤ 10), a brute-force approach is feasible, as the number of simple paths is manageable (up to approximately 109,601 for N=10 in the worst case).

2. **Intuition**: Since XOR operations are commutative and associative, the total XOR value for a path depends only on the set of edges traversed, not their order. This allows us to explore all possible paths without worrying about the sequence of edges, simplifying the traversal.

3. **Algorithm Selection**: We use a Depth-First Search (DFS) approach to explore all simple paths from vertex 1 to vertex N. Each state in the DFS is represented by:
   - The current vertex.
   - A bitmask indicating which vertices have been visited (to ensure no revisits).
   - The cumulative XOR value of the edges traversed so far.

4. **Optimization**: By using a bitmask to track visited vertices, we efficiently manage state transitions and avoid cycles. The DFS stack processes each state, updating the minimum XOR value whenever the destination vertex (N) is reached.

5. **Complexity Analysis**: For N vertices, the number of possible states is O(N * 2^N), but the actual number of paths is much less in practice. The worst-case scenario (complete graph) results in approximately 109,601 paths for N=10, which is computationally feasible.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    graph = [[] for _ in range(n)]
    idx = 2
    for _ in range(m):
        u = int(data[idx])
        v = int(data[idx+1])
        w = int(data[idx+2])
        idx += 3
        u -= 1
        v -= 1
        graph[u].append((v, w))
        graph[v].append((u, w))
    
    min_xor = None
    stack = [(0, 1, 0)]
    while stack:
        node, mask, xor_val = stack.pop()
        if node == n - 1:
            if min_xor is None or xor_val < min_xor:
                min_xor = xor_val
        for neighbor, weight in graph[node]:
            if mask & (1 << neighbor) == 0:
                new_mask = mask | (1 << neighbor)
                new_xor = xor_val ^ weight
                stack.append((neighbor, new_mask, new_xor))
    
    print(min_xor)

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input is read and parsed into the number of vertices (N), edges (M), and the graph structure. Each edge is stored in an adjacency list for both vertices it connects.
2. **Initialization**: The DFS stack starts at vertex 0 (representing vertex 1) with an initial bitmask (indicating vertex 0 is visited) and an initial XOR value of 0.
3. **DFS Traversal**: For each state (current vertex, visited bitmask, current XOR value):
   - If the current vertex is the destination (N-1), the XOR value is compared to the current minimum.
   - For each unvisited neighbor, a new state is pushed onto the stack with the updated bitmask and XOR value (current XOR ^ edge weight).
4. **Output**: After processing all paths, the minimum XOR value encountered is printed.

This approach efficiently explores all simple paths using DFS with bitmask state tracking, ensuring optimal performance for the given constraints.
