Takahashi has two ways of moving. If he is on the X-th floor (1\leq X\leq F_{i,j}) of the building in block (i,j), he can:

You are given Q queries to be answered. The i-th query (1\leq i\leq Q) is the following.

Find the minimum possible number of times that Takahashi uses stairs to move from the Y_i-th floor of the building in block (A_i,B_i) to the Z_i-th floor of the building in block (C_i,D_i).
The count of times using stairs is incremented each time he moves up or down one floor, possibly multiple times within the same building. (For example, moving from the 1st floor to the 6th floor of a building counts as 5 uses of stairs.)
Note that he does not have to minimize the number of times he uses walkways.

To solve this problem, we need to find the minimum number of times Takahashi uses stairs to move from a starting floor in one building to a destination floor in another building in a grid of buildings. The movement between buildings is constrained by the height of adjacent buildings and the current floor Takahashi is on. 

### Approach
1. **Problem Analysis**: The problem involves navigating a grid of buildings where each building has a certain number of floors. Takahashi can move between adjacent buildings only if the destination building has at least the same number of floors as his current floor. The goal is to minimize the number of stairs used (vertical movement within buildings) while allowing unlimited use of walkways (horizontal movement between adjacent buildings at the same floor).

2. **Key Insight**: The minimal stairs cost depends on the path taken between the start and destination buildings. The cost is determined by the absolute difference between the start and destination floors if all buildings along the path have sufficient height. Otherwise, the cost includes an additional penalty for dipping to the minimum building height along the path.

3. **Algorithm Selection**: 
   - **Union-Find Data Structure (DSU)**: Used to efficiently manage and merge sets of buildings as we process them in descending order of their heights. This helps in determining the maximum threshold (minimum building height) for which the start and destination buildings are connected.
   - **Offline Processing**: Process queries by sorting buildings by height in descending order and using DSU to connect adjacent buildings. For each query, the bottleneck (minimum building height on the path) is recorded when the start and destination buildings become connected in the DSU.

4. **Complexity Analysis**: 
   - **Time Complexity**: Sorting the buildings takes \(O(HW \log(HW))\). Processing each building and performing union operations with neighbors takes nearly linear time due to path compression and union by rank in DSU. Each query is processed in constant time after the initial setup, leading to an overall complexity of \(O(HW \log(HW) + Q)\).
   - **Space Complexity**: \(O(HW + Q)\) for storing the grid, DSU structures, and queries.
