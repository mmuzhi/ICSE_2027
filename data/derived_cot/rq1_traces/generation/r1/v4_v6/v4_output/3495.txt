=== PROMPT ===
You will be given a competitive programming problem.
There is an infinite 2D plane.
You are given a positive integer k. You are also given a 2D array queries, which contains the following queries:

queries[i] = [x, y]: Build an obstacle at coordinate (x, y) in the plane. It is guaranteed that there is no obstacle at this coordinate when this query is made.

After each query, you need to find the distance of the k^th nearest obstacle from the origin.
Return an integer array results where results[i] denotes the k^th nearest obstacle after query i, or results[i] == -1 if there are less than k obstacles.
Note that initially there are no obstacles anywhere.
The distance of an obstacle at coordinate (x, y) from the origin is given by |x| + |y|.
 
Example 1:

Input: queries = [[1,2],[3,4],[2,3],[-3,0]], k = 2
Output: [-1,7,5,3]
Explanation:

Initially, there are 0 obstacles.
After queries[0], there are less than 2 obstacles.
After queries[1], there are obstacles at distances 3 and 7.
After queries[2], there are obstacles at distances 3, 5, and 7.
After queries[3], there are obstacles at distances 3, 3, 5, and 7.


Example 2:

Input: queries = [[5,5],[4,4],[3,3]], k = 1
Output: [10,8,6]
Explanation:

After queries[0], there is an obstacle at distance 10.
After queries[1], there are obstacles at distances 8 and 10.
After queries[2], there are obstacles at distances 6, 8, and 10.


 
Constraints:

1 <= queries.length <= 2 * 10^5
All queries[i] are unique.
-10^9 <= queries[i][0], queries[i][1] <= 10^9
1 <= k <= 10^5

=== ANSWER ===
To solve this problem, we need to efficiently maintain the k-th smallest distance from the origin (0, 0) for obstacles added sequentially via queries. The distance of an obstacle at coordinate (x, y) is calculated as |x| + |y|. After each query, if there are at least k obstacles, we return the k-th smallest distance; otherwise, we return -1.

### Approach
1. **Heap Data Structure**: We use a max-heap (simulated using a min-heap with negative values) to keep track of the k smallest distances encountered so far. The max-heap allows us to efficiently access and remove the largest element among the k smallest elements when necessary.
2. **Processing Queries**: For each query (x, y):
   - Calculate the distance d = |x| + |y|.
   - Push the negative of this distance onto the heap to maintain it as a max-heap.
   - If the number of obstacles exceeds k, remove the largest element (smallest negative) from the heap to ensure only the k smallest distances remain.
   - If the number of obstacles is less than k, append -1 to the results. Otherwise, append the current k-th smallest distance, which is the negative of the top element of the heap.

### Solution Code
```python
import heapq

class Solution:
    def resultsArray(self, queries: list[list[int]], k: int) -> list[int]:
        heap = []
        results = []
        n = 0
        for x, y in queries:
            d = abs(x) + abs(y)
            heapq.heappush(heap, -d)
            n += 1
            if n > k:
                heapq.heappop(heap)
            if n < k:
                results.append(-1)
            else:
                results.append(-heap[0])
        return results
```

### Explanation
1. **Initialization**: We start with an empty heap and an empty results list. The variable `n` tracks the number of obstacles processed.
2. **Processing Each Query**:
   - **Distance Calculation**: For each coordinate (x, y), compute the Manhattan distance d = |x| + |y|.
   - **Heap Update**: Push the negative of d onto the heap to simulate a max-heap. If the number of obstacles `n` exceeds k, remove the largest element (using `heappop`) to maintain only the k smallest distances.
   - **Result Determination**: If `n` is less than k, append -1 to the results. Otherwise, the k-th smallest distance is the negative of the heap's top element (accessed as `heap[0]`), which is appended to the results.
3. **Efficiency**: The heap operations (push and pop) run in O(log k) time per query. With m queries, the total time complexity is O(m log k), which is efficient for the given constraints (m ≤ 2×10^5 and k ≤ 10^5). The space complexity is O(k) for storing the heap.
