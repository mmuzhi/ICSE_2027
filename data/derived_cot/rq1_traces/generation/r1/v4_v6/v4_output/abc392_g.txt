### Approach
1. **Problem Analysis**: The problem requires finding all ordered triplets (A, B, C) in a set S of distinct integers where A < B < C and 2*B = A + C. The key insight is that for each valid triplet, the sum of A and C must be even, and the midpoint B (which is (A + C)/2) must exist in the set S.

2. **Intuition**: For each possible even sum T = A + C, we can determine B as T/2. If B exists in S, then all pairs (A, C) that sum to T (with A < C) will form valid triplets (A, B, C). 

3. **Algorithm Selection**: 
   - **Frequency Array**: We first create a frequency array `F` where `F[x] = 1` if `x` is present in S, otherwise `0`.
   - **Convolution using FFT**: To efficiently count the number of ordered pairs (A, C) such that A + C = T, we use the Fast Fourier Transform (FFT) to compute the convolution of the frequency array with itself. The convolution at index T gives the count of ordered pairs (A, C) where A + C = T.
   - **Counting Valid Triplets**: For each even T, the number of unordered pairs (A, C) (with A < C) is half the convolution value (since ordered pairs are counted twice in the convolution). If T/2 exists in S, we add this count to our result.

4. **Complexity Analysis**: 
   - **FFT Complexity**: The FFT of an array of size N runs in O(N log N) time. Here, the size of the frequency array is 2*max_val (2,000,000), padded to the nearest power of two (4,194,304). Thus, the FFT operations dominate the time complexity, which is O(N log N) where N is approximately 4 million.
   - **Overall Complexity**: The preprocessing steps (reading input and setting up the frequency array) are O(N), and the convolution step is O(N log N). The final loop over T is O(N), making the overall complexity O(N log N).
