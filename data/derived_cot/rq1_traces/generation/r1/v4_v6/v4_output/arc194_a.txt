For each i = 1, 2, \ldots, N in this order, you perform exactly one of the following two operations:

- Append A_i as an element to the end of S.
- Delete the last element of S. You cannot choose this operation if S is empty.

Print the maximum possible value of the sum of the elements of S after all operations.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the answer.

Constraints


- 1 \leq N \leq 2 \times 10^5
- -10^9 \leq A_i \leq 10^9
- All input values are integers.

Sample Input 1

6
3 -1 -4 5 -9 2

Sample Output 1

8

- For i = 1, append A_1 = 3 to the end of S. Now, S = (3).

- For i = 5, append A_5 = -9 to the end of S. Now, S = (3, 5, -9).
- For i = 6, delete the last element of S. Now, S = (3, 5).

Here, the sum of the elements of S after all operations is 3 + 5 = 8, which is the maximum possible value.

The operations allowed are either appending the current element to `S` or deleting the last element of `S` (provided `S` is not empty). The challenge is to choose the optimal sequence of operations to maximize the sum of the elements in `S` after processing all elements of `A`.

### Approach
1. **Problem Analysis**: The problem requires processing each element of the sequence `A` in order, deciding for each element whether to append it to `S` or pop the last element from `S`. The goal is to maximize the sum of `S` after all operations. The key insight is that we should avoid retaining negative elements in `S` unless they are necessary to allow future positive elements to be included in the sum.

2. **Intuition**: We can use a greedy approach where we always append positive elements to `S` and handle negative elements by either appending them conditionally or popping them immediately if they cause the sum of `S` to become negative. Specifically:
   - For each element in `A`, if it is non-negative, we append it to `S` and add its value to the current sum.
   - If the element is negative, we still append it but then check if the current sum becomes negative. If it does, we pop the smallest (most negative) elements from `S` until the sum is non-negative again. This ensures that the sum of `S` remains as large as possible.

3. **Algorithm Selection**: The algorithm involves iterating through each element in `A`, maintaining a min-heap (priority queue) to keep track of the elements in `S` for efficient removal of the smallest elements when necessary. The min-heap allows us to quickly access and remove the smallest element in `S` to restore the sum to a non-negative state.

4. **Complexity Analysis**: The algorithm processes each element once, and each element is pushed and popped from the heap at most once. The heap operations (insertion and extraction) take O(log n) time, leading to an overall time complexity of O(n log n). The space complexity is O(n) to store the heap.
