=== ANSWER ===
To solve this problem, we need to split an array into three non-empty contiguous subarrays such that the sum of the distinct counts in each subarray is maximized. The challenge lies in efficiently computing the distinct counts for all possible splits without resorting to a brute-force approach, which would be computationally infeasible for large inputs.

### Approach
1. **Prefix and Suffix Arrays**: 
   - **Prefix Array (L)**: Compute the distinct count from the start of the array up to each index. This helps in quickly determining the distinct count in the left subarray for any split.
   - **Suffix Array (R)**: Compute the distinct count from each index to the end of the array. This helps in quickly determining the distinct count in the right subarray for any split.

2. **Next Occurrence Array**:
   - For each element in the array, precompute the next occurrence of the same element. This helps in efficiently updating the distinct counts when elements are removed from consideration in the middle subarray.

3. **Segment Tree for Range Queries and Updates**:
   - **Initialization**: Build a segment tree where each leaf node represents the sum of the distinct counts of the left subarray (up to a certain index) and the distinct counts of the right subarray (from the next index to the end). This initial setup helps in evaluating potential splits.
   - **Updates and Queries**: As we iterate through each possible split point for the left subarray, we update the segment tree to reflect the removal of the current element. This involves decrementing the distinct counts in the affected range (from the current index to the next occurrence of the same element). After updating, we query the segment tree to find the maximum sum of distinct counts for the middle and right subarrays starting from the next index.

4. **Maximizing the Sum**:
   - For each possible split point of the left subarray, combine the distinct count of the left subarray with the maximum value obtained from the segment tree query (which gives the best sum of distinct counts for the middle and right subarrays). Track the maximum sum encountered during this process.

### Solution Code
```python
import sys
sys.setrecursionlimit(3000000)

class SegmentTree:
    def __init__(self, data):
        self.n = len(data)
        self.size = 1
        while self.size < self.n:
            self.size *= 2
        self.tree = [-10**18] * (2 * self.size)
        self.lazy = [0] * (2 * self.size)
        for i in range(self.n):
            self.tree[self.size + i] = data[i]
        for i in range(self.n, self.size):
            self.tree[self.size + i] = -10**18
        for i in range(self.size-1, 0, -1):
            self.tree[i] = max(self.tree[2*i], self.tree[2*i+1])
    
    def push(self, i):
        if self.lazy[i] != 0:
            self.tree[2*i] += self.lazy[i]
            self.tree[2*i+1] += self.lazy[i]
            self.lazy[2*i] += self.lazy[i]
            self.lazy[2*i+1] += self.lazy[i]
            self.lazy[i] = 0

    def update_range(self, l, r, val, i=1, seg_l=0, seg_r=None):
        if seg_r is None:
            seg_r = self.size - 1
        if r < seg_l or l > seg_r:
            return
        if l <= seg_l and seg_r <= r:
            self.tree[i] += val
            self.lazy[i] += val
            return
        self.push(i)
        mid = (seg_l + seg_r) // 2
        if l <= mid:
            self.update_range(l, r, val, 2*i, seg_l, mid)
        if r > mid:
            self.update_range(l, r, val, 2*i+1, mid+1, seg_r)
        self.tree[i] = max(self.tree[2*i], self.tree[2*i+1])
    
    def query_range(self, l, r, i=1, seg_l=0, seg_r=None):
        if seg_r is None:
            seg_r = self.size - 1
        if r < seg_l or l > seg_r:
            return -10**18
        if l <= seg_l and seg_r <= r:
            return self.tree[i]
        self.push(i)
        mid = (seg_l + seg_r) // 2
        left_res = self.query_range(l, r, 2*i, seg_l, mid)
        right_res = self.query_range(l, r, 2*i+1, mid+1, seg_r)
        return max(left_res, right_res)

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    L = [0] * n
    freq = [0] * (n+1)
    distinct = 0
    for i in range(n):
        x = A[i]
        freq[x] += 1
        if freq[x] == 1:
            distinct += 1
        L[i] = distinct
        
    R = [0] * (n+1)
    freq = [0] * (n+1)
    distinct = 0
    for i in range(n-1, -1, -1):
        x = A[i]
        freq[x] += 1
        if freq[x] == 1:
            distinct += 1
        R[i] = distinct
    R[n] = 0
    
    next_occ = [n] * n
    last_occ = {}
    for i in range(n):
        x = A[i]
        if x in last_occ:
            prev_i = last_occ[x]
            next_occ[prev_i] = i
        last_occ[x] = i
    
    V = [-10**18] * n
    for j in range(n-1):
        V[j] = L[j] + R[j+1]
    
    seg_tree = SegmentTree(V)
    ans = 0
    for i in range(n-2):
        r = next_occ[i] - 1
        seg_tree.update_range(i, min(r, n-1), -1)
        current_max = seg_tree.query_range(i+1, n-2)
        candidate = L[i] + current_max
        if candidate > ans:
            ans = candidate
    print(ans)

if __name__ == "__main__":
    main()
```

1. **Prefix and Suffix Arrays**: The prefix array `L` stores the distinct counts from the start of the array up to each index, while the suffix array `R` stores the distinct counts from each index to the end. These arrays allow quick lookup of distinct counts for any left or right subarray.
2. **Next Occurrence Array**: This array helps in efficiently updating the segment tree by indicating the next position where each element reappears. This is crucial for adjusting the distinct counts in the middle subarray when elements are removed from consideration.
3. **Segment Tree**: The segment tree is initialized with values representing the sum of distinct counts for potential middle and right subarrays. As we iterate through each possible left subarray split, the tree is updated to reflect the removal of the current element, and queries are made to find the optimal split for the remaining subarrays.
4. **Efficiency**: The approach efficiently computes the solution in O(n log n) time using the segment tree for range updates and queries, making it suitable for large input sizes. The solution avoids the O(n^2) complexity of a brute-force approach by leveraging precomputed arrays and efficient data structures.
