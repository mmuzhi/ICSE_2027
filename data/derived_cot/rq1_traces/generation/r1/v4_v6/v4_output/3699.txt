To solve this problem, we need to count the number of special subsequences in an array of positive integers. A special subsequence is defined by four indices (p, q, r, s) such that p < q < r < s, with at least one element between each consecutive pair (i.e., q - p > 1, r - q > 1, and s - r > 1). Additionally, the elements at these indices must satisfy the condition nums[p] * nums[r] = nums[q] * nums[s].

### Approach
1. **Problem Analysis**: The problem requires finding all valid quadruples (p, q, r, s) in the array where the indices are strictly increasing with gaps of at least one element between each pair. The product of elements at p and r must equal the product of elements at q and s. Given the constraints (array length up to 1000), a brute-force O(n^4) solution is infeasible. Instead, we use a more efficient approach by leveraging preprocessing and nested loops with dictionary lookups.

2. **Preprocessing**:
   - **Left Dictionaries (left_dicts)**: For each index `q`, `left_dicts[q]` stores the frequency of elements from the start of the array up to `q-2` (ensuring q - p > 1). This helps in quickly accessing possible values for `nums[p]` when paired with a given `q`.
   - **Right Dictionaries (right_dicts)**: For each index `r`, `right_dicts[r]` stores the frequency of elements from `r+2` to the end of the array (ensuring s - r > 1). This helps in quickly accessing possible values for `nums[s]` when paired with a given `r`.

3. **Counting Valid Quadruples**: For each valid pair (q, r) where `q` ranges from 2 to `n-3` and `r` ranges from `q+2` to `n-2`, we iterate over all distinct values of `s_val` in `right_dicts[r]`. For each `s_val`, we check if the product `nums[q] * s_val` is divisible by `nums[r]`. If it is, we compute the required value `x = (nums[q] * s_val) // nums[r]` and check its frequency in `left_dicts[q]`. The total count of valid quadruples is incremented by the product of the frequencies of `x` in `left_dicts[q]` and `s_val` in `right_dicts[r]`.

### Solution Code
```python
class Solution:
    def numberOfSubsequences(self, nums: list) -> int:
        n = len(nums)
        left_dicts = [dict() for _ in range(n)]
        left_dicts[0] = {}
        left_dicts[1] = {}
        cur = {}
        for q in range(2, n):
            cur[nums[q-2]] = cur.get(nums[q-2], 0) + 1
            left_dicts[q] = cur.copy()
        
        right_dicts = [dict() for _ in range(n)]
        cur = {}
        for r in range(n-1, -1, -1):
            right_dicts[r] = cur.copy()
            if r + 1 < n:
                cur[nums[r+1]] = cur.get(nums[r+1], 0) + 1
        
        total = 0
        for q in range(2, n-2):
            for r in range(q+2, n-1):
                for s_val, cnt_s in right_dicts[r].items():
                    product = nums[q] * s_val
                    if product % nums[r] == 0:
                        x = product // nums[r]
                        cnt_p = left_dicts[q].get(x, 0)
                        total += cnt_p * cnt_s
        return total
```

### Explanation
1. **Preprocessing Left Dictionaries**: The `left_dicts` array is built by iterating from the start of the array. For each index `q` (starting from 2), it accumulates the frequency of elements from the start up to `q-2`. This allows O(1) access to the frequency of any element that could be `nums[p]` for a given `q`.
2. **Preprocessing Right Dictionaries**: The `right_dicts` array is built by iterating from the end of the array. For each index `r`, it stores the frequency of elements from `r+2` to the end of the array. This allows O(1) access to the frequency of any element that could be `nums[s]` for a given `r`.
3. **Counting Valid Quadruples**: The nested loops iterate over all valid pairs (q, r). For each pair, the inner loop checks all possible `s_val` values from `right_dicts[r]`. If the product `nums[q] * s_val` is divisible by `nums[r]`, it calculates the required `x` value (which would be `nums[p]`) and checks its frequency in `left_dicts[q]`. The total count is incremented by the product of the frequencies of `x` and `s_val`.

This approach efficiently reduces the problem to manageable nested loops with dictionary lookups, leveraging preprocessing to avoid the O(n^4) complexity of a brute-force solution.
