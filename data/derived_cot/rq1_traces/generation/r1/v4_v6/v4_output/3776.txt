To solve this problem, we need to find the minimum cost required to remove all elements from an integer array by performing specific operations. The operations allowed are:
1. **Remove two elements from the first three elements**: The cost is the maximum of the two elements removed.
2. **Remove all remaining elements in a single operation**: This is done when fewer than three elements remain, and the cost is the maximum of the remaining elements.

### Approach
The solution involves using dynamic programming to efficiently compute the minimum cost for removing elements from the array. The key idea is to break down the problem into smaller subproblems and solve them recursively, storing the results to avoid redundant calculations. Here's the step-by-step approach:

1. **Dynamic Programming Setup**: We use an array `dp` where `dp[i]` represents the minimum cost to remove the subarray starting from index `i` to the end of the array.
2. **Base Cases**:
   - If only one element remains (`n - i == 1`), the cost is simply that element.
   - If two elements remain (`n - i == 2`), the cost is the maximum of the two elements.
3. **Recursive Cases**: For subarrays of length three or more, we consider all three possible ways to remove two elements from the first three elements:
   - Remove the first and second elements.
   - Remove the first and third elements.
   - Remove the second and third elements.
   For each option, the cost is the maximum of the two removed elements plus the cost of removing the remaining subarray (which is computed using the `dp` array).
4. **Memoization**: The results of subproblems are stored in the `dp` array to ensure each subproblem is solved only once, optimizing both time and space complexity.

### Solution Code
```python
class Solution:
    def minCost(self, nums: List[int]) -> int:
        n = len(nums)
        dp = [0] * (n + 1)
        dp[n] = 0
        if n >= 1:
            dp[n - 1] = nums[n - 1]
        if n >= 2:
            dp[n - 2] = max(nums[n - 2], nums[n - 1])
        for i in range(n - 3, -1, -1):
            option1 = max(nums[i], nums[i + 1]) + dp[i + 2]
            option2 = max(nums[i], nums[i + 2]) + dp[i + 3] if i + 3 <= n else float('inf')
            option3 = max(nums[i + 1], nums[i + 2]) + dp[i + 3] if i + 3 <= n else float('inf')
            dp[i] = min(option1, option2, option3)
        return dp[0]
```

### Explanation
- **Initialization**: The `dp` array is initialized with zeros, and the base cases for the last one or two elements are set directly.
- **Filling the DP Array**: The array is filled from the end towards the beginning. For each position `i`, we consider the three possible pairs of elements to remove from the first three elements starting at `i`.
- **Cost Calculation**: For each pair, the cost is the maximum of the two elements plus the cost to remove the remaining elements (which is stored in `dp[i+2]` or `dp[i+3]` depending on which elements were removed).
- **Result Extraction**: The result is found in `dp[0]`, which contains the minimum cost to remove the entire array starting from the first element.

This approach efficiently computes the minimum cost by leveraging dynamic programming to avoid redundant calculations, ensuring optimal performance even for larger arrays.
