To solve this problem, we need to compute the sum of all good subsequences in an integer array. A good subsequence is defined as a subsequence where the absolute difference between any two consecutive elements is exactly 1. Note that a subsequence of size 1 is always considered good.

### Approach
1. **Dynamic Programming Setup**: We use dynamic programming to keep track of the count and sum of good subsequences ending with each number encountered so far. Specifically, we maintain two arrays:
   - `count`: An array where `count[x]` stores the number of good subsequences ending with the number `x`.
   - `sum_arr`: An array where `sum_arr[x]` stores the sum of all good subsequences ending with the number `x`.

2. **Processing Each Element**: For each element `x` in the array:
   - **Single Element Subsequence**: The element `x` itself forms a subsequence of length 1. Thus, we initialize `current_count` to 1 and `current_sum` to `x`.
   - **Extending Existing Subsequences**: We extend any existing good subsequences ending with `x-1` or `x+1` by appending `x`. The new count and sum for these extended subsequences are derived from the counts and sums of subsequences ending with `x-1` and `x+1`.
   - **Update DP Arrays**: We update `count[x]` and `sum_arr[x]` by adding the newly formed subsequences' counts and sums.

3. **Modulo Handling**: Since the result can be very large, all operations are performed modulo \(10^9 + 7\).

4. **Result Calculation**: After processing all elements, the total sum of all good subsequences is the sum of all values in `sum_arr`.

### Solution Code
```python
class Solution:
    def sumOfGoodSubsequences(self, nums: List[int]) -> int:
        mod = 10**9 + 7
        if not nums:
            return 0
        max_val = max(nums)
        count = [0] * (max_val + 2)
        sum_arr = [0] * (max_val + 2)
        
        for x in nums:
            c_minus = count[x-1] if x - 1 >= 0 else 0
            s_minus = sum_arr[x-1] if x - 1 >= 0 else 0
            c_plus = count[x+1]
            s_plus = sum_arr[x+1]
            
            current_count = 1 + c_minus + c_plus
            current_sum = x + (s_minus + x * c_minus) + (s_plus + x * c_plus)
            current_count %= mod
            current_sum %= mod
            
            count[x] = (count[x] + current_count) % mod
            sum_arr[x] = (sum_arr[x] + current_sum) % mod
        
        total_ans = 0
        for i in range(len(sum_arr)):
            total_ans = (total_ans + sum_arr[i]) % mod
        
        return total_ans
```

### Explanation
1. **Initialization**: We initialize `mod` to \(10^9 + 7\) to handle large numbers. We also initialize `count` and `sum_arr` arrays to store counts and sums of subsequences ending with each possible number in the array, sized to accommodate the maximum value in the array plus one (to handle `x+1` accesses).
2. **Processing Elements**: For each element `x` in the array:
   - **Single Element**: The element `x` itself is a valid subsequence, contributing a count of 1 and a sum of `x`.
   - **Extending Subsequences**: We check for subsequences ending with `x-1` and `x+1`. For each such subsequence, appending `x` creates new subsequences. The new count is the sum of counts from `x-1` and `x+1`, and the new sum is derived by adding the sum of these subsequences plus `x` multiplied by their counts.
   - **Updating Arrays**: We update `count[x]` and `sum_arr[x]` with the new counts and sums, modulo \(10^9 + 7\).
3. **Result Calculation**: The total sum of all good subsequences is obtained by summing all values in `sum_arr`, again modulo \(10^9 + 7\).

This approach efficiently computes the desired sum using dynamic programming, leveraging the properties of good subsequences and modular arithmetic to handle large numbers. The complexity is linear with respect to the size of the array and the maximum value in the array, making it suitable for large inputs.
