To solve this problem, we need to count the number of beautiful numbers between two given integers, `l` and `r`, inclusive. A beautiful number is defined as a positive integer where the product of its digits is divisible by the sum of its digits. Given the constraints (1 <= l <= r < 10^9), a brute-force approach is infeasible. Instead, we use a digit dynamic programming (DP) approach to efficiently count the beautiful numbers.

### Approach
1. **Digit Dynamic Programming (DP)**: The solution involves breaking down the problem into smaller subproblems using digit DP. For each number up to a certain limit, we process each digit position while keeping track of key states:
   - **Position (pos)**: Current digit being processed.
   - **Tight Constraint (tight)**: Whether the digits chosen so far match the prefix of the upper limit number.
   - **Started Flag (started)**: Indicates whether non-zero digits have been chosen (to avoid leading zeros).
   - **Current Sum (sum_so_far)**: Sum of digits chosen so far.
   - **Current Product Modulo (prod_mod)**: Product of digits chosen so far modulo the target sum (to handle divisibility).

2. **Iterate Over Possible Digit Sums (s)**: The sum of digits of a number can range from 1 to 90 (since the maximum digit sum for a 9-digit number is 9*9=81, but we consider up to 90 for safety). For each possible sum `s`, we use DP to count numbers where:
   - The sum of digits equals `s`.
   - The product of digits is divisible by `s` (i.e., product modulo `s` is zero).

3. **Count Beautiful Numbers in Range**: The count of beautiful numbers from 1 to `n` is computed using the digit DP approach. The result for the range `[l, r]` is then derived as `count_upto(r) - count_upto(l-1)`.

### Solution Code
```python
class Solution:
    def beautifulNumbers(self, l: int, r: int) -> int:
        return self.count_upto(r) - self.count_upto(l-1)
    
    def count_upto(self, n):
        if n == 0:
            return 0
        digits = list(map(int, str(n)))
        L = len(digits)
        max_sum = 9 * L
        total_count = 0
        for s in range(1, min(90, max_sum) + 1):
            memo = {}
            total_count += self._dp(s, digits, 0, True, False, 0, 1, memo)
        return total_count

    def _dp(self, s, digits, pos, tight, started, sum_so_far, prod_mod, memo):
        if pos == len(digits):
            if started and sum_so_far == s and prod_mod == 0:
                return 1
            else:
                return 0
        key = (pos, tight, started, sum_so_far, prod_mod)
        if key in memo:
            return memo[key]
        up = digits[pos] if tight else 9
        total = 0
        for d in range(0, up + 1):
            new_tight = tight and (d == up)
            if not started and d == 0:
                total += self._dp(s, digits, pos + 1, new_tight, False, sum_so_far, prod_mod, memo)
            else:
                new_started = True
                new_sum = sum_so_far + d
                if new_sum > s:
                    continue
                new_prod = (prod_mod * d) % s
                total += self._dp(s, digits, pos + 1, new_tight, new_started, new_sum, new_prod, memo)
        memo[key] = total
        return total
```

### Explanation
1. **Digit DP Function (`_dp`)**:
   - **Base Case**: When all digits are processed, check if the number is started (non-zero), the sum of digits equals `s`, and the product of digits modulo `s` is zero.
   - **Memoization**: Uses a dictionary to store intermediate results and avoid recomputation.
   - **Digit Processing**: For each digit position, iterates over possible digits (0 to 9, constrained by the tight condition). Handles leading zeros by skipping them until a non-zero digit is encountered.
   - **State Updates**: Updates the sum and product modulo `s` for each digit chosen. Prunes paths where the sum exceeds `s` early to optimize performance.

2. **Counting Function (`count_upto`)**:
   - **Initialization**: Converts the number to its digit list and computes the maximum possible digit sum.
   - **Iterate Over Possible Sums (`s`)**: For each possible digit sum `s` (from 1 to the minimum of 90 and the maximum digit sum), initializes memoization and invokes the DP function.
   - **Result Aggregation**: Sums the counts of beautiful numbers for each `s`.

3. **Range Query (`beautifulNumbers`)**:
   - Computes the count of beautiful numbers from 1 to `r` and subtracts the count from 1 to `l-1` to get the result for the range `[l, r]`.

This approach efficiently counts beautiful numbers by leveraging digit DP and memoization, handling large ranges within feasible computational limits.
