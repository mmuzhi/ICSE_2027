You are given an array points where points[i] = [x_i, y_i] represents the coordinates of a point on an infinite plane.
Your task is to find the maximum area of a rectangle that:

Can be formed using four of these points as its corners.
Does not contain any other point inside or on its border.
Has its edges parallel to the axes.

Return the maximum area that you can obtain or -1 if no such rectangle is possible.
 
Example 1:

Input: points = [[1,1],[1,3],[3,1],[3,3]]
Output: 4
Explanation:

We can make a rectangle with these 4 points as corners and there is no other point that lies inside or on the border. Hence, the maximum possible area would be 4.

Input: points = [[1,1],[1,3],[3,1],[3,3],[2,2]]
Output: -1
Explanation:

There is only one rectangle possible is with points [1,1], [1,3], [3,1] and [3,3] but [2,2] will always lie inside it. Hence, returning -1.

Input: points = [[1,1],[1,3],[3,1],[3,3],[1,2],[3,2]]

=== ANSWER ===
To solve this problem, we need to find the maximum area of a rectangle formed by four distinct points from a given list such that the rectangle's edges are parallel to the axes and no other point lies inside or on the border of the rectangle. If no such rectangle exists, we return -1.

### Approach
1. **Problem Analysis**: The problem involves checking all possible rectangles formed by four points where the rectangle's sides are parallel to the axes. The key insight is that such a rectangle can be defined by two distinct x-coordinates and two distinct y-coordinates. The four corners of the rectangle must be the points (x1, y1), (x1, y2), (x2, y1), and (x2, y2), where x1 and x2 are the distinct x-coordinates, and y1 and y2 are the distinct y-coordinates.

2. **Intuition**: For each pair of distinct x-coordinates (x1, x2), we find the common y-coordinates present at both x1 and x2. For each pair of common y-coordinates (y1, y2), we check if the rectangle formed by (x1, y1), (x1, y2), (x2, y1), (x2, y2) contains no other points inside or on its border besides these four corners.

3. **Algorithm Selection**: 
   - **Group Points by X-coordinate**: Use a dictionary to map each x-coordinate to a set of y-coordinates.
   - **Generate Valid Rectangles**: For each pair of x-coordinates (x1, x2), compute the intersection of their y-coordinate sets. For each pair of y-coordinates (y1, y2) in this intersection, form the rectangle defined by these x and y values.
   - **Check Validity**: Ensure that no other points lie inside or on the borders of this rectangle. The rectangle's area is calculated if valid.
   - **Track Maximum Area**: Keep track of the maximum valid area encountered during the checks.

4. **Complexity Analysis**: Given the constraints (n ≤ 10), the approach efficiently checks all possible rectangles. The worst-case complexity involves O(n^2) operations for x-pairs and O(m^2) operations for y-pairs, where m is the number of common y-coordinates (≤ 10), leading to a manageable total of about 2025 operations in the worst case.

```python
                for k in range(m):
                    for l in range(k+1, m):
                        y1 = common_ys_sorted[k]
                        y2 = common_ys_sorted[l]
                        left = min(x1, x2)
                        right = max(x1, x2)
                        bottom = min(y1, y2)
                        top = max(y1, y2)
                        valid = True
                        for (x, y) in all_points_set:
                            if (x, y) in [(x1, y1), (x1, y2), (x2, y1), (x2, y2)]:
                                continue
                            if left <= x <= right and bottom <= y <= top:
                                valid = False
                                break
                        if valid:
                            area = (right - left) * (top - bottom)
                            if area > max_area:
                                max_area = area
        return max_area if max_area != -1 else -1
```

### Explanation
1. **Initialization**: The solution starts by converting the list of points into a set for quick lookup and grouping points by their x-coordinates using a dictionary where each key is an x-coordinate and the value is a set of corresponding y-coordinates.
2. **Generating Rectangles**: For each pair of distinct x-coordinates, the solution checks for common y-coordinates. If there are at least two common y-coordinates, it forms all possible pairs of these y-coordinates.
3. **Checking Validity**: For each rectangle formed by the x and y pairs, the solution checks if any other points lie inside or on the borders of the rectangle. If no such points exist, the rectangle is considered valid, and its area is calculated.
4. **Tracking Maximum Area**: The solution keeps track of the maximum valid area encountered during the checks. If no valid rectangles are found, it returns -1; otherwise, it returns the maximum area.

This approach efficiently checks all possible valid rectangles while ensuring optimal performance given the problem constraints.
