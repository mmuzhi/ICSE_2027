=== ANSWER ===
To solve this problem, we need to determine the minimum total number of movements required to complete a sequence of tasks where each task requires moving a specific person to a designated coordinate on a road. The road extends infinitely in both directions, and persons can move one meter east or west at a time, provided the destination is unoccupied. The challenge is to compute the minimal total movements while ensuring that after each task, the specified person is at the required coordinate, and all positions remain distinct.

### Approach
1. **Problem Analysis**: The problem involves managing the positions of N persons on a line, with each task specifying that a particular person must be moved to a given coordinate. The key constraint is that no two persons can occupy the same position at any time. The solution must efficiently handle up to 200,000 persons and 200,000 tasks, making an optimal approach crucial.

2. **Intuition**: For each task, moving a person to a new coordinate might require moving other persons out of the way if the target coordinate is occupied. The minimal movement for each task involves:
   - Moving the specified person from their current position to the target coordinate.
   - If the target coordinate is occupied, move the occupying person to the nearest free coordinate, which could be adjacent or further away, depending on existing positions.

3. **Algorithm Selection**: 
   - **Coordinate Compression**: Given the large possible coordinate range (up to 10^8), we compress all relevant coordinates (initial positions, task targets, and potential free spots) into a manageable range.
   - **Segment Trees**: We use two segment trees to maintain the current positions: one for tracking the maximum coordinate in any range (to find predecessors) and another for the minimum coordinate (to find successors). This allows efficient updates and queries.
   - **Greedy Simulation**: For each task, we:
     1. Remove the current position of the specified person.
     2. Check if the target coordinate is free.
     3. If occupied, move the occupying person to the nearest free coordinate (either adjacent or calculated based on nearby positions).
     4. Update positions and compute movement costs.

4. **Complexity Analysis**: 
   - **Coordinate Compression**: Gathering and compressing coordinates is O((N + Q) log (N + Q)).
   - **Segment Tree Operations**: Each update and query operation on the segment trees is O(log M), where M is the number of compressed coordinates (up to 3.6 million). With 400,000 operations (2 per task), the total time is O((N + Q) log M), which is feasible for the constraints.

class SegmentTreeForMax:
    def __init__(self, size, rev_mapping):
        self.n = size
        self.data = [NEG_INF] * (4 * size)
        self.rev = rev_mapping

    def update(self, idx, value):
        self._update(1, 0, self.n-1, idx, value)
        
    def _update(self, node, l, r, idx, value):
        if l == r:
            self.data[node] = value
            return
        mid = (l + r) // 2
        if idx <= mid:
            self._update(node*2, l, mid, idx, value)
        else:
            self._update(node*2+1, mid+1, r, idx, value)
        self.data[node] = max(self.data[node*2], self.data[node*2+1])
        
    def query_range(self, ql, qr):
        if ql > qr:
            return NEG_INF
        return self._query(1, 0, self.n-1, ql, qr)
