=== ANSWER ===
To solve this problem, we need to simulate the process of repeatedly merging adjacent pairs in the array until the array becomes non-decreasing. The merging operation is deterministic: at each step, we must select the adjacent pair with the minimum sum. If multiple pairs have the same minimum sum, we choose the leftmost one. The goal is to count the number of such operations required to make the array non-decreasing.

### Approach
1. **Initial Check**: First, check if the array is already non-decreasing. If it is, return 0 immediately since no operations are needed.
2. **Simulate Merging Process**: While the array is not non-decreasing:
   - **Find the Minimum Sum Pair**: Iterate through the array to find the adjacent pair with the smallest sum. If multiple pairs have the same minimum sum, select the leftmost one.
   - **Merge the Pair**: Replace the identified pair with their sum, effectively reducing the array length by one.
   - **Increment Operation Count**: Each merge operation counts as one step.
3. **Check Non-Decreasing Condition**: After each merge, check if the resulting array is non-decreasing. If it is, return the current operation count.
4. **Termination Condition**: The loop terminates when the array becomes non-decreasing, which is guaranteed to happen eventually since merging reduces the array to a single element, which is trivially non-decreasing.

### Solution Code
```python
class Solution:
    def minimumPairRemoval(self, nums: List[int]) -> int:
        arr = nums
        ops = 0
        while True:
            if all(arr[i] >= arr[i-1] for i in range(1, len(arr))):
                return ops
            min_sum = float('inf')
            idx = -1
            for i in range(len(arr) - 1):
                s = arr[i] + arr[i+1]
                if s < min_sum:
                    min_sum = s
                    idx = i
            arr = arr[:idx] + [min_sum] + arr[idx+2:]
            ops += 1
```

### Explanation
- **Initial Check**: The loop starts by checking if the current array is non-decreasing using the condition `all(arr[i] >= arr[i-1] for i in range(1, len(arr)))`. If true, it returns the operation count `ops`.
- **Finding Minimum Sum Pair**: The code scans each adjacent pair in the array to find the pair with the smallest sum. The leftmost such pair is chosen if there are ties.
- **Merging the Pair**: The identified pair is replaced by their sum, creating a new array that excludes the two elements and includes their sum at the same position.
- **Operation Count**: Each merge operation increments the count `ops`.
- **Termination**: The loop continues until the array becomes non-decreasing, at which point the operation count is returned. The process ensures that the array length reduces by one with each operation, eventually leading to a non-decreasing array. The solution efficiently handles the constraints with a straightforward simulation approach.
