=== PROMPT ===
You will be given a competitive programming problem.
AtCoder Regular Contest (ARC) is divided into two divisions.

- In ARC Div. 1, participants whose rating at the start of the contest is between 1600 and 2799, inclusive, are subject to rating updates.
- In ARC Div. 2, participants whose rating at the start of the contest is between 1200 and 2399, inclusive, are subject to rating updates.

Takahashi decided to participate in N ARCs.
Initially, his rating is R.
The i-th (1\leq i\leq N) ARC is Div. D _ i, and his performance in that contest is represented by an integer A _ i.
If he is subject to a rating update in the i-th ARC, let T be his rating at the start of that contest. Then, after the contest, his rating becomes T+A _ i.
If his is not subject to a rating update, his rating does not change.
Rating updates for ARCs are performed immediately after the contest ends, and whether he is subject to rating updates in the next contest is determined based on his rating after the update.
Find his rating after finishing the N ARCs.
He does not participate in any contests other than these N ARCs, and his rating does not change in other ways.

Input

The input is given in the following format from Standard Input:
N R
D _ 1 A _ 1
D _ 2 A _ 2
\vdots
D _ N A _ N

Output

Print Takahashi's rating after finishing the N ARCs.

Constraints


- 1\leq N\leq 100
- 0\leq R\leq 4229
- 1\leq D _ i\leq 2\ (1\leq i\leq N)
- -1000\leq A _ i\leq 1000\ (1\leq i\leq N)
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to simulate the rating changes of Takahashi over a series of contests based on specific division rules. The key is to update his rating only when he is eligible for a rating update in each contest, according to the division's rating range constraints.

### Approach
1. **Initialization**: Start with the initial rating provided.
2. **Iterate through each contest**: For each contest, check the division (Div. 1 or Div. 2) and the current rating.
3. **Check eligibility for rating update**:
   - For Div. 1 contests, the current rating must be between 1600 and 2799 (inclusive).
   - For Div. 2 contests, the current rating must be between 1200 and 2399 (inclusive).
4. **Update rating if eligible**: If eligible, add the performance value (A_i) to the current rating.
5. **Proceed to next contest**: Move to the next contest with the updated (or unchanged) rating.
6. **Output final rating**: After processing all contests, print the final rating.

This approach efficiently handles each contest in sequence, applying the rating update rules dynamically based on the current rating before each contest.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    n, r = map(int, data[0].split())
    contests = []
    for i in range(1, n + 1):
        parts = data[i].split()
        d = int(parts[0])
        a = int(parts[1])
        contests.append((d, a))
    
    current_rating = r
    for d, a in contests:
        if d == 1:
            if 1600 <= current_rating <= 2799:
                current_rating += a
        else:
            if 1200 <= current_rating <= 2399:
                current_rating += a
    
    print(current_rating)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line contains the number of contests (N) and the initial rating (R). Subsequent lines contain the division (D_i) and performance (A_i) for each contest.
2. **Processing Each Contest**: For each contest, the division is checked. If it's a Div. 1 contest, the current rating must be within [1600, 2799] for an update. If it's a Div. 2 contest, the rating must be within [1200, 2399]. The performance value is added to the current rating if eligible.
3. **Output Result**: After processing all contests, the final rating is printed.

This approach ensures that the rating is updated according to the specific rules of each contest division, leveraging straightforward conditional checks and sequential processing. The solution efficiently handles the constraints and provides the correct final rating after all contests.
