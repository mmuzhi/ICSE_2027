You will be given a competitive programming problem.
Takahashi has a piano with 100 keys arranged in a row.
The i-th key from the left is called key i.
He will play music by pressing N keys one by one.
For the i-th press, he will press key A_i, using his left hand if S_i= L, and his right hand if S_i= R.
Before starting to play, he can place both of his hands on any keys he likes, and his fatigue level at this point is 0.
During the performance, if he moves one hand from key x to key y, the fatigue level increases by |y-x| (conversely, the fatigue level does not increase for any reason other than moving hands).
To press a certain key with a hand, that hand must be placed on that key.
Find the minimum possible fatigue level at the end of the performance.

Input

The input is given from Standard Input in the following format:
N
A_1 S_1
A_2 S_2
\vdots
A_N S_N

Output

Print the minimum fatigue level at the end of the performance.

Constraints


- 1 \leq N \leq 100
- 1 \leq A_i \leq 100
- N and A_i are integers.
- S_i is L or R.

Sample Input 1

4
3 L
6 R
9 L
1 R

Sample Output 1

11

For example, the performance can be done as follows:

- Initially, place the left hand on key 3 and the right hand on key 6.
- Press key 3 with the left hand.
- Press key 6 with the right hand.
- Move the left hand from key 3 to key 9. The fatigue level increases by |9-3| = 6.
- Move the right hand from key 6 to key 1. The fatigue level increases by |1-6| = 5.
- Press key 9 with the left hand.
- Press key 1 with the right hand.

In this case, the fatigue level at the end of the performance is 6+5 = 11, which is the minimum possible.

=== ANSWER ===
To solve this problem, we need to minimize the fatigue level Takahashi experiences when playing a sequence of notes on a piano. The fatigue level is defined as the total distance his hands move while pressing the specified keys with the designated hands (left or right). The challenge is to determine the optimal initial positions for his hands and the minimal movement required to press all the notes in sequence.

### Approach
1. **Dynamic Programming (DP) Setup**: We use a DP table `dp[l][r]` where `l` and `r` represent the current positions of the left and right hands, respectively. The value `dp[l][r]` stores the minimal fatigue level required to reach the state where the left hand is at key `l` and the right hand is at key `r` after processing the first `i` notes.

2. **Initialization**: Before any notes are pressed, Takahashi can place his hands on any keys. Thus, we initialize `dp[l][r]` to 0 for all possible initial positions (1 to 100 for both hands).

3. **Processing Notes**: For each note in the sequence:
   - **Left Hand (L)**: If the note must be pressed with the left hand, we update the DP table by moving the left hand from its current position to the target key `A_i`. The fatigue level increases by the absolute difference between the current and target positions. The right hand remains stationary.
   - **Right Hand (R)**: Similarly, if the note requires the right hand, we move the right hand to the target key, updating the DP table accordingly while the left hand stays in place.

4. **State Transition**: After processing each note, we prepare a new DP table (`next_dp`) to store the updated states and fatigue levels. The current DP table (`dp`) is then replaced by `next_dp` for the next iteration.

5. **Result Extraction**: After processing all notes, the answer is the minimal value in the DP table, representing the least fatigue level required to press all notes with the specified hands.

```python
        A = int(parts[0])
        S = parts[1]
        notes.append((A, S))
    
    INF = 10**9
    dp = [[INF] * 101 for _ in range(101)]
    for l in range(1, 101):
        for r in range(1, 101):
            dp[l][r] = 0
    
    for A, S in notes:
        next_dp = [[INF] * 101 for _ in range(101)]
        if S == 'L':
            for l_prev in range(1, 101):
                for r_prev in range(1, 101):
                    if dp[l_prev][r_prev] != INF:
                        cost = dp[l_prev][r_prev] + abs(l_prev - A)
                        if cost < next_dp[A][r_prev]:
                            next_dp[A][r_prev] = cost
        else:
            for l_prev in range(1, 101):
                for r_prev in range(1, 101):
                    if dp[l_prev][r_prev] != INF:
                        cost = dp[l_prev][r_prev] + abs(r_prev - A)
                        if cost < next_dp[l_prev][A]:
                            next_dp[l_prev][A] = cost
        dp = next_dp
    
    ans = INF
    for l in range(1, 101):
        for r in range(1, 101):
            if dp[l][r] < ans:
                ans = dp[l][r]
    print(ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line specifies the number of notes, followed by pairs of integers and characters representing the key and hand (L or R) for each note.
2. **Initialization**: The DP table is initialized to a large value (`INF`), except for all possible initial hand positions (1 to 100 for both hands), which are set to 0 fatigue.
3. **Processing Each Note**: For each note, a new DP table (`next_dp`) is created. Depending on whether the note requires the left or right hand, the current DP table is updated by moving the respective hand to the target key, increasing the fatigue by the distance moved. The other hand remains in place.
4. **Updating State**: After processing each note, the current DP table is replaced by `next_dp` to reflect the new state after pressing the note.
5. **Finding the Answer**: After processing all notes, the minimal value in the DP table is found and printed, representing the least fatigue level required to complete the sequence.

This approach efficiently tracks the minimal fatigue by dynamically updating hand positions and movement costs, ensuring optimal performance even for the maximum input size.
