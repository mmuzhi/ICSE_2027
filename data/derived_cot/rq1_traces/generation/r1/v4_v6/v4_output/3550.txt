=== PROMPT ===
You will be given a competitive programming problem.
You are given a m x n 2D array board representing a chessboard, where board[i][j] represents the value of the cell (i, j).
Rooks in the same row or column attack each other. You need to place three rooks on the chessboard such that the rooks do not attack each other.
Return the maximum sum of the cell values on which the rooks are placed.
 
Example 1:

Input: board = [[-3,1,1,1],[-3,1,-3,1],[-3,2,1,1]]
Output: 4
Explanation:

We can place the rooks in the cells (0, 2), (1, 3), and (2, 1) for a sum of 1 + 1 + 2 = 4.

Example 2:

Input: board = [[1,2,3],[4,5,6],[7,8,9]]
Output: 15
Explanation:
We can place the rooks in the cells (0, 0), (1, 1), and (2, 2) for a sum of 1 + 5 + 9 = 15.

Example 3:

Input: board = [[1,1,1],[1,1,1],[1,1,1]]
Output: 3
Explanation:
We can place the rooks in the cells (0, 2), (1, 1), and (2, 0) for a sum of 1 + 1 + 1 = 3.

 
Constraints:

3 <= m == board.length <= 100
3 <= n == board[i].length <= 100
-10^9 <= board[i][j] <= 10^9

To solve this problem, we need to place three rooks on a chessboard such that no two rooks attack each other (i.e., they are not in the same row or column), and the sum of the values of the cells where the rooks are placed is maximized. 

### Approach
1. **Problem Analysis**: The problem requires selecting three cells from an `m x n` chessboard where each cell is in a distinct row and column. The goal is to maximize the sum of the values of these three cells. Given the constraints (up to 100x100 board), a brute-force approach is infeasible. Instead, we use dynamic programming to efficiently explore possible placements.
2. **Dynamic Programming State**: The DP state is represented by a dictionary where keys are tuples of sorted column indices (representing columns already occupied by rooks), and values are the maximum sum achievable with those columns. The state starts as an empty tuple with a sum of 0.
3. **State Transitions**: For each row, we consider two options:
   - **Skip the row**: Carry over all current states without modification.
   - **Place a rook in the row**: For each state (tuple of columns) with fewer than three rooks, place a rook in any available column (not already in the state). Update the new state (sorted tuple of columns including the new column) with the new sum (current sum + cell value).
4. **Result Extraction**: After processing all rows, the solution is the maximum sum among all states that have exactly three columns (i.e., three rooks placed).

### Solution Code
```python
class Solution:
    def maximumValueSum(self, board: List[List[int]]) -> int:
        m = len(board)
        n = len(board[0])
        dp = {(): 0}
        
        for i in range(m):
            new_dp = {}
            for state, val in dp.items():
                if state in new_dp:
                    new_dp[state] = max(new_dp[state], val)
                else:
                    new_dp[state] = val
            
            for state, val in dp.items():
                if len(state) < 3:
                    s_set = set(state)
                    for j in range(n):
                        if j not in s_set:
                            new_state = tuple(sorted(state + (j,)))
                            new_val = val + board[i][j]
                            if new_state in new_dp:
                                new_dp[new_state] = max(new_dp[new_state], new_val)
                            else:
                                new_dp[new_state] = new_val
            dp = new_dp
        
        ans = -10**18
        for state, val in dp.items():
            if len(state) == 3:
                if val > ans:
                    ans = val
        return ans
```

### Explanation
1. **Initialization**: The DP dictionary starts with an empty state (no rooks placed) and a sum of 0.
2. **Row Processing**: For each row:
   - **State Carry-Over**: All current states are carried over to the next row without adding a rook (skipping the row).
   - **Rook Placement**: For each state with fewer than three rooks, place a rook in each available column (not in the state). The new state is formed by adding the new column to the existing columns (sorted to maintain uniqueness). The new sum is the current sum plus the value of the cell.
3. **Result Extraction**: After processing all rows, the solution is the maximum sum from all states that have exactly three columns (three rooks placed). The algorithm efficiently explores possible placements using dynamic programming, avoiding the need for a brute-force approach. The complexity is manageable due to the constraints (100 rows and columns), with the inner loops optimized to handle state transitions efficiently.
