Information about the edges is given by N^2 characters C_{1, 1}, C_{1, 2}, \ldots, C_{1, N}, C_{2, 1}, \ldots, C_{N, N}. Here, each C_{i, j} is either a lowercase English letter or -.

- Among all (not necessarily simple) paths from vertex i to vertex j whose concatenation of labels on the edges forms a palindrome, what is the length of the shortest such path? If there is no such path, the answer is -1.

Input

The input is given from Standard Input in the following format:
N

Output

Let A_{i, j} be the answer to the question for the pair (i, j). Print them in the following format:
A_{1, 1} A_{1, 2} \ldots A_{1, N}
A_{2, 1} A_{2, 2} \ldots A_{2, N}
\vdots
A_{N, 1} A_{N, 2} \ldots A_{N, N}

Constraints


- 1 \leq N \leq 100
- N is an integer.
- Each C_{i, j} is either a lowercase English letter or -.

--b-
---a
c---

Sample Output 1

-1 0 1 -1
3 -1 0 1

=== ANSWER ===
To solve this problem, we need to find the shortest path between every pair of vertices in a directed graph such that the concatenation of the edge labels along the path forms a palindrome. The solution involves a BFS-based approach combined with dynamic programming to efficiently compute these paths for all vertex pairs.

### Approach
1. **Problem Analysis**: The problem requires finding the shortest path between every pair of vertices (i, j) where the path's edge labels form a palindrome. The graph is directed, and each edge has a label (a lowercase letter) or no edge (denoted by '-').

2. **Key Insight**: The shortest palindrome path can be constructed by extending symmetrically from the middle. We start with trivial paths (single vertices or single edges) and expand outward by adding matching edges at both ends. This ensures the path remains a palindrome.

3. **Algorithm Selection**: We use a BFS approach initialized with:
   - **Base Cases**: Paths of length 0 (vertices themselves) and length 1 (direct edges).
   - **Extension**: For each state (u, v), we look for edges leading into u (backward) and out of v (forward) with the same label. This forms a new path from `u_prev` to `v_next` with length increased by 2.

4. **Complexity Analysis**: The algorithm processes each vertex pair (u, v) once. For each pair, it checks all possible predecessors of u and successors of v, leading to an O(N^4) worst-case complexity. Given N ≤ 100, this is feasible (100^4 = 100,000,000 operations).

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    n = int(data[0])
    graph = []
    for i in range(1, 1 + n):
        graph.append(data[i].strip())
    
    INF = 10**9
    dp = [[INF] * n for _ in range(n)]
    q = deque()
    
    for i in range(n):
        dp[i][i] = 0
        q.append((i, i))
        
    for i in range(n):
        for j in range(n):
            if graph[i][j] != '-':
                if 1 < dp[i][j]:
                    dp[i][j] = 1
                    q.append((i, j))
                    
    in_edges = [dict() for _ in range(n)]
    for prev in range(n):
        for u in range(n):
            c = graph[prev][u]
            if c != '-':
                if c not in in_edges[u]:
                    in_edges[u][c] = []
                in_edges[u][c].append(prev)
                
    out_edges = [dict() for _ in range(n)]
    for v in range(n):
        for nxt in range(n):
            c = graph[v][nxt]
            if c != '-':
                if c not in out_edges[v]:
                    out_edges[v][c] = []
                out_edges[v][c].append(nxt)
                
    while q:
        u, v = q.popleft()
        current = dp[u][v]
        for c, u_prev_list in in_edges[u].items():
            if c in out_edges[v]:
                v_next_list = out_edges[v][c]
                for u_prev in u_prev_list:
                    for v_next in v_next_list:
                        if dp[u_prev][v_next] > current + 2:
                            dp[u_prev][v_next] = current + 2
                            q.append((u_prev, v_next))
                            
    for i in range(n):
        for j in range(n):
            if dp[i][j] == INF:
                dp[i][j] = -1
                
    for i in range(n):
        print(" ".join(map(str, dp[i])))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The algorithm starts by reading the input and initializing a distance matrix `dp` with `INF` (representing initially unreachable paths). The BFS queue is initialized with all vertices (paths of length 0) and direct edges (paths of length 1).

2. **Preprocessing**: Two auxiliary data structures, `in_edges` and `out_edges`, are built. These store, for each vertex, the incoming and outgoing edges grouped by their labels for efficient access.

3. **BFS Processing**: For each state (u, v) dequeued, the algorithm checks all pairs of edges leading into `u` and out of `v` with the same label. For each such pair, a new state (u_prev, v_next) is enqueued with an updated path length (current length + 2).

4. **Result Extraction**: After BFS completes, the `dp` matrix contains the shortest path lengths (or -1 for unreachable paths). The results are printed row by row.

This approach efficiently computes the shortest palindrome paths by leveraging BFS and dynamic programming, ensuring optimal performance for the given constraints.
