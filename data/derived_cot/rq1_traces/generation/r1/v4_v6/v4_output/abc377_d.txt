Find the number of pairs of integers (l,r) that satisfy both of the following conditions:

- 1\le l \le r \le M
- For every 1\le i\le N, the interval [l,r] does not completely contain the interval [L_i,R_i].

Input

The input is given from Standard Input in the following format:
N M
L_1 R_1
L_2 R_2
\vdots
L_N R_N

Output

Print the answer.

- 1\le N,M\le 2\times 10^5
- 1\le L_i\le R_i\le M
- All input values are integers.

To solve this problem, we need to count the number of integer pairs (l, r) such that 1 ≤ l ≤ r ≤ M and the interval [l, r] does not completely contain any given interval [L_i, R_i]. The solution involves efficiently determining the pairs that violate the condition and subtracting them from the total possible pairs.

### Approach
1. **Problem Analysis**: The task is to find all intervals (l, r) that do not fully contain any of the given intervals (L_i, R_i). The total number of possible pairs (l, r) is M*(M+1)/2. We need to subtract the pairs that violate the condition, i.e., pairs (l, r) where there exists at least one interval [L_i, R_i] such that l ≤ L_i and r ≥ R_i.

2. **Key Insight**: For each starting index l, the condition for a pair (l, r) to be "bad" (i.e., it contains some interval [L_i, R_i]) is that r must be at least the minimum R_i among all intervals with L_i ≥ l. This insight allows us to efficiently compute the number of bad pairs for each l by maintaining the smallest R_i encountered as we iterate from M down to 1.

3. **Algorithm Selection**: 
   - **Preprocessing**: Store all intervals grouped by their starting index L_i in a list of lists.
   - **Iterate from M down to 1**: For each l, update the smallest R_i encountered so far (from intervals with L_i ≥ l). The number of bad pairs for the current l is the count of r values from max(l, min_val) to M.
   - **Calculate Result**: Subtract the total bad pairs from the total possible pairs (M*(M+1)/2) to get the answer.

4. **Complexity Analysis**: The algorithm processes each interval exactly once during preprocessing and once during the iteration. The loop runs M times, and each inner loop processes intervals in O(1) per interval. Thus, the overall complexity is O(N + M), which is efficient for the constraints (N, M ≤ 2×10^5).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    M = int(data[1])
    lists = [[] for _ in range(M+1)]
    idx = 2
    for _ in range(n):
        l_val = int(data[idx])
        r_val = int(data[idx+1])
        idx += 2
        if l_val <= M:
            lists[l_val].append(r_val)
    
    total_pairs = M * (M + 1) // 2
    min_val = M + 1
    bad = 0
    for l in range(M, 0, -1):
        for r_val in lists[l]:
            if r_val < min_val:
                min_val = r_val
        if min_val == M + 1:
            continue
        bad += (M - max(l, min_val) + 1)
    
    print(total_pairs - bad)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of intervals (N), the maximum value (M), and the intervals themselves.
2. **Preprocessing**: Intervals are grouped by their starting index (L_i) into a list of lists for efficient access.
3. **Initialization**: `total_pairs` calculates all possible (l, r) pairs. `min_val` is initialized to a value larger than any possible R_i to keep track of the smallest R_i encountered.
4. **Processing Intervals**: Iterating from M down to 1, for each l:
   - Update `min_val` with the smallest R_i from intervals starting at l.
   - If `min_val` is updated (i.e., an interval exists), compute the number of bad pairs (l, r) where r ranges from max(l, min_val) to M.
5. **Result Calculation**: The result is obtained by subtracting the total bad pairs from the total possible pairs and printed.

This approach efficiently counts the valid pairs by leveraging preprocessing and a single pass over the range of possible starting indices, ensuring optimal performance for large inputs.
