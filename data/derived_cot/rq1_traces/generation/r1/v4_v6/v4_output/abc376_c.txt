You will be given a competitive programming problem.
There are N toys numbered from 1 to N, and N-1 boxes numbered from 1 to N-1.
Toy i\ (1 \leq i \leq N) has a size of A_i, and box i\ (1 \leq i \leq N-1) has a size of B_i.
Takahashi wants to store all the toys in separate boxes, and he has decided to perform the following steps in order:

- Choose an arbitrary positive integer x and purchase one box of size x.
- Place each of the N toys into one of the N boxes (the N-1 existing boxes plus the newly purchased box).
Here, each toy can only be placed in a box whose size is not less than the toy's size, and no box can contain two or more toys.

He wants to execute step 2 by purchasing a sufficiently large box in step 1, but larger boxes are more expensive, so he wants to purchase the smallest possible box.
Determine whether there exists a value of x such that he can execute step 2, and if it exists, find the minimum such x.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \dots A_N
B_1 B_2 \dots B_{N-1}

Output

If there exists a value of x such that Takahashi can execute step 2, print the minimum such x. Otherwise, print -1.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq A_i, B_i \leq 10^9
- All input values are integers.

Sample Input 1

4
5 2 3 7
6 2 8

Sample Output 1

3

Consider the case where x=3 (that is, he purchases a box of size 3 in step 1).
If the newly purchased box is called box 4, toys 1,\dots,4 have sizes of 5, 2, 3, and 7, respectively, and boxes 1,\dots,4 have sizes of 6, 2, 8, and 3, respectively.
Thus, toy 1 can be placed in box 1, toy 2 in box 2, toy 3 in box 4, and toy 4 in box 3.
On the other hand, if x \leq 2, it is impossible to place all N toys into separate boxes.
Therefore, the answer is 3.

-1

To solve this problem, we need to determine the smallest size of an additional box that Takahashi must purchase so that all the toys can be placed into separate boxes, with each toy placed in a box whose size is at least the toy's size. The solution involves efficiently checking which toy can be placed in the new box such that the remaining toys can be placed in the existing boxes.

### Approach
1. **Problem Analysis**: 
   - We have `N` toys and `N-1` existing boxes. Each toy and box has a specific size. The goal is to place each toy in a separate box (including the new box) such that the toy's size does not exceed the box's size.
   - The new box's size `x` must be at least the size of the toy placed in it. We aim to find the smallest `x` that allows all toys to be placed in boxes.

2. **Key Insight**:
   - For each toy, if we place it in the new box, the remaining `N-1` toys must be placed in the existing `N-1` boxes. This requires that when both the remaining toys and existing boxes are sorted in ascending order, each toy's size must be less than or equal to the corresponding box's size (i.e., the `i-th` smallest toy must fit into the `i-th` smallest box).

3. **Algorithm Selection**:
   - **Sorting**: Sort the toys and existing boxes to facilitate efficient comparison.
   - **Prefix and Suffix Arrays**: 
     - **Prefix Array (`pre_arr`)**: Tracks whether the first `i` toys (when sorted) can be placed in the first `i` boxes (also sorted) for all `i` from `0` to `N-2`.
     - **Suffix Array (`suf_arr`)**: Tracks whether the last `N-i-1` toys can be placed in the last `N-i-1` boxes for all `i` from `0` to `N-2`.
   - **Validation**: For each toy, if placing it in the new box allows the remaining toys to be placed in the existing boxes (checked using `pre_arr` and `suf_arr`), consider the toy's size as a candidate for `x`. The smallest such candidate is the answer.

4. **Complexity Analysis**:
   - **Time Complexity**: Sorting the toys and boxes takes `O(N log N)` time. Computing the prefix and suffix arrays takes `O(N)` time. The validation step also takes `O(N)` time. Thus, the overall complexity is `O(N log N)`.
   - **Space Complexity**: The prefix and suffix arrays require `O(N)` space, leading to an overall space complexity of `O(N)`.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    B = list(map(int, data[1+n:1+n+n-1]))
    
    T = sorted(A)
    U = sorted(B)
    
    pre_arr = [False] * n
    if n > 1:
        pre_arr[0] = (T[0] <= U[0])
        for i in range(1, n-1):
            pre_arr[i] = pre_arr[i-1] and (T[i] <= U[i])
    
    suf_arr = [False] * n
    if n > 1:
        suf_arr[n-2] = (T[n-1] <= U[n-2])
        for i in range(n-3, -1, -1):
            suf_arr[i] = (T[i+1] <= U[i]) and suf_arr[i+1]
    
    ans = 10**18
    for i in range(n):
        if i == 0:
            if suf_arr[i]:
                ans = min(ans, T[i])
        elif i == n-1:
            if n > 1 and pre_arr[n-2]:
                ans = min(ans, T[i])
        else:
            if pre_arr[i-1] and suf_arr[i]:
                ans = min(ans, T[i])
                
    print(-1 if ans == 10**18 else ans)

if __name__ == "__main__":
    main()
```

2. **Sorting**: Both the toy sizes (`A`) and box sizes (`B`) are sorted to facilitate efficient comparison.
3. **Prefix Array (`pre_arr`)**: This array checks whether the first `i` toys can be placed in the first `i` boxes for each `i` from `0` to `n-2`.
4. **Suffix Array (`suf_arr`)**: This array checks whether the last `n-i-1` toys can be placed in the last `n-i-1` boxes for each `i` from `0` to `n-2`.
