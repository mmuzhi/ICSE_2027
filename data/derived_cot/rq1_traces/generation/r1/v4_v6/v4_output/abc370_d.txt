You will be given a competitive programming problem.
There is a grid with H rows and W columns. Let (i, j) denote the cell at the i-th row from the top and j-th column from the left.
Initially, there is one wall in each cell.
After processing Q queries explained below in the order they are given, find the number of remaining walls.
In the q-th query, you are given two integers R_q and C_q.
You place a bomb at (R_q, C_q) to destroy walls. As a result, the following process occurs.

- If there is a wall at (R_q, C_q), destroy that wall and end the process.
- If there is no wall at (R_q, C_q), destroy the first walls that appear when looking up, down, left, and right from (R_q, C_q). More precisely, the following four processes occur simultaneously:
- If there exists an i \lt R_q such that a wall exists at (i, C_q) and no wall exists at (k, C_q) for all i \lt k \lt R_q, destroy the wall at (i, C_q).
- If there exists an i \gt R_q such that a wall exists at (i, C_q) and no wall exists at (k, C_q) for all R_q \lt k \lt i, destroy the wall at (i, C_q).
- If there exists a j \lt C_q such that a wall exists at (R_q, j) and no wall exists at (R_q, k) for all j \lt k \lt C_q, destroy the wall at (R_q, j).
- If there exists a j \gt C_q such that a wall exists at (R_q, j) and no wall exists at (R_q, k) for all C_q \lt k \lt j, destroy the wall at (R_q, j).

Input

The input is given from Standard Input in the following format:
H W Q
R_1 C_1
R_2 C_2
\vdots
R_Q C_Q

Output

Print the number of remaining walls after processing all queries.

Constraints


- 1 \leq H, W
- H \times W \leq 4 \times 10^5
- 1 \leq Q \leq 2 \times 10^5
- 1 \leq R_q \leq H
- 1 \leq C_q \leq W
- All input values are integers.

Sample Input 1

1 2
1 3

Sample Output 1

2

After processing all queries, there are two remaining walls, at (2, 1) and (2, 4).

            class FenwickTree:
                def __init__(self, size):
                    self.n = size
                    self.tree = [0] * (self.n+1)

                def update(self, index, delta):
                    # index is 0-indexed
                    i = index+1
                    while i <= self.n:
                        self.tree[i] += delta
                        i += i & -i

                def prefix_sum(self, index):
                    # including index (0-indexed, so from 0 to index)
                    res = 0
                    i = index+1
                    while i:
                        res += self.tree[i]
                        i -= i & -i
                    return res

                def range_sum(self, l, r):
                    if l > r:
                        return 0
                    return self.prefix_sum(r) - self.prefix_sum(l-1)

=== ANSWER ===
To solve this problem, we need to efficiently process a series of queries on a grid where each query either destroys a wall at a specific cell or destroys the nearest walls in the four cardinal directions if the cell is already empty. The challenge lies in handling these operations efficiently, especially when the grid is large and the number of queries is high.

### Approach
1. **Initialization**: We start by initializing a grid of size \(H \times W\) where each cell initially has a wall. We use two sets of Fenwick Trees (Binary Indexed Trees) to keep track of the alive walls in each row and column. This allows us to efficiently query the nearest walls in any direction.
2. **Processing Queries**: For each query:
   - **If the cell has a wall**: We destroy the wall at that cell, update the Fenwick Trees to reflect this, and increment the count of destroyed walls.
   - **If the cell is empty**: We find the nearest walls in the four directions (up, down, left, right) using binary search on the Fenwick Trees. For each found wall, we destroy it, update the Fenwick Trees, and increment the destroyed count.
3. **Result Calculation**: After processing all queries, the result is the total number of initial walls minus the count of destroyed walls.

### Solution Code
```python
import sys

class FenwickTree:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)
    
    def update(self, index, delta):
        i = index + 1
        while i <= self.n:
            self.tree[i] += delta
            i += i & -i
            
    def prefix_sum(self, index):
        if index < 0:
            return 0
        res = 0
        i = index + 1
        while i:
            res += self.tree[i]
            i -= i & -i
        return res

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    H = int(data[0])
    W = int(data[1])
    Q = int(data[2])
    queries = []
    index = 3
    for i in range(Q):
        r = int(data[index])
        c = int(data[index + 1])
        index += 2
        queries.append((r - 1, c - 1))
    
    total = H * W
    destroyed_count = 0
    alive = [True] * total
    
    row_fw = [FenwickTree(W) for _ in range(H)]
    col_fw = [FenwickTree(H) for _ in range(W)]
    
    for r in range(H):
        for c in range(W):
            row_fw[r].update(c, 1)
            col_fw[c].update(r, 1)
    
    for (r, c) in queries:
        idx = r * W + c
        if alive[idx]:
            alive[idx] = False
            destroyed_count += 1
            row_fw[r].update(c, -1)
            col_fw[c].update(r, -1)
        else:
            to_destroy = []
            if c - 1 >= 0:
                T = row_fw[r].prefix_sum(c - 1)
                if T > 0:
                    low, high = 0, c - 1
                    ans = -1
                    while low <= high:
                        mid = (low + high) // 2
                        s = row_fw[r].prefix_sum(mid)
                        if s == T:
                            ans = mid
                            low = mid + 1
                        elif s < T:
                            low = mid + 1
                        else:
                            high = mid - 1
                    if ans != -1:
                        to_destroy.append((r, ans))
            if c + 1 < W:
                T = row_fw[r].prefix_sum(c)
                T_total = row_fw[r].prefix_sum(W - 1)
                if T_total > T:
                    low, high = c + 1, W - 1
                    ans = -1
                    while low <= high:
                        mid = (low + high) // 2
                        s = row_fw[r].prefix_sum(mid)
                        if s == T + 1:
                            ans = mid
                            high = mid - 1
                        elif s < T + 1:
                            low = mid + 1
                        else:
                            high = mid - 1
                    if ans != -1:
                        to_destroy.append((r, ans))
            if r - 1 >= 0:
                T = col_fw[c].prefix_sum(r - 1)
                if T > 0:
                    low, high = 0, r - 1
                    ans = -1
                    while low <= high:
                        mid = (low + high) // 2
                        s = col_fw[c].prefix_sum(mid)
                        if s == T:
                            ans = mid
                            low = mid + 1
                        elif s < T:
                            low = mid + 1
                        else:
                            high = mid - 1
                    if ans != -1:
                        to_destroy.append((ans, c))
            if r + 1 < H:
                T = col_fw[c].prefix_sum(r)
                T_total = col_fw[c].prefix_sum(H - 1)
                if T_total > T:
                    low, high = r + 1, H - 1
                    ans = -1
                    while low <= high:
                        mid = (low + high) // 2
                        s = col_fw[c].prefix_sum(mid)
                        if s == T + 1:
                            ans = mid
                            high = mid - 1
                        elif s < T + 1:
                            low = mid + 1
                        else:
                            high = mid - 1
                    if ans != -1:
                        to_destroy.append((ans, c))
            for (i, j) in to_destroy:
                idx_d = i * W + j
                if alive[idx_d]:
                    alive[idx_d] = False
                    destroyed_count += 1
                    row_fw[i].update(j, -1)
                    col_fw[j].update(i, -1)
    print(total - destroyed_count)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The grid is initialized with all walls present. Fenwick Trees for each row and column are set up to track the presence of walls.
2. **Query Processing**:
   - **Direct Destruction**: If a query targets a cell with a wall, the wall is destroyed, and the corresponding Fenwick Trees are updated.
   - **Indirect Destruction**: If the target cell is empty, the nearest walls in the four directions are located using binary search on the Fenwick Trees. These walls are then destroyed, and the Fenwick Trees are updated.
3. **Efficiency**: The use of Fenwick Trees allows efficient updates and prefix sum queries, which are crucial for quickly locating the nearest walls in large grids. The binary search ensures that each direction query is handled in logarithmic time relative to the grid dimensions.
4. **Result Calculation**: The final result is derived by subtracting the count of destroyed walls from the total number of initial walls, providing the number of remaining walls after all queries.
