=== ANSWER ===
To solve this problem, we need to determine the maximum number of kagamimochi pairs that can be formed from a given set of mochi (rice cakes) arranged in ascending order of size. Each pair consists of a top mochi and a bottom mochi, where the size of the top mochi must be at most half the size of the bottom mochi. 

### Approach
1. **Problem Analysis**: The problem involves pairing mochi such that each pair (top, bottom) satisfies the condition \( \text{top} \leq \text{bottom}/2 \). The goal is to maximize the number of such pairs. Given that the mochi are sorted in ascending order, we can leverage a two-pointer technique to efficiently form these pairs.
2. **Intuition**: The smallest mochi are more likely to satisfy the condition when paired with larger mochi. Therefore, we can use the smallest half of the mochi as potential tops and the largest half as potential bottoms. By iterating through these halves with two pointers, we can greedily form valid pairs.
3. **Algorithm Selection**: 
   - **Two Pointers**: We initialize one pointer at the start of the array (for tops) and another at the middle of the array (for bottoms). For each bottom, we check if the current top satisfies the condition. If it does, we form a pair and move both pointers forward. Otherwise, we only move the bottom pointer to explore larger bottoms.
4. **Complexity Analysis**: The algorithm runs in \( O(N) \) time since each mochi is processed at most once by the pointers. The space complexity is \( O(1) \) as no additional space is used beyond the input array and a few variables.

### Solution Code
```python
def main():
    n = int(input().strip())
    A = list(map(int, input().split()))
    i = 0
    j = n // 2
    count = 0
    while j < n:
        if i < (n // 2) and 2 * A[i] <= A[j]:
            count += 1
            i += 1
            j += 1
        else:
            j += 1
    print(count)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The input consists of the number of mochi \( N \) and their sizes in ascending order. We initialize two pointers, `i` starting at 0 (for tops) and `j` starting at the middle of the array (for bottoms).
2. **Pair Formation**: For each bottom starting from the middle, we check if the current top (starting from the smallest) satisfies \( 2 \times \text{top} \leq \text{bottom} \). If it does, we increment the pair count and move both pointers forward. If not, we only move the bottom pointer to try the next larger bottom.
3. **Termination**: The loop terminates when the bottom pointer reaches the end of the array. The total count of valid pairs is printed as the result.

This approach efficiently maximizes the number of valid pairs by leveraging the sorted order and a greedy strategy with two pointers, ensuring optimal performance even for large input sizes.
