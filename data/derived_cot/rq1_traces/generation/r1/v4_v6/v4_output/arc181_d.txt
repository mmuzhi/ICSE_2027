You are given a permutation P=(P_1,P_2,\dots,P_N) of (1,2,\dots,N).
Consider the following operations k\ (k=2,3,\dots,N) on this permutation.

- Operation k: For i=1,2,\dots,k-1 in this order, if P_i > P_{i+1}, swap the values of the i-th and (i+1)-th elements of P.

You are also given a non-decreasing sequence A=(A_1,A_2,\dots,A_M)\ (2 \leq A_i \leq N) of length M.

 What is the inversion number of a sequence?

The inversion number of a sequence x=(x_1,x_2,\dots,x_n) of length n is the number of pairs of integers (i,j)\ (1\leq i < j \leq n) such that x_i > x_j.

N

Output

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq M \leq 2 \times 10^5
- 2 \leq A_i \leq N
- P is a permutation of (1,2,\dots,N).
- A_i \leq A_{i+1} for i=1,2,\dots,M-1.
- All input values are integers.

To solve this problem, we need to efficiently compute the inversion number of a permutation after applying a series of operations. Each operation involves a bubble sort-like pass on the first k elements of the permutation, which may reduce the inversion number. Given the constraints, a brute-force approach would be too slow for large inputs, so we need an optimized solution.

### Approach
1. **Initial Inversion Number Calculation**: The inversion number of the initial permutation is computed using a Fenwick Tree (Binary Indexed Tree) for efficiency, which allows O(n log n) time complexity.
2. **Simulating Operations**: For each operation specified in the sequence, we simulate the bubble sort pass on the first k elements. During this pass, whenever adjacent elements are out of order, they are swapped, and the inversion number is decremented by the number of swaps performed in the operation.
3. **Output Results**: After processing each operation, the current inversion number is recorded and printed.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    
    n = int(data[0])
    P = list(map(int, data[1:1+n]))
    m = int(data[1+n])
    A = list(map(int, data[1+n+1:1+n+1+m]))
    
    fenw = [0] * (n + 1)
    
    def update(index, delta):
        while index <= n:
            fenw[index] += delta
            index += index & -index
            
    def query(index):
        s = 0
        while index:
            s += fenw[index]
            index -= index & -index
        return s

    total_inversions = 0
    for i in range(n-1, -1, -1):
        total_inversions += query(P[i] - 1)
        update(P[i], 1)
    
    current_arr = P[:]
    results = []
    for k in A:
        swaps = 0
        for i in range(k - 1):
            if current_arr[i] > current_arr[i + 1]:
                current_arr[i], current_arr[i + 1] = current_arr[i + 1], current_arr[i]
                swaps += 1
        total_inversions -= swaps
        results.append(str(total_inversions))
    
    sys.stdout.write("\n".join(results) + "\n")

if __name__ == '__main__':
    main()
```

1. **Initial Inversion Number Calculation**: The initial inversion number is computed using a Fenwick Tree. The tree is updated as we traverse the permutation from right to left, counting the number of elements smaller than the current element that have been processed so far.
