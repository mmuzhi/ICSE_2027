=== PROMPT ===
You will be given a competitive programming problem.
There is a grid with H rows and W columns.
Let (i,j) denote the cell at the i-th row from the top and the j-th column from the left.
There are N horizontal bars numbered from 1 to N placed on the grid.
Bar i consists of L_i blocks of size 1 \times 1 connected horizontally, and its leftmost block is initially at cell (R_i, C_i).
That is, initially, bar i occupies the cells (R_i, C_i), (R_i, C_i + 1), \dots, (R_i, C_i + L_i - 1).
It is guaranteed that there is no cell occupied by two different bars.
The current time is t = 0.
At every time t = 0.5 + n for some non-negative integer n, the following occurs in order of i = 1, 2, \dots, N:

- If bar i is not on the bottom row (the H-th row), and none of the cells directly below the cells occupied by bar i is occupied by any bar, then bar i moves down by one cell. That is, if at that time bar i occupies the cells (r,C_i),(r,C_i+1),\dots,(r,C_i+L_i-1)\ (r < H), and the cell (r + 1, C_i + j) is not occupied by any bar for all j (0 \leq j \leq L_i - 1), then bar i now occupies (r + 1, C_i), (r + 1, C_i + 1), \dots, (r + 1, C_i + L_i - 1).
- Otherwise, nothing happens.

Let (R'_i, C_i), (R'_i, C_i + 1), \dots, (R'_i, C_i + L_i - 1) be the cells occupied by bar i at time t = 10^{100}. Find R'_1, R'_2, \dots, R'_N.

Input

The input is given from Standard Input in the following format:
H W N
R_1 C_1 L_1
R_2 C_2 L_2
\vdots
R_N C_N L_N

Output

Print N lines.
The i-th line (1 \leq i \leq N) should contain R'_i.

Constraints


- 1 \leq H, W \leq 2 \times 10^5
- 1 \leq N \leq 2 \times 10^5
- 1 \leq R_i \leq H
- 1 \leq C_i \leq W
- 1 \leq L_i \leq W - C_i + 1
- In the initial state, there is no cell occupied by two different bars.
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to simulate the falling process of horizontal bars on a grid until they reach their final positions. The bars fall in discrete time steps, and each bar moves down if the cells directly below it are unoccupied. The challenge is to efficiently determine the final row of each bar after an infinite number of time steps without simulating each step individually, which would be computationally infeasible for large grids.

### Approach
1. **Problem Analysis**: The problem involves multiple horizontal bars placed on a grid. Each bar consists of consecutive horizontal cells. At each time step, bars are processed in order. A bar moves down by one row if all cells directly below it are unoccupied. The goal is to find the final row of each bar after all possible movements.

2. **Key Insight**: The final position of each bar is determined by the highest obstacle (either another bar or the ground) directly below it. The obstacle for a bar is the closest bar (in terms of row) below it that shares at least one column. If no such bar exists, the ground is the obstacle.

3. **Algorithm Selection**: 
   - **Sorting**: Process bars in descending order of their initial rows. For bars in the same row, process them in descending order of their input index. This ensures that when processing a bar, all bars below it (which could be obstacles) have already been processed.
   - **Segment Tree**: Use a segment tree to efficiently query and update the minimal row (highest obstacle) in any column range. The segment tree supports range updates (setting each element in a range to the minimum of its current value and a new value) and range queries (finding the minimum value in a range).

4. **Complexity Analysis**: 
   - Sorting the bars takes O(N log N) time.
   - Each segment tree operation (query or update) takes O(log W) time per bar, where W is the number of columns.
   - Overall complexity is O(N log N + N log W), which is efficient for the given constraints (H, W, N up to 200,000).

### Solution Code
```python
import sys

class SegmentTree:
    def __init__(self, size, default):
        self.n = size
        self.default = default
        self.size = 1
        while self.size < size:
            self.size *= 2
        self.data = [default] * (2 * self.size)
        self.lazy = [None] * (2 * self.size)
    
    def apply(self, node, x):
        if self.lazy[node] is None:
            self.lazy[node] = x
        else:
            self.lazy[node] = min(self.lazy[node], x)
        self.data[node] = min(self.data[node], x)
    
    def push(self, node):
        if self.lazy[node] is not None:
            self.apply(2*node, self.lazy[node])
            self.apply(2*node+1, self.lazy[node])
            self.lazy[node] = None

    def update_range(self, l, r, x, node, seg_l, seg_r):
        if r < seg_l or seg_r < l:
            return
        if l <= seg_l and seg_r <= r:
            self.apply(node, x)
            return
        self.push(node)
        mid = (seg_l + seg_r) // 2
        self.update_range(l, r, x, 2*node, seg_l, mid)
        self.update_range(l, r, x, 2*node+1, mid+1, seg_r)
        self.data[node] = min(self.data[2*node], self.data[2*node+1])
    
    def query_range(self, l, r, node, seg_l, seg_r):
        if r < seg_l or seg_r < l:
            return self.default
        if l <= seg_l and seg_r <= r:
            return self.data[node]
        self.push(node)
        mid = (seg_l + seg_r) // 2
        left_val = self.query_range(l, r, 2*node, seg_l, mid)
        right_val = self.query_range(l, r, 2*node+1, mid+1, seg_r)
        return min(left_val, right_val)
    
    def update(self, l, r, x):
        self.update_range(l, r, x, 1, 0, self.size-1)
    
    def query(self, l, r):
        return self.query_range(l, r, 1, 0, self.size-1)

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    H = int(data[0])
    W = int(data[1])
    N = int(data[2])
    bars = []
    index = 3
    for i in range(N):
        R = int(data[index])
        C = int(data[index+1])
        L = int(data[index+2])
        index += 3
        bars.append((R, C, L, i))
    
    sorted_bars = sorted(bars, key=lambda x: (-x[0], -x[3]))
    seg_tree = SegmentTree(W, H+1)
    R_prime = [0] * N
    
    for bar in sorted_bars:
        R, C, L, idx = bar
        l0 = C - 1
        r0 = C + L - 2
        if l0 > r0:
            obstacle = H+1
        else:
            obstacle = seg_tree.query(l0, r0)
        R_prime[idx] = min(obstacle, H+1) - 1
        seg_tree.update(l0, r0, R_prime[idx])
    
    for i in range(N):
        print(R_prime[i])

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The input is read and parsed into the grid dimensions (H, W) and the number of bars (N). Each bar's initial position (row, column, length) and index are stored.
2. **Sorting**: Bars are sorted by their initial row in descending order. For bars in the same row, they are sorted by their input index in descending order. This ensures that bars below or in the same row (but processed later in the input) are processed first.
3. **Segment Tree**: A segment tree is initialized to keep track of the minimal row (highest obstacle) in each column range. The tree starts with all values set to a large number (H+1), representing no obstacles initially.
4. **Processing Bars**: For each bar in the sorted list:
   - **Query**: The minimal row (obstacle) in the bar's column range is queried.
   - **Update**: The bar's final row is computed as one less than the obstacle's row (or the ground if no obstacle exists). The segment tree is updated to reflect this bar's final position as a new obstacle.
5. **Output**: The final rows of the bars are printed in the original input order.

This approach efficiently determines the final positions of all bars by leveraging sorting and a segment tree for range queries and updates, ensuring optimal performance for large inputs.
