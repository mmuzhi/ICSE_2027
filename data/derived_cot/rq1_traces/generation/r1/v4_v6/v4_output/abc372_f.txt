There is a simple directed graph G with N vertices and N+M edges. The vertices are numbered 1 to N, and the edges are numbered 1 to N+M.
Edge i (1 \leq i \leq N) goes from vertex i to vertex i+1. (Here, vertex N+1 is considered as vertex 1.)
Edge N+i (1 \leq i \leq M) goes from vertex X_i to vertex Y_i.
Takahashi is at vertex 1. At each vertex, he can move to any vertex to which there is an outgoing edge from the current vertex.
Compute the number of ways he can move exactly K times.
That is, find the number of integer sequences (v_0, v_1, \dots, v_K) of length K+1 satisfying all of the following three conditions:

- 1 \leq v_i \leq N for i = 0, 1, \dots, K.
- v_0 = 1.
- There is a directed edge from vertex v_{i-1} to vertex v_i for i = 1, 2, \ldots, K.

Since this number can be very large, print it modulo 998244353.

Input

The input is given from Standard Input in the following format:
N M K
X_1 Y_1
X_2 Y_2
\vdots
X_M Y_M

Output

Print the count modulo 998244353.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 0 \leq M \leq 50
- 1 \leq K \leq 2 \times 10^5
- 1 \leq X_i, Y_i \leq N, X_i \neq Y_i
- All of the N+M directed edges are distinct.
- All input values are integers.

To solve this problem, we need to count the number of walks of length exactly \( K \) starting from vertex 1 in a directed graph that consists of a cycle plus some additional edges. The graph has \( N \) vertices and \( N + M \) edges, where the cycle edges connect vertex \( i \) to vertex \( i+1 \) (with vertex \( N \) connected to vertex 1), and the additional \( M \) edges are given as input.

### Approach
1. **Problem Analysis**: The problem requires counting all possible walks of length \( K \) starting from vertex 1. Given the constraints (up to \( N = 2 \times 10^5 \), \( K = 2 \times 10^5 \), and \( M \leq 50 \)), a brute-force approach is infeasible. Instead, we leverage dynamic programming (DP) with optimizations based on the graph's structure.

2. **Key Insight**: The graph consists of a primary cycle and a few additional edges. The key observation is that vertices not involved in any additional edges (either as start or end) can be handled efficiently by referencing the nearest important vertex (those involved in additional edges). This reduces the state space significantly.

3. **Dynamic Programming Setup**:
   - **State Representation**: The DP state \( dp[k][v] \) represents the number of ways to reach vertex \( v \) in exactly \( k \) moves. However, storing the full \( N \times K \) matrix is infeasible. Instead, we track only the important vertices (those involved in additional edges or the start vertex).
   - **State Transition**: For each important vertex \( v \), transitions are derived from:
     - The predecessor vertex in the cycle (either an important vertex or referenced via the nearest important vertex).
     - Any additional edges ending at \( v \) (similarly handled by referencing important vertices).

4. **Efficiency Optimization**:
   - **Precomputation**: For vertices not in the set of important vertices \( S \), precompute the nearest important vertex and the distance (number of base edges) to reach them.
   - **DP Initialization**: Initialize DP for the start vertex (vertex 1) at time 0.
   - **DP Transition**: For each time step \( k \) and each important vertex, compute the next state by summing contributions from relevant previous states (considering both base and additional edges).

5. **Result Calculation**: After filling the DP table, sum the counts for all vertices at time \( K \), handling non-important vertices by referencing their nearest important vertex.

import sys
import bisect

mod = 998244353

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it)); m = int(next(it)); K = int(next(it))
    edges = []
    for _ in range(m):
        x = int(next(it)); y = int(next(it))
        edges.append((x, y))
    
    S0 = {1}
    for x, y in edges:
        S0.add(x)
        S0.add(y)
    S_list = sorted(S0)
    len_S = len(S_list)
    S_index = {v: idx for idx, v in enumerate(S_list)}
    
    last_important = [0] * (n + 1)
    distance = [0] * (n + 1)
    sorted_imp = S_list
    max_imp = S_list[-1] if S_list else 0
    for v in range(1, n + 1):
        if v in S_index:
            continue
        pos = bisect.bisect_right(sorted_imp, v) - 1
        if pos >= 0:
            last_important[v] = sorted_imp[pos]
            distance[v] = v - last_important[v]
        else:
            last_important[v] = max_imp
            distance[v] = (n - max_imp) + v

    in_edges = [[] for _ in range(n + 1)]
    for x, y in edges:
        in_edges[y].append(x)
    
    L = [[] for _ in range(len_S)]
    for idx, v in enumerate(S_list):
        w = v - 1 if v > 1 else n
        if w in S_index:
            j = S_index[w]
            L[idx].append((0, j))
        else:
            u_imp = last_important[w]
            d = distance[w]
            j = S_index[u_imp]
            L[idx].append((d, j))
        
        for u in in_edges[v]:
            if u in S_index:
                j = S_index[u]
                L[idx].append((0, j))
            else:
                u_imp = last_important[u]
                d = distance[u]
                j = S_index[u_imp]
                L[idx].append((d, j))
    
    size = (K + 1) * len_S
    dp_flat = [0] * size
    idx_1 = S_index[1]
    dp_flat[0 * len_S + idx_1] = 1

    for k in range(1, K + 1):
        base_index = k * len_S
        for i in range(len_S):
            total = 0
            for offset, j in L[i]:
                t = k - 1 - offset
                if t >= 0:
                    idx_src = t * len_S + j
                    total = (total + dp_flat[idx_src]) % mod
            dp_flat[base_index + i] = total

    ans = 0
    for v in range(1, n + 1):
        if v in S_index:
            i = S_index[v]

            i = S_index[u_imp]
