=== ANSWER ===
To solve this problem, we need to find the minimum number of button presses required to display a given string `S` on a calculator that has buttons for digits `0-9` and a special button `00` which appends two zeros at once. The initial display is an empty string. The goal is to minimize the number of presses by optimally using the `00` button wherever possible.

### Approach
1. **Problem Analysis**: The string `S` consists of digits where the first character is never `0`. The calculator buttons allow appending either a single digit or two zeros (using the `00` button). The challenge is to cover the entire string `S` with the fewest tokens (button presses), leveraging the `00` button for consecutive zeros to reduce the number of presses.
2. **Key Insight**: For any run of consecutive zeros of length `k`, the minimal number of presses required is `(k + 1) // 2`. This is because:
   - For an even number of zeros, we can use `k/2` presses of the `00` button.
   - For an odd number of zeros, we use `(k-1)/2` presses of the `00` button and one press of the `0` button, totaling `(k + 1) // 2` presses.
3. **Algorithm**: Traverse the string from left to right:
   - For each non-zero digit, increment the press count by 1 and move to the next character.
   - For each zero, count the length of the consecutive zeros starting at that position, compute the minimal presses for that run using `(k + 1) // 2`, and skip the entire run.

### Solution Code
```python
def main():
    S = input().strip()
    n = len(S)
    i = 0
    presses = 0
    while i < n:
        if S[i] != '0':
            presses += 1
            i += 1
        else:
            j = i
            while j < n and S[j] == '0':
                j += 1
            k = j - i
            presses += (k + 1) // 2
            i = j
    print(presses)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The input string `S` is read, and variables `n` (length of `S`), `i` (current index), and `presses` (total button presses) are initialized.
2. **Traversal Loop**: The loop processes each character in `S`:
   - **Non-zero characters**: Each non-zero character requires one press (the corresponding digit button), and the index `i` is incremented by 1.
   - **Zero characters**: When a zero is encountered, the algorithm counts the length `k` of consecutive zeros starting at `i`. The minimal presses for this run is `(k + 1) // 2`, which efficiently covers even and odd lengths of zero runs. The index `i` is then set to the next non-zero character.
3. **Output**: After processing all characters, the total number of presses is printed.

This approach efficiently minimizes the button presses by optimally using the `00` button for consecutive zeros, ensuring the solution is both correct and optimal.
