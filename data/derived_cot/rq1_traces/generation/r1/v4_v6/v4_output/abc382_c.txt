You will be given a competitive programming problem.
There are N people numbered from 1 to N visiting a conveyor belt sushi restaurant.
The gourmet level of person i is A_i.
Now, M pieces of sushi will be placed on the conveyor belt.
The deliciousness of the j-th sushi is B_j.
Each piece of sushi passes in front of people 1, 2, \dots, N in this order.
Each person, when a sushi whose deliciousness is not less than their gourmet level passes in front of them, will take and eat that sushi; otherwise, they do nothing.
A sushi that person i takes and eats will no longer pass in front of person j\ (j > i).
For each of the M pieces of sushi, determine who eats that sushi, or if nobody eats it.

Input

The input is given from Standard Input in the following format:
N M
A_1 A_2 \dots A_N
B_1 B_2 \dots B_M

Output

Print M lines.
The j-th line (1 \leq j \leq M) should contain the number representing the person who eats the j-th sushi, or -1 if nobody eats it.

Constraints


- 1 \leq N, M \leq 2 \times 10^5
- 1 \leq A_i, B_i \leq 2 \times 10^5
- All input values are integers.

Sample Input 1

3 3
3 8 2
5 2 1

Sample Output 1

1
3
-1


- For the 1st sushi:
- It first passes in front of person 1. Since B_1 \geq A_1, person 1 takes and eats it.
- It will not pass in front of person 2 and 3.


- For the 2nd sushi:
- It first passes in front of person 1. Since B_2 < A_1, person 1 does nothing.
- Next, it passes in front of person 2. Since B_2 < A_2, person 2 does nothing.
- Finally, it passes in front of person 3. Since B_2 \geq A_3, person 3 takes and eats it.


- For the 3rd sushi:
- It first passes in front of person 1. Since B_3 < A_1, person 1 does nothing.
- Next, it passes in front of person 2. Since B_3 < A_2, person 2 does nothing.
- Finally, it passes in front of person 3. Since B_3 < A_3, person 3 does nothing.
- Therefore, nobody eats this sushi.

To solve this problem, we need to determine for each piece of sushi on the conveyor belt, which person (if any) will take and eat it. The sushi passes in front of each person in order, and the first person whose gourmet level is less than or equal to the sushi's deliciousness will take it. 

### Approach
1. **Problem Analysis**: The problem involves processing multiple queries (M pieces of sushi) where each query requires finding the smallest index of a person whose gourmet level is less than or equal to the sushi's deliciousness. The challenge is to efficiently answer these queries without resorting to a brute-force approach, which would be too slow for large inputs (N and M can be up to 200,000).

2. **Key Insight**: For each sushi with deliciousness `b`, we need the smallest index `i` such that `A_i <= b`. This can be rephrased as finding the minimum index among all persons with gourmet levels ≤ `b`.

3. **Preprocessing**:
   - **Min-Index Array**: Create an array `min_index` where `min_index[a]` stores the smallest index of any person with gourmet level `a`.
   - **Global-Min Array**: Construct an array `global_min` where `global_min[x]` is the minimum index among all persons with gourmet levels ≤ `x`. This array is built by iterating from 1 to the maximum value (200,000) and keeping track of the smallest index encountered so far.

4. **Query Processing**: For each sushi's deliciousness `b`, if `global_min[b]` is a valid index (not infinity), it indicates the person who will take the sushi; otherwise, no one takes it, and we return -1.

```python
    BIG = 10**9
    min_index = [BIG] * (max_val + 1)
    
    for idx, a_val in enumerate(A):
        person_index = idx + 1
        if person_index < min_index[a_val]:
            min_index[a_val] = person_index
            
    global_min = [BIG] * (max_val + 1)
    global_min[0] = BIG
    for x in range(1, max_val + 1):
        global_min[x] = min(global_min[x - 1], min_index[x])
    
    res = []
    for b in B:
        if b < 1:
            res.append(-1)
        else:
            ans = global_min[b]
            res.append(ans if ans != BIG else -1)
    
    for r in res:
        print(r)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of people `n`, the number of sushi pieces `m`, the gourmet levels `A` of the people, and the deliciousness values `B` of the sushi pieces.
2. **Initialization**: The `min_index` array is initialized to a large value (`BIG`) for all possible gourmet levels (up to 200,000). This array will store the smallest index of each gourmet level encountered.
3. **Building Min-Index Array**: For each person, update `min_index[a]` to the smallest index where gourmet level `a` occurs.
4. **Building Global-Min Array**: The `global_min` array is constructed such that `global_min[x]` holds the smallest index of any person with a gourmet level ≤ `x`. This is done by iterating through each possible value from 1 to 200,000 and taking the minimum between the previous value and the current `min_index[x]`.
5. **Processing Queries**: For each sushi piece, use `global_min` to quickly determine the smallest index of a person who can take it. If no such person exists (i.e., `global_min[b]` remains `BIG`), output -1.
6. **Output Results**: The results for each sushi piece are printed in order.

This approach efficiently preprocesses the data to answer each query in constant time, making the solution scalable for large inputs.
