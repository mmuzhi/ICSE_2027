=== ANSWER ===
To solve this problem, we need to simulate the operations of a stack that initially contains 100 cards, each labeled with the integer 0. The stack processes two types of queries: adding a card with a specific integer on top of the stack, and removing the top card from the stack and outputting its value. 

### Approach
1. **Initialization**: Start by initializing the stack with 100 zeros, representing the initial state of the stack.
2. **Processing Queries**: For each query:
   - **Type 1 (Add)**: Read the integer `x` and append it to the stack. This operation places the new card on top of the stack.
   - **Type 2 (Remove)**: Remove the top card from the stack (the last element in the list) and print its value. The stack is guaranteed to have at least one card during removal operations.
3. **Output Handling**: For each removal operation, the value of the removed card is printed immediately in the order the queries are processed.

### Solution Code
```python
def main():
    stack = [0] * 100
    Q = int(input().strip())
    for _ in range(Q):
        data = input().split()
        if data[0] == '1':
            x = int(data[1])
            stack.append(x)
        else:
            top = stack.pop()
            print(top)

if __name__ == "__main__":
    main()
```

### Explanation
- **Initialization**: The stack is initialized with 100 zeros to represent the initial state of the stack as described in the problem.
- **Processing Queries**: 
  - For a Type 1 query, the integer `x` is read from the input and appended to the stack. This operation effectively places a new card labeled `x` on top of the stack.
  - For a Type 2 query, the top card (last element of the list) is removed from the stack, and its value is printed. This operation simulates removing the top card from the stack and outputting its value.
- **Output Handling**: The value of the removed card is printed immediately during the processing of each Type 2 query, ensuring the output order matches the order of removal queries in the input.

This approach efficiently simulates the stack operations by leveraging list operations (append for adding and pop for removing), which are O(1) operations, making the solution both optimal and straightforward for the given constraints.
