You will be given a competitive programming problem.
This problem shares a similar setting with Problem A. The way Takahashi chooses cookies and what you are required to find are different from Problem A.
There are N boxes arranged in a row, and some of these boxes contain cookies.
The state of these boxes is represented by a string S of length N.
Specifically, the i-th box (1\leq i \leq N) from the left contains one cookie if the i-th character of S is @, and is empty if it is ..
Over the next D days, Takahashi will choose and eat one cookie per day from among the cookies in these boxes. On each day, he chooses the cookie in the rightmost box that contains a cookie at that point.
Determine, for each of the N boxes, whether it will contain a cookie after D days have passed.
It is guaranteed that S contains at least D occurrences of @.

Input

The input is given from Standard Input in the following format:
N D
S

Output

Print a string of length N.
The i-th character (1 \leq i \leq N) of the string should be @ if the i-th box from the left contains a cookie after D days have passed, and . otherwise.

Constraints


- 1 \leq D \leq N \leq 100
- N and D are integers.
- S is a string of length N consisting of @ and ..
- S contains at least D occurrences of @.

Sample Input 1

5 2
.@@.@

Sample Output 1

.@...

Takahashi acts as follows:

- Day 1: There are cookies in the 2nd, 3rd, and 5th boxes from the left. Among these, the rightmost is the 5th box. He eats the cookie in this box.
- Day 2: There are cookies in the 2nd and 3rd boxes. Among these, the rightmost is the 3rd box. He eats the cookie in this box.
- After two days have passed, only the 2nd box from the left contains a cookie.

Therefore, the correct output is .@....

Sample Input 2

3 3
@@@

Sample Output 2

...

Sample Input 3

10 4
@@@.@@.@@.

Sample Output 3

@@@.......

=== ANSWER ===
To solve this problem, we need to determine the state of boxes after Takahashi eats D cookies, one per day, by always choosing the rightmost cookie available each day. The solution involves identifying the positions of all cookies, then marking the last D cookies (in terms of their rightmost positions) as eaten, and finally generating the resulting state of the boxes.

### Approach
1. **Problem Analysis**: The problem involves processing a string representing boxes, some containing cookies ('@') and others empty ('.'). Over D days, Takahashi eats the rightmost cookie each day. The task is to output the final state of the boxes after D days.
2. **Key Insight**: The cookies eaten are the D rightmost cookies in the original list. By collecting all positions of cookies and sorting them (which inherently orders them from left to right), the last D positions in this list correspond to the cookies that will be eaten.
3. **Algorithm Selection**: 
   - Collect all indices where the box contains a cookie ('@').
   - From this list, select the last D indices (these are the cookies to be removed).
   - Construct the result string by replacing these indices with '.' in the original string, while leaving other boxes unchanged.
4. **Complexity Analysis**: The solution involves a linear scan to collect cookie positions (O(N)), and another linear pass to update the result (O(D)), making the overall complexity O(N), which is efficient given the constraints (N ≤ 100).

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data: 
        return
    n_d = data[0].split()
    N = int(n_d[0])
    D = int(n_d[1])
    S = data[1].strip()
    
    cookies = []
    for i, char in enumerate(S):
        if char == '@':
            cookies.append(i)
    
    to_remove = cookies[-D:] if D > 0 else []
    
    res_list = list(S)
    for idx in to_remove:
        res_list[idx] = '.'
    
    print(''.join(res_list))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input consists of two lines: the first line contains N (number of boxes) and D (number of days), and the second line contains the string S representing the initial state of the boxes.
2. **Collecting Cookie Positions**: We iterate through each character in S, recording the indices where a cookie ('@') is found.
3. **Identifying Cookies to Remove**: The last D indices in the collected list of cookie positions are the ones Takahashi will eat over D days.
4. **Updating Result**: We convert the original string S into a list for easy modification. For each index in the list of cookies to remove, we set the corresponding position in the result list to '.'.
5. **Output**: The modified list is joined into a string and printed, representing the final state of the boxes after D days.

This approach efficiently identifies and processes the cookies to be removed, ensuring the solution meets the problem requirements with optimal performance.
