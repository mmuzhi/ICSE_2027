You are given an integer array nums and an integer k.
An integer h is called valid if all values in the array that are strictly greater than h are identical.

For each index i where nums[i] > h, set nums[i] to h.

Return the minimum number of operations required to make every element in nums equal to k. If it is impossible to make all elements equal to k, return -1.

Example 3:

Input: nums = [9,7,5,3], k = 1
Output: 4
Explanation:
The operations can be performed using valid integers in the order 7, 5, 3, and 1.

1 <= k <= 100

To solve this problem, we need to determine the minimum number of operations required to make every element in an array equal to a given integer \( k \). The operations involve selecting a valid integer \( h \) and setting all elements greater than \( h \) to \( h \). The integer \( h \) is valid if all elements in the array strictly greater than \( h \) are identical. 

### Approach
1. **Check for Impossibility**: If any element in the array is less than \( k \), it's impossible to make all elements equal to \( k \) because the operations can only reduce elements, not increase them. Hence, we return -1 in such cases.
2. **Count Distinct Values Above \( k \)**: If all elements are at least \( k \), the number of operations required is equal to the number of distinct values in the array that are strictly greater than \( k \). Each operation reduces one distinct value above \( k \) to the next lower distinct value or directly to \( k \), thereby requiring one operation per distinct value.

### Solution Code
```python
class Solution:
    def minOperations(self, nums: List[int], k: int) -> int:
        distinct_set = set(nums)
        if min(distinct_set) < k:
            return -1
        return sum(1 for x in distinct_set if x > k)
```

### Explanation
1. **Check for Impossibility**: The first step is to check if there exists any element in the array that is less than \( k \). This is done by converting the array into a set of distinct values and checking if the minimum value in this set is less than \( k \). If so, it's impossible to achieve the goal, and we return -1.
2. **Count Operations**: If all elements are at least \( k \), the number of operations needed is simply the count of distinct values in the array that are greater than \( k \). Each distinct value above \( k \) will require one operation to reduce it to the next lower distinct value or directly to \( k \). This count is obtained by iterating over the distinct values and summing those that exceed \( k \).

This approach efficiently checks for feasibility and computes the solution by leveraging the properties of distinct values and their relationship with the target \( k \). The complexity is linear with respect to the size of the array due to the use of a set for distinct values.
