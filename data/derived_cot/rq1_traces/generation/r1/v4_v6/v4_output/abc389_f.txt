### Approach
1. **Problem Analysis**: For each query with an initial rating X, we simulate the process of participating in each contest sequentially. For each contest, if the current rating is within the contest's range [L_i, R_i], the rating increases by 1. The challenge is to compute this efficiently for multiple queries, especially given the constraints where N and Q can be as large as 200,000 and 300,000 respectively.

2. **Intuition**: The key observation is that the rating changes dynamically after each contest based on whether the current rating falls within the contest's range. The final rating for a given initial rating X is simply X plus the number of contests where the rating at the start of the contest was within the contest's range.

3. **Algorithm Selection**: Given the constraints, a direct simulation for each query is feasible. For each query, we iterate through all contests, checking if the current rating (initially X, then updated after each contest) lies within the contest's range. If it does, we increment the rating by 1. This approach is straightforward and leverages the fact that each contest's effect is independent and sequential.

4. **Complexity Analysis**: The approach involves O(N) operations per query, leading to a total of O(N*Q) operations. import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    contests = []
    index = 1
    for i in range(n):
        L = int(data[index])
        R = int(data[index + 1])
        index += 2
        contests.append((L, R))
    
    q = int(data[index])
    index += 1
    queries = []
    for i in range(q):
        X = int(data[index])
        index += 1
        queries.append(X)
    
    res = []
    for X in queries:
        current = X
        for L, R in contests:
            if L <= current <= R:
                current += 1
        res.append(str(current))
    
    sys.stdout.write("\n".join(res))

if __name__ == '__main__':
    main()
