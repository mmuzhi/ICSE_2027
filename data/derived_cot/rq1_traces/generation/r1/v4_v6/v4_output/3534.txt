To solve this problem, we need to count the number of pairs (i, j) in an array of positive integers such that i < j and the integers at these indices are "almost equal". Two integers are considered almost equal if they can be made equal by performing at most one swap operation on any two digits of either number. The swap operation can lead to leading zeros, which are allowed when comparing the numerical values of the resulting numbers.

### Approach
1. **Problem Analysis**: The problem requires checking all pairs (i, j) where i < j to determine if the integers nums[i] and nums[j] can be made equal by swapping two digits in one of them at most once. The key insight is that after swapping, the numerical value of the modified number must match the other number. This involves:
   - **Direct Equality**: If the numbers are already equal, they trivially satisfy the condition.
   - **Single Swap**: For numbers that are not equal, we need to check if swapping any two digits in either number results in the other number.

2. **Intuition**: For each number in the array, precompute all possible numerical values that can be obtained by swapping any two digits (including the original number). This set of values represents all possible outcomes of the number after zero or one swap operations. Then, for each pair (i, j), check if nums[j] is in the precomputed set of nums[i] or if nums[i] is in the precomputed set of nums[j]. If either condition is true, the pair is counted.

3. **Algorithm Selection**:
   - **Precomputation**: For each number in the array, generate a set of integers that includes the original number and all numbers formed by swapping any two digits. This set is stored for quick lookup.
   - **Pair Checking**: Iterate over all pairs (i, j) where i < j. For each pair, check if nums[j] is in the set of nums[i] or if nums[i] is in the set of nums[j]. If so, increment the count.

4. **Complexity Analysis**:
   - **Precomputation**: For each number with d digits, generating all possible swaps takes O(d^2) time. Given the constraint d ≤ 7 (since numbers are up to 10^6), this step is efficient. For n numbers, the total precomputation time is O(n * d^2).
   - **Pair Checking**: Checking all pairs (i, j) takes O(n^2) time. For each pair, set membership checks are O(1) on average. With n ≤ 100, this results in 4950 operations, which is feasible.

### Solution Code
```python
class Solution:
    def countPairs(self, nums: List[int]) -> int:
        sets = []
        for x in nums:
            s = set()
            s.add(x)
            if x >= 10:
                digits_str = str(x)
                n_digits = len(digits_str)
                digits_list = list(digits_str)
                for i in range(n_digits):
                    for j in range(i+1, n_digits):
                        digits_list[i], digits_list[j] = digits_list[j], digits_list[i]
                        swapped_val = int(''.join(digits_list))
                        s.add(swapped_val)
                        digits_list[i], digits_list[j] = digits_list[j], digits_list[i]
            sets.append(s)
        
        count = 0
        n = len(nums)
        for i in range(n):
            for j in range(i+1, n):
                if nums[j] in sets[i] or nums[i] in sets[j]:
                    count += 1
        return count
```

### Explanation
1. **Precomputation**: For each number in the array, we generate a set of integers that includes the original number and all numbers formed by swapping any two digits. This is done by converting the number to a string, swapping each pair of digits, converting the resulting string back to an integer, and adding it to the set. This set is stored in a list for later access.
2. **Pair Checking**: We iterate over all pairs (i, j) where i < j. For each pair, we check if nums[j] is in the set of nums[i] or if nums[i] is in the set of nums[j]. If either condition is true, the pair is counted as almost equal.
3. **Result**: The total count of such valid pairs is returned as the result.

This approach efficiently checks all possible pairs by leveraging precomputed sets, ensuring optimal performance within the problem constraints.
