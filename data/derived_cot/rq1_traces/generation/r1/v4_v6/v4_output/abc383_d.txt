To solve this problem, we need to count the number of positive integers not greater than \( N \) that have exactly 9 positive divisors. The key insight is recognizing that numbers with exactly 9 divisors must be of one of two forms: either \( p^8 \) where \( p \) is a prime, or \( p^2 \times q^2 \) where \( p \) and \( q \) are distinct primes. 

### Approach
1. **Problem Analysis**: The number of divisors of a number \( n \) is determined by its prime factorization. For \( n \) to have exactly 9 divisors, the exponent pattern in its prime factorization must result in the product of (exponents + 1) equaling 9. The possible forms are:
   - \( p^8 \): Here, the number of divisors is \( 8 + 1 = 9 \).
   - \( p^2 \times q^2 \): Here, the number of divisors is \( (2 + 1) \times (2 + 1) = 9 \).

2. **Intuition**: 
   - **Case 1 (p^8)**: For each prime \( p \), check if \( p^8 \leq N \). The primes \( p \) are small because \( p^8 \) grows rapidly.
   - **Case 2 (p^2 \times q^2)**: For distinct primes \( p \) and \( q \), check if \( (p \times q)^2 \leq N \). This translates to \( p \times q \leq \sqrt{N} \). We generate all primes up to \( \sqrt{N} \) and count pairs \( (p, q) \) such that \( p < q \) and \( p \times q \leq \sqrt{N} \).

3. **Algorithm Selection**:
   - **Prime Generation**: Use the Sieve of Eratosthenes to generate primes up to \( \sqrt{N} \).
   - **Case 1 Handling**: Iterate through primes and count those where \( p^8 \leq N \).
   - **Case 2 Handling**: For each prime \( p \), use binary search to count primes \( q \) (greater than \( p \)) such that \( p \times q \leq \sqrt{N} \).

4. **Complexity Analysis**:
   - **Prime Generation**: The sieve runs in \( O(\sqrt{N} \log \log \sqrt{N}) \) time.
   - **Case 1**: Since \( p^8 \leq N \) implies \( p \) is small (up to \( N^{1/8} \)), this step is \( O(1) \).
   - **Case 2**: For each prime \( p \) (up to \( \sqrt{N}/2 \)), a binary search is performed, leading to \( O(\pi(\sqrt{N}) \log \pi(\sqrt{N})) \) time, where \( \pi(x) \) is the prime-counting function. Given \( \sqrt{N} \leq 2 \times 10^6 \), this is feasible.

### Solution Code
```python
import math
import bisect

def generate_primes(n):
    if n < 2:
        return []
    sieve = [True] * (n + 1)
    sieve[0] = sieve[1] = False
    i = 2
    while i * i <= n:
        if sieve[i]:
            sieve[i*i : n+1 : i] = [False] * ((n - i*i) // i + 1)
        i += 1
    return [i for i in range(n+1) if sieve[i]]

def main():
    N = int(input().strip())
    if N < 2:
        print(0)
        return
        
    max_product = math.isqrt(N)
    primes = generate_primes(max_product) if max_product >= 2 else []
    
    count1 = 0
    for p in primes:
        num = p ** 8
        if num <= N:
            count1 += 1
        else:
            break
    
    count2 = 0
    if len(primes) >= 2:
        for i in range(len(primes)):
            p = primes[i]
            if p > max_product // 2:
                break
            right_bound = max_product // p
            j = bisect.bisect_right(primes, right_bound)
            if j > i + 1:
                count2 += j - (i + 1)
    
    print(count1 + count2)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Prime Generation**: The `generate_primes` function uses the Sieve of Eratosthenes to efficiently generate all primes up to \( \sqrt{N} \).
2. **Case 1 Handling**: For each prime \( p \), it checks if \( p^8 \leq N \). If so, it increments `count1`.
3. **Case 2 Handling**: For each prime \( p \), it calculates the maximum prime \( q \) such that \( p \times q \leq \sqrt{N} \). Using binary search, it counts valid primes \( q \) greater than \( p \) and updates `count2`.
4. **Result**: The total count is the sum of `count1` (numbers of form \( p^8 \)) and `count2` (numbers of form \( p^2 \times q^2 \)), printed as the result.

This approach efficiently counts numbers with exactly 9 divisors by leveraging prime factorization properties and optimized prime generation and searching techniques.
