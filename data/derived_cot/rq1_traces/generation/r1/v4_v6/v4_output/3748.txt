=== ANSWER ===
To solve this problem, we need to sort the diagonals of a given square matrix in specific orders. The diagonals in the bottom-left triangle (including the main diagonal) should be sorted in non-increasing order, while the diagonals in the top-right triangle should be sorted in non-decreasing order. 

### Approach
1. **Identify Diagonals**: Each diagonal in the matrix can be uniquely identified by the difference `d = i - j`, where `i` is the row index and `j` is the column index. The diagonals with `d >= 0` belong to the bottom-left triangle (including the main diagonal), and those with `d < 0` belong to the top-right triangle.
2. **Collect Diagonal Elements**: For each diagonal (i.e., for each unique `d`), collect all elements along that diagonal. This involves traversing the matrix and grouping elements by their `d` value.
3. **Sort Diagonals**: For each diagonal:
   - If `d >= 0`, sort the elements in non-increasing order (descending).
   - If `d < 0`, sort the elements in non-decreasing order (ascending).
4. **Reassign Sorted Elements**: Place the sorted elements back into their original positions in the matrix.

### Solution Code
```python
class Solution:
    def sortMatrix(self, grid: List[List[int]]) -> List[List[int]]:
        n = len(grid)
        for d in range(-(n-1), n):
            coords = []
            if d >= 0:
                i, j = d, 0
                while i < n and j < n:
                    coords.append((i, j))
                    i += 1
                    j += 1
            else:
                i, j = 0, -d
                while i < n and j < n:
                    coords.append((i, j))
                    i += 1
                    j += 1
            values = [grid[i][j] for i, j in coords]
            if d >= 0:
                values.sort(reverse=True)
            else:
                values.sort()
            for (i, j), val in zip(coords, values):
                grid[i][j] = val
        return grid
```

### Explanation
1. **Loop Through Diagonals**: The loop iterates over all possible diagonal differences `d` from `-(n-1)` to `n-1`. Each `d` represents a unique diagonal in the matrix.
2. **Collect Coordinates**: For each `d`, collect the coordinates `(i, j)` of all elements on the diagonal. If `d >= 0`, start from `(d, 0)` and move down-right. If `d < 0`, start from `(0, -d)` and move down-right.
3. **Extract and Sort Values**: Extract the values from the collected coordinates. Sort these values in descending order if `d >= 0` (bottom-left triangle) or ascending order if `d < 0` (top-right triangle).
4. **Update Matrix**: Place the sorted values back into their original positions in the matrix using the collected coordinates.
5. **Return Result**: After processing all diagonals, return the modified matrix.

This approach efficiently processes each diagonal independently, ensuring the required sorting orders are applied, and the matrix is updated in place for optimal performance. The complexity is O(n^2 log n) due to sorting each diagonal, which is feasible given the constraint `n <= 10`.
