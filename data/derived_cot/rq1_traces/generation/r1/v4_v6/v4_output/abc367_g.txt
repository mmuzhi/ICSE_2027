You are given positive integers N, M, K, and a sequence of non-negative integers: A=(A_1,A_2,\ldots,A_N).
For a non-empty non-negative integer sequence B=(B_1,B_2,\ldots,B_{|B|}), we define its score as follows.

- If the length of B is a multiple of M: (B_1 \oplus B_2 \oplus \dots \oplus B_{|B|})^K
- Otherwise: 0

Here, \oplus represents the bitwise XOR.
Find the sum, modulo 998244353, of the scores of the 2^N-1 non-empty subsequences of A.
What is bitwise XOR? The bitwise XOR of non-negative integers A and B, denoted as A \oplus B, is defined as follows: - In the binary representation of A \oplus B, the digit at position 2^k (k \geq 0) is 1 if exactly one of A and B has a 1 in that position in their binary representations, and 0 otherwise. For example, 3 \oplus 5 = 6 (in binary: 011 \oplus 101 = 110). In general, the XOR of k integers p_1, \dots, p_k is defined as (\cdots ((p_1 \oplus p_2) \oplus p_3) \oplus \cdots \oplus p_k), and it can be proved that this is independent of the order of p_1, \dots, p_k.

Input

The input is given from Standard Input in the following format:
N M K
A_1 A_2 \ldots A_N

Output

Print the answer.

Constraints


- 1 \leq N,K \leq 2 \times 10^5
- 1 \leq M \leq 100
- 0 \leq A_i < 2^{20}
- All input values are integers.

To solve this problem, we need to compute the sum of scores of all non-empty subsequences of a given sequence, where the score of a subsequence is defined as the K-th power of the XOR of its elements if its length is a multiple of M, and zero otherwise. Given the constraints, a brute-force approach is infeasible, so we use generating functions and the Fast Walsh-Hadamard Transform (FWHT) to efficiently compute the result.

### Approach
1. **Problem Analysis**: The problem requires summing the scores of all non-empty subsequences where the length is a multiple of M. The score for such a subsequence is (XOR of its elements)^K. The challenge is to compute this sum efficiently given the constraints (N, K up to 200,000 and M up to 100).

2. **Key Insight**: The solution leverages generating functions and FWHT to handle the XOR convolution efficiently. The generating function tracks the number of subsequences for each possible XOR value and length modulo M. The FWHT transforms the problem into the Walsh basis, where polynomial multiplication becomes pointwise multiplication, simplifying the computation.

3. **Algorithm Selection**:
   - **Frequency Array**: First, we compute the frequency of each element in the sequence.
   - **FWHT Application**: We apply FWHT to transform the frequency array and another array where each entry is the K-th power of the index.
   - **Root of Unity**: For M-th roots of unity in the field, we compute the product of generating functions for each root, leveraging properties of cyclic polynomials.
   - **Inverse Transform**: After processing all elements, we perform an inverse FWHT to obtain the result in the original domain, then compute the final sum by dotting with the transformed K-th power array.

4. **Optimization**: To handle large N and K, the solution precomputes power arrays for bases derived from roots of unity, reducing the exponentiation overhead during the main computation loop.
