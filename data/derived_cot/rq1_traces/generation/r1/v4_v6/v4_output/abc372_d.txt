=== PROMPT ===
You will be given a competitive programming problem.
There are N buildings, Building 1, Building 2, \ldots, Building N, arranged in a line in this order. The height of Building i (1 \leq i \leq N) is H_i.
For each i = 1, 2, \ldots, N, find the number of integers j (i < j \leq N) satisfying the following condition:

- There is no building taller than Building j between Buildings i and j.

Input

The input is given from Standard Input in the following format:
N
H_1 H_2 \ldots H_N

Output

For each i = 1, 2, \ldots, N, let c_i be the number of j satisfying the condition. Print c_1, c_2, \ldots, c_N in order, separated by spaces.

Constraints


- 1 \leq N \leq 2 \times 10^5
- 1 \leq H_i \leq N
-  H_i\neq H_j\ (i\neq j)
- All input values are integers.

Sample Input 1

5
2 1 4 3 5

Sample Output 1

3 2 2 1 0

Sample Output 2

Sample Input 3

To solve this problem, we need to determine for each building \(i\) (from 1 to \(N\)) the number of buildings \(j\) (where \(j > i\)) such that there is no building taller than building \(j\) between buildings \(i\) and \(j\). 

### Approach
1. **Problem Analysis**: The problem requires us to count, for each building \(i\), the number of buildings \(j\) to its right that are "visible" in the sense that all buildings between \(i\) and \(j\) are not taller than \(j\). This can be rephrased as: for each \(j\), the buildings \(i\) that can "see" \(j\) must lie between the last building taller than \(j\) (to the left of \(j\)) and \(j\) itself.

2. **Key Insight**: For each building \(j\), the set of buildings \(i\) that can see \(j\) are those from the position immediately after the last building taller than \(j\) (or the first building if none exists) up to \(j-1\). This insight allows us to use a difference array to efficiently count the contributions of each \(j\) to the relevant \(i\) positions.

3. **Algorithm Selection**:
   - **Previous Greater Element (PGE)**: We first compute for each building \(j\) the position of the last building to its left that is taller than \(j\). This is done using a stack to maintain a decreasing sequence of heights.
   - **Difference Array**: For each \(j\), we update a difference array to mark the range of \(i\) positions (from the PGE position + 1 to \(j-1\)) that can see \(j\). This is efficiently handled by incrementing the start of the range and decrementing the end+1 in the difference array.
   - **Prefix Sum**: Finally, we compute the prefix sum of the difference array to get the count for each building \(i\).

4. **Complexity Analysis**: The algorithm processes each building once for the PGE computation and once for updating the difference array, leading to an overall time complexity of \(O(N)\). The space complexity is \(O(N)\) for storing the PGE, stack, and difference array.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    H = list(map(int, data[1:1+n]))
    
    prev_greater = [-1] * n
    stack = []
    for i in range(n):
        while stack and H[stack[-1]] <= H[i]:
            stack.pop()
        if stack:
            prev_greater[i] = stack[-1]
        else:
            prev_greater[i] = -1
        stack.append(i)
    
    diff = [0] * (n + 1)
    
    for j in range(1, n):
        lb = prev_greater[j] if prev_greater[j] != -1 else 0
        diff[lb] += 1
        diff[j] -= 1
    
    res = []
    current = 0
    for i in range(n):
        current += diff[i]
        res.append(str(current))
    
    print(" ".join(res))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of buildings \(N\) and their respective heights \(H\).
2. **Computing Previous Greater Element (PGE)**: Using a stack, we traverse the buildings from left to right. For each building, we pop elements from the stack until we find a building taller than the current one. The top of the stack at this point gives the PGE for the current building. If the stack is empty, there is no PGE.
3. **Difference Array Updates**: For each building \(j\) (from 1 to \(N-1\) in 0-based index), we determine the left boundary (PGE position or 0 if none exists). We then increment the start of the range (left boundary) in the difference array and decrement the position immediately after the end of the range (\(j\)) to mark the valid \(i\) positions that can see \(j\).
4. **Prefix Sum Calculation**: By computing the prefix sum of the difference array, we obtain the count of valid \(j\) buildings for each \(i\), which is then printed as the result.

This approach efficiently counts the required buildings for each \(i\) by leveraging the properties of the PGE and difference arrays, ensuring optimal performance even for large inputs.
