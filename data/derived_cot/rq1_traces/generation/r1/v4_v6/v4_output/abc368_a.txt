=== ANSWER ===
To solve this problem, we need to simulate the operation of taking K cards from the bottom of a stack and placing them on top, maintaining their order. The solution involves rearranging the elements of the given list such that the last K elements are moved to the beginning of the list.

### Approach
1. **Problem Analysis**: The problem requires us to transform the initial list of integers representing the stack of cards. The transformation involves moving the last K elements (bottom K cards) to the front of the list (top of the stack) while preserving the order of the remaining elements.
2. **Intuition**: The solution can be derived by splitting the original list into two parts. The first part consists of the elements from the beginning up to the (N-K)-th element, and the second part consists of the last K elements. The transformed list is then constructed by concatenating the second part followed by the first part.
3. **Algorithm Selection**: The algorithm involves slicing the original list into two segments and combining them in reverse order. Specifically, the segment from index (N-K) to the end is placed before the segment from the start to index (N-K).
4. **Complexity Analysis**: The solution involves slicing the list, which is an O(N) operation in Python. Since the constraints specify that N is at most 100, the solution is efficient and optimal for the given problem size.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    k = int(data[1])
    A = list(map(int, data[2:2+n]))
    result = A[n - k:] + A[:n - k]
    print(" ".join(map(str, result)))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line contains two integers, N (number of cards) and K (number of cards to move from the bottom to the top). The second line contains N integers representing the cards from top to bottom.
2. **Slicing the List**: The list `A` is split into two segments. The first segment `A[n - k:]` contains the last K elements (bottom K cards), and the second segment `A[:n - k]` contains the remaining elements (top N-K cards).
3. **Constructing Result**: The result is formed by concatenating the first segment (bottom K cards) followed by the second segment (top N-K cards). This effectively moves the bottom K cards to the top of the stack.
4. **Output**: The result is printed as space-separated integers, representing the new order of cards from top to bottom.

This approach efficiently rearranges the elements by leveraging list slicing and concatenation, ensuring the solution is both simple and optimal for the given constraints.
