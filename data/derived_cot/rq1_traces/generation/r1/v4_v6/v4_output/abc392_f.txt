There is an empty array A. For i = 1,2,\ldots,N, perform the following operation in order:

Output the final array A after all operations have been completed.

Input

The input is given from Standard Input in the following format:
N
P_1 P_2 \ldots P_N

Output

Let the final array be A = (A_1, A_2, \ldots, A_N).
Print A_1, A_2, \ldots, A_N in this order, separated by spaces.

Constraints


- 1 \leq N \leq 5\times 10^5
- 1 \leq P_i \leq i
- All input values are integers.

To solve this problem, we need to efficiently simulate the insertion operations into an array as specified, but without the high time complexity of a naive approach. The key insight is to process the operations in reverse order and use a segment tree to track available positions in the final array, allowing us to determine the correct position for each number in logarithmic time.

### Approach
1. **Problem Analysis**: The problem involves inserting numbers from 1 to N into an array in a specific order. Each number `i` is inserted at the `P_i`-th position of the current array. The challenge is to efficiently compute the final array after all insertions, especially given the constraint that N can be as large as 500,000, which makes a naive O(N^2) approach infeasible.

2. **Intuition**: By processing the insertions in reverse order (from N down to 1), we can leverage a segment tree to keep track of available positions in the final array. For each number `i`, we determine its position by finding the `P_i`-th available position in the segment tree, place `i` at that position, and then mark the position as occupied (update the segment tree).

3. **Algorithm Selection**: We use a segment tree that supports:
   - **Building**: Initialize the tree with all positions marked as available (1).
   - **Updating**: Mark a position as occupied (0) after placing a number.
   - **Querying**: Efficiently find the `k`-th available position (the position where the prefix sum of available positions reaches `k`).

4. **Complexity Analysis**: Each insertion (query and update) operation on the segment tree takes O(log N) time. With N insertions, the total time complexity is O(N log N), which is efficient for the given constraints.

### Solution Code
```python
import sys

class SegmentTree:
    def __init__(self, n):
        self.n = n
        self.size = 1
        while self.size < n:
            self.size *= 2
        self.tree = [0] * (2 * self.size)
    
    def build(self):
        for i in range(self.n):
            self.tree[self.size + i] = 1
        for i in range(self.size - 1, 0, -1):
            self.tree[i] = self.tree[2*i] + self.tree[2*i+1]
    
    def update(self, index, value):
        pos = self.size + index
        self.tree[pos] = value
        while pos > 1:
            pos //= 2
            self.tree[pos] = self.tree[2*pos] + self.tree[2*pos+1]
    
    def query_kth(self, k):
        idx = 1
        while idx < self.size:
            left_child = 2 * idx
            right_child = 2 * idx + 1
            if self.tree[left_child] >= k:
                idx = left_child
            else:
                k -= self.tree[left_child]
                idx = right_child
        return idx - self.size

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    p = list(map(int, data[1:1+n]))
    seg_tree = SegmentTree(n)
    seg_tree.build()
    res = [0] * n
    for i in range(n-1, -1, -1):
        k = p[i]
        pos = seg_tree.query_kth(k)
        res[pos] = i + 1
        seg_tree.update(pos, 0)
    print(" ".join(map(str, res)))

if __name__ == '__main__':
    main()
```

2. **Processing Insertions**: The algorithm processes numbers from N down to 1. For each number `i`, it queries the segment tree to find the `P_i`-th available position (using `query_kth`), places `i` at that position in the result array, and updates the tree to mark the position as occupied (using `update`).
3. **Output**: After processing all numbers, the result array is printed, which reflects the final state of the array after all insertions.
