- 0 \le K_i \le N^2

- All inputs are integers

After research online, I recall that the number of fixed elements in a (0,1)-matrix with given row and column sums is related to the notion of "bottleneck" and can be computed using the formula for the minimum and maximum number of 1's at each cell. 

And in fact, the total number of fixed elements is: 
      sum_{i,j} [ [min_number(i,j) == 1] or [max_number(i,j) == 0] ]

And the min_number(i,j) = max(0, r_i + c_j - (F - \min_{S \subseteq [N]\setminus\{i\}, T \subseteq [N]\setminus\{j\}} ( \sum_{k \in S} r_k + \sum_{l \in T} c_l - |S||T| ))) 
And similarly, the max_number(i,j) = min(r_i, c_j, \min_{S \subseteq [N]\setminus\{i\}, T \subseteq [N]\setminus\{j\}} ( \sum_{k \in S} r_k + \sum_{l \in T} c_l - |S||T| )) 

But then, to compute the min_{S,T} ( \sum_{k in S} r_k + \sum_{l in T} c_l - |S||T| ) for each (i,j), we need to iterate over subsets S and T. 

That is O(2^(2*(N-1)) per (i,j), which for N=30 is 2^58, which is about 3e17, which is not feasible for a single (i,j) let alone for each (i,j) and each row sum vector.

Actually, the max_{S,T} ( \sum_{k \in S} r_k + \sum_{l \ in T} c_l - |S||T| ) is known as the "maximum excess" and can be computed by a single DP. 

In fact, the expression inside is supermodular, and there is an O(N^2) algorithm to compute it. 

But then, for each (i,j), we have: 
      L(i,j) = max(0, r_i + c_j - ( \max_{S,T} ( \sum_{k \in S} r_k + \sum_{l \ in T} c_l - |S||T| ) )) 

But wait, the expression inside the max_{S,T} is over S not containing i and T not containing j? 

In the formula for L(i,j), the S is a subset of [N]\{i} and T is a subset of [N]\{j}.
