=== PROMPT ===
You will be given a competitive programming problem.
There is an undirected graph with N vertices and 0 edges. The vertices are numbered 1 to N.
You are given Q queries to process in order. Each query is of one of the following two types:

- Type 1: Given in the format 1 u v. Add an edge between vertices u and v.
- Type 2: Given in the format 2 v k. Print the k-th largest vertex number among the vertices connected to vertex v. If there are fewer than k vertices connected to v, print -1.

Input

The input is given from Standard Input in the following format:
N Q
\mathrm{query}_1
\mathrm{query}_2
\vdots
\mathrm{query}_Q

Here, \mathrm{query}_i is the i-th query and is given in one of the following formats:
1 u v

2 v k

Output

Let q be the number of Type 2 queries. Print q lines.
The i-th line should contain the answer to the i-th Type 2 query.

Constraints


- 1 \leq N, Q \leq 2 \times 10^5
- In a Type 1 query, 1 \leq u < v \leq N.
- In a Type 2 query, 1 \leq v \leq N, 1 \leq k \leq 10.
- All input values are integers.

To solve this problem, we need to efficiently process a series of queries on an initially empty undirected graph. The queries involve adding edges between vertices and querying the k-th largest vertex number in the connected component of a given vertex. Given the constraints, we need an approach that can handle up to 200,000 queries efficiently.

### Approach
1. **Union-Find Data Structure with Path Compression and Union by Rank**: This data structure helps manage the connected components efficiently. Each component is represented by a root node, and we use path compression during find operations to keep the tree flat, ensuring near-constant time complexity for find operations.
2. **Tracking Component Size and Top Values**: For each root node, we maintain the size of its component and a list of the top 10 largest vertex numbers in the component. This list is kept sorted in descending order to facilitate quick access to the k-th largest element.
3. **Efficient Merging of Components**: When merging two components, we combine their top 10 lists, sort the combined list in descending order, and retain only the top 10 values. This ensures that each root's top 10 list is always up-to-date and accurate for queries.
4. **Query Processing**: For each query of type 2, we find the root of the given vertex, check if the component size is at least k, and return the k-th largest value from the root's top 10 list if it exists; otherwise, we return -1.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    q = int(data[1])
    queries = []
    idx = 2
    for i in range(q):
        t = int(data[idx])
        if t == 1:
            u = int(data[idx+1])
            v = int(data[idx+2])
            idx += 3
            queries.append((t, u, v))
        else:
            v = int(data[idx+1])
            k = int(data[idx+2])
            idx += 3
            queries.append((t, v, k))
    
    parent = list(range(n+1))
    rank = [0] * (n+1)
    size = [1] * (n+1)
    top10 = [[] for _ in range(n+1)]
    for i in range(1, n+1):
        top10[i] = [i]
    
    def find(x):
        root = x
        while root != parent[root]:
            root = parent[root]
        temp = x
        while temp != root:
            nxt = parent[temp]
            parent[temp] = root
            temp = nxt
        return root
    
    def union(u, v):
        ru = find(u)
        rv = find(v)
        if ru == rv:
            return
        if rank[ru] < rank[rv]:
            ru, rv = rv, ru
        parent[rv] = ru
        size[ru] += size[rv]
        merged_list = top10[ru] + top10[rv]
        merged_list.sort(reverse=True)
        top10[ru] = merged_list[:10]
        if rank[ru] == rank[rv]:
            rank[ru] += 1
    
    output_lines = []
    for query in queries:
        if query[0] == 1:
            u, v = query[1], query[2]
            union(u, v)
        else:
            v, k = query[1], query[2]
            root = find(v)
            if k > size[root]:
                output_lines.append("-1")
            else:
                output_lines.append(str(top10[root][k-1]))
    
    print("\n".join(output_lines))

if __name__ == '__main__':
    main()
```

1. **Initialization**: The code starts by reading the input values and initializing the necessary data structures: `parent` to track each node's root, `rank` for union by rank optimization, `size` to store the size of each component, and `top10` to keep the top 10 largest vertex numbers for each component.
2. **Find Operation**: The `find` function uses path compression to flatten the tree structure, ensuring efficient future queries. It traverses up to find the root and then updates the parent pointers of all nodes along the path to point directly to the root.
3. **Union Operation**: The `union` function merges two components by rank. It updates the parent pointer of the smaller rank root to point to the larger rank root. The sizes are combined, and the top 10 lists are merged, sorted, and truncated to retain only the top 10 values.
4. **Query Processing**: For each query, if it's an edge addition (type 1), the corresponding vertices are unioned. For a k-th largest query (type 2), the root of the vertex is found, and if the component size is sufficient, the k-th largest value from the root's top 10 list is returned; otherwise, -1 is returned.
5. **Output**: The results for all type 2 queries are collected and printed at the end.

This approach efficiently handles the dynamic graph operations and queries by leveraging union-find with optimizations and maintaining auxiliary data for quick access to the required information.
