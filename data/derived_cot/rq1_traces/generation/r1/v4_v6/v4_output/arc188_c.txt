You will be given a competitive programming problem.
There is a village with N villagers numbered from 1 to N.
Each villager is honest or a liar. Additionally, some villagers are confused.
You have obtained M testimonies from the villagers. Each testimony is given by A_i, B_i, C_i for i=1,2,\ldots,M, representing:

- If C_i=0, villager A_i testified that villager B_i is honest.
- If C_i=1, villager A_i testified that villager B_i is a liar.

All villagers know whether every other villager is honest or a liar, and you know that they made their testimonies to you according to the following rules:

- An honest villager who is not confused always tells the truth.
- A liar who is not confused always tells lies.
- A confused honest villager always tells lies.
- A confused liar always tells the truth.

In other words, if they are not confused, honest villagers always tell the truth, and liars always tell lies, but if they are confused, it is reversed.
You have decided to guess the set of villagers who are confused.
Given a choice of villagers who are confused, whether the set of testimonies "contradicts" or not is determined.
Here, a set of testimonies is said to contradict if, no matter how you assign honest or liar statuses to the villagers, there is at least one testimony that violates the villagers' testimony rules.
Find a set of confused villagers such that the given set of testimonies does not contradict.
If no such set of confused villagers exists, indicate that fact.

Input

The input is given from Standard Input in the following format:
N M
A_1 B_1 C_1
A_2 B_2 C_2
\vdots
A_M B_M C_M

Output

If there exists a set of confused villagers such that the given set of testimonies does not contradict, print a string of length N representing the set of confused villagers. In this string, the i-th character should be 1 if villager i is confused, and 0 otherwise.
If no such set of confused villagers exists, print -1.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 0 \leq M \leq \mathrm{min} \lbrace 2 \times 10^5,N(N-1) \rbrace 
- 1 \leq A_i, B_i \leq N, A_i \neq B_i
- A_i \neq A_j or B_i \neq B_j for i \neq j.
- C_i = 0 or 1.
- All input values are integers.

   \]
   This equation must hold for all testimonies. Additionally, for each villager A, all expressions for \(d_A\) derived from their testimonies must be consistent.

3. **Graph Construction**: We build a graph where nodes are testifiers (villagers who give at least one testimony). For each villager B, if multiple villagers testify about B, we add edges between these testifiers with weights derived from their testimonies. The weights ensure that the constraints on \(x_B\) (the honesty status of B) are consistent.

4. **Bipartite Check**: Using BFS or DFS, we check if the constructed graph is bipartite (i.e., no odd-length cycles). If the graph is not bipartite, it means the constraints are contradictory, and we output -1.

5. **Assigning Values**: If the graph is bipartite, we assign values to \(x_B\) (honesty status of B) based on the testimonies. For each testifier A, we determine their confusion status \(d_A\) using the equation \(d_A = x_A \oplus v_A\), where \(v_A\) is the value assigned during the bipartite check. If a testifier is not testified about, we can arbitrarily set \(d_A = 0\).

6. **Output**: Finally, we output the confusion statuses of all villagers. Non-testifiers are set to 0 by default.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    m = int(data[1])
    is_testifier = [False] * (n + 1)
    lists = [[] for _ in range(n + 1)]
    idx = 2
    for _ in range(m):
        a = int(data[idx])
        b = int(data[idx + 1])
        c = int(data[idx + 2])
        idx += 3
        is_testifier[a] = True
        lists[b].append((a, c))
    
    graph = [[] for _ in range(n + 1)]
    for b in range(1, n + 1):
        if not lists[b]:
            continue
        ref = lists[b][0]
        for j in range(1, len(lists[b])):
            a, c_val = lists[b][j]
            w = ref[1] ^ c_val
            graph[ref[0]].append((a, w))
            graph[a].append((ref[0], w))
    
    v_arr = [-1] * (n + 1)
    for i in range(1, n + 1):
        if is_testifier[i] and v_arr[i] == -1:
            q = deque([i])
            v_arr[i] = 0
            while q:
                u = q.popleft()
                for neighbor, w in graph[u]:
                    if v_arr[neighbor] == -1:
                        v_arr[neighbor] = v_arr[u] ^ w
                        q.append(neighbor)
                    else:
                        if v_arr[neighbor] != (v_arr[u] ^ w):
                            print("-1")
                            return
    x = [-1] * (n + 1)
    for b in range(1, n + 1):
        if lists[b]:
            a0, c0 = lists[b][0]
            x[b] = v_arr[a0] ^ c0
    d = [0] * (n + 1)
    for i in range(1, n + 1):
        if is_testifier[i] and x[i] != -1:
            d[i] = x[i] ^ v_arr[i]
    res = ''.join(str(d[i]) for i in range(1, n + 1))
    print(res)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of villagers (N) and testimonies (M). Each testimony is stored with the testifier (A), the subject (B), and the claim (C).
2. **Graph Construction**: For each subject B, if multiple villagers testify about B, edges are added between these testifiers in a graph. The edge weights are derived from the XOR of their claims.
3. **Bipartite Check**: Using BFS, we traverse the graph to assign values (0 or 1) to each testifier. If a contradiction is found (i.e., inconsistent assignments), the solution outputs -1.
4. **Honesty Status Assignment**: For each villager B, if there are testimonies about them, their honesty status (x_B) is determined using the assigned values from the BFS.
5. **Confusion Status Assignment**: For each testifier, their confusion status (d_A) is computed based on their honesty status and the BFS-assigned values. Non-testifiers are set to 0 by default.
6. **Output**: The confusion statuses of all villagers are concatenated into a string and printed.

This approach efficiently checks for consistency in testimonies using graph theory and ensures a valid assignment of confusion statuses if possible.
