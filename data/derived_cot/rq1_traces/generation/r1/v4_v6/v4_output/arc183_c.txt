To solve this problem, we need to count the number of permutations of integers from 1 to N that satisfy a set of conditions. Each condition specifies that for a given segment [L_i, R_i], the maximum value in that segment should not be located at position X_i. The solution involves dynamic programming and combinatorial methods to efficiently handle the constraints.

### Approach
1. **Problem Analysis**: The problem requires counting permutations of integers from 1 to N that meet M conditions. Each condition (L_i, R_i, X_i) dictates that the maximum element in the segment [L_i, R_i] must not be the element at position X_i. Given the constraints (N ≤ 500 and M ≤ 10^5), a brute-force approach is infeasible.
2. **Key Insight**: For a permutation to satisfy a condition, there must exist at least one element in the segment [L_i, R_i] that is greater than the element at X_i. Since permutations consist of distinct integers, the largest element in any segment will be unique.
3. **Dynamic Programming with Bitmask**: For smaller values of N (N ≤ 15), we use a bitmask DP approach. The DP state represents a bitmask where each bit indicates whether a position has been assigned a number. For each state, we check if adding a new position satisfies all conditions for that position.
4. **Heap Method for Larger N**: For larger N, we use a greedy approach with a heap. We maintain a count of conditions each position needs to satisfy. Positions with no conditions are initially available. As positions are assigned, we update the counts for other positions, adding them to the available set if their conditions are met.
