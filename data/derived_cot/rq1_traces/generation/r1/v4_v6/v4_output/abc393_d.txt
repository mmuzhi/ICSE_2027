To solve this problem, we need to find the minimum number of adjacent swaps required to make all the '1's in a given string contiguous. The key insight is recognizing that the problem can be reduced to calculating the sum of absolute differences between the original positions of the '1's and their target positions in a contiguous block, where the target block is chosen optimally to minimize this sum.

### Approach
1. **Problem Analysis**: The problem involves transforming a binary string such that all '1's form a contiguous segment. The allowed operation is swapping adjacent characters. The challenge is to minimize the number of such swaps.
2. **Key Insight**: The minimum number of adjacent swaps required is equivalent to the sum of the distances each '1' needs to move to reach its target position in a contiguous block. The optimal target block is determined by the median of the adjusted positions of the '1's (adjusted by their index in the sequence of '1's).
3. **Algorithm Selection**: 
   - **Collect Positions**: First, collect the positions of all '1's in the string.
   - **Adjust Positions**: For each '1', adjust its position by subtracting its index in the sequence of '1's. This adjustment helps in aligning the '1's to their target positions in the contiguous block.
   - **Sum Absolute Differences**: The sum of absolute differences between the adjusted positions and their median gives the minimum number of swaps. This sum can be efficiently computed as the difference between the sum of the right half of the adjusted positions and the sum of the left half.

### Solution Code
```python
n = int(input().strip())
s = input().strip()
p = [i for i, ch in enumerate(s) if ch == '1']
k = len(p)
if k == 0:
    print(0)
else:
    left_sum = sum(p[i] - i for i in range(k // 2))
    right_sum = sum(p[i] - i for i in range(k - k // 2, k))
    print(right_sum - left_sum)
```

### Explanation
1. **Reading Input**: The input consists of the length of the string `n` and the string `s` itself.
2. **Collecting Positions**: The positions of all '1's in the string are stored in a list `p`.
3. **Handling Edge Case**: If there are no '1's (though the problem guarantees at least one), print `0`.
4. **Calculating Left and Right Sums**: 
   - **Left Sum**: For the first half of the '1's, compute the sum of each '1's original position minus its index in the sequence of '1's.
   - **Right Sum**: For the last half of the '1's, similarly compute the sum of each '1's original position minus its index.
5. **Result Calculation**: The difference between the right sum and the left sum gives the minimum number of adjacent swaps required to make all '1's contiguous. This approach efficiently minimizes the total movement by leveraging the properties of medians and absolute differences.
