=== ANSWER ===
To solve this problem, we need to compute the sum of the maximum and minimum elements of all subarrays with at most `k` elements in the given array `nums`. Given the constraints, a brute-force approach is infeasible, so we use an efficient approach leveraging monotonic stacks to compute the contribution of each element as the minimum or maximum in relevant subarrays.

### Approach
1. **Problem Analysis**: The problem requires summing the minimum and maximum elements of all subarrays of length at most `k`. A direct approach would involve enumerating all subarrays, which is O(n^2) and too slow for large `n` (up to 80,000). Instead, we use the concept of "contribution technique" where each element's contribution as a minimum or maximum in valid subarrays is calculated.

2. **Monotonic Stacks**: We use monotonic stacks to efficiently compute the boundaries for each element where it is the minimum or maximum. Specifically:
   - **Minimum Boundaries**: For each element, find the nearest previous element strictly less than it and the next element less than or equal to it.
   - **Maximum Boundaries**: For each element, find the nearest previous element strictly greater than it and the next element greater than or equal to it.

3. **Counting Valid Subarrays**: For each element, the number of subarrays where it is the minimum or maximum and has length ≤ `k` is computed using these boundaries. The calculation involves:
   - Determining the valid start and end indices for subarrays containing the element.
   - Using arithmetic series properties to efficiently count these subarrays in O(1) per element.

4. **Summing Contributions**: Multiply each element by the count of subarrays where it is the minimum or maximum, and sum these products to get the final result.

### Solution Code
```python
class Solution:
    def minMaxSubarraysSum(self, nums, k):
        n = len(nums)
        if n == 0:
            return 0
        
        left_min_b = [-1] * n
        right_min_b = [n] * n
        left_max_b = [-1] * n
        right_max_b = [n] * n
        
        stack = []
        for i in range(n):
            while stack and nums[stack[-1]] >= nums[i]:
                stack.pop()
            if stack:
                left_min_b[i] = stack[-1]
            else:
                left_min_b[i] = -1
            stack.append(i)
        
        stack = []
        for i in range(n-1, -1, -1):
            while stack and nums[stack[-1]] > nums[i]:
                stack.pop()
            if stack:
                right_min_b[i] = stack[-1]
            else:
                right_min_b[i] = n
            stack.append(i)
        
        stack = []
        for i in range(n):
            while stack and nums[stack[-1]] <= nums[i]:
                stack.pop()
            if stack:
                left_max_b[i] = stack[-1]
            else:
                left_max_b[i] = -1
            stack.append(i)
        
        stack = []
        for i in range(n-1, -1, -1):
            while stack and nums[stack[-1]] < nums[i]:
                stack.pop()
            if stack:
                right_max_b[i] = stack[-1]
            else:
                right_max_b[i] = n
            stack.append(i)
        
        def count_subarrays(L, R, i, k):
            s_min_val = max(L, i - k + 1)
            if s_min_val > i:
                return 0
            n_total = i - s_min_val + 1
            part1_end = min(i, R - k + 1)
            if part1_end < s_min_val:
                n1 = 0
            else:
                n1 = part1_end - s_min_val + 1
            if n1 == 0:
                sum1 = 0
            else:
                sum_s = (s_min_val + part1_end) * n1 // 2
                sum1 = sum_s + (k - 1) * n1
            part2_start = max(s_min_val, R - k + 2)
            if part2_start > i:
                n2 = 0
            else:
                n2 = i - part2_start + 1
            sum2 = n2 * R
            total_sum = sum1 + sum2
            return total_sum - (i - 1) * n_total
        
        total_sum = 0
        for i in range(n):
            L_min = left_min_b[i] + 1
            R_min = right_min_b[i] - 1
            count_min = count_subarrays(L_min, R_min, i, k)
            total_sum += nums[i] * count_min
            
            L_max = left_max_b[i] + 1
            R_max = right_max_b[i] - 1
            count_max = count_subarrays(L_max, R_max, i, k)
            total_sum += nums[i] * count_max
        
        return total_sum
```

### Explanation
1. **Boundary Calculation**: Using monotonic stacks, we compute for each element the nearest previous and next elements that are strictly less (for minimum) or strictly greater (for maximum). This helps in determining the range of subarrays where each element is the minimum or maximum.
2. **Counting Valid Subarrays**: For each element, we count the number of subarrays of length ≤ `k` where it is the minimum or maximum. This involves:
   - **Start and End Indices**: Valid subarrays start within `[L, i]` and end within `[i, R]`, where `L` and `R` are derived from the boundary indices.
   - **Arithmetic Series**: The count is efficiently computed by splitting the range into parts where the subarray length constraint simplifies the calculation.
3. **Summing Contributions**: Each element's value is multiplied by the count of subarrays where it is the minimum or maximum, and these products are summed to yield the final result.

This approach efficiently handles the constraints by leveraging monotonic stacks and arithmetic properties, ensuring optimal performance for large input sizes.
