=== ANSWER ===
To solve this problem, we need to determine if there exists any permutation of three given integers such that the product of the first two integers equals the third integer. The solution involves checking all possible permutations of the three integers to see if any permutation satisfies the condition \(B_1 \times B_2 = B_3\).

### Approach
1. **Problem Analysis**: Given three integers, we need to check if any permutation of these integers can satisfy the condition where the product of the first two integers equals the third integer. Since there are only three integers, there are six possible permutations. However, we can efficiently check only three scenarios where each integer is considered as the third integer (the product result), and the remaining two integers are multiplied to see if they equal the third integer.
2. **Key Insight**: For the three integers \(A_1, A_2, A_3\), the condition \(B_1 \times B_2 = B_3\) can be satisfied if any of the following are true:
   - \(A_1 \times A_2 = A_3\)
   - \(A_1 \times A_3 = A_2\)
   - \(A_2 \times A_3 = A_1\)
3. **Algorithm Selection**: Instead of generating all permutations, we directly check the three possible conditions mentioned above. This approach is efficient with a constant time complexity \(O(1)\) since the number of checks is fixed regardless of the input size.

### Solution Code
```python
def main():
    a, b, c = map(int, input().split())
    if a * b == c or a * c == b or b * c == a:
        print("Yes")
    else:
        print("No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input consists of three integers read from standard input.
2. **Checking Conditions**: The code checks three conditions:
   - Whether the product of the first and second integers equals the third integer.
   - Whether the product of the first and third integers equals the second integer.
   - Whether the product of the second and third integers equals the first integer.
3. **Output Result**: If any of the conditions is satisfied, the code prints "Yes"; otherwise, it prints "No".

This approach efficiently checks all possible scenarios without generating all permutations, ensuring optimal performance for the given problem constraints.
