You are given an N \times N grid S and an M \times M grid T. The cell at the i-th row from the top and the j-th column from the left is denoted by (i,j).

Print a and b in this order, separated by a space on one line.

=== ANSWER ===
To solve this problem, we need to find a smaller grid T within a larger grid S. The solution involves checking all possible positions in S where T could start and verifying if the subgrid of S at that position matches T exactly.

### Approach
1. **Problem Analysis**: The task is to locate a smaller M x M grid T within a larger N x N grid S. The grids are composed of characters '.' (white) and '#' (black). The solution must find the top-left position (a, b) in S such that the M x M subgrid starting at (a, b) matches T exactly. The indices a and b are 1-based.
2. **Intuition**: The solution involves iterating over all possible starting positions (a, b) in S where the subgrid of size M x M can fit. For each position, we compare each corresponding cell in the subgrid of S with the cells in T.
3. **Algorithm Selection**: We use nested loops to check each possible starting position (a, b) in S. For each position, we then loop through each cell in T and verify if the corresponding cell in S matches. If all cells match, we record the position (a, b) and exit the loops early since the problem guarantees exactly one valid solution.
4. **Complexity Analysis**: The worst-case time complexity is O((N-M+1)^2 * M^2). Given the constraints (N and M up to 50), this results in a manageable number of operations (around 6.25 million in the worst case), which is feasible for modern computing.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    n_m_line = data[0].split()
    N = int(n_m_line[0])
    M = int(n_m_line[1])
    
    grid_S = []
    for i in range(1, 1+N):
        grid_S.append(data[i].strip())
    
    grid_T = []
    for i in range(1+N, 1+N+M):
        grid_T.append(data[i].strip())
    
    for a0 in range(N - M + 1):
        for b0 in range(N - M + 1):
            match = True
            for i in range(M):
                for j in range(M):
                    if grid_S[a0 + i][b0 + j] != grid_T[i][j]:
                        match = False
                        break
                if not match:
                    break
            if match:
                print(f"{a0+1} {b0+1}")
                return

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input. The first line contains integers N (size of grid S) and M (size of grid T). The next N lines represent grid S, followed by M lines representing grid T.
2. **Grid Storage**: Grid S is stored as a list of strings, where each string is a row of the grid. Similarly, grid T is stored as a list of strings.
3. **Checking Positions**: The algorithm checks all possible starting positions (a0, b0) in S (0-based indices) where the M x M subgrid can fit. For each position, it verifies if every cell in the subgrid starting at (a0, b0) matches the corresponding cell in T.
4. **Output**: Once a matching position is found, the 1-based indices (a0+1, b0+1) are printed, and the program exits since there is exactly one valid solution.

This approach efficiently checks all possible positions in S to find the exact match for T, leveraging nested loops for thorough comparison while ensuring optimal performance within given constraints.
