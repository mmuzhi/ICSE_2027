Train i departs from city A_i at time S_i and arrives at city B_i at time T_i.
Given a positive integer X_1, find a way to set non-negative integers X_2,\ldots,X_M that satisfies the following condition with the minimum possible value of X_2+\ldots+X_M.

- 2 \leq N \leq 2\times 10^5
- 2 \leq M \leq 2\times 10^5
- 1 \leq A_i,B_i \leq N
- A_i \neq B_i
- 0 \leq S_i < T_i \leq 10^9
- 1 \leq X_1 \leq 10^9
- All input values are integers.

To solve this problem, we need to determine the optimal delays \(X_2, X_3, \ldots, X_M\) for each train such that for any pair of trains \((i, j)\) where the arrival city of train \(i\) is the departure city of train \(j\) and the arrival time of \(i\) is less than or equal to the departure time of \(j\) (i.e., \(B_i = A_j\) and \(T_i \leq S_j\)), the condition \(T_i + X_i \leq S_j + X_j\) holds. The goal is to minimize the sum of \(X_2 + \ldots + X_M\) while ensuring \(X_1\) is given as input.

### Approach
1. **Problem Analysis**: The problem involves scheduling train delays to maintain transfer feasibility between connecting trains. Each train \(i\) has a departure city \(A_i\), arrival city \(B_i\), departure time \(S_i\), and arrival time \(T_i\). The delay \(X_i\) for train \(i\) adjusts its departure and arrival times to \(S_i + X_i\) and \(T_i + X_i\), respectively. The key constraint is that for any transfer from train \(i\) to train \(j\) (where \(B_i = A_j\) and \(T_i \leq S_j\)), the adjusted arrival time of \(i\) must not exceed the adjusted departure time of \(j\).

2. **Graph Representation**: Treat each train as a node in a directed graph. An edge from train \(i\) to train \(j\) exists if \(B_i = A_j\) and \(T_i \leq S_j\). The graph is acyclic because any cycle would imply a contradiction in time constraints (i.e., \(T_i \leq S_j \leq T_j \leq \ldots \leq S_i < T_i\)).

3. **Topological Processing**: Process trains in increasing order of their departure times \(S_i\). For each train \(j\), compute \(X_j\) as the maximum of 0 and \((X_i + T_i - S_j)\) for all trains \(i\) arriving at \(A_j\) with \(T_i \leq S_j\). This ensures \(X_j\) is the minimal delay satisfying all constraints from incoming trains.

4. **Efficient Query Handling**: For each city, maintain a segment tree to track the maximum \((X_i + T_i)\) value for arrivals up to any given time. This allows efficient range maximum queries and updates, crucial for handling large input sizes.

import sys
import bisect
from collections import defaultdict

sys.setrecursionlimit(300000)

class SegTree:
    def __init__(self, n):
        self.n = n
        self.size = 1
        while self.size < n:
            self.size *= 2
        self.data = [-10**18] * (2 * self.size)
    
    def update(self, index, value):
        pos = self.size + index
        if value > self.data[pos]:
            self.data[pos] = value
            pos //= 2
            while pos:
                self.data[pos] = max(self.data[2*pos], self.data[2*pos+1])
                pos //= 2
                
    def query(self, l, r):
        if l > r:
            return -10**18
        res = -10**18
        l_pos = self.size + l
        r_pos = self.size + r
        while l_pos <= r_pos:
            if l_pos % 2 == 1:
                res = max(res, self.data[l_pos])
                l_pos += 1
            if r_pos % 2 == 0:
                res = max(res, self.data[r_pos])
                r_pos -= 1
            l_pos //= 2
            r_pos //= 2
        return res

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    N = int(next(it)); M = int(next(it)); X1 = int(next(it))
    trains = []
    for i in range(M):
        A = int(next(it)); B = int(next(it)); S = int(next(it)); T = int(next(it))
        trains.append((A, B, S, T, i))
    
    city_arrival_times = defaultdict(list)
    for A, B, S, T, idx in trains:
        city_arrival_times[B].append(T)
    
    city_segtrees = {}
    for c, times_list in city_arrival_times.items():
        times_list.sort()
        n = len(times_list)
        segtree = SegTree(n)
        city_segtrees[c] = (times_list, segtree)
    
    trains_sorted = sorted(trains, key=lambda x: (x[2], x[4]))
    
    ans = [0] * M
    
    for A, B, S, T, idx in trains_sorted:
        if idx == 0:
            x_val = X1
        else:
            if A in city_segtrees:
                times, segtree = city_segtrees[A]
                pos = bisect.bisect_right(times, S) - 1
                if pos < 0:
                    max_val = -10**18
                else:
                    max_val = segtree.query(0, pos)
                x_val = max(0, max_val - S)
            else:
                x_val = 0
        ans[idx] = x_val
        
        times, segtree = city_segtrees[B]
        pos = bisect.bisect_left(times, T)
        segtree.update(pos, x_val + T)
    
    output = []
    for i in range(1, M):
        output.append(str(ans[i]))
    print(" ".join(output))

if __name__ == "__main__":
    main()

### Explanation
1. **Input Parsing**: The input is read and parsed into the number of cities \(N\), trains \(M\), and the initial delay \(X_1\). Each train's details (departure city, arrival city, departure time, arrival time) are stored.
2. **Preprocessing**: For each city, arrival times of trains are collected and sorted. A segment tree is built for each city to facilitate efficient range maximum queries and updates.
3. **Processing Trains**: Trains are processed in ascending order of their departure times. For each train \(j\), if it's the first train, \(X_j\) is set to \(X_1\). Otherwise, \(X_j\) is computed based on the maximum \((X_i + T_i)\) from incoming trains to the departure city \(A_j\) that arrive before \(S_j\).
4. **Updating Segment Trees**: After processing each train, the segment tree of its arrival city \(B_j\) is updated with the value \(X_j + T_j\).
5. **Output**: The delays \(X_2\) to \(X_M\) are printed in the order corresponding to the original input trains.

This approach efficiently handles the constraints using segment trees for range queries and updates, ensuring optimal delays while maintaining transfer feasibility between trains. The complexity is \(O(M \log M)\) due to sorting and segment tree operations.
