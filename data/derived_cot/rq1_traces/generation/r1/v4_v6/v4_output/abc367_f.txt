=== PROMPT ===
You will be given a competitive programming problem.
You are given sequences of positive integers of length N: A=(A_1,A_2,\ldots,A_N) and B=(B_1,B_2,\ldots,B_N).
You are given Q queries to process in order. The i-th query is explained below.

- You are given positive integers l_i,r_i,L_i,R_i. Print Yes if it is possible to rearrange the subsequence (A_{l_i},A_{l_i+1},\ldots,A_{r_i}) to match the subsequence (B_{L_i},B_{L_i+1},\ldots,B_{R_i}), and No otherwise.

Input

The input is given from Standard Input in the following format:
N Q
A_1 A_2 \ldots A_N
B_1 B_2 \ldots B_N
l_1 r_1 L_1 R_1
l_2 r_2 L_2 R_2
\vdots
l_Q r_Q L_Q R_Q

Output

Print Q lines. The i-th line should contain the answer to the i-th query.

Constraints


-  1\leq N,Q\leq 2\times 10^5
-  1\leq A_i,B_i\leq N
-  1\leq l_i \leq r_i\leq N
-  1\leq L_i \leq R_i\leq N
- All input values are integers.

1 2 3 2 4
2 3 1 4 2
1 3 1 3
1 2 3 5
1 4 2 5
1 5 1 5

To solve this problem, we need to efficiently determine if two subsequences from arrays `A` and `B` can be rearranged to match each other. The key insight is that two sequences can be rearranged to match if and only if they are multisets of the same elements. Given the constraints, a direct comparison of the multisets for each query would be too slow. Instead, we use a randomized hashing technique to represent each multiset as a hash value, allowing us to compare the multisets in constant time per query.

### Approach
1. **Randomized Hashing**: Assign a unique random 64-bit integer to each distinct integer in the range 1 to N. This random value is used to represent each element in the arrays `A` and `B`.
2. **Prefix Arrays**: Construct prefix sum arrays for both `A` and `B` using two different hash functions. Each prefix array stores the cumulative sum of the hash values up to each index, modulo two large primes (or one prime and one power of two). This allows efficient range sum queries.
3. **Query Processing**: For each query, check if the lengths of the two subsequences are equal. If not, the answer is immediately "No". Otherwise, compute the hash values for both subsequences using the prefix arrays. If the hash values from both hash functions match, the multisets are the same, and the answer is "Yes"; otherwise, it's "No".

import random

def main():
    import sys
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    N = int(next(it)); Q = int(next(it))
    A = [int(next(it)) for _ in range(N)]
    B = [int(next(it)) for _ in range(N)]
    
    mod1 = (1 << 64)
    mod2 = (1 << 64) - 59

        PA1[i] = (PA1[i - 1] + rnd1[x]) % mod1
