You will be given a competitive programming problem.
The definition of an 11/22 string in this problem is the same as in Problems A and C.

A string T is called an 11/22 string when it satisfies all of the following conditions:

- |T| is odd. Here, |T| denotes the length of T.
- The 1-st through (\frac{|T|+1}{2} - 1)-th characters are all 1.
- The (\frac{|T|+1}{2})-th character is /.
- The (\frac{|T|+1}{2} + 1)-th through |T|-th characters are all 2.

For example, 11/22, 111/222, and / are 11/22 strings, but 1122, 1/22, 11/2222, 22/11, and //2/2/211 are not.
Given a string S of length N consisting of 1, 2, and /, process Q queries.
Each query provides two integers L and R. Let T be the (contiguous) substring of S from the L-th through R-th character. Find the maximum length of a subsequence (not necessarily contiguous) of T that is an 11/22 string. If no such subsequence exists, print 0.

Input

The input is given from Standard Input in the following format. Here, \mathrm{query}_i denotes the i-th query.
N Q
S
\mathrm{query}_1
\mathrm{query}_2
\vdots
\mathrm{query}_Q

Each query is given in the following format:
L R

Output

Print Q lines. The i-th line should contain the answer to the i-th query.

Constraints


- 1 \leq N \leq 10^5
- 1 \leq Q \leq 10^5
- S is a string of length N consisting of 1, 2, and /.
- 1 \leq L \leq R \leq N
- N, Q, L, and R are integers.

111/212/1122
1 7
9 12
3 6

1 12

An 11/22 string is defined as having an odd length, with the first half (excluding the middle) consisting of '1's, the middle character being '/', and the second half consisting of '2's.

### Approach
1. **Problem Analysis**: The key observation is that a valid 11/22 string must have the form: `(some number of '1's) + '/' + (same number of '2's)`. The total length is thus `2k + 1` where `k` is the number of '1's (and '2's). The challenge is to find, for each query substring, the maximum `k` such that there are at least `k` '1's before a '/' and at least `k` '2's after the same '/'.

2. **Intuition**: For each possible '/' in the substring, we can consider it as the middle character. Then, the maximum `k` for that '/' is the minimum of the number of '1's available before it and the number of '2's available after it within the substring. The answer for the query is the maximum over all such possible `2k+1` (for each '/') and also considering that we can choose a subsequence (so the characters don't have to be contiguous).

3. **Optimization**: Given the constraints (N, Q up to 10^5), a brute-force approach for each query is infeasible. Instead, we precompute prefix sums for '1's and suffix sums for '2's.
