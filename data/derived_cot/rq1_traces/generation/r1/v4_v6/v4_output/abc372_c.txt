=== PROMPT ===
You will be given a competitive programming problem.
You are given a string S of length N. You are also given Q queries, which you should process in order.
The i-th query is as follows:

- Given an integer X_i and a character C_i, replace the X_i-th character of S with C_i. Then, print the number of times the string ABC appears as a substring in S.

Here, a substring of S is a string obtained by deleting zero or more characters from the beginning and zero or more characters from the end of S.
For example, ab is a substring of abc, but ac is not a substring of abc.

Input

The input is given from Standard Input in the following format:
N Q
S
X_1 C_1
X_2 C_2
\vdots
X_Q C_Q

Output

Print Q lines.
The i-th line (1 \le i \le Q) should contain the answer to the i-th query.

Constraints


- 3 \le N \le 2 \times 10^5
- 1 \le Q \le 2 \times 10^5
- S is a string of length N consisting of uppercase English letters.
- 1 \le X_i \le N
- C_i is an uppercase English letter.

Sample Input 1

7 4
ABCDABC
4 B
3 A
5 C
4 G

Sample Output 1

2
1
1
0

After processing each query, S becomes as follows.

- After the first query: S= ABCBABC. In this string, ABC appears twice as a substring.
- After the second query: S= ABABABC. In this string, ABC appears once as a substring.
- After the third query: S= ABABCBC. In this string, ABC appears once as a substring.
- After the fourth query: S= ABAGCBC. In this string, ABC appears zero times as a substring.

Sample Input 2

3 3
ABC
1 A

3 C

Sample Output 2

1

There are cases where S does not change through processing a query.

=== ANSWER ===
To solve this problem, we need to efficiently handle multiple queries on a string, where each query updates a character at a specific position and then requires counting the number of times the substring "ABC" appears in the modified string. Given the constraints, a brute-force approach for each query would be too slow, so we need an optimized solution.

### Approach
1. **Initial Setup**: Read the input values, including the string length `N`, the number of queries `Q`, the initial string `S`, and the list of queries.
2. **Precompute Initial Count**: Before processing any queries, compute the initial count of "ABC" substrings in the string by scanning all possible starting positions (from 0 to `N-3`).
3. **Process Queries**: For each query:
   - **Check for Redundant Update**: If the new character is the same as the current character at the specified position, skip further processing for this query and output the current count.
   - **Update Affected Substrings**: For the position being updated, identify the three substrings that could be affected (those starting at `index-2`, `index-1`, and `index`). For each valid substring (within bounds), if it was "ABC" before the update, decrement the total count.
   - **Update Character**: Replace the character at the specified position with the new character.
   - **Recheck Affected Substrings**: For the same three substrings, if they now form "ABC" after the update, increment the total count.
4. **Output Results**: After processing each query, output the current count of "ABC" substrings.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    n, q = map(int, data[0].split())
    s = list(data[1].strip())
    queries = []
    for i in range(2, 2 + q):
        parts = data[i].split()
        x = int(parts[0])
        c = parts[1].strip()
        queries.append((x, c))
    
    total = 0
    for j in range(n - 2):
        if s[j] == 'A' and s[j+1] == 'B' and s[j+2] == 'C':
            total += 1
    
    out_lines = []
    for x, c in queries:
        index = x - 1
        old_char = s[index]
        if old_char == c:
            out_lines.append(str(total))
        else:
            for j in [index - 2, index - 1, index]:
                if j >= 0 and j <= n - 3:
                    if s[j] == 'A' and s[j+1] == 'B' and s[j+2] == 'C':
                        total -= 1
            s[index] = c
            for j in [index - 2, index - 1, index]:
                if j >= 0 and j <= n - 3:
                    if s[j] == 'A' and s[j+1] == 'B' and s[j+2] == 'C':
                        total += 1
            out_lines.append(str(total))
    
    print("\n".join(out_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first line contains `N` (string length) and `Q` (number of queries). The second line is the string `S`, and subsequent lines are the queries.
2. **Initial Count Calculation**: The initial count of "ABC" substrings is determined by iterating through all possible starting positions (0 to `N-3`) and checking for the sequence "A", "B", "C".
3. **Processing Queries**: For each query:
   - **Check for Redundant Update**: If the new character is the same as the existing one, the count remains unchanged.
   - **Update Substrings**: The update affects three potential substrings (starting at `index-2`, `index-1`, and `index`). Each valid substring is checked; if it was "ABC" before the update, the count is decremented.
   - **Apply Update**: The character at the specified position is updated to the new character.
   - **Recheck Substrings**: The same three substrings are rechecked; if they now form "ABC", the count is incremented.
4. **Output**: The results for each query are collected in a list and printed at the end to minimize I/O operations, which is crucial for handling large input sizes efficiently.

This approach ensures that each query is processed in constant time, making the solution efficient and scalable for large inputs.
