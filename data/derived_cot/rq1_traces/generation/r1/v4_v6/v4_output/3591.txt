To solve this problem, we need to find the minimum total cost required to transform string `s` into string `t` by performing operations on each character of `s`. Each operation involves either shifting a character to the next or previous letter in the alphabet (with wrap-around from 'z' to 'a' and vice versa). The cost of each shift depends on the starting letter of the shift, as specified by the `nextCost` and `previousCost` arrays.

### Approach
1. **Problem Analysis**: The problem requires transforming each character in `s` to the corresponding character in `t` with minimal cost. Each character can be shifted either forward or backward in the alphabet, and each shift operation's cost is determined by the starting letter of the shift. The challenge is to find the minimal cost path for each character transformation in a circular alphabet (26 letters).

2. **Graph Representation**: Represent the alphabet as a directed graph where each node is a letter (0 to 25). Each node has outgoing edges to its next and previous nodes (with wrap-around). The edge weights are the costs from `nextCost` and `previousCost` arrays.

3. **Shortest Path Calculation**: Use the Floyd-Warshall algorithm to compute the shortest path between all pairs of nodes (letters). This algorithm efficiently finds the minimal path cost between every pair of nodes in a graph, which is suitable here due to the small graph size (26 nodes).

4. **Cost Summation**: For each character in `s` and `t`, convert them to their respective indices (0 to 25), retrieve the precomputed minimal transformation cost from the Floyd-Warshall result, and sum these costs to get the total shift distance.

### Solution Code
```python
class Solution:
    def shiftDistance(self, s: str, t: str, nextCost: List[int], previousCost: List[int]) -> int:
        n = 26
        INF = 10**18
        dist = [[INF] * n for _ in range(n)]
        
        for i in range(n):
            dist[i][i] = 0
            nxt_node = (i + 1) % n
            prv_node = (i - 1) % n
            if nextCost[i] < dist[i][nxt_node]:
                dist[i][nxt_node] = nextCost[i]
            if previousCost[i] < dist[i][prv_node]:
                dist[i][prv_node] = previousCost[i]
        
        for k in range(n):
            for i in range(n):
                if dist[i][k] == INF:
                    continue
                for j in range(n):
                    new_dist = dist[i][k] + dist[k][j]
                    if new_dist < dist[i][j]:
                        dist[i][j] = new_dist
        
        total_cost = 0
        for a, b in zip(s, t):
            i = ord(a) - ord('a')
            j = ord(b) - ord('a')
            total_cost += dist[i][j]
        
        return total_cost
```

### Explanation
1. **Initialization**: Initialize a 26x26 distance matrix `dist` with `INF` (a very large number), and set the diagonal entries to 0 (cost to transform a letter to itself is zero).
2. **Graph Construction**: For each letter (node), set the cost to shift to the next letter using `nextCost` and to the previous letter using `previousCost`.
3. **Floyd-Warshall Algorithm**: Compute the minimal path cost between all pairs of nodes by checking all possible intermediate nodes. This updates the distance matrix to reflect the shortest paths.
4. **Cost Calculation**: For each corresponding character in `s` and `t`, convert them to indices, retrieve the minimal transformation cost from the distance matrix, and accumulate these costs to get the total shift distance.

This approach efficiently precomputes the minimal transformation costs between all letter pairs, allowing quick summation of costs for transforming the entire string `s` to `t`. The algorithm's complexity is dominated by the Floyd-Warshall step (O(26^3)), which is feasible due to the constant size of the alphabet. The subsequent summation over the strings is linear (O(n)), where n is the length of the strings.
