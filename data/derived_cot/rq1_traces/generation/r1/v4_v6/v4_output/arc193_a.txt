You will be given a competitive programming problem.
For integers l, r, let [l, r] denote the set of all integers from l through r. That is, [l, r] = \lbrace l, l+1, l+2, \ldots, r-1, r\rbrace.
You are given N pairs of integers (L_1, R_1), (L_2, R_2), \ldots, (L_N, R_N).
Based on these pairs, consider an undirected graph G defined as follows:

- It has N vertices numbered 1, 2, \ldots, N.
- For all i, j \in [1, N], there is an undirected edge between vertices i and j if and only if the intersection of [L_i, R_i] and [L_j, R_j] is empty.

In addition, for each i = 1, 2, \ldots, N, define the weight of vertex i to be W_i.
You are given Q queries about G. Process these queries in the order they are given.
For each i = 1, 2, \ldots, Q, the i-th query is the following:

You are given integers s_i and t_i (both between 1 and N, inclusive) such that s_i \neq t_i. Determine whether there exists a path from vertex s_i to vertex t_i in G. If it exists, print the minimum possible weight of such a path.

Here, the weight of a path from vertex s to vertex t is defined as the sum of the weights of the vertices on that path (including both endpoints s and t).

Input

The input is given from Standard Input in the following format:

L_2 R_2
\vdots
L_N R_N

s_2 t_2
\vdots
s_Q t_Q

Output

Print Q lines.
For each i = 1, 2, \ldots, Q, on the i-th line, if there exists a path from vertex s_i to vertex t_i, print the minimum possible weight of such a path, and print -1 otherwise.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq Q \leq 2 \times 10^5
- 1 \leq W_i \leq 10^9
- 1 \leq L_i \leq R_i \leq 2N
- 1 \leq s_i, t_i \leq N
- s_i \neq t_i
- All input values are integers.

To solve this problem, we need to determine if there exists a path between two vertices in a graph constructed from intervals, and if so, find the minimum weight of such a path. The graph is defined such that an edge exists between two vertices if their intervals do not overlap. The weight of a path is the sum of the weights of the vertices along the path.

### Approach
1. **Graph Construction**: The graph is built implicitly using coordinate nodes to avoid explicitly storing all edges. Each interval is connected to coordinate nodes representing `L_i - 1` and `R_i + 1`. These coordinate nodes help in efficiently determining disjoint intervals.
2. **Dijkstra's Algorithm**: For each query, we use Dijkstra's algorithm to find the minimum weight path from the source to the target vertex. The algorithm processes both interval nodes and coordinate nodes:
   - **Interval Nodes**: When processing an interval node, we exit to coordinate nodes `L_i - 1` and `R_i + 1`.
   - **Coordinate Nodes**: When processing a coordinate node, we propagate to adjacent coordinates (left and right) and enter any interval nodes that can be reached from this coordinate (intervals for which the coordinate is `L_i - 1` or `R_i + 1`).
3. **Early Termination**: The algorithm terminates early if the target vertex is reached during processing.
