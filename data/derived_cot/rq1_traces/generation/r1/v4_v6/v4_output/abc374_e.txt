You will be given a competitive programming problem.
The manufacturing of a certain product requires N processes numbered 1,2,\dots,N.
For each process i, there are two types of machines S_i and T_i available for purchase to handle it.

- Machine S_i: Can process A_i products per day per unit, and costs P_i yen per unit.
- Machine T_i: Can process B_i products per day per unit, and costs Q_i yen per unit.

You can purchase any number of each machine, possibly zero.
Suppose that process i can handle W_i products per day as a result of introducing machines.
Here, we define the production capacity as the minimum of W, that is, \displaystyle \min^{N}_{i=1} W_i.
Given a total budget of X yen, find the maximum achievable production capacity.

Input

The input is given from Standard Input in the following format:
N X
A_1 P_1 B_1 Q_1
A_2 P_2 B_2 Q_2
\vdots
A_N P_N B_N Q_N

Output

Print the answer as an integer.

Constraints


- All input values are integers.
- 1 \le N \le 100
- 1 \le A_i,B_i \le 100
- 1 \le P_i,Q_i,X \le 10^7

Sample Input 1

3 22
2 5 3 6
1 1 3 3
1 3 2 4

Sample Output 1

4

To solve this problem, we need to maximize the production capacity of a manufacturing process given a budget constraint. The production capacity is defined as the minimum number of products that can be processed per day across all processes. Each process can utilize two types of machines, each with different production rates and costs. The goal is to determine the optimal number of machines to purchase for each process within the budget to achieve the highest possible production capacity.

### Approach
1. **Binary Search on Production Capacity**: We perform a binary search over possible production capacities (Y). For each candidate Y, we check if it's feasible to achieve a production capacity of at least Y within the given budget.
2. **Cost Calculation for Each Process**: For each process, we compute the minimal cost required to achieve at least Y products per day. This involves:
   - **Iterating Over Possible Machine Combinations**: For each process, we consider all possible residues (k0) of S_i machines modulo B_i. This helps in efficiently exploring the solution space without exhaustive checks.
   - **Calculating Base Cost**: For each residue k0, we calculate the minimal number of T_i machines (t0_min) needed to meet the production requirement Y. The base cost is derived from the costs of these machines.
   - **Adjusting for Cost Efficiency**: If shifting between machine types (S_i and T_i) reduces the total cost, we adjust the cost accordingly. The shift is determined by the cost difference between the two machine types.
3. **Early Termination**: During the cost summation across all processes, if the total cost exceeds the budget at any point, we terminate early to save computation time.
4. **Binary Search Result**: The highest Y for which the total cost is within the budget is our answer.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    N = int(data[0])
    X = int(data[1])
    processes = []
    idx = 2
    for _ in range(N):
        A = int(data[idx])
        P = int(data[idx+1])
        B = int(data[idx+2])
        Q = int(data[idx+3])
        idx += 4
        processes.append((A, P, B, Q))
    
    low, high = 0, 10**18
    
    def total_minimal_cost(Y):
        total = 0
        for A, P, B, Q in processes:
            min_cost_process = float('inf')
            for k0 in range(B):
                rem = Y - k0 * A
                if rem <= 0:
                    t0_min = 0
                    base_cost = k0 * P + t0_min * Q
                else:
                    t0_min = (rem + B - 1) // B
                    base_cost = k0 * P + t0_min * Q
                slope = B * P - A * Q
                m_max = t0_min // A
                if slope >= 0:
                    candidate = base_cost
                else:
                    candidate = base_cost + m_max * slope
                if candidate < min_cost_process:
                    min_cost_process = candidate
            total += min_cost_process
            if total > X:
                break
        return total

    while low <= high:
        mid = (low + high) // 2
        cost_needed = total_minimal_cost(mid)
        if cost_needed <= X:
            low = mid + 1
        else:
            high = mid - 1
            
    print(high)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to a sufficiently large value (10^18) to cover all possible production capacities.
2. **Cost Calculation Function**: For each candidate production capacity (Y), the function `total_minimal_cost` calculates the minimal cost required to achieve at least Y products per day for all processes:
   - **Residue Handling**: For each process, we iterate over possible residues (k0) of S_i machines modulo B_i. This allows us to efficiently explore combinations without exhaustive checks.
   - **Base Cost Calculation**: For each residue k0, we compute the minimal number of T_i machines (t0_min) needed to meet the production requirement Y. The base cost is the sum of the costs of these machines.
   - **Cost Adjustment**: If shifting between machine types reduces costs (when the slope is negative), we adjust the cost by the maximum possible shifts (m_max) without violating constraints.
3. **Early Termination**: During the summation of minimal costs across processes, if the total cost exceeds the budget (X), we terminate early to save computation time.
4. **Binary Search Execution**: The binary search checks each candidate Y. If the total minimal cost is within the budget, it searches higher capacities; otherwise, it searches lower capacities. The highest feasible Y is printed as the result.

This approach efficiently narrows down the optimal production capacity using binary search and calculates minimal costs per process through residue analysis and cost adjustments, ensuring optimal performance even for large input sizes.
