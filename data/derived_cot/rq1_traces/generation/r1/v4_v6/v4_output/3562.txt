Two intervals are said to be non-overlapping if they do not share any points. In particular, intervals sharing a left or right boundary are considered overlapping.
An array a is lexicographically smaller than an array b if in the first position where a and b differ, array a has an element that is less than the corresponding element in b.
If the first min(a.length, b.length) elements do not differ, then the shorter array is the lexicographically smaller one.
 
Example 1:

Input: intervals = [[1,3,2],[4,5,2],[1,5,5],[6,9,3],[6,7,1],[8,9,1]]
Output: [2,3]
Explanation:
You can choose the intervals with indices 2, and 3 with respective weights of 5, and 3.

Explanation:
You can choose the intervals with indices 1, 3, 5, and 6 with respective weights of 7, 6, 3, and 5.

To solve this problem, we need to select at most 4 non-overlapping intervals from a given list such that the sum of their weights is maximized. If multiple solutions exist with the same maximum weight, we return the lexicographically smallest array of indices. 

### Approach
1. **Problem Analysis**: The problem involves selecting intervals that do not overlap (including sharing boundaries) and maximizing their total weight. The solution must handle up to 50,000 intervals efficiently.
2. **Key Insight**: We can use dynamic programming with segment trees to efficiently track the best states (maximum weight and lexicographically smallest indices) for selecting up to 4 intervals. The segment trees help in querying and updating states in logarithmic time.
3. **Algorithm Selection**: 
   - **Sorting**: Intervals are sorted by their end positions to process them in increasing order.
   - **Coordinate Compression**: Compress the interval endpoints to handle large values (up to 10^9) efficiently.
   - **Segment Trees**: Use four segment trees (for k=0 to 3) to store the best states (weight and indices) for intervals ending at specific positions. Each tree corresponds to selecting k intervals.
   - **State Processing**: For each interval, query the appropriate segment tree to find the best state from intervals ending before the current interval's start. Update the next segment tree with the new state formed by including the current interval.
   - **Global Best State**: Track the best state (maximum weight and lexicographically smallest indices) across all processed intervals and layers (k=1 to 4).

```python
INVALID = (-10**18, None)

    def __init__(self, n: int):
        self.n = n
        self.size = 1
        while self.size < n:
            self.size *= 2
        self.data = [INVALID] * (2 * self.size)
    
    def merge(self, a, b):
        if a[0] == INVALID[0] and b[0] == INVALID[0]:
            return INVALID
        if a[0] == INVALID[0]:
            return b
        if b[0] == INVALID[0]:
            return a

            return a
        elif a[0] < b[0]:
            return b
        else:
            if a[1] < b[1]:
                return a

        i //= 2
        while i:
            self.data[i] = self.merge(self.data[2*i], self.data[2*i+1])
            i //= 2

    def query(self, l, r):
        if l > r:
            return INVALID
        l0 = l + self.size
        r0 = r + self.size
        res = INVALID
        while l0 <= r0:
            if l0 % 2 == 1:
                res = self.merge(res, self.data[l0])
                l0 += 1
            if r0 % 2 == 0:
                res = self.merge(res, self.data[r0])
                r0 -= 1
            l0 //= 2
            r0 //= 2
        return res

class Solution:
    def maximumWeight(self, intervals: List[List[int]]) -> List[int]:
        intervals_with_index = []
        for idx, (l, r, w) in enumerate(intervals):
            intervals_with_index.append((l, r, w, idx))
        
        intervals_sorted = sorted(intervals_with_index, key=lambda x: x[1])
        
        endpoints = set()
        endpoints.add(0)
        for l, r, w, idx in intervals_sorted:
            endpoints.add(l)
            endpoints.add(r)
        coords = sorted(endpoints)
        n_coords = len(coords)
        comp = {x: i for i, x in enumerate(coords)}
        
        trees = [SegTree(n_coords) for _ in range(4)]
        trees[0].update(comp[0], (0, ()))
        
        best_ans = (0, ())
        
        for k in range(4):
            for l, r, w, idx_orig in intervals_sorted:
                pos_l = bisect.bisect_left(coords, l)
                if pos_l == 0:
                    state_prev = INVALID
                else:
                    state_prev = trees[k].query(0, pos_l-1)
                
                if state_prev[0] == INVALID[0]:
                    continue
                
                total_weight = state_prev[0] + w
                new_set = tuple(sorted(state_prev[1] + (idx_orig,)))
                
                if total_weight > best_ans[0] or (total_weight == best_ans[0] and new_set < best_ans[1]):
                    best_ans = (total_weight, new_set)
                
                if k < 3:
                    trees[k+1].update(comp[r], (total_weight, new_set))
        
        return list(best_ans[1])
```

### Explanation
1. **Initialization**: The intervals are augmented with their original indices and sorted by their end positions. The endpoints (start and end positions) are compressed to handle large values efficiently.
2. **Segment Trees**: Four segment trees are initialized to store the best states for selecting 0 to 3 intervals. The first tree is initialized with a dummy state (0 weight and empty set) at position 0.
3. **Processing Intervals**: For each interval, the algorithm queries the segment tree corresponding to the current layer (k) to find the best state from intervals ending before the current interval's start. The new state is formed by adding the current interval's weight and index to the queried state.
4. **Updating Best State**: The global best state is updated whenever a new state with higher weight or lexicographically smaller indices is found.
5. **Updating Next Layer**: If not at the last layer (k < 3), the new state is used to update the next segment tree for the next layer (k+1).
6. **Result Extraction**: After processing all intervals and layers, the lexicographically smallest array of indices from the best state is returned. 

This approach efficiently handles the constraints by leveraging segment trees for dynamic state updates and queries, ensuring optimal performance even for large input sizes.
