There are an infinite amount of bags on a number line, one bag for each coordinate. Some of these bags contain coins.
You are given a 2D array coins, where coins[i] = [l_i, r_i, c_i] denotes that every bag from l_i to r_i contains c_i coins.
The segments that coins contain are non-overlapping.
You are also given an integer k.
Return the maximum amount of coins you can obtain by collecting k consecutive bags.
 
Example 1:

Input: coins = [[8,10,1],[1,3,2],[5,6,4]], k = 4
Output: 10
Explanation:
Selecting bags at positions [3, 4, 5, 6] gives the maximum number of coins: 2 + 0 + 4 + 4 = 10.

Example 2:

Input: coins = [[1,10,3]], k = 2
Output: 6
Explanation:
Selecting bags at positions [1, 2] gives the maximum number of coins: 3 + 3 = 6.

 
Constraints:

1 <= coins.length <= 10^5
1 <= k <= 10^9
coins[i] == [l_i, r_i, c_i]
1 <= l_i <= r_i <= 10^9
1 <= c_i <= 1000
The given segments are non-overlapping.

To solve this problem, we need to find the maximum number of coins that can be collected by selecting k consecutive bags on an infinite number line. The coins are distributed in non-overlapping segments, where each segment [l_i, r_i, c_i] indicates that every bag from l_i to r_i contains c_i coins. The solution involves efficiently calculating the coins collected in any window of k consecutive bags and determining the maximum such value.

### Approach
1. **Event Creation**: For each segment [l, r, c], generate four events:
   - **Left Entry Event (l, -c)**: When the left end of the window enters the segment, the derivative decreases by c (since the leftmost bag starts contributing coins).
   - **Left Exit Event (r+1, c)**: When the left end exits the segment, the derivative increases by c (the leftmost bag no longer contributes coins).
   - **Right Entry Event (l - k, c)**: When the right end of the window enters the segment, the derivative increases by c (the new bag at the right end starts contributing coins).
   - **Right Exit Event (r+1 - k, -c)**: When the right end exits the segment, the derivative decreases by c (the bag at the right end no longer contributes coins).

2. **Event Processing**: Sort all events by their x-coordinate. Process events in order, grouping events at the same x-coordinate. For each event:
   - Update the current function value (coins collected) using the current derivative over the distance since the last event.
   - Adjust the current derivative by the sum of deltas from all events at the current x-coordinate.
   - Track the maximum function value encountered during processing.

3. **Result Extraction**: The maximum value recorded during event processing is the solution, representing the maximum coins obtainable from any k consecutive bags.

### Solution Code
```python
class Solution:
    def maximumCoins(self, coins: List[List[int]], k: int) -> int:
        events = []
        for l, r, c in coins:
            events.append((l, -c))
            events.append((r + 1, c))
            events.append((l - k, c))
            events.append((r + 1 - k, -c))
        
        if not events:
            return 0
        
        events.sort(key=lambda x: x[0])
        current_x = events[0][0] - 1
        current_deriv = 0
        current_f = 0
        max_val = 0
        i = 0
        n = len(events)
        
        while i < n:
            x = events[i][0]
            current_f += current_deriv * (x - current_x)
            current_x = x
            total_delta = 0
            while i < n and events[i][0] == x:
                total_delta += events[i][1]
                i += 1
            current_deriv += total_delta
            if current_f > max_val:
                max_val = current_f
        return max_val
```

1. **Event Creation**: For each segment, four events are created to mark changes in the derivative of the function representing coins collected. These events correspond to the left and right ends of the window entering or exiting the segment.
2. **Event Processing**: The events are sorted by their x-coordinate. The algorithm processes each event in order, updating the current function value based on the derivative and the distance since the last event. The derivative is adjusted by the sum of deltas from events at the same x-coordinate.
3. **Tracking Maximum**: During processing, the algorithm keeps track of the maximum function value encountered, which after processing all events, gives the maximum coins obtainable from any k consecutive bags.

This approach efficiently handles the large input constraints by leveraging event-based processing and linear updates between critical points, ensuring optimal performance.
