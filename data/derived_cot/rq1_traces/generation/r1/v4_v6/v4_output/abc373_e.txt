You will be given a competitive programming problem.
An election is being held with N candidates numbered 1, 2, \ldots, N. There are K votes, some of which have been counted so far.
Up until now, candidate i has received A_i votes.
After all ballots are counted, candidate i (1 \leq i \leq N) will be elected if and only if the number of candidates who have received more votes than them is less than M.  There may be multiple candidates elected.
For each candidate, find the minimum number of additional votes they need from the remaining ballots to guarantee their victory regardless of how the other candidates receive votes.
Formally, solve the following problem for each i = 1,2,\ldots,N.
Determine if there is a non-negative integer X not exceeding K - \displaystyle{\sum_{i=1}^{N}} A_i satisfying the following condition.  If it exists, find the minimum possible such integer.

- If candidate i receives X additional votes, then candidate i will always be elected.

Input

The input is given from Standard Input in the following format:
N M K
A_1 A_2 \ldots A_N

Output

Let C_i be the minimum number of additional votes candidate i needs from the remaining ballots to guarantee their victory regardless of how other candidates receive votes. Print C_1, C_2, \ldots, C_N separated by spaces.
If candidate i has already secured their victory, then let C_i = 0. If candidate i cannot secure their victory under any circumstances, then let C_i = -1.

Constraints


- 1 \leq M \leq N \leq 2 \times 10^5
- 1 \leq K \leq 10^{12}
- 0 \leq A_i \leq 10^{12}
- \displaystyle{\sum_{i=1}^{N} A_i} \leq K
- All input values are integers.

Sample Input 1

5 2 16
3 1 4 1 5

Sample Output 1

2 -1 1 -1 0

14 votes have been counted so far, and 2 votes are left.
The C to output is (2, -1, 1, -1, 0).  For example:

- Candidate 1 can secure their victory by obtaining 2 more votes, while not by obtaining 1 more vote.  Thus, C_1 = 2.
- Candidate 2 can never (even if they obtain 2 more votes) secure their victory, so C_2 = -1.

To solve this problem, we need to determine for each candidate the minimum number of additional votes they need to guarantee their election, regardless of how the remaining votes are distributed among other candidates. The candidate is elected if the number of candidates who have received more votes than them is less than M.

### Approach
1. **Problem Analysis**: 
   - **Input**: N (number of candidates), M (number of candidates that can be elected), K (total votes), and an array A where A_i represents the current votes for candidate i.
   - **Remaining Votes**: R = K - sum(A_i) votes are left to be distributed.
   - **Objective**: For each candidate i, find the minimum X (0 ≤ X ≤ R) such that if candidate i gets X additional votes, then no matter how the remaining R - X votes are distributed to others, candidate i is elected.

2. **Key Insight**:
   - A candidate i is elected if there are less than M candidates who have strictly more votes than i. To ensure this, even in the worst-case scenario where other candidates receive all remaining votes (except those given to i), candidate i must still be in the top M.
   - For candidate i, if we give them x additional votes, their total becomes A_i + x. The worst-case scenario is when all other candidates (especially the ones currently below or near) receive as many votes as possible to surpass candidate i.

3. **Binary Search for Minimum X**:
   - For each candidate i, we use binary search to find the minimum x such that even if we assign the remaining R - x votes to other candidates in the most adversarial way, candidate i still has no more than M-1 candidates with votes strictly greater than A_i + x.
   - To check a particular x for candidate i:
     - Let T = A_i + x.
     - Count C0 = number of candidates j (j ≠ i) with A_j > T.
     - If C0 ≥ M, x is insufficient.
     - Otherwise, compute the minimal votes required to push M - C0 candidates (from those currently ≤ T) above T. If this sum exceeds R - x, candidate i is safe.

4. **Efficiency**:
   - Precompute sorted array and prefix sums for efficient calculations.
   - For each candidate, binary search over x (0 to R) with each check operation O(log N) due to binary search in the sorted array.
   - Total complexity: O(N (log N + log R)).

```python
            available_candidates = total_count - 1
            k = m - C0
            if available_candidates < k:
                if (n - 1) < m:
                    ans_x = mid
                    high = mid - 1
                else:
                    low = mid + 1
                continue
            
            pos_i = pos_in_sorted[i]
            if total_count - k <= pos_i < total_count:
                if total_count - (k + 1) >= 0:
                    sum_top_k_plus_1 = prefix_sum[total_count] - prefix_sum[total_count - (k + 1)]
                else:
                    sum_top_k_plus_1 = prefix_sum[total_count]
                sum_top_k_plus_1_without_i = sum_top_k_plus_1 - a_i
                S = k * (T + 1) - sum_top_k_plus_1_without_i
            else:
                sum_top_k = prefix_sum[total_count] - prefix_sum[total_count - k]
                S = k * (T + 1) - sum_top_k
            
            if S > (R - mid):
                ans_x = mid
                high = mid - 1
            else:
                low = mid + 1
        
        results.append(str(ans_x))
    
    print(" ".join(results))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Precomputation**:
   - **Sorted Array and Prefix Sums**: The array of votes is sorted, and prefix sums are computed to facilitate quick range sum queries.
   - **Position Mapping**: Each candidate's position in the sorted array is stored for quick access during the check operation.

2. **Binary Search for Each Candidate**:
   - For each candidate, we perform a binary search over possible additional votes (0 to R).
   - **Check Function**: For a given x, compute the candidate's new vote count T = A_i + x. Determine how many candidates already have more votes than T (C0). If C0 ≥ M, x is insufficient.
   - **Safety Check**: If there aren't enough candidates to push above T to threaten the candidate's position (i.e., available_candidates < M - C0), check if the total candidates above T would still be less than M.
   - **Gap Calculation**: Otherwise, compute the minimal votes required to push M - C0 candidates above T. If this sum exceeds the remaining votes after giving x to the candidate, x is sufficient.

3. **Output**: For each candidate, the minimal x that ensures their election (or -1 if impossible) is collected and printed.

This approach efficiently handles the constraints by leveraging binary search and precomputed data structures, ensuring optimal performance for large inputs.
