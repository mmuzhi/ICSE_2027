There is a grid of N^2 squares with N rows and N columns.
Let (i,j) denote the square at the i-th row from the top (1\leq i\leq N) and j-th column from the left (1\leq j\leq N).
Each square is either empty or has a piece placed on it.
There are M pieces placed on the grid, and the k-th (1\leq k\leq M) piece is placed on square (a_k,b_k).
You want to place your piece on an empty square in such a way that it cannot be captured by any of the existing pieces.
A piece placed on square (i,j) can capture pieces that satisfy any of the following conditions:

- Placed in row i
- Placed in column j
- Placed on any square (a,b)\ (1\leq a\leq N,1\leq b\leq N) where i+j=a+b
- Placed on any square (a,b)\ (1\leq a\leq N,1\leq b\leq N) where i-j=a-b

For example, a piece placed on square (4,4) can capture pieces placed on the squares shown in blue in the following figure:

How many squares can you place your piece on?

Input

The input is given from Standard Input in the following format:
N M
a_1 b_1
a_2 b_2
\vdots
a_M b_M

Output

Print the number of empty squares where you can place your piece without it being captured by any existing pieces.

Constraints


- 1\leq N\leq10^9
- 1\leq M\leq10^3
- 1\leq a_k\leq N,1\leq b_k\leq N\ (1\leq k\leq M)
- (a_k,b_k)\neq(a_l,b_l)\ (1\leq k\lt l\leq M)
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to determine the number of empty squares in an N x N grid where we can place a piece such that it is not captured by any of the existing M pieces. The capturing conditions are that a piece can capture another piece if they share the same row, column, or either of the two diagonals (sum-diagonal or difference-diagonal).

### Approach
1. **Problem Analysis**: The problem involves a grid of size N x N with M pieces already placed. For any empty square (i, j), we need to ensure that it is not in the same row, column, or diagonal as any existing piece. The key challenge is efficiently computing the number of safe squares without iterating over the entire grid (which is infeasible for large N).

2. **Intuition**: The solution leverages inclusion-exclusion principles to count the number of unsafe squares (those attacked by existing pieces) and subtract this from the total number of squares (N²), adjusting for occupied squares. The unsafe squares are those that lie in any row, column, or diagonal of an existing piece.

3. **Algorithm Selection**: 
   - **Sets for Rows, Columns, and Diagonals**: We maintain sets of rows (R), columns (Col), sum-diagonals (D1), and difference-diagonals (D2) for all existing pieces.
   - **Inclusion-Exclusion Principle**: We compute the total number of squares attacked by any row, column, or diagonal using inclusion-exclusion over the four sets (rows, columns, sum-diagonals, difference-diagonals). This involves calculating the sizes of individual sets and their intersections (pairwise, triple, and quadruple).

4. **Complexity Analysis**: The approach efficiently handles large N (up to 10^9) by focusing on the M pieces (up to 1000). The nested loops over sets (R, Col, D1, D2) result in O(M²) operations, which is feasible since M is small (≤ 1000).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    m = int(data[1])
    pieces = []
    R = set()
    Col_set = set()
    D1 = set()
    D2 = set()
    
    index = 2
    for _ in range(m):
        a = int(data[index])
        b = int(data[index + 1])
        index += 2
        pieces.append((a, b))
        R.add(a)
        Col_set.add(b)
        D1.add(a + b)
        D2.add(a - b)
    
    sizeA = len(R) * n
    sizeB = len(Col_set) * n
    sizeC = 0
    for d in D1:
        if d <= n + 1:
            sizeC += d - 1
        else:
            sizeC += 2 * n - d + 1
    sizeD = 0
    for d in D2:
        sizeD += n - abs(d)
    
    AB = len(R) * len(Col_set)
    
    AC = 0
    for r in R:
        for d in D1:
            y = d - r
            if 1 <= y <= n:
                AC += 1
                
    AD = 0
    for r in R:
        for d in D2:
            y = r - d
            if 1 <= y <= n:
                AD += 1
                
    BC = 0
    for c in Col_set:
        for d in D1:
            x = d - c
            if 1 <= x <= n:
                BC += 1
                
    BD = 0
    for c in Col_set:
        for d in D2:
            x = d + c
            if 1 <= x <= n:
                BD += 1
                
    CD = 0
    for d1 in D1:
        for d2 in D2:
            total = d1 + d2
            if total % 2 != 0:
                continue
            x = total // 2
            y = d1 - x
            if 1 <= x <= n and 1 <= y <= n:
                CD += 1
                
    ABC = 0
    for r in R:
        for c in Col_set:
            if (r + c) in D1:
                ABC += 1
                
    ABD = 0
    for r in R:
        for c in Col_set:
            if (r - c) in D2:
                ABD += 1
                
    ACD = 0
    for r in R:
        for d1 in D1:
            y = d1 - r
            if 1 <= y <= n:
                d2 = 2 * r - d1
                if d2 in D2:
                    ACD += 1
                    
    BCD = 0
    for c in Col_set:
        for d1 in D1:
            x = d1 - c
            if 1 <= x <= n:
                d2 = d1 - 2 * c
                if d2 in D2:
                    BCD += 1
                    
    ABCD = 0
    for r in R:
        for c in Col_set:
            if (r + c) in D1 and (r - c) in D2:
                ABCD += 1
                
    union_size = (sizeA + sizeB + sizeC + sizeD) - (AB + AC + AD + BC + BD + CD) + (ABC + ABD + ACD + BCD) - ABCD
    ans = n * n - union_size
    print(ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the grid size (N), number of pieces (M), and their positions.
2. **Sets Initialization**: Sets for rows (R), columns (Col), sum-diagonals (D1), and difference-diagonals (D2) are populated from the piece positions.
3. **Set Sizes Calculation**: The sizes of the sets (A, B, C, D) are computed. Set A (rows) and B (columns) each contribute N squares per distinct row or column. Sets C (sum-diagonals) and D (difference-diagonals) contribute squares based on diagonal properties.
4. **Inclusion-Exclusion Application**: The inclusion-exclusion principle is applied to compute the total number of unsafe squares by considering intersections of the sets (AB, AC, AD, BC, BD, CD), triple intersections (ABC, ABD, ACD, BCD), and quadruple intersections (ABCD).
5. **Result Calculation**: The total number of safe squares is derived by subtracting the count of unsafe squares (union_size) from the total grid squares (N²), and the result is printed.

This approach efficiently narrows down the problem to manageable set operations, avoiding the need to process the entire grid directly, thus optimizing for large N.
