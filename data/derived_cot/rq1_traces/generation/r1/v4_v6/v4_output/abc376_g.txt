To solve this problem, we need to find the expected number of operations required to find a treasure hidden in a tree, where each operation involves selecting an unsearched node with a searched parent and marking it as searched. The goal is to minimize the expected number of operations by choosing the optimal order of searching nodes.

### Approach
1. **Tree Representation**: Represent the tree using adjacency lists where each node points to its children. The tree is rooted at node 0.
2. **Postorder Processing**: Process nodes from leaves to the root (postorder) to compute necessary values for each subtree.
3. **Key Values Calculation**: For each node, compute:
   - **w[u]**: The total probability weight (sum of `a_i`) of the subtree rooted at `u`.
   - **s[u]**: The number of nodes in the subtree rooted at `u`.
4. **Optimal Ordering**: For each node, sort its children in descending order of the ratio `w[v] / s[v]`. This ratio helps prioritize subtrees with higher probability density, minimizing the expected operations.
5. **Dynamic Programming**: For each node, compute the expected operations `F[u]` using the recurrence:
   - For each child `v` in sorted order, `F[u] += F[v] + w[v] * (1 + cumulative_size)`, where `cumulative_size` is the sum of sizes of previously processed subtrees.
6. **Modulo Handling**: Since the result needs to be reported modulo 998244353, compute the final result as `(F[0] * inverse(w[0], 998244353)) % 998244353`, where `inverse` is the modular multiplicative inverse.
