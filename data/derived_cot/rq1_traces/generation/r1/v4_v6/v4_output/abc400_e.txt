You will be given a competitive programming problem.
A positive integer N is a 400 number if and only if it satisfies both of the following two conditions:

- N has exactly 2 distinct prime factors.
- For each prime factor p of N, p divides N an even number of times. More formally, the maximum non-negative integer k such that p^k divides N is even.

Process Q queries. Each query gives you an integer A, so find the largest 400 number not exceeding A. Under the constraints of this problem, a 400 number not exceeding A always exists.

Input

The input is given from Standard Input in the following format:
Q
\text{query}_1
\text{query}_2
\vdots
\text{query}_Q

Here, \text{query}_i is the i-th query, given in the following format:
A

Output

Print Q lines. The i-th line should contain the answer to the i-th query.

Constraints


- 1 \leq Q \leq 2 \times 10^5
- For each query, 36 \leq A \leq 10^{12}.
- All input values are integers.

To solve this problem, we need to find the largest "400 number" not exceeding a given integer \( A \) for each query. A "400 number" is defined as a positive integer \( N \) that satisfies two conditions:
1. \( N \) has exactly two distinct prime factors.
2. For each prime factor \( p \) of \( N \), the exponent of \( p \) in the prime factorization of \( N \) is even.

### Approach
1. **Understanding 400 Numbers**: 
   - A 400 number \( N \) can be expressed as \( N = (p^a \cdot q^b)^2 \), where \( p \) and \( q \) are distinct primes, and \( a \) and \( b \) are positive integers. This is because the exponents in the prime factorization of \( N \) must be even, making \( N \) a perfect square. The base number \( m = p^a \cdot q^b \) must have exactly two distinct prime factors.
   
2. **Precomputation**:
   - We precompute all possible values of \( m \) up to \( 10^6 \) (since \( m^2 \) must be \( \leq 10^{12} \)) such that \( m \) has exactly two distinct prime factors. This is done using a sieve method to count distinct prime factors for each number up to \( 10^6 \).
   - For each valid \( m \), we compute \( N = m^2 \) and store these values in a sorted list \( L \). The list \( L \) is naturally sorted because \( m \) increases, leading to \( m^2 \) increasing.

3. **Query Processing**:
   - For each query \( A \), we perform a binary search on the sorted list \( L \) to find the largest \( N \) such that \( N \leq A \). This is efficiently done using the `bisect_right` function from the `bisect` module, which returns the insertion position of \( A \) in \( L \) to maintain sorted order. The largest valid \( N \) is then the element at the position immediately before this insertion point.

### Solution Code
```python
import bisect
import sys

def main():
    MAX_M = 10**6
    dpc = [0] * (MAX_M + 1)
    for i in range(2, MAX_M + 1):
        if dpc[i] == 0:
            for j in range(i, MAX_M + 1, i):
                dpc[j] += 1

    L = []
    for m in range(6, MAX_M + 1):
        if dpc[m] == 2:
            L.append(m * m)
    
    data = sys.stdin.read().split()
    q = int(data[0])
    queries = list(map(int, data[1:1 + q]))
    
    res = []
    for A in queries:
        idx = bisect.bisect_right(L, A)
        res.append(str(L[idx - 1]))
    
    print("\n".join(res))

    main()
```

1. **Precomputation**:
   - We use a sieve algorithm to compute the number of distinct prime factors for each number up to \( 10^6 \). The array `dpc` is initialized to zero, and for each prime number \( i \), we increment `dpc[j]` for all multiples \( j \) of \( i \).

2. **Query Handling**:
