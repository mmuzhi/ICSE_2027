Print all integer sequences of length N that satisfy the following conditions, in ascending lexicographical order.

- The i-th element is between 1 and R_i, inclusive.

 What is lexicographical order for sequences?
A sequence A = (A_1, \ldots, A_{|A|}) is lexicographically smaller than B = (B_1, \ldots, B_{|B|}) if either 1. or 2. below holds:

- |A|<|B| and (A_{1},\ldots,A_{|A|}) = (B_1,\ldots,B_{|A|}).
- There exists an integer 1\leq i\leq \min\{|A|,|B|\} such that both of the following are true:

- (A_{1},\ldots,A_{i-1}) = (B_1,\ldots,B_{i-1})
- A_i < B_i

=== ANSWER ===
To solve this problem, we need to generate all integer sequences of length N where each element at position i is between 1 and R_i (inclusive), and the sum of all elements in the sequence is a multiple of K. The sequences should be printed in ascending lexicographical order.

### Approach
1. **Problem Analysis**: The problem involves generating all possible sequences of integers where each element in the sequence adheres to constraints defined by the input values. The key constraints are:
   - The i-th element must be between 1 and R_i.
   - The sum of all elements in the sequence must be divisible by K.
   - The sequences must be printed in lexicographical order.

2. **Intuition**: Given the constraints (N ≤ 8, R_i ≤ 5), the total number of possible sequences is manageable (up to 5^8 = 390,625). We can generate all possible sequences using the Cartesian product of the ranges for each position. This approach ensures that sequences are generated in lexicographical order naturally.

3. **Algorithm Selection**: We use the `itertools.product` function to generate the Cartesian product of the ranges for each sequence position. For each generated sequence, we check if the sum of its elements is divisible by K. If it is, we print the sequence.

4. **Complexity Analysis**: The time complexity is O(Π R_i), which is the product of all R_i values. Given the constraints, this is feasible. The space complexity is O(N) for storing each sequence during generation.

### Solution Code
```python
import itertools
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    k = int(data[1])
    R_list = list(map(int, data[2:2+n]))
    
    ranges_list = [range(1, r+1) for r in R_list]
    
    for seq in itertools.product(*ranges_list):
        if sum(seq) % k == 0:
            print(" ".join(str(x) for x in seq))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input and split into individual components. The first two values are N (length of the sequence) and K (the divisor for the sum). The subsequent N values are the R_i constraints for each position in the sequence.
2. **Generating Ranges**: For each position i, a range from 1 to R_i (inclusive) is created. These ranges are stored in a list.
3. **Generating Sequences**: Using `itertools.product`, the Cartesian product of these ranges is computed. This generates all possible sequences in lexicographical order.
4. **Checking Sum Condition**: For each generated sequence, the sum of its elements is checked. If the sum is divisible by K, the sequence is printed.
5. **Output**: Valid sequences are printed in the required format, with elements separated by spaces.

This approach efficiently generates and checks all possible sequences, leveraging Python's built-in functions for simplicity and readability. The solution handles edge cases, such as no valid sequences, by simply producing no output.
