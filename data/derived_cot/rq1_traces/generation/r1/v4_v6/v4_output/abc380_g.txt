You are given a permutation P of (1,2,\dots,N) and an integer K.

- First, choose an integer i uniformly at random between 1 and N - K + 1, inclusive.

What is the inversion number?
The inversion number of a sequence (A_1, A_2, \dots, A_N) is the number of integer pairs (i, j) satisfying 1 \le i < j \le N and A_i > A_j.


What does "expected value modulo 998244353" mean?
It can be proved that the sought expected value is always rational. Under the constraints of this problem, when this value is represented as an irreducible fraction \frac{P}{Q}, it can also be proved that Q \not\equiv 0 \pmod{998244353}. Thus, there is a unique integer R satisfying R \times Q \equiv P \pmod{998244353}, \ 0 \le R < 998244353. Report this integer R.

Output

Print the answer in one line.

      mod = 998244353

      class FenwTree:
          def __init__(self, n):
              self.n = n
              self.tree = [0] * (n+1)
          
          def update(self, idx, delta):
              while idx <= self.n:
                  self.tree[idx] = (self.tree[idx] + delta) % mod
                  idx += idx & -idx
          
          def query(self, idx):
              s = 0
              while idx:
                  s = (s + self.tree[idx]) % mod
                  idx -= idx & -idx
              return s
          
          def range_query(self, l, r):
              if l > r:
                  return 0
              res = self.query(r) - self.query(l-1)
              res %= mod
              if res < 0:
                  res += mod
              return res

To solve this problem, we need to compute the expected inversion number of a permutation after performing a specific random operation. The operation involves selecting a random contiguous subarray of length K and shuffling it uniformly at random. The challenge is to efficiently compute the expected inversion number modulo 998244353.

### Approach
1. **Initial Inversion Count**: First, compute the initial inversion number of the permutation.
2. **Expected Inversion After Operation**: The operation affects only the inversions that involve elements within the chosen subarray. The inversions entirely outside the subarray or entirely inside the subarray (but not crossing the boundaries) need to be considered separately.
3. **Linearity of Expectation**: The total expected inversion number can be broken down into the sum of expectations for each pair of indices (i, j). We can categorize pairs based on whether both elements are in the subarray or one is in and the other is out.
4. **Handling Overlapping Subarrays**: For each possible subarray of length K, the inversion number within the subarray after shuffling becomes the expected inversion number of a random permutation of size K, which is \( \frac{K(K-1)}{4} \). However, we must adjust for the fact that the initial inversion count of the entire permutation includes inversions both within and outside the subarray.
5. **Adjustment for Overlaps**: The key insight is that for any pair of indices (i, j), the probability that both lie in a randomly chosen subarray is \( \frac{\max(0, K - |i-j|)}{N-K+1} \). Using this, we can compute the expected change in inversion number for each pair.
