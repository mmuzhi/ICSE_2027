You are given an integer array nums of length n and a 2D array queries where queries[i] = [l_i, r_i].
Each queries[i] represents the following action on nums:

Decrement the value at each index in the range [l_i, r_i] in nums by at most 1.
The amount by which the value is decremented can be chosen independently for each index.

A Zero Array is an array with all its elements equal to 0.
Return the maximum number of elements that can be removed from queries, such that nums can still be converted to a zero array using the remaining queries. If it is not possible to convert nums to a zero array, return -1.
 
Example 1:

Input: nums = [2,0,2], queries = [[0,2],[0,2],[1,1]]
Output: 1
Explanation:
After removing queries[2], nums can still be converted to a zero array.

Using queries[0], decrement nums[0] and nums[2] by 1 and nums[1] by 0.
Using queries[1], decrement nums[0] and nums[2] by 1 and nums[1] by 0.


Example 2:

Input: nums = [1,1,1,1], queries = [[1,3],[0,2],[1,3],[1,2]]
Output: 2
Explanation:
We can remove queries[2] and queries[3].

Example 3:

Input: nums = [1,2,3,4], queries = [[0,3]]
Output: -1
Explanation:
nums cannot be converted to a zero array even after using all the queries.

 
Constraints:

1 <= nums.length <= 10^5
0 <= nums[i] <= 10^5
1 <= queries.length <= 10^5
queries[i].length == 2

To solve this problem, we need to determine the maximum number of queries that can be removed such that the remaining queries can still convert the given array `nums` into a zero array. Each query allows decrementing elements in a specified range by at most 1 (independently for each index). The solution involves checking if the total coverage of all queries is sufficient to cover the decrement requirements of each element in `nums` and then finding the minimal subset of queries that still satisfies these requirements.

### Approach
1. **Check Feasibility**: 
   - Compute the total coverage each index in `nums` receives from all queries using a difference array. If any index has less coverage than its value in `nums`, it's impossible to convert `nums` to a zero array, so return -1.

2. **Minimal Query Subset Selection**:
   - **Initialization**: Prepare events for each query's start index and track their end indices. Use a max-heap (simulated with a min-heap storing negative values) to manage available queries that start at or before the current index and cover the current index.
   - **Processing Indices**: For each index from left to right:
     - Remove queries from the active set that no longer cover the current index (their end index is less than the current index).
     - Add new queries starting at the current index to the available heap.
     - While the current coverage is less than the required decrement (`nums[i]`), select the query from the available heap with the largest end index (to maximize future coverage), update the coverage, and add the query to the active set.
   - **Result Calculation**: The number of removable queries is the total number of queries minus the size of the minimal selected subset.

### Solution Code
```python
import heapq
from typing import List

class Solution:
    def maxRemoval(self, nums: List[int], queries: List[List[int]]) -> int:
        n = len(nums)
        m = len(queries)
        total_cover_arr = [0] * (n + 1)
        
        for l, r in queries:
            total_cover_arr[l] += 1
            if r + 1 < n:
                total_cover_arr[r + 1] -= 1
        
        for i in range(1, n):
            total_cover_arr[i] += total_cover_arr[i - 1]
        
        for i in range(n):
            if total_cover_arr[i] < nums[i]:
                return -1
        
        start_events = [[] for _ in range(n)]
        for l, r in queries:
            start_events[l].append(r)
        
        cover = 0
        selected = 0
        available = []
        active_end = []
        
        for i in range(n):
            while active_end and active_end[0] < i:
                heapq.heappop(active_end)
                cover -= 1
            
            for r in start_events[i]:
                heapq.heappush(available, -r)
            
            while cover < nums[i]:
                if not available:
                    return -1
                max_end = -heapq.heappop(available)
                selected += 1
                cover += 1
                heapq.heappush(active_end, max_end)
        
        return m - selected
```

### Explanation
1. **Feasibility Check**: The difference array `total_cover_arr` is used to compute the total coverage each index receives from all queries. If any index's coverage is less than its value in `nums`, converting `nums` to a zero array is impossible, and we return -1.
2. **Event Setup**: For each query, its end index is stored in `start_events` at its start index. This helps in efficiently accessing queries as we process each index.
3. **Processing Indices**: 
   - **Active Queries**: Queries in `active_end` that no longer cover the current index (end index < current index) are removed, and coverage is decremented.
   - **Available Queries**: Queries starting at the current index are added to `available` (a max-heap for end indices).
   - **Coverage Adjustment**: If the current coverage is insufficient, queries from `available` with the largest end indices are selected to maximize future coverage. The selected queries are added to `active_end`.
4. **Result**: The number of removable queries is the total queries minus the number of selected queries (`m - selected`), representing the maximal removable queries while ensuring `nums` can be converted to a zero array.
