There are N slimes lined up in a row from left to right.
For i = 1, 2, \ldots, N, the i-th slime from the left has size A_i.
For each K = 1, 2, \ldots, N, solve the following problem.

Takahashi is the K-th slime from the left in the initial state.
Find the maximum size that he can have after performing the following action any number of times, possibly zero:

- Choose a slime adjacent to him that is strictly smaller than him, and absorb it.

When a slime disappears due to absorption, the gap is immediately closed, and the slimes that were adjacent to the disappearing slime (if they exist) become adjacent (see the explanation in Sample Input 1).

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the answers B_K for each K = 1, 2, \ldots, N, separated by spaces, in the following format:
B_1 B_2 \ldots B_N

Constraints


- 2 \leq N \leq 5 \times 10^5
- 1 \leq A_i \leq 10^9
- All input values are integers.

6
4 13 2 3 2 6

Sample Output 1

4 30 2 13 2 13

To solve this problem, we need to determine the maximum size Takahashi can achieve for each starting position K (from 1 to N) in a row of slimes. The key observation is that Takahashi can only absorb adjacent slimes that are strictly smaller than his current size. The challenge is to efficiently compute the maximum size for each starting position without simulating each scenario individually, which would be too slow for large N (up to 500,000).

### Approach
1. **Problem Analysis**: The problem involves processing a sequence of slimes where Takahashi, starting at position K, can absorb adjacent smaller slimes. The absorption process is sequential and can propagate in both directions (left and right) as long as the adjacent slime is strictly smaller. The goal is to maximize Takahashi's final size for each starting position.

2. **Key Insight**: The absorption process can be viewed as merging adjacent slimes in increasing order of size. However, the direction of absorption (left or right) and the initial position play crucial roles. The solution involves precomputing the cumulative sums of slime sizes and determining the range of slimes Takahashi can absorb based on the initial conditions.

3. **Algorithm Selection**: 
   - **Prefix and Suffix Arrays**: We use prefix sums to quickly calculate the total size of absorbed slimes in a contiguous segment.
   - **Monotonic Stack**: To efficiently compute the next larger element to the left and right of each slime. This helps in determining the boundaries within which Takahashi can absorb slimes.

4. **Complexity Analysis**: The algorithm runs in O(N) time for each of the preprocessing steps (building prefix/suffix arrays and using the monotonic stack for next larger elements), leading to an overall O(N) time complexity, which is efficient for the given constraints.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    prefix = [0] * (n + 1)
    for i in range(1, n + 1):
        prefix[i] = prefix[i - 1] + A[i - 1]
    
    left_bound = [-1] * n
    stack = []
    for i in range(n):
        while stack and A[stack[-1]] < A[i]:
            stack.pop()
        if stack:
            left_bound[i] = stack[-1]
        stack.append(i)
    
    right_bound = [n] * n
    stack = []
    for i in range(n - 1, -1, -1):
        while stack and A[stack[-1]] < A[i]:
            stack.pop()
        if stack:
            right_bound[i] = stack[-1]
        stack.append(i)
    
    left_chain = [0] * n
    for i in range(n):
        if left_bound[i] == -1:
            left_chain[i] = prefix[i]
        else:
            left_chain[i] = left_chain[left_bound[i]] + (prefix[i] - prefix[left_bound[i] + 1])
    
    right_chain = [0] * n
    for i in range(n - 1, -1, -1):
        if right_bound[i] == n:
            right_chain[i] = prefix[n] - prefix[i]
        else:
            right_chain[i] = right_chain[right_bound[i]] + (prefix[right_bound[i]] - prefix[i])
    
    ans = [0] * n
    for i in range(n):
        ans[i] = A[i] + (left_chain[i] - A[i]) + (right_chain[i] - A[i])
    
    print(" ".join(map(str, ans)))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Prefix Sum Array**: The `prefix` array stores cumulative sums of the slime sizes, enabling quick calculation of the sum of any contiguous segment.
2. **Left and Right Boundaries**: Using a monotonic stack, we compute for each slime the nearest larger slime to its left (`left_bound`) and right (`right_bound`). These boundaries define the segments that can be absorbed without interruption.
3. **Left and Right Chains**: For each slime, `left_chain[i]` represents the total size of absorbable slimes to the left up to the nearest larger slime. Similarly, `right_chain[i]` does the same for the right side. These values are derived using the boundaries and prefix sums.
4. **Final Calculation**: The maximum size for Takahashi starting at position `i` is the initial size `A[i]` plus the total absorbable sizes from both the left and right chains, adjusted to avoid double-counting the initial size.

This approach efficiently computes the solution for each starting position in linear time, making it suitable for large input sizes.
