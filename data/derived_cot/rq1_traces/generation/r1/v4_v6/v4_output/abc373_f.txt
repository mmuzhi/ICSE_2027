You will be given a competitive programming problem.
There are N types of items. The i-th type of item has a weight of w_i and a value of v_i. Each type has 10^{10} items available.
Takahashi is going to choose some items and put them into a bag with capacity W. He wants to maximize the value of the selected items while avoiding choosing too many items of the same type. Hence, he defines the happiness of choosing k_i items of type i as k_i v_i - k_i^2. He wants to choose items to maximize the total happiness over all types while keeping the total weight at most W. Calculate the maximum total happiness he can achieve.

Input

The input is given from Standard Input in the following format:
N W
w_1 v_1
w_2 v_2
\vdots
w_N v_N

Output

Print the answer.

Constraints


- 1 \leq N \leq 3000
- 1 \leq W \leq 3000
- 1 \leq w_i \leq W
- 1 \leq v_i \leq 10^9
- All input values are integers.

Sample Input 1

2 10
3 4
3 2

Sample Output 1

5

By choosing 2 items of type 1 and 1 item of type 2, the total happiness can be 5, which is optimal.
Here, the happiness for type 1 is 2 \times 4 - 2^2 = 4, and the happiness for type 2 is 1 \times 2 - 1^2 = 1.
The total weight is 9, which is within the capacity 10.

=== ANSWER ===
To solve this problem, we need to maximize the total happiness of selected items under a weight constraint, where the happiness for each type of item is defined as \(k_i v_i - k_i^2\) for \(k_i\) items of type \(i\). The challenge is to efficiently select the optimal number of items from each type without exceeding the bag's capacity.

### Approach
1. **Problem Analysis**: The problem involves selecting items from multiple types, each with a weight and value. The happiness for each type is a quadratic function of the number of items selected, which is concave (i.e., the marginal gain decreases with each additional item). The goal is to maximize the total happiness while ensuring the total weight does not exceed the given capacity.

2. **Key Insight**: The concave nature of the happiness function allows us to use a greedy approach. Specifically, we prioritize items with the highest marginal gain (value of the next item) first. The marginal gain for the \(j\)-th item of type \(i\) is \(v_i - (2j - 1)\), which decreases as \(j\) increases.

3. **Algorithm Selection**: We use a max-heap (priority queue) to always pick the item with the highest current marginal gain. For each item taken, we update the remaining capacity and push the next item from the same type into the heap if its marginal gain is positive. This ensures we always consider the most beneficial items first.

4. **Complexity Analysis**: The algorithm processes each item type up to \(\min(W / w_i, (v_i + 1)/2)\) times. For \(N\) types, this results in up to \(N \times 3000 = 9 \times 10^6\) operations. Each heap operation (push and pop) is \(O(\log n)\), leading to an overall complexity of \(O(9 \times 10^6 \log 9 \times 10^6)\), which is feasible for the given constraints.

### Solution Code
```python
import heapq
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    W = int(data[1])
    items = []
    idx = 2
    for i in range(n):
        w = int(data[idx])
        v = int(data[idx + 1])
        idx += 2
        items.append((w, v))
    
    counts = [0] * n
    heap = []
    for i in range(n):
        w, v = items[i]
        value1 = v - 1
        heapq.heappush(heap, (-value1, w, i))
    
    total_val = 0
    cap = W
    while cap > 0 and heap:
        neg_val, w, i = heapq.heappop(heap)
        real_val = -neg_val
        if real_val <= 0:
            continue
        if w > cap:
            continue
        total_val += real_val
        cap -= w
        counts[i] += 1
        j = counts[i] + 1
        v_i = items[i][1]
        next_val = v_i - (2 * j - 1)
        if next_val > 0:
            heapq.heappush(heap, (-next_val, items[i][0], i))
    print(total_val)

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read and parsed into the number of item types \(N\), the bag capacity \(W\), and a list of tuples representing each item's weight and value.
2. **Initial Setup**: A count array tracks the number of items taken from each type. A max-heap (simulated using negative values) is initialized with the first item from each type, where the value is \(v_i - 1\).
3. **Processing Items**: While there is remaining capacity and items in the heap:
   - The item with the highest marginal gain is popped.
   - If the item's value is non-positive or it doesn't fit, it is skipped.
   - Otherwise, the item is taken, updating the total happiness and remaining capacity.
   - The next item from the same type is pushed into the heap if its marginal gain is positive.
4. **Output**: The total happiness value is printed after processing all possible items.

This approach efficiently maximizes happiness by leveraging the concave nature of the happiness function and a greedy selection strategy, ensuring optimal performance within the problem constraints.
