You will be given a competitive programming problem.
There is a printing machine that prints line segments on the xy-plane by emitting a laser.

- At the start of printing, the laser position is at coordinate (0, 0).
- 
When printing a line segment, the procedure below is followed.

- First, move the laser position to one of the endpoints of the line segment.
- One may start drawing from either endpoint.


- Then, move the laser position in a straight line from the current endpoint to the other endpoint while emitting the laser.
- It is not allowed to stop printing in the middle of a line segment.




- 
When not emitting the laser, the laser position can move in any direction at a speed of S units per second.

- When emitting the laser, the laser position can move along the line segment being printed at a speed of T units per second.
- The time required for operations other than moving the laser position can be ignored.

Takahashi wants to print N line segments using this printing machine.
The i-th line segment connects coordinates (A_i, B_i) and (C_i, D_i).
Some line segments may overlap, in which case he needs to print the overlapping parts for each line segment separately.
What is the minimum number of seconds required to complete printing all the line segments when he operates the printing machine optimally?

Input

The input is given from Standard Input in the following format:
N S T
A_1 B_1 C_1 D_1
\vdots
A_N B_N C_N D_N

Output

Your output will be considered correct if the absolute or relative error from the true value does not exceed 10^{-6}.

Constraints


- All input values are integers.
- 1 \le N \le 6
- 1 \le T \le S \le 1000
- -1000 \le A_i,B_i,C_i,D_i \le 1000
- (A_i,B_i) \neq (C_i,D_i) ( 1 \le i \le N )

To solve this problem, we need to find the minimum time required to print all given line segments on the xy-plane using a laser printing machine. The laser starts at the origin (0, 0) and can move without emitting the laser at speed S, or while emitting the laser along a line segment at speed T. The challenge is to determine the optimal order of printing the segments and the starting endpoint for each segment to minimize the total time.

### Approach
1. **Problem Analysis**: The problem involves selecting an order to print N line segments and choosing a starting endpoint for each segment. The laser must move from the end of one segment to the start of the next without emitting, and the movement during segment printing is constrained to the segment's path. The goal is to minimize the total time, which includes both non-emitting movement time and segment printing time.

2. **Intuition**: The solution can be approached using dynamic programming (DP) with a bitmask to represent the state of printed segments. The DP state will track the last printed segment and the endpoint where the laser finished, allowing transitions to unprinted segments by considering all possible starting endpoints.

3. **Algorithm Selection**: We use a DP approach where:
   - `dp[mask][i][e]` represents the minimum time to have printed the segments specified by the bitmask `mask`, ending at endpoint `e` of segment `i`.
   - The mask is a bitmask indicating which segments have been printed.
   - For each segment, we consider both possible starting endpoints (choices), which determine the endpoint where the laser finishes after printing the segment.

4. **Initialization**: For each segment, initialize the DP state by moving from the origin (0, 0) to the chosen starting endpoint (without emitting) and then printing the segment (emitting).

5. **Transitions**: For each state, transition to unprinted segments by moving from the current endpoint to the chosen starting endpoint of the next segment (without emitting), then printing the segment (emitting).

6. **Complexity Analysis**: The algorithm has a time complexity of O(2^N * N^2 * 2), where N is the number of segments (up to 6). This is feasible because 2^6 = 64, and N^2 = 36, resulting in 64 * 36 * 2 = 4608 operations, which is manageable.
