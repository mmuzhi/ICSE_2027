=== ANSWER ===
To solve this problem, we need to determine if a given string of brackets can be reduced to an empty string by repeatedly removing contiguous pairs of matching brackets (i.e., "()", "[]", or "<>"). The solution involves simulating the removal process using a stack to keep track of the opening brackets and ensuring that each closing bracket matches the most recent unmatched opening bracket of the same type.

### Approach
1. **Problem Analysis**: The problem requires checking if the string can be entirely reduced by removing adjacent matching bracket pairs in any order. The key insight is that the process is analogous to validating balanced parentheses, where each closing bracket must correspond to the most recent unmatched opening bracket of the same type.
2. **Intuition**: Using a stack, we can efficiently simulate the removal process. For each character in the string:
   - If it's an opening bracket (i.e., '(', '[', '<'), push it onto the stack.
   - If it's a closing bracket (i.e., ')', ']', '>'), check if it matches the top of the stack. If it does, pop the stack; otherwise, the string is invalid.
3. **Algorithm Selection**: The stack-based approach is optimal because it processes each character exactly once, resulting in an O(n) time complexity, where n is the length of the string. The space complexity is O(n) in the worst case when all characters are opening brackets.
4. **Optimization**: Early termination occurs if a closing bracket is encountered without a matching opening bracket on the stack, immediately indicating the string is invalid.
5. **Complexity Analysis**: The algorithm runs in linear time and uses linear space, making it efficient for the given constraints (string length up to 200,000).

### Solution Code
```python
def main():
    s = input().strip()
    stack = []
    mapping = {')': '(', ']': '[', '>': '<'}
    
    for char in s:
        if char in '([<':
            stack.append(char)
        else:
            if not stack or stack[-1] != mapping[char]:
                print("No")
                return
            stack.pop()
    
    print("Yes" if not stack else "No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The `stack` is initialized to keep track of opening brackets. The `mapping` dictionary defines the corresponding opening bracket for each closing bracket.
2. **Processing Characters**: For each character in the string:
   - If the character is an opening bracket, it is pushed onto the stack.
   - If the character is a closing bracket, the stack is checked. If the stack is empty or the top does not match the corresponding opening bracket, the string is invalid, and "No" is printed immediately.
   - If there's a match, the top of the stack is popped, simulating the removal of the matching pair.
3. **Final Check**: After processing all characters, if the stack is empty, all brackets were matched and removed, so "Yes" is printed. Otherwise, "No" is printed due to unmatched opening brackets.

This approach efficiently checks the validity of the string by leveraging stack operations to simulate the removal of matching bracket pairs in linear time.
