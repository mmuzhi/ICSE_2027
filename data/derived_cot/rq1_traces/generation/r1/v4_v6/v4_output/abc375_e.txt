There are N people divided into three teams.
The people are numbered 1, 2, \ldots, N, and the teams are numbered 1, 2, 3. Currently, person i belongs to team A_i.
Each person has a value called strength; person i has a strength of B_i. The strength of a team is defined as the sum of the strengths of its members.
Determine whether it is possible for zero or more people to switch teams so that all teams have equal strength. If it is possible, find the minimum number of people who need to switch teams to achieve this.
You cannot create new teams other than teams 1, 2, 3.

Input

The input is given from Standard Input in the following format:
N
A_1 B_1
A_2 B_2
\vdots
A_N B_N

Output

If it is possible to make all teams have equal strength, print the minimum number of people who need to switch teams. Otherwise, print -1.

Constraints


- 3 \leq N \leq 100
- A_i \in \lbrace 1, 2, 3 \rbrace
- For each x \in \lbrace 1, 2, 3 \rbrace, there exists some i with A_i = x.
- 1 \leq B_i
- \displaystyle\sum_{i = 1}^{N} B_i \leq 1500 
- All input values are integers.

To solve this problem, we need to determine whether it's possible to make the total strength of all three teams equal by allowing zero or more people to switch teams. If possible, we also need to find the minimum number of switches required. 

### Approach
1. **Problem Analysis**: The problem involves redistributing people among three teams such that the sum of strengths of each team is equal. Each person can switch teams, and each switch is counted as one move. The goal is to minimize the number of switches.
2. **Key Insight**: The total strength of all people must be divisible by 3 for it to be possible to split into three equal parts. If not, the answer is immediately -1.
3. **Dynamic Programming (DP) Setup**: We use a DP approach where `dp[s1][s2]` represents the minimum number of switches required to achieve sums `s1` for team 1 and `s2` for team 2 (the sum for team 3 is derived as `total_strength - s1 - s2`).
4. **State Transitions**: For each person, we consider assigning them to one of the three teams. The cost increases by 1 if the person's final team differs from their initial team. We update the DP state accordingly, ensuring that the sums do not exceed the target value `T` (which is `total_strength / 3`).
5. **Final Check**: After processing all people, if the state `dp[T][T]` is reachable, the answer is the value stored there; otherwise, it's -1.

```python
    if total_sum % 3 != 0:

        return
        
    T = total_sum // 3
    INF = 10**9
    dp = [[INF] * (T + 1) for _ in range(T + 1)]
    dp[0][0] = 0
    prefix_sum = 0

    for idx, (a, b) in enumerate(persons):
        new_dp = [[INF] * (T + 1) for _ in range(T + 1)]
        prefix_sum += b
        pre = prefix_sum - b
        
        for s1 in range(T + 1):
            for s2 in range(T + 1):
                if dp[s1][s2] == INF:
                    continue
                s3_old = pre - s1 - s2
                if s3_old > T:
                    continue
                if s1 + b <= T:
                    ns1 = s1 + b
                    ns2 = s2
                    cost_inc = 0 if a == 1 else 1
                    new_val = dp[s1][s2] + cost_inc
                    if new_val < new_dp[ns1][ns2]:
                        new_dp[ns1][ns2] = new_val
                if s2 + b <= T:
                    ns1 = s1
                    ns2 = s2 + b
                    cost_inc = 0 if a == 2 else 1
                    new_val = dp[s1][s2] + cost_inc
                    if new_val < new_dp[ns1][ns2]:
                        new_dp[ns1][ns2] = new_val
                if prefix_sum - s1 - s2 <= T:
                    ns1 = s1
                    ns2 = s2
                    cost_inc = 0 if a == 3 else 1
                    new_val = dp[s1][s2] + cost_inc
                    if new_val < new_dp[ns1][ns2]:
                        new_dp[ns1][ns2] = new_val
        dp = new_dp

    if dp[T][T] == INF:
        print(-1)
    else:
        print(dp[T][T])

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into a list of tuples, each representing a person's initial team and strength.
2. **Total Sum Check**: The total strength of all people is checked for divisibility by 3. If not, the answer is -1.
3. **Dynamic Programming Initialization**: A 2D DP array is initialized to store the minimum switches required to achieve sums `s1` and `s2` for teams 1 and 2, respectively.
4. **Processing Each Person**: For each person, the DP array is updated by considering their assignment to each of the three teams. The cost increases if the final team differs from the initial team.
5. **State Updates**: The DP state is updated only if the resulting sums do not exceed the target `T` (one-third of the total strength).
6. **Result Extraction**: After processing all people, the value at `dp[T][T]` gives the answer if reachable; otherwise, -1 is printed.

This approach efficiently checks all possible distributions of people among teams while minimizing the number of switches using dynamic programming.
