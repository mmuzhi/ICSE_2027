You are given a positive integer N.
Consider an N \times N grid. Let (i,j) denote the cell at the i-th row from the top and the j-th column from the left. Initially, no cell is colored.
Then, for i = 1,2,\dots,N in order, perform the following operation:

- Let j = N + 1 - i.
- If i \leq j, fill the rectangular region whose top-left cell is (i,i) and bottom-right cell is (j,j) with black if i is odd, or white if i is even. If some cells are already colored, overwrite their colors.  
- If i > j, do nothing.

After all these operations, it can be proved that there are no uncolored cells. Determine the final color of each cell.

Input

The input is given from Standard Input in the following format:
N

Output

Print N lines. The i-th line should contain a length-N string S_i representing the colors of the i-th row of the grid after all operations, as follows:

- If cell (i,j) is finally colored black, the j-th character of S_i should be #.
- If cell (i,j) is finally colored white, the j-th character of S_i should be ..

Constraints


- 1 \leq N \leq 50
- All input values are integers.

To solve this problem, we need to generate an N x N grid where each cell's color is determined by the last operation that affected it. The operations are performed in sequence from i = 1 to i = N, where each operation i colors a square region from (i, i) to (N+1-i, N+1-i) in black if i is odd or white if i is even. The key insight is that the last operation to color a cell (r, c) is determined by the smallest distance from the cell to any of the grid's borders. 

### Approach
1. **Problem Analysis**: The operations are performed in order, with each subsequent operation overwriting the colors of the previous operations within its square. The last operation affecting a cell (r, c) is the one corresponding to the smallest distance from (r, c) to any of the grid's borders. This distance can be computed as the minimum of the row index (r), column index (c), distance from the bottom (N-1-r), and distance from the right (N-1-c) in a 0-indexed grid.
2. **Key Insight**: The layer index (i) for a cell (r, c) is derived from this minimum distance (d) as i = d + 1. The color of the cell is black ('#') if i is odd, and white ('.') if i is even.
3. **Algorithm**: For each cell in the grid, compute the minimum distance to any border, determine the layer index, and then decide the color based on whether the layer index is odd or even.
4. **Complexity Analysis**: The algorithm involves iterating over each cell in the N x N grid once, resulting in a time complexity of O(N^2), which is efficient given the constraint N ≤ 50.

### Solution Code
```python
def main():
    N = int(input().strip())
    for r in range(N):
        s = []
        for c in range(N):
            d = min(r, c, N - 1 - r, N - 1 - c)
            layer_index = d + 1
            if layer_index % 2 == 1:
                s.append('#')
            else:
                s.append('.')
        print(''.join(s))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The integer N is read from standard input, representing the grid size.
2. **Grid Construction**: For each row r (0 to N-1) and each column c (0 to N-1):
   - **Distance Calculation**: The minimum distance (d) from the cell (r, c) to any of the grid's borders is computed. This distance is the smallest value among r, c, (N-1-r), and (N-1-c).
   - **Layer Index**: The layer index (i) is derived as i = d + 1, indicating the last operation that colors the cell.
   - **Color Assignment**: The cell is colored '#' if the layer index is odd, otherwise '.'.
3. **Output**: Each constructed row is printed as a string, forming the final grid.

This approach efficiently determines the color of each cell by leveraging the minimum distance to the grid's borders and the parity of the corresponding layer index, ensuring the solution meets the problem requirements.
