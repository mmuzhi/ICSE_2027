There are N buildings arranged in a line at equal intervals. The height of the i-th building from the front is H_i.

=== ANSWER ===
To solve this problem, we need to find the maximum number of buildings that can be chosen such that all selected buildings have the same height and are arranged at equal intervals. The solution involves grouping buildings by their heights and then, for each group, finding the longest arithmetic progression (AP) in their indices. The length of the longest AP found across all groups is the answer.

### Approach
1. **Grouping by Height**: First, we group the indices of buildings by their heights. This allows us to process each group separately, ensuring that all buildings in a group have the same height.
2. **Finding Longest Arithmetic Progression**: For each group of indices (sorted in ascending order), we use dynamic programming to find the longest arithmetic progression. The idea is to iterate over all pairs of indices (i, j) where i < j, compute the common difference `d = arr[j] - arr[i]`, and then update the DP table where `dp[j][d] = dp[i].get(d, 1) + 1`. This means the chain ending at index `j` with step `d` is one longer than the chain ending at index `i` with the same step.
3. **Tracking Maximum Chain Length**: For each group, we keep track of the maximum chain length encountered during the DP process. The global answer is the maximum of these values across all groups, with a minimum value of 1 (since selecting a single building is always valid).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    n = int(data[0])
    H = list(map(int, data[1:1+n]))
    
    groups = {}
    for i, h in enumerate(H):
        if h not in groups:
            groups[h] = []
        groups[h].append(i)
    
    ans = 1
    for arr in groups.values():
        k = len(arr)
        if k <= 1:
            continue
        arr.sort()
        dp = [dict() for _ in range(k)]
        max_chain = 1
        for j in range(1, k):
            for i in range(j):
                d = arr[j] - arr[i]
                cur = dp[i].get(d, 1) + 1
                dp[j][d] = cur
                if cur > max_chain:
                    max_chain = cur
        if max_chain > ans:
            ans = max_chain
    print(ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Handling**: The input is read and parsed into the number of buildings `n` and their heights `H`.
2. **Grouping by Height**: Buildings are grouped by their heights into a dictionary where each key is a height, and the value is a list of indices (positions) of buildings with that height.
3. **Dynamic Programming Setup**: For each group of indices (sorted), we initialize a list of dictionaries `dp` where each dictionary at index `j` will store step sizes `d` and the corresponding lengths of arithmetic progressions ending at `j` with step `d`.
4. **Finding Longest AP**: For each pair of indices `(i, j)` where `i < j`, we compute the common difference `d = arr[j] - arr[i]`. The length of the AP ending at `j` with step `d` is set to one more than the length of the AP ending at `i` with the same step (or 2 if no such AP exists at `i`).
5. **Tracking Maximum**: The maximum chain length encountered during processing each group is updated, and the global maximum `ans` is printed at the end.

This approach efficiently checks all possible arithmetic progressions within each height group, leveraging dynamic programming to avoid redundant calculations, and ensures optimal performance even for the upper constraint limits.
