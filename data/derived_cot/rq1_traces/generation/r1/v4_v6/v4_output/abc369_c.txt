You will be given a competitive programming problem.
You are given a sequence of N positive integers A=(A_1,A_2,\dots,A_N).
Find the number of pairs of integers (l,r) satisfying 1\leq l\leq r\leq N such that the subsequence (A_l,A_{l+1},\dots,A_r) forms an arithmetic progression.
A sequence (x_1,x_2,\dots,x_{|x|}) is an arithmetic progression if and only if there exists a d such that x_{i+1}-x_i=d\ (1\leq i < |x|).
In particular, a sequence of length 1 is always an arithmetic progression.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \dots A_N

Output

Print the answer.

Constraints


- 1\leq N \leq 2\times 10^5
- 1\leq A_i \leq 10^9
- All input values are integers.

4
3 6 9 3

Sample Output 1

8

However, when (l,r)=(2,4), (A_l,\dots,A_r)=(6,9,3) is not an arithmetic progression, so it does not satisfy the condition.

To solve this problem, we need to count the number of contiguous subarrays in a given sequence of integers that form an arithmetic progression (AP). An AP is defined as a sequence where the difference between consecutive elements is constant. 

### Approach
1. **Problem Analysis**: The problem requires counting all contiguous subarrays (subsequences) of the given array that form an AP. A single-element subarray is always an AP. Any two consecutive elements also form an AP. For subarrays longer than two elements, the difference between each consecutive pair of elements must be the same.
2. **Intuition**: The key observation is that for any subarray of length greater than two, the condition for being an AP can be checked by verifying that all consecutive differences in the subarray are the same. 
3. **Algorithm Selection**: 
   - **Base Case**: For an array of size `n`, there are `n` single-element subarrays, all of which are APs.
   - **Handling Larger Subarrays**: For subarrays of length two or more, we can iterate through the array, maintaining a running count of the current sequence of consecutive elements where the differences are equal. For each such sequence of length `k`, there are `k` valid subarrays ending at the current element (from length 2 to length `k+1`).
4. **Complexity Analysis**: The algorithm processes each element in the array exactly once, leading to an O(n) time complexity, which is optimal for the given constraints (n ≤ 2×10^5).

### Solution Code
```python
def main():
    n = int(input().strip())
    A = list(map(int, input().split()))
    
    total_ans = n
    if n >= 2:
        prev_diff = A[1] - A[0]
        total_ans += 1
        cur_run = 1
        for i in range(2, n):
            cur_diff = A[i] - A[i-1]
            if cur_diff == prev_diff:
                cur_run += 1
            else:
                cur_run = 1
            total_ans += cur_run
            prev_diff = cur_diff
            
    print(total_ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The variable `total_ans` starts with `n` to account for all single-element subarrays.
2. **First Pair Handling**: For arrays with at least two elements, the difference between the first two elements is computed, and `total_ans` is incremented by 1 to account for the subarray consisting of these two elements.
3. **Iterating Through the Array**: For each subsequent element, the difference with the previous element is computed. If this difference matches the previous difference, the current run length (`cur_run`) is incremented; otherwise, it is reset to 1. The value of `cur_run` is added to `total_ans` in each iteration, representing the number of new valid AP subarrays ending at the current element.
4. **Output**: After processing all elements, the total count of valid AP subarrays is printed.

This approach efficiently counts all valid AP subarrays by leveraging the properties of arithmetic progressions and maintaining a running count of consecutive elements with equal differences. The solution is optimal with linear time complexity.
