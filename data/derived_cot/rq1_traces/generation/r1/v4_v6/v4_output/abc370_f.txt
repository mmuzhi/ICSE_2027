There is a circular cake divided into N pieces by cut lines. Each cut line is a line segment connecting the center of the circle to a point on the arc.
The pieces and cut lines are numbered 1, 2, \ldots, N in clockwise order, and piece i has a mass of A_i. Piece 1 is also called piece N + 1.
Cut line i is between pieces i and i + 1, and they are arranged clockwise in this order: piece 1, cut line 1, piece 2, cut line 2, \ldots, piece N, cut line N.
We want to divide this cake among K people under the following conditions. Let w_i be the sum of the masses of the pieces received by the i-th person.

- Each person receives one or more consecutive pieces.
- There are no pieces that no one receives.
- Under the above two conditions, \min(w_1, w_2, \ldots, w_K) is maximized.

Find the value of \min(w_1, w_2, \ldots, w_K) in a division that satisfies the conditions, and the number of cut lines that are never cut in the divisions that satisfy the conditions. Here, cut line i is considered cut if pieces i and i + 1 are given to different people.

Input

The input is given from Standard Input in the following format:
N K
A_1 A_2 \ldots A_N

Output

Let x be the value of \min(w_1, w_2, \ldots, w_K) in a division that satisfies the conditions, and y be the number of cut lines that are never cut. Print x and y in this order, separated by a space.

Constraints


- 2 \leq K \leq N \leq 2 \times 10^5
- 1 \leq A_i \leq 10^4
- All input values are integers.

Sample Input 1

3 6 8 6 4

Sample Output 1

The value of \min(w_1, w_2) in divisions satisfying the conditions is 13, and there is one cut line that is not cut in either division: cut line 5.

To solve this problem, we need to divide a circular cake into K consecutive segments such that the minimum sum of the masses of the pieces in any segment is maximized. Additionally, we need to count the number of cut lines that are never cut in any optimal division.

### Approach
1. **Binary Search for Maximum Minimum Segment Sum (x):** 
   - The solution involves using binary search to determine the maximum possible value of the minimum segment sum (x). For each candidate x, we check if it's feasible to divide the cake into K segments where each segment has a sum of at least x.
   - The feasibility check is performed by:
     - Doubling the array to handle the circular nature of the cake.
     - Using a two-pointer technique to compute the next segment start pointers (next_ptr) for each position in the doubled array. This helps in efficiently determining the segments that meet the sum condition.
     - Building a sparse table (binary lifting) to quickly compute the number of segments that can be formed starting from any position within the circular array.
     - For each starting position in the original array, we use the sparse table to count the maximum segments that can be formed without exceeding the circular boundary. If any starting position yields at least K segments, x is feasible.

2. **Counting Uncut Cut Lines (y):**
   - Once the optimal x is found, we reuse the next_ptr and sparse table to determine which cut lines are never cut in any optimal division.
   - For each cut line, we check if breaking the cake at that line allows forming at least K segments with each segment sum at least x. If not, the cut line is never cut in any optimal division, and we count it.

def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    k = int(data[1])
    A = list(map(int, data[2:2+n]))
    total = sum(A)
    doubled = A + A
    LOG = 20

    def feasible(x):
        if total < k * x:
            return False
        
        next_ptr = [-1] * (2 * n)
        s = 0
        right = 0
        for left in range(2 * n):
            if right < left:
                right = left
                s = 0
            while right < min(2 * n, left + n + 1) and s < x:
                s += doubled[right]
                right += 1
            if s >= x:
                next_ptr[left] = right
            else:
                next_ptr[left] = -1
            if left < 2 * n:
                s -= doubled[left]
                if s < 0:
                    s = 0

        dp_table = [[-1] * (2 * n) for _ in range(LOG + 1)]
        for i in range(2 * n):
            if next_ptr[i] != -1:
                dp_table[0][i] = next_ptr[i]
            else:
                dp_table[0][i] = -1

        for j in range(1, LOG + 1):
            for i in range(2 * n):
                if dp_table[j - 1][i] == -1:
                    dp_table[j][i] = -1
                else:
                    nxt = dp_table[j - 1][i]
                    if nxt < 2 * n and dp_table[j - 1][nxt] != -1:
                        dp_table[j][i] = dp_table[j - 1][nxt]
                    else:
                        dp_table[j][i] = -1

        for start in range(n):
            current = start
            count = 0
            for j in range(LOG, -1, -1):
                if dp_table[j][current] == -1:
                    continue
                nxt_index = dp_table[j][current]
                if nxt_index <= start + n:
                    count += (1 << j)
                    current = nxt_index
                    if current >= 2 * n:
                        break
            if count >= k:
                return True
        return False

    low, high = 0, total
    ans_x = 0
    while low <= high:
        mid = (low + high + 1) // 2
        if feasible(mid):
            ans_x = mid
            low = mid + 1
        else:
            high = mid - 1

    x = ans_x
    next_ptr = [-1] * (2 * n)
    s = 0
    right = 0
    for left in range(2 * n):
        if right < left:

        if left < 2 * n:

            if s < 0:

    dp_table = [[-1] * (2 * n) for _ in range(LOG + 1)]
    for i in range(2 * n):
        if next_ptr[i] != -1:
            dp_table[0][i] = next_ptr[i]
        else:
            dp_table[0][i] = -1

    for j in range(1, LOG + 1):
        for i in range(2 * n):
            if dp_table[j - 1][i] == -1:
                dp_table[j][i] = -1
            else:
                nxt = dp_table[j - 1][i]
                if nxt < 2 * n and dp_table[j - 1][nxt] != -1:
                    dp_table[j][i] = dp_table[j - 1][nxt]
                else:
                    dp_table[j][i] = -1

    y = 0
    for i in range(n):
        start_index = (i + 1) % n
        current = start_index
        count = 0
        for j in range(LOG, -1, -1):

                continue
            nxt_index = dp_table[j][current]
            if nxt_index <= start_index + n:
                count += (1 << j)
                current = nxt_index

                    break
        if count < k:
            y += 1

    print(x, y)

2. **Feasibility Check:** For each candidate `x` during binary search:
   - **Initial Check:** If the total sum of the cake is less than `K * x`, it's impossible to form `K` segments each with sum `x`, so we skip further checks.
   - **Two-Pointer Technique:** We use two pointers to compute the `next_ptr` array, which helps in determining the start of the next segment for any given starting position.
   - **Sparse Table Construction:** The sparse table (`dp_table`) is built using binary lifting to allow efficient jumping over segments during the feasibility check.
   - **Segment Counting:** For each starting position in the original array, we use the sparse table to count the maximum segments that can be formed within the circular boundary. If any starting position yields at least `K` segments, `x` is feasible.
3. **Optimal Division and Cut Line Counting:** After determining the optimal `x`, we reuse the `next_ptr` and sparse table to count how many cut lines are never cut in any optimal division. This is done by checking each cut line to see if breaking the cake at that line allows forming at least `K` segments with each segment sum at least `x`. If not, the cut line is counted as never cut.
