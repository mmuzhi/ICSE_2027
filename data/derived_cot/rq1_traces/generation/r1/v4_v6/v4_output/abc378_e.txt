To solve this problem, we need to compute the sum of the remainders when each contiguous subarray sum of a given sequence is divided by a positive integer \( M \). Given the constraints, a brute-force approach is infeasible, so we use a more efficient method involving prefix sums and inversion count with a Fenwick tree (Binary Indexed Tree).

### Approach
1. **Prefix Sums and Modulo Calculation**: Compute the prefix sums \( P \) of the sequence \( A \), where each \( P_i \) is the sum of the first \( i \) elements of \( A \). Then, compute the residues \( X_i = P_i \mod M \) for each prefix sum.
2. **Part 1 Calculation**: Calculate the first part of the answer by summing \( i \times X_i \) for each \( i \) from 1 to \( N \) and subtracting \( (N - i) \times X_i \) for each \( i \) from 0 to \( N-1 \). This leverages the linearity of the non-wrap-around part of the modulo operation.
3. **Inversion Count for Wrap-around Part**: The second part of the answer involves counting the number of pairs \( (i, j) \) where \( i < j \) and \( X_i > X_j \). Each such pair contributes \( M \) to the sum. This count is efficiently computed using a Fenwick tree to maintain and query residue frequencies.
4. **Combining Results**: The final answer is the sum of the first part and \( M \) multiplied by the inversion count.

### Solution Code
```python
import sys

class Fenw:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)

    def update(self, index, delta):
        while index <= self.n:
            self.tree[index] += delta
            index += index & -index

    def query(self, index):
        s = 0
        while index:
            s += self.tree[index]
            index -= index & -index
        return s

def main():
    data = sys.stdin.read().split()
    if not data: 
        return
    n = int(data[0])
    M_val = int(data[1])
    A = list(map(int, data[2:2+n]))
    P = [0] * (n+1)
    for i in range(1, n+1):
        P[i] = P[i-1] + A[i-1]
    X = [p % M_val for p in P]

    part1 = 0
    for i in range(1, n+1):
        part1 += i * X[i]
    for i in range(0, n):
        part1 -= (n - i) * X[i]

    fenw = Fenw(M_val)
    inv_count = 0
    for j in range(0, n+1):
        x = X[j]
        cnt_le = fenw.query(x+1)
        inv_count += (j - cnt_le)
        fenw.update(x+1, 1)

    ans = part1 + M_val * inv_count
    print(ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Prefix Sums and Residues**: The prefix sums \( P \) are computed to facilitate the calculation of subarray sums. The residues \( X_i \) (i.e., \( P_i \mod M \)) are stored for each prefix sum.
2. **Part 1 Calculation**: The first part of the answer is derived by summing each residue \( X_i \) multiplied by its position \( i \) (for \( i \) from 1 to \( N \)) and subtracting each residue \( X_i \) multiplied by \( (N - i) \) (for \( i \) from 0 to \( N-1 \)). This handles the non-wrap-around contributions of the modulo operation.
3. **Inversion Count**: Using a Fenwick tree, we efficiently count the number of inversion pairs \( (i, j) \) where \( i < j \) and \( X_i > X_j \). Each such pair indicates a wrap-around in the modulo operation, contributing \( M \) to the sum.
4. **Final Answer**: The solution combines the results from Part 1 and the inversion count multiplied by \( M \), providing the final answer efficiently with \( O(N \log M) \) complexity. The Fenwick tree ensures that the inversion count is computed in logarithmic time per operation.
