You will be given a competitive programming problem.
Takahashi has 10^{100} flower pots. Initially, he is not growing any plants.
You are given Q queries to process in order.
There are three types of queries as follows.

- 1: Prepare one empty flower pot and put a plant in it. Here, the plant's height is 0.
- 2 T: Wait for T days. During this time, the height of every existing plants increases by T.
- 3 H: Harvest all plants with a height of at least H, and output the number of plants harvested. The harvested plants are removed from their flower pots.

Assume that performing queries of the first and third types takes zero time.

Input

The input is given from Standard Input in the following format:
Q
\text{query}_1
\text{query}_2
\vdots
\text{query}_Q

Each query is given in one of the following formats:
1

2 T

3 H

Output

Let there be K queries of the third type, and print K lines.
The i-th line (1 \leq i \leq K) should contain the answer to the i-th query of type 3.

Constraints


- 1 \leq Q \leq 2 \times 10^{5}
- 1 \leq T,H \leq 10^{9}
- There is at least one query of the third type.
- All input values are integers.

1
3 10
2 20
3 20

Sample Output 1

- In the fourth query, all plants with height at least 10 are harvested. Here, one plant of height 15 gets harvested, and one plant of height 0 remains. Since one plant was harvested, print 1 on the first line.
- In the fifth query, the height of the remaining plant increases to 20.
- In the sixth query, all plants with height at least 20 are harvested. Here, one plant of height 20 gets harvested. Thus, print 1 on the second line.

To solve this problem, we need to efficiently process a series of queries involving planting, growing, and harvesting plants in flower pots. The key challenge is to handle the harvesting operation, which requires removing all plants that have reached a certain height and counting them, all while managing the growth of plants over time.

### Approach
1. **Problem Analysis**: The problem involves three types of queries:
   - **Type 1**: Plant a new plant with initial height 0.
   - **Type 2**: Wait for T days, during which all existing plants grow by T units.
   - **Type 3**: Harvest all plants with height at least H, remove them, and output the count of harvested plants.

2. **Key Insight**: The height of each plant can be tracked using the total time elapsed (global_add) since the plant was added. Specifically, the height of a plant added at time `t0` is `current_global_add - t0`.

3. **Data Structure Selection**: A min-heap is used to store the birth times (base values) of the plants. This allows efficient access to the smallest base value, which corresponds to the plant that has been growing the longest. During a harvest operation, all plants with base values `<= (global_add - H)` are removed from the heap, as their heights are at least H.

4. **Algorithm**:
   - **Planting (Type 1)**: Push the current global_add value into the min-heap.
   - **Growing (Type 2)**: Add T to global_add, simulating the passage of time.
   - **Harvesting (Type 3)**: Calculate the threshold `X = global_add - H`. Pop all plants from the heap with base values `<= X`, counting them, and output the count.

5. **Complexity Analysis**: 
   - **Insertion (Type 1)**: O(log n) per operation.
   - **Growing (Type 2)**: O(1) per operation.
   - **Harvesting (Type 3)**: O(k log n) per operation, where k is the number of plants harvested. Since each plant is harvested at most once, the total operations over all queries is O(n log n), which is efficient for the constraint `n <= 200,000`.

import heapq

def main():
    import sys
    data = sys.stdin.read().splitlines()
    Q = int(data[0].strip())
    global_add = 0
    heap = []
    output_lines = []
    index = 1
    for _ in range(Q):

        index += 1
        if parts[0] == '1':
            heapq.heappush(heap, global_add)
        elif parts[0] == '2':

            global_add += T
        else:

                cnt += 1

if __name__ == "__main__":
