To solve this problem, we need to determine whether it's possible to transform sequence A into sequence B using a series of operations where each operation allows us to set an element in A to the value of another element within a distance of K. The solution involves checking specific conditions based on the last occurrences of each value in B and their proximity to corresponding values in A.

### Approach
1. **Problem Analysis**: The operations allow us to copy a value from one element to another within a distance K. The challenge is to ensure that every element in B can be achieved in A through such operations without violating the distance constraint, especially for the last occurrence of each value in B.
2. **Key Insight**: For each value in B, the last occurrence of that value must have a corresponding value in A within a distance K. This ensures that during the transformation, the last occurrence can be set correctly without losing necessary source values.
3. **Algorithm Selection**: 
   - **Preprocessing**: For each test case, we first build a dictionary (`occ`) that maps each value in A to the list of indices where it appears. We then sort these lists for efficient binary search.
   - **Last Occurrence Tracking**: We create another dictionary (`last_occurrence`) that records the last occurrence index of each value in B.
   - **Validation Check**: For each value in `last_occurrence`, we check if it exists in `occ`. If not, the transformation is impossible. If it exists, we use binary search to find the closest occurrence in A to the last occurrence in B. If the closest distance exceeds K, the transformation is impossible.
4. **Complexity Analysis**: The preprocessing step takes O(N log N) per test case due to sorting. The validation step involves O(M log N) operations per test case, where M is the number of distinct values in B. Given the constraints, the total complexity is manageable.

```python
                min_dist = min(min_dist, abs(arr[pos-1] - L))
            if min_dist > k:
                valid = False
                break
                
        results.append("Yes" if valid else "No")
    
    for res in results:
        print(res)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints.
2. **Building Occurrence Dictionary (`occ`)**: For each value in A, we store the indices where it appears and sort these indices to facilitate binary search.
3. **Tracking Last Occurrences (`last_occurrence`)**: For each value in B, we record the last occurrence index.
4. **Validation**: For each value in `last_occurrence`, we check if it exists in `occ`. If not, the result is "No". If it exists, we find the closest occurrence in A to the last occurrence in B using binary search. If the closest distance exceeds K, the result is "No"; otherwise, it's "Yes".
5. **Output**: The results for all test cases are collected and printed at the end.

This approach efficiently checks the feasibility of transforming A into B by leveraging sorted indices and binary search to ensure optimal performance.
