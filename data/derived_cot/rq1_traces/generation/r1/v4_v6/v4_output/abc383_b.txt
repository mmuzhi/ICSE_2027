You will be given a competitive programming problem.
The AtCoder company office can be represented as a grid of H rows and W columns. Let (i, j) denote the cell at the i-th row from the top and j-th column from the left.
The state of each cell is represented by a character S_{i,j}. If S_{i,j} is #, that cell contains a desk; if S_{i,j} is ., that cell is a floor. It is guaranteed that there are at least two floor cells.
You will choose two distinct floor cells and place a humidifier on each.
After placing the humidifiers, a cell (i,j) is humidified if and only if it is within a Manhattan distance D from at least one of the humidifier cells (i',j'). The Manhattan distance between (i,j) and (i',j') is defined as |i - i'| + |j - j'|.
Note that any floor cell on which a humidifier is placed is always humidified.
Find the maximum possible number of humidified floor cells.

Input

The input is given from Standard Input in the following format:
H W D
S_{1,1}S_{1,2}\cdotsS_{1,W}
S_{2,1}S_{2,2}\cdotsS_{2,W}
\vdots
S_{H,1}S_{H,2}\cdotsS_{H,W}

Output

Print the answer.

Constraints


- 1 \leq H \leq 10
- 1 \leq W \leq 10
- 2 \leq H \times W
- 0 \leq D \leq H+W-2
- H,W,D are integers.
- S_{i,j} is # or .. (1 \leq i \leq H, 1 \leq j \leq W)
- There are at least two floor cells.

In total, three cells are humidified. No configuration can humidify four or more floor cells, so the answer is 3.

To solve this problem, we need to maximize the number of humidified floor cells in a grid by strategically placing two humidifiers on distinct floor cells. The humidifiers will humidify all floor cells within a Manhattan distance of `D` from either humidifier. 

### Approach
1. **Problem Analysis**: The grid consists of `H` rows and `W` columns, each cell being either a desk (`#`) or a floor (`.`). We need to place two humidifiers on distinct floor cells such that the number of humidified floor cells (those within Manhattan distance `D` from either humidifier) is maximized.
2. **Intuition**: For each possible pair of distinct floor cells, we calculate the number of floor cells that are within Manhattan distance `D` from at least one of the humidifiers. The maximum count across all such pairs is our answer.
3. **Algorithm Selection**: Given the constraints (H and W up to 10), a brute-force approach is feasible. We generate all possible pairs of distinct floor cells. For each pair, we iterate over all floor cells, checking if they are within the required Manhattan distance from either humidifier in the pair.
4. **Complexity Analysis**: The number of floor cells can be up to 100 (since H*W ≤ 100). The number of pairs is O(N^2), where N is the number of floor cells (up to 4950 for N=100). For each pair, we check all floor cells (up to 100), leading to a total of 495,000 operations, which is computationally feasible.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    H, W, D = map(int, data[0].split())
    grid = []
    for i in range(1, 1+H):
        grid.append(data[i].strip())
    
    floor_cells = []
    for i in range(H):
        for j in range(W):
            if grid[i][j] == '.':
                floor_cells.append((i, j))
    
    n = len(floor_cells)
    max_count = 0
    for i in range(n):
        for j in range(i+1, n):
            a = floor_cells[i]
            b = floor_cells[j]
            count = 0
            for cell in floor_cells:
                d1 = abs(cell[0] - a[0]) + abs(cell[1] - a[1])
                d2 = abs(cell[0] - b[0]) + abs(cell[1] - b[1])
                if d1 <= D or d2 <= D:
                    count += 1
            if count > max_count:
                max_count = count
                
    print(max_count)

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input is read from standard input, where the first line contains `H`, `W`, and `D`, followed by `H` lines representing the grid.
2. **Floor Cells Identification**: We iterate through each cell in the grid, collecting coordinates of all floor cells (`.`) into a list `floor_cells`.
3. **Brute-Force Pair Evaluation**: For each distinct pair of floor cells (considered as potential humidifier placements), we count how many floor cells are within Manhattan distance `D` from either humidifier in the pair. This is done by checking each floor cell against both humidifiers in the pair.
4. **Maximization**: The maximum count of humidified floor cells across all evaluated pairs is printed as the result.

This approach efficiently checks all possible placements of the two humidifiers and determines the optimal solution by leveraging the manageable constraints.
