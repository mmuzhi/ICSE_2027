=== PROMPT ===
You will be given a competitive programming problem.
You are given a string source of size n, a string pattern that is a subsequence of source, and a sorted integer array targetIndices that contains distinct numbers in the range [0, n - 1].
We define an operation as removing a character at an index idx from source such that:

idx is an element of targetIndices.
pattern remains a subsequence of source after removing the character.

Performing an operation does not change the indices of the other characters in source. For example, if you remove 'c' from "acb", the character at index 2 would still be 'b'.
Return the maximum number of operations that can be performed.
 
Example 1:

Input: source = "abbaa", pattern = "aba", targetIndices = [0,1,2]
Output: 1
Explanation:
We can't remove source[0] but we can do either of these two operations:

Remove source[1], so that source becomes "a_baa".
Remove source[2], so that source becomes "ab_aa".


Example 2:

Input: source = "bcda", pattern = "d", targetIndices = [0,3]
Output: 2
Explanation:
We can remove source[0] and source[3] in two operations.

Example 3:

Input: source = "dda", pattern = "dda", targetIndices = [0,1,2]
Output: 0
Explanation:
We can't remove any character from source.

Example 4:

Input: source = "yeyeykyded", pattern = "yeyyd", targetIndices = [0,2,3,4]
Output: 2
Explanation:
We can remove source[2] and source[3] in two operations.

 
Constraints:

1 <= n == source.length <= 3 * 10^3
1 <= pattern.length <= n
1 <= targetIndices.length <= n
targetIndices is sorted in ascending order.
The input is generated such that targetIndices contains distinct elements in the range [0, n - 1].
source and pattern consist only of lowercase English letters.
The input is generated such that pattern appears as a subsequence in source.

The challenge is to efficiently compute this without checking all possible subsets, which would be computationally expensive.

2. **Key Insight**: The solution involves finding a matching of the pattern in the source string that minimizes the number of indices used from `targetIndices`. The maximum number of operations (removals) is then given by the total number of indices in `targetIndices` minus the number of indices from `targetIndices` used in this optimal matching.

3. **Dynamic Programming**: We use dynamic programming to track the minimal cost (number of indices from `targetIndices` used) to match the pattern up to each character. The DP array `dp` is initialized with infinity except for `dp[0]` which is 0. For each character in the source string, we update the DP array backwards (from the last character of the pattern to the first) to ensure we use the correct state from previous indices.

4. **Complexity Analysis**: The algorithm runs in O(n * m) time, where n is the length of the source string and m is the length of the pattern. This is efficient given the constraints (n and m are up to 3000).

### Solution Code
```python
class Solution:
    def maxRemovals(self, source: str, pattern: str, targetIndices: List[int]) -> int:
        m = len(pattern)
        n = len(source)
        dp = [float('inf')] * (m + 1)
        dp[0] = 0
        targetSet = set(targetIndices)
        for i in range(n):
            for j in range(m - 1, -1, -1):
                if source[i] == pattern[j] and dp[j] != float('inf'):
                    cost = 1 if i in targetSet else 0
                    if dp[j + 1] > dp[j] + cost:
                        dp[j + 1] = dp[j] + cost
        return len(targetIndices) - dp[m]
```

### Explanation
1. **Initialization**: The DP array `dp` is initialized to infinity, except for `dp[0]` which is set to 0. This array will store the minimal number of indices from `targetIndices` required to match the first `j` characters of the pattern.
2. **Iterate Over Source**: For each character in the source string, we check if it matches any character in the pattern. We iterate backwards through the pattern to avoid overwriting the DP state needed for subsequent updates.
3. **Update DP State**: When a match is found, we update the DP state for the next character in the pattern by adding the cost (1 if the current index is in `targetIndices`, 0 otherwise) to the previous state's value.
4. **Result Calculation**: The result is derived by subtracting the minimal cost (stored in `dp[m]`) from the total number of indices in `targetIndices`, giving the maximum number of removable indices while preserving the pattern as a subsequence.

This approach efficiently computes the solution by leveraging dynamic programming to minimize the overlap between the pattern matching indices and the target removal indices, thus maximizing the number of valid operations.
