=== ANSWER ===
To solve this problem, we need to simulate the process of building a string through a series of operations where each operation involves appending a transformed version of the current string. The transformation involves shifting each character in the string to its next character in the alphabet (with 'z' wrapping around to 'a'). The goal is to determine the k-th character in the resulting string after performing enough operations such that the string's length is at least k.

### Approach
1. **Initialization**: Start with the initial string "a".
2. **Operation Simulation**: While the length of the current string is less than k:
   - **Transformation**: For each character in the current string, compute its next character in the alphabet. If the character is 'z', it wraps around to 'a'.
   - **Appending**: Append the transformed string to the current string.
3. **Result Extraction**: Once the string's length is at least k, return the k-th character (1-based index).

The key insight is that each operation doubles the length of the string, which means the number of operations required to reach a length of at least k is logarithmic in k. For k up to 500, the maximum operations needed is 9 (since \(2^9 = 512\)), making the approach efficient.

### Solution Code
```python
class Solution:
    def kthCharacter(self, k: int) -> str:
        word = "a"
        while len(word) < k:
            next_str = []
            for c in word:
                next_char = chr((ord(c) - ord('a') + 1) % 26 + ord('a'))
                next_str.append(next_char)
            word += ''.join(next_str)
        return word[k-1]
```

### Explanation
- **Initialization**: The initial string is set to "a".
- **Loop Until String Length ≥ k**: The loop continues as long as the current string length is less than k. Inside the loop:
  - **Transformation**: Each character in the current string is processed to generate its next character. For example, 'a' becomes 'b', 'b' becomes 'c', and 'z' becomes 'a' (handled via modulo arithmetic).
  - **Appending**: The transformed string is appended to the current string, effectively doubling its length each iteration.
- **Return Result**: After exiting the loop, the k-th character (1-based index) is accessed at position `k-1` in the resulting string and returned.

This approach efficiently builds the string through iterative operations, leveraging the exponential growth in string length to minimize the number of operations needed, making it suitable for the given constraints.
