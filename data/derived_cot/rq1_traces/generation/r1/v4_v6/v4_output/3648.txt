Each child has specific movement rules, and they must make exactly n-1 moves. The goal is to maximize the sum of fruits collected, considering that if multiple children enter the same room, the fruits are counted only once.
2. **Key Insight**: The solution requires exploring all possible paths each child can take while adhering to their movement constraints. The paths must be explored simultaneously to account for overlaps where children might collect the same fruits.
3. **Dynamic Programming with Memoization**: Given the constraints (n ≤ 1000), a brute-force approach is infeasible. Instead, we use a memoized depth-first search (DFS) to explore the state space. The state is defined by the current positions of all three children. For each state, we compute the distinct fruits collected in that state (handling overlaps) and recursively explore all valid next moves.
4. **State Handling**: The state transitions involve moving each child according to their respective movement rules. The memoization stores the maximum fruits collectible from each state to avoid redundant computations.
5. **Final State Check**: The recursion terminates when all children reach the bottom-right corner (n-1, n-1) after exactly n-1 moves. Any state that doesn't meet this condition is considered invalid.

from functools import lru_cache

class Solution:
    def maxCollectedFruits(self, fruits: List[List[int]]) -> int:
        n = len(fruits)
        
        moves1 = [(0, 1), (1, 0), (1, 1)]
        moves2 = [-1, 0, 1]
        moves3 = [-1, 0, 1]
        
        def in_bounds(i, j):
            return 0 <= i < n and 0 <= j < n
        
        @lru_cache(maxsize=None)
        def dfs(i1, j1, i2, j2, i3, j3):
            if i2 == n-1 and j3 == n-1:
                if i1 == n-1 and j1 == n-1 and j2 == n-1 and i3 == n-1:
                    rooms = {(i1, j1), (i2, j2), (i3, j3)}
                    return sum(fruits[i][j] for i, j in rooms)
                else:
                    return -10**18
            rooms = {(i1, j1), (i2, j2), (i3, j3)}
            current_sum = sum(fruits[i][j] for i, j in rooms)
            best_next = -10**18
            for di, dj in moves1:
                ni1, nj1 = i1 + di, j1 + dj
                if not in_bounds(ni1, nj1):
                    continue
                for dj2_delta in moves2:
                    new_j2 = j2 + dj2_delta
                    if not in_bounds(i2 + 1, new_j2):
                        continue
                    for di3_delta in moves3:
                        new_i3 = i3 + di3_delta
                        if not in_bounds(new_i3, j3 + 1):
                            continue
                        next_val = dfs(ni1, nj1, i2 + 1, new_j2, new_i3, j3 + 1)
                        if next_val > best_next:
                            best_next = next_val
            return current_sum + best_next if best_next != -10**18 else -10**18
        
        return dfs(0, 0, 0, n-1, n-1, 0)

1. **Initialization**: The solution starts by defining the movement rules for each child. Child 1 can move right, down, or diagonally; Child 2 can move down and adjust left/right by one column; Child 3 can move right and adjust up/down by one row.
2. **Bounds Check**: A helper function checks if a position is within the grid boundaries.
3. **Memoized DFS**: The core of the solution is a memoized DFS function that explores all possible moves from the current state (positions of all children). The memoization (using `lru_cache`) stores results of previously computed states to optimize performance.
4. **State Handling**: For each state, the function calculates the distinct fruits collected in that state (handling overlaps by using a set). It then recursively explores all valid next moves, updating the children's positions according to their movement rules.
5. **Termination Condition**: The recursion terminates when all children reach the bottom-right corner (n-1, n-1). If they arrive at the correct position, the fruits collected in that state are returned; otherwise, a large negative value signifies an invalid state.
6. **Result Calculation**: The initial call to `dfs` starts with the children's initial positions, and the result is the maximum fruits collected over all valid paths.

This approach efficiently explores the state space while leveraging memoization to avoid redundant calculations, ensuring optimal performance for moderate grid sizes.
