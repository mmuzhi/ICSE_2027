This is a DAG in the grid from (W,H) to (0,0) in reverse. We can iterate from (W,H) down to (0,0). 

But the grid is 10^6 x 10^6, which is 10^12 states, not feasible.

We need a combinatorial approach.

Let me consider the following: 

The valid region is the complement of a rectangle. 

The region has a "hole" in the middle. 

The generating function for the number of paths might be factored into the parts: 

  - The left part: x < L
  - The bottom part: y < D and x in [L,R] 
  - The top part: y > U and x in [L,R] 
  - The right part: x > R

And the combinations of these.

But the path can weave between these regions. 

For example, a path might start in the left part, then go up above U (and then can go to the right part) or go up not above U and then enter the bottom part (x in [L,R] and y<D) and then go to the right part by going below the hole.

Actually, the only thing that is not allowed is to be in the hole. 

The valid region is therefore: 
  Region 1: x < L, any y
  Region 2: L<=x<=R and y < D
  Region 3: L<=x<=R and y > U
  Region 4: x > R, any y

And also, the Region 2 and Region 3 are separate.

The moves are only right and up. 

A path can be in Region 1, then move right and enter Region 2 (if y<D) or Region 1 (if x+1<L) or even Region 3 if y+1>U (but that's not possible from Region 1 because if you move up in Region 1, you stay in Region 1 until you move right into Region 2 or Region 3 or Region 4).

Similarly, from Region 2, you can move right and stay in Region 2 if x+1 <= R, or move to Region 4 if x+1 > R. Or move up and if y+1 < D, stay in Region 2, if y+1 >= D and y+1 <= U, then that's the hole, not allowed, or if y+1 > U, then move to Region 3.

But if in Region 2, you move up and y+1 becomes >= D and <= U, then the block (x, y+1) is in the hole, which is not allowed. So you cannot move up from Region 2 if y+1 would be in [D,U]. 

Similarly, from Region 3, you cannot move down if y-1 is in [D,U] (but you only move up or right, so you will not move down).

From Region 3, if you move up, you stay in Region 3. If you move right, if x+1 <= R, you stay in Region 3, if x+1 > R, you go to Region 4.

From Region 4, you can only move up or right within Region 4.

The challenge is that between Region 1 and Region 2, and Region 2 and Region 4, and Region 3 and Region 4, there are interactions. 

We can try to write generating functions for each region and then for the transitions between regions. 

Let F1(x,y) for x<L, any y.
Let F2(x,y) for L<=x<=R and y<D.
Let F3(x,y) for L<=x<=R and y>U.
Let F4(x,y) for x>R, any y.

For F1(x,y) = 1 + F1(x+1,y) + F1(x,y+1)   if x+1 < L, then it's F1, if x+1 >= L and x+1 <= R and y < D, then it's F2(x+1,y), if x+1 > R, then F4(x+1,y). For F1(x,y+1) = if y+1 is such that (x,y+1) is still in Region 1, then F1(x,y+1), but if y+1 >= D and y+1 <= U and x in [L,R]? But x<L, so (x,y+1) is still in Region 1 (because x<L). So from F1, moving up always stays in F1. Only when moving right might leave to F2 or F4.

Similarly for F2(x,y): 
  Move right: if x+1 <= R, then if y<D, then F2(x+1,y) if x+1 is in [L,R], else if x+1 > R, then F4(x+1,y). 
  Move up: 
      if y+1 < D, then F2(x,y+1)
      if y+1 >= D and y+1 <= U, then not allowed.
      if y+1 > U, then F3(x,y+1)

But the move is only one step: from (x,y) to (x,y+1). If y+1 is in [D,U], then (x,y+1) is in the hole, which is not a block. Therefore, if y+1 is in [D,U], then the move is not allowed. Only if y+1 > U, then the block (x,y+1) is in Region 3.

Similarly for F3(x,y):
  Move right: if x+1 <= R, then F3(x+1,y), else if x+1>R, then F4(x+1,y).
  Move up: F3(x,y+1)  [because if y+1 > U, then it's still in Region 3]

  Move right: F4(x+1,y)
  Move up: F4(x,y+1)

And for all, if the move would take you to a coordinate beyond W or H, then not allowed.

The blocks are defined as all lattice points (x, y) that satisfy 0 ≤ x ≤ W, 0 ≤ y ≤ H, and (x < L or R < x or y < D or U < y). The solution involves combinatorial mathematics and efficient computation using precomputed factorials and modular arithmetic.

### Approach
1. **Problem Analysis**: The problem requires counting all possible paths that start at any allowed block and move only right or up, ending at any allowed block, without passing through any forbidden blocks (the rectangle defined by [L, R] × [D, U]). The constraints are large (up to 10^6), so a brute-force approach is infeasible.
2. **Key Insight**: The solution leverages combinatorial mathematics to count valid paths. The total number of paths from (x1, y1) to (x2, y2) without any restrictions is given by the binomial coefficient C((x2 - x1) + (y2 - y1), (x2 - x1)). However, we must subtract paths that pass through the forbidden rectangle.
3. **Inclusion-Exclusion Principle**: To handle the forbidden rectangle, we use inclusion-exclusion by subtracting paths that enter the forbidden rectangle from the left or bottom. Specifically:
   - **A_total**: Paths that enter the forbidden rectangle from the left wall (x = L).
   - **B_total**: Paths that enter the forbidden rectangle from the bottom wall (y = D).
4. **Efficient Computation**: Precompute factorials and inverse factorials up to 3,000,000 to compute binomial coefficients in constant time. def precompute_factorials(n, mod=MOD):
    fact = [1] * (n+1)
    inv_fact = [1] * (n+1)
    for i in range(2, n+1):
        fact[i] = fact[i-1] * i % mod
    inv_fact[n] = pow(fact[n], mod-2, mod)
    for i in range(n, 0, -1):
        inv_fact[i-1] = inv_fact[i] * i % mod
    return fact, inv_fact
