=== ANSWER ===
To solve this problem, we need to determine whether a given sequence of integers contains at least one instance where the same element appears three or more times consecutively. 

### Approach
1. **Input Reading**: Read the integer `N` which represents the length of the sequence. Then, read the sequence of `N` integers.
2. **Iterate through the Sequence**: Loop through the sequence from the first element to the third-to-last element (i.e., indices 0 to N-3). For each element at index `i`, check if it is equal to the next two elements (i.e., elements at indices `i+1` and `i+2`).
3. **Check for Consecutive Triplets**: If during the iteration, any triplet of consecutive elements is found to be equal, immediately print "Yes" and exit the loop.
4. **Result if No Triplets Found**: If the loop completes without finding any such triplet, print "No".

This approach efficiently checks each possible triplet in the sequence in linear time, making it optimal for the given constraints (N ≤ 100).

### Solution Code
```python
def main():
    n = int(input().strip())
    arr = list(map(int, input().split()))
    
    found = False
    for i in range(n - 2):
        if arr[i] == arr[i+1] == arr[i+2]:
            found = True
            break
            
    print("Yes" if found else "No")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The first line of input is read as an integer `N`, the length of the sequence. The second line is read as a list of integers.
2. **Checking for Triplets**: The loop runs from the start of the list up to the third-to-last element. For each position `i`, it checks if the current element and the next two elements are equal.
3. **Early Termination**: If a triplet of equal elements is found, the loop breaks early, and "Yes" is printed. If no such triplet is found after checking all possible positions, "No" is printed.
4. **Efficiency**: The algorithm runs in O(N) time since each element is checked at most once (specifically, each element is part of up to three triplet checks), which is efficient given the problem constraints. The space complexity is O(1) as no additional space is used beyond the input array.
