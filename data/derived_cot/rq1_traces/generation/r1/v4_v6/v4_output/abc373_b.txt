=== ANSWER ===
To solve this problem, we need to calculate the minimal total distance traveled by a finger when typing all the letters from 'A' to 'Z' in order on a keyboard where the keys are arranged in a specific permutation. The finger starts at the key corresponding to 'A' and must move sequentially to each subsequent key in the alphabet order, pressing each key exactly once.

### Approach
1. **Mapping Key Positions**: The input string `S` represents the permutation of the 26 letters. Each character in `S` is mapped to its 1-based coordinate (position in the string). For example, the first character in `S` is at coordinate 1, the second at coordinate 2, and so on.
2. **Initial Position**: The finger starts at the coordinate of the letter 'A'.
3. **Sequential Movement**: For each subsequent letter from 'B' to 'Z', the finger moves from its current position to the position of the next letter. The distance moved is the absolute difference between the current and next positions.
4. **Summing Distances**: The total distance traveled is the sum of all individual distances moved between consecutive letters in the sequence from 'A' to 'Z'.

### Solution Code
```python
def main():
    s = input().strip()
    pos = {}
    for idx, char in enumerate(s):
        pos[char] = idx + 1
    
    current = pos['A']
    total = 0
    for char in "BCDEFGHIJKLMNOPQRSTUVWXYZ":
        next_pos = pos[char]
        total += abs(current - next_pos)
        current = next_pos
    
    print(total)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Mapping Creation**: The code first reads the input string `S` and creates a dictionary `pos` where each character in `S` is mapped to its 1-based position (index in the string plus one).
2. **Initial Setup**: The starting position is set to the coordinate of 'A' (found in the `pos` dictionary).
3. **Distance Calculation**: The loop iterates over each letter from 'B' to 'Z'. For each letter, the next position is retrieved from the `pos` dictionary. The distance between the current position and the next position is calculated as the absolute difference and added to the total distance. The current position is then updated to the next position.
4. **Output**: After processing all letters, the total distance traveled is printed.

This approach efficiently calculates the minimal distance by leveraging the fixed sequence of letters to be typed and the predefined key positions, ensuring optimal performance with a linear pass through the alphabet sequence.
