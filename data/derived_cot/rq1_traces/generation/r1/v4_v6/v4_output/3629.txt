To solve this problem, we need to determine the length of a string after performing exactly `t` transformations according to specific rules. Each transformation involves replacing every character in the string based on the following rules:
- If the character is 'z', it is replaced with the string "ab".
- Otherwise, the character is replaced with the next character in the alphabet (e.g., 'a' becomes 'b', 'b' becomes 'c', etc.).

The challenge is to compute the length of the resulting string after `t` transformations efficiently, especially since the result can be very large, requiring modulo operations (10^9 + 7).

### Approach
1. **Problem Analysis**: Each character in the string transforms independently. The transformation of a character depends on its current value and the number of remaining transformations. For non-'z' characters, each transformation simply shifts the character to the next alphabet (length remains 1). For 'z', it splits into two characters 'a' and 'b', which then undergo further transformations.
2. **Dynamic Programming (DP) Setup**: We use a DP array where `dp[i]` represents the length after transformations starting from the character corresponding to index `i` (0 for 'a', 25 for 'z'). Initialize `dp` with 1s since 0 transformations yield the original character (length 1).
3. **Transformation Simulation**: For each transformation step from 1 to `t`:
   - For characters 'a' to 'y' (indices 0-24), the next state is the next character (i+1) with one less transformation (hence, `dp[i] = dp[i+1]` from the previous step).
   - For 'z' (index 25), it splits into 'a' and 'b', so the length is the sum of their lengths after one less transformation (`dp[25] = (dp[0] + dp[1]) % mod`).
4. **Result Calculation**: After processing all transformations, sum the lengths contributed by each character in the original string, using the precomputed DP array.

### Solution Code
```python
class Solution:
    def lengthAfterTransformations(self, s: str, t: int) -> int:
        mod = 10**9 + 7
        dp = [1] * 26
        for _ in range(1, t + 1):
            new_dp = [0] * 26
            new_dp[:25] = dp[1:]
            new_dp[25] = (dp[0] + dp[1]) % mod
            dp = new_dp
        total = 0
        for char in s:
            idx = ord(char) - ord('a')
            total = (total + dp[idx]) % mod
        return total
```

### Explanation
1. **Initialization**: The `dp` array is initialized to 1 for all 26 characters, representing the length after 0 transformations (each character is 1 unit long).
2. **Transformation Loop**: For each transformation step (from 1 to `t`):

   - **'z' (index 25)**: The length is the sum of the lengths of 'a' and 'b' after `t-1` transformations, modulo 10^9 + 7.
3. **Result Calculation**: After processing all transformations, iterate over each character in the original string, use the precomputed `dp` array to get its contribution to the total length, and sum these contributions modulo 10^9 + 7.
