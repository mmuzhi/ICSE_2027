=== ANSWER ===
To solve this problem, we need to determine whether three given integers can be divided into two or more groups such that the sums of each group are equal. The solution involves checking specific conditions based on the values of the integers and their total sum.

### Approach
1. **Check for Equal Values**: If all three integers are equal, we can split them into three separate groups, each containing one integer. The sum of each group will naturally be the same as the integers are identical. Hence, the answer is "Yes".
2. **Check Total Sum Parity**: If the integers are not all equal, we then check if the total sum of the three integers is even. If the total sum is odd, it's impossible to split the integers into groups with equal sums, so the answer is "No".
3. **Check for Half Sum**: If the total sum is even, we then check if any single integer equals half of the total sum. If so, the remaining two integers must sum to the other half (which they will, as their sum is total sum minus the half sum, which equals the other half). Thus, we can form two groups: one with the single integer and the other with the remaining two integers. The answer is "Yes". If no such integer exists, the answer is "No".

### Solution Code
```python
def main():
    A, B, C = map(int, input().split())
    if A == B == C:
        print("Yes")
    else:
        total = A + B + C
        if total % 2 != 0:
            print("No")
        else:
            half = total // 2
            if A == half or B == half or C == half:
                print("Yes")
            else:
                print("No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Handling**: The code reads three integers from the standard input.
2. **Equal Values Check**: If all three integers are the same, it prints "Yes" because each can form a separate group with the same sum.
3. **Total Sum Check**: If the integers are not all equal, the code calculates their total sum. If this sum is odd, it prints "No" because an odd total cannot be split into two equal parts.
4. **Half Sum Check**: If the total sum is even, the code checks if any of the integers equals half of the total sum. If so, it prints "Yes" because the remaining two integers will sum to the other half, allowing two groups with equal sums. If no such integer exists, it prints "No".

This approach efficiently checks all possible scenarios where the integers can be divided into groups with equal sums, ensuring optimal performance with constant time complexity O(1).
