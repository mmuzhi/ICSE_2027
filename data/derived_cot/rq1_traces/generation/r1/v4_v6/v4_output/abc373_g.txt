You will be given a competitive programming problem.
There are 2N points P_1,P_2,\ldots,P_N, Q_1,Q_2,\ldots,Q_N on a two-dimensional plane.
The coordinates of P_i are (A_i, B_i), and the coordinates of Q_i are (C_i, D_i).
No three different points lie on the same straight line.
Determine whether there exists a permutation R = (R_1, R_2, \ldots, R_N) of (1, 2, \ldots, N) that satisfies the following condition. If such an R exists, find one.

- For each integer i from 1 through N, let segment i be the line segment connecting P_i and Q_{R_i}.  Then, segment i and segment j (1 \leq  i < j \leq N) never intersect.

Input

The input is given from Standard Input in the following format:
N
A_1 B_1
A_2 B_2
\vdots 
A_N B_N
C_1 D_1
C_2 D_2
\vdots
C_N D_N

Output

If there is no R satisfying the condition, print -1.
If such an R exists, print R_1, R_2, \ldots, R_N separated by spaces. If there are multiple solutions, you may print any of them.

Constraints


- 1 \leq N \leq 300
- 0 \leq A_i, B_i, C_i, D_i \leq 5000 (1 \leq i \leq N)
- (A_i, B_i) \neq (A_j, B_j) (1 \leq i < j \leq N)
- (C_i, D_i) \neq (C_j, D_j) (1 \leq i < j \leq N)
- (A_i, B_i) \neq (C_j, D_j) (1 \leq i, j \leq N)
- No three different points lie on the same straight line.

=== ANSWER ===
To solve this problem, we need to find a permutation of Q points such that when each P point is connected to its corresponding Q point, none of the resulting line segments intersect. The solution involves a recursive approach that ensures non-crossing segments by leveraging geometric properties and divide-and-conquer strategies.

### Approach
1. **Problem Analysis**: Given 2N points (N P points and N Q points) in a 2D plane, we need to pair each P point with a Q point such that the line segments connecting them do not intersect. The key insight is that a non-crossing matching can be achieved by ensuring that each segment divides the remaining points into two balanced subsets (equal number of P and Q points on each side of the segment).

2. **Algorithm Selection**: The solution uses a recursive divide-and-conquer approach:
   - **Identify the Leftmost Point**: The leftmost point (smallest x, then y) is selected as a reference point. If it's a P point, it must be paired with a Q point, and vice versa.
   - **Check Valid Candidates**: For each candidate point of the opposite type, verify if the line segment from the reference to the candidate splits the remaining points into two subsets with equal numbers of P and Q points in the left half-plane.
   - **Recursive Partitioning**: Once a valid candidate is found, partition the remaining points into left and right subsets relative to the line segment. Recursively solve each subset.
   - **Combine Results**: The solution combines the current pair with the results from the left and right subsets.

3. **Complexity Analysis**: The algorithm runs in O(N^3) time in the worst case. This is because, at each recursion level, we process up to N points, and the depth of recursion is N. The space complexity is O(N) due to the recursion stack.

### Solution Code
```python
import sys
sys.setrecursionlimit(10000)

def main():
    data = sys.stdin.read().splitlines()
    n = int(data[0])
    points_p = []
    for i in range(1, n + 1):
        a, b = map(int, data[i].split())
        points_p.append((a, b, 'P', i))
    points_q = []
    for i in range(n + 1, 2 * n + 1):
        c, d = map(int, data[i].split())
        points_q.append((c, d, 'Q', i - n))
    all_points = points_p + points_q

    def solve_matching(points):
        if not points:
            return []
        points_sorted = sorted(points, key=lambda x: (x[0], x[1]))
        ref = points_sorted[0]
        rest = points_sorted[1:]
        opp_type = 'Q' if ref[2] == 'P' else 'P'
        candidates = [pt for pt in rest if pt[2] == opp_type]
        
        for cand in candidates:
            left_p = 0
            left_q = 0
            for pt in rest:
                if pt == cand:
                    continue
                v1x = cand[0] - ref[0]
                v1y = cand[1] - ref[1]
                v2x = pt[0] - ref[0]
                v2y = pt[1] - ref[1]
                cross = v1x * v2y - v1y * v2x
                if cross > 0:
                    if pt[2] == 'P':
                        left_p += 1
                    else:
                        left_q += 1
            if left_p == left_q:
                left_list = []
                right_list = []
                for pt in rest:
                    if pt == cand:
                        continue
                    v1x = cand[0] - ref[0]
                    v1y = cand[1] - ref[1]
                    v2x = pt[0] - ref[0]
                    v2y = pt[1] - ref[1]
                    cross = v1x * v2y - v1y * v2x
                    if cross > 0:
                        left_list.append(pt)
                    else:
                        right_list.append(pt)
                left_match = solve_matching(left_list)
                if left_match is None:
                    continue
                right_match = solve_matching(right_list)
                if right_match is None:
                    continue
                if ref[2] == 'P':
                    cur_pair = (ref[3], cand[3])
                else:
                    cur_pair = (cand[3], ref[3])
                return [cur_pair] + left_match + right_match
        return None

    result = solve_matching(all_points)
    if result is None:
        print(-1)
    else:
        R = [0] * (n + 1)
        for p_id, q_id in result:
            R[p_id] = q_id
        print(' '.join(str(R[i]) for i in range(1, n + 1)))

if __name__ == "__main__":
    main()
```
