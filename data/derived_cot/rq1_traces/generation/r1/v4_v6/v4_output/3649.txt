To solve this problem, we need to determine the minimum time required for Bob to break all locks in a dungeon. The locks can be broken in any order, and each lock requires a certain amount of energy. The energy of Bob's sword starts at 0 and increases by a factor X each minute. After breaking a lock, the sword's energy resets to 0, and the factor X increases by a given value K. The goal is to minimize the total time taken to break all locks.

### Approach
1. **Problem Analysis**: The problem involves breaking `n` locks in any order. Each lock `i` requires `strength[i]` energy. The sword's energy starts at 0, and its factor X starts at 1. Each minute, the energy increases by the current factor X. To break a lock, the energy must be at least the lock's strength. After breaking a lock, the energy resets to 0, and X increases by K. The challenge is to find the optimal order of breaking locks to minimize total time.

2. **Key Insight**: The time to break a lock `i` when it is the `j-th` lock broken (0-based) is given by `ceil(strength[i] / (1 + j*K))`. The total time is the sum of these times for all locks. The optimal order assigns locks to positions (0 to `n-1`) such that the sum of `ceil(strength[i] / (1 + j*K))` is minimized.

3. **Algorithm Selection**: Given the small constraint `n <= 8`, we use dynamic programming with a bitmask to represent subsets of broken locks. The state `dp[mask]` stores the minimum total time to break the locks in the subset represented by `mask`. For each state, we consider breaking one additional lock, calculating the time based on the current number of locks broken (which determines the factor X).

4. **Complexity Analysis**: The algorithm processes all `2^n` subsets of locks. For each subset, it processes each of the `n` locks not yet broken. Thus, the time complexity is `O(2^n * n)`, which is feasible since `n` is at most 8 (resulting in 256 subsets and 2048 operations).

### Solution Code
```python
class Solution:
    def findMinimumTime(self, strength: List[int], K: int) -> int:
        n = len(strength)
        dp = [10**18] * (1 << n)
        dp[0] = 0
        for mask in range(1 << n):
            j = bin(mask).count("1")
            for i in range(n):
                if mask & (1 << i) == 0:
                    factor = 1 + j * K
                    t = (strength[i] - 1) // factor + 1
                    new_mask = mask | (1 << i)
                    if dp[mask] + t < dp[new_mask]:
                        dp[new_mask] = dp[mask] + t
        return dp[(1 << n) - 1]
```

### Explanation
1. **Initialization**: The DP array `dp` is initialized with a large value (10^18) to represent an initially high time cost for all states, except `dp[0]` (no locks broken) which is set to 0.
2. **Processing Subsets**: For each subset of locks represented by `mask`, the current number of broken locks `j` is counted. For each unbroken lock `i`, the time `t` to break it is calculated as `ceil(strength[i] / (1 + j*K))` using integer arithmetic to avoid floating-point operations.
3. **State Transition**: The new state `new_mask` (including lock `i`) is updated if the total time from the current state plus `t` is better than the stored time for `new_mask`.
4. **Result Extraction**: The result is found in `dp[(1 << n) - 1]`, which represents the minimum time to break all locks (all bits set in the mask).

This approach efficiently explores all possible orders of breaking locks using dynamic programming and bitmask techniques, ensuring optimal performance given the problem constraints.
