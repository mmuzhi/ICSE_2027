You will be given a competitive programming problem.
There is a grid of N^2 squares with N rows and N columns.
Let (i,j) denote the square at the i-th row from the top (1\leq i\leq N) and j-th column from the left (1\leq j\leq N).
Each square is either empty or has a piece placed on it.
There are M pieces placed on the grid, and the k-th (1\leq k\leq M) piece is placed on square (a_k,b_k).
You want to place your piece on an empty square in such a way that it cannot be captured by any of the existing pieces.
A piece placed on square (i,j) can capture pieces that satisfy any of the following conditions:

- Placed on square (i+2,j+1)
- Placed on square (i+1,j+2)
- Placed on square (i-1,j+2)
- Placed on square (i-2,j+1)
- Placed on square (i-2,j-1)
- Placed on square (i-1,j-2)
- Placed on square (i+1,j-2)
- Placed on square (i+2,j-1)

Here, conditions involving non-existent squares are considered to never be satisfied.
For example, a piece placed on square (4,4) can capture pieces placed on the squares shown in blue in the following figure:

How many squares can you place your piece on?

Input

The input is given from Standard Input in the following format:
N M
a_1 b_1
a_2 b_2
\vdots
a_M b_M

Output

Print the number of empty squares where you can place your piece without it being captured by any existing pieces.

Constraints


- 1\leq N\leq10^9
- 1\leq M\leq2\times10^5
- 1\leq a_k\leq N,1\leq b_k\leq N\ (1\leq k\leq M)
- (a_k,b_k)\neq(a_l,b_l)\ (1\leq k\lt l\leq M)
- All input values are integers.

=== ANSWER ===
To solve this problem, we need to determine the number of empty squares in an N x N grid where we can place a piece such that it is not captured by any existing pieces. The existing pieces capture other pieces using knight moves (L-shaped moves in chess). 

### Approach
1. **Problem Analysis**: The grid has N^2 squares, with M pieces already placed. Each existing piece can capture pieces in 8 specific directions (knight moves). We need to find all empty squares that are not threatened by any existing piece. 
2. **Intuition**: For each existing piece, compute the 8 squares it threatens. Collect these threatened squares if they are within the grid and not occupied. The solution involves:
   - Marking existing pieces in a set for O(1) lookups.
   - For each existing piece, calculate the 8 threatened squares.
   - Store these threatened squares in a set to avoid duplicates.
   - The answer is then computed as: `Total squares (N^2) - Existing pieces (M) - Threatened empty squares (size of threatened set)`.
3. **Complexity Analysis**: The algorithm processes each of the M pieces exactly once, and for each piece, it checks 8 moves. Thus, the time complexity is O(M), which is efficient given M can be up to 200,000. The space complexity is O(M) for storing existing pieces and O(M) for threatened squares (since each piece can threaten up to 8 unique squares).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    
    n = int(data[0])
    m = int(data[1])
    existing_set = set()
    index = 2
    for _ in range(m):
        a = int(data[index])
        b = int(data[index + 1])
        index += 2
        existing_set.add((a, b))
    
    moves = [(2, 1), (1, 2), (-1, 2), (-2, 1), (-2, -1), (-1, -2), (1, -2), (2, -1)]
    threatened_set = set()
    
    for a, b in existing_set:
        for dx, dy in moves:
            x = a + dx
            y = b + dy
            if 1 <= x <= n and 1 <= y <= n:
                if (x, y) not in existing_set:
                    threatened_set.add((x, y))
    
    total_squares = n * n
    safe_squares = total_squares - m - len(threatened_set)
    print(safe_squares)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, starting with grid size N and number of existing pieces M.
2. **Storing Existing Pieces**: The coordinates of existing pieces are stored in a set (`existing_set`) for efficient access.
3. **Calculating Threatened Squares**: For each existing piece, the 8 possible knight moves are computed. Each resulting coordinate is checked to ensure it lies within the grid boundaries (1 to N) and is not occupied by an existing piece. Valid threatened coordinates are stored in a set (`threatened_set`) to eliminate duplicates.
4. **Computing Safe Squares**: The total number of safe squares is calculated as the difference between the total grid squares (N^2), the number of existing pieces (M), and the number of unique threatened squares (`len(threatened_set)`).
5. **Output**: The result is printed, representing the number of safe squares where a new piece can be placed without being captured by any existing pieces. 

This approach efficiently handles the constraints by leveraging set operations for deduplication and O(1) lookups, ensuring optimal performance even for large grid sizes.
