To solve this problem, we need to select `k` points from a given set of points lying on the boundary of a square such that the minimum Manhattan distance between any two selected points is maximized. The Manhattan distance between two points is defined as the sum of the absolute differences of their coordinates. 

### Approach
1. **Linear Coordinate Transformation**: Convert each point on the square's boundary into a linear coordinate by "unrolling" the square's perimeter into a straight line. This transformation helps in simplifying the distance calculation between points on the perimeter. The linear coordinate is calculated based on which edge the point lies:
   - **Bottom edge (y = 0)**: Linear coordinate is `x`.
   - **Right edge (x = side)**: Linear coordinate is `side + y`.
   - **Top edge (y = side)**: Linear coordinate is `3 * side - x`.
   - **Left edge (x = 0)**: Linear coordinate is `4 * side - y`.

2. **Binary Search**: Perform a binary search over the possible Manhattan distances (from 0 to `2 * side`) to find the maximum distance `D` such that we can select `k` points where the minimum Manhattan distance between any two points is at least `D`.

3. **Feasibility Check**: For a given distance `D`, check if it's possible to select `k` points such that:
   - The clockwise distance between consecutive points (in the sorted linear coordinate order) is at least `D` and at most `4 * side - D`.
   - The last point's clockwise distance to the starting point is also within `[D, 4 * side - D]`.
   - This ensures that the Manhattan distance (minimum of clockwise and counter-clockwise distances) between any two consecutive points (including the last and first) is at least `D`.

4. **Greedy Selection**: For each starting point in the sorted list of linear coordinates, attempt to greedily select `k` points by picking the next point that lies within the valid range `[current + D, current + 4 * side - D]`. If we can select `k` points satisfying all conditions, then `D` is feasible.

### Solution Code
```python
import bisect

class Solution:
    def maxDistance(self, side: int, points: list[list[int]], k: int) -> int:
        n = len(points)
        linear = []
        L = 4 * side
        for x, y in points:
            if y == 0:
                linear.append(x)
            elif x == side:
                linear.append(side + y)
            elif y == side:
                linear.append(3 * side - x)
            elif x == 0:
                linear.append(4 * side - y)
        
        linear = sorted(linear)
        arr2 = linear + [x + L for x in linear]
        
        def check(D):
            if k * D > L:
                return False
            for i in range(len(linear)):
                count = 1
                current = arr2[i]
                idx = i
                for j in range(1, k):
                    next_min = current + D
                    next_max = current + L - D
                    pos = bisect.bisect_left(arr2, next_min, lo=idx+1, hi=i + n)
                    if pos >= i + n or arr2[pos] > next_max:
                        break
                    count += 1
                    current = arr2[pos]
                    idx = pos
                if count < k:
                    continue
                last_gap = arr2[i] + L - current
                if D <= last_gap <= L - D:
                    return True
            return False
        
        low, high = 0, 2 * side
        while low < high:
            mid = (low + high + 1) // 2
            if check(mid):
                low = mid
            else:
                high = mid - 1
        return low
```

### Explanation
1. **Linear Coordinate Transformation**: Each point on the square's boundary is mapped to a linear coordinate by traversing the perimeter clockwise. This transformation simplifies the distance calculation between points by converting the 2D problem into a 1D problem.
2. **Binary Search**: The binary search is used to efficiently determine the maximum possible minimum Manhattan distance `D` between any `k` selected points. The search space ranges from 0 to `2 * side` (the maximum possible Manhattan distance on a square of side length `side`).
3. **Feasibility Check**: For each candidate distance `D` during the binary search, the algorithm checks if selecting `k` points is feasible. This involves:
   - Ensuring the clockwise distance between consecutive points (in the sorted linear coordinate list) is within `[D, 4 * side - D]`.
   - Checking the clockwise distance from the last selected point back to the starting point is also within `[D, 4 * side - D]`.
4. **Greedy Selection**: Starting from each point in the sorted list, the algorithm attempts to select `k` points by picking the next valid point within the specified range. If successful, `D` is feasible; otherwise, the algorithm continues checking other starting points.

This approach efficiently narrows down the maximum possible minimum Manhattan distance using binary search and ensures the solution meets the problem constraints through careful linear coordinate transformation and greedy selection.
