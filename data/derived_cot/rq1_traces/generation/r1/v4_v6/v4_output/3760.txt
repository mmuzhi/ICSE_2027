You will be given a competitive programming problem.
You are given an integer array groups, where groups[i] represents the size of the i^th group. You are also given an integer array elements.
Your task is to assign one element to each group based on the following rules:

An element at index j can be assigned to a group i if groups[i] is divisible by elements[j].
If there are multiple elements that can be assigned, assign the element with the smallest index j.
If no element satisfies the condition for a group, assign -1 to that group.

Return an integer array assigned, where assigned[i] is the index of the element chosen for group i, or -1 if no suitable element exists.
Note: An element may be assigned to more than one group.
 
Example 1:

Input: groups = [8,4,3,2,4], elements = [4,2]
Output: [0,0,-1,1,0]
Explanation:

elements[0] = 4 is assigned to groups 0, 1, and 4.
elements[1] = 2 is assigned to group 3.
Group 2 cannot be assigned any element.


Example 2:

Input: groups = [2,3,5,7], elements = [5,3,3]
Output: [-1,1,0,-1]
Explanation:

elements[1] = 3 is assigned to group 1.
elements[0] = 5 is assigned to group 2.
Groups 0 and 3 cannot be assigned any element.


Example 3:

Input: groups = [10,21,30,41], elements = [2,1]
Output: [0,1,0,1]
Explanation:
elements[0] = 2 is assigned to the groups with even values, and elements[1] = 1 is assigned to the groups with odd values.

 
Constraints:

1 <= groups.length <= 10^5
1 <= elements.length <= 10^5

=== ANSWER ===
To solve this problem, we need to assign elements from the `elements` array to each group in the `groups` array based on specific rules. The key rule is that an element at index `j` can be assigned to a group `i` if the group size `groups[i]` is divisible by the element value `elements[j]`. If multiple elements satisfy this condition for a group, we choose the element with the smallest index `j`. If no such element exists, we assign `-1` to that group.

### Approach
1. **Precompute Divisors**: For efficient processing, we precompute all divisors for every number up to the maximum value present in the `groups` array (which is 100,000 as per constraints). This helps in quickly accessing all divisors of any group size during the assignment phase.
2. **Store Minimum Indices**: We create a dictionary (`min_index_dict`) that maps each unique element value in the `elements` array to the smallest index at which it appears. This ensures that when multiple elements with the same value exist, we always consider the smallest index.
3. **Assign Elements to Groups**: For each group size in the `groups` array, we retrieve its divisors. For each divisor, if it exists in `min_index_dict`, we consider the corresponding element index as a candidate. Among all such candidates, we select the smallest index. If no candidates are found, we assign `-1` to that group.

### Solution Code
```python
class Solution:
    def assignElements(self, groups: List[int], elements: List[int]) -> List[int]:
        max_val = 100000
        divisors = [[] for _ in range(max_val + 1)]
        for i in range(1, max_val + 1):
            for j in range(i, max_val + 1, i):
                divisors[j].append(i)
        
        min_index_dict = {}
        for idx, e in enumerate(elements):
            if e not in min_index_dict:
                min_index_dict[e] = idx
        
        res = []
        for g in groups:
            best_index = None
            for d in divisors[g]:
                if d in min_index_dict:
                    candidate = min_index_dict[d]
                    if best_index is None or candidate < best_index:
                        best_index = candidate
            res.append(-1 if best_index is None else best_index)
        return res
```

### Explanation
1. **Precompute Divisors**: The code first initializes a list `divisors` where each entry `divisors[i]` contains all divisors of `i`. This is done by iterating through numbers from 1 to 100,000 and marking multiples of each number as divisors.
2. **Store Minimum Indices**: The code then processes the `elements` array to create a dictionary `min_index_dict` where each key is an element value, and the value is the smallest index at which this element appears in the `elements` array.
3. **Assign Elements to Groups**: For each group size `g` in the `groups` array, the code retrieves all divisors of `g` from the precomputed `divisors` list. For each divisor, if it exists in `min_index_dict`, the corresponding element index is considered a candidate. The smallest index among all candidates is selected for the group. If no candidates exist, `-1` is assigned.

This approach efficiently handles the constraints by leveraging precomputation and dictionary lookups, ensuring optimal performance even for large input sizes.
