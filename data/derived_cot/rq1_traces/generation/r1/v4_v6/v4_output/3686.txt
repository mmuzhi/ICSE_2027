To solve this problem, we need to count the number of ways to split an array into three non-empty contiguous subarrays such that either the first subarray is a prefix of the second subarray or the second subarray is a prefix of the third subarray. 

### Approach
1. **Problem Analysis**: The problem requires splitting the array into three parts (nums1, nums2, and nums3) such that either nums1 is a prefix of nums2 or nums2 is a prefix of nums3. The challenge is to efficiently check these conditions for all possible splits without resorting to a brute-force O(n^3) solution.
2. **Intuition**: We can precompute a Longest Common Prefix (LCP) table to store the length of the longest common prefix between any two starting indices in the array. This table allows us to quickly check the prefix conditions in constant time during the main computation.
3. **Algorithm Selection**: 
   - **Precompute LCP Table**: Using dynamic programming, we fill a 2D array `lcp` where `lcp[i][j]` represents the length of the longest common prefix between the subarrays starting at indices `i` and `j`.
   - **Check Splits**: For each possible split (i, j) where `i` is the end of nums1 and `j` is the end of nums2, we check:
     - **Condition 1**: If the length of nums2 is at least the length of nums1 and the common prefix between the start of the array and position `i` is at least the length of nums1.
     - **Condition 2**: If the length of nums3 is at least the length of nums2 and the common prefix between positions `i` and `j` is at least the length of nums2.
4. **Complexity Analysis**: 
   - **Precomputation**: The LCP table is filled in O(n^2) time and space, where `n` is the length of the array.
   - **Main Loop**: Checking all possible splits (i, j) takes O(n^2) time in the worst case.
   - **Overall**: The solution efficiently handles the constraints with O(n^2) time and space complexity, which is feasible given the problem constraints (n ≤ 5000).

### Solution Code
```python
class Solution:
    def beautifulSplits(self, nums):
        n = len(nums)
        if n < 3:
            return 0
        
        lcp = [[0] * (n + 1) for _ in range(n + 1)]
        
        for i in range(n - 1, -1, -1):
            for j in range(n - 1, -1, -1):
                if nums[i] == nums[j]:
                    lcp[i][j] = lcp[i + 1][j + 1] + 1
                else:
                    lcp[i][j] = 0
        
        count = 0
        for i in range(1, n - 1):
            for j in range(i + 1, n):
                len1 = i
                len2 = j - i
                len3 = n - j
                if len2 >= len1 and lcp[0][i] >= len1:
                    count += 1
                else:
                    if len3 >= len2 and lcp[i][j] >= len2:
                        count += 1
        return count
```

### Explanation
1. **Initialization**: The solution starts by checking if the array length is less than 3, in which case no valid splits exist, returning 0.
2. **LCP Table Construction**: The `lcp` table is constructed using dynamic programming. For each pair of indices (i, j), if the elements at these indices match, the value of `lcp[i][j]` is set to `lcp[i + 1][j + 1] + 1`; otherwise, it remains 0. This table helps in determining the longest common prefix between any two subarrays starting at i and j.
3. **Counting Valid Splits**: For each possible split (i, j):
   - **Condition 1 Check**: Verifies if nums2 is long enough to contain nums1 as a prefix by comparing the common prefix length from the start of the array to position `i` with the length of nums1.
   - **Condition 2 Check**: If Condition 1 fails, checks if nums3 is long enough to contain nums2 as a prefix by comparing the common prefix length between positions `i` and `j` with the length of nums2.
4. **Result**: The count of valid splits is returned, which meets the problem's requirements efficiently using precomputation and constant-time checks during the main loop.
