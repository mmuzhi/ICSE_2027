To solve this problem, we need to determine for each time step whether smoke exists at a specific cell (R, C) after a series of wind movements and potential smoke generations. The campfire at (0, 0) generates smoke initially, and at each subsequent time step, existing smoke moves according to wind directions, and new smoke is generated at (0, 0) if it's empty after the movement.

### Approach
1. **Problem Analysis**: The problem involves tracking the movement of smoke particles over time steps. At each time step, all existing smoke moves in a specified direction (N, W, S, E), and if (0, 0) is empty after this movement, new smoke is generated there. We need to check for each time step if the cell (R, C) contains smoke after these operations.

2. **Key Insight**: The smoke at (R, C) at any time step t+0.5 can originate from two sources:
   - **Initial Smoke**: The smoke generated at time 0, which moves over t steps to a position determined by cumulative wind displacements.
   - **New Smoke**: Smoke generated at some step i (1 ≤ i ≤ t) if after the wind movement at step i, (0, 0) was empty. This smoke then moves from (0, 0) over the remaining (t - i) steps.

3. **Algorithm Selection**:
   - **Prefix Sum Array (A)**: Compute the cumulative displacement of smoke from the initial position (0, 0) after each wind direction in the sequence. This helps track positions of the initial smoke over time.
   - **Generation Tracking (gen)**: For each step i, determine if new smoke was generated by checking if the source cell (required to leave (0, 0) after the wind movement) was empty at the previous time step.
   - **Efficient Lookup (min_index)**: Use a dictionary to store the earliest step i where each coordinate (from the prefix sum) is reached with valid generation (gen[i] is true).

4. **Complexity Analysis**: 
   - **Time Complexity**: O(N) for computing prefix sums, generation conditions, and checking conditions for each time step.
   - **Space Complexity**: O(N) for storing prefix sums, generation flags, and the dictionary for minimum indices.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.readline().split()
    N = int(data[0])
    R = int(data[1])
    C = int(data[2])
    S = sys.stdin.readline().strip()
    
    A = [(0, 0)]
    for d in S:
        r, c = A[-1]
        if d == 'N':
            A.append((r - 1, c))
        elif d == 'S':
            A.append((r + 1, c))
        elif d == 'W':
            A.append((r, c - 1))
        elif d == 'E':
            A.append((r, c + 1))
    
    gen = [False] * (N + 1)
    set_of_A = set()
    set_of_A.add(A[0])
    
    for i in range(1, N + 1):
        d = S[i - 1]
        if d == 'N':
            Q_i = (1, 0)
        elif d == 'W':
            Q_i = (0, 1)
        elif d == 'S':
            Q_i = (-1, 0)
        elif d == 'E':
            Q_i = (0, -1)
        target = (A[i - 1][0] - Q_i[0], A[i - 1][1] - Q_i[1])
        if target in set_of_A:
            gen[i] = False
        else:
            gen[i] = True
        set_of_A.add(A[i - 1])
    
    min_index = {}
    for i in range(1, N + 1):
        if gen[i]:
            coord = A[i]
            if coord not in min_index:
                min_index[coord] = i
            else:
                min_index[coord] = min(min_index[coord], i)
    
    res = []
    for t in range(1, N + 1):
        if A[t] == (R, C):
            res.append('1')
        else:
            target_coord = (A[t][0] - R, A[t][1] - C)
            if target_coord in min_index and min_index[target_coord] <= t:
                res.append('1')
            else:
                res.append('0')
    
    print(''.join(res))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: Read input values for N (number of steps), R and C (target cell coordinates), and S (sequence of wind directions).
2. **Prefix Sum Array (A)**: Compute cumulative displacements starting from (0, 0) for each wind direction in S. This array helps track the position of the initial smoke after each step.
3. **Generation Tracking (gen)**: For each step i, check if the source cell (determined by the wind direction) was empty at the previous step. If empty, mark gen[i] as true, indicating new smoke was generated.
4. **Minimum Index Dictionary (min_index)**: Store the earliest step i where each coordinate (from A) is reached with valid generation (gen[i] is true).
5. **Result Construction**: For each time step t, check if either:
   - The initial smoke is at (R, C) (A[t] == (R, C)).
   - There exists a step i ≤ t where smoke generated at i reaches (R, C) after t steps (A[t] - (R, C) is in min_index with i ≤ t).
6. **Output**: Construct a binary string indicating smoke presence at (R, C) for each step and print it.

This approach efficiently tracks smoke movements and generations using prefix sums and a dictionary for quick lookups, ensuring optimal performance for large inputs.
