To solve this problem, we need to find the minimum possible value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a subset \(S\) of size \(K\) from given sequences \(A\) and \(B\). 

### Approach
1. **Problem Analysis**: The problem requires selecting a subset \(S\) of \(K\) elements such that the product of the maximum \(A_i\) in \(S\) and the sum of \(B_i\) in \(S\) is minimized. The challenge is to efficiently explore all possible subsets \(S\) without resorting to brute-force methods, which would be computationally infeasible for large inputs.

2. **Key Insight**: The solution leverages the observation that the maximum \(A_i\) in any subset \(S\) must be one of the elements in \(A\). For each candidate \(A_i\) considered as the maximum, we can restrict our selection to elements with \(A_j \leq A_i\). The optimal subset \(S\) for this candidate will consist of the smallest \(K\) elements in \(B\) from this restricted set, ensuring the sum of \(B_i\) is minimized. However, if the smallest \(K\) elements do not include any element from the group of \(A_i\) (which is necessary to set the maximum to \(A_i\)), we adjust the selection by including the smallest \(B_i\) from this group and replacing the largest element in the current selection.

3. **Algorithm Selection**: 
   - **Sorting and Grouping**: First, we sort the elements by \(A\) and group them by distinct \(A\) values. For each group (processed in increasing order of \(A\)), we consider its \(A\) value as a candidate for the maximum in \(S\).
   - **Fenwick Trees for Efficient Queries**: We use Fenwick Trees (Binary Indexed Trees) to maintain cumulative counts and sums of \(B\) values. This allows efficient computation of the \(K\)-th smallest \(B\) value and the sum of the smallest \(K\) \(B\) values in logarithmic time.
   - **Handling Edge Cases**: For each candidate group, if the smallest \(B\) in the group is not among the smallest \(K\) \(B\) values, we adjust the selection by including this smallest \(B\) and replacing the largest \(B\) in the current selection, or by taking additional elements from the group if necessary.

4. **Complexity Analysis**: 
   - **Time Complexity**: The solution processes each test case in \(O(N \log N)\) time due to sorting, grouping, and Fenwick Tree operations. Given \(T\) test cases and the sum of \(N\) across test cases being \(O(N)\), the overall complexity is \(O(N \log N)\).
   - **Space Complexity**: The space complexity is \(O(N)\) for storing the Fenwick Trees and other auxiliary data structures.

    def __init__(self, size):
        self.n = size
        self.tree = [0] * (size + 1)
    
    def update(self, i, delta):
        while i <= self.n:
            self.tree[i] += delta
            i += i & -i

        s = 0
        while i:
            s += self.tree[i]
            i -= i & -i

            val = rev_mapping[idx]
            need = k - cnt_so_far
            total_sum = sum_so_far + val * need
            return val, total_sum
        
        def total_sum():
            return sum_fenw.query(size_comp)
        
        prev_count = 0

        for g in range(len(groups)):
            a_val, blist = groups[g]
            n_current = len(blist)
            total_count = prev_count + n_current
            if total_count < K:
                for b_val in blist:
                    i = mapping[b_val]
                    cnt_fenw.update(i, 1)
                    sum_fenw.update(i, b_val)
