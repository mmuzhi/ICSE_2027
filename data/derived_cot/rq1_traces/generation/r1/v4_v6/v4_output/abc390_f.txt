For each integer pair (L,R) with 1 \le L \le R \le N, define f(L,R) as follows:

- Start with an empty blackboard. Write the R-L+1 integers A_L, A_{L+1}, \ldots, A_R on the blackboard in order.
- Repeat the following operation until all integers on the blackboard are erased:
- Choose integers l, r with l \le r such that every integer from l through r appears at least once on the blackboard. Then, erase all integers from l through r that are on the blackboard.


- Let f(L,R) be the minimum number of such operations needed to erase all the integers from the blackboard.

Find \displaystyle \sum_{L=1}^N \sum_{R=L}^N f(L,R).

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the answer.

- 1 \le N \le 3 \times 10^5
- 1 \le A_i \le N
- All input values are integers.

Sample Output 1

16

- Choose (l,r)=(3,4) and erase all occurrences of 3 and 4. The blackboard becomes empty.
- It cannot be done in fewer than two operations, so f(1,4) = 2.

To solve this problem, we need to compute the sum of the function \( f(L, R) \) over all possible subarrays \( A[L..R] \) of a given integer sequence \( A \). The function \( f(L, R) \) represents the minimum number of operations required to erase all integers from the blackboard where each operation erases all occurrences of a contiguous range of integers that appear at least once in the current blackboard.

### Approach
1. **Understanding \( f(L, R) \)**:
   - For a subarray \( A[L..R] \), \( f(L, R) \) is the number of connected components in the distinct integers of the subarray when sorted. For example, if the distinct integers are \( \{1, 3, 4\} \), they form two connected components: \( [1] \) and \( [3, 4] \), so \( f(L, R) = 2 \).
   - The key insight is that \( f(L, R) \) can be expressed as the number of distinct integers minus the number of adjacent value pairs (i.e., pairs \( (x, x+1) \)) that both appear in the subarray. This is because each adjacent pair reduces the number of connected components by one.

2. **Summing \( f(L, R) \) over all subarrays**:
   - The total sum \( \sum_{L,R} f(L, R) \) can be decomposed into two parts:
     - **Part 1**: The sum of the number of distinct integers in each subarray. This is a classic problem solved by iterating through the array and calculating the contribution of each element to all subarrays where it is the last occurrence of its value.
     - **Part 2**: The sum over all adjacent value pairs \( (x, x+1) \) of the number of subarrays where both \( x \) and \( x+1 \) appear. This is computed using inclusion-exclusion: for each pair, the count is \( \text{count}(x) + \text{count}(x+1) - \text{count}(x \text{ or } x+1) \), where \( \text{count}(x) \) is the number of subarrays containing \( x \).

3. **Efficiency**:
   - **Part 1** is computed in \( O(N) \) time by traversing the array and maintaining the last occurrence of each value.
   - **Part 2** involves merging sorted lists of positions for each value and its successor. The total operations across all pairs are \( O(N) \) because each element is processed at most twice (once for \( (x-1, x) \) and once for \( (x, x+1) \)).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    positions = [[] for _ in range(n+1)]
    for i in range(n):
        a = A[i]
        if 1 <= a <= n:
            positions[a].append(i)
    
    part1 = 0
    last_occurrence = [-1] * (n+1)
    for i in range(n):
        a = A[i]
        gap = i - last_occurrence[a]
        part1 += gap * (n - i)
        last_occurrence[a] = i
        
    cont = [0] * (n+1)
    for v in range(1, n+1):
        if not positions[v]:
            continue
        last = -1
        for p in positions[v]:
            cont[v] += (p - last) * (n - p)
            last = p
            
    part2 = 0
    for x in range(1, n):
        pos_x = positions[x]
        pos_x1 = positions[x+1]
        if not pos_x and not pos_x1:
            continue
            
        merged = []
        i, j = 0, 0
        len_x = len(pos_x)
        len_x1 = len(pos_x1)
        while i < len_x and j < len_x1:
            if pos_x[i] < pos_x1[j]:
                merged.append(pos_x[i])
                i += 1
            else:
                merged.append(pos_x1[j])
                j += 1
        if i < len_x:
            merged.extend(pos_x[i:])
        if j < len_x1:
            merged.extend(pos_x1[j:])
            
        total_or = 0
        last = -1
        for p in merged:
            total_or += (p - last) * (n - p)
            last = p
            
        both_count = cont[x] + cont[x+1] - total_or
        part2 += both_count
        
    ans = part1 - part2
    print(ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the sequence length \( N \) and the sequence \( A \).
2. **Position Tracking**: For each value in \( A \), store the indices where it occurs.
3. **Part 1 Calculation**: For each element in \( A \), compute its contribution to the total distinct count over all subarrays by considering it as the last occurrence of its value in relevant subarrays.
4. **Contribution Array (cont)**: For each value, compute the number of subarrays that contain it by summing contributions from each occurrence.
5. **Part 2 Calculation**: For each adjacent value pair \( (x, x+1) \), compute the number of subarrays containing both values using inclusion-exclusion based on the counts of subarrays containing each value individually and the count containing at least one of them.
6. **Result Calculation**: The final result is obtained by subtracting the Part 2 sum from the Part 1 sum, which gives the total \( f(L, R) \) over all subarrays. The result is printed. 

This approach efficiently computes the desired sum by leveraging classic combinatorial techniques and careful handling of value occurrences and their adjacencies.
