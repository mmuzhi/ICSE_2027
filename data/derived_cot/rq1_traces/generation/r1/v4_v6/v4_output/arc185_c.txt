To solve this problem, we need to find three distinct indices (i, j, k) in an array such that the sum of the elements at these indices equals a given integer X. The indices must satisfy the condition 1 ≤ i < j < k ≤ N. If no such triple exists, we should return -1.

### Approach
1. **Problem Analysis**: The problem requires finding three distinct elements in an array whose sum equals a given value X. The challenge is to efficiently search for such a triplet, especially given the constraints where N can be as large as 1,000,000, making a brute-force O(N^3) solution infeasible.
2. **Key Insight**: By sorting the array while keeping track of the original indices, we can use a combination of early termination checks and a two-pointer technique to reduce the average time complexity. The idea is to iterate through each element (using the outer loop) and for each element, use two pointers (inner loop) to find two other elements that sum up to the remaining target value (X - current element).
3. **Optimization**: 
   - **Early Termination**: Before entering the main loop, we check if the smallest possible sum (sum of the first three elements) is greater than X or the largest possible sum (sum of the last three elements) is less than X. If either condition is true, we immediately return -1 since no valid triplet can exist.
   - **Early Break in Loop**: During the iteration, if the smallest possible sum for the current element and the next two elements exceeds X, we break out of the loop early because subsequent elements will only yield larger sums.
4. **Two-pointer Technique**: For each element at position `i` in the sorted array, we set two pointers, `j` starting at `i+1` and `k` starting at the end of the array. We adjust these pointers based on whether the sum of the elements at `j` and `k` is less than, greater than, or equal to the target (X - element at `i`).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    X = int(data[1])
    A = list(map(int, data[2:2+n]))
    
    if n < 3:
        print(-1)
        return
        
    B = []
    for i in range(n):
        B.append((A[i], i+1))
    B.sort()
    
    min0 = B[0][0] + B[1][0] + B[2][0]
    max0 = B[-1][0] + B[-2][0] + B[-3][0]
    if min0 > X or max0 < X:
        print(-1)
        return
        
    found = False
    for i in range(n-2):
        a_val = B[i][0]
        if i + 2 < n:
            if a_val + B[i+1][0] + B[i+2][0] > X:
                break
        if a_val + B[-1][0] + B[-2][0] < X:
            continue
        
        target = X - a_val
        j = i + 1
        k = n - 1
        while j < k:
            b_val = B[j][0]
            c_val = B[k][0]
            s = b_val + c_val
            if s == target:
                indices = sorted([B[i][1], B[j][1], B[k][1]])
                print(f"{indices[0]} {indices[1]} {indices[2]}")
                found = True
                break
            elif s < target:
                j += 1
            else:
                k -= 1
        if found:
            break
            
    if not found:
        print(-1)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints.
2. **Edge Case Handling**: If the array has fewer than three elements, it's impossible to form a triplet, so we return -1 immediately.
3. **Sorting with Indices**: The array is sorted along with the original indices stored to facilitate the two-pointer technique while allowing us to retrieve the original indices for the solution.
4. **Early Termination Checks**: Before the main loop, we check if the smallest or largest possible triplet sums are outside the target range X. If so, we return -1 early.
5. **Main Loop**: For each element in the sorted array (outer loop), we compute the target sum (X - current element). Using two pointers (inner loop), we search for two elements in the remaining array that sum to this target. The pointers are adjusted based on whether their current sum is less than or greater than the target.
6. **Output**: If a valid triplet is found, the original indices are sorted and printed. If no triplet is found after all iterations, -1 is printed.

This approach efficiently narrows down the search space using sorting and early termination, making it feasible to handle large input sizes within reasonable time.
