To solve this problem, we need to maximize the total score obtained by repeatedly removing adjacent pairs from a sequence, where the score for each removal is the absolute difference between the two numbers. The challenge is to find the optimal order of removals to maximize the cumulative score.

### Approach
1. **Problem Analysis**: The problem involves processing a sequence of numbers by removing adjacent pairs in each operation. The key is to determine the order of removals that maximizes the sum of absolute differences of the removed pairs. The sequence can be of any length, and after each removal, the remaining elements become adjacent, potentially forming new pairs for future removals.

2. **Intuition**: The optimal strategy involves always selecting the pair with the largest absolute difference available at any step. This greedy approach ensures that we maximize the score from each removal, which in turn maximizes the total score. By using a max-heap (priority queue) to keep track of the current adjacent pairs and their differences, we can efficiently retrieve and process the largest difference pair in each step.

3. **Algorithm Selection**: 
   - **Heap (Priority Queue)**: To always access the pair with the largest absolute difference in logarithmic time.
   - **Linked List Pointers**: To efficiently manage the adjacency relationships after each removal. We maintain left and right pointers for each element to quickly update the adjacency list when elements are removed.
   - **Removal Tracking**: We mark elements as removed to avoid reprocessing them and update the adjacency list by connecting the left and right neighbors of the removed elements.

4. **Complexity Analysis**: 
   - **Time Complexity**: The algorithm processes each element once, and each insertion and extraction from the heap takes O(log N) time. Thus, the total time complexity is O(N log N), which is efficient for the constraint N ≤ 3 × 10^5.
   - **Space Complexity**: The algorithm uses O(N) space for storing the sequence, left and right pointers, and the heap.

import heapq
import sys

def main():
    data = sys.stdin.read().split()
