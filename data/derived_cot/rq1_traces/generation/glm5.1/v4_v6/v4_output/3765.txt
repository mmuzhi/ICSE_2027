You will be given a competitive programming problem.
You are given two integer arrays, nums and cost, of the same size, and an integer k.
You can divide nums into subarrays. The cost of the i^th subarray consisting of elements nums[l..r] is:

(nums[0] + nums[1] + ... + nums[r] + k * i) * (cost[l] + cost[l + 1] + ... + cost[r]).

Note that i represents the order of the subarray: 1 for the first subarray, 2 for the second, and so on.
Return the minimum total cost possible from any valid division.
 
Example 1:

Input: nums = [3,1,4], cost = [4,6,6], k = 1
Output: 110
Explanation:
The minimum total cost possible can be achieved by dividing nums into subarrays [3, 1] and [4].


The cost of the first subarray [3,1] is (3 + 1 + 1 * 1) * (4 + 6) = 50.
The cost of the second subarray [4] is (3 + 1 + 4 + 1 * 2) * 6 = 60.


Example 2:

Input: nums = [4,8,5,1,14,2,2,12,1], cost = [7,2,8,4,2,2,1,1,2], k = 7
Output: 985
Explanation:
The minimum total cost possible can be achieved by dividing nums into subarrays [4, 8, 5, 1], [14, 2, 2], and [12, 1].


The cost of the first subarray [4, 8, 5, 1] is (4 + 8 + 5 + 1 + 7 * 1) * (7 + 2 + 8 + 4) = 525.
The cost of the second subarray [14, 2, 2] is (4 + 8 + 5 + 1 + 14 + 2 + 2 + 7 * 2) * (2 + 2 + 1) = 250.
The cost of the third subarray [12, 1] is (4 + 8 + 5 + 1 + 14 + 2 + 2 + 12 + 1 + 7 * 3) * (1 + 2) = 210.

from collections import deque

class Solution:
    def minimumCost(self, nums: List[int], cost: List[int], k: int) -> int:
        n = len(nums)
        
        # Compute prefix sums
        prefix_nums = [0] * (n + 1)
        prefix_cost = [0] * (n + 1)
        for i in range(n):
            prefix_nums[i + 1] = prefix_nums[i] + nums[i]
            prefix_cost[i + 1] = prefix_cost[i] + cost[i]
        
        # dp_prev[j] = minimum cost to partition nums[0..j-1] into (t-1) subarrays
        dp_prev = [float('inf')] * (n + 1)
        dp_prev[0] = 0
        
        result = float('inf')
        
        for t in range(1, n + 1):
            dp_curr = [float('inf')] * (n + 1)
            
            # Convex hull trick
            # Lines: y = -prefix_cost[l] * x + dp_prev[l]
            # We want minimum y at x = prefix_nums[j] + k * t
            
            dq = deque()  # Store indices l
            
            for j in range(t, n + 1):
                # Add line for l = j - 1
                l = j - 1
                if dp_prev[l] < float('inf'):
                    # Add line for l to the deque
                    while len(dq) >= 2:
                        l1, l2, l3 = dq[-2], dq[-1], l
                        # Check if l3 makes l2 obsolete
                        lhs = (dp_prev[l3] - dp_prev[l1]) * (prefix_cost[l2] - prefix_cost[l1])
                        rhs = (dp_prev[l2] - dp_prev[l1]) * (prefix_cost[l3] - prefix_cost[l1])
                        
                        if lhs <= rhs:
                            dq.pop()
                        else:
                            break
                    
                    dq.append(l)
                
                # Query at x = prefix_nums[j] + k * t
                x = prefix_nums[j] + k * t
                
                # Remove lines from the front that are not optimal
                while len(dq) >= 2:
                    l1, l2 = dq[0], dq[1]
                    val1 = -prefix_cost[l1] * x + dp_prev[l1]
                    val2 = -prefix_cost[l2] * x + dp_prev[l2]
                    if val2 <= val1:
                        dq.popleft()
                    else:
                        break
                
                if dq:
                    best_l = dq[0]
                    min_val = -prefix_cost[best_l] * x + dp_prev[best_l]
                    dp_curr[j] = (prefix_nums[j] + k * t) * prefix_cost[j] + min_val
            
            result = min(result, dp_curr[n])
            dp_prev = dp_curr
        
        return result

Looking at this problem, I need to find the minimum total cost when dividing the array into subarrays, where the cost of the i-th subarray [l..r] is:
`(nums[0] + ... + nums[r] + k*i) * (cost[l] + ... + cost[r])`

**Key Observations:**
1. Let `prefix_nums[j]` = sum of nums[0..j-1] and `prefix_cost[j]` = sum of cost[0..j-1]
2. For the i-th subarray [l..r], cost = `(prefix_nums[r+1] + k*i) * (prefix_cost[r+1] - prefix_cost[l])`
3. Define `dp[j][t]` = minimum cost to partition nums[0..j-1] into exactly t subarrays

**Transition:**
```
dp[j][t] = min over l of (dp[l][t-1] + (prefix_nums[j] + k*t) * (prefix_cost[j] - prefix_cost[l]))
```

Expanding this:
```
dp[j][t] = (prefix_nums[j] + k*t) * prefix_cost[j] + min over l of (dp[l][t-1] - (prefix_nums[j] + k*t) * prefix_cost[l])
```

The second term is a linear function in `prefix_cost[l]`, making this a **Convex Hull Trick** problem!

- Lines: `y = -prefix_cost[l] * x + dp[l][t-1]`
- Query at: `x = prefix_nums[j] + k*t`
- Slopes are decreasing (since prefix_cost is increasing)
- Queries are increasing in x

This allows O(1) amortized per query using a deque.

**Verification with Example 1:** nums=[3,1,4], cost=[4,6,6], k=1
- t=1: dp[3][1] = (8+1)*16 = 144
- t=2: dp[3][2] = 110 (splitting into [3,1] and [4])
- t=3: dp[3][3] = 118
- Answer: min(144, 110, 118) = 110 ✓
