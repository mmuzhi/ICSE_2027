There is a game dungeon comprised of n x n rooms arranged in a grid.
You are given a 2D array fruits of size n x n, where fruits[i][j] represents the number of fruits in the room (i, j). Three children will play in the game dungeon, with initial positions at the corner rooms (0, 0), (0, n - 1), and (n - 1, 0).
The children will make exactly n - 1 moves according to the following rules to reach the room (n - 1, n - 1):

The child starting from (0, 0) must move from their current room (i, j) to one of the rooms (i + 1, j + 1), (i + 1, j), and (i, j + 1) if the target room exists.
The child starting from (0, n - 1) must move from their current room (i, j) to one of the rooms (i + 1, j - 1), (i + 1, j), and (i + 1, j + 1) if the target room exists.
The child starting from (n - 1, 0) must move from their current room (i, j) to one of the rooms (i - 1, j + 1), (i, j + 1), and (i + 1, j + 1) if the target room exists.

When a child enters a room, they will collect all the fruits there. If two or more children enter the same room, only one child will collect the fruits, and the room will be emptied after they leave.
Return the maximum number of fruits the children can collect from the dungeon.

Output: 100

Is there any possibility that `best_prev` remains `-10**9`?
Since we only iterate `j` in the valid range, and the valid range is contiguous, any valid `j` must be reachable from a valid `j` in the previous step.
Let's verify this.
For $k$, valid $j \in [\max(k, n-1-k), n-1]$.
For $k+1$, valid $j' \in [\max(k+1, n-2-k), n-1]$.
Since $j' - j \in [-1, 1]$, any $j'$ in the valid range for $k+1$ must be reachable from some $j$ in the valid range for $k$.
Proof:
Let $L_k = \max(k, n-1-k)$.
$L_{k+1} = \max(k+1, n-2-k)$.
If $L_k = k$, then $L_{k+1} = k+1$. Since $j \ge k$, $j' \ge k+1$ is reachable.
If $L_k = n-1-k$, then $L_{k+1} = n-2-k$. Since $j \ge n-1-k$, $j' \ge n-2-k$ is reachable.
So the valid range is always reachable. `best_prev` will never be `-10**9` for valid `j`.

Wait, if they could overlap elsewhere, we WOULD need to worry about double counting, because we would add the fruit twice in our independent DP sums.

```python
class Solution:
    def maxCollectedFruits(self, fruits: List[List[int]]) -> int:
        n = len(fruits)
        ans = 0
        for i in range(n):
            ans += fruits[i][i]
            fruits[i][i] = 0
            
        # Child 2
        prev_dp = [-10**9] * n
        prev_dp[n-1] = fruits[0][n-1]
        for k in range(1, n):
            curr_dp = [-10**9] * n
            min_j = max(k, n - 1 - k)
            max_j = n - 1
            for j in range(min_j, max_j + 1):
                best_prev = prev_dp[j]
                if j > 0:
                    best_prev = max(best_prev, prev_dp[j-1])
                if j < n - 1:
                    best_prev = max(best_prev, prev_dp[j+1])
                curr_dp[j] = fruits[k][j] + best_prev
            prev_dp = curr_dp
        ans += prev_dp[n-1]
        
        # Child 3
        prev_dp = [-10**9] * n
        prev_dp[n-1] = fruits[n-1][0]
        for k in range(1, n):
            curr_dp = [-10**9] * n
            min_i = max(k, n - 1 - k)
            max_i = n - 1
            for i in range(min_i, max_i + 1):
                best_prev = prev_dp[i]
                if i > 0:
                    best_prev = max(best_prev, prev_dp[i-1])
                if i < n - 1:
                    best_prev = max(best_prev, prev_dp[i+1])
                curr_dp[i] = fruits[i][k] + best_prev
            prev_dp = curr_dp
        ans += prev_dp[n-1]
        
        return ans
```

An elegant and optimal solution to this problem relies on analyzing the movement constraints of the three children.

### Path Analysis

1. **Child 1 (from `(0, 0)` to `(n-1, n-1)`):**
   The child must make exactly $n-1$ moves. In each move, the sum of the coordinates $i + j$ increases by either 1 or 2. To go from $0 + 0 = 0$ to $(n-1) + (n-1) = 2n-2$ in exactly $n-1$ moves, the average increase per move must be exactly $\frac{2n-2}{n-1} = 2$. Since the maximum increase per move is 2, *every single move* must increase $i+j$ by 2. This restricts Child 1 to only making diagonal moves `(i+1, j+1)`. Thus, Child 1 will **only ever visit rooms on the main diagonal** `(k, k)` and has exactly one valid path.

2. **Child 2 (from `(0, n-1)` to `(n-1, n-1)`):**
   The child moves downwards such that at step $k$, they are at row $k$. Their column $j_k$ satisfies $|j_k - j_{k-1}| \le 1$. Starting at $j_0 = n-1$ and ending at $j_{n-1} = n-1$, the column must satisfy $j_k \ge n-1-k$ (cannot decrease too fast) and $j_k \ge k$ (must be able to increase back to $n-1$ in time). Thus, $j_k \in [\max(k, n-1-k), n-1]$.

3. **Child 3 (from `(n-1, 0)` to `(n-1, n-1)`):**
   Symmetrically, the child moves rightwards such that at step $k$, they are at column $k$. Their row $i_k$ obeys the exact same boundaries: $i_k \in [\max(k, n-1-k), n-1]$.

### Overlap Analysis
Can Child 2 and Child 3 visit the same room $(r, c)$?
- If Child 2 visits $(r, c)$ at step $r$, then $c = j_r \ge r$.
- If Child 3 visits $(r, c)$ at step $c$, then $r = i_c \ge c$.
For both to be true, we must have $c \ge r$ and $r \ge c$, which means $r = c$. Therefore, **Child 2 and Child 3 can only overlap on the main diagonal!**

Since Child 1 exclusively visits the main diagonal and collects all fruits there, any overlap between Child 2 and Child 3 will simply result in them trying to collect from an already emptied room. We can safely set the fruits on the main diagonal to `0` and then compute the maximum path sums for Child 2 and Child 3 completely independently!
