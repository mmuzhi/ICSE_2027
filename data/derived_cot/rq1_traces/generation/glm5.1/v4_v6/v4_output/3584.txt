You will be given a competitive programming problem.
You are given two strings word1 and word2.
A string x is called almost equal to y if you can change at most one character in x to make it identical to y.
A sequence of indices seq is called valid if:

The indices are sorted in ascending order.
Concatenating the characters at these indices in word1 in the same order results in a string that is almost equal to word2.

Return an array of size word2.length representing the lexicographically smallest valid sequence of indices. If no such sequence of indices exists, return an empty array.
Note that the answer must represent the lexicographically smallest array, not the corresponding string formed by those indices.
 
Example 1:

Input: word1 = "vbcca", word2 = "abc"
Output: [0,1,2]
Explanation:
The lexicographically smallest valid sequence of indices is [0, 1, 2]:

Change word1[0] to 'a'.
word1[1] is already 'b'.
word1[2] is already 'c'.


Example 2:

Input: word1 = "bacdc", word2 = "abc"
Output: [1,2,4]
Explanation:
The lexicographically smallest valid sequence of indices is [1, 2, 4]:

word1[1] is already 'a'.
Change word1[2] to 'b'.
word1[4] is already 'c'.


Example 3:

Input: word1 = "aaaaaa", word2 = "aaabc"
Output: []
Explanation:
There is no valid sequence of indices.

Example 4:

Input: word1 = "abc", word2 = "ab"
Output: [0,1]

 
Constraints:

1 <= word2.length < word1.length <= 3 * 10^5
word1 and word2 consist only of lowercase English letters.

```python
class Solution:
    def validSequence(self, word1: str, word2: str) -> List[int]:
        n, m = len(word1), len(word2)
        
        # Compute next_pos: next_pos[j][c] = smallest index >= j in word1 where character c appears
        next_pos = [[-1] * 26 for _ in range(n + 1)]
        for c in range(26):
            next_pos[n][c] = -1
        for i in range(n - 1, -1, -1):
            for c in range(26):
                next_pos[i][c] = next_pos[i+1][c]
            next_pos[i][ord(word1[i]) - ord('a')] = i
        
        # Compute dp0: length of longest suffix of word2 that can be matched exactly starting from word1[j:]
        dp0 = [0] * (n + 1)
        j0 = m - 1
        for i in range(n - 1, -1, -1):
            if j0 >= 0 and word1[i] == word2[j0]:
                dp0[i] = m - j0
                j0 -= 1
            else:
                dp0[i] = dp0[i+1]
        
        # Compute dp1: length of longest suffix of word2 that can be matched with at most one change starting from word1[j:]
        dp1 = [0] * (n + 1)
        j1 = m - 1
        used_change = False
        for i in range(n - 1, -1, -1):
            if j1 >= 0 and word1[i] == word2[j1]:
                dp1[i] = m - j1
                j1 -= 1
            elif j1 >= 0 and not used_change:
                dp1[i] = m - j1
                j1 -= 1
                used_change = True
            else:
                dp1[i] = dp1[i+1]
        
        # Greedy algorithm to find the lexicographically smallest sequence
        result = []
        prev_idx = -1
        used_change = False
        
        for k in range(m):
            c = ord(word2[k]) - ord('a')
            
            if not used_change:
                # We haven't used the one change yet
                # Option 1: Use an exact match (word1[j] == word2[k])
                j_exact = next_pos[prev_idx + 1][c]
                can_use_exact = j_exact != -1 and dp1[j_exact + 1] >= m - k - 1
                
                # Option 2: Use the one change (word1[j] != word2[k] or word1[j] == word2[k] with the one change)
                # The smallest j is prev_idx + 1
                j_change = prev_idx + 1
                can_use_change = j_change < n and dp0[j_change + 1] >= m - k - 1
                
                if can_use_exact and (not can_use_change or j_exact <= j_change):
                    result.append(j_exact)
                    prev_idx = j_exact
                elif can_use_change:
                    result.append(j_change)
                    prev_idx = j_change
                    if word1[j_change] != word2[k]:
                        used_change = True
                else:
                    return []
            else:
                # We've already used the one change
                j = next_pos[prev_idx + 1][c]
                if j != -1 and dp0[j+1] >= m - k - 1:
                    result.append(j)
                    prev_idx = j
                else:
                    return []
        
        return result
```

=== ANSWER ===
Looking at this problem, I need to find the lexicographically smallest sequence of indices from word1 that forms a string "almost equal" to word2 (at most one character difference).

**Key Observations:**
1. For lexicographically smallest, I should greedily minimize each index
2. I need to track whether I've used the "one change" allowance
3. I need to know if the remaining characters can be matched after choosing an index

**Algorithm:**
1. Precompute `dp0[j]`: length of longest suffix of word2 that can be matched exactly starting from word1[j:]
2. Precompute `dp1[j]`: same but allowing at most one change
3. Precompute `next_pos[j][c]`: smallest index ≥ j where character c appears
4. Greedily select indices, preferring exact matches when possible

**Greedy Selection:**
- If haven't used change: Try exact match first (check dp1 for remaining), else use change (check dp0 for remaining)
- If used change: Must use exact match (check dp0 for remaining)

                j1 -= 1
