Looking at this problem, I need to determine when a set of people can all have their requirements satisfied simultaneously.

**Key Insight:** For a person traveling from S to T:
- If S < T: They need prefix sums p_j > p_{S-1} for all S-1 < j < T-1, and p_{T-1} = p_{S-1}
- If S > T: They need prefix sums p_j < p_{T-1} for all T-1 < j < S-1, and p_{S-1} = p_{T-1}

Two people with the **same direction** conflict if their intervals partially overlap (neither contains the other, but they do overlap). Two people with **different directions** never conflict.

For a query [L, R], the answer is "Yes" if and only if no pair of people with the same direction within [L, R] have partially overlapping intervals.

**Algorithm:**
1. For each person k, compute `prev[k]` = maximum index i < k where (i, k) conflicts
2. Query [L, R] has answer "No" if max{prev[k] : L ≤ k ≤ R} ≥ L
3. Use segment tree for efficient range maximum queries

For computing `prev[k]` efficiently, I use the observation that for direction +1, person i conflicts with k if:
- l_i < l_k < r_i < r_k (i stabs k from left), OR  
- l_k < l_i < r_k < r_i (k stabs i from left)

Similar for direction -1.
