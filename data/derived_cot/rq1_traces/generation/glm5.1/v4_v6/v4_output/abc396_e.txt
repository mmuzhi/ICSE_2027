=== PROMPT ===
You will be given a competitive programming problem.
You are given integers N, M and three integer sequences of length M: X = (X_1, X_2, \ldots, X_M), Y = (Y_1, Y_2, \ldots, Y_M), and Z = (Z_1, Z_2, \ldots, Z_M). It is guaranteed that all elements of X and Y are between 1 and N, inclusive.
We call a length-N sequence of non-negative integers A = (A_1, A_2, \ldots, A_N) a good sequence if and only if it satisfies the following condition:

- For every integer i with 1 \le i \le M, the XOR of A_{X_i} and A_{Y_i} is Z_i.

Determine whether a good sequence A=(A_1,A_2,\ldots,A_N) exists, and if it exists, find one good sequence that minimizes the sum of its elements \displaystyle \sum_{i=1}^N A_i.

Notes on XOR

For non-negative integers A and B, their XOR A \oplus B is defined as follows:


- In the binary representation of A \oplus B, the digit in the place corresponding to 2^k \,(k \ge 0) is 1 if and only if exactly one of the digits in the same place of A and B is 1; otherwise, it is 0.


For example, 3 \oplus 5 = 6 (in binary: 011 \oplus 101 = 110).

Input

The input is given from Standard Input in the following format:
N M
X_1 Y_1 Z_1
X_2 Y_2 Z_2
\vdots
X_M Y_M Z_M

Output

If no good sequence exists, print -1.
If a good sequence exists, print one good sequence that minimizes the sum of its elements, separated by spaces.
If there are multiple good sequences with the same minimum sum, printing any of them is accepted.

Constraints


- 1 \le N \le 2\times 10^5
- 0 \le M \le 10^5
- 1 \le X_i, Y_i \le N
- 0 \le Z_i \le 10^9
- All input values are integers.

1 3 4
1 2 3

Sample Output 1

0 3 4

Other good sequences include A=(1,2,5) and A=(7,4,3), but A=(0,3,4) has the smallest sum among all good sequences.

- If we set bit b of the root value to 0, then for each node u, bit b of A_u is bit b of val[u].

- If root bit b = 0: (number of nodes with bit b of val[u] = 1) * 2^b
- If root bit b = 1: (number of nodes with bit b of val[u] = 0) * 2^b = (size - count_1_b) * 2^b

To minimize the sum, we choose root bit b = 0 if count_1_b <= size - count_1_b, and root bit b = 1 otherwise.

Looking at this problem, I need to find a sequence of non-negative integers that satisfies XOR constraints and minimizes the sum.

**Key Observations:**

1. **Graph Structure**: The constraints form a graph where each edge (X_i, Y_i) with weight Z_i means A_{X_i} ⊕ A_{Y_i} = Z_i.

2. **Connected Components**: Within each connected component, if I fix one variable, all others are determined. If I set A_root = x, then for any node u, A_u = x ⊕ val[u], where val[u] is the XOR of edge weights along the path from root to u.

3. **Consistency Check**: For any cycle, the XOR of all edge weights must be 0. I can check this during BFS - when visiting an already-visited node, verify that val[v] == val[u] ⊕ z.

4. **Minimizing Sum**: For each bit position independently, I count how many nodes have that bit set in val[u]. If more than half have it set, I should flip that bit in the root value (setting it to 1), otherwise keep it 0. This minimizes the total contribution of each bit to the sum.

5. **Self-loops**: An edge (u, u, z) requires 0 = z, so if z ≠ 0, no solution exists.

**Algorithm:**
- Build adjacency list (ignoring self-loops with z=0)
- For each connected component:
  - BFS to compute val[u] and check consistency
  - Count bits across all nodes in component
  - Choose optimal root value bit-by-bit
  - Compute final values
