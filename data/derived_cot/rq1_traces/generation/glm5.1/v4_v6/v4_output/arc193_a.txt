For integers l, r, let [l, r] denote the set of all integers from l through r. That is, [l, r] = \lbrace l, l+1, l+2, \ldots, r-1, r\rbrace.
You are given N pairs of integers (L_1, R_1), (L_2, R_2), \ldots, (L_N, R_N).
Based on these pairs, consider an undirected graph G defined as follows:

- It has N vertices numbered 1, 2, \ldots, N.
- For all i, j \in [1, N], there is an undirected edge between vertices i and j if and only if the intersection of [L_i, R_i] and [L_j, R_j] is empty.

In addition, for each i = 1, 2, \ldots, N, define the weight of vertex i to be W_i.
You are given Q queries about G. Process these queries in the order they are given.
For each i = 1, 2, \ldots, Q, the i-th query is the following:

You are given integers s_i and t_i (both between 1 and N, inclusive) such that s_i \neq t_i. Determine whether there exists a path from vertex s_i to vertex t_i in G. If it exists, print the minimum possible weight of such a path.

Here, the weight of a path from vertex s to vertex t is defined as the sum of the weights of the vertices on that path (including both endpoints s and t).

Output

    def __init__(self, data, n):  # data list indexed from 1 to n (or 0..n-1)
        self.n = n
        self.size = 1
        while self.size < n: self.size <<= 1
        self.seg = [INF] * (2*self.size)
        for i in range(n):
            self.seg[self.size+i] = data[i+1] if 1-indexed else data[i]
        for i in range(self.size-1,0,-1):
            self.seg[i] = min(self.seg[2*i], self.seg[2*i+1])

    def query(self, l, r):  # inclusive, 1-indexed
        l += self.size-1
        r += self.size-1
        res = INF
        while l <= r:
            if l%2==1: res = min(res, self.seg[l]); l+=1
            if r%2==0: res = min(res, self.seg[r]); r-=1
            l//=2; r//=2
        return res

Now, we need to implement efficiently.

Complexities: O(N log N) for building segment trees, O(Q log N) for queries.

But we can also use sparse tables for O(1) range min queries after O(M log M) preprocessing. Since M=2N up to 4e5, log2(4e5)~19, building two sparse tables of size ~4e5*19 ~7.6e6 each, total ~15e6 entries. Each entry could be an integer (4 bytes) if we use array of ints, but in Python, list of lists of ints would be heavy (each int is object). Might exceed memory. Better to use segment tree with O(log N) queries, which is simpler and memory efficient (2*size ~ 2*2^19 ~ 1M). So segment tree is fine.

But we need two segment trees: one for R, one for L. Each size about 2*2^19 = 1,048,576 nodes, each storing an integer (INF). That's okay.

Alternatively, we could use Fenwick tree for prefix minima but Fenwick normally supports prefix min, not arbitrary range min. However, we could use segment tree.

We'll implement segment tree with 1-indexed leaves for coordinates 1..maxCoord.

Implementation steps in code:

1. Read N.
2. Read W list (1-indexed).
3. Initialize arrays L, R, W (size N+1).
4. For i in 1..N: read L[i], R[i].
5. Compute maxCoord = max(max(L), max(R)) maybe 2*N, but we can compute actual max to reduce size. Since L_i,R_i <= 2N, we can set maxCoord = 2*N. But to be safe, compute max_val = max(max(L), max(R)). Use that as size.

6. Initialize minR = [INF]*(maxCoord+2), minL = [INF]*(maxCoord+2).
7. For i in 1..N:
    minR[R[i]] = min(minR[R[i]], W[i])
    minL[L[i]] = min(minL[L[i]], W[i])

8. Build segment tree for minR over indices 1..maxCoord.
   We'll define a function build_seg(data, n) returning segment tree object.

9. Build segment tree for minL similarly.

10. Compute pref_min_R array of length maxCoord+2:
    pref = [INF]*(maxCoord+2)
    cur = INF
    for x in range(1, maxCoord+2):
        pref[x] = cur
        if x <= maxCoord:
            cur = min(cur, minR[x])

11. Compute suff_min_L array of length maxCoord+2:
    suff = [INF]*(maxCoord+2)
    cur = INF
    for x in range(maxCoord, 0, -1):
        suff[x] = cur
        cur = min(cur, minL[x])
    # For x=0, we might not need.

12. Read Q.
13. For each query:
    s, t = map(int, input().split())
    if s == t: not possible per constraints.
    Ls, Rs, Ws = L[s], R[s], W[s]
    Lt, Rt, Wt = L[t], R[t], W[t]

    # Direct edge
    if Rs < Lt or Rt < Ls:
        print(Ws + Wt)
        continue

    # Overlap
    left_min = min(Ls, Lt)
    right_max = max(Rs, Rt)
    single_min = min(pref[left_min], suff[right_max])
    best = INF
    if single_min != INF:
        best = Ws + Wt + single_min

    # Two-intermediate
    # Determine crossing
    if Ls > Lt and Rs > Rt:
        lowR = Lt
        highR = Ls - 1
        lowL = Rt + 1
        highL = Rs
        if lowR <= highR and lowL <= highL:
            min_a = segR.query(lowR, highR)
            min_b = segL.query(lowL, highL)
            if min_a != INF and min_b != INF:
                cand = Ws + Wt + min_a + min_b
                if cand < best:
                    best = cand
    elif Lt > Ls and Rt > Rs:
        lowR = Ls
        highR = Lt - 1
        lowL = Rs + 1
        highL = Rt
        if lowR <= highR and lowL <= highL:
            min_a = segR.query(lowR, highR)
            min_b = segL.query(lowL, highL)
            if min_a != INF and min_b != INF:
                cand = Ws + Wt + min_a + min_b
                if cand < best:
                    best = cand

    if best == INF:
        print(-1)
    else:
        print(best)
