class SegmentTree:
    def __init__(self, data):
        self.n = len(data)
        self.size = 1
        while self.size < self.n:
            self.size *= 2
        self.sum = [0] * (2 * self.size)
        self.min_val = [float('inf')] * (2 * self.size)
        self.max_val = [float('-inf')] * (2 * self.size)
        self.lazy = [None] * (2 * self.size)

            self.max_val[self.size + i] = data[i]

Looking at this problem, I need to understand the key constraints:
- People must maintain their relative order (can't pass through each other)
- When person T moves to position G, other people may need to move to maintain valid positions

**Key Insight:** Define "offset" as `a_i = p_i - i` for person at position `p_i`. Since positions must be strictly increasing with integer positions, offsets must be non-decreasing.

When person T needs to go to position G:
- New offset for T is `G - T`
- If decreasing: all people with index < T who have offset > new_offset must be pushed down
- If increasing: all people with index > T who have offset < new_offset must be pushed up

**Algorithm:**
1. Use a segment tree with lazy propagation to maintain offsets
2. For each task, binary search to find the boundary where offsets need adjustment
3. Calculate movement as difference of sums before/after the range assignment
4. Update the segment tree with range assignment
