We need to compute sum over all subarrays of number of distinct values in that subarray. N up to 2e5. Classic problem: sum of number of distinct elements over all subarrays. Usually we can compute by considering contribution of each value: for each value v, count number of subarrays where v appears at least once. Then sum over v. So answer = sum_{v} (total number of subarrays - number of subarrays where v is absent). Number of subarrays where v is absent is computed by considering positions where v occurs, and gaps between them, plus prefix and suffix. More precisely: Let positions of v be p1, p2, ..., pk (1-indexed). Then segments without v: from start to p1-1, between pi and p_{i+1} (i.e., from pi+1 to p_{i+1}-1), and from pk+1 to N. For each such segment of length L, number of subarrays completely within that segment (i.e., not containing v) is L*(L+1)/2. Sum these over all gaps. Then total subarrays = N*(N+1)/2. So contribution of v = total_subarrays - sum_{gaps} L*(L+1)/2.

We need to compute for all distinct values efficiently. Since Ai <= N, we can have up to N distinct values. But we can compute contributions in O(N) by scanning and maintaining last occurrence. Another approach: iterate over subarrays and count distinct? Not feasible.

Standard method: For each index i, consider the contribution of A_i as the first occurrence of that value from left? Actually we can compute sum of distinct counts by adding 1 for each subarray where the element at position i is the first occurrence of that value in the subarray. That is, for each i, count number of subarrays [l, r] such that l <= i <= r, and A_i is the first occurrence of its value in that subarray. Then sum over i gives answer. This is often computed by for each i, let last occurrence of A_i before i be prev[i] (or 0 if none). Then number of subarrays where i is the first occurrence of its value is (i - prev[i]) * (N - i + 1). Because we can choose l from prev[i]+1 to i, and r from i to N. Then sum over i of that product gives answer. Let's verify: For each subarray, its distinct count equals number of values that appear, which is sum over values of indicator that value appears. But summing over i of indicator that A_i is the first occurrence in the subarray counts each distinct value exactly once per subarray (the leftmost occurrence). So indeed answer = sum_{i=1}^N (i - prev[i]) * (N - i + 1). This is O(N) after computing prev.

=== ANSWER ===
```python

import sys

def main():
    input = sys.stdin.readline
    N = int(input())
    A = list(map(int, input().split()))

    last = {}          # last occurrence index for each value
    ans = 0

    for i, val in enumerate(A, start=1):
        prev = last.get(val, 0)      # 0 if not seen before
        ans += (i - prev) * (N - i + 1)
        last[val] = i

    print(ans)

if __name__ == "__main__":
    main()
```
