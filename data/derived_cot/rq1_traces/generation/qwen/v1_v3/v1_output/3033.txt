To solve this problem, we need to transform one binary string into another using the minimum cost operations. The operations allowed are flipping any two bits (cost `x`) or flipping two adjacent bits (cost 1). The solution involves determining the minimal cost to make the strings equal or returning -1 if it's impossible.

### Approach
1. **Problem Analysis**: The key observation is that each operation flips two bits. Therefore, the total number of differing bits (mismatches) between the two strings must be even; otherwise, it's impossible to make them equal. 
2. **Intuition**: The optimal strategy involves using adjacent operations (cost 1) wherever possible, especially for consecutive mismatches. For non-consecutive mismatches, we use arbitrary operations (cost `x`). The challenge is to maximize the number of adjacent operations to minimize the total cost.
3. **Algorithm Selection**: 
   - **Step 1**: Count the mismatches. If the count is odd, return -1.
   - **Step 2**: For each position, check if two consecutive bits are both mismatches. This helps in identifying potential adjacent operations.
   - **Step 3**: Use dynamic programming to compute the maximum number of non-overlapping adjacent operations (edges) that can be applied. The DP array `dp[i]` stores the maximum number of adjacent operations up to the i-th bit.
   - **Step 4**: Calculate the total cost by combining the cost of adjacent operations and arbitrary operations. The formula used is `(m // 2) * x + k * (1 - x)`, where `k` is the number of adjacent operations.

### Solution Code
```python
class Solution:
    def minOperations(self, s1: str, s2: str, x: int) -> int:
        n = len(s1)
        m = 0
        for i in range(n):
            if s1[i] != s2[i]:
                m += 1

            return -1
        
        if m == 0:
            return 0

        for i in range(n - 1):
            if s1[i] != s2[i] and s1[i+1] != s2[i+1]:

        dp[0] = 0
        if n >= 2:
            for i in range(1, n):
                dp[i] = dp[i-1]
                if i-1 < n-1 and avail[i-1]:
                    prev = dp[i-2] if i-2 >= 0 else 0
                    dp[i] = max(dp[i], prev + 1)
```

### Explanation
1. **Initialization**: The code starts by counting the number of mismatches between the two strings. If the count is odd, it returns -1 immediately since it's impossible to fix an odd number of mismatches with operations that flip two bits each.
2. **Edge Cases**: If there are no mismatches, the cost is zero. If the strings are of length 1 and there's a mismatch, it's impossible (handled by the odd check).
3. **Availability Array**: The `avail` array marks positions where two consecutive bits are both mismatches, indicating potential adjacent operations.
4. **Dynamic Programming**: The `dp` array is filled by iterating through the string. For each position, it checks if an adjacent operation can be applied (based on the `avail` array) and updates the DP value accordingly.
5. **Cost Calculation**: The total cost is computed by considering both adjacent and arbitrary operations. The formula `(m // 2) * x + k * (1 - x)` efficiently combines the costs, where `k` is the number of adjacent operations used.

This approach ensures that we minimize the cost by maximizing the use of cheaper adjacent operations while handling non-consecutive mismatches with arbitrary operations.
