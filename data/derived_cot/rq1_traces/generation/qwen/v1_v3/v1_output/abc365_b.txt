To solve this problem, we need to find the second largest element in a distinct integer sequence and then determine its 1-based index in the original array. The solution involves iterating through the array while keeping track of the largest and second largest elements and their respective positions.

### Approach
1. **Initialization**: Start by reading the input values, which include the length of the array and the array elements themselves.
2. **Tracking Largest and Second Largest**: Initialize two variables to keep track of the largest (`max1`) and second largest (`max2`) elements, along with their indices (`idx1` and `idx2`).
3. **Iteration**: Traverse the array starting from the second element. For each element:
   - If the element is greater than the current largest, update the second largest to the previous largest and the largest to the current element. Record the index of the new largest element.
   - If the element is greater than the current second largest (and not the largest), update the second largest and its index.
4. **Output**: After processing all elements, the index of the second largest element (1-based) is printed.

This approach efficiently tracks the required elements in a single pass through the array, ensuring optimal performance with a time complexity of O(N), where N is the number of elements in the array.

### Solution Code
```python
n = int(input().strip())
arr = list(map(int, input().split()))

max1 = arr[0]
idx1 = 0
max2 = None
idx2 = None

for i in range(1, n):
    if arr[i] > max1:
        max2 = max1
        idx2 = idx1
        max1 = arr[i]
        idx1 = i
    elif max2 is None or arr[i] > max2:
        max2 = arr[i]
        idx2 = i

print(idx2 + 1)
```

### Explanation
1. **Reading Input**: The first line of input is the number of elements `n`, and the second line contains the array elements.
2. **Initialization**: `max1` and `idx1` are initialized to the first element and its index (0). `max2` and `idx2` start as `None` and will be updated during iteration.
3. **Iteration**: The loop runs from the second element to the end of the array. For each element:
   - If the element is greater than `max1`, it becomes the new `max1`, and the previous `max1` is moved to `max2` with its index stored in `idx2`.
   - If the element is greater than `max2` (and not the largest), it updates `max2` and its index.
4. **Output**: The 1-based index of the second largest element (`idx2 + 1`) is printed.

This method efficiently identifies the second largest element and its position in a single pass, making it both time and space efficient. The solution handles all edge cases, including arrays with exactly two elements, ensuring correctness across various input scenarios.
