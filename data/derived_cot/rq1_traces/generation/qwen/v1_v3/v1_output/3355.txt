To solve this problem, we need to determine the minimum number of levels Alice should play to ensure her total points are strictly greater than Bob's points when both play optimally. The game consists of a binary array where each level can either be cleared (1 point) or not cleared (0 points, resulting in -1 point). Alice plays the first k levels, and Bob plays the remaining levels. The solution involves finding the smallest k such that Alice's total points exceed Bob's total points.

### Approach
1. **Problem Analysis**: The key observation is that the total points for Alice and Bob are determined by the sum of their respective segments of the array, where each 1 contributes +1 and each 0 contributes -1. The goal is to find the smallest k (1 ≤ k ≤ n-1) such that the sum of the first k elements (Alice's segment) is strictly greater than the sum of the remaining elements (Bob's segment).

2. **Insight**: The sum of the entire array (total) is fixed. For any split at position k, Alice's sum (S) and Bob's sum (total - S) must satisfy S > total - S, which simplifies to 2S > total. This condition allows us to check for the smallest k efficiently.

3. **Algorithm Selection**: 
   - First, compute the total sum of the entire array.
   - Then, iterate through the array, maintaining a running sum of the elements processed so far (Alice's segment). For each position k, check if twice the running sum exceeds the total sum. The first valid k found during this iteration is the answer.

4. **Complexity Analysis**: The algorithm involves two linear passes through the array: one to compute the total sum and another to find the smallest k. Both passes are O(n), making the overall complexity O(n), which is efficient for large n (up to 10^5).

### Solution Code
```python
class Solution:
    def minimumLevels(self, possible: List[int]) -> int:
        n = len(possible)
        total = 0
        for x in possible:
            total += 1 if x == 1 else -1
        
        running_sum = 0
        for i in range(n-1):
            running_sum += 1 if possible[i] == 1 else -1
            if 2 * running_sum > total:
                return i + 1
        return -1
```

### Explanation
1. **Initialization**: The code starts by calculating the total sum of the entire array, where each 1 contributes +1 and each 0 contributes -1.
2. **Iteration**: The code then iterates through the array, maintaining a running sum of the elements processed (Alice's segment). For each position, it checks if twice the running sum exceeds the total sum. If it does, the current position (adjusted for 1-based indexing) is returned as the answer.
3. **Termination**: If no such position is found during the iteration, the function returns -1, indicating it's impossible for Alice to have more points than Bob under the given constraints.

This approach efficiently checks all possible splits by leveraging the fixed total sum and a running sum, ensuring optimal performance even for large input sizes.
