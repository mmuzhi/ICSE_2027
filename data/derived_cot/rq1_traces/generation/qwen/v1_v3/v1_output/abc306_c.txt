You are given a sequence A=(A_1,A_2,\dots,A_{3N}) of length 3N where each of 1,2,\dots, and N occurs exactly three times.
For i=1,2,\dots,N, let f(i) be the index of the middle occurrence of i in A.
Sort 1,2,\dots,N in ascending order of f(i).
Formally, f(i) is defined as follows.

- Suppose that those j such that A_j = i are j=\alpha,\beta,\gamma\ (\alpha < \beta < \gamma).  Then, f(i) = \beta.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \dots A_{3N}

Output

Print the sequence of length N obtained by sorting 1,2,\dots,N in ascending order of f(i), separated by spaces.

Constraints


- 1\leq N \leq 10^5
- 1 \leq A_j \leq N
- i occurs in A exactly three times, for each i=1,2,\dots,N.
- All input values are integers.

3

Sample Output 1

1 3 2

=== ANSWER ===

To solve this problem, we need to sort a sequence of numbers from 1 to N in ascending order based on the middle occurrence index of each number in the given sequence. The sequence contains each number from 1 to N exactly three times. The middle occurrence index is the position of the second occurrence of each number when the occurrences are ordered by their positions.

### Approach
1. **Problem Analysis**: The problem requires us to process a sequence of integers where each number from 1 to N appears exactly three times. For each number, we need to find the index of its middle occurrence (i.e., the second occurrence when all occurrences are ordered by their positions). Then, we sort the numbers from 1 to N based on these middle occurrence indices in ascending order.

2. **Intuition**: By traversing the sequence once, we can record the first and second occurrences of each number. The second occurrence directly gives us the middle occurrence index. Once we have these indices for all numbers, we can sort the numbers based on these indices.

3. **Algorithm Selection**: We use two arrays, `first` and `second`, to store the positions of the first and second occurrences of each number. The sequence is traversed once, and for each number encountered, we update these arrays if the number's occurrence is the first or second time it appears. After processing the entire sequence, we sort the numbers from 1 to N based on the values stored in the `second` array.

4. **Complexity Analysis**: The algorithm involves a single pass through the sequence of length 3N, which is O(N) time complexity. Sorting the numbers based on their middle occurrence indices is O(N log N) due to the sorting step. The space complexity is O(N) to store the occurrence positions for each number.

### Solution Code
```python
n = int(input())
arr = list(map(int, input().split()))

first = [0] * (n + 1)
second = [0] * (n + 1)

for idx, num in enumerate(arr):
    pos = idx + 1
    if first[num] == 0:
        first[num] = pos
    elif second[num] == 0:
        second[num] = pos

sorted_numbers = sorted(range(1, n + 1), key=lambda x: second[x])
print(" ".join(map(str, sorted_numbers)))
```

### Explanation
1. **Reading Input**: The first line reads the integer N, and the second line reads the sequence of 3N integers.
2. **Tracking Occurrences**: Two arrays, `first` and `second`, are initialized to store the positions of the first and second occurrences of each number. As we iterate through the sequence, we update these arrays whenever a number is encountered for the first or second time.
3. **Sorting**: The numbers from 1 to N are sorted based on their second occurrence positions. The sorted numbers are then printed as a space-separated string.

This approach efficiently processes the sequence and sorts the numbers based on the required criteria, leveraging simple array operations and a standard sorting algorithm.
