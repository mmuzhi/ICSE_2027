=== PROMPT ===
You will be given a competitive programming problem.
You are given a 0-indexed array nums of integers.
A triplet of indices (i, j, k) is a mountain if:

i < j < k
nums[i] < nums[j] and nums[k] < nums[j]

Return the minimum possible sum of a mountain triplet of nums. If no such triplet exists, return -1.
 
Example 1:

Input: nums = [8,6,1,5,3]
Output: 9
Explanation: Triplet (2, 3, 4) is a mountain triplet of sum 9 since: 
- 2 < 3 < 4
- nums[2] < nums[3] and nums[4] < nums[3]
And the sum of this triplet is nums[2] + nums[3] + nums[4] = 9. It can be shown that there are no mountain triplets with a sum of less than 9.

Example 2:

Input: nums = [5,4,8,7,10,2]
Output: 13
Explanation: Triplet (1, 3, 5) is a mountain triplet of sum 13 since: 
- 1 < 3 < 5
- nums[1] < nums[3] and nums[5] < nums[3]
And the sum of this triplet is nums[1] + nums[3] + nums[5] = 13. It can be shown that there are no mountain triplets with a sum of less than 13.

Example 3:

Input: nums = [6,5,4,3,4,5]
Output: -1
Explanation: It can be shown that there are no mountain triplets in nums.

 
Constraints:

3 <= nums.length <= 50
1 <= nums[i] <= 50

To solve this problem, we need to find the minimum possible sum of a mountain triplet in an array. A mountain triplet is defined as three indices (i, j, k) such that i < j < k, nums[i] < nums[j], and nums[k] < nums[j]. The goal is to find the triplet with the smallest sum that meets these conditions.

### Approach
1. **Problem Analysis**: The problem requires checking all possible triplets (i, j, k) in the array where the middle element nums[j] is greater than both nums[i] and nums[k]. The challenge is to efficiently find the triplet with the smallest sum.
2. **Intuition**: Given the constraints (array length up to 50), a brute-force approach is feasible. We can iterate through all possible triplets and check if they form a mountain triplet, then keep track of the minimum sum encountered.
3. **Algorithm Selection**: The solution involves three nested loops to check all possible triplets (i, j, k) where i < j < k. For each triplet, we verify if nums[i] < nums[j] and nums[k] < nums[j]. If both conditions are met, we calculate the sum and update the minimum sum found.
4. **Complexity Analysis**: The algorithm has a time complexity of O(n^3) where n is the length of the array. For n = 50, this results in 125,000 operations, which is manageable given the constraints.

### Solution Code
```python
class Solution:
    def minimumSum(self, nums: List[int]) -> int:
        n = len(nums)
        min_val = float('inf')
        for i in range(n):
            for j in range(i+1, n):
                for k in range(j+1, n):
                    if nums[i] < nums[j] and nums[k] < nums[j]:
                        total = nums[i] + nums[j] + nums[k]
                        if total < min_val:
                            min_val = total
        return min_val if min_val != float('inf') else -1
```

### Explanation
1. **Initialization**: We start by initializing `min_val` to a very large value (infinity) to keep track of the smallest sum found.
2. **Triplet Iteration**: Using three nested loops, we iterate through all possible triplets (i, j, k) where i < j < k. The outer loop (i) runs from the start of the array, the middle loop (j) from i+1 to the end, and the inner loop (k) from j+1 to the end.
3. **Mountain Triplet Check**: For each triplet, we check if nums[i] < nums[j] and nums[k] < nums[j]. If both conditions are satisfied, we calculate the sum of the triplet.
4. **Update Minimum Sum**: If the calculated sum is smaller than the current `min_val`, we update `min_val`.
5. **Result Handling**: After checking all triplets, if a valid mountain triplet was found, we return the smallest sum; otherwise, we return -1.

This approach efficiently checks all possible mountain triplets and ensures the solution is found with the minimum sum, leveraging the constraints to keep the implementation straightforward.
