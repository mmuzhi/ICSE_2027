To solve this problem, we need to determine the maximum number of products that can be printed by a Keyence printer under specific constraints. The printer can print one product at a time, and after each print, it requires a 1-microsecond charge before it can print again. Each product enters the printer's range at time \( T_i \) and stays for \( D_i \) microseconds, meaning it leaves at time \( T_i + D_i \). The goal is to schedule the printing of as many products as possible without violating the printer's constraints.

### Approach
1. **Problem Analysis**: The problem requires selecting a subset of products and assigning each a printing time within their available interval \([T_i, T_i + D_i]\) such that consecutive prints are at least 1 microsecond apart. The challenge is to maximize the number of products printed efficiently, given the constraints.

2. **Intuition**: The optimal strategy involves scheduling products in a way that minimizes the time between consecutive prints. This can be achieved by prioritizing products that end earlier (i.e., have smaller \( T_i + D_i \)) and can be scheduled as soon as possible after the last print.

3. **Algorithm Selection**: 
   - **Sorting by End Time**: We sort all products by their end time \( T_i + D_i \). If two products have the same end time, we sort them by their start time \( T_i \) to ensure the one that starts earlier is considered first.
   - **Greedy Scheduling**: For each product in the sorted list, we calculate the earliest possible printing time, which is the maximum of the previous print time + 1 and the product's start time \( T_i \). If this time is within the product's interval, we schedule it and update the last print time.

4. **Complexity Analysis**: The algorithm involves sorting \( N \) products, which takes \( O(N \log N) \) time. The subsequent greedy pass through the sorted list is \( O(N) \). Given \( N \) can be up to \( 2 \times 10^5 \), this approach is efficient.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
        
    n = int(data[0])
    products = []
    index = 1
    for i in range(n):
        t = int(data[index])
        d = int(data[index+1])
        index += 2
        products.append((t, d, t + d))
        
    products.sort(key=lambda x: (x[2], x[0]))
    
    last_time = 0
    count = 0
    for t, d, end in products:
        candidate_time = max(last_time + 1, t)
        if candidate_time <= end:
            last_time = candidate_time
            count += 1
            
    print(count)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into a list of tuples, each representing a product's start time \( T_i \), duration \( D_i \), and end time \( T_i + D_i \).

2. **Sorting**: The products are sorted primarily by their end times and secondarily by their start times. This ensures that we consider products that become free earlier first, maximizing the number of products that can be scheduled.

3. **Greedy Scheduling**: We iterate through the sorted products. For each product, the earliest possible printing time is calculated as the maximum of the previous print time + 1 and the product's start time. If this time is within the product's interval, it is scheduled, and the last print time is updated.

4. **Output**: The count of successfully scheduled products is printed.

This approach efficiently maximizes the number of products printed by leveraging sorting and a greedy selection strategy, ensuring optimal performance even for large input sizes.
