To solve this problem, we need to determine the minimum number of operations required to remove all elements from an array using two types of operations: removing two elements of the same value or removing three elements of the same value. If it's not possible to empty the array, we should return -1.

### Approach
1. **Frequency Counting**: First, we count the frequency of each element in the array using a hash map (or a Counter in Python). This helps us understand how many times each element appears.
2. **Check for Impossibility**: If any element occurs exactly once, it's impossible to remove it using the given operations (since both operations require at least two elements). Hence, we immediately return -1 in such cases.
3. **Calculate Minimum Operations**: For each element's frequency, the minimal number of operations required is the ceiling of the division of the frequency by 3. This is because each operation can remove up to 3 elements, and combining operations of 2 and 3 elements allows us to cover any remainder efficiently. The ceiling can be computed as `(count + 2) // 3`.
4. **Sum Operations**: Sum the operations for all elements to get the total minimum operations required to empty the array.

### Solution Code
```python
from collections import Counter

class Solution:
    def minOperations(self, nums: List[int]) -> int:
        freq = Counter(nums)
        total_ops = 0
        for count in freq.values():
            if count == 1:
                return -1
            total_ops += (count + 2) // 3
        return total_ops
```

### Explanation
1. **Frequency Counting**: The `Counter` from the `collections` module is used to count occurrences of each element in the input array. This gives us a dictionary where keys are elements and values are their respective counts.
2. **Impossibility Check**: If any element's count is 1, the function returns -1 because we cannot remove a single element with the given operations.
3. **Operations Calculation**: For each element's count, the minimal operations needed is calculated as `(count + 2) // 3`. This formula effectively computes the ceiling of `count / 3` without floating-point operations. For example:
   - For a count of 4, `(4 + 2) // 3 = 6 // 3 = 2` operations (two operations of removing two elements each).
   - For a count of 5, `(5 + 2) // 3 = 7 // 3 = 2` operations (one operation of removing three elements and one operation of removing two elements).
4. **Summing Operations**: The operations for all elements are summed up to get the total minimum operations required to empty the array. This sum is returned as the result.

This approach efficiently checks for the possibility of emptying the array and calculates the minimal operations by leveraging simple arithmetic and frequency counting, ensuring optimal performance even for large input sizes.
