You will be given a competitive programming problem.
You are given a 0-indexed array of positive integers nums and a positive integer limit.
In one operation, you can choose any two indices i and j and swap nums[i] and nums[j] if |nums[i] - nums[j]| <= limit.
Return the lexicographically smallest array that can be obtained by performing the operation any number of times.
An array a is lexicographically smaller than an array b if in the first position where a and b differ, array a has an element that is less than the corresponding element in b. For example, the array [2,10,3] is lexicographically smaller than the array [10,2,3] because they differ at index 0 and 2 < 10.
 
Example 1:

Input: nums = [1,5,3,9,8], limit = 2
Output: [1,3,5,8,9]
Explanation: Apply the operation 2 times:
- Swap nums[1] with nums[2]. The array becomes [1,3,5,9,8]
- Swap nums[3] with nums[4]. The array becomes [1,3,5,8,9]
We cannot obtain a lexicographically smaller array by applying any more operations.
Note that it may be possible to get the same result by doing different operations.

Example 2:

Input: nums = [1,7,6,18,2,1], limit = 3
Output: [1,6,7,18,1,2]
Explanation: Apply the operation 3 times:
- Swap nums[1] with nums[2]. The array becomes [1,6,7,18,2,1]
- Swap nums[0] with nums[4]. The array becomes [2,6,7,18,1,1]
- Swap nums[0] with nums[5]. The array becomes [1,6,7,18,1,2]
We cannot obtain a lexicographically smaller array by applying any more operations.

Example 3:

Input: nums = [1,7,28,19,10], limit = 3
Output: [1,7,28,19,10]
Explanation: [1,7,28,19,10] is the lexicographically smallest array we can obtain because we cannot apply the operation on any two indices.

 
Constraints:

1 <= nums.length <= 10^5
1 <= nums[i] <= 10^9
1 <= limit <= 10^9

class Solution:
    def lexicographicallySmallestArray(self, nums: List[int], limit: int) -> List[int]:
        n = len(nums)
        # Create an array of (value, index)
        arr = [(nums[i], i) for i in range(n)]
        arr.sort(key=lambda x: x[0])  # sort by value
        
        # Group into components
        i = 0
        components = []
        while i < n:
            j = i
            # Extend the component while the next element is within limit
            while j + 1 < n and arr[j+1][0] - arr[j][0] <= limit:
                j += 1
            # Append the current component (from i to j)
            components.append(arr[i:j+1])
            i = j + 1
        
        # Initialize the result array
        ans = [0] * n
        
        # For each component, extract indices and values
        for comp in components:
            # Extract indices and values
            indices = sorted([idx for val, idx in comp])
            vals = [val for val, idx in comp]  # already sorted by value
            # Assign the sorted values to the sorted indices
            for idx, val in zip(indices, vals):
                ans[idx] = val
        
        return ans

To solve this problem, we need to find the lexicographically smallest array that can be obtained by performing any number of swaps between elements of the array, where each swap is allowed only if the absolute difference between the two elements is less than or equal to a given limit. The key insight is that elements can be grouped into connected components based on their values, and within each component, elements can be rearranged freely. The lexicographically smallest array is achieved by placing the smallest values in the smallest indices within each component.

### Approach
1. **Sort the Array with Indices**: First, we create a list of tuples where each tuple consists of an element from the original array and its original index. This list is then sorted based on the element values.
2. **Group Consecutive Elements**: We traverse the sorted list and group consecutive elements if their values differ by at most the given limit. This grouping helps identify connected components where elements can be swapped among themselves.
3. **Assign Sorted Values to Indices**: For each connected component, we sort the original indices of the elements in the component. We then assign the sorted values (from the component) to these sorted indices in increasing order. This ensures the lexicographically smallest configuration for the array.

### Solution Code
```python
class Solution:
    def lexicographicallySmallestArray(self, nums: List[int], limit: int) -> List[int]:
        n = len(nums)
        arr = [(nums[i], i) for i in range(n)]
        arr.sort(key=lambda x: x[0])
        
        i = 0
        components = []
        while i < n:
            j = i
            while j + 1 < n and arr[j+1][0] - arr[j][0] <= limit:
                j += 1
            components.append(arr[i:j+1])
            i = j + 1
        
        ans = [0] * n
        for comp in components:
            indices = sorted([idx for val, idx in comp])
            vals = [val for val, idx in comp]
            for idx, val in zip(indices, vals):
                ans[idx] = val
        
        return ans
```

### Explanation
1. **Sorting and Grouping**: The array elements along with their original indices are sorted by their values. Consecutive elements in this sorted array that meet the difference condition (≤ limit) are grouped into the same connected component. This grouping is based on the observation that elements can be swapped if they form a chain where each consecutive pair in the sorted array meets the condition.
2. **Component Processing**: For each connected component, we extract the original indices and sort them. The values within the component are already sorted due to the initial sorting step. These sorted values are then assigned to the sorted indices in the original array.
3. **Result Construction**: The final array is constructed by placing the smallest value from each component into the smallest index of that component, the next smallest value into the next smallest index, and so on. This ensures the lexicographically smallest possible array configuration under the given constraints.

This approach efficiently leverages sorting and grouping to determine the connected components and then optimally assigns values to indices to achieve the desired result. The complexity is dominated by the sorting step, making it O(n log n), which is efficient for large input sizes.
