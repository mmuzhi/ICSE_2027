You will be given a competitive programming problem.
Takahashi participated in N contests and earned a performance P_i in the i-th contest.
He wants to choose some (at least one) contests from these and maximize his rating calculated from the results of those contests.
Find the maximum possible rating he can achieve by optimally choosing the contests.
Here, Takahashi's rating R is calculated as the following, where k is the number of chosen contests and (Q_1, Q_2, \ldots, Q_k) are the performances in the chosen contests in the order he participated:

\displaystyle R=\frac{\sum_{i=1}^k (0.9)^{k-i}Q_i}{\sum_{i=1}^k (0.9)^{k-i}}-\frac{1200}{\sqrt{k}}.

Input

The input is given from Standard Input in the following format:
N
P_1 P_2 \ldots P_N

Output

Print the maximum possible rating that Takahashi can achieve.
Your output will be considered correct if the absolute or relative error from the true value is at most 10^{-6}.

Constraints


- 1\leq N\leq 5000
- 1\leq P_i\leq 5000
- All input values are integers.

Sample Input 1

3
1000 600 1200

Sample Output 1

256.735020470879931

\displaystyle R=\frac{0.9\times 1000+ 1.0\times 1200}{0.9+1.0}-\frac{1200}{\sqrt{2}}=256.73502....
This is the maximum possible rating.

To solve this problem, we need to find the maximum possible rating Takahashi can achieve by selecting a subset of contests from the given list. The rating is calculated based on the weighted sum of performances in the chosen contests, divided by a geometric series sum, minus a penalty term that depends on the number of contests selected.

### Approach
1. **Problem Analysis**: The problem requires selecting a non-empty subset of contests (in the order of participation) to maximize the rating. The rating is computed as the weighted sum of performances (with weights decreasing exponentially) divided by the sum of the weights, minus a penalty term that increases with the square root of the number of contests selected.

2. **Dynamic Programming (DP) Setup**: We use a DP approach where `best_prev[j]` stores the maximum weighted sum achievable by selecting `j` contests from the first `i` contests. The recurrence relation for `best_prev[j]` is derived by extending a subset of `j-1` contests from the first `i-1` contests by including the `i-th` contest, adjusting the weights accordingly.

3. **Algorithm Selection**: For each contest, we compute the maximum weighted sum for all possible subset sizes (from 1 to the total contests processed so far) by leveraging previously computed values. The key insight is that the optimal subset for `j` contests ending at the current contest can be derived from the optimal subset for `j-1` contests from the previous contests.

4. **Complexity Analysis**: The algorithm processes each contest and, for each contest, considers all possible subset sizes up to the current contest. This results in a time complexity of O(N^2), which is feasible given the constraint N ≤ 5000.

### Solution Code
```python
n = int(input().strip())
P = list(map(int, input().split()))

best_prev = [-1e18] * (n + 1)
```

1. **Initialization**: We start by reading the number of contests `n` and their performances `P`. The `best_prev` array is initialized to store the maximum weighted sums for subsets of sizes 1 to `n`, initialized to a very small number to represent uncomputed states.

2. **Processing Each Contest**: For each contest, we compute the maximum weighted sum for all subset sizes from 1 to the current contest. 3. **Updating Best Values**: After computing the weighted sums for the current contest, we update `best_prev` if the current computation yields a better (higher) value for any subset size.

4. **Calculating the Maximum Rating**: After processing all contests, we compute the rating for each possible subset size (1 to `n`) using the stored `best_prev` values. The rating is derived by dividing the weighted sum by the geometric series sum and subtracting the penalty term. The maximum rating across all subset sizes is then printed with high precision.
