You are given a binary string s and a positive integer k.
A substring of s is beautiful if the number of 1's in it is exactly k.
Let len be the length of the shortest beautiful substring.
Return the lexicographically smallest beautiful substring of string s with length equal to len. If s doesn't contain a beautiful substring, return an empty string.
A string a is lexicographically larger than a string b (of the same length) if in the first position where a and b differ, a has a character strictly larger than the corresponding character in b.

For example, "abcd" is lexicographically larger than "abcc" because the first position they differ is at the fourth character, and d is greater than c.

1. **Problem Analysis**: The problem requires scanning all possible substrings of the given binary string to find those that contain exactly `k` '1's. Among these substrings, we need to find the one with the shortest length. If there are multiple substrings of the same minimal length, we return the lexicographically smallest one. If no such substring exists, we return an empty string.

2. **Intuition**: The solution involves iterating over each possible starting index of the substring and expanding the substring to include subsequent characters until the count of '1's reaches `k`. For each starting index, the first substring that meets the `k` '1's condition is recorded. The length of this substring is compared with the current minimal length, updating the minimal length and the list of candidates if a shorter substring is found. If a substring of the same minimal length is found, it is added to the list of candidates.

3. **Algorithm Selection**: The algorithm uses a nested loop where the outer loop iterates over each starting index, and the inner loop expands the substring from the starting index to include subsequent characters. The count of '1's is maintained as the substring expands. Once the count reaches `k`, the substring is recorded if it is the shortest found so far or if it matches the current minimal length.

4. **Complexity Analysis**: The algorithm has a time complexity of O(n^2) where `n` is the length of the string. This is because for each starting index, the inner loop may scan up to `n` characters. Given the constraint `n <= 100`, this approach is efficient and feasible.

### Solution Code
```python
class Solution:
    def shortestBeautifulSubstring(self, s: str, k: int) -> str:
        n = len(s)
        min_len = float('inf')
        candidates = []
        for i in range(n):
            ones = 0
            for j in range(i, n):
                if s[j] == '1':
                    ones += 1
                if ones == k:
                    length = j - i + 1
                    if length < min_len:
                        min_len = length
                        candidates = [s[i:j+1]]
                    elif length == min_len:
                        candidates.append(s[i:j+1])
                    break
        if min_len == float('inf'):
            return ""
        return min(candidates)
```

### Explanation
1. **Initialization**: We initialize `min_len` to infinity and an empty list `candidates` to store substrings that meet the `k` '1's condition.
2. **Nested Loop**: The outer loop iterates over each character in the string as the starting index of a potential substring. The inner loop expands the substring from the starting index to include subsequent characters, counting the number of '1's.
3. **Condition Check**: When the count of '1's reaches `k`, the length of the current substring is compared with `min_len`. If it is shorter, `min_len` is updated, and the list `candidates` is reset to contain only this substring. If it matches `min_len`, the substring is added to `candidates`.
4. **Result Selection**: After processing all possible substrings, if `min_len` remains infinity, it means no beautiful substring was found, and an empty string is returned. Otherwise, the lexicographically smallest substring from `candidates` is returned using the `min` function.

This approach efficiently narrows down the candidates to those with the minimal length and ensures the lexicographically smallest one is selected, meeting the problem requirements.
