class Solution:
    def minOperations(self, nums: List[int]) -> int:
        n = len(nums)
        end = [0] * (n + 3)  # to avoid index out of bounds, allocate n+3 elements
        flip = 0  # current flip effect (mod 2)
        operations = 0
        for i in range(n):
            if i >= 3:
                flip ^= end[i]  # remove the operations that ended at i (started at i-3)
            # Calculate the effective value
            current = nums[i] ^ flip
            if current == 0:
                if i > n - 3:
                    return -1
                operations += 1
                flip ^= 1  # flip the current effect
                if i + 3 < n + 3:  # i+3 is within the allocated end array
                    end[i+3] ^= 1
        return operations

To solve this problem, we need to determine the minimum number of operations required to turn all elements of a binary array into 1s by flipping any three consecutive elements any number of times. If it's impossible to achieve this, we should return -1.

### Approach
1. **Problem Analysis**: The key observation here is that each flip operation affects three consecutive elements. The challenge is to find a sequence of flips that converts all elements to 1s with the least operations. The solution involves processing the array from left to right, flipping elements whenever necessary to ensure the current element becomes 1, while considering the effect of previous flips on subsequent elements.

2. **Insight**: The effect of each flip operation is cumulative and can be tracked using a variable that keeps the current flip state (mod 2). Additionally, we use an auxiliary array to mark the end of each flip's effect, allowing us to adjust the flip state as we move through the array.

3. **Algorithm Selection**: We use a greedy approach where we process each element from left to right. For each element, if it is not already 1 (considering previous flips), we perform a flip operation starting at the current position. This operation affects the current and next two elements. We maintain a flip state variable and an end marker array to efficiently manage the flip effects.

4. **Complexity Analysis**: The algorithm processes each element exactly once, making it O(n) time complexity, where n is the length of the array. The space complexity is O(n) due to the auxiliary array used to track flip end positions.

### Solution Code
```python
class Solution:
    def minOperations(self, nums: List[int]) -> int:
        n = len(nums)
        end = [0] * (n + 3)
        flip = 0
        operations = 0
        for i in range(n):
            if i >= 3:
                flip ^= end[i]
            current = nums[i] ^ flip
            if current == 0:
                if i > n - 3:
                    return -1
                operations += 1
                flip ^= 1
                if i + 3 < n + 3:
                    end[i + 3] ^= 1
        return operations
```

### Explanation
1. **Initialization**: We initialize an array `end` to keep track of where each flip's effect ends, a variable `flip` to maintain the current flip state (mod 2), and `operations` to count the number of flips performed.

2. **Processing Each Element**: For each element in the array:
   - **Adjust Flip State**: If the current index is at least 3, we adjust the flip state by considering the flips that ended at the current index (stored in `end`).
   - **Check Current Value**: The effective value of the current element is computed by XORing its original value with the current flip state. If this value is 0, we need to flip starting at the current index.
   - **Flip Operation**: If flipping is needed and possible (i.e., there are at least three elements left), we increment the operation count, adjust the flip state, and mark the end of this flip's effect in the `end` array.

3. **Result**: After processing all elements, if all were successfully converted to 1s, the total operations are returned; otherwise, -1 is returned if any element couldn't be flipped appropriately.
