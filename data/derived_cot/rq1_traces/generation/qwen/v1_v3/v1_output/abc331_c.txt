=== ANSWER ===

To solve this problem, we need to efficiently compute, for each element in a sequence, the sum of all elements greater than it. Given the constraints, a brute-force approach would be too slow, so we use a Fenwick Tree (Binary Indexed Tree) to achieve logarithmic time complexity for both updates and queries.

### Approach
1. **Problem Analysis**: For each element in the sequence, we need to find the sum of all elements strictly greater than it. The challenge is to do this efficiently for large sequences (up to 200,000 elements).

2. **Intuition**: The solution involves leveraging the fact that the sum of elements greater than a given element can be derived by subtracting the sum of elements less than or equal to it from the total sum of the sequence. 

3. **Algorithm Selection**: 
   - **Coordinate Compression**: Since the values can be as large as 10^6, we first map these values to a smaller range (based on distinct values in the sequence) to optimize the Fenwick Tree operations.
   - **Fenwick Tree**: This data structure allows efficient computation of prefix sums (sums of elements up to a certain value) and updates in logarithmic time. We use it to store the cumulative sum of elements up to each compressed value.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(N log M), where N is the number of elements and M is the number of distinct values. Each element is processed once for updating the tree and once for querying, each operation being O(log M).
   - **Space Complexity**: O(M), where M is the number of distinct values, due to the storage required for the Fenwick Tree.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    total_sum = sum(A)
    
    sorted_vals = sorted(set(A))
    m = len(sorted_vals)
    mapping = {val: idx+1 for idx, val in enumerate(sorted_vals)}
    
    tree = [0] * (m+1)
    
    def update(i, delta):
        while i <= m:
            tree[i] += delta
            i += i & -i
            
    def query(i):
        s = 0
        while i:
            s += tree[i]
            i -= i & -i
        return s
            
    for a in A:
        idx = mapping[a]
        update(idx, a)
        
    res = []
    for a in A:
        idx = mapping[a]
        prefix_sum = query(idx)
        res.append(str(total_sum - prefix_sum))
        
    print(" ".join(res))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the sequence length and the sequence itself.
2. **Total Sum Calculation**: The total sum of all elements in the sequence is computed to later determine the sum of elements greater than any given element.
3. **Coordinate Compression**: The distinct values in the sequence are sorted and mapped to a compressed range (1 to number of distinct values) to optimize Fenwick Tree operations.
4. **Fenwick Tree Operations**: 
   - **Update**: For each element, its value is added to the Fenwick Tree at its compressed index.
   - **Query**: For each element, the sum of all elements up to its compressed index is queried from the Fenwick Tree.
5. **Result Calculation**: For each element, the result is the total sum minus the queried sum (sum of elements less than or equal to it), giving the sum of elements strictly greater than it.
6. **Output**: The results for all elements are printed in the required format.

This approach efficiently handles the problem constraints by leveraging coordinate compression and the Fenwick Tree to achieve optimal performance.
