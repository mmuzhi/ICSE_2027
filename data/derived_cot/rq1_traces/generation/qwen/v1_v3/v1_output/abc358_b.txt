=== ANSWER ===

To solve this problem, we need to determine the finish times for each person in line at a ticket booth where each person takes a fixed amount of time to purchase their ticket. The key observation here is that each person starts their purchase either immediately upon arrival (if the booth is free) or when the previous person finishes (if the booth is busy). 

### Approach
1. **Problem Analysis**: The problem involves processing a sequence of people arriving at a ticket booth, each requiring a fixed processing time (A seconds). The challenge is to compute the finish time for each person based on their arrival time and the availability of the booth.
2. **Intuition**: The solution hinges on tracking the time when the previous person finishes their transaction. If the current person arrives after the previous finish time, they start immediately upon arrival. Otherwise, they wait until the previous transaction is complete.
3. **Algorithm Selection**: We use a simple linear scan through the arrival times, maintaining a variable that keeps track of the last finish time. For each arrival, we compare it with the last finish time to determine the start time, then compute the finish time by adding the processing time (A).
4. **Complexity Analysis**: The algorithm processes each arrival time exactly once, resulting in a time complexity of O(N), where N is the number of people. This is optimal for the given constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = int(data[1])
    T = list(map(int, data[2:2+n]))
    
    finish_times = []
    current_finish = 0
    
    for t in T:
        if t >= current_finish:
            current_finish = t + A
        else:
            current_finish = current_finish + A
        finish_times.append(current_finish)
    
    for ft in finish_times:
        print(ft)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the number of people (N), the processing time per person (A), and the list of arrival times (T).
2. **Initialization**: We initialize a list to store the finish times of each person and a variable to keep track of the last finish time (starting at 0, indicating the booth is free initially).
3. **Processing Each Arrival**: For each arrival time in the list:
   - If the arrival time is greater than or equal to the last finish time, the person starts immediately, and their finish time is calculated as the arrival time plus the processing time (A).
   - Otherwise, the person starts when the booth becomes free (at the last finish time), and their finish time is the last finish time plus A.
4. **Output**: The finish times for all people are printed in the order of their arrival.

This approach efficiently computes the finish times by leveraging the sequence of arrival times and the fixed processing time, ensuring optimal performance with minimal computational overhead.
