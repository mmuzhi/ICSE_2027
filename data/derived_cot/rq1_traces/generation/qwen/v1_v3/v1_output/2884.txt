To solve this problem, we need to find the longest valid substring of a given string `word` such that none of its substrings are present in a given list of forbidden strings. A valid substring is defined as one that does not contain any forbidden substring within it. 

### Approach
1. **Problem Analysis**: The problem requires checking all possible substrings of the input string `word` to determine if they contain any forbidden substring. Given the constraints (string length up to 10^5 and forbidden strings up to 10^5), a brute-force approach is infeasible. Instead, we use a sliding window technique combined with a hash set to efficiently track forbidden substrings.

2. **Intuition**: The key observation is that any forbidden substring must end at some position in the string. For each position `i` in the string, we check all substrings ending at `i` (of lengths 1 to 10, since forbidden strings are at most 10 characters long). If any of these substrings is forbidden, we adjust the left boundary of our window to exclude it.

3. **Algorithm Selection**: 
   - **Sliding Window**: We maintain a window `[j, i]` where `i` is the current right pointer. The left pointer `j` is adjusted whenever a forbidden substring is found ending at `i`.
   - **Hash Set for Forbidden Substrings**: We precompute a set of hash values for all forbidden strings to allow O(1) lookups. This helps in quickly determining if a substring ending at `i` is forbidden.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n * 10) = O(n), where n is the length of the string. For each character in the string, we check up to 10 substrings (each up to 10 characters long).
   - **Space Complexity**: O(m * 10), where m is the number of forbidden strings. We store hash values for all forbidden strings up to 10 characters long.

### Solution Code
```python
class Solution:
    def longestValidSubstring(self, word: str, forbidden: List[str]) -> int:
        n = len(word)
        if not forbidden:
            return n
        
        forbidden_set = set()
        base1 = 131
        base2 = 131
        mod1 = 10**9 + 7
        mod2 = 10**9 + 9
        
        for f in forbidden:
            L = len(f)
            hash1 = 0
            hash2 = 0
            for char in f:
                hash1 = (hash1 * base1 + ord(char)) % mod1
                hash2 = (hash2 * base2 + ord(char)) % mod2
            forbidden_set.add((hash1, hash2))
        
        j = 0
        ans = 0
        
        for i in range(n):
            j_required = j
            for L in range(1, 11):
                if i + 1 < L:
                    break
                substr = word[i - L + 1:i + 1]
                hash1 = 0
                hash2 = 0
                for char in substr:
                    hash1 = (hash1 * base1 + ord(char)) % mod1
                    hash2 = (hash2 * base2 + ord(char)) % mod2
                if (hash1, hash2) in forbidden_set:
                    j_required = max(j_required, i - L + 2)
            j = max(j, j_required)
            ans = max(ans, i - j + 1)
        
        return ans
```

1. **Initialization**: We start by checking if there are any forbidden strings. If not, the entire string is valid, and we return its length.
2. **Hash Set Construction**: For each forbidden string, we compute its hash value using two different bases and moduli to minimize collisions. These hash values are stored in a set for quick lookup.
3. **Sliding Window**: We iterate over each character in the string (right pointer `i`). For each position, we check all possible substrings ending at `i` (lengths 1 to 10). If any substring's hash is found in the forbidden set, we adjust the left pointer `j` to exclude the forbidden substring.
4. **Result Calculation**: After adjusting `j`, the length of the current valid substring is `i - j + 1`. We keep track of the maximum length encountered during the iteration.

This approach efficiently narrows down the valid substrings by leveraging hash sets and a sliding window, ensuring optimal performance even for large input sizes.
