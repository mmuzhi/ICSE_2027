To solve this problem, we need to efficiently handle a series of queries that involve moving balls between boxes and counting the distinct colors in the destination box after each move. The challenge lies in performing these operations efficiently, especially given the constraints where both the number of boxes (N) and the number of queries (Q) can be as large as 200,000.

### Approach
1. **Problem Analysis**: The problem involves moving all balls from one box to another and then counting the distinct colors in the destination box. The key observation is that each ball has a color, and moving all balls from one box to another means merging their color sets. The naive approach of merging sets directly would be too slow for large inputs, so we use a small-to-large merging strategy to optimize performance.

2. **Intuition**: The small-to-large merging strategy ensures that each color is moved at most O(log N) times. This is because each time a color is moved, it is from a smaller set to a larger one, doubling the size of its set each time it is moved. This keeps the total number of operations manageable (O(N log N)).

3. **Algorithm Selection**: We maintain two arrays:
   - `s[i]` stores the set of distinct colors in box `i`.
   - `cnt[i]` stores the count of distinct colors in box `i`.
   For each query, if the source box is non-empty, we merge its set into the destination box's set using the small-to-large strategy. If the source set is smaller or equal to the destination set, we merge the source into the destination. Otherwise, we merge the destination into the source and then swap the sets to comply with the query direction.

4. **Complexity Analysis**: The total number of operations (insertions and membership checks) is O(N log N) due to the small-to-large merging strategy. Each operation within the merge is O(1) on average for set operations, making the overall complexity efficient for large inputs.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it)); q = int(next(it))
    colors = [int(next(it)) for _ in range(n)]
    s = [set() for _ in range(n+1)]
    cnt = [0] * (n+1)
    for i in range(1, n+1):
        s[i] = {colors[i-1]}
        cnt[i] = 1

    out_lines = []
    for _ in range(q):
        a = int(next(it)); b = int(next(it))
        if cnt[a] == 0:
            out_lines.append(str(cnt[b]))
        else:
            if cnt[a] <= cnt[b]:
                for color in s[a]:
                    if color not in s[b]:
                        s[b].add(color)
                        cnt[b] += 1
                s[a] = set()
                cnt[a] = 0
                out_lines.append(str(cnt[b]))
            else:
                for color in s[b]:
                    if color not in s[a]:
                        s[a].add(color)
                        cnt[a] += 1
                s[b] = s[a]
                cnt[b] = cnt[a]
                s[a] = set()
                cnt[a] = 0
                out_lines.append(str(cnt[b]))
                
    print("\n".join(out_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: We read the input values and initialize two arrays, `s` (to store sets of colors for each box) and `cnt` (to store the count of distinct colors in each box). Each box starts with one ball of a given color.

2. **Query Processing**: For each query, we check if the source box is empty. If it is, the result is simply the count of distinct colors in the destination box. If the source box is non-empty, we merge its set into the destination box's set:
   - If the source set is smaller or equal to the destination set, we merge the source into the destination.
   - Otherwise, we merge the destination into the source and then swap the sets to ensure the result is in the destination box.
   
3. **Output**: After processing each query, the result (count of distinct colors in the destination box) is stored and printed at the end.

This approach efficiently handles the merging operations using the small-to-large strategy, ensuring optimal performance for large inputs.
