To solve this problem, we need to determine the minimum number of operations required to make the sum of elements in an array (initially containing a single element `1`) greater than or equal to a given integer `k`. The allowed operations are either increasing any element by 1 or duplicating any element and adding it to the end of the array.

### Approach
1. **Problem Analysis**: The key observation is that the optimal strategy involves either increasing the initial element to a certain value and then duplicating it, or duplicating first and then increasing all elements. However, the minimal operations are achieved by first increasing the initial element to a value `x` and then duplicating it `n-1` times, where `n` is the total number of elements in the final array. The value `x` must be at least the ceiling of `k/n` to ensure the sum of the array elements is at least `k`.

2. **Insight**: For each possible number of elements `n` (from 1 to `k`), compute the minimal value `x` (using ceiling division) such that `n * x >= k`. The operations required for each `n` is then `(x - 1) + (n - 1)`, which simplifies to `x + n - 2`. The solution involves finding the minimum value of this expression across all possible `n`.

3. **Algorithm Selection**: Iterate over all possible values of `n` (number of elements) from 1 to `k`. For each `n`, compute `x` as the ceiling of `k/n`, then calculate the operations required. Track the minimum operations encountered during the iteration.

4. **Complexity Analysis**: The algorithm runs in O(k) time since it iterates from 1 to `k`. Given the constraint `k <= 10^5`, this approach is efficient and feasible for the problem.

### Solution Code
```python
class Solution:
    def minOperations(self, k: int) -> int:
        ans = float('inf')
        for n in range(1, k + 1):
            x = (k + n - 1) // n
            operations = x + n - 2
            if operations < ans:

        return ans
```

### Explanation
1. **Initialization**: Start with `ans` set to a very large value to represent the initial minimum operations.
2. **Iteration**: For each possible number of elements `n` (from 1 to `k`):

   - **Operations Calculation**: The total operations required for `n` elements is `x + n - 2`, derived from `(x - 1)` increments on the initial element and `(n - 1)` duplications.
3. **Tracking Minimum**: Update `ans` whenever a lower operation count is found during the iteration.
4. **Result**: After checking all possible values of `n`, `ans` holds the minimum operations required, which is returned as the result.

This approach efficiently explores all possible configurations of the array to find the optimal solution with minimal operations, leveraging mathematical insights to simplify the problem.
