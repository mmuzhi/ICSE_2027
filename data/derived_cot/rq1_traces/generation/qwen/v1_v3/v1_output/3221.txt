You are given a 0-indexed array mountain. Your task is to find all the peaks in the mountain array.
Return an array that consists of indices of peaks in the given array in any order.
Notes:

A peak is defined as an element that is strictly greater than its neighboring elements.
The first and last elements of the array are not a peak.

 
Example 1:

Input: mountain = [2,4,4]
Output: []
Explanation: mountain[0] and mountain[2] can not be a peak because they are first and last elements of the array.
mountain[1] also can not be a peak because it is not strictly greater than mountain[2].
So the answer is [].

Example 2:

Input: mountain = [1,4,3,8,5]
Output: [1,3]
Explanation: mountain[0] and mountain[4] can not be a peak because they are first and last elements of the array.
mountain[2] also can not be a peak because it is not strictly greater than mountain[3] and mountain[1].
But mountain [1] and mountain[3] are strictly greater than their neighboring elements.
So the answer is [1,3].

1 <= mountain[i] <= 100

=== ANSWER ===

To solve this problem, we need to identify all the peaks in a given mountain array. A peak is defined as an element that is strictly greater than its immediate neighbors. The first and last elements of the array are automatically excluded from consideration since they cannot have elements on both sides.

### Approach
1. **Problem Analysis**: The task involves scanning through the array from the second element to the second-to-last element. For each element in this range, we check if it is strictly greater than both its left and right neighbors. If it is, the index of that element is added to the result list.
2. **Intuition**: The solution leverages a straightforward iteration through the relevant portion of the array, checking each element against its neighbors. This approach efficiently narrows down the candidates for peaks by focusing only on elements that can potentially have both neighbors.
3. **Algorithm Selection**: The algorithm involves a single pass through the array, making it efficient with a time complexity of O(n), where n is the length of the array. This is optimal for the given constraints.
4. **Complexity Analysis**: The algorithm runs in linear time, O(n), and uses constant space, O(1), as it only requires a few variables to store intermediate results and the final output.

### Solution Code
```python
class Solution:
    def findPeaks(self, mountain: List[int]) -> List[int]:
        n = len(mountain)
        result = []
        for i in range(1, n-1):
            if mountain[i] > mountain[i-1] and mountain[i] > mountain[i+1]:
                result.append(i)
        return result
```

### Explanation
1. **Initialization**: The solution starts by determining the length of the input array and initializing an empty list to store the indices of the peaks.
2. **Iteration**: The loop runs from index 1 to the second-to-last element (inclusive). For each element at index `i`, it checks if the element is greater than both its left neighbor (at index `i-1`) and its right neighbor (at index `i+1`).
3. **Condition Check**: If the element at index `i` meets the peak condition, its index is added to the result list.
4. **Return Result**: After processing all elements, the list of peak indices is returned.

This approach efficiently identifies all peaks in the array by leveraging a simple linear scan and neighbor comparison, ensuring optimal performance and correctness.
