To solve this problem, we need to find the K-th smallest positive integer that is divisible by exactly one of two given integers N and M. The solution involves leveraging mathematical insights and binary search to efficiently determine the answer without resorting to brute-force methods, especially given the constraints where K can be as large as 10^10.

### Approach
1. **Problem Analysis**: The problem requires identifying numbers divisible by either N or M but not both. The key observation here is that the count of such numbers up to a certain value X can be derived using inclusion-exclusion principles. Specifically, the count is given by:
   - Numbers divisible by N but not M: floor(X/N) - floor(X/L)
   - Numbers divisible by M but not N: floor(X/M) - floor(X/L)
   Where L is the least common multiple (LCM) of N and M.
   
2. **Binary Search**: Given the large possible value of K, a binary search is employed to efficiently find the smallest X such that the total count of numbers up to X (divisible by exactly one of N or M) is at least K. The search space is bounded between 0 and a sufficiently large upper limit (K * 2 * max(N, M)), ensuring coverage of all potential candidates.

3. **Mathematical Formulation**: The LCM of N and M is computed first. For each midpoint in the binary search, the count of valid numbers is calculated by subtracting the common multiples (those divisible by both N and M) from the counts of multiples of N and M individually. This count is then compared to K to adjust the binary search bounds.

### Solution Code
```python
import math

def main():
    N, M, K = map(int, input().split())
    g = math.gcd(N, M)
    L = (N * M) // g

    low, high = 0, K * 2 * max(N, M)

    while low < high:
        mid = (low + high) // 2
        count_A = mid // N - mid // L
        count_B = mid // M - mid // L
        total = count_A + count_B
        if total >= K:
            high = mid
        else:
            low = mid + 1

    print(low)

if __name__ == '__main__':
```
