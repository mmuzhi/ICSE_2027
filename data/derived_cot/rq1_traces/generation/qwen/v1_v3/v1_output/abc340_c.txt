You will be given a competitive programming problem.
There is a single integer N written on a blackboard.
Takahashi will repeat the following series of operations until all integers not less than 2 are removed from the blackboard:

- Choose one integer x not less than 2 written on the blackboard.
- Erase one occurrence of x from the blackboard. Then, write two new integers \left \lfloor \dfrac{x}{2} \right\rfloor and \left\lceil \dfrac{x}{2} \right\rceil on the blackboard.
- Takahashi must pay x yen to perform this series of operations.

Here, \lfloor a \rfloor denotes the largest integer not greater than a, and \lceil a \rceil denotes the smallest integer not less than a.
What is the total amount of money Takahashi will have paid when no more operations can be performed?
It can be proved that the total amount he will pay is constant regardless of the order in which the operations are performed.

Input

The input is given from Standard Input in the following format:
N

Output

Print the total amount of money Takahashi will have paid, in yen.

Constraints


- 2 \leq N \leq 10^{17}

Sample Input 1

3

Sample Output 1

5

Here is an example of how Takahashi performs the operations:

- Initially, there is one 3 written on the blackboard.
- He chooses 3. He pays 3 yen, erases one 3 from the blackboard, and writes \left \lfloor \dfrac{3}{2} \right\rfloor = 1 and \left\lceil \dfrac{3}{2} \right\rceil = 2 on the blackboard.
- There is one 2 and one 1 written on the blackboard.
- He chooses 2. He pays 2 yen, erases one 2 from the blackboard, and writes \left \lfloor \dfrac{2}{2} \right\rfloor = 1 and \left\lceil \dfrac{2}{2} \right\rceil = 1 on the blackboard.
- There are three 1s written on the blackboard.
- Since all integers not less than 2 have been removed from the blackboard, the process is finished.

Takahashi has paid a total of 3 + 2 = 5 yen for the entire process, so print 5.

To solve this problem, we need to determine the total cost Takahashi will incur when repeatedly splitting integers on a blackboard until only integers less than 2 remain. The key observation is that the total cost is independent of the order of operations performed. 

### Approach
1. **Problem Analysis**: The problem involves breaking down a number \( N \) into smaller parts by splitting it into its floor and ceiling halves recursively until all parts are less than 2. Each split operation costs the value of the number being split. The challenge is to compute the total cost efficiently, especially given the constraint \( 2 \leq N \leq 10^{17} \).

2. **Insight**: The total cost can be derived by considering the sum of all numbers that are split during the process. The sum of all numbers in the decomposition tree (including leaves) is \( S(N) \), and since the leaves are all 1's, their sum is \( N \). Therefore, the total cost is \( S(N) - N \).

3. **Algorithm Selection**: We use a recursive approach with memoization to compute \( S(N) \). The function \( S(n) \) is defined as \( n \) plus the sum of \( S(\text{floor}(n/2)) \) and \( S(\text{ceil}(n/2)) \). The base case is \( S(1) = 1 \). Memoization is used to store computed values and avoid redundant calculations, ensuring efficiency.

4. **Complexity Analysis**: The number of distinct states (values of \( n \)) encountered during recursion is \( O(\log N) \) because each split reduces \( n \) by roughly half. For \( N = 10^{17} \), this results in about 60 distinct states, making the approach feasible.

### Solution Code
```python
memo = {}

def S(n):
    if n == 1:
        return 1
    if n in memo:
        return memo[n]
    a = n // 2
    b = (n + 1) // 2
    memo[n] = n + S(a) + S(b)
    return memo[n]

def main():
    import sys
    N = int(sys.stdin.readline())
    result = S(N) - N
    print(result)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Memoization**: The function `S(n)` uses a dictionary `memo` to store previously computed values of \( S(n) \) to avoid redundant calculations and improve efficiency.
2. **Recursive Breakdown**: For each \( n \), the function splits it into its floor and ceiling halves, recursively computes \( S \) for each half, and combines the results.
3. **Base Case**: When \( n = 1 \), the function returns 1, as no further splitting is needed.
4. **Result Calculation**: The main function reads the input \( N \), computes the total cost as \( S(N) - N \), and prints the result. This works because \( S(N) \) includes the sum of all numbers in the decomposition tree, and subtracting \( N \) (the sum of leaves) gives the total cost of all splits.

This approach efficiently computes the total cost by leveraging memoization and recursive decomposition, handling even very large values of \( N \) within feasible time and space complexity.
