You will be given a competitive programming problem.
There are N people participating in an event, and the transportation cost for the i-th person is A_i yen.
Takahashi, the organizer of the event, decided to set a maximum limit x for the transportation subsidy. The subsidy for person i will be \min(x, A_i) yen. Here, x must be a non-negative integer.
Given that Takahashi's budget is M yen, and he wants the total transportation subsidy for all N people to be at most M yen, what is the maximum possible value of the subsidy limit x?
If the subsidy limit can be made infinitely large, report that instead.

Input

The input is given from Standard Input in the following format:
N M
A_1 A_2 \ldots A_{N}

Output

Print the maximum value of the subsidy limit x that satisfies the budget condition, as an integer.
If the subsidy limit can be made infinitely large, print infinite instead.

Constraints


- 1 \leq N \leq 2 \times 10^5
- 1 \leq M \leq 2 \times 10^{14}
- 1 \leq A_i \leq 10^9
- All input values are integers.

Sample Input 1

4 8
1 3 2 4

Sample Output 1

2

If the subsidy limit is set to 3 yen, the total transportation subsidy for all N people is \min(3,1) + \min(3,3) + \min(3,2) + \min(3,4) = 9 yen, which exceeds the budget of 8 yen.
Therefore, the maximum possible value of the subsidy limit is 2 yen.

Sample Input 2

3 20
5 3 2

Sample Output 2

infinite

The subsidy limit can be made infinitely large.

To solve this problem, we need to determine the maximum possible value of the subsidy limit \( x \) such that the total transportation subsidy for all participants does not exceed the given budget \( M \). The subsidy for each participant is the minimum of \( x \) and their individual transportation cost \( A_i \). 

### Approach
1. **Problem Analysis**: The key observation is that if the sum of all individual transportation costs (when \( x \) is set to a value greater than or equal to the maximum cost) is less than or equal to \( M \), then \( x \) can be set to any value (including very large values), and the answer is "infinite". Otherwise, we need to find the largest integer \( x \) such that the total subsidy is within the budget \( M \).

2. **Intuition**: The total subsidy when \( x \) is set to a value \( k \) is calculated as the sum of all \( A_i \) that are less than or equal to \( k \), plus \( k \) multiplied by the number of \( A_i \) values greater than \( k \). This function is piecewise linear and increasing with \( k \), allowing us to use binary search to efficiently find the maximum \( k \) that satisfies the budget constraint.

3. **Algorithm Selection**: 
   - **Binary Search**: We perform a binary search on \( x \) from 0 to the maximum value in the list \( A \) (minus 1, since values beyond the maximum are not considered as they would sum to the total cost which is already checked to be greater than \( M \)).
   - **Prefix Sum Array**: To optimize the calculation of the total subsidy for any \( x \), we precompute a prefix sum array of the sorted list of transportation costs. This allows us to quickly compute the sum of elements less than or equal to \( x \) and the count of elements greater than \( x \).

4. **Complexity Analysis**: 
   - **Sorting**: Sorting the list of \( N \) elements takes \( O(N \log N) \) time.
   - **Binary Search**: The binary search runs in \( O(\log \text{max}(A)) \) time, and each step involves a bisect operation which is \( O(\log N) \). Thus, the total time complexity is \( O(N \log N + \log \text{max}(A) \cdot \log N) \), which is efficient for the given constraints.

### Solution Code
```python
import sys
import bisect

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    A = list(map(int, data[2:2+n]))
    
    total_sum = sum(A)
    if total_sum <= m:
        print("infinite")
    else:
        sorted_A = sorted(A)
        prefix = [0] * (n + 1)
        for i in range(1, n + 1):
            prefix[i] = prefix[i - 1] + sorted_A[i - 1]
        max_A = sorted_A[-1]
        low, high = 0, max_A - 1
        while low <= high:
            mid = (low + high) // 2
            p = bisect.bisect_right(sorted_A, mid)
            total_sub = prefix[p] + mid * (n - p)
            if total_sub <= m:
                low = mid + 1
            else:
                high = mid - 1
        print(high)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are \( N \) (number of participants) and \( M \) (budget). The remaining values are the transportation costs \( A_i \).

2. **Total Sum Check**: If the sum of all transportation costs is less than or equal to \( M \), the answer is "infinite" because setting \( x \) to any value (including very large ones) will not exceed the budget.

3. **Binary Search Setup**: If the total sum exceeds \( M \), we proceed to find the maximum \( x \) by binary search. The list \( A \) is sorted, and a prefix sum array is constructed to facilitate quick sum calculations for elements less than or equal to any given \( x \).

4. **Binary Search Execution**: The binary search checks each candidate \( x \) (mid value) by determining how many elements are less than or equal to \( x \) (using bisect) and calculates the total subsidy. If the total subsidy is within the budget, the search continues in the higher half; otherwise, it continues in the lower half.

5. **Output Result**: After the binary search completes, the highest valid \( x \) is printed. If no \( x \) is found (though \( x = 0 \) is always valid), the loop ensures that the result is correctly identified. 

This approach efficiently narrows down the possible values of \( x \) using binary search and leverages sorting and prefix sums for optimal performance.
