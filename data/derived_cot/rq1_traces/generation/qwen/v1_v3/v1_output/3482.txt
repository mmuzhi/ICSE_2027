Append words[i] to s.
The cost of operation is costs[i].

Return the minimum cost to make s equal to target. If it's not possible, return -1.
 
Example 1:

Input: target = "abcdef", words = ["abdef","abc","d","def","ef"], costs = [100,1,1,10,5]
Output: 7
Explanation:
The minimum cost can be achieved by performing the following operations:

Select index 2 and append "d" to s at a cost of 1, resulting in s = "abcd".
Select index 4 and append "ef" to s at a cost of 5, resulting in s = "abcdef".


Example 2:

Input: target = "aaaa", words = ["z","zz","zzz"], costs = [1,10,100]
Output: -1
Explanation:
It is impossible to make s equal to target, so we return -1.

 
Constraints:

1 <= target.length <= 5 * 10^4
1 <= words.length == costs.length <= 5 * 10^4
1 <= words[i].length <= target.length
The total sum of words[i].length is less than or equal to 5 * 10^4.
target and words[i] consist only of lowercase English letters.
1 <= costs[i] <= 10^4

To solve this problem, we need to find the minimum cost to construct the target string by concatenating words from a given list, where each word has an associated cost. The solution involves dynamic programming combined with efficient string matching using a rolling hash technique to minimize the number of operations.

### Approach
1. **Problem Analysis**: The goal is to build the target string by appending words from the given list any number of times, with each operation costing the word's associated cost. The challenge is to determine the minimum cost or return -1 if it's impossible to form the target string.

2. **Intuition**: We use dynamic programming (DP) where `dp[i]` represents the minimum cost to form the first `i` characters of the target string. For each position `i` in the target, we check all possible words that could end at `i` and update `dp[i]` accordingly.

3. **Optimization**: To efficiently check for matching words, we preprocess the words using a rolling hash technique. This allows us to quickly compare substrings of the target with words from the list without performing expensive string comparisons for every possible word at each position.

4. **Algorithm Selection**: 
   - **Dynamic Programming**: We initialize a DP array where each entry `dp[i]` is the minimum cost to form the first `i` characters.
   - **Rolling Hash**: We compute hash values for all words (except those of length 1) and store them. For each position in the target, we compute the hash of the relevant substring and compare it with the stored hashes of words, reducing the number of full string comparisons.

5. **Complexity Analysis**: The preprocessing step involves computing hashes for all words, which is O(L) per word, where L is the word length. The DP step involves iterating over each position in the target and checking all relevant word lengths, which in the worst case is O(n * m), where n is the target length and m is the number of words. However, the rolling hash significantly reduces the average number of comparisons.

2. **Hashing Words**: For each word, we compute two hash values using different bases and moduli to minimize collision chances. These hashes are stored in `words_by_length_hash` keyed by word length.

3. **Prefix Hashes**: We compute prefix hashes for the target string to enable efficient substring hash comparisons using the rolling hash technique.

4. **Dynamic Programming**: We initialize a DP array where each entry `dp[i]` is the minimum cost to form the first `i` characters. For each position `i`, we check:
   - **Single-character words**: If the current character exists in `char_costs`, update `dp[i]` using the cost from the previous position.
   - **Multi-character words**: For each word length, compute the hash of the relevant target substring. If the hash matches any word in `words_by_length_hash`, perform a string comparison to confirm the match and update `dp[i]` if beneficial.

5. **Result**: After processing all positions, `dp[n]` holds the minimum cost to form the entire target string. If it remains infinity, return -1 indicating it's impossible to form the target.
