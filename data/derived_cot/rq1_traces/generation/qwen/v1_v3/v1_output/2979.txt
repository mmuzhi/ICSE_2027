To solve this problem, we need to maximize the profit from selling houses on a number line to various buyers, each offering to buy a contiguous range of houses for a certain amount of gold. The key constraint is that no two buyers can purchase the same house, so we must select non-overlapping offers to maximize the total profit.

### Approach
1. **Problem Analysis**: The problem resembles the classic interval scheduling problem where we need to select non-overlapping intervals (offers) to maximize the sum of their values (gold). The challenge is to efficiently compute the maximum profit given the constraints (n and offers can be up to 10^5).

2. **Intuition**: We can use dynamic programming (DP) to solve this problem. The idea is to process each house from left to right, keeping track of the maximum profit achievable up to that house. For each house, we consider two possibilities:
   - **Not selling the house**: The profit remains the same as the previous house.
   - **Selling the house via an offer**: If there are offers ending at the current house, we calculate the profit by adding the current offer's gold to the profit from the house just before the offer starts (if any) and update our DP value if this results in a higher profit.

3. **Algorithm Selection**: 
   - **Preprocessing**: Group all offers by their end indices for quick access during DP computation.
   - **Dynamic Programming Array**: `dp[i]` represents the maximum profit achievable from houses 0 to i. Initialize `dp` with zeros.
   - **DP Update**: For each house `i`, start with `dp[i]` set to `dp[i-1]` (if `i > 0`), then for each offer ending at `i`, update `dp[i]` to the maximum of its current value and the sum of the offer's gold and the profit up to the house before the offer starts.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n + m), where n is the number of houses and m is the number of offers. We process each house once and each offer once.
   - **Space Complexity**: O(n) for storing the DP array and the offers grouped by end indices.

### Solution Code
```python
from typing import List

class Solution:
    def maximizeTheProfit(self, n: int, offers: List[List[int]]) -> int:
        offers_by_end = [[] for _ in range(n)]
        for s, e, g in offers:
            if e < n:
                offers_by_end[e].append((s, g))
        
        dp = [0] * n
        
        for i in range(n):
            if i > 0:
                dp[i] = dp[i-1]
            for s, g in offers_by_end[i]:
                prev_profit = dp[s-1] if s > 0 else 0
                candidate = prev_profit + g
                if candidate > dp[i]:
                    dp[i] = candidate
        
        return dp[-1]
```

### Explanation
1. **Preprocessing**: The offers are grouped into a list `offers_by_end` where each index represents a house, and the list contains all offers that end at that house. This allows efficient access during DP computation.
2. **Dynamic Programming Initialization**: The `dp` array is initialized to store the maximum profit up to each house, starting with zeros.
3. **DP Update**: For each house `i`:
   - The initial value of `dp[i]` is set to `dp[i-1]` (if `i > 0`), representing the scenario where the current house is not sold.
   - For each offer ending at `i`, the profit is calculated by adding the offer's gold to the profit up to the house before the offer starts (`s-1`). If this results in a higher profit, `dp[i]` is updated.
4. **Result Extraction**: The final result is found in `dp[-1]`, which contains the maximum profit achievable for all houses from 0 to `n-1`.

This approach efficiently computes the maximum profit by leveraging dynamic programming and careful preprocessing, ensuring optimal performance even for large input sizes.
