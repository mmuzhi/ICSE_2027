=== ANSWER ===

To solve this problem, we need to efficiently handle a series of queries on a dragon's body, where the head moves in specific directions, and the rest of the body parts follow the head's previous positions. The challenge lies in processing these movements and queries efficiently, especially given the constraints where the number of parts and queries can be very large.

### Approach
1. **Problem Analysis**: The dragon consists of N parts, with the head initially at (1, 0) and each subsequent part at (i, 0) for i from 2 to N. The head moves in one of four directions (R, L, U, D), and each subsequent part moves to the previous position of the part in front. The task is to answer queries about the current position of any part after processing a series of moves.

2. **Intuition**: The key observation is that the position of any part p at time t is determined by the head's position at time t - (p - 1). If t - (p - 1) is negative, the part is still in its initial position (p, 0). However, directly storing all positions for each part for every time step is infeasible due to high constraints.

3. **Optimization**: Instead of storing all positions, we use a deque (double-ended queue) to maintain the positions of all parts. The deque starts with the initial positions of all parts. For each head move, we update the deque by adding the new head position to the front and removing the last position if the deque exceeds the number of parts. This ensures that the deque always contains the latest positions of all parts in order.

4. **Complexity Analysis**: 
   - **Initialization**: O(N) time to initialize the deque with N elements.
   - **Move Handling**: Each move involves O(1) operations (appending to the front and possibly popping from the back).
   - **Query Handling**: Each query involves O(1) time to access the required position in the deque.
   - **Overall Complexity**: O(N + Q) time, which is efficient for the given constraints (N ≤ 10^6 and Q ≤ 2×10^5).

### Solution Code
```python
import collections

def main():
    import sys
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    q = int(data[1])
    dq = collections.deque()
    for i in range(1, n+1):
        dq.append((i, 0))
    
    index = 2
    out_lines = []
    for _ in range(q):
        t = data[index]; index += 1
        if t == '1':
            direction = data[index]; index += 1
            x, y = dq[0]
            if direction == 'R':
                new_head = (x + 1, y)
            elif direction == 'L':
                new_head = (x - 1, y)
            elif direction == 'U':
                new_head = (x, y + 1)
            elif direction == 'D':
                new_head = (x, y - 1)
            dq.appendleft(new_head)
            if len(dq) > n:
                dq.pop()
        else:
            p = int(data[index]); index += 1
            x, y = dq[p-1]
            out_lines.append(f"{x} {y}")
    
    for line in out_lines:
        print(line)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The deque is initialized with the initial positions of all parts, where part 1 is at (1, 0), part 2 at (2, 0), and so on up to part N.
2. **Processing Moves**: For each move query, the head's position is updated based on the direction (R, L, U, D). The new head position is added to the front of the deque, and if the deque size exceeds N, the last position is removed to maintain the deque size.
3. **Processing Queries**: For each part query, the position is retrieved from the deque at the index corresponding to the part number (0-based index). The result is collected in a list and printed at the end.
4. **Efficiency**: The deque operations (append left and pop) are O(1) on average, making the solution efficient for large inputs. The space complexity is O(N) for storing the deque.
