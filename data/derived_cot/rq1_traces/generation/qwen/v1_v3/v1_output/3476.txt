To solve this problem, we need to determine the minimum number of operations required to make all elements in an array divisible by 3. Each operation allows us to either add or subtract 1 from any element of the array. 

### Approach
1. **Problem Analysis**: For each element in the array, we check its remainder when divided by 3. If the remainder is 0, the element is already divisible by 3, and no operations are needed. If the remainder is 1 or 2, we can either adjust the element by subtracting the remainder (for remainder 1) or adding (3 - remainder) (for remainder 2). The minimal number of operations for any non-divisible element is 1, as either subtracting 1 (for remainder 1) or adding 1 (for remainder 2) will make it divisible by 3.
2. **Algorithm Selection**: Iterate through each element in the array. For each element, if it is not divisible by 3, increment a counter by 1. The total count after processing all elements gives the answer.
3. **Complexity Analysis**: The algorithm processes each element exactly once, resulting in a time complexity of O(n), where n is the number of elements in the array. This is optimal for the given constraints.

### Solution Code
```python
class Solution:
    def minimumOperations(self, nums: List[int]) -> int:
        return sum(1 for n in nums if n % 3 != 0)
```

### Explanation
- **Problem Analysis**: The problem requires transforming each element in the array to a multiple of 3 with the least number of operations. Each operation can either add or subtract 1 from an element.
- **Insight**: For any number not divisible by 3, the remainder when divided by 3 is either 1 or 2. In both cases, one operation (either subtract 1 for remainder 1 or add 1 for remainder 2) suffices to make the number divisible by 3.
- **Algorithm**: The solution involves counting how many elements in the array are not divisible by 3. Each such element requires exactly one operation to make it divisible by 3.
- **Complexity**: The algorithm efficiently checks each element once, leading to a linear time complexity O(n), which is optimal for the problem constraints (array size up to 50). The space complexity is O(1) as no additional data structures are used.
