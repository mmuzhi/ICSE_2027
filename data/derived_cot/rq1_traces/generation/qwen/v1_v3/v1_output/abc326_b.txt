=== ANSWER ===

To solve this problem, we need to find the smallest three-digit number that is greater than or equal to a given integer N, where the product of the hundreds and tens digits equals the ones digit. This is known as a 326-like number.

### Approach
1. **Problem Analysis**: The task involves checking each number starting from N upwards to determine if it meets the 326-like condition. The condition requires that the product of the hundreds and tens digits of the number must equal the ones digit. For example, the number 326 satisfies this condition because 3 (hundreds digit) multiplied by 2 (tens digit) equals 6 (ones digit).
2. **Algorithm Selection**: The straightforward approach is to iterate through each number starting from N and check the condition for each number. This method is efficient given the constraints (N is between 100 and 919), as the loop will terminate once the first valid number is found.
3. **Optimization**: Since the numbers are three-digit, extracting the hundreds, tens, and ones digits is done through simple arithmetic operations. The loop runs from N to 999, ensuring that we cover all possible three-digit numbers if necessary.

### Solution Code
```python
def main():
    N = int(input().strip())
    num = N
    while num < 1000:
        h = num // 100
        t = (num % 100) // 10
        o = num % 10
        if h * t == o:
            print(num)
            return
        num += 1

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input integer N is read from standard input.
2. **Loop Through Numbers**: Starting from N, each subsequent number is checked to see if it meets the 326-like condition. The loop continues until a valid number is found.
3. **Digit Extraction**: For each number, the hundreds digit is obtained by integer division by 100, the tens digit by integer division of the remainder (after dividing by 100) by 10, and the ones digit by taking the remainder when divided by 10.
4. **Condition Check**: The product of the hundreds and tens digits is compared to the ones digit. If they match, the number is printed, and the program exits. Otherwise, the loop continues to the next number.
5. **Termination**: The loop is bounded by 999 to ensure it covers all possible three-digit numbers, though the problem guarantees that a solution exists within the constraints.

This approach efficiently checks each number in sequence, ensuring the smallest valid 326-like number greater than or equal to N is found quickly.
