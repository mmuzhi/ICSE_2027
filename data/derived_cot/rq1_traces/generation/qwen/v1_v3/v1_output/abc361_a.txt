=== ANSWER ===

To solve this problem, we need to insert a given integer X immediately after the K-th element in a sequence of integers A. The solution involves reading the input values, manipulating the sequence by inserting X at the correct position, and then printing the resulting sequence.

### Approach
1. **Read Input Values**: The first line of input contains three integers: N (length of sequence A), K (position after which to insert X), and X (the integer to insert). The second line contains N integers representing the sequence A.
2. **Insert X into Sequence**: The sequence B is constructed by splitting the original sequence A into two parts: the first K elements (from index 0 to K-1) and the remaining elements (from index K to the end). The integer X is then inserted between these two parts.
3. **Output the Result**: The resulting sequence B is printed as a space-separated list.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    k = int(data[1])
    x = int(data[2])
    A = list(map(int, data[3:3+n]))
    
    B = A[:k] + [x] + A[k:]
    
    print(" ".join(map(str, B)))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once and split into a list of strings. The first three elements are converted to integers representing N, K, and X. The remaining elements are converted to integers to form the sequence A.
2. **Constructing Sequence B**: The sequence B is created by concatenating three parts: the first K elements of A, the integer X, and the remaining elements of A starting from the K-th position.
3. **Output**: The resulting sequence B is converted to a string where elements are separated by spaces and printed.

This approach efficiently handles the insertion by leveraging Python's list slicing, which allows for easy manipulation of sequences. The solution is optimal with a time complexity of O(N) due to the slicing operations, which are linear in the size of the sequence. The space complexity is also O(N) as we create a new list for the result.
