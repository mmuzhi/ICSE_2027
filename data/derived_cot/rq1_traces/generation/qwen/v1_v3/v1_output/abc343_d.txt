=== ANSWER ===

To solve this problem, we need to efficiently track the number of distinct score values among N players after each event in a sequence of T events. Each event increases a specific player's score by a given amount. The challenge lies in maintaining the count of distinct scores dynamically as each event occurs, without resorting to a brute-force approach that would be too slow for large inputs.

### Approach
1. **Initialization**: Start with all players having a score of 0. Use a frequency dictionary to keep track of how many players have each score. Initially, all N players have score 0, so the frequency dictionary starts with `{0: N}` and the distinct count is 1.
2. **Processing Events**: For each event, update the specified player's score by adding the given points. Before updating, decrement the frequency of the old score. If the frequency drops to zero, it means no players have this score anymore, so the distinct count is decremented by one.
3. **Updating New Score**: After updating the player's score, increment the frequency of the new score. If this was the first occurrence of the new score, the distinct count is incremented by one.
4. **Output**: After processing each event, record the current distinct count. After all events are processed, output the recorded distinct counts for each event.

This approach efficiently maintains the distinct count by leveraging a frequency dictionary, ensuring each operation (updating and checking frequencies) is handled in constant time on average, making the solution suitable for large inputs.

### Solution Code
```python
import sys
from collections import defaultdict

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    t = int(data[1])
    events = []
    index = 2
    for i in range(t):
        a = int(data[index])
        b = int(data[index + 1])
        index += 2
        events.append((a, b))
        
    scores = [0] * (n + 1)
    freq = defaultdict(int)
    freq[0] = n
    distinct_count = 1
    
    out_lines = []
    
    for a, b in events:
        old = scores[a]
        new_score = old + b
        scores[a] = new_score
        
        freq[old] -= 1
        if freq[old] == 0:
            distinct_count -= 1
            
        freq[new_score] += 1
        if freq[new_score] == 1:
            distinct_count += 1
            
        out_lines.append(str(distinct_count))
        
    print("\n".join(out_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The code reads the input values and initializes the scores for all players to 0. The frequency dictionary is set up with an initial entry for score 0, indicating all N players start with this score.
2. **Event Processing**: For each event, the code updates the specified player's score. The old score's frequency is decremented, and if it reaches zero, the distinct count is reduced. The new score's frequency is incremented, and if it was not present before, the distinct count is increased.
3. **Output Handling**: After processing each event, the current distinct count is recorded. After all events are processed, the results are printed line by line.

This method efficiently tracks the distinct scores after each event by dynamically updating the frequency dictionary and adjusting the distinct count based on the changes in scores, ensuring optimal performance even for large inputs.
