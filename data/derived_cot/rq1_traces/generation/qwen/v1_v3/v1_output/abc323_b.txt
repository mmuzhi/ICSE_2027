To solve this problem, we need to determine the ranking of players in a round-robin tournament based on their wins. The ranking is primarily determined by the number of wins each player has, with more wins leading to a higher rank. If two players have the same number of wins, the player with the smaller player number (1-indexed) is ranked higher.

### Approach
1. **Problem Analysis**: The input consists of N strings, each representing a player's match results against all other players. Each string includes '-' for the diagonal (self-match), 'o' for a win, and 'x' for a loss. The goal is to count the number of 'o's in each player's string (excluding the diagonal) to determine their wins.
2. **Intuition**: The solution involves parsing each player's string to count their wins. After calculating the wins for all players, we sort them primarily by the number of wins in descending order. For players with the same number of wins, we sort them by their player numbers in ascending order (since a smaller player number is better).
3. **Algorithm Selection**: We use a straightforward approach where we iterate over each player's string, count the 'o's (excluding the diagonal), and store the wins. Then, we sort the players based on their wins and player numbers using a custom sorting key.
4. **Complexity Analysis**: The algorithm involves two nested loops to count wins (O(N^2)) and sorting (O(N log N)), which is efficient given the constraint N ≤ 100.

    for i in range(n):
        s = data[1+i]
        for j in range(n):
            if i == j:
                continue
            if s[j] == 'o':
                wins[i] += 1
    
    players = [(wins[i], i) for i in range(n)]
    players.sort(key=lambda x: (-x[0], x[1]))
    
    result = [str(i+1) for _, i in players]
    print(" ".join(result))

if __name__ == '__main__':
    main()

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line is the number of players (N), followed by N strings representing each player's match results.
2. **Counting Wins**: For each player (indexed from 0 to N-1), we examine their string. The diagonal element (where index i equals j) is skipped. For each 'o' found, the player's win count is incremented.
3. **Sorting Players**: Players are sorted based on their win counts in descending order. If two players have the same number of wins, they are sorted by their original indices (0-indexed) in ascending order, which corresponds to their 1-indexed player numbers.
4. **Output**: The sorted list of players is converted to 1-indexed player numbers and printed in the required order.

This approach efficiently processes the input, calculates the necessary statistics, and sorts the players to determine their ranks based on wins and player numbers. The solution is optimal for the given constraints.
