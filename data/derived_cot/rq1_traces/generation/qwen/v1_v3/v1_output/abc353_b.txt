The AtCoder amusement park has an attraction that can accommodate K people. Now, there are N groups lined up in the queue for this attraction.
The i-th group from the front (1\leq i\leq N) consists of A_i people. For all i (1\leq i\leq N), it holds that A_i \leq K.
Takahashi, as a staff member of this attraction, will guide the groups in the queue according to the following procedure.
Initially, no one has been guided to the attraction, and there are K empty seats.

- If there are no groups in the queue, start the attraction and end the guidance.
- Compare the number of empty seats in the attraction with the number of people in the group at the front of the queue, and do one of the following:
- If the number of empty seats is less than the number of people in the group at the front, start the attraction. Then, the number of empty seats becomes K again.
- Otherwise, guide the entire group at the front of the queue to the attraction. The front group is removed from the queue, and the number of empty seats decreases by the number of people in the group.


- Go back to step 1.

Here, no additional groups will line up after the guidance has started. Under these conditions, it can be shown that this procedure will end in a finite number of steps.
Determine how many times the attraction will be started throughout the guidance.

Input

The input is given from Standard Input in the following format:
N K
A_1 A_2 \ldots A_N

Output

Print the answer.

Constraints


- 1\leq N\leq 100
- 1\leq K\leq 100
- 1\leq A_i\leq K\ (1\leq i\leq N)
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to simulate the process of guiding groups through an attraction with a fixed capacity, counting how many times the attraction needs to be started. The attraction can accommodate K people, and each group in the queue has a certain number of people. The key is to follow the given procedure step-by-step, ensuring we correctly count each instance the attraction is started.

### Approach
1. **Initialization**: Read the number of groups (N) and the attraction's capacity (K), followed by the list of group sizes.
2. **Simulation Loop**: Use a loop to process each group in the queue until it is empty.
   - **Check Capacity**: For each group at the front of the queue, compare the current empty seats with the group size.
     - If the empty seats are less than the group size, start the attraction (increment the count and reset empty seats to K). The group remains in the queue.
     - Otherwise, guide the entire group into the attraction (decrement empty seats by the group size and remove the group from the queue).
3. **Final Start**: After processing all groups, the attraction must be started one final time if the queue is empty, as per the problem's procedure.
4. **Output**: Print the total count of attraction starts.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    k = int(data[1])
    A = list(map(int, data[2:2+n]))
    
    count = 0
    empty = k
    queue = A[:]
    
    while queue:
        if empty < queue[0]:
            count += 1
            empty = k
        else:
            empty -= queue[0]
            queue.pop(0)
            
    count += 1
    print(count)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the number of groups (N), the attraction's capacity (K), and the list of group sizes.
2. **Initialization**: The `count` variable keeps track of the number of times the attraction is started. The `empty` variable represents the current number of empty seats in the attraction, initialized to K. The `queue` is a list containing all group sizes.
3. **Processing Groups**: The loop continues as long as there are groups in the queue. For each group:
   - If the group size exceeds the current empty seats, the attraction is started (`count` increases by 1, and `empty` resets to K). The group remains in the queue.
   - Otherwise, the group is guided into the attraction (`empty` is reduced by the group size, and the group is removed from the queue).
4. **Final Start**: After processing all groups, the attraction is started one last time (`count` increases by 1) to handle the empty queue condition.
5. **Output**: The total count of attraction starts is printed.

This approach efficiently simulates the given procedure, ensuring accurate counting of attraction starts while adhering to the problem constraints. The use of a list for the queue allows straightforward manipulation, and the loop ensures all groups are processed correctly.
