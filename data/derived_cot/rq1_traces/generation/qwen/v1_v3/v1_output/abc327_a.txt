=== ANSWER ===

To solve this problem, we need to determine if there are any adjacent occurrences of the characters 'a' and 'b' in a given string. The order of 'a' and 'b' does not matter; thus, both "ab" and "ba" sequences in adjacent positions should result in a "Yes" response.

### Approach
1. **Problem Analysis**: The problem requires checking each adjacent pair of characters in the string to see if they form either "ab" or "ba". If any such pair is found, we immediately return "Yes". If no such pair is found after checking all adjacent pairs, we return "No".
2. **Intuition**: By iterating through the string from the first character to the second last character, we can compare each character with its next character. If the pair is "ab" or "ba", we can conclude the answer as "Yes".
3. **Algorithm Selection**: The solution involves a simple linear scan through the string. For each character at position `i`, we check the next character at position `i+1`. If the pair matches "ab" or "ba", we print "Yes" and exit. If the loop completes without finding such a pair, we print "No".
4. **Complexity Analysis**: The algorithm runs in O(N) time, where N is the length of the string, because each character is checked exactly once. The space complexity is O(1) as we only use a constant amount of additional space.

### Solution Code
```python
def main():
    N = int(input().strip())
    S = input().strip()
    for i in range(N - 1):
        if S[i] == 'a' and S[i+1] == 'b':
            print("Yes")
            return
        elif S[i] == 'b' and S[i+1] == 'a':
            print("Yes")
            return
    print("No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The code starts by reading the integer `N` (the length of the string) and then the string `S` itself.
2. **Checking Adjacent Pairs**: The loop runs from the first character to the second last character. For each character, it checks if the current character and the next one form either "ab" or "ba".
3. **Immediate Check**: If either sequence is found, the code prints "Yes" and exits immediately, as the problem only requires checking for the existence of at least one such pair.
4. **Final Check**: If the loop completes without finding any valid pairs, the code prints "No".

This approach efficiently checks all possible adjacent pairs in linear time, ensuring optimal performance for the given constraints.
