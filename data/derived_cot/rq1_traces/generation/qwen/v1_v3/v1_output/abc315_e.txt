=== PROMPT ===
You will be given a competitive programming problem.
We have N books numbered 1 to N.
Book i assumes that you have read C_i books, the j-th of which is book P_{i,j}: you must read all these C_i books before reading book i.
Here, you can read all the books in some order.
You are trying to read the minimum number of books required to read book 1.
Print the numbers of the books you must read excluding book 1 in the order they should be read. Under this condition, the set of books to read is uniquely determined.
If there are multiple reading orders that satisfy the condition, you may print any of them.

Input

The input is given from Standard Input in the following format:
N
C_1 P_{1,1} \ldots P_{1,C_1}
C_2 P_{2,1} \ldots P_{2,C_2}
\vdots
C_N P_{N,1} \ldots P_{N,C_N}

Output

Print the numbers of the books you must read to read book 1 in the order they should be read, with spaces in between.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 0 \leq C_i < N
- \sum_{i=1}^{N} C_i \leq 2 \times 10^5
- C_1 \geq 1
- 1 \leq P_{i,j} \leq N
- P_{i,j} \neq P_{i,k} for 1 \leq j < k \leq C_i.
- It is possible to read all the books.

Sample Input 1

6
3 2 3 4
2 3 5
0
1 5
0
0

Sample Output 1

5 3 4 2

To read book 1, you must read books 2,3,4; to read book 2, you must read books 3,5; to read book 4, you must read book 5. To read books 3,5,6, you do not have to read any other books.
For example, if you read books 5,3,4,2 in this order, you can read book 1. This is a correct answer, because you will never be able to read book 1 with three or fewer books read. As another example, reading books 3,5,4,2 in this order also allows you to read book 1 with 4 books read.

To solve this problem, we need to determine the minimum set of books that must be read before book 1, along with the order in which they should be read. The solution involves analyzing the dependencies between books and ensuring that all prerequisites are satisfied before reading book 1.

### Approach
1. **Problem Analysis**: The problem requires us to find all books that are prerequisites (direct or indirect) of book 1. These books must be read in an order that satisfies all their dependencies. The solution involves modeling the dependencies as a directed acyclic graph (DAG) and performing a topological sort to determine the reading order.

2. **Intuition**: The key observation is that the set of books required to read book 1 forms a closed subgraph in the dependency graph. We can start from book 1 and traverse its prerequisites (and their prerequisites) to collect all necessary books. Once we have this set, we can perform a topological sort to determine the reading order.

3. **Algorithm Selection**: 
   - **Breadth-First Search (BFS)**: Used to collect all prerequisite books starting from book 1.
   - **Kahn's Algorithm for Topological Sort**: Used to determine the order in which the collected books should be read, ensuring all dependencies are satisfied.

4. **Complexity Analysis**: 
   - **BFS**: O(N + E), where N is the number of books and E is the total number of dependencies. This is efficient given the constraints (N and E up to 200,000).
   - **Topological Sort (Kahn's Algorithm)**: Also O(N + E) as it processes each node and edge once. The overall complexity is linear with respect to the input size, making it suitable for large inputs.

### Solution Code
```python
from collections import deque
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    n = int(data[0].strip())
    lines = data[1:1+n]
    
    prereq = [[] for _ in range(n+1)]
    reverse_graph = [[] for _ in range(n+1)]
    
    index = 1
    for i in range(1, n+1):
        parts = lines[index].split()
        index += 1
        if not parts:
            continue
        c = int(parts[0])
        if c == 0:
            prereq[i] = []
        else:
            prereq[i] = list(map(int, parts[1:1+c]))
        
        for p in prereq[i]:
            reverse_graph[p].append(i)
    
    required = set()
    q = deque([1])
    while q:
        cur = q.popleft()
        for p in prereq[cur]:
            if p not in required:
                required.add(p)
                q.append(p)
    
    in_degree = {}
    for node in required:
        in_degree[node] = len(prereq[node])
    
    q = deque()
    for node in required:
        if in_degree[node] == 0:
            q.append(node)
    
    result = []
    while q:
        cur = q.popleft()
        result.append(str(cur))
        for neighbor in reverse_graph[cur]:
            if neighbor in required:
                in_degree[neighbor] -= 1
                if in_degree[neighbor] == 0:
                    q.append(neighbor)
    
    print(" ".join(result))

if __name__ == "__main__":
    main()
```

2. **Building Dependency Graph**: The `prereq` array stores the direct prerequisites for each book, while the `reverse_graph` is constructed to map each prerequisite to the books that depend on it.
3. **BFS for Prerequisite Collection**: Starting from book 1, BFS is used to traverse all prerequisite books (direct and indirect), collecting them into the set `required`.
4. **Topological Sort**: Using Kahn's algorithm, the books in `required` are processed in an order that ensures all dependencies are satisfied. The algorithm starts with books that have no remaining prerequisites and processes them, reducing the in-degree of dependent books until all are processed.
5. **Output**: The result is printed as the sequence of books to read, excluding book 1, in the order determined by the topological sort.
