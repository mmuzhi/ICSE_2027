Notes on bitwise XOR
The bitwise XOR of non-negative integers A and B, denoted as A \oplus B, is defined as follows:
- In the binary representation of A \oplus B, the digit at the 2^k (k \geq 0) position is 1 if and only if exactly one of the digits at the 2^k position in the binary representations of A and B is 1; otherwise, it is 0.
For example, 3 \oplus 5 = 6 (in binary: 011 \oplus 101 = 110).
In general, the bitwise XOR of k integers p_1, \dots, p_k is defined as (\cdots ((p_1 \oplus p_2) \oplus p_3) \oplus \cdots \oplus p_k).  It can be proved that this is independent of the order of p_1, \dots, p_k.

To solve this problem, we need to compute the sum of the XORs of all contiguous subarrays of length at least 2 in a given sequence. Given the constraints, a brute-force approach is infeasible, so we use a bitwise approach to efficiently calculate the result.

### Approach
1. **Bitwise Analysis**: For each bit position (from 0 to 30), we determine how many contiguous subarrays of length ≥2 have an odd number of elements with that bit set. The contribution of each bit to the total sum is then calculated as the number of such subarrays multiplied by \(2^{\text{bit position}}\).

2. **Prefix XOR Array**: For each bit position, we construct a prefix XOR array where each element at index \(i\) represents the XOR of the first \(i\) elements' bits at that position. This helps in efficiently computing the XOR of any subarray.

3. **Frequency Counting**: Using a frequency array, we count how many times each possible XOR value (0 or 1) occurs in the prefix array from a certain starting index to the end. This allows us to quickly determine how many subarrays starting at a given index have an odd number of set bits for the current bit position.

4. **Sliding Window**: As we iterate through each starting index, we adjust the frequency array by removing elements that are no longer in the current window, ensuring efficient counting without recalculating from scratch each time.

```python
        return
    n = int(data[0])
    arr = list(map(int, data[1:1+n]))
    max_bits = 31
    total_sum = 0
    for k in range(max_bits):
        Y = [0] * (n + 1)
        for i in range(1, n + 1):
            bit_val = (arr[i - 1] >> k) & 1
            Y[i] = Y[i - 1] ^ bit_val
        freq = [0, 0]
        for idx in range(2, n + 1):
            freq[Y[idx]] += 1
        count_for_bit = 0
        for i in range(0, n - 1):
            if Y[i] == 0:
                count_for_bit += freq[1]
            else:
                count_for_bit += freq[0]
            if i + 2 <= n:
                freq[Y[i + 2]] -= 1
        total_sum += count_for_bit * (1 << k)
    print(total_sum)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints.
2. **Bitwise Processing**: For each bit position (0 to 30), the code processes the array to count subarrays with an odd number of set bits at that position.
3. **Prefix XOR Array**: The array `Y` is constructed such that `Y[i]` holds the XOR of the first `i` elements' bits at the current position. This allows the XOR of any subarray to be computed as `Y[j] ^ Y[i]`.
4. **Frequency Array**: The frequency array counts occurrences of 0 and 1 in the prefix array from index 2 to `n`. This helps in quickly determining how many subarrays starting at a given index have an odd XOR value for the current bit.
5. **Sliding Window Adjustment**: As the starting index moves, the frequency array is updated by removing elements that are no longer relevant, ensuring the count remains accurate for the current window.
6. **Sum Calculation**: The total sum is computed by summing the contributions from each bit position, where each contribution is the count of valid subarrays multiplied by \(2^{\text{bit position}}\).

This approach efficiently breaks down the problem into manageable parts using bitwise operations and sliding window techniques, ensuring optimal performance even for large input sizes.
