Your task is to split nums into subarrays such that the total cost of the subarrays is maximized, ensuring each element belongs to exactly one subarray.

cost(0, i_1) + cost(i_1 + 1, i_2) + ... + cost(i_k − 1 + 1, n − 1)
Return an integer denoting the maximum total cost of the subarrays after splitting the array optimally.
Note: If nums is not split into subarrays, i.e. k = 1, the total cost is simply cost(0, n - 1).

Output: 10
Explanation:

To solve this problem, we need to maximize the total cost of subarrays obtained by splitting the given array optimally. The cost of a subarray is defined with alternating signs starting with a positive sign for the first element. The challenge is to determine the best way to split the array into subarrays such that the sum of their costs is maximized.

### Approach
1. **Dynamic Programming (DP) Setup**: We use a dynamic programming approach where we maintain two states for each position in the array:
   - `last0`: The maximum total cost up to the current index if the current element is taken with a positive sign.
   - `last1`: The maximum total cost up to the current index if the current element is taken with a negative sign.
   
2. **Initialization**: For the first element, the only possible state is positive (`last0 = nums[0]`), and the negative state is invalid (`last1 = -infinity`).

3. **Iteration**: For each subsequent element, we compute the new states based on the previous states:
   - **State0 (Positive)**: This can be achieved either by continuing the previous subarray (if the previous state was negative) or by splitting the array before the current element (taking the best of the previous states and adding the current element).
   - **State1 (Negative)**: This can only be achieved by continuing the previous subarray (if the previous state was positive) and subtracting the current element.

4. **Result Extraction**: After processing all elements, the result is the maximum value between the two final states (`last0` and `last1`).
