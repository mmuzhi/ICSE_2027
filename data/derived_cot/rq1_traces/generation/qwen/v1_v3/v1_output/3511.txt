=== ANSWER ===

To solve this problem, we need to determine the winner of a game between two players, Alice and Bob, who take turns picking up coins of values 75 and 10. Each turn, a player must pick coins totaling exactly 115. The player who cannot make a move loses the game. Both players play optimally.

### Approach
1. **Problem Analysis**: The key observation is that the only way to make a total of 115 with the given coins (75 and 10) is to use one 75-coin and four 10-coins. This combination is the only solution to the equation \(75a + 10b = 115\) for non-negative integers \(a\) and \(b\).
2. **Game Simulation**: The game proceeds in turns where each player must use one 75-coin and four 10-coins if available. The total number of such moves possible is the minimum of the number of 75-coins (x) and the number of 10-coins divided by 4 (y // 4).
3. **Determine Winner**: If the total number of moves (m) is odd, Alice wins because she makes the last move, leaving Bob unable to move. If m is even, Bob wins because he makes the last move, leaving Alice unable to move.

### Solution Code
```python
class Solution:
    def losingPlayer(self, x: int, y: int) -> str:
        m = min(x, y // 4)
        return "Alice" if m % 2 == 1 else "Bob"
```

### Explanation
1. **Input Handling**: The function takes two integers, x (number of 75-coins) and y (number of 10-coins).
2. **Calculate Moves**: The variable `m` is computed as the minimum of x and y divided by 4 (since each move requires four 10-coins). This gives the total number of complete moves possible.
3. **Determine Winner**: If `m` is odd, Alice wins because she will make the last move. If `m` is even, Bob wins as he makes the last move. The result is returned accordingly.

This approach efficiently checks the game's outcome by leveraging simple arithmetic operations and logical conditions, ensuring optimal play from both players.
