=== ANSWER ===

To solve this problem, we need to repeatedly remove the leftmost occurrence of the substring "ABC" from a given string until no more such substrings exist. The challenge is to efficiently process the string in linear time, especially given the constraint that the string can be as long as 200,000 characters.

### Approach
The approach involves using a stack to simulate the removal process. The stack helps in efficiently checking and removing the leftmost "ABC" substring by maintaining the characters of the string in order. Here's the step-by-step breakdown:

1. **Stack Initialization**: We start with an empty stack.
2. **Character Processing**: For each character in the input string, we push it onto the stack.
3. **Check for "ABC"**: After each push, we check if the top three characters of the stack form the substring "ABC". If they do, we remove these three characters from the stack.
4. **Result Construction**: Once all characters are processed, the remaining characters in the stack form the final result string.

This approach ensures that we efficiently remove each "ABC" substring in a left-to-right manner by leveraging the stack's LIFO (Last-In-First-Out) property, which allows us to handle the removals in a single pass through the string.

### Solution Code
```python
s = input().strip()
stack = []
for c in s:
    stack.append(c)
    if len(stack) >= 3 and stack[-3:] == ['A', 'B', 'C']:
        stack.pop()
        stack.pop()
        stack.pop()
print(''.join(stack))
```

### Explanation
1. **Stack Processing**: The algorithm processes each character of the input string sequentially. Each character is pushed onto the stack.
2. **Substring Check**: After each push, the algorithm checks if the last three characters in the stack form "ABC". If they do, these three characters are popped from the stack, effectively removing them from the string.
3. **Efficiency**: The stack operations (push and pop) are O(1) per character, leading to an overall O(n) time complexity, where n is the length of the input string. This is optimal for the given constraints.
4. **Result Output**: After processing all characters, the remaining characters in the stack are joined into a string, which is printed as the final result.

This method efficiently handles the removal of all leftmost "ABC" substrings in a single pass, ensuring optimal performance even for large input sizes.
