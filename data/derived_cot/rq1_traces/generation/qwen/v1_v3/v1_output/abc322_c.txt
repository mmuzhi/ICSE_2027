=== ANSWER ===

To solve this problem, we need to determine, for each day in a festival, how many days it will take until the next fireworks display, which is the first occurrence on or after the given day. The fireworks are launched on specific days provided in a sorted list, and the last day of the festival is guaranteed to have fireworks.

### Approach
1. **Problem Analysis**: The problem requires us to compute, for each day from 1 to N, the smallest number of days one must wait to see fireworks. If fireworks are launched on the given day, the wait time is 0. The challenge is to efficiently compute these values for large N (up to 200,000) without resorting to a brute-force method that would be too slow.

2. **Intuition**: The key observation is that the firework days are given in a sorted list. For any day i, the next firework day is the smallest day in the list that is greater than or equal to i. This can be efficiently found using a set for quick membership checks and a backward pass through the days to determine the next firework day for each day.

3. **Algorithm Selection**: 
   - **Set for Membership Checks**: We first create a set from the list of firework days to allow O(1) membership checks.
   - **Backward Pass**: We initialize an array `next_day` where each entry `next_day[i]` holds the next firework day starting from day i. Starting from the last day (which is a firework day), we move backward. For each day i, if it is a firework day, `next_day[i]` is i; otherwise, it is the same as `next_day[i+1]`. This ensures that each day's next firework day is correctly set by leveraging previously computed values.

4. **Complexity Analysis**: 
   - **Time Complexity**: The algorithm involves reading the input (O(N + M)), creating a set (O(M)), and a backward pass (O(N)). Thus, the overall time complexity is O(N + M), which is efficient for the given constraints.
   - **Space Complexity**: We use a set of size O(M) and an array of size O(N), leading to a space complexity of O(N + M), which is acceptable given the constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    A = list(map(int, data[2:2+m]))
    
    firework_set = set(A)
    
    next_day = [0] * (n + 1)
    next_day[n] = n
    
    for i in range(n - 1, 0, -1):
        if i in firework_set:
            next_day[i] = i
        else:
            next_day[i] = next_day[i + 1]
    
    for i in range(1, n + 1):
        print(next_day[i] - i)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are N (number of days) and M (number of firework days), followed by M integers representing the firework days.
2. **Set Creation**: A set is created from the firework days to enable O(1) membership checks, which is crucial for the backward pass to be efficient.
3. **Backward Pass**: Starting from the last day (N), which is a firework day, the algorithm moves backward. For each day i, if it is a firework day, `next_day[i]` is set to i; otherwise, it is set to the next firework day found in the subsequent day (i+1).
4. **Output**: For each day from 1 to N, the result is computed as the difference between the next firework day and the current day, then printed.

This approach efficiently computes the required values by leveraging the sorted nature of the firework days and using a set for quick lookups, ensuring optimal performance even for large input sizes.
