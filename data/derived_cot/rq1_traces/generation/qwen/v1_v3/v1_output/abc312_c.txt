To solve this problem, we need to find the minimum integer \( X \) such that the number of sellers willing to sell an apple at \( X \) yen or less is greater than or equal to the number of buyers willing to buy an apple at \( X \) yen or less. The solution involves efficiently determining this \( X \) using binary search, leveraging sorted lists to count the sellers and buyers for any given \( X \).

### Approach
1. **Problem Analysis**: The problem requires finding the smallest price \( X \) where the number of sellers (those with \( A_i \leq X \)) is at least the number of buyers (those with \( B_i \geq X \)). The challenge is to efficiently compute this \( X \) given the constraints (up to \( 2 \times 10^5 \) sellers and buyers).

2. **Intuition**: The condition \( F(X) \geq G(X) \) (where \( F(X) \) is the count of sellers and \( G(X) \) is the count of buyers for price \( X \)) is monotonic. Specifically, if the condition holds for \( X \), it will hold for all \( Y \geq X \). This property allows us to use binary search to find the minimal \( X \).

3. **Algorithm Selection**: 
   - **Binary Search**: We perform binary search on the possible values of \( X \) (from 1 to \( \text{max}(B) + 1 \)).
   - **Bisect Module**: For each mid value during binary search, we use the `bisect` module to quickly count the number of sellers (\( F(X) \)) and buyers (\( G(X) \)).

4. **Complexity Analysis**: 
   - Sorting the lists \( A \) and \( B \) takes \( O(N \log N + M \log M) \).
   - Each binary search step involves two bisect operations, each \( O(\log N) \) or \( O(\log M) \).
   - The binary search itself runs in \( O(\log (\text{max}(B)) \) steps.
   - Overall complexity is \( O((N + M) \log (\text{max}(B)) \), which is efficient for the given constraints.

### Solution Code
```python
import bisect

def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    A = list(map(int, data[2:2+n]))
    B = list(map(int, data[2+n:2+n+m]))
    
    A.sort()
    B.sort()
    max_B = B[-1] if m > 0 else 0
    low = 1
    high = max_B + 1
    
    while low < high:
        mid = (low + high) // 2
        count_seller = bisect.bisect_right(A, mid)
        idx = bisect.bisect_left(B, mid)
        count_buyer = m - idx
        
        if count_seller >= count_buyer:
            high = mid
        else:
            low = mid + 1
            
    print(low)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of sellers \( N \) and buyers \( M \), followed by their respective price lists \( A \) and \( B \).

2. **Sorting**: The lists \( A \) and \( B \) are sorted to facilitate efficient counting using binary search (via `bisect`).

3. **Binary Search Setup**: The search range is set from 1 to \( \text{max}(B) + 1 \), ensuring we cover all possible values of \( X \).

4. **Binary Search Execution**: For each midpoint \( mid \) in the search range:
   - **Count Sellers**: The number of sellers willing to sell at \( mid \) or less is found using `bisect.bisect_right` on the sorted list \( A \).
   - **Count Buyers**: The number of buyers willing to buy at \( mid \) or less is found using `bisect.bisect_left` on the sorted list \( B \), then subtracting from the total buyers \( M \).
   - **Condition Check**: If the number of sellers meets or exceeds the number of buyers, the search continues in the lower half; otherwise, it continues in the upper half.

5. **Output**: The loop exits when the minimal \( X \) is found, which is then printed.

This approach efficiently narrows down the search space using binary search and leverages sorted lists to count sellers and buyers quickly, ensuring optimal performance even for large input sizes.
