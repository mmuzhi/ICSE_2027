=== ANSWER ===

To solve this problem, we need to calculate the distance between two given points on a straight line. The points are arranged in a specific order with known distances between adjacent points. The solution involves mapping each point to its cumulative distance from the starting point (A) and then computing the absolute difference between the cumulative distances of the two given points.

### Approach
1. **Mapping Points to Cumulative Distances**: We first create a dictionary where each point (A to G) is mapped to its cumulative distance from point A. This is done by sequentially adding the distances between consecutive points.
2. **Reading Input**: The input consists of two distinct uppercase letters representing the points between which we need to find the distance.
3. **Calculating Distance**: Using the cumulative distances from the dictionary, we compute the absolute difference between the distances of the two input points. This difference gives the distance between them.

### Solution Code
```python
def main():
    distances = {
        'A': 0,
        'B': 3,
        'C': 4,
        'D': 8,
        'E': 9,
        'F': 14,
        'G': 23
    }
    p, q = input().split()
    pos_p = distances[p]
    pos_q = distances[q]
    result = abs(pos_p - pos_q)
    print(result)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Mapping Creation**: The dictionary `distances` is initialized with each point as a key and its cumulative distance from A as the value. For example, 'A' is at 0, 'B' is at 3 (0 + 3), 'C' is at 4 (3 + 1), and so on up to 'G' at 23 (14 + 9).
2. **Input Handling**: The input is read and split into two variables, `p` and `q`, representing the two points between which the distance needs to be calculated.
3. **Distance Calculation**: The cumulative distances of `p` and `q` are retrieved from the dictionary. The absolute difference between these distances is computed and printed, which gives the straight-line distance between the two points.

This approach efficiently leverages the given distances between adjacent points to precompute cumulative distances, allowing for quick lookup and straightforward distance calculation between any two points. The solution is optimal with a constant time complexity O(1) for both mapping and distance calculation.
