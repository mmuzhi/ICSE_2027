To solve this problem, we need to find the median of the uniqueness array of a given integer array. The uniqueness array is constructed by calculating the number of distinct elements in every possible contiguous subarray of the input array, sorting these counts, and then finding the median of this sorted array. The median is defined as the middle element in the sorted uniqueness array, or the smaller of the two middle elements if the array length is even.

### Approach
1. **Problem Analysis**: The uniqueness array consists of the distinct counts of all subarrays of the input array. Directly generating all subarrays and their distinct counts is infeasible for large arrays (up to 10^5 elements) due to the O(n^2) complexity. Instead, we use a binary search approach combined with a sliding window technique to efficiently determine the median without explicitly constructing the uniqueness array.

2. **Binary Search Setup**: The median is the smallest value `x` such that the number of subarrays with at most `x` distinct elements is at least half the total number of subarrays (rounded up). This leverages the fact that the uniqueness array is sorted, and the median can be found by determining the smallest `x` where the cumulative count of subarrays with distinct counts ≤ `x` meets or exceeds the median position.

3. **Sliding Window Technique**: For each candidate `x` during the binary search, we use a sliding window approach to count the number of subarrays with at most `x` distinct elements. The sliding window efficiently tracks the number of distinct elements in the current window and adjusts the window size to maintain the constraint, counting valid subarrays in linear time.

4. **Complexity Analysis**: The binary search runs in O(log n) time, and each sliding window operation runs in O(n) time. Thus, the overall complexity is O(n log n), which is efficient for the given constraints.

2. **Sliding Window**: The helper function `count_at_most_x` uses a sliding window to count valid subarrays. It maintains a frequency dictionary to track distinct elements in the current window. The window expands by including new elements and contracts when the distinct count exceeds `x`, ensuring efficient counting in O(n) time per `x`.

3. **Efficiency**: By combining binary search (O(log n)) with the sliding window technique (O(n)), the solution efficiently narrows down the median value without explicitly processing all subarrays, making it suitable for large input sizes. The approach ensures optimal performance while adhering to problem constraints.
