=== ANSWER ===

To solve this problem, we need to determine the minimum number of attacks required to reduce an enemy's stamina to zero or less. Each attack reduces the enemy's stamina by a fixed amount. The solution involves calculating the ceiling of the division of the enemy's initial stamina by the amount reduced per attack, which gives the exact number of attacks needed.

### Approach
1. **Problem Analysis**: The enemy starts with stamina `A`, and each attack reduces it by `B`. The goal is to find the smallest integer `n` such that `n * B >= A`. This translates to finding the ceiling of `A / B`.
2. **Insight**: For positive integers, the ceiling of `A / B` can be efficiently computed using integer arithmetic without floating-point operations. The formula `(A + B - 1) // B` effectively calculates the ceiling by adjusting the dividend before performing floor division.
3. **Algorithm Selection**: The chosen algorithm uses a mathematical formula to compute the ceiling division directly. This avoids the need for conditional checks or floating-point operations, ensuring efficiency and correctness even for very large numbers (up to \(10^{18}\)).
4. **Complexity Analysis**: The solution involves a constant-time arithmetic operation, making it O(1) in both time and space complexity. This is optimal for the given constraints.

### Solution Code
```python
def main():
    A, B = map(int, input().split())
    print((A + B - 1) // B)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The program reads two integers `A` (initial stamina) and `B` (stamina reduction per attack) from standard input.
2. **Ceiling Division Calculation**: The formula `(A + B - 1) // B` computes the smallest integer `n` such that `n * B` is at least `A`. This formula works by adjusting `A` by adding `B - 1` before performing integer division, which effectively rounds up the result to the nearest integer.
3. **Output Result**: The result of the ceiling division is printed, which represents the minimum number of attacks needed to reduce the enemy's stamina to zero or less.

This approach efficiently handles the constraints and provides the correct result with minimal computational overhead.
