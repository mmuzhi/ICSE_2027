To solve this problem, we need to count the number of Christmas trees set up by Snuke along an infinite road, starting from a point with coordinate A and spaced M meters apart, that lie within the interval [L, R] (inclusive). The solution involves determining the range of integers k such that the position A + k*M falls within the given interval.

### Approach
1. **Problem Analysis**: The problem requires counting the number of points in the arithmetic sequence defined by A + k*M (for integer k) that lie within the interval [L, R]. The key insight is to transform the problem into finding the range of k values that satisfy the condition L ≤ A + k*M ≤ R.
2. **Mathematical Transformation**: The condition can be rewritten as (L - A) ≤ k*M ≤ (R - A). This translates to finding the smallest integer k (k_low) such that k*M ≥ (L - A) and the largest integer k (k_high) such that k*M ≤ (R - A).
3. **Ceiling and Floor Operations**: 
   - **k_low** is the ceiling of (L - A)/M, computed as (L - A + M - 1) // M using integer arithmetic.
   - **k_high** is the floor of (R - A)/M, computed as (R - A) // M.
4. **Count Calculation**: The number of valid k values is the difference between k_high and k_low plus one, provided k_low ≤ k_high. Otherwise, the count is zero.

### Solution Code
```python
def main():
    A, M, L, R = map(int, input().split())
    k_low = (L - A + M - 1) // M
    k_high = (R - A) // M
    if k_low > k_high:
        print(0)
    else:
        print(k_high - k_low + 1)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input values for A (starting coordinate), M (spacing between trees), L (left boundary), and R (right boundary) are read.
2. **Calculating k_low**: This is the smallest integer k such that A + k*M ≥ L. The formula (L - A + M - 1) // M effectively computes the ceiling of (L - A)/M using integer arithmetic.
3. **Calculating k_high**: This is the largest integer k such that A + k*M ≤ R. The formula (R - A) // M computes the floor of (R - A)/M.
4. **Counting Valid k Values**: If k_low exceeds k_high, there are no valid k values, and the output is 0. Otherwise, the count of valid k values is k_high - k_low + 1, which is printed.

This approach efficiently narrows down the range of k values using mathematical transformations and integer arithmetic, ensuring optimal performance even for very large input values.
