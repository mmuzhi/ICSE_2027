Note that a bit i is set in x if (2^i AND x) == 2^i, where AND is the bitwise AND operator.

Input: nums = [7,12,9,8,9,15], k = 4
Output: 9
Explanation: Bit 0 is set at nums[0], nums[2], nums[4], and nums[5].

Bit 3 is set at nums[1], nums[2], nums[3], nums[4], and nums[5].

Output: 0
Explanation: Since k == 6 == nums.length, the 6-or of the array is equal to the bitwise AND of all its elements. Hence, the answer is 2 AND 12 AND 1 AND 11 AND 4 AND 5 = 0.

=== ANSWER ===

To solve this problem, we need to determine the K-or of a given array of integers. The K-or is defined as a non-negative integer where each bit is set if and only if it is set in at least `k` elements of the array. 

### Approach
1. **Problem Analysis**: The problem requires checking each bit position (from 0 to 31) in the binary representation of the numbers in the array. For each bit position, we count how many numbers in the array have that bit set. If the count is at least `k`, we set that bit in the result.
2. **Intuition**: By iterating over each bit position (0 to 31) and counting the occurrences of that bit in all numbers, we can determine which bits meet the threshold `k`. The result is constructed by setting these qualifying bits.
3. **Algorithm Selection**: The algorithm involves:
   - Iterating over each bit position from 0 to 31.
   - For each bit position, count how many numbers in the array have that bit set.
   - If the count meets or exceeds `k`, set the corresponding bit in the result.
4. **Complexity Analysis**: The algorithm runs in O(32 * n) time, where `n` is the number of elements in the array. This is efficient because 32 is a constant (the number of bits considered), and `n` is constrained to a maximum of 50.

### Solution Code
```python
class Solution:
    def findKOr(self, nums: List[int], k: int) -> int:
        n = len(nums)
        result = 0
        for i in range(32):
            count = 0
            for num in nums:
                if (num >> i) & 1:
                    count += 1
            if count >= k:
                result |= (1 << i)
        return result
```

### Explanation
1. **Initialization**: The result variable is initialized to 0. This will hold the final K-or value.
2. **Bit Position Iteration**: The outer loop iterates over each bit position from 0 to 31. For each bit position, a count of how many numbers in the array have that bit set is initialized to 0.
3. **Counting Set Bits**: The inner loop checks each number in the array. For each number, it shifts the number right by the current bit position and checks if the least significant bit is set (indicating the bit is set in the number). If so, the count is incremented.
4. **Setting Result Bits**: After processing all numbers for a bit position, if the count of set bits is at least `k`, the corresponding bit in the result is set using a bitwise OR operation.
5. **Return Result**: After processing all bit positions, the result, which now contains all bits set that meet the threshold `k`, is returned.

This approach efficiently checks each bit position independently, ensuring the solution is both correct and optimal for the given constraints.
