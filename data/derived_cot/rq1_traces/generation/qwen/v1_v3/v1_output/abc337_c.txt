=== ANSWER ===

To solve this problem, we need to determine the order of people standing in a line based on given parent-child relationships. Each person (except the front) is directly behind another person, and the arrangement must be unique as per the problem constraints.

### Approach
1. **Problem Analysis**: The problem involves reconstructing a line of people from a sequence where each person (except the front) is specified to be behind another person. The key insight is that the line forms a directed graph where each node (person) has at most one incoming edge (parent) and at most one outgoing edge (child). The front of the line is the node with no incoming edges (in-degree 0).

2. **Graph Construction**: We construct two arrays:
   - `in_degree` to count the number of incoming edges for each node.
   - `children` to store the child nodes for each parent node.

3. **Finding the Front**: The front of the line is the node with an in-degree of 0. This is found by scanning the `in_degree` array.

4. **Traversal**: Starting from the front node, we traverse the graph by following each node's child, building the sequence from front to back. The traversal is straightforward since each node has at most one child, ensuring a unique path.

### Solution Code
```python
n = int(input().strip())
A = list(map(int, input().split()))

in_degree = [0] * (n + 1)
children = [[] for _ in range(n + 1)]

for i in range(n):
    if A[i] != -1:
        parent = A[i]
        child = i + 1
        in_degree[child] += 1
        children[parent].append(child)

front = None
for i in range(1, n + 1):
    if in_degree[i] == 0:
        front = i
        break

if front is None:
    result = list(range(1, n + 1))
    print(" ".join(map(str, result)))
else:
    result = []
    current = front
    while current <= n and current is not None:
        result.append(str(current))
        if children[current]:
            current = children[current][0]
        else:
            break
    print(" ".join(result))
```

### Explanation
1. **Graph Construction**: The code reads the input values and constructs two arrays, `in_degree` and `children`. The `in_degree` array keeps track of how many people are behind each person (incoming edges), and the `children` array lists the people directly behind each person (outgoing edges).

2. **Finding the Front**: The front of the line is identified as the person with no one behind them (in-degree 0). This is found by iterating through the `in_degree` array.

3. **Traversal**: Starting from the front node, the code traverses the graph by moving to each node's child (if any) and appends each node to the result list. This process continues until there are no more children to visit, resulting in the line order from front to back.

This approach efficiently reconstructs the line by leveraging graph traversal principles, ensuring correctness under the problem's constraints. The solution handles large inputs efficiently with a time complexity of O(N), where N is the number of people.
