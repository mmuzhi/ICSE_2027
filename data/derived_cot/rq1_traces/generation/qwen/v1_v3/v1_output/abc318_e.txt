To solve this problem, we need to count the number of triples (i, j, k) in a sequence of integers such that:
1. \(1 \leq i < j < k \leq N\)
2. \(A_i = A_k\)
3. \(A_i \neq A_j\)

### Approach
1. **Problem Analysis**: The problem requires finding all valid triples (i, j, k) where the first and third elements are equal, and the middle element is different. The challenge is to efficiently count these triples without resorting to a brute-force approach, which would be computationally expensive for large sequences (up to 300,000 elements).

2. **Intuition**: For each distinct number in the sequence, we collect all its positions (indices). For each such number, we then consider all pairs of positions (i, k) where \(i < k\). For each pair, the number of valid middle elements (j) between i and k is the total number of elements between i and k minus the count of the same number between i and k.

3. **Algorithm Selection**: 
   - **Group Indices by Value**: First, we group the indices of each distinct number in the sequence. This helps in efficiently accessing all positions where a particular number occurs.
   - **Prefix Sum and Double Sum Calculation**: For each number with at least two occurrences, we calculate the total valid triples by leveraging prefix sums and a mathematical breakdown of the problem. Specifically, for each pair of indices (i, k), the number of valid j's is derived from the difference between the total elements between i and k and the count of the same number between them. This is optimized using prefix sums to avoid recalculating sums repeatedly.

4. **Complexity Analysis**: The algorithm processes each element in the sequence once to build the index groups, which is O(N). For each number, processing its indices involves a linear pass, but the total operations across all numbers sum to O(N) due to the constraint that the sum of all indices' lengths is N. Thus, the overall complexity is O(N), which is efficient for the given constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    n_val = int(data[0])
    A = list(map(int, data[1:1+n_val]))
    
    indices_list = [[] for _ in range(n_val+1)]
    
    for i, num in enumerate(A):
        if 1 <= num <= n_val:
            indices_list[num].append(i)
            
    total_ans = 0
    for x in range(1, n_val+1):
        L = indices_list[x]
        m = len(L)
        if m < 2:
            continue
        running = 0
        for q in range(1, m):
            running += L[q-1]
            first_part = q * L[q] - running
            second_part = (q*(q-1)//2) - q*q
            total_ans += first_part + second_part

    print(total_ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the sequence length \(N\) and the sequence \(A\) of \(N\) integers.
2. **Index Grouping**: We create a list of lists where each sublist contains the indices of all occurrences of a particular number (from 1 to \(N\)).
3. **Triple Counting**: For each number with at least two occurrences, we calculate the number of valid triples (i, j, k) by:
   - **Prefix Sum Calculation**: We maintain a running sum of indices to efficiently compute the sum of indices up to the current position.
   - **Mathematical Breakdown**: For each pair of indices (i, k), the number of valid j's is derived from the difference between the total elements between i and k and the count of the same number between them. This is optimized using the prefix sum and combinatorial mathematics to avoid nested loops.
4. **Output**: The total count of valid triples is printed as the result.

This approach efficiently narrows down the problem to manageable calculations by leveraging index grouping and mathematical insights, ensuring optimal performance even for large input sizes.
