To solve this problem, we need to determine the minimum number of days required for at least P people in a group to have their hair lengths reach or exceed a specified length T, given that each person's hair grows by 1 unit per day. The solution involves checking the current hair lengths and using binary search to efficiently find the minimal days needed if the condition isn't already met.

### Approach
1. **Initial Check**: First, we check if the condition (at least P people with hair length ≥ T) is already satisfied with zero days. If it is, we immediately return 0.
2. **Binary Search Setup**: If the condition isn't met initially, we set up a binary search between 0 and a sufficiently large upper bound (200 in this case) to find the minimal days required.
3. **Binary Search Execution**: For each midpoint in the binary search, we count how many people's hair lengths would be ≥ T after that many days. If the count meets or exceeds P, we adjust the upper bound; otherwise, we adjust the lower bound.
4. **Result**: The binary search converges to the minimal days required, which is then printed as the result.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    T = int(data[1])
    P = int(data[2])
    L = list(map(int, data[3:3+n]))
    
    count_now = sum(1 for length in L if length >= T)
    if count_now >= P:
        print(0)
        return
        
    low = 0
    high = 200
    
    while low < high:
        mid = (low + high) // 2
        count_mid = sum(1 for length in L if length + mid >= T)
        if count_mid >= P:
            high = mid
        else:
            low = mid + 1
            
    print(low)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initial Check**: The code reads the input values and checks if the current hair lengths already meet the condition (at least P people with hair length ≥ T). If so, it prints 0 and exits.
2. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to 200 (a safe upper bound given the constraints).
3. **Binary Search Execution**: The loop calculates the midpoint (`mid`) and checks how many people's hair lengths would be ≥ T after `mid` days. If the count meets or exceeds P, the upper bound is adjusted to `mid`; otherwise, the lower bound is increased.
4. **Result**: The loop exits when `low` equals `high`, which is the minimal days required. This value is printed as the result.

This approach efficiently narrows down the solution using binary search, ensuring optimal performance even for the upper constraint limits.
