You will be given a competitive programming problem.
You are given N strings S_1,S_2,\ldots,S_N consisting of lowercase English letters.
Determine if there are distinct integers i and j between 1 and N, inclusive, such that the concatenation of S_i and S_j in this order is a palindrome.
A string T of length M is a palindrome if and only if the i-th character and the (M+1-i)-th character of T are the same for every 1\leq i\leq M.

Input

The input is given from Standard Input in the following format:
N
S_1
S_2
\vdots
S_N

Output

If there are i and j that satisfy the condition in the problem statement, print Yes; otherwise, print No.

Constraints


- 2\leq N\leq 100
- 1\leq \lvert S_i\rvert \leq 50
- N is an integer.
- S_i is a string consisting of lowercase English letters.
- All S_i are distinct.

Sample Input 1

5
ab
ccef
da
a
fe

Sample Output 1

Yes

If we take (i,j)=(1,4), the concatenation of S_1=ab and S_4=a in this order is aba, which is a palindrome, satisfying the condition.
Thus, print Yes.  
Here, we can also take (i,j)=(5,2), for which the concatenation of S_5=fe and S_2=ccef in this order is feccef, satisfying the condition.

To solve this problem, we need to determine if there exist two distinct indices \(i\) and \(j\) such that the concatenation of the strings \(S_i\) and \(S_j\) forms a palindrome. A palindrome reads the same forwards and backwards. 

### Approach
1. **Problem Analysis**: The problem requires checking all possible pairs of distinct strings in the given list to see if their concatenation forms a palindrome. The key observation here is that the entire concatenated string must be a palindrome, meaning the first half of the string must mirror the second half in reverse order.
2. **Intuition**: For each string \(S_i\), we can check every other string \(S_j\) (where \(i \neq j\)) to see if \(S_i + S_j\) is a palindrome. This involves concatenating the two strings and checking if the resulting string is equal to its reverse.
3. **Algorithm Selection**: We use a nested loop to iterate over all pairs of distinct strings. For each pair, we concatenate the strings and check if the concatenated result is a palindrome. If any such pair is found, we immediately return "Yes". If no such pair is found after checking all possibilities, we return "No".
4. **Optimization**: The solution efficiently checks each pair in \(O(N^2)\) time, where \(N\) is the number of strings. Given the constraints \(N \leq 100\) and each string length \(\leq 50\), this approach is feasible as it involves checking up to 5000 pairs (each pair check involves reversing a string of length up to 100, which is computationally inexpensive).

### Solution Code
```python
n = int(input())
words = [input().strip() for _ in range(n)]

found = False
for i in range(n):
    for j in range(n):
        if i == j:
            continue
        s = words[i] + words[j]
        if s == s[::-1]:
            found = True
            break
    if found:
        break

print("Yes" if found else "No")
```

1. **Reading Input**: The code starts by reading the number of strings \(N\) and then reads each string into a list.
2. **Checking Pairs**: Using nested loops, the code checks every possible pair of distinct strings. For each pair, it concatenates the strings and checks if the result is a palindrome by comparing it to its reverse.
3. **Early Termination**: If a palindromic concatenation is found, the loops terminate early, and "Yes" is printed. If no such pair is found after all checks, "No" is printed.
4. **Efficiency**: The solution efficiently checks all possible pairs with a time complexity of \(O(N^2 \times M)\), where \(M\) is the maximum length of any concatenated string (up to 100). This is optimal for the given constraints. The space complexity is \(O(N \times M)\) to store the input strings.
