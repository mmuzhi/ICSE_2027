You will be given a competitive programming problem.
You are given an integer n and a 2D integer array queries.
There are n cities numbered from 0 to n - 1. Initially, there is a unidirectional road from city i to city i + 1 for all 0 <= i < n - 1.
queries[i] = [u_i, v_i] represents the addition of a new unidirectional road from city u_i to city v_i. After each query, you need to find the length of the shortest path from city 0 to city n - 1.
Return an array answer where for each i in the range [0, queries.length - 1], answer[i] is the length of the shortest path from city 0 to city n - 1 after processing the first i + 1 queries.
 
Example 1:

Input: n = 5, queries = [[2,4],[0,2],[0,4]]
Output: [3,2,1]
Explanation: 

After the addition of the road from 2 to 4, the length of the shortest path from 0 to 4 is 3.

After the addition of the road from 0 to 2, the length of the shortest path from 0 to 4 is 2.

After the addition of the road from 0 to 4, the length of the shortest path from 0 to 4 is 1.

Example 2:

Input: n = 4, queries = [[0,3],[0,2]]
Output: [1,1]
Explanation:

After the addition of the road from 0 to 3, the length of the shortest path from 0 to 3 is 1.

After the addition of the road from 0 to 2, the length of the shortest path remains 1.

3 <= n <= 500
1 <= queries.length <= 500
queries[i].length == 2
0 <= queries[i][0] < queries[i][1] < n
1 < queries[i][1] - queries[i][0]
There are no repeated roads among the queries.

To solve this problem, we need to find the shortest path from city 0 to city n-1 in a directed graph after each query adds a new unidirectional road. The initial graph consists of base roads connecting each city i to city i+1. The solution involves dynamically updating the graph with each query and then computing the shortest path using Breadth-First Search (BFS) for each query.

### Approach
1. **Graph Initialization**: Start by building the initial graph with base edges from each city i to city i+1 for all i from 0 to n-2.
2. **Processing Queries**: For each query, add the new road (edge) from city u to city v to the graph.
3. **BFS for Shortest Path**: After each query, perform a BFS starting from city 0 to compute the shortest path to city n-1. BFS is suitable here because the graph is unweighted (all edges have a weight of 1), and BFS efficiently finds the shortest path in such graphs.
4. **Result Collection**: After each BFS, record the shortest path distance from city 0 to city n-1 and append it to the result list.

### Solution Code
```python
from collections import deque

class Solution:
    def shortestDistanceAfterQueries(self, n: int, queries: List[List[int]]) -> List[int]:
        graph = [[] for _ in range(n)]
        for i in range(n-1):
            graph[i].append(i+1)
        
        result = []
        for u, v in queries:
            graph[u].append(v)
            dist = [-1] * n
            dist[0] = 0
            q = deque([0])
            while q:
                current = q.popleft()
                for neighbor in graph[current]:
                    if dist[neighbor] == -1:
                        dist[neighbor] = dist[current] + 1
                        q.append(neighbor)
            result.append(dist[n-1])
        return result
```

### Explanation
1. **Graph Initialization**: The graph is initialized with base edges. Each city i (except the last one) has a directed edge to city i+1.
2. **Query Processing**: For each query [u, v], the edge from u to v is added to the graph. This edge is unidirectional and connects two cities where v > u.
3. **BFS Execution**: For each updated graph, BFS is initiated from city 0. The BFS explores all reachable cities level by level, updating the shortest path distances. The distance to city n-1 is recorded after each BFS.
4. **Result Compilation**: The shortest path distance from city 0 to city n-1 after each query is collected into a result list, which is returned as the final output.

This approach efficiently handles each query by dynamically updating the graph and leveraging BFS to compute the shortest path in linear time relative to the number of edges and nodes, making it suitable for the given constraints.
