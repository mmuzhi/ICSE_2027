        if q_count == 0:

        cur = base[:]  # current counts for each letter

To solve this problem, we need to replace all occurrences of '?' in a given string with lowercase English letters such that the total "value" of the string is minimized. The value of the string is defined as the sum of the cost for each character, where the cost of a character at position `i` is the number of times that character has appeared before `i` in the string. If multiple strings yield the same minimal value, we return the lexicographically smallest one.

### Approach
1. **Problem Analysis**: The key observation is that the total value of the string depends on the frequency of each character. The cost for each character is determined by the number of previous occurrences of the same character. To minimize the total value, we should avoid repeating characters as much as possible. The optimal strategy involves distributing the replacement characters (those replacing '?') across different letters to keep their frequencies as balanced as possible.

2. **Intuition**: The minimal total value is achieved by ensuring that the sum of the triangular numbers (i.e., `n*(n-1)/2` for each character's total count) is minimized. This is done by greedily assigning each replacement to the letter with the smallest current count (and lexicographically smallest in case of ties).

3. **Algorithm Selection**: 
   - **Count Base Frequencies**: First, count the occurrences of each non-'?' character in the string.
   - **Greedy Replacement Distribution**: Use a min-heap to distribute the replacement characters. For each replacement, select the letter with the smallest current count (and lexicographically smallest if counts are equal). Update the count and push it back into the heap.
   - **Construct Result String**: After determining the counts for all replacement letters, sort these letters lexicographically and replace the '?' positions in the original string in order.

4. **Complexity Analysis**: 
   - **Time Complexity**: The algorithm involves processing each '?' once (O(q)) with heap operations (each operation is O(log 26)), leading to an overall time complexity of O(q log 26), which is efficient for the constraints (q ≤ 10^5).
   - **Space Complexity**: The space required is O(26) for the heap and counts, plus O(q) for storing replacement letters, which is manageable given the constraints.

### Solution Code
```python
import heapq

class Solution:
    def minimizeStringValue(self, s: str) -> str:
        n = len(s)
        base = [0] * 26
        q_count = 0
        for char in s:
            if char == '?':
                q_count += 1
            else:
                base[ord(char) - ord('a')] += 1
        
        if q_count == 0:
            return s
        
        cur = base[:]
        heap = [(cur[i], i) for i in range(26)]
        heapq.heapify(heap)
        
        for _ in range(q_count):
            while True:
                count_val, idx = heapq.heappop(heap)
                if count_val != cur[idx]:
                    continue
                else:
                    break
            cur[idx] += 1
            heapq.heappush(heap, (cur[idx], idx))
        
        replacements = []
        for i in range(26):
            replacements.extend([chr(i + ord('a'))] * (cur[i] - base[i]))
        
        replacements.sort()
        
        res = []
        rep_index = 0
        for char in s:
            if char == '?':
                res.append(replacements[rep_index])
                rep_index += 1
            else:
                res.append(char)
        
        return "".join(res)
```

### Explanation
1. **Initialization**: The code starts by counting the occurrences of each non-'?' character (base counts) and the number of '?' characters (q_count).
2. **Heap Setup**: A min-heap is initialized with tuples of (current count, character index) for each letter. This heap helps in efficiently retrieving the letter with the smallest current count for each replacement.
3. **Greedy Replacement**: For each '?', the smallest count letter (lexicographically smallest in case of ties) is selected, its count is incremented, and it is pushed back into the heap.
4. **Replacement Letters**: After processing all '?', the letters to be used for replacements are determined by the difference between the final counts and the base counts. These letters are then sorted lexicographically.
5. **Result Construction**: The original string is reconstructed by replacing each '?' with the next letter from the sorted list, ensuring the lexicographically smallest result while maintaining the minimal total value.

This approach efficiently minimizes the string's value by balancing character frequencies and ensures the lexicographically smallest result when multiple solutions exist.
