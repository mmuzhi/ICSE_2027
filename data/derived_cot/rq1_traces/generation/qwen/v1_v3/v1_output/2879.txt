Notes

A string with a length of len is considered a semi-palindrome if there exists a positive integer d such that 1 <= d < len and len % d == 0, and if we take indices that have the same modulo by d, they form a palindrome. To solve this problem, we need to partition a given string into `k` contiguous substrings such that the sum of the changes required to turn each substring into a semi-palindrome is minimized. A semi-palindrome is defined as a string that can be divided into `d` groups (where `d` is a divisor of the string's length and `1 <= d < length`), and each group forms a palindrome when read in sequence.

### Approach
1. **Precompute Divisors**: For each possible substring length (from 2 to 200), precompute all divisors `d` (where `1 <= d < length` and `length % d == 0`). This helps in efficiently checking all possible ways to split a substring into groups for semi-palindrome validation.

2. **Calculate Minimum Changes for Substrings**: For each possible substring (from index `l` to `r`), compute the minimum changes required to make it a semi-palindrome. This involves:
   - For each divisor `d` of the substring's length, split the substring into `d` groups.
   - For each group, calculate the number of changes needed to turn it into a palindrome by comparing corresponding characters from the start and end of the group.
   - Sum the changes for all groups and keep track of the minimum sum across all divisors.

3. **Dynamic Programming (DP) Setup**: Use a DP table where `dp[i][m]` represents the minimum changes needed to partition the first `i` characters of the string into `m` substrings. Initialize the table with a large value (infinity) except for `dp[0][0]` which is 0 (no changes needed for an empty string and no partitions).

4. **DP Table Population**: For each number of partitions `m` from 1 to `k`, and for each possible end index `i` (from `2*m` to the string length), compute the minimum changes by considering all valid starting indices `j` for the last substring. The value `dp[i][m]` is updated by taking the minimum of `dp[j][m-1] + min_cost[j][i-1]` for all valid `j`.

5. **Result Extraction**: The answer is found in `dp[n][k]`, which gives the minimum changes required to partition the entire string into `k` semi-palindromic substrings.

### Solution Code
```python
class Solution:
    def minimumChanges(self, s: str, k: int) -> int:
        n = len(s)
        # Precompute divisors for lengths from 2 to 200
        divisors = {}
        for L_val in range(2, 201):
            divisors[L_val] = []
            for d in range(1, L_val):
                if L_val % d == 0:
                    divisors[L_val].append(d)
        
        # Precompute min_cost for all substrings (l, r) with r-l+1 >= 2
        min_cost = [[10**9] * n for _ in range(n)]
        for l in range(n):
            for r in range(l+1, n):
                L = r - l + 1
                if L < 2:
                    continue
                min_val = 10**9
                for d in divisors[L]:
                    total_for_d = 0
                    for rmd in range(d):
                        p0 = l + rmd
                        m_group = (r - p0) // d + 1
                        cost_group = 0
                        for t in range(m_group // 2):
                            if s[p0 + t * d] != s[p0 + (m_group - 1 - t) * d]:
                                cost_group += 1
                        total_for_d += cost_group
                    if total_for_d < min_val:
                        min_val = total_for_d
                min_cost[l][r] = min_val
        
        # DP table: dp[i][m] = minimal cost for first i characters partitioned into m substrings
        dp = [[10**9] * (k+1) for _ in range(n+1)]
        dp[0][0] = 0
        
        for m_val in range(1, k+1):
            for i in range(2*m_val, n+1):
                for j in range(2*(m_val-1), i-1):
                    if dp[j][m_val-1] == 10**9 or min_cost[j][i-1] == 10**9:
                        continue
                    if dp[i][m_val] > dp[j][m_val-1] + min_cost[j][i-1]:
                        dp[i][m_val] = dp[j][m_val-1] + min_cost[j][i-1]
        
        return dp[n][k]
```

### Explanation
1. **Divisors Precomputation**: The code first computes all divisors for every possible substring length (2 to 200). This helps in efficiently determining how to split any substring into groups for semi-palindrome checks.

2. **Minimum Changes Calculation**: For each substring (defined by start and end indices), the code calculates the minimal changes required to make it a semi-palindrome by checking all valid divisors. Each divisor splits the substring into groups, and the cost to turn each group into a palindrome is computed by comparing characters symmetrically positioned within the group.

3. **Dynamic Programming**: The DP table is filled by considering all possible partitions of the string into `m` substrings. For each partition count `m`, the code checks all possible end indices `i` and valid starting indices `j` for the last substring, updating the DP table with the minimal changes required.

4. **Result Extraction**: The solution is derived from `dp[n][k]`, which contains the minimal changes needed to partition the entire string into `k` semi-palindromic substrings. This approach efficiently narrows down the solution using precomputation and dynamic programming, ensuring optimal performance even for the upper constraint limits.
