You will be given a competitive programming problem.
An ant is on a boundary. It sometimes goes left and sometimes right.
You are given an array of non-zero integers nums. The ant starts reading nums from the first element of it to its end. At each step, it moves according to the value of the current element:

If nums[i] < 0, it moves left by -nums[i] units.
If nums[i] > 0, it moves right by nums[i] units.

Return the number of times the ant returns to the boundary.
Notes:

There is an infinite space on both sides of the boundary.
We check whether the ant is on the boundary only after it has moved |nums[i]| units. In other words, if the ant crosses the boundary during its movement, it does not count.

Output: 1
Explanation: After the first step, the ant is 2 steps to the right of the boundary.
After the second step, the ant is 5 steps to the right of the boundary.
After the third step, the ant is on the boundary.
So the answer is 1.

After the third step, the ant is 2 steps to the right of the boundary.
After the fourth step, the ant is 2 steps to the left of the boundary.
The ant never returned to the boundary, so the answer is 0.

=== ANSWER ===

To solve this problem, we need to determine the number of times an ant returns to the boundary after moving according to the elements of a given array. The ant starts at the boundary (position 0) and moves left or right based on the sign of each element in the array. The key observation is that we need to check the ant's position after each move to see if it has returned to the boundary (position 0).

### Approach
1. **Initialization**: Start with the ant's initial position at 0 and a counter set to 0 to keep track of the number of returns to the boundary.
2. **Iterate through the array**: For each element in the array:
   - **Update the ant's position**: If the element is positive, move the ant right by the element's value. If the element is negative, move the ant left by the absolute value of the element.
   - **Check for boundary return**: After updating the position, check if the ant's position is 0. If it is, increment the counter.
3. **Return the result**: After processing all elements, the counter will hold the number of times the ant returned to the boundary during its journey.

This approach efficiently tracks the ant's position after each move and checks for boundary returns in constant time for each element, leading to an overall linear time complexity.

### Solution Code
```python
class Solution:
    def returnToBoundaryCount(self, nums: List[int]) -> int:
        current_position = 0
        count = 0
        for num in nums:
            current_position += num
            if current_position == 0:
                count += 1
        return count
```

### Explanation
- **Initialization**: The ant starts at position 0 (`current_position = 0`), and the counter (`count`) is initialized to 0 to count the boundary returns.
- **Processing each element**: For each number in the array:
  - The ant's position is updated by adding the number (if positive) or subtracting its absolute value (if negative).
  - After each update, if the ant's position is 0, the counter is incremented.
- **Result**: The counter, which now contains the total number of times the ant returned to the boundary during its movement, is returned as the result.

This method efficiently processes each element in the array exactly once, making it optimal with a time complexity of O(n), where n is the length of the array. The space complexity is O(1) as only a few variables are used.
