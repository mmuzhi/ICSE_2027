To solve this problem, we need to determine for each query whether there exists one element from each of the three given sequences (A, B, and C) such that their sum equals the query value. Given the constraints, a brute-force approach checking all possible combinations for each query would be too slow. Instead, we optimize by precomputing the set of all possible sums from sequences A and B, then for each query, we check if the difference between the query value and any element in C exists in the precomputed set.

### Approach
1. **Precompute Sums of A and B**: Calculate all possible sums of elements from sequences A and B and store them in a set for O(1) lookups.
2. **Process Queries Efficiently**: For each query value, iterate through the elements of sequence C. For each element in C, check if the difference between the query value and the element exists in the precomputed set of sums from A and B. If such a difference is found, the answer is "Yes"; otherwise, after checking all elements, the answer is "No".

```python
    M = int(next(it))
    B = [int(next(it)) for _ in range(M)]
    L_val = int(next(it))
    C = [int(next(it)) for _ in range(L_val)]
    Q = int(next(it))
    X = [int(next(it)) for _ in range(Q)]
    
    S_set = set()
    for a in A:
        for b in B:
            S_set.add(a + b)
    
    C_set_distinct = set(C)
    
    out_lines = []
    for x in X:
        found = False
        for c in C_set_distinct:
            if x >= c and (x - c) in S_set:
                found = True
                break
        out_lines.append("Yes" if found else "No")
    
    for line in out_lines:
        print(line)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The input includes the sequences A, B, and C, and the queries.
2. **Precompute Sums**: The set `S_set` is constructed by computing all possible sums of elements from A and B. This allows O(1) time complexity for checking if a particular sum exists.
3. **Processing Queries**: For each query value, the code checks each distinct element in C. For each element, it verifies if the difference between the query value and the element exists in `S_set`. If found, the answer is "Yes"; otherwise, after all elements are checked, the answer is "No".
4. **Output Results**: The results for all queries are collected in a list and printed at the end for efficiency, minimizing I/O operations.

This approach efficiently handles the problem by leveraging set operations to reduce the time complexity from O(N*M*L) per query to O(L) per query, where L is the size of sequence C, making it feasible for large input sizes.
