To solve this problem, we need to find the minimum travel distance required to start from any city and visit all cities at least once in a tree-structured network. The solution involves leveraging the properties of trees, specifically the diameter (the longest path between any two nodes) and the sum of all edge weights.

### Approach
1. **Problem Analysis**: The problem requires visiting all nodes in a tree (a connected acyclic graph) starting from any node, without needing to return to the start. The minimal travel distance is derived from the sum of all edge weights multiplied by two, minus the diameter of the tree. This works because the diameter represents the longest path that needs to be traversed only once, while all other edges are traversed twice (once to reach a node and once to backtrack).

2. **Key Insight**: The diameter of the tree is the longest path between any two nodes. The minimal travel distance is calculated as twice the sum of all edge weights minus the diameter. This formula accounts for the fact that all edges except those on the diameter are traversed twice in an optimal path.

3. **Algorithm Selection**: 
   - **Diameter Calculation**: Use two depth-first searches (DFS) to find the diameter of the tree. The first DFS starts from an arbitrary node to find the farthest node. The second DFS starts from this farthest node to find the node farthest from it, which gives the diameter.
   - **Sum of Edge Weights**: Sum all edge weights during input reading.

4. **Complexity Analysis**: The algorithm involves two DFS traversals, each operating in O(N) time where N is the number of nodes. Given the constraints (N ≤ 2×10^5), this approach is efficient and feasible.

import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])

                max_node = node

                stack.append((neighbor, dist + weight, node))
