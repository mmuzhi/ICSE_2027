=== ANSWER ===

To solve this problem, we need to determine the middle day of a year that consists of M months, each with a specified number of days. The key insight is that the total number of days in the year is odd, so the middle day is the ((total_days + 1) // 2)-th day. 

### Approach
1. **Calculate Total Days**: Sum all the days in the given months to get the total number of days in the year.
2. **Determine Target Day**: The target day is the middle day, which is computed as (total_days + 1) // 2.
3. **Locate the Target Day in the Calendar**: Iterate through each month, accumulating the total days until the cumulative sum meets or exceeds the target day. The day within the current month is then calculated by subtracting the cumulative days of all previous months from the target day.

### Solution Code
```python
def main():
    M = int(input().strip())
    D_list = list(map(int, input().split()))
    
    total_days = sum(D_list)
    target = (total_days + 1) // 2
    
    cumulative_prev = 0
    for i in range(M):
        if target <= cumulative_prev + D_list[i]:
            day_in_month = target - cumulative_prev
            month = i + 1
            break
        cumulative_prev += D_list[i]
    
    print(f"{month} {day_in_month}")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Reading**: The code reads the number of months (M) and the list of days in each month (D_list).
2. **Total Days Calculation**: The sum of all days in the months gives the total days in the year.
3. **Target Day Calculation**: The middle day is found using integer division to handle the odd total days.
4. **Month and Day Identification**: The loop checks each month's cumulative days. Once the cumulative days include the target day, the specific day within the current month is calculated by subtracting the days from previous months. The month and day are then printed.

This approach efficiently narrows down the target day by leveraging cumulative sums and ensures optimal performance with a linear scan through the months. The solution handles all edge cases, including a single month with one day, as demonstrated in the sample inputs.
