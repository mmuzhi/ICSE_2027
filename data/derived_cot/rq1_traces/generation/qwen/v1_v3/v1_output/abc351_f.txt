=== ANSWER ===

To solve this problem, we need to compute the sum of the maximum differences between all pairs of elements in a sequence where the second element is greater than the first. The straightforward approach of checking all pairs would result in a time complexity of O(N^2), which is infeasible for large N (up to 4×10^5). Instead, we use a more efficient approach leveraging coordinate compression and Fenwick Trees (Binary Indexed Trees) to reduce the time complexity to O(N log N).

### Approach
1. **Coordinate Compression**: Since the values in the sequence can be as large as 10^8, we first map these values to a smaller range (1 to the number of distinct values) to facilitate efficient processing with Fenwick Trees.
2. **Fenwick Trees for Count and Sum**: We maintain two Fenwick Trees—one to keep track of the count of elements processed so far and another to keep the sum of these elements. This allows us to efficiently query the number of elements and their sum that are less than the current element.
3. **Processing Elements**: For each element in the sequence, we query the Fenwick Trees to get the count and sum of elements less than the current element. The contribution of the current element to the total sum is then calculated as (count * current element value) - (sum of those elements). After processing, we update the Fenwick Trees with the current element's count and value.

### Solution Code
```python
class Fenw:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)
    
    def update(self, index, delta):
        while index <= self.n:
            self.tree[index] += delta
            index += index & -index
            
    def query(self, index):
        s = 0
        while index:
            s += self.tree[index]
            index -= index & -index
        return s

def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    if n == 0:
        print(0)
        return
        
    sorted_vals = sorted(set(A))
    comp = {val: idx+1 for idx, val in enumerate(sorted_vals)}
    size = len(sorted_vals)
    
    cnt_tree = Fenw(size)
    sum_tree = Fenw(size)
    
    total_ans = 0
    for i in range(n):
        x = A[i]
        pos = comp[x]
        
        if pos > 1:
            count_less = cnt_tree.query(pos - 1)
            sum_less = sum_tree.query(pos - 1)
        else:
            count_less = 0
            sum_less = 0
            
        total_ans += count_less * x - sum_less
        
        cnt_tree.update(pos, 1)
        sum_tree.update(pos, x)
        
    print(total_ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Coordinate Compression**: The distinct values in the sequence are sorted and mapped to a compressed range (1 to the number of distinct values). This step ensures that the Fenwick Trees can handle the values efficiently.
2. **Fenwick Trees**: The `Fenw` class implements a Fenwick Tree with `update` and `query` methods. The `update` method increments the tree at a given index by a delta, and the `query` method computes the prefix sum up to a given index.
3. **Processing Elements**: For each element in the sequence, we determine its compressed position. Using the Fenwick Trees, we query the count and sum of elements less than the current element. The contribution to the total sum is calculated and added. The current element's count and value are then updated in the Fenwick Trees.
4. **Efficiency**: The algorithm processes each element in O(log N) time due to the Fenwick Tree operations, resulting in an overall time complexity of O(N log N), which is optimal for the given constraints. The space complexity is O(N) due to the storage required for the Fenwick Trees and the compressed values.
