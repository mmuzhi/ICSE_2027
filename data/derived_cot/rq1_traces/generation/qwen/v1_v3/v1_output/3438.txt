A peak in an array arr is an element that is greater than its previous and next element in arr.
You are given an integer array nums and a 2D integer array queries.

queries[i] = [2, index_i, val_i], change nums[index_i] to val_i.

Return an array answer containing the results of the queries of the first type in order.
Notes:

The first and the last element of an array or a subarray cannot be a peak.

Output: [0]

First query: We change nums[3] to 4 and nums becomes [3,1,4,4,5].

class Solution:
    def countOfPeaks(self, nums: List[int], queries: List[List[int]]) -> List[int]:
        n = len(nums)
        # Initialize peaks array with zeros
        peaks = [0] * n
        # Precompute initial peaks for indices 1 to n-2 (0-indexed)
        for i in range(1, n-1):
            if nums[i] > nums[i-1] and nums[i] > nums[i+1]:
                peaks[i] = 1
        
        # Fenw tree for the peaks array (size n)
        class Fenw:
            def __init__(self, n):
                self.n = n
                self.tree = [0] * (n+1)  # 1-indexed, index from 1 to n

            def update(self, index, delta):
                # index is 0-indexed
                i = index + 1
                while i <= self.n:
                    self.tree[i] += delta
                    i += i & -i

            def query(self, index):
                # returns prefix sum from 0 to index (0-indexed)
                res = 0
                i = index + 1
                while i:
                    res += self.tree[i]
                    i -= i & -i
                return res

            def range_query(self, l, r):
                if l > r:
                    return 0
                if l == 0:
                    return self.query(r)
                return self.query(r) - self.query(l-1)
        
        fenw = Fenw(n)
        # Build the Fenw tree: for each index i, if peaks[i] is 1, update fenw at i by 1.
        for i in range(n):
            if peaks[i]:
                fenw.update(i, 1)
        
        ans = []
        # Process each query
        for q in queries:
            if q[0] == 1:
                l, r = q[1], q[2]
                # We need to count peaks from index l+1 to r-1 (0-indexed)
                if l+1 <= r-1:
                    count = fenw.range_query(l+1, r-1)
                else:
                    count = 0
                ans.append(count)
            else:  # update query
                idx = q[1]
                val = q[2]
                # Update nums[idx] to val
                nums[idx] = val
                # Update the peaks for indices idx-1, idx, idx+1 (if they are in [1, n-2] (0-indexed: from 1 to n-2))
                for i in [idx-1, idx, idx+1]:
                    if i < 1 or i > n-2:  # because peaks only defined for indices 1 to n-2 (0-indexed)

                    if new_peak != peaks[i]:
                        fenw.update(i, new_peak - peaks[i])
                        peaks[i] = new_peak
        return ans

=== ANSWER ===

To solve this problem, we need to efficiently handle two types of queries on an array: counting the number of peak elements in a subarray and updating an element in the array. A peak element is defined as an element greater than both its immediate neighbors, excluding the first and last elements of the array or any subarray.

### Approach
1. **Problem Analysis**: The challenge involves processing up to 100,000 queries on an array of size up to 100,000. Each query can either be a count request for peaks in a subarray or an update to an element in the array. The key insight is that each update affects the peak status of the updated element and its adjacent elements. 

2. **Intuition**: We can use a Fenwick Tree (Binary Indexed Tree) to maintain the count of peak elements. This allows efficient point updates and range queries. The peak status for each element is determined by comparing it with its neighbors. When an element is updated, we recheck the peak status for the updated element and its neighbors, updating the Fenwick Tree accordingly.

3. **Algorithm Selection**: 
   - **Fenwick Tree**: This data structure supports efficient prefix sum queries and point updates in logarithmic time, making it suitable for range queries and updates.
   - **Peak Identification**: For each element (except the first and last), check if it is greater than both neighbors. The result is stored in a boolean array, which is then used to build the Fenwick Tree.

4. **Complexity Analysis**: 
   - **Initialization**: O(n log n) time to build the Fenwick Tree.
   - **Update Query**: O(log n) per affected element (up to 3 elements per update).
   - **Count Query**: O(log n) per query to compute the range sum.
   - Overall, with q queries, the total time complexity is O(q log n), which is efficient for the given constraints.

class Solution:
    def countOfPeaks(self, nums: List[int], queries: List[List[int]]) -> List[int]:
        n = len(nums)
        peaks = [0] * n
        for i in range(1, n-1):
            if nums[i] > nums[i-1] and nums[i] > nums[i+1]:
                peaks[i] = 1
        
        class Fenw:
            def __init__(self, n):
                self.n = n
                self.tree = [0] * (n+1)
            
            def update(self, index, delta):
                i = index + 1
                while i <= self.n:
                    self.tree[i] += delta
                    i += i & -i
            
            def query(self, index):
                res = 0
                i = index + 1
                while i:
                    res += self.tree[i]
                    i -= i & -i
                return res
            
            def range_query(self, l, r):
                if l > r:
                    return 0
                if l == 0:
                    return self.query(r)
                return self.query(r) - self.query(l-1)
        
        fenw = Fenw(n)
        for i in range(n):
            if peaks[i]:
                fenw.update(i, 1)
        
        ans = []
        for q in queries:
            if q[0] == 1:
                l, r = q[1], q[2]
                if l+1 <= r-1:
                    count = fenw.range_query(l+1, r-1)
                else:
                    count = 0
                ans.append(count)
            else:
                idx = q[1]
                val = q[2]
                nums[idx] = val
                for i in [idx-1, idx, idx+1]:
                    if i < 1 or i > n-2:
                        continue
                    new_peak = 1 if (nums[i] > nums[i-1] and nums[i] > nums[i+1]) else 0
                    if new_peak != peaks[i]:
                        fenw.update(i, new_peak - peaks[i])
                        peaks[i] = new_peak
        return ans

### Explanation
1. **Initialization**: The `peaks` array is initialized by checking each element (except the first and last) to see if it is a peak. The Fenwick Tree is built using this array to store the count of peaks.
2. **Query Processing**:
   - **Count Query (Type 1)**: The range `[l+1, r-1]` is queried in the Fenwick Tree to get the number of peaks in the subarray from index `l` to `r`.
   - **Update Query (Type 2)**: The specified element is updated in the array. The peak status of the updated element and its neighbors is recalculated. If the peak status changes, the Fenwick Tree is updated accordingly.
3. **Efficiency**: The use of the Fenwick Tree ensures that both update and count operations are handled in logarithmic time, making the solution efficient for large input sizes.
