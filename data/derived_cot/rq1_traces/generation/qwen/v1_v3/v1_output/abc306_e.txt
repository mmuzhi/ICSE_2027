=== ANSWER ===

To solve this problem, we need to efficiently maintain a sequence of numbers and, after each update, compute the sum of the top K elements in descending order. Given the constraints, a brute-force approach of sorting the array after each update is infeasible. Instead, we use a combination of Fenwick Trees (Binary Indexed Trees) and binary search to dynamically track the necessary elements and their sums.

### Approach
1. **Problem Analysis**: The problem requires handling multiple updates to an initially zero-filled array and, after each update, computing the sum of the top K elements. The challenge lies in efficiently maintaining the top K elements and their sums under frequent updates.

2. **Intuition**: The solution involves using two Fenwick Trees (one for frequencies and one for sums) to keep track of the elements' occurrences and their contributions to the sum. By compressing the values, we can efficiently query the trees. The key insight is that the sum of the top K elements can be derived by subtracting the sum of the smallest (N-K) elements from the total sum of the array.

3. **Algorithm Selection**: 
   - **Fenwick Trees**: These data structures allow efficient point updates and prefix sum queries in logarithmic time.
   - **Binary Search**: Used to find the (N-K+1)-th smallest element, which helps in determining the sum of the smallest (N-K) elements.
   - **Coordinate Compression**: Reduces the range of values to a smaller set, optimizing the Fenwick Tree operations.

4. **Complexity Analysis**: 
   - **Initialization**: O(N + Q + M log M), where M is the number of distinct values.
   - **Each Update**: O(log M) for updating the Fenwick Trees and O(log M) for each binary search step, leading to O((log M)^2) per update.
   - **Overall Complexity**: O((N + Q) * (log M)^2), which is feasible given the constraints.

import sys

class Fenw:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)
    
    def update(self, index, delta):
        while index <= self.n:
            self.tree[index] += delta
            index += index & -index
            
    def query(self, index):
        s = 0
        while index:
            s += self.tree[index]
            index -= index & -index
        return s

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it))
    k_val = int(next(it))
    q = int(next(it))
    
    updates = []
    all_values = set()
    all_values.add(0)
    for _ in range(q):
        x = int(next(it))
        y = int(next(it))
        updates.append((x, y))
        all_values.add(y)
        
    comp = sorted(all_values)
    m = len(comp)
    mapping = {}
    for i, val in enumerate(comp):
        mapping[val] = i + 1
        
    arr = [0] * (n + 1)
    total_sum = 0
    
    fenw_freq = Fenw(m)
    fenw_sum = Fenw(m)
    
    pos0 = mapping[0]
    fenw_freq.update(pos0, n)
    fenw_sum.update(pos0, 0)
    
    out_lines = []
    for (x, y) in updates:
        old_val = arr[x]
        new_val = y
        total_sum += (new_val - old_val)
        
        pos_old = mapping[old_val]
        pos_new = mapping[new_val]
        
        fenw_freq.update(pos_old, -1)
        fenw_freq.update(pos_new, 1)
        
        fenw_sum.update(pos_old, -old_val)
        fenw_sum.update(pos_new, new_val)
        
        arr[x] = new_val
        
        required_count = n - k_val + 1
        low = 1
        high = m
        while low < high:
            mid = (low + high) // 2
            if fenw_freq.query(mid) >= required_count:
                high = mid
            else:
                low = mid + 1
                
        idx = low
        x_val = comp[idx - 1]
        count_less = fenw_freq.query(idx - 1)
        sum_less = fenw_sum.query(idx - 1)
        
        T = sum_less + x_val * ( (n - k_val) - count_less )
        ans = total_sum - T
