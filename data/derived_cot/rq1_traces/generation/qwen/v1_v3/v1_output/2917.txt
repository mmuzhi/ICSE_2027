To solve this problem, we need to count the number of pairs (i, j) in an array where the sum of the elements at these indices is strictly less than a given target. The solution involves checking all possible pairs (i, j) where i < j and counting those pairs that meet the condition.

### Approach
1. **Problem Analysis**: The problem requires us to find all pairs of indices (i, j) such that i < j and the sum of the elements at these indices is less than the target. Given the constraints (array length up to 50), a brute-force approach is feasible.
2. **Intuition**: For each element in the array, we can check all subsequent elements to see if their sum with the current element is less than the target. This involves nested loops where the outer loop runs from the start to the end of the array, and the inner loop runs from the current position of the outer loop to the end of the array.
3. **Algorithm Selection**: The straightforward approach of using nested loops to check all possible pairs ensures that we cover every possible pair exactly once. This method is efficient enough given the small input size constraint.
4. **Complexity Analysis**: The time complexity of this approach is O(n^2), where n is the length of the array. For n = 50, this results in 1275 operations, which is computationally feasible.

### Solution Code
```python
class Solution:
    def countPairs(self, nums: List[int], target: int) -> int:
        n = len(nums)
        count = 0
        for i in range(n):
            for j in range(i + 1, n):
                if nums[i] + nums[j] < target:
                    count += 1
        return count
```

### Explanation
1. **Initialization**: We start by determining the length of the array and initializing a counter `count` to zero.
2. **Nested Loops**: The outer loop iterates over each element in the array (index `i`). The inner loop iterates over all elements that come after `i` (index `j`).
3. **Condition Check**: For each pair (i, j), we check if the sum of the elements at these indices is less than the target. If it is, we increment the counter.
4. **Result**: After processing all pairs, the counter holds the total number of valid pairs, which is returned as the result.

This approach efficiently checks all possible pairs and counts those that meet the condition, leveraging the constraints to ensure optimal performance.
