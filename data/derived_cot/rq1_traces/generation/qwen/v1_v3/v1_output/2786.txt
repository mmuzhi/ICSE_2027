To solve this problem, we need to find the longest substring in a given string of digits that is semi-repetitive. A semi-repetitive substring is defined as one that contains at most one pair of consecutive identical digits. 

### Approach
1. **Problem Analysis**: The task is to identify the longest contiguous substring where there are no more than one adjacent duplicate digits. For example, "0010" is valid because it has only one pair of consecutive zeros, whereas "00101022" is invalid due to multiple consecutive duplicates.
2. **Intuition**: We can use a sliding window technique to efficiently explore all possible substrings. The window expands by moving the right pointer and contracts from the left when more than one consecutive duplicate is found within the current window.
3. **Algorithm Selection**: The sliding window approach is chosen for its efficiency in handling substring problems. We maintain a window [left, right] and track the number of consecutive duplicates within this window. If the count exceeds one, we move the left pointer to the right until the window meets the semi-repetitive condition again.
4. **Complexity Analysis**: The algorithm processes each character in the string once (O(n) time complexity) and uses a constant amount of additional space (O(1)), making it efficient for the given constraints (string length up to 50).

### Solution Code
```python
class Solution:
    def longestSemiRepetitiveSubstring(self, s: str) -> int:
        n = len(s)
        left = 0
        duplicates = 0
        max_len = 1
        
        for right in range(1, n):
            if s[right] == s[right-1]:
                duplicates += 1
            
            while duplicates > 1:
                if s[left] == s[left+1]:
                    duplicates -= 1
                left += 1
            
            max_len = max(max_len, right - left + 1)
        
        return max_len
```

### Explanation
1. **Initialization**: We start with the left pointer at the beginning of the string, a counter for duplicates set to zero, and the maximum length initialized to 1 (the minimum valid substring length).
2. **Sliding Window**: The right pointer moves from the second character to the end of the string. For each position of the right pointer, we check if the current character and the previous one form a consecutive duplicate. If they do, we increment the duplicates counter.
3. **Window Adjustment**: If the duplicates counter exceeds 1, we move the left pointer to the right until the duplicates counter is back to 1 or less. During this adjustment, we check if the leftmost character and its next character form a duplicate, decrementing the counter if they do.
4. **Update Maximum Length**: After each adjustment, we update the maximum length if the current window size (right - left + 1) is greater than the previously recorded maximum.
5. **Result**: The algorithm returns the length of the longest semi-repetitive substring found during the traversal.

This approach efficiently checks all possible valid substrings while maintaining optimal performance through the sliding window technique.
