To solve this problem, we need to simulate the movement of Takahashi on a toroidal grid, where he repaints cells and changes direction based on their color. The grid starts entirely white, and Takahashi begins at the top-left corner facing upwards. The operation is repeated N times, and after each operation, the grid's state must be updated according to the rules provided.

### Approach
1. **Initialization**: Create a grid of size H x W initialized with white cells ('.'). Start at position (0, 0) (0-indexed) facing upwards (direction 0).
2. **Direction Handling**: Define four possible directions (up, right, down, left) using direction vectors. The direction changes based on the current cell's color: rotate 90 degrees clockwise if the cell is white, and counterclockwise if it's black.
3. **Movement**: After repainting the current cell, move one cell in the new direction, wrapping around the grid using modulo arithmetic to handle the toroidal nature.
4. **Simulation**: Repeat the operation N times, updating the grid and position accordingly. After N operations, print the final grid.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    H = int(data[0])
    W = int(data[1])
    N = int(data[2])
    
    grid = [['.' for _ in range(W)] for _ in range(H)]
    r, c = 0, 0
    direction = 0
    directions = [(-1, 0), (0, 1), (1, 0), (0, -1)]
    
    for _ in range(N):
        if grid[r][c] == '.':
            grid[r][c] = '#'
            direction = (direction + 1) % 4
        else:
            grid[r][c] = '.'
            direction = (direction + 3) % 4
        
        dr, dc = directions[direction]
        r = (r + dr) % H
        c = (c + dc) % W
    
    for row in grid:
        print(''.join(row))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The grid is initialized with all cells set to '.' (white). The starting position is (0, 0), and the initial direction is 0 (upwards).
2. **Direction Handling**: The directions are stored as tuples representing movement in each direction. The direction changes based on the cell's color: clockwise for white cells and counterclockwise for black cells.
3. **Movement**: For each operation, the current cell is repainted (black if white, white if black), the direction is updated, and the position is recalculated using modulo arithmetic to stay within grid bounds, considering the toroidal nature of the grid.
4. **Output**: After processing all N operations, the grid is printed row by row, showing the final state of each cell.

This approach efficiently simulates the operations by leveraging modulo arithmetic for toroidal movement and straightforward conditional checks for cell color changes, ensuring correctness and simplicity.
