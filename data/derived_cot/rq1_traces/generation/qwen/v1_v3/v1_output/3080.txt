To solve this problem, we need to split an array into one or more contiguous subarrays such that the sum of the bitwise AND of all subarrays is minimized. The key insight is that the minimal sum is achieved either by having the entire array as a single subarray (if its bitwise AND is non-zero) or by splitting the array into multiple subarrays each with a bitwise AND of zero (if the entire array's bitwise AND is zero).

### Approach
1. **Check Entire Array's AND**: First, compute the bitwise AND of the entire array. If this value is non-zero, the minimal sum is this value, and it can only be achieved by keeping the entire array as a single subarray. Hence, the answer is 1 in this case.
2. **Handle Zero AND Case**: If the entire array's bitwise AND is zero, we need to split the array into the maximum number of subarrays where each subarray's bitwise AND is zero. This involves:
   - **Suffix AND Array**: Precompute a suffix array where each element at index `i` represents the bitwise AND of all elements from `i` to the end of the array.
   - **Greedy Splitting**: Traverse the array while maintaining a running bitwise AND. Whenever this running AND becomes zero and the remaining part of the array (if any) also has a bitwise AND of zero, we can split the array here. This ensures each subarray's AND is zero, contributing to the minimal sum of zero.

### Solution Code
```python
class Solution:
    def maxSubarrays(self, nums: List[int]) -> int:
        n = len(nums)
        x = nums[0]
        for i in range(1, n):
            x &= nums[i]
        
        if x != 0:
            return 1
        
        suf = [0] * n
        suf[-1] = nums[-1]
        for i in range(n-2, -1, -1):
            suf[i] = nums[i] & suf[i+1]
        
        segments = 0
        cur = (1 << 21) - 1
        i = 0
        while i < n:
            cur &= nums[i]
            if cur == 0:
                if i == n-1 or suf[i+1] == 0:
                    segments += 1
                    i += 1
                    cur = (1 << 21) - 1
                else:
                    i += 1
            else:
                i += 1
        
        return segments if segments != 0 else 1
```

### Explanation
1. **Initial Check**: The code first checks if the bitwise AND of the entire array is non-zero. If so, it returns 1 because the minimal sum is achieved only by keeping the array unsplit.
2. **Suffix Array Calculation**: If the entire array's AND is zero, the code computes a suffix array `suf` where each element `suf[i]` is the bitwise AND from index `i` to the end of the array.
3. **Greedy Splitting**: The code then traverses the array, maintaining a running bitwise AND (`cur`). Whenever `cur` becomes zero and either we are at the last element or the remaining array's AND (`suf[i+1]`) is zero, it increments the segment count and resets `cur` for the next subarray. This ensures each subarray's AND is zero, maximizing the number of subarrays while maintaining the minimal sum.
4. **Result Handling**: If no valid splits are found (though theoretically, at least one subarray should be valid), it defaults to returning 1, representing the entire array as a single subarray.

This approach efficiently balances between maximizing the number of subarrays and ensuring the sum of their bitwise ANDs is minimized, leveraging bitwise operations and greedy traversal for optimal performance.
