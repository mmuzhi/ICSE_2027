=== ANSWER ===

To solve this problem, we need to efficiently process a series of queries on a dynamic sequence. The sequence starts empty, and each query either appends an element to the end of the sequence or requests the k-th element from the end of the sequence. Given the constraints, a straightforward approach using a list is sufficient.

### Approach
1. **Problem Analysis**: The problem involves two types of queries:
   - **Type 1 (1 x)**: Append the integer `x` to the end of the sequence.
   - **Type 2 (2 k)**: Retrieve the k-th element from the end of the sequence, ensuring the sequence length is at least `k` at the time of the query.
2. **Intuition**: Since the sequence operations are sequential and the number of queries is limited (up to 100), we can use a simple list to store the sequence elements. For retrieval queries, accessing the k-th element from the end can be done using negative indexing (i.e., `A[-k]`).
3. **Algorithm Selection**: The solution involves maintaining a list where each append operation adds an element to the end, and each retrieval operation uses negative indexing to access the desired element from the end.
4. **Complexity Analysis**: Each append operation is O(1) time complexity, and each retrieval operation is O(1) time complexity due to direct indexing. The space complexity is O(N) where N is the number of elements in the sequence, which is optimal for the given constraints.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    q = int(data[0])
    A = []
    index = 1
    output_lines = []
    for _ in range(q):
        t = data[index]; index += 1
        if t == '1':
            x = int(data[index]); index += 1
            A.append(x)
        elif t == '2':
            k = int(data[index]); index += 1
            output_lines.append(str(A[-k]))
    print("\n".join(output_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first value is the number of queries, `Q`.
2. **Processing Queries**: For each query:
   - **Type 1 (Append)**: The integer `x` is appended to the list `A`.
   - **Type 2 (Retrieve)**: The k-th element from the end of `A` is accessed using negative indexing (`A[-k]`) and stored for output.
3. **Output**: The results of all retrieval queries are collected in a list and printed at the end, each on a new line.

This approach efficiently handles the problem requirements with minimal computational overhead, leveraging Python's list operations for simplicity and clarity.
