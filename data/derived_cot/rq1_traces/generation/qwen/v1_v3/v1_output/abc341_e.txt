A string consisting of 0 and 1 is called a good string if two consecutive characters in the string are always different.
You are given a string S of length N consisting of 0 and 1.
Q queries will be given and must be processed in order.
There are two types of queries:

- 1 L R: Flip each of the L-th to R-th characters of S. That is, for each integer i satisfying L\leq i\leq R, change the i-th character of S to 0 if it is 1, and vice versa.
- 2 L R: Let S' be the string of length (R-L+1) obtained by extracting the L-th to R-th characters of S (without changing the order). Print Yes if S' is a good string and No otherwise.

Input

The input is given from Standard Input in the following format:
N Q
S
query_1
query_2
\vdots
query_Q

Each query query_i (1\leq i\leq Q) is given in the form:
1 L R 

or:
2 L R

Output

Let K be the number of queries of type 2. Print K lines.
The i-th line should contain the response to the i-th query of type 2.

Constraints


- 1\leq N, Q\leq 5\times 10^5
- S is a string of length N consisting of 0 and 1.
- 1\leq L\leq R\leq N for queries of types 1 and 2.
- There is at least one query of type 2.
- N, Q, L, and R are integers.

10100

2 2 4

=== ANSWER ===

To solve this problem, we need to efficiently handle two types of queries on a binary string: flipping a substring and checking if a substring is "good". A "good" string is defined as one where no two consecutive characters are the same. Given the constraints, a naive approach would be too slow, so we use a segment tree with lazy propagation to manage these operations efficiently.

### Approach
1. **Segment Tree with Lazy Propagation**: The segment tree will store information about each segment of the string, including the first and last characters, whether the segment is "good", and a lazy flag for pending flips. This allows us to perform range updates (flips) and range queries efficiently.
2. **Handling Flips**: When flipping a range, we update the segment tree nodes by toggling the first and last characters of the affected segments. The lazy flag ensures that updates are propagated only when necessary.
3. **Checking Good Strings**: For a query asking if a substring is "good", we combine the relevant segments, checking if all consecutive characters in the substring differ. The segment tree's structure allows this check to be done in logarithmic time.

                lazy[2*idx+2] = (lazy[2*idx+2] + 1) % 2
            return

        mid = (l + r) // 2
        update(2*idx+1, l, mid, L, R)
        update(2*idx+2, mid+1, r, L, R)
        left_node = tree[2*idx+1]
        right_node = tree[2*idx+2]
        first = left_node[0]
        last = right_node[1]
        length = left_node[4] + right_node[4]
        is_good = left_node[2] and right_node[2] and (left_node[1] != right_node[0])
        tree[idx] = [first, last, is_good, 0, length]
    
    def query(idx, l, r, L, R):
        if lazy[idx] != 0:
            if tree[idx][4] > 0:
                tree[idx][0] = 1 - tree[idx][0]
                tree[idx][1] = 1 - tree[idx][1]
            if l != r:
                lazy[2*idx+1] = (lazy[2*idx+1] + lazy[idx]) % 2
                lazy[2*idx+2] = (lazy[2*idx+2] + lazy[idx]) % 2
            lazy[idx] = 0

        if r < L or R < l:
            return [0, 0, True, 0]
            
        if L <= l and r <= R:
            return [tree[idx][0], tree[idx][1], tree[idx][2], tree[idx][4]]
            
        mid = (l + r) // 2
        left_node = query(2*idx+1, l, mid, L, R)
        right_node = query(2*idx+2, mid+1, r, L, R)
        
        if left_node[3] == 0:
            return right_node
        elif right_node[3] == 0:
            return left_node
        else:
            first = left_node[0]
            last = right_node[1]
            length = left_node[3] + right_node[3]
            is_good = left_node[2] and right_node[2] and (left_node[1] != right_node[0])
            return [first, last, is_good, length]
    
    output_lines = []
    index = 2
    for _ in range(q):
        parts = data[index].split()
        index += 1
        t = int(parts[0])
        L = int(parts[1])
        R = int(parts[2])
        if t == 1:
            update(0, 0, n-1, L-1, R-1)
        else:
            node = query(0, 0, n-1, L-1, R-1)
            if node[2]:
                output_lines.append("Yes")
            else:
                output_lines.append("No")
    
    for line in output_lines:
        print(line)

if __name__ == "__main__":

### Explanation
1. **Segment Tree Construction**: The segment tree is built to store each segment's first and last characters, whether the segment is "good", and its length. The tree is constructed recursively, combining child segments into parent segments.
2. **Update Operation (Flip)**: When flipping a range, the segment tree nodes covering this range are updated by toggling their first and last characters. Lazy propagation ensures that updates are deferred until necessary, optimizing performance.
3. **Query Operation (Check Good String)**: For a query, the relevant segments are combined, checking if all consecutive characters in the substring differ. The result is derived by combining the segments' information, ensuring efficient query resolution in logarithmic time.
4. **Efficiency**: Both update and query operations leverage the segment tree's structure, ensuring they run in O(log N) time per operation, making the solution efficient for large inputs.
