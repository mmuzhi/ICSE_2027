The tree is constructed such that each vertex i (from 2 to N) has an edge connecting it to its parent vertex ⌊i/2⌋. The solution involves efficiently navigating both downward and upward paths from vertex X to count the nodes at distance K without explicitly traversing the entire tree, which is crucial given the constraints (N can be as large as 10^18).

### Approach
1. **Problem Analysis**: The tree is a binary heap-like structure where each node i has children 2*i and 2*i+1 (if they exist within N). The problem requires counting nodes at exactly K edges from X, considering both downward and upward paths.
2. **Intuition**: 
   - **Downward Path**: Nodes reachable by moving only downward from X (children, grandchildren, etc.) at depth K.
   - **Upward Path**: Nodes reachable by moving upward (towards the root) i steps and then downward (K - i) steps in a different branch (not the path back to X).
3. **Algorithm Selection**: 
   - **Count Nodes at a Level**: For a given node and depth, count the number of nodes in its subtree at that depth, considering the tree's boundary (N).
   - **Ancestors Path**: For each test case, compute the path from X to the root to determine ancestors and forbidden branches (branches leading back to X).
   - **Combining Results**: Sum the nodes from the downward path and valid nodes from the upward paths.
4. **Complexity Analysis**: The solution efficiently handles large N by leveraging mathematical calculations (bit shifts and divisions) to avoid explicit tree traversal. The complexity is O(T * K), where K is the maximum distance (depth) considered, which is feasible given the constraints (K is up to 60 for N=10^18).

def count_nodes_at_level(node, depth, N):
    if depth < 0:
        return 0
    if node == 0:
        return 0
    base = 1 << depth
    quotient = N // base
    if node > quotient:
        return 0
    low_bound = node * base
    high_bound = low_bound + base - 1
    if high_bound <= N:
        return base
    else:
        return N - low_bound + 1
