=== ANSWER ===

To solve this problem, we need to find the maximum value of the expression `(nums[i] - nums[j]) * nums[k]` for all triplets of indices `(i, j, k)` such that `i < j < k`. If all possible triplets yield a negative value, we should return 0.

### Approach
1. **Problem Analysis**: The key observation is that the product `(nums[i] - nums[j]) * nums[k]` is maximized when `nums[i]` is as large as possible (to make the difference `nums[i] - nums[j]` positive and large) and `nums[k]` is as large as possible. For each middle index `j`, the optimal `i` is the index of the maximum value to the left of `j`, and the optimal `k` is the index of the maximum value to the right of `j`.

2. **Intuition**: By precomputing two arrays:
   - `max_left[j]`: The maximum value in the subarray `nums[0..j-1]`.
   - `max_right[j]`: The maximum value in the subarray `nums[j+1..n-1]`.
   We can efficiently determine the best `i` and `k` for each `j` and compute the product `(max_left[j] - nums[j]) * max_right[j]`.

3. **Algorithm Selection**: 
   - **Precompute `max_left` and `max_right` arrays**: These arrays help in quickly accessing the maximum values to the left and right of any index `j`.
   - **Iterate through each possible `j`**: For each `j`, compute the product using the precomputed `max_left[j]` and `max_right[j]`, and keep track of the maximum product encountered.

4. **Complexity Analysis**: The algorithm runs in O(n) time for precomputing the `max_left` and `max_right` arrays and O(n) time for iterating through each `j`, resulting in an overall O(n) time complexity. The space complexity is O(n) due to the storage of the two auxiliary arrays.

### Solution Code
```python
class Solution:
    def maximumTripletValue(self, nums: List[int]) -> int:
        n = len(nums)
        max_left = [-10**18] * n
        max_right = [-10**18] * n
        
        if n > 1:
            max_left[0] = -10**18
            for i in range(1, n):
                max_left[i] = max(max_left[i-1], nums[i-1])
        
        if n > 1:
            max_right[n-1] = -10**18
            for i in range(n-2, -1, -1):
                max_right[i] = max(nums[i+1], max_right[i+1])
        
        ans = 0
        for j in range(1, n-1):
            candidate = (max_left[j] - nums[j]) * max_right[j]
            if candidate > ans:
                ans = candidate
        
        return ans
```

### Explanation
1. **Initialization**: The `max_left` and `max_right` arrays are initialized with a very small value (`-10^18`) to handle cases where no elements are present to the left or right of an index.
2. **Filling `max_left`**: For each index `i` from 1 to `n-1`, `max_left[i]` stores the maximum value encountered from the start of the array up to `i-1`.
3. **Filling `max_right`**: For each index `i` from `n-2` down to 0, `max_right[i]` stores the maximum value encountered from `i+1` to the end of the array.
4. **Finding Maximum Product**: For each middle index `j` (from 1 to `n-2`), the product `(max_left[j] - nums[j]) * max_right[j]` is computed. The maximum of these products is stored in `ans`, which is returned as the result. If no positive product is found, `ans` remains 0, as required.

This approach efficiently narrows down the problem to checking only the most promising triplets by leveraging precomputed maximum values, ensuring optimal performance.
