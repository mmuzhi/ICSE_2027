To solve this problem, we need to determine the maximum possible minimum path length between vertex 1 and vertex (N1 + N2) in an undirected graph after adding exactly one edge between the two disconnected components (A and B). The graph consists of two connected components: vertices 1 to N1 (component A) and vertices N1+1 to N1+N2 (component B). The solution involves leveraging BFS to compute the shortest distances from vertex 1 in component A and from vertex (N1+N2) in component B, then finding the maximum distances in each component to compute the desired result.

### Approach
1. **Problem Analysis**: The graph is divided into two connected components, A and B, with no edges between them initially. Adding an edge between any vertex in A and any vertex in B will connect the two components. The minimal path from vertex 1 to vertex (N1+N2) in the resulting graph must pass through this added edge. The minimal path length is the sum of the shortest path from vertex 1 to the chosen vertex in A, the added edge (length 1), and the shortest path from the chosen vertex in B to vertex (N1+N2).

2. **Intuition**: To maximize the minimal path length, we need to choose vertices in A and B that are farthest from vertex 1 and vertex (N1+N2), respectively. The BFS algorithm efficiently computes the shortest paths in unweighted graphs, which is suitable here since all edges have the same weight.

3. **Algorithm Selection**: 
   - **BFS from Vertex 1**: Compute the shortest distances from vertex 1 to all vertices in component A.
   - **BFS from Vertex (N1+N2)**: Compute the shortest distances from vertex (N1+N2) to all vertices in component B.
   - **Maximize Path Length**: The maximum minimal path length is obtained by adding the maximum distance from vertex 1 in A, the maximum distance from vertex (N1+N2) in B, and 1 (for the added edge).

4. **Complexity Analysis**: The BFS operations run in O(N + M) time, where N is the number of vertices and M is the number of edges. Given the constraints (N up to 300,000 and M up to 600,000), this approach is efficient.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    n1 = int(next(it))
    n2 = int(next(it))
    m = int(next(it))
    n = n1 + n2
    
    graph = [[] for _ in range(n+1)]
    
    for _ in range(m):
        a = int(next(it))
        b = int(next(it))
        graph[a].append(b)
        graph[b].append(a)
        
    dA = [-1] * (n+1)
    q = deque([1])
    dA[1] = 0
    while q:
        cur = q.popleft()
        for neighbor in graph[cur]:
            if dA[neighbor] == -1:
                dA[neighbor] = dA[cur] + 1
                q.append(neighbor)
                
    dB = [-1] * (n+1)
    q = deque([n])
    dB[n] = 0
    while q:
        cur = q.popleft()
        for neighbor in graph[cur]:
            if dB[neighbor] == -1:
                dB[neighbor] = dB[cur] + 1
                q.append(neighbor)
                
    max_dA = 0
    for i in range(1, n1+1):
        if dA[i] > max_dA:
            max_dA = dA[i]
            
    max_dB = 0
    for i in range(n1+1, n+1):
        if dB[i] > max_dB:
            max_dB = dB[i]
            
    print(max_dA + max_dB + 1)

if __name__ == "__main__":
    main()
```
