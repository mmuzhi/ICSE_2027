=== ANSWER ===

To solve this problem, we need to transform a given permutation into a semi-ordered permutation with the minimum number of adjacent swaps. A semi-ordered permutation requires the first element to be 1 and the last element to be n (the length of the permutation). 

### Approach
1. **Identify Positions**: Locate the indices of the elements 1 and n in the permutation.
2. **Case Handling**:
   - **Case 1 (1 is left of n)**: The minimal swaps required is the sum of the distance to move 1 to the front and the adjusted distance to move n to the back. The adjusted distance for n is calculated considering the initial positions.
   - **Case 2 (1 is right of n)**: There are two strategies:
     - **Strategy 1**: Move 1 to the front first, then move n to the back. The total swaps are the sum of the distance to move 1 and the adjusted distance to move n.
     - **Strategy 2**: Move n to the back first, then move 1 to the front. The total swaps are the sum of the distance to move n and the adjusted distance to move 1.
   The minimal swaps between these two strategies is chosen.

### Solution Code
```python
class Solution:
    def semiOrderedPermutation(self, nums: List[int]) -> int:
        n = len(nums)
        i = None
        j = None
        for idx, num in enumerate(nums):
            if num == 1:
                i = idx
            if num == n:
                j = idx
        if i < j:
            return i + (n - 1 - j)
        else:
            option1 = i + (n - 2 - j)
            option2 = (n - 1 - j) + (i - 1)
            return min(option1, option2)
```

### Explanation
1. **Finding Indices**: The code first identifies the positions of the elements 1 and n in the permutation.
2. **Case 1 (i < j)**: If 1 is to the left of n, the minimal swaps required is simply the sum of the distance from 1 to the start (i) and the distance from n to the end (n-1-j). This is because moving 1 to the front does not affect n's position.
3. **Case 2 (i > j)**: If 1 is to the right of n, there are two strategies:
   - **Option 1**: Move 1 to the front first (i swaps), then move n to the back (n-2-j swaps, as n's position is now j+1).
   - **Option 2**: Move n to the back first (n-1-j swaps), then move 1 to the front (i-1 swaps, as 1's position is now i-1 after n's movement).
   The minimal swaps between these two options is chosen.
   
This approach efficiently calculates the minimal adjacent swaps required to transform the permutation into a semi-ordered one by leveraging the relative positions of 1 and n.
