We have N cups of ice cream.
The flavor and deliciousness of the i-th cup are F_i and S_i, respectively (S_i is an even number).

Your satisfaction here is defined as follows.

- Let s and t (s \ge t) be the deliciousness of the eaten cups.
- If the two cups have different flavors, your satisfaction is \displaystyle s+t.
- Otherwise, your satisfaction is \displaystyle s + \frac{t}{2}.



Find the maximum achievable satisfaction.

Input

Input is given from Standard Input in the following format:
N
F_1 S_1
F_2 S_2
\vdots
F_N S_N

Output

Print the answer as an integer.

Constraints


- All input values are integers.
- 2 \le N \le 3 \times 10^5
- 1 \le F_i \le N
- 2 \le S_i \le 10^9
- S_i is even.

Sample Input 1

4
1 4
2 10
2 8
3 6

Sample Output 1

16

You cannot achieve a satisfaction greater than 16.

To solve this problem, we need to maximize the satisfaction derived from eating two cups of ice cream, considering either the same flavor or different flavors. The satisfaction is calculated differently based on whether the two chosen cups have the same flavor or different flavors. 

### Approach
1. **Problem Analysis**: The problem requires selecting two cups of ice cream from a list, where each cup has a flavor and a deliciousness value (an even number). The satisfaction is defined as:
   - If the two cups have different flavors: the sum of their deliciousness values.
   - If they have the same flavor: the sum of the highest deliciousness value and half of the second highest (since the value is even, division by 2 is integer).

2. **Intuition**: The solution involves two main cases:
   - **Same Flavor Case**: For each flavor with at least two cups, compute the satisfaction by taking the two highest deliciousness values and applying the formula `s + t/2`.
   - **Different Flavor Case**: For all distinct flavors, compute the satisfaction by taking the two highest deliciousness values from different flavors and summing them.

3. **Algorithm Selection**:
   - **Data Structures**: Use dictionaries to keep track of the highest (`max1`) and second highest (`max2`) deliciousness values for each flavor.
   - **Processing**: For each cup, update the dictionaries to maintain the top two values for each flavor.
   - **Result Calculation**: After processing all cups, compute the maximum satisfaction from both cases (same and different flavors) and return the higher value.

4. **Complexity Analysis**: The algorithm processes each cup exactly once, making it O(N) time complexity, where N is the number of cups. The space complexity is O(F), where F is the number of distinct flavors, which is efficient given the constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    n = int(data[0])
    max1 = {}
    max2 = {}
    index = 1
    for _ in range(n):
        parts = data[index].split()
        index += 1
        f = int(parts[0])
        s = int(parts[1])
        if f not in max1:
            max1[f] = s
        else:
            if s > max1[f]:
                max2[f] = max1[f]
                max1[f] = s
            elif s == max1[f]:
                if f not in max2 or s >= max2[f]:
                    max2[f] = s
            else:
                if f not in max2 or s > max2[f]:
                    max2[f] = s

    candidate_same = 0
    for f in max2:
        total = max1[f] + max2[f] // 2
        if total > candidate_same:
            candidate_same = total

    candidate_diff = 0
    if len(max1) >= 2:
        values = list(max1.values())
        values.sort(reverse=True)
        candidate_diff = values[0] + values[1]

    print(max(candidate_same, candidate_diff))

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints.
2. **Tracking Maximum Values**: For each flavor, we maintain the highest (`max1`) and second highest (`max2`) deliciousness values encountered. This is done by updating the dictionaries as we process each cup.
3. **Same Flavor Calculation**: For each flavor with at least two cups, the satisfaction is calculated as the sum of the highest value and half of the second highest value. The maximum value from all such flavors is stored in `candidate_same`.
4. **Different Flavor Calculation**: The highest values from all distinct flavors are collected, sorted in descending order, and the sum of the top two values is stored in `candidate_diff`.
5. **Result**: The higher value between `candidate_same` and `candidate_diff` is printed as the result, ensuring the optimal satisfaction is achieved. 

This approach efficiently handles both cases (same and different flavors) by leveraging dictionary operations and sorting, ensuring optimal performance even for large input sizes.
