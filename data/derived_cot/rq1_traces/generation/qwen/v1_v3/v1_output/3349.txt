### Approach

2. **Intuition**: For each possible starting index in the string, expand the substring to the right while maintaining a frequency count of each character. If at any point a character's count exceeds two, skip further expansion from that starting index.
3. **Algorithm Selection**: We use a nested loop approach where the outer loop sets the starting index of the substring, and the inner loop expands the substring to the right. A frequency dictionary keeps track of character counts. For each expansion, we check if any character count exceeds two. If not, update the maximum length found.
4. **Complexity Analysis**: The algorithm checks all possible substrings, leading to a time complexity of O(n^2), where n is the length of the string. Given the constraint n ≤ 100, this approach is efficient enough.

class Solution:

1. **Initialization**: We start by initializing `max_len` to keep track of the longest valid substring found.
