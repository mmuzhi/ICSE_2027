You will be given a competitive programming problem.
There is a grid with H rows and W columns. Let (i, j) denote the cell at the i-th row from the top and the j-th column from the left. The state of each cell is represented by the character A_{i,j}, which means the following:

- .: An empty cell.
- #: An obstacle.
- S: An empty cell and the start point.
- T: An empty cell and the goal point.

Takahashi can move from his current cell to a vertically or horizontally adjacent empty cell by consuming 1 energy. He cannot move if his energy is 0, nor can he exit the grid.
There are N medicines in the grid. The i-th medicine is at the empty cell (R_i, C_i) and can be used to set the energy to E_i. Note that the energy does not necessarily increase. He can use the medicine in his current cell. The used medicine will disappear.
Takahashi starts at the start point with 0 energy and wants to reach the goal point. Determine if this is possible.

Input

The input is given from Standard Input in the following format:
H W
A_{1, 1}A_{1, 2}\cdotsA_{1, W}
A_{2, 1}A_{2, 2}\cdotsA_{2, W}
\vdots
A_{H, 1}A_{H, 2}\cdotsA_{H, W}
N
R_1 C_1 E_1
R_2 C_2 E_2
\vdots
R_N C_N E_N

Output

If Takahashi can reach the goal point from the start point, print Yes; otherwise, print No.

Constraints


- 1 \leq H, W \leq 200
- A_{i, j} is one of ., #, S, and T.
- Each of S and T exists exactly once in A_{i, j}.
- 1 \leq N \leq 300
- 1 \leq R_i \leq H
- 1 \leq C_i \leq W
- (R_i, C_i) \neq (R_j, C_j) if i \neq j.
- A_{R_i, C_i} is not #.
- 1 \leq E_i \leq HW

Each state is defined by the current cell and whether a medicine has been used at that cell (state0: no medicine used, state1: medicine used). For each cell, we maintain two values: the maximum energy achievable without using a medicine (dp1) and the maximum energy achievable after using a medicine (dp2). 

4. **Complexity Analysis**: The algorithm processes each cell in two states (dp1 and dp2), leading to a time complexity of O(H * W * log(E)), where E is the maximum energy value (up to H*W). Given the constraints (H, W ≤ 200), this approach is feasible.

### Solution Code
```python
import heapq

def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        print("No")
        return
    
    H, W = map(int, data[0].split())
    grid = []
    for i in range(1, 1+H):
        grid.append(data[i].strip())
    
    N = int(data[1+H])
    med_energy = {}
    for i in range(1+H+1, 1+H+1+N):
        parts = data[i].split()
        if len(parts) < 3:
            continue
        r = int(parts[0]) - 1
        c = int(parts[1]) - 1
        e = int(parts[2])
        med_energy[(r, c)] = e

    start = None
    goal = None
    for i in range(H):
        for j in range(W):
            if grid[i][j] == 'S':
                start = (i, j)
            elif grid[i][j] == 'T':
                goal = (i, j)
    
    if start is None or goal is None:
        print("No")
        return

    dp1 = [[-1]*W for _ in range(H)]
    dp2 = [[-1]*W for _ in range(H)]
    
    sr, sc = start
    tr, tc = goal

    dp1[sr][sc] = 0
    if (sr, sc) in med_energy:
        dp2[sr][sc] = med_energy[(sr, sc)]
    
    queue = []
    heapq.heappush(queue, (-0, sr, sc, 0))
    if (sr, sc) in med_energy:
        heapq.heappush(queue, (-dp2[sr][sc], sr, sc, 1))

    while queue:
        neg_energy, r, c, state = heapq.heappop(queue)
        energy = -neg_energy
        
        if state == 0 and energy < dp1[r][c]:
            continue
        if state == 1 and energy < dp2[r][c]:
            continue
            
        if (r, c) == (tr, tc):
            print("Yes")
            return
            
        if state == 0 and (r, c) in med_energy and med_energy[(r, c)] > dp2[r][c]:
            dp2[r][c] = med_energy[(r, c)]
            heapq.heappush(queue, (-dp2[r][c], r, c, 1))
            
        for dr, dc in [(-1,0), (1,0), (0,-1), (0,1)]:
            nr, nc = r+dr, c+dc
            if not (0 <= nr < H and 0 <= nc < W):
                continue
            if grid[nr][nc] == '#':
                continue
            if energy <= 0:
                continue
            new_energy = energy - 1
            
            if state == 0:
                if new_energy > dp1[nr][nc]:
                    dp1[nr][nc] = new_energy
                    heapq.heappush(queue, (-new_energy, nr, nc, 0))
                    if (nr, nc) in med_energy and med_energy[(nr, nc)] > dp2[nr][nc]:
                        dp2[nr][nc] = med_energy[(nr, nc)]
                        heapq.heappush(queue, (-dp2[nr][nc], nr, nc, 1))
            else:
                if new_energy > dp1[nr][nc]:
                    dp1[nr][nc] = new_energy
                    heapq.heappush(queue, (-new_energy, nr, nc, 0))
                    if (nr, nc) in med_energy and med_energy[(nr, nc)] > dp2[nr][nc]:
                        dp2[nr][nc] = med_energy[(nr, nc)]
                        heapq.heappush(queue, (-dp2[nr][nc], nr, nc, 1))
                        
    print("No")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The grid is read, and the positions of 'S' and 'T' are identified. Medicines are stored in a dictionary with their positions and energy values.
2. **Priority Queue Setup**: The priority queue starts with the initial state (0 energy at 'S'). If 'S' has a medicine, it is used to set the energy to the medicine's value.
3. **Processing States**: The algorithm processes each state by first checking if the current cell is 'T'. If not, it checks if a medicine can be used to increase energy. Then, it explores adjacent cells, updating their energy levels if a higher value is found.
4. **Termination**: If 'T' is reached during processing, "Yes" is printed; otherwise, after exhausting all possibilities, "No" is printed.

This approach efficiently explores all reachable cells while managing energy levels through priority-based processing, ensuring optimal performance for the given constraints.
