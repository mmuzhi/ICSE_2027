To solve this problem, we need to determine the minimum number of operations required to obtain a subsequence from an array of non-negative powers of 2 that sums to a given target. Each operation involves selecting an element greater than 1, removing it, and adding two occurrences of its half-value to the end of the array. 

### Approach
1. **Initial Check**: First, we check if the sum of all elements in the array is less than the target. If it is, it's impossible to form the target, so we return -1.
2. **Exponent Calculation**: For each element in the array, determine its exponent (i.e., the power of 2 it represents) and maintain a count of how many times each exponent appears.
3. **Greedy Processing**: Iterate from the highest exponent down to the lowest. For each bit set in the target's binary representation, if the corresponding exponent's count is zero, break a higher exponent's coin (if available) into two coins of the current exponent. Each such break operation increments the operation count.
4. **Result**: The total number of break operations required to ensure all bits of the target are covered is returned. If at any point a required exponent's coin cannot be obtained by breaking higher exponents, return -1.
