You will be given a competitive programming problem.
Takahashi has prepared N dishes for Snuke.
The dishes are numbered from 1 to N, and dish i has a sweetness of A_i and a saltiness of B_i.
Takahashi can arrange these dishes in any order he likes.
Snuke will eat the dishes in the order they are arranged, but if at any point the total sweetness of the dishes he has eaten so far exceeds X or the total saltiness exceeds Y, he will not eat any further dishes.
Takahashi wants Snuke to eat as many dishes as possible.
Find the maximum number of dishes Snuke will eat if Takahashi arranges the dishes optimally.

Input

The input is given from Standard Input in the following format:
N X Y
A_1 B_1
A_2 B_2
\vdots
A_N B_N

Output

Print the answer as an integer.

Constraints


- 1 \leq N \leq 80
- 1 \leq A_i, B_i \leq 10000
- 1 \leq X, Y \leq 10000
- All input values are integers.

Sample Input 1

4 8 4
1 5
3 2
4 1
5 3

Sample Output 1

3

- Next, Snuke eats dish 3. The total sweetness so far is 7, and the total saltiness is 3.
- Next, Snuke eats dish 1. The total sweetness so far is 8, and the total saltiness is 8.
- The total saltiness has exceeded Y=4, so Snuke will not eat any further dishes.

Thus, in this arrangement, Snuke will eat three dishes.
No matter how Takahashi arranges the dishes, Snuke will not eat all four dishes, so the answer is 3.

To solve this problem, we need to determine the maximum number of dishes Snuke can eat without the cumulative sweetness exceeding X or the cumulative saltiness exceeding Y at any point during the meal. The dishes can be arranged in any order, and Snuke stops eating if either condition is violated after consuming a dish.

### Approach
1. **Problem Analysis**: The problem requires finding the longest sequence of dishes where the cumulative sweetness and saltiness do not exceed the given limits (X and Y) at any prefix of the sequence. The key insight is that if a subset of dishes has a total sweetness ≤ X and total saltiness ≤ Y, there exists an ordering of these dishes such that no intermediate cumulative sum exceeds the limits. This assumption simplifies the problem to finding the largest such subset and then potentially adding one more dish that may violate the conditions but is eaten regardless.

2. **Algorithm Selection**: We use a dynamic programming approach with a dictionary to keep track of achievable states (sweetness, saltiness) and the maximum number of dishes that can be consumed without violating the conditions. For each dish, we update the dictionary by considering adding the dish's sweetness and saltiness to existing states, provided the new state remains within the limits.

3. **Complexity Analysis**: The algorithm processes each dish and updates the dictionary with new states. The number of states is bounded by the product of X and Y, but in practice, the dictionary size is manageable due to the constraints (N ≤ 80). The overall complexity is O(N * X * Y) in the worst case, but optimizations like early termination and state pruning help in reducing the actual runtime.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        print(0)
        return
    it = iter(data)
    N = int(next(it))
    X = int(next(it))
    Y = int(next(it))
    dishes = []
    for _ in range(N):
        a = int(next(it))
        b = int(next(it))
        dishes.append((a, b))
    
    dp = {}
    dp[(0, 0)] = 0
    for a, b in dishes:
        new_dp = dict(dp)
        for s, t in list(dp.items()):
            ns = s + a
            nt = t + b
            if ns <= X and nt <= Y:
                new_count = dp[(s, t)] + 1
                if (ns, nt) not in new_dp or new_dp[(ns, nt)] < new_count:
                    new_dp[(ns, nt)] = new_count
        dp = new_dp
        
    if not dp:
        k0 = 0
    else:
        k0 = max(dp.values())
        
    if k0 < N:
        print(k0 + 1)
    else:
        print(k0)

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read and parsed into the number of dishes (N), the sweetness limit (X), the saltiness limit (Y), and the list of dishes with their respective sweetness and saltiness values.

2. **Dynamic Programming Setup**: A dictionary `dp` is initialized to keep track of achievable states (sweetness, saltiness) and the maximum number of dishes (count) that can be consumed without exceeding the limits. The initial state (0, 0) with count 0 represents no dishes consumed.

3. **Processing Dishes**: For each dish, the algorithm updates the dictionary by considering the addition of the dish's sweetness and saltiness to each existing state. If the new state remains within the limits, it is added to the dictionary with an incremented count.

4. **Result Calculation**: After processing all dishes, the maximum count in the dictionary (`k0`) is determined. If `k0` is less than the total number of dishes (N), it means at least one more dish can be consumed (even if it violates the conditions), so the answer is `k0 + 1`. Otherwise, the answer is `k0`.

This approach efficiently explores all possible sequences of dishes while leveraging dynamic programming to keep track of valid states, ensuring optimal performance for the given constraints.
