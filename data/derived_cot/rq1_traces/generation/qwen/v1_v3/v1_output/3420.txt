=== ANSWER ===

To solve this problem, we need to find the indices of the k-th occurrences of a specific element `x` in an array `nums` for each query in `queries`. If the k-th occurrence does not exist, we return -1 for that query.

### Approach
1. **Problem Analysis**: The task involves processing an array `nums` to collect all indices where the element `x` appears. Then, for each query in `queries`, we need to determine if the specified occurrence (k-th) exists in the collected indices. The solution requires efficient handling of large input sizes (up to 10^5 elements), so the approach must be optimized for performance.

2. **Intuition**: By first traversing `nums` and recording all indices where `x` occurs, we can then directly access these indices for each query. This avoids repeatedly scanning `nums` for each query, thus optimizing both time and space complexity.

3. **Algorithm Selection**: The algorithm involves:
   - **Index Collection**: Iterate through `nums` once to gather all indices of `x` into a list `indices`.
   - **Query Processing**: For each query, check if the requested occurrence (k-th) is within the bounds of `indices`. If it is, return the corresponding index; otherwise, return -1.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n + q), where n is the length of `nums` and q is the number of queries. The traversal of `nums` is O(n), and each query is processed in O(1) time.
   - **Space Complexity**: O(m), where m is the number of occurrences of `x` in `nums`. This space is used to store the indices of `x`.

### Solution Code
```python
class Solution:
    def occurrencesOfElement(self, nums: List[int], queries: List[int], x: int) -> List[int]:
        indices = []
        for idx, num in enumerate(nums):
            if num == x:
                indices.append(idx)
                
        res = []
        for q in queries:
            if q <= len(indices):
                res.append(indices[q-1])
            else:
                res.append(-1)
        return res
```

### Explanation
1. **Index Collection**: The code first iterates through the `nums` array, checking each element. Whenever an element matches `x`, its index is added to the `indices` list. This step ensures that all positions of `x` are recorded in a single pass through the array.

2. **Query Processing**: For each query in `queries`, the code checks if the query value (k) is a valid index (1-based) in the `indices` list. If valid, it returns the corresponding index (adjusted to 0-based by subtracting 1). If the query exceeds the number of occurrences, it returns -1.

This approach efficiently handles the problem by leveraging a single traversal for index collection and constant-time lookups for each query, making it suitable for large input sizes as specified in the constraints.
