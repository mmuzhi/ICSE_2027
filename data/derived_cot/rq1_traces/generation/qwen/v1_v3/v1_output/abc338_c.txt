Your refrigerator has N kinds of ingredients. Let us call them ingredient 1, \dots, ingredient N. You have Q_i grams of ingredient i.
You can make two types of dishes. To make one serving of dish A, you need A_i grams of each ingredient i (1 \leq i \leq N). To make one serving of dish B, you need B_i grams of each ingredient i. You can only make an integer number of servings of each type of dish.
Using only the ingredients in the refrigerator, what is the maximum total number of servings of dishes you can make?

Input

The input is given from Standard Input in the following format:
N
Q_1 Q_2 \dots Q_N
A_1 A_2 \dots A_N
B_1 B_2 \dots B_N

Output

Assuming that you can make a maximum total of S servings of dishes, print the integer S.

Constraints


- 1 \leq N \leq 10
- 1 \leq Q_i \leq 10^6
- 0 \leq A_i \leq 10^6
- There is an i such that A_i \geq 1.
- 0 \leq B_i \leq 10^6
- There is an i such that B_i \geq 1.
- All input values are integers.

Sample Input 1

2
800 300
100 100
200 10

Sample Output 1

5

To solve this problem, we need to determine the maximum number of servings of two types of dishes that can be made using the available ingredients in a refrigerator. The dishes require specific amounts of each ingredient, and we can only make an integer number of servings. The challenge is to find the optimal combination of servings for both dishes without exceeding the available ingredients.

### Approach
1. **Binary Search on Total Servings**: The solution involves performing a binary search on the possible number of total servings (S) to determine the maximum feasible value. The binary search range is from 0 to a sufficiently large number (10^18) to cover all possible values.
2. **Feasibility Check**: For each candidate value of S during the binary search, we check if there exists a valid number of servings of dish A (x) and dish B (S - x) such that all ingredient constraints are satisfied. This involves:
   - **Linear Constraints**: For each ingredient, derive constraints on x based on the amounts required by both dishes. These constraints can be upper bounds (for ingredients where dish A requires more than dish B) or lower bounds (for ingredients where dish B requires more than dish A).
   - **Combining Constraints**: Ensure that the constraints derived from all ingredients can be satisfied simultaneously by checking if the intersection of all upper and lower bounds for x is non-empty.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
        
    n = int(data[0])
    Q = list(map(int, data[1:1+n]))
    A = list(map(int, data[1+n:1+2*n]))
    B = list(map(int, data[1+2*n:1+3*n]))
    
    low = 0
    high = 10**18
    
    def feasible(S):
        x_min = 0
        x_max = S
        for i in range(n):
            a_i = A[i]
            b_i = B[i]
            q_i = Q[i]
            d = a_i - b_i
            if d == 0:
                if b_i != 0 and S * b_i > q_i:
                    return False
            elif d > 0:
                if S * b_i > q_i:
                    return False
                x_max_i = (q_i - S * b_i) // d
                if x_max_i < 0:
                    return False
                if x_max_i < x_max:
                    x_max = x_max_i
            else:
                if q_i - S * b_i < 0:
                    a_val = S * b_i - q_i
                    b_val = -d
                    x_min_i = (a_val + b_val - 1) // b_val
                    if x_min_i > x_max:
                        return False
                    if x_min_i > x_min:
                        x_min = x_min_i
        return x_min <= x_max

    while low <= high:
        mid = (low + high) // 2
        if feasible(mid):
            low = mid + 1
        else:
            high = mid - 1
            
    print(high)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to a large number (10^18) to cover all possible values of total servings.
2. **Feasibility Function**: For each candidate value of S (mid in binary search), the function `feasible` checks if there exists a valid x (number of dish A servings) such that:
   - **Upper Bound Constraint**: For ingredients where dish A requires more than dish B, the maximum x is constrained by `(Q_i - S * B_i) // (A_i - B_i)`.
   - **Lower Bound Constraint**: For ingredients where dish B requires more than dish A, the minimum x is constrained by `(S * B_i - Q_i + (B_i - A_i) - 1) // (B_i - A_i)`.
3. **Binary Search Execution**: The binary search adjusts `low` and `high` based on whether the current mid value is feasible. If feasible, it searches for a higher value; otherwise, it searches for a lower value.
4. **Output**: The highest feasible value found during the binary search is printed as the result, representing the maximum number of servings possible.

This approach efficiently narrows down the search space using binary search and ensures the solution meets all ingredient constraints through careful feasibility checks.
