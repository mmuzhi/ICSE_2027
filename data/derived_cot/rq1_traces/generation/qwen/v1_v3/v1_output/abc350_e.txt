The deterministic operation divides \( N \) by \( A \) (floor division) at a cost \( X \), while the stochastic operation divides \( N \) by a random integer between 1 and 6 (floor division) at a cost \( Y \). The goal is to find the optimal sequence of operations that minimizes the expected cost.

### Approach
1. **Problem Analysis**: The problem involves making a series of decisions (either deterministic or stochastic operations) to reduce \( N \) to 0. Each operation has an associated cost, and the stochastic operation introduces uncertainty (expected value calculation). The challenge is to compute the minimum expected cost efficiently, considering the large possible value of \( N \) (up to \( 10^{18} \)).

2. **Intuition**: The solution involves dynamic programming (DP) with memoization. The key insight is that each state (current value of \( N \)) can be reduced by either the deterministic or stochastic operation, and the cost for each state can be derived from its subproblems (states resulting from these operations). The stochastic operation's expected cost is calculated by considering all possible outcomes (1 through 6) and their probabilities.

3. **Algorithm Selection**: We use a recursive DP approach with memoization to avoid recalculating the same states. The function `E(n)` computes the minimum expected cost from state \( n \) (current value of \( N \)) by considering both operations:
   - **Deterministic Operation**: Cost \( X \) plus the cost to reach \( \left\lfloor \frac{n}{A} \right\rfloor \).
   - **Stochastic Operation**: Cost \( Y \) plus the average cost of reaching \( \left\lfloor \frac{n}{b} \right\rfloor \) for \( b \) from 2 to 6 (since dividing by 1 doesn't change the state and is handled separately).

4. **Complexity Analysis**: The state space is defined by the distinct integers obtained by repeatedly applying the operations. The depth of recursion is logarithmic in \( N \) (base \( A \)), and each state branches into up to 6 subproblems (for the stochastic operation). While this could lead to an exponential number of states in the worst case, the logarithmic depth ensures the solution is feasible for large \( N \) (up to \( 10^{18} \)).

import sys
sys.setrecursionlimit(10000)

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    N = int(data[0])
    A = int(data[1])
    X = int(data[2])
    Y = int(data[3])
    memo = {}
    
    def E(n):
        if n == 0:
            return 0.0
        if n in memo:
            return memo[n]
        next_d = n // A
        option1 = X + E(next_d)
        
        s = 0.0
        for b in [2, 3, 4, 5, 6]:
            next_b = n // b
            s += E(next_b)
            
        option2 = (6/5) * Y + s / 5.0
        
        res = min(option1, option2)
        memo[n] = res
        return res
        
    ans = E(N)
