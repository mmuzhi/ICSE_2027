There are H \times W cookies in H rows and W columns.
The color of the cookie at the i-row from the top and j-th column from the left is represented by a lowercase English letter c_{i,j}.  
We will perform the following procedure.
1. For each row, perform the following operation: if there are two or more cookies remaining in the row and they all have the same color, mark them.  
2. For each column, perform the following operation: if there are two or more cookies remaining in the column and they all have the same color, mark them.  
3. If there are any marked cookies, remove them all and return to 1; otherwise, terminate the procedure.
Find the number of cookies remaining at the end of the procedure.

Input

The input is given from Standard Input in the following format:
H W
c_{1,1}c_{1,2} \ldots c_{1,W}
c_{2,1}c_{2,2} \ldots c_{2,W}
\vdots
c_{H,1}c_{H,2} \ldots c_{H,W}

Output

Print the answer.

Constraints


- 2 \leq H, W \leq 2000
- c_{i,j} is a lowercase English letter.

Sample Input 1

4 3
aaa
aaa
abc
abd

Sample Output 1

2

=== ANSWER ===

To solve this problem, we need to simulate the process of removing cookies from a grid based on specific conditions until no more cookies can be removed. The key challenge is efficiently handling the removals and updating the grid state in each iteration without resorting to a brute-force approach that would be too slow for large grids.

### Approach
1. **Initialization**: Read the grid dimensions and the grid itself. Initialize arrays to keep track of the number of remaining cookies in each row and column, and dictionaries to maintain the frequency of each color in the remaining cookies for each row and column.
2. **Initial Setup**: Populate the frequency dictionaries for rows and columns based on the initial grid state. Identify all initially removable rows and columns (those with at least two remaining cookies of the same color) and enqueue them.
3. **Processing Rounds**: While there are rows or columns to process:
   - **Marking Removable Rows/Columns**: Collect all rows and columns that are removable at the start of the current round.
   - **Building Removal Set**: Construct a set of all cells that need to be removed (union of cells in removable rows and columns).
   - **Updating State**: For each cell in the removal set, update the row and column counts, adjust the color frequencies, and mark the cell as removed.
   - **Enqueue New Removables**: After processing the current round, check all rows and columns to see if they become removable in the next round based on the updated state.
4. **Result Calculation**: The total number of remaining cookies is the sum of remaining cookies across all rows.

import sys
from collections import deque

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        print(0)

    H, W = map(int, data[0].split())

    col_remaining = [H] * W

            row_freq[i][c] = row_freq[i].get(c, 0) + 1
            col_freq[j][c] = col_freq[j].get(c, 0) + 1
            
    row_queue = deque()
    col_queue = deque()
    
    for i in range(H):
        if row_remaining[i] >= 2 and len(row_freq[i]) == 1:
            row_queue.append(i)
            
    for j in range(W):
        if col_remaining[j] >= 2 and len(col_freq[j]) == 1:
            col_queue.append(j)

        rows_to_remove_set = set(row_queue)
        cols_to_remove_set = set(col_queue)
        removed_set = set()
        for i in rows_to_remove_set:
            for j in range(W):
                removed_set.add((i, j))

                removed_set.add((i, j))

        col_queue = deque()
        
        for (i, j) in removed_set:
            if grid[i][j] == '.':
                continue
            c = grid[i][j]
            row_remaining[i] -= 1
            col_remaining[j] -= 1
            if c in row_freq[i]:
                row_freq[i][c] -= 1
                if row_freq[i][c] == 0:
                    del row_freq[i][c]
            if c in col_freq[j]:
                col_freq[j][c] -= 1
                if col_freq[j][c] == 0:
                    del col_freq[j][c]
            grid[i][j] = '.'

### Explanation
1. **Initialization**: The grid is read from the input, and arrays `row_remaining` and `col_remaining` are initialized to keep track of the number of cookies left in each row and column. Dictionaries `row_freq` and `col_freq` are used to count occurrences of each color in the remaining cookies for each row and column.
2. **Initial Setup**: The initial state of the grid is processed to populate the frequency dictionaries. Rows and columns with all remaining cookies of the same color and at least two cookies are identified and enqueued for processing.
3. **Processing Rounds**: In each round, the algorithm processes all rows and columns that are removable (based on the current state). The set of cells to be removed is constructed as the union of cells in these rows and columns. For each cell removed, the row and column counts are decremented, and the color frequencies are updated. Cells marked as removed are set to '.'.
4. **Enqueue New Removables**: After processing each round, the algorithm checks all rows and columns again to see if any new rows or columns meet the removal criteria for the next round.
5. **Result Calculation**: The total number of remaining cookies is computed by summing the remaining cookies across all rows and printed as the result.

This approach efficiently handles the removal process by leveraging queues to manage removable rows and columns, updating the grid state dynamically, and ensuring optimal performance even for large grids.
