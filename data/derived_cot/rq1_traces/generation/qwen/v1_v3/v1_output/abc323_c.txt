- At least how many of the problems that player i has not yet solved must player i solve to exceed all other players' current total scores?

- A_i is a multiple of 100.
- S_i is a string of length M consisting of o and x.
- S_i contains at least one x.

=== ANSWER ===

To solve this problem, we need to determine the minimum number of unsolved problems each player must solve to exceed the current total scores of all other players. The solution involves calculating each player's current score, identifying the highest score among other players, and then determining the smallest subset of unsolved problems whose combined scores, when added to the player's current score, surpass this highest score.

### Approach
1. **Problem Analysis**: 
   - Each player has a current score derived from the sum of the scores of solved problems plus a bonus equal to their player index.
   - The goal is to find the minimum number of unsolved problems a player must solve (selecting the highest-scoring problems first) to ensure their total score exceeds the highest current score of all other players.

2. **Intuition**:
   - For each player, compute their current score by summing the scores of solved problems and adding their bonus.
   - Determine the highest current score among all other players.
   - If the player's current score already exceeds this highest score, the answer is 0.
   - Otherwise, sort the unsolved problems by their scores in descending order and accumulate the highest scores until the player's total score surpasses the highest score of other players.

3. **Algorithm Selection**:
   - **Current Score Calculation**: For each player, sum the scores of solved problems and add the player's index (bonus).
   - **Max Score Calculation**: For each player, find the maximum current score among all other players.
   - **Greedy Selection**: For unsolved problems, select the highest-scoring problems first to minimize the number needed to exceed the max score.

4. **Complexity Analysis**:
   - **Time Complexity**: O(N*M) for processing each player's score and O(N*M log M) for sorting unsolved problems (though M is small, up to 100).
   - **Space Complexity**: O(N*M) for storing player strings and scores, which is manageable given the constraints.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    first_line = data[0].split()
    n = int(first_line[0])
    m = int(first_line[1])
    
    scores = list(map(int, data[1].split()))
    
    players = []
    for i in range(2, 2+n):
        players.append(data[i].strip())
    
    base_scores = []
    for i in range(n):
        s = players[i]
        total = 0
        for j in range(m):
            if s[j] == 'o':
                total += scores[j]
        total += (i+1)
        base_scores.append(total)
    
    max_other = [0] * n
    for i in range(n):
        other_scores = [base_scores[j] for j in range(n) if j != i]
        max_other[i] = max(other_scores) if other_scores else 0
    
    results = []
    for i in range(n):
        base_score = base_scores[i]
        max_o = max_other[i]
        if base_score > max_o:
            results.append("0")
            continue
        
        unsolved = []
        s = players[i]
        for j in range(m):
            if s[j] == 'x':
                unsolved.append(scores[j])
        
        unsolved.sort(reverse=True)
        current_sum = 0
        k = 0
        for score in unsolved:
            k += 1
            current_sum += score
            if base_score + current_sum > max_o:
                results.append(str(k))
                break
        
    for res in results:
        print(res)

if __name__ == '__main__':
    main()
```

3. **Max Score Calculation**: For each player, the highest score among all other players is determined.
4. **Greedy Selection**: For each player, if their base score is already higher than the max score of others, the answer is 0. Otherwise, the unsolved problems are sorted by score in descending order, and the highest scores are added until the player's total score exceeds the max score of others.
5. **Output**: The results for all players are printed, indicating the minimum number of problems each must solve to surpass the competition. 

This approach efficiently addresses the problem by leveraging greedy selection and straightforward arithmetic to determine the solution for each player.
