To solve this problem, we need to determine the minimum number of operations required to remove all black cells from a strip of paper. Each operation allows selecting any `k` consecutive cells and turning them all white. The goal is to cover all black cells with the fewest operations possible.

### Approach
1. **Problem Analysis**: The problem involves covering all black cells in a string with the minimum number of operations, where each operation covers `k` consecutive cells. The key observation is that each operation can cover multiple black cells if they lie within a window of `k` consecutive cells. The challenge is to efficiently determine the optimal starting positions for these operations to maximize the number of black cells covered in each step.

2. **Intuition**: The solution involves processing the black cells in order from left to right. For each uncovered black cell, we determine the latest possible starting position for an operation that includes this cell and extends as far right as possible (to cover more black cells). The operation's starting position is constrained by the current black cell's position and the strip's length to ensure it stays within bounds.

3. **Algorithm Selection**: The algorithm uses a greedy approach where, for each uncovered black cell, it selects the optimal operation start position (the rightmost valid position that includes the current black cell). It then skips all black cells covered by this operation and repeats the process until all black cells are covered.

4. **Complexity Analysis**: The algorithm processes each test case by first identifying all black cell positions (O(n) per test case). For each uncovered black cell, it uses binary search (O(log m)) to find the next uncovered black cell, where `m` is the number of black cells. The total complexity across all test cases is O(t * n log n), which is efficient given the constraints (sum of `n` across test cases is ≤ 200,000).

### Solution Code
```python
import bisect

t = int(input().strip())
results = []

for _ in range(t):
    data = input().split()
    n = int(data[0])
    k = int(data[1])
    s = input().strip()
    
    blacks = [i for i, char in enumerate(s) if char == 'B']
    
    if not blacks:
        results.append(0)
        continue
        
    operations = 0
    i = 0
    while i < len(blacks):
        p = blacks[i]
        x_max = min(p, n - k)
        r = x_max + k - 1
        
        j = bisect.bisect_right(blacks, r)
        operations += 1
        i = j
        
    results.append(operations)

for res in results:
    print(res)
```

### Explanation
1. **Reading Input**: The code starts by reading the number of test cases `t`. For each test case, it reads the strip length `n`, the operation size `k`, and the string `s` representing the strip.
2. **Black Cell Identification**: It collects all positions of black cells ('B') in the string into a list `blacks`.
3. **No Black Cells Handling**: If there are no black cells, it appends `0` to the results since no operations are needed.
4. **Greedy Operation Selection**: For each uncovered black cell (starting from the left), it calculates the latest valid starting position `x_max` for an operation that includes the current black cell. The operation covers from `x_max` to `x_max + k - 1`.
5. **Binary Search for Next Uncovered Cell**: Using binary search (`bisect_right`), it finds the first black cell beyond the current operation's coverage. This reduces the problem size by skipping covered cells.
6. **Output Results**: After processing all test cases, it prints the results, each representing the minimum operations needed for the corresponding test case.

This approach efficiently covers all black cells with minimal operations by leveraging greedy selection and binary search for optimal performance.
