To solve this problem, we need to maximize the total reward collected by selecting rewards from an array such that each subsequent reward is greater than the current total reward. The solution involves sorting the rewards and using dynamic programming to keep track of the maximum achievable total at each step.

### Approach
1. **Sort and Deduplicate Rewards**: First, we sort the reward values and remove duplicates since duplicates do not contribute to a higher total reward under the given conditions.
2. **Dynamic Programming Setup**: We use a dynamic programming array `best_total` where `best_total[i]` represents the maximum total reward achievable using the first `i+1` rewards (after sorting).
3. **Binary Search for Valid Previous States**: For each reward, we find the largest previous reward's total that is less than the current reward. This is done using binary search to efficiently locate the appropriate previous state.
4. **Update DP Array**: For each reward, if there exists a valid previous state (i.e., a previous total less than the current reward), we update the current reward's total by adding the largest valid previous total. The `best_total` array is updated to reflect the maximum total achievable up to the current reward.

### Solution Code
```python
class Solution:
    def maxTotalReward(self, rewardValues: List[int]) -> int:
        rewards = sorted(set(rewardValues))
        n = len(rewards)
        if n == 0:
            return 0
        best_total = [0] * n
        best_total[0] = rewards[0]
        
        for i in range(1, n):
            left, right = 0, i - 1
            target = -1
            while left <= right:
                mid = (left + right) // 2
                if best_total[mid] < rewards[i]:
                    target = mid
                    left = mid + 1
                else:
                    right = mid - 1
            if target != -1:
                best_total[i] = max(best_total[i-1], rewards[i] + best_total[target])
            else:
                best_total[i] = best_total[i-1]
        
        return best_total[-1]
```

### Explanation
1. **Sorting and Deduplication**: The rewards are sorted and duplicates are removed to simplify the selection process and ensure each reward is considered only once.
2. **Dynamic Programming Initialization**: The `best_total` array is initialized to store the maximum reward achievable up to each index. The first element is simply the first reward in the sorted list.
3. **Binary Search for Valid States**: For each subsequent reward, we use binary search to find the largest previous reward's total that is less than the current reward. This helps in efficiently determining the best previous state to extend the current total.
4. **Updating the DP Array**: The current reward's total is updated by adding the largest valid previous total to the current reward. If no valid previous state exists, the current total remains the same as the previous maximum. The final result is the last element in the `best_total` array, representing the maximum total reward achievable.

This approach efficiently narrows down the problem to manageable subproblems using dynamic programming and binary search, ensuring optimal performance even for the upper constraint limits.
