- The digits of x are strictly decreasing from top to bottom.

- All input values are integers.

To solve this problem, we need to find the K-th smallest 321-like number, which is defined as a positive integer with strictly decreasing digits from left to right. All one-digit positive integers are considered 321-like numbers. The solution involves generating these numbers in increasing order and selecting the K-th smallest one.

### Approach
1. **Problem Analysis**: The 321-like numbers are numbers where each digit is strictly decreasing. This includes all one-digit numbers (1-9), two-digit numbers (like 10, 20, 21), and so on. The key observation is that these numbers can be generated by selecting subsets of digits from 0-9, arranging them in decreasing order, and ensuring the first digit is non-zero (to form a positive integer).

2. **Intuition**: The numbers can be categorized by their digit length. For each digit length `d`, the count of such numbers is given by combinations of digits from 0-9 taken `d` at a time. For `d=1`, we exclude the digit 0, so the count is 9. For `d>=2`, the count is the combination of 10 digits taken `d` at a time (since including 0 is allowed as long as the first digit is non-zero).

3. **Algorithm Selection**: 
   - **Precompute Counts**: Calculate the number of 321-like numbers for each digit length from 1 to 10.
   - **Determine Digit Length**: Use cumulative counts to find the digit length `d` where the K-th number resides.
   - **Generate Numbers**: For the identified digit length `d`, generate all possible numbers by selecting combinations of digits, sorting them in decreasing order, and converting them to integers. Sort these numbers to find the K-th smallest one.

4. **Complexity Analysis**: The algorithm generates combinations for a specific digit length, which is manageable since the maximum digit length is 10, and the number of combinations for `d=5` is 252 (the highest). The sorting step for these combinations is efficient given the constraints.

### Solution Code
```python
import itertools

def main():
    K = int(input().strip())
    
    counts = [0, 9, 45, 120, 210, 252, 210, 120, 45, 10, 1]
    
    cum_arr = [0] * 11
    for i in range(1, 11):
        cum_arr[i] = cum_arr[i-1] + counts[i]
    
    d_found = 1
    for i in range(1, 11):
        if cum_arr[i] >= K:
            d_found = i
            break
            
    pos_in_d = K - cum_arr[d_found-1]
    
    if d_found == 1:
        numbers = [i for i in range(1, 10)]
    else:
        digits = list(range(0, 10))
        numbers = []
        for comb in itertools.combinations(digits, d_found):
            sorted_comb = sorted(comb, reverse=True)
            num_str = ''.join(str(x) for x in sorted_comb)
            num_int = int(num_str)
            numbers.append(num_int)
        numbers.sort()
    
    result = numbers[pos_in_d-1]
    print(result)

if __name__ == "__main__":
    main()
```

1. **Input Handling**: The input value `K` is read, representing the position of the desired 321-like number in the ordered list.
2. **Precompute Counts**: The `counts` array holds the number of 321-like numbers for each digit length from 1 to 10. For `d=1`, it's 9 (digits 1-9). For `d>=2`, it's the combination of 10 digits taken `d` at a time.
3. **Cumulative Counts**: The `cum_arr` array stores cumulative counts up to each digit length to determine where the K-th number falls.
4. **Digit Length Determination**: The loop finds the smallest digit length `d_found` such that the cumulative count up to `d_found` is at least `K`.
5. **Number Generation**: For `d_found=1`, numbers are 1-9. For other lengths, all combinations of digits are generated, sorted in decreasing order, converted to integers, and then sorted to find the K-th smallest number.
