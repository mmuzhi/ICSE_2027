To solve this problem, we need to determine the number of ways to select a group of students such that all students remain happy based on the given conditions. The conditions specify that a selected student must have the total number of selected students strictly greater than their corresponding number in the array, while a non-selected student must have the total number of selected students strictly less than their corresponding number.

### Approach
1. **Problem Analysis**: The key observation is that for a given number of selected students \( k \), the selection must include all students with values less than \( k \) and exclude all students with values greater than or equal to \( k \). Additionally, no student should have a value exactly equal to \( k \), as this would violate the happiness conditions for that student.

2. **Intuition**: For each possible \( k \) (from 0 to \( n \), where \( n \) is the number of students), we check two conditions:
   - **Condition 1**: No student has a value equal to \( k \).
   - **Condition 2**: The number of students with values less than \( k \) is exactly \( k \).

3. **Algorithm Selection**: 
   - **Sorting**: First, we sort the array to efficiently count the number of elements less than \( k \) using a two-pointer technique.
   - **Set for Lookup**: We use a set to quickly check if \( k \) is present in the array for Condition 1.
   - **Two-pointer Technique**: After sorting the array, we maintain a pointer that moves forward as we iterate through possible \( k \) values, counting how many elements are less than \( k \). This count must equal \( k \) for Condition 2 to hold.

4. **Complexity Analysis**: 
   - **Time Complexity**: Sorting the array takes \( O(n \log n) \). The two-pointer technique and set operations (for Condition 1) take \( O(n) \) time. Thus, the overall complexity is \( O(n \log n) \).
   - **Space Complexity**: We use \( O(n) \) space to store the sorted array and the set of elements.

### Solution Code
```python
class Solution:
    def countWays(self, nums: List[int]) -> int:
        n = len(nums)
        nums_sorted = sorted(nums)
        set_nums = set(nums)
        ways = 0
        i = 0
        for k in range(0, n + 1):
            if k in set_nums:
                continue
            while i < n and nums_sorted[i] < k:
                i += 1
            if i == k:
                ways += 1
        return ways
```

### Explanation
1. **Sorting and Set Initialization**: The array is sorted to facilitate the two-pointer technique. A set is created from the array for quick lookup to check if \( k \) is present in the array.
2. **Iterating Over Possible \( k \) Values**: For each \( k \) from 0 to \( n \):
   - **Condition 1 Check**: If \( k \) is found in the set, it skips further checks for this \( k \).
   - **Condition 2 Check**: The two-pointer technique counts how many elements in the sorted array are less than \( k \). If this count equals \( k \), it means selecting exactly those students satisfies the happiness conditions, and the count of valid ways is incremented.
3. **Result**: The total number of valid \( k \) values (ways to select students) is returned as the result.

This approach efficiently checks all possible group sizes \( k \) by leveraging sorting and set operations, ensuring optimal performance even for large input sizes.
