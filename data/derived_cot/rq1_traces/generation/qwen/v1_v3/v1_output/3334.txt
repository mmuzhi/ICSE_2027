To solve this problem, we need to determine the minimum number of boxes required to redistribute all the apples from given packs into the boxes, where each box has a specific capacity. The key observation here is that apples from the same pack can be split across multiple boxes, and the total capacity of the selected boxes must be at least the total number of apples.

### Approach
1. **Calculate Total Apples**: First, compute the total number of apples by summing up all the elements in the `apple` array.
2. **Sort Boxes by Capacity**: Sort the `capacity` array in descending order to prioritize using boxes with the largest capacities first. This greedy approach ensures that we use the fewest boxes possible by maximizing the capacity added with each selection.
3. **Select Boxes**: Iterate through the sorted list of box capacities, accumulating their capacities until the accumulated capacity meets or exceeds the total number of apples. The count of boxes used at this point is our answer.

### Solution Code
```python
class Solution:
    def minimumBoxes(self, apple: List[int], capacity: List[int]) -> int:
        total = sum(apple)
        capacity.sort(reverse=True)
        current = 0
        count = 0
        for cap in capacity:
            current += cap
            count += 1
            if current >= total:
                return count
        return count
```

### Explanation
1. **Total Apples Calculation**: The sum of all elements in the `apple` array gives the total number of apples that need to be redistributed.
2. **Sorting Boxes**: By sorting the `capacity` array in descending order, we ensure that we always consider the largest available boxes first. This is crucial for minimizing the number of boxes used.
3. **Greedy Selection**: As we iterate through each box in the sorted list, we add its capacity to a running total. If at any point this running total meets or exceeds the total number of apples, we return the current count of boxes used. This approach efficiently leverages the largest available capacities first, ensuring the solution is both optimal and straightforward.

This method efficiently solves the problem by leveraging a greedy algorithm, which is optimal here due to the nature of the problem constraints and requirements. The algorithm ensures minimal box usage by always selecting the most capacious boxes first, thus minimizing the total number of boxes needed.
