class Solution:
    def maximumSetSize(self, nums1: List[int], nums2: List[int]) -> int:
        n = len(nums1)  # since nums1 and nums2 have same length
        set1 = set(nums1)
        set2 = set(nums2)
        A_only = set1 - set2
        B_only = set2 - set1
        common = set1 & set2
        k1 = k2 = n // 2
        
        # The maximum distinct numbers we can keep from A_only and B_only
        T = min(len(A_only), k1) + min(len(B_only), k2)
        remaining = n - T  # total remaining slots in both arrays
        common_kept = min(len(common), remaining)
        max_distinct = T + common_kept
        
        # Total distinct numbers in both arrays
        total_distinct = len(A_only) + len(B_only) + len(common)
        return min(max_distinct, total_distinct)

To solve this problem, we need to maximize the size of a set formed by the remaining elements of two arrays after removing exactly half of the elements from each array. The key insight is to strategically remove elements such that the remaining elements contribute the maximum number of distinct values to the set.

### Approach
1. **Problem Analysis**: The problem requires removing `n/2` elements from each of the two arrays and then combining the remaining elements into a set. The goal is to maximize the size of this set. The challenge lies in efficiently selecting which elements to remove to retain as many distinct values as possible.

2. **Intuition**: The solution involves categorizing elements into three groups:
   - **A_only**: Elements present only in the first array.
   - **B_only**: Elements present only in the second array.
   - **Common**: Elements present in both arrays.

3. **Algorithm Selection**:
   - **Step 1**: Calculate the sets `A_only`, `B_only`, and `common` by finding the symmetric difference and intersection of the two arrays.
   - **Step 2**: Determine the maximum number of distinct elements (`T`) that can be retained from `A_only` and `B_only` without exceeding the allowed removals (`n/2` from each array).
   - **Step 3**: Use the remaining slots (after retaining `T` elements) to include as many distinct elements from the `common` set as possible.
   - **Step 4**: The maximum distinct elements is the sum of `T` and the number of distinct common elements retained. The final answer is the minimum between this sum and the total distinct elements available in both arrays.

4. **Complexity Analysis**: The algorithm involves set operations which are efficient with a time complexity of O(n), where n is the length of the arrays. This is optimal given the problem constraints.

### Solution Code
```python
class Solution:
    def maximumSetSize(self, nums1: List[int], nums2: List[int]) -> int:
        n = len(nums1)
        set1 = set(nums1)
        set2 = set(nums2)
        A_only = set1 - set2
        B_only = set2 - set1
        common = set1 & set2
        k1 = k2 = n // 2
        
        T = min(len(A_only), k1) + min(len(B_only), k2)
        remaining = n - T
        common_kept = min(len(common), remaining)
        max_distinct = T + common_kept
        
        total_distinct = len(A_only) + len(B_only) + len(common)
        return min(max_distinct, total_distinct)
```

### Explanation
1. **Set Operations**: The code first converts both arrays into sets to efficiently compute the elements unique to each array (`A_only` and `B_only`) and the common elements (`common`).
2. **Removal Constraints**: The allowed number of elements to retain from each array is `n/2` (denoted as `k1` and `k2`).
3. **Maximizing Distinct Elements**: The variable `T` calculates the maximum distinct elements that can be retained from `A_only` and `B_only` without exceeding the removal constraints. The remaining slots (`remaining`) are used to include as many distinct elements from the `common` set as possible (`common_kept`).
4. **Result Calculation**: The result is the minimum between the total distinct elements (`total_distinct`) and the sum of `T` and `common_kept` (`max_distinct`), ensuring we do not exceed the total available distinct elements.

This approach efficiently maximizes the set size by leveraging set operations and careful consideration of removal constraints.
