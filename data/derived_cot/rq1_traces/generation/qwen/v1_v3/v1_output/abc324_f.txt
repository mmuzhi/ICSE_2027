To solve this problem, we need to find the maximum ratio of the total beauty to the total cost for any path from vertex 1 to vertex N in a directed acyclic graph (DAG) where edges are directed from lower-numbered to higher-numbered vertices. The solution involves using binary search combined with a dynamic programming approach to efficiently determine the maximum ratio.

### Approach
1. **Binary Search on the Ratio**: The core idea is to perform a binary search on the possible values of the ratio (beauty/cost). For each candidate ratio during the binary search, we check if there exists a path from vertex 1 to vertex N such that the total beauty is at least the candidate ratio times the total cost.
2. **Dynamic Programming in DAG**: Since the graph is a DAG (edges go from lower to higher vertices), we can process vertices in increasing order. For each vertex, we compute the maximum value of (total beauty - ratio * total cost) from vertex 1 to that vertex. If this value for vertex N is non-negative, the candidate ratio is feasible.
3. **Efficiency**: The binary search runs in logarithmic time (O(log(max_ratio))), and each iteration of the binary search involves a dynamic programming pass over the graph (O(N + M)), making the overall complexity O((N + M) * log(max_ratio)), which is efficient for the given constraints.

```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it)); m = int(next(it))
    graph = [[] for _ in range(n+1)]
    
    for i in range(m):
        u = int(next(it)); v = int(next(it)); b = int(next(it)); c = int(next(it))
        graph[u].append((v, b, c))
    
    low = 0.0
    high = 200000 * 10000
    
    def check(X):
        dp = [-10**18] * (n+1)
        dp[1] = 0.0
        for i in range(1, n+1):
            if dp[i] == -10**18:
                continue
            for edge in graph[i]:
                j, b_val, c_val = edge
                new_val = dp[i] + b_val - X * c_val
                if new_val > dp[j]:
                    dp[j] = new_val
        return dp[n] >= 0

    for _ in range(100):
        mid = (low + high) / 2.0
        if check(mid):
            low = mid
        else:
            high = mid

    print("{:.15f}".format(low))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `low` as 0.0 and `high` as a sufficiently large value (2e9) to cover the maximum possible ratio.
2. **Feasibility Check (check function)**: For a given ratio `X`, the function computes the maximum value of (total beauty - X * total cost) for each vertex. It starts from vertex 1 with an initial value of 0.0 and propagates this value to higher vertices using the graph's edges. If the value for vertex N is non-negative, the ratio `X` is feasible.
3. **Binary Search Execution**: The binary search runs for a fixed number of iterations (100) to ensure precision. In each iteration, it checks the feasibility of the midpoint ratio and adjusts the search range accordingly.
4. **Output**: After completing the binary search, the highest feasible ratio is printed with a precision of 15 decimal places.

This approach efficiently narrows down the maximum ratio by leveraging binary search and dynamic programming, ensuring optimal performance even for large input sizes.
