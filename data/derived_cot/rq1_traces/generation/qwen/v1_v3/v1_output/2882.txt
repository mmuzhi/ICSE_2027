=== ANSWER ===

To solve this problem, we need to count the number of ways to express a given integer `n` as the sum of unique positive integers each raised to the `x`-th power. The solution involves dynamic programming to efficiently compute the number of valid combinations without revisiting the same sets of integers.

### Approach
1. **Problem Analysis**: The problem requires finding all unique sets of integers such that the sum of their `x`-th powers equals `n`. Each integer in the set must be unique, and the solution must efficiently count these sets even for large values of `n` (up to 300) and small values of `x` (up to 5).

2. **Dynamic Programming Setup**: We use a dynamic programming array `dp` where `dp[j]` represents the number of ways to achieve the sum `j` using the computed integers. The array is initialized with `dp[0] = 1` (the base case representing the empty set).

3. **Coin Change Analogy**: Treat each integer raised to the `x`-th power as a unique "coin". The goal is to find the number of ways to form the sum `n` using these coins, ensuring each coin is used at most once. This is analogous to the classic coin change problem but with the constraint that each coin can be used only once.

4. **Backward Iteration**: To avoid counting the same set multiple times, we iterate the DP array from `n` down to the current coin's value. For each value `j`, we update `dp[j]` by adding the number of ways to form `j - coin` (i.e., `dp[j - coin]`).

5. **Modulo Operation**: Since the result can be very large, all operations are performed modulo \(10^9 + 7\).

### Solution Code
```python
class Solution:
    def numberOfWays(self, n: int, x: int) -> int:
        mod = 10**9 + 7
        dp = [0] * (n + 1)
        dp[0] = 1
        i = 1
        while i**x <= n:
            coin = i**x
            for j in range(n, coin - 1, -1):
                dp[j] = (dp[j] + dp[j - coin]) % mod
            i += 1
        return dp[n] % mod
```

### Explanation
1. **Initialization**: The `dp` array is initialized to store the number of ways to achieve each sum from `0` to `n`. The base case `dp[0] = 1` signifies one way to achieve a sum of zero (using no integers).

2. **Coin Processing**: For each integer `i` starting from `1`, compute the coin value as `i^x`. If this value exceeds `n`, break the loop as further integers will only produce larger values.

3. **DP Array Update**: For each valid coin, update the `dp` array from `n` down to the coin's value. For each `j`, the value `dp[j]` is incremented by `dp[j - coin]`, representing the new ways to form `j` by including the current coin.

4. **Result Extraction**: The value `dp[n]` gives the number of ways to form the sum `n` using unique integers raised to the `x`-th power. The result is returned modulo \(10^9 + 7\) to handle large numbers.

This approach efficiently counts all unique sets of integers by leveraging dynamic programming and careful iteration, ensuring optimal performance and correctness.
