To solve this problem, we need to find the minimum number of adjacent swaps required to group all black balls (represented by '1') to the right and all white balls (represented by '0') to the left. The key observation here is that each swap of adjacent elements (specifically, a '1' followed by a '0') reduces the number of inversions by one. The total number of such inversions in the string gives the minimal number of swaps needed.

### Approach
1. **Problem Analysis**: The problem requires transforming the binary string into a configuration where all '0's are on the left and all '1's are on the right. The allowed operation is swapping adjacent elements. The minimal number of swaps needed is equivalent to the number of inversions in the string, where an inversion is a pair (i, j) such that i < j, s[i] = '1', and s[j] = '0'.

2. **Intuition**: For each '1' in the string, the number of '0's that appear after it contributes to the inversion count. Summing these counts for all '1's gives the total minimal swaps required. 

3. **Algorithm Selection**: 
   - Count the total number of '0's in the string.
   - Traverse the string from left to right, keeping track of the number of '0's encountered so far.
   - For each '1' encountered, the number of '0's remaining to its right (total_zeros - zeros_so_far) is added to the steps count.

4. **Complexity Analysis**: The algorithm involves a single pass through the string to count '0's and another pass to compute the steps, resulting in an O(n) time complexity, where n is the length of the string. The space complexity is O(1) as we only use a few extra variables.

### Solution Code
```python
class Solution:
    def minimumSteps(self, s: str) -> int:
        total_zeros = s.count('0')
        zeros_so_far = 0
        steps = 0
        for char in s:
            if char == '0':
                zeros_so_far += 1
            else:
                steps += total_zeros - zeros_so_far
        return steps
```

### Explanation
1. **Initialization**: We start by counting the total number of '0's in the string (`total_zeros`). This helps in determining how many '0's need to be moved past each '1'.
2. **Traversal**: As we traverse the string from left to right:
   - For each '0' encountered, we increment `zeros_so_far`. This keeps track of how many '0's have been processed so far.
   - For each '1' encountered, the number of '0's that still need to be moved past this '1' is `total_zeros - zeros_so_far`. This value is added to `steps`.
3. **Result**: The variable `steps` accumulates the total number of swaps needed, which is returned as the result.

This approach efficiently calculates the minimal swaps by leveraging the inversion count, ensuring optimal performance even for large input sizes.
