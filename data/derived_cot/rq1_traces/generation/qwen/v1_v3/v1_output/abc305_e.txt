You will be given a competitive programming problem.
There is a simple undirected graph with N vertices and M edges, where vertices are numbered from 1 to N, and edges are numbered from 1 to M. Edge i connects vertex a_i and vertex b_i.
K security guards numbered from 1 to K are on some vertices. Guard i is on vertex p_i and has a stamina of h_i. All p_i are distinct.
A vertex v is said to be guarded when the following condition is satisfied:

- there is at least one guard i such that the distance between vertex v and vertex p_i is at most h_i.

Here, the distance between vertex u and vertex v is the minimum number of edges in the path connecting vertices u and v.
List all guarded vertices in ascending order.

Input

The input is given from Standard Input in the following format:
N M K
a_1 b_1
a_2 b_2
\vdots
a_M b_M
p_1 h_1
p_2 h_2
\vdots
p_K h_K

Output

Print the answer in the following format. Here,

- G is the number of guarded vertices,
- and v_1, v_2, \dots, v_G are the vertex numbers of the guarded vertices in ascending order.

G
v_1 v_2 \dots v_G

Constraints


- 1 \leq N \leq 2 \times 10^5
- 0 \leq M \leq \min \left(\frac{N(N-1)}{2}, 2 \times 10^5 \right)
- 1 \leq K \leq N
- 1 \leq a_i, b_i \leq N
- The given graph is simple.
- 1 \leq p_i \leq N
- All p_i are distinct.
- 1 \leq h_i \leq N
- All input values are integers.

To solve this problem, we need to determine all vertices in a graph that are guarded by at least one security guard. A vertex is considered guarded if the shortest path distance from the vertex to any guard's position is less than or equal to the guard's stamina. 

### Approach
1. **Graph Representation**: Represent the graph using an adjacency list for efficient traversal.
2. **Multi-source Dijkstra-like BFS**: Use a priority queue (max-heap) to propagate the remaining stamina from each guard. Each guard starts with their initial stamina, and as we move from a vertex to its neighbors, we decrement the remaining stamina. If the remaining stamina at a neighbor is higher than previously recorded, we update it and continue the propagation.
3. **Tracking Guarded Vertices**: After processing all guards and their possible reach, collect all vertices where the remaining stamina is non-negative, indicating they are guarded.

### Solution Code
```python
import heapq
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it)); m = int(next(it)); k = int(next(it))
    
    graph = [[] for _ in range(n+1)]
    for _ in range(m):
        u = int(next(it)); v = int(next(it))
        graph[u].append(v)
        graph[v].append(u)
    
    rem = [-10**18] * (n+1)
    heap = []
    
    for _ in range(k):
        p = int(next(it)); h = int(next(it))
        if h > rem[p]:
            rem[p] = h
            heapq.heappush(heap, (-h, p))
    
    while heap:
        neg_rem, u = heapq.heappop(heap)
        current_rem = -neg_rem
        if current_rem < rem[u]:
            continue
        for v in graph[u]:
            new_rem = current_rem - 1
            if new_rem > rem[v]:
                rem[v] = new_rem
                heapq.heappush(heap, (-new_rem, v))
    
    guarded_vertices = []
    for i in range(1, n+1):
        if rem[i] >= 0:
            guarded_vertices.append(i)
            
    guarded_vertices.sort()
    
    print(len(guarded_vertices))
    if guarded_vertices:
        print(" ".join(map(str, guarded_vertices)))
    else:
        print()

if __name__ == '__main__':
    main()
```

3. **Priority Queue Processing**: The algorithm processes each vertex from the priority queue, propagating the remaining stamina to neighboring vertices. If a neighbor's stamina can be improved (higher value), it is updated and added back to the queue.
4. **Result Collection**: After processing, vertices with non-negative remaining stamina are collected, sorted, and printed.
