To solve this problem, we need to find the remainder when the sum of elements from index L to R in a hidden sequence A is divided by 100. The sequence A consists of integers between 0 and 99, and we can query the sum of any contiguous block of elements that is a power of two in length. The challenge is to determine the minimal number of such queries required to compute the desired sum.

### Approach
1. **Problem Analysis**: The problem involves querying sums of specific blocks (aligned to power-of-two lengths) in a hidden array. The goal is to compute the sum of a given range [L, R] modulo 100 with the fewest queries possible.
2. **Intuition**: The solution leverages the observation that any range [L, R] can be broken down into non-overlapping blocks of power-of-two lengths. The sum of these blocks can then be combined to get the desired result. The key insight is to always query the largest possible block within the current range, then proceed recursively with the remaining elements.
3. **Algorithm Selection**: The algorithm uses a greedy approach to find the largest power-of-two block within the current range [low, high]. If the block starts at 'low' and fits within the range, it is queried. The process repeats with the remaining elements until the entire range is covered. If no such block exists, individual elements are queried.
4. **Complexity Analysis**: The number of queries depends on the binary representation of the range length (R - L + 1). In the worst case, each element is queried individually, leading to O(N) queries where N is the length of the range. However, for ranges aligned to power-of-two boundaries, fewer queries are needed, typically logarithmic in the range length.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.readline().split()
    if not data:
        return
    N = int(data[0])
    L = int(data[1])
    R = int(data[2])
    
    low = L
    high = R
    total = 0
    
    while low <= high:
        len_interval = high - low + 1
        block_size = 1
        while block_size <= len_interval:
            block_size <<= 1
        block_size >>= 1
        
        if block_size == 0:
            break
            
        start = low - (low % block_size)
        while start < low or start + block_size - 1 > high:
            if block_size == 0:
                break
            block_size //= 2
            if block_size == 0:
                break
            start = low - (low % block_size)
            
        if block_size == 0:
            for i in range(low, high + 1):
                print("0", i)
                sys.stdout.flush()
                T = int(sys.stdin.readline().strip())
                total = (total + T) % 100
            low = high + 1
        else:
            i_val = block_size.bit_length() - 1
            j_val = start // block_size
            print(f"{i_val} {j_val}")
            sys.stdout.flush()
            T = int(sys.stdin.readline().strip())
            total = (total + T) % 100
            low = start + block_size

    print("!" + str(total))
```
