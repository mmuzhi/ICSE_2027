To solve this problem, we need to find the length of the longest valid subsequence in an integer array. A subsequence is considered valid if all consecutive pairs in the subsequence have the same parity when their sums are taken modulo 2. The parity can be either 0 (for pairs of even-even or odd-odd numbers) or 1 (for pairs of even-odd or odd-even numbers).

### Approach
1. **Problem Analysis**: The problem requires checking two separate conditions for subsequences:
   - **Same Parity (Case A)**: All consecutive pairs in the subsequence must have a sum modulo 2 of 0. This means the entire subsequence must consist of either all even numbers or all odd numbers.
   - **Alternating Parity (Case B)**: All consecutive pairs in the subsequence must have a sum modulo 2 of 1. This means the subsequence must alternate between even and odd numbers.

2. **Intuition**: 
   - For **Case A**, the longest valid subsequence is simply the count of all even or all odd numbers in the array, whichever is larger.
   - For **Case B**, we use dynamic programming to track the longest alternating subsequence. We maintain two variables, `max_dp0` and `max_dp1`, which represent the lengths of the longest alternating subsequences ending with an even or odd number, respectively. For each number in the array, if it's even, we update `max_dp0` using `max_dp1` and vice versa for odd numbers.

3. **Algorithm Selection**: 
   - **Same Parity**: Count the number of even and odd numbers in the array. The maximum of these counts gives the solution for Case A.
   - **Alternating Parity**: Use dynamic programming to compute the longest alternating subsequence by iterating through the array and updating the DP values based on the parity of the current number.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n), where n is the length of the array. We traverse the array twice: once to count even and odd numbers, and once to compute the alternating subsequence.
   - **Space Complexity**: O(1), as we only use a few extra variables for counting and tracking DP values.

### Solution Code
```python
class Solution:
    def maximumLength(self, nums: List[int]) -> int:
        even_count = 0
        for num in nums:
            if num % 2 == 0:
                even_count += 1
        odd_count = len(nums) - even_count
        same = max(even_count, odd_count)
        
        max_dp0 = 0
        max_dp1 = 0
        
        for num in nums:
            if num % 2 == 0:
                current = max_dp1 + 1
                if current > max_dp0:
                    max_dp0 = current
            else:
                current = max_dp0 + 1
                if current > max_dp1:
                    max_dp1 = current
        
        alternating = max(max_dp0, max_dp1)
        
        return max(same, alternating)
```

### Explanation
1. **Same Parity Calculation**: The code first counts the number of even and odd numbers in the array. The maximum of these counts gives the length of the longest subsequence where all elements have the same parity (either all even or all odd).
2. **Alternating Parity Calculation**: The code then iterates through the array, maintaining two variables, `max_dp0` and `max_dp1`, which store the lengths of the longest alternating subsequences ending with an even or odd number, respectively. For each number, if it's even, it updates `max_dp0` using `max_dp1` and vice versa for odd numbers.
3. **Result Calculation**: The final result is the maximum between the longest same-parity subsequence and the longest alternating-parity subsequence, ensuring the solution covers both possible valid subsequence types. 

This approach efficiently checks both conditions for valid subsequences and leverages dynamic programming to compute the longest alternating subsequence in linear time, making it suitable for large input sizes.
