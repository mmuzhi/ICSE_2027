Here, P is a constant given in the input.
There are NM ways to choose a main dish and a side dish for a set meal. Find the total price of all these set meals.

Input

The input is given from Standard Input in the following format:
N M P
A_1 A_2 \dots A_N
B_1 B_2 \dots B_M

Output

Print the answer as an integer.
Under the constraints of this problem, it can be proved that the answer fits into a 64-bit signed integer.

Constraints


- 1\leq N,M \leq 2\times 10^5
- 1\leq A_i,B_j \leq 10^8
- 1\leq P \leq 2\times 10^8
- All input values are integers.

3 5
6 1

Sample Output 1

24

=== ANSWER ===

To solve this problem, we need to compute the total price of all possible set meals, where each set meal consists of one main dish and one side dish. The price of each set meal is the minimum of the sum of the prices of the selected main and side dishes and a given constant P. 

### Approach
1. **Problem Analysis**: The problem requires calculating the total price of all possible combinations of main and side dishes, where each combination's price is capped at P. The challenge is to efficiently compute this without checking all N*M combinations individually, which would be computationally expensive for large N and M (up to 200,000 each).

2. **Key Insight**: The total price can be derived by first calculating the sum of all possible combinations without the cap (S) and then adjusting this sum by subtracting the excess amounts for combinations where the sum of the dish prices exceeds P. Specifically, for each combination where the sum (s) is greater than or equal to P, we subtract (s - P) from the total sum.

3. **Algorithm Selection**:
   - **Sorting and Prefix Sums**: Sort the side dishes' prices and compute prefix sums to efficiently determine the number of side dishes and their total sum that, when added to a given main dish price, result in a sum ≥ P.
   - **Binary Search**: For each main dish price, use binary search to find the smallest side dish price that, when added to the main dish price, meets or exceeds P. This allows us to count the valid combinations and their total sum efficiently.

4. **Complexity Analysis**: Sorting the side dishes takes O(M log M). For each main dish, a binary search and prefix sum lookup take O(log M) and O(1) time, respectively. With N main dishes, the total time complexity is O(N log M + M log M), which is efficient for the given constraints.

### Solution Code
```python
import sys
import bisect

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
        
    it = iter(data)
    n = int(next(it)); m = int(next(it)); p = int(next(it))
    A = [int(next(it)) for _ in range(n)]
    B = [int(next(it)) for _ in range(m)]
    
    total_sum = sum(A) * m + sum(B) * n
    
    B_sorted = sorted(B)
    prefix_sum = [0] * (m + 1)
    for i in range(1, m + 1):
        prefix_sum[i] = prefix_sum[i - 1] + B_sorted[i - 1]
        
    sum_ge = 0
    count_ge = 0
    
    for a in A:
        x = p - a
        idx = bisect.bisect_left(B_sorted, x)
        num_ge = m - idx
        if num_ge > 0:
            s_b = prefix_sum[m] - prefix_sum[idx]
            sum_ge += num_ge * a + s_b
            count_ge += num_ge
            
    ans = total_sum - sum_ge + count_ge * p
    print(ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints.
2. **Initial Total Calculation**: The initial total sum (S) is computed as if there were no price cap, i.e., the sum of all main dishes multiplied by the number of side dishes plus the sum of all side dishes multiplied by the number of main dishes.
3. **Sorting and Prefix Sums**: The side dishes' prices are sorted, and a prefix sum array is built to quickly access the sum of any subarray of the sorted side dishes.
4. **Binary Search for Valid Combinations**: For each main dish price, the algorithm calculates the minimum side dish price required to meet or exceed P (i.e., P - main dish price). Using binary search, it finds the first side dish in the sorted list that meets this requirement. The count and sum of valid side dishes are then used to adjust the total sum.
5. **Adjusting Total Sum**: The total sum is adjusted by subtracting the excess amounts (sum_ge) for combinations where the dish prices sum to more than P and adding the capped price (count_ge * P) for these combinations.
6. **Output**: The final result is printed, which is the total price of all set meals after applying the price cap.

This approach efficiently handles the large input sizes by leveraging sorting, prefix sums, and binary search to minimize the number of operations required.
