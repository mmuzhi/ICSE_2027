=== PROMPT ===
You will be given a competitive programming problem.
N AtCoder users have gathered to play AtCoder RPS 2. The i-th user's name is S_i and their rating is C_i.
AtCoder RPS 2 is played as follows:

- Assign the numbers 0, 1, \dots, N - 1 to the users in lexicographical order of their usernames.
- Let T be the sum of the ratings of the N users. The user assigned the number T \bmod N is the winner.

Print the winner's username.

What is lexicographical order?

Lexicographical order, simply put, means "the order in which words appear in a dictionary." More precisely, the algorithm to determine the order of two distinct strings S and T consisting of lowercase English letters is as follows:

Here, "the i-th character of S" is denoted as S_i. If S is lexicographically smaller than T, we write S \lt T, and if S is larger, we write S \gt T.

-  Let L be the length of the shorter string among S and T. Check if S_i and T_i match for i=1,2,\dots,L. 
-  If there exists an i such that S_i \neq T_i, let j be the smallest such i. Compare S_j and T_j. If S_j is alphabetically smaller than T_j, then S \lt T. Otherwise, S \gt T. The algorithm ends here.
  
-  If there is no i such that S_i \neq T_i, compare the lengths of S and T. If S is shorter than T, then S \lt T. If S is longer, then S \gt T. The algorithm ends here.

Input

The input is given from Standard Input in the following format:
N
S_1 C_1
S_2 C_2
\vdots
S_N C_N

Output

Print the answer on a single line.

Constraints


- 1 \leq N \leq 100
- S_i is a string consisting of lowercase English letters with length between 3 and 16, inclusive.
- S_1, S_2, \dots, S_N are all distinct.
- 1 \leq C_i \leq 4229
- C_i is an integer.

Sample Output 1

Since 13 \bmod 3 = 1, print snuke, who is assigned number 1.

=== ANSWER ===

To solve this problem, we need to determine the winner of a competitive programming contest based on specific rules. The winner is determined by the sum of all user ratings modulo the number of users, and the users are ordered lexicographically by their usernames.

### Approach
1. **Read Input Data**: The first line of input gives the number of users, followed by each user's username and rating.
2. **Sort Users Lexicographically**: The users are sorted based on their usernames in lexicographical order. This means arranging the usernames as they would appear in a dictionary.
3. **Calculate Total Rating Sum**: Sum all the ratings of the users.
4. **Determine Winner**: The winner is the user at the position given by the total sum modulo the number of users (0-indexed). This position is used to select the winner from the sorted list.

### Solution Code
```python
n = int(input().strip())
users = []
for _ in range(n):
    data = input().split()
    users.append((data[0], int(data[1])))

users.sort(key=lambda x: x[0])

total = sum(r for _, r in users)
index = total % n

print(users[index][0])
```

### Explanation
1. **Reading Input**: The code starts by reading the number of users, `n`. It then reads each subsequent line, which contains a username and a rating, storing them as tuples in a list.
2. **Sorting Users**: The list of users is sorted lexicographically by their usernames using the `sort` method with a lambda function as the key.
3. **Total Rating Calculation**: The sum of all ratings is computed by iterating through the sorted list and adding up the ratings.
4. **Finding the Winner**: The winner's position is determined by taking the total rating sum modulo `n`. The user at this position in the sorted list is selected as the winner, and their username is printed.

This approach efficiently combines sorting and modular arithmetic to determine the winner according to the given rules. The solution handles all edge cases, including when the modulo operation results in zero, ensuring the correct user is selected from the sorted list.
