To solve this problem, we need to determine the position of a ball after it has been passed among a line of children for a given number of seconds. The ball starts at the first child (child 0) and moves rightwards. Upon reaching either end of the line, the direction reverses. The challenge is to efficiently compute the child holding the ball after `k` seconds without simulating each second step-by-step, especially since `k` can be up to 50 and `n` up to 50.

### Approach
1. **Problem Analysis**: The ball moves in a back-and-forth pattern between the first and last children. Each full cycle (from child 0 to child `n-1` and back to child 0) takes `2*(n-1)` seconds. 
2. **Key Insight**: The position of the ball after `k` seconds can be determined by reducing `k` modulo the cycle length (`2*(n-1)`). This gives the effective time within the current cycle. The ball's position is then the minimum of the effective time and the remaining steps in the cycle (cycle length minus effective time). This works because the movement is symmetric; the position at time `t` is the same as at time `cycle - t` if `t` is beyond the midpoint of the cycle.
3. **Algorithm Selection**: Compute the effective time by taking `k % (2*(n-1))`. The result is the minimum of this effective time and the remaining steps in the cycle. This avoids unnecessary iterations and efficiently computes the result using arithmetic operations.

### Solution Code
```python
class Solution:
    def numberOfChild(self, n: int, k: int) -> int:
        cycle = 2 * (n - 1)
        effective_k = k % cycle
        return min(effective_k, cycle - effective_k)
```

### Explanation
1. **Cycle Calculation**: The cycle length is `2*(n-1)` because the ball moves from child 0 to child `n-1` (taking `n-1` steps) and then back to child 0 (another `n-1` steps).
2. **Effective Time**: By taking `k % cycle`, we find the position within the current cycle. This handles cases where `k` is larger than the cycle length by wrapping around to the start of the cycle.
3. **Position Calculation**: The ball's position is the minimum of the effective time and the remaining steps in the cycle. This works because the movement is symmetric; the ball's position at time `t` is the same as at time `cycle - t` if `t` exceeds half the cycle length.
