=== ANSWER ===

To solve this problem, we need to generate all possible triples of non-negative integers (x, y, z) such that their sum is less than or equal to a given integer N. The triples should be printed in ascending lexicographical order.

### Approach
1. **Problem Analysis**: The problem requires generating all ordered triples (x, y, z) where each component is a non-negative integer and their sum does not exceed N. The lexicographical order means that we first compare the first elements (x), then the second elements (y), and finally the third elements (z) of the triples.
2. **Intuition**: The solution involves iterating through all possible values of x, y, and z in a nested loop manner. For each x, y ranges from 0 to N - x. For each y, z ranges from 0 to N - x - y. This ensures that the sum of the triple is always ≤ N and the triples are generated in lexicographical order.
3. **Algorithm Selection**: A straightforward three nested loop approach is chosen due to the constraints (N ≤ 21), which makes the solution efficient and easy to understand. The loops efficiently cover all valid triples without redundancy.
4. **Complexity Analysis**: The time complexity is O(N^3) in the worst case, but given the constraint N ≤ 21, this is manageable. The space complexity is O(1) as no additional data structures are used beyond simple variables.

### Solution Code
```python
def main():
    N = int(input().strip())
    for x in range(0, N + 1):
        for y in range(0, N - x + 1):
            for z in range(0, N - x - y + 1):
                print(f"{x} {y} {z}")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The integer N is read from standard input.
2. **Nested Loops**: The outermost loop iterates over all possible values of x from 0 to N. For each x, the middle loop iterates over y from 0 to N - x. The innermost loop then iterates over z from 0 to N - x - y, ensuring the sum x + y + z ≤ N.
3. **Output**: Each valid triple (x, y, z) is printed in the specified format. The loops naturally ensure the triples are printed in lexicographical order because x is varied first, followed by y, and then z.

This approach efficiently generates all required triples by leveraging nested loops and simple arithmetic checks, adhering to the problem constraints and requirements.
