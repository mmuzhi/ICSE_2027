To solve this problem, we need to determine the minimum number of operations required to collect all elements from 1 to k by repeatedly removing the last element of the array. The key observation here is that the set of removed elements must include all numbers from 1 to k. The solution involves checking the last x elements of the original array for each possible x until the set of these elements contains all required numbers.

### Approach
1. **Problem Analysis**: The problem requires us to collect all numbers from 1 to k by removing elements from the end of the array. The challenge is to find the smallest number of such operations needed.
2. **Intuition**: The solution hinges on the observation that the set of the last x elements of the original array must include all numbers from 1 to k. By traversing the array from the end towards the beginning, we can keep track of the collected numbers and stop as soon as all required numbers are found.
3. **Algorithm Selection**: We use a greedy approach where we iterate from the end of the array, adding each encountered number to a set if it falls within the range [1, k]. The iteration stops when the set contains all numbers from 1 to k, and the number of operations is the count of elements processed from the end.
4. **Complexity Analysis**: The algorithm processes each element at most once, resulting in a linear time complexity O(n), where n is the length of the array. This is efficient given the constraints (n ≤ 50).

### Solution Code
```python
class Solution:
    def minOperations(self, nums: List[int], k: int) -> int:
        target_set = set(range(1, k + 1))
        current_set = set()
        n = len(nums)
        for i in range(n - 1, -1, -1):
            if nums[i] in target_set:
                current_set.add(nums[i])
            if current_set == target_set:
                return n - i
        return n
```

### Explanation
1. **Initialization**: We start by creating a target set containing all numbers from 1 to k.
2. **Traversal**: We traverse the array from the end to the beginning. For each element, if it is within the target range, it is added to a current set.
3. **Check Completion**: After each addition, we check if the current set matches the target set. If it does, the number of operations required is the number of elements processed from the end (calculated as `n - i`, where `i` is the current index).
4. **Termination**: The loop terminates as soon as all target numbers are found, ensuring the solution is both correct and efficient. If the loop completes without finding all numbers (though the problem guarantees it will), the default return is the length of the array, implying all elements must be removed.

This approach efficiently narrows down the minimal operations by leveraging set operations and a single traversal of the array, ensuring optimal performance.
