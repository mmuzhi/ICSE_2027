=== ANSWER ===

To solve this problem, we need to efficiently compute the number of adjacent elements with the same color after each query in a dynamic array where elements are colored one by one. The challenge lies in maintaining the count of adjacent same-color pairs without recalculating from scratch after each query, which would be too slow for large inputs.

### Approach
1. **Initialization**: Start with an array of zeros (uncolored elements) and a variable `total` to keep track of the current count of adjacent same-color pairs.
2. **Processing Queries**: For each query:
   - **Check for No Change**: If the new color is the same as the current color at the specified index, the count remains unchanged.
   - **Remove Old Contributions**: If the index was previously colored, remove the contributions of the old color to the adjacent pairs. Specifically, check the left and right neighbors; if they were the same as the old color, decrement `total`.
   - **Update Color**: Set the new color at the specified index.
   - **Add New Contributions**: Check the left and right neighbors again; if they are the same as the new color, increment `total`.
3. **Result Collection**: After processing each query, record the current `total` in the result list.

This approach ensures that each query is processed in constant time, making the solution efficient for large inputs.

```python
        ans = []
        for idx, color in queries:
            old_color = arr[idx]
            if old_color == color:
                ans.append(total)
                continue
            if old_color != 0:
                if idx - 1 >= 0 and arr[idx-1] == old_color:
                    total -= 1
                if idx + 1 < n and arr[idx+1] == old_color:
                    total -= 1
            arr[idx] = color
            if idx - 1 >= 0 and arr[idx-1] == color:
                total += 1
            if idx + 1 < n and arr[idx+1] == color:
                total += 1
            ans.append(total)
        return ans
```

### Explanation
- **Initialization**: The array `arr` is initialized to all zeros, representing uncolored elements. The variable `total` starts at zero, indicating no adjacent same-color pairs initially.
- **Query Processing**: For each query:
  - If the new color matches the current color at the index, the result for this query is simply the current `total`.
  - If the index was previously colored, the code checks its neighbors. If any neighbor had the same old color, those adjacent pairs are removed from `total`.
  - The color at the specified index is updated to the new color.
  - The code then checks the neighbors again. If any neighbor now has the same new color, those pairs are added to `total`.
- **Result Collection**: After processing each query, the current `total` is appended to the result list `ans`, which is returned after all queries are processed.

This method efficiently tracks the number of adjacent same-color pairs by only considering the immediate neighbors of the updated index, ensuring optimal performance even for large inputs.
