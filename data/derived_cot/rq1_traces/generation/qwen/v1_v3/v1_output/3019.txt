The moves can be 'L' (left), 'R' (right), or '_' (underscore, which can be chosen as either 'L' or 'R'). The goal is to maximize the absolute distance from the origin by strategically choosing the direction for each underscore move.

### Approach
1. **Problem Analysis**: The key observation here is that the underscores can be treated as flexible moves that can either add to the left or right displacement. The fixed moves ('L' and 'R') contribute directly to the base displacement. The underscores can then be used to extend this displacement in the direction that maximizes the absolute distance from the origin.

2. **Intuition**: The maximum absolute distance is achieved by either:
   - Using all underscores to move in the direction that aligns with the base displacement (i.e., adding their count to the base displacement).
   - Or using all underscores to move in the opposite direction (i.e., subtracting their count from the base displacement).
   
   The furthest point will be the one with the larger absolute value between these two scenarios.

3. **Algorithm Selection**: 
   - Count the number of 'L' and 'R' moves to compute the base displacement (`base = countR - countL`).
   - Determine the number of underscores (`U = total_moves - (countR + countL)`).
   - The maximum absolute distance is the larger of `|base + U|` and `|base - U|`.

4. **Complexity Analysis**: The solution involves counting characters in the string, which is an O(n) operation where n is the length of the string. This is efficient given the constraint that n ≤ 50.
