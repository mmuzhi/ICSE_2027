To solve this problem, we need to find the lexicographically smallest string `t` that can be obtained by changing characters of the original string `s` such that the total cyclic distance between `s` and `t` is less than or equal to `k`. The cyclic distance between two characters is defined as the minimum number of steps required to move from one character to another in a circular arrangement of the alphabet (from 'a' to 'z').

### Approach
1. **Problem Analysis**: The key observation is that to achieve the lexicographically smallest string, we should prioritize changing the leftmost characters to the smallest possible characters (i.e., 'a') while ensuring the total cost (sum of cyclic distances) does not exceed `k`. For each character in the string, we consider changing it to any character from 'a' up to the original character (to maintain lexicographical order) and choose the smallest character that can be afforded with the remaining `k`.

2. **Intuition**: By processing each character from left to right, we can greedily set each character to the smallest possible value (starting from 'a') that the remaining `k` allows. This ensures the resulting string is lexicographically smallest while keeping the total cost within the constraint.

3. **Algorithm Selection**: The algorithm involves iterating over each character in the string. For each character, it checks all possible target characters from 'a' to the original character, calculates the cyclic distance, and selects the smallest character that can be changed within the remaining `k`. The cost of changing the character is subtracted from `k`, and the process continues with the next character.

4. **Complexity Analysis**: The algorithm processes each character in the string once, and for each character, it checks up to 26 possible target characters (from 'a' to 'z'). Thus, the time complexity is O(n * 26), where `n` is the length of the string. This is efficient given the constraints (string length up to 100).

### Solution Code
```python
class Solution:
    def getSmallestString(self, s: str, k: int) -> str:
        s_list = list(s)
        remaining_k = k
        n = len(s_list)
        for i in range(n):
            current_char = s_list[i]
            for target_ord in range(97, ord(current_char) + 1):
                target_char = chr(target_ord)
                diff = abs(ord(current_char) - ord(target_char))
                cost = min(diff, 26 - diff)
                if cost <= remaining_k:
                    s_list[i] = target_char
                    remaining_k -= cost
                    break
        return ''.join(s_list)
```

### Explanation
1. **Initialization**: Convert the input string `s` into a list of characters for easy manipulation.
2. **Processing Each Character**: For each character in the string, iterate from 'a' to the original character (inclusive). For each candidate character, compute the cyclic distance (cost) required to change the original character to the candidate.
3. **Cost Check**: If the computed cost is within the remaining `k`, update the character in the list, subtract the cost from `k`, and break out of the loop to proceed to the next character.
4. **Result Construction**: After processing all characters, join the list back into a string and return it. The result is the lexicographically smallest string achievable within the given cost constraint.

This approach efficiently balances the need to minimize the string lexicographically while adhering to the cost constraint by leveraging a greedy selection of the smallest affordable characters from left to right.
