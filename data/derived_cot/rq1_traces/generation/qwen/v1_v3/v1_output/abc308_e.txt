To solve this problem, we need to find the sum of the minimum excluded values (mex) of all valid triples (i, j, k) in a sequence where the characters at these positions form the string "MEX" and the indices are in increasing order. The mex of a set of numbers is the smallest non-negative integer not present in the set.

### Approach
1. **Problem Analysis**: The problem requires processing all valid triples (i, j, k) such that the characters at these positions in the string S are 'M', 'E', and 'X' respectively, and the indices are in increasing order. For each such triple, we compute the mex of the corresponding values from the sequence A. The challenge is to efficiently count these triples and compute their mex values without resorting to a brute-force approach, which would be too slow for large inputs.

2. **Intuition**: The key observation is that the values in sequence A are limited to 0, 1, or 2. This allows us to precompute a mex table for all possible combinations of three values (each from 0 to 2). Additionally, we can use Fenwick Trees (Binary Indexed Trees) to efficiently count the occurrences of each value in the relevant ranges (left of 'E' for 'M' and right of 'E' for 'X').

3. **Algorithm Selection**: 
   - **Fenwick Trees**: We use three Fenwick Trees for 'M' values (0, 1, 2) and three for 'X' values. These trees help in efficiently querying the count of specific values in any range.
   - **Mex Table**: A precomputed table that stores the mex value for every possible combination of three values (each 0, 1, or 2). This allows quick lookup during the main computation.

4. **Complexity Analysis**: 
   - **Fenwick Tree Operations**: Each update and query operation on a Fenwick Tree is O(log n). We perform O(n) updates (for 'M' and 'X' values) and O(n) queries (for each 'E' in the string), leading to an overall complexity of O(n log n).
   - **Mex Table**: The table is computed once and has a fixed size (3x3x3 = 27 entries), making it O(1) in terms of space and time.

### Solution Code
```python
class FenwTree:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)
    
    def update(self, index, delta):
        i = index + 1
        while i <= self.n:
            self.tree[i] += delta
            i += i & -i
            
    def query(self, index):
        if index < 0:
            return 0
        i = index + 1
        s = 0
        while i:
            s += self.tree[i]
            i -= i & -i
        return s
            
    def range_query(self, l, r):
        if l > r:
            return 0
        res = self.query(r)
        if l > 0:
            res -= self.query(l - 1)
        return res

def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    n = int(data[0].strip())
    A = list(map(int, data[1].split()))
    S = data[2].strip()
    
    mex_table = {}
    for a in range(3):
        for b in range(3):
            for c in range(3):
                s = {a, b, c}
                m = 0
                while m in s:
                    m += 1
                mex_table[(a, b, c)] = m
                
    FenwM = [FenwTree(n) for _ in range(3)]
    FenwX = [FenwTree(n) for _ in range(3)]
    
    for i in range(n):
        if S[i] == 'M':
            val = A[i]
            FenwM[val].update(i, 1)
        elif S[i] == 'X':
            val = A[i]
            FenwX[val].update(i, 1)
            
    E_list = []
    for i in range(n):
        if S[i] == 'E':
            E_list.append((i, A[i]))
            
    total = 0
    for idx, val_E in E_list:
        count_M = [FenwM[a].range_query(0, idx - 1) for a in range(3)]
        count_X = [FenwX[c].range_query(idx + 1, n - 1) for c in range(3)]
        for a in range(3):
            for c in range(3):
                total += count_M[a] * count_X[c] * mex_table[(a, val_E, c)]
```
