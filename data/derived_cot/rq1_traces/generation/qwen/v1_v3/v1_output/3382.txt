To solve this problem, we need to count the number of subarrays where the first and last elements are equal to the largest element in the subarray. The key insight is that for any subarray to satisfy the condition, the first and last elements must be the maximum element in that subarray, and no element in the subarray can be greater than this maximum.

### Approach
1. **Problem Analysis**: The problem requires identifying subarrays where the first and last elements are the maximum elements in the subarray. This means the subarray must not contain any element greater than its first and last elements. 

2. **Intuition**: For each element in the array, we can determine the segment in which it is the maximum element by finding the next greater elements to its left and right. This segment defines the boundaries within which any subarray starting and ending with this element will have this element as the maximum.

3. **Algorithm Selection**:
   - **Monotonic Stack**: We use a monotonic stack to compute the next greater elements to the left and right for each element in the array. This helps in determining the boundaries for each element where it is the maximum.
   - **Binary Search**: For each element, we use binary search on the list of its occurrences to count how many times it appears within its valid segment (defined by the boundaries from the monotonic stack). This count gives the number of valid subarrays starting at that element.

4. **Complexity Analysis**: The algorithm involves two passes over the array with a stack (O(n) each) and a binary search for each element (O(log n) per element). The total complexity is O(n log n), which is efficient for the given constraints (n ≤ 10^5).

### Solution Code
```python
import bisect
from collections import defaultdict
from typing import List

class Solution:
    def numberOfSubarrays(self, nums: List[int]) -> int:
        n = len(nums)
        left_bound = [-1] * n
        stack = []
        for i in range(n):
            while stack and nums[stack[-1]] <= nums[i]:
                stack.pop()
            left_bound[i] = stack[-1] if stack else -1
            stack.append(i)
        
        right_bound = [n] * n
        stack = []
        for i in range(n-1, -1, -1):
            while stack and nums[stack[-1]] <= nums[i]:
                stack.pop()
            right_bound[i] = stack[-1] if stack else n
            stack.append(i)
        
        indices = defaultdict(list)
        for i, num in enumerate(nums):
            indices[num].append(i)
        
        ans = 0
        for i in range(n):
            x = nums[i]
            r_bound_val = right_bound[i] - 1
            lst = indices[x]
            left_idx = bisect.bisect_left(lst, i)
            right_idx = bisect.bisect_right(lst, r_bound_val) - 1
            if left_idx <= right_idx:
                ans += (right_idx - left_idx + 1)
        return ans
```

### Explanation
1. **Monotonic Stack for Boundaries**: The algorithm first computes the `left_bound` and `right_bound` arrays using a monotonic stack. These arrays store the indices of the next greater elements to the left and right for each element, respectively. This helps in defining the segment where each element is the maximum.

2. **Dictionary of Indices**: A dictionary maps each number to a list of its indices in the array. This allows efficient lookup of all occurrences of a number.

3. **Binary Search for Valid Subarrays**: For each element, the algorithm uses binary search to count how many times the element appears within its valid segment (from `left_bound[i] + 1` to `right_bound[i] - 1`). The count of such occurrences gives the number of valid subarrays starting at that element.

4. **Efficiency**: The use of a monotonic stack ensures the boundary computation is done in O(n) time, and binary search for counting valid subarrays is O(log n) per element, leading to an overall O(n log n) time complexity, which is optimal for the problem constraints.
