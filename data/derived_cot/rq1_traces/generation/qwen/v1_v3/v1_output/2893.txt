=== ANSWER ===

To solve this problem, we need to maximize the score obtained by traversing a sequence of numbers in an array, starting from the first element, while accounting for penalties when moving between numbers of different parities. The key insight is to use dynamic programming to keep track of the best scores achievable when ending with an even or odd number at each step.

### Approach
1. **Initialization**: Start by initializing two variables, `dp_even` and `dp_odd`, to negative infinity. These variables will store the maximum scores achievable when the last number in the sequence is even or odd, respectively.
2. **Base Case**: The first element of the array determines the initial values of `dp_even` or `dp_odd`. If the first element is even, set `dp_even` to its value; otherwise, set `dp_odd` to its value.
3. **Iteration**: For each subsequent element in the array:
   - If the element is even, update `dp_even` to be the maximum of:
     - Extending an existing even-ending sequence (no penalty).
     - Extending an existing odd-ending sequence (with penalty).
   - If the element is odd, update `dp_odd` similarly.
4. **Result**: After processing all elements, the result is the maximum value between `dp_even` and `dp_odd`, representing the highest score achievable by ending with either an even or odd number.

This approach efficiently tracks the best possible scores for sequences ending with even or odd numbers, leveraging dynamic programming to avoid recomputation and ensure optimal performance.

### Solution Code
```python
class Solution:
    def maxScore(self, nums: List[int], x: int) -> int:
        dp_even = -10**18
        dp_odd = -10**18
        
        if nums[0] % 2 == 0:
            dp_even = nums[0]
        else:
            dp_odd = nums[0]
            
        for i in range(1, len(nums)):
            if nums[i] % 2 == 0:
                dp_even = max(dp_even + nums[i], dp_odd + nums[i] - x)
            else:
                dp_odd = max(dp_odd + nums[i], dp_even + nums[i] - x)
                
        return max(dp_even, dp_odd)
```

### Explanation
- **Initialization**: The variables `dp_even` and `dp_odd` are initialized to a very low value (`-10^18`) to represent that no sequence ending with an even or odd number has been processed initially.
- **Base Case**: The first element of the array sets the initial state. If it's even, `dp_even` is set to its value; if it's odd, `dp_odd` is set to its value.
- **Iteration**: For each subsequent element, the algorithm checks its parity. If it's even, `dp_even` is updated by considering both extending an even-ending sequence (no penalty) or an odd-ending sequence (with penalty). The same logic applies for odd elements.
- **Result**: The final result is the maximum value between `dp_even` and `dp_odd`, representing the highest score achievable by optimally traversing the array while accounting for parity changes.

This approach efficiently computes the maximum score by dynamically updating the best possible scores for sequences ending with even or odd numbers, ensuring optimal performance with a linear pass through the array.
