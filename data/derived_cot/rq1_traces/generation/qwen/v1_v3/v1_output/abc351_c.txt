### Approach

The challenge is to efficiently manage the sequence by merging adjacent balls of the same size whenever they appear at the end of the sequence. The merging process can cascade, meaning that after each merge, the new ball might merge again with the next ball if they are of the same size.
2. **Intuition**: Using a stack data structure is ideal for this problem because it allows us to efficiently access and modify the top elements of the sequence. Each time we add a new ball, we check if the top two balls are of the same size. If they are, we merge them and repeat the check until no more merges are possible.
3. **Algorithm Selection**: The algorithm processes each ball in the input sequence, pushing it onto a stack. After each insertion, it checks if the top two elements of the stack are equal. If they are, it pops them, pushes a new element (the sum of the two popped elements), and repeats the check. This approach ensures that all possible merges are handled efficiently.
4. **Complexity Analysis**: The algorithm processes each ball exactly once, and each merge operation reduces the total number of balls by one. Since there are at most \(N\) balls, the total number of merge operations is \(O(N)\). Each operation involves stack operations (push and pop), which are \(O(1)\) on average.
