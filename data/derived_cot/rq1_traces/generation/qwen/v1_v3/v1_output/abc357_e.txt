You will be given a competitive programming problem.
There is a directed graph with N vertices numbered 1 to N and N edges.
The out-degree of every vertex is 1, and the edge from vertex i points to vertex a_i.
Count the number of pairs of vertices (u, v) such that vertex v is reachable from vertex u.
Here, vertex v is reachable from vertex u if there exists a sequence of vertices w_0, w_1, \dots, w_K of length K+1 that satisfies the following conditions. In particular, if u = v, it is always reachable.

- w_0 = u.
- w_K = v.
- For every 0 \leq i \lt K, there is an edge from vertex w_i to vertex w_{i+1}.

Input

The input is given from Standard Input in the following format:
N
a_1 a_2 \dots a_N

Output

Print the number of pairs of vertices (u, v) such that vertex v is reachable from vertex u.

Constraints


- 1 \leq N \leq 2 \times 10^5
- 1 \leq a_i \leq N
- All input values are integers.

Sample Input 1

4
2 1 1 4

Sample Output 1

8

The vertices reachable from vertex 3 are vertices 1, 2, 3.
The vertex reachable from vertex 4 is vertex 4.
Therefore, the number of pairs of vertices (u, v) such that vertex v is reachable from vertex u is 8.
Note that the edge from vertex 4 is a self-loop, that is, it points to vertex 4 itself.

=== ANSWER ===

To solve this problem, we need to count the number of pairs of vertices (u, v) in a directed graph where vertex v is reachable from vertex u. The graph is given with N vertices and N edges, meaning each vertex has exactly one outgoing edge, forming a structure of cycles with trees attached to them.

### Approach
1. **Cycle Detection**: The graph consists of one or more cycles, each with trees (directed towards the cycle). We first detect all cycles in the graph. For each node in a cycle, we record the cycle's length.
2. **Reverse Graph Construction**: We build a reverse graph where each edge is reversed. This helps in propagating information from cycle nodes to tree nodes.
3. **BFS for Distance and Component Assignment**: Using BFS from cycle nodes, we compute the distance of each node from its nearest cycle node. During BFS, we also assign the cycle length of the component to each node (all nodes in the same component share the same cycle length).
4. **Counting Reachable Pairs**: For each node, the number of reachable nodes is the sum of its distance to the cycle and the cycle length. Summing this value for all nodes gives the total number of reachable pairs.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    a_list = list(map(int, data[1:1+n]))
    next_node = [x-1 for x in a_list]

    visited = [False] * n
    in_cycle = [False] * n
    cycle_length_arr = [-1] * n

    for i in range(n):
        if visited[i]:
            continue
        path_dict = {}
        path_list = []
        cur = i
        while not visited[cur]:
            visited[cur] = True
            path_dict[cur] = len(path_list)
            path_list.append(cur)
            cur = next_node[cur]
        if cur in path_dict:
            idx = path_dict[cur]
            cycle_nodes = path_list[idx:]
            L = len(cycle_nodes)
            for node in cycle_nodes:
                in_cycle[node] = True
                cycle_length_arr[node] = L

    rev_graph = [[] for _ in range(n)]
    for i in range(n):
        j = next_node[i]
        rev_graph[j].append(i)

    dist = [-1] * n
    comp_cycle_length = [-1] * n

    q = deque()
    for i in range(n):
        if in_cycle[i]:
            dist[i] = 0
            comp_cycle_length[i] = cycle_length_arr[i]
            q.append(i)

    while q:
        u = q.popleft()
        for v in rev_graph[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                comp_cycle_length[v] = comp_cycle_length[u]
                q.append(v)

    total = 0
    for i in range(n):
        total += (dist[i] + comp_cycle_length[i])

    print(total)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Cycle Detection**: The algorithm traverses each node, marking visited nodes and detecting cycles using a path dictionary to track node indices in the current traversal path. When a node is revisited within the same path, a cycle is identified, and its length is recorded.
2. **Reverse Graph Construction**: The reverse graph is built to facilitate BFS, allowing us to propagate distances from cycle nodes to tree nodes efficiently.
3. **BFS for Distance and Component Assignment**: Starting from cycle nodes, BFS computes the shortest distance to each node from its cycle node and assigns the cycle length of the component. This helps in determining the number of reachable nodes for each node.
4. **Counting Reachable Pairs**: For each node, the number of reachable nodes is the sum of its distance to the cycle and the cycle length. Summing these values across all nodes gives the total number of reachable pairs, which is printed as the result. 

This approach efficiently handles the graph structure by leveraging cycle detection and BFS, ensuring optimal performance even for large graphs.
