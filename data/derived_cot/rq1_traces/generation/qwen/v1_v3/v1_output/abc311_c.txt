=== PROMPT ===
You will be given a competitive programming problem.
There is a directed graph with N vertices and N edges.
The i-th edge goes from vertex i to vertex A_i. (The constraints guarantee that i \neq A_i.)
Find a directed cycle without the same vertex appearing multiple times.
It can be shown that a solution exists under the constraints of this problem.
Notes
The sequence of vertices B = (B_1, B_2, \dots, B_M) is called a directed cycle when all of the following conditions are satisfied:

- M \geq 2
- The edge from vertex B_i to vertex B_{i+1} exists. (1 \leq i \leq M-1)
- The edge from vertex B_M to vertex B_1 exists.
- If i \neq j, then B_i \neq B_j.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \dots A_N

Output

Print a solution in the following format:
M
B_1 B_2 \dots B_M

M is the number of vertices, and B_i is the i-th vertex in the directed cycle.
The following conditions must be satisfied:

- 2 \le M
- B_{i+1} = A_{B_i} ( 1 \le i \le M-1 )
- B_{1} = A_{B_M}
- B_i \neq B_j ( i \neq j )

If multiple solutions exist, any of them will be accepted.

Constraints


- All input values are integers.
- 2 \le N \le 2 \times 10^5
- 1 \le A_i \le N
- A_i \neq i

7
6 7 2 1 3 4 5

=== ANSWER ===

To solve this problem, we need to find a directed cycle in a graph with N vertices and N edges. The graph is structured such that each vertex has exactly one outgoing edge, forming a functional graph. This structure guarantees the presence of at least one cycle. The solution involves traversing the graph to detect the first cycle encountered during the traversal.

### Approach
1. **Graph Representation**: The graph is represented using an array `next_node` where each element `next_node[i]` indicates the vertex that the current vertex `i` points to. This array is derived from the input by converting vertex numbers to 0-based indices.
2. **Cycle Detection**: We iterate over each vertex in reverse order (from N-1 down to 0). For each unvisited vertex, we follow the edges until we encounter a visited node. If the visited node is part of the current traversal path, a cycle is detected.
3. **Cycle Extraction**: Once a cycle is detected, the cycle is extracted from the traversal path starting from the first occurrence of the repeated node. The cycle is then converted to 1-based indexing and printed.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    next_node = [x-1 for x in A]
    
    visited = [False] * n
    for i in range(n-1, -1, -1):
        if not visited[i]:
            path = []
            cur = i
            while not visited[cur]:
                visited[cur] = True
                path.append(cur)
                cur = next_node[cur]
            if cur in path:
                idx = path.index(cur)
                cycle = path[idx:]
                print(len(cycle))
                print(" ".join(str(x+1) for x in cycle))
                return

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first value is the number of vertices `N`, followed by `N` integers representing the edges.
2. **Graph Setup**: The `next_node` array is created by converting each vertex in the input to a 0-based index, forming a directed graph where each vertex points to exactly one other vertex.
3. **Traversal and Cycle Detection**: Starting from the highest vertex index down to 0, each unvisited vertex is processed. The traversal follows the edges until a visited node is encountered. If this visited node is part of the current path, a cycle is identified.
4. **Cycle Output**: The cycle is extracted from the path starting at the first occurrence of the repeated node. The cycle's vertices are converted to 1-based indexing and printed, along with the cycle length.

This approach efficiently detects the first cycle encountered during the traversal, ensuring optimal performance with a time complexity of O(N) due to the functional graph structure and the use of a visited array to track nodes. The solution guarantees a valid cycle is found and printed in the required format.
