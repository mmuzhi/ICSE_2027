You will be given a competitive programming problem.
Given an array of strings words and a character separator, split each string in words by separator.
Return an array of strings containing the new strings formed after the splits, excluding empty strings.
Notes

separator is used to determine where the split should occur, but it is not included as part of the resulting strings.
A split may result in more than two strings.
The resulting strings must maintain the same order as they were initially given.

 
Example 1:

Input: words = ["one.two.three","four.five","six"], separator = "."
Output: ["one","two","three","four","five","six"]
Explanation: In this example we split as follows:

"one.two.three" splits into "one", "two", "three"
"four.five" splits into "four", "five"
"six" splits into "six" 

Hence, the resulting array is ["one","two","three","four","five","six"].
Example 2:

Input: words = ["$easy$","$problem$"], separator = "$"
Output: ["easy","problem"]
Explanation: In this example we split as follows: 

"$easy$" splits into "easy" (excluding empty strings)
"$problem$" splits into "problem" (excluding empty strings)

Hence, the resulting array is ["easy","problem"].

Example 3:

Input: words = ["|||"], separator = "|"
Output: []
Explanation: In this example the resulting split of "|||" will contain only empty strings, so we return an empty array []. 
 
Constraints:

1 <= words.length <= 100
1 <= words[i].length <= 20
characters in words[i] are either lowercase English letters or characters from the string ".,|$#@" (excluding the quotes)
separator is a character from the string ".,|$#@" (excluding the quotes)

=== ANSWER ===

To solve this problem, we need to split each string in a given list of words by a specified separator and then collect all the resulting non-empty strings into a single list while maintaining the original order of the words.

### Approach
1. **Problem Analysis**: The task involves processing each string in the input list by splitting it at every occurrence of the given separator. The key requirement is to exclude any resulting empty strings from the final output. The solution must efficiently handle all edge cases, such as strings that split entirely into empty strings (like "|||" with separator "|").

2. **Intuition**: The solution leverages Python's built-in `split()` method, which splits a string into a list of substrings based on a specified separator. By iterating over each word, splitting it, and filtering out empty strings, we can construct the desired output list.

3. **Algorithm Selection**: The algorithm involves a straightforward iteration over each word in the input list. For each word, it splits the string using the given separator and then filters out any empty strings from the resulting list. The non-empty parts are collected into the result list.

4. **Complexity Analysis**: The time complexity is O(n * m), where n is the number of words and m is the average length of each word. This is because each word is processed once, and splitting the word takes time proportional to its length. The space complexity is O(n * k), where k is the average number of non-empty parts per word, as we store all non-empty parts in the result list.

### Solution Code
```python
class Solution:
    def splitWordsBySeparator(self, words: List[str], separator: str) -> List[str]:
        return [part for word in words for part in word.split(separator) if part]
```

### Explanation
- **Problem Analysis**: The problem requires splitting each string in the input list by a specified separator and collecting all non-empty resulting parts in order.
- **Intuition**: Using Python's `split()` method simplifies the splitting process, and a list comprehension efficiently filters out empty strings.
- **Algorithm Selection**: The solution iterates over each word, splits it using the separator, and includes only non-empty parts in the result.
- **Complexity Analysis**: The solution efficiently processes each word in linear time relative to its length, making it suitable for the given constraints. The space complexity is linear with respect to the total number of non-empty parts across all words.
