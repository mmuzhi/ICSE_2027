To solve this problem, we need to efficiently answer multiple queries regarding the maximum sum of elements from two arrays, `nums1` and `nums2`, under specific constraints. Each query provides two values, `x_i` and `y_i`, and we need to find the maximum value of `nums1[j] + nums2[j]` for any index `j` where `nums1[j] >= x_i` and `nums2[j] >= y_i`. If no such index exists, we return -1.

### Approach
1. **Problem Analysis**: The problem requires processing up to 100,000 queries on two arrays of size up to 100,000. A brute-force approach would be too slow, so we need an efficient method. The key observation is that each query is independent, and we can leverage offline processing and a segment tree to answer these queries efficiently.

2. **Intuition**: By sorting the queries in descending order of `x_i` and processing them, we can add points (from `nums1` and `nums2`) to a data structure as their `x_i` becomes smaller or equal. This allows us to answer each query by considering all relevant points that meet the `x_i` condition and then querying the maximum sum where the `y_i` condition is also satisfied.

3. **Algorithm Selection**: 
   - **Coordinate Compression**: Compress the values of `nums2` and the `y_i` values from queries to a smaller range for efficient segment tree operations.
   - **Segment Tree**: Use a segment tree to keep track of the maximum sum (`a + b`) for each compressed `b` value. The segment tree supports point updates and range maximum queries.
   - **Offline Processing**: Sort both the points (from `nums1` and `nums2`) and the queries by `x_i` in descending order. Process each query, adding all points with `a >= x_i` to the segment tree, then query the segment tree for the maximum sum where `b >= y_i`.

4. **Complexity Analysis**: 
   - **Preprocessing**: Sorting the points and queries takes O(n log n + q log q) time. Coordinate compression also takes O(n + q log n) time.
   - **Query Processing**: Each point is added once, and each query is processed in O(log m) time (where m is the number of distinct compressed values). Thus, the total time complexity is O((n + q) log m), which is efficient for the given constraints.

import bisect
from typing import List

class SegmentTree:
    def __init__(self, n):
        self.n = n
        self.size = 1
        while self.size < n:
            self.size *= 2
        self.data = [-1] * (2 * self.size)

    def update(self, index, value):
        idx = index + self.size
        if self.data[idx] < value:
            self.data[idx] = value
        idx //= 2
        while idx:
            self.data[idx] = max(self.data[2*idx], self.data[2*idx+1])
            idx //= 2

    def query(self, l, r):
        if l > r:
            return -1
        l += self.size
        r += self.size
        res = -1
        while l <= r:
            if l % 2 == 1:
                res = max(res, self.data[l])
                l += 1
            if r % 2 == 0:
                res = max(res, self.data[r])
                r -= 1
            l //= 2
            r //= 2
        return res

class Solution:
    def maximumSumQueries(self, nums1: List[int], nums2: List[int], queries: List[List[int]]) -> List[int]:
        n = len(nums1)
        points = [(nums1[i], nums2[i]) for i in range(n)]
        all_vals = set()
        for a, b in points:
            all_vals.add(b)
        for q in queries:
            all_vals.add(q[1])
        all_vals = sorted(all_vals)
        m = len(all_vals)
        points_sorted = sorted(points, key=lambda x: x[0], reverse=True)
        queries_sorted = [(x, y, i) for i, (x, y) in enumerate(queries)]
        queries_sorted.sort(key=lambda q: q[0], reverse=True)
        seg_tree = SegmentTree(m)
        res = [-1] * len(queries)
        ptr = 0
        for x, y, orig_idx in queries_sorted:
            while ptr < n and points_sorted[ptr][0] >= x:
                a_val, b_val = points_sorted[ptr]
                pos = bisect.bisect_left(all_vals, b_val)
                seg_tree.update(pos, a_val + b_val)
                ptr += 1
            y_pos = bisect.bisect_left(all_vals, y)
            ans = seg_tree.query(y_pos, m-1)
            res[orig_idx] = ans if ans != -1 else -1
        return res
