To solve this problem, we need to determine if a graph constructed by sequentially adding edges from multiple operations is connected and, if it is, find the total weight of the edges in its minimum spanning tree (MST). The graph starts with no edges, and each operation adds a complete subgraph (clique) on a given subset of vertices with a specified weight for all edges in that subgraph. 

### Approach
1. **Problem Analysis**: The graph is built by processing M operations, each adding all edges of a complete subgraph with a uniform weight. The challenge is to efficiently determine the MST of the resulting graph without explicitly generating all edges, which would be computationally infeasible for large N (up to 200,000 vertices).

2. **Key Insight**: The MST of the graph can be constructed by considering the operations in increasing order of their edge weights. For each operation, we use its weight to merge the connected components of its vertices. The minimal edges required to connect these components are provided by the operation's weight, ensuring the MST is built with the smallest possible edges.

3. **Algorithm Selection**: We use the Union-Find (Disjoint Set Union, DSU) data structure with path compression and union by rank to efficiently manage and merge connected components. The operations are sorted by their edge weights to process them from smallest to largest, leveraging Kruskal's algorithm principles without explicitly listing all edges.

4. **Complexity Analysis**: The DSU operations (find and union) are nearly constant time due to path compression and union by rank. The total number of operations is O(M log M) for sorting, and each operation processes up to K_i vertices, leading to an overall complexity of O((N + sum K_i) α(N)), where α is the inverse Ackermann function, making it efficient for large inputs.

### Solution Code
```python
import sys

sys.setrecursionlimit(300000)

class DSU:
    def __init__(self, n):
        self.parent = list(range(n+1))
        self.rank = [0]*(n+1)
    
    def find(self, a):
        if self.parent[a] != a:
            self.parent[a] = self.find(self.parent[a])
        return self.parent[a]
    
    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return
        if self.rank[a] < self.rank[b]:
            self.parent[a] = b
        elif self.rank[a] > self.rank[b]:
            self.parent[b] = a
        else:
            self.parent[b] = a
            self.rank[a] += 1

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    tokens = list(map(int, data))
    n = tokens[0]
    m = tokens[1]
    index = 2
    operations = []
    for i in range(m):
        k_i = tokens[index]; index += 1
        c_i = tokens[index]; index += 1
        vertices = []
        for j in range(k_i):
            vertices.append(tokens[index]); index += 1
        operations.append((c_i, vertices))
    
    operations.sort(key=lambda x: x[0])
    
    dsu = DSU(n)
    total_weight = 0
    
    for c_i, vertices in operations:
        root_set = set()
        for v in vertices:
            root_set.add(dsu.find(v))
        
        roots_list = list(root_set)
        k = len(roots_list)
        if k > 1:
            total_weight += (k-1) * c_i
            base = roots_list[0]
            for i in range(1, k):
                r_i = dsu.find(roots_list[i])
                if dsu.find(base) != r_i:
                    dsu.union(base, r_i)
                    base = dsu.find(base)
    
    components = set()
    for i in range(1, n+1):
        components.add(dsu.find(i))
    if len(components) > 1:
        print(-1)
    else:
        print(total_weight)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The DSU structure is initialized to keep track of connected components, with each vertex initially its own component.
2. **Operation Processing**: Operations are sorted by edge weight to process them from smallest to largest. For each operation, the roots of all vertices in its subset are found. If these roots form multiple components, they are merged using the operation's weight, and the total weight of the MST is updated.
3. **Connectivity Check**: After processing all operations, the number of connected components is checked. If there's more than one component, the graph is disconnected, and -1 is printed. Otherwise, the total weight of the MST is printed.

This approach efficiently handles the problem by leveraging the DSU structure to dynamically manage components and ensures the MST is constructed with minimal edge weights.
