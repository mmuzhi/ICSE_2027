To solve this problem, we need to determine the minimum number of right shifts required to sort a given array of distinct positive integers. A right shift moves each element to the next position, with the last element wrapping around to the first position. The solution involves checking if the array can be sorted into ascending order through any number of right shifts and, if possible, calculating the minimal number of shifts needed.

### Approach
1. **Check if the array is already sorted**: Iterate through the array to verify if it is in ascending order. If it is, return 0 as no shifts are needed.
2. **Count the number of descents**: A descent is identified where an element is greater than the next element (considering the array circularly). The array can be sorted by right shifts only if there is exactly one descent. If there are zero or more than one descents, return -1.
3. **Find the index of the minimum element**: The minimum element's position in the original array determines the number of right shifts needed. The formula used is `n - index_of_min`, where `n` is the length of the array. This formula works because rotating the array right by `n - index_of_min` positions brings the minimum element to the start, resulting in the sorted array.

### Solution Code
```python
class Solution:
    def minimumRightShifts(self, nums: List[int]) -> int:
        n = len(nums)
        is_sorted = True
        for i in range(n-1):
            if nums[i] > nums[i+1]:
                is_sorted = False
                break
        if is_sorted:
            return 0
        
        descents = 0
        for i in range(n):
            if nums[i] > nums[(i+1) % n]:
                descents += 1
        
        if descents != 1:
            return -1
        
        min_val = min(nums)
        idx = nums.index(min_val)
        
        return n - idx
```

### Explanation
1. **Check for Sorted Array**: The loop checks each adjacent pair in the array. If any element is greater than the next, the array is not sorted, and we proceed to the next step.
2. **Count Descents**: The loop checks each element against the next element (considering the array as circular). If there is exactly one descent, the array can be sorted by right shifts; otherwise, it's impossible.
3. **Calculate Right Shifts**: The position of the smallest element in the original array is found. The number of right shifts needed to bring this element to the start (thus sorting the array) is calculated as `n - index_of_min`. This works because each right shift moves the array one position to the right, and `n - index_of_min` positions are needed to align the smallest element at the beginning.

This approach efficiently checks the feasibility of sorting the array with right shifts and calculates the minimal shifts required if possible. The solution handles edge cases, such as already sorted arrays or arrays that cannot be sorted, gracefully.
