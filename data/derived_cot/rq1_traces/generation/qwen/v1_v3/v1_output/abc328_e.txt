To solve this problem, we need to find the minimum cost of a spanning tree in a given weighted, connected, undirected graph, where the cost is defined as the sum of the edge weights modulo a given integer K. The solution involves generating all possible spanning trees of the graph, computing their costs, and selecting the minimum cost.

### Approach
1. **Problem Analysis**: The problem requires finding a spanning tree (a connected acyclic subgraph that includes all vertices) with the minimum cost, where the cost is the sum of edge weights modulo K. Given the constraints (N ≤ 8), a brute-force approach is feasible.
2. **Intuition**: Since the graph is small (up to 8 vertices), we can generate all combinations of edges of size (N-1) (the number of edges in any spanning tree) and check if they form a spanning tree. For each valid spanning tree, compute the sum of its edge weights modulo K and keep track of the minimum value encountered.
3. **Algorithm Selection**: The solution involves:
   - Reading the input values and constructing the list of edges.
   - Generating all combinations of (N-1) edges from the given edges.
   - For each combination, checking if it forms a spanning tree by ensuring all vertices are included.
   - Calculating the sum of edge weights modulo K for each valid spanning tree and updating the minimum cost accordingly.
4. **Complexity Analysis**: The number of combinations to check is C(M, N-1), where M is the number of edges (up to 28 for N=8). For N=8, this is C(28,7) = 1184040 combinations, which is manageable given the constraints.

### Solution Code
```python
import itertools
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it))
    m = int(next(it))
    K = int(next(it))
    edges = []
    for _ in range(m):
        u = int(next(it))
        v = int(next(it))
        w = int(next(it))
        edges.append((u, v, w))
        
    min_cost = K
    
    for comb in itertools.combinations(edges, n-1):
        nodes_used = set()
        total_weight = 0
        for u, v, w in comb:
            nodes_used.add(u)
            nodes_used.add(v)
            total_weight = (total_weight + w) % K
        if nodes_used != set(range(1, n+1)):
            continue
        if total_weight < min_cost:
            min_cost = total_weight
            
    print(min_cost)

if __name__ == "__main__":
    main()
```

3. **Checking Spanning Tree Validity**: For each combination, the set of vertices covered by the edges is compared to the set of all vertices (1 to N). If they match, the combination is a valid spanning tree.
4. **Calculating Cost**: For each valid spanning tree, the sum of its edge weights is computed modulo K. The minimum cost encountered during this process is updated and printed at the end.
