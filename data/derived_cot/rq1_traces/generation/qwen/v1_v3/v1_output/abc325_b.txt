To solve this problem, we need to determine the optimal one-hour meeting time in UTC that allows the maximum number of employees from different bases to participate. The key insight is that each base's local time at the meeting start must be within the 9:00 to 18:00 window for all employees at that base to participate. 

### Approach
1. **Problem Analysis**: The problem involves calculating the local time at each base for every possible UTC meeting start time (from 0 to 23) and checking if the entire meeting hour (from start time to start time + 1) falls within the 9:00 to 18:00 window. 
2. **Intuition**: For each base, the local start time of the meeting is derived by adding the base's UTC offset to the meeting start time (mod 24). If this local start time is between 9 and 17 (inclusive), the entire meeting hour will be within the 9:00 to 18:00 window.
3. **Algorithm Selection**: We iterate over all 24 possible UTC start times. For each start time, we check all bases to see if their local start time falls within the valid range. The total number of employees available for each start time is summed, and the maximum value across all start times is our answer.
4. **Complexity Analysis**: The algorithm runs in O(N * 24) time, where N is the number of bases. Given the constraint N ≤ 1000, this approach is efficient with a maximum of 24,000 operations.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    n = int(data[0])
    bases = []
    for i in range(1, n + 1):
        parts = data[i].split()
        w = int(parts[0])
        x = int(parts[1])
        bases.append((w, x))
    
    best = 0
    for T in range(24):
        total_employees = 0
        for w, x in bases:
            local_time = (x + T) % 24
            if 9 <= local_time <= 17:
                total_employees += w
        if total_employees > best:
            best = total_employees
            
    print(best)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line is the number of bases, followed by lines containing each base's employee count and UTC offset.
2. **Processing Each Start Time**: For each possible UTC start time (0 to 23), the code checks every base to see if the local meeting start time (calculated as `(offset + start_time) % 24`) falls between 9 and 17.
3. **Summing Employees**: For each valid start time, the number of employees from all bases meeting the condition is summed.
4. **Finding Maximum Participation**: The maximum number of employees across all start times is tracked and printed as the result.

This approach efficiently checks all possible meeting times to find the one that maximizes employee participation, leveraging modular arithmetic to handle the 24-hour clock and simple range checks for the valid meeting window.
