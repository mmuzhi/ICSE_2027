=== ANSWER ===

To solve this problem, we need to partition a given array into two non-empty subarrays such that the value of the partition, defined as the absolute difference between the maximum element of the first subarray and the minimum element of the second subarray, is minimized. 

### Approach
1. **Problem Analysis**: The key observation here is that the optimal partition value can be achieved by sorting the array. Once sorted, the smallest possible partition value between any two consecutive elements in the array will be the minimal difference between elements in the sorted array. This is because splitting the array between two consecutive elements will ensure that the maximum of the left subarray is the first element and the minimum of the right subarray is the next element, and their difference will be minimized.

2. **Intuition**: By sorting the array, we can systematically check the differences between consecutive elements. The smallest difference between any two consecutive elements in the sorted array will be the answer, as splitting the array at this point will yield the minimal partition value.

3. **Algorithm Selection**: The algorithm involves sorting the array and then iterating through it to find the smallest difference between consecutive elements. This approach efficiently narrows down the problem to checking adjacent elements in the sorted array, leveraging the fact that the minimal partition value must be the smallest gap between any two elements in the sorted order.

4. **Complexity Analysis**: Sorting the array takes O(n log n) time, where n is the number of elements in the array. The subsequent iteration to find the minimal difference is O(n). Thus, the overall time complexity is dominated by the sorting step, O(n log n), which is efficient for the given constraints (n ≤ 10^5).

### Solution Code
```python
class Solution:
    def findValueOfPartition(self, nums: List[int]) -> int:
        nums.sort()
        min_diff = float('inf')
        for i in range(1, len(nums)):
            diff = nums[i] - nums[i-1]
            if diff < min_diff:
                min_diff = diff
        return min_diff
```

### Explanation
1. **Sorting the Array**: The array is sorted to bring consecutive elements in ascending order. This allows us to easily compute the differences between adjacent elements, which are potential candidates for the minimal partition value.
2. **Finding Minimal Difference**: After sorting, we iterate through the array starting from the second element. For each element, we calculate its difference with the previous element. The smallest difference encountered during this iteration is the answer, as it represents the minimal partition value achievable by splitting the array at that point.
3. **Efficiency**: The solution efficiently leverages sorting and a single pass through the array to determine the minimal partition value, ensuring optimal performance even for large input sizes. The use of sorting ensures that we consider all possible partitions in a systematic manner, while the linear scan guarantees that we find the minimal difference in linear time.
