    def count(self, num1: str, num2: str, min_sum: int, max_sum: int) -> int:
        mod = 10**9 + 7
        num1_minus_one = self.subtract_one(num1)
        count2 = self.count_up_to(num2, min_sum, max_sum)
        count1 = self.count_up_to(num1_minus_one, min_sum, max_sum)
        return (count2 - count1) % mod

    def subtract_one(self, s):
        digits = list(map(int, s))
        n = len(digits)
        i = n - 1
        while i >= 0 and digits[i] == 0:
            digits[i] = 9
            i -= 1
        if i < 0:
            return "0"
        digits[i] -= 1
        # Remove leading zeros
        res = ""
        leading = True
        for d in digits:
            if leading and d == 0:
                continue
            else:
                leading = False
                res += str(d)
        return res if res != "" else "0"

To solve this problem, we need to count the number of integers between two given strings `num1` and `num2` (inclusive) such that the sum of their digits lies within a specified range `[min_sum, max_sum]`. Given the constraints where `num1` and `num2` can be as large as \(10^{22}\), a brute-force approach is infeasible. Instead, we use a digit dynamic programming (digit DP) approach to efficiently count the numbers meeting the criteria.

### Approach
1. **Digit Dynamic Programming (Digit DP)**: The digit DP technique is used to count numbers up to a given number `s` that satisfy certain conditions (in this case, the sum of their digits being between `min_sum` and `max_sum`). The approach involves processing each digit of the number from left to right, keeping track of:
   - **Position**: The current digit being processed.
   - **Tight Constraint**: Whether the digits processed so far match the corresponding digits in `s` (limiting the next digits).
   - **Sum So Far**: The cumulative sum of the digits processed.
   - **Started Flag**: Indicates if non-zero digits have been encountered (to handle leading zeros).

2. **Subtracting One from a String**: Since `num1` and `num2` are given as strings, we need a helper function to subtract 1 from `num1` (to compute the range from `num1` to `num2` as `count_up_to(num2) - count_up_to(num1 - 1)`).

3. **Modulo Operation**: Given the potentially large result, all operations are performed modulo \(10^9 + 7\).

### Solution Code
```python
class Solution:
    def count(self, num1: str, num2: str, min_sum: int, max_sum: int) -> int:
        mod = 10**9 + 7
        num1_minus_one = self.subtract_one(num1)
        count2 = self.count_up_to(num2, min_sum, max_sum)
        count1 = self.count_up_to(num1_minus_one, min_sum, max_sum)
        return (count2 - count1) % mod

    def subtract_one(self, s):
        digits = list(map(int, s))
        n = len(digits)
        i = n - 1
        while i >= 0 and digits[i] == 0:
            digits[i] = 9
            i -= 1
        if i < 0:
            return "0"
        digits[i] -= 1
        res = ""
        leading = True
        for d in digits:
            if leading and d == 0:
                continue
            else:
                leading = False
                res += str(d)
        return res if res != "" else "0"

    def count_up_to(self, s, min_sum, max_sum):
        mod = 10**9 + 7
        digits = [int(ch) for ch in s]
        n = len(digits)

        from functools import lru_cache
        @lru_cache(maxsize=None)
        def dp(pos, tight, sum_so_far, started):
            if sum_so_far > max_sum:
                return 0
            if pos == n:
                if started and min_sum <= sum_so_far <= max_sum:
                    return 1
                elif not started:
                    if min_sum <= 0 <= max_sum:
                        return 1
                    else:
                        return 0
                else:
                    return 0
            total = 0
            limit = digits[pos] if tight else 9
            for d in range(0, limit + 1):
                new_tight = tight and (d == limit)
                new_started = started or (d != 0)
                new_sum = sum_so_far + d if new_started else sum_so_far
                total = (total + dp(pos + 1, new_tight, new_sum, new_started)) % mod
            return total % mod

        return dp(0, True, 0, False) % mod
```

1. **Digit DP Function (`count_up_to`)**:
   - **Parameters**: The function processes a string `s` (the upper bound), and integers `min_sum` and `max_sum`.
   - **DP State**: The state consists of the current digit position, a tight constraint flag, the current digit sum, and a flag indicating if non-zero digits have started.
   - **Base Case**: When all digits are processed, check if the number (if started) has a digit sum within the specified range.
   - **Recursive Case**: For each digit position, iterate through possible digits (0 to the current digit if tight, else 0-9), update the tight constraint, digit sum, and started flag, then recursively count valid numbers.

2. **Subtract One Helper (`subtract_one`)**:
   - Converts the string to a list of digits, subtracts 1 by handling borrows, and removes leading zeros from the result.

3. **Main Function (`count`)**:
   - Computes the count of valid numbers from 0 to `num2` and subtracts the count from 0 to `num1 - 1` (using `count_up_to`), then returns the result modulo \(10^9 + 7\).
