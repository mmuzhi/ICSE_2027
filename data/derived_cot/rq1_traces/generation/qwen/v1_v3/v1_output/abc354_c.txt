To solve this problem, we need to determine the set of cards that remain after repeatedly discarding cards that can be dominated by others. A card is dominated if there exists another card with higher strength and lower cost. The solution involves sorting the cards by strength in descending order and using a segment tree to efficiently check for dominated cards during the process.

### Approach
1. **Problem Analysis**: The problem requires us to find the maximal set of cards such that no card in the set can be dominated by any other card in the set. A card is dominated if there exists another card with higher strength and lower cost.
2. **Intuition**: By sorting the cards in descending order of strength, we can process each card and check if there exists any previously processed card (with higher strength) that has a lower cost. If such a card exists, the current card is dominated and should be discarded.
3. **Algorithm Selection**: 
   - **Sorting**: Cards are sorted by their strength in descending order to process stronger cards first.
   - **Coordinate Compression**: Since the costs can be very large (up to 10^9), we compress the cost values to a smaller range for efficient segment tree operations.
   - **Segment Tree**: A segment tree is used to keep track of the minimum cost encountered so far for the processed cards. For each card, we query the segment tree to check if any card with higher strength (already processed) has a lower cost.
4. **Complexity Analysis**: 
   - Sorting the cards takes O(N log N) time.
   - Coordinate compression also takes O(N log N) time due to sorting.
   - Each segment tree operation (query and update) takes O(log N) time, and we perform these operations for each card, leading to O(N log N) time overall.
   - The space complexity is O(N) for storing the segment tree and other auxiliary data structures.
