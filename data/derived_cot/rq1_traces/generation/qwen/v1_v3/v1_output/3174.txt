To solve this problem, we need to determine the minimum number of changes required to transform a given binary string into a beautiful string. A beautiful string is defined as one that can be partitioned into one or more substrings of even lengths, where each substring contains only 0s or only 1s. 

### Approach
1. **Problem Analysis**: The key observation here is that the minimal changes required to make the string beautiful can be achieved by partitioning the string into segments of length 2. For each segment of length 2, if the two characters are different, we need exactly one change to make them the same (either both 0s or both 1s). If they are already the same, no changes are needed. 
2. **Insight**: By breaking the string into segments of length 2, we ensure that each segment can be made uniform with minimal changes. The total number of changes required is simply the count of segments where the two characters differ.
3. **Algorithm Selection**: The algorithm involves iterating over the string in steps of 2, checking each pair of characters. For each pair, if the characters are different, we increment our change count. The sum of these increments gives the minimal changes needed.
4. **Complexity Analysis**: The algorithm runs in O(n) time, where n is the length of the string, since it processes each character exactly once. The space complexity is O(1) as no additional data structures are used.

### Solution Code
```python
class Solution:
    def minChanges(self, s: str) -> int:
        changes = 0
        for i in range(0, len(s), 2):
            if s[i] != s[i+1]:
                changes += 1
        return changes
```

### Explanation
- **Initialization**: We start with a `changes` counter set to 0.
- **Iteration**: We loop through the string in steps of 2, checking each pair of characters at positions `i` and `i+1`.
- **Comparison**: For each pair, if the characters are different, we increment the `changes` counter by 1.
- **Result**: After processing all pairs, the `changes` counter holds the minimal number of changes required to make the entire string beautiful, which is returned as the result.

This approach efficiently checks each necessary pair and ensures the solution is optimal with minimal computational overhead.
