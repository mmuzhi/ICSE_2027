=== ANSWER ===

To solve this problem, we need to determine if a given positive integer is a 321-like number. A 321-like number is defined as a number where each digit from left to right is strictly decreasing. This means that for any two consecutive digits in the number, the left digit must be greater than the right digit.

### Approach
1. **Convert the Number to a String**: This allows us to easily access each digit in the number.
2. **Check for Single-Digit Numbers**: Any single-digit number is trivially a 321-like number, so we immediately return "Yes" for such cases.
3. **Iterate Through the Digits**: For numbers with more than one digit, we check each pair of consecutive digits. If any digit is found to be less than or equal to the next digit, the number is not 321-like, and we return "No".
4. **Return Result**: If all consecutive digits satisfy the strictly decreasing condition, we return "Yes".

### Solution Code
```python
def main():
    N = input().strip()
    s = N
    if len(s) == 1:
        print("Yes")
    else:
        valid = True
        for i in range(len(s) - 1):
            if s[i] <= s[i+1]:
                valid = False
                break
        print("Yes" if valid else "No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Handling**: The input is read as a string to facilitate digit-by-digit processing.
2. **Single-Digit Check**: If the input string has only one character, it is automatically a 321-like number, so the output is "Yes".
3. **Digit Comparison**: For strings longer than one character, each digit is compared to the next one. If any digit is not strictly greater than the next, the loop breaks early, and "No" is printed.
4. **Result Determination**: If all consecutive digits meet the strictly decreasing condition, "Yes" is printed; otherwise, "No" is printed.

This approach efficiently checks the required condition by leveraging simple string manipulation and iteration, ensuring correctness for all valid inputs within the given constraints.
