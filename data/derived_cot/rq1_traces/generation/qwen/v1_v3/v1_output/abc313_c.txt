To solve this problem, we need to transform an integer sequence such that the difference between the maximum and minimum values in the sequence is at most one, using the minimum number of operations. Each operation allows us to decrease one element by one and increase another by one. 

### Approach
1. **Problem Analysis**: The key observation is that the total sum of the sequence remains constant during any operation. The goal is to distribute the elements such that most elements are floor(total_sum / n) and some are ceil(total_sum / n), where n is the number of elements. The difference between these two values is at most one.
2. **Insight**: The optimal target sequence consists of (n - r) elements of value m (floor(total_sum / n)) and r elements of value m + 1, where r is the remainder when total_sum is divided by n. This ensures the difference between the maximum and minimum values is either 0 or 1.
3. **Algorithm Selection**: 
   - Calculate the total sum of the sequence.
   - Determine m (floor(total_sum / n)) and r (total_sum % n).
   - Sort the sequence to efficiently assign the smallest (n - r) elements to m and the largest r elements to m + 1.
   - Compute the total surplus, which is the sum of all elements exceeding their target values. This surplus directly gives the number of operations needed since each operation reduces one surplus unit.

### Solution Code
```python
n = int(input().strip())
A = list(map(int, input().split()))
total = sum(A)
m = total // n
r = total % n

A.sort()

surplus = 0
for i in range(n - r):
    if A[i] > m:
        surplus += A[i] - m

for i in range(n - r, n):
    if A[i] > m + 1:
        surplus += A[i] - (m + 1)

print(surplus)
```

### Explanation
1. **Reading Input**: The input consists of an integer n (number of elements) followed by the sequence of integers.
2. **Total Sum Calculation**: The sum of all elements is computed to determine the target values.
3. **Target Values**: The base value m is the floor of the total sum divided by n, and r is the remainder. This gives (n - r) elements of m and r elements of m + 1 in the target sequence.
4. **Sorting**: The sequence is sorted to efficiently assign the smallest elements to the base value m and the largest elements to m + 1.
5. **Surplus Calculation**: The surplus is the sum of all elements exceeding their target values. This surplus must be reduced through operations, each of which decreases one surplus unit. Hence, the total surplus gives the minimum number of operations required.
6. **Output**: The computed surplus is printed, representing the minimal operations needed to achieve the desired sequence. 

This approach efficiently minimizes the operations by leveraging sorting and arithmetic properties to distribute the elements optimally. The complexity is dominated by the sorting step, making it O(n log n), which is efficient for the given constraints.
