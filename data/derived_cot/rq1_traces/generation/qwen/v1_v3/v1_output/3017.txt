To solve this problem, we need to count the number of beautiful integers within a given range [low, high]. A beautiful integer is defined as a number that meets two conditions: the count of even digits equals the count of odd digits, and the number is divisible by a given integer k. Given the constraints where low and high can be as large as 10^9, a brute-force approach is infeasible. Instead, we use digit dynamic programming (digit DP) to efficiently count the numbers that satisfy both conditions without iterating through each number in the range.

### Approach
1. **Digit Dynamic Programming (Digit DP)**: This technique is used to solve problems involving digits of numbers by breaking down the problem into smaller subproblems. The key idea is to process each digit of the number from left to right, keeping track of certain states (like whether the current number being formed is still within the bounds of the given range, the remainder modulo k, and the difference between the counts of even and odd digits).

2. **State Tracking**: The DP state includes:
   - **Position**: The current digit being processed.
   - **Tight Constraint**: A boolean indicating if the digits processed so far match the corresponding digits of the upper bound (to ensure the number does not exceed the upper bound).
   - **Started**: A boolean indicating if non-zero digits have been processed (to handle leading zeros).
   - **Remainder**: The remainder of the number formed so far when divided by k.
   - **Difference**: The difference between the count of even and odd digits processed so far.

3. **Base Case**: When all digits are processed, we check if the number is valid (non-zero, divisible by k, and has equal counts of even and odd digits).

4. **Recursive Case**: For each digit position, iterate through possible digits (0 to 9, constrained by the tight condition), update the states, and recursively compute the count of valid numbers.

5. **Range Handling**: The solution computes the count of beautiful numbers from 1 to high and subtracts the count from 1 to low-1 to get the result for the range [low, high].
