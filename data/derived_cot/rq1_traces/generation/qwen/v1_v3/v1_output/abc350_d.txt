To solve this problem, we need to determine the maximum number of operations that can be performed on an SNS where each operation involves adding a friendship between three users who form a specific pattern: two users are friends, and the third is a mutual friend but not directly connected. The solution involves analyzing the graph structure of the SNS to count the number of possible operations efficiently.

### Approach
1. **Graph Representation**: Represent the SNS as an undirected graph where nodes are users and edges are friendships.
2. **Connected Components Identification**: Use BFS to identify all connected components in the graph. Each connected component is a subgraph where any two nodes are connected by edges.
3. **Edge Counting per Component**: For each connected component, count the number of edges within it.
4. **Operation Calculation**: For each connected component, calculate the maximum number of operations possible by subtracting the existing edges from the total edges required to make the component a complete graph (i.e., every pair of nodes is connected by an edge). The formula used is \( \frac{n \times (n-1)}{2} - m \), where \( n \) is the number of nodes in the component and \( m \) is the number of edges in it.
5. **Summing Operations**: Sum the operations across all connected components to get the final answer.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    it = iter(data)
    n = int(next(it))
    m = int(next(it))
    graph = [[] for _ in range(n+1)]
    edges_list = []
    for _ in range(m):
        u = int(next(it))
        v = int(next(it))
        edges_list.append((u, v))
        graph[u].append(v)
        graph[v].append(u)
    
    visited = [False] * (n+1)
    comp_id = [0] * (n+1)
    comp_count = 0
    for i in range(1, n+1):
        if not visited[i]:
            comp_count += 1
            q = deque([i])
            visited[i] = True
            comp_id[i] = comp_count
            while q:
                node = q.popleft()
                for neighbor in graph[node]:
                    if not visited[neighbor]:
                        visited[neighbor] = True
                        comp_id[neighbor] = comp_count
                        q.append(neighbor)
    
    node_count_per_comp = [0] * (comp_count+1)
    for i in range(1, n+1):
        cid = comp_id[i]
        node_count_per_comp[cid] += 1
    
    edge_count_per_comp = [0] * (comp_count+1)
    for (u, v) in edges_list:
        cid = comp_id[u]
        edge_count_per_comp[cid] += 1
    
    total_operations = 0
    for cid in range(1, comp_count+1):
        n_nodes = node_count_per_comp[cid]
        n_edges = edge_count_per_comp[cid]
        total_operations += (n_nodes * (n_nodes - 1) // 2 - n_edges)
    
    print(total_operations)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Graph Representation**: The graph is built using adjacency lists from the input edges. Each edge is added in both directions to represent undirected friendships.
2. **Connected Components Identification**: Using BFS, we traverse the graph starting from each unvisited node to mark all reachable nodes with the same component ID. This helps in identifying all connected components.
3. **Edge Counting per Component**: For each component, we count the number of edges by iterating through the input edges and checking if both endpoints belong to the same component.
4. **Operation Calculation**: For each component, the maximum operations needed to make it a complete graph is calculated by subtracting the existing edges from the total possible edges in a complete graph of that size. The result is summed across all components to get the final answer.
5. **Efficiency**: The BFS and edge counting steps operate in linear time relative to the number of nodes and edges, making the solution efficient for large inputs as specified in the problem constraints.
