class Solution:
    def maxSubarrayLength(self, nums: List[int], k: int) -> int:
        left = 0
        freq = {}
        max_len = 0
        for right in range(len(nums)):
            # Increase the count for nums[right]

            # If the current element's count exceeds k, shrink the window from the left

                left += 1
            # Now, the window [left, right] is good

To solve this problem, we need to find the length of the longest contiguous subarray where the frequency of each element is at most `k`. The solution involves using a sliding window technique to efficiently track the frequencies of elements within the current window and adjust the window size whenever any element's frequency exceeds `k`.

### Approach
1. **Sliding Window Technique**: We maintain a window defined by two pointers, `left` and `right`, which represent the start and end of the current subarray being considered.
2. **Frequency Tracking**: A dictionary is used to keep track of the frequency of each element within the current window.
3. **Window Adjustment**: As we expand the window by moving the `right` pointer, we increment the count of the current element in the dictionary. If any element's count exceeds `k`, we shrink the window from the `left` until the condition is satisfied again.
4. **Max Length Update**: After each adjustment, we update the maximum length of the valid subarray encountered so far.

This approach ensures that we efficiently explore all possible valid subarrays in linear time, making it optimal for large input sizes.

### Solution Code
```python
class Solution:
    def maxSubarrayLength(self, nums: List[int], k: int) -> int:
        left = 0
        freq = {}
        max_len = 0
        for right in range(len(nums)):
            freq[nums[right]] = freq.get(nums[right], 0) + 1
            while freq[nums[right]] > k:
                freq[nums[left]] -= 1
                left += 1
            max_len = max(max_len, right - left + 1)
        return max_len
```

### Explanation
- **Initialization**: We start with `left` pointer at 0, an empty dictionary `freq` to store element frequencies, and `max_len` set to 0 to keep track of the longest valid subarray found.
- **Expanding the Window**: For each element at `right` pointer, we update its frequency in the dictionary.
- **Shrinking the Window**: If the frequency of the current element exceeds `k`, we move the `left` pointer to the right, decrementing the count of the elements that are left behind, until the frequency of the current element is within the allowed limit.
- **Updating Maximum Length**: After ensuring the window is valid, we update `max_len` if the current window size (`right - left + 1`) is greater than the previously recorded maximum.
- **Result**: After processing all elements, `max_len` holds the length of the longest subarray where each element's frequency is at most `k`.

This approach efficiently processes each element exactly once, leading to an optimal solution with a time complexity of O(n), where n is the number of elements in the input array. The space complexity is O(m), where m is the number of distinct elements in the array, due to the dictionary used for frequency tracking.
