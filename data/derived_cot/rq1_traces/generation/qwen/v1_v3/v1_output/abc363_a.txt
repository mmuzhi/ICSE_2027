=== ANSWER ===

To solve this problem, we need to determine the minimum increase in a user's rating required to display one more '^' in AtCoder's rating system. The rating system categorizes users based on their rating into specific ranges, each displaying a different number of '^' symbols. The goal is to find the smallest rating increase needed to move from the current category to the next higher one.

### Approach
1. **Problem Analysis**: The problem involves categorizing the user's current rating into one of four predefined ranges (1-99, 100-199, 200-299, and 300-399) and then calculating the minimal rating increase required to reach the next higher category. The key observation is that each category starts at a specific rating (100 for 2 '^', 200 for 3 '^', and 300 for 4 '^'). 

2. **Intuition**: For any given rating, the next category's starting point is determined by the current category. For instance:
   - If the rating is between 1 and 99, the next category starts at 100.
   - If the rating is between 100 and 199, the next category starts at 200.
   - If the rating is between 200 and 299, the next category starts at 300.

3. **Algorithm Selection**: The solution involves checking the current rating against these thresholds and computing the difference between the next category's start rating and the current rating. This difference is the minimal increase required.

4. **Complexity Analysis**: The solution involves a simple conditional check and arithmetic operation, making it an O(1) time complexity solution, which is optimal for the given constraints.

### Solution Code
```python
def main():
    R = int(input().strip())
    if R < 100:
        next_tier = 100
    elif R < 200:
        next_tier = 200
    else:
        next_tier = 300
    print(next_tier - R)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Handling**: The code reads the current rating `R` from the input.
2. **Category Determination**: Using conditional statements, the code checks which category the current rating falls into:
   - If `R` is less than 100, the next category starts at 100.
   - If `R` is between 100 and 199, the next category starts at 200.
   - If `R` is 200 or higher (but within the constraint of 299), the next category starts at 300.
3. **Calculation**: The minimal increase required is the difference between the next category's start rating and the current rating `R`.
4. **Output**: The result is printed, which is the computed minimal increase.

This approach efficiently categorizes the rating and calculates the minimal increase needed, ensuring optimal performance and correctness for the given problem constraints.
