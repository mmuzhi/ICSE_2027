You will be given a competitive programming problem.
You are given an array of positive integers nums of length n.
A polygon is a closed plane figure that has at least 3 sides. The longest side of a polygon is smaller than the sum of its other sides.
Conversely, if you have k (k >= 3) positive real numbers a_1, a_2, a_3, ..., a_k where a_1 <= a_2 <= a_3 <= ... <= a_k and a_1 + a_2 + a_3 + ... + a_k-1 > a_k, then there always exists a polygon with k sides whose lengths are a_1, a_2, a_3, ..., a_k.
The perimeter of a polygon is the sum of lengths of its sides.
Return the largest possible perimeter of a polygon whose sides can be formed from nums, or -1 if it is not possible to create a polygon.
 
Example 1:

Input: nums = [5,5,5]
Output: 15
Explanation: The only possible polygon that can be made from nums has 3 sides: 5, 5, and 5. The perimeter is 5 + 5 + 5 = 15.

Example 2:

Input: nums = [1,12,1,2,5,50,3]
Output: 12
Explanation: The polygon with the largest perimeter which can be made from nums has 5 sides: 1, 1, 2, 3, and 5. The perimeter is 1 + 1 + 2 + 3 + 5 = 12.
We cannot have a polygon with either 12 or 50 as the longest side because it is not possible to include 2 or more smaller sides that have a greater sum than either of them.
It can be shown that the largest possible perimeter is 12.

Example 3:

Input: nums = [5,5,50]
Output: -1
Explanation: There is no possible way to form a polygon from nums, as a polygon has at least 3 sides and 50 > 5 + 5.

=== ANSWER ===

To solve this problem, we need to find the largest possible perimeter of a polygon that can be formed from an array of positive integers. The polygon must have at least 3 sides, and the longest side must be strictly less than the sum of the other sides. The solution involves sorting the array and then iterating through it to find the largest valid subset of sides that meets the polygon condition.

### Approach
1. **Sort the Array**: Sorting helps in efficiently checking subsets of sides from the smallest to the largest.
2. **Iterate and Accumulate**: For each element in the sorted array, accumulate the sum of the elements up to that point.
3. **Check Polygon Condition**: For each accumulated sum (representing a potential polygon), check if the largest side in the current subset is less than the sum of the remaining sides. This is verified by the condition `total > 2 * nums[i]`, where `total` is the sum of the subset and `nums[i]` is the largest side in the subset.
4. **Track Maximum Perimeter**: Whenever the condition is met, update the answer to the current sum. The last valid sum encountered during the iteration will be the largest possible perimeter.

### Solution Code
```python
class Solution:
    def largestPerimeter(self, nums: List[int]) -> int:
        nums.sort()
        total = 0
        ans = 0
        for i in range(len(nums)):
            total += nums[i]
            if i >= 2 and total > 2 * nums[i]:
                ans = total
        return ans if ans != 0 else -1
```

### Explanation
1. **Sorting**: The array is sorted in ascending order to facilitate the efficient checking of subsets. This allows us to consider subsets from the smallest to the largest sides.
2. **Accumulating Sum**: As we iterate through the sorted array, we maintain a running sum (`total`) of the elements processed so far. This sum represents the perimeter of the potential polygon formed by the current subset of sides.
3. **Condition Check**: For each element (starting from the third element to ensure at least 3 sides), we check if the largest side in the current subset (`nums[i]`) is less than the sum of the remaining sides (`total - nums[i]`). This is simplified to `total > 2 * nums[i]`.
4. **Updating Answer**: If the condition is satisfied, the current sum (`total`) is a valid perimeter, and we update our answer (`ans`) to this value. Since we iterate from smallest to largest, the last valid `ans` encountered will be the largest possible perimeter.
5. **Return Result**: If no valid polygon is found, return -1; otherwise, return the largest valid perimeter.

This approach efficiently narrows down the solution by leveraging sorting and a single pass through the array, ensuring optimal performance with a time complexity of O(n log n) due to the sorting step.
