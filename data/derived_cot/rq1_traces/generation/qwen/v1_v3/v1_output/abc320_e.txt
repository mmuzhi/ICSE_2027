You will be given a competitive programming problem.
There are N people gathered for an event called Flowing Noodles. The people are lined up in a row, numbered 1 to N in order from front to back.
During the event, the following occurrence happens M times:

- At time T_i, a quantity W_i of noodles is flown down. The person at the front of the row gets all of it (if no one is in the row, no one gets it). That person then steps out of the row and returns to their original position in the row at time T_i+S_i.

A person who returns to the row at time X is considered to be in the row at time X.
After all the M occurrences, report the total amount of noodles each person has got.

Input

The input is given from Standard Input in the following format:
N M
T_1 W_1 S_1
\vdots
T_M W_M S_M

Output

Print N lines.
The i-th line should contain the amount of noodles person i has got.

Constraints


- 1 \leq N \leq 2\times 10^5
- 1 \leq M \leq 2\times 10^5
- 0 <T_1 <\ldots < T_M \leq 10^9
- 1 \leq S_i \leq 10^9
- 1 \leq W_i \leq 10^9
- All input values are integers.

Sample Input 1

3 5
1 1 3
2 10 100
4 100 10000
10 1000 1000000000
100 1000000000 1

Sample Output 1

101
10
1000

To solve this problem, we need to simulate the process of distributing noodles to people in a queue over multiple events, where people temporarily leave the queue and return at specified times. The key challenge is efficiently managing the availability of people in the queue and processing the noodle distribution events in chronological order.

### Approach
1. **Initialization**: We start by reading the input values, which include the number of people (N), the number of noodle distribution events (M), and the details of each event (time T_i, noodles W_i, and return time S_i).

2. **Data Structures**: 
   - **Available People**: A min-heap to keep track of the indices of people currently available in the queue. Initially, all people (1 to N) are available.
   - **Return Events**: Another min-heap to manage the return events, where each event is a tuple (return_time, person_index). This heap helps in efficiently processing people who return to the queue at specific times.

3. **Processing Events**: For each noodle distribution event:
   - **Process Returns**: Before handling the current event, process all return events where the return time is less than or equal to the current event time. For each such event, the person is added back to the available people heap.
   - **Distribute Noodles**: If there are available people, the person with the smallest index (front of the queue) receives the noodles. This person is then scheduled to return at the current event time plus the given return interval (S_i).

4. **Output**: After processing all events, the total noodles received by each person is printed in order from person 1 to person N.

### Solution Code
```python
import heapq
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    m = int(data[1])
    events = []
    index = 2
    for _ in range(m):
        t = int(data[index])
        w = int(data[index+1])
        s = int(data[index+2])
        index += 3
        events.append((t, w, s))
    
    available = list(range(1, n+1))
    heapq.heapify(available)
    return_events = []
    ans = [0] * (n+1)

    for T, W, S in events:
        while return_events and return_events[0][0] <= T:
            time_return, person = heapq.heappop(return_events)
            heapq.heappush(available, person)
            
        if available:
            p = heapq.heappop(available)
            ans[p] += W
            heapq.heappush(return_events, (T + S, p))
            
    for i in range(1, n+1):
        print(ans[i])

if __name__ == "__main__":
    main()
```

### Explanation
- **Initialization**: The input is read and parsed into the number of people (N), the number of events (M), and the list of events.
- **Data Structures**: The `available` heap starts with all people (1 to N), and the `return_events` heap is initially empty.
- **Event Processing**: For each event, all return events scheduled to occur at or before the current event time are processed, moving people back to the available heap. If people are available, the smallest index person receives the noodles and is scheduled to return at the calculated return time.
- **Output**: After processing all events, the total noodles received by each person (from 1 to N) is printed.

This approach efficiently handles the dynamic availability of people in the queue using min-heaps, ensuring optimal performance even for large input sizes.
