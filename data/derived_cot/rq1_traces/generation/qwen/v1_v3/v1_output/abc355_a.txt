=== ANSWER ===

To solve this problem, we need to determine if the culprit can be uniquely identified based on the memories of two witnesses. The witnesses provide information about which persons are not the culprit. The solution involves checking if the two memories together eliminate all but one suspect.

### Approach
1. **Problem Analysis**: There are three suspects (persons 1, 2, and 3). Two witnesses each state that a specific person is not the culprit. The task is to determine if these statements uniquely identify the culprit.
2. **Key Insight**: If both witnesses mention the same person, then the culprit must be one of the other two suspects, making it impossible to uniquely identify the culprit. If the witnesses mention two different suspects, the remaining suspect is the culprit.
3. **Algorithm Selection**: 
   - Read the input values for the two memories (A and B).
   - If A and B are the same, output -1 because the culprit cannot be uniquely identified.
   - If A and B are different, the culprit is the one not mentioned by either witness. This can be found by subtracting the sets {A, B} from the set {1, 2, 3}.

### Solution Code
```python
def main():
    A, B = map(int, input().split())
    if A == B:
        print(-1)
    else:
        suspects = {1, 2, 3}
        eliminated = {A, B}
        culprit = (suspects - eliminated).pop()
        print(culprit)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The code reads two integers, A and B, which represent the suspects ruled out by the two witnesses.
2. **Check for Identical Memories**: If both witnesses rule out the same suspect (A == B), the culprit cannot be uniquely identified, so the output is -1.
3. **Identify Culprit**: If the witnesses rule out two different suspects, the remaining suspect is the culprit. This is found by taking the set difference between the set of all suspects {1, 2, 3} and the set of eliminated suspects {A, B}. The result is a set containing only the culprit, which is then printed.
This approach efficiently checks the conditions and leverages set operations to determine the solution concisely.
