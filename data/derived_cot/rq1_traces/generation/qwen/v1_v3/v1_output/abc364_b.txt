There is a grid with H rows and W columns. Let (i, j) denote the cell at the i-th row from the top and j-th column from the left.

Takahashi is currently at cell (S_i, S_j), and he will act according to the following rules for i = 1, 2, \ldots, |X| in order.

- If the i-th character of X is L, and the cell to the left of his current cell exists and is empty, he moves to the cell to the left. Otherwise, he stays in the current cell.
- If the i-th character of X is R, and the cell to the right of his current cell exists and is empty, he moves to the cell to the right. Otherwise, he stays in the current cell.
- If the i-th character of X is U, and the cell above his current cell exists and is empty, he moves to the cell above. Otherwise, he stays in the current cell.

Print the cell where he is after completing the series of actions.

C_{H, 1}C_{H, 2}\ldotsC_{H, W}
X

Output

Let (x, y) be the cell where Takahashi is after completing the series of actions. Print x and y, separated by a space.

- 1 \leq H, W \leq 50
- 1 \leq S_i \leq H

- H, W, S_i, S_j are integers.
- C_{i, j} is . or #.

=== ANSWER ===

To solve this problem, we need to simulate the movement of Takahashi on a grid according to a sequence of directional commands. The grid consists of empty cells (denoted by '.') and blocked cells (denoted by '#'). The movement is restricted to moving into empty cells only, and the movement must stay within the grid boundaries.

### Approach
1. **Initialization**: Read the grid dimensions (H rows and W columns), the starting position (S_i, S_j), the grid itself, and the sequence of movement commands (X).
2. **Convert Starting Position**: Convert the 1-indexed starting position to 0-indexed for easier array manipulation.
3. **Process Commands**: For each character in the command sequence:
   - **Left (L)**: Move left if the left cell exists and is empty.
   - **Right (R)**: Move right if the right cell exists and is empty.
   - **Up (U)**: Move up if the top cell exists and is empty.
   - **Down (D)**: Move down if the bottom cell exists and is empty.
   - If the movement condition is not met, the position remains unchanged.
4. **Output Result**: After processing all commands, convert the final 0-indexed position back to 1-indexed and print it.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    H, W = map(int, data[0].split())
    start_pos = data[1].split()
    S_i = int(start_pos[0])
    S_j = int(start_pos[1])
    
    grid = []
    for i in range(2, 2 + H):
        grid.append(data[i].strip())
    
    X = data[2 + H].strip()
    
    r = S_i - 1
    c = S_j - 1
    
    for move in X:
        new_r = r
        new_c = c
        if move == 'L':
            new_c = c - 1
            if new_c >= 0 and grid[new_r][new_c] == '.':
                c = new_c
        elif move == 'R':
            new_c = c + 1
            if new_c < W and grid[new_r][new_c] == '.':
                c = new_c
        elif move == 'U':
            new_r = r - 1
            if new_r >= 0 and grid[new_r][new_c] == '.':
                r = new_r
        elif move == 'D':
            new_r = r + 1
            if new_r < H and grid[new_r][new_c] == '.':
                r = new_r
                
    print(f"{r + 1} {c + 1}")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the grid dimensions, starting position, the grid itself, and the sequence of movement commands.
2. **Initial Position Setup**: The starting position is converted from 1-indexed to 0-indexed for easier manipulation within the grid.
3. **Command Processing**: For each command in the sequence, the code checks the direction and attempts to move Takahashi in that direction if the target cell is within bounds and is empty. If the movement is not possible, the position remains unchanged.
4. **Output**: After processing all commands, the final position is converted back to 1-indexed format and printed.

This approach efficiently simulates Takahashi's movement by carefully checking each command's validity and updating the position only when the movement conditions are satisfied. The solution ensures that all edge cases, such as moving out of bounds or into blocked cells, are handled correctly.
