To solve this problem, we need to compute the sum of the power of all non-empty groups of heroes, where the power of a group is defined as the square of the maximum element in the group multiplied by the minimum element. Given the constraints, a brute-force approach is infeasible, so we need an efficient method leveraging sorting and dynamic programming.

### Approach
1. **Sorting the Array**: First, we sort the array in increasing order. This allows us to efficiently determine the maximum and minimum elements in any subset by considering elements in a structured manner.
2. **Dynamic Programming Setup**: We use a dynamic programming array `dp` where `dp[i]` represents the sum of the minimum values of all non-empty subsets of the first `i` elements in the sorted array. The recurrence relation for `dp` is derived by considering each new element added to the set of previous elements.
3. **Contribution Calculation**: For each element in the sorted array, we calculate its contribution to the total power. The contribution is based on the element being the maximum in its group, and the sum of the minimum values of all subsets formed by including or excluding previous elements (which are smaller or equal).

```python
class Solution:
    def sumOfPower(self, nums: List[int]) -> int:
        mod = 10**9 + 7
        nums.sort()
        n = len(nums)
        dp = [0] * (n + 1)
        for i in range(1, n + 1):
            dp[i] = (2 * dp[i - 1] + nums[i - 1]) % mod
        
        ans = 0
        for i in range(n):
            term1 = (nums[i] * nums[i]) % mod
            term2 = (dp[i] + nums[i]) % mod
            ans = (ans + term1 * term2) % mod
        
        return ans
```

1. **Sorting**: The array is sorted to facilitate the dynamic programming approach, ensuring that when we consider an element as the maximum, all previous elements are less than or equal to it.
2. **Dynamic Programming Array**: The `dp` array is built such that `dp[i]` holds the sum of the minimum values of all non-empty subsets of the first `i` elements. This is computed using the recurrence relation `dp[i] = 2 * dp[i-1] + nums[i-1]`, which accounts for all subsets including the new element.
3. **Contribution Calculation**: For each element `nums[i]`, its contribution to the total power is calculated as `(nums[i]^2) * (dp[i] + nums[i])`. Here, `dp[i]` sums the minimum values of all non-empty subsets of elements before `nums[i]`, and `nums[i]` accounts for the subset containing only `nums[i]`.
4. **Modulo Operation**: Since the result can be very large, all operations are performed modulo \(10^9 + 7\) to keep numbers manageable and avoid overflow.

This approach efficiently computes the desired sum by leveraging sorting and dynamic programming, reducing the problem's complexity from exponential to linear, making it feasible for large input sizes.
