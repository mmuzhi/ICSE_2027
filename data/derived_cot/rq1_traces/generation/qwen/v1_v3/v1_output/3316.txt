To solve this problem, we need to compute the sum of the minimum absolute differences between any two elements in all subsequences of a given array that have a specified length. Given the constraints, a brute-force approach is infeasible, so we use a dynamic programming approach combined with critical point analysis to efficiently compute the result.

### Approach
1. **Sorting the Array**: The first step is to sort the array to facilitate the calculation of absolute differences and to ensure that the minimum difference in any subsequence is between consecutive elements in the sorted array.
2. **Critical Points Identification**: We identify critical points (values of `d`) where the count of valid subsequences changes. These points include all unique gaps between elements in the sorted array, their increments by 1, and the boundaries (0 and the maximum possible gap + 1).
3. **Dynamic Programming for F(d)**: For each critical point `d`, we use dynamic programming to count the number of subsequences of length `k` where all consecutive differences are at least `d`. This involves iterating through the array and using a DP table to keep track of valid subsequences.
4. **Summing Contributions**: Using the counts from the DP step, we compute the sum of contributions from each critical point interval. The contribution of each interval is derived from the difference in counts between consecutive critical points, weighted by the interval's sum of values.

### Solution Code
```python
class Solution:
    def sumOfPowers(self, nums: List[int], k: int) -> int:
        mod_val = 10**9 + 7
        nums.sort()
        n = len(nums)
        if k > n:
            return 0
        
        max_gap = nums[-1] - nums[0]
        C = set()
        C.add(0)
        C.add(max_gap + 1)
        for i in range(n):
            for j in range(i + 1, n):
                gap = nums[j] - nums[i]
                C.add(gap)
                C.add(gap + 1)
        C = sorted(C)
        
        F_vals = {}
        for d in C:
            dp = [1] * n
            if k == 1:
                F_vals[d] = n % mod_val
                continue
            for length in range(2, k + 1):
                new_dp = [0] * n
                for i in range(length - 1, n):
                    total = 0
                    for p in range(i):
                        if nums[i] - nums[p] >= d:
                            total = (total + dp[p]) % mod_val
                    new_dp[i] = total
                dp = new_dp
            F_vals[d] = sum(dp) % mod_val
        
        ans = 0
        m = len(C)
        for i in range(m - 1):
            low = C[i]
            high = C[i + 1]
            num_terms = high - low
            sum_d = (num_terms * (low + high - 1)) // 2
            F_current = F_vals[low]
            F_next = F_vals[high]
            term = sum_d * (F_current - F_next)
            ans = (ans + term) % mod_val
        
        ans %= mod_val
        if ans < 0:
            ans += mod_val
        return ans
```

### Explanation
1. **Sorting the Array**: The array is sorted to simplify the calculation of absolute differences and to ensure that the minimum difference in any subsequence is between consecutive elements.
2. **Critical Points Identification**: The set `C` is built to include all unique gaps between elements, their increments by 1, and the boundaries (0 and `max_gap + 1`). These points help in efficiently determining the ranges where the count of valid subsequences remains constant.
3. **Dynamic Programming for F(d)**: For each critical point `d`, a DP table is used to count the number of subsequences of length `k` where all consecutive differences are at least `d`. The DP table is updated iteratively for each subsequence length from 2 to `k`.
4. **Summing Contributions**: For each interval between consecutive critical points, the sum of values in the interval is multiplied by the difference in counts of valid subsequences between the endpoints. This gives the total contribution of the interval to the final result, which is accumulated modulo \(10^9 + 7\).

This approach efficiently narrows down the problem to manageable critical points and uses dynamic programming to count valid subsequences, ensuring optimal performance even for the upper constraint limits.
