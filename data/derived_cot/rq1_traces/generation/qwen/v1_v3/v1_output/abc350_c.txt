You are given a permutation A=(A_1,\ldots,A_N) of (1,2,\ldots,N).
Transform A into (1,2,\ldots,N) by performing the following operation between 0 and N-1 times, inclusive:

- Operation: Choose any pair of integers (i,j) such that 1\leq i < j \leq N. Swap the elements at the i-th and j-th positions of A.

It can be proved that under the given constraints, it is always possible to transform A into (1,2,\ldots,N).

Input

The input is given from Standard Input in the following format:
N
A_1 \ldots A_N

Output

Let K be the number of operations. Print K+1 lines.
The first line should contain K.
The (l+1)-th line (1\leq l \leq K) should contain the integers i and j chosen for the l-th operation, separated by a space.
Any output that satisfies the conditions in the problem statement will be considered correct.

- 2 \leq N \leq 2\times 10^5
- (A_1,\ldots,A_N) is a permutation of (1,2,\ldots,N).

To solve this problem, we need to transform a given permutation into the sorted sequence (1, 2, ..., N) using the minimum number of swaps. The solution involves identifying cycles in the permutation and then performing swaps to bring each element to its correct position.

### Approach
1. **Problem Analysis**: The task is to convert a permutation of integers from 1 to N into the sorted sequence using swaps. Each swap can be between any two elements. The key insight is that each cycle in the permutation requires (cycle length - 1) swaps to sort the elements within the cycle.
2. **Intuition**: By decomposing the permutation into cycles, we can determine the minimal number of swaps needed. For each cycle, we can fix the elements by performing (cycle length - 1) swaps, ensuring each element moves to its correct position.
3. **Algorithm Selection**: We use a greedy approach where we iterate through each element. If the current element is not in its correct position, we find the element that should be at this position, swap them, and update the positions of the involved elements. This process continues until all elements are in their correct positions.
4. **Complexity Analysis**: The algorithm runs in O(N) time, where N is the number of elements in the permutation. This is because each element is processed exactly once, and each swap operation is O(1). The space complexity is O(N) due to the storage required for the permutation and the position mapping array.

```python
    pos = [0] * n
    for idx in range(n):
        pos[arr[idx]] = idx
        
    swaps = []
    
    for i in range(n):
        if arr[i] != i:
            j = pos[i]
            x = arr[i]
            arr[i], arr[j] = arr[j], arr[i]
            pos[i] = i
            pos[x] = j
            swaps.append((i, j))
            
    print(len(swaps))
    for a, b in swaps:
        print(f"{a+1} {b+1}")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, which includes the permutation size N and the permutation itself.
2. **Initialization**: The permutation is converted to 0-indexed values for easier manipulation. A position array `pos` is created to map each value to its current index.
3. **Cycle Handling**: For each element in the permutation, if it is not in its correct position (i.e., `arr[i] != i`), the algorithm finds the element that should be at position `i` (using the `pos` array), swaps the two elements, and updates their positions in the `pos` array. Each swap is recorded.
4. **Output**: The total number of swaps is printed first, followed by each swap operation in 1-indexed format (since the problem requires 1-based indexing).

This approach efficiently sorts the permutation by leveraging cycle decomposition and ensures minimal swaps, providing a clear and optimal solution.
