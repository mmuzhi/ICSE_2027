You are given a 0-indexed array nums of size n consisting of positive integers.
You are also given a 2D array queries of size m where queries[i] = [index_i, k_i].
Initially all elements of the array are unmarked.
You need to apply m queries on the array in order, where on the i^th query you do the following:

Mark the element at index index_i if it is not already marked.
Then mark k_i unmarked elements in the array with the smallest values. If multiple such elements exist, mark the ones with the smallest indices. And if less than k_i unmarked elements exist, then mark all of them.

Return an array answer of size m where answer[i] is the sum of unmarked elements in the array after the i^th query.
 
Example 1:

Input: nums = [1,2,2,1,2,3,1], queries = [[1,2],[3,3],[4,2]]
Output: [8,3,0]
Explanation:

Example 2:

Input: nums = [1,4,2,3], queries = [[0,1]]
Output: [7]
Explanation:  We do one query which is mark the element at index 0 and mark the smallest element among unmarked elements. The marked elements will be nums = [1,4,2,3], and the sum of unmarked elements is 4 + 3 = 7.

To solve this problem, we need to efficiently handle a series of queries on an array where each query involves marking specific elements and then marking the smallest remaining unmarked elements. The challenge lies in processing these queries quickly, especially given the constraints where both the array size and the number of queries can be as large as 100,000.

### Approach
1. **Initial Setup**: Calculate the total sum of all elements in the array. This sum will be adjusted as elements are marked.
2. **Tracking Marked Elements**: Use a boolean array to keep track of which elements have been marked.
3. **Min-Heap for Efficient Selection**: Construct a min-heap containing tuples of (value, index) for all elements in the array. This heap allows us to efficiently retrieve the smallest unmarked elements during each query.
4. **Processing Queries**: For each query:
   - **Mark the Specified Element**: If the element at the given index is unmarked, mark it and subtract its value from the total sum.
   - **Mark Smallest Unmarked Elements**: Use the min-heap to find and mark the smallest `k` unmarked elements. For each element popped from the heap, if it is unmarked, mark it, subtract its value from the total sum, and decrement `k` until either `k` elements are marked or the heap is exhausted.
5. **Result Collection**: After processing each query, the current total sum of unmarked elements is recorded as part of the result.

This approach efficiently handles the queries by leveraging a min-heap to quickly access the smallest unmarked elements, ensuring optimal performance even for large input sizes.

### Solution Code
```python
import heapq

class Solution:
    def unmarkedSumArray(self, nums: List[int], queries: List[List[int]]) -> List[int]:
        n = len(nums)
        total = sum(nums)
        marked = [False] * n
        heap = [(nums[i], i) for i in range(n)]
        heapq.heapify(heap)
        
        res = []
        for idx, k in queries:
            if not marked[idx]:
                marked[idx] = True
                total -= nums[idx]
            while k > 0 and heap:
                val, i = heapq.heappop(heap)
                if marked[i]:
                    continue
                marked[i] = True
                total -= val
                k -= 1
            res.append(total)
        return res
```

### Explanation
1. **Initialization**: The total sum of the array is computed, and a boolean array `marked` is initialized to keep track of which elements have been marked. A min-heap is built from the array elements to facilitate efficient retrieval of the smallest unmarked elements.
2. **Query Processing**: For each query:
   - The specified index is checked and marked if not already marked, and its value is subtracted from the total sum.
   - The smallest `k` unmarked elements are then marked by repeatedly extracting the smallest element from the heap. If the extracted element is already marked, it is skipped; otherwise, it is marked, its value is subtracted from the total, and `k` is decremented.
3. **Result Handling**: After processing each query, the current total sum of unmarked elements is added to the result list, which is returned after all queries are processed.

This approach ensures that each query is processed in logarithmic time relative to the number of elements, making the solution efficient and scalable for large inputs.
