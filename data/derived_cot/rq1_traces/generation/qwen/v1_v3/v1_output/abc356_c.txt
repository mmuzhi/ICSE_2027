=== PROMPT ===
You will be given a competitive programming problem.
You have N keys numbered 1, 2, \dots, N.
Some of these are real keys, while the others are dummies.
There is a door, Door X, into which you can insert any number of keys. Door X will open if and only if at least K real keys are inserted.
You have conducted M tests on these keys. The i-th test went as follows:

- You inserted C_i keys A_{i,1}, A_{i,2}, \dots, A_{i,C_i} into Door X.
- The test result is represented by a single English letter R_i.
- R_i = o means that Door X opened in the i-th test.
- R_i = x means that Door X did not open in the i-th test.



There are 2^N possible combinations of which keys are real and which are dummies. Among these, find the number of combinations that do not contradict any of the test results.
It is possible that the given test results are incorrect and no combination satisfies the conditions. In such a case, report 0.

Input

The input is given from Standard Input in the following format:
N M K
C_1 A_{1,1} A_{1,2} \dots A_{1,C_1} R_1
C_2 A_{2,1} A_{2,2} \dots A_{2,C_2} R_2
\vdots
C_M A_{M,1} A_{M,2} \dots A_{M,C_M} R_M

Output

Print the answer as an integer.

Constraints


- N, M, K, C_i, and A_{i,j} are integers.
- 1 \le K \le N \le 15
- 1 \le M \le 100
- 1 \le C_i \le N
- 1 \le A_{i,j} \le N
- A_{i,j} \neq A_{i,k} if j \neq k.
- R_i is o or x.

Sample Input 1

3 2 2
3 1 2 3 o
2 2 3 x

Sample Output 1

2

To solve this problem, we need to determine the number of valid key combinations (each key can be either real or dummy) that satisfy all given test results. The door opens only if at least K real keys are inserted. Each test specifies a set of keys inserted and whether the door opened ('o') or not ('x'). 

### Approach
1. **Problem Analysis**: The problem involves checking all possible combinations of real and dummy keys (2^N possibilities) to see which combinations meet all test conditions. Each test requires that the number of real keys in the inserted set meets a specific condition (either ≥ K for 'o' or < K for 'x').

2. **Intuition**: Since N is small (up to 15), we can use a brute-force approach by iterating over all possible 2^N combinations. For each combination, we check against all M tests to see if the number of real keys in each test's inserted set meets the required condition.

3. **Optimization**: To efficiently count the number of real keys in the inserted set for each test, we use bitmask representation. Each key combination is represented as a bitmask where each bit indicates if the corresponding key is real (1) or dummy (0). For each test, we compute the intersection of the test's key set (also represented as a bitmask) with the combination's bitmask and count the number of set bits (real keys) in the intersection.

4. **Complexity Analysis**: The algorithm checks 2^N combinations, each against M tests. For each test, the bit count operation is O(1) with a lookup table, making the overall complexity O(2^N * M). Given N ≤ 15, 2^15 = 32768, and M ≤ 100, the total operations are around 3.2768 million, which is feasible.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data: 
        print(0)
        return
        
    first_line = data[0].split()
    N = int(first_line[0])
    M = int(first_line[1])
    K = int(first_line[2])
    
    tests = []
    index = 1
    for i in range(M):
        parts = data[index].split()
        index += 1
        c = int(parts[0])
        keys = list(map(int, parts[1:1+c]))
        res_char = parts[1+c]
        mask_val = 0
        for key in keys:
            mask_val |= (1 << (key-1))
        tests.append((mask_val, res_char))
    
    max_mask = 1 << N
    popcount_arr = [0] * max_mask
    for i in range(max_mask):
        popcount_arr[i] = bin(i).count("1")
        
    ans = 0
    for mask in range(max_mask):
        valid = True
        for test_mask, res_char in tests:
            count = popcount_arr[mask & test_mask]
            if res_char == 'o':
                if count < K:
                    valid = False
                    break
            else:
                if count >= K:
                    valid = False
                    break
        if valid:
            ans += 1
            
    print(ans)

if __name__ == "__main__":
    main()
```

2. **Bitmask Representation**: Each test's keys are converted into a bitmask where each bit represents a key (1 if the key is in the test, 0 otherwise). This allows efficient computation of the number of real keys in the test set for any combination.

3. **Popcount Lookup Table**: A lookup table (`popcount_arr`) is precomputed to store the number of set bits (real keys) for all possible bitmasks (0 to 2^N - 1). This table enables quick bit counting during the combination checks.

4. **Combination Check**: For each possible combination (represented as a bitmask from 0 to 2^N - 1), the algorithm checks if the number of real keys in each test's inserted set meets the required condition (≥ K for 'o', < K for 'x'). If all tests pass, the combination is counted as valid.

5. **Output**: The total number of valid combinations is printed, which is the solution to the problem.
