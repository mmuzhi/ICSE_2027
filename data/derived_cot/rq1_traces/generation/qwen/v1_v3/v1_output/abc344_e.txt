=== PROMPT ===
You will be given a competitive programming problem.
You are given a sequence A=(A_1,\ldots,A_N) of length N. The elements of A are distinct.
Process Q queries in the order they are given. Each query is of one of the following two types:

- 1 x y : Insert y immediately after the element x in A. It is guaranteed that x exists in A when this query is given.
- 2 x : Remove the element x from A. It is guaranteed that x exists in A when this query is given.

It is guaranteed that after processing each query, A will not be empty, and its elements will be distinct.
Print A after processing all the queries.

Input

The input is given from Standard Input in the following format:
N 
A_1 \ldots A_N
Q
\mathrm{Query}_1
\vdots 
\mathrm{Query}_Q

Here, \mathrm{Query}_i represents the i-th query and is given in one of the following formats:
1 x y

2 x

Output

Let A=(A_1,\ldots,A_K) be the sequence after processing all the queries. Print A_1,\ldots,A_K in this order, separated by spaces.

Constraints


- 1 \leq N \leq 2\times 10^5 
- 1 \leq Q \leq 2\times 10^5
- 1 \leq A_i \leq 10^9
- A_i \neq A_j 
- For queries of the first type, 1 \leq x,y \leq 10^9.
- When a query of the first type is given, x exists in A.
- For queries of the second type, 1 \leq x \leq 10^9.
- When a query of the second type is given, x exists in A.
- After processing each query, A is not empty, and its elements are distinct.
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to efficiently handle a series of insertions and deletions in a sequence of distinct elements while maintaining the order of elements. The challenge lies in performing these operations in constant time per query, which is crucial given the constraints where both the initial sequence length and the number of queries can be as large as 200,000.

### Approach
1. **Data Structures**: We use a doubly linked list to allow efficient insertions and deletions. Each node in the linked list contains a value, a pointer to the previous node, and a pointer to the next node. Additionally, we maintain a dictionary (hash map) to quickly locate any node by its value.
2. **Initialization**: The initial sequence is processed to build the linked list, with the first element as the head and the last element as the tail. Each element is stored in the dictionary for O(1) access.
3. **Query Processing**:
   - **Insertion (Type 1)**: For each insertion query, we locate the node containing the specified element `x`, then insert the new element `y` immediately after `x` in the linked list. The dictionary is updated to include the new element.
   - **Deletion (Type 2)**: For each deletion query, we locate the node containing the specified element `x`, remove it from the linked list by updating the previous and next nodes' pointers, and remove the element from the dictionary. If the removed node was the head or tail, we update the head or tail pointers accordingly.
4. **Output**: After processing all queries, the final sequence is reconstructed by traversing the linked list from the head to the tail, collecting each element's value.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    idx = 0
    n = int(data[idx]); idx += 1
    initial_list = list(map(int, data[idx:idx+n])); idx += n
    q = int(data[idx]); idx += 1
    
    head = None
    tail = None
    mapping = {}
    
    for num in initial_list:
        node = {'val': num, 'prev': None, 'next': None}
        mapping[num] = node
        if head is None:
            head = node
            tail = node
        else:
            tail['next'] = node
            node['prev'] = tail
            tail = node

    for _ in range(q):
        t = data[idx]; idx += 1
        if t == '1':
            x = int(data[idx]); idx += 1
            y = int(data[idx]); idx += 1
            node_x = mapping[x]
            next_node = node_x['next']
            new_node = {'val': y, 'prev': node_x, 'next': next_node}
            mapping[y] = new_node
            node_x['next'] = new_node
            if next_node is not None:
                next_node['prev'] = new_node
            else:
                tail = new_node
        elif t == '2':
            x = int(data[idx]); idx += 1
            node = mapping[x]
            prev_node = node['prev']
            next_node = node['next']
            if prev_node is not None:
                prev_node['next'] = next_node
            else:
                head = next_node
            if next_node is not None:
                next_node['prev'] = prev_node
            else:
                tail = prev_node
            del mapping[x]
    
    current = head
    result = []
    while current is not None:
        result.append(str(current['val']))
        current = current['next']
    
    print(" ".join(result))

if __name__ == "__main__":
    main()
```

### Explanation
- **Initialization**: The initial sequence is read and stored in a doubly linked list. The head points to the first element, and the tail points to the last element. Each element is also stored in a dictionary for quick lookup.
- **Insertion (Type 1)**: When inserting an element `y` after `x`, the new node is created and linked between `x` and its next node (if any). The dictionary is updated to include the new element.
- **Deletion (Type 2)**: When removing an element `x`, the node is removed from the linked list by updating the pointers of its previous and next nodes. If `x` was the head or tail, these pointers are adjusted accordingly. The element is removed from the dictionary.
- **Output**: The final sequence is obtained by traversing the linked list from head to tail, collecting each element's value, and printing them in order.

This approach efficiently handles each query in constant time, leveraging the doubly linked list for O(1) insertions and deletions and the dictionary for O(1) node access, making it suitable for large input sizes.
