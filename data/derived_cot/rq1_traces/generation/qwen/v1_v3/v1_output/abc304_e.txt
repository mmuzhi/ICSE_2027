To solve this problem, we need to determine if adding a new edge to an undirected graph maintains the graph's "good" property. A graph is considered good if there are no paths between certain specified vertex pairs. 

### Approach
1. **Graph Representation and Connectivity**: We represent the graph using a Disjoint Set Union (DSU) data structure to efficiently manage and query the connected components of the graph. The DSU helps in determining if two vertices are in the same component and merging components when edges are added.
2. **Initial Conditions Processing**: For each condition pair (x_i, y_i), we store the pair of their root components in a set. This set helps in quickly checking if adding an edge between two vertices would connect components that are part of any condition pair.
3. **Query Handling**: For each query, we check if the vertices of the new edge are in the same component. If they are, adding the edge doesn't affect the graph's goodness. If they are in different components, we check if merging these components would connect any condition pair. If it does, the graph is no longer good; otherwise, it remains good.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    it = iter(data)
    
    n = int(next(it))
    m = int(next(it))
    
    parent = [i for i in range(n+1)]
    rank = [0] * (n+1)
    
    def find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]
    
    def union(x, y):
        rx = find(x)
        ry = find(y)
        if rx == ry:
            return
        if rank[rx] < rank[ry]:
            parent[rx] = ry
        elif rank[rx] > rank[ry]:
            parent[ry] = rx
        else:
            parent[ry] = rx
            rank[rx] += 1
    
    for _ in range(m):
        u = int(next(it))
        v = int(next(it))
        if u == v:
            continue
        union(u, v)
    
    comp = [0] * (n+1)
    for i in range(1, n+1):
        comp[i] = find(i)
    
    k = int(next(it))
    conditions_set = set()
    for _ in range(k):
        x = int(next(it))
        y = int(next(it))
        cx = comp[x]
        cy = comp[y]
        conditions_set.add((min(cx, cy), max(cx, cy)))
    
    q = int(next(it))
    out_lines = []
    for _ in range(q):
        p = int(next(it))
        q_val = int(next(it))
        cp = comp[p]
        cq = comp[q_val]
        if cp == cq:
            out_lines.append("Yes")
        else:
            key = (min(cp, cq), max(cp, cq))
            if key in conditions_set:
                out_lines.append("No")
            else:
                out_lines.append("Yes")
    
    print("\n".join(out_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Graph Representation**: The graph is built using the DSU structure, which efficiently handles dynamic connectivity queries and updates. The `find` function uses path compression to optimize future queries, and the `union` function uses union by rank to keep the tree flat.
2. **Conditions Processing**: For each condition pair (x_i, y_i), we determine their root components and store the sorted pair of these roots in a set. This set allows O(1) lookups during query processing.
3. **Query Handling**: For each query, we check if the vertices of the new edge are in the same component. If they are, the graph remains good. If not, we check if merging their components would connect any condition pair by verifying if the sorted component pair exists in the conditions set. If it does, the graph is no longer good; otherwise, it remains good.

This approach efficiently checks the impact of each potential edge addition on the graph's goodness by leveraging DSU for connectivity and a set for quick condition checks. The overall complexity is nearly linear, making it suitable for large input sizes.
