=== PROMPT ===
You will be given a competitive programming problem.
There is an undirected graph with N vertices numbered 1 through N, and initially with 0 edges.
Given Q queries, process them in order.  After processing each query,
print the number of vertices that are not connected to any other vertices by an edge.
The i-th query, \mathrm{query}_i, is of one of the following two kinds.

- 
1 u v: connect vertex u and vertex v with an edge.  It is guaranteed that, when this query is given, vertex u and vertex v are not connected by an edge.

- 
2 v: remove all edges that connect vertex v and the other vertices.  (Vertex v itself is not removed.)

Input

The input is given from Standard Input in the following format:
N Q
\mathrm{query}_1
\mathrm{query}_2
\vdots
\mathrm{query}_Q

Output

Print Q lines.
The i-th line (1\leq i\leq Q) should contain the number of vertices that are not connected to any other vertices by an edge.

Constraints


- 2 \leq N\leq 3\times 10^5
- 1 \leq Q\leq 3\times 10^5
- For each query of the first kind, 1\leq u,v\leq N and u\neq v.
- For each query of the second kind, 1\leq v\leq N.
- Right before a query of the first kind is given, there is no edge between vertices u and v.
- All values in the input are integers.

3 7
1 1 2
1 1 3
1 2 3
2 1
1 1 2
2 2
1 1 2

Sample Output 1

1
0
0
1

1

Thus, 1 should be printed in the first line.
After the third query, all pairs of different vertices are connected by an edge.
However, the fourth query asks to remove all edges that connect vertex 1 and the other vertices, specifically to remove the edge between vertex 1 and vertex 2, and another between vertex 1 and vertex 3.
As a result, vertex 2 and vertex 3 are connected to each other, while vertex 1 is not connected to any other vertices by an edge.
Thus, 0 and 1 should be printed in the third and fourth lines, respectively.

To solve this problem, we need to process a series of queries on an initially empty undirected graph and, after each query, output the number of vertices that are isolated (i.e., not connected to any other vertices by an edge). The queries can either add an edge between two vertices or remove all edges incident to a specified vertex.

### Approach
1. **Initialization**: Start with all vertices isolated (degree 0). Maintain a count of isolated vertices, an array to track the degree of each vertex, and an adjacency list to represent the graph.
2. **Processing Queries**:
   - **Type 1 (Add Edge)**: For each edge addition between vertices `u` and `v`, decrement the isolated count if either vertex was previously isolated. Then, update their degrees and the adjacency list.
   - **Type 2 (Remove Edges)**: For the specified vertex `v`, if it has edges, iterate over all its neighbors, remove `v` from their adjacency lists, and update their degrees. If any neighbor's degree drops to zero, increment the isolated count. Finally, reset `v`'s adjacency list and degree to zero, and increment the isolated count since `v` becomes isolated.
3. **Output**: After processing each query, the current count of isolated vertices is recorded and printed.

### Solution Code

import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    q = int(data[1])
    index = 2
    
    isolated_count = n
    degree = [0] * (n + 1)
    graph = [set() for _ in range(n + 1)]
    
    output_lines = []
    
    for _ in range(q):
        t = int(data[index]); index += 1
        if t == 1:
            u = int(data[index]); index += 1
            v = int(data[index]); index += 1
            if degree[u] == 0:
                isolated_count -= 1
            if degree[v] == 0:
                isolated_count -= 1
            graph[u].add(v)
            graph[v].add(u)
            degree[u] += 1
            degree[v] += 1

            v_val = int(data[index]); index += 1
            if degree[v_val] == 0:

                    graph[u].discard(v_val)

                graph[v_val] = set()

- **Initialization**: The graph starts with `n` vertices, all isolated. The `isolated_count` is initialized to `n`, and the `degree` array tracks the number of edges per vertex. The `graph` is an adjacency list where each vertex points to a set of its neighbors.
