You will be given a competitive programming problem.
N people, person 1, person 2, \ldots, person N, are playing roulette.
The outcome of a spin is one of the 37 integers from 0 to 36.
For each i = 1, 2, \ldots, N, person i has bet on C_i of the 37 possible outcomes: A_{i, 1}, A_{i, 2}, \ldots, A_{i, C_i}.
The wheel has been spun, and the outcome is X.
Print the numbers of all people who have bet on X with the fewest bets, in ascending order.
More formally, print all integers i between 1 and N, inclusive, that satisfy both of the following conditions, in ascending order:

- Person i has bet on X.
- For each j = 1, 2, \ldots, N, if person j has bet on X, then C_i \leq C_j.

Note that there may be no number to print (see Sample Input 2).

Input

The input is given from Standard Input in the following format:
N
C_1
A_{1, 1} A_{1, 2} \ldots A_{1, C_1}
C_2
A_{2, 1} A_{2, 2} \ldots A_{2, C_2}
\vdots
C_N
A_{N, 1} A_{N, 2} \ldots A_{N, C_N}
X

Output

Let B_1, B_2, \ldots, B_K be the sequence of numbers to be printed in ascending order.
Using the following format, print the count of numbers to be printed, K, on the first line,
and B_1, B_2, \ldots, B_K separated by spaces on the second line:
K
B_1 B_2 \ldots B_K

Constraints


- 1 \leq N \leq 100
- 1 \leq C_i \leq 37
- 0 \leq A_{i, j} \leq 36
- A_{i, 1}, A_{i, 2}, \ldots, A_{i, C_i} are all different for each i = 1, 2, \ldots, N.
- 0 \leq X \leq 36
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to determine which people have bet on the outcome of a roulette spin with the fewest number of bets, given that multiple people can bet on different outcomes. The solution involves reading the input data, processing it to identify relevant people, and then outputting the results in the specified format.

### Approach
1. **Read Input Data**: The first line contains the number of people, followed by each person's bet count and their specific bets. The last line is the outcome of the roulette spin.
2. **Process Each Person's Bets**: For each person, store their bet count and the list of outcomes they bet on.
3. **Identify Relevant People**: Check which people have bet on the outcome of the roulette spin.
4. **Determine Minimum Bet Count**: Among the relevant people, find the smallest number of bets.
5. **Collect People with Minimum Bets**: Gather all people who have bet on the outcome with the minimum number of bets.
6. **Output Results**: If no one bet on the outcome, output 0. Otherwise, output the count of such people followed by their indices in ascending order.

### Solution Code
```python
def main():
    n = int(input().strip())
    persons = []
    for i in range(n):
        c = int(input().strip())
        bets = list(map(int, input().split()))
        persons.append((i + 1, c, bets))
    
    x = int(input().strip())
    
    candidates = []
    for idx, c, bets in persons:
        if x in bets:
            candidates.append((idx, c))
            
    if not candidates:
        print(0)
    else:
        min_bet = min(c for idx, c in candidates)
        selected_indices = [idx for idx, c in candidates if c == min_bet]
        selected_indices.sort()
        print(len(selected_indices))
        print(' '.join(map(str, selected_indices)))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The code starts by reading the number of people, `n`. For each person, it reads their bet count and the specific outcomes they bet on, storing this information along with their index (1-based) in a list called `persons`.
2. **Processing the Outcome**: The outcome of the roulette spin, `x`, is read next.
3. **Finding Relevant People**: The code checks each person's bets to see if they include the outcome `x`. Those who do are added to a list of candidates along with their bet count.
4. **Determining Minimum Bets**: If there are no candidates, the code outputs 0. Otherwise, it finds the smallest bet count among the candidates.
5. **Collecting and Sorting Indices**: All candidates with the minimum bet count are collected, and their indices are sorted in ascending order.
6. **Output**: The code prints the count of these indices followed by the indices themselves, each separated by a space. If no candidates exist, only the count (0) is printed.

This approach efficiently processes the input data, identifies the relevant people, and outputs the results in the required format, handling edge cases such as no relevant people gracefully.
