You will be given a competitive programming problem.
We have a grid with H horizontal rows and W vertical columns.
We denote by (i,j) the cell at the i-th row from the top and j-th column from the left.
Each cell in the grid has a lowercase English letter written on it.  The letter written on (i,j) equals the j-th character of a given string S_i.
Snuke will repeat moving to an adjacent cell sharing a side to travel from (1,1) to (H,W).
Determine if there is a path
in which the letters written on the visited cells (including initial (1,1) and final (H,W)) are
s \rightarrow n \rightarrow u \rightarrow k
\rightarrow e \rightarrow s \rightarrow n \rightarrow \dots, in the order of visiting.
Here, a cell (i_1,j_1) is said to be an adjacent cell of (i_2,j_2) sharing a side if and only if |i_1-i_2|+|j_1-j_2| = 1.
Formally, determine if there is a sequence of cells ((i_1,j_1),(i_2,j_2),\dots,(i_k,j_k)) such that:

- (i_1,j_1) = (1,1),(i_k,j_k) = (H,W);
- (i_{t+1},j_{t+1}) is an adjacent cell of (i_t,j_t) sharing a side, for all t\ (1 \leq t < k); and
- the letter written on (i_t,j_t) coincides with the (((t-1) \bmod 5) + 1)-th character of snuke, for all t\ (1 \leq t \leq k).

Input

The input is given from Standard Input in the following format:
H W
S_1
S_2
\vdots
S_H

Output

Print Yes if there is a path satisfying the conditions in the problem statement; print No otherwise.

Constraints


- 2\leq H,W \leq 500
- H and W are integers.
- S_i is a string of length W consisting of lowercase English letters.

To solve this problem, we need to determine if there exists a path from the top-left corner (1,1) to the bottom-right corner (H,W) of a given grid, such that the letters on the visited cells form a repeating sequence "snuke" in order. The path must move only to adjacent cells (up, down, left, or right) and must start with 's' and follow the sequence cyclically.

### Approach
1. **Initial Check**: The starting cell (0,0 in 0-based indexing) must contain the letter 's'. If not, the answer is immediately "No".
2. **Breadth-First Search (BFS)**: We use BFS to explore all possible paths from the starting cell. Each state in the BFS is represented by the current cell coordinates (i, j) and the current step count modulo 5 (since the sequence "snuke" repeats every 5 letters). This helps in determining the expected next letter in the sequence.
3. **State Tracking**: For each cell, we keep track of whether we have visited it with a specific step count modulo 5 to avoid redundant checks and cycles.
4. **Termination Condition**: During BFS, if we reach the destination cell (H-1, W-1) with a step count such that the next letter in the sequence matches the cell's letter, we immediately return "Yes". If the BFS completes without finding such a path, we return "No".

### Solution Code
```python
from collections import deque

def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        print("No")
        return
    
    H, W = map(int, data[0].split())
    grid = []
    for i in range(1, H+1):
        grid.append(data[i].strip())
    
    if grid[0][0] != 's':
        print("No")
        return
        
    if H == 1 and W == 1:
        print("Yes")
        return
        
    visited = [[[False] * 5 for _ in range(W)] for __ in range(H)]
    directions = [(1,0), (-1,0), (0,1), (0,-1)]
    q = deque()
    visited[0][0][0] = True
    q.append((0,0,0))
    
    while q:
        i, j, r = q.popleft()
        for dx, dy in directions:
            ni, nj = i + dx, j + dy
            if 0 <= ni < H and 0 <= nj < W:
                next_r = (r + 1) % 5
                if grid[ni][nj] == "snuke"[next_r]:
                    if ni == H-1 and nj == W-1:
                        print("Yes")
                        return
                    if not visited[ni][nj][next_r]:
                        visited[ni][nj][next_r] = True
                        q.append((ni, nj, next_r))
                        
    print("No")

if __name__ == "__main__":
    main()
```

2. **BFS Initialization**: The BFS starts from (0,0) with step count 0 (mod 5). The visited array tracks which states (i, j, step_mod) have been processed to avoid revisiting.
3. **BFS Execution**: For each cell, the code checks its adjacent cells. If the adjacent cell's letter matches the next expected letter in the sequence (determined by `(r + 1) % 5`), the state is enqueued for further exploration.
4. **Termination**: If during BFS the destination cell (H-1, W-1) is reached with a matching sequence letter, "Yes" is printed. If the BFS completes without finding the destination, "No" is printed.

This approach efficiently explores all possible paths while leveraging modulo arithmetic to manage the cyclic nature of the sequence, ensuring optimal performance even for large grids.
