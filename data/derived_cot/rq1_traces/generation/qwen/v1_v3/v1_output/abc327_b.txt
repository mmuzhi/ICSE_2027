To solve this problem, we need to determine if there exists a positive integer \( A \) such that \( A^A \) equals a given integer \( B \). If such an \( A \) exists, we should output it; otherwise, we should output -1. 

### Approach
1. **Problem Analysis**: The problem requires checking if \( B \) can be expressed as \( A^A \) for some positive integer \( A \). Given the constraints \( 1 \leq B \leq 10^{18} \), we note that \( A \) can range from 1 to 15 because \( 15^{15} \) is approximately \( 4.37 \times 10^{17} \) (which is less than \( 10^{18} \)), and \( 16^{16} \) is approximately \( 1.84 \times 10^{19} \) (which exceeds \( 10^{18} \)). Thus, we only need to check values of \( A \) from 1 to 15.

2. **Algorithm Selection**: We iterate over each integer \( A \) from 1 to 15. For each \( A \), we compute \( A^A \) and compare it with \( B \). If they match, we immediately return \( A \). If during the iteration \( A^A \) exceeds \( B \), we break out of the loop early since subsequent values of \( A \) will only produce larger results due to the strictly increasing nature of \( A^A \).

3. **Complexity Analysis**: The loop runs at most 15 times, and each iteration involves a single exponentiation operation. Since 15 is a constant, the time complexity is \( O(1) \). The space complexity is also \( O(1) \) as we only use a few variables.

### Solution Code
```python
B = int(input().strip())
ans = -1
for A in range(1, 16):
    power = A ** A
    if power == B:
        ans = A
        break
    if power > B:
        break
print(ans)
```

### Explanation
1. **Reading Input**: The input integer \( B \) is read from standard input.
2. **Initialization**: The variable `ans` is initialized to -1 to indicate no solution has been found yet.
3. **Loop Through Possible Values of \( A \)**: The loop runs from \( A = 1 \) to \( A = 15 \). For each \( A \), it calculates \( A^A \).
4. **Check for Match**: If \( A^A \) equals \( B \), `ans` is set to \( A \), and the loop breaks early.
5. **Early Termination**: If \( A^A \) exceeds \( B \), the loop breaks early since further values of \( A \) will only produce larger results.
6. **Output Result**: After the loop, the value of `ans` is printed. If no match was found, `ans` remains -1; otherwise, it holds the valid \( A \).

This approach efficiently checks all possible values of \( A \) within the constrained range, leveraging early termination to optimize performance when \( B \) is small.
