The AtCoder Archipelago consists of N islands connected by N bridges.
The islands are numbered from 1 to N, and the i-th bridge (1\leq i\leq N-1) connects islands i and i+1 bidirectionally, while the N-th bridge connects islands N and 1 bidirectionally.
There is no way to travel between islands other than crossing the bridges.
On the islands, a tour that starts from island X_1 and visits islands X_2, X_3, \dots, X_M in order is regularly conducted.
The tour may pass through islands other than those being visited, and the total number of times bridges are crossed during the tour is defined as the length of the tour.
More precisely, a tour is a sequence of l+1 islands a_0, a_1, \dots, a_l that satisfies all the following conditions, and its length is defined as l:

- For all j\ (0\leq j\leq l-1), islands a_j and a_{j+1} are directly connected by a bridge.
- There are some 0 = y_1 < y_2 < \dots < y_M = l such that for all k\ (1\leq k\leq M), a_{y_k} = X_k.

Due to financial difficulties, the islands will close one bridge to reduce maintenance costs.
Determine the minimum possible length of the tour when the bridge to be closed is chosen optimally.

Input

The input is given from Standard Input in the following format:
N M
X_1 X_2 \dots X_M

Output

Print the answer as an integer.

Constraints


- 3\leq N \leq 2\times 10^5
- 2\leq M \leq 2\times 10^5
- 1\leq X_k\leq N
- X_k\neq X_{k+1}\ (1\leq k\leq M-1)
- All input values are integers.

Sample Input 1

1 3 2

Sample Output 1

2

To solve this problem, we need to determine the minimum possible length of a tour that visits a sequence of islands in a circular archipelago after optimally closing one bridge. The tour's length is defined as the number of bridges crossed. The solution involves calculating the minimal path lengths between consecutive islands in the sequence and efficiently determining the impact of closing each bridge on the total tour length.

### Approach
1. **Problem Analysis**: The archipelago is a circular graph with N islands connected by N bridges. The tour is a sequence of M islands. Closing any bridge breaks the cycle into a linear path, potentially increasing the tour length if the minimal path between some consecutive islands in the sequence uses the closed bridge.

2. **Key Insight**: The minimal path between two islands in a circular graph is the shorter of the two possible directions (clockwise or counter-clockwise). When a bridge is closed, the minimal path for any consecutive pair that used this bridge must switch to the other direction, increasing the tour length by the difference between the two path lengths.

3. **Algorithm Selection**:
   - **Compute Base Length (D)**: Calculate the total tour length without closing any bridge by summing the minimal path lengths between consecutive islands.
   - **Difference Array for Efficient Updates**: Use a difference array to efficiently compute the extra length added for each bridge closure. For each consecutive island pair, determine the bridges used in their minimal path and update the difference array accordingly.
   - **Prefix Sum Calculation**: After processing all pairs, compute the prefix sums of the difference array to get the total extra length for each bridge closure. The minimal tour length is then D plus the minimal extra length across all bridges.

4. **Complexity Analysis**: The algorithm processes each of the M-1 consecutive pairs once, performing constant-time operations per pair. The difference array updates and prefix sum calculation are linear in N. Thus, the overall complexity is O(N + M), which is efficient for the given constraints.

import sys

def main():
