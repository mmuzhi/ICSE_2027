class Solution:
    def maximumSum(self, nums: List[int]) -> int:
        n = len(nums)
        # Dictionary to store the sum for each square-free part
        group_sums = {}
        # Function to compute the square-free part of a number
        def square_free(x):
            res = x
            i = 2
            # Loop while i*i <= res
            while i * i <= res:
                # Remove squares of i from res
                while res % (i * i) == 0:
                    res //= (i * i)
                i += 1
            return res
        
        # Iterate over indices 1 to n
        for i in range(1, n+1):
            # Get the square-free part of index i
            s = square_free(i)
            # Add the corresponding array value
            if s not in group_sums:
                group_sums[s] = 0
            group_sums[s] += nums[i-1]  # because nums is 0-indexed, index i is at nums[i-1]
        
        # Return the maximum sum from the groups
        return max(group_sums.values())

=== ANSWER ===

To solve this problem, we need to find the maximum element-sum of a complete subset of indices from a given 1-indexed array. A subset of indices is considered complete if the product of every pair of indices in the subset is a perfect square. The element-sum is the sum of the array values at these indices.

### Approach
1. **Problem Analysis**: The key observation is that for any two indices \(i\) and \(j\) in a complete subset, their product \(i \times j\) must be a perfect square. This condition holds if and only if the square-free parts of \(i\) and \(j\) are identical. The square-free part of a number is obtained by removing all square factors from its prime factorization. For example, the square-free part of 8 (which is \(2^3\)) is 2, and the square-free part of 9 (which is \(3^2\)) is 1.

2. **Intuition**: Indices with the same square-free part can form a complete subset because the product of any two such indices will be a perfect square. Therefore, the problem reduces to grouping indices by their square-free parts and then finding the group with the maximum sum of corresponding array values.

3. **Algorithm Selection**: 
   - **Square-Free Calculation**: For each index from 1 to \(n\), compute its square-free part by repeatedly dividing out square factors.
   - **Grouping Indices**: Use a dictionary to accumulate the sum of array values for each square-free part.
   - **Maximization**: The solution is the maximum sum found in the dictionary values, which corresponds to the largest complete subset's element-sum.

4. **Complexity Analysis**: The algorithm processes each index once, and for each index, it computes its square-free part in \(O(\sqrt{i})\) time in the worst case (when \(i\) is a prime number). Given \(n\) can be up to \(10^4\), the total operations are around \(10^4 \times 100 = 10^6\), which is efficient for Python.

### Solution Code
```python
class Solution:
    def maximumSum(self, nums: List[int]) -> int:
        n = len(nums)
        group_sums = {}
        def square_free(x):
            res = x
            i = 2
            while i * i <= res:
                while res % (i * i) == 0:
                    res //= (i * i)
                i += 1
            return res
        
        for i in range(1, n + 1):
            s = square_free(i)
            group_sums[s] = group_sums.get(s, 0) + nums[i - 1]
        
        return max(group_sums.values())
```

### Explanation
1. **Square-Free Calculation**: The helper function `square_free(x)` computes the square-free part of \(x\) by dividing out all square factors. For example, for \(x = 8\), it divides by \(2^2\) (since \(8 = 2^3\)) to get the square-free part 2.
2. **Grouping Indices**: The main loop iterates over each index from 1 to \(n\). For each index, it calculates its square-free part and accumulates the corresponding array value in a dictionary where keys are square-free parts.
3. **Result Extraction**: The maximum value in the dictionary, which represents the largest sum of array values for any complete subset, is returned as the result.

This approach efficiently groups indices by their square-free parts and leverages the properties of perfect squares to determine complete subsets, ensuring optimal performance even for the upper constraint limits.
