To solve this problem, we need to process a list of words where each word is either a positive integer string or the string "prev". The goal is to generate an output array where each element corresponds to a "prev" word encountered in the input list. For each "prev" word, the output should be the last visited integer from the history of integers seen so far, based on the number of consecutive "prev" words encountered up to that point.

### Approach
1. **Initialization**: We maintain a list `nums` to keep track of all integers encountered in the input list and a variable `consecutive_count` to count the current streak of consecutive "prev" words.
2. **Iteration**: For each word in the input list:
   - If the word is "prev", increment `consecutive_count` and check if the count is within the bounds of the `nums` list. If it is, the result for this "prev" is the element at the position `-consecutive_count` in `nums` (using negative indexing to access elements from the end of the list). If the count exceeds the list length, the result is -1.
   - If the word is an integer string, append its integer value to `nums` and reset `consecutive_count` to 0, as the streak of "prev" words is broken.
3. **Output**: After processing all words, return the output array containing the results for each "prev" encountered.

### Solution Code
```python
class Solution:
    def lastVisitedIntegers(self, words: List[str]) -> List[int]:
        nums = []
        consecutive_count = 0
        output = []
        for word in words:
            if word == "prev":
                consecutive_count += 1
                if consecutive_count <= len(nums):
                    output.append(nums[-consecutive_count])
                else:
                    output.append(-1)
            else:
                nums.append(int(word))
                consecutive_count = 0
        return output
```

### Explanation
- **Initialization**: The `nums` list stores all integers from the input words, and `consecutive_count` tracks the current sequence of "prev" words.
- **Processing Words**: For each word:
  - **"prev" Handling**: The `consecutive_count` is incremented. If the count is valid (i.e., it does not exceed the length of `nums`), the corresponding integer from the reversed `nums` list (accessed via negative indexing) is added to the output. Otherwise, -1 is added.
  - **Integer Handling**: Non-"prev" words are converted to integers and appended to `nums`, resetting `consecutive_count` to 0 since a new sequence of "prev" words starts here.
- **Output**: The output array is built during iteration and returned after processing all words, containing the results for each "prev" encountered.

This approach efficiently processes each word in linear time, maintaining the necessary state to compute the output dynamically. The use of negative indexing simplifies accessing elements from the end of the `nums` list, ensuring optimal performance.
