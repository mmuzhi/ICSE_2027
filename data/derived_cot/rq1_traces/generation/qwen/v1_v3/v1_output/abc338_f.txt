You will be given a competitive programming problem.
There is a weighted simple directed graph with N vertices and M edges.
The vertices are numbered 1 to N, and the i-th edge has a weight of W_i and extends from vertex U_i to vertex V_i.
The weights can be negative, but the graph does not contain negative cycles.
Determine whether there is a walk that visits each vertex at least once. If such a walk exists, find the minimum total weight of the edges traversed.
If the same edge is traversed multiple times, the weight of that edge is added for each traversal.
Here, "a walk that visits each vertex at least once" is a sequence of vertices v_1,v_2,\dots,v_k that satisfies both of the following conditions:

- For every i (1\leq i\leq k-1), there is an edge extending from vertex v_i to vertex v_{i+1}.
- For every j\ (1\leq j\leq N), there is i (1\leq i\leq k) such that v_i=j.

Input

The input is given from Standard Input in the following format:
N M
U_1 V_1 W_1
U_2 V_2 W_2
\vdots
U_M V_M W_M

Output

If there is a walk that visits each vertex at least once, print the minimum total weight of the edges traversed. Otherwise, print No.

Constraints


- 2\leq N \leq 20
- 1\leq M \leq N(N-1)
- 1\leq U_i,V_i \leq N
- U_i \neq V_i
- (U_i,V_i) \neq (U_j,V_j) for i\neq j
- -10^6\leq W_i \leq 10^6
- The given graph does not contain negative cycles.
- All input values are integers.

To solve this problem, we need to determine if there exists a walk in a directed graph that visits each vertex at least once, and if such a walk exists, find the minimum total weight of the edges traversed. The graph may contain negative weights but does not contain any negative cycles.

### Approach
1. **Problem Analysis**: The problem requires finding a walk that covers all vertices at least once with minimal edge weight. The key observation is that the minimal walk will not include any cycles because the absence of negative cycles ensures that cycles do not reduce the total weight. Thus, the solution reduces to finding a simple path (without revisiting vertices) that visits each vertex exactly once (Hamiltonian path) with minimal total weight.

2. **Graph Preprocessing**: We first compute the shortest paths between all pairs of vertices using the Floyd-Warshall algorithm. This helps in determining if the graph is strongly connected (all vertices are reachable from each other) and provides the minimal path weights between any two vertices.

3. **Hamiltonian Path Calculation**: Using dynamic programming, we compute the minimal Hamiltonian path (visiting each vertex exactly once) in a complete graph where edge weights are the shortest path distances from the original graph. The DP state `dp[mask][i]` represents the minimal cost to reach vertex `i` having visited the set of vertices represented by the bitmask `mask`.

4. **Result Extraction**: After filling the DP table, the answer is the minimal value among all Hamiltonian paths that visit all vertices, i.e., the minimal value in `dp[(1 << N) - 1][i]` for all vertices `i`.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it)); m = int(next(it))
    
    INF = 10**18
    dist = [[INF] * n for _ in range(n)]

        dist[i][i] = 0

        v = int(next(it)) - 1

        for i in range(n):
            if dist[i][k] == INF:
                continue
            for j in range(n):
                if dist[k][j] == INF:
                    continue
                if dist[i][j] > dist[i][k] + dist[k][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
                    
    for i in range(n):
        for j in range(n):
            if dist[i][j] == INF:
                print("No")
                return
                
    dp = [[INF] * n for _ in range(1 << n)]
```
