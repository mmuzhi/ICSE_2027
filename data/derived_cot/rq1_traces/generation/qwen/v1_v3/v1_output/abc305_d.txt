You will be given a competitive programming problem.
Takahashi keeps a sleep log.
The log is represented as an odd-length sequence A=(A _ 1(=0), A _ 2,\ldots,A _ N), where odd-numbered elements represent times he got up, and even-numbered elements represent times he went to bed.
More formally, he had the following sleep sessions after starting the sleep log.

- For every integer i such that 1\leq i\leq\dfrac{N-1}2, he fell asleep exactly A _ {2i} minutes after starting the sleep log and woke up exactly A _ {2i+1} minutes after starting the sleep log.
- He did not fall asleep or wake up at any other time.

Answer the following Q questions.
For the i-th question, you are given a pair of integers (l _ i,r _ i) such that 0\leq l _ i\leq r _ i\leq A _ N.

- What is the total number of minutes for which Takahashi was asleep during the r _ i-l _ i minutes from exactly l _ i minutes to r _ i minutes after starting the sleep log?

Input

The input is given from Standard Input in the following format:
N
A _ 1 A _ 2 \ldots A _ N
Q
l _ 1 r _ 1
l _ 2 r _ 2
\vdots
l _ Q r _ Q

Output

Print the answer in Q lines.
The i-th line should contain an integer answering to the i-th question.

Constraints


- 3\leq N\lt2\times10^5
- N is odd.
- 0=A _ 1\lt A _ 2\lt\cdots\lt A _ N\leq10^9
- 1\leq Q\leq2\times10^5
- 0\leq l _ i\leq r _ i\leq A _ N\ (1\leq i\leq Q)
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to calculate the total time Takahashi was asleep during specific intervals given his sleep log. The sleep log is represented as an odd-length sequence where even-indexed elements (1-based) represent the times he went to bed, and odd-indexed elements (1-based) represent the times he woke up. The challenge is to efficiently answer multiple queries about the total sleep time within arbitrary intervals.

### Approach
1. **Problem Analysis**: The sleep log consists of disjoint sleep sessions. Each session starts at an even index and ends at the next odd index in the sequence. The goal is to compute the total sleep time between any two given times, `l` and `r`, for multiple queries efficiently.

2. **Intuition**: The sleep sessions can be broken down into segments between consecutive elements in the sequence. For each segment, if its index (0-based) is odd, it represents a sleep session, and its length contributes to the total sleep time. 

3. **Algorithm Selection**: 
   - **Prefix Sum Array**: We precompute a prefix sum array (`P`) where each entry `P[i]` is the cumulative sleep time up to the `i-th` segment. This allows us to quickly compute the total sleep time up to any point `x` by leveraging binary search to find the relevant segment.
   - **Binary Search**: For each query, we use binary search to determine the segment containing `x` (either `l` or `r`). The sleep time up to `x` is then derived from the prefix sum array and adjusted if `x` lies within a sleep segment.

4. **Complexity Analysis**: 
   - **Preprocessing**: Building the prefix sum array takes O(N) time, where N is the number of elements in the sleep log.
   - **Query Processing**: Each query requires O(log N) time for binary search and O(1) time to compute the result using the prefix sum array. With Q queries, the total time is O(Q log N), which is efficient given the constraints.

### Solution Code
```python
import bisect

def main():
    import sys
    data = sys.stdin.read().strip().split()
    if not data:
        return
    
    it = iter(data)
    N = int(next(it))
    A = [int(next(it)) for _ in range(N)]
    Q = int(next(it))
    
    num_segments = N - 1
    sleep_seg = [0] * num_segments
    for i in range(num_segments):
        if i % 2 == 1:
            sleep_seg[i] = A[i+1] - A[i]
    
    P = [0] * num_segments
    if num_segments > 0:
        P[0] = sleep_seg[0]
        for i in range(1, num_segments):
            P[i] = P[i-1] + sleep_seg[i]
    total_sleep = P[-1] if num_segments > 0 else 0

    def F(x):
        if x >= A[-1]:
            return total_sleep
        j = bisect.bisect_right(A, x) - 1
        if j < 0:
            return 0
        if j == 0:
            base = 0
        else:
            base = P[j-1]
        if j % 2 == 1:
            base += (x - A[j])
        return base

    out_lines = []
    for _ in range(Q):
        l = int(next(it))
        r = int(next(it))
        res = F(r) - F(l)
        out_lines.append(str(res))
    
    sys.stdout.write("\n".join(out_lines))

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The sequence `A` and the number of queries `Q` are parsed first.
2. **Prefix Sum Calculation**: The `sleep_seg` array is constructed where each element at an odd index (0-based) holds the length of the corresponding sleep session. The prefix sum array `P` is then built to store cumulative sleep times up to each segment.
3. **Query Processing**: For each query, the function `F(x)` computes the total sleep time from the start of the log up to time `x`. This involves:
   - Checking if `x` is beyond the last element in `A`, in which case the total sleep time is returned.
   - Using binary search (`bisect_right`) to locate the segment containing `x`.
   - Calculating the base sleep time from the prefix sum array and adjusting if `x` is within a sleep segment.
4. **Output**: The results for all queries are collected and printed at once for efficiency.

This approach efficiently handles the problem by leveraging prefix sums and binary search to answer each query in logarithmic time relative to the input size.
