Determine whether it is possible to monitor all N sections, and if it is possible, find the minimum total cost of the necessary sensors.

Input

The input is given from Standard Input in the following format:
N
D_1 D_2 \dots D_N
L_1 C_1 K_1
L_2 C_2 K_2

Output

If it is impossible to monitor all N sections, print -1. Otherwise, print the minimum total cost of the necessary sensors.

Constraints


- 1\leq N \leq 100
- 1\leq D_i,L_j \leq 10^5
- 1\leq C_j \leq 10^9
- 1\leq K_j \leq 10^3
- All input values are integers.

3

4 3 3
2 2 6

17

In this case, the total cost of the necessary sensors is 3\times 3 + 2\times 4 = 17, which is the minimum.

To solve this problem, we need to determine whether it's possible to monitor all given sections on a conveyor belt using a combination of two types of sensors, each with specific constraints. The goal is to find the minimum total cost if monitoring is possible, or return -1 if it's not.

### Approach
1. **Problem Analysis**: The problem involves covering each section of the conveyor belt with sensors such that the combined coverage from the sensors meets or exceeds the section's length. Each sensor type has a fixed coverage length, cost, and a maximum number of sensors that can be used. The solution requires checking all possible combinations of sensor usage across all sections while staying within the constraints.

2. **Dynamic Programming (DP) Setup**: We use a DP array where `dp[a]` represents the minimal number of type-2 sensors needed when `a` type-1 sensors are used across all sections processed so far. The DP array is initialized with a large number (infinity) except for `dp[0]` which is 0 (no sensors used initially).

3. **Processing Each Section**: For each section, we update the DP array by considering all possible numbers of type-1 sensors (from 0 to the remaining available sensors). For each possible count of type-1 sensors, we calculate the minimal number of type-2 sensors required to cover the section's length. The DP array is updated to reflect the minimal type-2 sensor usage for each total type-1 sensor count.

4. **Cost Calculation**: After processing all sections, we iterate through the DP array to find the minimal cost by combining the number of type-1 and type-2 sensors used, ensuring the total does not exceed the given constraints.

```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    
    it = iter(data)
    N = int(next(it))
    D_list = [int(next(it)) for _ in range(N)]
    L1 = int(next(it))
    C1 = int(next(it))
    K1_limit = int(next(it))
    L2 = int(next(it))
    C2 = int(next(it))
    K2_limit = int(next(it))
    
    dp = [10**18] * (K1_limit + 1)
    dp[0] = 0
    
    for d in D_list:
        new_dp = [10**18] * (K1_limit + 1)
        for a in range(0, K1_limit + 1):
            for x in range(0, a + 1):
                if x * L1 >= d:
                    y = 0
                else:
                    y = (d - x * L1 + L2 - 1) // L2
                if dp[a - x] == 10**18:
                    continue
                candidate = dp[a - x] + y
                if candidate > K2_limit:
                    continue
                if candidate < new_dp[a]:
                    new_dp[a] = candidate
        dp = new_dp

    ans = 10**18
    for a in range(0, K1_limit + 1):
        if dp[a] <= K2_limit:
            cost = a * C1 + dp[a] * C2
            if cost < ans:
                ans = cost
```
