=== ANSWER ===

To solve this problem, we need to encrypt a given string by replacing each character with the k-th character after it in a cyclic manner. The encryption involves shifting each character k positions to the right in the original string, wrapping around if necessary. 

### Approach
1. **Problem Analysis**: The task is to transform each character in the string `s` by finding the character located k positions ahead of it, considering the string as cyclic. For example, if `s = "dart"` and `k = 3`, the first character 'd' is replaced by the 3rd character after it, which is 't' (since 'd' → 'a' → 'r' → 't'). The same process is applied to each character in the string.
2. **Intuition**: For each character at position `i` in the string, the replacement character is found at position `(i + k) % n`, where `n` is the length of the string. This modulo operation ensures the position wraps around cyclically if it exceeds the string length.
3. **Algorithm Selection**: The algorithm involves iterating over each character in the string, computing the target position using the formula `(i + k) % n`, and collecting the characters from these positions to form the encrypted string.
4. **Complexity Analysis**: The algorithm runs in O(n) time, where `n` is the length of the string, since each character is processed exactly once. The space complexity is also O(n) to store the result.

### Solution Code
```python
class Solution:
    def getEncryptedString(self, s: str, k: int) -> str:
        n = len(s)
        res = []
        for i in range(n):
            idx = (i + k) % n
            res.append(s[idx])
        return ''.join(res)
```

### Explanation
1. **Initialization**: The length of the string `n` is determined, and an empty list `res` is initialized to store the encrypted characters.
2. **Iteration and Position Calculation**: For each index `i` in the string, the target index is calculated as `(i + k) % n`. This ensures the index wraps around the string's bounds if `i + k` exceeds `n-1`.
3. **Character Replacement**: The character at the calculated index is appended to the result list.
4. **Result Construction**: The list of characters is joined into a string and returned as the encrypted result.

This approach efficiently handles the cyclic nature of the problem, ensuring each character is replaced by the correct one in the encrypted string. The solution is optimal with linear time and space complexity.
