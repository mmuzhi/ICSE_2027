To solve this problem, we need to efficiently apply a series of character replacement operations to a given string. The challenge lies in handling a large number of operations (up to 200,000) on a string of up to 200,000 characters without resorting to a brute-force approach that would be computationally expensive.

### Approach
1. **Problem Analysis**: The problem requires processing a sequence of operations where each operation replaces all occurrences of a specific character in the string with another character. The key observation is that each operation affects the entire string, and subsequent operations may override previous replacements. Directly applying each operation to the string for every query would result in O(N*Q) time complexity, which is infeasible for large inputs.

2. **Insight**: Instead of processing each operation on the string directly, we can maintain a mapping of each original character to its final character after all operations. This involves tracking how each character transforms through each operation. For each operation (c, d), we update the mapping for all characters that currently map to c, setting them to d. This approach leverages the fact that the final character for any original character is determined by the cumulative effect of all operations.

3. **Algorithm Selection**: We use a dictionary to map each character to its final form. For each operation, we iterate over all 26 lowercase letters, updating those that map to the operation's source character to the target character. This ensures that each operation's effect is incorporated into the mapping in constant time per operation, making the overall time complexity O(26*Q), which is efficient given the constraints.

4. **Complexity Analysis**: The algorithm processes each operation by checking and updating up to 26 characters, leading to 26*Q operations. For Q up to 200,000, this results in approximately 5.2 million operations, which is feasible. The final string construction is O(N), where N is the string length, also efficient.

### Solution Code
```python
def main():
    n = int(input().strip())
    s = input().strip()
    q = int(input().strip())
    operations = []
    for _ in range(q):
        data = input().split()
        if len(data) < 2:
            continue
        c = data[0]
        d = data[1]
        operations.append((c, d))
    
    mapping = {}
    for i in range(26):
        letter = chr(ord('a') + i)
        mapping[letter] = letter
        
    for c, d in operations:
        for letter in mapping:
            if mapping[letter] == c:
                mapping[letter] = d
                
    result = ''.join(mapping[char] for char in s)
    print(result)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The code reads the string length, the string itself, and the number of operations. Each operation is stored as a pair of characters (source and target).

2. **Mapping Setup**: A dictionary is initialized where each lowercase letter maps to itself. This dictionary will track the cumulative effect of all operations.

3. **Processing Operations**: For each operation (c, d), the code updates the mapping for all characters that currently map to c, changing them to d. This step efficiently captures the sequential nature of the operations by updating the mapping in a single pass for each operation.

4. **Result Construction**: After processing all operations, the final string is constructed by replacing each character in the original string according to the computed mapping. The result is printed, representing the string after all operations have been applied.

This approach efficiently handles the operations by leveraging a mapping dictionary, ensuring optimal performance even for large inputs.
