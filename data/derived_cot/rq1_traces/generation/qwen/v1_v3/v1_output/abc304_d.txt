You will be given a competitive programming problem.
There is a rectangular cake with some strawberries on the xy-plane. The cake occupies the rectangular area \lbrace (x, y) : 0 \leq x \leq W, 0 \leq y \leq H \rbrace.
There are N strawberries on the cake, and the coordinates of the i-th strawberry are (p_i, q_i) for i = 1, 2, \ldots, N. No two strawberries have the same coordinates.
Takahashi will cut the cake into several pieces with a knife, as follows.

- First, cut the cake along A different lines parallel to the y-axis: lines x = a_1, x = a_2, \ldots, x = a_A.
- Next, cut the cake along B different lines parallel to the x-axis: lines y = b_1, y = b_2, \ldots, y = b_B.

As a result, the cake will be divided into (A+1)(B+1) rectangular pieces. Takahashi will choose just one of these pieces to eat. Print the minimum and maximum possible numbers of strawberries on the chosen piece.
Here, it is guaranteed that there are no strawberries along the edges of the final pieces. For a more formal description, refer to the constraints below.

Input

The input is given from Standard Input in the following format:
W H
N
p_1 q_1
p_2 q_2
\vdots
p_N q_N
A
a_1 a_2 \ldots a_A
B
b_1 b_2 \ldots b_B

Output

Print the minimum possible number of strawberries m and the maximum possible number M on the chosen piece in the following format, separated by a space.
m M

Constraints


- 3 \leq W, H \leq 10^9
- 1 \leq N \leq 2 \times 10^5
- 0 \lt p_i \lt W
- 0 \lt q_i \lt H
- i \neq j \implies (p_i, q_i) \neq (p_j, q_j)
- 1 \leq A, B \leq 2 \times 10^5
- 0 \lt a_1 \lt a_2 \lt \cdots \lt a_A \lt W
- 0 \lt b_1 \lt b_2 \lt \cdots \lt b_B \lt H
- p_i \not \in \lbrace a_1, a_2, \ldots, a_A \rbrace
- q_i \not \in \lbrace b_1, b_2, \ldots, b_B \rbrace
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to determine the minimum and maximum number of strawberries that can be found in any piece of a rectangular cake after it has been cut along specified vertical and horizontal lines. The cake is divided into several rectangular pieces by these cuts, and we need to efficiently count the strawberries in each piece without explicitly checking every piece, which would be computationally expensive due to the large possible number of pieces.

### Approach
1. **Problem Analysis**: The cake is divided into (A+1) vertical segments and (B+1) horizontal segments, resulting in (A+1)*(B+1) pieces. Each strawberry's position determines which segment it falls into. The challenge is to count the strawberries in each segment efficiently, especially since the number of pieces can be very large (up to 400,000 in the worst case).

2. **Key Insight**: For each strawberry, we can determine its segment by using binary search (via `bisect_right`) on the lists of vertical and horizontal cuts. This allows us to map each strawberry to its respective segment in logarithmic time relative to the number of cuts.

3. **Algorithm Selection**: 
   - **Binary Search**: Using `bisect_right` to find the segment indices for each strawberry's x and y coordinates. This efficiently determines which vertical and horizontal segments the strawberry belongs to.
   - **Dictionary for Counting**: A dictionary is used to keep track of the number of strawberries in each segment. This allows us to aggregate counts without iterating over all possible segments.

4. **Complexity Analysis**: 
   - **Time Complexity**: The solution involves processing N strawberries, each requiring two binary searches (O(log A) and O(log B)), leading to O(N*(log A + log B)) time. The space complexity is O(N) for storing the counts in the dictionary.
   - **Efficiency**: Given the constraints (N, A, B up to 200,000), this approach is efficient and feasible.

5. **Determine Minimum and Maximum**: 
   - The minimum number of strawberries in any piece is 0 if there exists at least one piece without any strawberries (i.e., the total number of pieces exceeds the number of segments with strawberries). Otherwise, the minimum is the smallest count among all segments.
   - The maximum is simply the highest count among all segments.

### Solution Code
```python
import sys
import bisect
from collections import defaultdict

def main():
    data = sys.stdin.read().split()
    if not data:
        print("0 0")
        return

    tokens = list(map(int, data))
    W = tokens[0]
    H = tokens[1]
    N = tokens[2]
    index = 3
    strawberries = []
    for i in range(N):
        p = tokens[index]
        q = tokens[index+1]
        index += 2
        strawberries.append((p, q))
    
    A = tokens[index]
    index += 1
    x_cuts = tokens[index:index+A]
    index += A

    B = tokens[index]
    index += 1
    y_cuts = tokens[index:index+B]
    index += B

    count_dict = defaultdict(int)
    for p, q in strawberries:
        x_index = bisect.bisect_right(x_cuts, p)
        y_index = bisect.bisect_right(y_cuts, q)
        count_dict[(x_index, y_index)] += 1

    total_cells = (A+1) * (B+1)
    if total_cells > len(count_dict):
        min_val = 0
    else:
        min_val = min(count_dict.values())

    max_val = max(count_dict.values()) if count_dict else 0

    print(f"{min_val} {max_val}")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints. The first three values are the cake dimensions (W, H) and the number of strawberries (N).
2. **Processing Strawberries**: For each strawberry, its coordinates are read and stored. The x and y cuts are read next, defining the segments of the cake.
3. **Binary Search for Segments**: For each strawberry, `bisect_right` is used to find the indices of the vertical and horizontal segments it belongs to. This is efficient and ensures correct segment assignment even with large datasets.
4. **Counting Strawberries**: A dictionary (`count_dict`) keeps track of how many strawberries are in each segment, using a tuple of segment indices as the key.
5. **Calculating Results**: The total number of segments is (A+1)*(B+1). If this exceeds the number of segments with strawberries, the minimum count is 0; otherwise, it's the smallest count in the dictionary. The maximum count is the highest value in the dictionary.
6. **Output**: The results are printed as specified, providing the minimum and maximum number of strawberries in any cake piece after the cuts. This approach efficiently handles the problem constraints by leveraging binary search and dictionary aggregation.
