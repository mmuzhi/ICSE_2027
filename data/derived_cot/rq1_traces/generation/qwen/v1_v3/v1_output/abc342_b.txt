=== ANSWER ===

To solve this problem, we need to determine which of two given persons in a line is standing further to the front. The line is represented as a list where each element corresponds to a person at a specific position from front to back. The solution involves processing multiple queries, each asking for the person with the smaller index (i.e., closer to the front) between two given persons.

### Approach
1. **Problem Analysis**: The problem requires us to compare the positions of two distinct persons in a line. The line is given as a list where the order represents the front to back positions. For each query, we need to check which of the two persons appears earlier in this list.
2. **Intuition**: By mapping each person to their index in the list, we can quickly determine their positions. For any two persons, the one with the smaller index is closer to the front.
3. **Algorithm Selection**: We use a dictionary to map each person to their index in the list. This allows O(1) access to the position of any person. For each query, we simply compare the indices of the two persons and return the one with the smaller index.
4. **Complexity Analysis**: 
   - **Time Complexity**: O(N) for building the dictionary and O(Q) for processing the queries, where N is the number of people and Q is the number of queries. Each query is processed in constant time.
   - **Space Complexity**: O(N) for storing the dictionary.

### Solution Code
```python
n = int(input())
arr = list(map(int, input().split()))
mapping = {person: idx for idx, person in enumerate(arr)}
q = int(input())
for _ in range(q):
    a, b = map(int, input().split())
    if mapping[a] < mapping[b]:
        print(a)
    else:
        print(b)
```

### Explanation
1. **Reading Input**: The first line reads the number of people, `n`. The next line reads the list representing the line from front to back.
2. **Building the Dictionary**: A dictionary is created where each key is a person (from the list) and the value is their 0-based index in the list. This allows quick lookup of any person's position.
3. **Processing Queries**: For each query, the two person numbers are read. The dictionary is used to find their indices. The person with the smaller index is printed as they are closer to the front.
4. **Output**: The result for each query is printed immediately after processing.

This approach efficiently handles each query by leveraging a precomputed dictionary for constant-time lookups, ensuring optimal performance even for the upper constraint limits.
