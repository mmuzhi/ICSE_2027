class Solution:
    def findKthSmallest(self, coins: List[int], k: int) -> int:

        import math
        from itertools import combinations

        # But we need to generate all non-empty subsets. We can use bit masks or combinations.

        # But we need to consider all subsets, not just contiguous. 

        # Using bit masks:

        n = len(coins)
        total_subsets = 1 << n
        precomputed = []  # (lcm_val, sign)

        # Iterate over non-empty subsets
        for bitmask in range(1, total_subsets):
            # get the subset
            subset = []
            for j in range(n):
                if bitmask & (1 << j):
                    subset.append(coins[j])
            # Compute LCM for the subset
            current = subset[0]
            # If the subset has one element, then LCM is that element.
            # For more, compute iteratively.
            for i in range(1, len(subset)):
                # Compute LCM of current and subset[i]
                g = math.gcd(current, subset[i])
                current = current * subset[i] // g
            # Determine sign: 1 if subset length is odd, -1 if even
            sign = 1 if len(subset) % 2 == 1 else -1
            precomputed.append((current, sign))

        # Now, define the count function
        def count(X):
            total_count = 0
            for (lcm_val, sign) in precomputed:
                if lcm_val > X:
                    continue
                total_count += sign * (X // lcm_val)

The challenge is to efficiently count these numbers without enumerating all possible values, especially since k can be as large as 2 * 10^9.

2. **Intuition**: The solution involves two main steps:
   - **Precompute Subsets and LCMs**: For each non-empty subset of the given coins, compute the least common multiple (LCM) of the subset. The inclusion-exclusion principle is then used to count the numbers up to a certain value X that are divisible by any coin in the subset.
   - **Binary Search**: Use binary search to find the smallest number X such that the count of numbers ≤ X that are multiples of any coin is at least k. The binary search bounds are set from 1 to the product of the smallest coin and k, ensuring the solution is within a feasible range.

3. **Algorithm Selection**: The inclusion-exclusion principle is chosen for its ability to efficiently count the union of multiple arithmetic sequences (the multiples of each coin). Binary search is used to handle the large possible value of k by narrowing down the search space logarithmically.

4. **Complexity Analysis**: The algorithm involves generating all non-empty subsets of the coins, which is O(2^N) where N is the number of coins (up to 15, so 32,768 subsets). For each subset, the LCM is computed in O(N) time, making the precomputation step manageable. The binary search runs in O(log(min_coin * k)) time, and each count operation during the search is O(2^N). Given the constraints, this approach is efficient and feasible.

### Solution Code
```python
from typing import List
import math

class Solution:
    def findKthSmallest(self, coins: List[int], k: int) -> int:
        n = len(coins)
        total_subsets = 1 << n
        precomputed = []
        for bitmask in range(1, total_subsets):
            subset = []
            for j in range(n):
                if bitmask & (1 << j):
                    subset.append(coins[j])
            current = subset[0]
            for i in range(1, len(subset)):
                g = math.gcd(current, subset[i])
                current = current * subset[i] // g
            sign = 1 if len(subset) % 2 == 1 else -1
            precomputed.append((current, sign))
        
        def count(X):
            total = 0
            for lcm_val, sign in precomputed:
                if lcm_val > X:
                    continue
                total += sign * (X // lcm_val)
            return total
        
        min_coin = min(coins)
        low, high = 1, min_coin * k
        
        while low < high:
            mid = (low + high) // 2
            if count(mid) >= k:
                high = mid
            else:
                low = mid + 1
        
        return low
```

### Explanation
1. **Precompute Subsets and LCMs**: The code first generates all non-empty subsets of the given coins. For each subset, it calculates the LCM of the coins in the subset and determines the sign based on the subset size (positive for odd-sized subsets, negative for even-sized subsets). This is stored in a list for later use.

2. **Count Function**: The `count` function uses the inclusion-exclusion principle to determine how many numbers up to X are divisible by any coin. It iterates over all precomputed subsets, adjusting the count based on the subset's LCM and sign.

3. **Binary Search**: The binary search is performed between 1 and the product of the smallest coin and k. For each midpoint value, the `count` function checks if the number of valid multiples is at least k. If it is, the upper bound is adjusted; otherwise, the lower bound is increased. The loop continues until the smallest such value is found.

This approach efficiently narrows down the search space using binary search and leverages the inclusion-exclusion principle to count the relevant multiples without exhaustive enumeration, making it suitable for large values of k.
