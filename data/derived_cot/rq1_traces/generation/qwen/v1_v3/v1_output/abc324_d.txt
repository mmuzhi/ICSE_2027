Input

The input is given from Standard Input in the following format:
N
S

Output

Print the answer in a single line.

Constraints


- 1\leq N\leq 13
- S is a string of length N consisting of digits.
- N is an integer.

Sample Input 1

4320

Sample Output 1

2

To solve this problem, we need to count the number of distinct square numbers that can be formed by interpreting any permutation of a given string of digits as a decimal integer. The solution involves generating all possible square numbers within the range defined by the length of the digit string and checking if their digit multiset matches the given string's multiset when represented as an N-digit number (including leading zeros).

### Approach
1. **Problem Analysis**: The problem requires us to consider all permutations of the given digit string and check if the resulting number (when interpreted as a decimal integer) is a perfect square. However, directly generating all permutations is computationally infeasible for larger strings (N up to 13). Instead, we can leverage the fact that the number must be a perfect square within the range [0, 10^N - 1].

2. **Intuition**: For each number in the range [0, 10^N - 1], we check if it is a perfect square. For each perfect square found, we convert it to an N-digit string (padding with leading zeros if necessary) and compare its digit multiset with the multiset of the original string. If they match, the square is counted.

3. **Algorithm Selection**: 
   - **Frequency Array**: We first build a frequency array for the digits of the input string.
   - **Iterate Over Squares**: For each integer `i` from 0 to the integer square root of (10^N - 1), compute `i*i` and check if it lies within the range.
   - **Digit Multiset Check**: For each valid square, convert it to an N-digit string (with leading zeros), build its digit frequency array, and compare it with the original string's frequency array. If they match, increment the count.

4. **Complexity Analysis**: The algorithm's time complexity is O(sqrt(10^N)), which for N=13 is approximately 31.6 million operations. Each operation involves digit manipulation and comparison, which is efficient given the constraints.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data: 
        return
    N = int(data[0].strip())
    S = data[1].strip()

    freq_S = [0] * 10
    for char in S:
        digit = int(char)
        freq_S[digit] += 1

    total = 10**N - 1
    count = 0
    i = 0
    while i * i <= total:
        x = i * i
        s = str(x)
        L = len(s)
        freq_x = [0] * 10
        for char in s:
            digit = int(char)
            freq_x[digit] += 1
        if L < N:
            freq_x[0] += (N - L)
        if freq_x == freq_S:
            count += 1
        i += 1

    print(count)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input consists of an integer N (length of the digit string) and the string S itself.
2. **Frequency Array Construction**: We construct a frequency array `freq_S` that counts occurrences of each digit (0-9) in the string S.
3. **Square Generation**: We iterate over all integers `i` from 0 to the integer square root of (10^N - 1). For each `i`, we compute the square `x = i*i`.
4. **Digit Multiset Check**: For each square `x`, we convert it to a string and pad it with leading zeros to make it an N-digit string. We then build a frequency array `freq_x` for the digits of this string. If the length of the string representation of `x` is less than N, we account for the leading zeros by incrementing the count of '0's in `freq_x`.
5. **Comparison**: We compare `freq_x` with `freq_S`. If they match, the square `x` is counted.
6. **Output**: Finally, we print the count of distinct square numbers that match the digit multiset of the input string.

This approach efficiently checks all possible squares within the specified range, leveraging digit frequency comparison to determine valid permutations without explicitly generating all permutations, thus optimizing both time and space complexity.
