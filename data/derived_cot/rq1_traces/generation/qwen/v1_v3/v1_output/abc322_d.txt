=== PROMPT ===
You will be given a competitive programming problem.
A polyomino is a puzzle piece in the shape of a connected polygon made by connecting several squares by their edges.
There is a grid with four rows and four columns, and three polyominoes that fit within the grid.
The shape of the i-th polyomino is represented by 16 characters P_{i,j,k} (1 \leq j, k \leq 4). They describe the state of the grid when the i-th polyomino is placed on it. If P_{i, j, k} is #, the square at the j-th row from the top and k-th column from the left is occupied by the polyomino; if it is ., the square is not occupied. (Refer to the figures at Sample Input/Output 1.)
You want to fill the grid with all three polyominoes so that all of the following conditions are satisfied.

- All squares of the grid are covered by the polyominoes.
- The polyominoes must not overlap each other.
- The polyominoes must not stick out of the grid.
- The polyominoes may be freely translated and rotated but may not be flipped over.

Can the grid be filled with the polyominoes to satisfy these conditions?

Input

The input is given from Standard Input in the following format:
P_{1,1,1}P_{1,1,2}P_{1,1,3}P_{1,1,4}
P_{1,2,1}P_{1,2,2}P_{1,2,3}P_{1,2,4}
P_{1,3,1}P_{1,3,2}P_{1,3,3}P_{1,3,4}
P_{1,4,1}P_{1,4,2}P_{1,4,3}P_{1,4,4}
P_{2,1,1}P_{2,1,2}P_{2,1,3}P_{2,1,4}
P_{2,2,1}P_{2,2,2}P_{2,2,3}P_{2,2,4}
P_{2,3,1}P_{2,3,2}P_{2,3,3}P_{2,3,4}
P_{2,4,1}P_{2,4,2}P_{2,4,3}P_{2,4,4}
P_{3,1,1}P_{3,1,2}P_{3,1,3}P_{3,1,4}
P_{3,2,1}P_{3,2,2}P_{3,2,3}P_{3,2,4}
P_{3,3,1}P_{3,3,2}P_{3,3,3}P_{3,3,4}
P_{3,4,1}P_{3,4,2}P_{3,4,3}P_{3,4,4}

Output

If it is possible to fill the grid with the polyominoes to satisfy the conditions in the problem statement, print Yes; otherwise, print No.

Constraints


- P_{i, j, k} is # or ..
- The given polyominoes are connected. In other words, the squares that make up a polyomino can be reached from each other by following only the squares up, down, left, and right.
- The given polyominoes are not empty.

To solve this problem, we need to determine if three given polyominoes can be placed on a 4x4 grid such that they cover the entire grid without overlapping or sticking out of the grid. The polyominoes can be translated and rotated (but not flipped) to fit into the grid.

### Approach
1. **Input Parsing and Initial Check**: Read the input to get the three polyominoes, each represented by a 4x4 grid. Count the total number of '#' characters across all three grids. If the total is not 16, immediately return "No" because the grid cannot be fully covered.
2. **Generate Placements**: For each polyomino, generate all possible placements on the 4x4 grid by considering all rotations (0°, 90°, 180°, 270°) and translations (shifting the polyomino within the grid bounds). Each placement is stored as a set of coordinates.
3. **Check Valid Combinations**: Iterate through all combinations of placements (one from each polyomino) to check if any combination covers the entire grid without overlapping. If such a combination is found, return "Yes"; otherwise, return "No".

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if len(data) < 12:
        print('No')
        return

    grids = []
    for i in range(0, 12, 4):
        grid = data[i:i+4]
        grids.append(grid)

    sets = []
    for grid in grids:
        s = set()
        for i in range(4):
            for j in range(4):
                if grid[i][j] == '#':
                    s.add((i, j))
        sets.append(s)

    total = sum(len(s) for s in sets)
    if total != 16:
        print('No')
        return

    def generate_placements(S):
        if not S:
            return set()
        rotations_map = {
            0: lambda p: p,
            90: lambda p: (p[1], 3-p[0]),
            180: lambda p: (3-p[0], 3-p[1]),
            270: lambda p: (3-p[1], p[0])
        }
        placements = set()
        for rot in [0, 90, 180, 270]:
            rotated_set = set()
            for (i, j) in S:
                new_point = rotations_map[rot]((i, j))
                rotated_set.add(new_point)
            if not rotated_set:
                continue
            min_i = min(x for x, y in rotated_set)
            max_i = max(x for x, y in rotated_set)
            min_j = min(y for x, y in rotated_set)
            max_j = max(y for x, y in rotated_set)

            for dx in range(0, 4 - max_i):
                for dy in range(0, 4 - max_j):
                    new_set = set()
                    for (x, y) in rotated_set:
                        nx, ny = x + dx, y + dy
                        new_set.add((nx, ny))
                    placements.add(frozenset(new_set))
        return placements

    placements_list = []
    for s in sets:
        placements_list.append(generate_placements(s))

    found = False
    for s1 in placements_list[0]:
        for s2 in placements_list[1]:
            if s1 & s2:
                continue
            for s3 in placements_list[2]:
                if s1 & s3 or s2 & s3:
                    continue
                found = True
                break
            if found:
                break
        if found:
            break

    print('Yes' if found else 'No')

if __name__ == '__main__':
    main()
```

2. **Placement Generation**: For each polyomino, all possible placements (considering rotations and translations) are generated. Each placement is a set of coordinates where the polyomino would occupy squares in the 4x4 grid.
3. **Combination Check**: The code checks all combinations of placements (one from each polyomino) to see if they cover the entire grid without overlapping. If such a combination is found, it prints "Yes"; otherwise, it prints "No".

This approach efficiently explores all possible configurations of the polyominoes on the grid, ensuring the solution is both correct and optimal for the given constraints.
