=== PROMPT ===
You will be given a competitive programming problem.
There is an election to choose one winner from N candidates with candidate numbers 1, 2, \ldots, N, and there have been M votes cast.
Each vote is for exactly one candidate, with the i-th vote being for candidate A_i.
The votes will be counted in order from first to last, and after each vote is counted, the current winner will be updated and displayed.
The candidate with the most votes among those counted is the winner. If there are multiple candidates with the most votes, the one with the smallest candidate number is the winner.
For each i = 1, 2, \ldots, M, determine the winner when counting only the first i votes.

Input

The input is given from Standard Input in the following format:
N M
A_1 A_2 \ldots A_M

Output

Print M lines.
The i-th line should contain the winner's candidate number when counting only the first i votes.

Constraints


- 1 \leq N, M \leq 200000
- 1 \leq A_i \leq N
- All input values are integers.

Sample Input 1

3 7
1 2 2 3 1 3 3

Sample Output 1

1
1
2
2
1
1
3

- After the first vote is counted, (C_1, C_2, C_3) = (1, 0, 0), so the winner is 1.
- After the second vote is counted, (C_1, C_2, C_3) = (1, 1, 0), so the winner is 1.
- After the third vote is counted, (C_1, C_2, C_3) = (1, 2, 0), so the winner is 2.
- After the fourth vote is counted, (C_1, C_2, C_3) = (1, 2, 1), so the winner is 2.
- After the fifth vote is counted, (C_1, C_2, C_3) = (2, 2, 1), so the winner is 1.
- After the sixth vote is counted, (C_1, C_2, C_3) = (2, 2, 2), so the winner is 1.

To solve this problem, we need to determine the winner after each vote is counted in an election. The winner is the candidate with the most votes at that point. If multiple candidates have the same maximum number of votes, the candidate with the smallest number is chosen. 

### Approach
1. **Initialization**: We start by reading the input values, which include the number of candidates (N), the number of votes (M), and the sequence of votes.
2. **Tracking Votes**: We maintain an array to keep track of the current vote counts for each candidate.
3. **Updating Winner**: For each vote, we update the corresponding candidate's vote count. After each update, we check if the candidate's new count exceeds the current maximum count. If it does, we update the maximum count and set this candidate as the new winner. If the new count equals the current maximum, we compare the candidate's number with the current winner and update the winner if the candidate has a smaller number.
4. **Output Results**: After processing each vote, we record the current winner and print all results after processing all votes.

This approach efficiently tracks the winner after each vote in linear time, O(M), by only updating the necessary information after each vote, making it suitable for large input sizes as specified in the problem constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    votes = list(map(int, data[2:2+m]))
    
    count = [0] * (n + 1)
    max_count = 0
    best_candidate = 0
    results = []
    
    for vote in votes:
        c = vote
        old_count = count[c]
        new_count = old_count + 1
        count[c] = new_count
        
        if new_count > max_count:
            max_count = new_count
            best_candidate = c
        elif new_count == max_count:
            if c < best_candidate:
                best_candidate = c
                
        results.append(str(best_candidate))
        
    print("\n".join(results))

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are the number of candidates (N) and the number of votes (M), followed by the sequence of votes.
2. **Initialization**: An array `count` of size N+1 is initialized to zero to keep track of each candidate's votes. Variables `max_count` and `best_candidate` are initialized to zero to start tracking the highest vote count and the corresponding candidate.
3. **Processing Votes**: For each vote in the sequence:
   - The candidate's vote count is incremented.

   - If the new count equals `max_count`, `best_candidate` is updated only if the current candidate has a smaller number than the existing `best_candidate`.
4. **Output**: After processing all votes, the results (the winner after each vote) are printed line by line.
