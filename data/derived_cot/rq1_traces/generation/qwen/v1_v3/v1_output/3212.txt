To solve this problem, we need to determine the number of good partitions of an array where no two subarrays in the partition contain the same number. A good partition is defined as one where all occurrences of any number are confined to a single contiguous subarray. 

### Approach
1. **Problem Analysis**: The key observation is that for any number appearing multiple times in the array, all its occurrences must lie within the same contiguous subarray. This means that the partition boundaries must be such that no number is split across different subarrays.
2. **Intuition**: The solution involves identifying the minimal segments of the array where each segment contains all occurrences of any number. These segments are determined by the last occurrence of each number in the array. By tracking the farthest last occurrence of any number encountered up to each point, we can determine the boundaries of these segments.
3. **Algorithm Selection**: 
   - **Step 1**: Traverse the array from right to left to record the last occurrence of each number.
   - **Step 2**: Traverse the array from left to right, keeping track of the farthest last occurrence encountered so far. Whenever the current index matches this farthest last occurrence, it signifies the end of a minimal segment.
   - **Step 3**: The number of such segments determines the number of ways to partition the array. Specifically, the answer is \(2^{(k-1)}\) where \(k\) is the number of segments, as each segment boundary can either be included or excluded in the partition.

### Solution Code
```python
class Solution:
    def numberOfGoodPartitions(self, nums: List[int]) -> int:
        n = len(nums)
        last_occurrence = {}
        for i in range(n-1, -1, -1):
            if nums[i] not in last_occurrence:
                last_occurrence[nums[i]] = i
        
        segments = 0
        current_end = 0
        for j in range(n):
            current_end = max(current_end, last_occurrence[nums[j]])
            if j == current_end:
                segments += 1
        
        mod = 10**9 + 7
        return pow(2, segments - 1, mod)
```

### Explanation
1. **Recording Last Occurrences**: The first loop processes the array from right to left, storing the last occurrence index of each number in a dictionary. This helps in knowing the furthest position each number appears in the array.
2. **Determining Segments**: The second loop processes the array from left to right, tracking the farthest last occurrence of any number encountered. When the current index matches this farthest position, it marks the end of a segment. The number of such segments is counted.
3. **Calculating Partitions**: The number of good partitions is derived from the number of segments. Each segment boundary can either be a partition point or not, leading to \(2^{(k-1)}\) possible partitions, where \(k\) is the number of segments. The result is computed modulo \(10^9 + 7\) to handle large numbers.

This approach efficiently identifies the necessary segments and leverages combinatorial mathematics to count the valid partitions, ensuring optimal performance even for large input sizes.
