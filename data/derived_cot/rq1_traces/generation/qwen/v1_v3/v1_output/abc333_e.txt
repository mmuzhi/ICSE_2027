### Approach
1. **Problem Analysis**: The problem involves processing a sequence of events where Takahashi either finds potions or encounters monsters. For each monster encounter, he must have a potion of the same type to defeat it. The goal is to ensure all monsters are defeated while minimizing the maximum number of potions carried at any time.

2. **Key Insight**: The minimal peak number of potions (K_min) is the smallest value such that there exists a strategy to defeat all monsters without exceeding K_min potions at any point. This can be determined using binary search on the possible values of K (from 0 to the total number of potion events).

3. **Binary Search**: For each candidate K during binary search, simulate the events:
   - For potion events, decide to pick up the potion only if the current total potions is less than K and there are future monsters of the same type that haven't been defeated.
   - For monster events, use a potion if available; otherwise, the candidate K is not feasible.

4. **Feasibility Check**: The simulation checks if all monsters can be defeated without exceeding the candidate K. The smallest feasible K is the answer.

5. **Action Sequence**: Once the minimal K is found, the actions (picking up or discarding potions) that achieve this K are recorded and output.

import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    events = []
    index = 1
    for i in range(n):
        t = int(data[index]); x = int(data[index+1]); index += 2
        events.append((t, x))
    
    type_counts = {}
    for i in range(1, n+1):
        type_counts[i] = 0
        
    monsters = {}
    for i in range(1, n+1):
        monsters[i] = 0
        
    for t, x in events:
        if t == 2:
            monsters[x] += 1
        else:
            type_counts[x] += 1
            
    for x in list(type_counts.keys()):
        if monsters.get(x, 0) > type_counts[x]:
            print(-1)
            return
            
    low = 0
    high = n
    
    def feasible(K):
        total = 0
        counts = [0] * (n+1)
        actions = [0] * n
        for i in range(n):
            t, x = events[i]
            if t == 2:
                if counts[x] > 0:
                    counts[x] -= 1
                    total -= 1
                else:
                    return False, None
            else:
                if total < K:
                    if monsters[x] > 0:
                        counts[x] += 1
                        total += 1
                        actions[i] = 1
                    else:
                        actions[i] = 0
                else:
                    actions[i] = 0
        return True, actions
        
    ansK = -1
    actions_list = None
    while low <= high:
        mid = (low + high) // 2
        feasible_flag, actions_mid = feasible(mid)
        if feasible_flag:
            ansK = mid
            actions_list = actions_mid
            high = mid - 1
        else:
            low = mid + 1
            
    if ansK == -1:
        print(-1)
    else:
        print(ansK)
        action_strings = []
