=== ANSWER ===

To solve this problem, we need to determine the number of cells in a grid that are painted with each distinct color after performing a series of row and column repaint operations. The grid starts with all cells painted color 0, and each operation repaints either a row or a column with a specified color. The challenge is to efficiently compute the final color counts without explicitly processing each cell, given the constraints that the grid dimensions and operations can be very large (up to 200,000).

### Approach
1. **Track Last Operations**: For each row and column, record the last operation that affected it, including the operation's time (index) and the color used. This helps in determining the final color of any cell based on whether the last operation was a row or column operation.
2. **Count Cells by Operations**: For each row operation, count the number of columns that were last updated before the row operation's time. Similarly, for each column operation, count the number of rows updated before the column operation's time. These counts give the number of cells that retain the color from the row or column operation.
3. **Handle Unupdated Cells**: Cells that are never updated by any operation (neither row nor column) remain color 0. The number of such cells is calculated as the product of the number of unupdated rows and columns.
4. **Aggregate Results**: Combine the counts from row and column operations, then add the count of unupdated cells to color 0. Finally, output the distinct colors in ascending order along with their respective cell counts.

### Solution Code
```python
import sys
from bisect import bisect_left
from collections import defaultdict

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    H = int(next(it))
    W = int(next(it))
    M = int(next(it))
    
    row_time = [-1] * (H + 1)
    col_time = [-1] * (W + 1)
    row_color = [0] * (H + 1)
    col_color = [0] * (W + 1)
    
    for idx in range(M):
        t = int(next(it))
        a = int(next(it))
        x = int(next(it))
        time_index = idx + 1
        if t == 1:
            if time_index > row_time[a]:
                row_time[a] = time_index
                row_color[a] = x
        else:
            if time_index > col_time[a]:
                col_time[a] = time_index
                col_color[a] = x
                
    col_time_list = []
    for j in range(1, W + 1):
        col_time_list.append(col_time[j])
    col_time_list.sort()
    
    row_time_list = []
    for i in range(1, H + 1):
        row_time_list.append(row_time[i])
    row_time_list.sort()
    
    count_dict = defaultdict(int)
    
    for i in range(1, H + 1):
        if row_time[i] != -1:
            pos = bisect_left(col_time_list, row_time[i])
            count_dict[row_color[i]] += pos
            
    for j in range(1, W + 1):
        if col_time[j] != -1:
            pos = bisect_left(row_time_list, col_time[j])
            count_dict[col_color[j]] += pos
            
    R0 = 0
    for i in range(1, H + 1):
        if row_time[i] == -1:
            R0 += 1
            
    C0 = 0
    for j in range(1, W + 1):
        if col_time[j] == -1:
            C0 += 1
            
    Set0_count = R0 * C0
    if 0 in count_dict:
        count_dict[0] += Set0_count
    else:
        count_dict[0] = Set0_count
        
    result = []
    for color, cnt in count_dict.items():
        if cnt > 0:
            result.append((color, cnt))
            
    result.sort(key=lambda x: x[0])
    
    print(len(result))
    for color, cnt in result:
        print(color, cnt)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints.
2. **Tracking Operations**: For each operation, we update the last operation time and color for the respective row or column. This ensures we know the most recent color applied to any cell in that row or column.
3. **Counting Cells**: Using binary search (via `bisect_left`), we efficiently count how many columns (for row operations) or rows (for column operations) were last updated before the current operation's time. This count gives the number of cells that will retain the operation's color.
4. **Unupdated Cells**: Cells not covered by any operation remain color 0. The count of such cells is derived from the number of unupdated rows and columns.
5. **Result Compilation**: The counts for each color are aggregated, with color 0 including any unupdated cells. The results are then sorted and printed in ascending order of color numbers.

This approach efficiently handles the large constraints by leveraging sorting and binary search to avoid per-cell processing, ensuring optimal performance.
