To solve this problem, we need to determine the number of ice squares a player can touch (either by passing through or resting on) in an N x M grid. The player starts at a specific position (2,2 in 1-based indexing) and can move in any of the four cardinal directions until hitting a rock. The player's movement continues in the specified direction until a rock is encountered, and the player rests on the last ice square before the rock. The goal is to count all unique ice squares the player can reach during any sequence of moves.

### Approach
1. **Problem Analysis**: The player starts at (2,2) (converted to 0-based indexing as (1,1)) and can move in four directions (up, down, left, right) until a rock is encountered. Each move allows the player to traverse all ice squares in the chosen direction until a rock stops the movement. The challenge is to count all unique ice squares touched during any sequence of such moves.

2. **Intuition**: The solution involves a Breadth-First Search (BFS) approach where each cell processed is an ice square the player can rest on. From each cell, the player can move in all four directions, marking all traversed ice squares as visited. The BFS ensures that every reachable ice square is processed exactly once, avoiding redundant checks.

3. **Algorithm Selection**: BFS is suitable here because it explores all reachable cells level by level, ensuring that all ice squares reachable from the starting position are visited. For each cell, we explore all four directions, marking each ice square encountered as visited and adding it to the queue for further exploration.

4. **Complexity Analysis**: The grid size is up to 200x200, leading to a maximum of 40,000 cells. For each cell, we may check up to 200 steps in each direction, resulting in a worst-case scenario of 40,000 * 4 * 200 = 32,000,000 operations, which is feasible for modern programming environments.

### Solution Code
```python
import collections
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data: 
        return
    N, M = map(int, data[0].split())
    grid = []
    for i in range(1, 1+N):
        grid.append(data[i].strip())

    visited = [[False] * M for _ in range(N)]
    q = collections.deque()
    start_r, start_c = 1, 1

    if grid[start_r][start_c] == '.':
        visited[start_r][start_c] = True
        q.append((start_r, start_c))
    else:
        print(0)
        return

    while q:
        r, c = q.popleft()
        for dr, dc in [(0,1), (0,-1), (1,0), (-1,0)]:
            nr, nc = r+dr, c+dc
            while 0 <= nr < N and 0 <= nc < M and grid[nr][nc] == '.':
                if not visited[nr][nc]:
                    visited[nr][nc] = True
                    q.append((nr, nc))
                nr += dr
                nc += dc

    count = 0
    for i in range(N):
        for j in range(M):
            if visited[i][j]:
                count += 1

    print(count)

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read from standard input, where the first line contains the grid dimensions (N, M), followed by N lines representing the grid itself.
2. **Initialization**: A 2D `visited` array is initialized to keep track of visited ice squares. The starting position (1,1 in 0-based indexing) is marked as visited and added to the queue.
3. **BFS Execution**: For each cell dequeued, the algorithm explores all four possible directions. In each direction, it moves until a rock or grid boundary is encountered, marking each ice square as visited and enqueuing it for further exploration.
4. **Counting Visited Cells**: After BFS completes, the algorithm counts all marked ice squares in the `visited` array and prints the result.

This approach efficiently explores all reachable ice squares using BFS, ensuring optimal performance and correctness for the given problem constraints.
