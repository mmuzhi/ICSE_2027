To solve this problem, we need to find the lexicographically smallest string that can be obtained by swapping adjacent digits in the original string at most once, provided the digits have the same parity (both even or both odd). The solution involves checking all possible adjacent pairs of digits with the same parity, swapping them, and comparing the resulting string with the original to determine the smallest lexicographical string.

### Approach
1. **Problem Analysis**: The task is to find the smallest lexicographical string by performing at most one adjacent swap of digits with the same parity. Lexicographical order means the string should be as small as possible when compared character by character from left to right.
2. **Intuition**: The key observation is that swapping digits at an earlier position can have a more significant impact on the lexicographical order. Therefore, we need to check all possible adjacent swaps (same parity) and choose the one that results in the smallest string.
3. **Algorithm Selection**: We iterate through each adjacent pair in the string. For each pair with the same parity, we generate a candidate string by swapping the digits. We then compare this candidate with the current best string (initially the original string) and update the best string if the candidate is lexicographically smaller.
4. **Complexity Analysis**: The algorithm processes each adjacent pair once, leading to O(n) time complexity where n is the length of the string. For each pair, we perform a constant amount of work (checking parity and constructing the candidate string), making the overall complexity O(n). The space complexity is O(n) due to the storage of the candidate string during comparisons.

This approach efficiently explores all possible beneficial swaps and ensures the smallest lexicographical result by leveraging simple string operations and comparisons.
