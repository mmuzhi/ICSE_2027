A length-K sequence of indices (i_1, i_2, \dots, i_K) is called a good index sequence if it satisfies both of the following conditions:

- 1 \leq i_1 < i_2 < \dots < i_K \leq N.

Formally, there exists an integer a such that \lbrace P_{i_1},P_{i_2},\dots,P_{i_K} \rbrace = \lbrace a,a+1,\dots,a+K-1 \rbrace.

Input

The input is given from Standard Input in the following format:
N K
P_1 P_2 \dots P_N

Output

Print the minimum value of i_K - i_1 among all good index sequences.

Constraints


- 1 \leq K \leq N \leq 2 \times 10^5
- 1 \leq P_i \leq N
- P_i \neq P_j if i \neq j.
- All input values are integers.

A good index sequence is defined as a strictly increasing sequence of indices where the corresponding elements in the permutation form a set of K consecutive integers when rearranged.

### Approach
1. **Problem Analysis**: The problem requires identifying a sequence of K indices in a permutation such that the elements at these indices form a set of K consecutive integers. The challenge is to find such a sequence with the smallest possible span (difference between the last and first indices).

2. **Key Insight**: For any K consecutive integers (say from `a` to `a+K-1`), their positions in the permutation must be considered. The span for this set is the difference between the maximum and minimum positions of these elements. The solution involves checking all possible consecutive sets of K integers and computing their spans, then selecting the minimum span.

3. **Algorithm Selection**: 
   - **Sliding Window Technique**: To efficiently compute the minimum and maximum positions for all contiguous segments of length K in the array of positions (for values 1 to N), we use two deques—one for maintaining the minimum values and another for the maximum values in the current window. This allows us to compute the min and max for each window in constant time per element, leading to an overall O(N) time complexity.

4. **Complexity Analysis**: The algorithm processes each element in the permutation once for both the minimum and maximum deques, resulting in a time complexity of O(N). The space complexity is O(N) due to the storage required for the position array and the deques.

### Solution Code
```python
import collections

def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    k = int(data[1])
    P = list(map(int, data[2:2+n]))
    
    pos_arr = [0] * (n + 1)
    for idx, value in enumerate(P):
        pos_arr[value] = idx
        
    arr = [0] * n
    for i in range(n):
        arr[i] = pos_arr[i + 1]
    
    if k == 1:
        print(0)
        return
        
    min_deque = collections.deque()
    max_deque = collections.deque()
    min_vals = [0] * n
    max_vals = [0] * n
    
    for i in range(n):
        if min_deque and min_deque[0] < i - k + 1:
            min_deque.popleft()
        if max_deque and max_deque[0] < i - k + 1:
            max_deque.popleft()
            
        while min_deque and arr[min_deque[-1]] > arr[i]:
            min_deque.pop()
        min_deque.append(i)
        
        while max_deque and arr[max_deque[-1]] < arr[i]:
            max_deque.pop()
        max_deque.append(i)
        
        if i >= k - 1:
            min_vals[i] = arr[min_deque[0]]
            max_vals[i] = arr[max_deque[0]]
            
    ans = 10**15
    for j in range(0, n - k + 1):
        end_index = j + k - 1
        span = max_vals[end_index] - min_vals[end_index]
        if span < ans:
            ans = span
            
    print(ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input consists of the permutation length `N`, the sequence length `K`, and the permutation itself.
2. **Position Array**: We create an array `pos_arr` where `pos_arr[value]` stores the 0-based index of `value` in the permutation.
3. **Position Array for Values**: The array `arr` is constructed such that `arr[i]` holds the position of the value `i+1` in the permutation.
4. **Special Case Handling**: If `K` is 1, the answer is 0 since any single element trivially satisfies the condition.
5. **Sliding Window for Min and Max**: Using two deques, we efficiently compute the minimum and maximum values in all sliding windows of size `K` in `arr`.
6. **Finding Minimum Span**: For each window, the span is the difference between the maximum and minimum positions. The smallest span across all windows is the solution.

This approach efficiently narrows down the problem to checking all possible consecutive integer sets in the permutation, leveraging sliding window techniques to maintain optimal performance.
