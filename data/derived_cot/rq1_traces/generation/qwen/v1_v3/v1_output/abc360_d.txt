            # Fenwick tree class
            class Fenw:
                def __init__(self, size):
                    self.n = size
                    self.tree = [0]*(self.n+1)
                
                def update(self, index, delta):
                    # index is the 1-indexed index in the Fenw tree
                    while index <= self.n:
                        self.tree[index] += delta
                        index += index & -index
                
                def query(self, index):
                    # prefix sum from 1 to index
                    s = 0
                    while index:
                        s += self.tree[index]
                        index -= index & -index
                    return s

                def range_query(self, l, r):
                    if l > r:
                        return 0
                    return self.query(r) - self.query(l-1)

To solve this problem, we need to count the number of pairs of ants (i, j) where i < j, such that the ants pass each other before time T + 0.1. The ants move in opposite directions (one left and one right) and their initial positions must be such that they meet within the given time frame.

### Approach
1. **Problem Analysis**: The problem involves ants moving in opposite directions on a number line. The key observation is that two ants will pass each other if they are moving towards each other (one left and one right) and their initial positions allow them to meet within the specified time. The meeting time is determined by the distance between their starting positions and their directions.

2. **Intuition**: For each ant, we need to check all previous ants (since i < j) to see if they form a valid pair. However, a brute-force approach would be too slow for large N (up to 200,000). Instead, we use a Fenwick Tree (Binary Indexed Tree) to efficiently count valid pairs by processing ants in order and querying the tree for positions that meet the distance condition.

3. **Algorithm Selection**: 
   - **Fenwick Tree**: This data structure is used to maintain counts of ant positions dynamically, allowing efficient range queries and updates.
   - **Integer Arithmetic**: To avoid floating-point precision issues, we transform the distance condition into an integer comparison. Specifically, the condition (X_i - X_j) < 2*(T + 0.1) is rewritten as 10*(X_i - X_j) < 20*T + 2.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(N log N) due to sorting and Fenwick tree operations (each update and query is O(log N)).
   - **Space Complexity**: O(N) for storing positions and the Fenwick tree.

### Solution Code
```python
import sys
from bisect import bisect_left, bisect_right

class Fenw:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)
    
    def update(self, index, delta):
        while index <= self.n:
            self.tree[index] += delta
            index += index & -index
    
    def query(self, index):
        s = 0
        while index:
            s += self.tree[index]
            index -= index & -index
        return s

    def range_query(self, l, r):
        if l > r:
            return 0
        return self.query(r) - self.query(l - 1)

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        print(0)
        return
    it = iter(data)
    N = int(next(it))
    T = int(next(it))
    S = next(it).strip()
    X = [int(next(it)) for _ in range(N)]
    
    all_positions_set = set(X)
    all_positions = sorted(all_positions_set)
    size = len(all_positions)
    mapping = {}
    for idx, pos in enumerate(all_positions):
        mapping[pos] = idx + 1
    
    threshold = 20 * T + 2
    fenw_case1 = Fenw(size)
    count_case1 = 0
    for j in range(N):
        if S[j] == '0':
            pos = X[j]
            if pos in mapping:
                idx_fenw = mapping[pos]
                fenw_case1.update(idx_fenw, 1)
        else:
            pos_j = X[j]
            low_bound = pos_j + 1
            high_bound = (threshold + 10 * pos_j) // 10
            left_idx = bisect_left(all_positions, low_bound)
            right_idx = bisect_right(all_positions, high_bound) - 1
            if left_idx <= right_idx and left_idx < size:
                count_case1 += fenw_case1.range_query(left_idx + 1, right_idx + 1)
    
    fenw_case2 = Fenw(size)
    count_case2 = 0
    for j in range(N):
        if S[j] == '1':
            pos = X[j]
            if pos in mapping:
                idx_fenw = mapping[pos]
                fenw_case2.update(idx_fenw, 1)
        else:
            pos_j = X[j]
            low_bound_val = (10 * pos_j - threshold) // 10 + 1
            high_bound_val = pos_j - 1
            if low_bound_val > high_bound_val:
                continue
            left_idx = bisect_left(all_positions, low_bound_val)
            right_idx = bisect_right(all_positions, high_bound_val) - 1
            if left_idx <= right_idx and left_idx < size:
                count_case2 += fenw_case2.range_query(left_idx + 1, right_idx + 1)
    
    print(count_case1 + count_case2)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of ants (N), time (T), directions (S), and initial positions (X).
2. **Position Compression**: The positions are sorted and mapped to a compressed range to facilitate efficient Fenwick tree operations.
3. **Fenwick Tree Setup**: Two Fenwick trees are initialized to handle counts of left-moving and right-moving ants separately.
4. **Case Handling**:
   - **Case 1**: For each right-moving ant, count left-moving ants before it that are to the right and within the time constraint.
   - **Case 2**: For each left-moving ant, count right-moving ants before it that are to the left and within the time constraint.
5. **Output**: The total count of valid pairs from both cases is printed.

This approach efficiently counts the valid pairs using integer arithmetic and Fenwick trees, ensuring optimal performance for large input sizes.
