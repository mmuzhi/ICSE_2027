To solve this problem, we need to find the N-th smallest good integer, where a good integer is defined as a non-negative integer whose all digits are even (0, 2, 4, 6, or 8). The challenge is to efficiently determine the N-th smallest such number without generating all possible good integers up to N, especially since N can be as large as 10^12.

### Approach
1. **Problem Analysis**: The key observation is that good integers can be mapped to numbers in base-5. Specifically, each digit in the base-5 representation of a number (excluding leading zeros) can be mapped to an even digit (0, 2, 4, 6, or 8) by multiplying each digit by 2. This mapping ensures that the sequence of good integers is generated in ascending numerical order.
2. **Insight**: The N-th good integer corresponds to the base-5 representation of (N-1), where each digit in this base-5 number is then mapped to an even digit by doubling it. This approach leverages the fact that the base-5 numbers, when converted to good integers via digit doubling, maintain their natural order.
3. **Algorithm Selection**: The algorithm involves:
   - Converting (N-1) into its base-5 representation.
   - Mapping each digit of this base-5 number to an even digit by doubling it.
   - Combining these digits to form the N-th good integer.
4. **Complexity Analysis**: The algorithm efficiently handles large N (up to 10^12) by processing the number in logarithmic time relative to N, specifically O(log5(N)), which is feasible even for the upper constraint.

### Solution Code
```python
def main():
    N_val = int(input().strip())
    x = N_val - 1
    if x == 0:
        print(0)
    else:
        digits = []
        num = x
        while num:
            digits.append(num % 5)
            num //= 5
        digits.reverse()
        s = ''.join(str(2 * d) for d in digits)
        print(int(s))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read as an integer N_val, which represents the position of the desired good integer in the sequence.
2. **Special Case Handling**: If N_val is 1, the output is 0, the first good integer.
3. **Base-5 Conversion**: For N_val greater than 1, the algorithm converts (N_val - 1) into its base-5 representation by repeatedly dividing the number by 5 and collecting remainders. These remainders are then reversed to form the base-5 digits in the correct order.
4. **Digit Mapping**: Each digit in the base-5 number is doubled to map it to an even digit (0, 2, 4, 6, or 8). These digits are concatenated to form the final good integer.
5. **Output**: The resulting number is printed as an integer, which is the N-th smallest good integer.

This approach efficiently narrows down the problem to base-5 conversion and digit manipulation, ensuring optimal performance even for very large values of N.
