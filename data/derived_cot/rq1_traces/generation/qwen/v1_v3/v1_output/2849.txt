- Subarray [3, 1] with an imbalance number of 1.

To solve this problem, we need to calculate the sum of imbalance numbers for all contiguous subarrays of a given array. The imbalance number of a subarray is defined as the number of indices in the sorted version of the subarray where the difference between consecutive elements is greater than 1.

### Approach
1. **Problem Analysis**: The problem requires us to consider every contiguous subarray of the input array, sort each subarray, and then count the number of gaps (consecutive elements in the sorted subarray with a difference greater than 1). The challenge is to efficiently compute this for all subarrays without resorting to a brute-force method, which would be too slow for large arrays.

2. **Intuition**: For each subarray, the imbalance number is determined by the gaps between consecutive elements in its sorted form. Instead of sorting each subarray from scratch, we can dynamically maintain the sorted order as we expand each subarray. By tracking the left and right neighbors of each element in the sorted subarray, we can efficiently compute the change in the number of gaps when a new element is added.

3. **Algorithm Selection**: We use a nested loop approach where the outer loop fixes the starting index of the subarray, and the inner loop expands the subarray to include subsequent elements. For each new element added, we insert it into a sorted list and update the count of gaps by examining its neighbors. The change in the number of gaps is calculated based on the differences between the new element and its neighbors.

4. **Complexity Analysis**: The algorithm involves iterating over all possible subarrays (O(n^2) for an array of size n). For each subarray, inserting an element into a sorted list takes O(n) time in the worst case, leading to an overall time complexity of O(n^3). Given the constraint n ≤ 1000, this approach may be borderline in terms of performance in Python, but it is correct and handles the problem requirements.

```python
                idx = bisect.bisect_left(sorted_list, x)
                sorted_list.insert(idx, x)
                L = sorted_list[idx-1] if idx > 0 else None
                R = sorted_list[idx+1] if idx < len(sorted_list)-1 else None
                if L is not None and R is not None:
                    new_gap_at_x = (1 if (x - L > 1) else 0) + (1 if (R - x > 1) else 0)
                    old_gap_at_LR = (1 if (R - L > 1) else 0)
                    delta = new_gap_at_x - old_gap_at_LR
                elif L is not None or R is not None:
                    new_gap_at_x = (1 if (x - L > 1) else 0) + (1 if (R - x > 1) else 0)
                    delta = new_gap_at_x
                else:
                    delta = 0
                gaps += delta
                res += gaps
        return res
```

### Explanation
1. **Initialization**: We start by initializing the result variable `res` to zero. This will store the sum of imbalance numbers for all subarrays.

2. **Outer Loop**: The outer loop iterates over each starting index `i` of the subarray.

3. **Inner Loop**: For each starting index `i`, the inner loop expands the subarray by including elements from `i` to the end of the array. For each new element `x` added to the subarray:
   - **Insertion**: The element `x` is inserted into the sorted list `sorted_list` at the correct position using binary search (via `bisect.bisect_left`).
   - **Neighbor Check**: The left and right neighbors of `x` in the sorted list are identified.
   - **Gap Calculation**: The change in the number of gaps (`delta`) is computed based on the differences between `x` and its neighbors. If both neighbors exist, the delta is the sum of new gaps created by `x` minus any gap that was previously present between its neighbors. If only one neighbor exists, the delta is simply the sum of new gaps involving `x`.

4. **Update Result**: The current count of gaps (`gaps`) is added to the result (`res`), which accumulates the imbalance numbers for all subarrays.

5. **Output**: After processing all subarrays, the result is returned.

This approach efficiently computes the required sum by dynamically tracking the sorted subarray and the gaps, ensuring optimal performance for the given constraints.
