=== ANSWER ===

To solve this problem, we need to find the minimum weight of a path from vertex 1 to every other vertex in a connected undirected graph. The weight of a path is defined as the sum of the weights of the vertices and edges encountered along the path. 

### Approach
1. **Problem Analysis**: The problem involves finding the shortest path in a graph where both vertices and edges have weights. The key observation is that the path weight includes the starting vertex's weight, each subsequent vertex's weight, and the edge weights between them. 
2. **Intuition**: We can model this problem using Dijkstra's algorithm, which efficiently finds the shortest paths in graphs with non-negative edge weights. The modification here is that the weight of a vertex is added when first visiting it (except the starting vertex, which is included initially).
3. **Algorithm Selection**: Dijkstra's algorithm is chosen because it efficiently handles graphs with non-negative weights and can be adapted to include vertex weights during traversal. The algorithm starts from vertex 1, exploring neighbors and updating their distances by adding the edge weight and the neighbor's vertex weight.
4. **Complexity Analysis**: The algorithm uses a priority queue (min-heap) to manage node exploration, leading to a time complexity of O((V + E) log V), where V is the number of vertices and E is the number of edges. This is efficient given the constraints (V and E up to 2×10^5).

### Solution Code
```python
import heapq
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    weights = list(map(int, data[2:2+n]))
    graph = [[] for _ in range(n)]
    index = 2 + n
    for _ in range(m):
        u = int(data[index])
        v = int(data[index+1])
        b = int(data[index+2])
        index += 3
        u -= 1
        v -= 1
        graph[u].append((v, b))
        graph[v].append((u, b))
    
    INF = 10**18
    dist = [INF] * n
    dist[0] = weights[0]
    heap = [(weights[0], 0)]
    heapq.heapify(heap)
    
    while heap:
        d, node = heapq.heappop(heap)
        if d != dist[node]:
            continue
        for neighbor, weight_edge in graph[node]:
            new_d = d + weight_edge + weights[neighbor]
            if new_d < dist[neighbor]:
                dist[neighbor] = new_d
                heapq.heappush(heap, (new_d, neighbor))
    
    print(" ".join(str(x) for x in dist[1:]))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints. The first two values are the number of vertices (n) and edges (m). The next n values are the vertex weights. The remaining lines describe the edges.
2. **Graph Construction**: The graph is built using an adjacency list where each entry stores connected vertices and the corresponding edge weight.
3. **Dijkstra's Algorithm**: The algorithm starts by initializing the distance of the starting vertex (vertex 1, index 0) to its weight. A priority queue is used to explore vertices in order of increasing path weight. For each vertex, its neighbors are processed by adding the edge weight and the neighbor's vertex weight to the current path weight. If this new weight is better than the previously recorded weight, it updates the distance and pushes the neighbor into the queue.
4. **Output**: After processing all vertices, the distances to all vertices except the starting one are printed in order.

This approach efficiently computes the minimum path weights by leveraging Dijkstra's algorithm with a slight modification to account for vertex weights during traversal. The solution handles large input sizes within feasible time complexity.
