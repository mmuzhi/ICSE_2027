You will be given a competitive programming problem.
There is an exam structured as follows.

- The exam consists of N rounds called round 1 to N.
- In each round, you are given an integer score between 0 and 100, inclusive.
- Your final grade is the sum of the N-2 of the scores earned in the rounds excluding the highest and lowest.
- Formally, let S=(S_1,S_2,\dots,S_N) be the sequence of the scores earned in the rounds sorted in ascending order, then the final grade is S_2+S_3+\dots+S_{N-1}.



Now, N-1 rounds of the exam have ended, and your score in round i was A_i.
Print the minimum score you must earn in round N for a final grade of X or higher.
If your final grade will never be X or higher no matter what score you earn in round N, print -1 instead.
Note that your score in round N can only be an integer between 0 and 100.

Input

The input is given from Standard Input in the following format:
N X
A_1 A_2 \dots A_{N-1}

Output

Print the answer.

Constraints


- All input values are integers.
- 3 \le N \le 100
- 0 \le X \le 100 \times (N-2)
- 0 \le A_i \le 100

Sample Input 1

5 180
40 60 80 50

Sample Output 1

70

It can be shown that 70 is the minimum score you must earn for a final grade of 180 or higher.

=== ANSWER ===

To solve this problem, we need to determine the minimum score a student must achieve in the final round of an exam to ensure their final grade meets or exceeds a given target. The final grade is calculated by summing all scores except the highest and lowest scores from all rounds.

### Approach
1. **Problem Analysis**: The exam consists of N rounds, with N-1 scores already given. The final grade is the sum of the middle N-2 scores when all N scores (including the final round's score) are sorted in ascending order. The goal is to find the smallest possible score in the final round (between 0 and 100) that meets or exceeds the target final grade. If it's impossible to achieve the target, we return -1.

2. **Key Insights**:
   - **Case 1**: If the score in the final round is the new minimum (≤ smallest existing score), the final grade is the sum of all existing scores minus the largest existing score.
   - **Case 2**: If the score in the final round is the new maximum (≥ largest existing score), the final grade is the sum of all existing scores minus the smallest existing score.
   - **Case 3**: If the score in the final round is between the smallest and largest existing scores, the final grade is the sum of all existing scores minus the smallest and largest existing scores plus the final round's score.

3. **Algorithm Selection**:
   - Calculate the sum of the given scores (`base`), the smallest score (`m0`), and the largest score (`M0`).
   - Check if the maximum possible final grade (when the final round's score is 100) is less than the target. If so, output -1.
   - Otherwise, determine the minimal score in each of the three cases (new minimum, new maximum, or middle value) that meets the target, and choose the smallest valid score among them.

4. **Complexity Analysis**: The solution involves basic arithmetic operations and comparisons, making it efficient with a constant time complexity O(1) since the number of rounds N is constrained to a maximum of 100.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    X_target = int(data[1])
    A = list(map(int, data[2:2+n-1]))
    
    base = sum(A)
    m0 = min(A)
    M0 = max(A)
    
    if base - m0 < X_target:
        print(-1)
    else:
        candidate = 101
        if base - M0 >= X_target:
            candidate = min(candidate, 0)
        candidate = min(candidate, M0)
        required = X_target - (base - m0 - M0)
        s0 = max(m0 + 1, required)
        if s0 < M0 and s0 <= 100:
            candidate = min(candidate, s0)
        print(candidate)

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read in one go for efficiency, then parsed into the number of rounds (n), the target grade (X_target), and the list of existing scores (A).
2. **Initial Checks**: The sum of existing scores (`base`), smallest (`m0`), and largest (`M0`) scores are computed. If the maximum possible final grade (base - m0) is less than the target, -1 is printed immediately.
3. **Case Handling**:
   - **Case 1**: If setting the final score to 0 (or any value ≤ m0) meets the target, 0 is a candidate.
   - **Case 2**: The minimal score in this case is M0 (since any score ≥ M0 will yield the maximum final grade).
   - **Case 3**: The minimal score here is the maximum between (m0 + 1) and the required score derived from the target. If this score is within bounds, it's considered.
4. **Output**: The smallest valid score from all cases is printed, or -1 if no solution exists. The solution efficiently checks all possible scenarios to determine the minimal score needed.
