We have a grid with H rows and W columns.
Let (i,j) denote the square at the i-th row from the top and j-th column from the left.
Each square in the grid is one of the following: the start square, the goal square, an empty square, a wall square, and a candy square.
(i,j) is represented by a character A_{i,j}, and is the start square if A_{i,j}= S, the goal square if A_{i,j}= G, an empty square if A_{i,j}= ., a wall square if A_{i,j}= #, and a candy square if A_{i,j}= o.
Here, it is guaranteed that there are exactly one start, exactly one goal, and at most 18 candy squares.
Takahashi is now at the start square.

Each candy square counts only once, even if it is visited multiple times.

Input

H W T

Output

If it is impossible to reach the goal square in at most T moves, print -1.
Otherwise, print the maximum number of candy squares that can be visited on the way to the goal square, where Takahashi must finish.

Constraints


- 1\leq H,W \leq 300
- 1 \leq T \leq 2\times 10^6
- H, W, and T are integers.
- A_{i,j} is one of S, G, ., #, and o.
- Exactly one pair (i,j) satisfies A_{i,j}= S.
- Exactly one pair (i,j) satisfies A_{i,j}= G.
- At most 18 pairs (i,j) satisfy A_{i,j}= o.

To solve this problem, we need to determine if it's possible for Takahashi to reach the goal square from the start square within a given number of moves (T) and, if possible, find the maximum number of distinct candy squares he can visit along the way. The solution involves using BFS to compute shortest paths between key points (start, goal, and candies) and then applying a dynamic programming approach to explore all possible paths that collect candies while staying within the move limit.

### Approach
1. **Problem Analysis**: The grid contains a start (S), goal (G), empty squares (.), walls (#), and candies (o). The goal is to reach G from S in at most T moves, collecting as many distinct candies as possible. The challenge is to efficiently explore all possible paths without exceeding the move limit, especially given the constraints (grid up to 300x300 and up to 18 candies).

2. **Key Insights**: 
   - **BFS for Shortest Paths**: Since the grid can have walls, BFS is used to compute the shortest path between any two key points (S, G, and candies). This helps in building a distance matrix where each entry represents the minimal steps between two points.
   - **Dynamic Programming (DP)**: After computing the distance matrix, a DP approach is used where the state is defined by a bitmask representing visited candies and the current position. The DP state tracks the minimal steps required to reach each state, allowing us to explore all possible paths efficiently.

3. **Algorithm Selection**:
   - **BFS**: For each key point (S, G, and candies), perform BFS to compute distances to all other key points. This ensures we have the shortest path between any two points in the set.
   - **DP with Bitmasking**: The DP state `dp[mask][node]` represents the minimal steps to visit the set of candies indicated by `mask` and end at `node`. The bitmask efficiently tracks which candies have been visited.

4. **Complexity Analysis**:
   - **BFS**: Each BFS runs in O(H*W) time, and there are O(1) BFS calls (since there are at most 20 key points). Thus, the total BFS time is O(20*H*W), which is feasible for H, W ≤ 300.
   - **DP**: The DP has 2^18 (≈ 262,144) states for the bitmask and 20 positions, leading to about 5.2 million operations, which is manageable in Python.
