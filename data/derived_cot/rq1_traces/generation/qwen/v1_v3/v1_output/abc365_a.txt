=== ANSWER ===

To solve this problem, we need to determine the number of days in a given year Y of the Gregorian calendar based on specific rules for leap years. The solution involves checking the year against these rules to decide whether it is a leap year (366 days) or a common year (365 days).

### Approach
The approach involves checking the year Y against the following conditions in order of their specificity:
1. If the year is a multiple of 400, it is a leap year (366 days).
2. If the year is a multiple of 100 but not a multiple of 400, it is not a leap year (365 days).
3. If the year is a multiple of 4 but not a multiple of 100, it is a leap year (366 days).
4. Otherwise, the year is a common year (365 days).

By following this order of checks, we ensure that the most specific conditions (like multiples of 400) are evaluated first, which is crucial for correctly identifying leap years according to the Gregorian calendar rules.

### Solution Code
```python
Y = int(input().strip())

if Y % 400 == 0:
    print(366)
elif Y % 100 == 0:
    print(365)
elif Y % 4 == 0:
    print(366)
else:
    print(365)
```

### Explanation
1. **Input Handling**: The code reads the integer input Y, which represents the year.
2. **Leap Year Check**: The code checks if Y is a multiple of 400. If true, it prints 366 (leap year).
3. **Century Year Check**: If Y is a multiple of 100 but not 400, it prints 365 (not a leap year).
4. **Divisible by 4 Check**: If Y is divisible by 4 but not by 100, it prints 366 (leap year).
5. **Default Case**: If none of the above conditions are met, the year is a common year with 365 days.

This approach efficiently narrows down the conditions to determine the correct number of days in the year Y by leveraging modular arithmetic to check divisibility. The solution is both concise and adheres to the Gregorian calendar rules.
