=== ANSWER ===

To solve this problem, we need to convert a sequence of 64 binary digits into their corresponding integer value. The sequence is given in little-endian order, where the first digit represents the least significant bit (2^0) and the last digit represents the most significant bit (2^63). 

### Approach
1. **Input Reading**: Read the input string and split it into a list of 64 individual bits.
2. **Bit Processing**: Iterate over each bit in the list. For each bit that is '1', calculate its contribution to the final integer by raising 2 to the power of its position index (0 to 63).
3. **Summation**: Accumulate the contributions of all '1' bits to get the final integer value.
4. **Output**: Print the accumulated sum.

The key insight here is recognizing that each bit's position directly corresponds to its power of two, allowing us to compute the integer value efficiently by leveraging bit shifting (1 << i) for each set bit.

### Solution Code
```python
def main():
    data = input().split()
    total = 0
    for i in range(len(data)):
        if data[i] == '1':
            total += (1 << i)
    print(total)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read as a single string and split into a list of strings, each representing a binary digit.
2. **Processing Bits**: For each bit in the list, if the bit is '1', the code calculates 2 raised to the power of the bit's position (using bit shifting for efficiency) and adds this value to a running total.
3. **Output**: After processing all bits, the total is printed, which is the integer representation of the binary sequence.

This approach efficiently converts the binary sequence into its integer form by leveraging the positional value of each bit and the properties of binary numbers. The use of bit shifting ensures optimal performance, especially given the constraints of handling up to 64 bits.
