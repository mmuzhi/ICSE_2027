The process continues as long as each subsequent roll results in a number greater than the previous highest roll. The challenge is to efficiently calculate this expected value modulo 998244353.

### Approach
1. **Problem Analysis**: The salary is the sum of payments made during the process, which depends on the sequence of die rolls. The expected value is the average salary over all possible sequences. The key insight is to use dynamic programming to compute the expected value by considering the contribution of each die face independently.
2. **Dynamic Programming Setup**: We define two arrays, `F` and `T`, where `F[x]` represents the expected salary starting from state `x` (current maximum roll), and `T[x]` is the sum of expected values from states `x+1` to `N`.
3. **Modular Arithmetic**: Since the result needs to be modulo 998244353, we use modular inverses for division operations. Specifically, we compute the modular inverse of `N` to handle division by `N` in the recurrence relation.
4. **Suffix Sum Array**: We precompute a suffix sum array `S_arr` where `S_arr[x]` is the sum of payments for die faces from `x+1` to `N`. This helps in efficiently calculating the sum of payments in the recurrence relation.
5. **Backward Iteration**: We iterate from the highest state `N` down to `0`, updating `T[x]` and `F[x]` based on previous values. The recurrence relations ensure that we efficiently compute the expected value without recalculating sums repeatedly.

mod = 998244353
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)

        F[x] = (inv_N * (S_arr[x] + T[x])) % mod
