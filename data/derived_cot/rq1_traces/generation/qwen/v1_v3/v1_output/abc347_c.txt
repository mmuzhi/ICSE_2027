=== ANSWER ===

To solve this problem, we need to determine if there exists a starting day in a week such that all given plans fall on holidays. The week consists of A+B days, with the first A days being holidays and the next B days being weekdays. Each plan is scheduled D_i days later from the starting day. The challenge is to check if there's any starting day that ensures all plans land on holidays.

### Approach
1. **Problem Analysis**: The key observation is that the week repeats every W = A + B days. For each plan D_i, the condition for it to be a holiday is that the day (starting day + D_i) modulo W must be less than A. This translates to checking if the computed interval for each plan does not cover all possible starting days.

2. **Insight**: For each plan D_i, we compute the complement of the valid starting days (those days that would make D_i a holiday). The valid starting days for all plans must be the intersection of these complements. If the union of all complements covers all possible starting days (0 to W-1), then no valid starting day exists; otherwise, there is at least one valid starting day.

3. **Algorithm Selection**: 
   - For each plan D_i, compute t_i = D_i % W.
   - Determine the complement interval for each t_i. If t_i < A, the complement is [A - t_i, W - t_i - 1]. If t_i >= A, the complement is [0, W - t_i - 1] or [W - t_i - 1 + 1, W - 1] if the interval is non-empty.
   - Merge all complement intervals and check if their total length equals W. If it does, output "No"; otherwise, "Yes".

4. **Complexity Analysis**: The algorithm processes each plan once, computes intervals, and merges them. The merging step is O(N log N) due to sorting, where N is the number of plans (up to 2×10^5). This is efficient given the constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    a = int(data[1])
    b = int(data[2])
    D_list = list(map(int, data[3:3+n]))
    W = a + b
    t_list = [(d % W) for d in D_list]

    intervals = []
    for t in t_list:
        if a - t - 1 >= 0:
            low = a - t
            high = W - t - 1
            intervals.append((low, high))
        else:
            low2 = W - t
            high2 = min(W-1, W + a - t - 1)
            if low2 <= high2:
                if low2 - 1 >= 0:
                    intervals.append((0, low2-1))
                if high2 + 1 <= W-1:
                    intervals.append((high2+1, W-1))
            else:
                intervals.append((0, W-1))
                
    if not intervals:
        print("Yes")
        return
        
    intervals.sort(key=lambda x: x[0])
    merged = []
    start, end = intervals[0]
    for i in range(1, len(intervals)):
        a_val, b_val = intervals[i]
        if a_val <= end + 1:
            end = max(end, b_val)
        else:
            merged.append((start, end))
            start, end = a_val, b_val
    merged.append((start, end))
    
    total_length = 0
    for (a_val, b_val) in merged:
        total_length += (b_val - a_val + 1)
        
    if total_length == W:
        print("No")
    else:
        print("Yes")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of plans (N), the number of holiday days (A), the number of weekday days (B), and the list of plans (D_list).
2. **Week Length Calculation**: The week length W is computed as A + B.
3. **Modulo Calculation**: For each plan D_i, compute t_i = D_i % W to handle the cyclical nature of the week.
4. **Interval Calculation**: For each t_i, determine the complement interval (days that would not make D_i a holiday). If t_i < A, the complement is [A - t_i, W - t_i - 1]. If t_i >= A, the complement is derived from [0, W - t_i - 1] or [W - t_i - 1 + 1, W - 1] if applicable.
5. **Merging Intervals**: All complement intervals are merged to form non-overlapping intervals. The total length of these merged intervals is checked against W. If the total length equals W, no valid starting day exists ("No"); otherwise, there is at least one ("Yes").
6. **Output**: Based on the merged interval length, the result is printed.

This approach efficiently checks for the existence of a valid starting day by leveraging interval arithmetic and merging, ensuring optimal performance even for large inputs.
