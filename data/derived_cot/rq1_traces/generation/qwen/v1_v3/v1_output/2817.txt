To solve this problem, we need to find the minimum cost required to make all characters of a binary string equal by performing a series of operations. The operations allowed are flipping a prefix or a suffix of the string, each with a specific cost. The key insight is that the minimal cost is determined by the sum of the minimal costs to remove each boundary between differing adjacent characters.

### Approach
1. **Problem Analysis**: The problem involves transforming a binary string into a uniform string (all '0's or all '1's) using two types of operations: flipping a prefix or flipping a suffix. The cost of each operation depends on the index at which it is applied. The challenge is to find the sequence of operations that results in a uniform string with the least total cost.

2. **Intuition**: The solution leverages the observation that each boundary (a position where adjacent characters differ) can be resolved by either flipping the left character (using a prefix operation) or the right character (using a suffix operation). The minimal cost to resolve each boundary is the lesser of the two operation costs associated with flipping the respective characters.

3. **Algorithm Selection**: The algorithm involves:
   - Identifying all boundaries in the string where adjacent characters differ.
   - For each boundary, calculating the minimal cost to resolve it by comparing the costs of flipping the left or right character.
   - Summing these minimal costs across all boundaries to get the total minimal cost.

4. **Complexity Analysis**: The algorithm processes each character in the string once to identify boundaries and compute the minimal costs, resulting in a linear time complexity of O(n), where n is the length of the string. This is efficient for large input sizes up to 10^5.

### Solution Code
```python
class Solution:
    def minimumCost(self, s: str) -> int:
        n = len(s)
        total_cost = 0
        i = 0
        while i < n - 1:
            if s[i] != s[i+1]:
                total_cost += min(i+1, n - i - 1)
                while i < n - 1 and s[i] != s[i+1]:
                    i += 1
            else:
                i += 1
        return total_cost
```

### Explanation
1. **Initialization**: The algorithm starts by initializing the total cost to zero and setting the initial index to 0.
2. **Boundary Identification**: The loop iterates through the string to identify boundaries where adjacent characters differ.
3. **Cost Calculation**: For each identified boundary at index `i`, the algorithm calculates the minimal cost to resolve it by comparing the cost of flipping the prefix up to `i` (cost `i+1`) and the cost of flipping the suffix starting at `i+1` (cost `n - i - 1`). The lesser of these two costs is added to the total cost.
4. **Boundary Resolution**: After resolving a boundary, the loop skips subsequent boundaries that are automatically resolved by the same operation (due to the nature of prefix and suffix flips).
5. **Result**: The loop continues until all boundaries are processed, and the accumulated total cost is returned as the result.

This approach efficiently computes the minimal cost by focusing on resolving each boundary independently and leveraging the optimal operation cost for each boundary, ensuring the solution is both correct and optimal.
