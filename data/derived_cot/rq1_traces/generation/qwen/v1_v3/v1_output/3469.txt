To solve this problem, we need to determine the maximum height of a triangle that can be formed using red and blue balls, where each row must consist of balls of the same color, and adjacent rows must have different colors. The triangle starts with 1 ball in the first row, 2 in the second, and so on. 

### Approach
1. **Problem Analysis**: The problem requires forming a triangle where each row alternates between red and blue balls. The key observation is that the sum of the balls in the rows of one color must not exceed the available count of that color. The solution involves checking two possible starting scenarios: starting with red or starting with blue.
2. **Insight**: For each possible height `h`, we calculate the total number of red and blue balls required for both starting scenarios. If either scenario's required balls do not exceed the given counts, the height is feasible.
3. **Algorithm Selection**: We iterate over possible heights starting from 1. For each height, we compute the required red and blue balls for both starting scenarios. If both scenarios fail to meet the color counts, we return the previous height as the maximum feasible height.
4. **Complexity Analysis**: The loop runs until the color counts are exhausted, which is efficient given the constraints (red and blue are up to 100). The worst-case time complexity is O(h), where h is the maximum feasible height, which is linear in the input size.

### Solution Code
```python
class Solution:
    def maxHeightOfTriangle(self, red: int, blue: int) -> int:
        h = 0
        while True:
            h += 1
            red1 = ((h + 1) // 2) ** 2
            blue1 = (h // 2) * (h // 2 + 1)
            blue2 = ((h + 1) // 2) ** 2
            red2 = (h // 2) * (h // 2 + 1)
            if (red >= red1 and blue >= blue1) or (red >= red2 and blue >= blue2):
                continue
            else:
                return h - 1
```

### Explanation
1. **Initialization**: We start with height `h` set to 0.
2. **Loop Through Heights**: For each increment of `h`, we calculate the number of red and blue balls needed for two scenarios:
   - **Scenario 1**: Start with red. The red balls are the sum of squares of the number of odd-numbered rows, and blue balls are the sum of even-numbered rows.
   - **Scenario 2**: Start with blue. The blue balls are the sum of squares of the number of odd-numbered rows, and red balls are the sum of even-numbered rows.
3. **Feasibility Check**: If either scenario's required balls do not exceed the given counts, the loop continues to the next height. Otherwise, the previous height is returned as the maximum feasible height.
This approach efficiently checks all possible triangle heights by leveraging mathematical sums of sequences, ensuring optimal performance for the given constraints.
