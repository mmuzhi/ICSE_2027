To solve this problem, we need to determine the minimum number of operations required to modify a given string such that no two adjacent characters are almost-equal. Two characters are considered almost-equal if they are the same or adjacent in the alphabet (i.e., their ASCII values differ by at most 1). 

### Approach
1. **Problem Analysis**: The problem requires us to ensure that no two adjacent characters in the string are almost-equal. Each operation allows us to change any character to any lowercase English letter. The goal is to minimize the number of such operations.
2. **Intuition**: The solution involves identifying maximal sequences of characters where each adjacent pair in the sequence is almost-equal. For each such sequence of length `L`, the minimal operations needed to break all adjacent almost-equal pairs within the sequence is `L // 2`. This is because changing every other character in the sequence ensures no two adjacent characters remain almost-equal.
3. **Algorithm Selection**: We traverse the string to identify these maximal sequences. For each sequence found, we calculate the required operations as the integer division of the sequence length by 2. The sum of these operations across all sequences gives the total minimal operations needed.
4. **Complexity Analysis**: The algorithm processes each character in the string exactly once, resulting in a linear time complexity of O(n), where n is the length of the string. This is efficient given the constraint that the string length is at most 100.

### Solution Code
```python
class Solution:
    def removeAlmostEqualCharacters(self, word: str) -> int:
        n = len(word)
        i = 0
        operations = 0
        while i < n:
            j = i
            while j + 1 < n and abs(ord(word[j+1]) - ord(word[j])) <= 1:
                j += 1
            L = j - i + 1
            operations += L // 2
            i = j + 1
        return operations
```

### Explanation
1. **Initialization**: We start by initializing the index `i` to 0 and the operations count to 0.
2. **Traversal**: For each starting index `i`, we expand the current sequence as far as possible. The sequence includes characters where each adjacent pair differs by at most 1 in their ASCII values (i.e., they are almost-equal).
3. **Operations Calculation**: For each identified sequence of length `L`, the number of operations needed is `L // 2`. This is because changing every other character in the sequence breaks all adjacent almost-equal pairs.
4. **Update Index**: After processing a sequence, we move the index `i` to the position right after the sequence to continue processing the remaining string.
5. **Result**: The total operations required to modify the entire string is returned as the result.

This approach efficiently breaks down the problem into manageable sequences and calculates the minimal operations by leveraging simple arithmetic on sequence lengths, ensuring optimal performance.
