=== ANSWER ===

To solve this problem, we need to find the maximum length of a non-decreasing array that can be obtained by performing any number of operations where each operation involves replacing a contiguous subarray with the sum of its elements. The key insight is that the problem reduces to partitioning the original array into contiguous segments such that the sequence of segment sums is non-decreasing, and we aim to maximize the number of segments.

### Approach
1. **Prefix Sum Calculation**: Compute the prefix sum array to efficiently calculate the sum of any contiguous subarray.
2. **Binary Search for Segment Partitioning**: For each segment starting at index `i`, use binary search to find the smallest `j` such that the sum of the subarray from `i` to `j` is at least the sum of the previous segment. This ensures the non-decreasing property.
3. **Handling Non-Partitionable Segments**: If no such `j` is found, extend the previous segment to include the remaining elements, updating the segment sum accordingly.
4. **Count Segments**: The number of valid segments found during the process is the answer, as each segment corresponds to a contiguous part of the original array, and their sums form a non-decreasing sequence.

### Solution Code
```python
class Solution:
    def findMaximumLength(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 0:
            return 0
        pre = [0] * (n + 1)
        for idx in range(n):
            pre[idx + 1] = pre[idx] + nums[idx]
        
        count = 0
        last = 0
        i = 0
        starts = []
        
        while i < n:
            low = i
            high = n - 1
            candidate = n
            while low <= high:
                mid = (low + high) // 2
                s = pre[mid + 1] - pre[i]
                if s >= last:
                    candidate = mid
                    high = mid - 1
                else:
                    low = mid + 1
            
            if candidate < n:
                last = pre[candidate + 1] - pre[i]
                count += 1
                starts.append(i)
                i = candidate + 1
            else:
                seg_start = starts[-1]
                last = pre[n] - pre[seg_start]
                i = n
        
        return count
```

### Explanation
1. **Prefix Sum Array**: The prefix sum array `pre` is computed to allow quick calculation of the sum of any subarray. `pre[i]` holds the sum of the first `i` elements of the array.
2. **Binary Search for Segment Ends**: For each starting index `i`, binary search is used to find the smallest `j` such that the sum of elements from `i` to `j` is at least the sum of the previous segment (`last`). This ensures the non-decreasing property of segment sums.
3. **Segment Count Update**: If a valid `j` is found, the segment count is incremented, and the start of the new segment is recorded. The search continues from `j + 1`.
4. **Handling Remaining Elements**: If no valid `j` is found, the previous segment is extended to include all remaining elements. The segment sum is updated to the total sum from the start of the previous segment to the end of the array.
5. **Result**: The total count of valid segments is returned, representing the maximum length of a non-decreasing array obtainable after any number of operations.

This approach efficiently partitions the array into the maximum number of contiguous segments with non-decreasing sums, leveraging binary search for optimal performance. The complexity is O(n log n) due to the binary search for each segment.
