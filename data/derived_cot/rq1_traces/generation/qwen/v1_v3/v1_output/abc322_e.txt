You will be given a competitive programming problem.
AtCoder Inc. is planning to develop a product. The product has K parameters, whose values are currently all zero. The company aims to raise all parameter values to at least P.
There are N development plans. Executing the i-th development plan (1 \le i \le N) increases the value of the j-th parameter by A_{i,j} for every integer j such that 1 \le j \le K, at the cost of C_i.
A development plan cannot be executed more than once. Determine whether the company can achieve its goal, and if it can, find the minimum total cost required to achieve the goal.

Input

The input is given from Standard Input in the following format:
N K P
C_1 A_{1,1} A_{1,2} \dots A_{1,K}
C_2 A_{2,1} A_{2,2} \dots A_{2,K}
\dots
C_N A_{N,1} A_{N,2} \dots A_{N,K}

Output

If AtCoder Inc. can achieve its goal, print the minimum total cost required to achieve the goal; otherwise, print -1.

Constraints


- 1 \le N \le 100
- 1 \le K,P \le 5
- 0 \le A_{i,j} \le P(1 \le i \le N,1 \le j \le K)
- 1 \le C_i \le 10^9(1 \le i \le N)
- All input values are integers.

Sample Input 1

4 3 5
5 3 0 2
3 1 2 3
3 2 4 0
1 0 1 4

Sample Output 1

9

If you execute the first, third, and fourth development plans, each parameter will be 3+2+0=5,0+4+1=5,2+0+4=6, all of which are at least 5, so the goal is achieved. The total cost in this case is 5 + 3 + 1 = 9.
It is impossible to achieve the goal at a total cost of 8 or less. Thus, the answer is 9.

To solve this problem, we need to determine whether it's possible to increase all parameters of a product to at least a specified value using a subset of available development plans, each of which can be used at most once. The goal is to find the minimum total cost required to achieve this, or return -1 if it's not possible.

### Approach
1. **Problem Analysis**: The problem involves selecting a subset of development plans such that each parameter of the product reaches at least a specified value \( P \). Each plan provides a certain increase to each parameter and has an associated cost. The challenge is to find the combination of plans that meets the parameter requirements with the least cost.

2. **Intuition**: The solution leverages dynamic programming (DP) to explore all possible combinations of parameter values achievable by the given plans. The DP state represents the current values of all parameters, capped at \( P \) since exceeding \( P \) does not provide additional benefit. The DP transitions involve considering each plan and updating the state by adding the plan's parameter increases, capped at \( P \).

3. **Algorithm Selection**: The DP approach efficiently explores the state space by iteratively updating possible parameter values as each plan is considered. The state space is manageable due to the constraints (each parameter value ranges from 0 to \( P \), and \( P \) is at most 5, leading to a state space of \( 6^5 = 7776 \) for 5 parameters).

4. **Complexity Analysis**: The algorithm processes each of the \( N \) plans, and for each plan, it updates all possible states in the DP. The number of states is \( (P+1)^K \), which is \( 7776 \) for the given constraints. Thus, the total operations are \( O(N \times (P+1)^K) \), which is feasible since \( N \) is up to 100 and \( (P+1)^K \) is 7776.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(-1)
        return
    
    it = iter(data)
    n = int(next(it))
    k = int(next(it))
    p = int(next(it))
    
    plans = []
    for _ in range(n):
        cost = int(next(it))
        a_list = [int(next(it)) for _ in range(k)]
        plans.append((cost, a_list))
    
    dp = {tuple(0 for _ in range(k)): 0}
    
    for cost, a_list in plans:
        new_dp = dp.copy()
        for state, total_cost in dp.items():
            new_state = []
            for j in range(k):
                new_val = state[j] + a_list[j]
                if new_val > p:
                    new_val = p
                new_state.append(new_val)
            new_state = tuple(new_state)
            new_total_cost = total_cost + cost
            if new_state not in new_dp or new_dp[new_state] > new_total_cost:
                new_dp[new_state] = new_total_cost
        dp = new_dp
    
    target = tuple(p for _ in range(k))
    print(dp.get(target, -1))

if __name__ == '__main__':
    main()
```

3. **Processing Plans**: For each plan, the algorithm updates the DP by considering the new states formed by applying the plan's parameter increases to each existing state. The new state values are capped at \( P \) to avoid unnecessary increases beyond the target.

4. **State Update**: After processing each plan, the DP is updated to include new states formed by combining the current plan with all previous states, ensuring the minimum cost for each achievable state is recorded.

5. **Result Extraction**: After processing all plans, the algorithm checks if the target state (all parameters at least \( P \)) is achievable. If it is, the minimum cost to reach this state is printed; otherwise, -1 is printed indicating it's not possible to meet the requirements.
