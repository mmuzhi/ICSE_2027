=== ANSWER ===

To solve this problem, we need to determine the minimum number of changes required to convert the binary representation of integer `n` into the binary representation of integer `k` by only changing bits that are set to 1 in `n` to 0. If it's impossible to make `n` equal to `k` using these operations, we should return -1.

### Approach
1. **Problem Analysis**: The key observation is that we can only change bits that are 1 in `n` to 0. Therefore, for `n` to be convertible to `k`, every bit that is 1 in `k` must also be 1 in `n`. If any bit is 1 in `k` but 0 in `n`, it's impossible to set that bit to 1, making the conversion impossible.
2. **Bitwise Comparison**: For each bit position from 0 to 20 (since numbers up to 10^6 can be represented with 20 bits), we compare the corresponding bits of `n` and `k`:
   - If `k` has a 1 and `n` has a 0 in the same bit position, return -1 immediately.
   - If `k` has a 0 and `n` has a 1, increment the change count.
3. **Result Calculation**: After processing all relevant bits, if no impossibility condition is met, return the total count of changes needed.

### Solution Code
```python
class Solution:
    def minChanges(self, n: int, k: int) -> int:
        changes = 0
        for i in range(0, 21):
            n_bit = (n >> i) & 1
            k_bit = (k >> i) & 1
            if k_bit == 1 and n_bit == 0:
                return -1
            elif k_bit == 0 and n_bit == 1:
                changes += 1
        return changes
```

### Explanation
- **Bitwise Shifting**: For each bit position `i`, we shift both `n` and `k` right by `i` positions and mask the result with 1 to isolate the `i-th` bit.
- **Impossibility Check**: If `k` requires a bit to be 1 but `n` has it as 0, we return -1 because we cannot set a 0 to 1.
- **Change Count**: For each bit where `n` has a 1 and `k` requires a 0, we increment the change count since we need to flip that bit from 1 to 0.
- **Efficiency**: The loop runs for a fixed number of iterations (21 times), making the solution efficient with a constant time complexity O(1). This is optimal given the constraints (numbers up to 10^6). 

This approach efficiently checks each bit position, ensuring correctness by leveraging bitwise operations and early termination upon encountering an impossible condition.
