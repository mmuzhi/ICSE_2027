To solve this problem, we need to compute the sum of the remainders of the sums of all pairs of elements in a given sequence when divided by \(10^8\). The challenge lies in efficiently calculating this sum without resorting to a brute-force approach, which would be computationally expensive for large sequences.

### Approach
1. **Problem Analysis**: The problem requires summing the remainders of the sums of all pairs of elements in a sequence when divided by \(10^8\). The remainder operation is straightforward, but the key insight is recognizing that the sum of all pairs can be broken down into two parts: the sum of all elements multiplied by (N-1) and subtracting \(10^8\) multiplied by the number of pairs whose sum is at least \(10^8\).

2. **Intuition**: For each pair of elements \((A_i, A_j)\), the remainder when \(A_i + A_j\) is divided by \(10^8\) is \(A_i + A_j - 10^8 \times \text{floor}((A_i + A_j)/10^8)\). Since \(A_i\) and \(A_j\) are less than \(10^8\), their sum is less than \(2 \times 10^8\), so the floor division result is either 0 or 1. This allows us to simplify the problem to counting the number of pairs whose sum is at least \(10^8\) and then adjusting the total sum accordingly.

3. **Algorithm Selection**: 
   - **Sorting**: First, sort the array to facilitate efficient counting of pairs using binary search.
   - **Binary Search**: For each element \(A_i\), determine how many elements \(A_j\) (where \(j > i\)) satisfy \(A_j \geq 10^8 - A_i\). This count is found using binary search (via `bisect_left`), which efficiently narrows down the valid elements in the sorted array.

4. **Complexity Analysis**: Sorting the array takes \(O(N \log N)\) time. The binary search for each element also takes \(O(\log N)\) time per element, leading to an overall time complexity of \(O(N \log N)\), which is efficient for the given constraints (N up to \(3 \times 10^5\)).

### Solution Code
```python
import bisect

def main():
    M = 10**8
    n = int(input().strip())
    A = list(map(int, input().split()))
    A.sort()
    total_sum = sum(A)
    count_ge = 0
    for i in range(n):
        target = M - A[i]
        j = bisect.bisect_left(A, target, i+1)
        if j < n:
            count_ge += (n - j)
    answer = (n - 1) * total_sum - M * count_ge
    print(answer)
```
