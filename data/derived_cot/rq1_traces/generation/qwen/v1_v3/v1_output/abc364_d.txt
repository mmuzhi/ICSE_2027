=== PROMPT ===
You will be given a competitive programming problem.
There are N+Q points A_1,\dots,A_N,B_1,\dots,B_Q on a number line, where point A_i has a coordinate a_i and point B_j has a coordinate b_j.
For each j=1,2,\dots,Q, answer the following question:

- Let X be the point among A_1,A_2,\dots,A_N that is the k_j-th closest to point B_j. Find the distance between points X and B_j.
More formally, let d_i be the distance between points A_i and B_j. Sort (d_1,d_2,\dots,d_N) in ascending order to get the sequence (d_1',d_2',\dots,d_N'). Find d_{k_j}'.

Input

The input is given from Standard Input in the following format:
N Q
a_1 a_2 \dots a_N
b_1 k_1
b_2 k_2
\vdots
b_Q k_Q

Output

Print Q lines.
The l-th line (1 \leq l \leq Q) should contain the answer to the question for j=l as an integer.

Constraints


- 1 \leq N, Q \leq 10^5
- -10^8 \leq a_i, b_j \leq 10^8
- 1 \leq k_j \leq N
- All input values are integers.

Sample Input 1

4 3
-3 -1 5 6
-2 3
2 1
10 4

Sample Output 1

7
3
13

=== ANSWER ===

To solve this problem, we need to efficiently answer multiple queries about the k-th closest point from a fixed set of points to various query points on a number line. The challenge lies in handling large input sizes (up to 10^5 points and queries) without resorting to brute-force methods, which would be computationally infeasible.

### Approach
1. **Problem Analysis**: For each query, we need to find the k-th smallest distance from a query point to any point in a fixed set of points. The straightforward approach of calculating all distances for each query and sorting them is too slow for large inputs.

2. **Intuition**: The key insight is that we can use binary search to determine the minimal distance such that at least k points lie within that distance from the query point. This leverages the fact that the sorted list of distances will have the k-th smallest distance as the smallest value where the count of points within that distance meets or exceeds k.

3. **Algorithm Selection**: 
   - **Binary Search**: For each query, perform a binary search on the possible distance values. The search range is from 0 to the maximum distance between the query point and any point in the fixed set.
   - **Bisect for Counting**: For each midpoint in the binary search, count how many points in the fixed set fall within the interval [query_point - midpoint, query_point + midpoint] using binary search (via `bisect` functions) on the sorted list of fixed points.

4. **Complexity Analysis**: 
   - **Preprocessing**: Sorting the fixed set of points takes O(N log N) time.
   - **Query Processing**: Each query involves O(log(max_distance)) binary search steps, each requiring O(log N) time for counting points in the interval. With Q queries, the total time is O(Q log(max_distance) log N), which is efficient for the given constraints.

### Solution Code
```python
import bisect

def main():
    import sys
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it))
    q = int(next(it))
    A = [int(next(it)) for _ in range(n)]
    queries = []
    for _ in range(q):
        b_val = int(next(it))
        k_val = int(next(it))
        queries.append((b_val, k_val))
    
    sorted_A = sorted(A)
    min_A = min(A)
    max_A = max(A)
    
    results = []
    for b, k in queries:
        d1 = abs(b - min_A)
        d2 = abs(b - max_A)
        high = max(d1, d2)
        low = 0
        
        while low < high:
            mid = (low + high) // 2
            L = b - mid
            R = b + mid
            left_index = bisect.bisect_left(sorted_A, L)
            right_index = bisect.bisect_right(sorted_A, R)
            count = right_index - left_index
            
            if count >= k:
                high = mid
            else:
                low = mid + 1
                
        results.append(str(low))
    
    sys.stdout.write("\n".join(results))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints. The input includes the number of fixed points (N), the number of queries (Q), the coordinates of the fixed points, and each query's point and k value.

2. **Preprocessing**: The fixed points are sorted, and their minimum and maximum values are stored to quickly determine the maximum possible distance for any query.

3. **Query Processing**: For each query:
   - **Binary Search Setup**: The search range is set from 0 to the maximum distance between the query point and any fixed point.
   - **Binary Search Execution**: For each midpoint, the interval [query_point - midpoint, query_point + midpoint] is checked. The number of fixed points within this interval is counted using binary search (via `bisect_left` and `bisect_right`).
   - **Adjusting Search Range**: If the count of points within the interval meets or exceeds k, the search continues in the lower half; otherwise, it continues in the upper half.

4. **Output**: The results for all queries are collected and printed at once for efficiency.

This approach efficiently handles the problem constraints by leveraging binary search and sorted data structures to minimize computation time per query.
