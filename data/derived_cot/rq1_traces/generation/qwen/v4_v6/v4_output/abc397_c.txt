To solve this problem, we need to find the maximum sum of the counts of distinct integers in two non-empty contiguous subarrays formed by splitting the given array at any position. The solution involves efficiently computing the distinct counts for all possible prefixes and suffixes of the array and then finding the optimal split point where the sum of the distinct counts of the left and right subarrays is maximized.

### Approach
1. **Problem Analysis**: The problem requires splitting the array into two non-empty parts and calculating the sum of distinct elements in each part. The challenge is to do this efficiently for large arrays (up to 300,000 elements) without resorting to a brute-force approach, which would be too slow.

2. **Intuition**: The key insight is that we can precompute two arrays:
   - **Left Array (`left_arr`)**: This array stores the count of distinct elements from the start of the array up to each index `i`.
   - **Right Array (`right_arr`)**: This array stores the count of distinct elements from each index `i` to the end of the array.

3. **Algorithm Selection**: 
   - **Frequency Tracking**: For the `left_arr`, we traverse the array from left to right, maintaining a frequency array to keep track of elements encountered. Each time a new element is encountered, we increment the distinct count.
   - **Similar for Right Array**: For the `right_arr`, we traverse the array from right to left, again using a frequency array to track distinct elements.
   - **Maximization**: After computing both arrays, we iterate through all possible split points (from 1 to N-1 in 0-based indexing) and compute the sum of `left_arr[i-1]` and `right_arr[i]` to find the maximum value.

4. **Complexity Analysis**: The algorithm involves two linear passes (one for each array) and another linear pass to find the maximum sum. Each pass is O(N), making the overall complexity O(N), which is optimal for the given constraints.
