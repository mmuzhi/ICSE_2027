- If there exists a j \lt C_q such that a wall exists at (R_q, j) and no wall exists at (R_q, k) for all j \lt k \lt C_q, destroy the wall at (R_q, j).

To solve this problem, we need to efficiently process a series of queries on a grid where each query involves destroying walls in specific directions until the first wall in each direction is encountered. The challenge is to handle these queries in an optimal manner given the constraints.

### Approach
1. **Initial Setup**: We start by reading the grid dimensions (H rows and W columns) and the number of queries (Q). Each query provides coordinates (R_q, C_q) for a bomb placement.
2. **Wall Tracking**: We maintain a set of all wall positions initially, where each cell (i, j) in the grid starts as a wall.
3. **Query Processing**: For each query:
   - **Center Check**: If the bomb is placed on a wall, we simply remove that wall from the set.
   - **Directional Check**: If the bomb is placed on an already destroyed cell, we check the four cardinal directions (up, down, left, right) to find the first wall in each direction. We then destroy those walls by removing them from the set.
4. **Result Calculation**: After processing all queries, the remaining number of walls is the size of the set, which we print as the result.

This approach leverages a set for O(1) average time complexity for checking and removing elements. However, the worst-case time complexity is O(Q*(H + W)) due to potential long scans in each direction for each query, which might not be optimal for large grids but works well for the given constraints.
