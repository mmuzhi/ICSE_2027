To solve this problem, we need to compute the sum of all possible substrings of a given string of digits, where each substring is interpreted as a decimal number. The challenge lies in efficiently calculating this sum without resorting to a brute-force approach, which would be computationally expensive for large strings.

### Approach
1. **Problem Analysis**: The problem requires summing the values of all substrings of a given string. A substring is a contiguous sequence of characters within the string, and its value is the number formed by interpreting those characters as digits. For example, the string "379" has substrings "3", "7", "9", "37", "79", and "379", which sum to 514.

2. **Insight**: The key insight is recognizing that each digit in the string contributes to the sum based on its position and the number of substrings that include it. Specifically, a digit at position `i` (0-indexed) will appear in `(i+1)` substrings as the most significant digit and in `n-i` substrings as a less significant digit, where `n` is the length of the string.

3. **Dynamic Programming**: We use a dynamic programming approach where `dp[i]` represents the sum of all substrings ending at position `i`. The recurrence relation for `dp[i]` is derived by considering:
   - The digit at position `i` alone, which contributes `digit * (i+1)` (since there are `i+1` substrings ending at `i` that include this digit as the most significant digit).
   - The sum of all substrings ending at `i-1`, each of which can be extended by the digit at `i`, multiplying their values by 10 and adding the digit.

4. **Efficiency**: The algorithm processes each digit in the string once, making it linear in time complexity, O(n), where `n` is the length of the string. This is efficient and feasible for large strings up to 200,000 characters.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    n = int(data[0].strip())
    S = data[1].strip()
    
    if n == 0:
        print(0)
        return
        
    dp = [0] * n
    dp[0] = int(S[0])
    total = dp[0]
    
    for i in range(1, n):
        digit_val = int(S[i])
        dp[i] = digit_val * (i + 1) + 10 * dp[i - 1]
        total += dp[i]
        
    print(total)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The code reads the input values, including the string length `n` and the string `S` itself. If the string is empty, it prints 0 and exits.
2. **Dynamic Programming Array**: The `dp` array is initialized to store the sum of substrings ending at each position. The first element `dp[0]` is simply the first digit of the string.
3. **Processing Each Digit**: For each subsequent digit (from index 1 to `n-1`), the code calculates `dp[i]` as the sum of:
   - The digit multiplied by the number of new substrings it forms (which is `i+1`).
   - Ten times the sum of substrings ending at the previous position (`10 * dp[i-1]`), accounting for extending each substring ending at `i-1` by the current digit.
4. **Summing Results**: The total sum is accumulated by adding each `dp[i]` to a running total.
5. **Output**: The final result is printed, which is the sum of all substring values.

This approach efficiently computes the desired sum by leveraging dynamic programming to avoid the O(n^2) complexity of a brute-force solution, making it suitable for large input sizes.
