To solve this problem, we need to find the minimum value of the expression \((\max_{i \in S} A_i) \times (\sum_{i \in S} B_i)\) for a subset \(S\) of size \(K\) from a given set of elements. The solution involves efficiently selecting elements such that the product of the maximum \(A_i\) in the subset and the sum of \(B_i\) values is minimized.

### Approach
1. **Problem Analysis**: The problem requires selecting a subset of size \(K\) from sequences \(A\) and \(B\) such that the product of the maximum \(A_i\) in the subset and the sum of \(B_i\) values is minimized. The challenge is to efficiently determine the optimal subset without resorting to brute-force methods, which would be computationally expensive.

2. **Intuition**: The key insight is that for any candidate maximum \(A_i\) (denoted as \(x\)), the optimal subset must include at least one element with \(A_i = x\) and the remaining elements should be the smallest \(B_i\) values from elements with \(A_i \leq x\). This reduces the problem to finding, for each distinct \(x\) in \(A\), the minimal sum of \(K\) elements (including at least one \(x\)) from elements with \(A_i \leq x\).

3. **Algorithm Selection**: 
   - **Sorting and Grouping**: First, combine the sequences \(A\) and \(B\) into tuples and sort them by \(A_i\). Group elements by their \(A_i\) values to process them in increasing order.
   - **Max-Heap for Efficient Selection**: Use a max-heap (simulated with a min-heap storing negative values) to keep track of the smallest \(K\) \(B_i\) values encountered so far. This allows efficient updates and queries.
   - **Sliding Window and Batch Processing**: For each distinct \(A_i\) value \(x\), process all elements with \(A_i = x\). After each insertion, maintain the heap size to exactly \(K\) by removing the largest elements (if necessary). The minimal sum for the current \(x\) is then derived from the heap, ensuring at least one element from the current batch is included.

4. **Complexity Analysis**: The algorithm processes each test case in \(O(N \log N)\) time due to sorting and heap operations, where \(N\) is the number of elements. Given the constraints (sum of \(N\) across test cases is \(2 \times 10^5\)), this approach is efficient and feasible.
