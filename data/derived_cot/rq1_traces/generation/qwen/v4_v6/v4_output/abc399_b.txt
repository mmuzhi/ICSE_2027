=== ANSWER ===

To solve this problem, we need to determine the ranks of N participants in a contest based on their scores. The ranking procedure involves repeatedly identifying the highest score among the remaining participants, assigning the current rank to all participants with that score, and then updating the rank variable by adding the count of participants with that score. 

### Approach
1. **Input Reading**: Read the number of participants (N) and their respective scores.
2. **Data Preparation**: Create a list of tuples where each tuple consists of a participant's score and their original index (to maintain the input order).
3. **Sorting**: Sort this list in descending order based on the scores to process the highest scores first.
4. **Rank Assignment**: Iterate through the sorted list, grouping participants with the same score. Assign the current rank to all participants in the current group, then update the rank by adding the size of the group.
5. **Output**: After processing all groups, output the ranks in the original input order.

### Solution Code
```python
n = int(input())
p_list = list(map(int, input().split()))
result = [0] * n
arr = [(p_list[i], i) for i in range(n)]
arr.sort(key=lambda x: x[0], reverse=True)

r = 1
i = 0
while i < n:
    j = i
    while j < n and arr[j][0] == arr[i][0]:
        j += 1
    count = j - i
    for k in range(i, j):
        original_index = arr[k][1]
        result[original_index] = r
    r += count
    i = j

for i in range(n):
    print(result[i])
```

### Explanation
1. **Input Reading**: The code starts by reading the number of participants (N) and their scores.
2. **Data Preparation**: It creates a list of tuples where each tuple contains a participant's score and their original index. This helps in maintaining the input order during output.
3. **Sorting**: The list of tuples is sorted in descending order based on scores. This allows the algorithm to process the highest scores first.
4. **Rank Assignment**: The algorithm processes the sorted list by grouping participants with the same score. For each group, it assigns the current rank to all participants in the group and updates the rank by adding the size of the group.
5. **Output**: Finally, the ranks are printed in the original input order, ensuring each participant's rank is correctly displayed as per their position in the input.

This approach efficiently handles the ranking by leveraging sorting and grouping, ensuring optimal performance even for the upper constraint limits.
