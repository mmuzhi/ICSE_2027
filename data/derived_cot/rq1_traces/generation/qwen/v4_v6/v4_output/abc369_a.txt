=== ANSWER ===

To solve this problem, we need to determine the number of integers \( x \) such that the three integers \( A \), \( B \), and \( x \) can be arranged in some order to form an arithmetic sequence. An arithmetic sequence of three numbers \( p, q, r \) satisfies the condition \( q - p = r - q \), which simplifies to \( 2q = p + r \). 

### Approach
1. **Problem Analysis**: The key observation is that for three numbers to form an arithmetic sequence, one of the numbers must be the average of the other two. This leads us to three possible conditions:
   - \( 2A = B + x \)
   - \( 2B = A + x \)
   - \( 2x = A + B \)
   
   Each condition corresponds to one of the three numbers being the middle element in the arithmetic sequence. 

2. **Intuition**: By solving these three equations for \( x \), we can generate all possible values of \( x \) that satisfy the problem's condition. The solution involves:
   - Calculating \( x \) from the first two equations directly.
   - Checking if \( x \) from the third equation is an integer (i.e., \( A + B \) must be even for \( x \) to be an integer).
   - Collecting all distinct \( x \) values from these conditions.

3. **Algorithm Selection**: The algorithm involves:
   - Using a set to store distinct \( x \) values to avoid duplicates.
   - Adding \( 2A - B \) and \( 2B - A \) to the set.
   - Adding \( (A + B) // 2 \) to the set if \( A + B \) is even.
   - The size of the set gives the number of valid integers \( x \).

4. **Complexity Analysis**: The approach involves constant-time operations (set insertions and arithmetic checks), making it \( O(1) \) in time and space complexity, which is efficient for the given constraints.

### Solution Code
```python
def main():
    A, B = map(int, input().split())
    s = {2 * A - B, 2 * B - A}
    if (A + B) % 2 == 0:
        s.add((A + B) // 2)
    print(len(s))

if __name__ == '__main__':
    main()
```
