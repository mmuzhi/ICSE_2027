Takahashi, a patissier working at the ABC pastry shop, decided to sell assorted cakes to commemorate AtCoder Beginner Contest 400.
The shop sells N kinds of cakes: cake 1, cake 2, \ldots, cake N.
Each cake has three non-negative integer values: beauty, tastiness, and popularity. Specifically, cake i has beauty X_i, tastiness Y_i, and popularity Z_i.
He considers pairing up these cakes into K pairs without overlaps.
Formally, he will choose 2K distinct integers a_1,b_1,a_2,b_2,\ldots,a_K,b_K between 1 and N (inclusive), and pair cake a_i with cake b_i.
The price of a pair formed by cakes a_i and b_i is \max(X_{a_i} + X_{b_i},\, Y_{a_i} + Y_{b_i},\, Z_{a_i} + Z_{b_i}).
Here, \max(P,Q,R) denotes the greatest value among P,Q,R.
Find the maximum possible total price of the K pairs.
You are given T test cases; solve each of them.

Input

The input is given from Standard Input in the following format:
T

\vdots

N K
X_1 Y_1 Z_1
X_2 Y_2 Z_2
\vdots
X_N Y_N Z_N

Output

Print T lines. The i-th line (1\leq i\leq T) should contain the answer to the i-th test case.

Constraints


- 1\leq T\leq 1000
- 2\leq N \leq 10^5
- The sum of N over all test cases in each input file is at most 10^5.
- 1\leq K \leq \lfloor \frac{N}{2}\rfloor (For a real number x, \lfloor x\rfloor denotes the greatest integer not exceeding x.)
- 0\leq X_i,Y_i,Z_i \leq 10^9
- All input values are integers.

Sample Output 1

12

To solve this problem, we need to maximize the total price of forming K pairs of cakes, where the price of each pair is the maximum of the sums of their beauty, tastiness, and popularity values. The challenge is to efficiently find the optimal pairs without resorting to brute-force methods, especially given the constraints where N can be as large as 100,000.

### Approach
1. **Problem Analysis**: The problem requires forming K disjoint pairs of cakes such that the sum of the maximum values (from beauty, tastiness, and popularity) of each pair is maximized. The key observation is that the optimal solution can be derived by considering each of the three metrics (beauty, tastiness, popularity) separately. For each metric, we select the top 2K cakes (since each pair requires two cakes) and pair them in descending order of the metric's value. The total price for each metric's pairing is the sum of the maximum values (from all three metrics) of each pair. The maximum of these three totals across all metrics gives the solution.

2. **Intuition**: By focusing on each metric individually, we ensure that the selected pairs are the best possible for that metric. The total price for each metric's pairing is computed by considering the maximum value (from all three metrics) for each pair. The highest total price among these three computations will be the answer, as it guarantees that at least one metric's pairing contributes to the maximum possible sum.

3. **Algorithm Selection**: For each test case, we process the cakes three times—once for each metric (beauty, tastiness, popularity). For each metric, we sort the cakes in descending order based on that metric's value, select the top 2K cakes, and pair them consecutively. The total price for each metric is the sum of the maximum values (from all three metrics) of each pair. The solution is the maximum of these three totals.

4. **Complexity Analysis**: The algorithm involves sorting the list of cakes three times (once per metric), each taking O(N log N) time. Since N can be up to 100,000 and the sum of N across all test cases is also up to 100,000, the total time complexity is O(3 * N log N), which is efficient for the given constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    t = int(data[0])
    index = 1
    results = []
    
    for _ in range(t):
        N = int(data[index])
        index += 1
        K = int(data[index])
        index += 1
        cakes = []
        for i in range(N):
            x = int(data[index])
            y = int(data[index+1])
            z = int(data[index+2])
            index += 3
            cakes.append((x, y, z))
            
        if K == 0:
            results.append("0")
            continue
            
        T_vals = [0, 0, 0]
        
        for metric in range(3):
            sorted_cakes = sorted(cakes, key=lambda cake: cake[metric], reverse=True)
            selected = sorted_cakes[:2*K]
            for i in range(0, 2*K, 2):
                cake1 = selected[i]
                cake2 = selected[i+1]
                s1 = cake1[0] + cake2[0]
                s2 = cake1[1] + cake2[1]
                s3 = cake1[2] + cake2[2]
                T_vals[metric] += max(s1, s2, s3)
                
        answer = max(T_vals)
        results.append(str(answer))
        
    print("\n".join(results))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first value is the number of test cases, T.
2. **Processing Each Test Case**: For each test case, we read N (number of cakes) and K (number of pairs to form). Then, we read the details of each cake (beauty, tastiness, popularity).
3. **Metric Processing**: For each metric (0 for beauty, 1 for tastiness, 2 for popularity), we sort the cakes in descending order based on that metric's value. We then select the top 2K cakes.
4. **Pairing and Price Calculation**: The selected cakes are paired consecutively (highest with next highest, etc.). For each pair, we compute the sums of beauty, tastiness, and popularity, and take the maximum of these three sums. This maximum is added to the total price for the current metric.
5. **Result Calculation**: After processing all three metrics, the maximum total price among them is the answer for the test case.
6. **Output**: The results for all test cases are printed at the end.

This approach efficiently narrows down the problem to three separate but similar subproblems, leveraging sorting and pairing to find the optimal solution without excessive computational overhead.
