- You are given positive integers l_i,r_i,L_i,R_i. Print Yes if it is possible to rearrange the subsequence (A_{l_i},A_{l_i+1},\ldots,A_{r_i}) to match the subsequence (B_{L_i},B_{L_i+1},\ldots,B_{R_i}), and No otherwise.

Input

The input is given from Standard Input in the following format:
N Q
A_1 A_2 \ldots A_N
B_1 B_2 \ldots B_N
l_1 r_1 L_1 R_1
l_2 r_2 L_2 R_2
\vdots
l_Q r_Q L_Q R_Q

Output

Print Q lines. The i-th line should contain the answer to the i-th query.

Constraints


-  1\leq N,Q\leq 2\times 10^5
-  1\leq A_i,B_i\leq N
-  1\leq l_i \leq r_i\leq N
-  1\leq L_i \leq R_i\leq N
- All input values are integers.

The key observation is that if the lengths of the subsequences differ, the answer is immediately "No". If the lengths are the same, we need to verify that both subsequences contain the exact same elements with the same frequencies. 

2. **Intuition**: Instead of comparing the entire subsequences for each query (which would be inefficient for large inputs), we use a hashing technique. We assign a random 64-bit integer to each number in the range [1, N]. For each array (A and B), we compute prefix sums of these random values. For any subsequence, the sum of the random values of its elements serves as a hash. If the hashes of the two subsequences match and their lengths are equal, the subsequences are multisets of each other.

3. **Algorithm Selection**: 
   - **Prefix Sums**: We precompute prefix sums for both arrays A and B using two different random value assignments (R1 and R2) to minimize the chance of hash collisions.
   - **Query Processing**: For each query, we first check if the lengths of the subsequences are equal. If not, the answer is "No". If lengths match, we compute the hash values for both subsequences using the prefix sums. If both hashes match, the answer is "Yes"; otherwise, it's "No".

4. **Complexity Analysis**: 
   - **Preprocessing**: O(N) time to generate random values and compute prefix sums for both arrays.
   - **Query Processing**: O(1) per query after preprocessing, as each query involves simple arithmetic operations and comparisons.
   - **Space Complexity**: O(N) to store the prefix sums and random values.

### Solution Code
```python
import sys
import random

def main():
    data = sys.stdin.read().split()
    it = iter(data)
    n = int(next(it)); q = int(next(it))
    A = [int(next(it)) for _ in range(n)]
    B = [int(next(it)) for _ in range(n)]
    
    R1 = [0]*(n+1)
    R2 = [0]*(n+1)
    for i in range(1, n+1):
        R1[i] = random.getrandbits(64)
        R2[i] = random.getrandbits(64)
    
    PA1 = [0]*(n+1)
    PA2 = [0]*(n+1)
    for i in range(1, n+1):
        PA1[i] = PA1[i-1] + R1[A[i-1]]
        PA2[i] = PA2[i-1] + R2[A[i-1]]
    
    PB1 = [0]*(n+1)
    PB2 = [0]*(n+1)
    for i in range(1, n+1):
        PB1[i] = PB1[i-1] + R1[B[i-1]]
        PB2[i] = PB2[i-1] + R2[B[i-1]]
    
    out_lines = []
    for _ in range(q):
        l = int(next(it)); r = int(next(it)); L_val = int(next(it)); R_val = int(next(it))
        lenA = r - l + 1
        lenB = R_val - L_val + 1
        if lenA != lenB:
            out_lines.append("No")
            continue
        
        a = l - 1
        b = r - 1
        segA1 = PA1[b+1] - PA1[a]
        segA2 = PA2[b+1] - PA2[a]
        
        c = L_val - 1
        d = R_val - 1
        segB1 = PB1[d+1] - PB1[c]
        segB2 = PB2[d+1] - PB2[c]
        
        if segA1 == segB1 and segA2 == segB2:
            out_lines.append("Yes")
        else:
            out_lines.append("No")
    
    print("\n".join(out_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first two values are N (length of arrays A and B) and Q (number of queries).
2. **Random Value Assignment**: Two arrays, R1 and R2, are generated where each entry holds a random 64-bit integer. These arrays map each number from 1 to N to a unique random value.
3. **Prefix Sum Calculation**: For both arrays A and B, prefix sums are computed using the random values. This allows efficient range sum queries.
4. **Query Processing**: For each query, the lengths of the specified subsequences are compared. If they differ, the answer is "No". If lengths match, the sum of random values for both subsequences is compared. If both sums match, the answer is "Yes"; otherwise, it's "No".
5. **Output**: The results for all queries are collected and printed at the end for efficiency.

This approach efficiently checks for multiset equality using hashing, ensuring optimal performance even for large input sizes.
