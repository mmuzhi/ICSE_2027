If a fruit type cannot be placed in any basket, it remains unplaced.

Return the number of fruit types that remain unplaced after all possible allocations are made.
 
Example 1:

Input: fruits = [4,2,5], baskets = [3,5,4]
Output: 1
Explanation:

fruits[0] = 4 is placed in baskets[1] = 5.
fruits[1] = 2 is placed in baskets[0] = 3.
fruits[2] = 5 cannot be placed in baskets[2] = 4.

Since one fruit type remains unplaced, we return 1.

fruits[2] = 1 is placed in baskets[1] = 4.

Since all fruits are successfully placed, we return 0.

      class SegmentTree:
          def __init__(self, data):
              self.n = len(data)
              self.size = 1
              while self.size < self.n:
                  self.size *= 2
              self.tree = [float('inf')] * (2 * self.size)
              # Build the tree: the leaves are from [self.size, self.size + self.n)
              for i in range(self.n):
                  self.tree[self.size + i] = data[i]
              for i in range(self.size - 1, 0, -1):
                  self.tree[i] = min(self.tree[2*i], self.tree[2*i+1])

          def update(self, index, value):
              # index is the index in the original data (0 to n-1)
              pos = self.size + index
              self.tree[pos] = value
              while pos > 1:
                  pos //= 2
                  self.tree[pos] = min(self.tree[2*pos], self.tree[2*pos+1])

          def query(self, l, r):
              # [l, r] inclusive
              l += self.size
              r += self.size
              res = float('inf')
              while l <= r:
                  if l % 2 == 1:
                      res = min(res, self.tree[l])
                      l += 1
                  if r % 2 == 0:
                      res = min(res, self.tree[r])
                      r -= 1
                  l //= 2
                  r //= 2
              return res

### Approach
1. **Problem Analysis**: The problem requires matching each fruit to the smallest index basket that can hold it, ensuring each basket is used at most once. The key insight is to efficiently find the leftmost available basket for each fruit without resorting to a brute-force O(n^2) approach.

2. **Intuition**: By sorting the basket capacities and using a segment tree to keep track of the smallest available basket index for each capacity, we can efficiently query and update the available baskets. This allows us to handle each fruit in logarithmic time relative to the number of baskets.

3. **Algorithm Selection**: 
   - **Segment Tree**: Used to maintain the minimum index of available baskets for each capacity. This enables efficient range queries to find the smallest index basket with capacity >= current fruit's quantity.
   - **Binary Search**: Helps in quickly locating the smallest capacity in the sorted list of basket capacities that is >= the current fruit's quantity.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n log n) due to sorting and segment tree operations (each query and update is O(log n)).
   - **Space Complexity**: O(n) for storing the segment tree and auxiliary data structures.
