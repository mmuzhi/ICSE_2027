Toy i\ (1 \leq i \leq N) has a size of A_i, and box i\ (1 \leq i \leq N-1) has a size of B_i.

Determine whether there exists a value of x such that he can execute step 2, and if it exists, find the minimum such x.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \dots A_N
B_1 B_2 \dots B_{N-1}

Output

If there exists a value of x such that Takahashi can execute step 2, print the minimum such x. Otherwise, print -1.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq A_i, B_i \leq 10^9
- All input values are integers.

4
5 2 3 7
6 2 8

Sample Output 1

3

To solve this problem, we need to determine the smallest box size Takahashi should purchase so that all toys can be placed into separate boxes, either existing or the newly purchased one, with each box's size being at least the toy's size. 

### Approach
1. **Problem Analysis**: The problem requires us to ensure that after purchasing one additional box, all toys can be placed into boxes such that each box's size is not less than the toy's size. The key insight is that the new box must be large enough to accommodate the smallest toy that cannot fit into any of the existing boxes when the remaining toys are optimally matched with the existing boxes.

2. **Intuition**: The solution involves sorting both the list of toy sizes and the list of existing box sizes. For each toy, we check if removing it allows the remaining toys to be matched with the existing boxes such that each remaining toy's size is less than or equal to the corresponding box's size in the sorted order. The smallest such toy size that satisfies this condition is our answer.

3. **Algorithm Selection**: 
   - **Sorting**: Both the toy sizes and existing box sizes are sorted in ascending order to facilitate a greedy matching approach.
   - **Prefix and Suffix Arrays**: We precompute two arrays:
     - `left_cond` checks if the first `i` smallest toys can fit into the first `i` smallest boxes.
     - `right_cond` checks if the remaining toys (after removing a toy) can fit into the remaining boxes.
   - **Efficient Checking**: Using prefix and suffix arrays, we efficiently determine for each candidate toy whether the matching condition holds, allowing us to find the smallest valid box size in linear time.

4. **Complexity Analysis**: 
   - **Time Complexity**: The dominant operations are sorting the lists, which takes O(N log N) time, and then processing the lists with prefix and suffix arrays, which takes O(N) time. Thus, the overall complexity is O(N log N).
   - **Space Complexity**: We use additional space for the prefix and suffix arrays, which is O(N), making the space complexity O(N).

import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    B = list(map(int, data[1+n:1+n+n-1]))

    A_sorted = sorted(A)
    B_sorted = sorted(B)
    M = n - 1

    left_cond = []
    for i in range(M):
        if A_sorted[i] <= B_sorted[i]:
            left_cond.append(True)
        else:
            left_cond.append(False)

    prefix_left = [True] * (n + 1)
    for i in range(1, n + 1):
        if i - 1 < M:
            prefix_left[i] = prefix_left[i - 1] and left_cond[i - 1]
        else:
            prefix_left[i] = prefix_left[i - 1]

    right_cond = []
    for i in range(M):
        if A_sorted[i + 1] <= B_sorted[i]:
            right_cond.append(True)
        else:
            right_cond.append(False)

    suffix_right = [False] * M
    if M > 0:
        suffix_right[M - 1] = right_cond[M - 1]
        for i in range(M - 2, -1, -1):
            suffix_right[i] = right_cond[i] and suffix_right[i + 1]

    ans = float('inf')
    for i in range(n):
        if i < M:
            left_part = prefix_left[i]
            right_part = suffix_right[i]
        else:
            left_part = prefix_left[i]
            right_part = True

        if left_part and right_part:
            if A_sorted[i] < ans:
                ans = A_sorted[i]

    print(ans if ans != float('inf') else -1)

if __name__ == '__main__':
    main()

1. **Reading Input**: The input is read and parsed into the number of toys `n`, the list of toy sizes `A`, and the list of existing box sizes `B`.
2. **Sorting**: Both `A` and `B` are sorted to facilitate the greedy matching process.
3. **Left Condition Check**: For each position in the sorted toy list, we check if the toy can fit into the corresponding box in the sorted box list. This forms the `left_cond` array.
4. **Prefix Array**: The `prefix_left` array stores cumulative results of the `left_cond` checks to quickly determine if the first `i` toys can fit into the first `i` boxes.
5. **Right Condition Check**: For each position in the sorted toy list (except the last), we check if the next toy can fit into the corresponding box. This forms the `right_cond` array.
6. **Suffix Array**: The `suffix_right` array stores cumulative results of the `right_cond` checks to quickly determine if the remaining toys can fit into the remaining boxes after removing a toy.
7. **Finding Minimum Box Size**: For each toy, we check if removing it allows the remaining toys to fit into the existing boxes using the precomputed arrays. The smallest valid toy size is our answer.
8. **Output**: The smallest valid box size is printed, or -1 if no such size exists. 

This approach efficiently checks all possible candidates for the smallest box size by leveraging sorting and prefix/suffix arrays, ensuring optimal performance even for large input sizes.
