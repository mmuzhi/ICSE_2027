3. **State Transition**: For each round, based on Alice's move and Bob's possible moves (excluding the last move to avoid consecutive duplicates), we update the DP state by calculating the new score difference. The DP table is updated iteratively for each round.

4. **Initialization**: For the first round, all three possible moves are considered, and the initial score differences are recorded.

5. **Result Calculation**: After processing all rounds, the result is the sum of all DP states where the score difference is strictly positive (Bob's points > Alice's points).

```python
            a = s[0]
            change_val = changes[a][move]
            score_index = change_val + offset
            if 0 <= score_index < 2 * n + 1:
                dp[move][score_index] = (dp[move][score_index] + 1) % MOD
        
        for i in range(1, n):
            new_dp = [[0] * (2 * n + 1) for _ in range(3)]
            a_round = s[i]
            for last in range(3):
                for score_index in range(2 * n + 1):
                    count = dp[last][score_index]
                    if count == 0:
                        continue
                    current_score = score_index - offset
                    for nxt in range(3):
                        if nxt == last:
                            continue
                        change_val = changes[a_round][nxt]
                        new_score = current_score + change_val
                        if -n <= new_score <= n:
                            new_score_index = new_score + offset
                            if 0 <= new_score_index < 2 * n + 1:
                                new_dp[nxt][new_score_index] = (new_dp[nxt][new_score_index] + count) % MOD
            dp = new_dp
        
        ans = 0
        for last in range(3):
            for score_index in range(offset + 1, 2 * n + 1):
                ans = (ans + dp[last][score_index]) % MOD
        return ans
```

### Explanation
1. **Initialization**: The `changes` dictionary maps each creature combination to the score change (Bob's points minus Alice's points) for that round. The DP array `dp` is initialized to store the number of ways to reach each state (last move and score difference) after each round.

2. **First Round Processing**: For the first round, all three possible moves by Bob are considered, and the initial score differences are recorded in the DP array.

3. **Subsequent Rounds**: For each subsequent round, the DP array is updated by considering all valid moves (excluding the last move to avoid consecutive duplicates) and calculating the new score difference. The new states are stored in a temporary array `new_dp`, which becomes the current DP array for the next round.

4. **Result Calculation**: After processing all rounds, the result is obtained by summing all DP states where the score difference is positive, indicating Bob's total points exceed Alice's. The result is returned modulo \(10^9 + 7\).

This approach efficiently counts all valid sequences using dynamic programming, leveraging the constraints to optimize the solution and ensure correctness.
