To solve this problem, we need to maximize the amount of the initial currency after performing any number of currency conversions over two days. The conversions on each day are governed by given exchange rates, and we can convert currencies in any order, including converting back to the initial currency using the reciprocal rates.

### Approach
1. **Problem Analysis**: The problem involves two days of currency conversions. On each day, we can perform any number of conversions using the provided exchange rates. The key insight is that after the first day's conversions, we can use the second day's rates to further convert the accumulated currencies back to the initial currency or other currencies to maximize the final amount.

2. **Graph Representation**: For each day, we represent the currency conversions as a directed graph where each node is a currency, and each edge from currency `u` to `v` with rate `r` means we can convert 1 unit of `u` to `r` units of `v`. The reverse edge (from `v` to `u`) is included with rate `1/r` to allow conversions back to the original currency.

3. **Bellman-Ford Algorithm**: For each day, we use a modified Bellman-Ford algorithm to compute the maximum amount of each currency achievable starting from the initial currency. The algorithm relaxes all edges repeatedly until no further improvements can be made or the amount exceeds a specified limit (5e10).

4. **Combining Results**: After computing the maximum amounts for each currency on the first day, we use the second day's graph to compute the maximum amount of the initial currency achievable from each currency obtained on the first day. The final result is the maximum value obtained by combining the results from both days.

### Solution Code
```python
class Solution:
    def maxAmount(self, initialCurrency: str, pairs1: List[List[str]], rates1: List[float], pairs2: List[List[str]], rates2: List[float]) -> float:
        # Collect all distinct currencies
        currencies = set()
        currencies.add(initialCurrency)
        for pair in pairs1:
            currencies.add(pair[0])
            currencies.add(pair[1])
        for pair in pairs2:
            currencies.add(pair[0])
            currencies.add(pair[1])
        currencies = list(currencies)
        
        # Build graph1 (day1) with both directions
        graph1 = []
        for i in range(len(pairs1)):
            u = pairs1[i][0]
            v = pairs1[i][1]
            r = rates1[i]
            graph1.append((u, v, r))
            graph1.append((v, u, 1/r))
        
        # Build graph2 (day2) with both directions
        graph2 = []
        for i in range(len(pairs2)):
            u = pairs2[i][0]
            v = pairs2[i][1]
            r = rates2[i]
            graph2.append((u, v, r))
            graph2.append((v, u, 1/r))
        
        # Day1 computation
        dp1 = {c: 0.0 for c in currencies}
        dp1[initialCurrency] = 1.0
        
        changed = True
        while changed and max(dp1.values()) <= 5e10:
            changed = False
            for u, v, r in graph1:
                candidate = r * dp1[v]
                if candidate > dp1[u]:
                    dp1[u] = candidate
                    changed = True
        
        # Day2 computation
        dp2 = {c: 0.0 for c in currencies}
        dp2[initialCurrency] = 1.0
        
        changed = True
        while changed and max(dp2.values()) <= 5e10:
            changed = False
            for u, v, r in graph2:
                candidate = r * dp2[v]
                if candidate > dp2[u]:
                    dp2[u] = candidate
                    changed = True
        
        # Calculate the final result
        result = 0.0
        for c in currencies:
            if dp1[c] > 0 and dp2[c] > 0:
                amount = dp1[c] * dp2[c]
                if amount > result:
                    result = amount
        
        return round(result, 5)
```

### Explanation
1. **Graph Construction**: The solution starts by building two directed graphs, one for each day, where each edge represents a currency conversion with its corresponding rate and the reverse conversion with the reciprocal rate.
2. **Bellman-Ford Algorithm**: For each day, the algorithm computes the maximum amount of each currency achievable starting from the initial currency by repeatedly relaxing all edges. This ensures that the best conversion paths are considered.
3. **Result Calculation**: After processing both days, the solution calculates the maximum amount of the initial currency by combining the results from both days. The result is the highest value obtained by converting currencies from the first day and then further converting them on the second day.
4. **Output**: The final result is rounded to five decimal places for precision, as specified in the problem constraints.

This approach efficiently handles the currency conversions over two days, leveraging graph theory and dynamic programming to maximize the final amount of the initial currency.
