=== ANSWER ===

To solve this problem, we need to determine if a given permutation of the integers 1 through 5 can be sorted in ascending order by performing exactly one adjacent swap. The solution involves checking all possible adjacent swaps to see if any of them results in the sorted array [1, 2, 3, 4, 5].

### Approach
1. **Problem Analysis**: The input is a permutation of the integers 1 to 5. The goal is to check if swapping any two adjacent elements in this permutation results in the sorted array [1, 2, 3, 4, 5]. If the array is already sorted, no single adjacent swap can restore it to the sorted state because swapping any two adjacent elements in a sorted array will break the order.
2. **Intuition**: For each adjacent pair in the array, swap them and check if the resulting array matches the sorted array. If any of these swaps results in the sorted array, the answer is "Yes"; otherwise, it's "No".
3. **Algorithm Selection**: The algorithm involves iterating over each possible adjacent pair in the array, performing a swap, and comparing the result with the sorted array. If a match is found during this iteration, we immediately return "Yes". If no matches are found after checking all possible swaps, we return "No".
4. **Complexity Analysis**: The algorithm checks 4 possible adjacent swaps (since an array of length 5 has 4 adjacent pairs). Each check involves creating a copy of the array and comparing it with the sorted array, which is O(5) per check. Thus, the overall complexity is O(1) due to the fixed input size.

### Solution Code
```python
def main():
    arr = list(map(int, input().split()))
    target = [1, 2, 3, 4, 5]
    n = len(arr)
    found = False
    for i in range(n-1):
        cand = arr.copy()
        cand[i], cand[i+1] = cand[i+1], cand[i]
        if cand == target:
            found = True
            break
    print("Yes" if found else "No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read as a list of integers.
2. **Target Array**: The target sorted array is defined as [1, 2, 3, 4, 5].
3. **Checking Adjacent Swaps**: For each adjacent pair in the input array, the algorithm swaps them and checks if the resulting array matches the target array. If a match is found, it sets a flag and breaks out of the loop.
4. **Output Result**: Based on whether a matching array was found during the checks, the algorithm prints "Yes" or "No".

This approach efficiently checks all possible single adjacent swaps to determine if the array can be sorted in one operation, leveraging the constraints of the problem to keep the solution simple and optimal.
