To solve this problem, we need to determine if a given string meets all the conditions specified for a "1122 string". The conditions are:
1. The string length must be even.
2. Each pair of consecutive characters (at positions 2i-1 and 2i for each i) must be identical.
3. Each character in the string must appear exactly zero or two times.

### Approach
1. **Check String Length**: The first condition requires the string length to be even. If the length is odd, the string immediately fails the condition.
2. **Check Consecutive Pairs**: The second condition mandates that every pair of consecutive characters (i.e., characters at positions 0-1, 2-3, etc.) must be the same. If any pair does not meet this requirement, the string is invalid.
3. **Check Character Frequencies**: The third condition states that each character in the string must appear exactly twice. We use a frequency counter to verify this. If any character appears a number of times other than zero or two, the string is invalid.

### Solution Code
```python
from collections import Counter

def main():
    S = input().strip()
    n = len(S)
    if n % 2 != 0:
        print("No")
    elif any(S[i] != S[i+1] for i in range(0, n, 2)):
        print("No")
    else:
        freq = Counter(S)
        if all(count == 2 for count in freq.values()):
            print("Yes")
        else:
            print("No")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Input Handling**: The input string is read and stripped of any leading or trailing whitespace.
2. **Length Check**: The length of the string is checked for evenness. If it's odd, the output is "No".
3. **Pair Check**: The string is scanned in steps of two characters to ensure each pair of consecutive characters is identical. If any pair fails this check, the output is "No".
4. **Frequency Check**: Using a `Counter` from the `collections` module, the frequency of each character is counted. If all characters appear exactly twice, the output is "Yes"; otherwise, it's "No".

This approach efficiently verifies all three conditions in sequence, ensuring the string meets the criteria for being a "1122 string". The solution handles edge cases such as odd-length strings and strings with characters appearing more or less than twice, providing the correct output in all scenarios.
