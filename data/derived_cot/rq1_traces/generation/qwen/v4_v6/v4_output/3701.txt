A good caption is defined as a string where every character appears in groups of at least 3 consecutive occurrences. The allowed operations are changing a character to its immediate predecessor or successor in the alphabet, if available. ### Approach
1. **Problem Analysis**: The key observation is that the resulting string must be divided into segments (runs) of the same character, each of length at least 3. The challenge is to find the minimal operations required to achieve this, considering the constraints on character changes (only adjacent characters in the alphabet are allowed).

2. **Dynamic Programming (DP) Setup**: We use a DP approach where the state is defined by the current position in the string and the last character used. The DP state keeps track of the minimal cost (operations) to reach that state and the length of the current run (to ensure it meets the minimum length requirement of 3).

3. **State Transition**: For each position, we consider two possibilities:
   - **Extending the current run**: If the current run length is less than 3, we continue extending it by choosing the same character (if beneficial).
   - **Starting a new run**: If the current run length is at least 3, we can start a new run with a different character. The new character must be different from the previous one to ensure valid segmentation.

4. **Cost Calculation**: The cost for changing a character to another is the absolute difference in their ASCII values. The total cost is the sum of individual costs for each position.

5. **Lexicographical Order**: Among all solutions with minimal operations, we select the lexicographically smallest string by prioritizing smaller characters during the DP transitions.

6. **Complexity**: The DP approach efficiently handles the problem with a time complexity of O(n * 26 * n), where n is the length of the string. This is feasible for the given constraints (n ≤ 5 * 10^4) due to optimizations in Python.
