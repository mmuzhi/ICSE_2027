There are N squares arranged in a row, labeled 1, 2, \ldots, N from left to right.
You are given M pairs of integers (L_1, R_1), \ldots, (L_M, R_M).
A square j is defined to be bad if and only if there exists some i such that L_i \leq j \leq R_i.

- Let your current square be x. Choose an integer i that satisfies all of the following conditions, and move to square x + i.
- A \leq i \leq B
- x + i \leq N
- Square x + i is not bad.

Input

The input is given from Standard Input in the following format:
N M A B
L_1 R_1
L_2 R_2
\vdots
L_M R_M

Output

If it is possible to reach square N by repeating the action described in the problem statement, print Yes. Otherwise, print No.

- 2 \leq N \leq 10^{12}
- 0 \leq M \leq 2 \times 10^4
- 1 \leq A \leq B \leq 20
- 1 < L_i \leq R_i < N \ (1 \leq i \leq M)
- R_i < L_{i+1} \ (1 \leq i \leq M - 1)
- All input values are integers.

The movement is restricted to jumping forward by an integer between A and B, inclusive, and landing on a non-bad square within the bounds of the row.

### Approach
1. **Problem Analysis**: The problem involves navigating through a sequence of squares, avoiding bad squares, by making forward jumps of specific lengths. The key challenge is efficiently determining reachability given the constraints (N can be as large as 10^12, but the number of bad intervals is up to 20,000).

2. **Insight**: The bad squares divide the row into contiguous segments of good squares. The solution involves checking if we can jump from the initial segment (containing square 1) to the segment containing square N, ensuring each jump lands within a good segment and adheres to the jump length constraints.

3. **Algorithm Selection**: 
   - **Good Segments Identification**: First, we identify all segments of good squares by processing the given bad intervals. These segments are the complement of the bad intervals in the range [1, N].
   - **BFS on Segments**: Using a BFS-like approach, we check if we can jump from each segment to subsequent segments. The jump condition between segments is derived from ensuring that the jump length (between A and B) allows landing within the target segment without passing through any bad squares.

4. **Complexity Analysis**: The number of good segments is at most 20,001 (due to the constraints on bad intervals). The BFS involves checking each segment against all subsequent segments, leading to a worst-case O(n^2) complexity, where n is the number of segments. Given n is up to 20,001, the worst-case operations are around 400 million, which is manageable with optimizations.

def main():

            e_next = bad_intervals[i+1][0] - 1

        s_last = bad_intervals[-1][1] + 1
        e_last = N
        if s_last <= e_last:
            good_segments.append((s_last, e_last))
            
    if not good_segments:

        return
        
    n_segments = len(good_segments)
    reachable = [False] * n_segments
    reachable[0] = True

    for i in range(n_segments):
        if not reachable[i]:
            continue
        s_i, e_i = good_segments[i]
        if e_i >= N:
            print("Yes")
            return
            
        for j in range(i+1, n_segments):
            s_j, e_j = good_segments[j]
            if s_j > e_i + B:

            low_bound = max(A, s_j - e_i)
            high_bound = min(B, e_j - s_i)
            if low_bound <= high_bound:
                reachable[j] = True

    for i in range(n_segments):
        if reachable[i] and good_segments[i][1] >= N:

            return

2. **Good Segments Identification**: The good segments are derived by processing the bad intervals. The first segment starts at 1 and ends just before the first bad interval. Subsequent segments are the gaps between consecutive bad intervals, and the last segment starts after the last bad interval up to N.
3. **BFS on Segments**: For each segment, if it contains N, we immediately return "Yes". Otherwise, we check all subsequent segments to see if a jump from the current segment can reach them. The jump condition ensures that the landing square is within the target segment and adheres to the jump length constraints.
4. **Output**: If any segment containing N is reachable, we print "Yes"; otherwise, "No".

This approach efficiently checks reachability by leveraging segment processing and BFS, ensuring optimal performance even for large N.
