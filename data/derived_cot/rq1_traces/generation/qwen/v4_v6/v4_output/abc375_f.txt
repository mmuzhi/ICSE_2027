=== ANSWER ===

To solve this problem, we need to handle dynamic graph queries where roads can be removed, and we need to answer shortest path queries efficiently. The solution involves maintaining the all-pairs shortest path (APSP) matrix and updating it whenever a road is removed. Given the constraints, we use the Floyd-Warshall algorithm to compute the APSP matrix after each road removal, leveraging the fact that the number of removals is limited (at most 300).

### Approach
1. **Initial Setup**: Read the number of cities (N), roads (M), and queries (Q). Store the roads in a list.
2. **Initial APSP Matrix**: Compute the initial APSP matrix using the Floyd-Warshall algorithm, which works well for dense graphs (N ≤ 300).
3. **Query Processing**:
   - **Type 1 (Road Removal)**: When a road is removed, rebuild the APSP matrix by excluding the removed road and recompute the shortest paths using Floyd-Warshall.
   - **Type 2 (Shortest Path Query)**: Use the current APSP matrix to answer the shortest path query between two cities. If the path is unreachable (indicated by a large value), return -1.
4. **Efficiency**: Although rebuilding the APSP matrix after each removal is computationally expensive (O(N³) per removal), the limited number of removals (≤ 300) ensures the solution is feasible within the problem constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    it = iter(data)
    n = int(next(it))
    m = int(next(it))
    q = int(next(it))
    
    edges = []
    for _ in range(m):
        a = int(next(it))
        b = int(next(it))
        c = int(next(it))
        edges.append((a, b, c))
    
    removed_roads = set()
    INF = 10**18
    dist = [[INF] * n for _ in range(n)]
    for i in range(n):
        dist[i][i] = 0
        
    for idx in range(m):
        a, b, c = edges[idx]
        i_node = a - 1
        j_node = b - 1
        if c < dist[i_node][j_node]:
            dist[i_node][j_node] = c
            dist[j_node][i_node] = c
            
    for k in range(n):
        for i in range(n):
            for j in range(n):
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
                    
    out_lines = []
    for _ in range(q):
        t = next(it)
        if t == '1':
            i_road = int(next(it))
            removed_roads.add(i_road)
            new_dist = [[INF] * n for _ in range(n)]
            for i_node in range(n):
                new_dist[i_node][i_node] = 0
            for idx in range(m):
                a, b, c = edges[idx]
                road_num = idx + 1
                if road_num in removed_roads:
                    continue
                i_node = a - 1
                j_node = b - 1
                if c < new_dist[i_node][j_node]:
                    new_dist[i_node][j_node] = c
                    new_dist[j_node][i_node] = c
            for k in range(n):
                for i_val in range(n):
                    for j_val in range(n):
                        if new_dist[i_val][k] + new_dist[k][j_val] < new_dist[i_val][j_val]:
                            new_dist[i_val][j_val] = new_dist[i_val][k] + new_dist[k][j_val]
            dist = new_dist
        else:
            x = int(next(it))
            y = int(next(it))
            x -= 1
            y -= 1
            if dist[x][y] == INF:
                out_lines.append("-1")
            else:
                out_lines.append(str(dist[x][y]))
                
    print("\n".join(out_lines))

if __name__ == "__main__":
    main()
```

2. **Initial APSP Matrix**: The Floyd-Warshall algorithm initializes the shortest paths between all pairs of cities. The matrix `dist` is updated to reflect the shortest paths considering all roads.
3. **Handling Queries**:
   - **Type 1 Queries**: When a road is removed, the APSP matrix is rebuilt by excluding the removed road. The Floyd-Warshall algorithm is reapplied to the modified graph.
   - **Type 2 Queries**: The current APSP matrix is queried to find the shortest path between the specified cities. If no path exists (indicated by the large value INF), -1 is returned.
4. **Output**: Results for Type 2 queries are collected and printed at the end to minimize I/O operations, which is crucial for handling large input sizes efficiently.
