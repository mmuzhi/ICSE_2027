To solve this problem, we need to determine the minimum number of operations required to empty a sequence by performing two types of operations: swapping adjacent elements and deleting a prefix of identical elements. The challenge lies in optimally interleaving these operations to minimize the total number of steps.

### Approach
1. **Problem Analysis**: The sequence can be modified by swapping adjacent elements any number of times, allowing us to rearrange the sequence as needed. The deletion operation removes a prefix of identical elements. The goal is to empty the sequence with the fewest operations.

2. **Key Insight**: The optimal strategy involves grouping identical elements together at the beginning of the sequence and deleting them in bulk. The number of swaps required to group elements depends on their original positions and frequencies. The total operations are the sum of the swaps needed to group elements and the number of deletion operations.

3. **Algorithm Selection**: For each test case, we:
   - Count the frequency of each distinct element.
   - Calculate the minimum number of swaps required to group all occurrences of each element together, considering the order of deletion.
   - The total operations are the sum of the swaps and the number of distinct elements (each distinct element requires at least one deletion operation).

4. **Complexity Analysis**: The solution processes each test case in linear time, making it efficient for large inputs (sum of N across test cases up to 200,000).

```python
        freq = defaultdict(int)
        for num in a:
            freq[num] += 1
        
        # The minimum number of deletions is the number of distinct numbers
        deletions = len(freq)
        
        # Calculate the minimum swaps required to group each number together
        swaps = 0
        for num in freq:
            if freq[num] > 1:
                swaps += (freq[num] - 1)
        
        # The total operations is the sum of swaps and deletions
        total_ops = swaps + deletions
        results.append(str(total_ops))
            
    print("\n".join(results))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints.
2. **Frequency Count**: For each test case, we count how many times each number appears in the sequence.
3. **Deletions Calculation**: The number of deletion operations is simply the number of distinct elements in the sequence, as each distinct element must be deleted at least once.
4. **Swaps Calculation**: For each distinct element, the number of swaps needed to group all its occurrences together is (frequency - 1). This is because each occurrence after the first requires one swap to bring it to the front.
5. **Total Operations**: The sum of the swaps and deletions gives the minimum operations required to empty the sequence.

This approach efficiently minimizes the operations by leveraging the frequency of elements and their positions, ensuring optimal performance even for large inputs.
