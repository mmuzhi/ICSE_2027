The grid is defined such that tiles with the same parity (both i and j even or both odd) cover a K-unit width and 1-unit height, while tiles with different parity cover a 1-unit width and K-unit height. The challenge is to find the shortest path between two tiles (one containing the start point and another containing the target point) considering their positions and parities.

2. **Intuition**: The solution involves:
   - **Tile Identification**: For both the start and target points, determine the corresponding tiles (i, j, k) using floor division and parity checks. Each point can be contained in one or two tiles, depending on whether it lies on the boundary between two tiles.
   - **Manhattan Distance Calculation**: The minimum number of moves is the Manhattan distance between the start and target tiles in their 3D (i, j, k) space. If the target point is contained in multiple tiles, select the one that minimizes this distance.

3. **Algorithm Selection**: The algorithm involves:
   - **Floor Division**: Use floor division to compute tile indices (i, j, k) for both start and target points.
   - **Parity Check**: Ensure the computed tiles meet the parity condition (same or different) for their respective definitions.
   - **Distance Calculation**: Compute the Manhattan distance between the start and target tiles, considering all possible candidate tiles for the target if it lies on a boundary.

4. **Complexity Analysis**: The solution efficiently computes the required values using basic arithmetic operations (floor division, modulus) and comparisons, making it suitable for large input sizes (up to 10^16) with a time complexity of O(1) per test case.

def main():
    import sys
    data = sys.stdin.read().split()
    t = int(data[0])

        K = int(data[index]); index += 1

        def get_candidates(a, b, K):
            candidates = []
            x = a + 0.5
            y_val = b + 0.5
            i1 = math.floor(x / K) if K != 0 else 0
            n_y = math.floor(y_val)
            j1 = math.floor(n_y / K) if K != 0 else 0
            k1 = n_y - j1 * K
            if (i1 % 2) == (j1 % 2):
                candidates.append((i1, j1, k1))
            
            n_val = math.floor(x)
            i2 = math.floor(n_val / K) if K != 0 else 0
            k2 = n_val % K

                candidates.append((i2, j2, k2))
