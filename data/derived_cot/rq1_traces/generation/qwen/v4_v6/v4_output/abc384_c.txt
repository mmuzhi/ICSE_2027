You will be given a competitive programming problem.
Takahashi decided to hold a programming contest.
The contest consists of five problems: A, B, C, D, E, with scores a, b, c, d, e, respectively.
There are 31 participants, and all of them solved at least one problem.
More specifically, for every non-empty subsequence (not necessarily contiguous) of the string ABCDE, there is a participant named after that subsequence who solved the problems corresponding to the letters in their name and did not solve the other problems.
For example, participant A solved only problem A, and participant BCE solved problems B, C, and E.
Print the names of the participants in order of their obtained scores, from the largest to the smallest.
The score obtained by a participant is the sum of the scores of the problems they solved.
If two participants obtained the same score, print the one whose name is lexicographically smaller first.
What does "lexicographically smaller" mean?
In short, "lexicographically smaller" refers to the order in which words would appear in a dictionary.
More precisely, for distinct strings S,T consisting of uppercase English letters, S is lexicographically smaller than T if either of the following conditions holds:

- The length |S| of S is less than the length of T, and the first |S| characters of T match S.
- There exists an integer 1\leq i\leq\min\{ |S|,|T|\} that satisfy both of the following two conditions:
- For every integer j with 1\leq j\lt i, the j-th character of S equals the j-th character of T.
- The i-th character of S is alphabetically smaller than the i-th character of T.


For example, if S= AB and T= ABC, the first condition holds, so S is lexicographically smaller than T.
If S= ABD and T= ACD, the second condition holds for i=2, so S is lexicographically smaller than T.

=== ANSWER ===

To solve this problem, we need to generate all non-empty subsequences of the string "ABCDE", calculate the scores for each subsequence based on given problem scores, and then sort these subsequences in descending order of their scores. If two subsequences have the same score, they should be sorted lexicographically in ascending order.

### Approach
1. **Generate Subsequences**: For each bitmask from 1 to 31 (since there are 5 problems, leading to 2^5 - 1 = 31 non-empty subsequences), generate the corresponding subsequence of "ABCDE" by checking each bit in the bitmask. Each bit represents whether a particular problem is included in the subsequence.
2. **Calculate Scores**: For each generated subsequence, compute its score by summing the scores of the included problems.
3. **Sort Subsequences**: Sort the subsequences first by their scores in descending order. If two subsequences have the same score, sort them lexicographically in ascending order.
4. **Output Results**: Print each subsequence in the sorted order, one per line.

### Solution Code
```python
def main():
    data = input().split()
    a, b, c, d, e = map(int, data)
    s = "ABCDE"
    scores = [a, b, c, d, e]
    
    participants = []
    for mask in range(1, 32):
        name = ""
        total = 0
        for i in range(5):
            if mask & (1 << i):
                name += s[i]
                total += scores[i]
        participants.append((total, name))
    
    sorted_participants = sorted(participants, key=lambda x: (-x[0], x[1]))
    
    for _, name in sorted_participants:
        print(name)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input consists of five integers representing the scores for problems A, B, C, D, and E.
2. **Generating Subsequences**: Using a bitmask from 1 to 31, each bit in the mask determines whether a corresponding problem is included in the subsequence. For example, a mask value of 1 (binary 00001) includes only problem A.
3. **Calculating Scores**: For each subsequence, the score is the sum of the scores of the included problems. The subsequence name is constructed by concatenating the letters of the included problems in order.
4. **Sorting**: The subsequences are sorted primarily by their scores in descending order. For subsequences with the same score, they are sorted lexicographically in ascending order.
5. **Output**: The sorted list of subsequences is printed, each on a new line, from highest to lowest score, with lexicographical order breaking ties.

This approach efficiently generates all possible non-empty subsequences, calculates their scores, and sorts them according to the specified criteria, ensuring the correct output order.
