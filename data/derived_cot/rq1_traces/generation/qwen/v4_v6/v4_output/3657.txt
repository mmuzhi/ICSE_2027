To solve this problem, we need to determine if we can make either two horizontal or two vertical cuts on an n x n grid such that each of the three resulting sections contains at least one rectangle, and every rectangle belongs to exactly one section. The rectangles are given as non-overlapping axis-aligned rectangles.

### Approach
1. **Problem Analysis**: The problem requires checking two scenarios: horizontal cuts and vertical cuts. For horizontal cuts, we need to partition the grid into three horizontal strips (bottom, middle, top) such that each rectangle is entirely within one strip. The same applies to vertical cuts, partitioning the grid into three vertical strips (left, middle, right).

2. **Key Insight**: For both horizontal and vertical cuts, the solution involves finding two cut lines (either horizontal or vertical) such that:
   - The bottom/left section contains rectangles entirely below the first cut line.
   - The top/right section contains rectangles entirely above the second cut line.
   - The middle section contains the remaining rectangles, which must be entirely between the two cut lines.

3. **Algorithm Selection**: 
   - **Horizontal Cuts**: For each candidate cut line (from start and end y-coordinates of rectangles), compute the required second cut line ensuring all rectangles are covered and sections are non-empty.
   - **Vertical Cuts**: Similarly, for each candidate cut line (from start and end x-coordinates of rectangles), compute the required second cut line ensuring all rectangles are covered and sections are non-empty.

4. **Complexity Analysis**: The algorithm involves sorting and binary searches, leading to an O(m log m) complexity per direction (horizontal or vertical), where m is the number of unique coordinates (up to 200,000). This is efficient given the constraints.

### Solution Code
```python
import bisect
from typing import List

class Solution:
    def checkValidCuts(self, n: int, rectangles: List[List[int]]) -> bool:
        def check_direction(intervals):
            if not intervals:
                return False
            starts = [s for s, e in intervals]
            ends = [e for s, e in intervals]
            candidates = sorted(set(starts) | set(ends))
            min_end = min(ends) if ends else 10**18
            max_start = max(starts) if starts else -10**18
            
            if min_end == 10**18 or max_start == -10**18:
                return False
            
            sorted_intervals = sorted(intervals, key=lambda x: x[0])
            sy_list = [si[0] for si in sorted_intervals]
            ey_list = [si[1] for si in sorted_intervals]
            
            prefix_max = [0] * len(sorted_intervals)
            prefix_max[0] = ey_list[0]
            for i in range(1, len(sorted_intervals)):
                prefix_max[i] = max(prefix_max[i-1], ey_list[i])
            
            suffix_min = [0] * len(sorted_intervals)
            suffix_min[-1] = ey_list[-1]
            for i in range(len(sorted_intervals)-2, -1, -1):
                suffix_min[i] = min(suffix_min[i+1], ey_list[i])
            
            for cand in candidates:
                if cand < min_end:
                    continue
                pos = bisect.bisect_left(sy_list, cand)
                if pos == len(sy_list):
                    n_val = 10**18
                else:
                    n_val = suffix_min[pos]
                
                if pos == 0:
                    m_val = -10**18
                else:
                    m_val = prefix_max[pos-1]
                
                y2_req = max(m_val, n_val)
                if y2_req > max_start:
                    continue
                
                return True
            
            return False

        horizontal_intervals = [[r[1], r[3]] for r in rectangles]
        vertical_intervals = [[r[0], r[2]] for r in rectangles]
        
        if check_direction(horizontal_intervals):
            return True
        if check_direction(vertical_intervals):
            return True
        
        return False
```

### Explanation
1. **check_direction Function**: This helper function checks if valid cuts exist for a given set of intervals (either horizontal or vertical). It:
   - Collects all unique start and end coordinates.
   - Sorts intervals and precomputes prefix maxima and suffix minima for efficient range queries.
   - For each candidate cut line, it determines the required second cut line ensuring all rectangles are covered and sections are non-empty.

2. **Main Function**: The main function checks both horizontal and vertical cuts by transforming the rectangles' coordinates accordingly. If either direction yields valid cuts, it returns `True`; otherwise, it returns `False`.

This approach efficiently checks the feasibility of making two cuts in either direction by leveraging sorting and binary search, ensuring optimal performance even for large input sizes.
