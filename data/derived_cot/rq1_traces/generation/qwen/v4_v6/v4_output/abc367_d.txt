To solve this problem, we need to count the number of ordered pairs (s, t) of distinct rest areas around a lake such that the clockwise distance from s to t is a multiple of a given integer M. The rest areas are arranged in a clockwise circular order, and moving from any rest area to the next in the clockwise direction takes a specified number of steps. 

### Approach
1. **Problem Analysis**: The problem involves calculating the clockwise distance between every pair of distinct rest areas (s, t) and checking if this distance is a multiple of M. The challenge is to efficiently compute these distances for all possible pairs without resorting to a brute-force O(N^2) approach, especially given the constraints (N can be up to 200,000).

2. **Key Insight**: The clockwise distance from rest area i to rest area j can be broken into two cases:
   - **Case 1**: If i < j, the distance is the sum of segments from i to j-1.
   - **Case 2**: If i > j, the distance is the sum of segments from i to the end of the circle plus the segments from the start to j-1.

3. **Modulo Arithmetic**: Instead of dealing with potentially very large distances, we use modulo arithmetic to check divisibility by M. For each rest area, we compute prefix sums modulo M and use frequency arrays to count valid pairs efficiently.

4. **Algorithm Selection**:
   - **Case 1**: For each rest area, count the number of previous rest areas (i < j) with the same prefix sum modulo M. This is done using a backward pass through the rest areas.
   - **Case 2**: For each rest area, count the number of previous rest areas (i > j) where the adjusted prefix sum (based on the total circle sum) modulo M matches the required value. This is done using a forward pass.

5. **Complexity Analysis**: The algorithm processes each rest area in two linear passes (O(N)), with each pass involving constant-time operations for each element. The space complexity is O(M) for the frequency arrays, which is feasible given M ≤ 10^6.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    N = int(data[0])
    M = int(data[1])
    A = list(map(int, data[2:2+N]))
    
    total_sum = sum(A)
    total_mod = total_sum % M

    T_rest = [0] * N
    for i in range(1, N):
        T_rest[i] = T_rest[i-1] + A[i-1]

    freq = [0] * M
    ans1 = 0
    for i in range(N-1, -1, -1):
        r = T_rest[i] % M
        ans1 += freq[r]
        freq[r] += 1

    freq = [0] * M
    ans2 = 0
    for i in range(N):
        r_i = T_rest[i] % M
        req_i = (r_i - total_mod) % M
        ans2 += freq[req_i]
        freq[r_i] += 1

    print(ans1 + ans2)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of rest areas (N), the divisor (M), and the list of segment lengths (A).

2. **Prefix Sum Calculation**: The array `T_rest` is computed where each element `T_rest[i]` represents the sum of segments from the start up to (but not including) rest area i. This helps in quickly calculating the clockwise distance between any two rest areas.

3. **Case 1 Handling**: A backward pass through the rest areas counts pairs (i, j) where i < j and the modulo of their prefix sums matches. This is done using a frequency array to keep track of encountered residues.

4. **Case 2 Handling**: A forward pass counts pairs (i, j) where i > j and the adjusted prefix sum (based on the total circle sum) modulo M matches the required value. Again, a frequency array is used for efficiency.

5. **Output**: The results from both cases are summed and printed, giving the total number of valid ordered pairs (s, t) where the clockwise distance is a multiple of M. This approach efficiently handles the constraints by leveraging modulo arithmetic and linear passes through the data.
