=== ANSWER ===

To solve this problem, we need to simulate the operation of taking K cards from the bottom of a stack and placing them on top while maintaining their order. The stack is represented as a list of integers where the first element is the top card and the last element is the bottom card.

### Approach
1. **Problem Analysis**: The problem requires us to rearrange the stack such that the last K elements (from the bottom) are moved to the top of the stack. The relative order of the cards should remain unchanged. This operation can be efficiently performed using list slicing.
2. **Intuition**: By splitting the original list into two parts—the last K elements and the remaining elements—we can simply concatenate these two parts in reverse order to achieve the desired result. The last K elements form the new top of the stack, followed by the remaining elements.
3. **Algorithm Selection**: The solution involves slicing the list into two parts: the last K elements and the first (N - K) elements. Concatenating these two parts in the order of last K elements followed by the first (N - K) elements gives the desired result.
4. **Complexity Analysis**: The slicing operation in Python is O(N) due to the creation of new lists, but given the constraints (N ≤ 100), this approach is efficient and feasible.

### Solution Code
```python
n, k = map(int, input().split())
a = list(map(int, input().split()))
result = a[-k:] + a[:-k]
print(*result)
```

### Explanation
1. **Reading Input**: The first line reads two integers, N (number of cards) and K (number of cards to move from the bottom). The second line reads the list of integers representing the stack of cards.
2. **Slicing the List**: The list is split into two parts using slicing. `a[-k:]` extracts the last K elements, and `a[:-k]` extracts all elements except the last K.
3. **Concatenation**: The two slices are concatenated such that the last K elements (from the bottom) come first, followed by the remaining elements (from the top). This forms the new stack configuration.
4. **Output**: The result is printed as space-separated integers.

This approach efficiently rearranges the stack by leveraging Python's slicing capabilities, ensuring clarity and correctness with minimal computational overhead.
