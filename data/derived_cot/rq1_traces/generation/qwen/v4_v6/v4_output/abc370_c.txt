You are given two strings S and T consisting of lowercase English letters. Here, S and T have equal lengths.
Let X be an empty array, and repeat the following operation until S equals T:

- Change one character in S, and append S to the end of X.

Find the array of strings X with the minimum number of elements obtained in this way. If there are multiple such arrays with the minimum number of elements, find the lexicographically smallest one among them.
 What is lexicographical order on arrays of strings?
A string S = S_1 S_2 \ldots S_N of length N is lexicographically smaller than a string T = T_1 T_2 \ldots T_N of length N if there exists an integer 1 \leq i \leq N such that both of the following are satisfied:

-  S_1 S_2 \ldots S_{i-1} = T_1 T_2 \ldots T_{i-1}
-  S_i comes earlier than T_i in alphabetical order.

An array of strings X = (X_1,X_2,\ldots,X_M) with M elements is lexicographically smaller than an array of strings Y = (Y_1,Y_2,\ldots,Y_M) with M elements if there exists an integer 1 \leq j \leq M such that both of the following are satisfied:

-  (X_1,X_2,\ldots,X_{j-1}) = (Y_1,Y_2,\ldots,Y_{j-1})
-  X_j is lexicographically smaller than Y_j.

=== ANSWER ===

To solve this problem, we need to transform string `S` into string `T` with the minimum number of operations, where each operation changes one character in `S` and appends the resulting string to an array `X`. If multiple solutions exist with the same minimum number of operations, we must choose the lexicographically smallest sequence of strings.

### Approach
1. **Problem Analysis**: The key observation is that the minimal number of operations required to transform `S` into `T` is equal to the number of positions where `S` and `T` differ. Each operation fixes one such mismatch by changing the corresponding character in `S` to the target character in `T`.

2. **Greedy Selection**: To ensure the lexicographically smallest sequence, at each step, we select the mismatch that, when fixed, produces the lexicographically smallest string. This involves comparing all possible candidates generated by fixing each remaining mismatch and choosing the smallest one.

3. **Algorithm**:
   - **Initialization**: Start with the original string `S` and identify all positions where `S` and `T` differ.
   - **Iterative Fixing**: While there are mismatches left, for each remaining mismatch, generate the string that would result from fixing that mismatch. Select the string that is lexicographically smallest.
   - **Update State**: Update the current string by fixing the chosen mismatch and append the generated string to the result array.
   - **Termination**: Once all mismatches are fixed, the current string equals `T`, and the result array contains the sequence of transformations.

### Solution Code
```python
def main():
    S = input().strip()
    T = input().strip()
    
    if S == T:
        print(0)
        return
        
    current = list(S)
    mismatches = [i for i in range(len(S)) if S[i] != T[i]]
    X = []

    while mismatches:
        best_candidate = None
        best_index = None
        for i in mismatches:
            cand = current[:]
            cand[i] = T[i]
            cand_str = ''.join(cand)
            if best_candidate is None or cand_str < best_candidate:
                best_candidate = cand_str
                best_index = i
                
        current[best_index] = T[best_index]
        X.append(best_candidate)
        mismatches.remove(best_index)
        
    print(len(X))
    for x in X:
        print(x)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The code reads the input strings `S` and `T`. If they are already identical, it immediately outputs `0` since no operations are needed.
2. **Mismatch Identification**: The code identifies all positions where `S` and `T` differ and initializes the current state as `S`.
3. **Greedy Selection**: In each iteration, the code generates all possible strings by fixing each remaining mismatch. The lexicographically smallest string is selected.
4. **State Update**: The current string is updated by fixing the chosen mismatch, and the generated string is added to the result array `X`.
5. **Output**: After processing all mismatches, the length of `X` (number of operations) is printed, followed by each string in `X` on a separate line.

This approach efficiently ensures the minimal number of operations and the lexicographically smallest sequence by leveraging a greedy strategy at each step. The complexity is manageable due to the constraints (string length up to 100 and mismatches up to 100), making it suitable for the problem requirements.
