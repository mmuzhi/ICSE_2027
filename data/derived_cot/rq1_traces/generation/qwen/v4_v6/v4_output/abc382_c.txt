You will be given a competitive programming problem.
There are N people numbered from 1 to N visiting a conveyor belt sushi restaurant.
The gourmet level of person i is A_i.
Now, M pieces of sushi will be placed on the conveyor belt.
The deliciousness of the j-th sushi is B_j.
Each piece of sushi passes in front of people 1, 2, \dots, N in this order.
Each person, when a sushi whose deliciousness is not less than their gourmet level passes in front of them, will take and eat that sushi; otherwise, they do nothing.
A sushi that person i takes and eats will no longer pass in front of person j\ (j > i).
For each of the M pieces of sushi, determine who eats that sushi, or if nobody eats it.

Input

The input is given from Standard Input in the following format:
N M
A_1 A_2 \dots A_N
B_1 B_2 \dots B_M

Output

Print M lines.
The j-th line (1 \leq j \leq M) should contain the number representing the person who eats the j-th sushi, or -1 if nobody eats it.

Constraints


- 1 \leq N, M \leq 2 \times 10^5
- 1 \leq A_i, B_i \leq 2 \times 10^5
- All input values are integers.

Sample Input 1

3 3
3 8 2
5 2 1

Sample Output 1

1
3
-1


- For the 1st sushi:
- It first passes in front of person 1. Since B_1 \geq A_1, person 1 takes and eats it.
- It will not pass in front of person 2 and 3.


- For the 2nd sushi:
- It first passes in front of person 1. Since B_2 < A_1, person 1 does nothing.
- Next, it passes in front of person 2. Since B_2 < A_2, person 2 does nothing.
- Finally, it passes in front of person 3. Since B_2 \geq A_3, person 3 takes and eats it.


- For the 3rd sushi:
- It first passes in front of person 1. Since B_3 < A_1, person 1 does nothing.
- Next, it passes in front of person 2. Since B_3 < A_2, person 2 does nothing.
- Finally, it passes in front of person 3. Since B_3 < A_3, person 3 does nothing.
- Therefore, nobody eats this sushi.

Sample Output 2

Sample Input 3

60 83 76 45 70 91 37 58 94 22
70 39 52 33 18

=== ANSWER ===

To solve this problem, we need to determine for each sushi piece on a conveyor belt who the first person is that will eat it based on their gourmet level. The sushi passes in front of each person in order, and a person will eat the sushi if its deliciousness is not less than their gourmet level. The challenge is to efficiently compute this for each sushi piece given the constraints.

### Approach
1. **Problem Analysis**: The problem requires checking for each sushi piece, the first person in line whose gourmet level is less than or equal to the sushi's deliciousness. Given the constraints (up to 200,000 people and sushi pieces), a brute-force approach would be too slow.
2. **Key Insight**: For each sushi with deliciousness `b`, we need the smallest index `i` (0-based) such that `A[i] <= b`. This can be rephrased as finding the minimum index where the condition holds for all values up to `b`.
3. **Algorithm Selection**: 
   - **Preprocessing**: We first create an array `min_index_at_value` where each entry at index `x` holds the smallest index `i` such that `A[i] = x`. This is done by iterating through the list of gourmet levels and recording the earliest occurrence of each value.
   - **Prefix Minimum Array**: Using the `min_index_at_value` array, we construct another array `ans_arr` where each entry at index `x` holds the smallest index `i` such that `A[i] <= x`. This is achieved by a forward pass through the `min_index_at_value` array, keeping track of the minimum index encountered so far.
4. **Efficiency**: The preprocessing steps (filling `min_index_at_value` and `ans_arr`) each run in O(max_val) time, where `max_val` is 200,000. The space complexity is O(max_val) to store these arrays. For each sushi piece, the solution is then a simple lookup in `ans_arr`, making the overall complexity O(max_val + M), which is efficient for the given constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m = int(data[1])
    A = list(map(int, data[2:2+n]))
    B = list(map(int, data[2+n:2+n+m]))
    
    max_val = 200000
    min_index_at_value = [n] * (max_val + 1)
    
    for i, a in enumerate(A):
        if a <= max_val:
            if i < min_index_at_value[a]:
                min_index_at_value[a] = i
                
    ans_arr = [n] * (max_val + 1)
    current_min = n
    for x in range(0, max_val + 1):
        if min_index_at_value[x] < current_min:
            current_min = min_index_at_value[x]
        ans_arr[x] = current_min
        
    output_lines = []
    for b in B:
        if b > max_val:
            output_lines.append("-1")
        else:
            idx = ans_arr[b]
            if idx == n:
                output_lines.append("-1")
            else:
                output_lines.append(str(idx + 1))
                
    print("\n".join(output_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints. The first two values are `N` (number of people) and `M` (number of sushi pieces). The next `N` values are the gourmet levels of the people, followed by `M` values representing the deliciousness of each sushi piece.
2. **Preprocessing**: The `min_index_at_value` array is initialized to store the smallest index for each possible gourmet level (up to 200,000). For each person's gourmet level, we update this array to keep the earliest occurrence.
3. **Building Answer Array**: The `ans_arr` array is constructed by iterating from 0 to 200,000. For each value `x`, `ans_arr[x]` holds the smallest index `i` such that `A[i] <= x`, derived from the `min_index_at_value` array.
4. **Processing Sushi Pieces**: For each sushi piece, if its deliciousness exceeds 200,000, it's impossible for any person to eat it, so the result is -1. Otherwise, the result is derived from `ans_arr` by converting the 0-based index to a 1-based person number or -1 if no person meets the condition.
5. **Output**: The results for all sushi pieces are collected in a list and printed at once for efficiency.

This approach efficiently handles the problem constraints by leveraging preprocessing and lookup, ensuring optimal performance.
