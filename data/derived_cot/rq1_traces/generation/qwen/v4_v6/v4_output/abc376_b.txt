Initially, your left hand is holding part 1, and your right hand is holding part 2.
In one operation, you can do the following:

- Move one of your hands to an adjacent part of the part it is currently holding. However, you can do this only if the other hand is not on the destination part.

The following figure shows the initial state and examples of operations that can and cannot be made from there. The number written on each part of the ring represents the part number, and the circles labeled L and R represent your left and right hands, respectively.

You need to follow Q instructions given to you in order.
The i-th (1 \leq i \leq Q) instruction is represented by a character H_i and an integer T_i, meaning the following:

- Perform some number of operations (possibly zero) so that your left hand (if H_i is L) or your right hand (if H_i is R) is holding part T_i.
  Here, you must not move the other hand not specified by H_i.

It is guaranteed that only achievable instructions are given.

Details
Under the settings of this problem, it can be proved that the positions of both hands are uniquely determined just before following the i-th instruction for each i.
At that time, if we denote the positions of the left and right hands as parts l_i and r_i, respectively, it is guaranteed that T_i \neq r_i when H_i is L, and T_i \neq l_i when H_i is R.


Find the minimum total number of operations required to follow all the instructions.

Input

The Input is given from Standard Input in the following format:
N Q
H_1 T_1
H_2 T_2
\vdots
H_Q T_Q

Output

Print the minimum total number of operations required to follow all the instructions.

Constraints


- 3 \leq N \leq 100
- 1 \leq Q \leq 100
- H_i is L or R.
- 1 \leq T_i \leq N
- N, Q, and T_i are integers.
- Only achievable instructions are given (see the problem statement for details).

To solve this problem, we need to determine the minimum number of operations required to follow a sequence of instructions on a ring with N parts, where each instruction specifies moving one hand (left or right) to a target part while the other hand remains fixed. The key challenge is to compute the shortest path for the moving hand to reach the target part without stepping on the fixed hand's position.

### Approach
1. **Problem Analysis**: The ring is circular, so movement can be clockwise or counter-clockwise. The problem requires moving one hand (either left or right) to a specified target part, avoiding the fixed hand's position. The solution involves calculating the shortest path for the moving hand, considering both directions (clockwise and counter-clockwise) and ensuring the path does not intersect the fixed hand's position.

2. **Key Insight**: For each instruction, the moving hand can take either the clockwise or counter-clockwise path to reach the target. The path must avoid the fixed hand's position. The shortest valid path (either direction) is chosen, and if both paths are blocked, the instruction is considered achievable (as per problem constraints).

3. **Algorithm Selection**: 
   - **Clockwise Path Check**: Determine if the clockwise path from the current position to the target contains the fixed hand. If not, the clockwise distance is the minimal steps required.
   - **Counter-Clockwise Path Check**: Similarly, check if the counter-clockwise path (equivalent to the clockwise path from the target to the current position) is free of the fixed hand. If not, the counter-clockwise distance is the minimal steps.
   - **Path Calculation**: For each direction, compute the distance by considering the circular nature of the ring. The distance is calculated based on the positions, adjusting for wrap-around if necessary.

4. **Complexity Analysis**: The algorithm processes each instruction in constant time, making the overall complexity O(Q), where Q is the number of instructions. This is efficient given the constraints (Q ≤ 100).

import sys

def moves_for_L(l, T, r, N):
    if l == T:
        return 0, (T, r)
    if l <= T:
        if l <= r <= T:
            clockwise_clear = False
        else:
            clockwise_clear = True
    else:
        if r >= l or r <= T:
            clockwise_clear = False
        else:
            clockwise_clear = True
            
    if clockwise_clear:
        if l <= T:
            d = T - l
        else:
            d = T - l + N
        return d, (T, r)
        
    if T <= l:
        if T <= r <= l:
            counterclockwise_clear = False
        else:
            counterclockwise_clear = True
    else:
        if r >= T or r <= l:
            counterclockwise_clear = False
        else:
            counterclockwise_clear = True
            
    if counterclockwise_clear:
        if T <= l:
            d = l - T
        else:
            d = l - T + N
        return d, (T, r)
    else:
        return float('inf'), (T, r)

def moves_for_R(l, r, T, N):
    if r == T:
        return 0, (l, T)
    if r <= T:
        if r <= l <= T:
            clockwise_clear = False
        else:
            clockwise_clear = True
    else:
        if l >= r or l <= T:
            clockwise_clear = False
        else:
            clockwise_clear = True
            
    if clockwise_clear:
        if r <= T:

            d = r - T
        else:

    else:

def main():
    data = sys.stdin.read().strip().splitlines()
    if not data:
        return
    first_line = data[0].split()
    N = int(first_line[0])
    Q = int(first_line[1])
    instructions = []
    for i in range(1, Q+1):
        parts = data[i].split()
        H = parts[0]
        T = int(parts[1])
        instructions.append((H, T))
        
    l, r = 1, 2
    total_moves = 0
    
    for H, T in instructions:
        if H == 'L':
            if l == T:
                new_state = (l, r)
            else:
                d, new_state = moves_for_L(l, T, r, N)
                if d == float('inf'):
                    d = 0
                total_moves += d
                l, r = new_state
        else:
            if r == T:
                new_state = (l, r)
            else:
                d, new_state = moves_for_R(l, r, T, N)
                if d == float('inf'):
                    d = 0
                total_moves += d
                l, r = new_state
                
    print(total_moves)

if __name__ == "__main__":
    main()

### Explanation
1. **Initialization**: The program reads the input values, including the number of parts (N) and instructions (Q). The initial positions of the hands are set to left=1 and right=2.
2. **Instruction Processing**: For each instruction, the program checks if the target part is already occupied by the specified hand. If not, it calculates the minimal steps required to move the hand to the target without stepping on the fixed hand's position.
3. **Path Calculation**: The algorithm checks both clockwise and counter-clockwise paths. For each direction, it verifies if the path is clear (does not contain the fixed hand). The shortest clear path is chosen, and the total operations are accumulated.
4. **Output**: After processing all instructions, the total number of operations is printed.

This approach efficiently handles the circular nature of the problem and ensures minimal operations by leveraging path checks and distance calculations in both directions.
