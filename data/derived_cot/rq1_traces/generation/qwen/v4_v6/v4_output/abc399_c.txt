To solve this problem, we need to determine the minimum number of edges to delete from a given undirected graph so that the resulting graph becomes a forest (i.e., contains no cycles). The solution involves analyzing the graph's structure to identify cycles and connected components, then calculating the required edge deletions based on these findings.

### Approach
1. **Problem Analysis**: The key observation is that a forest (acyclic graph) with \( N \) vertices and \( k \) connected components can have at most \( N - k \) edges. The minimum number of edges to delete is the total edges minus this maximum, i.e., \( M - (N - k) \).

2. **Graph Traversal**: We use BFS (Breadth-First Search) to traverse the graph and count the number of connected components. Each unvisited node initiates a BFS, marking all reachable nodes as visited, thus incrementing the component count each time.

3. **Edge Calculation**: Once the number of connected components \( k \) is determined, the answer is computed as \( M - (N - k) \). This formula accounts for the maximum edges allowed in a forest (each component being a tree) and subtracts this from the total edges to find the deletions needed.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    it = iter(data)
    n = int(next(it))
    m = int(next(it))
    graph = [[] for _ in range(n)]
    for _ in range(m):
        u = int(next(it)) - 1
        v = int(next(it)) - 1
        graph[u].append(v)
        graph[v].append(u)
    
    visited = [False] * n
    components = 0
    for i in range(n):
        if not visited[i]:
            components += 1
            queue = deque([i])
            visited[i] = True
            while queue:
                node = queue.popleft()
                for neighbor in graph[node]:
                    if not visited[neighbor]:
                        visited[neighbor] = True
                        queue.append(neighbor)
    
    ans = m - (n - components)
    print(ans)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are \( N \) (number of vertices) and \( M \) (number of edges).
2. **Graph Construction**: An adjacency list is built to represent the graph, converting vertex labels from 1-based to 0-based for easier manipulation.
3. **BFS for Component Counting**: The BFS algorithm traverses each unvisited node, marking all reachable nodes, and counts each traversal as a separate connected component.
4. **Edge Deletion Calculation**: The result is derived by subtracting the maximum edges allowed in a forest (which is \( N - k \)) from the total edges \( M \). This gives the number of edges to delete to eliminate all cycles, resulting in a forest.

This approach efficiently handles the problem by leveraging graph traversal to determine connectivity and applying a straightforward mathematical formula to compute the solution. The complexity is linear with respect to the number of vertices and edges, making it suitable for large inputs.
