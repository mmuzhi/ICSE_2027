   - For each -1, the chosen value (x or y) lies within the interval defined by its adjacent fixed numbers (if any) and D.
   - The two numbers x and y must be chosen such that they cover all intervals derived from the gaps (missing values).

3. **Feasibility Check**: For each candidate D, we:
   - Compute the interval for each gap based on its adjacent fixed numbers (if any). The interval must be non-empty for the gap to be feasible.
   - Use a greedy algorithm to determine if two numbers (x, y) can cover all intervals. The greedy algorithm sorts intervals by their right endpoints and checks if two points can cover all intervals.

4. **Complexity Analysis**: The binary search runs in O(log(max_val)) time, where max_val is the upper bound (10^9). Each feasibility check involves O(n log n) time due to sorting (n is the number of gaps, up to 10^5). Thus, the overall complexity is O(n log n log(max_val)), which is efficient for the given constraints.

### Solution Code
```python
class Solution:
    def minDifference(self, nums: List[int]) -> int:
        n = len(nums)
        if all(x == -1 for x in nums):
            return 0
        
        fixed_adjacent_max = 0
        for i in range(n-1):
            if nums[i] != -1 and nums[i+1] != -1:
                diff = abs(nums[i] - nums[i+1])
                if diff > fixed_adjacent_max:
                    fixed_adjacent_max = diff
        
        if fixed_adjacent_max == 0:
            return 0
        
        low = fixed_adjacent_max
        high = 10**9
        
        def feasible(D):
            intervals = []
            for i in range(n):
                if nums[i] == -1:
                    left = nums[i-1] if i-1 >= 0 and nums[i-1] != -1 else None
                    right = nums[i+1] if i+1 < n and nums[i+1] != -1 else None
                    if left is not None and right is not None:
                        l_interval = max(left - D, right - D)
                        r_interval = min(left + D, right + D)
                        if l_interval > r_interval:
                            return False
                        intervals.append((l_interval, r_interval))
                    elif left is not None:
                        l_interval = left - D
                        r_interval = left + D
                        intervals.append((l_interval, r_interval))
                    elif right is not None:
                        l_interval = right - D
                        r_interval = right + D
                        intervals.append((l_interval, r_interval))
            if not intervals:
                return True
            
            intervals_sorted = sorted(intervals, key=lambda x: x[1])
            count = 0
            i = 0
            total = len(intervals_sorted)
            while i < total:
                count += 1
                if count > 2:
                    return False
                current_point = intervals_sorted[i][1]
                j = i
                while j < total and intervals_sorted[j][0] <= current_point:
                    j += 1
                i = j
            return True
        
        ans = high
        while low <= high:
            if feasible(low):
                ans = low
                high = low - 1
            else:
                low += 1
        return ans
```

### Explanation
1. **Initial Checks**: The solution first checks if all elements are -1, in which case the answer is 0 since replacing all with the same number (x = y) results in zero adjacent differences.
2. **Fixed Adjacent Differences**: It calculates the maximum difference between adjacent fixed numbers (non -1) to set the initial lower bound for binary search.
3. **Binary Search**: The binary search between `low` (maximum fixed adjacent difference) and `high` (10^9) checks each candidate D for feasibility.
4. **Feasibility Function**: For each candidate D, the function computes intervals for each -1 based on adjacent fixed numbers. If any interval is empty, D is not feasible. Otherwise, it uses a greedy algorithm to check if two numbers can cover all intervals, ensuring the solution is efficient.
5. **Result**: The binary search narrows down the smallest feasible D, which is returned as the result. This approach efficiently minimizes the maximum adjacent difference by leveraging binary search and interval covering.
