To solve this problem, we need to determine whether it's possible to select five cards from a given set of seven cards such that the selected cards form a full house. A full house is defined as three cards of one integer and two cards of another distinct integer.

### Approach
1. **Frequency Counting**: First, we count the frequency of each distinct integer in the input list. This helps us understand how many times each number appears.
2. **Check for Full House Condition**: We then check if there exists any pair of distinct integers where one integer appears at least three times and the other appears at least two times. If such a pair exists, we can form a full house by selecting three cards of the first integer and two cards of the second integer.
3. **Edge Cases**: If there are fewer than two distinct integers, it's impossible to form a full house, so we immediately return "No" in such cases. Additionally, if no such pair is found during the iteration, we return "No".

### Solution Code
```python
from collections import Counter

def main():
    data = list(map(int, input().split()))
    freq = Counter(data)
    distinct_nums = list(freq.keys())
    
    if len(distinct_nums) < 2:
        print("No")
        return
        
    found = False
    for i in range(len(distinct_nums)):
        for j in range(i+1, len(distinct_nums)):
            if freq[distinct_nums[i]] >= 3 and freq[distinct_nums[j]] >= 2:
                found = True
                break
        if found:
            break
            
    print("Yes" if found else "No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Frequency Counting**: The `Counter` from the `collections` module is used to count occurrences of each integer in the input list. This gives us a dictionary where keys are the distinct integers and values are their respective counts.
2. **Distinct Numbers Check**: If there are fewer than two distinct integers, it's impossible to form a full house (since we need two different integers), so we print "No" and exit.
3. **Pair Validation**: We iterate over all pairs of distinct integers. For each pair, we check if one integer appears at least three times and the other appears at least twice. If such a pair is found, we set a flag and break out of the loops.
4. **Result Determination**: Based on the flag, we print "Yes" if a valid pair was found, otherwise "No".

This approach efficiently checks the necessary conditions to form a full house by leveraging frequency counting and simple iteration over distinct pairs, ensuring optimal performance for the given constraints.
