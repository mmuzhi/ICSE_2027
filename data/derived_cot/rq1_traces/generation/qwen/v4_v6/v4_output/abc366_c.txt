- 1 \leq x \leq 10^{6}
- When a query of the second type is given, the bag has a ball with the integer x written on it.
- There is at least one query of the third type.
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to efficiently process a series of queries on a bag of balls, where each query either adds a ball, removes a ball, or asks for the number of distinct integers currently in the bag. The challenge lies in handling these operations quickly, especially given the constraints that the number of queries can be as high as 200,000.

### Approach
1. **Problem Analysis**: The problem involves three types of operations on a bag of balls: adding a ball, removing a ball, and querying the count of distinct integers in the bag. The key observation is that we need to keep track of the count of each integer in the bag and the number of distinct integers present.
2. **Intuition**: Using a frequency array (or hash map) to keep track of the count of each integer allows us to efficiently update counts and determine distinct integers. However, given the constraints (integers up to 10^6), an array-based approach is more efficient in terms of both time and space complexity.
3. **Algorithm Selection**: We use an array of size 1,000,001 (to cover integers from 1 to 1,000,000) to store the frequency of each integer. For each query:
   - **Addition (1 x)**: If the frequency of `x` is zero, increment the distinct count. Then, increment the frequency of `x`.
   - **Removal (2 x)**: Decrement the frequency of `x`. If the frequency drops to zero, decrement the distinct count.
   - **Query (3)**: Simply output the current distinct count.
4. **Complexity Analysis**: Each operation (addition, removal, query) is processed in constant time O(1) due to the array operations. The space complexity is O(1,000,001) for the frequency array, which is acceptable given the problem constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    q = int(data[0])
    n_max = 1000000
    size = n_max + 1
    arr = [0] * size
    distinct_count = 0
    
    output_lines = []
    
    for i in range(1, q + 1):
        parts = data[i].split()
        if not parts:
            continue
            
        if parts[0] == '1':
            x = int(parts[1])
            if arr[x] == 0:
                distinct_count += 1
            arr[x] += 1
        elif parts[0] == '2':
            x = int(parts[1])
            arr[x] -= 1
            if arr[x] == 0:
                distinct_count -= 1
        elif parts[0] == '3':
            output_lines.append(str(distinct_count))
    
    print("\n".join(output_lines))

if __name__ == "__main__":
    main()
```

### Explanation
- **Initialization**: The code reads all input lines at once for efficiency, especially given the large number of queries. The frequency array `arr` is initialized to keep track of how many times each integer appears in the bag. The `distinct_count` variable maintains the number of unique integers currently in the bag.
- **Processing Queries**: For each query:
  - **Addition (1 x)**: The code checks if the integer `x` was not present before (frequency is zero). If so, it increments `distinct_count`. Then, it updates the frequency of `x`.
  - **Removal (2 x)**: The code decrements the frequency of `x`. If the frequency reaches zero, it decrements `distinct_count`.
  - **Query (3)**: The current value of `distinct_count` is recorded for output.
- **Output**: After processing all queries, the results of all type 3 queries are printed in order.

This approach efficiently handles the operations with constant time per query, making it suitable for large input sizes.
