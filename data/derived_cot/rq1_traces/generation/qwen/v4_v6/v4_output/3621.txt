To solve this problem, we need to determine the minimum number of operations required to make all elements in an array equal to a given integer \( k \). Each operation involves selecting a valid integer \( h \) and setting all elements strictly greater than \( h \) to \( h \). The solution involves checking if it's possible to achieve the goal and, if possible, calculating the minimal operations needed.

### Approach
1. **Check for Feasibility**: First, we check if any element in the array is less than \( k \). If such an element exists, it's impossible to make all elements equal to \( k \) because operations can only reduce elements that are strictly greater than \( h \). Hence, we return -1 in this case.
2. **Count Distinct Values Above \( k \)**: If all elements are at least \( k \), the next step is to count the number of distinct values in the array that are strictly greater than \( k \). Each distinct value above \( k \) requires at least one operation to reduce it to \( k \) or another value closer to \( k \). The minimal number of operations is equal to the number of such distinct values.

### Solution Code
```python
class Solution:
    def minOperations(self, nums: List[int], k: int) -> int:
        for num in nums:
            if num < k:
                return -1
        distinct = set()
        for num in nums:
            if num > k:
                distinct.add(num)
        return len(distinct)
```

### Explanation
1. **Feasibility Check**: The loop iterates through each element in the array. If any element is found to be less than \( k \), the function immediately returns -1, indicating it's impossible to achieve the goal.
2. **Distinct Values Count**: The second loop collects all distinct values in the array that are strictly greater than \( k \) into a set. The size of this set gives the number of operations needed because each distinct value must be reduced to \( k \) through at least one operation. The operations can be performed in any order, but each distinct value above \( k \) requires its own operation to be flattened down to \( k \).

This approach efficiently checks for feasibility and calculates the minimal operations by leveraging the properties of set operations to count distinct values above \( k \). The solution is optimal with a time complexity of \( O(n) \), where \( n \) is the number of elements in the array, as it involves two linear passes through the array.
