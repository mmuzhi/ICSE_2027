The solution involves checking specific conditions derived from the initial sequence to decide the possibility of forming a non-decreasing sequence.

### Approach
1. **Problem Analysis**: The key observation is that the operations allow transferring values from right to left, but the total sum of the sequence remains unchanged. The challenge is to check if there exists a non-decreasing sequence with the same total sum as the original sequence, where each element in the sequence is at least the value obtained by subtracting the sum of all elements to its right from the original element.

2. **Intuition**: For each element in the sequence, the minimal value it can take in the final non-decreasing sequence is determined by the original value minus the sum of all elements to its right. The sequence must be non-decreasing and sum to the original total sum. 

3. **Algorithm Selection**: 
   - **Suffix Sum Calculation**: Compute the suffix sums for the sequence to efficiently determine the sum of elements to the right of each element.
   - **Minimal Value Calculation**: For each element, compute the minimal value it can have in the final sequence (original value minus the sum of elements to its right).
   - **Non-decreasing Sequence Construction**: Construct a non-decreasing sequence starting from the minimal values, ensuring each subsequent element is at least as large as the previous one.
   - **Feasibility Check**: Verify if the sum of this constructed sequence is less than or equal to the original total sum. If it is, forming the sequence is possible; otherwise, it's not.

4. **Complexity Analysis**: The algorithm processes each test case in linear time, O(N), where N is the length of the sequence. Given the constraints, this approach efficiently handles the problem within acceptable limits.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    t = int(data[0])
    index = 1
    results = []
    for _ in range(t):
        n = int(data[index])
        index += 1
        A = list(map(int, data[index:index+n]))
        index += n
        
        S = sum(A)
        suffix = [0] * (n + 1)
        for i in range(n - 1, -1, -1):
            suffix[i] = suffix[i + 1] + A[i]
        
        minB = [0] * n
        for i in range(n):
            minB[i] = A[i] - suffix[i + 1]
        
        B = [0] * n
        B[0] = minB[0]
        for i in range(1, n):
            if minB[i] > B[i - 1]:
                B[i] = minB[i]
            else:
                B[i] = B[i - 1]
                
        total_min = sum(B)
        if total_min <= S:
            results.append("Yes")
        else:
            results.append("No")
            
    for res in results:
        print(res)

if __name__ == "__main__":
```
