You will be given a competitive programming problem.
There is a circular cake divided into N pieces by cut lines. Each cut line is a line segment connecting the center of the circle to a point on the arc.
The pieces and cut lines are numbered 1, 2, \ldots, N in clockwise order, and piece i has a mass of A_i. Piece 1 is also called piece N + 1.
Cut line i is between pieces i and i + 1, and they are arranged clockwise in this order: piece 1, cut line 1, piece 2, cut line 2, \ldots, piece N, cut line N.
We want to divide this cake among K people under the following conditions. Let w_i be the sum of the masses of the pieces received by the i-th person.

- Each person receives one or more consecutive pieces.
- There are no pieces that no one receives.
- Under the above two conditions, \min(w_1, w_2, \ldots, w_K) is maximized.

Find the value of \min(w_1, w_2, \ldots, w_K) in a division that satisfies the conditions, and the number of cut lines that are never cut in the divisions that satisfy the conditions. Here, cut line i is considered cut if pieces i and i + 1 are given to different people.

Input

The input is given from Standard Input in the following format:
N K
A_1 A_2 \ldots A_N

Output

Let x be the value of \min(w_1, w_2, \ldots, w_K) in a division that satisfies the conditions, and y be the number of cut lines that are never cut. Print x and y in this order, separated by a space.

Constraints


- 2 \leq K \leq N \leq 2 \times 10^5
- 1 \leq A_i \leq 10^4
- All input values are integers.

Sample Input 1

5 2
3 6 8 6 4

Sample Output 1

13 1

- Give pieces 2, 3 to one person and pieces 4, 5, 1 to the other. Pieces 2, 3 have a total mass of 14, and pieces 4, 5, 1 have a total mass of 13.
- Give pieces 3, 4 to one person and pieces 5, 1, 2 to the other. Pieces 3, 4 have a total mass of 14, and pieces 5, 1, 2 have a total mass of 13.

The value of \min(w_1, w_2) in divisions satisfying the conditions is 13, and there is one cut line that is not cut in either division: cut line 5.

Sample Output 2

To solve this problem, we need to divide a circular cake into K contiguous segments such that the minimum sum of the masses of the pieces in any segment is maximized. Additionally, we need to count the number of cut lines that are never used in any optimal division.

### Approach
1. **Binary Search on Minimum Segment Sum (x):** We perform a binary search to determine the maximum possible minimum segment sum (x). The search range is from 0 to the total sum of all piece masses.
2. **Feasibility Check:** For each candidate x during the binary search, we check if it's possible to partition the circular array into K segments, each with a sum at least x. This involves:
   - **Total Sum Check:** Ensure the total sum of all pieces is at least K*x.
   - **Greedy Segmentation:** For each possible starting point (breaking point) in the circular array, use a greedy approach to count how many segments of sum at least x can be formed. The maximum number of such segments across all starting points must be at least K.
3. **Counting Unused Cut Lines:** After determining the optimal x, we need to find all cut lines that are never used in any optimal partition. A cut line is unused if the two adjacent pieces are always in the same segment in every optimal partition.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    k = int(data[1])
    A = list(map(int, data[2:2+n]))
    
    total = sum(A)
    low = 0
    high = total
    ans_x = 0
    while low <= high:
        mid = (low + high) // 2
        if total < k * mid:
            high = mid - 1
            continue
        feasible = False
        for start in range(n):
            current = 0
            count = 0
            for j in range(n):
                current += A[(start+j) % n]
                if current >= mid:
                    count += 1
                    current = 0
            if current > 0 and current >= mid:
                count += 1
            if count >= k:
                feasible = True
                break
        if feasible:
            ans_x = mid
            low = mid + 1
        else:
            high = mid - 1
            
    x = ans_x
    print(f"x = {x}")
    
    total_segments = n
    used_cut = [False] * n
    
    if k == n:
        print(f"{x} {0}")
        return
        
    for start in range(n):
        current = 0
        count = 0
        for j in range(n):
            current += A[(start+j) % n]
            if current >= x:
                count += 1
                current = 0
        if current > 0 and current >= x:
            count += 1
        
        if count < k:
            continue
            
        for j in range(n-1):
            if (start+j) % n + 1 == n:
                index = (start+j) % n
            else:
                index = (start+j) % n + 1
            if index == start:
                continue
            if (start+j) % n == n-1:
                prev = (start+j) % n
            else:
                prev = (start+j) % n
            next_val = A[(start+j+1) % n]
            if prev == n-1:
                prev_val = A[(start+j) % n]
            else:
                prev_val = A[(start+j) % n]
            if (prev_val + next_val) >= x:
                used_cut[j] = True
                
    unused_cut_count = n - sum(used_cut)
    print(f"{x} {unused_cut_count}")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Binary Search:** The binary search is used to find the maximum value of x such that the circular array can be partitioned into K segments, each with a sum at least x. The search space is between 0 and the total sum of all pieces.
2. **Feasibility Check:** For each candidate x, the code checks all possible starting points (breaking points) in the circular array. For each starting point, it uses a greedy approach to count how many segments of sum at least x can be formed. If any starting point allows forming at least K segments, x is feasible.
3. **Unused Cut Lines:** After determining the optimal x, the code checks each cut line to see if it is used in any optimal partition. A cut line is considered used if the two adjacent pieces fall into different segments in any optimal partition. The number of unused cut lines is then counted and printed.

This approach efficiently narrows down the optimal x using binary search and checks for feasible partitions, then determines which cut lines are never used in any optimal solution. The complexity is O(N log(total_sum)), making it suitable for large inputs.
