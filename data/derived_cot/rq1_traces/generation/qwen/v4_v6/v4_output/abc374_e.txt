Print the answer as an integer.

- 1 \le N \le 100
- 1 \le A_i,B_i \le 100
- 1 \le P_i,Q_i,X \le 10^7

To solve this problem, we need to maximize the production capacity of a manufacturing process by strategically purchasing machines that can handle each of the N processes. The production capacity is defined as the minimum production rate across all processes. Given a budget constraint, we need to determine the highest achievable production capacity.

### Approach
1. **Problem Analysis**: The problem involves selecting machines for each process to maximize the bottleneck (minimum production rate) across all processes while staying within a given budget. Each process can be handled by two types of machines, each with different production rates and costs. The challenge is to find the optimal combination of machines for each process to achieve the highest possible bottleneck production rate without exceeding the budget.

2. **Key Insight**: The solution involves binary search over possible production capacities (M). For each candidate M, we check if it's feasible to achieve at least M production for each process within the budget. The feasibility is determined by calculating the minimal cost for each process to reach at least M production, considering both machine types and their combinations.

3. **Algorithm Selection**: 
   - **Binary Search**: We perform binary search on the possible production capacity values (M) from 0 to an upper bound derived from the budget and machine costs.
   - **Extended Euclidean Algorithm**: For each process and candidate M, we use the extended Euclidean algorithm to solve a congruence equation that helps in efficiently determining the minimal cost to achieve the production capacity M by considering combinations of machines.

4. **Complexity Analysis**: The binary search runs in O(log M) time, and for each candidate M, we process each of the N processes. For each process, we solve a congruence equation in O(B_i) time, where B_i is the production rate of one machine type. Given that N is up to 100 and B_i is up to 100, the overall complexity is manageable (around 30 * 100 * 100 operations).
