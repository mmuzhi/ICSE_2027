=== ANSWER ===

To solve this problem, we need to find a subgrid within a given N x N grid S that exactly matches a given M x M grid T. The solution involves checking all possible positions in grid S where the M x M subgrid could fit and verifying if the subgrid matches T.

### Approach
1. **Problem Analysis**: The problem requires identifying the top-left corner (a, b) in grid S such that the subgrid starting at (a, b) and of size M x M matches grid T exactly. Both grids consist of '.' (white) and '#' (black) cells.
2. **Intuition**: Since the grids are small (N and M up to 50), a brute-force approach is feasible. We can iterate over all possible starting positions (a, b) in S where a subgrid of size M x M can fit (i.e., from 1 to N-M+1 for both rows and columns).
3. **Algorithm Selection**: For each candidate position (a, b), we check every cell in the corresponding M x M subgrid of S against the cells in T. If all cells match, we return (a, b) as the solution.
4. **Optimization**: The solution leverages early termination once a match is found, ensuring efficiency given the constraints.
5. **Complexity Analysis**: The worst-case time complexity is O((N-M+1)^2 * M^2), which is manageable for the given constraints (N, M ≤ 50).

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    n_m_line = data[0].split()
    N = int(n_m_line[0])
    M = int(n_m_line[1])
    
    grid_S = []
    for i in range(1, 1+N):
        grid_S.append(data[i].strip())
        
    grid_T = []
    for i in range(1+N, 1+N+M):
        grid_T.append(data[i].strip())
        
    for a in range(0, N - M + 1):
        for b in range(0, N - M + 1):
            match = True
            for i in range(M):
                for j in range(M):
                    if grid_S[a+i][b+j] != grid_T[i][j]:
                        match = False
                        break
                if not match:
                    break
            if match:
                print(a+1, b+1)
                return

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line contains integers N (size of grid S) and M (size of grid T). The next N lines represent grid S, and the following M lines represent grid T.
2. **Grid Processing**: Grid S and T are stored as lists of strings for easy access.
3. **Brute-Force Search**: The code iterates over all possible starting positions (a, b) in grid S where a subgrid of size M x M can fit. For each position, it checks if the corresponding subgrid matches grid T.
4. **Output**: The first valid position (a, b) found is printed in 1-indexed format (since the problem requires 1-based indexing), and the program exits immediately upon finding the solution.

This approach efficiently checks all possible subgrids in S against T, leveraging early termination to optimize performance. The solution is both straightforward and effective for the given problem constraints.
