=== ANSWER ===

To solve this problem, we need to compute the sum of the remainders when the sums of all contiguous subarrays of a given sequence are divided by a positive integer M. The challenge lies in efficiently calculating this sum without resorting to a brute-force approach, which would be computationally infeasible for large input sizes.

### Approach
1. **Prefix Sum Calculation**: Compute the prefix sums of the given sequence. This helps in efficiently calculating the sum of any contiguous subarray.
2. **Modulo Operation**: For each prefix sum, compute its value modulo M. This transforms the problem into working with values in the range [0, M-1].
3. **Modular Arithmetic Insight**: The sum of the remainders can be broken down into two parts:
   - The sum of the differences between the modulo values of the end and start indices of each subarray.
   - The adjustment for cases where the start index's modulo value is greater than the end index's modulo value, requiring an additional M to be added to the remainder.
4. **Fenwick Tree for Inversion Count**: Use a Fenwick Tree (Binary Indexed Tree) to efficiently count the number of inversions in the modulo values array. This helps in determining how many times we need to add M to the sum of remainders.
5. **Combining Results**: The final result is obtained by combining the sum of the differences (Part1) and the adjustment terms (Part2), which is M multiplied by the inversion count.

### Solution Code
```python
import sys

class Fenw:
    def __init__(self, size):
        self.n = size
        self.tree = [0] * (self.n + 1)
    
    def update(self, index, delta):
        while index <= self.n:
            self.tree[index] += delta
            index += index & -index
            
    def query(self, index):
        s = 0
        while index:
            s += self.tree[index]
            index -= index & -index
        return s

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    m_val = int(data[1])
    A = list(map(int, data[2:2 + n]))
    
    Q = [0] * (n + 1)
    for i in range(1, n + 1):
        Q[i] = Q[i - 1] + A[i - 1]
    
    B = [q % m_val for q in Q]
    
    part1a = 0
    part1b = 0
    for i in range(n + 1):
        part1a += B[i] * i
        part1b += B[i] * (n - i)
    
    fenw_tree = Fenw(m_val)
    inv_count = 0
    for i in range(n + 1):
        count_greater = i - fenw_tree.query(B[i] + 1)
        inv_count += count_greater
        fenw_tree.update(B[i] + 1, 1)
    
    ans = part1a - part1b + m_val * inv_count
    print(ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Prefix Sum Calculation**: The prefix sum array `Q` is computed to store cumulative sums of the sequence `A`. This allows efficient calculation of any subarray sum as `Q[r] - Q[l]`.
2. **Modulo Operation**: Each element in the prefix sum array is taken modulo `M` to transform the problem into working with smaller values, simplifying subsequent calculations.
3. **Part1 Calculation**: The sum of the differences between the modulo values of subarray ends and starts is computed. This involves summing `B[i] * i` for all positions and subtracting `B[i] * (n - i)` to account for all possible subarrays.
4. **Fenwick Tree for Inversion Count**: The Fenwick Tree is used to count the number of inversions (pairs where `B[l] > B[r]` for `l < r`). This count determines how many times M needs to be added to the sum of remainders.
5. **Result Combination**: The final result is the sum of Part1 (the sum of differences) and Part2 (M multiplied by the inversion count), providing the desired sum of remainders for all contiguous subarrays modulo M.

This approach efficiently handles the problem by leveraging prefix sums, modular arithmetic, and a Fenwick Tree to avoid the O(N^2) complexity of a brute-force solution.
