To solve this problem, we need to maximize the minimum value in the `gameScore` array after performing at most `m` moves. The `gameScore` array is initially zeros, and each move allows us to move left or right by one index (if within bounds) and add the corresponding `points` value to the `gameScore` at that index. The challenge is to distribute the moves optimally to ensure the smallest value in `gameScore` is as large as possible.

### Approach
1. **Binary Search on Answer**: The solution involves performing a binary search on the possible values of the minimum `gameScore` (denoted as `X`). For each candidate `X`, we check if it's feasible to distribute the moves such that every index in the array has a `gameScore` of at least `X`.
2. **Feasibility Check**: For each candidate `X`, calculate the minimum number of visits required for each index `i` to ensure `gameScore[i] >= X`. This is computed as `ceil(X / points[i])` for each index. The sum of these values across all indices must be less than or equal to `m + 1` (since each move lands on an index, and the first move is from the starting position -1 to index 0).
3. **Binary Search Adjustment**: If the total required visits for a candidate `X` is within the allowed moves (`m + 1`), we try a higher `X`; otherwise, we try a lower `X`.

### Solution Code
```python
class Solution:
    def maxScore(self, points: List[int], m: int) -> int:
        n = len(points)
        high = 10**18
        low = 0
        ans = 0
        while low <= high:
            mid = (low + high) // 2
            total = 0
            for i in range(n):
                if mid == 0:
                    add = 0
                else:
                    add = (mid + points[i] - 1) // points[i]
                total += add
                if total > m + 1:
                    break
            if total <= m + 1:
                ans = mid
                low = mid + 1
            else:
                high = mid - 1
        return ans
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to a very large number (10^18) to cover all possible values of `X`.
2. **Feasibility Check for Mid Value**: For each mid value `X` during the binary search, compute the total number of visits required for all indices to have a `gameScore` of at least `X`. This is done by summing `ceil(X / points[i])` for each index `i`.
3. **Adjust Search Range**: If the total visits required are within the allowed moves (`m + 1`), it means `X` is feasible, and we try a higher value. Otherwise, we try a lower value.
4. **Result**: The binary search concludes with the highest feasible `X` stored in `ans`, which is the maximum possible minimum value in `gameScore` achievable with at most `m` moves.

This approach efficiently narrows down the possible values of `X` using binary search and checks feasibility in linear time for each candidate, making it suitable for large input sizes as specified in the problem constraints.
