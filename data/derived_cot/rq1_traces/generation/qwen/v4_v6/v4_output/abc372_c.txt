- 1 \le X_i \le N

=== ANSWER ===

To solve this problem, we need to efficiently count the occurrences of the substring "ABC" in a string after each query that modifies a single character in the string. Given the constraints, a brute-force approach of scanning the entire string for each query is infeasible. Instead, we leverage the fact that each query affects only a limited number of potential "ABC" substrings, allowing us to update the count in constant time per query.

### Approach
1. **Initial Setup**: Convert the input string into a list for mutable operations. Precompute the initial count of "ABC" substrings by scanning the entire string once.
2. **Query Processing**: For each query:
   - **Check for No Change**: If the character to be inserted is the same as the current character, skip further processing for this query.
   - **Update Affected Triplets**: The query modifies a single character, which can affect up to three potential "ABC" substrings (those starting at positions `pos-2`, `pos-1`, and `pos`). Before updating the character, decrement the total count if any of these triplets were "ABC". After updating the character, increment the total count if any of these triplets are now "ABC".
3. **Output the Result**: After processing each query, output the current count of "ABC" substrings.

This approach efficiently handles each query in constant time by focusing only on the relevant triplets affected by the character modification, thus optimizing both time and space complexity.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    first_line = data[0].split()
    n = int(first_line[0])
    q = int(first_line[1])
    s_line = data[1].strip()
    queries = []
    for i in range(2, 2 + q):
        parts = data[i].split()
        if len(parts) < 2:
            continue
        x = int(parts[0])
        c = parts[1]
        queries.append((x, c))
    
    s_list = list(s_line)
    total = 0
    for i in range(n - 2):
        if s_list[i] == 'A' and s_list[i+1] == 'B' and s_list[i+2] == 'C':
            total += 1
            
    output_lines = []
    for (x, c) in queries:
        pos = x - 1
        old_char = s_list[pos]
        if old_char == c:
            output_lines.append(str(total))
            continue
            
        for start in (pos-2, pos-1, pos):
            if 0 <= start <= n - 3:
                if s_list[start] == 'A' and s_list[start+1] == 'B' and s_list[start+2] == 'C':
                    total -= 1
                    
        s_list[pos] = c
        
        for start in (pos-2, pos-1, pos):
            if 0 <= start <= n - 3:
                if s_list[start] == 'A' and s_list[start+1] == 'B' and s_list[start+2] == 'C':
                    total += 1
                    
        output_lines.append(str(total))
        
    print("\n".join(output_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initial Setup**: The input is read and parsed into the string length `n`, number of queries `q`, the initial string, and the list of queries.
2. **Initial Count Calculation**: The initial count of "ABC" substrings is computed by scanning the string once.
3. **Query Processing**: For each query:
   - **No Change Handling**: If the character to be inserted is the same as the current character, the count remains unchanged.
   - **Triplet Adjustment**: Before updating the character, the algorithm checks if any of the three triplets (substrings of length 3 starting at positions `pos-2`, `pos-1`, or `pos`) were "ABC". If so, the count is decremented. After updating the character, the algorithm checks the same triplets again and increments the count if they form "ABC".
4. **Output**: After processing each query, the current count is recorded and printed at the end.

This method ensures that each query is processed in constant time, making the solution efficient even for large input sizes.
