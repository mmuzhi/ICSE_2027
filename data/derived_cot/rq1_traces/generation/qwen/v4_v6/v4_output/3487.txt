idx is an element of targetIndices.
pattern remains a subsequence of source after removing the character.

Return the maximum number of operations that can be performed.

Input: source = "abbaa", pattern = "aba", targetIndices = [0,1,2]
Output: 1
Explanation:
We can't remove source[0] but we can do either of these two operations:

Remove source[1], so that source becomes "a_baa".
Remove source[2], so that source becomes "ab_aa".


Example 2:

Input: source = "bcda", pattern = "d", targetIndices = [0,3]
Output: 2
Explanation:
We can remove source[0] and source[3] in two operations.

Example 3:

Input: source = "dda", pattern = "dda", targetIndices = [0,1,2]
Output: 0
Explanation:
We can't remove any character from source.

Example 4:

Input: source = "yeyeykyded", pattern = "yeyyd", targetIndices = [0,2,3,4]
Output: 2
Explanation:
We can remove source[2] and source[3] in two operations.

 
Constraints:

1 <= n == source.length <= 3 * 10^3
1 <= pattern.length <= n
1 <= targetIndices.length <= n
targetIndices is sorted in ascending order.
The input is generated such that targetIndices contains distinct elements in the range [0, n - 1].

To solve this problem, we need to determine the maximum number of operations that can be performed on the source string such that after each operation (removing a character at a specified index), the pattern remains a subsequence of the source string. The operations can only remove characters from the given `targetIndices` list.

### Approach
1. **Problem Analysis**: The problem requires us to maximize the number of characters removed from the `targetIndices` list while ensuring that the pattern remains a subsequence of the source string after each removal. The key insight is that we need to match the pattern in the source string using as few `targetIndices` indices as possible, allowing the remaining indices in `targetIndices` to be removed.

2. **Dynamic Programming (DP) Setup**: We use a DP table `dp[i][j]` where `i` represents the number of characters processed in the source string, and `j` represents the number of characters matched in the pattern. The value `dp[i][j]` stores the minimal number of `targetIndices` indices used to match the first `j` characters of the pattern up to the `i-th` character in the source string.

3. **DP Transition**: For each character in the source string, we consider two options:
   - **Not using the current character** for matching the pattern, thus propagating the DP state from `dp[i][j]` to `dp[i+1][j]`.
   - **Using the current character** if it matches the next character in the pattern, updating `dp[i+1][j+1]` by adding 1 to the count if the current index is in `targetIndices`.

4. **Result Calculation**: After processing all characters in the source string, the minimal number of `targetIndices` indices used to match the entire pattern is found in `dp[n][m]`. The maximum number of operations is then the total number of `targetIndices` minus this minimal count.

### Solution Code
```python
class Solution:
    def maxRemovals(self, source: str, pattern: str, targetIndices: List[int]) -> int:
        n = len(source)
        m = len(pattern)
        if m == 0:
            return len(targetIndices)
        
        dp = [[float('inf')] * (m + 1) for _ in range(n + 1)]
        dp[0][0] = 0
        
        target_set = set(targetIndices)
        
        for i in range(n):
            for j in range(m + 1):
                if dp[i][j] == float('inf'):
                    continue
                if dp[i+1][j] > dp[i][j]:
                    dp[i+1][j] = dp[i][j]
                if j < m and source[i] == pattern[j]:
                    add = 1 if i in target_set else 0
                    if dp[i+1][j+1] > dp[i][j] + add:
                        dp[i+1][j+1] = dp[i][j] + add
        
        minimal_used = dp[n][m]
        return len(targetIndices) - minimal_used
```

### Explanation
1. **Initialization**: The DP table is initialized with infinity to represent unreachable states, except for `dp[0][0]` which is set to 0 (no characters processed, no matches made).

2. **DP Table Population**: For each character in the source string, the algorithm checks all possible states of pattern matching. If the current character matches the next character in the pattern, it updates the DP state by considering whether the current index is in `targetIndices` (incrementing the count if it is).

3. **Result Extraction**: The solution is derived by subtracting the minimal number of `targetIndices` indices used from the total number of `targetIndices` indices, giving the maximum number of operations (removals) that can be performed while keeping the pattern as a subsequence.

This approach efficiently leverages dynamic programming to explore all possible ways to match the pattern while minimizing the use of `targetIndices` indices, thus maximizing the number of allowed operations.
