To solve this problem, we need to count the number of permutations of length N that satisfy M given conditions. Each condition specifies that in a subarray from L_i to R_i, the element at position X_i should not be the maximum. The solution involves a dynamic programming approach that processes the permutation from the largest element down to the smallest, ensuring that at each step, the chosen position does not violate any active conditions.

### Approach
1. **Problem Analysis**: The problem requires counting permutations where specific elements in certain subarrays are not the maximum. Each condition must be satisfied for the entire permutation. The challenge is to efficiently handle these constraints without resorting to brute-force methods, which would be computationally infeasible for large N and M.

2. **Intuition**: The solution leverages the idea of processing elements from largest to smallest. For each element, we determine valid positions where it can be placed without violating any active conditions. Active conditions are those that have not been satisfied by placing a larger element in their respective subarrays.

3. **Algorithm Selection**: 
   - **Preprocessing**: For each condition, store the subarray it affects. For each position, precompute the list of conditions that would be affected if an element is placed there.
   - **Dynamic Programming**: Iterate from the largest element (N) down to 1. For each element, count the number of valid positions where it can be placed without violating any active conditions. Update the active conditions by deactivating those that are satisfied by placing the current element in their subarray.

4. **Complexity Analysis**: The algorithm involves preprocessing the conditions, which takes O(M*N) time. The main loop processes each element once, and for each position, it checks all relevant conditions, leading to an overall time complexity of O(N^2 * M) in the worst case. Given the constraints (N ≤ 500, M ≤ 10^5), this approach is feasible with optimizations.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    it = iter(data)
    n = int(next(it))
    m = int(next(it))
    conditions = []
    for _ in range(m):
        L = int(next(it))
        R = int(next(it))
        X = int(next(it))
        conditions.append((L, R, X))
    
    intervals = [[] for _ in range(n+1)]
    for j in range(m):
        L, R, X = conditions[j]
        for p in range(L, R+1):
            if p <= n:
                intervals[p].append(j)
    
    active = [True] * m
    assigned = [False] * (n+1)
    forbidden_count = [0] * (n+1)
    for j in range(m):
        X = conditions[j][2]
        forbidden_count[X] += 1
    
    ways = 1
    mod = 998244353
    
    for k in range(n, 0, -1):
        available_positions = [p for p in range(1, n+1) if not assigned[p]]
        if not available_positions:
            ways = 0
            break
        
        allowed_positions = []
        for p in available_positions:
            if forbidden_count[p] == 0:
                allowed_positions.append(p)
        
        if not allowed_positions:
            ways = 0
            break
        
        ways = (ways * len(allowed_positions)) % mod
        
        p0 = allowed_positions[0]
        assigned[p0] = True
        for j in intervals[p0]:
            if active[j]:
                active[j] = False
                Xj = conditions[j][2]
                forbidden_count[Xj] -= 1
    
    print(ways % mod)

if __name__ == '__main__':
    main()
```

5. **Updating Conditions**: After placing an element, the algorithm deactivates any conditions that are satisfied by this placement, updating the `forbidden_count` accordingly.
