To solve this problem, we need to count the number of substrings in a given binary string that satisfy a k-constraint. The k-constraint is met if either the number of '0's or the number of '1's in the substring is at most k. 

### Approach
1. **Problem Analysis**: The problem requires checking all possible substrings of the given binary string to determine if they meet the k-constraint. A substring is valid if it contains at most k '0's or at most k '1's. 
2. **Intuition**: For each starting index in the string, we can expand the substring to the right, counting the number of '0's and '1's. If at any point the counts exceed k for both characters, we can break early since any further expansion will only increase these counts, making the substring invalid.
3. **Algorithm Selection**: We use a nested loop where the outer loop runs from each starting index, and the inner loop expands the substring to the right. For each expansion, we update the counts of '0's and '1's. If the counts of either character exceed k, we break the inner loop early as further substrings starting at the same index will also be invalid.
4. **Complexity Analysis**: The algorithm has a time complexity of O(n^2) in the worst case, where n is the length of the string. This is because for each starting index, we might need to check up to n substrings. However, in practice, the inner loop often breaks early, optimizing the process.

### Solution Code
```python
class Solution:
    def countKConstraintSubstrings(self, s: str, k: int) -> int:
        n = len(s)
        total = 0
        for i in range(n):
            zeros = 0
            ones = 0
            for j in range(i, n):
                if s[j] == '0':
                    zeros += 1
                else:
                    ones += 1
                if zeros <= k or ones <= k:
                    total += 1
                else:
                    break
        return total
```

### Explanation
1. **Initialization**: We start by initializing a counter `total` to zero, which will keep track of the number of valid substrings.
2. **Outer Loop**: The outer loop iterates over each character in the string, treating it as the starting index of a substring.
3. **Inner Loop**: For each starting index, the inner loop expands the substring to the right, updating the counts of '0's (`zeros`) and '1's (`ones`) as it goes.
4. **Check Condition**: For each expanded substring, we check if either the count of '0's or '1's is at most k. If so, we increment the `total` counter. If both counts exceed k, we break the inner loop early since no further substrings starting at the same index can be valid.
5. **Result**: After processing all possible substrings, the `total` variable holds the count of all valid substrings, which is returned as the result.

This approach efficiently checks all possible substrings while leveraging early termination to optimize performance, especially for larger strings.
