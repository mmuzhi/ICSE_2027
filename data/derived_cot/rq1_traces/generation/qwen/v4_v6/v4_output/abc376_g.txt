To solve this problem, we need to find the expected number of operations required to find a treasure hidden in one of the vertices of a tree, where each vertex has a given probability of containing the treasure. The operations involve marking vertices as searched, starting from the root, and choosing the next unsearched vertex with the highest probability (based on given weights) at each step. The solution involves simulating the search process using a priority queue to determine the order in which vertices are searched, then calculating the expected operations using modular arithmetic to handle large numbers.

### Approach
1. **Tree Construction**: Read the input to construct the tree structure, where each node knows its children.
2. **Priority Queue Simulation**: Use a max-heap (simulated with a min-heap by storing negative values) to simulate the search process. The heap prioritizes vertices with higher weights (a_i) to minimize the expected number of operations.
3. **Order Tracking**: Track the order in which vertices are searched, recording the position of each vertex in this order.
4. **Expected Operations Calculation**: For each vertex, compute its contribution to the expected operations as the product of its probability (a_i / total_sum) and its search position (order + 1). Sum these contributions and use modular arithmetic to output the result modulo 998244353.

```python
import sys
import heapq

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    t = int(data[0])
    index = 1
    results = []
    for _ in range(t):
        N = int(data[index]); index += 1
        parents = list(map(int, data[index:index+N])); index += N
        a = list(map(int, data[index:index+N])); index += N
        
        T_val = sum(a)
        
        children = [[] for _ in range(N+1)]
        for i in range(N):
            parent = parents[i]
            child_vertex = i+1
            children[parent].append(child_vertex)
        
        if N == 0:
            results.append("0")
            continue
            
        heap = []
        for child in children[0]:
            heapq.heappush(heap, (-a[child-1], child))
        
        order = [-1] * (N+1)
        count = 0
        
        while heap:
            neg_a, vertex = heapq.heappop(heap)
            a_val = -neg_a
            if order[vertex] == -1:
                order[vertex] = count
                count += 1
                for child in children[vertex]:
                    heapq.heappush(heap, (-a[child-1], child))
        
        S_val = 0
        for i in range(1, N+1):
            S_val = (S_val + a[i-1] * (order[i] + 1))
        
        inv_T = pow(T_val, 998244351, 998244353)
        result = S_val * inv_T % 998244353
        results.append(str(result))
    
    print("\n".join(results))
```
