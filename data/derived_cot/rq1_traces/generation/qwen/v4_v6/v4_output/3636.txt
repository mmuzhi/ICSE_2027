=== ANSWER ===

To solve this problem, we need to determine if a given string of digits is balanced. A string is considered balanced if the sum of the digits at even indices (0-indexed) is equal to the sum of the digits at odd indices.

### Approach
1. **Problem Analysis**: The problem requires checking two separate sums from the string: one for digits at even indices and another for digits at odd indices. The solution involves iterating through the string once, calculating these sums, and then comparing them.
2. **Intuition**: By traversing the string, we can categorize each digit based on its index (even or odd) and accumulate their respective sums. The comparison of these sums will tell us if the string is balanced.
3. **Algorithm Selection**: The algorithm involves a single pass through the string. For each character in the string, we check if its index is even or odd and add its numeric value to the corresponding sum. After processing all characters, we compare the two sums.
4. **Complexity Analysis**: The algorithm runs in O(n) time, where n is the length of the string, since it processes each character exactly once. The space complexity is O(1) as we only use a few extra variables for storing the sums.

### Solution Code
```python
class Solution:
    def isBalanced(self, num: str) -> bool:
        even_sum = 0
        odd_sum = 0
        for i, char in enumerate(num):
            if i % 2 == 0:
                even_sum += int(char)
            else:
                odd_sum += int(char)
        return even_sum == odd_sum
```

### Explanation
1. **Initialization**: We initialize two variables, `even_sum` and `odd_sum`, to zero. These will hold the cumulative sums of digits at even and odd indices, respectively.
2. **Iteration**: Using a for loop with `enumerate`, we iterate over each character in the string along with its index.
3. **Sum Calculation**: For each character, if the index is even (i.e., `i % 2 == 0`), we add its integer value to `even_sum`. Otherwise, we add it to `odd_sum`.
4. **Comparison**: After processing all characters, we simply check if `even_sum` equals `odd_sum`. If they are equal, the function returns `True`, indicating the string is balanced; otherwise, it returns `False`.

This approach efficiently checks the balance condition by leveraging a single pass through the string, ensuring optimal performance and correctness.
