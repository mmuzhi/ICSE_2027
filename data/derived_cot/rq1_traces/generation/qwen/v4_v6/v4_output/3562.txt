To solve this problem, we need to select up to 4 non-overlapping intervals from a given list of intervals to maximize the total weight. The solution should return the lexicographically smallest array of indices corresponding to the chosen intervals. The intervals are considered non-overlapping if they do not share any points, including their boundaries.

### Approach
1. **Problem Analysis**: The problem requires selecting a combination of up to 4 non-overlapping intervals with the maximum total weight. The challenge is to efficiently explore all possible combinations without resorting to brute-force methods, which would be computationally infeasible for large input sizes (up to 50,000 intervals).

2. **Intuition**: The solution leverages dynamic programming (DP) combined with a segment tree to efficiently query and update the best combinations of intervals. The key insight is that for any interval, the best combination including it can be derived from the best combination of fewer intervals that end before its start.

3. **Algorithm Selection**: 
   - **Coordinate Compression**: Compress all start and end points of intervals to a smaller range to optimize segment tree operations.
   - **Segment Trees**: Use segment trees for each possible count of intervals (1 to 4) to store the best combinations (maximum weight and lexicographically smallest indices) ending at specific points.
   - **Processing Order**: Process intervals in increasing order of their end points to ensure that when processing an interval, all potential preceding intervals (those ending before its start) have already been processed and stored in the segment trees.

4. **Complexity Analysis**: 
   - **Time Complexity**: The algorithm processes each interval and performs segment tree operations (query and update) in logarithmic time relative to the number of unique coordinates. Thus, the overall complexity is O(n log n), where n is the number of intervals.
   - **Space Complexity**: The space required is O(n) for storing the segment trees, as each tree is built on the compressed coordinates.

class SegmentTree:
    def __init__(self, n):
        self.n = n
        self.size = 1
        while self.size < n:
            self.size *= 2
        self.data = [(-10**18, ()) for _ in range(2 * self.size)]
    
    def merge(self, a, b):
        if a[0] > b[0]:
            return a
        elif a[0] < b[0]:
            return b
        else:
            return a if a[1] < b[1] else b

    def update(self, index, weight, combination):
        i = index + self.size
        self.data[i] = (weight, combination)
        i //= 2
        while i:
            self.data[i] = self.merge(self.data[2*i], self.data[2*i+1])
            i //= 2

    def query(self, l, r):
        if l > r:
            return (-10**18, ())
        l += self.size
        r += self.size
        res = (-10**18, ())
        while l <= r:
            if l % 2 == 1:
                res = self.merge(res, self.data[l])
                l += 1
            if r % 2 == 0:
                res = self.merge(res, self.data[r])
                r -= 1
            l //= 2
            r //= 2
        return res

class Solution:
    def maximumWeight(self, intervals: List[List[int]]) -> List[int]:
        coords = set()
        for l, r, w in intervals:
            coords.add(l)
            coords.add(r)
        coords = sorted(coords)
        comp = {val: idx for idx, val in enumerate(coords)}
        size = len(coords)
        
        intervals.sort(key=lambda x: x[1])
        n = len(intervals)
        
        seg_trees = [None] * 5
        for k in range(1, 5):
            seg_trees[k] = SegmentTree(size)
        
        for idx, (l, r, w) in enumerate(intervals):
            for k in range(1, 5):
                if k == 1:
                    weight_sum = w
                    indices_tuple = (idx,)
                else:
                    query_result = seg_trees[k-1].query(0, comp[l]-1)
                    if query_result[0] == -10**18:
                        continue
                    weight_sum = query_result[0] + w
                    new_comb = sorted(query_result[1] + [idx])
                    indices_tuple = tuple(new_comb)
                seg_trees[k].update(comp[r], weight_sum, indices_tuple)
        
        best = None
        for k in range(1, 5):
            candidate = seg_trees[k].query(0, size-1)
            if best is None or candidate[0] > best[0] or (candidate[0] == best[0] and candidate[1] < best[1]):
                best = candidate
        
        return list(best[1]) if best is not None else []

1. **Coordinate Compression**: The start and end points of all intervals are compressed into a smaller range to facilitate efficient segment tree operations.
2. **Segment Trees**: Four segment trees are initialized (one for each possible count of intervals from 1 to 4). Each tree stores the best combination of intervals (maximum weight and lexicographically smallest indices) ending at specific points.
3. **Processing Intervals**: Intervals are processed in order of their end points. For each interval, the algorithm queries the segment trees of counts 1 to 4 to find the best combination that includes the current interval.
4. **Combination Update**: For each interval, the algorithm updates the segment trees with the new combination formed by the current interval and the best previous combination.
5. **Result Extraction**: After processing all intervals, the algorithm queries each segment tree for the best combination (maximum weight and lexicographically smallest indices) and returns the result.

This approach efficiently narrows down the search space using dynamic programming and segment trees, ensuring optimal performance even for large input sizes.
