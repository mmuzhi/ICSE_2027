=== ANSWER ===

To solve this problem, we need to determine whether it's possible to redistribute the people among three teams such that each team's total strength is equal, and if possible, find the minimum number of people who need to switch teams to achieve this. 

### Approach
1. **Problem Analysis**: The problem involves redistributing people across three teams to balance their total strengths. The key observation is that the total strength of all people must be divisible by 3 for it to be possible to partition them into three teams with equal strength. If not, the answer is immediately -1.

2. **Dynamic Programming (DP) Setup**: We use a DP approach where the state `dp[a][b]` represents the minimum number of switches required to achieve team1 with total strength `a` and team2 with total strength `b`. The third team's strength is derived as the remaining strength (total_sum - a - b).

3. **State Transitions**: For each person, we consider three options:
   - Assign to team1, updating its strength and checking if it doesn't exceed the target strength.
   - Assign to team2, similarly updating its strength.
   - Assign to team3, ensuring the combined strength of team3 does not exceed the target.

4. **Initial and Final States**: The initial state is `dp[0][0] = 0` (no switches, zero strength in both teams). The goal is to reach the state `dp[S][S]`, where `S` is one-third of the total strength.

5. **Complexity Analysis**: The DP has a state space of `O(S^2)`, where `S` is the target team strength (up to 500). For each of the `N` people, we process up to `S^2` states, leading to a total complexity of `O(N * S^2)`, which is feasible given the constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    
    n = int(data[0])
    people = []
    index = 1
    for i in range(n):
        team = int(data[index])
        strength = int(data[index+1])
        index += 2
        people.append((team, strength))
    
    total_sum = sum(strength for (_, strength) in people)
    if total_sum % 3 != 0:
        print(-1)
        return
        
    S = total_sum // 3
    
    prefix = [0] * (n+1)
    for i in range(1, n+1):
        prefix[i] = prefix[i-1] + people[i-1][1]
        
    if S == 0:
        print(0)
        return
        
    INF = 10**9
    dp = [[INF] * (S+1) for _ in range(S+1)]
    dp[0][0] = 0
    
    for i in range(n):
        s = people[i][1]
        t = people[i][0]
        new_dp = [[INF] * (S+1) for _ in range(S+1)]
        for a in range(S+1):
            for b in range(S+1):
                if dp[a][b] == INF:
                    continue
                if prefix[i] - a - b > S:
                    continue
                    
                if a + s <= S:
                    new_a = a + s
                    new_b = b
                    new_cost = dp[a][b] + (0 if t == 1 else 1)
                    if new_cost < new_dp[new_a][new_b]:
                        new_dp[new_a][new_b] = new_cost
                        
                if b + s <= S:
                    new_a = a
                    new_b = b + s
                    new_cost = dp[a][b] + (0 if t == 2 else 1)
                    if new_cost < new_dp[new_a][new_b]:
                        new_dp[new_a][new_b] = new_cost
                        
                if prefix[i] - a - b + s <= S:
                    new_a = a
                    new_b = b
                    new_cost = dp[a][b] + (0 if t == 3 else 1)
                    if new_cost < new_dp[new_a][new_b]:
                        new_dp[new_a][new_b] = new_cost
                        
        dp = new_dp
        
    print(dp[S][S] if dp[S][S] < INF else -1)

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read and parsed into a list of tuples, each representing a person's initial team and strength.
2. **Total Strength Check**: The total strength of all people is computed. If it's not divisible by 3, the answer is -1 since equal partitioning is impossible.
3. **Prefix Sum Array**: A prefix sum array is built to efficiently compute cumulative strengths up to any point.
4. **DP Initialization**: The DP array is initialized with a large value (INF) except for the initial state `dp[0][0] = 0`, representing zero switches and zero strength in both teams.
5. **DP State Transitions**: For each person, the DP state is updated by considering assignments to each of the three teams, ensuring the target strength is not exceeded and tracking the number of switches.
6. **Result Extraction**: The solution is found in `dp[S][S]`, which holds the minimum switches needed. If this state is unreachable, -1 is printed.

This approach efficiently checks all possible ways to redistribute people while minimizing the number of switches, leveraging dynamic programming to explore feasible states without excessive computational overhead.
