You will be given a competitive programming problem.
You are given a string s.
We define the mirror of a letter in the English alphabet as its corresponding letter when the alphabet is reversed. For example, the mirror of 'a' is 'z', and the mirror of 'y' is 'b'.
Initially, all characters in the string s are unmarked.
You start with a score of 0, and you perform the following process on the string s:

Iterate through the string from left to right.
At each index i, find the closest unmarked index j such that j < i and s[j] is the mirror of s[i]. Then, mark both indices i and j, and add the value i - j to the total score.
If no such index j exists for the index i, move on to the next index without making any changes.

Return the total score at the end of the process.
 
Example 1:

Input: s = "aczzx"
Output: 5
Explanation:

i = 0. There is no index j that satisfies the conditions, so we skip.
i = 1. There is no index j that satisfies the conditions, so we skip.

i = 3. There is no index j that satisfies the conditions, so we skip.
i = 4. The closest index j that satisfies the conditions is j = 1, so we mark both indices 1 and 4, and then add 4 - 1 = 3 to the score.

Output: 0

=== ANSWER ===

To solve this problem, we need to calculate the score obtained by iteratively processing a string from left to right, marking pairs of characters that are mirrors of each other and adding their distance to the score. The mirror of a character is defined as its corresponding character in the reversed alphabet (e.g., 'a' mirrors 'z', 'b' mirrors 'y', etc.). The process involves finding the closest unmarked mirror character to the left of the current character, marking both characters, and adding their distance to the score.

### Approach
1. **Mirror Mapping**: Create a dictionary that maps each lowercase letter to its mirror counterpart based on the reversed alphabet.
2. **Character Indices**: Build a dictionary where each key is a character from the string, and the value is a list of indices where that character appears, sorted in ascending order.
3. **Iterate Through String**: For each character in the string, determine its mirror character. Use binary search to find the largest index (closest to the current position) in the mirror character's list that is less than the current index and unmarked.
4. **Mark Characters**: If such an index is found, remove both the current index and the found index from their respective lists (mark them) and add their distance to the score.
5. **Efficiency**: Utilize binary search (via `bisect`) to efficiently find the closest mirror character, ensuring the solution handles the constraints effectively.

### Solution Code
```python
import bisect

class Solution:
    def calculateScore(self, s: str) -> int:
        mirror_map = {}
        for i in range(26):
            char1 = chr(ord('a') + i)
            char2 = chr(ord('z') - i)
            mirror_map[char1] = char2
        
        char_to_indices = {}
        for char in set(s):
            char_to_indices[char] = []
        
        for idx, char in enumerate(s):
            char_to_indices[char].append(idx)
        
        for char in char_to_indices:
            char_to_indices[char].sort()
        
        score = 0
        n = len(s)
        
        for i in range(n):
            c = s[i]
            m = mirror_map[c]
            
            if m in char_to_indices and char_to_indices[m]:
                lst_m = char_to_indices[m]
                pos = bisect.bisect_left(lst_m, i)
                if pos > 0:
                    j = lst_m[pos - 1]
                    new_lst_m = lst_m[:pos-1] + lst_m[pos:]
                    char_to_indices[m] = new_lst_m
                    
                    lst_c = char_to_indices[c]
                    pos_c = bisect.bisect_left(lst_c, i)
                    if pos_c < len(lst_c) and lst_c[pos_c] == i:
                        new_lst_c = lst_c[:pos_c] + lst_c[pos_c+1:]
                        char_to_indices[c] = new_lst_c
                    
                    score += (i - j)
        
        return score
```

### Explanation
1. **Mirror Mapping**: The `mirror_map` dictionary is created by iterating over each character in the alphabet and mapping it to its reversed counterpart (e.g., 'a' maps to 'z', 'b' to 'y', etc.).
2. **Character Indices**: The `char_to_indices` dictionary is populated by scanning the input string and recording the positions of each character. Each list of indices is then sorted to facilitate binary search operations.
3. **Processing Each Character**: For each character in the string, the algorithm checks if its mirror character exists in the dictionary. If so, it uses binary search (`bisect_left`) to find the largest index (closest to the current position) that is less than the current index and unmarked.
4. **Marking and Scoring**: If a valid mirror index is found, both indices are removed from their respective lists (marking them as used), and their distance is added to the score. This ensures each character is only used once.
5. **Efficiency**: The use of binary search (`bisect`) ensures that each lookup and deletion operation is performed in logarithmic time relative to the number of occurrences of each character, making the solution efficient even for large input sizes. The overall complexity is O(n log k) per character, where k is the number of occurrences of the mirror character.
