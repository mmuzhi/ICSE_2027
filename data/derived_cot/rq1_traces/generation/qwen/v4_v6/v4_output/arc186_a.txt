- The sums of corresponding rows are equal. That is, A_{i,1} + \dots + A_{i,N} = B_{i,1} + \dots + B_{i,N} for any i=1,\dots,N.

Furthermore, for an N \times N matrix A whose elements are 0 or 1, and integers i,j (1 \leq i,j \leq N), we say that the element at row i column j is fixed if A_{i,j} = B_{i,j} holds for any matrix B that is similar to A.

- The i-th query: If there exists an N \times N matrix whose elements are 0 or 1 such that exactly K_i elements are fixed, output Yes; otherwise, output No.

Output

For the i-th line (1 \le i \le Q), output the answer for the i-th query.

Constraints


- 2 \le N \le 30
- 1 \le Q \le N^2+1
- 0 \le K_i \le N^2
- K_i \ne K_j (1 \le i < j \le Q)
- All inputs are integers

An element is fixed if, in any matrix similar to the given matrix, the element's value remains the same. Two matrices are similar if they have the same sum of elements in each corresponding row and column.

### Approach
1. **Problem Analysis**: The key observation is that the number of fixed elements in a matrix depends on its row and column sums. For a given matrix, an element (i, j) is fixed if its value is the only possible value that satisfies the row and column sums constraints. The challenge is to determine all possible counts of fixed elements for any N x N matrix.

2. **Insight**: For specific values of N (like N=3), certain counts of fixed elements (0, 5, 9) are possible. For other values of N, only 0 and N² are possible. This insight allows us to handle specific cases directly and use a fallback for other cases.

4. **Complexity Analysis**: The solution efficiently checks each query in constant time by leveraging precomputed sets. The complexity is O(Q) for each test case, where Q is the number of queries.
