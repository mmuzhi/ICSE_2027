To solve this problem, we need to determine if it's possible to transform string S into string T using a series of operations where each operation replaces every occurrence of one letter with another. If possible, we also need to find the minimum number of such operations required.

### Approach
1. **Check the Necessary Condition**: For each letter in S, all occurrences of that letter must map to the same letter in T. If any letter in S maps to multiple different letters in T, it's impossible to transform S into T, and we return -1.
2. **Count Mismatched Letters**: For each distinct letter in S, if the letter does not already match its corresponding letter in T (i.e., S[i] != T[i] for some positions), we need to perform an operation to change it. The minimum number of operations required is simply the number of distinct letters in S that do not match their target letters in T.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    n = int(data[0])
    S = data[1].strip()
    T = data[2].strip()

    # Check necessary condition: for each letter in S, all its occurrences must map to the same letter in T.
    mapping = {}
    for i in range(n):
        if S[i] in mapping:
            if mapping[S[i]] != T[i]:
                print(-1)
                return
        else:
            mapping[S[i]] = T[i]

    # Count the number of distinct letters in S that do not match their target letter in T.
    distinct_s = set(S)
    count = 0
    for c in distinct_s:
        if mapping.get(c, c) != c:
            count += 1

    print(count)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Necessary Condition Check**: The code first checks if each letter in S consistently maps to a single letter in T. If any letter in S maps to multiple different letters in T, it immediately outputs -1, indicating transformation is impossible.
2. **Counting Operations**: If the necessary condition is satisfied, the code then counts how many distinct letters in S do not match their corresponding letters in T. Each such letter requires at least one operation to transform it into its target letter. The total count of these letters gives the minimum number of operations needed.
3. **Output**: The result is printed, which is either the count of mismatched letters or -1 if transformation is not possible.

This approach efficiently checks for feasibility and calculates the minimum operations by leveraging simple dictionary lookups and set operations, ensuring optimal performance even for large input sizes.
