Return the length of such a subsequence.

To solve this problem, we need to find the length of the longest subsequence in an array such that the absolute differences between consecutive elements form a non-increasing sequence. A subsequence is derived by deleting some or no elements without changing the order of the remaining elements. The key insight is to use dynamic programming to keep track of the longest subsequences ending with each possible value and last difference, ensuring the sequence of absolute differences is non-increasing.

### Approach
1. **Dynamic Programming Setup**: We use two 2D arrays, `dp` and `max_dp`, where:
   - `dp[x][d]` stores the length of the longest subsequence ending with value `x` and last absolute difference `d`.
   - `max_dp[x][d]` stores the maximum value of `dp[x][d']` for all `d' >= d`, allowing efficient lookup of the best subsequence length for a given difference constraint.

2. **Initialization**: Initialize `dp` and `max_dp` with zeros. The `max_dp` array is updated in reverse order for each value `x` to store the suffix maximum values of `dp[x][d]`.

3. **Processing Each Element**: For each element `y` in the array:
   - **New Subsequence**: Start a new subsequence with `y` by setting `dp[y][300] = 1` (using 300 as a large number to represent no previous element).
   - **Extending Subsequences**: For each possible previous value `x`, compute the new absolute difference `d_new = |y - x|`. If `d_new` is less than or equal to the last difference stored in `max_dp[x][d_new]`, extend the subsequence ending with `x` to include `y`.
   - **Update `max_dp`**: After processing all previous values, update `max_dp[y]` to reflect the best subsequence lengths for all possible differences ending with `y`.

4. **Result Extraction**: The maximum value encountered in `dp` during the processing gives the length of the longest valid subsequence.

### Solution Code
```python
class Solution:
    def longestSubsequence(self, nums: List[int]) -> int:
        size = 301
        dp = [[0] * size for _ in range(size)]
        max_dp = [[0] * size for _ in range(size)]
        ans = 0
        
        for y in nums:
            dp[y][300] = max(dp[y][300], 1)
            ans = max(ans, dp[y][300])
            
            for x in range(1, 301):
                d_new = abs(y - x)
                candidate = max_dp[x][d_new]
                if candidate + 1 > dp[y][d_new]:
                    dp[y][d_new] = candidate + 1
                    ans = max(ans, dp[y][d_new])
            
            max_dp[y][300] = dp[y][300]
            for d in range(299, -1, -1):
                max_dp[y][d] = max(dp[y][d], max_dp[y][d+1])
        
        return ans
```

### Explanation
1. **Initialization**: The `dp` and `max_dp` arrays are initialized to store the lengths of the longest subsequences and their suffix maxima, respectively.
2. **New Subsequence Handling**: For each element `y`, we start a new subsequence by setting `dp[y][300]` to 1, indicating a subsequence of length 1 ending at `y` with no prior difference (represented by 300).
3. **Extending Subsequences**: For each previous value `x`, we check if extending the subsequence ending with `x` (if the new absolute difference `d_new` is less than or equal to `x`'s last difference) results in a longer valid subsequence. The `max_dp` array helps efficiently find the best previous subsequence length.
4. **Updating `max_dp`**: After processing all previous values, `max_dp[y]` is updated to store the best subsequence lengths for all possible differences ending with `y`, facilitating quick lookups in subsequent iterations.
5. **Result Extraction**: The maximum value encountered in `dp` during the entire process is the answer, representing the longest subsequence with non-increasing absolute differences between consecutive elements. 

This approach efficiently leverages dynamic programming and suffix maxima to handle the constraints and ensure optimal performance.
