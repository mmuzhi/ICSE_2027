To solve this problem, we need to compute the sum of the maximum and minimum elements of all subarrays with at most `k` elements in an integer array `nums`. The solution involves efficiently calculating the sum of minimums and sum of maximums for all such subarrays and then combining these sums.

### Approach
1. **Problem Analysis**: The problem requires processing all subarrays of length 1 to `k` in the given array and for each subarray, summing its minimum and maximum elements. A brute-force approach would be too slow for large arrays (up to 80,000 elements), so we need an optimized method.

2. **Key Insight**: For each element in the array, determine how many subarrays it is the minimum or maximum of, considering the constraint that subarrays can be at most `k` elements long. This involves:
   - **Preprocessing**: Use monotonic stacks to compute the nearest smaller (for minimum) or greater (for maximum) elements to the left and right of each element. This helps in defining boundaries within which the element is the minimum or maximum.
   - **Sum Calculation**: For each element, calculate the number of valid subarrays where it is the minimum or maximum by considering the boundaries and the length constraint `k`. The sum of these values for all elements gives the total sum of minimums and maximums.

3. **Algorithm Selection**: 
   - **Monotonic Stacks**: Efficiently compute the nearest smaller or greater elements for each position in O(n) time.
   - **Sliding Window**: For each element, determine the valid range of subarrays where it is the minimum or maximum by considering the boundaries and the maximum allowed length `k`.

4. **Complexity Analysis**: The preprocessing step using monotonic stacks runs in O(n) time. The subsequent calculation for each element also runs in O(n) time, making the overall complexity O(n), which is optimal for the given constraints.

class Solution:
    def minMaxSubarraySum(self, nums: List[int], k: int) -> int:
        n = len(nums)
        
        def compute_left_boundaries(arr, greater=False):
            n = len(arr)
            res = [-1] * n
            stack = []
            for i in range(n):
                while stack and (arr[stack[-1]] <= arr[i] if greater else arr[stack[-1]] >= arr[i]):
                    stack.pop()
                res[i] = stack[-1] if stack else -1
                stack.append(i)
            return res
        
        def compute_right_boundaries(arr, greater=False):
            n = len(arr)
            res = [n] * n
            stack = []
            for i in range(n-1, -1, -1):
                while stack and (arr[stack[-1]] <= arr[i] if greater else arr[stack[-1]] >= arr[i]):
                    stack.pop()
                res[i] = stack[-1] if stack else n
                stack.append(i)
            return res
        
        left_min = compute_left_boundaries(nums, greater=False)
        right_min = compute_right_boundaries(nums, greater=False)
        left_max = compute_left_boundaries(nums, greater=True)
        right_max = compute_right_boundaries(nums, greater=True)
        
        def calculate_sum(boundaries_left, boundaries_right, k, greater=False):
            n = len(nums)
            total = 0
            if greater:
                for i in range(n):
                    L = boundaries_left[i] + 1
                    R = boundaries_right[i] - 1
                    s_low = max(L, i - k + 1)
                    s_high = min(i, R - k + 1)
                    Sum1 = 0
                    if s_low <= s_high:
                        terms = s_high - s_low + 1
                        first_term = s_low + k - 1 - i + 1
                        last_term = s_high + k - 1 - i + 1
                        Sum1 = terms * (first_term + last_term) // 2
                    Sum2 = 0
                    if s_high < i and R >= i:
                        terms2 = i - s_high
                        Sum2 = terms2 * (R - i + 1)
                    total += nums[i] * (Sum1 + Sum2)
            else:
                for i in range(n):
                    L = boundaries_left[i] + 1
                    R = boundaries_right[i] - 1
                    s_low = max(L, i - k + 1)
                    s_high = min(i, R - k + 1)
                    Sum1 = 0
                    if s_low <= s_high:
                        terms = s_high - s_low + 1
                        first_term = s_low + k - 1 - i + 1
                        last_term = s_high + k - 1 - i + 1
                        Sum1 = terms * (first_term + last_term) // 2
                    Sum2 = 0
                    if s_high < i and R >= i:
                        terms2 = i - s_high
                        Sum2 = terms2 * (R - i + 1)
                    total += nums[i] * (Sum1 + Sum2)
            return total
        
        S_min = calculate_sum(left_min, right_min, k, greater=False)
        S_max = calculate_sum(left_max, right_max, k, greater=True)
        return S_min + S_max

### Explanation
1. **Preprocessing**: The `compute_left_boundaries` and ` compute_right_boundaries` functions use monotonic stacks to find the nearest smaller (for minimum) or greater (for maximum) elements to the left and right of each element in O(n) time.
2. **Sum Calculation**: The `calculate_sum` function processes each element to determine how many subarrays of length up to `k` it is the minimum or maximum of. It uses the boundaries computed earlier to define valid subarrays and sums their contributions.
3. **Combining Results**: The total result is the sum of the minimums and maximums of all valid subarrays, computed by calling `calculate_sum` for both minimum and maximum cases.

This approach efficiently handles the problem constraints by leveraging preprocessing and sliding window techniques to avoid the O(n^2) complexity of a brute-force solution.
