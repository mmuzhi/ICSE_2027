Output: false
Explanation:
There is no substring matching the pattern.

Explanation:
The substrings "u", "uc", and "uck" match the pattern.

To solve this problem, we need to determine if a given pattern string `p` (which contains exactly one '*') can be transformed into a substring of the string `s` by replacing the '*' with any sequence of zero or more characters. The substring must be contiguous and non-empty.

### Approach
1. **Problem Analysis**: The pattern `p` consists of two parts separated by a '*': a left part (`left_pat`) and a right part (`right_pat`). The '*' can be replaced by any sequence of characters (including an empty sequence). The goal is to check if there exists a contiguous substring in `s` that starts with `left_pat` and ends with `right_pat`, with any characters (possibly empty) in between.

2. **Key Insight**: The solution involves iterating over all possible starting positions in `s` where `left_pat` can be matched. For each such starting position, we then look for a position in `s` where `right_pat` appears at the end of a substring starting from the current position. The substring between `left_pat` and `right_pat` can be any sequence of characters (including empty).

3. **Algorithm Selection**: 
   - **Splitting the Pattern**: The pattern `p` is split into `left_pat` and `right_pat` using the '*'.
   - **Handling Edge Cases**: If both `left_pat` and `right_pat` are empty (i.e., the pattern is "*"), any non-empty substring in `s` will suffice.
   - **Iterating Over Possible Substrings**: For each starting position in `s` where `left_pat` matches, we calculate the minimal length of the substring (if `left_pat` and `right_pat` are non-empty, the minimal length is the sum of their lengths; otherwise, it's 1 to ensure the substring is non-empty). We then check if `right_pat` appears at the end of any substring starting from the current position with the calculated minimal length.

4. **Complexity Analysis**: The algorithm involves nested loops. The outer loop runs in O(n) time, and the inner loop runs in O(n) time in the worst case. Given that the length of `s` is at most 50, the total complexity is O(n^2), which is efficient for the problem constraints.

### Solution Code
```python
class Solution:
    def hasMatch(self, s: str, p: str) -> bool:
        parts = p.split('*')
        left_pat = parts[0]
        right_pat = parts[1]
        n = len(s)
        left_len = len(left_pat)
        right_len = len(right_pat)
        min_length = left_len + right_len
        if min_length == 0:
            min_length = 1
        
        for i in range(0, n - left_len + 1):
            if s[i:i+left_len] != left_pat:
                continue
            j_min = i + min_length - 1
            if j_min >= n:
                continue
            if right_len == 0:
                return True
            else:
                for j in range(j_min, n):
                    if s[j-right_len+1:j+1] == right_pat:
                        return True
        return False
```

### Explanation
1. **Splitting the Pattern**: The pattern `p` is split into `left_pat` and `right_pat` using the '*' character. This separates the pattern into the parts before and after the wildcard.
2. **Initial Setup**: The lengths of `left_pat` and `right_pat` are calculated. The minimal length of the substring is determined as the sum of the lengths of `left_pat` and `right_pat`. If both are empty (pattern is "*"), the minimal length is set to 1 to ensure the substring is non-empty.
3. **Iterating Over `s`**: For each starting position `i` in `s` where `left_pat` matches, the algorithm checks if the substring starting at `i` can be extended to include `right_pat` at the end. The minimal length of the substring is used to determine the earliest position `j_min` where `right_pat` could start.
4. **Checking for `right_pat`**: If `right_pat` is empty, any substring starting at `i` with length at least `min_length` is valid. Otherwise, the algorithm checks all positions from `j_min` to the end of `s` to see if `right_pat` appears at the end of the substring.
5. **Result**: If a valid substring is found, the function returns `True`; otherwise, after all possibilities are exhausted, it returns `False`.
