To solve this problem, we need to determine whether there exists a cycle in a directed graph that includes vertex 1, and if such a cycle exists, find the minimum number of edges in it. The solution involves checking for cycles of length 2 first, as these are the smallest possible cycles in a directed graph without self-loops. If no such cycle exists, we then look for longer cycles by leveraging a multi-source BFS approach to find the shortest path from successors of vertex 1 to predecessors of vertex 1, excluding vertex 1 itself.

### Approach
1. **Check for Cycle of Length 2**: A cycle of length 2 exists if there is a vertex \( u \) such that both edges \( 1 \rightarrow u \) and \( u \rightarrow 1 \) exist in the graph. If such a vertex is found, the answer is immediately 2.
2. **Check for Successors and Predecessors**: If no cycle of length 2 exists, verify that there are edges both outgoing from vertex 1 (successors) and incoming to vertex 1 (predecessors). If either set is empty, output -1 as no cycle can include vertex 1.
3. **Build Modified Graph**: Construct a subgraph excluding vertex 1 and all edges incident to it. This helps in finding paths that do not revisit vertex 1.
4. **Multi-source BFS**: Perform a BFS starting from all successors of vertex 1. The goal is to find the shortest path from any successor to any predecessor of vertex 1 in the modified graph. The length of the cycle is then 2 plus the path length found (accounting for the edges \( 1 \rightarrow u \) and \( w \rightarrow 1 \)).

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        print(-1)
        return
        
    it = iter(data)
    n = int(next(it)); m = int(next(it))
    
    edges = []
    for _ in range(m):
        a = int(next(it)); b = int(next(it))
        edges.append((a, b))
        
    S = set()
    T = set()
    
    for a, b in edges:
        if a == 1:
            S.add(b)
        if b == 1:
            T.add(a)
            
    if S & T:
        print(2)
        return
        
    if not S or not T:
        print(-1)
        return
        
    G_prime = {}
    for a, b in edges:
        if a != 1 and b != 1:
            if a not in G_prime:
                G_prime[a] = []
            G_prime[a].append(b)
            
    dist = [-1] * (n + 1)
    q = deque()
    
    for node in S:
        dist[node] = 0
        q.append(node)
        
    while q:
        current = q.popleft()
        if current in G_prime:
            for neighbor in G_prime[current]:
                if dist[neighbor] == -1:
                    dist[neighbor] = dist[current] + 1
                    q.append(neighbor)
                    
    ans = float('inf')
    for node in T:
        if dist[node] != -1:
            ans = min(ans, 2 + dist[node])
            
    print(ans if ans != float('inf') else -1)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are the number of vertices \( N \) and edges \( M \).
2. **Building Sets S and T**: Sets \( S \) (successors of vertex 1) and \( T \) (predecessors of vertex 1) are constructed by examining each edge.
3. **Cycle of Length 2 Check**: If any vertex is present in both \( S \) and \( T \), a cycle of length 2 exists, and the answer is 2.
4. **Empty Sets Check**: If either \( S \) or \( T \) is empty, no cycle can include vertex 1, so the answer is -1.
5. **Modified Graph Construction**: The graph \( G' \) is built by excluding vertex 1 and all edges connected to it. This helps in finding paths that do not revisit vertex 1.
6. **Multi-source BFS**: Starting from all vertices in \( S \), BFS is used to explore the graph \( G' \). The shortest path from any successor to any predecessor is recorded.
7. **Result Calculation**: The minimum cycle length is calculated as 2 plus the shortest path length found during BFS. If no such path exists, the answer remains -1.

This approach efficiently checks for the smallest cycles containing vertex 1 by leveraging graph traversal and set operations, ensuring optimal performance even for large graphs.
