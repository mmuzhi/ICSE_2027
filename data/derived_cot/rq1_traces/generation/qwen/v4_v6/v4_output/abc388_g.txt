You will be given a competitive programming problem.
There are N mochi (rice cakes), arranged in ascending order of size.
The size of the i-th mochi (1\leq i\leq N) is A_i.
Given two mochi A and B, with sizes a and b respectively, you can make one kagamimochi (a stacked rice cake) by placing mochi A on top of mochi B if and only if a is at most half of b.
You are given Q integer pairs. Let (L_i, R_i) be the i-th pair (1\leq i\leq Q), and solve the following problem for each i:

Using only the R_i - L_i + 1 mochi from the L_i-th to the R_i-th, how many kagamimochi can you make simultaneously?  
More precisely, find the maximum non-negative integer K such that:

- Out of the R_i - L_i + 1 mochi from the L_i-th to the R_i-th, choose 2K mochi and form K pairs. For each pair, place one mochi on top of the other, to make K kagamimochi.

The input is given from Standard Input in the following format:
N

Q

L_2 R_2
\vdots
L_Q R_Q

Output

Print Q lines. The i-th line (1\leq i\leq Q) should contain the answer to the i-th query.

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq A_i \leq 10^9 \ (1 \leq i \leq N)
- A_i \leq A_{i+1} \ (1 \leq i < N)
- 1 \leq Q \leq 2 \times 10^5
- 1 \leq L_i < R_i \leq N \ (1 \leq i \leq Q)
- All input values are integers.

To solve this problem, we need to determine the maximum number of kagamimochi that can be formed from a given subarray of mochi. Each kagamimochi requires two mochi, where one mochi (the top) must be at most half the size of the other (the base). The solution involves efficiently processing each query to find the optimal number of pairs in the specified subarray.

### Approach
1. **Problem Analysis**: The problem requires forming the maximum number of pairs (top, base) from a subarray where each top is at most half the size of its base. The challenge is to do this efficiently for multiple queries on a large dataset.
2. **Intuition**: Given that the array is sorted in ascending order, we can use a two-pointer technique to find valid pairs. The idea is to use the smallest available base that can pair with the smallest available top, ensuring each element is used at most once.
3. **Algorithm Selection**: 
   - **Sorting**: The input array is already sorted, so each query's subarray is a contiguous sorted segment.
   - **Two-pointer Technique**: For each query, we sort the subarray (though it's already sorted) and use two pointers to traverse the array. One pointer starts at the beginning (smallest top) and the other at the end (largest base). We move the base pointer forward if the current base is too small for the current top, and both pointers move forward if a valid pair is found.
4. **Complexity Analysis**: The solution involves sorting (O(m log m) for each query, where m is the subarray size. Given the constraints (N and Q up to 2×10^5), the overall complexity is O(Q * m log m), which is acceptable for the given limits.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    q = int(data[1+n])
    queries = []
    index = 1+n+1
    for i in range(q):
        L = int(data[index]); R = int(data[index+1]); index += 2
        queries.append((L, R))
    
    out_lines = []
    for L, R in queries:
        segment = A[L-1:R]
        segment.sort()
        m = len(segment)
        i = 0
        j = 0
        count = 0
        while i < m and j < m and i < j:
            if segment[j] >= 2 * segment[i]:
                count += 1
                i += 1
                j += 1
            else:
                j += 1
        out_lines.append(str(count))
    
    sys.stdout.write("\n".join(out_lines))

if __name__ == '__main__':
    main()
```

The condition `segment[j] >= 2 * segment[i]` checks if the current base can pair with the current top. If true, both pointers are incremented; otherwise, only the base pointer is incremented.
4. **Output**: The count of valid pairs for each query is collected and printed at the end.

This approach efficiently processes each query by leveraging the sorted property of the array and the two-pointer technique to maximize the number of valid pairs while ensuring optimal performance.
