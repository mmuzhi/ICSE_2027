To solve this problem, we need to count the number of strings of length M (composed of lowercase English letters) that have a longest common subsequence (LCS) of exactly k with a given string S of length N, for each k from 0 to N. The solution involves dynamic programming to efficiently compute these counts without resorting to brute-force methods, which would be computationally infeasible for large M.

### Approach
1. **Problem Analysis**: The problem requires counting strings T of length M such that their LCS with S is exactly k for each k from 0 to N. Given the constraints (N ≤ 10, M ≤ 100), a brute-force approach is impractical. Instead, we use dynamic programming to leverage the properties of LCS.
2. **Dynamic Programming Setup**: We define a 3D DP array `dp[i][j][k]` where:
   - `i` ranges from 0 to N (length of the prefix of S).
   - `j` ranges from 0 to M (length of the string T being constructed).
   - `k` ranges from 0 to min(i, j) (possible LCS lengths).
   The value `dp[i][j][k]` represents the number of strings T of length j that have an LCS of exactly k with the prefix S[0:i] of S.
3. **Recurrence Relation**: The recurrence involves two cases for each character added to T:
   - **Case 1**: The new character does not extend the LCS. The count is derived from strings of length j-1 with the same LCS length k.
   - **Case 2**: The new character extends the LCS by matching the next character in S. The count is derived from strings of length j-1 with LCS length k-1.
4. **Base Case**: For strings of length 0 (empty string), the LCS is 0, so `dp[i][0][0] = 1` for all i.
5. **Modulo Operation**: Since the counts can be very large, all operations are performed modulo 998244353.
