There is an infinite 2D plane.
You are given a positive integer k. You are also given a 2D array queries, which contains the following queries:

queries[i] = [x, y]: Build an obstacle at coordinate (x, y) in the plane. It is guaranteed that there is no obstacle at this coordinate when this query is made.

After each query, you need to find the distance of the k^th nearest obstacle from the origin.
Return an integer array results where results[i] denotes the k^th nearest obstacle after query i, or results[i] == -1 if there are less than k obstacles.
Note that initially there are no obstacles anywhere.
The distance of an obstacle at coordinate (x, y) from the origin is given by |x| + |y|.
 
Example 1:

Input: queries = [[1,2],[3,4],[2,3],[-3,0]], k = 2
Output: [-1,7,5,3]
Explanation:

Initially, there are 0 obstacles.

After queries[1], there are obstacles at distances 3 and 7.
After queries[2], there are obstacles at distances 3, 5, and 7.
After queries[3], there are obstacles at distances 3, 3, 5, and 7.

Output: [10,8,6]
Explanation:

After queries[0], there is an obstacle at distance 10.
After queries[1], there are obstacles at distances 8 and 10.
After queries[2], there are obstacles at distances 6, 8, and 10.


 
Constraints:

1 <= queries.length <= 2 * 10^5
All queries[i] are unique.

class Solution:
    def resultsArray(self, queries: List[List[int]], k: int) -> List[int]:
        left = []  # max-heap (store negative)
        right = [] # min-heap (store positive)
        count = 0
        results = []
        for x, y in queries:
            d = abs(x) + abs(y)
            count += 1
            if count <= k:
                heapq.heappush(left, -d)
            else:
                # Compare d with the current k-th smallest (the top of left)
                if d < -left[0]:
                    heapq.heappush(left, -d)
                    popped = heapq.heappop(left)
                    heapq.heappush(right, -popped)
                else:
                    heapq.heappush(right, d)
            # After each query, if we have at least k obstacles, output the k-th smallest (the top of left)
            if count < k:
                results.append(-1)
            else:
                results.append(-left[0])
        return results

class Solution:
    def resultsArray(self, queries: List[List[int]], k: int) -> List[int]:
        left = []  # max-heap (store negative)
        right = []  # min-heap (store positive)
        count = 0
        res = []
        for x, y in queries:
            d = abs(x) + abs(y)
            count += 1
            if count <= k:
                heapq.heappush(left, -d)
            else:
                # Compare d with the current k-th smallest (the top of left)
                if d < -left[0]:
                    heapq.heappush(left, -d)
                    # Remove the largest from left (the k-th smallest) and put it in right
                    popped = heapq.heappop(left)
                    heapq.heappush(right, -popped)
                else:
                    heapq.heappush(right, d)
            # After each query, if count < k, then no k-th smallest, else the k-th smallest is -left[0]
            if count < k:
                res.append(-1)
            else:
                res.append(-left[0])
        return res

To solve this problem, we need to efficiently determine the k-th nearest obstacle to the origin after each query in a dynamic 2D plane. The distance of an obstacle at coordinate (x, y) from the origin is defined as |x| + |y|. The challenge is to handle up to 200,000 queries efficiently, ensuring that each query's result is computed in logarithmic time relative to the number of obstacles.

### Approach
1. **Problem Analysis**: The problem requires maintaining a dynamic set of obstacles and, after each insertion, finding the k-th smallest Manhattan distance from the origin. The key insight is that we only need the k smallest distances at any point to determine the k-th nearest obstacle.
2. **Data Structures**: We use two heaps:
   - A **max-heap** (simulated using negative values) to keep track of the k smallest distances. The top of this heap gives the k-th smallest distance.
   - A **min-heap** to store distances larger than the k-th smallest distance, allowing efficient insertion and retrieval.
3. **Algorithm**:
   - For each query, compute the Manhattan distance of the new obstacle.
   - If the number of obstacles is less than or equal to k, add the distance to the max-heap.
   - If the number of obstacles exceeds k, compare the new distance with the k-th smallest distance (top of the max-heap). If the new distance is smaller, it replaces the k-th smallest distance in the max-heap, which is then moved to the min-heap. Otherwise, the new distance is added to the min-heap.
   - After each query, if there are at least k obstacles, the k-th smallest distance is the top of the max-heap; otherwise, the result is -1.

### Solution Code
```python
import heapq
from typing import List

class Solution:
    def resultsArray(self, queries: List[List[int]], k: int) -> List[int]:
        left = []
        right = []
        count = 0
        results = []
        
        for x, y in queries:

            count += 1

                if d < -left[0]:

            else:

        return results
```

### Explanation
1. **Initialization**: Two heaps, `left` (max-heap for the k smallest distances) and `right` (min-heap for larger distances), are initialized.
2. **Processing Queries**:
   - For each query, the Manhattan distance `d` of the new obstacle is calculated.
   - If the number of obstacles (`count`) is less than or equal to k, the distance is added to `left` (as a negative value to simulate a max-heap).
   - If `count` exceeds k, the new distance is compared with the k-th smallest distance (top of `left`). If it's smaller, it replaces the top of `left`, which is then moved to `right`. Otherwise, the new distance is added to `right`.
3. **Result Calculation**: After processing each query, if `count` is less than k, the result is -1. Otherwise, the result is the top of `left` (converted back to positive value), representing the k-th smallest distance.

This approach efficiently maintains the k smallest distances using heap operations, ensuring optimal performance even for large input sizes.
