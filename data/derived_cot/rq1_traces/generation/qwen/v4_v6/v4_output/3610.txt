You will be given a competitive programming problem.
You are given an array nums of n integers and two integers k and x.
The x-sum of an array is calculated by the following procedure:

Count the occurrences of all elements in the array.
Keep only the occurrences of the top x most frequent elements. If two elements have the same number of occurrences, the element with the bigger value is considered more frequent.
Calculate the sum of the resulting array.

Note that if an array has less than x distinct elements, its x-sum is the sum of the array.
Return an integer array answer of length n - k + 1 where answer[i] is the x-sum of the subarray nums[i..i + k - 1].
 
Example 1:

Input: nums = [1,1,2,2,3,4,2,3], k = 6, x = 2
Output: [6,10,12]
Explanation:

For subarray [1, 1, 2, 2, 3, 4], only elements 1 and 2 will be kept in the resulting array. Hence, answer[0] = 1 + 1 + 2 + 2.
For subarray [1, 2, 2, 3, 4, 2], only elements 2 and 4 will be kept in the resulting array. Hence, answer[1] = 2 + 2 + 2 + 4. Note that 4 is kept in the array since it is bigger than 3 and 1 which occur the same number of times.
For subarray [2, 2, 3, 4, 2, 3], only elements 2 and 3 are kept in the resulting array. Hence, answer[2] = 2 + 2 + 2 + 3 + 3.

Output: [11,15,15,15,12]
Explanation:
Since k == x, answer[i] is equal to the sum of the subarray nums[i..i + k - 1].

=== ANSWER ===

To solve this problem, we need to compute the x-sum for every contiguous subarray of length k in the given array. The x-sum is defined as the sum of the top x most frequent elements in the subarray, with ties broken by the element's value (bigger value is more frequent). If there are fewer than x distinct elements, the x-sum is simply the sum of all elements in the subarray.

### Approach
1. **Sliding Window Technique**: We iterate over each possible window of size k in the array. For each window, we count the frequency of each element.
2. **Frequency Calculation**: For each window, we use a dictionary to count occurrences of each element.
3. **Sorting by Frequency and Value**: We sort the elements based on their frequencies in descending order. If two elements have the same frequency, the element with the higher value is placed first.
4. **Summing Top Elements**: We select the top x elements from the sorted list (or all elements if there are fewer than x distinct elements) and compute their sum by multiplying each element's frequency by its value.

### Solution Code
```python
class Solution:
    def findXSum(self, nums: List[int], k: int, x: int) -> List[int]:
        n = len(nums)
        res = []
        for i in range(n - k + 1):
            freq = {}
            for j in range(i, i + k):
                freq[nums[j]] = freq.get(nums[j], 0) + 1
            freq_list = [(count, num) for num, count in freq.items()]
            freq_list.sort(key=lambda x: (-x[0], -x[1]))
            top = freq_list[:x]
            s = 0
            for count, num in top:
                s += count * num
            res.append(s)
        return res
```

### Explanation
1. **Sliding Window**: The outer loop iterates over each starting index of the window (from 0 to n - k). For each starting index, the inner loop processes the window from that index to the index plus k-1.
2. **Frequency Dictionary**: For each element in the current window, we update its count in a dictionary.
3. **Sorting**: The elements are sorted by their frequencies in descending order. If frequencies are equal, elements are sorted by their values in descending order.
4. **Top Elements Selection**: The top x elements (or all elements if there are fewer than x distinct elements) are selected from the sorted list. The sum of these elements, considering their frequencies, is computed and added to the result list.

This approach efficiently computes the x-sum for each subarray by leveraging a sliding window and sorting, ensuring correctness and clarity within the problem constraints.
