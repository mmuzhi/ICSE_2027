To solve this problem, we need to find the maximum number of kagamimochi that can be formed from a given list of mochi sizes. A kagamimochi is formed by stacking one mochi on top of another, where the top mochi's size is at most half of the bottom mochi's size. The goal is to maximize the number of such pairs without reusing any mochi.

### Approach
1. **Sorting the Array**: The first step is to sort the array of mochi sizes in ascending order. This allows us to efficiently find pairs where the top mochi's size is at most half of the bottom mochi's size using a two-pointer technique.
2. **Two-Pointer Technique**: We use two pointers, `i` and `j`, where `i` starts at the beginning of the array (smallest mochi) and `j` starts at the end of the array (largest mochi). The idea is to pair the smallest available top mochi with the smallest available bottom mochi that satisfies the condition (top size ≤ half the bottom size).
3. **Greedy Matching**: For each top mochi (at index `i`), we find the smallest bottom mochi (at index `j`) that is at least twice the top mochi's size. If such a bottom mochi is found, we increment both pointers and count this pair. If not, we move the bottom pointer to the next larger mochi to find a suitable match.
4. **Termination Condition**: The process continues until we either exhaust the top or bottom pointers, ensuring all possible pairs are considered.

This approach efficiently maximizes the number of pairs by always using the smallest available top mochi with the smallest available bottom mochi that meets the size condition, thus optimizing the pairing process.

### Solution Code
```python
n = int(input().strip())
A = list(map(int, input().split()))
A.sort()
i = 0
j = 0
count = 0
while i < n and j < n:
    if A[i] * 2 <= A[j]:
        count += 1
        i += 1
        j += 1
    else:
        j += 1
print(count)
```
