You want to place your piece on an empty square in such a way that it cannot be captured by any of the existing pieces.

- Placed in row i
- Placed in column j
- Placed on any square (a,b)\ (1\leq a\leq N,1\leq b\leq N) where i+j=a+b
- Placed on any square (a,b)\ (1\leq a\leq N,1\leq b\leq N) where i-j=a-b

N M

a_2 b_2
\vdots
a_M b_M

Output

Print the number of empty squares where you can place your piece without it being captured by any existing pieces.

Constraints


- 1\leq N\leq10^9
- 1\leq M\leq10^3
- 1\leq a_k\leq N,1\leq b_k\leq N\ (1\leq k\leq M)
- (a_k,b_k)\neq(a_l,b_l)\ (1\leq k\lt l\leq M)
- All input values are integers.

To solve this problem, we need to determine the number of empty squares in an N x N grid where a new piece can be placed without being captured by any existing pieces. The existing pieces are given, and each piece can capture any square in its row, column, or either of the two diagonals (sum or difference of row and column indices) it lies on.

### Approach
1. **Problem Analysis**: The problem requires counting the number of empty squares not captured by any existing pieces. A square is captured if it shares the same row, column, or diagonal (sum or difference of indices) with any existing piece. Given the grid size can be as large as 10^9, a brute-force approach is infeasible. Instead, we use inclusion-exclusion principle on sets defined by rows, columns, diagonals, and anti-diagonals of the existing pieces.

2. **Key Insights**: 
   - **Rows and Columns**: The union of all rows and columns containing existing pieces will cover many captured squares.
   - **Diagonals and Anti-diagonals**: Squares where the sum (i+j) or difference (i-j) of their row and column indices matches that of any existing piece are also captured.
   - **Inclusion-Exclusion Principle**: To avoid overcounting squares captured by multiple conditions, we use inclusion-exclusion over the sets of rows, columns, diagonals, and anti-diagonals.

3. **Algorithm Selection**:
   - **Set Operations**: We first identify all distinct rows, columns, diagonals (sums), and anti-diagonals (differences) from the given pieces.
   - **Counting Captured Squares**: Using inclusion-exclusion, we compute the total number of squares captured by at least one of these sets. This involves:
     - Calculating individual set sizes.
     - Subtracting pairwise intersections.
     - Adding back triple intersections.
     - Subtracting the quadruple intersection.
   - **Final Count**: The result is derived by subtracting the total captured squares from the grid's total squares.

4. **Complexity Analysis**: The algorithm efficiently handles large grids (up to 10^9) by leveraging set operations and binary search for range queries, reducing the complexity from O(N^2) to O(M^2 log M), where M is the number of pieces (up to 1000).

### Solution Code
```python
import bisect

def main():
    import sys
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    m = int(data[1])
    pieces = []
    index = 2
    for i in range(m):
        a = int(data[index])
        b = int(data[index+1])
        index += 2
        pieces.append((a, b))
    
    R = set()
    C = set()
    D_set = set()
    A_set = set()
    for a, b in pieces:
        R.add(a)
        C.add(b)
        D_set.add(a + b)
        A_set.add(a - b)
    
    def diag_count(s, N):
        low_i = max(1, s - N)
        high_i = min(N, s - 1)
        if low_i > high_i:
            return 0
        return high_i - low_i + 1

    def anti_diag_count(d, N):
        if abs(d) > N - 1:
            return 0
        return N - abs(d)
    
    total = n * n
    
    F1 = len(R) * n
    F2 = len(C) * n
    F3 = sum(diag_count(s, n) for s in D_set)
    F4 = sum(anti_diag_count(d, n) for d in A_set)
    
    sorted_D = sorted(D_set)
    sorted_A = sorted(A_set)
    
    def count_in_range(arr, low_val, high_val):
        if not arr:
            return 0
        left = bisect.bisect_left(arr, low_val)
        right = bisect.bisect_right(arr, high_val)
        return max(0, right - left)
    
    F12 = len(R) * len(C)
    F13 = 0
    for i in R:
        low_val = i + 1
        high_val = i + n
        cnt = count_in_range(sorted_D, low_val, high_val)
        F13 += cnt
        
    F14 = 0
    for i in R:
        low_val = i - n
        high_val = i - 1
        cnt = count_in_range(sorted_A, low_val, high_val)
        F14 += cnt
        
    F23 = 0
    for j in C:
        low_val = j + 1
        high_val = j + n
        cnt = count_in_range(sorted_D, low_val, high_val)
        F23 += cnt
        
    F24 = 0
    for j in C:
        low_val = 1 - j
        high_val = n - j
        cnt = count_in_range(sorted_A, low_val, high_val)
        F24 += cnt
        
    F34 = 0
    for s1 in D_set:
        for s2 in A_set:
            if (s1 + s2) % 2 != 0:
                continue
            i_val = (s1 + s2) // 2
            j_val = (s1 - s2) // 2
            if 1 <= i_val <= n and 1 <= j_val <= n:
                F34 += 1
                
    F123 = 0
    for i in R:
        for j in C:
            if i + j in D_set:
                F123 += 1
                
    F124 = 0
    for i in R:
        for j in C:
            if i - j in A_set:
                F124 += 1
                
    F134 = 0
    for i in R:
        for s1 in D_set:
            j_val = s1 - i
            if 1 <= j_val <= n:
                d_val = 2 * i - s1
                if d_val in A_set:
                    F134 += 1
                    
    F234 = 0
    for j in C:
        for s1 in D_set:
            i_val = s1 - j
            if 1 <= i_val <= n:
                d_val = i_val - j
                if d_val in A_set:
                    F234 += 1
                    
    F1234 = 0
    for i in R:
        for j in C:
            if i + j in D_set and i - j in A_set:
                F1234 += 1
                
    union_size = F1 + F2 + F3 + F4 - F12 - F13 - F14 - F23 - F24 - F34 + F123 + F124 + F134 + F234 - F1234
    answer = total - union_size
    print(answer)

if __name__ == '__main__':
    main()
```
