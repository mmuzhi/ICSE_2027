=== PROMPT ===
You will be given a competitive programming problem.
You are given two integers, n and threshold, as well as a directed weighted graph of n nodes numbered from 0 to n - 1. The graph is represented by a 2D integer array edges, where edges[i] = [A_i, B_i, W_i] indicates that there is an edge going from node A_i to node B_i with weight W_i.
You have to remove some edges from this graph (possibly none), so that it satisfies the following conditions:

Node 0 must be reachable from all other nodes.
The maximum edge weight in the resulting graph is minimized.
Each node has at most threshold outgoing edges.

Return the minimum possible value of the maximum edge weight after removing the necessary edges. If it is impossible for all conditions to be satisfied, return -1.
 
Example 1:

Input: n = 5, edges = [[1,0,1],[2,0,2],[3,0,1],[4,3,1],[2,1,1]], threshold = 2
Output: 1
Explanation:

Remove the edge 2 -> 0. The maximum weight among the remaining edges is 1.

Example 2:

Input: n = 5, edges = [[0,1,1],[0,2,2],[0,3,1],[0,4,1],[1,2,1],[1,4,1]], threshold = 1
Output: -1
Explanation: 
It is impossible to reach node 0 from node 2.

Example 3:

Input: n = 5, edges = [[1,2,1],[1,3,3],[1,4,5],[2,3,2],[3,4,2],[4,0,1]], threshold = 1
Output: 2
Explanation: 

Remove the edges 1 -> 3 and 1 -> 4. The maximum weight among the remaining edges is 2.

Example 4:

Input: n = 5, edges = [[1,2,1],[1,3,3],[1,4,5],[2,3,2],[4,0,1]], threshold = 1
Output: -1

 
Constraints:

2 <= n <= 10^5
1 <= threshold <= n - 1
1 <= edges.length <= min(10^5, n * (n - 1) / 2).
edges[i].length == 3
0 <= A_i, B_i < n
A_i != B_i
1 <= W_i <= 10^6
There may be multiple edges between a pair of nodes, but they must have unique weights.

class Dinic:
    def __init__(self, graph, source, sink):
        self.graph = graph
        self.source = source
        self.sink = sink
        self.n = len(graph)
        self.level = [-1] * self.n
        self.ptr = [0] * self.n
    
    def bfs(self):
        # Assign levels
        self.level = [-1] * self.n
        q = deque()
        q.append(self.source)
        self.level[self.source] = 0
        while q:
            u = q.popleft()
            for i, (v, cap, rev) in enumerate(self.graph[u]):
                if cap > 0 and self.level[v] == -1:
                    self.level[v] = self.level[u] + 1
                    q.append(v)
        return self.level[self.sink] != -1
    
    def dfs(self, u, flow):
        if u == self.sink:
            return flow
        for i in range(self.ptr[u], len(self.graph[u])):
            v, cap, rev = self.graph[u][i]
            if cap > 0 and self.level[v] == self.level[u] + 1:
                pushed = self.dfs(v, min(flow, cap))
                if pushed > 0:
                    self.graph[u][i][1] -= pushed
                    self.graph[v][rev][1] += pushed
                    return pushed
            self.ptr[u] += 1
        return 0
    
    def max_flow(self):
        flow = 0
        while self.bfs() and self.level[self.sink] != -1:
            self.ptr = [0] * self.n
            while True:
                pushed = self.dfs(self.source, 10**18)
                if pushed == 0:
                    break
                flow += pushed
        return flow

class Solution:
    def minMaxWeight(self, n: int, edges: List[List[int]], threshold: int) -> int:
        if not edges:
            return -1
        
        edges.sort(key=lambda x: x[2])
        max_w = edges[-1][2]
        
        lo, hi = 0, max_w
        ans = -1
        
        def is_feasible(X_val):
            size = 2 * n + 2
            source = 2 * n
            sink = 2 * n + 1
            graph = [[] for _ in range(size)]
            
            # Node-splitting: for each node i, add edge from i to i+n with capacity = threshold
            for i in range(n):
                self.add_edge(graph, i, i + n, threshold)
            
            # Add edge from source to 0's in-node
            self.add_edge(graph, source, 0, 10**18)
            
            # Add edges for the reverse graph: for each original edge (u, v, w) with w<=X_val
            for u, v, w in edges:
                if w > X_val:
                    continue
                # The reverse graph's edge is (v, u)
                # In the flow network, add an edge from v_out to u_in
                self.add_edge(graph, v + n, u, 1)
            
            # Add edges from each node's out-node to sink
            for i in range(n):
                self.add_edge(graph, i + n, sink, 1)
            
            dinic = Dinic(graph, source, sink)
            flow = dinic.max_flow()
            return flow >= n - 1
        
        while lo <= hi:
            mid = (lo + hi) // 2
            if is_feasible(mid):
                ans = mid
                hi = mid - 1
            else:
                lo = mid + 1
        
        return ans if ans != -1 else -1

=== ANSWER ===

To solve this problem, we need to find the minimum possible value of the maximum edge weight in a directed weighted graph after removing some edges such that:
1. Node 0 must be reachable from all other nodes.
2. The maximum edge weight in the resulting graph is minimized.
3. Each node has at most `threshold` outgoing edges.

### Approach
1. **Binary Search on Edge Weight**: We perform a binary search on the maximum edge weight (X) to determine the smallest X such that the graph conditions are satisfied. The binary search ranges from 0 to the maximum edge weight in the graph.
2. **Flow Network Construction**: For each candidate X during the binary search, we construct a flow network where:
   - Each node is split into two nodes (in-node and out-node) connected by an edge with capacity equal to the threshold.
   - Edges from the reverse graph (original edges reversed) are added from the out-node of one node to the in-node of another with capacity 1.
   - A source node connects to the in-node of node 0 with infinite capacity, and each out-node connects to a sink node with capacity 1.
3. **Max-Flow Calculation**: Using Dinic's algorithm, we compute the maximum flow from the source to the sink. If the flow equals the number of nodes minus 1 (indicating all nodes are reachable from node 0 in the reverse graph), then X is feasible.
4. **Binary Search Adjustment**: Based on the feasibility of X, we adjust the binary search range to find the smallest feasible X or determine if no solution exists.

class Solution:
    def minMaxWeight(self, n: int, edges: List[List[int]], threshold: int) -> int:
        if not edges:
            return -1
        
        edges.sort(key=lambda x: x[2])
        max_w = edges[-1][2]
        
        lo, hi = 0, max_w
        ans = -1
        
        def is_feasible(X_val):
            size = 2 * n + 2
            source = 2 * n
            sink = 2 * n + 1
            graph = [[] for _ in range(size)]
            
            for i in range(n):

            self.add_edge(graph, source, 0, 10**18)
            
            for u, v, w in edges:
                if w > X_val:

                self.add_edge(graph, i + n, sink, 1)

                if cap > 0 and self.level[v] == -1:
                    self.level[v] = self.level[u] + 1
                    q.append(v)
        return self.level[self.sink] != -1

    def dfs(self, u, flow):
        if u == self.sink:
            return flow

            v, cap, rev = self.graph[u][i]

1. **Binary Search**: The binary search helps efficiently narrow down the smallest maximum edge weight (X) that satisfies the graph conditions.
2. **Flow Network**: The network is constructed by splitting each node into two (in-node and out-node) to enforce the threshold on outgoing edges. Reverse graph edges are added to model reachability from node 0.
3. **Dinic's Algorithm**: This algorithm efficiently computes the maximum flow in the network. If the flow reaches the number of nodes minus 1, it confirms that all nodes can reach node 0 in the original graph under the given constraints.
4. **Feasibility Check**: For each candidate X, the solution checks if the flow condition is met, adjusting the binary search range accordingly until the smallest feasible X is found or deemed non-existent. This approach ensures optimal performance even for large input sizes.
