To solve this problem, we need to find the minimum number of front kicks required for Takahashi to move from his starting position to the fish shop in a grid where some cells are walls and others are roads. The front kick action allows him to break walls up to two steps away in any direction, turning them into roads permanently. The challenge is to determine the optimal sequence of moves and kicks to reach the destination with the least number of kicks.

### Approach
1. **Problem Analysis**: The problem involves navigating a grid where walls can be converted to roads by performing front kicks. Each kick costs one action and can convert up to two walls into roads. The goal is to reach the destination with the minimum number of such kicks.
2. **Intuition**: The solution requires a modified BFS or Dijkstra's algorithm where each state tracks the current position and the number of kicks used. The key insight is that once a wall is broken, it remains a road for the rest of the journey, so we can update the grid globally as we explore.
3. **Algorithm Selection**: We use a 0-1 BFS (using a deque) where moving to an adjacent cell costs 0 and performing a front kick costs 1. This approach efficiently explores the grid, prioritizing moves over kicks to minimize the total kick count.
4. **Complexity Analysis**: The algorithm processes each cell in the grid with a priority queue, leading to a time complexity of O(H * W * log(H * W)), where H and W are the grid dimensions. The space complexity is O(H * W) for storing distances and the grid state.

### Solution Code
```python
from collections import deque
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data: 
        return
    H, W = map(int, data[0].split())
    grid = []
    for i in range(1, 1+H):
        grid.append(data[i].strip())
    A, B, C, D = map(int, data[H+1].split())
    start = (A-1, B-1)
    goal = (C-1, D-1)

    if start == goal:
        print(0)
        return

    grid_original = grid
    grid_current = [list(row) for row in grid]
    moves = [(1,0), (-1,0), (0,1), (0,-1)]
    
    dist = [[-1] * W for _ in range(H)]
    dist[start[0]][start[1]] = 0
    dq = deque()
    dq.appendleft((0, start[0], start[1]))
    
    while dq:
        k, r, c = dq.popleft()
        if k != dist[r][c]:
            continue
        if (r, c) == goal:
            print(k)
            return
            
        for dr, dc in moves:
            nr = r + dr
            nc = c + dc
            if 0 <= nr < H and 0 <= nc < W:
                if grid_current[nr][nc] == '.':
                    if k < dist[nr][nc]:
                        dist[nr][nc] = k
                        dq.appendleft((k, nr, nc))
                        
        for dx, dy in moves:
            for step in [1, 2]:
                nr = r + dx * step
                nc = c + dy * step
                if 0 <= nr < H and 0 <= nc < W:
                    if grid_original[nr][nc] == '#' and grid_current[nr][nc] == '#':
                        grid_current[nr][nc] = '.'
            if k + 1 < dist[r][c]:
                dist[r][c] = k + 1
                dq.append((k + 1, r, c))
                
    print(-1)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The grid is read from the input, and the start and goal positions are converted to 0-based indexing. If the start and goal are the same, the answer is 0.
2. **BFS Setup**: A deque is initialized for 0-1 BFS. The distance matrix `dist` keeps track of the minimum kicks required to reach each cell, initialized to -1 (unvisited).
3. **BFS Execution**: The BFS processes each cell by first moving to adjacent cells (0 cost) if they are roads (either original or broken). Then, it performs front kicks (1 cost) in all four directions, breaking walls up to two steps away. The grid is updated globally to reflect broken walls.
4. **Termination**: If the goal is reached, the number of kicks is printed. If the queue is exhausted without reaching the goal, -1 is printed (though the problem guarantees a solution).

This approach efficiently explores the grid, leveraging BFS to prioritize moves and minimize the number of front kicks required to reach the destination.
