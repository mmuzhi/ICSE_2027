It is guaranteed that S contains at least D occurrences of @.

Input

The input is given from Standard Input in the following format:
N D
S

Output

Constraints


- 1 \leq D \leq N \leq 100
- N and D are integers.
- S is a string of length N consisting of @ and ..

=== ANSWER ===

To solve this problem, we need to determine which cookies remain in a series of boxes after Takahashi eats one cookie per day for D days, always selecting the rightmost available cookie each day. The solution involves identifying the positions of the cookies, sorting them from right to left, and then marking the rightmost D cookies as eaten.

### Approach
1. **Problem Analysis**: The problem requires us to simulate the process of Takahashi eating cookies over D days. Each day, he eats the cookie from the rightmost box that still contains a cookie. The challenge is to efficiently determine which cookies remain after D days without simulating each day step-by-step, which could be inefficient for large N and D.

2. **Intuition**: Since Takahashi always picks the rightmost cookie available, the cookies eaten will be the D rightmost cookies in the original string. The remaining cookies will be all other cookies (if any) that are not among these D rightmost ones.

3. **Algorithm Selection**: 
   - **Step 1**: Collect all indices of the boxes that initially contain cookies (i.e., where the character is '@').
   - **Step 2**: Sort these indices in descending order to prioritize the rightmost cookies.
   - **Step 3**: The first D indices in this sorted list are the cookies eaten. The remaining cookies (if any) are those not in this list.
   - **Step 4**: Construct the result string by marking the eaten cookies as '.' and leaving the others as '@' or '.' based on the original string.

4. **Complexity Analysis**: The algorithm involves a linear scan to collect cookie indices (O(N)), sorting these indices (O(N log N) in the worst case, and then building the result string (O(N)). Given the constraints (N ≤ 100), this approach is efficient.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data: 
        return
    parts = data[0].split()
    n = int(parts[0])
    d = int(parts[1])
    s = data[1].strip()
    
    indices = [i for i, char in enumerate(s) if char == '@']
    indices.sort(reverse=True)
    removed_set = set(indices[:d])
    
    res = []
    for i in range(n):
        if i in removed_set:
            res.append('.')
        else:
            res.append(s[i])
            
    print(''.join(res))

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input is read from standard input, which includes the number of boxes (N), the number of days (D), and the initial state of the boxes (S).
2. **Collecting Cookie Indices**: We iterate through the string S to collect all indices where cookies ('@') are present.
3. **Sorting Indices**: The collected indices are sorted in descending order to prioritize the rightmost cookies.
4. **Identifying Removed Cookies**: The first D indices from the sorted list are the cookies eaten. These indices are stored in a set for quick lookup.
5. **Building Result String**: For each box, if its index is in the removed set, it is marked as '.'; otherwise, it retains its original character ('@' or '.').
6. **Output**: The resulting string is printed, showing which cookies remain after D days.

This approach efficiently determines the state of each box after D days by leveraging the rightmost selection criterion and set operations for quick lookups.
