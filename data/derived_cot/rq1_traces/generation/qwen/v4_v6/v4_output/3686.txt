To solve this problem, we need to count the number of ways to split an array into three non-empty subarrays such that either the first subarray is a prefix of the second subarray or the second subarray is a prefix of the third subarray. 

### Approach
1. **Problem Analysis**: The problem requires checking all possible splits of the array into three parts and verifying the prefix conditions. The key insight is that for each possible split, we need to check two conditions:
   - The first subarray (nums1) is a prefix of the second subarray (nums2).
   - The second subarray (nums2) is a prefix of the third subarray (nums3).

2. **Intuition**: To efficiently check these conditions, we use a rolling hash technique (double hashing with two different bases and moduli) to compare segments of the array in constant time. This avoids the O(n^3) time complexity that would result from naive substring comparisons.

3. **Algorithm Selection**: 
   - **Rolling Hash**: We precompute prefix hashes and power arrays for two different bases and moduli. This allows us to compute the hash of any segment of the array in constant time.
   - **Iterate Over Splits**: For each possible split (i, j), where i is the end of the first subarray and j is the end of the second subarray, we check the two conditions using the precomputed hashes. If either condition is satisfied, we count the split.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n^2) due to iterating over all possible splits (O(n^2) splits) and each split check is O(1) with the rolling hash.
   - **Space Complexity**: O(n) for storing the prefix hashes and power arrays.

```python
        mod2 = 10**9 + 9
        base1 = 131
        base2 = 313
        
        max_len = n
        pow1 = [1] * (max_len + 1)
        for i in range(1, max_len + 1):
            pow1[i] = (pow1[i-1] * base1) % mod1
        
        pow2 = [1] * (max_len + 1)
        for i in range(1, max_len + 1):
            pow2[i] = (pow2[i-1] * base2) % mod2
        
        P1 = [0] * (n + 1)
        for i in range(1, n + 1):
            P1[i] = (P1[i-1] * base1 + nums[i-1]) % mod1
        
        P2 = [0] * (n + 1)
        for i in range(1, n + 1):
            P2[i] = (P2[i-1] * base2 + nums[i-1]) % mod2
        
        def get_hash(l, r):
            if l == 0:
                return (P1[r+1], P2[r+1])
            h1 = (P1[r+1] - P1[l] * pow1[r-l+1]) % mod1
            h2 = (P2[r+1] - P2[l] * pow2[r-l+1]) % mod2
            return (h1, h2)
        
        count = 0
        for i in range(n - 2):
            for j in range(i+1, n-1):
                if (j - i >= i + 1):
                    h1_1, h1_2 = get_hash(0, i), get_hash(i+1, i+1+i)
                    if h1_1 == h1_2:
                        count += 1
                if (n - j - 1 >= j - i):
                    h2_1, h2_2 = get_hash(i+1, j), get_hash(j+1, j + (j-i))
                    if h2_1 == h2_2:
                        count += 1
        return count
```

### Explanation
1. **Initialization**: The solution starts by checking if the array length is less than 3, in which case no valid split exists, and returns 0.
2. **Precompute Hashing Parameters**: Two sets of base and modulus values (mod1, base1 and mod2, base2) are chosen to minimize hash collisions. Power arrays (pow1 and pow2) are computed for efficient rolling hash calculations.
3. **Prefix Hash Arrays**: Arrays P1 and P2 store prefix hashes for the input array using the two different bases and moduli. These arrays allow quick hash computation for any segment.
4. **Hash Function**: The `get_hash` function computes the hash of a segment [l, r] in constant time using the precomputed prefix hashes and power arrays.
5. **Iterate Over Splits**: For each possible split (i, j), the solution checks if either the first subarray is a prefix of the second or the second is a prefix of the third. This is done by comparing the hashes of the relevant segments. If either condition is met, the split is counted.
6. **Result**: The total count of valid splits is returned after processing all possible splits.

This approach efficiently checks all possible splits using rolling hashes, ensuring optimal performance even for the upper constraint limits.
