You will be given a competitive programming problem.
You are given an integer array groups, where groups[i] represents the size of the i^th group. You are also given an integer array elements.
Your task is to assign one element to each group based on the following rules:

An element at index j can be assigned to a group i if groups[i] is divisible by elements[j].
If there are multiple elements that can be assigned, assign the element with the smallest index j.
If no element satisfies the condition for a group, assign -1 to that group.

Return an integer array assigned, where assigned[i] is the index of the element chosen for group i, or -1 if no suitable element exists.
Note: An element may be assigned to more than one group.
 
Example 1:

Input: groups = [8,4,3,2,4], elements = [4,2]
Output: [0,0,-1,1,0]
Explanation:

elements[0] = 4 is assigned to groups 0, 1, and 4.
elements[1] = 2 is assigned to group 3.
Group 2 cannot be assigned any element.


Example 2:

Input: groups = [2,3,5,7], elements = [5,3,3]
Output: [-1,1,0,-1]
Explanation:

elements[1] = 3 is assigned to group 1.
elements[0] = 5 is assigned to group 2.
Groups 0 and 3 cannot be assigned any element.


Example 3:

Input: groups = [10,21,30,41], elements = [2,1]
Output: [0,1,0,1]
Explanation:
elements[0] = 2 is assigned to the groups with even values, and elements[1] = 1 is assigned to the groups with odd values.

 
Constraints:

1 <= groups.length <= 10^5

To solve this problem, we need to assign elements from a given array to groups based on specific divisibility conditions. Each group must be assigned the smallest index element from the elements array that divides the group's size. If no such element exists, the group should be assigned -1. 

### Approach
1. **Problem Analysis**: The problem requires checking for each group size whether any element in the elements array divides it. The challenge is to efficiently determine the smallest index element that meets this condition for each group without resorting to a brute-force approach, which would be computationally expensive for large inputs.

2. **Intuition**: For each element in the elements array, we can determine all group sizes that this element can divide. By precomputing the smallest index for each element value and then updating all multiples of these elements, we can efficiently find the smallest index element that divides each group size.

3. **Algorithm Selection**: 
   - **Precompute Minimum Indices**: First, we create an array `min_index` where each entry at index `d` holds the smallest index of the element with value `d` in the elements array.
   - **Update Multiples**: For each element value `d` present in the elements array, update all multiples of `d` (up to the maximum group size) in a separate array `best`, setting each entry to the smallest index encountered so far.
   - **Generate Result**: For each group size, use the precomputed `best` array to determine the smallest index element that divides it, or -1 if no such element exists.

4. **Complexity Analysis**: The algorithm efficiently precomputes necessary values in O(n log n) time, where n is the maximum group size (100,000). This is due to the nested loop that updates multiples of each element, which runs in O(n log n) time. The space complexity is O(n) for storing the `min_index` and `best` arrays.

### Solution Code
```python
class Solution:
    def assignElements(self, groups: List[int], elements: List[int]) -> List[int]:
        max_val = 100000
        min_index = [10**9] * (max_val + 1)
        for idx, val in enumerate(elements):
            if idx < min_index[val]:
                min_index[val] = idx
        
        best = [10**9] * (max_val + 1)
        
        for d in range(1, max_val + 1):
            if min_index[d] == 10**9:
                continue
            multiple = d
            while multiple <= max_val:
                if min_index[d] < best[multiple]:
                    best[multiple] = min_index[d]
                multiple += d
        
        res = []
        for n in groups:
            if n > max_val:
                res.append(-1)
            else:
                if best[n] == 10**9:

                else:
                    res.append(best[n])
        return res
```

### Explanation
1. **Initialization**: The `min_index` array is initialized to store the smallest index for each element value encountered in the elements array.
2. **Precompute Minimum Indices**: For each element in the elements array, update `min_index` to keep track of the smallest index for each value.
3. **Update Multiples**: For each element value `d` present in the elements array, update all multiples of `d` in the `best` array. This ensures that for each group size, the smallest index of a dividing element is recorded.
4. **Result Generation**: For each group size in the groups array, the solution checks the `best` array to determine the smallest index element that divides the group size or returns -1 if no such element exists.

This approach efficiently leverages precomputation to handle large input sizes within feasible time complexity, ensuring optimal performance and correctness.
