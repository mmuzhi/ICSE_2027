### Approach
1. **Problem Analysis**: The problem requires selecting cells from different rows with unique values to maximize the sum. Each row can contribute at most one cell, and all selected cell values must be distinct. The grid dimensions are up to 10x10, making a brute-force approach infeasible due to the high computational complexity.

2. **Intuition**: The solution involves modeling the problem as a bipartite graph matching problem where one set of nodes represents distinct numbers from the grid, and the other set represents rows. An edge connects a distinct number to a row if the number appears in that row. The task is to find a matching that maximizes the sum of the distinct numbers used, with each row used at most once.

3. **Algorithm Selection**: We use dynamic programming with a bitmask to represent the rows used so far. The DP state `dp[i][mask]` keeps track of the maximum sum achievable using the first `i` distinct numbers and a subset of rows represented by `mask`. For each distinct number, we update the DP state by either including it in the current mask (if the corresponding row is available) or excluding it.

4. **Complexity Analysis**: The algorithm processes each distinct number once, and for each, it processes all possible row combinations (bitmask states). The number of distinct numbers is at most 100, and the number of bitmask states is \(2^{10} = 1024\), leading to a total complexity of \(O(d \times 2^m)\), where \(d\) is the number of distinct numbers and \(m\) is the number of rows. This is feasible given the constraints.
