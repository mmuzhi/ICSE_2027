You are given simple undirected graphs G and H, each with N vertices: vertices 1, 2, \ldots, N.
Graph G has M_G edges, and its i-th edge (1\leq i\leq M_G) connects vertices u_i and v_i.

You can perform the following operation on graph H any number of times, possibly zero.

- Choose a pair of integers (i,j) satisfying 1\leq i<j\leq N. Pay A_{i,j} yen, and if there is no edge between vertices i and j in H, add one; if there is, remove it.

Find the minimum total cost required to make G and H isomorphic.
What is a simple undirected graph?
 A simple undirected graph is a graph without self-loops or multi-edges, where edges have no direction.

What does it mean for graphs to be isomorphic?
 Two graphs G and H with N vertices are isomorphic if and only if there exists a permutation (P_1,P_2,\ldots,P_N) of (1,2,\ldots,N) such that for all 1\leq i\lt j\leq N:

-  an edge exists between vertices i and j in G if and only if an edge exists between vertices P_i and P_j in H.

### Approach
1. **Problem Analysis**: The problem requires us to find the minimal cost to make graphs G and H isomorphic by flipping edges in H. The key insight is that two graphs are isomorphic if there exists a permutation of vertices in H such that the edge connections match those in G. The cost is determined by the sum of costs for flipping edges not present in both graphs.

2. **Intuition**: Since the number of vertices N is small (up to 8), we can generate all possible permutations of vertices (N! possibilities) and for each permutation, create a graph T that is the image of G under that permutation. For each such T, we compute the cost to transform H into T by summing the costs of flipping edges that differ between H and T.

3. **Algorithm Selection**: 
   - **Permutations**: Generate all permutations of vertex labels to explore all possible vertex mappings.
   - **Edge Set Generation**: For each permutation, generate the edge set of the transformed graph T.
   - **Cost Calculation**: For each distinct T, calculate the cost to transform H into T by comparing edge sets and summing the costs of necessary flips.

4. **Complexity Analysis**: The algorithm's complexity is dominated by the number of permutations (N! for N vertices). For N=8, this is 40320 permutations. For each permutation, we process O(N^2) edges, leading to an overall complexity of O(N! * N^2), which is feasible for N ≤ 8.

### Explanation
1. **Input Parsing**: The input is read and parsed into the number of vertices (N), edges of graph G, edges of graph H, and the cost matrix for flipping edges.
2. **Cost Matrix Construction**: The cost matrix is built from the input to store the cost of flipping each edge.
3. **Permutation Generation**: All permutations of vertex labels are generated to explore all possible vertex mappings.
4. **Edge Set Generation**: For each permutation, the edge set of the transformed graph T is generated by mapping edges of G according to the permutation.
5. **Cost Calculation**: For each distinct T, the cost to transform H into T is calculated by comparing edge sets and summing the costs of flipping edges that differ between H and T.
6. **Result Output**: The minimal cost across all possible transformations is printed.

This approach efficiently explores all possible isomorphic graphs to G and computes the minimal transformation cost, leveraging the constraints on N to ensure feasibility.
