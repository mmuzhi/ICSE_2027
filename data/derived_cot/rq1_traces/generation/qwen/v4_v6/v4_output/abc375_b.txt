To solve this problem, we need to calculate the total cost for Takahashi to travel from the origin (0, 0) to a sequence of N points in order and then return to the origin. The cost of each move is the Euclidean distance between the two points. The challenge is to compute this efficiently and accurately for a large number of points.

### Approach
1. **Problem Analysis**: The problem involves calculating the sum of Euclidean distances between consecutive points, including the initial move from the origin to the first point and the final move from the last point back to the origin. The Euclidean distance between two points (a, b) and (c, d) is given by \(\sqrt{(a - c)^2 + (b - d)^2}\).

2. **Intuition**: The solution requires processing each segment of the journey separately. For each segment, we compute the Euclidean distance and accumulate the total cost. The key insight is that each segment can be processed independently, allowing us to handle the problem efficiently even for large N (up to 200,000 points).

3. **Algorithm Selection**: The algorithm involves:
   - Reading the input values.
   - Calculating the distance from the origin to the first point.
   - Calculating the distances between each consecutive pair of points.
   - Calculating the distance from the last point back to the origin.
   - Summing all these distances to get the total cost.

4. **Complexity Analysis**: The algorithm processes each point exactly once for the initial and final segments, and each pair of consecutive points once for the intermediate segments. Thus, the time complexity is O(N), which is optimal for the given constraints (N ≤ 2 × 10^5).

5. **Precision Handling**: The use of floating-point arithmetic (via Python's `math.sqrt`) ensures that the calculations are efficient and accurate within the problem's required precision (absolute or relative error ≤ 1e-6).

### Solution Code
```python
import math

def main():
    n = int(input().strip())
    points = []
    for _ in range(n):
        data = input().split()
        x = int(data[0])
        y = int(data[1])
        points.append((x, y))
    
    total = 0.0
    x1, y1 = points[0]
    total += math.sqrt(x1*x1 + y1*y1)
    
    for i in range(n-1):
        x_prev, y_prev = points[i]
        x_curr, y_curr = points[i+1]
        dx = x_curr - x_prev
        dy = y_curr - y_prev
        total += math.sqrt(dx*dx + dy*dy)
        
    x_last, y_last = points[-1]
    total += math.sqrt(x_last*x_last + y_last*y_last)
    
    print("{:.15f}".format(total))

if __name__ == "__main__":
    main()
```

3. **Intermediate Distances**: For each subsequent point, the Euclidean distance from the previous point is calculated and added to the total cost.
4. **Final Distance Calculation**: The distance from the last point back to the origin is computed and added to the total cost.
5. **Output**: The total cost is printed with a precision of 15 decimal places to ensure the result meets the problem's accuracy requirements.
