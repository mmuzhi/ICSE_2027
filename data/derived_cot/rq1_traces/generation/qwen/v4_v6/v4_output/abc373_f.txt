There are N types of items. The i-th type of item has a weight of w_i and a value of v_i. Each type has 10^{10} items available.

- 1 \leq W \leq 3000

- All input values are integers.

5

To solve this problem, we need to maximize the total happiness Takahashi can achieve by selecting items from different types without exceeding the bag's capacity. The happiness for each type is defined by a quadratic function, and the goal is to choose the optimal number of items from each type to maximize the sum of their happiness values while keeping the total weight within the given limit.

### Approach
1. **Problem Analysis**: The problem involves selecting items from multiple types, each with a specific weight and value. The happiness for selecting `k` items of a type is given by `k * v_i - k^2`, where `v_i` is the value per item. The challenge is to maximize the total happiness across all types while ensuring the total weight does not exceed the bag's capacity.

2. **Dynamic Programming (DP) Setup**: We use a DP array where `dp[j]` represents the maximum happiness achievable with a total weight of `j`. The array is initialized to negative infinity except for `dp[0]` which is 0 (indicating zero happiness with zero weight).

3. **DP Update**: For each item type, we update the DP array by considering all possible numbers of items (from 0 to the maximum possible given the weight constraint). For each weight `j`, we check all possible counts `k` of the current item type that fit into `j` and update `dp[j]` if a higher happiness value is found.

4. **Complexity**: The algorithm has a time complexity of O(N * W * K_i), where `N` is the number of item types, `W` is the bag's capacity, and `K_i` is the maximum number of items of type `i` that can be taken (i.e., `W // w_i`). This can be up to 27 billion operations in the worst case, which is manageable for the given constraints using efficient implementation.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    W = int(data[1])
    items = []
    index = 2
    for i in range(n):
        w = int(data[index])
        v = int(data[index + 1])
        index += 2
        items.append((w, v))
    
    dp = [-10**18] * (W + 1)
    dp[0] = 0
    
    for w, v in items:
        new_dp = dp[:]
        K = W // w
        for j in range(W + 1):
            best = -10**18
            max_k = j // w
            if max_k > K:
                max_k = K
            for k in range(max_k + 1):
                remainder = j - k * w
                if remainder < 0:
                    continue
                candidate = dp[remainder] + k * v - k * k
                if candidate > best:
                    best = candidate
            new_dp[j] = best
        dp = new_dp
    
    print(max(dp))

if __name__ == '__main__':
    main()
```

1. **Initialization**: The input is read and parsed into the number of item types `n`, the bag's capacity `W`, and a list of tuples representing each item's weight and value.

2. **DP Array Setup**: The `dp` array is initialized to store the maximum happiness achievable for each possible weight up to `W`. The initial value `dp[0] = 0` means zero happiness with zero weight.

3. **Processing Each Item Type**: For each item type, the algorithm calculates the maximum number of items (`K_i`) that can be taken without exceeding the weight capacity. For each possible weight `j`, it checks all valid counts `k` of the current item type (from 0 to `K_i`). The happiness for each count `k` is computed and used to update the `dp` array.

4. **Result Extraction**: After processing all item types, the maximum value in the `dp` array is printed, representing the highest achievable happiness without exceeding the weight limit.

This approach efficiently explores all possible combinations of items while leveraging dynamic programming to keep track of the best solutions for each weight, ensuring optimal performance within the problem constraints.
