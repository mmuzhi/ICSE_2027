To solve this problem, we need to determine the maximum number of units that can be purchased without exceeding a given budget, where each product's cost is based on the square of the number of units bought multiplied by a given price per unit squared. The challenge lies in efficiently calculating the minimal cost for a given number of units and then using binary search to find the maximum number of units that can be bought within the budget.

### Approach
1. **Problem Analysis**: The problem involves purchasing units from multiple products where each product's cost is quadratic in the number of units bought. The goal is to maximize the total number of units bought without exceeding the budget. The key insight is that the minimal cost for a given number of units is achieved by always selecting the product with the smallest marginal cost for the next unit.

2. **Binary Search on Total Units**: We use binary search to determine the maximum number of units (T) that can be bought. For each candidate T, we compute the minimal cost required to buy T units.

3. **Binary Search for Marginal Cost**: For each candidate T, we perform another binary search to find the T-th smallest marginal cost (X). The marginal cost for the k-th unit of a product is (2*k - 1) * P_i.

4. **Cost Calculation**: Once X is found, the total cost is computed by summing the costs of all units up to X (excluding the T-th unit) and adding the cost of the remaining units at X.

### Solution Code
```python
import sys

def minimal_cost(T, P):
    if T == 0:
        return 0
        
    lo, hi = 0, 10**10
    while lo < hi:
        mid = (lo + hi) // 2
        count_mid = 0
        for p in P:
            k = (mid + p) // (2 * p)
            count_mid += k
        if count_mid >= T:
            hi = mid
        else:
            lo = mid + 1
    X = lo

    count_prev = 0
    S = 0
    for p in P:
        k_prev = (X - 1 + p) // (2 * p)
        count_prev += k_prev
        S += p * (k_prev * k_prev)

    count_eq = 0
    for p in P:
        k_curr = (X + p) // (2 * p)
        if (2 * k_curr - 1) * p == X:
            count_eq += 1
            
    total_cost = S + remaining * X
    return total_cost
```
