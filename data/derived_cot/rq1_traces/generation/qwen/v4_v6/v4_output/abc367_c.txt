=== PROMPT ===
You will be given a competitive programming problem.
Print all integer sequences of length N that satisfy the following conditions, in ascending lexicographical order.

- The i-th element is between 1 and R_i, inclusive.
- The sum of all elements is a multiple of K.

 What is lexicographical order for sequences?
A sequence A = (A_1, \ldots, A_{|A|}) is lexicographically smaller than B = (B_1, \ldots, B_{|B|}) if either 1. or 2. below holds:

- |A|<|B| and (A_{1},\ldots,A_{|A|}) = (B_1,\ldots,B_{|A|}).
- There exists an integer 1\leq i\leq \min\{|A|,|B|\} such that both of the following are true:

- (A_{1},\ldots,A_{i-1}) = (B_1,\ldots,B_{i-1})
- A_i < B_i

Input

The input is given from Standard Input in the following format:
N K
R_1 R_2 \dots R_N

Output

Print the answer in the following format, where X is the number of sequences to print, the i-th of which is A_i=(A_{i,1},A_{i,2},\dots,A_{i,N}):
A_{1,1} A_{1,2} \dots A_{1,N}
A_{2,1} A_{2,2} \dots A_{2,N}
\vdots
A_{X,1} A_{X,2} \dots A_{X,N}

Constraints


- All input values are integers.
- 1 \le N \le 8
- 2 \le K \le 10
- 1 \le R_i \le 5

Sample Input 1

3 2
2 1 3

Sample Output 1

1 1 2
2 1 1
2 1 3

There are three sequences to be printed, which are (1,1,2),(2,1,1),(2,1,3) in lexicographical order.

1

### Approach

   - Each element in the sequence must lie between 1 and R_i for its respective position.
   - The sum of all elements in the sequence must be divisible by K.
   - The sequences must be printed in lexicographical order.

2. **Intuition**: Since the constraints are small (N ≤ 8 and each R_i ≤ 5), a brute-force approach is feasible. We can generate all possible sequences using the Cartesian product of the ranges [1, R_i] for each position, then filter these sequences based on the sum condition.

3. **Algorithm Selection**: We use the `itertools.product` function to generate the Cartesian product of the ranges for each R_i. This function naturally generates sequences in lexicographical order. For each generated sequence, we check if the sum of its elements is divisible by K. If it is, we print the sequence.

4. **Complexity Analysis**: The number of sequences generated is the product of all R_i values. In the worst case, this is 5^8 = 390,625 sequences. For each sequence, we compute the sum, which takes O(N) time. Thus, the total time complexity is O(M * N), where M is the total number of sequences (up to 390,625) and N is the length of the sequence (up to 8). This is manageable given the constraints.

### Solution Code
```python
import sys
from itertools import product

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    k = int(data[1])
    R = list(map(int, data[2:2+n]))
    ranges_list = [range(1, r+1) for r in R]
    for seq in product(*ranges_list):
        if sum(seq) % k == 0:
            print(" ".join(map(str, seq)))

if __name__ == '__main__':
```

3. **Filtering Sequences**: For each generated sequence, we check if the sum of its elements is divisible by K. If it is, the sequence is printed.
