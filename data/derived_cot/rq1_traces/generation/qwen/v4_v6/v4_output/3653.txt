To solve this problem, we need to find the maximum sum of a contiguous subarray (subarray) in an array of integers where the length of the subarray is divisible by a given integer `k`. The solution involves efficiently checking all possible subarrays that meet the length condition and computing their sums to find the maximum.

### Approach
1. **Prefix Sum Array**: First, we compute a prefix sum array where each element at index `i` represents the sum of all elements from the start of the array up to, but not including, the element at index `i` in the original array. This allows us to compute the sum of any subarray in constant time.

2. **Residue Handling**: For each possible residue `r` (0 ≤ `r` < `k`), we maintain a list of indices from the prefix sum array that are congruent to `r` modulo `k`. This helps in efficiently identifying subarrays whose lengths are multiples of `k`.

3. **Suffix Maximum Array**: For each residue list, we build a suffix maximum array. This array helps in quickly finding the maximum prefix sum value from any given index to the end of the residue list, which is crucial for determining the maximum subarray sum efficiently.

4. **Binary Search for Valid Subarrays**: For each starting index `i` in the prefix sum array, we use binary search to find the smallest index `j` (≥ `i+1`) in the corresponding residue list. The maximum subarray sum starting at `i` is then derived from the suffix maximum array of the residue list, ensuring optimal performance.

### Solution Code
```python
import bisect

class Solution:
    def maxSubarraySum(self, nums: List[int], k: int) -> int:
        n = len(nums)
        prefix = [0] * (n + 1)
        for i in range(1, n + 1):
            prefix[i] = prefix[i - 1] + nums[i - 1]
        
        residues = [[] for _ in range(k)]
        for j in range(n + 1):
            r = j % k
            residues[r].append((j, prefix[j]))
        
        res_indices = [[] for _ in range(k)]
        res_smax = [[] for _ in range(k)]
        
        for r in range(k):
            lst = residues[r]
            indices = [j for j, _ in lst]
            vals = [p for _, p in lst]
            m = len(vals)
            if m == 0:
                continue
            smax_arr = [0] * m
            smax_arr[-1] = vals[-1]
            for idx in range(m - 2, -1, -1):
                smax_arr[idx] = max(vals[idx], smax_arr[idx + 1])
            res_indices[r] = indices
            res_smax[r] = smax_arr
        
        ans = -10**18
        for i in range(n + 1):
            r = i % k
            indices_r = res_indices[r]
            smax_arr_r = res_smax[r]
            if not indices_r:
                continue
            pos = bisect.bisect_left(indices_r, i + 1)
            if pos < len(indices_r):
                candidate = smax_arr_r[pos] - prefix[i]
                if candidate > ans:
                    ans = candidate
        return ans
```

### Explanation
1. **Prefix Sum Array**: The prefix array is constructed such that `prefix[i]` holds the sum of the first `i` elements of the original array. This allows for O(1) sum calculation of any subarray.

2. **Residue Lists**: For each residue `r` (0 to `k-1`), we collect all indices `j` from the prefix array where `j % k == r`. This helps in identifying subarrays whose lengths are multiples of `k`.

3. **Suffix Maximum Array**: For each residue list, we compute a suffix maximum array. This array stores the maximum prefix sum value from each position to the end of the residue list, enabling quick lookup of the maximum subarray sum starting from any given index.

4. **Binary Search and Candidate Calculation**: For each starting index `i`, we use binary search to find the smallest index `j` in the corresponding residue list that is at least `i+1`. The maximum subarray sum starting at `i` is then calculated using the suffix maximum array, and the overall maximum is updated accordingly.

This approach efficiently narrows down the search space using mathematical properties of residues and binary search, ensuring optimal performance even for large input sizes.
