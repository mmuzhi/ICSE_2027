The challenge is to determine whether this sum exceeds \(10^9\) without actually computing the entire sum, which could be prohibitively large for large values of \(N\) and \(M\).

### Approach
1. **Problem Analysis**: The problem requires calculating the sum \(X = \sum_{i=0}^{M} N^i\). Given the constraints \(1 \leq N \leq 10^9\) and \(1 \leq M \leq 100\), the sum can become very large quickly. However, we need to check if \(X\) exceeds \(10^9\) early to avoid unnecessary computations and potential overflow.

2. **Key Insight**: The sum of the series can be broken down into individual terms. For each term \(N^i\), if at any point this term exceeds \(10^9\), the entire sum \(X\) will definitely exceed \(10^9\) because subsequent terms will only add more positive values. Additionally, after adding each term, if the cumulative sum exceeds \(10^9\), we can immediately conclude that \(X\) is too large.

3. **Algorithm Selection**: We iterate through each term from \(i = 0\) to \(i = M\). For each term:
   - Check if the current term \(N^i\) exceeds \(10^9\). If it does, print 'inf' and exit.
   - Add the current term to the cumulative sum. If the sum exceeds \(10^9\), print 'inf' and exit.
   - Multiply the current term by \(N\) to compute the next term.
   - If all terms are processed without the sum exceeding \(10^9\), print the cumulative sum.

4. **Complexity Analysis**: The loop runs at most \(M+1\) times (from 0 to \(M\)), and each iteration involves a constant amount of work (multiplication and comparison). Since \(M\) is at most 100, this approach is efficient and feasible within the problem constraints.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    N = int(data[0])
    M = int(data[1])
    
    total = 0
    current = 1
    for _ in range(M + 1):
        if current > 10**9:
            print('inf')
            return
        total += current
        if total > 10**9:
            print('inf')
            return
        current *= N
    print(total)
```

   - **Next Term Calculation**: Multiply `current` by \(N\) to get the next term in the series.
