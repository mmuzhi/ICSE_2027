To solve this problem, we need to determine for each road in a given graph whether its removal affects the shortest path distance between two specific nodes (city 1 and city N). The solution involves leveraging Dijkstra's algorithm to compute shortest paths and their counts from both the source and target nodes, then analyzing each edge's contribution to these shortest paths.

If removing an edge increases the shortest path distance or makes it unreachable, the answer is "Yes"; otherwise, it's "No".

2. **Intuition**: 
   - **Dijkstra's Algorithm**: Compute the shortest distances from city 1 to all other cities (d1) and from city N to all other cities (d2).
   - **Path Counting**: Alongside the shortest distances, count the number of shortest paths (dp1 from city 1 and dp2 from city N) to determine if an edge is critical.
   - **Edge Criticality Check**: For each edge, verify if it lies on any shortest path (using d1 and d2) and if all shortest paths include it (using dp1 and dp2). If both conditions are met, removing the edge will change the shortest path distance.

3. **Algorithm Selection**: 
   - **Dijkstra's Algorithm**: Efficiently computes shortest paths in graphs with non-negative edge weights.
   - **Priority Queue (Min-Heap)**: Used in Dijkstra's to explore nodes in order of increasing distance.
   - **Dynamic Path Counting**: During Dijkstra's execution, accumulate the number of shortest paths to each node.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(M log N) for each Dijkstra run (two passes: from city 1 and city N), where M is the number of edges and N is the number of nodes. This is efficient for the given constraints (N, M ≤ 2×10^5).
   - **Space Complexity**: O(N + M) to store the graph and auxiliary arrays (d1, dp1, d2, dp2).

### Solution Code
```python
import heapq
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it)); m = int(next(it))
    edges = []
    graph = [[] for _ in range(n+1)]
    
    for _ in range(m):
        u = int(next(it)); v = int(next(it)); c = int(next(it))
        edges.append((u, v, c))
        graph[u].append((v, c))
        graph[v].append((u, c))
    
    INF = 10**18
    
    d1 = [INF] * (n+1)
    dp1 = [0] * (n+1)
    d1[1] = 0
    dp1[1] = 1
    heap = []
    heapq.heappush(heap, (0, 1))
    
    while heap:
        dist, node = heapq.heappop(heap)
        if dist != d1[node]:
            continue
        for neighbor, c in graph[node]:
            new_dist = dist + c
            if new_dist < d1[neighbor]:
                d1[neighbor] = new_dist
                dp1[neighbor] = dp1[node]
                heapq.heappush(heap, (new_dist, neighbor))
            elif new_dist == d1[neighbor]:
                dp1[neighbor] += dp1[node]
    
    d2 = [INF] * (n+1)
    dp2 = [0] * (n+1)
    d2[n] = 0
    dp2[n] = 1
    heap = []
    heapq.heappush(heap, (0, n))
    
    while heap:
        dist, node = heapq.heappop(heap)
        if dist != d2[node]:
            continue
        for neighbor, c in graph[node]:
            new_dist = dist + c
            if new_dist < d2[neighbor]:
                d2[neighbor] = new_dist
                dp2[neighbor] = dp2[node]
                heapq.heappush(heap, (new_dist, neighbor))
            elif new_dist == d2[neighbor]:
                dp2[neighbor] += dp2[node]
    
    total = dp1[n]
    
    out_lines = []
    for (u, v, c) in edges:
        count = 0
        if d1[u] + c + d2[v] == d1[n]:
            count += dp1[u] * dp2[v]
        if d1[v] + c + d2[u] == d1[n]:
            count += dp1[v] * dp2[u]
        if count == total:
            out_lines.append("Yes")
        else:
            out_lines.append("No")
    
    for line in out_lines:
        print(line)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of cities (n), roads (m), and the list of edges with their weights.
2. **Graph Construction**: An adjacency list is built to represent the graph.
3. **Dijkstra's Algorithm**: 
   - **Forward Pass (d1, dp1)**: Computes the shortest distance from city 1 to all other cities and the number of shortest paths to each city.
   - **Backward Pass (d2, dp2)**: Computes the shortest distance from city N to all other cities and the number of shortest paths from each city to city N.
4. **Edge Analysis**: For each edge, checks if it lies on any shortest path (using d1 and d2) and if all shortest paths include it (using dp1 and dp2). If both conditions are met, the edge is critical, and removing it changes the shortest path distance.
5. **Output**: For each edge, "Yes" is printed if it is critical; otherwise, "No" is printed.

This approach efficiently checks each edge's impact on the shortest path by leveraging Dijkstra's algorithm and dynamic path counting, ensuring optimal performance for large input sizes.
