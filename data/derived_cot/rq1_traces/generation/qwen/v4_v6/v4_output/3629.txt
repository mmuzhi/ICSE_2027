To solve this problem, we need to determine the length of a string after performing exactly `t` transformations. Each transformation replaces each character in the string according to specific rules: if the character is 'z', it is replaced by the string "ab"; otherwise, it is replaced by the next character in the alphabet. The challenge is to compute the length efficiently, especially since the string can become very long after multiple transformations, and `t` can be as large as 10^5.

### Approach
1. **Problem Analysis**: The key observation is that each transformation either replaces a character with a single character (if it's not 'z') or with two characters ("ab" if it's 'z'). The length of the string after `t` transformations depends on the initial characters and the number of transformations. 

2. **Dynamic Programming (DP) Setup**: We use a DP table where `dp[i][j]` represents the length of the string obtained by transforming the character at position `i` (0-based index) exactly `j` times. The table is initialized with `dp[i][0] = 1` for all `i` since no transformations mean the character remains as is.

3. **DP Table Population**: For each transformation count `j` from 1 to `t`:
   - For 'z' (i=25), the length is the sum of the lengths of the expansions of 'a' and 'b' after `j-1` transformations.
   - For other characters (i < 25), if `j` transformations are within the range to reach 'z' (i.e., `j <= 25 - i`), the length remains 1. Otherwise, the length is the same as the expansion of 'z' after `j - (25 - i)` transformations.

4. **Modulo Operation**: Since the result can be very large, all operations are performed modulo \(10^9 + 7\).

5. **Result Calculation**: After populating the DP table, the result is obtained by summing the lengths corresponding to each character in the initial string after `t` transformations, modulo \(10^9 + 7\).

### Solution Code
```python
class Solution:
    def lengthAfterTransformations(self, s: str, t: int) -> int:
        mod = 10**9 + 7
        n = len(s)
        dp = [[0] * (t + 1) for _ in range(26)]
        
        for i in range(26):
            dp[i][0] = 1
        
        for j in range(1, t + 1):
            for i in range(25, -1, -1):
                if i == 25:
                    dp[i][j] = (dp[0][j - 1] + dp[1][j - 1]) % mod
                else:
                    if j <= 25 - i:
                        dp[i][j] = 1
                    else:
                        k = j - (25 - i)
                        dp[i][j] = dp[25][k] % mod
        
        total = 0
        for char in s:
            idx = ord(char) - ord('a')
            total = (total + dp[idx][t]) % mod
        
        return total
```

### Explanation
1. **Initialization**: The DP table `dp` is initialized with dimensions 26 (for each lowercase letter) by `t+1` (for each transformation count from 0 to `t`). Each entry `dp[i][0]` is initialized to 1, representing the length of the character itself with no transformations.

2. **DP Table Population**: For each transformation count `j` from 1 to `t`, and for each character from 'z' down to 'a':
   - If the character is 'z' (i=25), its length after `j` transformations is the sum of the lengths of 'a' and 'b' after `j-1` transformations.
   - For other characters, if `j` transformations do not reach 'z', the length remains 1. Otherwise, the length is derived from the expansion of 'z' after the remaining transformations.

3. **Result Calculation**: The total length is computed by summing the lengths of each character in the initial string after `t` transformations, using the precomputed DP table values. The result is returned modulo \(10^9 + 7\) to handle large numbers.

This approach efficiently computes the result using dynamic programming, leveraging the structure of the transformation rules to avoid explicitly constructing the transformed strings, thus optimizing both time and space complexity.
