To solve this problem, we need to select k points from a given list of points lying on the boundary of a square such that the minimum Manhattan distance between any two selected points is maximized. The Manhattan distance between two points (x1, y1) and (x2, y2) is defined as |x1 - x2| + |y1 - y2|.

### Approach
1. **Binary Search on Minimum Distance**: The solution involves performing a binary search on the possible minimum Manhattan distances (d) between any two selected points. The search range is from 0 to 2 * side (the maximum possible Manhattan distance in the square).

2. **Greedy Validation of Minimum Distance**: For each candidate distance (d) during the binary search, we check if it's possible to select k points such that the minimum Manhattan distance between any two selected points is at least d. This is done using a greedy algorithm:
   - **Initialization**: Start with an empty set of selected points.
   - **Iterative Selection**: In each iteration, select the point that is farthest from all currently selected points, ensuring it meets the minimum distance requirement (d).
   - **Termination**: If we can select k points meeting the distance requirement, the candidate distance (d) is feasible; otherwise, it is not.

### Solution Code
```python
class Solution:
    def maxDistance(self, side: int, points: List[List[int]], k: int) -> int:
        low = 0
        high = 2 * side
        while low <= high:
            mid = (low + high) // 2
            if self.can_select(mid, points, k):
                low = mid + 1
            else:
                high = mid - 1
        return high

    def can_select(self, d, points, k):
        n = len(points)
        selected = []
        available = set(range(n))
        for _ in range(k):
            if not available:
                return False
            best_index = -1
            best_min_dist = -1
            for i in available:
                min_dist = float('inf')
                for s in selected:
                    dist = abs(points[i][0] - points[s][0]) + abs(points[i][1] - points[s][1])
                    if dist < min_dist:
                        min_dist = dist
                if min_dist >= d and min_dist > best_min_dist:
                    best_min_dist = min_dist
                    best_index = i
            if best_index == -1:
                return False
            selected.append(best_index)
            available.remove(best_index)
        return True
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to 2 * side, representing the range of possible minimum Manhattan distances.

2. **Feasibility Check (can_select method)**:
   - **Initialization**: The method starts with an empty list of selected points and a set of available points.
   - **Greedy Selection**: For each of the k points to select, it iterates over all available points to find the one that is farthest from all already selected points while maintaining a minimum distance (d). The point with the highest minimum distance to the selected points is chosen.
   - **Termination**: If at any point no suitable point is found, the method returns False, indicating that the candidate distance (d) is not feasible. If k points are successfully selected, it returns True.

3. **Binary Search Execution**: The binary search adjusts the range based on the feasibility of the mid-value distance. The loop continues until the search range is exhausted, returning the highest feasible distance found.

This approach efficiently narrows down the maximum possible minimum Manhattan distance by leveraging binary search and a greedy validation method, ensuring optimal performance even for large input sizes.
