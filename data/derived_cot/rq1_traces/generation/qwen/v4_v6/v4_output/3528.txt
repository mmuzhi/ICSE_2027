To solve this problem, we need to find the maximum possible score when moving from the first index to the last index of an array, where each jump from index `i` to `j` (with `j > i`) contributes a score of `(j - i) * nums[i]` to the total score. The challenge is to determine the optimal sequence of jumps that maximizes the total score efficiently, especially given the constraint that the array can be very large (up to 10^5 elements).

### Approach
1. **Problem Analysis**: The problem requires maximizing the score obtained by moving from the start index (0) to the end index (n-1) with jumps only forward. The score for each jump depends on the starting index and the distance jumped. A naive dynamic programming approach would be too slow for large arrays due to its O(n^2) complexity.

2. **Insight**: The key observation is that the score for reaching any index `i` can be expressed as the maximum value derived from all previous indices `j < i` of the form `dp[j] + (i - j) * nums[j]`. This can be rewritten as `(dp[j] - j * nums[j]) + i * nums[j]`, which resembles a linear function in terms of `i` for each `j`.

3. **Algorithm Selection**: We use a LiChao Tree, a data structure that efficiently handles dynamic insertion of linear functions and queries for the maximum value at any point. The LiChao Tree allows us to maintain and query these linear functions in logarithmic time, making the solution feasible for large input sizes.

4. **Complexity Analysis**: The LiChao Tree operations (insertion and query) each take O(log n) time. Since we perform these operations for each of the `n` elements, the total time complexity is O(n log n), which is optimal for the given constraints.

```python
from typing import List

class Line:
    __slots__ = ('m', 'c')
    def __init__(self, m: int, c: int) -> None:
        self.m = m
        self.c = c
    def eval(self, x: int) -> int:
        return self.m * x + self.c

class LiChaoTree:
    __slots__ = ('low', 'high', 'line', 'l', 'r')
    def __init__(self, low: int, high: int) -> None:
        self.low = low
        self.high = high
        self.line = None
        self.l = None
        self.r = None

    def update(self, line) -> None:
        if self.line is None:
            self.line = line
            return
        mid = (self.low + self.high) // 2
        if self.line.eval(mid) < line.eval(mid):
            self.line, line = line, self.line
        if self.low == self.high:
            return
        if line.eval(self.low) > self.line.eval(self.low):
            if self.l is None:
                self.l = LiChaoTree(self.low, mid)
            self.l.update(line)
        if line.eval(self.high) > self.line.eval(self.high):
            if self.r is None:
                self.r = LiChaoTree(mid+1, self.high)
            self.r.update(line)

    def query(self, x: int) -> int:
        res = self.line.eval(x) if self.line else -10**18
        mid = (self.low + self.high) // 2
        if x <= mid and self.l:
            res = max(res, self.l.query(x))
        elif x > mid and self.r:
            res = max(res, self.r.query(x))
        return res

class Solution:
    def findMaximumScore(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 1:
            return 0
        tree = LiChaoTree(0, n-1)
        tree.update(Line(nums[0], 0))
        last = 0
        for i in range(1, n):
            last = tree.query(i)
            tree.update(Line(nums[i], last - i * nums[i]))
        return last
```
