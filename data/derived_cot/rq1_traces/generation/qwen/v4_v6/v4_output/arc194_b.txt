### Approach
1. **Problem Analysis**: The problem involves sorting a permutation using adjacent swaps where each swap has a cost equal to the index of the left element (1-based index). The challenge is to minimize the total cost of these swaps.
2. **Insight**: The optimal strategy involves moving each element from its initial position to its target position (sorted position) in a way that minimizes the total cost. The cost for moving an element right by `k` positions is the sum of the operation indices (left element's index) for each swap, which can be calculated using the formula for the sum of an arithmetic series.
3. **Algorithm Selection**: We use a greedy approach where we process elements from largest to smallest. For each element, we move it to its target position by performing adjacent swaps, updating the positions of affected elements and accumulating the cost.
4. **Complexity Analysis**: The algorithm processes each element and performs swaps proportional to the distance it needs to move. In the worst case, this results in O(N^2) time complexity, which may not be efficient for large N (up to 200,000). However, for smaller inputs, this approach is feasible.

import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    n = int(data[0])
    a = list(map(int, data[1:1+n]))
    
    if a == sorted(a):

        return

    for i in range(n):
        pos[a[i]] = i
