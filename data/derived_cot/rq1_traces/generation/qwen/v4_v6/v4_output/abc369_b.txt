You will be given a competitive programming problem.
Takahashi has a piano with 100 keys arranged in a row.
The i-th key from the left is called key i.
He will play music by pressing N keys one by one.
For the i-th press, he will press key A_i, using his left hand if S_i= L, and his right hand if S_i= R.
Before starting to play, he can place both of his hands on any keys he likes, and his fatigue level at this point is 0.
During the performance, if he moves one hand from key x to key y, the fatigue level increases by |y-x| (conversely, the fatigue level does not increase for any reason other than moving hands).
To press a certain key with a hand, that hand must be placed on that key.
Find the minimum possible fatigue level at the end of the performance.

Input

The input is given from Standard Input in the following format:
N
A_1 S_1
A_2 S_2
\vdots
A_N S_N

Output

Print the minimum fatigue level at the end of the performance.

Constraints


- 1 \leq N \leq 100
- 1 \leq A_i \leq 100
- N and A_i are integers.
- S_i is L or R.

Sample Input 1

4
3 L
6 R
9 L
1 R

Sample Output 1

11

For example, the performance can be done as follows:

- Initially, place the left hand on key 3 and the right hand on key 6.
- Press key 3 with the left hand.
- Press key 6 with the right hand.
- Move the left hand from key 3 to key 9. The fatigue level increases by |9-3| = 6.
- Move the right hand from key 6 to key 1. The fatigue level increases by |1-6| = 5.
- Press key 9 with the left hand.
- Press key 1 with the right hand.

In this case, the fatigue level at the end of the performance is 6+5 = 11, which is the minimum possible.

=== ANSWER ===

To solve this problem, we need to minimize the fatigue level Takahashi experiences while playing a sequence of piano keys with his left and right hands. The fatigue level increases only when a hand moves from one key to another, and the cost is the absolute difference between the keys. The initial positions of both hands can be chosen arbitrarily to minimize the total fatigue.

### Approach
1. **Problem Analysis**: The problem involves processing a sequence of key presses, each specified by a key number and the hand (left or right) used to press it. The goal is to determine the minimal total movement (fatigue) required for both hands to press all keys in sequence, starting from optimally chosen initial positions.
2. **Key Insight**: The movements of the left and right hands are independent. Therefore, we can separately compute the minimal fatigue for each hand and sum them up. The minimal fatigue for each hand is the sum of the absolute differences between consecutive keys it needs to press, as the initial position can be set to the first key of the sequence for that hand, making the first move cost-free.
3. **Algorithm Selection**: For each hand (left or right), collect all keys it needs to press in the order they appear. Then, for each sequence of keys, compute the sum of absolute differences between consecutive keys. The total minimal fatigue is the sum of these values for both hands.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        print(0)
        return
        
    n = int(data[0])
    left_keys = []
    right_keys = []
    
    for i in range(1, n+1):
        parts = data[i].split()
        if not parts:
            continue
        key = int(parts[0])
        hand = parts[1] if len(parts) > 1 else ''
        if hand == 'L':
            left_keys.append(key)
        elif hand == 'R':
            right_keys.append(key)
            
    left_fatigue = 0
    if len(left_keys) > 1:
        for i in range(1, len(left_keys)):
            left_fatigue += abs(left_keys[i] - left_keys[i-1])
            
    right_fatigue = 0
    if len(right_keys) > 1:
        for i in range(1, len(right_keys)):
            right_fatigue += abs(right_keys[i] - right_keys[i-1])
            
    print(left_fatigue + right_fatigue)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, where the first line is the number of key presses, and subsequent lines specify the key and the hand (L or R) for each press.
2. **Separating Keys by Hand**: The keys pressed by the left and right hands are stored in separate lists (`left_keys` and `right_keys`) while maintaining their order of appearance.
3. **Calculating Fatigue**: For each list of keys (left or right), if there are multiple keys, the fatigue is calculated as the sum of absolute differences between consecutive keys. This sum represents the minimal movement required for the hand to press all its keys in sequence, starting from the first key (which sets the initial position optimally).
4. **Output Result**: The total minimal fatigue is the sum of the left and right hand fatigues, printed as the result.

This approach efficiently breaks down the problem into independent subproblems for each hand, leveraging the fact that optimal initial positions and minimal movements can be computed separately for each hand. The solution ensures minimal fatigue by focusing on consecutive key movements and optimal initial positioning.
