To solve this problem, we need to minimize the total dissatisfaction accumulated from shipping orders under specific constraints. The key challenge is to schedule shipments optimally, considering that each shipment can include at most K orders, orders can only be shipped on or after their placement day, and there must be a waiting period of X days between consecutive shipments.

### Approach
1. **Problem Analysis**: The problem involves scheduling shipments of orders such that each shipment adheres to constraints on the number of orders, the earliest shipping day, and the waiting period between shipments. The dissatisfaction for each order is the difference between its shipping day and its placement day. The goal is to minimize the sum of these differences across all orders.

2. **Intuition**: The optimal solution involves grouping orders into contiguous segments (since orders are placed in non-decreasing time order) and scheduling each group's shipment as early as possible, considering the constraints. The shipping day for a group is the maximum of the waiting period from the previous shipment and the latest placement day in the current group.

3. **Dynamic Programming (DP) Setup**: We use a DP approach where `dp[m][i]` represents the minimum total dissatisfaction for the first `i+1` orders using `m` shipments. Each state in `dp[m][i]` stores a tuple `(last_ship_day, total_dissatisfaction)`, where `last_ship_day` is the day the last shipment in the group was made.

4. **DP Initialization**: For the base case (single shipment), we consider all possible contiguous groups of size up to K and compute the dissatisfaction and the corresponding shipping day.

5. **DP Transition**: For each subsequent shipment, we consider all possible contiguous groups ending at the current order, updating the DP state based on the previous shipment's details and the current group's constraints.

6. **Result Extraction**: After processing all possible shipments, the answer is the minimum total dissatisfaction found in the states that cover all orders.

```python
            for j in range(start_j, i + 1):
                group_size = i - j + 1
                if group_size > K:
                    continue
                if j == 0:
                    continue
                if not dp[m_val - 1][j - 1]:
                    continue
                M_val_seg = max_val[j][i]
                for prev in dp[m_val - 1][j - 1]:
                    prev_last = prev[0]
                    total_prev = prev[1]
                    S = max(prev_last + X, M_val_seg)
                    diss_group = group_size * S - (prefix[i + 1] - prefix[j])
                    total_here = total_prev + diss_group
                    dp[m_val][i].append((S, total_here))
                    
    ans = None
    for m_val in range(1, M + 1):
        if not dp[m_val][N - 1]:
            continue
        for state in dp[m_val][N - 1]:
            if ans is None or state[1] < ans:
                ans = state[1]
                
    print(ans if ans is not None else 0)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of orders (N), maximum orders per shipment (K), waiting days between shipments (X), and the placement days of the orders (T).

2. **Prefix Sum and Maximum Calculation**: The prefix sum array helps in quickly calculating the sum of any segment of orders. The `max_val` 2D array stores the maximum placement day in any segment from index `j` to `i`.

3. **DP Initialization**: The DP array `dp` is initialized to store states for each possible number of shipments (up to `M`, the ceiling of N/K) and each possible ending order index.

4. **Base Case Handling**: For the first shipment, all possible contiguous groups of size up to K are considered, and the dissatisfaction and shipping day are calculated and stored.

5. **DP Transition**: For each subsequent shipment, the algorithm considers all valid contiguous groups ending at the current order, updating the DP state based on the previous shipment's details and the current group's constraints.

6. **Result Extraction**: The minimum total dissatisfaction is found by examining the DP states that cover all orders, ensuring the solution adheres to all constraints and minimizes dissatisfaction. The result is printed or 0 if no valid schedule exists. 

This approach efficiently explores all possible shipment schedules using dynamic programming, leveraging contiguous segments and constraints to minimize total dissatisfaction.
