=== ANSWER ===

To solve this problem, we need to determine the minimum number of button presses required to display a given string on a calculator that starts with an empty string. The calculator has buttons for digits 0-9 and a special "00" button that appends two zeros at once. The challenge is to find the optimal sequence of button presses to construct the string with the least number of presses.

### Approach
1. **Dynamic Programming (DP) Setup**: We use a dynamic programming array `dp` where `dp[i]` represents the minimum button presses needed to build the first `i` characters of the string.
2. **Initialization**: The base case is `dp[0] = 0` (no presses needed for an empty string).
3. **DP Transition**:
   - For each position `i` in the string, the default is to press the corresponding digit button, costing `dp[i-1] + 1` presses.
   - If the last two characters (at positions `i-2` and `i-1`) are both '0', we can use the "00" button, costing `dp[i-2] + 1` presses, which might be cheaper than using two separate '0' buttons.
4. **Result Extraction**: The answer is found in `dp[n]`, where `n` is the length of the input string.

### Solution Code
```python
def main():
    S = input().strip()
    n = len(S)
    if n == 0:
        print(0)
        return
        
    dp = [0] * (n + 1)
    
    for i in range(1, n + 1):
        dp[i] = dp[i - 1] + 1
        if i >= 2 and S[i - 2] == '0' and S[i - 1] == '0':
            dp[i] = min(dp[i], dp[i - 2] + 1)
            
    print(dp[n])

if __name__ == "__main__":
    main()
```

### Explanation
- **Initialization**: The `dp` array is initialized to store the minimum presses for each prefix of the string.
- **Base Case**: `dp[0]` is 0 because no presses are needed for an empty string.
- **DP Transition**: For each character in the string, we consider two scenarios:
  - Pressing the digit button directly, which increments the count by 1 from the previous state.
  - Using the "00" button if the current and previous characters are '0', which might reduce the press count by combining two zeros into one button press.
- **Result**: The value `dp[n]` gives the minimum presses needed to construct the entire string, leveraging the optimal use of the "00" button where applicable.

This approach efficiently computes the solution by breaking down the problem into smaller subproblems and reusing previously computed results, ensuring optimal performance even for the maximum input size.
