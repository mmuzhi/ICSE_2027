=== ANSWER ===

To solve this problem, we need to determine the minimum cost required to reach each position in a line of people, starting from the end of the line. The key observation here is that once you pay to swap with a person at a certain position, you can move freely to any position behind (higher index) without additional cost. Therefore, the minimal cost to reach any position `i` is the minimum cost encountered from the start of the line up to and including position `i`.

### Approach
1. **Problem Analysis**: The problem involves moving from the end of a line (position `n`) to each position `i` (from `0` to `n-1`). Each person in front (lower index) charges a specific amount to swap places, while people behind (higher index) can swap for free. The goal is to find the minimal cost to reach each position `i`.
2. **Insight**: The minimal cost to reach position `i` is the minimum value between all costs from position `0` to `i`. This is because paying the minimal cost at any position `j` (where `j <= i`) allows you to reach position `i` for free.
3. **Algorithm Selection**: We iterate through the cost array, keeping track of the minimum cost encountered so far. For each position `i`, the answer is the minimum cost between the current position's cost and the previously recorded minimum cost.
4. **Complexity Analysis**: The algorithm runs in O(n) time, where `n` is the length of the cost array, as it requires a single pass through the array. The space complexity is O(n) to store the result.

### Solution Code
```python
class Solution:
    def minCosts(self, cost: List[int]) -> List[int]:
        ans = []
        min_val = float('inf')
        for c in cost:
            if c < min_val:
                min_val = c
            ans.append(min_val)
        return ans
```

### Explanation
- **Initialization**: We start with an empty list `ans` to store the results and initialize `min_val` to a very large number to represent the initial minimum cost.
- **Iteration**: For each cost in the input array, we check if the current cost is less than `min_val`. If it is, we update `min_val` to this cost. We then append `min_val` to the result list `ans`.
- **Result**: After processing all elements, `ans` contains the minimal cost to reach each position from the start of the line up to that position. This approach efficiently computes the solution by leveraging the prefix minimum property, ensuring optimal performance.
