To solve this problem, we need to split a given sequence of integers into three non-empty contiguous subarrays such that the sum of the counts of distinct integers in each subarray is maximized. The solution involves efficiently determining the optimal split points (i, j) where 1 ≤ i < j ≤ N-1, and the sum of distinct integers in the three subarrays is maximized.

### Approach
1. **Problem Analysis**: The problem requires splitting the array into three parts and calculating the sum of distinct integers in each part. The challenge is to find the split points (i, j) that maximize this sum without resorting to a brute-force O(N^2) approach, which would be inefficient for large N.

2. **Intuition**: The solution leverages precomputed arrays to store the distinct counts for all prefixes (left part) and suffixes (right part) of the array. For each possible left part end (i), we then compute the distinct count for the middle part (from i to j-1) by iterating through possible right part starts (j). The sum of the distinct counts from the left, middle, and right parts is then evaluated to find the maximum value.

3. **Algorithm Selection**: 
   - **Prefix and Suffix Arrays**: We first compute two arrays, `left` and `right`, where `left[i]` is the number of distinct integers from the start up to index i-1, and `right[j]` is the number of distinct integers from index j to the end of the array.
   - **Sliding Window for Middle Part**: For each possible left part end (i), we use a sliding window approach to compute the distinct count for the middle part (from i to j-1) as we iterate j from i+1 to N-1. This involves maintaining a frequency dictionary to track distinct elements in the middle part dynamically.

4. **Complexity Analysis**: The algorithm runs in O(N^2) time in the worst case, where N is the length of the array. This is because for each i (O(N) possibilities), we iterate j from i+1 to N-1 (O(N) operations each). However, this approach is feasible for the given constraints (N ≤ 3×10^5) only if the input size is small. For larger inputs, a more efficient approach using data structures like segment trees or sparse tables would be necessary.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    left = [0] * (n + 1)
    seen = {}
    for i in range(1, n + 1):
        if A[i - 1] not in seen:
            seen[A[i - 1]] = 1
            left[i] = left[i - 1] + 1
        else:
            seen[A[i - 1]] += 1
            left[i] = left[i - 1]
            
    right = [0] * (n + 1)
    seen = {}
    for i in range(n - 1, -1, -1):
        if A[i] not in seen:
            seen[A[i]] = 1
            right[i] = right[i + 1] + 1
        else:
            seen[A[i]] += 1
            right[i] = right[i + 1]
            
    best = 0
    for i in range(1, n - 1):
        freq = {}
        distinct_mid = 0
        for j in range(i + 1, n):
            num = A[j - 1]
            if num not in freq:
                distinct_mid += 1
            freq[num] = freq.get(num, 0) + 1
            current_sum = left[i] + distinct_mid + right[j]
            if current_sum > best:
                best = current_sum
                
    print(best)

if __name__ == "__main__":
    main()
```
