To solve this problem, we need to format a given real number such that its decimal part does not have any trailing zeros, and there is no unnecessary trailing decimal point. The input is provided to the third decimal place, meaning it may have up to three digits after the decimal point, but we need to remove any trailing zeros beyond the necessary precision.

### Approach
1. **Input Handling**: Read the input string and strip any leading or trailing whitespace.
2. **Check for Decimal Point**: Determine if the input contains a decimal point.
3. **Process Decimal Input**: If a decimal point is present, split the input into integer and fractional parts. Remove leading zeros from the integer part and trailing zeros from the fractional part. If the fractional part becomes empty after removing trailing zeros, omit the decimal point and its trailing zeros.
4. **Process Non-Decimal Input**: If there is no decimal point, simply remove any leading zeros from the input string. If the resulting string is empty, output "0".
5. **Edge Cases**: Handle cases where the input is a single decimal point or all zeros by converting them to "0".

### Solution Code
```python
def main():
    s = input().strip()
    
    if '.' in s:
        parts = s.split('.', 1)
        a = parts[0]
        b = parts[1]
        a = a.lstrip('0') or '0'
        b = b.rstrip('0')
        if b == '':
            print(a)
        else:
            print(a + '.' + b)
    else:
        s_clean = s.lstrip('0')
        if s_clean == '':
            print('0')
        else:
            print(s_clean)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read as a string and stripped of any extraneous whitespace.
2. **Decimal Handling**: If the input contains a decimal point, it is split into two parts: the integer part and the fractional part. The integer part is processed to remove any leading zeros, and if it becomes empty, it defaults to "0". The fractional part is processed to remove trailing zeros. If the fractional part becomes empty after this processing, only the integer part is printed; otherwise, both parts are printed with a decimal point.
3. **Non-Decimal Handling**: If the input does not contain a decimal point, leading zeros are stripped. If the result is an empty string, "0" is printed; otherwise, the cleaned string is printed.
4. **Edge Cases**: The solution handles cases like "." (converted to "0") and "000.000" (converted to "0") correctly by ensuring no unnecessary characters are present in the output.

This approach efficiently formats the input number by removing unnecessary trailing zeros and ensuring the output adheres to the specified conditions.
