        def feasible(H, U_list, D_list, X):
            n = len(U_list)
            # If H is greater than min_sum, it's not allowed, but our binary search ensures H<=min_sum.
            L = [max(0, H - d) for d in D_list]
            R = [min(u, H) for u in U_list]

            # Forward pass
            forward_low = [0]*n
            forward_high = [0]*n
            forward_low[0] = L[0]
            forward_high[0] = R[0]
            for i in range(1, n):
                # The previous tooth's interval is [forward_low[i-1], forward_high[i-1]]
                # The constraint from the left is that the current tooth's value must be in [forward_low[i-1]-X, forward_high[i-1]+X]
                # But we must intersect with [L[i], R[i]]
                new_low = max(L[i], forward_low[i-1] - X)
                new_high = min(R[i], forward_high[i-1] + X)
                if new_low > new_high:
                    return False
                forward_low[i] = new_low
                forward_high[i] = new_high

            # Backward pass
            backward_low = [0]*n
            backward_high = [0]*n
            backward_low[-1] = L[-1]
            backward_high[-1] = R[-1]
            for i in range(n-2, -1, -1):
                new_low = max(L[i], backward_low[i+1] - X)
                new_high = min(R[i], backward_high[i+1] + X)
                if new_low > new_high:
                    return False
                backward_low[i] = new_low
                backward_high[i] = new_high

            # Now, for each tooth i, the value must be in the intersection of forward and backward intervals.
            for i in range(n):
                if forward_low[i] > backward_high[i] or backward_low[i] > forward_high[i]:

To solve this problem, we need to determine the minimum cost required to adjust the lengths of Takahashi's upper and lower teeth such that they fit together well. The teeth are considered to fit together well if two conditions are met: there exists a common sum H for each pair of upper and lower teeth, and the sequence of upper teeth lengths has consecutive differences of at most X. The cost is the total number of operations needed to reduce the teeth lengths to meet these conditions.

### Approach
1. **Problem Analysis**: The problem requires finding the maximum possible integer H such that for each tooth, the sum of its upper and lower lengths equals H, and the sequence of upper teeth lengths has consecutive differences ≤ X. The cost is calculated as the total reduction in teeth lengths, which is minimized by maximizing H.

2. **Key Insight**: The maximum feasible H is the largest value ≤ the minimum initial sum of any tooth pair (min_sum) that allows the sequence of upper teeth to meet the consecutive difference condition. This is determined using a two-pass method to check connectivity in the sequence of teeth lengths.

3. **Binary Search**: We perform a binary search on H from 0 to min_sum to find the maximum feasible H. For each candidate H, we check if the sequence of upper teeth can be adjusted to meet the conditions using a two-pass method.

4. **Two-Pass Method**: The two-pass method involves:
   - **Forward Pass**: Propagate constraints from left to right to determine the range of possible upper teeth lengths.
   - **Backward Pass**: Propagate constraints from right to left to verify the sequence can be formed with the given constraints.
   - **Intersection Check**: Ensure that for each tooth, the ranges from both passes have a non-empty intersection.

5. **Complexity**: The binary search runs in O(log(min_sum)) time, and each feasibility check is O(N) due to the two-pass method. Given N can be up to 2×10^5 and min_sum up to 10^9, the total operations are manageable (around 30×2×N).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    X = int(data[1])
    U_list = []
    D_list = []
    index = 2
    for i in range(n):
        u = int(data[index])
        d = int(data[index+1])
        index += 2
        U_list.append(u)
        D_list.append(d)
    
    total_sum = sum(U_list) + sum(D_list)
    min_sum = min(u + d for u, d in zip(U_list, D_list))
    
    def feasible(H):
        L = [max(0, H - d) for d in D_list]
        R = [min(u, H) for u in U_list]
        
        forward_low = L[:]
        forward_high = R[:]
        
        for i in range(1, n):
            new_low = max(L[i], forward_low[i-1] - X)
            new_high = min(R[i], forward_high[i-1] + X)
            if new_low > new_high:
                return False
            forward_low[i] = new_low
            forward_high[i] = new_high
        
        backward_low = L[:]
        backward_high = R[:]
        
        for i in range(n-2, -1, -1):
            new_low = max(L[i], backward_low[i+1] - X)
            new_high = min(R[i], backward_high[i+1] + X)
            if new_low > new_high:
                return False
            backward_low[i] = new_low
            backward_high[i] = new_high
        
        for i in range(n):
            if forward_low[i] > backward_high[i] or backward_low[i] > forward_high[i]:
                return False
        
        return True

    low = 0
    high = min_sum
    ans_H = 0
    while low <= high:
        mid = (low + high) // 2
        if feasible(mid):
            ans_H = mid
            low = mid + 1
        else:
            high = mid - 1
            
    answer = total_sum - n * ans_H
    print(answer)

if __name__ == "__main__":
    main()
```

3. **Feasibility Check**: The `feasible` function checks if a given H allows the sequence of upper teeth to meet the conditions. It uses forward and backward passes to propagate constraints and checks for overlapping valid ranges.
4. **Binary Search**: The binary search efficiently narrows down the maximum feasible H by checking mid-values and adjusting the search range based on feasibility.
5. **Result Calculation**: The minimum cost is computed as the total initial sum minus N times the maximum feasible H, and the result is printed.
