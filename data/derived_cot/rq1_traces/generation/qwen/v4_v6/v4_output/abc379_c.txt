To solve this problem, we need to determine the minimum number of operations required to distribute stones across N cells such that each cell contains exactly one stone. The operations allowed are moving one stone from cell i to cell i+1 any number of times. The solution involves checking feasibility conditions and calculating the minimal operations if feasible.

### Approach
1. **Feasibility Check**: 
   - **Total Stones Check**: The total number of stones must be exactly N, as each cell needs one stone. If not, it's impossible to achieve the goal.
   - **Cumulative Stones Check**: For each cell i (from 1 to N), the cumulative number of stones from cells with indices ≤ i must be at least i. This ensures that we can cover all cells up to i with the available stones.

2. **Calculate Minimum Operations**:
   - If the feasibility conditions are met, the minimal operations required is computed as the difference between the sum of all target cell indices (1 to N) and the sum of the initial positions of all stones. This is based on the observation that each stone moved from its initial position x_i to a target position j contributes (j - x_i) to the total operations.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    m = int(data[1])
    X = list(map(int, data[2:2+m]))
    A = list(map(int, data[2+m:2+2*m]))
    
    total_stones = sum(A)
    if total_stones != n:
        print(-1)
        return
        
    total_initial = 0
    for i in range(m):
        total_initial += X[i] * A[i]
        
    stones = []
    for i in range(m):
        stones.append((X[i], A[i]))
    stones.sort(key=lambda t: t[0])
    
    cumulative_count = 0
    prev = 0
    for x, a in stones:
        if prev + 1 <= x - 1:
            if cumulative_count < x - 1:
                print(-1)
                return
                
        cumulative_count += a
        if cumulative_count < x:
            print(-1)
            return
            
        prev = x
        
    total_moves = n*(n+1)//2 - total_initial
    print(total_moves)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Feasibility Check**:
   - **Total Stones Check**: The code first checks if the total number of stones (sum of all A_i) equals N. If not, it immediately returns -1.
   - **Cumulative Stones Check**: The stones are sorted by their cell indices. For each stone, the code checks if the cumulative stones from previous cells can cover the gap up to the current cell. If at any point the cumulative stones are insufficient, it returns -1.

2. **Minimum Operations Calculation**:
   - If all feasibility conditions are satisfied, the minimal operations are calculated as the difference between the sum of all target cell indices (1 to N) and the sum of the initial positions of all stones (sum of x_i * a_i for each stone). This difference gives the total number of moves required to position each stone in its target cell.

This approach efficiently checks for feasibility and computes the minimal operations using basic arithmetic and sorting, ensuring optimal performance even for large input sizes.
