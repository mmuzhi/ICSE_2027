To solve this problem, we need to simulate the process of typing a target string using a special keyboard with two keys. The first key appends an 'a' to the current string, and the second key changes the last character of the string to its next character in the alphabet (cyclically). The goal is to generate all intermediate strings that appear on the screen while typing the target string with the minimum number of key presses.

### Approach
1. **Initialization**: Start with an empty string and an empty result list.
2. **Iterate through each character in the target string**:
   - **Extend the string**: For each character at position `i`, extend the current string by appending 'a's until its length is `i+1`. Each extension operation appends one 'a' and adds the resulting string to the result list.
   - **Change the last character**: After extending, if the last character is not the target character, use the second key to change it. The number of operations needed is determined by the difference between the ASCII values of the target character and 'a'. Each operation changes the last character to the next one in the alphabet (cyclically), and each intermediate string is added to the result list.
3. **Result Construction**: The result list, which contains all intermediate strings in the order they appear, is returned after processing all characters.

### Solution Code
```python
class Solution:
    def stringSequence(self, target: str) -> List[str]:
        s = ""
        res = []
        for i in range(len(target)):
            while len(s) < i + 1:
                s += 'a'
                res.append(s)
            steps = ord(target[i]) - ord('a')
            for _ in range(steps):
                s = s[:-1] + chr(ord(s[-1]) + 1)
                res.append(s)
        return res
```

### Explanation
1. **Initialization**: The variables `s` (current string) and `res` (result list) are initialized to empty strings and lists, respectively.
2. **Character Processing**:
   - For each character in the target string, the code first ensures the current string is of length `i+1` by appending 'a's. Each appended 'a' generates a new string added to the result list.
   - Once the string length is correct, the code calculates the number of operations needed to change the last character (currently 'a') to the target character. This is done by computing the difference in their ASCII values.
   - Each operation changes the last character to the next one in the alphabet (cyclically), and each intermediate string is recorded in the result list.
3. **Result**: After processing all characters, the result list, containing all intermediate strings in the order they were generated, is returned.

This approach efficiently simulates the typing process, ensuring minimal key presses by always extending the string first and then adjusting the last character as needed. The solution handles all edge cases, including cyclic changes (e.g., 'z' changing to 'a') and varying target lengths.
