To solve this problem, we need to find a non-empty subsequence of a given integer array such that the alternating sum of the subsequence equals a specified value `k`, and the product of all elements in the subsequence is maximized without exceeding a given `limit`. The alternating sum is defined as the sum of elements at even indices minus the sum of elements at odd indices in the subsequence.

### Approach
1. **Problem Analysis**: The problem requires checking all possible non-empty subsequences of the array to find those that meet the alternating sum condition and have a product within the specified limit. Given the constraints (array length up to 150 and elements up to 12), a brute-force approach is infeasible. Instead, we use dynamic programming to efficiently explore possible subsequences.

2. **Dynamic Programming Setup**: We use two dictionaries, `dp_even` and `dp_odd`, to keep track of the maximum product achievable for each alternating sum. `dp_even` stores states for subsequences of even length, and `dp_odd` stores states for subsequences of odd length. The initial state is an empty subsequence with alternating sum 0 and product 1.

3. **State Transitions**: For each element in the non-zero part of the array, we update the DP states:
   - **Even-Length Subsequences**: Adding an element at an even index (next position) increases the alternating sum by the element's value.
   - **Odd-Length Subsequences**: Adding an element at an odd index (next position) decreases the alternating sum by the element's value.
   - The product is updated multiplicatively, and we only keep states where the product does not exceed the limit.

4. **Handling Zeros**: For the case where `k` is 0, we also consider subsequences consisting solely of zeros (if any exist), which trivially satisfy the alternating sum condition with a product of 0.

5. **Result Extraction**: After processing all elements, we check `dp_even` and `dp_odd` for the target `k`. If found, the maximum product (if any) is returned. For `k = 0`, we also check if any zero exists in the array to form a valid subsequence with product 0.

### Solution Code
```python
class Solution:
    def maxProduct(self, nums: list[int], k: int, limit: int) -> int:
        non_zero = [x for x in nums if x != 0]
        zeros = [x for x in nums if x == 0]
        
        dp_even = {0: 1}
        dp_odd = {}
        
        for x in non_zero:
            new_dp_even = dp_even.copy()
            new_dp_odd = dp_odd.copy()
            
            for s, p in dp_even.items():
                new_s = s + x
                new_p = p * x
                if new_p <= limit:
                    if new_s not in new_dp_odd or new_p > new_dp_odd[new_s]:
                        new_dp_odd[new_s] = new_p
            
            for s, p in dp_odd.items():
                new_s = s - x
                new_p = p * x
                if new_p <= limit:
                    if new_s not in new_dp_even or new_p > new_dp_even[new_s]:
                        new_dp_even[new_s] = new_p
            
            dp_even = new_dp_even
            dp_odd = new_dp_odd
        
        if 0 in dp_even and dp_even[0] == 1:
            del dp_even[0]
        
        if k != 0:
            even_val = dp_even.get(k, -1)
            odd_val = dp_odd.get(k, -1)
            ans = max(even_val, odd_val)
            return ans if ans != -1 else -1
        else:
            non_zero_ans = max(dp_even.get(0, -1), dp_odd.get(0, -1))
            if non_zero_ans != -1:
                return non_zero_ans
            else:
                return 0 if zeros else -1
```

### Explanation
1. **Initialization**: The array is split into non-zero and zero elements. The DP dictionaries `dp_even` and `dp_odd` are initialized to track subsequences of even and odd lengths, respectively.
2. **DP State Updates**: For each non-zero element, the DP states are updated by considering both including and excluding the element. The alternating sum and product are adjusted based on the subsequence length.
3. **Empty Subsequence Removal**: After processing all elements, the initial empty subsequence state is removed from `dp_even`.
4. **Result Handling**: For `k ≠ 0`, the solution checks both DP dictionaries for the target `k` and returns the maximum valid product. For `k = 0`, it checks non-zero subsequences first, and if none exist, it checks for zeros to form a valid subsequence with product 0.

This approach efficiently narrows down the search space using dynamic programming, ensuring optimal performance even for the upper constraint limits.
