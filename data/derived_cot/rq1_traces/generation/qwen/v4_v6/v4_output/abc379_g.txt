To solve this problem, we need to count the number of ways to replace each '?' in a given grid with digits 1, 2, or 3 such that no two adjacent cells (edge-sharing) contain the same digit. The solution involves different approaches based on the grid dimensions to ensure efficiency and correctness.

### Approach
1. **Single Row Handling (H = 1)**: For grids with only one row, we use a dynamic programming approach to generate all valid sequences of digits that satisfy the adjacent condition within the row. Each cell in the row is processed sequentially, ensuring that each digit is different from its left neighbor if it is a '?'.

2. **Small Grid Handling (W ≤ 10)**: For grids with more than one row and up to 10 columns, we use a two-dimensional dynamic programming approach. We generate all valid states for each row considering both within-row and between-row conditions (adjacent cells in the same column must also differ). The DP state tracks the number of ways to fill each row while respecting the constraints from the previous row.

3. **Large Grid Handling (W > 10)**: For grids with more than 10 columns, we use a depth-first search (DFS) approach with backtracking. The DFS explores all possible digit assignments for '?' cells, checking adjacent conditions on the fly. This method is chosen due to the constraint limitations (H * W ≤ 200), ensuring that the grid size is manageable even though the worst-case complexity is high.
