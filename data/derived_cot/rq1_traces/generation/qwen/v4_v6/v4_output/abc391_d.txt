The use of prefix sums and binary search allows efficient range queries, making the solution feasible for large inputs.

### Explanation
1. **Initial Setup**: The input is read and parsed into block positions and queries.
2. **Sorting and Grouping**: Blocks are sorted by their initial row positions. Blocks in each column are grouped and sorted to facilitate processing.
3. **Falling Time Calculation**: For each column, the time it takes for the lowest block to reach the bottom row is calculated. This helps in determining when a block might be removed.
4. **Prefix Sum for Efficient Queries**: A prefix sum array is built to quickly determine if the bottom row is filled by a certain time, which is crucial for checking block removal conditions.
5. **Query Processing**: For each query, the solution checks if the block exists at the given time by verifying if it has been removed (i.e., it reached the bottom row when the entire bottom row was filled) or if it is still present.

This approach efficiently handles the large constraints by leveraging sorting, prefix sums, and binary search to answer each query in logarithmic time relative to the number of columns.
