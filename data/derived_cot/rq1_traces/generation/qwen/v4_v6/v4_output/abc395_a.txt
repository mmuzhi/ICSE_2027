=== ANSWER ===

To solve this problem, we need to determine whether a given sequence of integers is strictly increasing. A sequence is strictly increasing if each element is less than the next element in the sequence. 

### Approach
1. **Input Reading**: Read the first integer, which represents the length of the sequence (N). Then read the next line, which contains the sequence of N integers.
2. **Check Strictly Increasing Condition**: Iterate through each element of the sequence (except the last one) and compare it with the next element. If any element is found to be greater than or equal to the next element, the sequence is not strictly increasing.
3. **Output Result**: If all consecutive elements satisfy the condition (each element is less than the next), print "Yes". Otherwise, print "No".

### Solution Code
```python
def main():
    n = int(input().strip())
    data = list(map(int, input().split()))
    
    if n == 0:
        print("No")
        return
    
    is_strictly_increasing = True
    for i in range(n - 1):
        if data[i] >= data[i + 1]:
            is_strictly_increasing = False
            break
            
    print("Yes" if is_strictly_increasing else "No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The first line of input is read as an integer `n`, which is the length of the sequence. The second line is read and split into individual integers, which are then stored in a list.
2. **Checking Condition**: The code then checks each consecutive pair in the list. If any element is found to be greater than or equal to the next element, the variable `is_strictly_increasing` is set to `False`, and the loop breaks early for efficiency.
3. **Output**: Based on the value of `is_strictly_increasing`, the code prints "Yes" if the sequence is strictly increasing, otherwise it prints "No".

This approach efficiently checks the strictly increasing condition by iterating through the sequence once, making it optimal with a time complexity of O(N), where N is the length of the sequence. The space complexity is O(1) as no additional data structures are used beyond the input list.
