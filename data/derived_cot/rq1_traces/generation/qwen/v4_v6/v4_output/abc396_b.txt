=== ANSWER ===

To solve this problem, we need to simulate a stack of cards where each card is initially labeled with the integer 0. The stack starts with 100 such cards. We then process a series of queries, each either adding a new card labeled with an integer x on top of the stack or removing the top card and printing its value. The solution involves maintaining a stack data structure to handle these operations efficiently.

### Approach
1. **Initialization**: Start with a stack containing 100 zeros. This represents the initial stack as described in the problem.
2. **Processing Queries**: For each query:
   - **Type 1 (Push)**: Read the integer x and push it onto the stack.
   - **Type 2 (Pop)**: Remove the top card from the stack and print its value. The problem guarantees that there will always be at least one card in the stack when a pop operation is performed.
3. **Output**: The output consists of the values of the cards popped during type 2 queries, each printed on a separate line in the order they are popped.

The approach leverages the Last-In-First-Out (LIFO) behavior of a stack to efficiently handle the push and pop operations. The solution efficiently processes each query in constant time, making it suitable for the given constraints.

### Solution Code
```python
Q = int(input().strip())
stack = [0] * 100

for _ in range(Q):
    data = input().split()
    if data[0] == '1':
        x = int(data[1])
        stack.append(x)
    else:
        print(stack.pop())
```

### Explanation
1. **Initialization**: The stack is initialized with 100 zeros, representing the initial state as specified.
2. **Query Processing**: For each query:
   - **Type 1 Queries**: The integer x is read and appended to the stack, effectively placing it on top of the current stack.
   - **Type 2 Queries**: The top element of the stack is removed and printed. The stack's LIFO nature ensures that the most recently added elements are removed first.
3. **Output**: Each pop operation's result is printed immediately, providing the required output in the sequence of operations.

This approach efficiently handles the stack operations, ensuring correct behavior for both push and pop queries while adhering to the problem constraints. The use of a list as a stack allows for straightforward implementation of the required operations.
