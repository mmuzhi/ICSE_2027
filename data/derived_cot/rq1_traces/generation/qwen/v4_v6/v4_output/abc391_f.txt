To solve this problem, we need to find the K-th largest value among all possible combinations of elements from three given sequences A, B, and C. The value for each combination (i, j, k) is computed as \(A_i \times B_j + B_j \times C_k + C_k \times A_i\). Given the constraints, a brute-force approach is infeasible, so we need an optimized method.

### Approach
1. **Problem Analysis**: The key observation is that the value for each triplet (i, j, k) can be rewritten as \(B_j \times (A_i + C_k) + A_i \times C_k\). This expression is symmetric and depends on the three elements chosen from the sequences. The top K values must come from the largest elements in each sequence, so we can limit our search to the top M elements from each sequence, where M is the smallest integer such that \(M^3 \geq K\).

2. **Algorithm Selection**: 
   - **Step 1**: Read the input values and parse the three sequences.
   - **Step 2**: Determine M, the smallest integer such that \(M^3 \geq K\).
   - **Step 3**: Sort each sequence in descending order and select the top M elements from each.
   - **Step 4**: Generate all possible combinations of the top M elements from each sequence, compute the value for each combination, and collect these values.
   - **Step 5**: Sort the collected values in descending order and return the K-th largest value.

3. **Complexity Analysis**: 
   - Sorting each sequence takes \(O(N \log N)\) time.
   - Selecting the top M elements from each sequence is \(O(N)\) per sequence.
   - Generating all combinations of M elements from each sequence results in \(O(M^3)\) combinations, which is manageable since M is at most 80 for K up to 500,000.
   - Sorting \(M^3\) values is \(O(M^3 \log M^3)\), which is feasible given the constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    k = int(data[1])
    A = list(map(int, data[2:2+n]))
    B = list(map(int, data[2+n:2+2*n]))
    C = list(map(int, data[2+2*n:2+3*n]))
    
    M = 1
    while M**3 < k:
        M += 1
        
    A.sort(reverse=True)
    B.sort(reverse=True)
    C.sort(reverse=True)
    
    A_top = A[:M]
    B_top = B[:M]
    C_top = C[:M]
    
    values = []
    for a in A_top:
        for b in B_top:
            for c in C_top:
                val = a * b + b * c + c * a
                values.append(val)
                
    values.sort(reverse=True)
    print(values[k-1])

if __name__ == '__main__':
    main()
```

3. **Sorting and Selecting Top Elements**: Each sequence is sorted in descending order, and the top M elements from each are selected.
4. **Generating Combinations**: All combinations of the top M elements from each sequence are generated, and the value for each combination is computed.
5. **Sorting and Output**: The computed values are sorted in descending order, and the K-th largest value is printed.

This approach efficiently narrows down the problem to a manageable subset of elements, leveraging sorting and combinatorial generation to avoid the infeasible \(O(N^3)\) brute-force solution.
