There is an undirected graph with N vertices and 0 edges. The vertices are numbered 1 to N.
You are given Q queries to process in order. Each query is of one of the following two types:

- Type 1: Given in the format 1 u v. Add an edge between vertices u and v.
- Type 2: Given in the format 2 v k. Print the k-th largest vertex number among the vertices connected to vertex v. If there are fewer than k vertices connected to v, print -1.

Input

The input is given from Standard Input in the following format:
N Q
\mathrm{query}_1
\mathrm{query}_2
\vdots
\mathrm{query}_Q

Here, \mathrm{query}_i is the i-th query and is given in one of the following formats:
1 u v

2 v k

Output

Let q be the number of Type 2 queries. Print q lines.
The i-th line should contain the answer to the i-th Type 2 query.

Constraints


- 1 \leq N, Q \leq 2 \times 10^5
- In a Type 1 query, 1 \leq u < v \leq N.
- In a Type 2 query, 1 \leq v \leq N, 1 \leq k \leq 10.
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to efficiently handle two types of queries on an initially empty undirected graph: adding an edge between two vertices and querying the k-th largest vertex number connected to a given vertex. The challenge lies in processing these queries quickly, especially since the graph can grow large (up to 200,000 vertices and queries), and the k-th largest query must be answered in constant time per query.

### Approach
1. **Union-Find Data Structure with Path Compression and Union by Rank**: This helps efficiently manage and merge connected components as edges are added. Path compression ensures that future queries for the same component are faster, and union by rank keeps the tree shallow, optimizing merge operations.
2. **Maintaining Top 10 Vertex Numbers per Component**: For each connected component, we maintain a list of the top 10 largest vertex numbers (sorted in descending order). This allows us to answer k-th largest queries in constant time if the component has at least k vertices; otherwise, we return -1.
3. **Handling Queries**: 
   - For edge addition queries (Type 1), we merge the connected components of the two vertices if they are not already connected. During the merge, we combine and sort the top 10 lists of both components to form a new top 10 list for the merged component.
   - For k-th largest queries (Type 2), we simply check the top 10 list of the component containing the queried vertex and return the k-th largest element if it exists.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it)); q = int(next(it))
    parent = [i for i in range(n+1)]
    rank = [0] * (n+1)
    top10 = [[] for _ in range(n+1)]
    for i in range(1, n+1):
        top10[i] = [i]

    def find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]

    def union(a, b):
        a = find(a)
        b = find(b)
        if a == b:
            return
        if rank[a] < rank[b]:
            a, b = b, a
        parent[b] = a
        combined = top10[a] + top10[b]
        combined.sort(reverse=True)
        top10[a] = combined[:10]
        if rank[a] == rank[b]:
            rank[a] += 1

    output_lines = []
    for _ in range(q):
        t = int(next(it))
        if t == 1:
            u = int(next(it)); v = int(next(it))
            if find(u) != find(v):
                union(u, v)
        else:
            v_val = int(next(it)); k = int(next(it))
            root = find(v_val)
            if len(top10[root]) < k:
                output_lines.append("-1")
            else:
                output_lines.append(str(top10[root][k-1]))
    
    print("\n".join(output_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The graph starts with `n` vertices, each initially in its own component. The `top10` array is initialized to contain just the vertex number for each vertex.
2. **Union-Find Operations**: The `find` function uses path compression to keep the tree flat, ensuring efficient future queries. The `union` function merges two components by attaching the smaller tree to the larger one (based on rank) and combines their top 10 lists, keeping only the top 10 largest vertex numbers.
3. **Query Processing**: 
   - **Type 1 Queries**: When an edge is added between two vertices, their components are merged if they are separate. The top 10 lists of both components are combined and sorted to form the new component's top 10 list.
   - **Type 2 Queries**: For a given vertex, the root of its component is found. If the component has at least `k` vertices, the k-th largest vertex number is taken from the top 10 list; otherwise, -1 is returned.
4. **Efficiency**: The union-find operations with path compression and union by rank ensure near-constant time per operation. The top 10 lists are maintained in constant size (up to 10 elements), making each merge operation efficient (O(1) per merge). This approach efficiently handles the constraints, ensuring quick query responses even for large graphs.
