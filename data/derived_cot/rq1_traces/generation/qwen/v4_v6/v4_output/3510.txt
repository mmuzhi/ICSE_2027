=== ANSWER ===

To solve this problem, we need to assign distinct positive integers to each tower such that no two towers have the same height, and each tower's height does not exceed its given maximum height. The goal is to maximize the total sum of these heights. If it's not possible to assign such heights, we should return -1.

### Approach
1. **Sort the Array**: First, we sort the array of maximum heights in descending order. This allows us to process the largest maximum heights first, which helps in assigning the largest possible distinct numbers initially.
2. **Greedy Assignment**: We start by assigning the largest possible number to the first tower, which is simply its maximum height. For each subsequent tower, we assign the largest number that is less than the previously assigned number and does not exceed the tower's maximum height. This ensures that we are always choosing the highest possible distinct number available for each tower.
3. **Check for Validity**: During the assignment process, if at any point the required number to assign becomes less than 1, it means it's impossible to assign distinct positive integers to all towers, and we return -1 immediately.
4. **Sum Calculation**: We accumulate the sum of all assigned numbers during the process and return it once all towers are processed.

### Solution Code
```python
class Solution:
    def maximumTotalSum(self, maximumHeight: List[int]) -> int:
        maximumHeight.sort(reverse=True)
        current = maximumHeight[0]
        total = current
        n = len(maximumHeight)
        for i in range(1, n):
            current = min(current - 1, maximumHeight[i])
            if current < 1:
                return -1
            total += current
        return total
```

### Explanation
1. **Sorting**: The array is sorted in descending order to prioritize assigning the largest numbers first. This helps in maximizing the sum by utilizing the highest possible values early in the process.
2. **Greedy Assignment**: The first tower is assigned its maximum height. For each subsequent tower, the assigned number is the minimum of (previous number - 1) and the tower's maximum height. This ensures the number is distinct and as large as possible without exceeding the tower's constraints.
3. **Validation Check**: If during the assignment process, the number to be assigned drops below 1, it means there are not enough distinct positive integers available, making the assignment impossible, hence returning -1.
4. **Sum Calculation**: The sum of all assigned numbers is computed and returned once all towers are processed. This sum represents the maximum possible total under the given constraints.

This approach efficiently checks for the feasibility of assigning distinct heights while maximizing the total sum, leveraging sorting and a greedy strategy to ensure optimal performance.
