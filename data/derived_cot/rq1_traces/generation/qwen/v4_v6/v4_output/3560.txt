To solve this problem, we need to determine the maximum total number of moves Alice can achieve in a game where she and Bob alternately capture pawns with a knight on a 50x50 chessboard, with both players playing optimally. The game starts with Alice, and each player selects a pawn to capture in the fewest possible moves, updating the knight's position to the captured pawn's location. The goal is to maximize the total moves (sum of moves by both players) until all pawns are captured.

### Approach
1. **Problem Analysis**: The problem involves a turn-based game on a chessboard where players alternately capture pawns with a knight. The key observation is that the minimal moves to capture each pawn from the current knight position is independent of other pawns. The challenge is to find the optimal sequence of captures that maximizes the total moves, considering both players play optimally (Alice maximizes, Bob minimizes).

2. **Intuition**: The solution involves dynamic programming (DP) with memoization. The state is defined by the set of remaining pawns (represented as a bitmask) and the current knight position. The DP function computes the optimal total moves from each state by considering all possible pawn captures, alternating between maximizing and minimizing the total moves based on the player's turn.

3. **Algorithm Selection**: 
   - **Breadth-First Search (BFS)**: For each of the 16 points (initial position and 15 pawns), compute the minimal moves to all other points using BFS. This precomputes a distance matrix for quick lookup during the DP phase.
   - **Dynamic Programming with Memoization**: The DP function uses the precomputed distance matrix to explore all possible captures, recursively calculating the optimal moves from each state. The memoization stores results of previously computed states to avoid redundant calculations.

4. **Complexity Analysis**: 
   - **BFS**: For each of the 16 points, BFS runs in O(50*50) time, leading to O(16*2500) = O(40,000) operations.
   - **DP State Space**: The state is defined by a bitmask (2^15 possible values) and a position (16 possible values), resulting in 32768 * 16 = 524,288 states. Each state processes up to 15 pawns, leading to a total of around 7.8 million operations in the worst case, which is feasible with memoization.

### Solution Code
```python
from collections import deque
from functools import lru_cache

class Solution:
    def maxMoves(self, kx: int, ky: int, positions: List[List[int]]) -> int:
        n_pawns = len(positions)
        points = [(kx, ky)] + positions
        
        if n_pawns == 0:
            return 0
        
        dist_matrix = [[0] * (n_pawns + 1) for _ in range(n_pawns + 1)]
        
        moves = [(2, 1), (1, 2), (-1, 2), (-2, 1), (-2, -1), (-1, -2), (1, -2), (2, -1)]
        
        for i in range(n_pawns + 1):
            dist_grid = [[-1] * 50 for _ in range(50)]
            q = deque()
            start_x, start_y = points[i]
            dist_grid[start_x][start_y] = 0
            q.append((start_x, start_y))
            while q:
                x, y = q.popleft()
                for dx, dy in moves:
                    nx, ny = x + dx, y + dy
                    if 0 <= nx < 50 and 0 <= ny < 50 and dist_grid[nx][ny] == -1:
                        dist_grid[nx][ny] = dist_grid[x][y] + 1
                        q.append((nx, ny))
            for j in range(n_pawns + 1):
                x_j, y_j = points[j]
                dist_matrix[i][j] = dist_grid[x_j][y_j]
        
        @lru_cache(maxsize=None)
        def dp(mask, pos):
            if mask == 0:
                return 0
            count = bin(mask).count("1")
            captures_done = n_pawns - count
            if captures_done % 2 == 0:
                best = -1
                for j in range(n_pawns):
                    if mask & (1 << j):
                        new_mask = mask ^ (1 << j)
                        cost = dist_matrix[pos][j + 1]
                        total_moves = cost + dp(new_mask, j + 1)
                        if best == -1 or total_moves > best:
                            best = total_moves
                return best
            else:
                best = float('inf')
                for j in range(n_pawns):
                    if mask & (1 << j):
                        new_mask = mask ^ (1 << j)
                        cost = dist_matrix[pos][j + 1]
                        total_moves = cost + dp(new_mask, j + 1)
                        if total_moves < best:
                            best = total_moves
                return best
        
        return dp((1 << n_pawns) - 1, 0)
```

### Explanation
1. **Initialization**: The initial position of the knight and the positions of the pawns are stored in a list. If there are no pawns, the function returns 0 immediately.
2. **Distance Matrix Calculation**: Using BFS, the minimal moves between each pair of the 16 points (initial position and 15 pawns) are precomputed and stored in a distance matrix. This matrix allows quick lookup of minimal moves during the DP phase.
3. **Dynamic Programming Function**: The `dp` function is defined with memoization to store results of subproblems. It takes the current state (bitmask of remaining pawns and knight position) and computes the optimal total moves:
   - **Base Case**: If no pawns remain, return 0.
   - **Player Turn**: Depending on the number of captures done (even for Alice, odd for Bob), the function either maximizes or minimizes the total moves by considering all possible pawn captures.
4. **Result Calculation**: The initial state (all pawns present, knight at starting position) is passed to the `dp` function, which returns the maximum total moves Alice can achieve with both players playing optimally.
