A sequence X = (X_1, X_2, \ldots) of positive integers (possibly empty) is called a 1122 sequence if and only if it satisfies all of the following three conditions: (The definition of a 1122 sequence is the same as in Problem F.)

- \lvert X \rvert is even. Here, \lvert X \rvert denotes the length of X.
- For each integer i satisfying 1\leq i\leq \frac{|X|}{2}, X_{2i-1} and X_{2i} are equal.
- Each positive integer appears in X either not at all or exactly twice. That is, every positive integer contained in X appears exactly twice in X.

Given a sequence A = (A_1, A_2, \ldots, A_N) of length N consisting of positive integers, print the maximum length of a (contiguous) subarray of A that is a 1122 sequence.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the maximum length of a (contiguous) subarray of A that is a 1122 sequence.

Constraints


- 1\leq N \leq 2 \times 10^5
- 1\leq A_i \leq N
- All input values are integers.

Sample Input 1

8
2 3 1 1 2 2 1 1

Sample Output 1

4

There is no longer (contiguous) subarray that satisfies the conditions for a 1122 sequence, so the answer is 4.

Sample Input 3

### Approach
1. **Problem Analysis**: The key observation is that a valid subarray must:
   - Have an even length.
   - Be composed of consecutive pairs where each pair consists of two identical elements.
   - Ensure that each distinct element appears exactly twice in the entire subarray.

2. **Intuition**: For small input sizes (n ≤ 1000), a brute-force approach is feasible. We can check all possible even-length subarrays and verify the conditions. However, for larger inputs, this approach would be too slow (O(n^2)), so we need an optimized solution.

3. **Optimization**: For large inputs (n > 1000), we use a two-pointer technique combined with a frequency dictionary to efficiently check subarrays. The two-pointer approach helps in maintaining a sliding window that meets the conditions, reducing the number of checks needed.

4. **Complexity Analysis**:
   - **Brute-force approach (n ≤ 1000)**: O(n^2) time complexity, which is acceptable for small n.
   - **Two-pointer approach (n > 1000)**: O(n) time complexity, which is efficient for large n. The frequency dictionary allows us to track element counts in constant time, and the two-pointer ensures we only process relevant subarrays.

import sys
from collections import defaultdict

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    if n <= 1000:
        max_len = 0
        for i in range(n):
            for j in range(i, n):
                length = j - i + 1
                if length % 2 != 0:
                    continue
                valid = True
                for k in range(i, j, 2):
                    if A[k] != A[k+1]:
                        valid = False
                        break
                if not valid:
                    continue
                freq = defaultdict(int)
                for k in range(i, j+1):
                    freq[A[k]] += 1
                good = True
                for count in freq.values():
                    if count != 2:
                        good = False
                        break
                if good:
                    if length > max_len:
                        max_len = length

    else:
        max_len = 0
        freq = defaultdict(int)
        left = 0
        for right in range(n):
            freq[A[right]] += 1

                if (right - left + 1) % 2 != 0:
                    left += 1
                else:

                    for k in range(left, right, 2):
