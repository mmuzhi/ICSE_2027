To solve this problem, we need to compute the sum of scores of all non-empty subsequences for each prefix of a given sequence, where the score of a subsequence is defined as the sum of the greatest common divisors (gcd) of all consecutive elements in the subsequence. The challenge lies in efficiently calculating this for large sequences without resorting to brute-force methods.

### Approach
1. **Problem Analysis**: The problem requires summing the scores of all non-empty subsequences for each prefix of the input sequence. The score of a subsequence is the sum of gcd values between consecutive elements. Given the constraints (N up to 500,000), a brute-force approach is infeasible.

2. **Key Insight**: For each pair of elements (i, j) where i < j, the gcd(A_i, A_j) contributes to the score of any subsequence that includes both elements consecutively. The number of such subsequences is determined by the positions of elements before i and after j, which can be computed using powers of 2.

3. **Mathematical Formulation**: The total score for a prefix of length m can be broken down into contributions from all pairs (i, j) where i < j ≤ m. The contribution of each pair is gcd(A_i, A_j) multiplied by 2^(i-1) * 2^(m-j). 

4. **Efficient Calculation**: 
   - **Precompute Divisors and Euler Totient Function (φ)**: For each number up to the maximum value in the sequence, precompute its divisors and φ values. This helps in leveraging the identity that the gcd of two numbers can be expressed as the sum of φ values over their common divisors.
   - **Dynamic Updates**: For each element in the sequence, update a data structure that keeps track of the sum of 2^(i-1) for all previous elements divisible by each divisor. This allows efficient computation of the contribution of each element to the total score.

5. **Modulo Handling**: Since the results can be very large, all calculations are performed modulo 998244353.

### Solution Code
```python
import sys

mod = 998244353
max_val = 100000
max_n = 500000

# Precompute divisors for 1 to max_val
divisors = [[] for _ in range(max_val+1)]
for i in range(1, max_val+1):
    for j in range(i, max_val+1, i):
        divisors[j].append(i)

# Precompute phi for 1 to max_val
phi = list(range(max_val+1))
for i in range(2, max_val+1):
    if phi[i] == i:
        for j in range(i, max_val+1, i):
            phi[j] -= phi[j] // i

# Precompute powers of 2 up to max_n
pow2 = [1] * (max_n+1)
for i in range(1, max_n+1):
    pow2[i] = (pow2[i-1] * 2) % mod

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    T = 0
    div_sum = [0] * (max_val+1)
    
    out_lines = []
    for j in range(1, n+1):
        x = A[j-1]
        S_j = 0
        for d in divisors[x]:
            if d <= max_val:
                S_j = (S_j + phi[d] * div_sum[d]) % mod
        
        T = (T * 2 + S_j) % mod
        out_lines.append(str(T))
        
        for d in divisors[x]:
            if d <= max_val:
                div_sum[d] = (div_sum[d] + pow2[j-1]) % mod

    print("\n".join(out_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Precomputation**: 
   - **Divisors and φ Values**: For each number up to 100,000, we precompute its divisors and φ values. This allows us to efficiently use the identity that the gcd of two numbers can be expressed as the sum of φ values over their common divisors.
   - **Powers of 2**: We precompute powers of 2 modulo 998244353 up to 500,000 to handle large exponents efficiently during the main computation.

2. **Main Loop**:
   - For each element in the sequence, we calculate its contribution to the total score by summing the φ values of its divisors multiplied by the sum of 2^(i-1) for all previous elements divisible by each divisor.
   - The total score for the current prefix is updated by doubling the previous total (to account for the new position) and adding the current element's contribution.
   - After processing each element, we update the data structure (div_sum) to include the current element's contributions for all its divisors.

3. **Output**: After processing each prefix, the current total score is stored and printed at the end.

This approach efficiently computes the required scores by leveraging mathematical insights and dynamic updates, ensuring optimal performance even for large input sizes.
