To solve this problem, we need to determine the minimum number of operations required to make an array non-decreasing by performing operations on its elements. Each operation allows selecting an element and dividing it by its greatest proper divisor. The solution involves analyzing each element's possible values after any number of operations and ensuring the sequence of chosen values is non-decreasing while minimizing the total operations.

### Approach
1. **Problem Analysis**: The key observation is that each element can be reduced to a limited set of values (its original value or its smallest prime factor if it's composite). The array must be non-decreasing, meaning each subsequent element's chosen value must be greater than or equal to the previous one.

2. **Intuition**: For each element, generate all possible values it can take after any number of operations (either the original value or its smallest prime factor if composite). Then, use dynamic programming with a segment tree to efficiently compute the minimum operations required to form a non-decreasing sequence by considering previous elements' valid values.

3. **Algorithm Selection**: 
   - **Precompute Smallest Prime Factors (SPF)**: Use the Sieve of Eratosthenes to precompute the smallest prime factor for all numbers up to the maximum value in the input array.
   - **Generate Candidate Values**: For each element, determine its candidate values (original value if prime or 1, or original and smallest prime factor if composite).
   - **Coordinate Compression**: Compress the candidate values to a smaller range for efficient segment tree operations.
   - **Segment Tree for Range Minimum Queries**: Use a segment tree to keep track of the minimum operations required for each candidate value from previous elements, allowing efficient range minimum queries to ensure the non-decreasing property.

4. **Complexity Analysis**: 
   - **SPF Computation**: O(N log log N) for numbers up to 10^6.
   - **Candidate Generation**: O(N) for each element, with each element having at most 2 candidates.
   - **Coordinate Compression**: O(N log N) for sorting and mapping candidate values.
   - **Segment Tree Operations**: Each update and query is O(log M), where M is the number of distinct candidate values (up to 200,000). With N elements, the total complexity is O(N log M).

import math
from typing import List

class SegmentTree:
    def __init__(self, size, init_val=10**18):
        self.n = size
        self.size = 1
        while self.size < self.n:
            self.size *= 2
        self.tree = [init_val] * (2 * self.size)
    
    def update(self, index, value):
        index += self.size
        self.tree[index] = min(self.tree[index], value)
        while index > 1:
            index //= 2
            self.tree[index] = min(self.tree[2*index], self.tree[2*index+1])
    
    def query(self, l, r):
        if l > r:
            return 10**18
        l += self.size
        r += self.size
        res = 10**18
        while l <= r:
            if l % 2 == 1:
                res = min(res, self.tree[l])
                l += 1
            if r % 2 == 0:
                res = min(res, self.tree[r])
                r -= 1
            l //= 2
            r //= 2
        return res

class Solution:
    def minOperations(self, nums: List[int]) -> int:
        max_val = max(nums) if nums else 0
        if max_val == 0:
            return 0
        
        spf = list(range(max_val + 1))
        for i in range(2, int(math.isqrt(max_val)) + 1):
            if spf[i] == i:
                for j in range(i*i, max_val+1, i):
                    if spf[j] == j:
                        spf[j] = i
        
        candidates = set()
        for num in nums:
            if num == 1:
                candidates.add(1)
            else:
                if spf[num] == num:
                    candidates.add(num)
                else:
                    candidates.add(num)
                    candidates.add(spf[num])
        
        sorted_vals = sorted(candidates)
        mapping = {val: idx+1 for idx, val in enumerate(sorted_vals)}
        M_size = len(sorted_vals)
        
        seg_tree = SegmentTree(M_size, 10**18)
        
        for num in nums:
            if num == 1:
                cand_set = {1}
            else:
                if spf[num] == num:
                    cand_set = {num}
                else:
                    cand_set = {num, spf[num]}
            for cand in cand_set:
                cost = 0 if cand == num else 1
                idx = mapping[cand]
                seg_tree.update(idx, cost)
        
        n = len(nums)
        for i in range(1, n):
            num = nums[i]
            if num == 1:
                cand_set = {1}
            else:
                if spf[num] == num:
                    cand_set = {num}
                else:
                    cand_set = {num, spf[num]}
            new_values = {}
            for cand in cand_set:
                cost = 0 if cand == num else 1
                idx = mapping[cand]
                min_prev = seg_tree.query(1, idx)
                if min_prev < 10**18:
                    new_values[idx] = min_prev + cost
            for idx, val in new_values.items():
                seg_tree.update(idx, val)
        
        ans = seg_tree.query(1, M_size)
        return ans if ans < 10**18 else -1

2. **Candidate Generation**: For each number in the array, generate its candidate values (original value if prime or 1, or original and smallest prime factor if composite). These candidates represent all possible values the number can be reduced to.
3. **Coordinate Compression**: The distinct candidate values are sorted and mapped to a compressed range to facilitate efficient segment tree operations.
4. **Segment Tree Operations**: The segment tree is used to keep track of the minimum operations required for each candidate value from previous elements. 5. **Result Extraction**: After processing all elements, the minimum value in the segment tree gives the answer. If no valid sequence is found, return -1.
