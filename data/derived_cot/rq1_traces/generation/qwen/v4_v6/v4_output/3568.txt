Initially, if any number has less than four digits, it is padded with leading zeros.
The i^th digit (1 <= i <= 4) of the key is generated by taking the smallest digit among the i^th digits of num1, num2, and num3.

Return the key of the three numbers without leading zeros (if any).
 
Example 1:

Input: num1 = 1, num2 = 10, num3 = 1000
Output: 0
Explanation:
On padding, num1 becomes "0001", num2 becomes "0010", and num3 remains "1000".

The 1^st digit of the key is min(0, 0, 1).
The 2^nd digit of the key is min(0, 0, 0).
The 3^rd digit of the key is min(0, 1, 0).
The 4^th digit of the key is min(1, 0, 0).

Hence, the key is "0000", i.e. 0.

Example 2:

Input: num1 = 987, num2 = 879, num3 = 798
Output: 777

Example 3:

Input: num1 = 1, num2 = 2, num3 = 3
Output: 1

=== ANSWER ===

To solve this problem, we need to generate a four-digit key by taking the smallest digit from each corresponding position of three given numbers. The numbers are first padded with leading zeros to ensure they are all four digits long. The key is then formed by concatenating the smallest digits from each position and converting the resulting string to an integer to remove any leading zeros.

### Approach
1. **Convert Numbers to Strings**: Each of the three numbers is converted into a string to facilitate digit-by-digit processing.
2. **Pad with Leading Zeros**: Each string is padded with leading zeros to ensure it has exactly four digits. This step ensures that all numbers are of the same length, making it easier to compare digits at each position.
3. **Find Minimum Digits**: For each of the four positions (from left to right), the smallest digit among the three numbers is selected.
4. **Construct the Key**: The selected digits are combined into a single string, which is then converted to an integer. This conversion automatically removes any leading zeros, resulting in the desired key.

### Solution Code
```python
class Solution:
    def generateKey(self, num1: int, num2: int, num3: int) -> int:
        s1 = str(num1).zfill(4)
        s2 = str(num2).zfill(4)
        s3 = str(num3).zfill(4)
        
        key_str = ''.join(str(min(int(s1[i]), int(s2[i]), int(s3[i]))) for i in range(4))
        
        return int(key_str)
```

### Explanation
1. **Conversion and Padding**: The numbers `num1`, `num2`, and `num3` are converted to strings and then padded with leading zeros using `zfill(4)` to ensure each string is exactly four characters long.
2. **Digit-wise Comparison**: For each of the four positions (indices 0 to 3), the smallest digit among the three strings is found by converting each character to an integer, comparing them, and then converting the result back to a string.
3. **Key Construction**: The smallest digits from each position are concatenated into a single string. This string is then converted to an integer, which removes any leading zeros, providing the final key.

For example, given the inputs `num1 = 1`, `num2 = 10`, and `num3 = 1000`, the strings become "0001", "0010", and "1000" respectively. The smallest digits at each position are '0', '0', '0', and '0', forming the key "0000", which converts to the integer 0. Similarly, for inputs like `num1 = 987`, `num2 = 879`, and `num3 = 798`, the key is formed by taking the minimum digits from each position, resulting in "777". 

This approach efficiently handles the problem by leveraging string manipulation and simple arithmetic operations to generate the desired key.
