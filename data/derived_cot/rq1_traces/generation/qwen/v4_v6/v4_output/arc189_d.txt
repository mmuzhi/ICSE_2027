You will be given a competitive programming problem.
There are N slimes lined up in a row from left to right.
For i = 1, 2, \ldots, N, the i-th slime from the left has size A_i.
For each K = 1, 2, \ldots, N, solve the following problem.

Takahashi is the K-th slime from the left in the initial state.
Find the maximum size that he can have after performing the following action any number of times, possibly zero:

- Choose a slime adjacent to him that is strictly smaller than him, and absorb it.
As a result, the absorbed slime disappears, and Takahashi's size increases by the size of the absorbed slime.

When a slime disappears due to absorption, the gap is immediately closed, and the slimes that were adjacent to the disappearing slime (if they exist) become adjacent (see the explanation in Sample Input 1).

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the answers B_K for each K = 1, 2, \ldots, N, separated by spaces, in the following format:
B_1 B_2 \ldots B_N

Constraints


- 2 \leq N \leq 5 \times 10^5
- 1 \leq A_i \leq 10^9
- All input values are integers.

Sample Input 1

6
4 13 2 3 2 6

Sample Output 1

4 30 2 13 2 13

- He absorbs the slime to his right. As a result, the absorbed slime disappears, and his size becomes 3 + 2 = 5. The state becomes (4, 13, 2, [5], 6).

There are no slimes adjacent to him that are strictly smaller than him, so he cannot perform any more actions. His final size is 13, which is the maximum possible.

To solve this problem, we need to determine the maximum size a slime can achieve by absorbing adjacent slimes that are strictly smaller than it. The absorption process can be performed any number of times, and the goal is to find the maximum possible size for each slime when it starts as the K-th slime in the initial array.

### Approach
1. **Problem Analysis**: The problem involves simulating the absorption process for each slime in the array. The key observation is that the absorption can be done in any order (left or right first), and the final size depends on the sum of the starting slime's size and all the sizes of the absorbed slimes. The challenge is to efficiently compute this for each slime in the array without resorting to a brute-force approach, which would be computationally expensive for large arrays.

2. **Insight**: The maximum size a slime can achieve is the sum of its own size and all the sizes of slimes it can absorb from both directions (left and right). The absorption is possible only if the adjacent slime is strictly smaller than the current size. The order of absorption (left or right first) does not affect the final size, as demonstrated by sample cases.

3. **Algorithm Selection**: For small arrays (N ≤ 10), we use a BFS approach to explore all possible absorption sequences, tracking the current position and the set of absorbed slimes. For larger arrays, we use a two-pass method where we calculate the maximum absorption from the left and right separately for each slime. The final size is derived from the sum of the starting slime's size and the absorbed sizes from both directions.

4. **Complexity Analysis**: The BFS approach for small arrays has a time complexity of O(N * 2^N) in the worst case, which is feasible for N ≤ 10. For larger arrays, the two-pass method operates in O(N^2) time due to nested loops for each slime, but optimizations like precomputing barriers (next greater or equal elements) can reduce this to O(N log N) in practice.

                    new_absorbed = absorbed | {right_pos}
                    new_state = (right_pos, frozenset(new_absorbed))
                    if new_state not in visited:
                        visited[new_state] = current_size + A[right_pos]
                        queue.append((right_pos, new_absorbed, current_size + A[right_pos]))

            if stack:
                left_barrier[i] = stack[-1]

### Explanation
1. **BFS for Small Arrays**: For arrays with 10 or fewer elements, the solution uses a BFS approach to explore all possible absorption sequences. Each state in the BFS is defined by the current position and the set of absorbed slimes. The BFS ensures that all possible absorption paths are considered, and the maximum size encountered during the BFS is recorded.

2. **Two-Pass for Large Arrays**: For larger arrays, the solution first computes the barriers (next greater or equal elements) for each slime on both sides. Using these barriers, it calculates the sum of all slimes that can be absorbed (those strictly smaller than the current size) in the segments between the barriers. The final size for each slime is derived by summing its own size and the absorbed sizes from both directions, adjusting for double-counting its own size.

This approach efficiently handles both small and large input sizes by leveraging BFS for small arrays and a two-pass method with barrier computation for larger arrays, ensuring optimal performance and correctness.
