To solve this problem, we need to determine if a given string consisting of various types of brackets can be reduced to an empty string by repeatedly deleting any contiguous substring that matches one of the specified bracket pairs: "()", "[]", or "<>". The solution involves checking if the string is a valid sequence of balanced brackets, considering all three types.

### Approach
1. **Check String Length**: The string must have an even length because each deletion operation removes exactly two characters. If the string length is odd, it's immediately impossible to reduce it to an empty string.
2. **Stack-Based Validation**: Use a stack to keep track of opening brackets. For each closing bracket encountered, check if it matches the top of the stack. If it does, pop the stack; otherwise, the string is invalid.
3. **Matching Brackets**: Maintain a dictionary to map each closing bracket to its corresponding opening bracket for quick validation.
4. **Final Check**: After processing all characters, if the stack is empty, the string is a colorful bracket sequence; otherwise, it is not.

### Solution Code
```python
def main():
    S = input().strip()
    n = len(S)
    if n % 2 != 0:
        print("No")
        return

    stack = []
    reverse_matching = {'(': ')', '[': ']', '<': '>'}
    reverse_dict = {')': '(', ']': '[', '>': '<'}

    for c in S:
        if c in '([{<':
            stack.append(c)
        else:
            if not stack:
                print("No")
                return
            if reverse_dict[c] != stack[-1]:
                print("No")
                return
            stack.pop()

    if stack:
        print("No")
    else:
        print("Yes")

if __name__ == "__main__":
    main()
```

2. **Stack Processing**: The algorithm processes each character in the string:
   - **Opening Brackets**: When an opening bracket is encountered, it is pushed onto the stack.
   - **Closing Brackets**: For a closing bracket, the algorithm checks if the top of the stack matches the corresponding opening bracket. If not, the string is invalid. If it matches, the top of the stack is popped.
3. **Final Validation**: After processing all characters, if the stack is empty, all brackets were properly matched and the string can be reduced to an empty string, so it prints "Yes". Otherwise, it prints "No".

This approach efficiently checks the validity of the bracket sequence by leveraging stack operations and a dictionary for quick lookups, ensuring optimal performance even for large input sizes.
