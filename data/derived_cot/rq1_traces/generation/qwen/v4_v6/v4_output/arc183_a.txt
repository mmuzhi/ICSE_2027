Here, \operatorname{floor}(x) represents the largest integer not exceeding x.
 What is lexicographical order for sequences?
A sequence S = (S_1,S_2,\ldots,S_{|S|}) is lexicographically smaller than a sequence T = (T_1,T_2,\ldots,T_{|T|}) if either 1. or 2. below holds.
Here, |S| and |T| represent the lengths of S and T, respectively.

-  |S| \lt |T| and (S_1,S_2,\ldots,S_{|S|}) = (T_1,T_2,\ldots,T_{|S|}). 
-  There exists an integer 1 \leq i \leq \min\lbrace |S|, |T| \rbrace such that both of the following hold:

-  (S_1,S_2,\ldots,S_{i-1}) = (T_1,T_2,\ldots,T_{i-1})
-  S_i is (numerically) smaller than T_i.

The solution involves generating the sequence by iteratively selecting the next element based on the remaining counts of each number and the current rank.

### Approach
1. **Problem Analysis**: The problem requires generating the median sequence (when sequences are ordered lexicographically) of all possible sequences of length NK where each number from 1 to N appears exactly K times. Given the constraints (N and K up to 500), a brute-force approach is infeasible due to the enormous number of sequences.

2. **Key Insight**: The solution leverages combinatorial mathematics to determine the next element in the sequence at each step. For each candidate number, the number of sequences starting with that number is calculated using logarithmic values to avoid overflow. The rank is adjusted by subtracting the counts of sequences starting with smaller numbers until the correct candidate is found.

3. **Algorithm Selection**: The algorithm uses a priority-based selection where the candidate number with the largest remaining count is chosen if it contains the current rank. The counts are computed using logarithmic values to handle large numbers efficiently.

4. **Complexity Analysis**: The algorithm processes each element of the sequence (NK elements) and for each element, it checks up to N candidates. Each candidate check involves logarithmic computations, making the overall complexity manageable for the given constraints.

### Solution Code
```python
import math
from math import lgamma

def main():
    import sys
    data = sys.stdin.read().split()
    N = int(data[0])
    K = int(data[1])
    total_length = N * K
    
    # If there's only one number, output that number K times.
    if N == 1:
        print(" ".join([str(1)] * K))
        return
        
    max_n = total_length
    log_fact = [0.0] * (max_n + 1)
    for i in range(1, max_n + 1):
        log_fact[i] = lgamma(i + 1)
        
    def count_sequences(counts, candidate_index):
        L = sum(counts)
        if L == 0:
            return 0.0
        c_i = counts[candidate_index]
        if c_i == 0:
            return 0.0
        S_log = sum(log_fact[c] for c in counts)
        log_count_i = log_fact[L - 1] + math.log(c_i) - S_log
        return log_count_i

    def find_sequence(counts, rank):
        result = []
        L = total_length
        current_counts = counts[:]
        while L > 0:
            for i in range(len(current_counts)):
                if current_counts[i] == 0:
                    continue
                log_count_i = count_sequences(current_counts, i)
                count_i = math.exp(log_count_i) if math.isfinite(log_count_i) else float('inf')
                if rank <= count_i:
                    result.append(i + 1)
                    current_counts[i] -= 1
                    L -= 1
                    break
                else:
                    rank -= count_i
        return result

    initial_rank = (total_length) // 2 + 1
    counts = [K] * N
    sequence = find_sequence(counts, initial_rank)
    print(" ".join(map(str, sequence)))

if __name__ == '__main__':
    main()
```
