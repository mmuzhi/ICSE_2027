You will be given a competitive programming problem.
There is a grid of H \times W cells. Let (i, j) denote the cell at the i-th row from the top and the j-th column from the left.
Cell (i, j) is empty if S_{i,j} is ., and blocked if it is #.
Count the number of ways to start from an empty cell and make K moves to adjacent cells (up, down, left, or right), without passing through blocked squares and not visiting the same cell more than once.
Specifically, count the number of sequences of length K+1, ((i_0, j_0), (i_1, j_1), \dots, (i_K, j_K)), satisfying the following.

- 1 \leq i_k \leq H, 1 \leq j_k \leq W, and S_{i_k, j_k} is ., for each 0 \leq k \leq K.
- |i_{k+1} - i_k| + |j_{k+1} - j_k| = 1 for each 0 \leq k \leq K-1.
- (i_k, j_k) \neq (i_l, j_l) for each 0 \leq k < l \leq K.

Input

The input is given from Standard Input in the following format:
H W K
S_{1,1}S_{1,2}\dots S_{1,W}
S_{2,1}S_{2,2}\dots S_{2,W}
\vdots
S_{H,1}S_{H,2}\dots S_{H,W}

Output

Print the answer.

Constraints


- 1 \leq H, W \leq 10
- 1 \leq K \leq 11
- H, W, and K are integers.
- Each S_{i,j} is . or #.
- There is at least one empty cell.

Sample Input 1

.#
..

Sample Output 1

2

=== ANSWER ===

To solve this problem, we need to count the number of ways to start from an empty cell in a grid and make exactly K moves to adjacent cells (up, down, left, or right) without passing through blocked cells or revisiting any cell more than once. The solution involves using a depth-first search (DFS) approach with memoization to efficiently explore all possible paths of length K+1 (K moves) from each empty cell.

### Approach
1. **Problem Analysis**: The problem requires counting all valid sequences of cells of length K+1 starting from any empty cell, where each subsequent cell is adjacent and not blocked, and no cell is revisited. The grid dimensions are up to 10x10, and K is up to 11, making a brute-force approach infeasible without optimizations.

2. **Intuition**: Given the constraints, a DFS with memoization is suitable. The memoization stores the number of valid paths from a given cell with a specific set of visited cells (represented as a bitmask) to avoid redundant calculations.

3. **Algorithm Selection**: 
   - **DFS with Memoization**: For each empty cell, initiate a DFS that tracks the current position, the number of moves taken, and a bitmask representing visited cells. The memoization cache stores the number of valid paths from each state (position and visited set) to avoid recalculating the same state multiple times.
   - **Bitmask Representation**: Each cell in the grid is represented by a bit in an integer, allowing efficient tracking of visited cells and quick checks for adjacency and blockage.

4. **Complexity Analysis**: The worst-case scenario involves exploring all possible paths of length K+1 from each empty cell. With K up to 11 and grid size 10x10, the number of states per starting cell is manageable (up to around 700,000 for a fully empty grid), and the memoization ensures each state is processed only once.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data: 
        print(0)
        return
    parts = data[0].split()
    H = int(parts[0])
    W = int(parts[1])
    K = int(parts[2])
    grid = []
    for i in range(1, 1+H):
        grid.append(data[i].strip())
    
    if K == 0:
        count = 0
        for i in range(H):
            for j in range(W):
                if grid[i][j] == '.':
                    count += 1
        print(count)
        return

    def dfs(i, j, mask, steps, memo, grid, H, W, K):
        if steps == K:
            return 1
        key = (i, j, mask)
        if key in memo:
            return memo[key]
        total = 0
        for di, dj in [(1,0), (-1,0), (0,1), (0,-1)]:
            ni, nj = i + di, j + dj
            if 0 <= ni < H and 0 <= nj < W and grid[ni][nj] == '.' and not (mask >> (ni*W + nj) & 1):
                new_mask = mask | (1 << (ni*W + nj))
                total += dfs(ni, nj, new_mask, steps+1, memo, grid, H, W, K)
        memo[key] = total
        return total

    total_count = 0
    for i in range(H):
        for j in range(W):
            if grid[i][j] == '.':
                memo = {}
                mask0 = 1 << (i*W + j)
                count = dfs(i, j, mask0, 0, memo, grid, H, W, K)
                total_count += count
    print(total_count)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read from standard input, parsing the grid dimensions (H, W), the number of moves (K), and the grid itself.
2. **Special Case Handling for K=0**: If no moves are required, the solution simply counts all empty cells in the grid.
3. **DFS with Memoization**: The `dfs` function recursively explores all valid paths from a given cell, using a bitmask to track visited cells. The memoization cache (`memo`) stores the number of valid paths from each state (current cell and visited set) to avoid redundant calculations.
4. **Bitmask Handling**: Each cell is represented by a bit in an integer. The bitmask is updated as cells are visited, and checks for adjacency and blockage are performed using bitwise operations.
5. **Iterating Over Starting Cells**: For each empty cell in the grid, the DFS is initiated with an initial bitmask set to include that cell. The results from all starting cells are summed to produce the final count of valid paths.

This approach efficiently explores all possible paths while leveraging memoization to optimize performance, making it suitable for the given problem constraints.
