There is a grid with H rows and W columns.
Let (i,j) denote the cell at the i-th row from the top and the j-th column from the left.
There are N horizontal bars numbered from 1 to N placed on the grid.
Bar i consists of L_i blocks of size 1 \times 1 connected horizontally, and its leftmost block is initially at cell (R_i, C_i).
That is, initially, bar i occupies the cells (R_i, C_i), (R_i, C_i + 1), \dots, (R_i, C_i + L_i - 1).
It is guaranteed that there is no cell occupied by two different bars.
The current time is t = 0.
At every time t = 0.5 + n for some non-negative integer n, the following occurs in order of i = 1, 2, \dots, N:

- If bar i is not on the bottom row (the H-th row), and none of the cells directly below the cells occupied by bar i is occupied by any bar, then bar i moves down by one cell. - Otherwise, nothing happens.

Let (R'_i, C_i), (R'_i, C_i + 1), \dots, (R'_i, C_i + L_i - 1) be the cells occupied by bar i at time t = 10^{100}. Find R'_1, R'_2, \dots, R'_N.

To solve this problem, we need to simulate the movement of bars on a grid over time until they reach their final positions. The bars move down one cell at each time step if the cells directly below them are free of any other bars. The challenge is to determine the final positions of all bars after an extremely large number of time steps (10^100), which implies that we need an efficient way to simulate the movement without iterating through each time step individually.

### Approach
1. **Problem Analysis**: The problem involves simulating the movement of multiple bars on a grid. Each bar moves down one cell per time step if the cells directly below it are unoccupied. The movement is processed in a specific order (from bar 1 to bar N) at each time step. The goal is to find the final positions of all bars after all possible movements.

2. **Intuition**: The key observation is that each bar will move as far down as possible until it encounters a bar or the bottom row. The movement of one bar can affect the movement of subsequent bars in the same time step, so we need to simulate the movement step by step, updating the grid after each bar's movement.

3. **Algorithm Selection**: We use a grid representation where each row contains a set of columns that are currently occupied by any bar. For each bar, we check if the cells directly below it are free. If they are, we move the bar down and update the grid. This process repeats until no bar can move further down.

4. **Optimization**: To efficiently check if the cells below a bar are free, we use binary search on a sorted list of occupied columns for the next row. This reduces the time complexity of each check from O(L) to O(log W), where L is the length of the bar and W is the width of the grid.

5. **Complexity Analysis**: The worst-case time complexity is O(N * M * log W), where N is the number of bars and M is the maximum number of moves a bar can make (which is up to H, the number of rows). Given the constraints (H, W, N ≤ 2×10^5), this approach may not be feasible for large inputs, but it works well for smaller cases.

        for col in range(c, c+l):
            grid[r].add(col)

    while moved:
