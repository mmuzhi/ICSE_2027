You are given N pairs of integers (L_1, R_1), (L_2, R_2), \ldots, (L_N, R_N).
Based on these pairs, consider an undirected graph G defined as follows:

- It has N vertices numbered 1, 2, \ldots, N.
- For all i, j \in [1, N], there is an undirected edge between vertices i and j if and only if the intersection of [L_i, R_i] and [L_j, R_j] is empty.

In addition, for each i = 1, 2, \ldots, N, define the weight of vertex i to be W_i.
You are given Q queries about G. Process these queries in the order they are given.
For each i = 1, 2, \ldots, Q, the i-th query is the following:

You are given integers s_i and t_i (both between 1 and N, inclusive) such that s_i \neq t_i. Determine whether there exists a path from vertex s_i to vertex t_i in G. If it exists, print the minimum possible weight of such a path.

Here, the weight of a path from vertex s to vertex t is defined as the sum of the weights of the vertices on that path (including both endpoints s and t).

Output

Constraints


- 2 \leq N \leq 2 \times 10^5
- 1 \leq Q \leq 2 \times 10^5
- 1 \leq W_i \leq 10^9
- 1 \leq L_i \leq R_i \leq 2N
- 1 \leq s_i, t_i \leq N
- s_i \neq t_i
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to determine the minimum weight path between two vertices in a graph where edges exist between vertices if their corresponding intervals do not overlap. The weight of a path is the sum of the weights of the vertices on that path. 

### Approach
1. **Graph Construction**: For each pair of vertices, check if their intervals are disjoint. If they are, add an undirected edge between them. This step is feasible for small N (up to 1000) using nested loops, but for larger N, this approach would be too slow.
2. **Transformed Graph**: To handle the vertex weights in the path calculation, transform the original graph into a new graph where each vertex is split into two nodes (entrance and exit). The entrance node connects to the exit node with the vertex's weight, and edges between vertices in the original graph are represented as connections between exit and entrance nodes of respective vertices with zero weight.
3. **Shortest Path Calculation**: Use the Floyd-Warshall algorithm on the transformed graph to compute the shortest path between all pairs of vertices. This algorithm efficiently handles the shortest path calculation in graphs with both vertex and edge weights.
4. **Query Processing**: For each query, use the precomputed shortest path matrix to quickly determine if a path exists between the given vertices and, if so, return the minimum weight.

```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return

    Q = int(next(it))

        queries.append((s, t))

        sys.stdout.write("\n".join(out_lines))
        return

    for i in range(N):
        for j in range(i+1, N):
            L1, R1 = intervals[i]
            L2, R2 = intervals[j]
            if R1 < L2 or R2 < L1:
                graph[i].append(j)
                graph[j].append(i)

            new_graph[2*i+1][2*j] = 0

            for j in range(size):

                    new_graph[i][j] = new_graph[i][k] + new_graph[k][j]
```
