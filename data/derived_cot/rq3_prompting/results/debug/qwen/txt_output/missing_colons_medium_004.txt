=== RAW ANSWER ===

To solve this problem, we need to compute the minimum edit distance between two strings, `word1` and `word2`. The edit distance is defined as the minimum number of operations required to transform one string into another, where each operation can be a deletion, insertion, or substitution of a character.

### Approach
The approach involves using dynamic programming to build a 2D table `dp` where `dp[i][j]` represents the minimum edit distance between the first `i` characters of `word1` and the first `j` characters of `word2`. The solution involves the following steps:

1. **Initialization**: 
   - The base cases are set where converting an empty string to the other string requires either deletions or insertions. Specifically, `dp[i][0] = i` (deleting all characters of `word1`) and `dp[0][j] = j` (inserting all characters of `word2`).

2. **Filling the DP Table**:
   - For each character in both strings, if the characters match, the edit distance remains the same as the diagonal value (`dp[i-1][j-1]`). If they don't match, we consider three operations:
     - **Deletion**: Remove the character from `word1` (cost `dp[i-1][j] + 1`).
     - **Insertion**: Add the character from `word2` to `word1` (cost `dp[i][j-1] + 1`).
     - **Substitution**: Replace the character in `word1` with the corresponding character in `word2` (cost `dp[i-1][j-1] + 1`).
   - The value `dp[i][j]` is the minimum of these three operations.

3. **Result Extraction**: The value `dp[m][n]` (where `m` and `n` are the lengths of `word1` and `word2`, respectively) gives the minimum edit distance.

### Solution Code
```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m = len(word1)
        n = len(word2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        
        for i in range(m + 1):
            dp[i][0] = i
        
        for j in range(n + 1):
            dp[0][j] = j
        
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i-1] == word2[j-1]:
                    dp[i][j] = dp[i-1][j-1]
                else:
                    dp[i][j] = min(dp[i-1][j] + 1, dp[i][j-1] + 1, dp[i-1][j-1] + 1)
        
        return dp[m][n]
```

### Explanation
- **Initialization**: The `dp` table is initialized with dimensions `(m+1) x (n+1)`. The first row and column are filled to represent the base cases of converting an empty string to the respective strings.
- **DP Table Filling**: For each character in both strings, if the characters match, the edit distance is carried forward from the diagonal cell. If they don't match, the minimum cost among deletion, insertion, or substitution is chosen.
- **Result**: The bottom-right cell of the `dp` table contains the minimum edit distance between the entire strings `word1` and `word2`.

This approach efficiently computes the edit distance in O(m*n) time and space complexity, making it suitable for strings of moderate length.
