To solve this problem, we need to determine if a singly-linked list contains a cycle. The classic approach to detect cycles in a linked list involves using two pointers, one moving at twice the speed of the other. If the fast pointer catches up to the slow pointer, a cycle exists.

### Approach
1. **Initialization**: Start both pointers (`a` and `b`) at the head of the linked list.
2. **Traversal**: Move the slow pointer (`a`) one step at a time and the fast pointer (`b`) two steps at a time.
3. **Cycle Detection**: If the fast pointer meets the slow pointer during traversal, a cycle exists. If the fast pointer reaches the end of the list (i.e., `None`), the list is acyclic.

This approach efficiently detects cycles in O(n) time complexity, where n is the number of nodes in the linked list, as both pointers traverse the list. The space complexity is O(1) since only two pointers are used.

### Solution Code
```python
class Solution:
    def hasCycle(self, head: Optional[ListNode]) -> bool:
        a = b = head
        while b and b.next:
            a = a.next
            b = b.next.next
            if a == b:
                return True
        return False
```

### Explanation
- **Initialization**: Both pointers `a` and `b` start at the head of the linked list.
- **Traversal Loop**: The loop continues as long as the fast pointer (`b`) and its next node exist. In each iteration:
  - The slow pointer (`a`) moves to the next node.
  - The fast pointer (`b`) moves to the node after the next.
- **Cycle Check**: If at any point the slow pointer and fast pointer point to the same node, a cycle is detected, and the function returns `True`.
- **Termination**: If the loop exits without detecting a cycle, the function returns `False`, indicating the list is acyclic.

This method efficiently checks for cycles by leveraging the relative speeds of the two pointers, ensuring optimal performance with minimal space usage.
