To solve this problem, we need to determine if it's possible to finish all courses given the prerequisites. The solution involves checking if the graph of courses and their prerequisites forms a directed acyclic graph (DAG). If there's a cycle in the graph, it's impossible to finish all courses, as there would be a circular dependency.

### Approach
1. **Graph Construction**: We construct a graph where each node represents a course, and edges represent prerequisites. Specifically, for each prerequisite pair `[a, b]`, we add an edge from `b` to `a`, indicating that course `a` is a prerequisite for course `b`.
2. **Cycle Detection**: Using Depth-First Search (DFS), we detect cycles in the graph. During DFS, we maintain a set `recStack` to keep track of nodes currently in the recursion stack. If we encounter a node that is already in `recStack`, a cycle exists.
3. **Visited Tracking**: We use a `visited` array to mark nodes that have been processed. This helps in avoiding redundant checks and ensures each node is processed only once.

### Solution Code
```python
class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        graph = [[] for _ in range(numCourses)]
        for a, b in prerequisites:
            graph[b].append(a)
        
        visited = [False] * numCourses
        recStack = set()
        
        def dfs(course):
            if visited[course]:
                return True
            if course in recStack:
                return False

            recStack.add(course)
            for p in graph[course]:
                if not dfs(p):
                    return False
            recStack.remove(course)
            return True

            if not visited[i]:
                if not dfs(i):

        return True
```

### Explanation
1. **Graph Construction**: The graph is built by iterating through the prerequisites. For each pair `[a, b]`, we add `a` to the list of prerequisites for `b`. This correctly models the dependency where `a` must be completed before `b`.
2. **Cycle Detection**: The DFS function checks for cycles by tracking nodes in the current recursion stack (`recStack`). If a node is revisited while it's still in the stack, a cycle is detected.
3. **Visited Tracking**: The `visited` array ensures each node is processed only once. After processing a node, it is removed from `recStack` to allow other paths to be explored without interference.
4. **Cycle Handling**: If any cycle is detected during DFS, the function immediately returns `False`. If all nodes are processed without detecting cycles, it returns `True`, indicating all courses can be finished.
