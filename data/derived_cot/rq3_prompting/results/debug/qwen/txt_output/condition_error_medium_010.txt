=== RAW ANSWER ===

To solve this problem, we need to find the k-th smallest factor of a given integer `n`. The factors of a number are the integers that divide it completely without leaving a remainder. The solution involves iterating through all possible divisors of `n` in ascending order and counting them until we reach the k-th divisor.

### Approach
1. **Initialization**: Start with a counter `count` set to 0 to keep track of the number of factors found.
2. **Iteration**: Loop through all integers from 1 to `n` (inclusive). For each integer `i`, check if `n` is divisible by `i`.
3. **Counting Factors**: If `i` is a divisor of `n`, increment the `count` and check if it matches `k`. If it does, return `i` immediately as it is the k-th factor.
4. **Completion Check**: If the loop completes without finding `k` factors, return -1 indicating that the k-th factor does not exist.

This approach efficiently checks each potential divisor in ascending order, ensuring that the k-th smallest factor is found if it exists.

### Solution Code
```python
class Solution:
    def kthFactor(self, n: int, k: int) -> int:
        count = 0
        for i in range(1, n + 1):
            if n % i == 0:
                count += 1
                if count == k:
                    return i
        return -1
```

### Explanation
- **Initialization**: The variable `count` is initialized to 0 to keep track of how many factors have been found.
- **Loop Through Divisors**: The loop runs from 1 to `n`, checking each number `i` to see if it is a divisor of `n` (i.e., `n % i == 0`).
- **Factor Counting**: Each time a divisor is found, `count` is incremented. If `count` reaches `k`, the current divisor `i` is returned immediately as it is the k-th factor.
- **Return -1 if Not Found**: If the loop completes without finding `k` factors, the function returns -1, indicating that the k-th factor does not exist.

This approach ensures that we efficiently and correctly identify the k-th smallest factor of `n` or determine that it does not exist. The solution handles all edge cases, including when `k` is larger than the number of divisors of `n`.
