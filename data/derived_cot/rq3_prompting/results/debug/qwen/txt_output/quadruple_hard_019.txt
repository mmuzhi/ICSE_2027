To solve this problem, we need to determine the maximum number of times we can form the string `s2` by selecting characters from `n1` copies of the string `s1` in order, without requiring the characters to be contiguous. The solution involves using a cycle detection approach to efficiently compute the result without explicitly constructing the concatenated string, which can be very large when `n1` is large.

### Approach
1. **Initial Check**: If `s2` contains any character not present in `s1`, it's impossible to form `s2`, so return 0.
2. **Cycle Detection**: The core idea is to simulate the process of forming `s2` from `s1` while tracking the position in the concatenated string (`n1` copies of `s1`). We use a dictionary to record the start positions of cycles encountered during this simulation.
3. **Cycle Utilization**: Once a cycle is detected, we calculate how many full cycles can be completed and the remaining characters. The result is derived from the cycle's properties and the remaining characters.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def getMaxRepetitions(self, s1: str, n1: int, s2: str, n2: int) -> int:
        if not set(s2).issubset(set(s1)):
            return 0
        
        rec = [0]
        track = defaultdict(int)
        start = 0
        ptr = 0
        
        while True:
            for char in s2:
                idx = s1.find(char, start)
                if idx == -1:
                    idx = s1.find(char)
                    start = idx + 1 if idx != -1 else -1
                else:
                    start = idx + 1
                if start == -1:

            rec.append(start)
            if rec[-1] >= n1 * len(s1):
                return (len(rec) - 1) // n2

                track[start] = len(rec) - 1

                cycle_length = len(rec) - cycle_start - 1
```
