To solve this problem, we need to find the longest cycle in a directed graph. A cycle is a path that starts and ends at the same node without repeating any other node. The solution involves traversing the graph using Depth-First Search (DFS) to detect cycles and determine their lengths.

### Approach
1. **Graph Representation**: The graph is represented by an adjacency list where each node points to its outgoing edge.
2. **Cycle Detection**: We use DFS to traverse the graph. During traversal, we keep track of visited nodes and the order in which they were visited (time-stamps).
3. **Cycle Length Calculation**: When we encounter a node that has already been visited in the current DFS path, we calculate the cycle length by subtracting the time-stamp of the encountered node from the current time-stamp. This gives the number of nodes in the cycle.
4. **Tracking Maximum Cycle**: We keep track of the maximum cycle length encountered during the traversal.

### Solution Code
```python
class Solution:
    def longestCycle(self, edges: List[int]) -> int:
        n = len(edges)
        bl = [0] * n
        mp = defaultdict(int)
        mx = -1
        for i in range(n):
            if bl[i] == 0:
                x = i
                l = 0
                st = set()
                while x >= 0 and x < n and bl[x] == 0:
                    bl[x] = 1
                    mp[x] = l
                    l += 1
                    st.add(x)
                    x = edges[x]
                if x in st:
                    mx = max(mx, l - mp[x])
        return mx
```

### Explanation
1. **Initialization**: We initialize an array `bl` to keep track of visited nodes (0 means unvisited, 1 means visited). A dictionary `mp` records the time-stamp (order of visit) for each node.
2. **DFS Traversal**: For each unvisited node, we start a DFS. During traversal, we mark nodes as visited and record their time-stamps.
3. **Cycle Detection**: If during traversal we encounter a node that is already in the current DFS path (i.e., it's in the set `st`), we calculate the cycle length by subtracting the time-stamp of that node from the current time-stamp.
4. **Result Update**: The maximum cycle length encountered during the traversal is updated and returned. If no cycle is found, the result remains -1.

This approach efficiently detects cycles in the graph using DFS and calculates their lengths, ensuring optimal performance and correctness.
