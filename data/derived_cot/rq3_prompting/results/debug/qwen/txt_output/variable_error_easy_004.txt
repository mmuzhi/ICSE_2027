2. **Prefix Sum Calculation**: Compute the prefix sums of the sorted array. The prefix sum array helps in quickly determining the sum of the first `i` elements.
3. **Binary Search for Queries**: For each query, use binary search to find the smallest index in the prefix sum array where the sum is at least the query value. This index directly gives the number of elements needed.

### Solution Code
```python
class Solution:
    def answerQueries(self, nums: List[int], queries: List[int]) -> List[int]:
        nums.sort()
        sums = [0]
        for num in nums:
            sums.append(sums[-1] + num)
        
        def bSearch(q):
            low, high = 0, len(sums)
            while low < high:
                mid = (low + high) // 2
                if sums[mid] < q:
                    low = mid + 1
                else:
                    high = mid
            return low
        
        return [bSearch(q) for q in queries]
```

### Explanation
1. **Sorting the Array**: The input array `nums` is sorted in ascending order. This ensures that the smallest elements are considered first, which is optimal for minimizing the number of elements needed to reach a given sum.
2. **Prefix Sum Array**: The prefix sum array `sums` is constructed where each element at index `i` is the sum of the first `i` elements of the sorted array. This allows for efficient range sum queries.
3. **Binary Search**: For each query value `q`, the binary search function `bSearch` locates the smallest index `i` in the prefix sum array where `sums[i] >= q`. The index `i` represents the number of elements from the sorted array needed to achieve a sum that meets or exceeds `q`.

This approach efficiently handles each query in logarithmic time relative to the size of the prefix sum array, making it suitable for large input sizes.
