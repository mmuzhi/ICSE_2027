=== RAW ANSWER ===

To solve this problem, we need to perform a level-order traversal (BFS) of a binary tree and return the nodes level by level from bottom to top. The initial code had two main issues: it returned `None` for an empty tree instead of an empty list, and the children nodes were not being properly added to the queue during BFS due to incorrect indentation.

### Approach
1. **Base Case Handling**: If the tree is empty (root is `None`), return an empty list.
2. **BFS Traversal**: Use a queue to traverse the tree level by level. For each node, add its value to the current level's list and enqueue its children if they exist.
3. **Reverse the Levels**: After collecting all levels from top to bottom, reverse the list of levels to get the bottom-up order.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        def checkReverse(ans):
            l, r = 0, len(ans)-1
            while l <= r:
                ans[l], ans[r] = ans[r], ans[l]
                l += 1
                r -= 1
            return ans

        if not root:
            return []
        q, ans = [root], []
        while q:
            n, level = len(q), []
            for i in range(n):
                curr = q.pop(0)
                level.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(level)
        return checkReverse(ans)
```

### Explanation
1. **Base Case Handling**: The function first checks if the root is `None`. If so, it returns an empty list, which is the correct behavior for an empty tree.
2. **BFS Traversal**: The queue starts with the root node. For each node dequeued, its value is added to the current level's list. The node's left and right children are enqueued if they exist.
3. **Level Reversal**: After collecting all levels from top to bottom, the `checkReverse` function reverses the list of levels in place by swapping elements from the start and end, moving towards the center. This ensures the levels are ordered from bottom to top.

This approach efficiently handles the traversal and reversal, ensuring the correct output for both empty and non-empty trees.
