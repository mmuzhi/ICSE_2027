### Approach
The key observation is that the range function in Python, when used as `range(1, n)`, generates numbers from 1 to `n-1`. To include `n`, we should use `range(1, n+1)`. The rest of the logic remains the same: convert each number to a string, sort the strings lexicographically, and then convert them back to integers.
```python
class Solution:
    def lexicalOrder(self, n: int) -> List[int]:
        lst = [str(i) for i in range(1, n+1)]
        lst.sort()
        return [int(i) for i in lst]
```

### Explanation
1. **Range Correction**: The range is adjusted from `range(1, n)` to `range(1, n+1)` to ensure all numbers from 1 to `n` are included.
2. **String Conversion**: Each number is converted to a string to facilitate lexicographical sorting.
3. **Sorting**: The list of strings is sorted lexicographically, which corresponds to the desired numerical order when converted back to integers.
4. **Integer Conversion**: The sorted strings are converted back to integers to produce the final result.

This approach efficiently handles the lexicographical ordering by leveraging string comparison, ensuring correctness and simplicity.
