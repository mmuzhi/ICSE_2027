=== RAW ANSWER ===

To solve this problem, we need to perform a level-order traversal of a binary tree and return the nodes level by level from bottom to top. The initial code attempted to do this using a breadth-first search (BFS) approach but had several bugs, including incorrect reversal of the level order traversal and handling of empty trees.

### Approach
1. **Handle Empty Tree Case**: If the root is `None`, return an empty list immediately.
2. **Breadth-First Search (BFS)**: Use a queue to traverse the tree level by level. For each level, collect all node values and then enqueue their children (if they exist) for the next level.
3. **Reverse the Result**: After collecting all levels from top to bottom, reverse the list of levels to present the result from bottom to top.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []
        q, ans = [root], []
        while q:
            n, l = len(q), []
            for i in range(n):
                curr = q.pop(0)
                l.append(curr.val)
                if curr.left:
                    q.append(curr.left)
                if curr.right:
                    q.append(curr.right)
            ans.append(l)
        return ans[::-1]
```

### Explanation
1. **Empty Tree Check**: The function first checks if the tree is empty (root is `None`) and returns an empty list if true.
2. **BFS Initialization**: The queue `q` is initialized with the root node. The list `ans` will store each level's node values.
3. **Level Processing**: For each level, the number of nodes `n` is determined. Each node is dequeued, its value is added to the current level list `l`, and its children (if any) are enqueued for processing in the next level.
4. **Result Construction**: After processing all levels, the list `ans` contains levels from top to bottom. Reversing this list (`ans[::-1]`) gives the levels from bottom to top, which is the required output.

This approach efficiently handles the level-order traversal and ensures the result is presented in the correct order while avoiding common pitfalls like processing `None` children or incorrect list reversal.
