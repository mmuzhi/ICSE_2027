=== RAW ANSWER ===

To solve this problem, we need to determine if it's possible to finish all courses given the prerequisites. The solution involves checking for cycles in the directed graph formed by the courses and their prerequisites. If there is a cycle, it means some courses cannot be scheduled, and the answer is False. Otherwise, it's True.

### Approach
1. **Graph Construction**: Represent the courses and their prerequisites as a directed graph using a adjacency list. Each course points to its prerequisites.
2. **Cycle Detection**: Use Depth-First Search (DFS) to detect cycles in the graph. During the DFS traversal, we maintain three states for each course:
   - **0**: Unvisited.
   - **1**: Visiting (currently in the recursion stack).
   - **2**: Visited (all prerequisites are satisfied).
3. **DFS Traversal**: For each course, if it hasn't been visited, perform DFS. During DFS, mark the course as visiting, then recursively check all its prerequisites. If any prerequisite is already visiting, a cycle is detected. If a prerequisite hasn't been visited, continue the DFS. If all prerequisites are visited (and no cycle is found), mark the course as visited.

### Solution Code
```python
class Solution:
    def canFinish(self, numCourses: int, prerequisites: List[List[int]]) -> bool:
        from collections import defaultdict
        pre = defaultdict(list)
        for a, b in prerequisites:
            pre[a].append(b)
        
        visited = [0] * numCourses
        
        def dfs(course):
            if visited[course] == 1:
                return False
            if visited[course] == 2:
                return True
            visited[course] = 1
            for p in pre[course]:
                if not dfs(p):
                    return False
            visited[course] = 2
            return True
        
        for i in range(numCourses):
            if visited[i] == 0:
                if not dfs(i):
                    return False
        return True
```

### Explanation
1. **Graph Construction**: The graph is built using a defaultdict where each course maps to a list of its prerequisites.
2. **Cycle Detection**: The DFS function checks for cycles by tracking the visit states. If a course is revisited while in the visiting state (1), a cycle exists, and the function returns False. If a course is marked visited (2), it means all its prerequisites are satisfied.
3. **DFS Traversal**: The main loop iterates over all courses. For each unvisited course, it initiates DFS. If DFS finds a cycle, it returns False immediately. If no cycles are found after all courses are processed, it returns True.

This approach efficiently checks for cycles in the graph, ensuring that all courses can be scheduled without any circular dependencies. The use of DFS with state tracking ensures optimal performance and correctness.
