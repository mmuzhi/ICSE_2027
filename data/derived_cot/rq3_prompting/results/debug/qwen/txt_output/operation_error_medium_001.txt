### Approach
1. **Graph Construction**: Build a graph where each recipe node points to its required ingredients. Additionally, maintain a reverse graph where each ingredient points to the recipes that use it.
2. **Topological Sort (Kahn's Algorithm)**: Use a queue to process nodes (recipes and ingredients) with zero incoming dependencies (indegree). Start with the available supplies (ingredients) as base nodes.
3. **Dependency Tracking**: For each node processed, reduce the indegree of its dependent nodes (recipes that use this ingredient). If a recipe's indegree drops to zero, all its ingredients are available, and it can be added to the result list.

### Solution Code
```python
class Solution:
    def findAllRecipes(self, recipes: List[str], ingredients: List[List[str]], supplies: List[str]) -> List[str]:
        from collections import defaultdict, deque
        
        graph = defaultdict(list)
        reverse_graph = defaultdict(list)
        indegree = defaultdict(int)
        
        n = len(recipes)
        
        for i in range(n):
            recipe = recipes[i]
            for ing in ingredients[i]:
                graph[recipe].append(ing)
                reverse_graph[ing].append(recipe)
                indegree[ing] += 1
        
        for recipe in recipes:
            indegree[recipe] = len(ingredients[recipes.index(recipe)])
        
        q = deque()
        for node in supplies:
            q.append(node)
        
        result = []
        while q:
            node = q.popleft()
            if node in recipes:
                result.append(node)
            for neighbor in reverse_graph.get(node, []):
                indegree[neighbor] -= 1
                if indegree[neighbor] == 0:
                    q.append(neighbor)
        
        return result
```

### Explanation
1. **Graph Construction**: The graph is built such that each recipe points to its required ingredients. The reverse graph allows us to efficiently find all recipes that depend on a particular ingredient.
2. **Topological Sort Initialization**: The queue starts with all available supplies (ingredients). These are nodes with no unresolved dependencies.
3. **Processing Nodes**: For each node processed, if it is a recipe, it is added to the result. The indegree of its dependent nodes (recipes that use this ingredient) is decremented. If a dependent node's indegree reaches zero, it is added to the queue, indicating all its ingredients are available.
4. **Result Compilation**: The result list accumulates all recipes that can be prepared once all their ingredients are available, processed in topological order.

This approach efficiently handles the dependencies between recipes and ingredients, ensuring that only recipes with all their ingredients available are included in the result.
