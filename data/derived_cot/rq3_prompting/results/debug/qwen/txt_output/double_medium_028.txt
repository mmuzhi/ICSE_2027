To solve this problem, we need to find the kth largest element in an array of integers. The initial approach attempts to use sorting and negative indexing but contains two main issues: the `sort` function is not defined, and the indexing is incorrect for finding the kth largest element.

### Approach
1. **Sorting the Array**: The solution involves sorting the array in ascending order. This allows us to easily access the kth largest element by leveraging negative indexing.
2. **Negative Indexing**: After sorting the array, the kth largest element can be found at the index `-k` in the sorted array. For example, if the array is sorted as `[1, 2, 3]` and k is 2, the element at index `-2` is `2`, which is the second largest element.

### Solution Code
```python
class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:

```

### Explanation
- **Sorting the Array**: The `sorted(nums)` function sorts the input array in ascending order. This creates a new sorted list without modifying the original array.
- **Negative Indexing**: By using `[-k]`, we directly access the kth largest element. For instance, in the sorted list `[1, 2, 3]`, the element at index `-1` is the largest, `-2` is the second largest, and so on. This approach efficiently retrieves the desired element with minimal computational overhead.

This solution is both concise and efficient, leveraging Python's built-in sorting and indexing capabilities to solve the problem in O(n log n) time complexity due to the sorting step, which is optimal for this problem given the constraints.
