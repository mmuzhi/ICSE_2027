=== RAW ANSWER ===

To solve this problem, we need to find the complement of a given positive integer by flipping all its bits. The complement should be computed with respect to the same number of bits as the original number. 

### Approach
1. **Problem Analysis**: The problem requires us to flip all the bits of a given positive integer. For example, the binary representation of 5 (101) becomes 010 (2) after flipping all bits. The challenge is to correctly process each bit of the number and construct the complement string.
2. **Insight**: The solution involves processing each bit of the number from the least significant bit (LSB) to the most significant bit (MSB). For each bit, if it is 1, we append '0' to the complement string; if it is 0, we append '1'. This effectively flips each bit.
3. **Algorithm Selection**: The algorithm uses a loop to repeatedly divide the number by 2 (integer division) and check the remainder (modulus operation). The remainder gives the current bit, which is then flipped and added to the result string.
4. **Complexity Analysis**: The time complexity is O(log n) because we process each bit of the number once. The space complexity is O(log n) to store the complement string.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        com = ''
        while num > 0:
            if num % 2 == 1:
                com += '0'
            else:
                com += '1'
            num //= 2
        return int(com, 2) if com else 0
```

### Explanation
1. **Initialization**: We start with an empty string `com` to build the binary complement.
2. **Loop Until Zero**: The loop continues as long as the number `num` is greater than zero. In each iteration:
   - **Check Least Significant Bit**: Using `num % 2`, we determine if the least significant bit is 0 or 1. If it's 1, we append '0' to `com`; otherwise, we append '1'.
   - **Update Number**: The number is updated by performing integer division by 2 (`num //= 2`), effectively removing the least significant bit.
3. **Result Construction**: After processing all bits, the string `com` is converted from binary to an integer using `int(com, 2)`. If the input was zero, the loop doesn't execute, and we return 0.

This approach efficiently flips each bit of the input number and constructs the complement by processing each bit from the least significant to the most significant, ensuring correctness for all positive integers.
