=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays efficiently. The median is the middle value in an ordered list of numbers. If the combined length of the two arrays is even, the median is the average of the two middle numbers.

### Approach
1. **Merge the Arrays**: We start by merging the two sorted arrays into a single sorted array. This is done by comparing elements from both arrays and appending the smaller element to the result list. This step ensures that the merged array is sorted.
2. **Handle Edge Cases**: If the merged array is empty, we return 0.0 as the median since the median of an empty set is undefined.
3. **Calculate the Median**: Depending on whether the length of the merged array is odd or even, we compute the median. For an odd length, the median is the middle element. For an even length, the median is the average of the two middle elements.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        new = []
        p1 = 0
        p2 = 0
        
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
        
        n = len(new)
        if n == 0:
            return 0.0
        
        if n % 2 == 0:
            index = n // 2
            median = (new[index] + new[index-1]) / 2
        else:
            median = float(new[n//2])
        
        return median
```

### Explanation
1. **Merging Arrays**: The code merges the two sorted arrays by iterating through both arrays with two pointers (`p1` and `p2`). The smaller element from the current positions of the two arrays is appended to the result list (`new`), and the corresponding pointer is incremented.
2. **Handling Remaining Elements**: After the initial loop, any remaining elements from either array are appended to the result list to ensure all elements are included.
3. **Edge Case Handling**: If the merged array is empty, the function returns 0.0 to handle undefined median cases.
4. **Median Calculation**: The median is calculated based on the length of the merged array. For an even length, it averages the two middle elements; for an odd length, it picks the middle element. The result is returned as a float to accommodate both scenarios.

This approach efficiently combines the two sorted arrays and computes the median in linear time, making it suitable for large input sizes as required by the problem constraints.
