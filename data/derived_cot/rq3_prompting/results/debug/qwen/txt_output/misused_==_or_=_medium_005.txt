To solve this problem, we need to determine if a given singly-linked list is a sub-path of a binary tree. A sub-path means that the linked list must appear as a contiguous sequence of nodes in the binary tree, starting from some node and moving only to its left or right children.

### Approach
1. **Base Cases Handling**: 
   - If the linked list is empty (i.e., `head` is `None`), it trivially is a sub-path of any tree, so return `True`.
   - If the tree node is `None` but the linked list is not empty, return `False` since there's no node to match.

2. **Node Matching**:
   - If the current node's value matches the head of the linked list, recursively check if the rest of the linked list can be found starting from the current node's left or right child.

3. **Recursive Search**:
   - If the current node does not match the head of the linked list, continue the search in the left and right subtrees of the current node.

4. **Path Validation**:
   - The helper function `isSame` checks if the remaining linked list nodes match the path starting from a given tree node. It recursively verifies each node in the linked list against the tree nodes, moving only to left or right children.

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def isSubPath(self, head: Optional[ListNode], root: Optional[TreeNode]) -> bool:
        if head is None:
            return True
        if root is None:
            return False
        if head.val == root.val:
            if self.isSame(head, root):
                return True
        return self.isSubPath(head, root.left) or self.isSubPath(head, root.right)
    
    def isSame(self, head, root):
        if head is None:
            return True
        if root is None:
            return False
        if head.val == root.val:
            return self.isSame(head.next, root.left) or self.isSame(head.next, root.right)
        return False
```

### Explanation
1. **Base Cases**: The function `isSubPath` checks if the linked list is empty (`head is None`), in which case it returns `True`. If the tree node is `None` but the linked list is not, it returns `False`.
2. **Node Matching**: If the current node's value matches the head of the linked list, the helper function `isSame` is called to check if the remaining linked list nodes can be matched starting from this node.
3. **Recursive Search**: The function `isSubPath` continues searching in the left and right subtrees if the current node does not match the head of the linked list.
4. **Path Validation**: The helper function `isSame` checks if the remaining linked list nodes match the tree path by recursively comparing values and moving to left or right children. If the entire linked list is matched, it returns `True`; otherwise, it returns `False`.

This approach efficiently checks all possible paths in the binary tree to determine if the linked list exists as a contiguous sub-path, leveraging recursion and careful base case handling.
