=== RAW ANSWER ===

To solve this problem, we need to find the smallest subsequence (by the number of elements) in an array such that the sum of its elements is strictly greater than half of the total sum of the array. The solution involves sorting the array in descending order and then iteratively adding elements to a subsequence until the sum condition is met.

### Approach
1. **Calculate Total Sum**: First, compute the total sum of all elements in the array.
2. **Sort the Array**: Sort the array in descending order to prioritize larger elements, which helps in achieving the sum condition with the fewest elements.
3. **Build Subsequence**: Iterate through the sorted array, adding each element to a running sum and a subsequence list. After each addition, check if the running sum exceeds half of the total sum. If it does, return the current subsequence immediately, as it is the smallest valid subsequence.
4. **Return Result**: If the loop completes without meeting the condition (which shouldn't happen for non-empty arrays as the entire array's sum will eventually exceed half), return the entire array.

### Solution Code
```python
class Solution:
    def minSubsequence(self, nums: List[int]) -> List[int]:
        total = sum(nums)
        nums.sort(reverse=True)
        sub_sum = 0
        sub_seq = []
        for x in nums:
            sub_sum += x
            sub_seq.append(x)
            if 2 * sub_sum > total:
                return sub_seq
        return sub_seq
```

### Explanation
1. **Total Sum Calculation**: The total sum of the array is computed once at the beginning to avoid recalculating it during each iteration.
2. **Sorting**: The array is sorted in descending order to ensure that we consider the largest elements first. This allows us to reach the sum condition with the minimal number of elements.
3. **Subsequence Construction**: As we iterate through each element, we add it to the running sum and the subsequence. The condition `2 * sub_sum > total` checks if the current subsequence's sum is strictly greater than half of the total sum. This condition is used to avoid floating-point operations and ensure integer comparisons.
4. **Early Termination**: The loop terminates as soon as the sum condition is met, returning the smallest valid subsequence. If the loop completes without meeting the condition, the entire array is returned, though this scenario is unlikely for non-empty arrays.

This approach efficiently finds the minimal subsequence by leveraging sorting and early termination, ensuring optimal performance and correctness.
