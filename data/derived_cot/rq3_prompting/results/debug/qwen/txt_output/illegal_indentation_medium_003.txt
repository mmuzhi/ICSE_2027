To solve this problem, we need to generate all numbers within a given range [low, high] that have sequential digits in increasing order. The solution involves using a breadth-first search (BFS) approach starting from each single-digit number (1 through 9) and extending them by appending the next digit until the number exceeds the high limit.

### Approach
1. **Initialization**: Start with a queue containing all single-digit numbers from 1 to 9.
2. **BFS Traversal**: For each number dequeued, check if it lies within the specified range [low, high]. If it does, add it to the result list.
3. **Extension**: For each number, if the last digit is less than 9, extend the number by appending the next digit (last digit + 1) and enqueue the new number. This ensures that only numbers with strictly increasing consecutive digits are generated.
4. **Termination**: The BFS continues until all possible numbers (up to 9 digits) are processed.

This approach efficiently generates all valid numbers by leveraging BFS, ensuring that each number is built from the previous one by appending the next digit in sequence.

```python
from collections import deque

class Solution:
    def sequentialDigits(self, low: int, high: int) -> list:
        out = []
        queue = deque(range(1, 10))
        while queue:
            elem = queue.popleft()
            if low <= elem <= high:
                out.append(elem)
            last_digit = elem % 10
            if last_digit < 9:
                new_num = elem * 10 + (last_digit + 1)
                queue.append(new_num)
        return out
```

### Explanation
- **Initialization**: The queue starts with numbers 1 through 9, which are the base cases for building sequential digits.
- **BFS Loop**: For each number dequeued, the algorithm checks if it falls within the range [low, high]. If it does, the number is added to the output list.
- **Extension**: The last digit of the current number is used to determine the next number. If the last digit is less than 9, the next number is formed by appending the next digit (last digit + 1). This new number is then enqueued for further processing.
- **Efficiency**: The BFS ensures that all numbers are processed in the order of their digit length, and the extension step efficiently builds numbers with consecutive digits. The algorithm terminates once all possible numbers (up to 9 digits) are processed, ensuring optimal performance.
