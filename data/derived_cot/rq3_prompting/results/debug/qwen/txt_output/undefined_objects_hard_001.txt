To solve this problem, we need to reverse the nodes of a linked list in groups of size k. If the number of nodes is not a multiple of k, the remaining nodes should remain unchanged. The solution involves using a dummy node to simplify edge cases and iteratively reversing each group of k nodes.

### Approach
1. **Dummy Node**: We start by creating a dummy node to act as a sentinel, which helps in handling edge cases such as an empty list or a list with fewer than k nodes.
2. **Iterate Through Groups**: For each group of k nodes, we first locate the k-th node from the current position. If we cannot find k nodes, we break the loop.
3. **Reverse the Group**: Once the k nodes are identified, we reverse them. The reversal is done by iterating through the group and adjusting the next pointers of each node.
4. **Connect Reversed Group**: After reversing the group, we connect the previous part of the list to the new head of the reversed group and the new tail of the reversed group to the next group.
5. **Update Pointers**: We then move the current group pointer to the start of the next group.

### Solution Code
```python
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        prev_group = dummy
        
        while True:
            kth = self.getKth(prev_group, k)
            if not kth:
                break
            next_group_start = kth.next
            
            start = prev_group.next
            prev = None
            curr = start
            for _ in range(k):
                next_node = curr.next
                curr.next = prev
                prev = curr
                curr = next_node
            
            prev_group.next = prev
            prev_group = next_group_start
        
        return dummy.next

    def getKth(self, node, k):
        while node and k > 0:
            node = node.next
            k -= 1
        return node
```

1. **Dummy Node**: The dummy node is used to simplify the reversal process, especially when dealing with the head of the list.
2. **Finding k-th Node**: The `getKth` method locates the k-th node from the given node. This is crucial for identifying the end of each group to be reversed.

4. **Connecting Groups**: After reversing a group, the previous group's next pointer is updated to point to the new head of the reversed group. The current group pointer is then moved to the start of the next group.
5. **Edge Handling**: If fewer than k nodes are left, the loop breaks, and the remaining nodes remain in their original order.

            kth = self.getKth(prev_group, k)
            if not kth:
                break
            next_group_start = kth.next
            
            start = prev_group.next
            prev = None
            curr = start
            for _ in range(k):
                next_node = curr.next
                curr.next = prev
                prev = curr
                curr = next_node
            
            prev_group.next = prev
            prev_group = next_group_start
        
        return dummy.next

    def getKth(self, node, k):
        while node and k > 0:
            node = node.next
            k -= 1
        return node
