To solve this problem, we need to generate all valid IP addresses from a given string. An IP address consists of four parts, each ranging from 0 to 255. The solution involves using a backtracking approach to explore all possible ways to split the string into four parts that form a valid IP address.

### Approach
1. **Backtracking Function**: The core of the solution is a recursive backtracking function that tries to split the string into four parts. The function checks if the current segment is valid (i.e., a number between 0 and 255) and proceeds with the next segments if the current segment is valid.
2. **Base Case**: When the entire string is processed (i.e., the current index reaches the end of the string and exactly four parts are formed), the current combination of segments is added to the result.
3. **Handling Leading Zeros**: If a segment starts with '0', it can only be a single '0' to avoid invalid numbers like '01' or '001'.
4. **Segment Validation**: For segments not starting with '0', the function checks if the segment's integer value is less than 256 and ensures the segment does not exceed three digits (since numbers with more than three digits are automatically greater than 255).

### Solution Code
```python
class Solution():
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i > len(s):
                return
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 4 and i + j < len(s):
                if int(s[i:i+j+1]) < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Initialization**: The `res` list stores all valid IP addresses, and `cur` keeps track of the current segments being formed.
2. **Backtracking Function**: The `backtrack` function is defined to explore all valid segments starting from index `i`.
3. **Base Case**: If the current index `i` reaches the end of the string and exactly four segments are formed, the segments are joined into a string and added to `res`.
4. **Invalid Cases**: If more than four segments are formed or the index exceeds the string length, the function returns immediately.
5. **Leading Zero Handling**: If the current character is '0', only the single '0' is added as a segment, and the function proceeds to the next index.
6. **Segment Validation**: For segments not starting with '0', the function checks all possible lengths (up to three digits) to ensure the segment's integer value is less than 256. Valid segments are added to `cur`, and the function proceeds recursively.
7. **Result Compilation**: After exploring all valid segments, the function returns the list of all valid IP addresses stored in `res`.

This approach efficiently explores all possible valid IP addresses by leveraging backtracking and careful validation of each segment, ensuring correctness and completeness.
