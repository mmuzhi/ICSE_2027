=== RAW ANSWER ===

To solve this problem, we need to generate all valid IP addresses from a given string. An IP address consists of four parts, each ranging from 0 to 255, and each part must not have leading zeros unless the part is exactly "0". The solution involves using a backtracking approach to explore all possible ways to split the string into four parts while adhering to these constraints.

### Approach
1. **Backtracking with Pruning**: The algorithm uses a recursive backtracking method to build possible IP addresses by splitting the string into four parts. It prunes invalid paths early by checking:
   - The number of parts does not exceed four.
   - The current index does not exceed the string length.
   - Parts do not have leading zeros (unless the part is "0").
   - Each part is a number between 0 and 255.

2. **Handling Leading Zeros**: Special handling is done for parts starting with '0' to ensure they are not split into multiple digits (e.g., "01" is invalid). If a part starts with '0', it must be exactly "0".

3. **Digit Limit**: Each part can be at most three digits long (since numbers larger than 255 are invalid).

### Solution Code
```python
class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) > 4 or i > len(s):
                return
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            j = 0
            while j < 3 and i + j < len(s):
                part = s[i:i+j+1]
                if int(part) < 256:
                    cur.append(part)
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Base Case Handling**: The recursion stops when the entire string is processed (i == len(s)) and exactly four parts are formed (len(cur) == 4). The current parts are joined into an IP address and added to the result list.
2. **Invalid Path Pruning**: If more than four parts are formed or the current index exceeds the string length, the recursion backtracks.
3. **Leading Zero Handling**: If the current character is '0', it is directly added as a part (since "0" is valid), and the recursion proceeds. No further digits are considered to prevent invalid parts like "01".
4. **Digit Limit and Value Check**: For parts not starting with '0', the algorithm considers up to three digits (j < 3). Each substring is converted to an integer and checked to ensure it is less than 256. Valid parts are added to the current list, and the recursion continues from the next index.

This approach efficiently explores all valid IP addresses by leveraging backtracking with careful pruning of invalid paths, ensuring correctness and optimal performance.
