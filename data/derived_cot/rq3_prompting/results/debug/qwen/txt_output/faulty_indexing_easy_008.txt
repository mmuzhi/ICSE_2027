=== RAW ANSWER ===

To solve this problem, we need to compute the maximum depth of a tree where each node can have multiple children. The initial code attempts to use a BFS (Breadth-First Search) approach with two queues to track nodes at each level, but it contains a bug related to the indexing of children nodes. 

### Approach
The approach involves using a BFS traversal to explore each level of the tree. The key steps are:
1. **Initialization**: Start with the root node in a queue and initialize the depth to 0.
2. **BFS Traversal**: Process each node in the current level, adding its children to the next level's queue.
3. **Level Tracking**: After processing all nodes in the current level, increment the depth counter and switch to the next level's queue.

The main bug in the original code was that it incorrectly accessed the children of a node using an index (specifically `[1]`), which would fail if the node had fewer than two children. The fix involves iterating over all children of a node without using any index.

### Solution Code
```python
"""
# Definition for a Node.
class Node:
    def __init__(self, val=None, children=None):
        self.val = val
        self.children = children

class Solution:
    def maxDepth(self, root: 'Node') -> int:
        if root is None:
            return 0
        depth = 0
        nodes_queue = [root]
        next_nodes_queue = []
        while nodes_queue:
            node_out = nodes_queue.pop(0)
            for child in node_out.children:
                next_nodes_queue.append(child)
            if not nodes_queue:
                nodes_queue, next_nodes_queue = next_nodes_queue, nodes_queue
                depth += 1
        return depth
```

### Explanation
1. **Initialization**: The function starts by checking if the root node is `None`, returning 0 if true. Otherwise, it initializes the depth to 0 and sets up two queues: `nodes_queue` for the current level and `next_nodes_queue` for the next level.
2. **BFS Loop**: The loop processes each node in `nodes_queue`:
   - **Node Processing**: The first node is removed from `nodes_queue`, and all its children are added to `next_nodes_queue`.
   - **Level Check**: After processing a node, if `nodes_queue` is empty, it means all nodes in the current level have been processed. The queues are swapped (moving to the next level), and the depth is incremented.
3. **Return Depth**: Once all nodes are processed, the depth variable holds the maximum depth of the tree, which is returned.

This approach efficiently traverses each level of the tree, ensuring the maximum depth is accurately calculated by incrementing the depth counter only after all nodes in the current level are processed.
