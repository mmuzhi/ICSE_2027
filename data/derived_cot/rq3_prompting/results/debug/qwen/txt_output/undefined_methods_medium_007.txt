```python
            else:
                high = mid - 1
        return -1
```

### Explanation
1. **Binary Search Setup**: The binary search is initialized with `low` set to 0 and `high` set to the last index of the array. If the array length is 1, the only element is trivially a peak.
2. **Mid Calculation**: The mid index is calculated as the average of `low` and `high`.
3. **Peak Check**: The code checks if the mid element is a peak by verifying it is strictly greater than both its left and right neighbors (if they exist). This ensures the condition of strict inequality is met.
4. **Binary Search Adjustment**: If the mid element is not a peak, the search space is adjusted. If the mid element is less than or equal to its right neighbor, the peak must lie to the right, so `low` is set to `mid + 1`. Otherwise, the peak must lie to the left, and `high` is set to `mid - 1`.
5. **Termination**: The loop terminates when a peak is found or the search space is exhausted, returning -1 if no peak is found (though the problem guarantees at least one peak exists).

This approach efficiently narrows down the search space using binary search, ensuring optimal performance with a time complexity of O(log n).

        n = len(nums)
        low = 0
        high = n - 1
        if n == 1:

        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] > nums[mid-1]) and (mid == n-1 or nums[mid] > nums[mid+1]):
                return mid
            elif nums[mid] <= nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
