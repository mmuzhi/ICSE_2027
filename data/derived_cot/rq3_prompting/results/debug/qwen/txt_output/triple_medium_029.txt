=== RAW ANSWER ===

To solve this problem, we need to generate all possible valid IP addresses from a given string. An IP address consists of four segments, each ranging from 0 to 255, and segments cannot have leading zeros unless the segment is exactly "0". The solution involves using a backtracking approach to explore all possible ways to split the string into four valid segments.

### Approach
1. **Backtracking with DFS**: The solution uses a depth-first search (DFS) approach to explore all possible ways to split the string into four segments. The algorithm starts from the beginning of the string and tries to form each segment by considering 1, 2, or 3 digits (if the segment starts with '0', it can only be one digit).
2. **Base Case Handling**: When the entire string is processed (i.e., the current index equals the string length) and exactly four segments are formed, the result is added to the output list.
3. **Pruning Invalid Paths**: The algorithm prunes paths where the number of segments exceeds four or the current index exceeds the string length. Additionally, it checks if four segments are already formed but the string isn't fully processed, which would be invalid.

### Solution Code
```python
class Solution:
    def restoreIpAddresses(self, s: str) -> List[str]:
        res = []
        cur = []
        
        def backtrack(i):
            if i == len(s) and len(cur) == 4:
                res.append(".".join(cur))
                return
            if len(cur) == 4 and i != len(s):
                return
            if len(cur) > 4 or i > len(s):
                return
            
            if s[i] == '0':
                cur.append('0')
                backtrack(i + 1)
                cur.pop()
                return
            
            j = 0
            while j < 4 and i + j < len(s):
                if int(s[i:i+j+1]) < 256:
                    cur.append(s[i:i+j+1])
                    backtrack(i + j + 1)
                    cur.pop()
                j += 1
        
        backtrack(0)
        return res
```

### Explanation
1. **Initialization**: The `res` list stores all valid IP addresses, and `cur` is a temporary list to hold the current segments being formed.
2. **Backtracking Function**: The `backtrack` function is defined to explore all possible segmentations starting from index `i`.
   - **Base Case**: If the entire string is processed (`i == len(s)`) and exactly four segments are formed (`len(cur) == 4`), the segments are joined into a string and added to `res`.
   - **Pruning**: If four segments are already formed but the string isn't fully processed (`len(cur) == 4 and i != len(s)`), the function returns early. Similarly, if the number of segments exceeds four or the index goes out of bounds, it returns early.
   - **Zero Handling**: If the current character is '0', it is added as a segment, and the function proceeds to the next index.
   - **Non-zero Handling**: For non-zero characters, the function tries segments of length 1 to 3 digits (up to 4 digits to avoid exceeding segment limits). Each valid segment (numeric value < 256) is added to `cur`, and the function proceeds recursively.
3. **Result Compilation**: After initiating the backtracking from the start of the string, the function returns the list of all valid IP addresses found.

This approach efficiently explores all possible valid segmentations while pruning invalid paths early, ensuring optimal performance and correctness.
