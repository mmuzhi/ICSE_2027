=== RAW ANSWER ===

To solve this problem, we need to find the maximum performance achievable by selecting at most `k` employees from a list, where performance is defined as the product of the sum of their speeds and the maximum efficiency among them. 

### Approach
1. **Problem Analysis**: The key observation here is that the maximum efficiency in the selected set will be the highest efficiency encountered so far when processing employees in descending order of efficiency. For each employee, we maintain a set of up to `k` employees with the highest speeds, using a min-heap to efficiently manage the smallest speed when the set size exceeds `k`.
2. **Algorithm Selection**: 
   - **Sorting**: Employees are sorted by efficiency in descending order to ensure the current employee has the highest efficiency in the set being considered.
   - **Min-Heap**: A min-heap is used to keep track of the smallest speed in the selected set, allowing efficient removal of the slowest employee when the set size exceeds `k`.
   - **Greedy Selection**: For each employee, we add their speed to the heap and update the sum of speeds. If the heap size exceeds `k`, we remove the smallest speed and adjust the sum accordingly. The performance is then calculated as the product of the current efficiency and the sum of speeds, updating the maximum performance found.
3. **Complexity Analysis**: 
   - **Sorting**: O(n log n) time to sort the employees by efficiency.
   - **Heap Operations**: Each insertion and removal operation on the heap takes O(log k) time, and these operations are performed for each of the `n` employees, leading to O(n log k) time.
   - **Overall Complexity**: O(n log n + n log k) = O(n log n) since `k` can be up to `n`.

        MOD = 10**9 + 7
        
        top_k_speeds = []
        res, top_k_sum = 0, 0
        
        pairs = [(e, s) for e, s in zip(efficiency, speed)]
        for e, s in sorted(pairs, reverse=True):
            heapq.heappush(top_k_speeds, s)
            top_k_sum += s
            if len(top_k_speeds) > k:
                top_k_sum -= heapq.heappop(top_k_speeds)
            res = max(res, e * top_k_sum)
        
        return res % MOD

1. **Initialization**: We initialize a min-heap to keep track of the speeds of the selected employees and variables to store the maximum performance (`res`) and the sum of speeds in the current set (`top_k_sum`).
2. **Sorting and Processing**: Employees are paired by efficiency and speed, then sorted in descending order of efficiency. For each employee, their speed is added to the heap. If the heap size exceeds `k`, the smallest speed is removed from the heap, and the sum is adjusted accordingly.
3. **Performance Calculation**: The performance for the current set is calculated as the product of the current employee's efficiency and the sum of speeds in the set. The maximum performance encountered during processing is stored.
4. **Modulo Operation**: The result is returned modulo \(10^9 + 7\) to handle large numbers and prevent overflow.

This approach efficiently selects the optimal set of employees by leveraging sorting and heap operations to maintain the highest performance possible while adhering to the constraint of selecting at most `k` employees.
