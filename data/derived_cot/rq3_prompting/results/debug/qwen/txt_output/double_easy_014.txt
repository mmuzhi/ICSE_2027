To solve this problem, we need to find the bitwise complement of a given integer. The bitwise complement operation involves flipping all the bits of the binary representation of the number. The challenge is to correctly implement this operation while handling edge cases such as zero and ensuring the result is accurate for all non-negative integers.

### Approach
1. **Problem Analysis**: The task is to flip each bit of the input number. For example, the binary representation of 5 (101) should become 010, which is 2 in decimal. The key observation is that we need to process each bit of the number from least significant to most significant, flipping each bit (0 becomes 1 and vice versa), and then combine these bits to form the result.

2. **Intuition**: By repeatedly dividing the number by 2 (integer division), we can isolate each bit from the least significant to the most significant. For each bit, if it is 1, we append '0' to the result string, and if it is 0, we append '1'. After processing all bits, the result string is reversed to form the correct binary representation, which is then converted to an integer.

3. **Algorithm Selection**: The algorithm involves a loop that continues until the number becomes zero. In each iteration, the least significant bit is checked, flipped, and then the number is right-shifted by dividing by 2. The flipped bits are stored in a string, which is reversed at the end to get the correct binary representation.

4. **Complexity Analysis**: The time complexity is O(log n) because each iteration processes one bit of the number. The space complexity is also O(log n) due to the storage required for the binary string.

### Solution Code
```python
class Solution:
    def findComplement(self, num: int) -> int:
        com = ''
        while num > 0:
            if num & 1:
                com += '0'
            else:
                com += '1'
            num = num // 2
        return int(com[::-1], 2)
```

### Explanation
1. **Initialization**: We start with an empty string `com` to store the binary complement.
2. **Loop Until Zero**: The loop continues as long as the number `num` is greater than zero.
3. **Bit Processing**: In each iteration, the least significant bit of `num` is checked using `num & 1`. If the bit is set (1), we append '0' to `com`; otherwise, we append '1'.
4. **Right Shift**: The number is right-shifted by performing integer division by 2 (`num = num // 2`), effectively removing the least significant bit.
5. **Result Construction**: After processing all bits, the string `com` is reversed (`com[::-1]`) to form the correct binary representation. This string is then converted to an integer using base 2 (`int(com[::-1], 2)`), which gives the desired complement.

This approach efficiently handles the bitwise complement operation by leveraging bit manipulation and string reversal, ensuring correctness for all non-negative integers.
