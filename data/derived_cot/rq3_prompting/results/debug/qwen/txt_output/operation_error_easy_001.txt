=== RAW ANSWER ===

To solve this problem, we need to determine if the frequency of each unique element in the array is unique. This means that no two different elements should share the same frequency count. 

### Approach
1. **Count Frequencies**: Use the `Counter` class from the `collections` module to count the occurrences of each element in the array. This gives us a dictionary where keys are the unique elements and values are their respective counts.
2. **Check Uniqueness of Frequencies**: Convert the values (frequencies) of the dictionary into a set to eliminate duplicates. If the number of unique frequencies matches the number of unique elements (i.e., the length of the dictionary), then all frequencies are unique. Otherwise, there are duplicate frequencies.

### Solution Code
```python
from collections import Counter

class Solution:
    def uniqueOccurrences(self, arr: List[int]) -> bool:
        d = Counter(arr)
        return len(d) == len(set(d.values()))
```

### Explanation
1. **Counting Frequencies**: The `Counter` class efficiently counts the occurrences of each element in the input array. For example, if the input array is `[1, 1, 2, 3]`, the `Counter` will produce `{1: 2, 2: 1, 3: 1}`.
2. **Checking Uniqueness**: By converting the values of the `Counter` dictionary into a set, we remove any duplicate frequency counts. If the length of this set equals the number of unique elements (i.e., the size of the dictionary), it means all frequencies are unique. For the example `[1, 1, 2, 3]`, the set of frequencies is `{1, 2}`, which has a length of 2, matching the number of unique elements (3 elements but frequencies are 2 and 1, so unique frequencies are 2). Hence, the function returns `True`. If frequencies were not unique, the set size would be less, and the function returns `False`.

This approach efficiently checks for the uniqueness of element frequencies using basic set operations, ensuring optimal performance and correctness.
