=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays efficiently. The solution involves merging the two arrays while maintaining their sorted order and then computing the median based on the merged array's length. The key insight is to avoid fully merging the arrays, which could be inefficient for large arrays, but instead, we can use a two-pointer technique to merge them in linear time.

### Approach
1. **Merge Two Sorted Arrays**: We use two pointers to traverse both arrays simultaneously, comparing elements and appending the smaller one to a new merged list. This ensures the merged list remains sorted.
2. **Compute Median**: After merging, the median is determined based on the length of the merged list. If the length is even, the median is the average of the two middle elements. If the length is odd, the median is the middle element.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        new = []
        p1 = 0
        p2 = 0
        
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
                
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
            
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
            
        if len(new) % 2 == 0:
            index = len(new) // 2
            median = (new[index] + new[index-1]) / 2
        else:
            median = float(new[len(new)//2])
        return median
```

### Explanation
1. **Initialization**: We start by initializing an empty list `new` to store the merged elements and two pointers `p1` and `p2` to traverse `nums1` and `nums2` respectively.
2. **Merging Process**: The first while loop runs as long as both pointers are within their respective array bounds. In each iteration, the smaller of the two current elements (pointed to by `p1` and `p2`) is appended to `new`, and the corresponding pointer is incremented.
3. **Remaining Elements**: After one array is exhausted, the remaining elements from the other array are appended to `new`.
4. **Median Calculation**: The median is calculated based on the length of `new`. If the length is even, the median is the average of the two middle elements. If the length is odd, the median is the middle element.

This approach efficiently merges the two arrays in linear time and computes the median in constant time, making it optimal for large input sizes.
