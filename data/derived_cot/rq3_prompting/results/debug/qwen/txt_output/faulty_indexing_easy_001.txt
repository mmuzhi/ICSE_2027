=== RAW ANSWER ===

To solve this problem, we need to find the longest harmonious subsequence in an array of integers. A harmonious subsequence is defined as a subsequence where the difference between the maximum and minimum elements is exactly 1. The solution involves counting the frequency of each number in the array and then checking for pairs of consecutive numbers (like n and n+1) to determine the longest possible subsequence.

### Approach
1. **Frequency Map Creation**: First, we create a frequency map (or hash map) to count the occurrences of each number in the input array. This helps in efficiently accessing the counts of numbers during the next step.
2. **Finding Consecutive Pairs**: For each number in the frequency map, we check if the next consecutive number (i.e., num + 1) exists in the map. If it does, the potential length of the harmonious subsequence is the sum of the counts of the current number and its consecutive number.
3. **Tracking Maximum Length**: We keep track of the maximum length encountered during the iteration over the frequency map. If no consecutive numbers are found, the result remains 0, indicating no harmonious subsequence exists.

### Solution Code
```python
class Solution:
    def findLHS(self, nums: List[int]) -> int:
        result = 0
        count_map = {}
        for num in nums:
            if num not in count_map:
                count_map[num] = 1
            else:
                count_map[num] += 1
        for num, count in count_map.items():
            if num + 1 in count_map:
                result = max(result, count + count_map[num + 1])
        return result
```

### Explanation
- **Frequency Map Creation**: The code starts by initializing an empty dictionary `count_map` to store the frequency of each number in the input array. It then iterates through the array, updating the dictionary to reflect the count of each number.
- **Finding Consecutive Pairs**: For each number in the dictionary, the code checks if the next consecutive number (num + 1) exists in the dictionary. If it does, the sum of the counts of the current number and its consecutive number is compared with the current maximum length (`result`). The larger value is retained.
- **Result Calculation**: After processing all numbers, the code returns the maximum length found. If no consecutive numbers are found, the result remains 0, indicating no harmonious subsequence exists.

This approach efficiently checks all possible harmonious subsequences by leveraging the frequency map, ensuring optimal performance with a time complexity of O(n), where n is the number of elements in the input array.
