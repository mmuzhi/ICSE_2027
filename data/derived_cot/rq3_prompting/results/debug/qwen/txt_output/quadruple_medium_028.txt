To solve this problem, we need to find the maximum gap between any two consecutive elements in a sorted list of numbers. The initial code has two main issues: a syntax error in initializing the minimum gap and a typo in the variable used for calculating the gap between consecutive elements.

### Approach
1. **Sort the List**: The first step is to sort the input list of numbers in ascending order. This allows us to easily compute the gaps between consecutive elements.
2. **Handle Edge Cases**: If the list has fewer than two elements, return 0 since no gaps can exist.
3. **Calculate Gaps**: Iterate through the sorted list, compute the absolute difference between each pair of consecutive elements, and keep track of the maximum gap encountered.
4. **Return Result**: After processing all consecutive pairs, return the maximum gap found.

### Solution Code
```python
class Solution:
    def maximumGap(self, nums: list) -> int:
        nums = sorted(nums)
        min_gap = float("inf")
        if len(nums) < 2:
            return 0
        for i in range(len(nums) - 1):
            gap = abs(nums[i] - nums[i+1])
            if min_gap < gap:
                min_gap = gap
        return min_gap
```

### Explanation
1. **Sorting the List**: The input list `nums` is sorted to ensure that the elements are in ascending order, which simplifies the process of finding the maximum gap between consecutive elements.
2. **Edge Case Handling**: If the list contains fewer than two elements, the function immediately returns 0 because no gaps can be computed.
3. **Gap Calculation**: The loop iterates through each pair of consecutive elements in the sorted list. The absolute difference between each pair is calculated and compared to the current maximum gap (`min_gap`). If a larger gap is found, `min_gap` is updated.
4. **Result Return**: After processing all consecutive pairs, the function returns the maximum gap found, which is stored in `min_gap`.

This approach efficiently computes the maximum gap by leveraging sorting and a single pass through the list, ensuring optimal performance with a time complexity of O(n log n) due to the sorting step.
