    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0

            # Check if mid is a peak with strict inequality and proper boundary conditions.

To solve this problem, we need to find a peak element in an array. A peak element is defined as an element that is strictly greater than its neighbors. The array can have multiple peaks, and the solution should return the index of any one of them. The challenge is to efficiently locate a peak element using a binary search approach, which ensures optimal performance with a time complexity of O(log n).

### Approach
1. **Binary Search**: The solution leverages binary search to efficiently narrow down the search space. The key idea is to determine whether the current element is a peak by comparing it with its neighbors.
2. **Peak Condition**: An element is a peak if it is greater than both its left and right neighbors (if they exist). The binary search checks this condition for the middle element of the current search range.
3. **Adjusting Search Range**: If the middle element is not a peak, the search range is adjusted based on the comparison with the next element. If the middle element is less than or equal to the next element, the peak must be on the right side; otherwise, it must be on the left side.

### Explanation
1. **Initialization**: The binary search starts with `low` set to 0 and `high` set to the last index of the array. If the array has only one element, that element is trivially a peak.
2. **Binary Search Loop**: The loop continues as long as `low` is less than or equal to `high`.
3. **Peak Check**: For the middle index `mid`, check if it is a peak by verifying it is greater than its left neighbor (if it exists) and its right neighbor (if it exists).
4. **Adjusting Range**: 
   - If `mid` is the last element and not a peak, move the `high` pointer left.
   - If the middle element is not a peak and less than or equal to the next element, move the `low` pointer right.
   - Otherwise, move the `high` pointer left.
5. **Termination**: If the loop exits without finding a peak, return -1 (though the problem guarantees at least one peak exists, this handles edge cases gracefully).

This approach efficiently narrows down the search space by leveraging binary search and ensures optimal performance with logarithmic time complexity.
