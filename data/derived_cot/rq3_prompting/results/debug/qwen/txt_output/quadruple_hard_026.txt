=== RAW ANSWER ===

To solve this problem, we need to determine if one string can be transformed into another by swapping adjacent characters any number of times. The key observation here is that any permutation of a string can be achieved through adjacent swaps. Therefore, the solution reduces to checking if the two strings have the same characters with the same frequencies.

### Approach
1. **Frequency Check**: The primary condition for transforming one string into another via adjacent swaps is that both strings must have the same characters with the same counts. If the character counts differ, the transformation is impossible.
2. **Simplicity**: The solution involves a straightforward frequency comparison using a counter. If the counts match, the transformation is possible; otherwise, it is not.

### Solution Code
```python
from collections import Counter

class Solution:
    def isTransformable(self, s: str, t: str) -> bool:
        return Counter(s) == Counter(t)
```

### Explanation
- **Frequency Check**: The `Counter` from the `collections` module is used to count the occurrences of each character in both strings. If the counts of all characters in `s` match those in `t`, it means `s` can be transformed into `t` by swapping adjacent characters any number of times.
- **Efficiency**: The solution efficiently checks the character counts in linear time relative to the length of the strings, making it optimal for this problem. The use of `Counter` simplifies the implementation and ensures correctness.
