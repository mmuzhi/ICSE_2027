The sequence is constructed recursively, where each term is built by appending the complement of the previous term. The challenge is to efficiently determine the k-th bit using a recursive approach with proper handling of base cases and recursive conditions.

### Approach
1. **Base Case Handling**: When `N` is 1, the sequence is "01". The first bit (k=1) is '0' if the parameter `R` is True, otherwise '1'. The second bit (k=2) is the complement of the first bit, so it is '1' if `R` is False, otherwise '0'.
2. **Recursive Case**: For `N > 1`, the sequence length is `2^N`. The sequence is split into two halves:
   - **First Half**: The first `2^(N-1)` bits are the same as the (N-1)-th term.
   - **Second Half**: The remaining bits are the complement of the (N-1)-th term.
3. **Recursive Conditions**:
   - If `K` is in the first half, recursively find the bit in the (N-1)-th term without flipping `R`.
   - If `K` is the middle bit, recursively find the bit in the (N-1)-th term with `R` flipped.
   - If `K` is in the second half, recursively find the bit in the mirrored position (relative to the first half) with `R` flipped.

class Solution:
    def findKthBit(self, N, K, R=True):
