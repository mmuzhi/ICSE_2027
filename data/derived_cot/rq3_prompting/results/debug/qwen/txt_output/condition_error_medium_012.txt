To solve this problem, we need to perform a level-order traversal of a binary tree from bottom to top. The level-order traversal involves visiting each node level by level, starting from the root, and then moving to the next level. The challenge is to return the nodes' values in the order of the levels from bottom to top.

### Approach
1. **Problem Analysis**: The task is to traverse the binary tree level by level (BFS) and then reverse the order of the levels to get the bottom-up traversal. The key insight is to collect nodes level by level and then reverse the list of levels.
2. **Intuition**: Using a queue, we can process each level's nodes sequentially. For each node, we add its children to the queue if they exist. After processing all nodes in a level, we record their values and move to the next level. Finally, we reverse the list of levels to present the bottom-up order.
3. **Algorithm Selection**: Breadth-First Search (BFS) is suitable here because it naturally processes nodes level by level. We use a queue to keep track of nodes to be processed. For each level, we process all nodes in the queue, record their values, and enqueue their children (if any).
4. **Complexity Analysis**: The algorithm visits each node exactly once, resulting in a time complexity of O(n), where n is the number of nodes in the tree. The space complexity is also O(n) due to the storage required for the queue and the result list.

### Solution Code
```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
```

### Explanation
1. **Initial Check**: If the tree is empty (root is None), return an empty list.
2. **Queue Initialization**: Start with the root node in the queue.
3. **BFS Traversal**: While the queue is not empty, process each node in the current level:
   - For each node, append its value to the current level's list.
   - Add its left and right children to the queue if they exist.
4. **Level Collection**: After processing all nodes in the current level, append the level's values to the result list.
5. **Reverse Levels**: Finally, reverse the list of levels to get the bottom-up order and return it.

This approach efficiently traverses the tree level by level and ensures the result is presented in the desired bottom-up order.
