To solve this problem, we need to determine if we can transform string `s` into string `t` by removing some characters while maintaining the relative order of the remaining characters. The key insight is that each character in `t` must appear in `s` in the same order, and no smaller digit (in terms of numeric value) that is required by `t` should appear earlier in `s` than the current character being processed.

### Approach
1. **Frequency Check**: First, we check if `s` contains at least as many occurrences of each digit in `t` as required by `t`. If not, it's immediately impossible to form `t` from `s`.
2. **Order Check**: For each character in `t`, we ensure that there are no unmatched smaller digits (digits required by `t` that haven't been used yet) appearing earlier in `s`. If such a digit exists, we cannot form `t` from `s` because the order would be violated.
3. **Character Matching**: We process each character in `t` in order, removing the earliest occurrence of the digit in `s` that hasn't been used yet. If at any point the conditions are violated, we return `False`. If all characters in `t` are successfully matched in order, we return `True`.

### Solution Code
```python
import collections

class Solution:
    def isTransformable(self, s: str, t: str) -> bool:
        indexes = collections.defaultdict(collections.deque)
        for index, char in enumerate(s):
            digit = int(char)
            indexes[digit].append(index)
        
        count_t = {}
        for char in t:
            digit = int(char)
            count_t[digit] = count_t.get(digit, 0) + 1
        
        for digit, cnt in count_t.items():
            if digit not in indexes or len(indexes[digit]) < cnt:
                return False
        
        unique_t_values = set(count_t.keys())
        
        for char in t:
            digit = int(char)
            for d in range(digit):
                if d in unique_t_values and d in indexes and indexes[d] and indexes[d][0] < indexes[digit][0]:
                    return False
            indexes[digit].popleft()
            if len(indexes[digit]) == 0:
                del indexes[digit]
        
        return True
```

### Explanation
1. **Frequency Check**: We first build a dictionary `indexes` that maps each digit in `s` to a deque of its positions. Then, we count the frequency of each digit in `t` and verify that `s` contains at least these frequencies. If not, we return `False`.
2. **Order Check**: For each digit in `t`, we check all smaller digits (digits less than the current digit) that are required by `t`. If any such digit appears earlier in `s` than the current digit's earliest occurrence, we return `False` because the order would be violated.
3. **Character Matching**: We process each digit in `t` by removing its earliest occurrence from `s`. If, after removal, there are no more occurrences of that digit, we remove it from the dictionary. If all digits in `t` are processed without violating the order, we return `True`.

This approach efficiently checks both the frequency and order constraints to determine if `t` can be formed from `s` by removing characters while maintaining the original order.
