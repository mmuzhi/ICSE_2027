=== RAW ANSWER ===

To solve this problem, we need to determine two lists from a series of matches: one containing teams that have never lost any match (undefeated teams) and another containing teams that have lost exactly once. 

### Approach
1. **Problem Analysis**: The problem requires processing a list of matches where each match is represented as a list of three integers: two teams playing and the winner. The goal is to track the number of wins and losses for each team. 
2. **Intuition**: Teams that have never lost are those not present in the losers' dictionary. Teams that have lost exactly once are those in the losers' dictionary with a count of 1.
3. **Algorithm Selection**: We use two dictionaries, `winners` and `losers`, to count the number of wins and losses for each team. For each match, we increment the winner's count in the `winners` dictionary and the loser's count in the `losers` dictionary.
4. **Complexity Analysis**: The algorithm processes each match exactly once, making it O(n) time complexity, where n is the number of matches. The space complexity is O(m) in the worst case, where m is the number of distinct teams.

### Solution Code
```python
class Solution:
    def findWinners(self, matches: List[List[int]]) -> List[List[int]]:

        winners, losers = defaultdict(int), defaultdict(int)

        for match in matches:
            winner = match[2]
            winners[winner] += 1
            if match[0] != winner:
                losers[match[0]] += 1
            else:
                losers[match[1]] += 1

        res_1, res_2 = [], []

        for k, v in winners.items():
            if k not in losers:
                res_1.append(k)
        
        for k, v in losers.items():
            if v == 1:
                res_2.append(k)

        res_1.sort()
        res_2.sort()
        
        return [res_1, res_2]
```

### Explanation
1. **Initialization**: Two dictionaries, `winners` and `losers`, are initialized to keep track of the number of wins and losses for each team.
2. **Processing Matches**: For each match, the winner is identified (third element in the match list). The winner's count is incremented in the `winners` dictionary. The loser (either the first or second team, whichever is not the winner) is identified and their loss count is incremented in the `losers` dictionary.
3. **Building Result Lists**: 
   - The first list (`res_1`) includes all teams not present in the `losers` dictionary (teams with zero losses).
   - The second list (`res_2`) includes all teams in the `losers` dictionary with exactly one loss.
4. **Sorting**: Both result lists are sorted alphabetically to meet the output requirements.
5. **Return**: The solution returns a list containing the two processed lists.

This approach efficiently tracks wins and losses, processes each match in linear time, and ensures the results are correctly formatted and sorted.
