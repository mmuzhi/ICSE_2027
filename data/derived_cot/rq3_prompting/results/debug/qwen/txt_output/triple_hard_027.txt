=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays efficiently. The median is the middle value in an ordered list, or the average of the two middle values if the list length is even. The challenge is to compute this without merging the arrays entirely, which would take O(n) time and O(n) space.

### Approach
1. **Merge the Arrays**: We use a two-pointer technique to merge the two sorted arrays into a single sorted array. This involves comparing elements from both arrays and appending the smaller one to the result array until one of the arrays is exhausted.
2. **Handle Remaining Elements**: After one array is exhausted, we append the remaining elements of the other array to the result array.
3. **Calculate the Median**: Once the merged array is complete, we check its length. If the length is odd, the median is the middle element. If even, it's the average of the two middle elements.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        if not nums1 and not nums2:
            return 0.0
        
        p1 = 0
        p2 = 0
        new = []
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
        
        n = len(new)
        if n % 2 == 1:
            return float(new[n//2])
        else:
            index = n // 2
            return (new[index] + new[index-1]) / 2
```

### Explanation
1. **Initial Check**: The code first checks if both input arrays are empty and returns 0.0 if true, as the median of an empty array is undefined.
2. **Merging Arrays**: Using two pointers (`p1` and `p2`), the code iterates through both arrays, comparing elements and appending the smaller one to the result array (`new`). This ensures the merged array remains sorted.
3. **Appending Remaining Elements**: After one array is fully traversed, the remaining elements of the other array are appended to `new`.
4. **Finding the Median**: The length of `new` determines the median calculation. For an odd length, the median is the middle element. For an even length, it's the average of the two middle elements. The result is returned as a float to handle both integer and non-integer cases.

This approach efficiently merges the arrays in O(n) time and space, where n is the total number of elements in both arrays, and computes the median in constant time, making it optimal for the given problem constraints.
