=== RAW ANSWER ===

To solve this problem, we need to split a given linked list into `k` parts, where each part consists of a contiguous sequence of nodes. The parts should be as evenly sized as possible. If the list cannot be split evenly, the extra nodes should be distributed one by one from left to right.

### Approach
1. **Calculate the Length of the Linked List**: First, we traverse the linked list to determine its total number of nodes.
2. **Determine the Size of Each Part**: Using the total length and the number of parts `k`, we calculate the base size (`maxi`) of each part and the number of parts that will have an extra node (`remain`).
3. **Split the Linked List**: We traverse the linked list again, cutting it into parts of the calculated sizes. For each part, we set the `next` pointer of the last node in the part to `None` and store the start of the part in the result list.
4. **Handle Remaining Parts**: If there are fewer than `k` parts after splitting, we append `None` for the remaining parts.

### Solution Code
```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def splitListToParts(self, head: Optional[ListNode], k: int) -> List[Optional[ListNode]]:
        # Calculate the length of the linked list
        length = 0
        ptr = head
        while ptr:
            length += 1
            ptr = ptr.next
        
        # Calculate the base size and remaining nodes
        maxi = length // k
        remain = length % k
        
        # Create a list to store the size of each part
        arrange = []
        for i in range(k):
            if i < remain:
                arrange.append(maxi + 1)
            else:
                arrange.append(maxi)
        
        # Split the linked list into k parts
        parts = []
        current = head
        for size in arrange:
            parts.append(current)
            if size == 0:
                continue
            for _ in range(size - 1):
                current = current.next
            if current:
                temp = current.next
                current.next = None
                current = temp
        
        return parts
```

### Explanation
1. **Calculate Length**: The code first traverses the linked list to count the total number of nodes. This helps in determining how to split the list into `k` parts.
2. **Determine Part Sizes**: The base size of each part (`maxi`) is calculated as the integer division of the total length by `k`. The remainder (`remain`) indicates how many parts will have one extra node.
3. **Split the List**: The code then traverses the list again, cutting it into parts of the calculated sizes. For each part, it moves the `current` pointer `size - 1` times to reach the last node of the part, then sets the `next` pointer of this node to `None` to terminate the part.
4. **Handle Remaining Parts**: The result list contains the heads of each part. If the list cannot be split into `k` non-empty parts, the remaining parts in the result list will be `None`.

This approach ensures that the linked list is split into `k` parts as evenly as possible, with any extra nodes distributed from left to right. The solution efficiently handles edge cases such as an empty list or when `k` exceeds the number of nodes.
