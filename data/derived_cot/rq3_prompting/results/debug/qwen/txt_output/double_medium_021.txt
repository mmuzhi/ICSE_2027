To solve this problem, we need to perform a level order bottom-up traversal of a binary tree. The level order traversal means visiting nodes level by level from top to bottom, and the bottom-up part requires reversing the order of levels. The key challenge is to ensure that the nodes are traversed correctly and that the levels are collected in the right order.

### Approach
1. **Initialization**: Use a queue to facilitate the level-order traversal. Start by enqueueing the root node.
2. **Traversal**: For each level, process all nodes in the current queue. For each node, append its value to the current level's list. Then, enqueue its left and right children (if they exist) in that order to maintain the correct left-to-right order within the level.
3. **Level Collection**: After processing all nodes in the current level, append the collected values to the result list. This ensures that levels are collected from top to bottom.
4. **Reversal**: Finally, reverse the result list to present the levels from bottom to top.

### Solution Code
```python
from collections import deque
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def levelOrderBottom(self, root: Optional[TreeNode]) -> List[List[int]]:
        queue = deque()
        queue.append(root)
        lst = []
        while queue:
            levels = []
            level_size = len(queue)
            for _ in range(level_size):
                tmp = queue.popleft()
                if tmp:
                    levels.append(tmp.val)
                    if tmp.left:
                        queue.append(tmp.left)
                    if tmp.right:
                        queue.append(tmp.right)
            if levels:
                lst.append(levels)
        return lst[::-1]
```

1. **Initialization**: The queue starts with the root node. If the root is `None`, the queue remains empty, and the loop is skipped, returning an empty list.

3. **Level Collection**: After processing all nodes in the current level, the collected values are added to the result list (`lst`). This ensures levels are stored from top to bottom.
4. **Reversal**: The result list is reversed before returning, which presents the levels from bottom to top, achieving the bottom-up level order traversal.
