To solve this problem, we need to find the longest cycle in a directed graph where each node has exactly one outgoing edge. This is known as a functional graph, which consists of cycles with trees attached. The approach involves traversing each node and detecting cycles by tracking the time of first visit to each node during the traversal.

### Approach
1. **Initialization**: We initialize an array `v` to keep track of the time when each node was first visited. Initially, all nodes are unvisited (marked as 0).
2. **Traversal**: For each node, we perform a depth-first search (DFS) starting from that node. During the traversal:
   - If a node is encountered that has been visited in the current traversal (i.e., its entry in `v` starts with the current node's index), we calculate the cycle length as the difference between the current time and the time of the first visit to that node.
   - If a node is unvisited, we record its first visit time and continue the traversal.
3. **Cycle Detection**: The cycle length is determined by the difference in visit times between the current traversal and the first visit. The maximum cycle length encountered during all traversals is the answer.

### Solution Code
```python
class Solution:
    def longestCycle(self, edges: List[int]) -> int:
        v = [0] * len(edges)
        ans = -1
        for i in range(len(edges)):
            t = 1
            c = i
            while c >= 0:
                if v[c] != 0:
                    if v[c][0] == i:
                        ans = max(ans, t - v[c][1])
                        break
                else:
                    v[c] = [i, t]
                    t += 1
                    c = edges[c]
        return ans
```

### Explanation
- **Initialization**: The array `v` is initialized to keep track of the time each node was first visited. The variable `ans` is initialized to -1 to store the longest cycle length found.
- **Traversal Loop**: For each node `i`, we start a traversal. The variable `t` is initialized to 1, representing the start time of the traversal.
- **Cycle Detection**: During the traversal, if a node `c` is found that was visited in the current traversal (i.e., `v[c][0] == i`), the cycle length is calculated as `t - v[c][1]` and stored in `ans`. The traversal breaks after detecting a cycle to avoid redundant checks.
- **Unvisited Nodes**: If a node is unvisited, its first visit time is recorded in `v`, and the traversal continues to the next node via the edge.
- **Edge Handling**: If an edge points to a node that has already been visited in the current traversal, the cycle is detected and the traversal breaks. If the edge points to -1, the traversal ends as `c` becomes -1.

This approach efficiently detects cycles by leveraging the functional graph property where each node has exactly one outgoing edge, ensuring that cycles are uniquely identifiable through visit times. The algorithm runs in O(n) time, where n is the number of nodes, as each node and edge is processed exactly once.
