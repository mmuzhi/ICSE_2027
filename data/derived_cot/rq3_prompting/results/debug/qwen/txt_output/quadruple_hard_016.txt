=== RAW ANSWER ===

To solve this problem, we need to find the median of two sorted arrays. The median is the middle value in an ordered list of numbers. If the combined length of the two arrays is even, the median is the average of the two middle numbers. 

### Approach
The approach involves merging the two sorted arrays into a single sorted array and then determining the median based on the length of the merged array. The steps are as follows:
1. **Merge the Arrays**: Use two pointers to traverse both arrays simultaneously, appending the smaller element to a new list each time.
2. **Handle Remaining Elements**: After one array is exhausted, append the remaining elements of the other array to the new list.
3. **Calculate the Median**: If the merged array has an even length, compute the average of the two middle elements. If the length is odd, the median is the middle element.

### Solution Code
```python
class Solution:
    def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
        new = []
        p1 = 0
        p2 = 0
        
        while p1 < len(nums1) and p2 < len(nums2):
            if nums1[p1] < nums2[p2]:
                new.append(nums1[p1])
                p1 += 1
            else:
                new.append(nums2[p2])
                p2 += 1
        
        while p1 < len(nums1):
            new.append(nums1[p1])
            p1 += 1
        
        while p2 < len(nums2):
            new.append(nums2[p2])
            p2 += 1
        
        if len(new) % 2 == 0:
            index = len(new) // 2
            median = (new[index] + new[index-1]) / 2
        else:
            median = float(new[len(new)//2])
        
        return median
```

### Explanation
1. **Initialization**: We start by initializing an empty list `new` to store the merged elements and two pointers `p1` and `p2` to traverse `nums1` and `nums2` respectively.
2. **Merging Process**: In the first loop, we compare elements from both arrays and append the smaller element to `new`, advancing the corresponding pointer. This continues until one of the arrays is exhausted.
3. **Appending Remaining Elements**: The subsequent loops handle the remaining elements of the non-exhausted array by simply appending them to `new`.
4. **Median Calculation**: Depending on whether the merged array length is even or odd, we compute the median. For an even length, the median is the average of the two middle elements; for an odd length, it is the middle element.

This approach efficiently merges the arrays and calculates the median in linear time relative to the combined length of the input arrays, ensuring optimal performance.
