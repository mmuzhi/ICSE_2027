To solve this problem, we need to reverse every k-group in a linked list. A k-group is defined as a sequence of k nodes. The challenge is to reverse each of these groups while maintaining the order of the groups and the nodes within each group.

### Approach
1. **Dummy Node and Initial Setup**: We start by creating a dummy node to simplify edge cases, such as when the head of the list is None.
2. **Traversal and Group Identification**: Using a helper function `getKth`, we traverse the list to find the k-th node from the current position. This helps in identifying the start and end of each group.
3. **Reversal of the Group**: Once the k-th node is found, we reverse the group of k nodes starting from the current position. This involves standard linked list reversal techniques but with careful handling to ensure the next group is correctly linked.
4. **Updating Pointers**: After reversing the group, we update the pointers to connect the reversed group to the rest of the list. The dummy node's next pointer is updated to the start of the reversed group, and the current group pointer is moved to the next group's start.

```python
class Solution:
    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        dummy = ListNode(0, head)
        currGroup = dummy

        while True:
            kth = self.getKth(currGroup, k)
```
