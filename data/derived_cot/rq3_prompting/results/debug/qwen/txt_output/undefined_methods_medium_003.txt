To solve this problem, we need to determine the minimum eating speed Koko needs to eat all the bananas within `h` hours. The solution involves using binary search to efficiently find the minimal eating speed that satisfies the condition where the total time taken to eat all bananas is less than or equal to `h` hours.

### Approach
1. **Binary Search Setup**: The binary search is performed over the possible eating speeds, ranging from 1 to the maximum number of bananas in any pile. This range is chosen because Koko must at least eat one banana per hour and can't eat more than the largest pile in one hour if she were to eat all bananas in one go.
2. **Check Function**: For each candidate eating speed `x`, the `check` function calculates the total hours required to eat all bananas by summing the ceiling of each pile's bananas divided by `x`. This is done using integer arithmetic to avoid floating-point operations, ensuring correctness for positive integers.
3. **Binary Search Execution**: The binary search adjusts the search range based on whether the current speed `x` allows Koko to eat all bananas within `h` hours. If the total hours exceed `h`, the search moves to higher speeds; otherwise, it searches for lower speeds to find the minimal valid speed.

### Solution Code
```python
class Solution:
    def minEatingSpeed(self, piles: List[int], h: int) -> int:
        def check(x):
            return sum((ele + x - 1) // x for ele in piles) <= h

        l = 1
        r = max(piles)
        while l < r:
            mid = (l + r) // 2
            if not check(mid):
                l = mid + 1
            else:
                r = mid
        return l
```

### Explanation
1. **Binary Search Initialization**: The search starts with `l` set to 1 (minimum possible speed) and `r` set to the maximum number of bananas in any pile (maximum possible speed).
2. **Mid Calculation**: In each iteration, the midpoint `mid` is calculated as the average of `l` and `r`. This midpoint represents the current candidate eating speed.
3. **Check Function**: The `check` function computes the total time required to eat all bananas at speed `mid`. For each pile, the time taken is the ceiling of the division of bananas by the speed, calculated using integer arithmetic `(ele + mid - 1) // mid`.
4. **Adjust Search Range**: If the total time exceeds `h`, the search continues in the upper half by setting `l` to `mid + 1`. Otherwise, the search continues in the lower half by setting `r` to `mid`.
5. **Termination**: The loop terminates when `l` equals `r`, which is the minimal eating speed that allows Koko to eat all bananas within `h` hours. This value is then returned as the solution.

This approach efficiently narrows down the possible speeds using binary search, ensuring optimal performance with a time complexity of O(n log m), where `n` is the number of piles and `m` is the maximum number of bananas in any pile.
