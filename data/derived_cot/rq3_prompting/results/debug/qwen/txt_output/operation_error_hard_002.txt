### Approach
The approach involves using dynamic programming to efficiently count all such slices. The key idea is to recognize that for any element at index `i`, the number of new arithmetic slices ending at `i` depends on the difference between `nums[i]` and `nums[i-1]`. If this difference matches the difference between `nums[i-1]` and `nums[i-2]`, then we can extend the slices ending at `i-1` to include `i`. Otherwise, we start a new slice at `i`.

1. **Dynamic Programming Setup**: We use an array `dp` where `dp[i]` represents the number of arithmetic slices ending at index `i`.
2. **Iteration**: For each index `i` starting from 2, we check if the difference between consecutive elements (from `i-1` to `i` and `i-2` to `i-1`) is the same. If they are, we increment `dp[i]` by `dp[i-1] + 1` (extending existing slices plus a new slice of length 2). If not, `dp[i]` is reset to 0.
3. **Summing Results**: The total number of arithmetic slices is the sum of all `dp[i]` values for `i` from 2 to `n-1`.

This approach efficiently counts all possible arithmetic slices in O(n) time with O(n) space complexity.

class Solution:
