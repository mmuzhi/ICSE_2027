=== RAW ANSWER ===

To solve this problem, we need to determine the K-th bit in the N-th term of a sequence defined recursively. The sequence starts with "0", and each subsequent term is formed by appending the previous term, its complement, and then the previous term again. The challenge is to efficiently find the K-th bit without generating the entire sequence up to the N-th term.

### Approach
1. **Base Case Handling**: The first bit (K=1) of the first term (N=1) is either '0' or '1' based on the flag `R`.
2. **Midpoint Calculation**: For each term, calculate the midpoint `mid` as \(2^{(N-1)}\). This midpoint divides the term into three parts:
   - The first part is the (N-1)-th term.
   - The second part is the complement of the (N-1)-th term.
   - The third part is again the (N-1)-th term.
3. **Recursive Handling**:
   - If `K` is in the first half (K ≤ mid), the bit is the same as the K-th bit in the (N-1)-th term.
   - If `K` is exactly at the midpoint, the bit is the complement of the (2*mid - K)-th bit in the (N-1)-th term.
   - If `K` is in the second half (K > mid), the bit is the complement of the (2*mid - K)-th bit in the (N-1)-th term.

### Solution Code
```python
class Solution:
    def findKthBit(self, N, K, R=True):
        if K == 1:
            return '0' if R else '1'
        mid = (1 << (N - 1))
        if K <= mid:
            return self.findKthBit(N - 1, K, R)
        elif K == mid:
            return self.findKthBit(N - 1, 2 * mid - K, not R)
        else:
            bit = self.findKthBit(N - 1, 2 * mid - K, not R)
            return '1' if bit == '0' else '0'
```

### Explanation
- **Base Case**: The function checks if `K` is 1. If so, it returns '0' if `R` is True, otherwise '1'.
- **Midpoint Calculation**: The midpoint `mid` is calculated as \(2^{(N-1)}\), which splits the current term into three parts.
- **Recursive Calls**:
  - For `K` in the first half (K ≤ mid), the function recursively calls itself with `N-1`, the same `K`, and `R` unchanged.
  - For `K` exactly at the midpoint, the function recursively calls itself with `N-1`, the mirrored position (2*mid - K), and `R` inverted.
  - For `K` in the second half (K > mid), the function recursively calls itself with `N-1`, the mirrored position (2*mid - K), and `R` inverted. The result is then complemented (i.e., '0' becomes '1' and vice versa).
This approach efficiently narrows down the search space by leveraging the recursive structure of the sequence, ensuring optimal performance without explicitly constructing the entire sequence.
