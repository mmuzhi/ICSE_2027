   - **Base-(-2) Addition (addNegabinary)**: Similar to base-2, the code processes bits from least to most significant. However, the carry handling is different due to the negative base. Additionally, trailing zeros in the result need to be removed before reversing the result.

2. **Key Insights**:
   - For both methods, the result list is built from least significant bit to most significant bit, so reversing the result is necessary.
   - The `addNegabinary` method requires removing trailing zeros before reversing to ensure the result is in the correct format.

3. **Algorithm Selection**:
   - **Base-2 Addition**: Process each bit from the least significant to the most, compute the sum with carry, and reverse the result.
   - **Base-(-2) Addition**: Similar to base-2, but handle the carry differently (using negative carry). After processing all bits, remove trailing zeros and then reverse the result.

4. **Complexity Analysis**:
   - **Time Complexity**: O(n), where n is the maximum length of the input lists. Each bit is processed exactly once.
   - **Space Complexity**: O(n), for storing the result.

### Solution Code
```python
class Solution:
    def addBinary(self, A, B):
        res = []
        carry = 0
        while A or B or carry:
            carry += (A or [0]).pop() + (B or [0]).pop()
            res.append(carry & 1)
            carry = carry >> 1
        res.reverse()
        return res

    def addNegabinary(self, A, B):
        res = []
        carry = 0
        while A or B or carry:
            carry += (A or [0]).pop() + (B or [0]).pop()
            res.append(carry & 1)
            carry = -(carry >> 1)
        while len(res) > 1 and res[-1] == 0:
            res.pop()
        res.reverse()
        return res
```

### Explanation
- **Base-2 Addition (addBinary)**:
  - The method processes each bit from the least significant to the most significant, summing the bits along with any carry.
  - The result bits are stored in reverse order, so reversing the result list at the end gives the correct binary representation.

- **Base-(-2) Addition (addNegabinary)**:
  - The method handles each bit similarly but uses a negative carry due to the negative base.
  - Trailing zeros are removed from the result before reversing to ensure the output is in the correct format without unnecessary leading zeros.
  - The carry is negated during each iteration to account for the negative base.

This approach ensures both methods correctly handle their respective base systems and produce the expected results in the correct format.
