=== RAW ANSWER ===

To solve this problem, we need to compute the median for each sliding window of size `k` in an array `nums`. The median is the middle value in an ordered list of numbers. If the window size `k` is even, the median is the average of the two middle values.

### Approach
The approach involves maintaining a balanced binary search tree (BST) to efficiently keep track of the elements in the current window. The BST allows us to insert and remove elements in logarithmic time and retrieve the k-th smallest element efficiently. The steps are as follows:

1. **Insertion**: For each element in the array, insert it into the BST.
2. **Removal**: If the window size exceeds `k`, remove the element that is no longer in the window (the element at position `i - k`).
3. **Median Calculation**: After ensuring the BST contains exactly `k` elements, compute the median. For odd `k`, the median is the middle element. For even `k`, it is the average of the two middle elements.

The BST operations (insert, remove, size, and k-th smallest element retrieval) are assumed to be implemented elsewhere, as they are not part of the solution code but are necessary for the sliding window median calculation.

### Solution Code
```python
class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        tree = None
        ans = []
        for i, x in enumerate(nums):
            tree = insert(tree, x)
            if i >= k:
                tree = remove(tree, nums[i - k])
            if size(tree) == k:
                if k % 2 == 1:
                    ans.append(get(tree, k // 2 + 1))
                else:
                    ans.append((get(tree, k // 2) + get(tree, k // 2 + 1)) / 2)
        return ans
```

### Explanation
1. **Initialization**: The BST (`tree`) is initialized as `None`, and the result list (`ans`) is empty.
2. **Iteration**: For each element in the array:
   - **Insertion**: The current element is inserted into the BST.
   - **Removal**: If the current index `i` is at least `k`, the element that is no longer in the window (at position `i - k`) is removed from the BST.
   - **Median Calculation**: If the BST contains exactly `k` elements, the median is calculated. For odd `k`, the middle element is retrieved. For even `k`, the average of the two middle elements is computed.
3. **Result**: The results for all windows are collected in the `ans` list and returned after processing all elements.

This approach efficiently maintains the sliding window and computes the median using BST operations, ensuring optimal performance for large input sizes.
