=== RAW ANSWER ===

To solve this problem, we need to determine the maximum profit achievable by buying and selling a stock given an array of daily prices, where multiple transactions are allowed but short selling is not permitted. The solution involves identifying all upward trends in the price array, where buying and selling on consecutive days with a price increase maximizes the profit.

### Approach
The approach to solve this problem efficiently is to iterate through the price array once and sum all the positive differences between consecutive days. This works because each upward trend (where the price increases from one day to the next) represents a profitable transaction. By summing these positive differences, we account for all possible profitable transactions without needing to track buy and sell points explicitly.

1. **Initialization**: Start with a profit of 0.
2. **Iteration**: Traverse the price array from the second day to the last day.
3. **Profit Calculation**: For each day, if the price is higher than the previous day, add the difference to the total profit.
4. **Return Result**: After processing all days, return the accumulated profit.

This approach ensures that we capture every possible profitable transaction in linear time, making it optimal with a time complexity of O(n), where n is the number of days.

### Solution Code
```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        profit = 0
        for i in range(1, len(prices)):
            if prices[i] > prices[i-1]:
                profit += prices[i] - prices[i-1]
        return profit
```

### Explanation
- **Initialization**: The variable `profit` is initialized to 0, which will hold the total profit from all transactions.
- **Iteration**: The loop runs from the second day (index 1) to the last day of the price array. For each day, it checks if the current day's price is higher than the previous day's price.
- **Profit Calculation**: If the current day's price is higher, the difference between the current and previous prices is added to `profit`. This difference represents the profit from buying on the previous day and selling on the current day.
- **Return Result**: After processing all days, the total profit is returned. This method efficiently captures all profitable transactions by leveraging the fact that every upward trend in consecutive days contributes to the maximum profit.

This solution is both time and space efficient, with O(n) time complexity and O(1) space complexity, making it suitable for large input arrays.
