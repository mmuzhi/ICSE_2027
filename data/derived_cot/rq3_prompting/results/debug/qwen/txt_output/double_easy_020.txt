To solve this problem, we need to find the third maximum distinct number in a given list of integers. If there are fewer than three distinct numbers, we should return the maximum number from the list. 

### Approach
1. **Problem Analysis**: The task involves identifying the third largest unique number in a list. If there are fewer than three distinct numbers, the solution should return the maximum number. 
2. **Intuition**: By converting the input list into a set, we eliminate duplicate values, leaving us with only distinct numbers. We then check if there are at least three distinct numbers. If not, we return the maximum number from the set. Otherwise, we sort the set and return the third element from the end (which is the third largest number).
3. **Algorithm Selection**: The algorithm involves:
   - Converting the input list to a set to get distinct numbers.
   - Checking the size of the set. If it has fewer than three elements, return the maximum of the set.
   - Otherwise, sort the set and return the third largest element.
4. **Complexity Analysis**: 
   - **Time Complexity**: Converting the list to a set takes O(n) time. Sorting the set takes O(k log k) time, where k is the number of distinct elements. In the worst case, k can be O(n), making the overall complexity O(n log n).
   - **Space Complexity**: The space required is O(k) for storing the distinct elements in the set.

### Solution Code
```python
class Solution:
    def thirdMax(self, nums) -> int:
        s = set(nums)
        return max(s) if len(s) < 3 else sorted(s)[-3]
```

### Explanation
1. **Initialization**: The input list `nums` is converted into a set `s` to remove duplicates.
2. **Condition Check**: If the set `s` has fewer than three elements, the maximum value in the set is returned.
3. **Sorting and Selection**: If there are three or more distinct numbers, the set is sorted in ascending order. The third largest element is then accessed using negative indexing (`[-3]`), which retrieves the element at position `-3` (the third from the end in a zero-indexed list).
This approach efficiently handles both scenarios (fewer than three distinct numbers and three or more distinct numbers) by leveraging set operations and sorting, ensuring correctness and simplicity.
