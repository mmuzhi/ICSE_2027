1. **Initialization**: Set the initial search range from `low = 0` to `high = n-1`, where `n` is the length of the array.
2. **Binary Search Loop**: While `low <= high`, compute the middle index `mid`.
3. **Peak Check**: If the middle element is greater than both its neighbors (or is the first/last element), return it as the peak.
4. **Adjust Search Range**: If the middle element is less than its right neighbor, move the search to the right half. Otherwise, move to the left half.

This approach efficiently reduces the problem size by half in each iteration, leading to a time complexity of O(log n).

### Solution Code
```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        low = 0
        high = n - 1
        
        while low <= high:
            mid = (low + high) // 2
            if (mid == 0 or nums[mid] > nums[mid-1]) and (mid == n-1 or nums[mid] > nums[mid+1]):
                return mid
            elif nums[mid] < nums[mid+1]:
                low = mid + 1
            else:
                high = mid - 1
        return -1
```

### Explanation
1. **Initialization**: The search range starts from the entire array, with `low` set to 0 and `high` set to `n-1`.
2. **Binary Search Loop**: The loop continues as long as `low` is less than or equal to `high`.
3. **Peak Check**: For the middle element `nums[mid]`, if it is greater than both its left and right neighbors (or is the first or last element), it is identified as a peak, and its index is returned.
4. **Adjust Search Range**: 
   - If `nums[mid]` is less than its right neighbor, the peak must be in the right half, so `low` is updated to `mid + 1`.
   - If `nums[mid]` is not less than its right neighbor, the peak must be in the left half, so `high` is updated to `mid - 1`.

This method efficiently narrows down the search space by leveraging binary search, ensuring optimal performance even for large arrays.
