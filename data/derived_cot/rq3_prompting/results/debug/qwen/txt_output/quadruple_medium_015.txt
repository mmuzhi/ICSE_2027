To solve this problem, we need to construct a Quad-Tree from a given 2D grid of integers. The Quad-Tree is a hierarchical data structure where each node can have up to four children, representing the four quadrants of the current node. The solution involves recursively dividing the grid into quadrants until each quadrant is a single cell (leaf node) or until all cells in a quadrant have the same value, allowing the node to be a leaf.

### Approach
1. **Problem Analysis**: The input is a grid of size n x n where each cell is either 0 or 1. The goal is to build a Quad-Tree where each leaf node represents a single cell, and non-leaf nodes have four children representing the four quadrants.
2. **Intuition**: The Quad-Tree can be constructed by recursively dividing the grid into four equal parts. For each part, if all cells in the part have the same value, the corresponding node is a leaf. Otherwise, the node is a non-leaf node with four children.
3. **Algorithm Selection**: The solution uses a recursive approach. The base case is when the grid part is 1x1 (leaf node). For larger grids, the grid is split into four quadrants, and each quadrant is processed recursively. If all four resulting nodes are leaves and have the same value, the current node is also a leaf; otherwise, it is a non-leaf node.
4. **Complexity Analysis**: The time complexity is O(n^2) because each cell in the grid is processed once. The space complexity is O(log n) due to the recursion stack depth, which is proportional to the grid's depth (logarithmic in n for a grid divided into quadrants).
