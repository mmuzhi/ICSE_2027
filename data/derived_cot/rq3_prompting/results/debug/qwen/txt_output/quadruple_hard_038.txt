1. **Problem Analysis**: The problem requires generating all possible permutations of numbers from 0 to n (inclusive) such that each subsequent number in the permutation adheres to the direction specified by the input string (either 'D' for decreasing or 'I' for increasing). The challenge is to efficiently count these permutations without enumerating all possible permutations explicitly, which would be computationally infeasible for larger n.

2. **Intuition**: The solution involves using dynamic programming with memoization to avoid recalculating subproblems. The state in our DP is defined by the current position in the string and the last number chosen. For each state, we consider all valid next numbers based on the current direction (either increasing or decreasing) and recursively compute the number of valid permutations from that state.

3. **Algorithm Selection**: We use a depth-first search (DFS) approach with memoization to explore all valid paths. The memoization stores the number of valid permutations starting from each state (current position and last number chosen) to avoid redundant calculations. The base case is when we reach the end of the string, indicating a complete permutation.

4. **Complexity Analysis**: The time complexity is O(n^2) where n is the length of the input string, as each state (i, val) is computed once, and for each state, we consider up to n possible next values. The space complexity is also O(n^2) due to the memoization table storing results for each state.

### Solution Code
```python
class Solution:
    def numPermsDISequence(self, s: str) -> int:
        n = len(s)
        mod = 10**9 + 7
        from collections import defaultdict
        mem = defaultdict(int)
        
        def dfs(i, val):
            if i == n:
                return 1
            key = (i, val)
            if key in mem:
                return mem[key]
            p = 0
            if s[i] == 'D':
                for ind in range(0, val):
                    p = (p + dfs(i+1, ind)) % mod
            else:
                for ind in range(val+1, n+1):
                    p = (p + dfs(i+1, ind)) % mod
            mem[key] = p
            return p
        
        return dfs(0, n)
```

### Explanation
1. **Initialization**: We start by defining the length of the input string `n` and the modulo value for result size. A `defaultdict` is used to memoize results of subproblems.
2. **DFS Function**: The `dfs` function takes the current index `i` and the last number `val` chosen. If we've reached the end of the string (`i == n`), it returns 1, indicating a valid permutation.
3. **Memoization Check**: Before processing a state, we check if it has been computed before using the `mem` dictionary to avoid redundant calculations.
4. **Direction Handling**: Depending on whether the current character is 'D' or 'I', we iterate over possible next numbers. For 'D', numbers must be less than `val`; for 'I', numbers must be greater than `val`.
5. **Modulo Operation**: The result is computed modulo \(10^9 + 7\) to handle large numbers and prevent overflow.
6. **Base Case**: When the entire string is processed, a valid permutation is counted (return 1).

This approach efficiently counts all valid permutations by leveraging memoization and dynamic programming, ensuring optimal performance even for larger input sizes.
