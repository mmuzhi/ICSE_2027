To solve this problem, we need to find the restaurant(s) that appear in both given lists, `list1` and `list2`, and return them in the order of the smallest index sum. The index sum for a restaurant is the sum of its indices in `list1` and `list2`. If multiple restaurants have the same smallest index sum, all of them should be included in the result.

### Approach
1. **Mapping Indices**: First, we create a dictionary to map each restaurant in `list2` to its index. This allows us to quickly look up the index of any restaurant from `list2`.
2. **Collecting Common Restaurants**: We iterate through `list1` and for each restaurant present in the dictionary (i.e., common to both lists), we calculate its index sum (sum of indices in `list1` and `list2`) and store it along with the restaurant name in a list.
3. **Handling Edge Cases**: If there are no common restaurants, we return an empty list.
4. **Sorting and Grouping**: We sort the list of index sums and restaurant names. Then, we collect all restaurants with the smallest index sum, ensuring they are in the order of their restaurant names if index sums are the same.

### Solution Code
```python
class Solution:
    def findRestaurant(self, list1: List[str], list2: List[str]) -> List[str]:
        d2 = {}
        for i in range(len(list2)):
            d2[list2[i]] = i
        
        l = []
        for i in range(len(list1)):
            if list1[i] in d2:
                l.append([i + d2[list1[i]], list1[i]])
        
        if not l:
            return []
        
        l.sort()
        l1 = [l[0][1]]
        for i in range(1, len(l)):
            if l[i][0] == l[i-1][0]:
                l1.append(l[i][1])
            else:
                break
        
        return l1
```

### Explanation
1. **Mapping Indices**: The dictionary `d2` is built by iterating over `list2`, storing each restaurant's index. This allows O(1) lookups for restaurant indices in `list2`.
2. **Collecting Common Restaurants**: For each restaurant in `list1`, if it exists in `d2`, we compute its index sum (current index in `list1` plus its index in `list2`) and store this sum along with the restaurant name in list `l`.
3. **Edge Case Handling**: If `l` is empty (no common restaurants), we return an empty list immediately.
4. **Sorting and Grouping**: The list `l` is sorted primarily by index sum and secondarily by restaurant name. We then iterate through the sorted list to collect all restaurants with the smallest index sum, appending their names to the result list `l1`. The loop breaks as soon as a different index sum is encountered, ensuring only the smallest sums are considered.

This approach efficiently finds the desired restaurants by leveraging dictionary lookups and careful sorting, ensuring optimal performance and correctness.
