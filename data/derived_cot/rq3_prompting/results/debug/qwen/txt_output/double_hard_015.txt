To solve this problem, we need to count the number of ways to cut a given pizza into exactly `k` pieces such that each piece contains at least one apple. The cuts can be made either horizontally or vertically, and each cut must separate at least one piece of apple. The solution involves dynamic programming with memoization to efficiently count the valid cutting configurations.

### Approach
1. **Prefix Sum Calculation**: We first compute two prefix sum arrays, `pfsum_row` and `pfsum_col`, to efficiently calculate the number of apples in any sub-pizza. `pfsum_row[i][j]` gives the number of apples in row `i` from column 0 to `j`, and `pfsum_col[i][j]` gives the number of apples in column `j` from row 0 to `i`.

2. **Total Apples Check**: Before proceeding, we check if the total number of apples in the pizza is at least `k+1` (since each of the `k` pieces must contain at least one apple). If not, return 0.

3. **Recursive Counting with Memoization**: Using a recursive function `getWays(i, j, k)`, we count the ways to cut the sub-pizza starting at position `(i, j)` with `k` remaining cuts. The function uses memoization to store intermediate results and avoid redundant calculations.

4. **Base Case (k = 1)**: When only one cut is left, we check if the sub-pizza can be cut horizontally or vertically into two parts, each containing at least one apple.

5. **Recursive Case (k > 1)**: For each possible horizontal or vertical cut, we ensure that both parts of the sub-pizza contain at least one apple. The function then recursively counts the ways to cut the resulting sub-pizzas with `k-1` cuts.
