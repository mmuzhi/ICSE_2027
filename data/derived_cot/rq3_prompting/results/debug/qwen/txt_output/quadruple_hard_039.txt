=== RAW ANSWER ===

To solve this problem, we need to compute the median of a sliding window in an array of integers. The sliding window moves one element at a time, and we need to efficiently compute the median for each window. The median is defined as the middle value in an ordered list of numbers. For an even number of elements, the median is the average of the two middle values.

### Approach
1. **Balanced Binary Search Tree (BST)**: The solution uses a balanced BST to maintain the elements of the current window. This allows efficient insertion, deletion, and median retrieval operations.
2. **Insertion and Deletion**: For each new element entering the window, it is inserted into the BST. When the window size exceeds `k`, the oldest element is removed from the BST.
3. **Median Calculation**: Depending on whether the window size `k` is odd or even, the median is calculated differently:
   - **Odd `k`**: The median is the middle element in the BST.
   - **Even `k`**: The median is the average of the two middle elements in the BST.
4. **Efficiency**: The BST operations (insertion, deletion, and median retrieval) are performed in logarithmic time, making the overall complexity O(n log k), where `n` is the number of elements in the array.

### Solution Code
```python
class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        tree = None
        ans = []
        for i, x in enumerate(nums):
            tree = insert(tree, x)
            if size(tree) > k:
                tree = remove(tree, nums[i - k + 1])
            if size(tree) == k:
                if k % 2 == 1:
                    ans.append(get(tree, k // 2 + 1))
                else:
                    a = get(tree, k//2)
                    b = get(tree, k//2+1)
                    ans.append((a + b) / 2.0)
        return ans
```

### Explanation
1. **Initialization**: The `tree` variable is initialized to `None`, and `ans` is an empty list to store the results.
2. **Iterating through the array**: For each element in the array:
   - **Insertion**: The current element is inserted into the BST.
   - **Window Size Check**: If the BST size exceeds `k`, the oldest element (at index `i - k + 1`) is removed from the BST.
   - **Median Calculation**: When the BST size equals `k`, the median is calculated:
     - For an odd `k`, the median is the middle element (k//2 + 1-th smallest element).
     - For an even `k`, the median is the average of the two middle elements (k//2-th and k//2 + 1-th smallest elements).
3. **Result Compilation**: The results are collected in the `ans` list and returned after processing all elements.

This approach efficiently maintains the sliding window and computes the median using a balanced BST, ensuring optimal performance for large input sizes.
