=== RAW ANSWER ===

To solve this problem, we need to determine if a given string containing '(', ')', and '*' is valid. The string is considered valid if all opening brackets '(' are closed by corresponding closing brackets ')', and any '*' can be treated as either an opening or closing bracket as needed. 

### Approach
The approach involves using two counters, `low` and `high`, to keep track of the minimum and maximum number of open brackets at any point while processing each character in the string. 
1. **Initialization**: Start with `low` and `high` set to 0. These variables represent the minimum and maximum number of open brackets possible at any point.
2. **Processing Each Character**:
   - **Opening Bracket '('**: Increment both `low` and `high` by 1, as this bracket adds to the open count.
   - **Closing Bracket ')'**: Decrement both `low` and `high` by 1, as this bracket closes an open bracket. If `low` becomes negative, it means the string is invalid.
   - **Wildcard '*'**: This can act as either an opening or closing bracket. Decrease `low` by 1 (as it can close an open bracket) and increase `high` by 1 (as it can open a new bracket). Ensure `low` does not go below 0.
3. **Check Validity**: After processing all characters, if `low` is 0, it means all open brackets were properly closed, making the string valid. Otherwise, the string is invalid.

### Solution Code
```python
class Solution:
    def checkValidString(self, s):
        low, high = 0, 0
        for c in s:
            if c == '(':
                low += 1
                high += 1
            elif c == ')':
                low -= 1
                high -= 1
            else:  # c == '*'
                low = max(low - 1, 0)
                high += 1
            if low < 0 or high < 0:
                return False
        return low == 0
```

### Explanation
- **Initialization**: The variables `low` and `high` start at 0, representing no open brackets initially.
- **Processing Characters**:
  - For each '(', both `low` and `high` increase, indicating an additional open bracket.
  - For each ')', both `low` and `high` decrease, indicating a closing bracket. If `low` becomes negative, the string is immediately deemed invalid.
  - For each '*', `low` is decremented (as it can close an open bracket) and set to a minimum of 0 (to avoid negative values), while `high` is incremented (as it can open a new bracket).
- **Validity Check**: After processing all characters, if `low` is 0, all open brackets were properly closed, and the string is valid. Otherwise, it is invalid.

This approach efficiently checks the validity of the string by dynamically adjusting the possible range of open brackets (`low` to `high`) and ensuring no point violates the balance condition.
