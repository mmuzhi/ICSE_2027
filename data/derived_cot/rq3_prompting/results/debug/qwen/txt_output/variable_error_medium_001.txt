### Approach
The approach involves using a binary search to efficiently locate a peak element. The key insight is that for any given element in the array, if it is greater than or equal to its next element, the peak must lie on the right side (including the current element). Conversely, if it is less than its next element, the peak must lie on the left side. 

1. **Binary Search Setup**: Initialize two pointers, `low` and `high`, to the start and end of the array, respectively.
2. **Binary Search Execution**: While `low` is less than `high`, compute the middle index `mid`.
3. **Comparison**: Compare the element at `mid` with the element at `mid+1`. If `nums[mid]` is greater than `nums[mid+1]`, it means the peak could be at `mid` or to its left. Adjust `high` to `mid`. Otherwise, adjust `low` to `mid+1`.
4. **Termination**: When `low` equals `high`, the loop terminates, and `low` (or `high`) is the index of a peak element.

This approach efficiently narrows down the search space by leveraging the properties of a peak element, ensuring optimal performance with a time complexity of O(log n).

```python
class Solution:
    def findPeakElement(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 1:
```
