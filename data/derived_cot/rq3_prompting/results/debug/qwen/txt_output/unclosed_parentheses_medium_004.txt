To solve this problem, we need to count the number of pairs of strings in the given array that concatenate to form the target string. The solution involves efficiently checking all possible pairs of strings in the array to see if their concatenation matches the target string.

### Approach
1. **Frequency Dictionary**: We first create a frequency dictionary to count occurrences of each string in the input array. This helps in quickly checking the availability of strings during the pairing process.
2. **Prefix Check**: For each string in the frequency dictionary, we check if it is a prefix of the target string. If it is, we determine the remaining part of the target string (suffix) that would complete the pair.
3. **Suffix Check**: We then check if this remaining part (suffix) exists in the frequency dictionary. If it does, we increment the count of valid pairs by the product of the frequencies of the prefix and the suffix strings.
4. **Edge Cases**: We handle cases where the prefix length exceeds the target length by skipping such strings. The solution assumes distinct strings in the input array, but the frequency dictionary accommodates potential duplicates.

### Solution Code
```python
from collections import defaultdict

class Solution:
    def numOfPairs(self, nums: List[str], target: str) -> int:
        d = defaultdict(int)
        for char in nums:
            d[char] += 1
        
        pairs = 0
        for s in d:
            if len(s) > len(target):
                continue
            if target.startswith(s):
                t1 = target[len(s):]
                if t1 in d:
                    pairs += d[s] * d[t1]
        return pairs
```

### Explanation
1. **Frequency Dictionary**: The code starts by building a dictionary (`d`) that maps each string in the input array to its frequency. This allows efficient lookups during the pairing process.
2. **Prefix Check**: For each string `s` in the dictionary, the code checks if `s` is a prefix of the target string. If `s` is a prefix, the remaining part of the target string (after removing `s`) is computed as `t1`.
3. **Suffix Check**: The code then checks if `t1` exists in the dictionary. If it does, the number of valid pairs is increased by the product of the frequencies of `s` and `t1`. This accounts for all possible combinations of `s` and `t1` that form the target string.
4. **Edge Handling**: Strings longer than the target are skipped to avoid unnecessary checks. The solution efficiently counts all valid pairs by leveraging the frequency dictionary and prefix checks.

This approach ensures that we efficiently count all valid pairs without resorting to a brute-force method, making it optimal for larger input sizes.
