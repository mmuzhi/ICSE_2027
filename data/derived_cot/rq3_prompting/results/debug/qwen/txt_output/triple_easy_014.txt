To solve this problem, we need to calculate the total score based on a series of operations applied to a stack of scores. The operations can be "D" to double the last score, "C" to remove the last score, "+" to add the last two scores, or a digit string representing a score.

### Approach
1. **Initialization**: We use a stack to keep track of the scores.
2. **Processing Operations**: For each operation in the input list:
   - If the operation is "D", double the last score in the stack and push it.
   - If the operation is "C", pop the last score from the stack.
   - If the operation is "+", add the last two scores in the stack and push the result.
   - For any other operation (a digit string), convert it to an integer and push it onto the stack.
3. **Summing Scores**: After processing all operations, sum all the scores in the stack to get the final result.

### Solution Code
```python
class Solution:
    def calPoints(self, operations: List[str]) -> int:
        stack = []
        for i in operations:
            if i == 'D':
                stack.append(2 * stack[-1])
            elif i == 'C':
                stack.pop()
            elif i == '+':
                stack.append(stack[-1] + stack[-2])
            else:
                stack.append(int(i))
        return sum(stack)
```

### Explanation
- **Stack Handling**: The stack is used to maintain the current scores dynamically as operations are processed.
- **Operations Processing**:
  - **"D" Operation**: The last score is doubled and pushed onto the stack.
  - **"C" Operation**: The last score is removed from the stack.
  - **"+" Operation**: The sum of the last two scores is computed and pushed onto the stack.
  - **Digit String**: The string is converted to an integer and pushed onto the stack.
- **Result Calculation**: The sum of all elements in the stack after processing all operations gives the total score.

This approach efficiently handles each operation in sequence, leveraging the stack to manage intermediate scores, and finally computes the total by summing the stack contents. The solution ensures correctness by addressing each operation type appropriately and handles edge cases such as empty stacks by relying on the problem's implicit assumption that operations are valid.
