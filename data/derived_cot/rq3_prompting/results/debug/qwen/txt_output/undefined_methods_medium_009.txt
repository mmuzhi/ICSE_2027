=== RAW ANSWER ===

To solve this problem, we need to partition a given string into the largest possible substrings such that each substring contains all occurrences of any character that appears in it. The solution involves using a dictionary to track the last occurrence of each character in the string and then iterating through the string to determine the optimal partition points.

### Approach
1. **Track Last Occurrences**: First, we create a dictionary that maps each character to its last occurrence index in the string. This helps in determining the farthest point each character appears.
2. **Iterate Through String**: Using two pointers, `start` and `end`, we traverse the string. For each character at position `start`, we determine the farthest `end` it can reach by considering the last occurrences of all characters encountered from `start` to `end`.
3. **Update Partitions**: Once the `end` is determined for the current partition, we record the length of the partition and move `start` to `end + 1` to continue processing the remaining string.

### Solution Code
```python
class Solution:
    def partitionLabels(self, s: str) -> List[int]:
        last_occurrence = {char: idx for idx, char in enumerate(s)}
        n = len(s)
        start = 0
        max_last = 0
        ans = []
        for i in range(n):
            if i > max_last:
                ans.append(max_last - start + 1)
                start = i
                max_last = i
            else:
                if last_occurrence[s[i]] > max_last:
                    max_last = last_occurrence[s[i]]
        ans.append(max_last - start + 1)
        return ans
```

### Explanation
1. **Tracking Last Occurrences**: The dictionary `last_occurrence` is built by iterating from the end of the string to the beginning, ensuring each character maps to its last index.
2. **Partition Determination**: The algorithm initializes `start` to 0 and `max_last` to 0. As it iterates through the string, for each character, it updates `max_last` to the maximum of its current value and the last occurrence of the current character.
3. **Optimal Partitioning**: Whenever the current index `i` exceeds `max_last`, it means we've reached the end of a valid partition. The length of this partition is recorded, and `start` is reset to the current index. Finally, after processing all characters, the last partition's length is appended to the result.

This approach efficiently partitions the string by leveraging the last occurrence information to ensure each partition contains all necessary characters without any duplicates, adhering to the problem constraints.
