To solve this problem, we need to find the closest pizza cost to a given target by selecting base costs and toppings (each topping can be used at most twice). The solution involves a dynamic programming approach with memoization to efficiently explore all possible topping combinations.

### Approach
1. **Problem Analysis**: The goal is to minimize the absolute difference between the total cost (base cost + selected toppings) and the target. Each topping can be used up to twice, which is handled by doubling the list of topping costs.
2. **Dynamic Programming with Memoization**: The function `fn(i, x)` computes the minimal cost from the toppings starting at index `i` to reach the remaining amount `x`. The function uses memoization to cache results of subproblems for efficiency.
3. **Base Cases**: If `x` becomes negative or all toppings are considered, return 0 (no more toppings can be added). Otherwise, recursively explore skipping the current topping or taking it, then choose the option with the smallest absolute difference to `x`.
4. **Main Loop**: For each base cost, compute the minimal topping cost to reach the remaining target (target - base cost) and update the answer with the best total cost (base cost + topping cost).

### Solution Code
```python
from functools import cache

class Solution:
    def closestCost(self, baseCosts: List[int], toppingCosts: List[int], target: int) -> int:
        toppingCosts *= 2
        
        @cache
        def fn(i, x):
            if x < 0 or i == len(toppingCosts):
                return 0
            option1 = fn(i+1, x)
            option2 = toppingCosts[i] + fn(i+1, x - toppingCosts[i])
            return min(option1, option2, key=lambda y: (abs(y - x), y))
        
        ans = float('inf')
        for bc in baseCosts:
            candidate = bc + fn(0, target - bc)
            if abs(candidate - target) < abs(ans - target) or (abs(candidate - target) == abs(ans - target) and candidate < ans):
```
