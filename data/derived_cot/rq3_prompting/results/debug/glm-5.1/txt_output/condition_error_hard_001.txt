        backward, res = {}, []

3. **Duplicate issue**: With the corrected checks, j=0 would produce pairs already handled by the separate full-reverse and empty-string checks. The original buggy checks accidentally prevented j=0 from firing. Fix: start loop from j=1.

Verification: With `range(1, len(word))`, case 1 at j=0 (full reverse pair `[backward[word], i]`) is already found by the first check when processing the reversed word. Case 2 at j=0 (palindrome + empty string `[i, backward[""]]`) is already handled by the explicit empty string check. No valid pairs are missed.

class Solution:
    def palindromePairs(self, words: List[str]) -> List[List[int]]:
        backward, res = {}, []
        for i, word in enumerate(words):
            backward[word[::-1]] = i

        for i, word in enumerate(words):
            
            if word in backward and backward[word] != i:
                res.append([i, backward[word]])
                
            if word != "" and "" in backward and word == word[::-1]:
                res.append([i, backward[""]])
                res.append([backward[""], i])

                    res.append([backward[word[j:]], i])

        return res
