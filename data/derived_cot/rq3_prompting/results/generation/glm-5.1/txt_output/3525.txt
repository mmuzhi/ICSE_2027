Problem: Maximize total energy boost over `n` hours. Two arrays `A` and `B`. Drink one per hour. Switching requires a 1-hour gap (0 boost). Start with either.
This is a standard DP problem.
State:
`dp_a[i]`: max energy at hour `i` if we drink A at hour `i`.
`dp_b[i]`: max energy at hour `i` if we drink B at hour `i`.
If we drink A at hour `i`, we could have either:
- Drunk A at hour `i-1`: `dp_a[i-1] + A[i]`
- Drunk B at hour `i-2`: `dp_b[i-2] + A[i]`
Wait, what if we didn't drink anything at hour `i-1`?
If we don't drink anything at hour `i-1`, it must be because we switched. So we drank B at `i-2`, skipped `i-1`, and drink A at `i`.
Is it ever optimal to skip an hour without switching? No, because all boosts are positive (1 <= A[i], B[i] <= 10^5). So we only skip if we switch.
Thus, the transitions are:
`dp_a[i] = A[i] + max(dp_a[i-1], dp_b[i-2] if i>=2 else 0)`
`dp_b[i] = B[i] + max(dp_b[i-1], dp_a[i-2] if i>=2 else 0)`

Wait, is it possible that we skip hour 0? No, we start with either, and skipping hour 0 gives 0, which is less than drinking either A[0] or B[0] since they are positive.
What about `i=1`?
`dp_a[1] = A[1] + max(dp_a[0], dp_b[-1])`. Since `dp_b[-1]` is not valid, we just use `dp_a[0]`.
Wait, if we drink B at hour 0, we can't drink A at hour 1 without skipping. So we skip hour 1? No, if we skip hour 1, we drink A at hour 2.
So at `i=1`, if we drink A, we must have drunk A at `i=0`.
`dp_a[1] = A[1] + dp_a[0]`
`dp_b[1] = B[1] + dp_b[0]`

dp_b[2] = 3 + max(dp_b[1], dp_a[0]) = 3 + max(2, 4) = 7

Max is 7. Correct.

Constraints: n up to 10^5.
O(n) time and O(1) space is easily achievable by just keeping the last two values for dp_a and dp_b.

Let's refine the DP variables:
`a_prev` = dp_a[i-1]
`a_prev2` = dp_a[i-2]
`b_prev` = dp_b[i-1]

For i=0:
`a_prev2`, `b_prev2` can be 0.

`dp_b[1] = B[1] + b_prev`

Wait, is it possible to skip more than 1 hour?
No, because all elements are positive. Skipping a hour gives 0, drinking A or B gives at least 1. So we only skip when switching drinks, and we only skip exactly 1 hour.
Wait, what if we switch, skip 1 hour, and then skip another hour? That would be equivalent to just not drinking for an hour, which is strictly worse than drinking something. Since we can switch to either, we would just drink whatever we were going to drink next.

Wait, is there any case where we switch from A to B, skip 1 hour, and then switch back to A, skipping another hour?
If we do A -> skip -> B -> skip -> A, we get A[i] + B[i+2] + A[i+4].
If we just do A -> A -> A -> A, we get A[i] + A[i+1] + A[i+2] + A[i+3] + A[i+4], which is obviously >= A[i] + A[i+4].
So skipping without switching is never optimal. Switching and then immediately switching back is never optimal.
Therefore, the state transition is perfectly correct.

class Solution:
    def maxEnergyBoost(self, energyDrinkA: List[int], energyDrinkB: List[int]) -> int:
        a_prev2, b_prev2 = 0, 0
        a_prev, b_prev = energyDrinkA[0], energyDrinkB[0]
        
        for i in range(1, len(energyDrinkA)):
            if i == 1:
                curr_a = energyDrinkA[i] + a_prev
                curr_b = energyDrinkB[i] + b_prev
            else:

                curr_b = energyDrinkB[i] + max(b_prev, a_prev2)

            a_prev, b_prev = curr_a, curr_b
            
        return max(a_prev, b_prev)
