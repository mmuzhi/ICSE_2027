Key observations:
1. The operation is linear over GF(2), so I can think of it as multiplying by matrix L (lower triangular of 1s)
2. L is invertible, so every sequence has an orbit that forms a cycle
3. Two sequences are in the same orbit iff one can be transformed to the other
4. f(i,j) is the position of A_j in A_i's orbit, or 0 if different orbits
5. The order of L is 2^ceil(log2(M)), so orbit sizes are powers of 2

Algorithm:
- For each unique sequence, compute its full orbit by repeatedly applying L
- Store each sequence's orbit ID and position within that orbit
- For each pair (i,j), if same orbit, compute f(i,j) from positions; otherwise f(i,j)=0
