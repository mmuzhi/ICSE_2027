If the number of $P$'s equals the number of $Q$'s in each half-plane, then we can match them, and recursively solve the two subproblems!
Since no three points are collinear, no point lies on the line $P_i Q_j$.
So the remaining $2N-2$ points are strictly partitioned into two half-planes.
Let the number of $P$'s in the left half-plane be $L_P$, and $Q$'s be $L_Q$.
For a perfect matching to exist in both half-planes, we must have $L_P = L_Q$.
If $L_P = L_Q$, then the right half-plane will also have $R_P = R_Q$.
So we can just find ANY pair $(P_i, Q_j)$ such that $L_P = L_Q$, match them, and recursively solve the two subproblems!
Does such a pair always exist?
Yes, it is a known theorem that a non-crossing matching always exists, and it can be found by this divide and conquer method.
To find such a pair, we can just iterate over all pairs $(P_i, Q_j)$.
For each pair, compute $L_P$ and $L_Q$.
If $L_P = L_Q$, we match them, and recurse.
This takes $O(N^3)$ time because there are $N$ pairs to match, finding the pair takes $O(N^2)$ (checking all $N^2$ pairs, each taking $O(N)$ to count $L_P, L_Q$).
Wait, $N \le 300$. $O(N^3)$ is $2.7 \times 10^7$ operations, which is very fast in Python!
Let's trace this algorithm.
Function `solve(points)`:
  `points` is a list of tuples `(x, y, type, id)`, where `type` is 'P' or 'Q', and `id` is the original index.
  If `points` is empty, return `[]`.
  For each `P` in `points`:
    For each `Q` in `points`:
      If `P.type == 'P'` and `Q.type == 'Q'`:

        If $L_P == L_Q$:
          Match `P` and `Q`.
          Create `left_points` and `right_points`.
          `left_res = solve(left_points)`
          `right_res = solve(right_points)`
          Return `[(P.id, Q.id)] + left_res + right_res`

Consider the line $P Q_1$. Let the difference between the number of $P$'s and $Q$'s on the left side be $\Delta_1 = L_P - L_Q$.
As we rotate the line from $P Q_1$ to $P Q_N$, the value of $\Delta$ changes.
When the line passes through another point, the value of $\Delta$ changes by $\pm 1$.
Actually, if we consider the line $P Q_i$, the left side has some $\Delta_i$.
When we rotate to $P Q_{i+1}$, the line sweeps over the points between $Q_i$ and $Q_{i+1}$.
The points between $Q_i$ and $Q_{i+1}$ in angular order are either $P$'s or $Q$'s.
So $\Delta_{i+1} = \Delta_i \pm 1$ or something.
Notice that the line $P Q_i$ and the line $P Q_i$ (opposite direction) have $\Delta$ values that are opposite in sign (or differ by something).
Actually, the line $P Q_i$ divides the plane into two half-planes. The left half-plane has $\Delta_i = L_P - L_Q$.
The right half-plane has $R_P - R_Q$.
Since $L_P + R_P = N - 1$ and $L_Q + R_Q = N - 1$ (excluding $P$ and $Q_i$),
we have $L_P - L_Q = -(R_P - R_Q)$.
So the two half-planes have opposite $\Delta$.
As we rotate the line $P Q_i$ continuously, $\Delta$ changes.
When the line rotates by 180 degrees, the left half-plane becomes the right half-plane, so $\Delta$ becomes $-\Delta$.
Since $\Delta$ changes by $\pm 1$ when we cross a point, and it goes from $\Delta_1$ to $-\Delta_1$ (or similar), it must cross 0!
Therefore, there MUST exist a $Q_i$ such that $\Delta_i = 0$, which means $L_P = L_Q$!
This is a solid proof that for ANY $P$, there exists a $Q$ such that $L_P = L_Q$.
So the algorithm will NEVER get stuck! We will always find a pair with $L_P = L_Q$.

```python
def solve(points):
    if not points:
        return []
    
    for i in range(len(points)):
        for j in range(len(points)):
            if points[i][2] == 'P' and points[j][2] == 'Q':
                P = points[i]
                Q = points[j]
                # count L_P and L_Q
                L_P = 0
                L_Q = 0
                for k in range(len(points)):
                    if k == i or k == j:
                        continue
                    # cross product to determine side
                    # (Q.x - P.x) * (pt.y - P.y) - (Q.y - P.y) * (pt.x - P.x)
                    cross = (Q[0] - P[0]) * (points[k][1] - P[1]) - (Q[1] - P[1]) * (points[k][0] - P[0])
                    if cross > 0:
                        if points[k][2] == 'P':
                            L_P += 1
                        else:
                            L_Q += 1
                if L_P == L_Q:
                    left_points = []
                    right_points = []
                    for k in range(len(points)):
                        if k == i or k == j:
                            continue
                        cross = (Q[0] - P[0]) * (points[k][1] - P[1]) - (Q[1] - P[1]) * (points[k][0] - P[0])
                        if cross > 0:
                            left_points.append(points[k])
                        else:
                            right_points.append(points[k])
                    
                    res = [(P[3], Q[3])]
                    res.extend(solve(left_points))
                    res.extend(solve(right_points))
                    return res
```

```python
        return []
```

```python
        for j in range(len(points)):
            if points[i][2] == 'P' and points[j][2] == 'Q':
                P = points[i]
                Q = points[j]

                L_Q = 0
```

The sector swept by the ray is the region between $P \to V_k$ and $P \to V_{k+1}$.

Wait, what about the points that are exactly 180 degrees opposite?

Let's formalize this.
Fix a point $P$. Sort all other points by angle around $P$.
Let the sorted points be $V_0, V_1, \ldots, V_{M-2}$.
We want to find a $Q$ such that $L_P = L_Q$.
We can maintain $L_P$ and $L_Q$ for the ray $P \to V_k$.

When we move from ray $P \to V_k$ to ray $P \to V_{k+1}$:
1. The point $V_k$ was on the ray. Now it moves to the Right half-plane.

2. The opposite ray sweeps over some points. These points move from the Left half-plane to the Right half-plane.
   Which points are these? They are the points whose angle is in the range $[\theta_k + \pi, \theta_{k+1} + \pi]$.
   Since the points are sorted by angle, we can maintain a pointer $ptr$ that points to the first point with angle $\ge \theta_k + \pi$.
   As $k$ increases, $ptr$ also increases!
   So we can just move $ptr$ forward, and for each point that moves from Left to Right, we update $L_P, L_Q, R_P, R_Q$.
3. The point $V_{k+1}$ is now on the ray, so it is removed from the Right half-plane.
   So if $V_{k+1}$ is a $P$, $R_P$ decreases by 1. If $Q$, $R_Q$ decreases by 1.
This update takes $O(1)$ amortized time per step!
So for a fixed $P$, we can check all $Q$'s in $O(M)$ time!
Since there are $O(M)$ choices for $P$, the total time to find a valid pair is $O(M^2)$!
This reduces the total time complexity to $O(N^3)$, which is extremely fast for $N=300$!

```python
def solve(points):
    if not points:
        return []
    
    M = len(points)
    
    for i in range(M):
        P = points[i]
        # Sort other points by angle around P
        others = []
        for j in range(M):
            if i != j:
                angle = math.atan2(points[j][1] - P[1], points[j][0] - P[0])
                others.append((angle, points[j]))
        others.sort()
        
        # Compute initial L_P, L_Q for ray P -> others[0]
        # Left half-plane is angle in (theta, theta + pi)
        # Right half-plane is angle in (theta + pi, theta + 2pi)
        # Wait, the ray is P -> V0. Left is (theta0, theta0 + pi).
        # Points on the ray are V0.
        # So initially, V0 is on the ray, not in Left or Right.
        
        # Let's compute initial L_P, L_Q
        L_P = 0
        L_Q = 0
        # others[k] is in Left if its angle is in (theta0, theta0 + pi)
        # We can use a two-pointer approach.
        # Since angles are in [-pi, pi], we need to handle wrap-around.
        # To avoid wrap-around, we can duplicate the array of angles by adding 2pi to the second half.
        
        angles = [a for a, p in others]
        pts = [p for a, p in others]
        
        # Duplicate
        for k in range(M - 1):
            angles.append(angles[k] + 2 * math.pi)
            pts.append(pts[k])
            
        # Find initial Left points
        # Ray is P -> pts[0]. Angle is angles[0].
        # Left points have angle in (angles[0], angles[0] + pi)
        # Right points have angle in (angles[0] + pi, angles[0] + 2pi)
        
        # ptr points to the first point in Right
        ptr = 1
        while ptr < 2 * (M - 1) and angles[ptr] < angles[0] + math.pi - 1e-9:
            if pts[ptr][2] == 'P':
                L_P += 1
            else:
                L_Q += 1
            ptr += 1
            
        # Now ptr is the first point in Right.
        # We check if pts[0] is a Q and L_P == L_Q
        if P[2] != pts[0][2] and L_P == L_Q:
            # Found a valid pair!
            Q = pts[0]
            # We need to partition the points into left and right
            left_points = pts[1:ptr]
            right_points = pts[ptr:M-1]
            
            res = [(P[3], Q[3])] if P[2] == 'P' else [(Q[3], P[3])]
            res.extend(solve(left_points))
            res.extend(solve(right_points))
            return res
            
        # Iterate k from 0 to M-2
        for k in range(M - 2):
            # Move ray from P -> pts[k] to P -> pts[k+1]
            # pts[k] moves from Ray to Right
            if pts[k][2] == 'P':
                pass # it was on the ray, now in Right. We don't explicitly track Right, only Left.
            else:
                pass
            
            # Opposite ray sweeps from angles[k] + pi to angles[k+1] + pi
            # Points in this range move from Left to Right
            while ptr < 2 * (M - 1) and angles[ptr] < angles[k+1] + math.pi - 1e-9:
                if pts[ptr][2] == 'P':
                    L_P -= 1
                else:
                    L_Q -= 1
                ptr += 1
                
            # pts[k+1] moves from Right to Ray
            # Wait, pts[k+1] was in Right. So we need to remove it from Right.
            # But we didn't track Right. We only tracked Left.
            # Since pts[k+1] was in Right, it was NOT in Left. So we don't need to change L_P, L_Q.
            
            # Check if pts[k+1] is a valid Q
            if P[2] != pts[k+1][2] and L_P == L_Q:
                Q = pts[k+1]
                # Partition points
                # Left points are those between k+1 and ptr
                left_points = pts[k+2:ptr]
                # Right points are the rest
                # We need to be careful with indices.

                right_points = pts[ptr:M-1] + pts[0:k+1]

                res.extend(solve(left_points))
                res.extend(solve(right_points))
```

```python
import math

def solve(points):
    if not points:
        return []
    
    M = len(points)
    
    for i in range(M):
        P = points[i]
```
```python
        px, py = P[0], P[1]
        
        others = []
        for j in range(M):
            if i != j:
                pt = points[j]
                angle = math.atan2(pt[1] - py, pt[0] - px)
                others.append((angle, pt))
        others.sort()
        
        angles = [a for a, p in others]
        pts = [p for a, p in others]
        
        for k in range(M - 1):
            angles.append(angles[k] + 2 * math.pi)
            pts.append(pts[k])
            
        L_P = 0
        L_Q = 0
        ptr = 1
        limit = angles[0] + math.pi
        while ptr < 2 * (M - 1) and angles[ptr] < limit:
            if pts[ptr][2] == 'P':
                L_P += 1
            else:
                L_Q += 1
            ptr += 1
            
        if P[2] != pts[0][2] and L_P == L_Q:
            Q = pts[0]
            left_points = pts[1:ptr]
            right_points = pts[ptr:M-1]
            
            res = [(P[3], Q[3])] if P[2] == 'P' else [(Q[3], P[3])]
            res.extend(solve(left_points))
            res.extend(solve(right_points))
            return res
            
        for k in range(M - 2):
            if pts[k+1][2] == 'P':
                L_P -= 1
            else:
                L_Q -= 1
                
            limit = angles[k+1] + math.pi
            while ptr < 2 * (M - 1) and angles[ptr] < limit:
                if pts[ptr][2] == 'P':
                    L_P += 1
                else:
                    L_Q += 1
                ptr += 1

                left_points = pts[k+2:ptr]
                right_points = pts[ptr:M-1] + pts[0:k+1]
```

```python
import sys
import math

sys.setrecursionlimit(2000)
```

```python
def solve(points):
    if not points:
        return []
    
    M = len(points)
    
    for i in range(M):
        P = points[i]
        px, py = P[0], P[1]
        
        others = []
        for j in range(M):
            if i != j:
                pt = points[j]
                angle = math.atan2(pt[1] - py, pt[0] - px)
                others.append((angle, pt))
        others.sort()
        
        angles = [a for a, p in others]
        pts = [p for a, p in others]
        
        for k in range(M - 1):
            angles.append(angles[k] + 2 * math.pi)
            pts.append(pts[k])
            
        L_P = 0
        L_Q = 0
        ptr = 1
        limit = angles[0] + math.pi
        while ptr < 2 * (M - 1) and angles[ptr] < limit:
            if pts[ptr][2] == 'P':
                L_P += 1
            else:
                L_Q += 1
            ptr += 1
            
        if P[2] != pts[0][2] and L_P == L_Q:
            Q = pts[0]
            left_points = pts[1:ptr]
            right_points = pts[ptr:M-1]
            
            res = [(P[3], Q[3])] if P[2] == 'P' else [(Q[3], P[3])]
            res.extend(solve(left_points))
            res.extend(solve(right_points))
            return res
            
        for k in range(M - 2):
            if pts[k+1][2] == 'P':
                L_P -= 1
            else:
                L_Q -= 1
                
            limit = angles[k+1] + math.pi
            while ptr < 2 * (M - 1) and angles[ptr] < limit:
                if pts[ptr][2] == 'P':
                    L_P += 1
                else:
                    L_Q += 1
                ptr += 1
                
            if P[2] != pts[k+1][2] and L_P == L_Q:
                Q = pts[k+1]
                left_points = pts[k+2:ptr]
                right_points = pts[ptr:M-1] + pts[0:k+1]
                
                res = [(P[3], Q[3])] if P[2] == 'P' else [(Q[3], P[3])]
                res.extend(solve(left_points))
                res.extend(solve(right_points))
                return res
```
