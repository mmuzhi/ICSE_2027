```python
class Solution:
    def maxSum(self, nums: List[int], k: int, m: int) -> int:
        N = len(nums)
        prefix = [0] * (N + 1)
        for i in range(N):
            prefix[i+1] = prefix[i] + nums[i]
            
        dp = [float('-inf')] * (N + 1)
        dp[0] = 0
        
        for j in range(1, k + 1):
            curr_dp = [float('-inf')] * (N + 1)
            best = float('-inf')
            for i in range(j * m, N + 1):
                s = i - m
                best = max(best, dp[s] - prefix[s])
                curr_dp[i] = max(curr_dp[i-1], prefix[i] + best)
            dp = curr_dp
            
        return dp[N]
```

    def maxSum(self, nums: List[int], k: int, m: int) -> int:
        N = len(nums)
        prefix = [0] * (N + 1)

Is there any case where `best` is updated from a suboptimal `s` but later a better `s` appears, and we miss it?
No, `best = max(best, dp[s] - prefix[s])` keeps the maximum over all `s <= i - m`.
Since we only add to `best`, and we take `max`, `best` is always the maximum over all valid `s` seen so far.

Wait, what if `curr_dp[i-1]` is chosen, meaning we don't end the `j`-th subarray at `i-1`.

This correctly adds `nums[i-1]` to the sum.
What if `curr_dp[i-1]` was chosen, but we want to START a new subarray at `i-m` and end it at `i-1`?
Then `s = i - m`. `best` is updated with `dp[s] - prefix[s]`.
`prefix[i] + best` will be `prefix[i] + dp[s] - prefix[s] = dp[s] + sum(nums[s...i-1])`.
This is exactly the sum of `j-1` subarrays in `0...s-1` plus the new subarray `nums[s...i-1]`.
This is also correctly handled.

What if we want to start a new subarray at `i-m` and extend it to `i`?
At `i`, `s = i - m`. `best` gets `dp[s] - prefix[s]`.
At `i+1`, `best` is at least `dp[s] - prefix[s]`.
`prefix[i+1] + best` will be `dp[s] + sum(nums[s...i])`.
This correctly extends the new subarray.

Everything is perfectly consistent.
The space optimization is also correct because `dp[j][i]` only depends on `dp[j-1][s]` for `s <= i - m`.
Since we iterate `i` from `j*m` to `N`, `s` goes from `(j-1)*m` to `N - m`.
We only read from `dp[s]`, which is the `dp` array from the *previous* iteration of `j`.
We never overwrite `dp[s]` in the current `j` iteration before reading it, because `curr_dp` is a separate array.
So the space optimization is safe.

class Solution:
    def maxSum(self, nums: List[int], k: int, m: int) -> int:

                best = max(best, dp[s] - prefix[s])
                curr_dp[i] = max(curr_dp[i-1], prefix[i] + best)
            dp = curr_dp
            
        return dp[N]
