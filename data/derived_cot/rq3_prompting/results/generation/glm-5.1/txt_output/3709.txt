- Problem: Find if there's a substring of length exactly `k` in `s` that is all the same character, and the characters immediately before and after it (if they exist) are different from this character.
- Constraints: `1 <= k <= s.length <= 100`. String is lowercase English letters.
- Candidate: Simple linear scan / sliding window. Group consecutive same characters.
- Feasibility: O(N) time, O(1) space. Very simple.
- Implementation ledger:
    - Iterate through `s`, keep track of the current run length of the same character.
    - When the character changes, or at the end of the string, check if the previous run length was exactly `k`. If so, return `True`.
    - Actually, if a run length is exactly `k`, it automatically means the characters before and after are different (or it's at the boundary), because it's a maximal run of length `k`!
    - Wait, is that true? Yes! A maximal run of identical characters of length `k` means the characters adjacent to it (if they exist) are different.
    - Let's double check. "aaabaaa", k=3. Runs: "aaa" (3), "b" (1), "aaa" (3). Both runs of 'a' have length 3. Return True.
    - "aaabaaa", k=2. Runs: "aaa" (3), "b" (1), "aaa" (3). None have length 2. Return False.
    - "aabaaa", k=1. Runs: "aa" (2), "b" (1), "aaa" (3). 'b' has length 1. Return True.
    - What if `s = "a"`, `k = 1`. Run: "a" (1). Return True.
    - So the problem reduces to: Does any maximal contiguous block of identical characters have length exactly `k`?
- Edge cases:
    - `k` equals string length, all same char: `s="aaa", k=3`. Run: "aaa" (3). Return True.
    - `k` equals string length, different chars: `s="abc", k=3`. Runs: "a", "b", "c". Return False.
- Implementation details:
    - `count = 1`
    - Loop `i` from 1 to `len(s)`:
        - if `s[i] == s[i-1]`: `count += 1`
        - else:
            - if `count == k`: return `True`
            - `count = 1`
    - After loop, check `if count == k: return True`
    - Return `False`
- Check: `s = "aaabaaa", k = 3`
    - i=1: 'a'=='a', count=2
    - i=2: 'a'=='a', count=3

- Wait, is it possible to have a substring of length `k` that is part of a longer run, but bounded by different characters? No, because if it's part of a longer run, the characters immediately before/after the substring would be the *same* character, violating the condition.
- So the substring *must* be a maximal run of length exactly `k`.
- This is extremely simple. Let's write the code.

```python
class Solution:
    def hasSpecialSubstring(self, s: str, k: int) -> bool:
        count = 1
        for i in range(1, len(s)):
            if s[i] == s[i-1]:
                count += 1
            else:
                if count == k:
                    return True
                count = 1
        return count == k
```
