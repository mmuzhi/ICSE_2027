=== ANSWER ===
Looking at this problem: N ≤ 300 cities, at most 300 road closures, up to 2×10⁵ queries.

**Key insight**: Process queries in reverse. Start from the final state (after all closures), then "open" roads as we go backwards. This converts deletions into additions, which are easier to handle.

**Algorithm**:
1. Determine which roads are closed at the end
2. Build initial APSP with only open roads using Floyd-Warshall O(N³)
3. Process queries in reverse:
   - Type 1 (reversed): Add road back — update APSP in O(N²) using: `new_dist[i][j] = min(dist[i][j], dist[i][a]+c+dist[b][j], dist[i][b]+c+dist[a][j])`
   - Type 2 (reversed): Record answer

The edge addition formula works because any new shortest path must use the added edge exactly once (positive weights), and the parts before/after are already shortest paths.

**Complexity**: O(N³ + 300·N² + Q) — feasible with numpy vectorization.

**Verification with Sample 1**:

- Reverse query 5: answer -1 ✓
- Add road 1 (0,1,5): dist[0][2] = 11 ✓  
- Reverse query 3: answer 11 ✓
- Add road 2 (0,2,10): dist[0][2] = 10 ✓
- Reverse query 1: answer 10 ✓
