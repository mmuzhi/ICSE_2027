The problem asks for the number of integers $x$ such that $A, B, x$ can be arranged to form an arithmetic sequence.
Let the three numbers in the arithmetic sequence be $y_1, y_2, y_3$.
Then $y_2 - y_1 = y_3 - y_2 \implies 2y_2 = y_1 + y_3$.
This means one of the three numbers is the average of the other two.
So, either:
1) $2A = B + x \implies x = 2A - B$
2) $2B = A + x \implies x = 2B - A$
3) $2x = A + B \implies x = (A + B) / 2$

For case 3, $x$ must be an integer, so $A + B$ must be even.
The possible values of $x$ are $2A - B$, $2B - A$, and $(A + B) / 2$ (if $A+B$ is even).
We just need to count the number of distinct values among these.
Since $A$ and $B$ are up to 100, we can just use a set to store the valid $x$ values.

Let's trace Sample 1: A=5, B=7

Implementation:
```python
A, B = map(int, input().split())
ans = set()
ans.add(2 * A - B)
ans.add(2 * B - A)
if (A + B) % 2 == 0:
    ans.add((A + B) // 2)
print(len(ans))
```
