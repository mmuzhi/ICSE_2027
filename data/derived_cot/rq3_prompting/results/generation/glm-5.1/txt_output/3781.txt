```python
        for x, y in points:
            if y == 0:
                p.append(x)
            elif x == side:
                p.append(side + y)
            elif y == side:
                p.append(2 * side + (side - x))
            else:
                p.append(3 * side + (side - y))

        high = 4 * side
```

Does greedy *always* find a valid sequence if one exists?
Suppose we have a fixed starting point `i`.
We want to find a sequence of `k` points starting at `i` such that the span is `<= 4 * side - D`.
Greedy minimizes the position of the `m`-th point for all `m`.
Therefore, greedy minimizes the span of the `k` points.
If greedy fails (either by hitting `-1` or the span is too large), then NO OTHER SEQUENCE starting at `i` can have a smaller span.
Thus, no other sequence starting at `i` can satisfy the span constraint.
So greedy is optimal for a fixed starting point `i`.

What if the optimal sequence does NOT start at `i`?
We check all `i` from `0` to `n-1`.
Any valid sequence must have a first point (the one with the smallest `p` value in the sequence).
When we check that `i`, we will find the valid sequence (or another one) using greedy.
So checking all `i` is complete.

Wait, what if the optimal sequence has multiple points with the SAME smallest `p` value?
Impossible, `p` values are unique.

What if the optimal sequence has a wrap-around, and the "first" point in the sorted order is NOT the best starting point to find the sequence?
Let the optimal sequence be $P_1, P_2, \dots, P_k$ in sorted order of `p`.
When we start at $P_1$ (i.e., `i` is the index of $P_1$), greedy will find a sequence.
Could greedy find a *different* sequence that fails, while the optimal sequence exists?
Greedy minimizes the span. The optimal sequence has a span $\le 4 \times side - D$.
Since greedy minimizes the span, the span of the greedy sequence is $\le$ the span of the optimal sequence.

```python
class Solution:
    def maxDistance(self, side: int, points: List[List[int]], k: int) -> int:
        n = len(points)
        p = []
        for x, y in points:
            if y == 0:
                p.append(x)
            elif x == side:
                p.append(side + y)
            elif y == side:
                p.append(2 * side + (side - x))
            else: # x == 0
                p.append(3 * side + (side - y))
        
        p.sort()
        A = p + [x + 4 * side for x in p]
        
        low = 0
        high = 4 * side
        ans = 0
        
        while low <= high:
            mid = (low + high) // 2
            D = mid
            
            nxt = [-1] * (2 * n)
            j = 0
            for i in range(2 * n):
                while j < 2 * n and (A[j] - A[i] < D or j <= i):
                    j += 1
                if j < 2 * n:
                    nxt[i] = j
            
            possible = False
            for i in range(n):
                curr = i
                valid = True
                for _ in range(k - 1):
                    curr = nxt[curr]
                    if curr == -1:
                        valid = False
                        break
                if valid:
                    if A[curr] - A[i] <= 4 * side - D:
                        possible = True
                        break
            
            if possible:
                ans = mid
                low = mid + 1
            else:
                high = mid - 1
                
        return ans
```
