- `sum`: sum of elements in the segment
- `lazy_chmin`: lazy tag for chmin
- `lazy_chmax`: lazy tag for chmax

Wait, maintaining both `lazy_chmin` and `lazy_chmax` is a bit tricky but standard.
Let's recall the pushdown logic for Segment Tree Beats with both chmin and chmax.
Suppose a node has `lazy_chmin` and `lazy_chmax`.
When we apply `chmin(x)` to a node:
- `mx = min(mx, x)`
- `mn = min(mn, x)`
- `sum` is updated accordingly.
- `lazy_chmin = min(lazy_chmin, x)`
- `lazy_chmax = min(lazy_chmax, x)`  // because chmin(x) also bounds the chmax tag!

When we apply `chmax(x)` to a node:
- `mn = max(mn, x)`
- `mx = max(mx, x)`
- `sum` is updated accordingly.
- `lazy_chmax = max(lazy_chmax, x)`
- `lazy_chmin = max(lazy_chmin, x)`  // because chmax(x) also bounds the chmin tag!

Wait, is it that simple?
Let's trace carefully.
A node represents a segment. We apply `chmin(C)` and `chmax(L)`.
The effective value of an element $v$ is $\max(L, \min(C, v))$.
If we have a node with `mx`, `mx2`, `mxc`, `mn`, `mn2`, `mnc`.
When we apply `chmin(C)`:
If `C >= mx`, nothing happens.
If `mn < C < mx`:
  We only need to change the elements that are $> C$, which are exactly the maximum elements.
  So `sum -= mxc * (mx - C)`.
  `mx = C`.
  `lazy_chmin = min(lazy_chmin, C)`.
  What about `mn`? Since `C > mn`, `mn` is unchanged.
If `C <= mn`:
  All elements become $C$.
  `sum = len * C`.
  `mx = mn = C`.
  `mx2 = -inf`, `mn2 = inf`.
  `lazy_chmin = C`, `lazy_chmax = C`.

When we apply `chmax(L)`:
If `L <= mn`, nothing happens.
If `mn < L < mx`:
  We only need to change the elements that are $< L$, which are exactly the minimum elements.
  `sum += mnc * (L - mn)`.
  `mn = L`.
  `lazy_chmax = max(lazy_chmax, L)`.
  What about `mx`? Since `L < mx`, `mx` is unchanged.
If `L >= mx`:
  All elements become $L$.
  `sum = len * L`.
  `mx = mn = L`.
  `mx2 = -inf`, `mn2 = inf`.
  `lazy_chmin = L`, `lazy_chmax = L`.

Wait, what if we apply `chmin(C)` and then `chmax(L)`?
If $L > C$, then all elements become $C$, and then `chmax(L)` makes them $L$.
But $L > C$ means the operations are contradictory.
In our problem, for $j < T$, we only apply `chmin(C)`.
For $j > T$, we only apply `chmax(C)`.
For $j = T$, we do `point_assign(T, C)`.
So we NEVER apply both `chmin` and `chmax` to the same segment in the same step!
Wait, over multiple steps, a segment might have both `chmin` and `chmax` tags.
For example, step 1 applies `chmin(C1)`. Step 2 applies `chmax(C2)`.
If we maintain `lazy_chmin` and `lazy_chmax`, the effective value is $\max(C2, \min(C1, v))$.
Let's check if the tags commute.
If we apply `chmin(C1)`, then `chmax(C2)`, the result is $\max(C2, \min(C1, v))$.
If we apply `chmax(C2)`, then `chmin(C1)`, the result is $\min(C1, \max(C2, v))$.
These are NOT the same if $C2 > C1$.
But wait, in our problem, the sequence $A$ is ALWAYS non-decreasing!
So when we apply `chmin(C)` to $[1, T-1]$, we are just reducing some suffix of $A$ to $C$.
When we apply `chmax(C)` to $[T+1, N]$, we are just increasing some prefix of $A$ to $C$.
Since $A$ is non-decreasing, the elements in $[1, T-1]$ are $\le A_T$.
The elements in $[T+1, N]$ are $\ge A_T$.
So we don't really need a full Segment Tree Beats!
Since $A$ is ALWAYS non-decreasing, we can just use a balanced binary search tree (like `SortedList` in Python) or a Segment Tree that maintains the non-decreasing property!
Wait, if $A$ is non-decreasing, then `range_chmin(1, T-1, C)` just means: find the largest index $m < T$ such that $A_m > C$, and set $A_j = C$ for all $j \in [m, T-1]$.
`range_chmax(T+1, N, C)` means: find the smallest index $m > T$ such that $A_m < C$, and set $A_j = C$ for all $j \in [T+1, m]$.
This is just a range assignment!
Since $A$ is non-decreasing, the set of indices where $A_j > C$ is a contiguous suffix of $[1, T-1]$.
The set of indices where $A_j < C$ is a contiguous prefix of $[T+1, N]$.
So we just need to find the boundaries and do a range assignment!
And we need to calculate the sum of the changed elements to find the cost.

Let's verify this!
Suppose $A$ is non-decreasing.
For $j < T$, we want to apply $A_j = \min(A_j, C)$.
Since $A$ is non-decreasing, there is some index $m$ such that $A_j \le C$ for $j < m$, and $A_j > C$ for $m \le j < T$.
So we just need to set $A_j = C$ for $j \in [m, T-1]$.
The cost is $\sum_{j=m}^{T-1} (A_j - C)$.
For $j > T$, we want to apply $A_j = \max(A_j, C)$.
Since $A$ is non-decreasing, there is some index $k$ such that $A_j < C$ for $T < j \le k$, and $A_j \ge C$ for $j > k$.
So we just need to set $A_j = C$ for $j \in [T+1, k]$.
The cost is $\sum_{j=T+1}^k (C - A_j)$.

This means we don't need Segment Tree Beats! We just need a data structure that supports:
- Find the first index $j$ in $[1, T-1]$ such that $A_j > C$.
- Find the last index $j$ in $[T+1, N]$ such that $A_j < C$.
- Range assign a value $C$.
- Range sum query.

This is a standard Segment Tree with lazy propagation!
Since we only do range assignment (set to $C$), the lazy tag is just a single value!
This is MUCH simpler than Segment Tree Beats!
Let's double check if $A$ is always non-decreasing.
Yes, we proved that if $A$ is non-decreasing, after the updates, $A$ remains non-decreasing.
The updates are:
$A_j = \min(A_j, C)$ for $j < T$
$A_T = C$
$A_j = \max(A_j, C)$ for $j > T$
Since $A$ is non-decreasing, $\min(A_j, C)$ is non-decreasing.
$A_{T-1} = \min(A_{T-1}, C) \le C$.
$A_T = C$.
$A_{T+1} = \max(A_{T+1}, C) \ge C$.
$\max(A_j, C)$ is non-decreasing.
So $A$ is definitely always non-decreasing!

Let's refine the Segment Tree implementation.
We need a segment tree that supports:
- `build(A)`
- `query_point(idx)`
- `range_sum(l, r)`
- `range_assign(l, r, val)`
- `find_first_greater(l, r, C)` -> returns smallest $j \in [l, r]$ with $A_j > C$, or $r+1$ if none.

Wait, `find_first_greater` and `find_last_smaller` can be implemented easily.
Since the array is non-decreasing, we can just maintain the maximum and minimum values in each node.
For `find_first_greater(l, r, C)`:
If node's max $\le C$, return $r+1$.
If it's a leaf, return its index.
Otherwise, search left child first, then right child.

For `find_last_smaller(l, r, C)`:
If node's min $\ge C$, return $l-1$.
If it's a leaf, return its index.
Otherwise, search right child first, then left child.

```python
def build(node, l, r):
    if l == r:
        mn[node] = A[l]
        mx[node] = A[l]
        sum_tree[node] = A[l]
    else:
        mid = (l + r) // 2
        build(node * 2, l, mid)
        build(node * 2 + 1, mid + 1, r)
        pull(node)

def pull(node):
    mn[node] = min(mn[node * 2], mn[node * 2 + 1])
    mx[node] = max(mx[node * 2], mx[node * 2 + 1])
    sum_tree[node] = sum_tree[node * 2] + sum_tree[node * 2 + 1]

def apply(node, l, r, val):
    mn[node] = val
    mx[node] = val
    sum_tree[node] = val * (r - l + 1)
    lazy[node] = val

def push(node, l, r):
    if lazy[node] is not None and l < r:
        mid = (l + r) // 2
        apply(node * 2, l, mid, lazy[node])
        apply(node * 2 + 1, mid + 1, r, lazy[node])
        lazy[node] = None

def range_assign(node, l, r, ql, qr, val):
    if ql <= l and r <= qr:
        apply(node, l, r, val)
        return
    push(node, l, r)
    mid = (l + r) // 2
    if ql <= mid:
        range_assign(node * 2, l, mid, ql, qr, val)
    if qr > mid:
        range_assign(node * 2 + 1, mid + 1, r, ql, qr, val)
    pull(node)

def range_sum(node, l, r, ql, qr):
    if ql <= l and r <= qr:
        return sum_tree[node]
    push(node, l, r)
    mid = (l + r) // 2
    res = 0
    if ql <= mid:
        res += range_sum(node * 2, l, mid, ql, qr)
    if qr > mid:
        res += range_sum(node * 2 + 1, mid + 1, r, ql, qr)
    return res

def find_first_greater(node, l, r, ql, qr, C):
    if l > qr or r < ql or mx[node] <= C:
        return qr + 1
    if l == r:
        return l
    push(node, l, r)
    mid = (l + r) // 2
    res = qr + 1
    if ql <= mid:
        res = find_first_greater(node * 2, l, mid, ql, qr, C)
    if res == qr + 1 and qr > mid:
        res = find_first_greater(node * 2 + 1, mid + 1, r, ql, qr, C)
    return res

def find_last_smaller(node, l, r, ql, qr, C):
    if l > qr or r < ql or mn[node] >= C:
        return ql - 1
    if l == r:
        return l
    push(node, l, r)
    mid = (l + r) // 2
    res = ql - 1
    if qr > mid:
        res = find_last_smaller(node * 2 + 1, mid + 1, r, ql, qr, C)
    if res == ql - 1 and ql <= mid:
        res = find_last_smaller(node * 2, l, mid, ql, qr, C)
    return res
```

```python
ans = 0

            range_assign(1, 1, N, m, T - 1, C)

            range_assign(1, 1, N, T + 1, k, C)
            ans += C * (k - (T + 1) + 1) - old_sum
            
    # Update T
    range_assign(1, 1, N, T, T, C)
```

```python
import sys

def solve():
    input = sys.stdin.read
    data = input().split()
    if not data:
        return
    
    N = int(data[0])
    X = [int(x) for x in data[1:N+1]]
    Q = int(data[N+1])
    
    A = [X[i] - (i + 1) for i in range(N)]
    
    mn = [0] * (4 * N)
    mx = [0] * (4 * N)
    sum_t = [0] * (4 * N)
    lazy = [None] * (4 * N)
    
    def build(node, l, r):
        if l == r:
            val = A[l]
            mn[node] = val
            mx[node] = val
            sum_t[node] = val
        else:
            mid = (l + r) // 2
            build(node * 2, l, mid)
            build(node * 2 + 1, mid + 1, r)
            pull(node)
            
    def pull(node):
        mn[node] = min(mn[node * 2], mn[node * 2 + 1])
        mx[node] = max(mx[node * 2], mx[node * 2 + 1])
        sum_t[node] = sum_t[node * 2] + sum_t[node * 2 + 1]
        
    def apply(node, l, r, val):
        mn[node] = val
        mx[node] = val
        sum_t[node] = val * (r - l + 1)
        lazy[node] = val
        
    def push(node, l, r):
        if lazy[node] is not None and l < r:
            mid = (l + r) // 2
            apply(node * 2, l, mid, lazy[node])
            apply(node * 2 + 1, mid + 1, r, lazy[node])
            lazy[node] = None
            
    def range_assign(node, l, r, ql, qr, val):
        if ql <= l and r <= qr:
            apply(node, l, r, val)
            return
        push(node, l, r)
        mid = (l + r) // 2
        if ql <= mid:
            range_assign(node * 2, l, mid, ql, qr, val)
        if qr > mid:
            range_assign(node * 2 + 1, mid + 1, r, ql, qr, val)
        pull(node)
        
    def range_sum(node, l, r, ql, qr):
        if ql <= l and r <= qr:
            return sum_t[node]
        push(node, l, r)
        mid = (l + r) // 2
        res = 0
        if ql <= mid:
            res += range_sum(node * 2, l, mid, ql, qr)
        if qr > mid:
            res += range_sum(node * 2 + 1, mid + 1, r, ql, qr)
        return res
        
    def find_first_greater(node, l, r, ql, qr, C):
        if l > qr or r < ql or mx[node] <= C:
            return qr + 1
        if l == r:
            return l
        push(node, l, r)
        mid = (l + r) // 2
        res = qr + 1
        if ql <= mid:
            res = find_first_greater(node * 2, l, mid, ql, qr, C)
        if res == qr + 1 and qr > mid:
            res = find_first_greater(node * 2 + 1, mid + 1, r, ql, qr, C)
        return res
        
    def find_last_smaller(node, l, r, ql, qr, C):
        if l > qr or r < ql or mn[node] >= C:
            return ql - 1
```

```python
import sys

def solve():
    input = sys.stdin.read
    data = input().split()
    if not data:
        return
    
    sys.setrecursionlimit(400000)
    
    N = int(data[0])
    X = [int(x) for x in data[1:N+1]]
    Q = int(data[N+1])
    
    A = [X[i] - (i + 1) for i in range(N)]
    
    mn = [0] * (4 * N)
    mx = [0] * (4 * N)
    sum_t = [0] * (4 * N)
    lazy = [None] * (4 * N)
    
    def build(node, l, r):
        if l == r:
            val = A[l]
            mn[node] = val
            mx[node] = val
            sum_t[node] = val
        else:
            mid = (l + r) // 2
            build(node * 2, l, mid)
            build(node * 2 + 1, mid + 1, r)
            pull(node)
            
    def pull(node):
        mn[node] = min(mn[node * 2], mn[node * 2 + 1])
        mx[node] = max(mx[node * 2], mx[node * 2 + 1])
        sum_t[node] = sum_t[node * 2] + sum_t[node * 2 + 1]
        
    def apply(node, l, r, val):
        mn[node] = val
        mx[node] = val
        sum_t[node] = val * (r - l + 1)
        lazy[node] = val
        
    def push(node, l, r):
        if lazy[node] is not None and l < r:
            mid = (l + r) // 2
            apply(node * 2, l, mid, lazy[node])
            apply(node * 2 + 1, mid + 1, r, lazy[node])
            lazy[node] = None
            
    def sum_and_assign(node, l, r, ql, qr, val):
        if ql <= l and r <= qr:
            old_sum = sum_t[node]
            apply(node, l, r, val)
            return old_sum
        push(node, l, r)
        mid = (l + r) // 2
        res = 0
        if ql <= mid:
            res += sum_and_assign(node * 2, l, mid, ql, qr, val)
        if qr > mid:
            res += sum_and_assign(node * 2 + 1, mid + 1, r, ql, qr, val)
        pull(node)
        return res
        
    def find_first_greater(node, l, r, ql, qr, C):
        if l > qr or r < ql or mx[node] <= C:
            return qr + 1
        if l == r:
            return l
        push(node, l, r)
        mid = (l + r) // 2
        res = qr + 1
        if ql <= mid:
            res = find_first_greater(node * 2, l, mid, ql, qr, C)
        if res == qr + 1 and qr > mid:
            res = find_first_greater(node * 2 + 1, mid + 1, r, ql, qr, C)
        return res
        
    def find_last_smaller(node, l, r, ql, qr, C):
        if l > qr or r < ql or mn[node] >= C:
            return ql - 1
        if l == r:
            return l
        push(node, l, r)
        mid = (l + r) // 2
        res = ql - 1
        if qr > mid:
            res = find_last_smaller(node * 2 + 1, mid + 1, r, ql, qr, C)
        if res == ql - 1 and ql <= mid:
            res = find_last_smaller(node * 2, l, mid, ql, qr, C)
        return res

    build(1, 0, N - 1)
    
    ans = 0
    idx = N + 2
    for _ in range(Q):
        T = int(data[idx]) - 1
        G = int(data[idx + 1])
        idx += 2
        
        C = G - (T + 1)
        
        # Cost for T
        old_val = sum_and_assign(1, 0, N - 1, T, T, C)
        ans += abs(old_val - C)
        
        # Cost for j < T
        if T > 0:
            m = find_first_greater(1, 0, N - 1, 0, T - 1, C)
            if m <= T - 1:
                old_sum = sum_and_assign(1, 0, N - 1, m, T - 1, C)
                ans += old_sum - C * (T - m)
                
        # Cost for j > T
        if T < N - 1:
            k = find_last_smaller(1, 0, N - 1, T + 1, N - 1, C)
            if k >= T + 1:
                old_sum = sum_and_assign(1, 0, N - 1, T + 1, k, C)
                ans += C * (k - T) - old_sum
                
    print(ans)

solve()
```
