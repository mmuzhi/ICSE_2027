Looking at this problem, I need to find the expected inversion number after randomly choosing an interval of length K and shuffling it.

**Key Insight:** Using linearity of expectation, I can compute the expected contribution of each pair (a,b) with a<b.

After careful analysis, the expected inversion count simplifies to:
$$E[\text{inv}] = I + \frac{K(K-1)}{4} - \frac{\sum_{d < K}(K-d) \cdot I_d}{N-K+1}$$

where:
- $I$ = total inversions
- $I_d$ = inversions with distance exactly $d$

**Computing the weighted sum:** For each position $b$, I need to sum $(K - (b-a))$ over all inversions $(a,b)$ with $b-a < K$. This equals $\sum_{a \in \text{window}, P_a > P_b}(a + K - b)$.

I use two Fenwick trees with a sliding window of size K-1:
1. Count of values in window
2. Sum of positions with those values

For each position $b$:
- Query count and sum of positions with values > $P_b$
- Contribution = sum_positions + (K - pos_b) × count
- Add current element, remove element leaving window
