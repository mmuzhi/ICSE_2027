For a path, define its weight as the maximum weight of an edge in the path.
Define f(x, y) as the minimum possible path weight of a path from vertex x to vertex y.
You are given two sequences of length K: (A_1, A_2, \ldots, A_K) and (B_1, B_2, \ldots, B_K). It is guaranteed that A_i \neq B_j (1 \leq i,j \leq K).
Permute the sequence B freely so that \displaystyle \sum_{i=1}^{K} f(A_i, B_i) is minimized.

Input

The input is given from Standard Input in the following format:
N M K
u_1 v_1 w_1
u_2 v_2 w_2
\vdots
u_M v_M w_M
A_1 A_2 \ldots A_K
B_1 B_2 \ldots B_K

Output

Print the minimum value of \displaystyle \sum_{i=1}^{K} f(A_i, B_i).

Constraints


- 2 \leq N  \leq 2 \times 10^5
- N-1 \leq M  \leq \min(\frac{N \times (N-1)}{2},2 \times 10^5)
- 1 \leq K \leq N
- 1 \leq u_i<v_i \leq N (1 \leq i \leq M) 
- 1 \leq w_i \leq 10^9
- 1 \leq A_i,B_i \leq N (1 \leq i \leq K)
- The given graph is simple and connected.
- All input values are integers.

```python
class DSU:
    def __init__(self, n):
        self.parent = list(range(n + 1))
        self.c_A = [0] * (n + 1)
        self.c_B = [0] * (n + 1)

    def find(self, i):
        if self.parent[i] == i:
            return i
        self.parent[i] = self.find(self.parent[i])
        return self.parent[i]

    def union(self, u, v, w):
        r_u = self.find(u)
        r_v = self.find(v)
        if r_u == r_v:
            return 0
        
        # Always merge to r_u
        k = min(self.c_A[r_u], self.c_B[r_v]) + min(self.c_A[r_v], self.c_B[r_u])
        cost = w * k
        
        self.c_A[r_u] += self.c_A[r_v] - k
        self.c_B[r_u] += self.c_B[r_v] - k
        self.parent[r_v] = r_u
        
        return cost
```
Wait, is there any case where merging to `r_u` is bad?
In DSU with path compression, we usually merge by rank or size to keep the tree flat.
If we don't merge by size, the tree might become deep, and `find` could take longer.
But with path compression, even without union by rank, it's fast enough. However, adding union by rank is better.
Wait, if we use union by rank, we might merge `r_u` to `r_v`.
If we do that, we need to update `c_A[r_v]` and `c_B[r_v]` instead.
Let's just add union by size/rank to be safe.
```python
class DSU:
    def __init__(self, n):
        self.parent = list(range(n + 1))
        self.size = [1] * (n + 1)
        self.c_A = [0] * (n + 1)
        self.c_B = [0] * (n + 1)

    def find(self, i):
        if self.parent[i] == i:
            return i
        self.parent[i] = self.find(self.parent[i])
        return self.parent[i]

    def union(self, u, v, w):
        r_u = self.find(u)
        r_v = self.find(v)
        if r_u == r_v:
            return 0
        
        if self.size[r_u] < self.size[r_v]:
            r_u, r_v = r_v, r_u
            
        k = min(self.c_A[r_u], self.c_B[r_v]) + min(self.c_A[r_v], self.c_B[r_u])
        cost = w * k
        
        self.c_A[r_u] += self.c_A[r_v] - k
        self.c_B[r_u] += self.c_B[r_v] - k
        
        self.parent[r_v] = r_u
        self.size[r_u] += self.size[r_v]
        
        return cost
```
