$\Delta(P) = \sum_{R=b_P}^n [M(R) == a_P] (M(R) - M_2(R))$.
Notice that $M(R)$ is a non-decreasing function of $R$.
Let's analyze the structure of $M(R)$.
$M(R)$ changes only at $R$ where there is a pair $[a, R]$.
Actually, $M(R)$ can change at any $R$, but it only changes when we add a new pair.
Wait, $M(R)$ is the max $a$ for pairs with $b \le R$.
So $M(R)$ only increases, and it only increases at $R$ where there is a pair $[a, R]$ with $a > M(R-1)$.
So $M(R)$ is a step function.
We can find all the intervals where $M(R)$ is constant.
For an interval $[L, R]$ where $M$ is constant and equal to $v$, and $M_2$ is constant and equal to $u$,
the contribution of this interval to $\Delta(P)$ for $P=[a, b]$ is:
If $v == a$, it adds $(R - \max(L, b) + 1) \times (v - u)$. (Assuming $R \ge b$).
Since $M(R)$ is non-decreasing, the intervals where $M(R) == a_P$ will form a single contiguous segment (or empty).
Wait, is it a single contiguous segment?
Yes, because $M(R)$ is non-decreasing. So the set of $R$ where $M(R) == a_P$ is an interval $[R_{start}, R_{end}]$.
Wait, what if $M(R)$ increases to $a_P$, stays at $a_P$ for a while, and then increases?
Yes, it will be a single interval $[R_{start}, R_{end}]$.
So for each pair $P=[a, b]$, we just need to find the interval $I = [L, R]$ where $M(R) == a$.
Then the sum is just the intersection of $[b, n]$ and $[L, R]$, multiplied by $(a - M_2)$.
But wait! $M_2(R)$ might NOT be constant on the interval where $M(R) == a$.
Let's check.
Suppose pairs are $[2, 3], [1, 4], [1, 5]$.
R=3: M=2, M2=0.
R=4: M=2, M2=1.
R=5: M=2, M2=1.
Here $M(R) == 2$ for $R \in [3, 5]$.
But $M_2(R)$ is 0 for $R=3$, and 1 for $R \in [4, 5]$.
So $M(R) - M_2(R)$ is 2 for $R=3$, and 1 for $R \in [4, 5]$.
So we cannot just assume $M_2$ is constant.

However, $M_2(R)$ is also a step-like function, but it can fluctuate?
No, $M_2(R)$ is the second maximum. It can fluctuate!
Wait, let's trace:
Pairs: $[5, 2], [4, 3], [3, 4]$.
R=1: M=0, M2=0
R=2: add 5. M=5, M2=0.
R=3: add 4. M=5, M2=4.
R=4: add 3. M=5, M2=4.
Here M2 is non-decreasing.

What if pairs are $[5, 2], [3, 3], [4, 4]$?
R=1: M=0, M2=0
R=2: add 5. M=5, M2=0.
R=3: add 3. M=5, M2=3.
R=4: add 4. M=5, M2=4.
Still non-decreasing.

Is $M_2(R)$ always non-decreasing?
Suppose pairs are $[5, 2], [4, 4], [3, 3]$.
Wait, pairs are added in order of $R$.
So at R=2, add 5. M=5, M2=0.
At R=3, add 3. M=5, M2=3.
At R=4, add 4. M=5, M2=4.
It's still non-decreasing!
Because we only ADD elements to the set of active $a$'s.
When we add an element $a$, the new top 2 maximums can only INCREASE or stay the same!
Let's prove this.
Let $S_R$ be the set of $a$'s for pairs with $b \le R$.
$S_{R} = S_{R-1} \cup \{a \mid [a, R] \in \text{Pairs}\}$.
Since we only add elements to the set, the maximum element can only increase or stay the same.
The second maximum element can only increase or stay the same.
So $M(R)$ and $M_2(R)$ are BOTH non-decreasing functions of $R$!
Wow! This is a huge realization.
Since $M(R)$ and $M_2(R)$ are non-decreasing, the function $M(R) - M_2(R)$ is... well, not necessarily monotonic, but we can just find the points where they change.
Since $M(R)$ and $M_2(R)$ only change when we add a pair, they change at most $O(\text{number of pairs})$ times!
So there are at most $O(\text{number of pairs})$ intervals where both $M(R)$ and $M_2(R)$ are constant.
Let's call these "segments".
For each segment $[L, R]$, $M$ is constant $v$, $M_2$ is constant $u$.
The contribution of this segment to $\Delta(P)$ for $P=[a, b]$ is:
If $v == a$, it adds $(R - \max(L, b) + 1) \times (v - u)$, provided $R \ge b$.
Since $M$ is non-decreasing, for a fixed $a$, the segments where $v == a$ form a single contiguous interval of $R$.
Wait, is it a single contiguous interval?
Yes, because $M(R)$ is non-decreasing. So the set of $R$ where $M(R) == a$ is an interval $[R_{start}, R_{end}]$.
Within this interval, $M_2(R)$ is non-decreasing.
So $M(R) - M_2(R)$ is non-increasing within this interval!
This means we can just find the segments where $M(R) == a_P$.
For each pair $P=[a, b]$, we can just sum over the segments where $v == a$.
Since the number of segments is $O(\text{number of pairs})$, and we have $O(\text{number of pairs})$ pairs, doing this naively would still be $O(P^2)$, which is $O(n^2)$.

We need a faster way to compute $\Delta(P)$ for all $P$.
Let's rewrite $\Delta(P)$:
$\Delta(P) = \sum_{R=b_P}^n [M(R) == a_P] (M(R) - M_2(R))$.
Notice that $P$ only appears in the condition $R \ge b_P$.
Let $f(a, R) = [M(R) == a] (M(R) - M_2(R))$.
Then $\Delta(P) = \sum_{R=b_P}^n f(a_P, R)$.
This is just a suffix sum!
For a fixed $a$, let $S_a(R) = \sum_{i=R}^n f(a, i)$.
Then $\Delta(P) = S_{a_P}(b_P)$.
So if we precompute $S_a(R)$ for all $a$ and $R$, we can answer each $\Delta(P)$ in $O(1)$.
But $a$ can be up to $n$, and $R$ up to $n$. Precomputing a 2D array is $O(n^2)$, too large.

Wait, $f(a, R)$ is non-zero only when $M(R) == a$.
Since $M(R)$ is non-decreasing, for each $a$, the set of $R$ where $M(R) == a$ is a single interval $[L_a, R_a]$.
And within this interval, $M_2(R)$ is non-decreasing, so $f(a, R)$ is non-increasing.
We can just store the segments where $M(R)$ and $M_2(R)$ are constant.
There are at most $P$ such segments.
For each segment, we know its range of $R$, its value of $v = M(R)$, and $u = M_2(R)$.
The contribution of this segment to $\Delta(P)$ for $P=[a, b]$ is:
If $v == a$, it adds $(R_{seg} - \max(L_{seg}, b) + 1) \times (v - u)$, provided $R_{seg} \ge b$.
This means for a fixed $a$, we just need to look at the segments where $v == a$.
Since $M(R)$ is non-decreasing, all segments with $v == a$ are contiguous.
Let these contiguous segments be $seg_1, seg_2, \ldots, seg_k$.
For a query with $P=[a, b]$, we just need to find the first segment $seg_i$ that ends at or after $b$, and then sum the contributions.
Since $k$ is the total number of segments across all $a$, and $k \le P$, we can just process the pairs for each $a$.
Wait, for a fixed $a$, how many pairs $P=[a, b]$ can there be?
There can be multiple pairs with the same $a$.
For a fixed $a$, the segments where $v == a$ are already computed.
Let the segments for $a$ be $seg_1, \ldots, seg_k$.
The total length of these segments is $R_a - L_a + 1$.
For each pair $P=[a, b]$, we need to compute the sum of contributions from $b$ to $n$.
This is equivalent to: for each segment, its contribution is $\max(0, R_{seg} - \max(L_{seg}, b) + 1) \times (v - u)$.
Since the segments are sorted by $R$, and the pairs are sorted by $b$, we can compute this efficiently.
Wait, even simpler: we can just compute the prefix/suffix sums of the contributions for each $a$.
For a fixed $a$, we can create an array `contrib` of size $n+1$, where `contrib[R] = f(a, R)`.
Then $\Delta(P)$ is just the suffix sum of `contrib` starting at $b_P$.
But creating an array of size $n$ for each $a$ is $O(n^2)$.
However, `contrib` is piecewise constant!
So we can just represent `contrib` as a list of segments.
For a query $b_P$, we want the suffix sum.
We can precompute the suffix sums of the segments!
Let the segments for $a$ be $s_1, s_2, \ldots, s_k$.
$s_i$ has range $[L_i, R_i]$ and value $V_i = a - M_2(R)$.
The suffix sum starting at $b$ is:
$\sum_{i=1}^k \max(0, R_i - \max(L_i, b) + 1) \times V_i$.
We can precompute the suffix sums of the segments:
Let $S_i = \sum_{j=i}^k (R_j - L_j + 1) \times V_j$.
Let $W_i = \sum_{j=i}^k V_j$.
Then for a query $b$, we find the first segment $i$ such that $R_i \ge b$.
If $L_i \le b \le R_i$, the answer is $(R_i - b + 1) \times V_i + S_{i+1}$.
If $b > R_i$, we go to the next segment.
Since the segments are sorted, we can find the segment $i$ using binary search for each query $P=[a, b]$.
Since the total number of segments is $O(P)$, and the total number of queries is $P$, this will take $O(P \log P)$ time, which is well within the limits!

   Then for a query $b$:
   Find the first segment $i$ such that $R_i \ge b$.

   Query the suffix sum for $v = a$ and $b = b_P$.
   This gives $\Delta(P)$.
   Keep track of the maximum $\Delta(P)$.

Let's refine the segment suffix sum logic.
For a list of segments `S = [(L_1, R_1, u_1), ...]` for a fixed `v`:
```python
k = len(S)
len_arr = [R - L + 1 for L, R, u in S]
val_arr = [v - u for L, R, u in S]

S_suffix = [0] * (k + 1)
for i in range(k - 1, -1, -1):
    S_suffix[i] = S_suffix[i + 1] + len_arr[i] * val_arr[i]

def query(b):
    # find first segment with R >= b
    # since R is increasing, we can use bisect_right on R's
    # R_list = [R_1, R_2, ...]
    # idx = bisect_left(R_list, b)
    idx = bisect_left(R_list, b)
    if idx == k:
        return 0
    L_i, R_i, u_i = S[idx]
    if b <= L_i:
        return S_suffix[idx]
    else:
        return (R_i - b + 1) * val_arr[idx] + S_suffix[idx + 1]
```
This is extremely clean and $O(\log k)$ per query.
Since we need to do this for each pair $P = [a, b]$, the total time for queries is $O(P \log P)$.
And building the segments takes $O(n)$.
So the overall time complexity is $O(n + P \log P)$, which easily passes for $n, P \le 10^5$.

```python
class Solution:
    def maxSubarrays(self, n: int, conflictingPairs: List[List[int]]) -> int:
        P = len(conflictingPairs)
        
        # Normalize pairs
        norm_pairs = []
        active_at_R = [[] for _ in range(n + 1)]
        for a, b in conflictingPairs:
            left = min(a, b)
            right = max(a, b)
            norm_pairs.append((left, right))
            active_at_R[right].append(left)
            
        # Compute M and M2
        M = [0] * (n + 1)
        M2 = [0] * (n + 1)
        max1 = 0
        max2 = 0
        
        for R in range(1, n + 1):
            for left in active_at_R[R]:
                if left > max1:
                    max2 = max1
                    max1 = left
                elif left > max2:
                    max2 = left
            M[R] = max1
            M2[R] = max2

        V_S = 0
        for R in range(1, n + 1):
            V_S += R - M[R]
            
        # Extract segments
        segments = []
        L = 1
        for R in range(1, n + 1):
            if R == n or M[R] != M[R + 1] or M2[R] != M2[R + 1]:
                segments.append((L, R, M[R], M2[R]))
                L = R + 1

            if v not in segs_by_v:
                segs_by_v[v] = []
            segs_by_v[v].append((L_seg, R_seg, u))

        # We can just store the necessary components for binary search
        # For a given v, we have a list of segments.
        # We need R_list, S_suffix, val_arr, L_list
        query_data = {}
        for v, segs in segs_by_v.items():
            k = len(segs)
            R_list = [R for L, R, u in segs]
            L_list = [L for L, R, u in segs]
            val_arr = [v - u for L, R, u in segs]
            S_suffix = [0] * (k + 1)
            for i in range(k - 1, -1, -1):
                S_suffix[i] = S_suffix[i + 1] + (R_list[i] - L_list[i] + 1) * val_arr[i]
            query_data[v] = (R_list, L_list, val_arr, S_suffix)
            
        # Compute max Delta
        max_delta = 0
        for left, right in norm_pairs:
            v = left
            b = right
            if v in query_data:
                R_list, L_list, val_arr, S_suffix = query_data[v]
                # Binary search for first segment with R >= b
                idx = bisect_left(R_list, b)
                if idx < len(R_list):
                    if b <= L_list[idx]:
                        delta = S_suffix[idx]
                    else:
                        delta = (R_list[idx] - b + 1) * val_arr[idx] + S_suffix[idx + 1]
                    max_delta = max(max_delta, delta)
                    
        return V_S + max_delta
```

    def maxSubarrays(self, n: int, conflictingPairs: List[List[int]]) -> int:
        P = len(conflictingPairs)
        
        # Normalize pairs and group by right endpoint
        norm_pairs = []
        active_at_R = [[] for _ in range(n + 1)]
        for a, b in conflictingPairs:
            left = min(a, b)
            right = max(a, b)
            norm_pairs.append((left, right))
            active_at_R[right].append(left)
            
        # Compute M and M2
        M = [0] * (n + 1)
        M2 = [0] * (n + 1)
        max1 = 0
        max2 = 0
        
        for R in range(1, n + 1):

                if left > max1:
                    max2 = max1
                    max1 = left
                elif left > max2:
                    max2 = left
            M[R] = max1
            M2[R] = max2

                L = R + 1

            if v not in segs_by_v:

            segs_by_v[v].append((L_seg, R_seg, u))

        for v, segs in segs_by_v.items():

            R_list = [R for L, R, u in segs]
