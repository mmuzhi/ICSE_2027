The i-th query is as follows:

- Given an integer X_i and a character C_i, replace the X_i-th character of S with C_i. Then, print the number of times the string ABC appears as a substring in S.

Here, a substring of S is a string obtained by deleting zero or more characters from the beginning and zero or more characters from the end of S.
For example, ab is a substring of abc, but ac is not a substring of abc.

Input

The input is given from Standard Input in the following format:

X_2 C_2

The i-th line (1 \le i \le Q) should contain the answer to the i-th query.

Constraints


- 3 \le N \le 2 \times 10^5
- 1 \le Q \le 2 \times 10^5
- S is a string of length N consisting of uppercase English letters.
- 1 \le X_i \le N
- C_i is an uppercase English letter.

The problem asks us to maintain a string $S$ of length $N$ and process $Q$ queries.
Each query updates a character in $S$ and asks for the number of occurrences of the substring "ABC" in $S$.

A naive approach would take $O(N)$ per query, which is too slow.
We can maintain the count of "ABC" occurrences.
When we update the character at position $X_i$ to $C_i$, only the substrings of length 3 that include position $X_i$ can be affected.
There are at most 3 such substrings: starting at $X_i-2$, $X_i-1$, and $X_i$.

Then, we update the character in $S$.
After the update, we check the same 3 substrings again and increment the count if any of them is "ABC".
Then we print the count.

Let's write down the steps for the implementation:
1. Read N, Q.
2. Read S.
3. Compute initial count of "ABC".
   ```python
   count = 0
   for i in range(N - 2):
       if S[i:i+3] == "ABC":
           count += 1
   ```
4. Loop Q times:
   - Read X, C.
   - Convert X to 0-based index: `idx = X - 1`.
   - For `j` in `idx-2, idx-1, idx`:
     - If `0 <= j <= N - 3` and `S[j:j+3] == "ABC"`:
       - `count -= 1`
   - `S = S[:idx] + C + S[idx+1:]`
   - For `j` in `idx-2, idx-1, idx`:
     - If `0 <= j <= N - 3` and `S[j:j+3] == "ABC"`:
       - `count += 1`
   - Print `count`.

Wait, string slicing `S[j:j+3]` takes $O(1)$ in Python? No, string slicing creates a new string which takes $O(K)$ time where $K$ is the length of the slice. Here $K=3$, so it takes $O(1)$ time. But doing it multiple times is fine.
However, string concatenation `S = S[:idx] + C + S[idx+1:]` takes $O(N)$ time because strings are immutable in Python.
For $N=2 \times 10^5$ and $Q=2 \times 10^5$, $O(NQ)$ time will be $4 \times 10^{10}$ operations, which will TLE.
We must use a mutable data structure for $S$, like a `list` of characters.

Let's adjust the implementation to use a list:
1. `S = list(S)`
2. To check `S[j:j+3] == "ABC"`, we can do:
   `S[j] == 'A' and S[j+1] == 'B' and S[j+2] == 'C'`
3. To update: `S[idx] = C`

import sys

def solve():
    input = sys.stdin.read
    data = input().split()
    if not data:
        return
    N = int(data[0])
    Q = int(data[1])
    S = list(data[2])
    
    queries = data[3:]
    
    count = 0
    for i in range(N - 2):
        if S[i] == 'A' and S[i+1] == 'B' and S[i+2] == 'C':
            count += 1
            
    out = []
    idx = 0
    for i in range(Q):
        X = int(queries[idx]) - 1
        C = queries[idx+1]
        idx += 2
        
        for j in range(max(0, X - 2), min(N - 2, X) + 1):
            if S[j] == 'A' and S[j+1] == 'B' and S[j+2] == 'C':
                count -= 1
                
        S[X] = C
        
        for j in range(max(0, X - 2), min(N - 2, X) + 1):
            if S[j] == 'A' and S[j+1] == 'B' and S[j+2] == 'C':
                count += 1
                
        out.append(str(count))
        
    print('\n'.join(out))

if __name__ == '__main__':
    solve()
