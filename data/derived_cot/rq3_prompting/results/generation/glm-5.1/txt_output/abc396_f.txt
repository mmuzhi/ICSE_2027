```python
import sys

def solve():
    input = sys.stdin.read
    data = input().split()
    if not data:
        return
    N = int(data[0])
    M = int(data[1])
    A = [int(x) for x in data[2:]]

    pos = [[] for _ in range(M)]
    for i in range(N):
        pos[A[i]].append(i + 1)

    # Fenwick Tree for initial inversions
    bit = [0] * (M + 1)

    def bit_add(i, x):
        i += 1
        while i <= M:
            bit[i] += x
            i += i & -i

    def bit_sum(i):
        s = 0
        i += 1
        while i > 0:
            s += bit[i]
            i -= i & -i
        return s

    inv = 0
    for x in A:
        inv += bit_sum(M - 1) - bit_sum(x)
        bit_add(x, 1)

    ans = [0] * M
    ans[0] = inv

    for k in range(M - 1):
        v = M - 1 - k
        C = len(pos[v])
        S = sum(pos[v])
        inv += 2 * S - C * N - C
        ans[k + 1] = inv

    print('\n'.join(map(str, ans)))

if __name__ == '__main__':
    solve()
```

```python
        i = x + 1
```

Wait, is there an O(N + M) way to compute initial inversions?
Yes, using a simple array and prefix sums, but since we need the count of elements *before* a given position, it's not straightforward without position BIT.
However, there's a trick.
Since we want to count pairs (i, j) with A[i] > A[j] and i < j.
This is equivalent to:
For each j, count the number of i < j such that A[i] > A[j].
We can maintain an array `cnt` of size M.
`cnt[v]` is the number of times `v` has appeared so far.
When we are at index j, we want the sum of `cnt[w]` for `w > A[j]`.
This is exactly a suffix sum.
Instead of a BIT, we can maintain a prefix sum array and update it.
But updating a suffix sum takes O(M) time, leading to O(NM) which is too slow.
So BIT is necessary.
Is there any other way?
We can use an array `cnt` and a variable `total`.
When we process A[j], the number of inversions ending at j is `total - sum(cnt[w] for w <= A[j])`.
This still requires a range sum query, which BIT does in O(log M).
So O(N log M) is the best we can do easily.
Given N, M <= 2e5, O(N log M) is ~3.4e6 operations, which is extremely fast in Python.

One final check on the inversion number formula:
`inv += 2 * S - C * N - C`
Let's re-verify its derivation.
For a wrapping element at position `p_r`:
- It loses an inversion with each element to its right that does NOT wrap.
  Number of elements to its right = `N - p_r`.
  Number of elements to its right that DO wrap = `C - r`.
  So elements to its right that do NOT wrap = `N - p_r - (C - r)`.
- It gains an inversion with each element to its left that does NOT wrap.
  Number of elements to its left = `p_r - 1`.
  Number of elements to its left that DO wrap = `r - 1`.
  So elements to its left that do NOT wrap = `p_r - 1 - (r - 1) = p_r - r`.

Net change for this element:
`(p_r - r) - (N - p_r - C + r) = 2p_r - 2r - N + C`.

Summing over all `r` from `1` to `C`:
`\sum_{r=1}^C (2p_r - 2r - N + C)`
`= 2 \sum p_r - 2 \frac{C(C+1)}{2} - CN + C^2`
`= 2 \sum p_r - C(C+1) - CN + C^2`
`= 2 \sum p_r - C^2 - C - CN + C^2`
`= 2 \sum p_r - CN - C`.

Yes, the derivation is completely solid.

```python
import sys

def solve():
    input = sys.stdin.read
    data = input().split()
    if not data:
        return
    N = int(data[0])
    M = int(data[1])
    A = [int(x) for x in data[2:]]

    # Store 1-based positions for each value in A
    pos = [[] for _ in range(M)]
    for i in range(N):
        pos[A[i]].append(i + 1)

    # Fenwick Tree (Binary Indexed Tree) to compute initial inversion count for k=0
    bit = [0] * (M + 1)

    inv = 0
    for x in A:
        # Query sum of elements > x (which is total elements - elements <= x)
        s_total = 0
        i = M
        while i > 0:
            s_total += bit[i]
            i -= i & -i
        
        s_x = 0
        i = x + 1
        while i > 0:
            s_x += bit[i]
            i -= i & -i
            
        inv += s_total - s_x
        
        # Add current element to BIT
        i = x + 1
        while i <= M:
            bit[i] += 1
            i += i & -i

    ans = [0] * M
    ans[0] = inv

    # Calculate inversion counts for k = 1 to M-1 based on the change from k-1
    for k in range(M - 1):
        v = M - 1 - k
        C = len(pos[v])
        S = sum(pos[v])
        # The change in inversion count is given by:
        # Delta = 2 * S - C * N - C
        inv += 2 * S - C * N - C
        ans[k + 1] = inv

    print('\n'.join(map(str, ans)))

if __name__ == '__main__':
    solve()
```
