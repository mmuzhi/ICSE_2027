Problem: Given a string `s`, iterate left to right. At each index `i`, find the closest unmarked index `j < i` such that `s[j]` is the mirror of `s[i]`. Mark both `i` and `j`, add `i - j` to score. If no such `j`, skip. Return total score.
Mirror: `a`<->`z`, `b`<->`y`, etc. Formula: `chr(ord('a') + ord('z') - ord(c))`.

We need to find the *closest* unmarked `j < i` for each `i`.
"Closest" means the largest `j < i` that is unmarked and `s[j]` is the mirror of `s[i]`.
Since we iterate `i` from left to right, the most recently seen unmarked `j` will be the closest!
So, for each character `c`, we can maintain a stack of unmarked indices where `s[index] == c`.
When we are at `i` and `s[i] == c`, we need the mirror of `c`, let's call it `m`.
If the stack for `m` is not empty, the top of the stack is the closest unmarked index `j`.
We pop `j` from the stack of `m`, and we DO NOT push `i` to the stack of `c` (since `i` is now marked).
If the stack for `m` is empty, we push `i` to the stack of `c` (since `i` is unmarked).

Let's trace Example 1: `s = "aczzx"`
Stacks for a..z are empty.
i=0, s[0]='a', m='z'. Stack['z'] is empty. Push 0 to Stack['a']. Stack['a']=[0].
i=1, s[1]='c', m='x'. Stack['x'] is empty. Push 1 to Stack['c']. Stack['c']=[1].
i=2, s[2]='z', m='a'. Stack['a'] has [0]. Pop 0. Score += 2 - 0 = 2. Stack['a']=[]. (Don't push 2).
i=3, s[3]='z', m='a'. Stack['a'] is empty. Push 3 to Stack['z']. Stack['z']=[3].
i=4, s[4]='x', m='c'. Stack['c'] has [1]. Pop 1. Score += 4 - 1 = 3. Stack['c']=[]. (Don't push 4).
Total score = 5. Correct.

Constraints: length up to 10^5. Time complexity O(N), Space O(N).
Data structure: 26 stacks (or just lists with append/pop).
State: 26 lists.
Loop order: i from 0 to len(s)-1.
Invariant: Stack[c] contains the indices of unmarked characters `c` seen so far, in increasing order.
Risks: None, this is very straightforward.

Implementation:
```python
class Solution:
    def calculateScore(self, s: str) -> int:
        stacks = [[] for _ in range(26)]
        score = 0
        for i, char in enumerate(s):
            c = ord(char) - ord('a')
            m = 25 - c
            if stacks[m]:
                j = stacks[m].pop()
                score += i - j
            else:
                stacks[c].append(i)
        return score
```

Check edge cases:
- No matches: score 0.
- All matches: e.g., "az". i=0, push 'a'. i=1, m='a', pop 0, score += 1.
- "aza". i=0, push 'a'. i=1, m='a', pop 0, score += 1. i=2, push 'a'.
- "azza". i=0, push 'a'. i=1, m='a', pop 0, score += 1. i=2, push 'z'. i=3, m='z', pop 2, score += 1. Total 2. Correct.

Wait, is it possible that a closer `j` exists but it's not at the top of the stack?
Since we process `i` from left to right, any index added to the stack will have a larger index than those already in the stack.
So the top of the stack always has the largest index (closest to `i`).
Yes, this perfectly maintains the "closest unmarked index" property.
