```python
def count(L, R, k):
    l_max = min(L, k)
    l_split = k + 1 - R
    if l_split < 1:
        m = l_max
        return m * (2 * k - m + 1) // 2
    else:

        res = l1 * R
        if l_max > l_split:
            m = l_max - l_split
            res += m * (2 * R - 1 - m) // 2
        return res
```

Now, we need to compute $L[i]$ and $R[i]$ for each element $A[i]$.
$L[i]$ is the distance to the previous strictly smaller element.
$R[i]$ is the distance to the next smaller or equal element.
We can use a monotonic stack to find these in $O(N)$ time.
For minimums:
```python
stack = []
for i in range(N):
    while stack and nums[stack[-1]] >= nums[i]:
        stack.pop()
    L[i] = i - stack[-1] if stack else i + 1
    stack.append(i)

stack = []
for i in range(N - 1, -1, -1):
    while stack and nums[stack[-1]] > nums[i]:
        stack.pop()
    R[i] = stack[-1] - i if stack else N - i
    stack.append(i)
```

For `nums = [2, 2, 2]`, i=0 covers [2], [2, 2], [2, 2, 2].
i=1 covers [2], [2, 2], [2, 2, 2].
i=2 covers [2], [2, 2], [2, 2, 2].
This is too much overlap.

Yes! Use "prev strictly smaller" and "next smaller or equal", BUT define "next smaller or equal" as the LAST element in the contiguous block of equal elements!

For L, we pop `>=`. This means L stops at the FIRST strictly smaller.
For R, we pop `>`. This means R stops at the FIRST smaller or equal.

Let's verify why.
L pops `>` means L is the distance to the prev SMALLER OR EQUAL.
R pops `>=` means R is the distance to the next STRICTLY SMALLER.
So for minimums:
L: prev smaller or equal.
R: next strictly smaller.
This means an element can extend to the left only up to the first element that is strictly smaller (wait, if L is smaller or equal, it stops at smaller or equal. So it CANNOT extend past smaller or equal. So it can only extend to elements that are STRICTLY GREATER).
And it can extend to the right up to the first element that is strictly smaller. So it CANNOT extend past strictly smaller. It can extend to elements that are GREATER OR EQUAL.
So an element is the minimum for subarrays that contain it and only contain elements to the left that are STRICTLY GREATER, and elements to the right that are GREATER OR EQUAL.
This exactly means it is the LEFTMOST MINIMUM!
If there are multiple equal minimums, the leftmost one will be the minimum, because the ones to its right cannot extend to the left past the leftmost one (since L stops at smaller or equal).
And the leftmost one CAN extend to the right past the equal ones (since R only stops at strictly smaller).
This is brilliant! Leftmost minimum!

`l_max \le L`. `l_split = k + 1 - R`.

```python
class Solution:
    def minMaxSubarraySum(self, nums: List[int], k: int) -> int:
        def count(L, R, k):
            l_max = min(L, k)
            l_split = k + 1 - R
            if l_split < 1:
                m = l_max
                return m * (2 * k - m + 1) // 2
            else:
                l1 = min(l_max, l_split)
                res = l1 * R
                if l_max > l_split:
                    m = l_max - l_split
                    res += m * (2 * R - 1 - m) // 2
                return res

        def get_sum(is_max):
            N = len(nums)
            L = [0] * N
            R = [0] * N
            
            # Calculate L
            stack = []
            for i in range(N):
                if is_max:
                    while stack and nums[stack[-1]] < nums[i]:
                        stack.pop()
                else:
                    while stack and nums[stack[-1]] > nums[i]:
                        stack.pop()
                L[i] = i - stack[-1] if stack else i + 1
                stack.append(i)
                
            # Calculate R
            stack = []
            for i in range(N - 1, -1, -1):
                if is_max:
                    while stack and nums[stack[-1]] <= nums[i]:
                        stack.pop()
                else:
                    while stack and nums[stack[-1]] >= nums[i]:
                        stack.pop()
                R[i] = stack[-1] - i if stack else N - i
                stack.append(i)
                
            # Calculate total sum
            total = 0
            for i in range(N):
                total += nums[i] * count(L[i], R[i], k)
            return total

        return get_sum(False) + get_sum(True)
```

```python
class Solution:
    def minMaxSubarraySum(self, nums: List[int], k: int) -> int:
        def count(L, R, k):
            l_max = min(L, k)
            l_split = k + 1 - R
            if l_split < 1:
                m = l_max
                return m * (2 * k - m + 1) // 2
            else:
                l1 = min(l_max, l_split)
                res = l1 * R
                if l_max > l_split:
                    m = l_max - l_split
                    res += m * (2 * R - 1 - m) // 2
                return res

        def get_sum(is_max):
            N = len(nums)
            L = [0] * N
            R = [0] * N
            
            # Calculate L: distance to prev smaller or equal (if min) / larger or equal (if max)
            stack = []
            for i in range(N):
                if is_max:
                    while stack and nums[stack[-1]] < nums[i]:
                        stack.pop()
                else:
                    while stack and nums[stack[-1]] > nums[i]:
                        stack.pop()
                L[i] = i - stack[-1] if stack else i + 1
                stack.append(i)
                
            # Calculate R: distance to next strictly smaller (if min) / strictly larger (if max)
            stack = []
            for i in range(N - 1, -1, -1):
                if is_max:
                    while stack and nums[stack[-1]] <= nums[i]:
                        stack.pop()
                else:
                    while stack and nums[stack[-1]] >= nums[i]:
                        stack.pop()
                R[i] = stack[-1] - i if stack else N - i
                stack.append(i)
                
            # Calculate total sum
            total = 0
            for i in range(N):
                total += nums[i] * count(L[i], R[i], k)
            return total

        return get_sum(False) + get_sum(True)
```
