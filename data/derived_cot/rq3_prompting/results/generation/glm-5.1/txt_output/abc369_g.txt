Can we implement a `Skew Heap` in Python?
Yes, a `Skew Heap` is very simple:
```python
class SkewHeap:
    def __init__(self, val=None, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def merge(h1, h2):
    if not h1: return h2
    if not h2: return h1
    if h1.val < h2.val: # max-heap
        h1, h2 = h2, h1
    h1.right = merge(h1.right, h2)
    h1.left, h1.right = h1.right, h1.left
    return h1
```
Wait, Python objects are slow. Creating $N$ objects might take some time, but $N=2 \times 10^5$ is fine.

2. Merge $u$'s heap and $v$'s heap.

How to add $L$ to the maximum element of $v$'s heap?
We can pop the maximum element, add $L$, and push it back!
Pop: `val = h.val; h = merge(h.left, h.right)`. Cost $O(\log N)$.
Push: `h = merge(h, SkewHeap(val + L))`. Cost $O(\log N)$.
Merge $u$ and $v$: `h_u = merge(h_u, h_v)`. Cost $O(\log N)$.
Total time for all operations: $O(N \log N)$!
This is incredibly fast and elegant!

            else:

                h_v = merge(h_v, SkewHeap(max_val + L))
            h_u = merge(h_u, h_v)
