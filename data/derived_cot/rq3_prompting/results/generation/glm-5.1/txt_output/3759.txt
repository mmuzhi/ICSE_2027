Naive approach is `O(n^2)`, which is too slow.
We need an `O(n log n)` approach.

Let's sort the elements by `nums1`.
If we process elements in increasing order of `nums1`, we can maintain a min-heap of size at most `k` containing the top `k` values of `nums2` seen so far.
Wait, if there are ties in `nums1`, we should process all elements with the same `nums1` value together, because for elements with the same `nums1`, none of them is strictly less than another.
So, we group by `nums1`.
For a group of elements with the same `nums1`, the answer for all of them is the sum of the top `k` elements in `nums2` from groups with strictly smaller `nums1`.
After computing the answers for the current group, we then add the `nums2` values of the current group to our min-heap.

Wait, the min-heap maintains the top `k` elements seen so far.
When we process a group of elements with the same `nums1`:
1. The current sum of the min-heap is exactly the answer for all elements in this group.
2. We record this answer for all elements in the group.
3. Then, we iterate through the elements in the group and add their `nums2` values to the min-heap.
   If the heap size exceeds `k`, we pop the smallest element and subtract it from the current sum.

Algorithm steps:
1. Create a list of tuples `(nums1[i], nums2[i], i)`.
2. Sort the list by `nums1[i]`.
3. Initialize a min-heap, `current_sum = 0`, and `ans` array of size `n`.
4. Iterate through the sorted list. Since we need to process groups of the same `nums1` together:
   - Find the boundaries of the current group.
   - For each element in the group, set `ans[i] = current_sum`.
   - For each element in the group, push `nums2[i]` to the min-heap, add to `current_sum`.
   - While `len(heap) > k`, pop from heap and subtract from `current_sum`.
5. Return `ans`.

Complexity:
Sorting takes `O(n log n)`.
Iterating and heap operations take `O(n log k)`.
Overall `O(n log n)`, which fits `n <= 10^5`.

```python
class Solution:
    def findMaxSum(self, nums1: List[int], nums2: List[int], k: int) -> List[int]:
        n = len(nums1)
        arr = [(nums1[i], nums2[i], i) for i in range(n)]
        arr.sort()
        
        ans = [0] * n
        min_heap = []
        current_sum = 0
        
        i = 0
        while i < n:
            j = i
            while j < n and arr[j][0] == arr[i][0]:
                ans[arr[j][2]] = current_sum
                j += 1
            
            for l in range(i, j):
                val = arr[l][1]
                heapq.heappush(min_heap, val)
                current_sum += val
                if len(min_heap) > k:
                    smallest = heapq.heappop(min_heap)
                    current_sum -= smallest
            
            i = j
            
        return ans
```

                j += 1
            
            # Add the nums2 values of the current group to the heap
            for l in range(i, j):
                val = arr[l][1]
                heapq.heappush(min_heap, val)

                if len(min_heap) > k:

            i = j
