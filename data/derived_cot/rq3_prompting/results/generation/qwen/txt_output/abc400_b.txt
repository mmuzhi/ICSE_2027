If it is, we output the sum; otherwise, we output 'inf'. The challenge lies in efficiently handling large values of \(N\) and \(M\) without directly computing very large numbers when possible.

### Approach
1. **Special Case Handling for N = 1**: When \(N\) is 1, the sum simplifies to \(M + 1\) because each term in the series is 1. Since \(M\) can be at most 100, \(M + 1\) will always be less than \(10^9\), so we can directly output \(M + 1\).

2. **Geometric Series Sum Calculation for N > 1**: For \(N > 1\), we compute the sum iteratively. Starting with the first term (1), we add each subsequent term \(N^i\) to a running total. If at any point the running total exceeds \(10^9\), we immediately output 'inf' and terminate the loop. If the loop completes without the total exceeding \(10^9\), we output the computed sum.

3. **Early Termination**: The loop breaks as soon as the running total exceeds \(10^9\), leveraging the fact that each subsequent term will only increase the sum further. This avoids unnecessary computations and handles very large numbers efficiently.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    N = int(data[0])
    M = int(data[1])
    
    if N == 1:
        print(M + 1)
    else:
        total = 0
        term = 1
        for _ in range(M + 1):
            total += term
            if total > 10**9:
                print('inf')
                return
            term *= N
        print(total)

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input consists of two integers, \(N\) and \(M\), read from standard input.

   - **Early Termination Check**: If `total` exceeds \(10^9\), 'inf' is printed, and the program exits.
