import heapq

class Solution:
    def findMaxSum(self, nums1: List[int], nums2: List[int], k: int) -> List[int]:
        n = len(nums1)
        # Combine the arrays and sort by nums1
        arr = [(nums1[i], nums2[i], i) for i in range(n)]
        arr.sort(key=lambda x: x[0])
        
        # Group by nums1
        groups = {}
        for a, b, i in arr:
            groups.setdefault(a, []).append((b, i))
        
        # Get the sorted keys (nums1 values)
        keys = sorted(groups.keys())
        
        heap = []
        current_sum = 0
        ans_arr = [0] * n
        
        for key in keys:
            # First, set the answer for all indices in this group
            for b, orig_idx in groups[key]:
                ans_arr[orig_idx] = current_sum
            
            # Then, update the candidate set (heap) with all nums2 values from this group
            for b, orig_idx in groups[key]:
                if len(heap) < k:
                    heapq.heappush(heap, b)
                    current_sum += b
                else:
                    # If the heap is full, then we only update if b is larger than the smallest in the heap
                    if b > heap[0]:
                        popped = heapq.heappop(heap)
                        heapq.heappush(heap, b)
                        current_sum = current_sum - popped + b
        
        return ans_arr

To solve this problem, we need to determine for each index `i` in two given arrays `nums1` and `nums2` the maximum sum obtainable by selecting at most `k` values from `nums2` at indices `j` where `nums1[j]` is strictly less than `nums1[i]`. The solution involves efficiently processing the arrays to avoid the O(n^2) brute-force approach, which would be infeasible for large inputs.

### Approach
1. **Problem Analysis**: For each index `i`, we need to consider all indices `j` where `nums1[j] < nums1[i]` and then select the top `k` values from `nums2[j]` to maximize the sum. The challenge is to do this efficiently for large arrays (up to 10^5 elements).

2. **Intuition**: By sorting the indices based on `nums1` values, we can process elements in increasing order. For each element, the candidate set consists of all previously processed elements (those with smaller `nums1` values). We maintain a min-heap to keep track of the top `k` values from this candidate set, allowing us to efficiently compute the maximum sum.

3. **Algorithm Selection**: 
   - **Sorting and Grouping**: Combine `nums1` and `nums2` into tuples and sort them by `nums1`. Group indices with the same `nums1` value together.
   - **Min-Heap for Top k Values**: Use a min-heap to store the top `k` values from the candidate set. The heap helps in efficiently updating and retrieving the top `k` values as we process each group.
   - **Processing Groups**: For each group (sorted by `nums1`), first compute the answer for all indices in the group using the current top `k` values from the candidate set. Then, update the candidate set by adding the values from the current group.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(n log n) due to sorting and O(n log k) for heap operations, where n is the number of elements. This is efficient for the given constraints.
   - **Space Complexity**: O(n) for storing the grouped data and the heap.

### Solution Code
```python
import heapq
from typing import List

class Solution:
    def findMaxSum(self, nums1: List[int], nums2: List[int], k: int) -> List[int]:
        n = len(nums1)
        arr = [(nums1[i], nums2[i], i) for i in range(n)]
        arr.sort(key=lambda x: x[0])
        
        groups = {}
        for a, b, i in arr:
            groups.setdefault(a, []).append((b, i))
        
        keys = sorted(groups.keys())
        
        heap = []
        current_sum = 0
        ans_arr = [0] * n
        
        for key in keys:
            for b, orig_idx in groups[key]:
                ans_arr[orig_idx] = current_sum
            
            for b, orig_idx in groups[key]:
                if len(heap) < k:
                    heapq.heappush(heap, b)
                    current_sum += b
                else:
                    if b > heap[0]:
                        popped = heapq.heappop(heap)
                        heapq.heappush(heap, b)
                        current_sum = current_sum - popped + b
        
        return ans_arr
```

### Explanation
1. **Sorting and Grouping**: The combined array of tuples (from `nums1` and `nums2`) is sorted by `nums1` values. This allows processing elements in increasing order of `nums1`, ensuring that for any element, all previous elements have smaller `nums1` values and thus form the candidate set.
2. **Processing Groups**: For each group of elements with the same `nums1` value, the solution first records the current top `k` values' sum (from the candidate set) for each index in the group. Then, it updates the candidate set by adding the values from the current group, maintaining the top `k` values using a min-heap.
3. **Heap Operations**: The min-heap efficiently keeps track of the top `k` values. When adding a new value, if the heap has fewer than `k` elements, the value is added directly. Otherwise, if the new value is larger than the smallest value in the heap, the smallest value is replaced to maintain the top `k` values.
4. **Result Construction**: The results for each original index are stored in an answer array during the processing of their respective groups, ensuring the solution meets the problem requirements efficiently.
