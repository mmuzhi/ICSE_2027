To solve this problem, we need to determine the minimum number of edges to remove from a given undirected graph so that the resulting graph becomes a forest (i.e., contains no cycles). The solution involves analyzing the graph's structure and leveraging the properties of connected components and spanning trees.

### Approach
1. **Problem Analysis**: The key observation is that a forest is a collection of trees, where each tree is a connected acyclic graph. The maximum number of edges a forest can have is given by \( N - K \), where \( N \) is the number of vertices and \( K \) is the number of connected components in the forest. The minimum number of edges to remove is then the total edges minus this maximum, i.e., \( M - (N - K) \).

2. **Insight**: The solution hinges on calculating the number of connected components \( K \) in the original graph. Each connected component must be a tree in the final forest, which means each component can have at most \( (number\ of\ vertices\ in\ component - 1) \) edges. The total edges to keep is \( N - K \), so the edges to remove is \( M - (N - K) \).

3. **Algorithm Selection**: We use the Union-Find (Disjoint Set Union, DSU) data structure to efficiently compute the number of connected components. Union-Find helps in merging sets and tracking the number of distinct components with path compression and union by rank for optimal performance.

4. **Complexity Analysis**: The Union-Find operations (find and union) are nearly constant time due to path compression and union by rank, making the overall complexity \( O(N + M \alpha(N)) \), where \( \alpha \) is the inverse Ackermann function, which is efficient for large inputs (up to \( 2 \times 10^5 \) vertices and edges).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
        
    it = iter(data)
    n = int(next(it)); m = int(next(it))
    
    if m == 0:
        print(0)
        return
        
    parent = list(range(n+1))
    rank = [0] * (n+1)
    
    def find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])
        return parent[x]
    
    def union(x, y):
        rx = find(x)
        ry = find(y)
        if rx == ry:
            return
        if rank[rx] < rank[ry]:
            parent[rx] = ry
        elif rank[rx] > rank[ry]:
            parent[ry] = rx
        else:
            parent[ry] = rx
            rank[rx] += 1
            
    for _ in range(m):
        u = int(next(it)); v = int(next(it))
        union(u, v)
        
    components = set()
    for i in range(1, n+1):
        components.add(find(i))
        
    k = len(components)
    result = m - (n - k)
    print(result)

if __name__ == '__main__':
    main()
```

3. **Union Operations**: For each edge, the union operation merges the sets containing the two vertices. Path compression and union by rank ensure the operations are efficient.

4. **Component Counting**: After processing all edges, the number of connected components \( K \) is determined by counting the unique root parents in the `parent` array.

5. **Result Calculation**: The result is computed as \( M - (N - K) \), which gives the minimum edges to remove to make the graph a forest. This is printed as the final answer.
