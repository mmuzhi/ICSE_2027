To solve this problem, we need to determine the minimum number of operations required to distribute exactly one stone in each cell of a row of N cells, starting from M cells that initially contain stones. The operation allowed is moving one stone from cell i to cell i+1 any number of times. The solution involves checking feasibility conditions and then calculating the minimal operations if feasible.

### Approach
1. **Feasibility Check**:
   - **Total Stones**: The total number of stones must exactly equal N, as each cell needs one stone. If not, it's impossible.
   - **Initial Positions**: The stones must be positioned such that for any cell j, the cumulative number of stones from positions 1 to j is at least j. This ensures we can cover all cells from left to right without gaps.

2. **Calculation of Minimal Operations**:
   - If feasible, the minimal operations required is the difference between the sum of the first N natural numbers (1 to N) and the sum of the products of each initial stone's position and its count. This is derived from the fact that each stone must be moved to its target cell, and the total displacement is minimized by optimally assigning stones to cells.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    m = int(data[1])
    X = list(map(int, data[2:2+m]))
    A = list(map(int, data[2+m:2+2*m]))
    
    total_stones = sum(A)
    if total_stones != n:
        print(-1)
        return
        
    stones = sorted(zip(X, A))
    
    current_total = 0
    prev = 0
    for i in range(m):
        x, a = stones[i]
        if prev + 1 <= x - 1:
            if current_total < x - 1:
                print(-1)
                return
                
        current_total += a
        if current_total < x:
            print(-1)
            return
            
        prev = x
        
    total_assigned = n * (n + 1) // 2
    initial_sum = 0
    for x, a_val in stones:
        initial_sum += x * a_val
        
    print(total_assigned - initial_sum)

if __name__ == '__main__':
    main()
```

1. **Feasibility Check**:
   - The code first checks if the total number of stones matches N. If not, it immediately returns -1.
   - The initial positions of the stones are sorted. For each stone, it checks if the cumulative stones from previous positions can cover the gap up to the current stone's position. If not, it returns -1.
   - After processing all stones, it ensures the cumulative stones up to each position meet the required condition.

2. **Minimal Operations Calculation**:
   - The sum of the first N natural numbers (total_assigned) represents the ideal target positions for all stones.
   - The initial_sum is the sum of each stone's initial position multiplied by its count.
   - The difference between total_assigned and initial_sum gives the minimal operations needed to move stones from their initial positions to cover all cells exactly once.

This approach efficiently checks for feasibility and calculates the minimal operations using basic arithmetic and sorting, ensuring optimal performance even for large inputs.
