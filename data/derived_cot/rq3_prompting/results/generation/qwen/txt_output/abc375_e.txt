To solve this problem, we need to determine if it's possible to redistribute the people among three teams such that each team has the same total strength, and if possible, find the minimum number of people who need to switch teams to achieve this. 

### Approach
1. **Problem Analysis**: The problem involves redistributing people among three teams to ensure each team's total strength is one-third of the overall sum of all people's strengths. The solution requires checking if such a redistribution is possible and finding the minimal number of people who need to switch teams.

2. **Key Insight**: The total sum of all people's strengths must be divisible by 3 for an equal distribution. If not, it's impossible, and we return -1. 

3. **Dynamic Programming (DP) Setup**: We use a DP approach where the state is defined by the current sums of two teams (since the third team's sum can be derived from the total). The DP dictionary keeps track of the minimal number of reassignments needed to reach each state.

4. **State Pruning**: To optimize, we prune states where the sum of any team exceeds the target value or the combined sum of two teams exceeds twice the target value, as these states cannot lead to a valid solution.

5. **DP State Transitions**: For each person, we consider three options: assigning them to team 1, team 2, or team 3. For each option, we update the DP state and the count of reassignments if the person is moved from their initial team.

6. **Final Check**: After processing all people, we check if the state where both teams have the target sum exists. If it does, the minimal reassignments for this state is our answer; otherwise, it's impossible, and we return -1.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        print(0)
        return
        
    n = int(data[0])
    A = []
    B = []
    for i in range(1, n+1):
        parts = data[i].split()
        if len(parts) < 2:
            continue
        A.append(int(parts[0]))
        B.append(int(parts[1]))
    
    total = sum(B)
    if total % 3 != 0:
        print(-1)
        return
        
    T = total // 3
    
    dp = {(0, 0): 0}
    
    for i in range(n):
        new_dp = {}
        for (s1, s2), count in dp.items():
            if s1 > T or s2 > T or s1 + s2 > 2 * T:
                continue
                
            new_s1 = s1 + B[i]
            if new_s1 <= T and s2 <= T and new_s1 + s2 <= 2 * T:
                new_state = (new_s1, s2)
                add = 0 if A[i] == 1 else 1
                new_count = count + add
                if new_state in new_dp:
                    if new_count < new_dp[new_state]:
                        new_dp[new_state] = new_count
                else:
                    new_dp[new_state] = new_count
                    
            new_s2 = s2 + B[i]
            if s1 <= T and new_s2 <= T and s1 + new_s2 <= 2 * T:
                new_state = (s1, new_s2)
                add = 0 if A[i] == 2 else 1
                new_count = count + add
                if new_state in new_dp:
                    if new_count < new_dp[new_state]:
                        new_dp[new_state] = new_count
                else:
                    new_dp[new_state] = new_count
                    
            if s1 <= T and s2 <= T and s1 + s2 <= 2 * T:
                new_state = (s1, s2)
                add = 0 if A[i] == 3 else 1
                new_count = count + add
                if new_state in new_dp:
                    if new_count < new_dp[new_state]:
                        new_dp[new_state] = new_count
                else:
                    new_dp[new_state] = new_count
                    
        dp = new_dp
        
    if (T, T) in dp:
        print(dp[(T, T)])
    else:
        print(-1)

if __name__ == "__main__":
    main()
```

3. **DP Initialization**: The DP starts with a state (0, 0) representing zero strength in both teams, requiring zero reassignments.
4. **Processing Each Person**: For each person, we explore three possibilities: assigning them to team 1, team 2, or team 3. For each assignment, we update the DP state and the reassignment count if the person moves from their initial team.
5. **State Pruning**: States where any team's sum exceeds the target or the combined sum of two teams exceeds twice the target are discarded to keep the state space manageable.
