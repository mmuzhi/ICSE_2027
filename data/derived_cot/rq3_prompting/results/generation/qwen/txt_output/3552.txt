=== ANSWER ===

To solve this problem, we need to find the largest integer with exactly `n` digits that is both a palindrome and divisible by `k`. The solution involves generating the largest possible palindrome by considering the first half of the number and then mirroring it appropriately to form the full palindrome. We then check if this palindrome is divisible by `k`. If not, we decrement the first half and repeat the process until we find a valid palindrome.

### Approach
1. **Special Case Handling for k=1**: Any number is divisible by 1, so the largest `n`-digit palindrome (which is a string of `n` '9's) is the answer.
2. **Determine the First Half**: For an `n`-digit palindrome, the first half (or first half plus middle digit for odd `n`) determines the entire palindrome. The length of this first half is `m = (n + 1) // 2`.
3. **Modular Arithmetic**: To efficiently check divisibility by `k`, we use modular arithmetic. The entire number can be expressed as `A * 10^p + B`, where `A` is the first half, `B` is the mirrored part, and `p` is the number of digits in the second half. The modulus of this number is computed using the moduli of `A`, `B`, and `10^p`.
4. **Iterate from Maximum Candidate**: Start with the largest possible first half (all '9's) and decrement it until the resulting palindrome is divisible by `k`. The decrement operation is handled digit-wise to avoid overflow and ensure correct borrowing.

### Solution Code
```python
class Solution:
    def largestPalindrome(self, n: int, k: int) -> str:
        if k == 1:
            return '9' * n
        
        m = (n + 1) // 2
        p = n - m
        power_mod = pow(10, p, k)
        
        def decrement(s):
            digits = list(map(int, list(s)))
            i = len(digits) - 1
            while i >= 0 and digits[i] == 0:
                digits[i] = 9
                i -= 1
            if i < 0:
                return None
            digits[i] -= 1
            return ''.join(map(str, digits))
        
        s = '9' * m
        while s[0] != '0':
            A_mod = 0
            for digit in s:
                A_mod = (A_mod * 10 + int(digit)) % k
            
            if n % 2 == 1:
                B_str = s[:-1][::-1]
            else:
                B_str = s[::-1]
            B_mod = 0
            for digit in B_str:
                B_mod = (B_mod * 10 + int(digit)) % k
            
            total_mod = (A_mod * power_mod + B_mod) % k
            if total_mod == 0:
                return s + B_str
            
            s = decrement(s)
            if s is None or s[0] == '0':
                break
        
        return None
```

### Explanation
1. **Special Case Handling**: If `k` is 1, the solution is straightforward: the largest `n`-digit palindrome is a string of `n` '9's.
2. **First Half Calculation**: The first half of the palindrome is determined by `m = (n + 1) // 2`. The second half is derived by mirroring the first half (excluding the last digit for odd `n`).
3. **Modular Arithmetic**: The modulus of the entire number is computed using the moduli of the first half (`A_mod`), the mirrored second half (`B_mod`), and `10^p mod k`. This avoids dealing with very large numbers directly.
4. **Decrement and Check**: Starting from the largest possible first half (all '9's), the solution checks each subsequent smaller first half (by decrementing) until a palindrome divisible by `k` is found. The decrement operation is handled carefully to maintain the correct digit length and avoid leading zeros.
5. **Efficiency**: The algorithm efficiently checks each candidate by leveraging modular arithmetic, ensuring that even very large `n` (up to 10^5) can be handled without directly computing large numbers. The use of modular arithmetic keeps the computations within feasible limits.
