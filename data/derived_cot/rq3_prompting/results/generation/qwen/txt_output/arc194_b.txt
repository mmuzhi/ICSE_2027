To solve this problem, we need to find the minimum total cost required to sort a permutation in ascending order using adjacent swaps, where each swap at position `i` (1-indexed) costs `i`. The challenge lies in determining the optimal sequence of swaps to minimize the total cost while ensuring the permutation is sorted.

### Approach
1. **Problem Analysis**: The problem involves transforming a given permutation into a sorted array using adjacent swaps. Each swap operation between elements at positions `i` and `i+1` costs `i`. The goal is to minimize the total cost of these operations.

2. **Key Insight**: The minimal cost is derived from the sum of the costs of all swaps performed. Each swap's cost is determined by its left index (1-indexed). The optimal sequence of swaps must efficiently move elements to their target positions with minimal cost.

3. **Algorithm Selection**: 
   - **Tracking Positions**: For each element in the permutation, determine its initial and target positions. The target position for element `x` is `x-1` (0-indexed) or `x` (1-indexed).
   - **Cost Calculation**: For each element, calculate the cost of moving it from its initial position to its target position. The cost is the sum of the indices of the swaps that move the element, which can be derived from the initial and target positions.
   - **Total Cost**: The total cost is the sum of the individual costs for all elements, divided by 2, since each swap operation is counted twice (once for each element involved).

4. **Complexity Analysis**: The algorithm processes each element once, making it linear in time complexity, O(N), where N is the length of the permutation. This is efficient and handles the constraint N ≤ 2 × 10^5 effectively.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    n = int(data[0])
    arr = list(map(int, data[1:1+n]))
    
    if arr == list(range(1, n+1)):
        print(0)
        return
        
    pos = [0] * (n+1)
    for i in range(n):
        pos[arr[i]] = i
        
    total = 0
    for x in range(1, n+1):
        initial = pos[x]
        target = x-1
        if initial < target:
            total += sum(range(initial+1, target+1))
        else:
            total += sum(range(target+1, initial+1))
            
    total //= 2
    print(total)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into an integer `n` (length of the permutation) and a list `arr` (the permutation itself).
2. **Check for Sorted Array**: If the array is already sorted, the cost is 0.
3. **Position Tracking**: An array `pos` is used to store the initial positions of each element in the permutation.
4. **Cost Calculation**: For each element `x` from 1 to N:
   - If the element's initial position is less than its target position, the cost is the sum of indices from `initial+1` to `target+1`.
   - If the element's initial position is greater than its target position, the cost is the sum of indices from `target+1` to `initial+1`.
5. **Total Cost**: The total cost is computed by summing the individual costs for all elements and then dividing by 2 (since each swap is counted twice). The result is printed as the minimal total cost.

This approach efficiently calculates the minimal cost by leveraging the positions of elements and the cost structure of the swaps, ensuring optimal performance even for large input sizes.
