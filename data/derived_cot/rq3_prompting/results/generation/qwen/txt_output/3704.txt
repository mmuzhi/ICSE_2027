To solve this problem, we need to determine the number of ways to partition an array such that the difference between the sum of the left and right subarrays is even. 

### Approach
1. **Problem Analysis**: The key observation here is that the difference between the left and right subarray sums can be expressed as \(2 \times \text{left\_sum} - \text{total\_sum}\). Since \(2 \times \text{left\_sum}\) is always even, the parity of the difference depends solely on the parity of the total sum of the array. 
2. **Insight**: If the total sum of the array is even, then the difference between the left and right subarray sums will be even for every possible partition. Conversely, if the total sum is odd, no partition will satisfy the condition.
3. **Algorithm Selection**: The solution involves calculating the total sum of the array. If the total sum is even, the answer is simply the number of possible partitions, which is \(n - 1\) (where \(n\) is the length of the array). If the total sum is odd, the answer is 0.

```python
class Solution:
    def countPartitions(self, nums: List[int]) -> int:
        total = sum(nums)
        n = len(nums)
        if total % 2 == 0:
            return n - 1
        else:
            return 0
```

### Explanation
1. **Total Sum Calculation**: The first step is to compute the total sum of all elements in the array. This sum determines whether any partition can satisfy the given condition.
2. **Parity Check**: If the total sum is even, every partition will result in an even difference between the left and right subarray sums. The number of such partitions is \(n - 1\) because the array can be partitioned at any index from 0 to \(n-2\) (each partition splits the array into two non-empty parts).
3. **Result Determination**: If the total sum is odd, no partition will yield an even difference, so the result is 0. This approach efficiently checks the necessary condition without explicitly evaluating each partition, leading to an optimal solution with linear time complexity.
