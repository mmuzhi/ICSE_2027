To solve this problem, we need to transform a given integer `n` into another integer `m` of the same digit length using specific operations while ensuring that no number encountered during the transformation (including the initial and final numbers) is a prime. The operations allowed are increasing a digit (if it's not 9) or decreasing a digit (if it's not 0). The goal is to find the minimal cost, which is the sum of all numbers visited during the transformation. If the transformation is impossible, we return -1.

### Approach
1. **Initial Checks**: 
   - If `n` and `m` are the same, return `n` since no operations are needed.
   - If either `n` or `m` is a prime number, return -1 immediately since the problem requires all numbers (including the start and end) to be non-prime.

2. **Digit Length and Composite Set**:
   - Determine the number of digits `d` in `n` (and `m`).
   - Generate all composite numbers (non-prime) with `d` digits. These numbers will form the nodes of our graph.

3. **Graph Construction**:
   - For each composite number, generate neighboring numbers by incrementing or decrementing each digit (if allowed by the operation rules). The operation rules ensure that the number of digits remains the same and the resulting number is composite.
   - Construct a graph where each node (composite number) has edges to its valid neighbors.

4. **Dijkstra's Algorithm**:
   - Use Dijkstra's algorithm to find the shortest path from `n` to `m` in the graph, where the "distance" is the sum of the node values along the path. The priority queue (min-heap) is used to explore the least costly paths first.

### Solution Code
```python
import heapq

class Solution:
    max_val = 10000
    is_prime_arr = None

    def __init__(self):
        if Solution.is_prime_arr is None:
            Solution.is_prime_arr = self._sieve(self.max_val)

    @staticmethod
    def _sieve(n):
        is_prime = [True] * (n + 1)
        is_prime[0] = is_prime[1] = False
        for i in range(2, int(n**0.5) + 1):
            if is_prime[i]:
                for j in range(i * i, n + 1, i):
                    is_prime[j] = False
        return is_prime

    def minOperations(self, n: int, m: int) -> int:
        if n == m:
            return n

        if n <= self.max_val and m <= self.max_val and (self.is_prime_arr[n] or self.is_prime_arr[m]):
            return -1

        d = len(str(n))
        low = 10 ** (d - 1)
        high = 10 ** d - 1

        composite_set = set()
        for num in range(low, high + 1):
            if not self.is_prime_arr[num]:
                composite_set.add(num)

        graph = {}
        for num in composite_set:
            s = str(num)
            for i in range(len(s)):
                if i == 0:
                    if int(s[i]) < 9:
                        new_num_str = str(int(s[i]) + 1) + s[1:]
                        new_num = int(new_num_str)
                        if new_num in composite_set:
                            self.add_edge(graph, num, new_num)
                else:
                    if int(s[i]) < 9:
                        new_num_str = s[:i] + str(int(s[i]) + 1) + s[i+1:]
                        new_num = int(new_num_str)
                        if new_num in composite_set:
                            self.add_edge(graph, num, new_num)

                if i == 0:
                    if int(s[i]) > 1:
                        new_num_str = str(int(s[i]) - 1) + s[1:]
                        new_num = int(new_num_str)
                        if new_num in composite_set:
                            self.add_edge(graph, num, new_num)
                else:
                    if int(s[i]) > 0:
                        new_num_str = s[:i] + str(int(s[i]) - 1) + s[i+1:]
                        new_num = int(new_num_str)
                        if new_num in composite_set:
                            self.add_edge(graph, num, new_num)

        dist = {num: float('inf') for num in composite_set}
        dist[n] = n
        heap = [(n, n)]
        while heap:
            cost, node = heapq.heappop(heap)
            if node == m:
                return cost
            if cost != dist[node]:
                continue
            for neighbor in graph.get(node, []):
                new_cost = cost + neighbor
                if new_cost < dist[neighbor]:
                    dist[neighbor] = new_cost
                    heapq.heappush(heap, (new_cost, neighbor))

        return -1

    @staticmethod
    def add_edge(graph, u, v):
        if u not in graph:
            graph[u] = []
        graph[u].append(v)
```

### Explanation
1. **Initial Checks**: The solution starts by checking if `n` and `m` are the same or if either is a prime number, returning -1 if either condition is met.
2. **Digit Length and Composite Set**: The number of digits `d` is determined, and all composite numbers (non-prime) with `d` digits are collected into a set.
3. **Graph Construction**: For each composite number, neighbors are generated by incrementing or decrementing each digit (ensuring the resulting number remains composite and has the same digit length). These neighbors form edges in the graph.
4. **Dijkstra's Algorithm**: The algorithm explores the graph using a priority queue to find the path from `n` to `m` with the minimal sum of node values. If `m` is reached, the sum is returned; otherwise, -1 is returned if no path exists.

This approach efficiently navigates the problem constraints by leveraging graph theory and Dijkstra's algorithm to find the optimal path while ensuring all intermediate numbers are composite.
