To solve this problem, we need to count the number of substrings in a given string that contain every vowel ('a', 'e', 'i', 'o', 'u') at least once and exactly k consonants. 

### Approach
1. **Problem Analysis**: The task involves checking all possible substrings of the given string to determine if they meet two conditions: containing all five vowels at least once and having exactly k consonants. The challenge is to efficiently count these substrings without resorting to a brute-force approach which would be computationally expensive.

2. **Intuition**: The solution leverages prefix sums and a sliding window technique. First, we precompute a prefix array to keep track of the cumulative count of vowels up to each position in the string. This allows us to quickly calculate the number of vowels in any substring. 

3. **Algorithm Selection**: 
   - **Prefix Sum Array**: We create an array `vowel_prefix` where each entry at index `i` holds the number of vowels from the start of the string up to (but not including) the character at position `i`. This helps in efficiently calculating the number of vowels in any substring.
   - **Minimal Window Calculation**: For each starting index `i`, we find the smallest ending index `j` such that the substring from `i` to `j` contains all five vowels. This is done using a frequency array to track the count of each vowel encountered.
   - **Substring Validation**: For each valid minimal window (from `i` to `j`), we extend the window to the right and check if the number of consonants in the substring equals `k`. The consonant count is derived from the substring length minus the vowel count.

4. **Complexity Analysis**: The algorithm involves two main steps:
   - **Prefix Sum Calculation**: O(n) time.
   - **Minimal Window Calculation**: O(n^2) time in the worst case, as for each starting index, we might scan up to `n` characters.
   - **Substring Validation**: O(n^2) time in the worst case, as for each valid minimal window, we might scan up to `n` characters.
   Given the constraint `n <= 250`, the total operations are manageable (around 62,500), making the approach feasible.

### Solution Code
```python
class Solution:
    def countOfSubstrings(self, word: str, k: int) -> int:
        n = len(word)
        vowel_prefix = [0] * (n + 1)
        for i in range(1, n + 1):
            if word[i - 1] in 'aeiou':
                vowel_prefix[i] = vowel_prefix[i - 1] + 1
            else:
                vowel_prefix[i] = vowel_prefix[i - 1]
        
        min_end = [n] * n
        vowels_set = "aeiou"
        for i in range(n):
            freq = [0] * 5
            j = i
            while j < n:
                if word[j] in vowels_set:
                    idx = vowels_set.index(word[j])
                    freq[idx] += 1
                    if all(x > 0 for x in freq):
                        min_end[i] = j
                        break
                j += 1
        
        ans = 0
        for i in range(n):
            if min_end[i] == n:
                continue
            for j in range(min_end[i], n):
                vowel_count = vowel_prefix[j + 1] - vowel_prefix[i]
                length = j - i + 1
                consonant_count = length - vowel_count
                if consonant_count == k:
                    ans += 1
        return ans
```

### Explanation
1. **Prefix Sum Array (`vowel_prefix`)**: This array is built by iterating through the string, incrementing the count at each position if the character is a vowel. This allows O(1) vowel count queries for any substring.
2. **Minimal Window Calculation (`min_end`)**: For each starting index `i`, we scan the string to find the smallest `j` such that the substring from `i` to `j` contains all five vowels. This is tracked using a frequency array.
3. **Substring Validation**: For each valid minimal window (from `i` to `j`), we extend the window to the right and use the prefix array to compute the vowel count in the substring. The consonant count is then derived from the substring length minus the vowel count. If this count matches `k`, the substring is counted.
4. **Efficiency**: The approach efficiently narrows down potential substrings by leveraging prefix sums and minimal window checks, ensuring optimal performance even for the upper constraint limits.
