To solve this problem, we need to efficiently answer multiple queries about the longest strictly increasing subsequence (LIS) in a prefix of an array where all elements are at most a given value. The challenge lies in handling these queries quickly for large input sizes, which requires an optimized approach.

### Approach
1. **Problem Analysis**: The problem involves processing up to 200,000 elements and 200,000 queries. Each query asks for the length of the longest strictly increasing subsequence in the first R elements of the array, with all elements not exceeding X. The solution must efficiently handle these queries without resorting to O(N) per query, which would be too slow for large N.

2. **Intuition**: The key insight is to preprocess the array and use a segment tree to maintain the maximum LIS length ending at each value. By compressing the array values and processing queries in increasing order of their prefix lengths, we can update the segment tree incrementally and answer each query by querying the segment tree for the maximum LIS length in the relevant range (values ≤ X).

3. **Algorithm Selection**: 
   - **Coordinate Compression**: Compress the distinct values of the array to a smaller range for efficient segment tree operations.
   - **Segment Tree**: Use a segment tree to support range maximum queries and point updates. The tree stores the maximum LIS length for each value, allowing us to query the maximum LIS length for values ≤ X quickly.
   - **Offline Processing**: Sort queries by their prefix length and process them in order, updating the segment tree as we go. This ensures that each query is answered using the most up-to-date segment tree state for its prefix.

4. **Complexity Analysis**: 
   - **Coordinate Compression**: O(N log N) for sorting and deduplication.
   - **Segment Tree Operations**: Each update and query takes O(log M) time, where M is the number of distinct values. With N elements and Q queries, the total time is O((N + Q) log M), which is feasible for the given constraints.

### Solution Code
```python
import sys
import bisect

sys.setrecursionlimit(300000)

class SegmentTree:
    def __init__(self, size):
        self.n = size
        self.size = 1
        while self.size < size:
            self.size *= 2
        self.data = [0] * (2 * self.size)
    
    def update(self, index, value):
        pos = index + self.size
        if value > self.data[pos]:
            self.data[pos] = value
            while pos > 1:
                pos //= 2
                self.data[pos] = max(self.data[2 * pos], self.data[2 * pos + 1])
                
    def query(self, l, r):
        if l > r:
            return 0
        l += self.size
        r += self.size
        res = 0
        while l <= r:
            if l % 2 == 1:
                res = max(res, self.data[l])
                l += 1
            if r % 2 == 0:
                res = max(res, self.data[r])
                r -= 1
            l //= 2
            r //= 2
        return res

        X = int(next(it))
        queries.append((R, X, i))
    
    comp = sorted(set(A))
    m = len(comp)
    mapping = {val: idx for idx, val in enumerate(comp)}
    
    seg_tree = SegmentTree(m)
    
    queries.sort(key=lambda x: x[0])
    
    index_ptr = 0
    ans = [0] * q
```
