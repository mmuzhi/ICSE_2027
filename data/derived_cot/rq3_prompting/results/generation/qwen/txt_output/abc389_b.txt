The solution involves efficiently determining \( N \) by leveraging the fact that factorials grow very rapidly, allowing us to use a simple loop to compute factorials until we reach or exceed \( X \).

### Approach
1. **Problem Analysis**: The problem requires finding \( N \) such that \( N! = X \). Given the constraints \( 2 \leq X \leq 3 \times 10^{18} \), we know that \( N \) will be at most 20 because 20! is approximately \( 2.4 \times 10^{18} \), and 21! exceeds \( 3 \times 10^{18} \).

2. **Insight**: Since factorials grow quickly, we can start from \( N = 2 \) and compute factorials iteratively until we either find the factorial equal to \( X \) or exceed \( X \). The problem guarantees that exactly one such \( N \) exists, so we can stop once the computed factorial matches \( X \).

3. **Algorithm Selection**: We use a simple loop starting from \( N = 2 \) and compute the factorial by multiplying the current value by the next integer in sequence. If at any point the computed factorial equals \( X \), we return the current \( N \). The loop condition ensures we stop when the factorial exceeds \( X \).

4. **Complexity Analysis**: The loop runs at most 20 times (since 20! is the largest factorial within the constraint), making the approach efficient with a time complexity of \( O(N) \), where \( N \) is the solution (at most 20).

### Solution Code
```python
X = int(input().strip())
n = 2
current = 2
while current < X:
    n += 1
    current *= n
print(n)
```

1. **Initialization**: Read the input integer \( X \) and initialize \( n \) to 2 (since \( 1! = 1 \) is not considered as \( X \geq 2 \)) and `current` to 2 (which is \( 2! \)).

2. **Loop**: The loop continues to multiply `current` by the next integer (starting from 3) and increments \( n \) until `current` is no longer less than \( X \). This effectively computes the factorial of increasing integers.

3. **Termination**: When `current` equals \( X \), the loop exits, and \( n \) is printed, which is the desired integer whose factorial is \( X \).

This approach efficiently narrows down the solution by leveraging the rapid growth of factorials, ensuring minimal computation steps even for the upper constraint. The solution is both optimal and straightforward, adhering to the problem constraints and guarantees.
