To solve this problem, we need to assign distinct positive integers to each tower such that no two towers have the same height, and each tower's height does not exceed its given maximum height. The goal is to maximize the total sum of these heights. If it's impossible to assign such heights, we should return -1.

### Approach
1. **Sort the Array in Descending Order**: By sorting the array in descending order, we can prioritize assigning the largest possible distinct numbers first. This ensures that we maximize the sum while adhering to the constraints.
2. **Greedy Assignment**: Start with the largest maximum height. Assign the largest possible distinct number (which is the maximum height itself for the first tower). For subsequent towers, assign the largest available number that is less than the previously assigned number and does not exceed the current tower's maximum height.
3. **Check for Feasibility**: During the assignment process, if at any point the next number to assign would be non-positive, it means we cannot assign a valid distinct positive integer to the current tower, making the entire assignment impossible. In such cases, we return -1.

### Solution Code
```python
class Solution:
    def maximumTotalSum(self, maximumHeight: List[int]) -> int:
        maximumHeight.sort(reverse=True)
        n = len(maximumHeight)
        if n == 0:
            return 0
        total = maximumHeight[0]
        current = maximumHeight[0] - 1
        for i in range(1, n):
            next_num = min(current, maximumHeight[i])
            if next_num <= 0:
                return -1
            total += next_num
            current = next_num - 1
        return total
```

### Explanation
1. **Sorting**: The array is sorted in descending order to facilitate the greedy assignment of the largest possible distinct numbers first.
2. **Initialization**: The first tower is assigned its maximum height, which is the largest number in the sorted array. The variable `current` is initialized to `maximumHeight[0] - 1` to start the next assignment.
3. **Loop Through Remaining Heights**: For each subsequent tower, the next number to assign is the minimum of `current` (to ensure distinctness and descending order) and the tower's maximum height. If this number is non-positive, the assignment is impossible, and we return -1. Otherwise, we add this number to the total sum and update `current` for the next iteration.
4. **Result**: If all towers are processed successfully, the total sum of the assigned heights is returned.

This approach efficiently checks for the feasibility of the assignment while maximizing the sum by leveraging sorting and a greedy strategy, ensuring optimal performance even for large input sizes.
