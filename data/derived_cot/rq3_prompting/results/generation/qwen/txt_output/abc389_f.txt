To solve this problem, we need to determine Takahashi's final rating after participating in N contests, given that his rating increases by 1 if it falls within a specific range [L_i, R_i] for each contest. The challenge is to efficiently handle multiple queries where each query provides a different initial rating X.

### Approach
1. **Problem Analysis**: For each query, we start with an initial rating X and simulate the process of participating in all N contests. For each contest, if the current rating is within the specified range [L_i, R_i], the rating increases by 1; otherwise, it remains unchanged. The final rating is the initial rating plus the number of contests where the condition was met.

2. **Intuition**: The straightforward approach is to simulate each contest for every query. However, given the constraints (N up to 200,000 and Q up to 300,000), a naive simulation would be too slow. Despite this, the solution involves reading the contests and queries, then for each query, iterating through all contests to compute the final rating by checking the condition for each contest.

3. **Algorithm Selection**: The chosen algorithm is a direct simulation for each query. For each query, we process each contest in sequence, updating the rating if it falls within the contest's range. The final rating is then output for each query.

4. **Complexity Analysis**: The time complexity is O(N*Q) in the worst case, where N is the number of contests and Q is the number of queries. Given the constraints, this approach may not be efficient for large inputs, but it is straightforward and correct for the given problem constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    contests = []
    index = 1
    for _ in range(n):
        L = int(data[index])
        R = int(data[index + 1])
        index += 2
        contests.append((L, R))
    
    q = int(data[index])
    index += 1
    queries = []
    for _ in range(q):
        X = int(data[index])
        index += 1
        queries.append(X)
    
    results = []
    for X in queries:
        rating = X
        for L, R in contests:
            if L <= rating <= R:
                rating += 1
        results.append(str(rating))
    
    sys.stdout.write("\n".join(results) + "\n")

if __name__ == '__main__':
    main()
```

1. **Reading Input**: The input is read in one go for efficiency, especially given the large constraints. Then, the number of queries (Q) is read, followed by Q integers representing the initial ratings for each query.

2. **Processing Queries**: For each query, the initial rating X is processed through all contests. For each contest, if the current rating is within the contest's range, it is incremented by 1. The final rating after processing all contests is stored.

3. **Output**: The results for all queries are collected in a list and printed at once for efficiency, minimizing I/O operations.

This approach ensures that each query is processed in linear time relative to the number of contests, making it straightforward and easy to understand, albeit potentially inefficient for very large inputs.
