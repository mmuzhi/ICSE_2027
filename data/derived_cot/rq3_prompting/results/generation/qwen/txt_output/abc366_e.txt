To solve this problem, we need to count the number of integer pairs (x, y) such that the sum of Manhattan distances from (x, y) to all given points is less than or equal to a given integer D. The Manhattan distance between two points (a, b) and (c, d) is |a - c| + |b - d|. 

### Approach
1. **Problem Analysis**: The problem requires us to find all integer pairs (x, y) such that the sum of Manhattan distances from (x, y) to each given point is ≤ D. The Manhattan distance sum can be broken down into separate sums for x and y coordinates. Specifically, the total distance is the sum of absolute differences for x-coordinates plus the sum for y-coordinates. 

2. **Key Insight**: The sum of absolute differences for a single coordinate (say x) is minimized at the median of the x-coordinates of the given points. The function F(x) = sum(|x - x_i|) is piecewise linear and decreases until the median and then increases. Similarly for y. 

3. **Algorithm Selection**:
   - **Preprocessing**: For both x and y coordinates, sort the lists and compute prefix sums to efficiently calculate the sum of absolute differences for any given x or y.
   - **Range Determination**: Determine the range [L, R] for x and y such that F(x) ≤ D and F(y) ≤ D. This involves scanning from a safe lower bound (min coordinate - D) to a safe upper bound (max coordinate + D) until the condition is met.
   - **Frequency Arrays**: For each coordinate (x and y), compute a frequency array where each entry at index t counts how many integers in the determined range have a sum of absolute differences equal to t.
   - **Cumulative Sum**: For the y-coordinate, compute a cumulative sum array to quickly determine how many y values satisfy F(y) ≤ k for any k.
   - **Counting Valid Pairs**: For each possible value t1 of F(x), count the number of pairs (x, y) such that F(x) + F(y) ≤ D by leveraging the cumulative sum array for y.

4. **Complexity Analysis**: The algorithm processes each coordinate independently. For each coordinate, it scans a range of size up to 2,000,000 (due to constraints), and for each point, it updates counts in linear time. The overall complexity is O(N + (range size)), which is efficient given the constraints.

import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
        
    it = iter(data)
    n = int(next(it))
    D_val = int(next(it))
    xs = []
    ys = []
    for _ in range(n):
        x = int(next(it))
        y = int(next(it))
        xs.append(x)
        ys.append(y)
        
    def solve_for_axis(arr, D_val):
        if not arr:
            return None
        arr_sorted = sorted(arr)
        n_arr = len(arr_sorted)
        total_sum = sum(arr_sorted)
        low_bound = arr_sorted[0] - D_val
        high_bound = arr_sorted[-1] + D_val
        
        freq_arr = [0] * (D_val + 1)
        i = 0
        left_count = 0
        left_sum = 0
        
        for s_val in range(low_bound, high_bound + 1):
            while i < n_arr and arr_sorted[i] <= s_val:
                left_count += 1
                left_sum += arr_sorted[i]
                i += 1
                
            t_val = s_val * left_count - left_sum + (total_sum - left_sum) - s_val * (n_arr - left_count)
            if t_val <= D_val and t_val >= 0:
                if t_val <= D_val:
                    freq_arr[t_val] += 1
                    
        return freq_arr

    freq_x = solve_for_axis(xs, D_val) if xs else None
    freq_y = solve_for_axis(ys, D_val) if ys else None

    if freq_x is None or freq_y is None or D_val < 0:
        print(0)
        return
        
    H_arr = [0] * (D_val + 1)
    H_arr[0] = freq_y[0]
    for t_val in range(1, D_val + 1):
        H_arr[t_val] = H_arr[t_val - 1] + freq_y[t_val]
        
    ans = 0
    for t1 in range(0, D_val + 1):
        if t1 > D_val:
            break
        t2 = D_val - t1
        if t2 < 0:
            break
        if t2 > D_val:
            continue
        if t1 <= D_val and t2 <= D_val:
            ans += freq_x[t1] * H_arr[t2]
            
    print(ans)

if __name__ == '__main__':

3. **Frequency Arrays**: For each coordinate, a frequency array is built where each index represents a possible sum of absolute differences, and the value at each index counts how many integers in the valid range produce that sum.
4. **Cumulative Sum**: For the y-coordinate, a cumulative sum array (H_arr) is constructed to quickly determine how many y values satisfy F(y) ≤ k for any k.
5. **Counting Valid Pairs**: The solution iterates over all possible values of F(x) (t1) and uses the cumulative sum array to count valid y values (F(y) ≤ D_val - t1), summing the product of counts for each valid pair (t1, t2) where t1 + t2 ≤ D_val.
