To solve this problem, we need to determine whether three given integers can be divided into two or more groups such that the sum of the integers in each group is equal. The solution involves checking two main conditions: one for dividing the integers into three equal groups and another for dividing them into two groups with equal sums.

### Approach
1. **Three Equal Groups**: The simplest case is when all three integers are equal. In this scenario, each integer can form its own group, resulting in three groups each with the same sum. This condition is straightforward: check if all three integers are identical.
2. **Two Groups with Equal Sum**: If the integers cannot be divided into three equal groups, we then check if they can be divided into two groups with equal sums. This requires that one of the integers is equal to the sum of the other two. The total sum of all three integers must be even for this to be possible, but this is implicitly ensured if any of the three conditions (A == B+C, B == A+C, or C == A+B) holds true.

### Solution Code
```python
def main():
    A, B, C = map(int, input().split())
    if A == B == C:
        print("Yes")
    elif A == B + C or B == A + C or C == A + B:
        print("Yes")
    else:
        print("No")

if __name__ == '__main__':
    main()
```

### Explanation
1. **Three Equal Groups Check**: The code first checks if all three integers are the same. If they are, it prints "Yes" because each integer can form its own group, and all groups will have the same sum.
2. **Two Groups Check**: If the integers are not all equal, the code checks if any one integer is equal to the sum of the other two. If any of these conditions is true, it prints "Yes" because the integers can be divided into two groups (one containing a single integer and the other containing the sum of the remaining two) with equal sums.
3. **Output "No"**: If neither of the above conditions is met, the code prints "No", indicating that it's not possible to divide the integers into two or more groups with equal sums.

This approach efficiently covers all possible scenarios for dividing the integers into the required groups, ensuring correctness with minimal computational overhead.
