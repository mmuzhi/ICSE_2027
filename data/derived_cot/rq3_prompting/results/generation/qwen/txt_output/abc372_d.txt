To solve this problem, we need to determine for each building in a sequence the number of subsequent buildings that do not have any taller building between them. The solution involves efficiently identifying these buildings using a combination of a stack-based approach to find previous greater elements and a difference array to count valid buildings for each position.

### Approach
1. **Problem Analysis**: For each building at position `i`, we need to count the number of buildings `j` (where `j > i`) such that there are no buildings taller than the `j`-th building between `i` and `j`. This translates to checking if the maximum building height in the segment `[i+1, j-1]` is less than or equal to the height of the `j`-th building.

2. **Intuition**: The key insight is that for each building `j`, the valid starting positions `i` are those where the previous greater element (if it exists) is not in the segment `[i+1, j-1]`. This can be determined using the previous greater element indices (L array) computed with a stack.

3. **Algorithm Selection**:
   - **Stack-based Previous Greater Element**: We use a stack to efficiently compute for each building the index of the nearest previous building that is taller than it. This helps in determining the range of valid starting positions `i` for each `j`.
   - **Difference Array**: To efficiently count the valid buildings for each `i`, we use a difference array. For each `j`, we mark the range `[L[j], j-1]` (or `[0, j-1]` if no previous greater element exists) by incrementing the start of the range and decrementing the position right after the end of the range in the difference array.

4. **Complexity Analysis**: The stack-based approach runs in O(N) time since each building is pushed and popped from the stack at most once. The difference array operations also run in O(N) time, making the overall complexity O(N), which is optimal for the given constraints (N ≤ 2×10^5).

import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
