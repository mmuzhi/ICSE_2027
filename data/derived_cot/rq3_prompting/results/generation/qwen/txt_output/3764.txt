To solve this problem, we need to find the maximum sum of at most `k` elements from a 2D matrix `grid`, where the number of elements taken from each row `i` does not exceed `limits[i]`. The solution involves a greedy approach where we always select the largest available elements while respecting the row constraints.

### Approach
1. **Problem Analysis**: The problem requires selecting elements from a matrix such that the total number of elements selected does not exceed `k` and the number of elements taken from each row `i` is at most `limits[i]`. The goal is to maximize the sum of these elements.
2. **Intuition**: Since all elements are non-negative, the optimal strategy is to select the largest elements first. This suggests a greedy approach where we use a max-heap to always pick the next largest available element, ensuring we do not violate any row's limit.
3. **Algorithm Selection**: 
   - **Sorting Rows**: Each row is sorted in descending order to easily access the largest elements.
   - **Max-Heap**: A max-heap (simulated using a min-heap with negative values) is used to keep track of the largest available elements across all rows.
   - **Row Pointers**: For each row, a pointer tracks the next element to consider. After selecting an element from a row, the pointer is incremented, and the next element (if within row limits) is added to the heap.
4. **Complexity Analysis**: The algorithm processes each row and each element up to `k` times. Sorting each row takes O(m log m) time, where `m` is the number of columns. The heap operations (insertions and deletions) take O(log n) time per operation, where `n` is the number of rows. The overall complexity is O(n * m log m + k log n), which is efficient given the constraints.

### Solution Code
```python
import heapq
from typing import List

class Solution:
    def maxSum(self, grid: List[List[int]], limits: List[int], k: int) -> int:
        if k == 0:
            return 0
        
        n = len(grid)
        rows = []
        for i in range(n):
            rows.append(sorted(grid[i], reverse=True))
        
        pointers = [0] * n
        heap = []
        ans = 0
        
        for i in range(n):
            if limits[i] > 0 and rows[i]:
                heapq.heappush(heap, (-rows[i][0], i))
        
        for _ in range(k):
            if not heap:
                break
            neg_val, i = heapq.heappop(heap)
            val = -neg_val
            ans += val
            pointers[i] += 1
            if pointers[i] < len(rows[i]) and pointers[i] + 1 <= limits[i]:
                next_val = rows[i][pointers[i]]
                heapq.heappush(heap, (-next_val, i))
        
        return ans
```

### Explanation
1. **Initialization**: The solution starts by checking if `k` is zero, in which case the answer is zero. Each row is sorted in descending order to facilitate easy access to the largest elements.
2. **Heap Setup**: A max-heap is initialized by pushing the largest element from each row (if the row's limit allows at least one element to be taken).
3. **Greedy Selection**: The algorithm then performs `k` iterations, each time selecting the largest available element from the heap. After selecting an element, the corresponding row's pointer is incremented, and the next largest element (if within row limits) is added to the heap.
4. **Termination**: The loop terminates if the heap is empty before `k` elements are selected, returning the accumulated sum of the selected elements.

This approach efficiently maximizes the sum by always selecting the largest available elements while respecting the constraints on the number of elements per row. The use of a heap ensures optimal selection, and sorting rows allows quick access to the largest elements.
