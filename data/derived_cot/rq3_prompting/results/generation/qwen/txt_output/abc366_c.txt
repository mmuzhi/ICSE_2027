=== ANSWER ===

To solve this problem, we need to efficiently handle a series of queries on a bag of balls, where each query either inserts a ball, removes a ball, or requests the current count of distinct integers in the bag. The challenge lies in maintaining an accurate count of distinct integers after each insertion and removal operation, especially since multiple balls can have the same integer.

### Approach
1. **Problem Analysis**: The problem involves processing three types of queries: inserting a ball with a specific integer, removing a ball with a specific integer, and querying the number of distinct integers in the bag. The key observation is that the count of distinct integers changes only when the first or last occurrence of an integer is added or removed, respectively.
2. **Intuition**: We can use a dictionary to keep track of the frequency of each integer in the bag. The distinct count is the number of integers with a non-zero frequency. For each insertion, if the integer's frequency transitions from 0 to 1, we increment the distinct count. For each removal, if the integer's frequency transitions from 1 to 0, we decrement the distinct count.
3. **Algorithm Selection**: A dictionary (or defaultdict for simplicity) is suitable here because it allows O(1) average time complexity for insertions and removals. The distinct count is maintained dynamically during each operation.
4. **Complexity Analysis**: The time complexity for each query is O(1), making the overall time complexity O(Q), where Q is the number of queries. The space complexity is O(N), where N is the number of distinct integers inserted, which is efficient given the constraints.

### Solution Code
```python
import sys
from collections import defaultdict

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    Q = int(data[0].strip())
    freq = defaultdict(int)
    distinct_count = 0
    index = 1
    output_lines = []
    for _ in range(Q):
        parts = data[index].split()
        index += 1
        if not parts:
            continue
        if parts[0] == '1':
            x = int(parts[1])
            freq[x] += 1
            if freq[x] == 1:
                distinct_count += 1
        elif parts[0] == '2':
            x = int(parts[1])
            freq[x] -= 1
            if freq[x] == 0:
                distinct_count -= 1
        elif parts[0] == '3':
            output_lines.append(str(distinct_count))
    print("\n".join(output_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first line specifies the number of queries (Q).
2. **Dictionary Initialization**: A defaultdict is used to keep track of the frequency of each integer in the bag. This allows for O(1) access and updates.
3. **Processing Queries**: For each query:
   - **Insertion (1 x)**: The integer x is added to the dictionary. If its frequency becomes 1, the distinct count is incremented.
   - **Removal (2 x)**: The integer x is removed from the dictionary. If its frequency drops to 0, the distinct count is decremented.
   - **Query (3)**: The current distinct count is recorded for output.
4. **Output**: After processing all queries, the results of all type 3 queries are printed in order.

This approach efficiently handles each query in constant time, leveraging the dictionary to dynamically track the distinct count, ensuring optimal performance even for large input sizes.
