To solve this problem, we need to determine the minimum number of operations required to make an array non-decreasing by performing operations where each operation divides an element by its greatest proper divisor. If it's not possible to achieve a non-decreasing array, we return -1.

### Approach
1. **Precompute Smallest Prime Factors (SPF):** We first compute the smallest prime factor for every number up to \(10^6\) using the Sieve of Eratosthenes. This helps in efficiently determining the greatest proper divisor for any composite number.
2. **Build Chains for Each Element:** For each element in the array, we build a chain of possible values it can take after any number of operations. For a number 1, the chain is just [1]. For a prime number, the chain is just the number itself. For composite numbers, the chain includes the original number and its smallest prime factor.
3. **Dynamic Programming (DP) Setup:** We use dynamic programming to keep track of the minimum operations required to make the array non-decreasing up to each element. For each element, we consider all possible values from its chain and check if they can follow the previous element's value in the non-decreasing sequence.
4. **DP Transition:** For each element, we compute the minimum operations by considering all valid previous values (those less than or equal to the current value) and adding the current operation count if applicable.
5. **Result Extraction:** After processing all elements, the answer is the minimum value in the DP array for the last element. If no valid sequence is found, we return -1.

### Solution Code
```python
import math

class Solution:
    def minOperations(self, nums: List[int]) -> int:
        max_val = 10**6
        spf = list(range(max_val + 1))
        for i in range(2, int(math.isqrt(max_val)) + 1):
            if spf[i] == i:
                for j in range(i * i, max_val + 1, i):
                    if spf[j] == j:
                        spf[j] = i
        
        chains = []
        for x in nums:
            if x == 1:
                chains.append([(1, 0)])
            elif spf[x] == x:
                chains.append([(x, 0)])
            else:
                chains.append([(x, 0), (spf[x], 1)])
        
        n = len(nums)
        if n == 0:
            return 0
        
        dp_prev = [0] if len(chains[0]) == 1 else [0, 1]
        
        for i in range(1, n):
            dp_current = [10**18] * len(chains[i])
            current_chain = chains[i]
            prev_chain = chains[i-1]
            prev_dp = dp_prev
            
            for j in range(len(current_chain)):
                value, cost = current_chain[j]
                min_prev = 10**18
                for k in range(len(prev_chain)):
                    prev_val, _ = prev_chain[k]
                    if prev_val <= value:
                        if prev_dp[k] < min_prev:
                            min_prev = prev_dp[k]
                if min_prev == 10**18:
                    continue
                dp_current[j] = cost + min_prev
            
            if all(val == 10**18 for val in dp_current):
                return -1
            
            dp_prev = dp_current
        
        return min(dp_prev) if min(dp_prev) < 10**18 else -1
```

### Explanation
1. **SPF Computation:** The Sieve of Eratosthenes is used to precompute the smallest prime factor for every number up to \(10^6\). This allows efficient determination of the greatest proper divisor for any composite number.
2. **Chain Construction:** For each number in the input array, we construct a chain of possible values it can take. If the number is 1 or prime, the chain is straightforward. For composite numbers, the chain includes the original number and its smallest prime factor.
3. **Dynamic Programming:** We use a DP array where each entry represents the minimum operations required to reach a certain value in the chain of the current element. For each element, we check all possible previous values that are less than or equal to the current value and update the DP array accordingly.
4. **Result Calculation:** After processing all elements, the minimum value in the DP array for the last element gives the answer. If no valid sequence is found, we return -1, indicating it's impossible to make the array non-decreasing with the given operations. 

This approach efficiently handles the constraints by leveraging prime factorization and dynamic programming to minimize the number of operations required.
