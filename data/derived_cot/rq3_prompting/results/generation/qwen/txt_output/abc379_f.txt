=== ANSWER ===

To solve this problem, we need to determine the number of buildings to the east of a given building that can be seen from both specified buildings in a sequence. The key insight is that a building is visible from another if there are no taller buildings between them. 

### Approach
1. **Problem Analysis**: The problem requires counting buildings to the east of a given building that are visible from both specified buildings. Visibility is determined by the absence of any taller buildings between the two buildings. 
2. **Key Insight**: For a building to be visible from a starting building, all buildings between them must not exceed its height. This can be rephrased using the concept of the previous greater element (PGE). Specifically, a building `j` is visible from building `i` if the PGE of `j` (the nearest building to the left with a greater height) is to the left of `i`.
3. **Algorithm Selection**: 
   - **Precompute PGE**: For each building, compute the index of the previous greater element using a stack. This helps in quickly determining if a building is visible from another.
   - **Offline Query Processing**: Sort the queries based on the left building's index. For each query, count the number of buildings to the east that meet the visibility condition by leveraging a Fenwick Tree (Binary Indexed Tree) for efficient range queries.
4. **Complexity Analysis**: The algorithm involves:
   - **Stack Processing**: O(N) time to compute the PGE for all buildings.
   - **Query Processing**: O(Q log N) time for sorting and processing queries with the Fenwick Tree.
   - **Overall Complexity**: O((N + Q) log N), which is efficient for the given constraints.

import sys

class Fenw:
    def __init__(self, n):
        self.n = n
        self.fw = [0] * (n + 1)

    def update(self, i, delta):
        idx = i + 1
        while idx <= self.n:
            self.fw[idx] += delta
            idx += idx & -idx

    def query_prefix(self, i):
        idx = i + 1
        s = 0
        while idx:
            s += self.fw[idx]
            idx -= idx & -idx
        return s

    def range_query(self, l, r):
        if l > r:
            return 0
        res = self.query_prefix(r)
        if l > 0:
            res -= self.query_prefix(l - 1)
        return res

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it))
    q = int(next(it))
    H = [int(next(it)) for _ in range(n)]
    
    L_arr = [-1] * n
    stack = []
    for i in range(n):
        while stack and H[stack[-1]] <= H[i]:
            stack.pop()
        if stack:
            L_arr[i] = stack[-1]
        else:
            L_arr[i] = -1
        stack.append(i)
    
    events = []
    for j in range(n):
        events.append((L_arr[j], j))
    
    events.sort(key=lambda x: x[0])
    
    queries = []
    for i in range(q):
        l_val = int(next(it))
        r_val = int(next(it))
        l0 = l_val - 1
        r0 = r_val - 1
        queries.append((l0, r0, i))
    
    queries.sort(key=lambda x: x[0])
    
    fenw = Fenw(n)
    pointer = 0
    ans = [0] * q
    
    for l0, r0, idx in queries:
        while pointer < len(events) and events[pointer][0] < l0:
            j_index = events[pointer][1]
            fenw.update(j_index, 1)
            pointer += 1
        
        count = fenw.range_query(r0 + 1, n - 1)
        ans[idx] = count
    
    for a in ans:
        print(a)

3. **Event Processing**: Events are created as tuples of (PGE index, building index) and sorted by the PGE index. This allows us to process queries in a specific order.
4. **Query Processing**: Queries are sorted by the left building's index. For each query, we update the Fenwick Tree with buildings that meet the visibility condition and then count the valid buildings to the east using range queries.
5. **Output**: The results for each query are stored and printed in the original order.

This approach efficiently handles the problem by leveraging stack-based PGE computation and Fenwick Tree range queries, ensuring optimal performance for large input sizes.
