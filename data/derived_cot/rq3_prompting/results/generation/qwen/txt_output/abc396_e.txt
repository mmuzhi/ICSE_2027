You are given integers N, M and three integer sequences of length M: X = (X_1, X_2, \ldots, X_M), Y = (Y_1, Y_2, \ldots, Y_M), and Z = (Z_1, Z_2, \ldots, Z_M). It is guaranteed that all elements of X and Y are between 1 and N, inclusive.
We call a length-N sequence of non-negative integers A = (A_1, A_2, \ldots, A_N) a good sequence if and only if it satisfies the following condition:

- For every integer i with 1 \le i \le M, the XOR of A_{X_i} and A_{Y_i} is Z_i.

Determine whether a good sequence A=(A_1,A_2,\ldots,A_N) exists, and if it exists, find one good sequence that minimizes the sum of its elements \displaystyle \sum_{i=1}^N A_i.

Notes on XOR

For non-negative integers A and B, their XOR A \oplus B is defined as follows:


- In the binary representation of A \oplus B, the digit in the place corresponding to 2^k \,(k \ge 0) is 1 if and only if exactly one of the digits in the same place of A and B is 1; otherwise, it is 0.


For example, 3 \oplus 5 = 6 (in binary: 011 \oplus 101 = 110).

X_1 Y_1 Z_1

Output

If a good sequence exists, print one good sequence that minimizes the sum of its elements, separated by spaces.
If there are multiple good sequences with the same minimum sum, printing any of them is accepted.

Constraints


- 1 \le N \le 2\times 10^5
- 0 \le M \le 10^5
- 1 \le X_i, Y_i \le N
- 0 \le Z_i \le 10^9

To solve this problem, we need to determine if there exists a sequence of non-negative integers such that for each given constraint, the XOR of two elements in the sequence equals a specified value. If such a sequence exists, we must find one that minimizes the sum of its elements.

### Approach
1. **Graph Representation**: Treat each node as an element in the sequence and each constraint as an edge between two nodes with a weight (the XOR result). This forms an undirected graph where edges may have duplicate or self-loops.
2. **Cycle Consistency Check**: For the sequence to exist, the graph must satisfy certain cycle conditions. Specifically, for any cycle in the graph, the XOR of all edge weights along the cycle must be zero. This is checked during a BFS traversal.
3. **Component Processing**: For each connected component in the graph, compute the minimal base value that, when XORed with node-specific values, results in the minimal sum for the component. This involves:
   - **BFS Traversal**: Assign values to nodes based on the base node (root) of the component.
   - **Bitwise Optimization**: For each bit position (0 to 31), determine if setting the bit in the base value minimizes the total contribution of that bit across all nodes in the component.
4. **Result Construction**: Combine the results from all components to form the final sequence, ensuring the sum of all elements is minimized.

4. **Result Calculation**: The final sequence is derived by XORing the base value with each node's assigned value in the component. The sequence is printed if all constraints are satisfied; otherwise, -1 is printed if any constraint is inconsistent.
