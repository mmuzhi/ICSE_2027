To solve this problem, we need to count the number of ways Takahashi can move exactly K times in a directed graph with N vertices and N+M edges. The graph consists of a cycle (base edges) and M additional edges (extra edges). The solution involves dynamic programming to efficiently compute the number of paths of length K starting from vertex 1, considering both base and extra edges.

### Approach
1. **Problem Analysis**: The graph is structured with a cycle (base edges) and additional edges (extra edges). Each vertex has outgoing edges via base edges (moving to the next vertex in the cycle) and possibly extra edges (direct jumps to other vertices). The challenge is to count all possible paths of length K starting from vertex 1, considering both types of edges.

2. **Dynamic Programming (DP) Setup**: We use a DP array where `dp[v]` represents the number of ways to reach vertex `v` at the current step. The initial state is `dp[1] = 1` (starting at vertex 1).

3. **Base and Extra Edges Handling**: For each step from 1 to K:
   - **Base Edges**: The base contribution to each vertex `v` is the number of ways to reach the previous vertex (v-1) in the cycle. This is handled by cyclically shifting the DP array.
   - **Extra Edges**: For each vertex `u`, if there is an extra edge from `u` to `v`, we add the number of ways to reach `u` in the previous step to `dp[v]`.

4. **Efficiency**: The algorithm processes each step in O(N + M) time, where N is the number of vertices and M is the number of extra edges. Given the constraints (N and K up to 200,000), this approach is feasible with careful implementation.

### Solution Code
```python
import sys

mod = 998244353

def main():
    data = sys.stdin.read().split()
    it = iter(data)
    N = int(next(it))
    M = int(next(it))
    K = int(next(it))
    
    extra_edges = []
    for _ in range(M):
        u = int(next(it))
        v = int(next(it))
        extra_edges.append((u, v))
    
    extra_sources = [[] for _ in range(N+1)]
    extra_edges_from = [[] for _ in range(N+1)]
    for u, v in extra_edges:
        extra_sources[v].append(u)
        extra_edges_from[u].append(v)
    
    dp = [0] * (N+1)
    dp[1] = 1
    
    for _ in range(K):
        new_dp = [0] * (N+1)
        for v in range(1, N+1):
            if v == 1:
                new_dp[v] = dp[N]
            else:
                new_dp[v] = dp[v-1]
        for u in range(1, N+1):
            for v in extra_edges_from[u]:
                new_dp[v] = (new_dp[v] + dp[u]) % mod
        dp = new_dp
    
    ans = sum(dp[1:]) % mod
    print(ans)

if __name__ == '__main__':
    main()
```

1. **Initialization**: The input is read and parsed. The extra edges are stored in two adjacency lists: `extra_sources` (for vertices reachable via extra edges) and `extra_edges_from` (for vertices that can jump to a given vertex via extra edges).

2. **Dynamic Programming Array**: The `dp` array is initialized with `dp[1] = 1` since Takahashi starts at vertex 1.

3. **Processing Each Step**: For each step from 1 to K:
   - **Base Contribution**: The base part of the DP is computed by cyclically shifting the previous DP values. For vertex `v`, the base contribution comes from vertex `v-1` (with vertex 1 taking the value from vertex N).
   - **Extra Contribution**: For each vertex `u`, the extra edges from `u` contribute to the DP values of their target vertices `v` by adding the previous DP value of `u`.

4. **Result Calculation**: After processing all K steps, the result is the sum of all values in the DP array (vertices 1 to N) modulo 998244353.

This approach efficiently handles the large constraints by leveraging dynamic programming and careful management of base and extra edges, ensuring optimal performance.
