You are given a string S of length N. You are also given Q queries, which you should process in order.
The i-th query is as follows:

- Given an integer X_i and a character C_i, replace the X_i-th character of S with C_i. Then, print the number of times the string ABC appears as a substring in S.

Here, a substring of S is a string obtained by deleting zero or more characters from the beginning and zero or more characters from the end of S.
For example, ab is a substring of abc, but ac is not a substring of abc.

Input

The input is given from Standard Input in the following format:
N Q

X_2 C_2
\vdots
X_Q C_Q

Output

The i-th line (1 \le i \le Q) should contain the answer to the i-th query.

Constraints


- 3 \le N \le 2 \times 10^5
- 1 \le Q \le 2 \times 10^5
- S is a string of length N consisting of uppercase English letters.
- 1 \le X_i \le N
- C_i is an uppercase English letter.

To solve this problem, we need to efficiently handle multiple queries on a string, where each query involves updating a character and then counting the occurrences of the substring "ABC" in the modified string. The challenge lies in performing these operations efficiently given the constraints, which require an optimal solution to handle large input sizes.

### Approach
1. **Initial Setup**: Read the input values, including the string length, number of queries, and the initial string.
2. **Precompute Initial Counts**: Before processing any queries, precompute an array where each entry indicates whether the substring "ABC" starts at that position in the string. The total count of "ABC" substrings is the sum of this array.
3. **Process Queries**: For each query:
   - **Check for No Change**: If the character to be updated is the same as the current character, skip the update and output the current count.
   - **Update the String**: Modify the specified character in the string.
   - **Recalculate Affected Substrings**: The update can affect up to three substrings (the one starting at the updated position, the one with the updated character in the middle, and the one with the updated character at the end). Recalculate the presence of "ABC" in these three positions and adjust the total count accordingly.
4. **Output the Result**: After each update, output the current count of "ABC" substrings.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    first_line = data[0].split()
    N = int(first_line[0])
    Q = int(first_line[1])
    S = list(data[1].strip())
    
    if N < 3:
        arr = []
        total_count = 0
    else:
        arr = [0] * (N - 2)
        total_count = 0
        for i in range(N - 2):
            if S[i] == 'A' and S[i+1] == 'B' and S[i+2] == 'C':
                arr[i] = 1
                total_count += 1
    
    output_lines = []
    index_line = 1
    for _ in range(Q):
        parts = data[index_line].split()
        index_line += 1
        if not parts:
            continue
        X = int(parts[0])
        C = parts[1]
        idx = X - 1
        
        if S[idx] == C:
            output_lines.append(str(total_count))
            continue
            
        S[idx] = C
        positions_to_recalc = set()
        
        if idx <= N - 3:
            positions_to_recalc.add(idx)
        if idx - 1 >= 0 and idx + 1 < N:
            positions_to_recalc.add(idx - 1)
        if idx - 2 >= 0:
            positions_to_recalc.add(idx - 2)
        
        for pos in positions_to_recalc:
            if pos >= len(arr):
                continue
            old_val = arr[pos]
            new_val = 1 if S[pos] == 'A' and S[pos+1] == 'B' and S[pos+2] == 'C' else 0
            total_count += (new_val - old_val)
            arr[pos] = new_val
            
        output_lines.append(str(total_count))
        
    sys.stdout.write("\n".join(output_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first line contains the string length and number of queries, followed by the initial string and the queries themselves.
2. **Initial Count Calculation**: The array `arr` is initialized to keep track of "ABC" substrings. Each entry in `arr` corresponds to a potential start position of "ABC" in the string. The total count is the sum of all 1s in `arr`.
3. **Query Processing**: For each query, the specified character is updated if it differs from the new character. The positions affected by this update (up to three) are identified, and the presence of "ABC" in these positions is recalculated. The total count is adjusted based on these recalculations.
4. **Output**: After processing each query, the current count of "ABC" substrings is recorded. After all queries are processed, the results are outputted.

This approach efficiently handles each query by focusing only on the affected parts of the string, ensuring optimal performance even for large inputs.
