What is XOR?

The XOR of two non-negative integers A and B, denoted as A \oplus B, is defined as follows:


- In the binary representation of A \oplus B, the bit at position 2^k \, (k \geq 0) is 1 if and only if exactly one of the bits at position 2^k in the binary representations of A and B is 1. Otherwise, it is 0.


For example, 3 \oplus 5 = 6 (in binary: 011 \oplus 101 = 110).

In general, the bitwise XOR of k integers p_1, \dots, p_k is defined as (\cdots ((p_1 \oplus p_2) \oplus p_3) \oplus \cdots \oplus p_k).  It can be proved that this is independent of the order of p_1, \dots, p_k.

To solve this problem, we need to determine if there exists a way to assign integers between 1 and \(2^{60} - 1\) to each vertex of a given undirected graph such that for every vertex with at least one neighbor, the XOR of the numbers on its adjacent vertices is zero. 

### Approach
1. **Problem Analysis**: The key observation here is that the condition for each vertex (the XOR of its adjacent vertices must be zero) can be translated into a system of linear equations over GF(2). Each bit in the number assigned to a vertex can be considered independently. The solution involves checking if the graph's adjacency matrix has a non-trivial kernel (i.e., there exists a non-zero solution vector) and ensuring that for each vertex, there exists a solution vector that assigns a 1 to that vertex.

2. **Insight**: The problem can be reduced to checking if the graph has all vertices with even degrees. If any vertex has an odd degree, the system of equations will have only the trivial solution (all zeros), which is invalid since the numbers must be between 1 and \(2^{60} - 1\). If all vertices have even degrees, the all-ones vector is a valid solution, and we can assign the same number (e.g., 1) to all vertices.

3. **Algorithm Selection**: The solution involves:
   - Reading the graph's adjacency list.
   - Checking the degree of each vertex.
   - If any vertex has an odd degree, output "No".
   - Otherwise, output "Yes" followed by the number 1 for each vertex.

4. **Complexity Analysis**: The algorithm efficiently checks the degrees of all vertices in \(O(N + M)\) time, where \(N\) is the number of vertices and \(M\) is the number of edges. This is optimal for the given constraints (N ≤ 60).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        print("Yes")
        print("1")
        return
        
    it = iter(data)
    n = int(next(it))
```
