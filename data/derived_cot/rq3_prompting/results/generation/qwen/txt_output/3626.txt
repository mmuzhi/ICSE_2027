If a zero is found, the number is immediately returned. Otherwise, the product of the digits is computed, and if it is divisible by `t`, the number is returned.
4. **Optimization**: The algorithm avoids unnecessary product calculations for numbers containing zero digits, thus optimizing the solution by reducing computations for such cases.
5. **Complexity Analysis**: Given the constraints (n ≤ 100), the algorithm efficiently checks numbers in linear time, making it suitable for the problem's constraints.

### Solution Code
```python
class Solution:
    def smallestNumber(self, n: int, t: int) -> int:
        num = n
        while True:
            s = str(num)
            if '0' in s:
                return num
            product = 1
            for char in s:
                product *= int(char)
            if product % t == 0:
                return num
            num += 1
```

### Explanation
1. **Initialization**: Start with the given number `n`.
2. **Loop**: For each number in sequence:
   - Convert the number to a string to check for any zero digits.
   - If a zero digit is found, return the number immediately since the product of its digits will be zero, which is divisible by any `t`.
   - If no zero is found, compute the product of all digits.
   - Check if this product is divisible by `t`. If yes, return the number.
3. **Termination**: The loop continues until the smallest valid number is found, ensuring the solution meets the problem's requirements efficiently.

This approach efficiently checks each candidate number in ascending order, leveraging early termination when possible (either by finding a zero digit or meeting the product divisibility condition), ensuring optimal performance for the given constraints.
