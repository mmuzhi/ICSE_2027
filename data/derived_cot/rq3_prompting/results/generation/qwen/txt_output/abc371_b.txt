=== ANSWER ===

To solve this problem, we need to determine for each baby born in a sequence whether the name given is "Taro" based on specific conditions. The name "Taro" is given only to the eldest son (the earliest born male) in each family. 

### Approach
1. **Problem Analysis**: The problem requires processing each baby in chronological order and checking if the baby is the first male in its family. If it is, the name "Taro" is assigned; otherwise, it is not. The solution involves tracking the first occurrence of a male in each family.
2. **Intuition**: For each family, the first male encountered in the sequence of births will be named "Taro". Subsequent males in the same family do not receive this name. Females do not affect the naming process.
3. **Algorithm Selection**: We use a boolean array to keep track of whether a family has already had its first male. For each baby, if it is male and the family hasn't had a male before, we mark the family as having a male and output "Yes". Otherwise, we output "No".
4. **Complexity Analysis**: The algorithm processes each baby exactly once, making it efficient with a time complexity of O(M), where M is the number of babies. The space complexity is O(N) due to the boolean array used to track families.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    n, m = map(int, data[0].split())
    first_male = [False] * (n + 1)
    
    output_lines = []
    for i in range(1, m + 1):
        parts = data[i].split()
        a = int(parts[0])
        b = parts[1]
        if b == 'M':
            if not first_male[a]:
                output_lines.append("Yes")
                first_male[a] = True
            else:
                output_lines.append("No")
        else:
            output_lines.append("No")
    
    for line in output_lines:
        print(line)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first line contains the number of families (N) and babies (M).
2. **Tracking First Male**: A boolean array `first_male` is initialized to keep track of whether a family has had its first male baby. The array is indexed from 1 to N (1-based).
3. **Processing Each Baby**: For each baby, if it is male (`B_i == 'M'`), the code checks if the family (A_i) has already had a male. If not, it outputs "Yes" and marks the family as having a male. If the baby is female or the family already has a male, it outputs "No".
4. **Output**: The results for all babies are collected in a list and printed at the end for efficiency, though printing as each baby is processed would also work.

This approach efficiently checks the conditions for each baby in linear time, ensuring optimal performance even for the upper constraint limits.
