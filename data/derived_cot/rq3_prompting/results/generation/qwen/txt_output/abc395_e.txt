To solve this problem, we need to find the minimum cost to travel from vertex 1 to vertex N in a directed graph, where we can either move along an edge (cost 1) or reverse all edges (cost X). The reversal operation changes the direction of all edges, and subsequent moves are made in the reversed graph. 

### Approach
1. **Graph Representation**: We model the problem using a two-layer graph where each vertex appears in two layers: one for the original graph (even number of reversals) and one for the reversed graph (odd number of reversals). This allows us to track the state of the graph (original or reversed) during traversal.
2. **Edge Construction**: 
   - **Original Layer (Layer 0)**: For each edge (u, v) in the original graph, add an edge from u to v in Layer 0 with cost 1.
   - **Reversed Layer (Layer 1)**: For each edge (u, v) in the original graph, add an edge from v to u in Layer 1 with cost 1.
   - **Reversal Edges**: For each vertex, add edges between its corresponding nodes in Layer 0 and Layer 1 with cost X. This represents the cost of reversing all edges.
3. **Dijkstra's Algorithm**: We use Dijkstra's algorithm to find the shortest path from the starting vertex (vertex 1 in Layer 0) to the target vertex (vertex N in either layer). The algorithm efficiently explores the graph, considering both moving along edges and reversing the graph.

### Solution Code
```python
import sys
import heapq

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    it = iter(data)
    N = int(next(it)); M = int(next(it)); X = int(next(it))
    edges = []
    for _ in range(M):
        u = int(next(it)); v = int(next(it))
        edges.append((u-1, v-1))
    
    total_nodes = 2 * N
    graph = [[] for _ in range(total_nodes)]
    
    for u, v in edges:
        graph[u].append((v, 1))
        graph[v + N].append((u + N, 1))
    
    for i in range(N):
        graph[i].append((i + N, X))
        graph[i + N].append((i, X))
    
    dist = [10**18] * total_nodes
    dist[0] = 0
    heap = [(0, 0)]
    
    while heap:
        d, node = heapq.heappop(heap)
        if d != dist[node]:
            continue
        for neighbor, cost in graph[node]:
            new_cost = d + cost
            if new_cost < dist[neighbor]:
                dist[neighbor] = new_cost
                heapq.heappush(heap, (new_cost, neighbor))
    
    answer = min(dist[N-1], dist[2*N-1])
    print(answer)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Graph Construction**: The graph is built with two layers (total_nodes = 2*N). Each original edge (u, v) is represented in Layer 0 (original graph) and Layer 1 (reversed graph). Reversal edges allow switching between layers with cost X.
2. **Dijkstra's Algorithm**: The algorithm starts from vertex 0 (vertex 1 in Layer 0) and explores all possible moves (edges in the current graph) and reversals (edges between layers). The priority queue ensures the shortest path is found by always expanding the least costly node first.
3. **Result Extraction**: The minimum cost to reach vertex N is the smaller of the costs to reach it in Layer 0 or Layer 1 (nodes N-1 and 2*N-1, respectively).

This approach efficiently handles the problem by leveraging graph theory and Dijkstra's algorithm to explore all possible paths, including those involving edge reversals, ensuring the minimum cost is found.
