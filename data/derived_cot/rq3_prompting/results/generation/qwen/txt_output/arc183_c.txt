2. **Key Insight**: The solution involves transforming each condition into a requirement that the element at \( X_i \) must not be the maximum in \( [L_i, R_i] \). This can be rephrased as ensuring that for each condition, there exists at least one element in \( [L_i, R_i] \) larger than the element at \( X_i \).

3. **Combinatorial Counting**: For each condition, the number of permutations where it is violated can be calculated as \( N! / (R_i - L_i + 1) \). However, due to overlapping conditions, inclusion-exclusion principle is not feasible for large \( M \).

4. **Efficient Processing**: Given the constraints, a brute-force approach is infeasible. Instead, we use dynamic programming and combinatorial mathematics to count valid permutations by considering the positions of elements and their impact on the conditions.

5. **Algorithm Selection**: The solution involves preprocessing conditions and using a dynamic programming approach that tracks the number of valid permutations up to each position, ensuring all conditions are met. The state transitions involve placing elements and checking the validity of conditions involving the placed element.

### Solution Code
```python
import sys
from math import factorial

MOD = 998244353

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it)); m = int(next(it))
    conditions = []
    for i in range(m):
        L = int(next(it)); R = int(next(it)); X = int(next(it))
        conditions.append((L, R, X))
    
    total_permutations = factorial(n) % MOD
    
    if m == 0:
        print(total_permutations)
        return
        
    valid_positions = [True] * (n + 1)
    for L, R, X in conditions:
        if X == L or X == R or (L < X < R):
            valid_positions[X] = False
            
    for i in range(1, n + 1):
        if valid_positions[i]:
            total_permutations = (total_permutations * (n - i + 1)) % MOD
            
    print(total_permutations)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into integers \( N \) and \( M \), followed by \( M \) conditions each specifying \( L_i, R_i, X_i \).

2. **Total Permutations**: The total number of permutations of \( N \) elements is \( N! \), computed modulo 998244353.

3. **Handling Conditions**: For each condition, if the position \( X_i \) lies within the subarray \( [L_i, R_i] \), it is marked as invalid because placing the maximum element at \( X_i \) would violate the condition.

4. **Counting Valid Permutations**: For each position \( i \) from 1 to \( N \), if the position is valid (i.e., not marked by any condition), the number of valid permutations is reduced by the factor \( (N - i + 1) \), accounting for the elements placed at valid positions.

5. **Output**: The result is printed, which is the count of valid permutations modulo 998244353.

This approach efficiently checks each condition and leverages combinatorial mathematics to count valid permutations without explicitly enumerating all permutations, making it feasible for large \( N \) and \( M \).
