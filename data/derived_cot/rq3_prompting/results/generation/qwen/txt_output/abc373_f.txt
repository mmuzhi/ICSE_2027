=== ANSWER ===

To solve this problem, we need to maximize the total happiness Takahashi can achieve by selecting items from different types, given a weight capacity constraint. Each item type has a weight and a value, and the happiness for selecting \( k_i \) items of type \( i \) is given by \( k_i \times v_i - k_i^2 \). The challenge is to determine the optimal number of items from each type to maximize the total happiness without exceeding the weight capacity.

### Approach
1. **Problem Analysis**: The problem is a variant of the bounded knapsack problem where the value function for each item type is concave (quadratic). The happiness function \( k_i \times v_i - k_i^2 \) decreases with each additional item from the same type, making it a diminishing returns scenario.
2. **Intuition**: The optimal solution involves selecting items in a way that maximizes the happiness per unit weight. Given the concave nature of the happiness function, the greedy approach of always selecting the item with the highest marginal happiness first is effective.
3. **Algorithm Selection**: We use a priority queue (min-heap) to always select the item with the highest marginal happiness that fits within the remaining weight capacity. For each type, we generate items (each representing an additional item from that type) with their respective happiness and weight, updating the heap dynamically as items are selected.
4. **Complexity Analysis**: The algorithm processes each item in the heap once, and the number of items is bounded by the sum of the maximum items per type (up to \( 10^{10} \)), but practically limited by the weight capacity \( W \). The heap operations (insertion and extraction) are logarithmic in the number of items, making the overall complexity manageable for the given constraints.

### Solution Code
```python
import heapq
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it))
    W_val = int(next(it))
    types = []
    for _ in range(n):
        w_i = int(next(it))
        v_i = int(next(it))
        types.append((w_i, v_i))
    
    next_index = [0] * n
    heap = []
    for i in range(n):
        w_i, v_i = types[i]
        if v_i - 1 > 0:
            heapq.heappush(heap, (-(v_i - 1), w_i, i))
    
    total_weight = 0
    total_happiness = 0
    
    while heap and total_weight < W_val:
        neg_val, weight_i, type_i = heapq.heappop(heap)
        val = -neg_val
        if total_weight + weight_i <= W_val:
            total_weight += weight_i
            total_happiness += val
            next_index[type_i] += 1
            next_val = types[type_i][1] - (2 * next_index[type_i] + 1)
            if next_val > 0:
                heapq.heappush(heap, (-next_val, weight_i, type_i))
    
    print(total_happiness)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of item types \( N \) and the maximum weight capacity \( W \). Each item type's weight and value are stored in a list.
2. **Priority Queue Initialization**: A min-heap is initialized to always process the item with the highest marginal happiness first. Each item in the heap is represented by its negative happiness value (to simulate a max-heap), its weight, and its type index.
3. **Processing Items**: The loop continues as long as there are items in the heap and the total weight does not exceed \( W \). For each item, if it fits within the remaining capacity, its happiness is added to the total, and the next item from the same type (if it has positive happiness) is added to the heap.
4. **Output**: The accumulated total happiness is printed after processing all possible items.

This approach efficiently maximizes happiness by leveraging a greedy strategy and a priority queue to handle the dynamic selection of items based on their diminishing returns.
