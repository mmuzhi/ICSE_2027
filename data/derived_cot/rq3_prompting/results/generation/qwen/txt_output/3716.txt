To solve this problem, we need to find the length of the longest subsequence in an array such that the absolute differences between consecutive elements form a non-increasing sequence. The solution involves dynamic programming where we keep track of the longest subsequence ending with each value and each possible last absolute difference.

### Approach
1. **Problem Analysis**: The task is to find a subsequence (not necessarily contiguous) where the sequence of absolute differences between consecutive elements is non-increasing. The challenge is to efficiently compute this using dynamic programming without resorting to a brute-force approach, which would be computationally expensive.

2. **Intuition**: For each element in the array, we consider all previous elements and check if appending the current element to a subsequence ending with a previous element maintains the non-increasing condition on the absolute differences. The dynamic programming state `dp[v][d]` represents the length of the longest subsequence ending with value `v` and last absolute difference `d`.

3. **Algorithm Selection**: We use a dynamic programming approach where we maintain two 2D arrays:
   - `dp[v][d]`: Stores the maximum length of a subsequence ending with value `v` and last absolute difference `d`.
   - `suffix_max[v][d]`: For each value `v`, stores the maximum `dp[v][d]` for all `d' >= d`, allowing efficient lookup of the best previous subsequence length.

4. **Optimization**: By precomputing `suffix_max`, we avoid nested loops over all possible differences, reducing the complexity from O(n^2 * 300) to O(n * 300), which is feasible given the constraints.

5. **Complexity Analysis**: The algorithm processes each element in the array once (O(n)) and for each element, it processes up to 300 possible values (O(300)), resulting in an overall complexity of O(n * 300), which is efficient for the given constraints (n ≤ 10^4).

### Solution Code
```python
class Solution:
    def longestSubsequence(self, nums: List[int]) -> int:
        MAX_V = 300
        MAX_D = 300
        
        dp = [[0] * (MAX_D + 1) for _ in range(MAX_V + 1)]
        suffix_max = [[0] * (MAX_D + 1) for _ in range(MAX_V + 1)]
        
        best_ans = 0
        
        for x in nums:
            new_dp = [0] * (MAX_D + 1)
            new_dp[MAX_D] = 1
            
            for v in range(1, MAX_V + 1):
                d_new = abs(x - v)
                if d_new > MAX_D:
                    continue
                if suffix_max[v][d_new] > 0:
                    if suffix_max[v][d_new] + 1 > new_dp[d_new]:
                        new_dp[d_new] = suffix_max[v][d_new] + 1
            
            for d in range(MAX_D, -1, -1):
                if d == MAX_D:
                    dp[x][d] = new_dp[d]
                    suffix_max[x][d] = new_dp[d]
                else:
                    dp[x][d] = new_dp[d]
                    suffix_max[x][d] = max(new_dp[d], suffix_max[x][d + 1])
            
            current_best = max(new_dp)
            if current_best > best_ans:
                best_ans = current_best
        
        return best_ans
```

### Explanation
1. **Initialization**: We initialize two 2D arrays, `dp` and `suffix_max`, to store the lengths of the longest subsequences and their suffix maxima for all possible values and differences.

2. **Processing Each Element**: For each element `x` in the input array:
   - **Base Case**: The subsequence containing only `x` has length 1, stored at `dp[x][300]` (since 300 is a large number representing no prior difference).
   - **Extending Subsequences**: For each previous value `v`, if the absolute difference `|x - v|` is within bounds, we check `suffix_max[v][d_new]` to find the longest valid subsequence that can be extended by `x`.

3. **Updating DP and Suffix Max**: After processing all previous values for `x`, we update `dp[x]` and `suffix_max[x]` to reflect the new subsequences ending with `x`.

4. **Tracking Best Answer**: The maximum length found during each iteration is tracked and returned as the result.

This approach efficiently leverages dynamic programming and suffix maxima to solve the problem within feasible time complexity.
