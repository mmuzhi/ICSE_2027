To solve this problem, we need to determine the minimum number of queries required to transform the given array into a zero array by processing the queries in sequence. Each query allows us to decrement any subset of elements within a specified range by a given value. The solution involves checking for each query whether the combination of previous queries can exactly reduce each element to zero.

### Approach
1. **Initialization**: First, we check if the array is already a zero array. If so, we return 0 immediately as no queries are needed.
2. **Dynamic Programming Setup**: For each element in the array, we initialize a dynamic programming (DP) array where `dp[i][s]` indicates whether it's possible to achieve a total decrement of `s` for the element using the queries processed so far.
3. **Query Processing**: For each query, we update the DP arrays of all elements within its range. The update involves checking if including the current query's value can help achieve the target decrement for each element.
4. **Early Termination**: After processing each query, we check if all elements can be reduced to zero. If they can, we return the current number of queries used. If no such number is found after processing all queries, we return -1.

### Solution Code
```python
class Solution:
    def minZeroArray(self, nums: List[int], queries: List[List[int]]) -> int:
        n = len(nums)
        m = len(queries)
        
        if all(x == 0 for x in nums):
            return 0
        
        dp = []
        for i in range(n):
            size = nums[i] + 1
            arr = [False] * size
            arr[0] = True
            dp.append(arr)
        
        for q_idx, (l, r, val) in enumerate(queries):
            for i in range(l, r + 1):
                if i >= n:
                    continue
                current_nums = nums[i]
                for s in range(current_nums, val - 1, -1):
                    if s - val >= 0 and dp[i][s - val]:
                        dp[i][s] = True
            all_done = True
            for i in range(n):
                if not dp[i][nums[i]]:
                    all_done = False
                    break
            if all_done:
                return q_idx + 1
        
        return -1
```

### Explanation
1. **Initialization Check**: The code first checks if the array is already all zeros. If true, it returns 0 since no queries are needed.
2. **DP Array Setup**: For each element in the array, a DP array is initialized to keep track of achievable decrements. The initial state is `dp[i][0] = True` for all elements, indicating that zero decrements are always possible without any queries.
3. **Query Processing**: For each query, the code updates the DP arrays for all elements within the query's range. The update is done in reverse order to avoid overwriting values prematurely. For each element, the code checks if including the current query's value can help achieve the target decrement.
4. **Early Termination**: After processing each query, the code checks if all elements can be reduced to zero. If they can, it returns the current query count. If no solution is found after all queries, it returns -1.

This approach efficiently checks for the minimal number of queries by leveraging dynamic programming to track achievable decrements, ensuring optimal performance even for the upper constraint limits.
