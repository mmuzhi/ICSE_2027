To solve this problem, we need to compute the sum of scores of all non-empty subsequences for each prefix of a given sequence, where the score of a subsequence is defined as the sum of the greatest common divisors (gcd) of consecutive elements in the subsequence. Given the constraints, a brute-force approach is infeasible, so we leverage mathematical insights and dynamic programming to efficiently compute the required sums.

### Approach
1. **Problem Analysis**: The score of a subsequence is the sum of gcd values between consecutive elements. For each prefix of the sequence, we need to consider all non-empty subsequences and sum their scores. The challenge is to compute this efficiently without enumerating all subsequences explicitly.

2. **Key Insight**: The sum of scores for all subsequences can be broken down by considering the contribution of each pair of elements (i, j) (i < j) in the sequence. Specifically, for each pair (i, j), the gcd(A_i, A_j) contributes to the score of any subsequence that includes both elements and excludes all elements between them. The number of such subsequences is \(2^{i-1} \times 2^{m-j}\), where m is the current prefix length.

3. **Mathematical Optimization**: Using the property that \(\gcd(a, b) = \sum_{d|\gcd(a,b)} \phi(d)\), where \(\phi\) is Euler's totient function, we can decompose the gcd into a sum over divisors. This allows us to precompute the contributions of each divisor and use dynamic programming to accumulate these contributions efficiently.

4. **Algorithm Selection**: 
   - **Precomputation**: Compute divisors and Euler's totient function for all numbers up to the maximum value in the sequence (100,000).
   - **Dynamic Programming**: For each element in the sequence, update a running sum that tracks the contributions of its divisors. This sum is used to compute the total score for each prefix by leveraging modular arithmetic to handle large numbers.

5. **Complexity Analysis**: The precomputation steps (divisors and totient function) are \(O(M \log M)\) where \(M\) is the maximum value (100,000). Processing each element involves iterating over its divisors, which is \(O(N \times D)\) where \(D\) is the average number of divisors (around 140 for numbers up to 100,000), making the overall complexity feasible for \(N\) up to 500,000.

### Solution Code
```python
import sys

mod = 998244353
max_val = 100000

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    A = list(map(int, data[1:1+n]))

    phi = list(range(max_val+1))
    for i in range(2, max_val+1):
        if phi[i] == i:
            for j in range(i, max_val+1, i):
                phi[j] -= phi[j] // i

    divisors = [[] for _ in range(max_val+1)]
    for i in range(1, max_val+1):
        for j in range(i, max_val+1, i):
            divisors[j].append(i)

    pow2 = [1] * (n+1)
    for i in range(1, n+1):
        pow2[i] = (pow2[i-1] * 2) % mod

    inv2 = pow(2, mod-2, mod)
    inv_pow2 = [1] * (n+1)
    for i in range(1, n+1):
        inv_pow2[i] = (inv_pow2[i-1] * inv2) % mod

    sums_arr = [0] * (max_val+1)
    cur = 0
    ans_arr = [0] * (n+1)

    for m in range(1, n+1):
        num = A[m-1]
        if m >= 2:
            G_val = 0
            for d in divisors[num]:
                if d > max_val:
                    continue
                G_val = (G_val + phi[d] * sums_arr[d]) % mod
            cur = (cur + G_val * inv_pow2[m]) % mod

        for d in divisors[num]:
            if d > max_val:
                continue
            sums_arr[d] = (sums_arr[d] + pow2[m-1]) % mod

        ans_arr[m] = pow2[m] * cur % mod

    for i in range(1, n+1):
        print(ans_arr[i])

if __name__ == '__main__':
    main()
```

### Explanation
1. **Precomputation**: The code starts by precomputing Euler's totient function values and the list of divisors for all numbers up to 100,000. This allows efficient decomposition of gcd values later.
2. **Dynamic Programming Setup**: It initializes arrays to store powers of 2 and their modular inverses, which are used to handle large numbers and division under modulo arithmetic.
3. **Processing Each Element**: For each element in the sequence, the code calculates its contribution to the total score by summing the contributions of all its divisors. This involves multiplying the totient value of each divisor by the accumulated sum of relevant subsequences.
4. **Modular Arithmetic**: The results are computed using modular arithmetic to prevent overflow and ensure correctness, especially given the large constraints.
5. **Output**: After processing each element, the code outputs the total score for the current prefix, leveraging the precomputed values to efficiently compute the result for each prefix length from 1 to N.

This approach efficiently breaks down the problem using mathematical insights and dynamic programming, ensuring optimal performance even for large input sizes.
