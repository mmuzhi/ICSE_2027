    def minDamage(self, power: int, damage: List[int], health: List[int]) -> int:
        n = len(damage)
        # Calculate h_i for each enemy

        # Create list of (damage[i], h_i)

        # Sort by ratio damage[i] / h_i in descending order

        # Now, compute the prefix sum of h_i
        prefix = 0
        total_damage = 0
        for dmg, h_val in enemies:
            prefix += h_val
            total_damage += dmg * prefix
        return total_damage

To solve this problem, we need to determine the minimum total damage Bob takes before all enemies are dead, given that Bob can attack one enemy per second with a fixed power. The key insight is to prioritize enemies that deal the most damage per unit of processing time (time required to kill them) to minimize the cumulative damage Bob takes.

### Approach
1. **Problem Analysis**: Bob needs to eliminate all enemies, each dealing damage per second while alive. The goal is to minimize the total damage Bob takes by strategically choosing the order of attacks. The optimal strategy involves prioritizing enemies with the highest damage per unit of processing time (damage divided by the number of attacks needed to kill them, calculated as the ceiling of their health divided by Bob's power).

2. **Intuition**: Enemies that deal more damage per attack should be eliminated first to reduce the total damage over time. The number of attacks needed to kill each enemy is given by the ceiling of their health divided by Bob's power. The total damage is the sum of each enemy's damage multiplied by the time they are alive (which is the cumulative number of attacks taken to kill all enemies up to and including them).

3. **Algorithm Selection**: 
   - **Calculate Processing Time**: For each enemy, compute the number of attacks needed (h_i) as the ceiling of health[i] divided by power.
   - **Sort Enemies**: Sort enemies in descending order of their damage per attack (damage[i] / h_i). This ensures that enemies contributing more damage per attack are eliminated first.
   - **Compute Total Damage**: For each enemy in the sorted order, the time they are alive is the sum of all previous enemies' processing times plus their own. Multiply this by their damage and accumulate the total damage.

4. **Complexity Analysis**: The algorithm involves a sorting step with O(n log n) complexity, where n is the number of enemies. The subsequent linear pass to compute the total damage is O(n). Given the constraints (n ≤ 10^5), this approach is efficient.

### Solution Code
```python
class Solution:
    def minDamage(self, power: int, damage: List[int], health: List[int]) -> int:
        n = len(damage)
        h_list = [(health[i] + power - 1) // power for i in range(n)]
        enemies = [(damage[i], h_list[i]) for i in range(n)]
        enemies.sort(key=lambda x: x[0] / x[1], reverse=True)
        prefix = 0
        total_damage = 0
        for dmg, h_val in enemies:
            prefix += h_val
            total_damage += dmg * prefix
        return total_damage
```

### Explanation
1. **Processing Time Calculation**: For each enemy, the number of attacks needed (h_i) is calculated using ceiling division: `(health[i] + power - 1) // power`. This gives the minimum number of attacks required to reduce the enemy's health to zero.
2. **Sorting Enemies**: Enemies are sorted based on their damage per attack (damage[i] / h_i) in descending order. This ensures that enemies contributing more damage per attack are prioritized for elimination.
3. **Total Damage Calculation**: For each enemy in the sorted list, the time they are alive is the cumulative sum of processing times of all enemies up to and including them. The total damage is the sum of each enemy's damage multiplied by this cumulative time.
This approach efficiently minimizes the total damage by focusing on high-damage enemies first, leveraging sorting and a linear pass for optimal performance.
