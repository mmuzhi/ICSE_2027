   - Using a set to record distinct valid splits to avoid counting duplicates.
4. **Complexity Analysis**: The algorithm has a time complexity of O(n^2) due to nested loops iterating over all possible splits. Given the constraint n ≤ 5000, the total operations are around 12.5 million, which is feasible with efficient hash comparisons.

### Solution Code
```python
class Solution:
    def beautifulSplits(self, nums: List[int]) -> int:
        n = len(nums)
        if n < 3:
            return 0
        
        mod1 = 10**9 + 7
        mod2 = 10**9 + 9
        base = 131
        
        H1 = [0] * n
        H2 = [0] * n
        pow1 = [1] * (n + 1)
        pow2 = [1] * (n + 1)
        
        for i in range(1, n + 1):
            pow1[i] = (pow1[i-1] * base) % mod1
            pow2[i] = (pow2[i-1] * base) % mod2
        
        H1[0] = nums[0] % mod1
        H2[0] = nums[0] % mod2
        for i in range(1, n):
            H1[i] = (H1[i-1] * base + nums[i]) % mod1
            H2[i] = (H2[i-1] * base + nums[i]) % mod2
        
        def get_hash(l, r, mod, H, pow_base):
            if l > r:
                return 0
            if l == 0:
                return H[r]
            else:
                return (H[r] - H[l-1] * pow_base[r - l + 1]) % mod
        
        seen = set()
        for i in range(n - 1):
            for j in range(i + 1, n - 1):
                cond1 = False
                cond2 = False
                if j - i >= i + 1:
                    if j >= i + 1 + i:
                        hash1_val1 = get_hash(0, i, mod1, H1, pow1)
                        hash1_val2 = get_hash(i + 1, i + 1 + i, mod1, H1, pow1)
                        hash2_val1 = get_hash(0, i, mod2, H2, pow2)
                        hash2_val2 = get_hash(i + 1, i + 1 + i, mod2, H2, pow2)
                        if hash1_val1 == hash1_val2 and hash2_val1 == hash2_val2:
                            cond1 = True
                if n - 1 - j >= j - i:
                    hash1_val1 = get_hash(i + 1, j, mod1, H1, pow1)
                    hash1_val2 = get_hash(j + 1, j + (j - i), mod1, H1, pow1)
                    hash2_val1 = get_hash(i + 1, j, mod2, H2, pow2)
                    hash2_val2 = get_hash(j + 1, j + (j - i), mod2, H2, pow2)
                    if hash1_val1 == hash1_val2 and hash2_val1 == hash2_val2:
                        cond2 = True
                if cond1 or cond2:
                    seen.add((i, j))
        return len(seen)
```

### Explanation
1. **Initialization**: The solution starts by initializing two large prime moduli and a base for the rolling hash. It then computes prefix hashes and power arrays for efficient substring comparison.
2. **Hash Calculation**: The `get_hash` function computes the hash of any substring in constant time using precomputed prefix hashes and power values.
3. **Iterate Over Splits**: The nested loops iterate over all possible split points (i, j). For each split, it checks if either the first subarray is a prefix of the second or the second is a prefix of the third using the double hash comparison.
4. **Tracking Valid Splits**: Valid splits are stored in a set to ensure each distinct split is counted only once. The size of this set gives the final answer.

This approach efficiently checks all possible splits using hashing to minimize comparison time, ensuring optimal performance even for the upper constraint limits.
