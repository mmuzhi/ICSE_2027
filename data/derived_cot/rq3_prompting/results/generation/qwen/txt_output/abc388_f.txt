=== ANSWER ===

To solve this problem, we need to determine if we can move from square 1 to square N in a row of N squares by repeatedly jumping from the current square to a square that is within a specified range [A, B] steps ahead, without landing on any bad squares. The bad squares are defined by M non-overlapping intervals.

### Approach
1. **Problem Analysis**: The problem involves navigating from the first square to the last square in a row, avoiding bad squares, using jumps of size between A and B. The key challenge is efficiently checking reachability given the constraints, especially since N can be as large as 10^12, making a brute-force approach infeasible.

2. **Insight**: The solution involves breaking the problem into segments of safe squares (those not in any bad interval) and modeling the reachability between these segments using a graph. Each node in the graph represents a segment of safe squares, and edges exist if a jump from one segment can land in another segment.

3. **Algorithm Selection**: 
   - **Safe Segments Identification**: The safe segments are derived by considering the gaps between bad intervals. The first segment starts at square 1 and ends just before the first bad interval starts. Subsequent segments are the gaps between consecutive bad intervals, and the last segment starts after the last bad interval ends and goes up to square N.
   - **Graph Construction**: For each segment, we check if there exists a jump from any square in the current segment to any square in a subsequent segment. This is done by verifying if the current segment overlaps with the interval [L_j - B, R_j - A] for the next segment, where L_j and R_j are the bounds of the next segment.
   - **BFS for Reachability**: Using Breadth-First Search (BFS), we traverse the graph starting from the first segment to determine if the last segment is reachable.

4. **Complexity Analysis**: The number of safe segments is O(M), where M is the number of bad intervals (up to 20,000). For each segment, we check all subsequent segments, leading to O(M^2) edge checks. BFS on O(M) nodes is efficient, making the overall complexity O(M^2), which is feasible given the constraints.

from collections import deque

        L0, R0 = intervals[0]
        safe_segments.append((1, L0 - 1))
