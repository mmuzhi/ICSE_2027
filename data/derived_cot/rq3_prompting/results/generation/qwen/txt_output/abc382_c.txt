There are N people numbered from 1 to N visiting a conveyor belt sushi restaurant.
The gourmet level of person i is A_i.

A sushi that person i takes and eats will no longer pass in front of person j\ (j > i).
For each of the M pieces of sushi, determine who eats that sushi, or if nobody eats it.

Input

The input is given from Standard Input in the following format:
N M
A_1 A_2 \dots A_N
B_1 B_2 \dots B_M

Output

Constraints


- 1 \leq N, M \leq 2 \times 10^5
- 1 \leq A_i, B_i \leq 2 \times 10^5
- All input values are integers.

To solve this problem, we need to determine which person on a conveyor belt sushi restaurant will eat each piece of sushi based on their gourmet levels. The sushi passes by each person in order, and the first person whose gourmet level is less than or equal to the sushi's deliciousness will eat it. 

### Approach
1. **Problem Analysis**: The problem involves processing M pieces of sushi, each with a specific deliciousness value. For each sushi piece, we need to find the first person (lowest index) whose gourmet level is not greater than the sushi's deliciousness. If no such person exists, the sushi remains uneaten.

2. **Intuition**: The key observation is that for each sushi piece, we need the smallest index person whose gourmet level is ≤ the sushi's deliciousness. To efficiently answer these queries, we preprocess the gourmet levels of all people to create an array where each entry at index `v` holds the smallest index person with a gourmet level exactly `v`. Then, we perform a forward update to this array to ensure each entry at index `v` contains the smallest index person with a gourmet level ≤ `v`.

3. **Algorithm Selection**: 
   - **Preprocessing**: We first populate an array `min_index_arr` where `min_index_arr[v]` is the smallest index of a person with gourmet level `v`. 
   - **Forward Update**: We then create another array `min_index` where `min_index[v]` is the smallest index of a person with gourmet level ≤ `v`. This is done by iterating through possible values and taking the minimum between the current value and the previous value in the array.

4. **Complexity Analysis**: 
   - **Time Complexity**: O(MAX_VAL + M), where MAX_VAL is the maximum possible value for gourmet levels (200,000). The preprocessing step takes O(N) time to populate `min_index_arr` and O(MAX_VAL) time to build `min_index`. Each query is answered in O(1) time.
   - **Space Complexity**: O(MAX_VAL), as we use two arrays of size MAX_VAL + 1.

```python
    n = int(data[0])
    m = int(data[1])
    A = list(map(int, data[2:2+n]))
    B = list(map(int, data[2+n:2+n+m]))
    
    MAX_VAL = 200000
    min_index_arr = [n + 1] * (MAX_VAL + 1)
    
    for i in range(n):
        a_val = A[i]
        if i + 1 < min_index_arr[a_val]:
            min_index_arr[a_val] = i + 1
            
    min_index = [n + 1] * (MAX_VAL + 1)
    min_index[0] = min_index_arr[0]
    for v in range(1, MAX_VAL + 1):
        min_index[v] = min(min_index_arr[v], min_index[v - 1])
        
    out_lines = []
    for b_val in B:
        if b_val > MAX_VAL:
            out_lines.append("-1")
        else:
            if min_index[b_val] == n + 1:
                out_lines.append("-1")
            else:
                out_lines.append(str(min_index[b_val]))
                
    print("\n".join(out_lines))

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are N (number of people) and M (number of sushi pieces). The next N values are the gourmet levels of the people, and the following M values are the deliciousness of the sushi pieces.

2. **Preprocessing**: 
   - **min_index_arr**: This array is initialized to store the smallest index of a person for each possible gourmet level. For each person, we update this array if their index is smaller than the currently stored index for their gourmet level.
   - **min_index**: This array is built by iterating through all possible values up to MAX_VAL. For each value `v`, `min_index[v]` is the minimum of `min_index_arr[v]` and `min_index[v-1]`, ensuring it holds the smallest index of a person with a gourmet level ≤ `v`.

3. **Query Processing**: For each sushi piece, if its deliciousness exceeds MAX_VAL, it's impossible for anyone to eat it (though constraints ensure this doesn't happen). Otherwise, we check `min_index` for the smallest index person whose gourmet level is ≤ the sushi's deliciousness. If no such person exists, we return -1; otherwise, we return the smallest index.

This approach efficiently preprocesses the data to allow each query to be answered in constant time, making it suitable for large input sizes as specified in the problem constraints.
