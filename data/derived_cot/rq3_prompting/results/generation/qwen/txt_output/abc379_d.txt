To solve this problem, we need to efficiently process a series of queries involving planting, waiting, and harvesting plants in a large number of flower pots. The challenge lies in handling these queries in a way that optimizes both time and space, especially given the constraints where the number of queries can be as high as 200,000.

### Approach
1. **Problem Analysis**: The problem involves three types of queries:
   - **Type 1**: Plant a new plant with initial height 0.
   - **Type 2**: Increment the height of all existing plants by a given value T.
   - **Type 3**: Harvest all plants with height at least H and output the count of such plants.

2. **Key Insight**: The height of a plant created at time `t0` is `current_total_time - t0`, where `current_total_time` is the cumulative time passed from all previous Type 2 queries. For a Type 3 query with parameter H, we need to harvest all plants where `current_total_time - t0 >= H`, i.e., `t0 <= current_total_time - H`.

3. **Data Structure**: We use a list to keep track of the creation times of active plants. Since the creation times are strictly increasing (as `current_total_time` only increases), the list remains sorted. For each Type 3 query, we use binary search (via `bisect_right`) to find the position up to which plants should be harvested and then remove those plants from the list.

4. **Complexity Analysis**: 
   - **Insertion (Type 1)**: O(1) per operation since we append to the list.
   - **Binary Search (Type 3)**: O(log n) per query.
   - **Removal (Type 3)**: O(n) per operation due to list deletion, but this is manageable given the constraints.

### Solution Code
```python
import bisect

def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    Q = int(data[0].strip())
    total_time = 0
    active_plants = []
    output_lines = []
    
    index = 1
    for _ in range(Q):
        parts = data[index].split()
        index += 1
        if not parts:
            continue
        if parts[0] == '1':
            active_plants.append(total_time)
        elif parts[0] == '2':
            T = int(parts[1])
            total_time += T
        elif parts[0] == '3':
            H = int(parts[1])
            threshold = total_time - H
            pos = bisect.bisect_right(active_plants, threshold)
            if pos > 0:
                del active_plants[:pos]
                output_lines.append(str(pos))
            else:
                output_lines.append("0")
    
    sys.stdout.write("\n".join(output_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: We start by reading all input queries. The `total_time` variable keeps track of the cumulative time passed from all Type 2 queries. The `active_plants` list stores the creation times of plants that haven't been harvested yet.
2. **Processing Queries**:
   - **Type 1**: The current `total_time` is appended to `active_plants`.
   - **Type 2**: The given value T is added to `total_time`.
   - **Type 3**: The threshold for harvesting is calculated as `total_time - H`. Using binary search (`bisect_right`), we find the position in `active_plants` where plants should be harvested. The first `pos` plants (if any) are removed from the list, and the count is recorded.
3. **Output**: After processing all queries, the results of Type 3 queries are printed.

This approach efficiently handles the queries by leveraging the sorted nature of creation times and binary search, ensuring optimal performance for large input sizes.
