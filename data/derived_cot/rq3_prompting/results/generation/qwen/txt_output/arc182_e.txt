The challenge lies in efficiently handling large values of \(K\) (up to \(10^9\)) and \(M\) (up to \(10^9\)) without resorting to brute-force methods.

### Approach
1. **Problem Analysis**: The problem requires calculating the minimum value of \((C \cdot k + A_i) \mod M\) for each \(k\) in the range [0, K-1] and summing these minima. Direct computation for each \(k\) is infeasible for large \(K\) due to high computational complexity.

2. **Key Insight**: The sequence of values \((C \cdot k) \mod M\) is periodic with a period \(T = M / \gcd(C, M)\). This periodicity allows us to break the problem into cycles of length \(T\), where each cycle's sum can be computed once and then multiplied by the number of full cycles. The remaining terms (if \(K\) is not a multiple of \(T\)) are handled separately.

3. **Efficiency Consideration**: For each cycle, we compute the sum of minima over one period. However, directly iterating over each \(k\) in the cycle is still too slow for large \(T\). Instead, we leverage the fact that the sequence of \((C \cdot k) \mod M\) is an arithmetic progression modulo \(M\), and use binary search to efficiently determine the minimum value for each \(k\) in the cycle.

4. **Algorithm Selection**: 
   - **Sorting**: The sequence \(A\) is sorted to facilitate binary search operations.
   - **Binary Search**: For each \(k\), determine the smallest value in \(A\) such that \((C \cdot k + A_i) \mod M\) is minimized by checking if \(A_i \geq M - (C \cdot k \mod M)\).
   - **Cycle Handling**: Compute the sum over one full cycle of length \(T\) and multiply by the number of full cycles. Sum the remaining terms separately.

### Solution Code
```python
import bisect
import math

def main():
    import sys
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it))
    m = int(next(it))
    c = int(next(it))
    k = int(next(it))
    A = [int(next(it)) for _ in range(n)]
    
    A.sort()
    min_val = A[0]
    total = 0
    g = math.gcd(c, m)
    T = m // g
    full_cycles = k // T
    remainder = k % T
    
    def g_function(B):
        if min_val + B < m:
            pos = bisect.bisect_left(A, m - B)
            if pos == n:
                return min_val + B
            else:
                candidate = A[pos] + B - m
                return min(min_val + B, candidate)
        else:
            return min_val + B - m

    for i in range(T):
        B = (c * i) % m
        total += g_function(B)
    
    cycle_sum = total
    total = full_cycles * cycle_sum
    for i in range(remainder):
        B = (c * i) % m
        total += g_function(B)
            
    print(total)

if __name__ == '__main__':
    main()
```

5. **Sum Calculation**: The sum over one full cycle is computed. This sum is then multiplied by the number of full cycles. The remaining terms are processed individually and added to the total sum.
