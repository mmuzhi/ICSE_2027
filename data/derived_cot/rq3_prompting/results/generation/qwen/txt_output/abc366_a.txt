=== ANSWER ===

To solve this problem, we need to determine if the outcome of an election between two candidates, Takahashi and Aoki, is already decided based on the current vote counts and the total number of votes. The solution involves checking if either candidate has a majority of the total votes, ensuring that no matter how the remaining votes are distributed, the leading candidate will win.

### Approach
1. **Problem Analysis**: The election has a total of N votes, which is an odd number. The current counts for Takahashi and Aoki are T and A, respectively. The remaining votes are N - T - A. The outcome is decided if one candidate has more than half of the total votes (N), meaning even if all remaining votes go to the other candidate, the leading candidate still has a majority.
2. **Key Insight**: A candidate's victory is secured if their current votes exceed half of the total votes (N). This is because the maximum votes the opponent can achieve is N minus the candidate's current votes. If the candidate's votes are greater than this value, they have a majority.
3. **Algorithm Selection**: The solution involves checking if either candidate's votes multiplied by 2 is greater than N. This condition ensures the candidate has a majority (since 2*T > N implies T > N/2, and similarly for Aoki).

### Solution Code
```python
def main():
    n, t, a = map(int, input().split())
    if 2 * t > n or 2 * a > n:
        print("Yes")
    else:
        print("No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Handling**: The code reads three integers: N (total votes), T (Takahashi's votes), and A (Aoki's votes).
2. **Majority Check**: The code checks if either candidate's votes (T or A) is more than half of N. This is done by verifying if 2*T > N or 2*A > N. If either condition is true, the outcome is decided, and "Yes" is printed; otherwise, "No" is printed.
3. **Output**: Based on the majority check, the code outputs whether the election outcome is already decided.

This approach efficiently determines the election outcome by leveraging simple arithmetic checks to ensure correctness and optimal performance. The solution handles all edge cases, including scenarios where votes are evenly split or one candidate leads by a significant margin.
