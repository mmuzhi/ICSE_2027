=== ANSWER ===

To solve this problem, we need to determine whether a given sequence of integers is strictly increasing. A sequence is strictly increasing if each element is less than the next element in the sequence. 

### Approach
1. **Problem Analysis**: The problem requires checking each consecutive pair of elements in the sequence to ensure that each element is strictly less than the next one. If any element in the sequence does not satisfy this condition, the sequence is not strictly increasing.
2. **Intuition**: The solution involves iterating through the sequence once, comparing each element with its successor. If any element is found to be greater than or equal to the next element, the sequence is not strictly increasing, and we can immediately return "No". If all consecutive pairs satisfy the condition, we return "Yes".
3. **Algorithm Selection**: A simple linear scan through the sequence is sufficient. The algorithm starts from the first element and checks each subsequent element in order, ensuring each is less than the next. This approach is efficient with a time complexity of O(N), where N is the length of the sequence, which is optimal for this problem.
4. **Optimization**: Since the constraints are small (N ≤ 100), no further optimization is necessary. The linear scan is straightforward and efficient enough for the given problem constraints.

### Solution Code
```python
def main():
    n = int(input().strip())
    data = list(map(int, input().split()))
    
    for i in range(n - 1):
        if data[i] >= data[i + 1]:
            print("No")
            return
            
    print("Yes")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The first line of input is read as an integer `n`, representing the length of the sequence. The second line is read as a list of integers.
2. **Checking Strictly Increasing Condition**: The code iterates through each element in the sequence from the first to the second last element. For each element, it checks if it is less than the next element. If any element is not less than the next, it prints "No" and exits.
3. **Output Result**: If all consecutive pairs satisfy the condition, the code prints "Yes" after the loop completes.

This approach efficiently checks the strictly increasing condition with minimal computational overhead, making it suitable for the given problem constraints.
