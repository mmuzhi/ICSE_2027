There is an undirected graph with N vertices and 0 edges. The vertices are numbered 1 to N.
You are given Q queries to process in order. Each query is of one of the following two types:

- Type 1: Given in the format 1 u v. Add an edge between vertices u and v.
- Type 2: Given in the format 2 v k. Print the k-th largest vertex number among the vertices connected to vertex v. If there are fewer than k vertices connected to v, print -1.

- In a Type 1 query, 1 \leq u < v \leq N.
- In a Type 2 query, 1 \leq v \leq N, 1 \leq k \leq 10.
- All input values are integers.

=== ANSWER ===

To solve this problem, we need to process a series of queries on an initially empty undirected graph and answer specific queries about the vertices connected to a given vertex. The graph starts with no edges, and we are to handle two types of queries: adding an edge between two vertices and querying the k-th largest vertex number among the neighbors of a given vertex.

### Approach
1. **Initialization**: Each vertex is initialized to include itself in its neighbor list. This ensures that even without any edges, each vertex considers itself as a connected vertex.
2. **Handling Queries**:
   - **Type 1 (Add Edge)**: For each edge addition between vertices `u` and `v`, we update both vertices' neighbor sets and their respective top 10 largest neighbors lists. The neighbor sets ensure distinct neighbors, while the top 10 lists maintain the largest neighbors efficiently.
   - **Type 2 (Query)**: For a query asking for the k-th largest neighbor of vertex `v`, we first check if there are at least `k` distinct neighbors. If not, we return -1. Otherwise, we retrieve the k-th largest neighbor from the pre-maintained top 10 list.
3. **Efficiency**: By maintaining a set of distinct neighbors and a sorted list (in descending order) of the top 10 largest neighbors for each vertex, we ensure that each edge addition and query is handled in constant time (since the list size is capped at 10).

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    it = iter(data)
    n = int(next(it)); q = int(next(it))
    graph_set = [set() for _ in range(n+1)]
    graph_list = [[] for _ in range(n+1)]
    
    for i in range(1, n+1):
        graph_set[i].add(i)
        graph_list[i] = [i]
    
    out_lines = []
    for _ in range(q):
        t = next(it)
        if t == '1':
            u = int(next(it)); v = int(next(it))
            if v not in graph_set[u]:
                graph_set[u].add(v)
                new_list = graph_list[u] + [v]
                new_list.sort(reverse=True)
                graph_list[u] = new_list[:10]
                if u not in graph_set[v]:
                    graph_set[v].add(u)
                    new_list_v = graph_list[v] + [u]
                    new_list_v.sort(reverse=True)
                    graph_list[v] = new_list_v[:10]
        else:
            v_val = int(next(it)); k = int(next(it))
            if k > len(graph_set[v_val]):
                out_lines.append("-1")
            else:
                out_lines.append(str(graph_list[v_val][k-1]))
    
    sys.stdout.write("\n".join(out_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Initialization**: The graph is initialized such that each vertex `i` has itself as a neighbor. This is stored in `graph_set` (a set for distinct neighbors) and `graph_list` (a list of the top 10 largest neighbors in descending order).
2. **Type 1 Query (Add Edge)**: When adding an edge between vertices `u` and `v`, we update both vertices' neighbor sets and their top 10 lists. The neighbor set ensures no duplicates, and the list is updated by inserting the new neighbor and maintaining the top 10 largest values.
3. **Type 2 Query (k-th Largest Neighbor)**: For a query on vertex `v` asking for the k-th largest neighbor, we first check if there are at least `k` neighbors. If so, we directly access the k-th largest from the pre-sorted list; otherwise, we return -1.
This approach efficiently handles the constraints by leveraging sets for distinct neighbors and maintaining a fixed-size list for quick access to the top 10 neighbors, ensuring optimal performance even for large input sizes.
