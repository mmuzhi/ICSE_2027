=== ANSWER ===

To solve this problem, we need to efficiently handle dynamic updates to a graph where edges can be removed, and answer shortest path queries between nodes in the updated graph. Given the constraints, especially the limited number of edge removals (at most 300), we can leverage the Floyd-Warshall algorithm to compute all-pairs shortest paths (APSP) after each removal. This approach ensures that we can answer each query in constant time by simply looking up the precomputed shortest paths.

### Approach
1. **Problem Analysis**: The problem involves processing two types of queries on a graph: removing an edge and querying the shortest path between two nodes. The key observation is that the number of edge removals is limited (at most 300), allowing us to recompute the APSP after each removal efficiently.
2. **Intuition**: The Floyd-Warshall algorithm is chosen because it computes the APSP for a graph in O(N^3) time, which is feasible given that N (number of nodes) is at most 300. After each edge removal, we recompute the APSP for the updated graph.
3. **Algorithm Selection**: The Floyd-Warshall algorithm is used to compute the shortest paths between all pairs of nodes. This algorithm is suitable because it can handle both directed and undirected graphs with positive edge weights and efficiently updates the shortest paths when edges are removed.
4. **Complexity Analysis**: The initial APSP computation takes O(N^3) time. Each edge removal triggers a recomputation of the APSP, which also takes O(N^3) time. With up to 300 removals, the total time complexity is O(300 * N^3), which is manageable for N ≤ 300.
5. **Optimization**: The solution avoids unnecessary recomputations by updating the graph and re-running Floyd-Warshall only when edges are removed. For each query of type 2, the answer is directly retrieved from the last computed APSP matrix.

### Solution Code
```python
import sys

INF = 10**18

def floyd_warshall(graph, N):
    dist = [list(row) for row in graph]
    for k in range(N):
        for i in range(N):
            if dist[i][k] == INF:
                continue
            for j in range(N):
                if dist[k][j] == INF:
                    continue
                if dist[i][j] > dist[i][k] + dist[k][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    return dist

def main():
    data = sys.stdin.read().split()
    it = iter(data)
    N = int(next(it)); M = int(next(it)); Q = int(next(it))
    
    graph = [[INF] * N for _ in range(N)]
    edges = []
    
    for _ in range(M):
        a = int(next(it)); b = int(next(it)); c = int(next(it))
        a -= 1; b -= 1
        graph[a][b] = c
        graph[b][a] = c
        edges.append((a, b, c))
        
    current_dist = floyd_warshall(graph, N)
    
    output_lines = []
    for _ in range(Q):
        t = next(it)
        if t == '1':
            i_val = int(next(it))
            idx = i_val - 1
            if idx < len(edges):
                a, b, c = edges[idx]
                graph[a][b] = INF
                graph[b][a] = INF
                current_dist = floyd_warshall(graph, N)
        else:
            x = int(next(it)); y = int(next(it))
            x -= 1; y -= 1
            if current_dist[x][y] == INF:
                output_lines.append("-1")
            else:
                output_lines.append(str(current_dist[x][y]))
                
    print("\n".join(output_lines))

if __name__ == "__main__":
    main()
```

1. **Initialization**: The graph is initialized with a large value (INF) for non-existent edges. Each edge from the input is added to the graph, and the edges are stored in a list for later removal.
2. **APSP Computation**: The Floyd-Warshall algorithm is used to compute the initial APSP matrix, which stores the shortest paths between all pairs of nodes.
3. **Query Processing**: For each query:
   - **Type 1 (Edge Removal)**: The specified edge is removed by setting its weight to INF in the graph, and the APSP matrix is recomputed.
   - **Type 2 (Shortest Path Query)**: The shortest path between the specified nodes is retrieved from the current APSP matrix. If the path does not exist (distance remains INF), -1 is returned; otherwise, the shortest distance is returned.
4. **Output**: The results of all type 2 queries are collected and printed at the end.

This approach efficiently handles dynamic edge removals and queries by leveraging the Floyd-Warshall algorithm, ensuring optimal performance within the problem constraints.
