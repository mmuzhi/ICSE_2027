- If he lets a monster go, he gains 0 experience points.
- If he defeats a monster with strength X, he gains X experience points.
  If it is an even-numbered defeated monster (2nd, 4th, ...), he gains an additional X experience points.

Find the maximum total experience points he can gain from the N monsters.

Input

The input is given from Standard Input in the following format:
N
A_1 A_2 \ldots A_N

Output

Print the maximum total experience points he can gain from the N monsters as an integer.

Constraints


- 1\leq N\leq 2\times 10^5
- 1\leq A_i\leq 10^9
- All input values are integers.

Sample Input 1

Therefore, in this case, he gains 1+(5+5)+3+0+(7+7)=28 experience points.

To solve this problem, we need to maximize the total experience points Takahashi can gain by defeating monsters in order, where each defeated monster contributes its strength to the experience, and every even-numbered defeated monster (2nd, 4th, etc.) adds an additional strength point. The challenge is to select a subsequence of monsters to defeat (in order) such that the total experience is maximized.

### Approach
1. **Problem Analysis**: The key observation is that the experience gained from defeating a monster depends on its position in the sequence of defeats. The first defeat gives the monster's strength, the second gives twice the strength, the third gives the strength again, and so on. This pattern repeats for each subsequent defeat.
2. **Dynamic Programming (DP) Setup**: We use a DP approach where we maintain two states:
   - `dp0`: Maximum experience when the number of defeats so far is even (next defeat will be odd, contributing once the monster's strength).
   - `dp1`: Maximum experience when the number of defeats so far is odd (next defeat will be even, contributing twice the monster's strength).
3. **State Transitions**: For each monster, we update the states:
   - If we skip the monster, the states remain unchanged.
   - If we defeat the monster, we update the states based on the current parity of defeats:
     - From `dp0` (even defeats), defeating the monster transitions to `dp1` (odd defeats) by adding the monster's strength once.
     - From `dp1` (odd defeats), defeating the monster transitions to `dp0` (even defeats) by adding twice the monster's strength.
4. **Initialization**: Start with `dp0 = 0` (no defeats) and `dp1 = -infinity` (invalid state for odd defeats initially).
5. **Result Extraction**: After processing all monsters, the answer is the maximum value between `dp0` and `dp1`.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().splitlines()
    n = int(data[0].strip())
    A = list(map(int, data[1].split()))
    
    dp0 = 0
    dp1 = -10**18
    
    for a in A:
        new_dp0 = max(dp0, dp1 + 2*a)
        new_dp1 = max(dp1, dp0 + a)
        dp0, dp1 = new_dp0, new_dp1
        
    print(max(dp0, dp1))
```

3. **Processing Each Monster**: For each monster's strength `a`:
   - `new_dp0` is updated to the maximum of skipping the monster (`dp0`) or defeating it from an odd state (`dp1 + 2*a`).
   - `new_dp1` is updated to the maximum of skipping the monster (`dp1`) or defeating it from an even state (`dp0 + a`).
4. **State Update**: The states `dp0` and `dp1` are updated to their new values after processing each monster.
5. **Result**: After processing all monsters, the maximum value between `dp0` and `dp1` gives the highest possible experience points, which is printed as the result.
