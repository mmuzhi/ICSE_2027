That is, find \underset{1\leq i_1\lt i_2\lt \ldots\lt i_K\leq N}{\max} A_{i_1}\oplus A_{i_2}\oplus \ldots \oplus A_{i_K}.

About XOR

For non-negative integers A,B, the XOR A \oplus B is defined as follows:


- In the binary representation of A \oplus B, the bit corresponding to 2^k (k \ge 0) is 1 if and only if exactly one of the bits corresponding to 2^k in A and B is 1, and is 0 otherwise.


For example, 3 \oplus 5 = 6 (in binary notation: 011 \oplus 101 = 110).
In general, the XOR of K integers p_1, \dots, p_k is defined as (\cdots((p_1 \oplus p_2) \oplus p_3) \oplus \cdots \oplus p_k). It can be proved that it does not depend on the order of p_1, \dots, p_k.

A_1 A_2 \ldots A_N

Output

Print the answer.

Constraints


- 1\leq K\leq N\leq 2\times 10^5
- 0\leq A_i<2^{60}
- \dbinom{N}{K}\leq 10^6
- All input values are integers.

To solve this problem, we need to find the maximum XOR value obtained by selecting exactly K distinct elements from a given sequence of non-negative integers. The solution leverages combinatorial generation and bitwise operations to efficiently compute the desired result, especially considering the constraint that the binomial coefficient C(N, K) is at most 1,000,000, which ensures the problem is computationally feasible.

### Approach
1. **Problem Analysis**: The problem requires selecting K distinct elements from a sequence of N integers such that their XOR is maximized. The key observation is that the binomial coefficient C(N, K) is small (≤ 1,000,000), allowing us to generate all possible combinations of K elements without excessive computational overhead.
2. **Intuition**: The maximum XOR value is derived from combinations where higher bits are set as much as possible. By generating all combinations of K elements (or their complements if K is large), we can compute the XOR for each combination and determine the maximum value.
3. **Algorithm Selection**:
   - **Small K**: If K is small (≤ 20), generate all combinations of K elements directly and compute their XOR values.
   - **Large K**: If K is large (close to N), generate all combinations of the complement set (size N-K, which is ≤ 20) and use the property that the XOR of the chosen set can be derived from the XOR of the complement set and the total XOR of all elements.
4. **Optimization**: Using `itertools.combinations` for generating combinations ensures that we efficiently handle the combinatorial generation. The solution leverages bitwise properties to minimize redundant calculations and ensures optimal performance within the problem constraints.

### Solution Code
```python
import itertools
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    
    n = int(data[0])
    k = int(data[1])
    A = list(map(int, data[2:2+n]))
    
    if k == 0:
        print(0)
        return
    if k == 1:
        print(max(A))
        return
        
    if k <= 20:
        size = k
        use_chosen = True
    else:
        size = n - k
        use_chosen = False
        
    if use_chosen:
        max_xor = 0
        for comb in itertools.combinations(range(n), size):
            xor_val = 0
            for index in comb:
                xor_val ^= A[index]
            if xor_val > max_xor:
                max_xor = xor_val
        print(max_xor)
    else:
        T = 0
        for a in A:
            T ^= a
            
        all_xors = set()
        for comb in itertools.combinations(range(n), size):
            xor_val = 0
            for index in comb:
                xor_val ^= A[index]
            all_xors.add(xor_val)
            
        best = 0
        for x in all_xors:
            candidate = T ^ x
            if candidate > best:
                best = candidate
        print(best)

if __name__ == '__main__':
    main()
```
