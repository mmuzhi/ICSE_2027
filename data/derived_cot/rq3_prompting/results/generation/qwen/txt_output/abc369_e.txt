### Approach
1. **Problem Analysis**: The problem involves navigating a graph of islands connected by bridges, where each bridge must be used at least once in the journey from island 1 to island N. The challenge is to compute the minimal path for each query, considering the constraints that the number of bridges per query is small (up to 5).

2. **Insight**: The solution leverages the fact that the set of required bridges is small (K ≤ 5). This allows us to use a state-space search approach where each state is defined by the current island and a bitmask representing which bridges have been used. The bitmask ensures we track the progress in covering all required bridges.

3. **Algorithm Selection**: We use Dijkstra's algorithm on a state-space graph where each node is (current island, bitmask). The priority queue helps explore states in order of increasing path cost. For each state, we consider two types of moves:
   - **Type A**: Move to any island via the shortest path in the entire graph, updating the bitmask if any new bridges are used in this path.

```python
            if cost != dp_state[node][mask]:
                continue
                
            if node == N-1 and mask == goal_mask:
                out_lines.append(str(cost))
                found = True
                break
                
            for v in range(N):
                if node == v:
                    continue
                total_cost = cost + dist[node][v]
                new_mask = mask
                for (u_b, v_b, t_b) in bridges_query:
                    if (dist[node][u_b] + t_b + dist[v_b][v] == total_cost) or (dist[node][v_b] + t_b + dist[u_b][v] == total_cost):
                        new_mask |= (1 << bridges_query.index((u_b, v_b, t_b)))
                
                if new_mask == goal_mask and v == N-1:
                    out_lines.append(str(total_cost))
                    found = True
                    break
                    
                if total_cost < dp_state[v][new_mask]:
                    dp_state[v][new_mask] = total_cost
                    heapq.heappush(pq, (total_cost, v, new_mask))
                    
            for (u_b, v_b, t_b) in bridges_query:
                idx_bridge = bridges_query.index((u_b, v_b, t_b))
                if mask & (1 << idx_bridge):
                    continue
                if node == u_b:
                    new_node = v_b
                    new_mask = mask | (1 << idx_bridge)
                    new_cost = cost + t_b
                    if new_node == N-1 and new_mask == goal_mask:
                        out_lines.append(str(new_cost))
                        found = True
                        break
                    if new_cost < dp_state[new_node][new_mask]:
                        dp_state[new_node][new_mask] = new_cost
                        heapq.heappush(pq, (new_cost, new_node, new_mask))
                elif node == v_b:
                    new_node = u_b
                    new_mask = mask | (1 << idx_bridge)
                    new_cost = cost + t_b
                    if new_node == N-1 and new_mask == goal_mask:
                        out_lines.append(str(new_cost))
                        found = True
                        break
                    if new_cost < dp_state[new_node][new_mask]:
                        dp_state[new_node][new_mask] = new_cost
                        heapq.heappush(pq, (new_cost, new_node, new_mask))
                        
            if found:
                break
                
        if not found:
            ans = 10**18
            for v in range(N):
                if dp_state[v][goal_mask] < ans:
                    ans = dp_state[v][goal_mask]
            if ans == 10**18:
                out_lines.append(str(-1))
            else:
                out_lines.append(str(ans))
                
    print("\n".join(out_lines))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into the number of islands (N), bridges (M), and their details. The bridges are stored with 0-indexed nodes for easier manipulation.
2. **Floyd-Warshall Algorithm**: This algorithm computes the shortest paths between all pairs of islands, which is essential for determining the minimal path costs during the state-space search.
3. **Query Processing**: For each query, the required bridges are identified. The state-space search (Dijkstra's algorithm) is initiated from island 1 (node 0) with an initial bitmask of 0 (no bridges used).
4. **State Exploration**: For each state (current island, bitmask), the algorithm explores:
   - **Type A Moves**: Moving to any island via the shortest path, updating the bitmask if any new bridges are used.
   - **Type B Moves**: Explicitly traversing a bridge not yet used, moving to its other endpoint and updating the bitmask.
5. **Termination**: The search terminates when island N (node N-1) is reached with all required bridges used (bitmask equals (1 << K) - 1). The minimal cost is recorded and printed for each query.

This approach efficiently handles the constraints by leveraging bitmask tracking and Dijkstra's algorithm, ensuring optimal pathfinding while adhering to the problem requirements.
