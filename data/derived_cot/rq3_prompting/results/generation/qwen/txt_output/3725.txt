To solve this problem, we need to compute the sum of the maximum and minimum elements of all subarrays with at most `k` elements in a given integer array `nums`. The challenge is to do this efficiently given the constraints, where the array length can be as large as 80,000.

### Approach
1. **Problem Analysis**: The problem requires summing the maximum and minimum values of all contiguous subarrays of length 1 to `k`. A brute-force approach would involve checking all possible subarrays, which is infeasible for large `n` (up to 80,000) due to its O(n^2) complexity.

2. **Key Insight**: For each element in the array, we can determine how many subarrays it is the minimum or maximum in, considering the length constraint. This involves:
   - **Left and Right Boundaries**: For each element, find the nearest elements to the left and right that are smaller (for minimum) or larger (for maximum) than the current element within a distance `k`.
   - **Count Calculation**: Using these boundaries, calculate the number of valid subarrays where the current element is the minimum or maximum, adjusted for the length constraint.

3. **Algorithm Selection**: 
   - **Monotonic Deques**: Use two deques for each part (minimum and maximum) to efficiently compute the left and right boundaries for each element. This allows us to process the array in linear time.
   - **Virtual Boundaries**: Adjust the boundaries to ensure subarrays do not exceed length `k` by setting virtual boundaries at `i-k` and `i+k+1`.

4. **Complexity Analysis**: The algorithm processes each element in the array with two passes (left to right and right to left) using deques, each pass being O(n). The overall complexity is O(n), which is efficient for large inputs.

class Solution:
    def minMaxSubarraySum(self, nums: List[int], k: int) -> int:
        n = len(nums)
        if n == 0:
            return 0
        
        from collections import deque
        
        # For minimum part
        left_min_bound = [-1] * n
        right_min_bound = [n] * n
        
        dq = deque()
        for i in range(n):
            while dq and nums[dq[-1]] >= nums[i]:
                dq.pop()
            if dq:
                left_min_bound[i] = dq[-1]
            else:
                left_min_bound[i] = -1
            dq.append(i)
        
        dq = deque()
        for i in range(n-1, -1, -1):
            while dq and nums[dq[-1]] <= nums[i]:
                dq.pop()
            if dq:
                right_min_bound[i] = dq[-1]
            else:
                right_min_bound[i] = n
            dq.append(i)
        
        # For maximum part
        left_max_bound = [-1] * n
        right_max_bound = [n] * n
        
        dq = deque()
        for i in range(n):
            while dq and nums[dq[-1]] < nums[i]:
                dq.pop()
            if dq:
                left_max_bound[i] = dq[-1]
            else:
                left_max_bound[i] = -1
            dq.append(i)
        
        dq = deque()
        for i in range(n-1, -1, -1):
            while dq and nums[dq[-1]] <= nums[i]:
                dq.pop()
            if dq:
                right_max_bound[i] = dq[-1]
            else:
                right_max_bound[i] = n
            dq.append(i)
        
        sum_min = 0
        sum_max = 0
        
        for i in range(n):
            # Adjust left boundary for minimum
            if left_min_bound[i] == -1:
                left_eff_min = i - k if i - k >= 0 else -1
            else:
                left_eff_min = max(left_min_bound[i], i - k)
            
            right_eff_min = min(right_min_bound[i], i + k + 1)
            if right_eff_min > n:
                right_eff_min = n
            
            count_min = (i - left_eff_min) * (right_eff_min - i)
            sum_min += nums[i] * count_min
            
            # Adjust left boundary for maximum
            if left_max_bound[i] == -1:
                left_eff_max = i - k if i - k >= 0 else -1
            else:
                left_eff_max = max(left_max_bound[i], i - k)
            
            right_eff_max = min(right_max_bound[i], i + k + 1)
            if right_eff_max > n:
                right_eff_max = n
            
            count_max = (i - left_eff_max) * (right_eff_max - i)
            sum_max += nums[i] * count_max
        
        return sum_min + sum_max

### Explanation
1. **Initialization**: The code initializes two arrays, `left_min_bound` and `right_min_bound`, to store the nearest smaller elements to the left and right for each element (for minimum subarrays). Similarly, `left_max_bound` and `right_max_bound` are initialized for larger elements (for maximum subarrays).

2. **Deque Processing**: Using deques, the code processes each element to find the nearest smaller or larger elements within the array. This is done in two passes: left to right for left boundaries and right to left for right boundaries.

3. **Boundary Adjustment**: For each element, the left and right boundaries are adjusted to ensure subarrays do not exceed length `k`. Virtual boundaries at `i-k` and `i+k+1` are used if no valid elements are found within the distance `k`.

4. **Count Calculation**: For each element, the number of valid subarrays where it is the minimum or maximum is calculated using the adjusted boundaries. The results are summed up to get the final answer.

This approach efficiently computes the required sum by leveraging monotonic deques and boundary adjustments, ensuring optimal performance even for large input sizes.
