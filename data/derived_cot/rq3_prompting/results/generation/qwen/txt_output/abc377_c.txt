=== ANSWER ===

To solve this problem, we need to determine the number of empty squares in an N x N grid where a new piece can be placed without being captured by any existing pieces. The existing pieces are given, and each piece can capture squares according to specific knight-like moves. The solution involves identifying all squares that are either occupied or attacked by any existing piece and then subtracting this count from the total number of squares in the grid.

### Approach
1. **Problem Analysis**: The grid is N x N, and there are M pieces placed on it. Each piece can attack up to 8 squares (knight moves). The goal is to count all empty squares not attacked by any existing piece.
2. **Intuition**: The solution involves two main steps:
   - **Identify Attacked Squares**: For each existing piece, compute all squares it can attack (within grid boundaries) and store them in a set to avoid duplicates.
   - **Count Forbidden Squares**: The forbidden squares are the union of occupied squares and attacked squares. The answer is the total squares minus the count of forbidden squares.
3. **Algorithm Selection**: 
   - Use a set to store occupied squares for quick lookup.
   - For each occupied square, compute all possible attacked squares (using predefined moves) and add them to another set (ensuring no duplicates).
   - The union of occupied and attacked squares gives the forbidden squares. Subtracting the size of this union from N² yields the answer.
4. **Complexity Analysis**: The algorithm processes each of the M pieces and their 8 possible attacks, leading to O(M) operations. Since M can be up to 200,000, this is efficient. The space complexity is O(M) for storing the sets, which is feasible given the constraints.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        print(0)
        return
        
    it = iter(data)
    N = int(next(it))
    M = int(next(it))
    
    occupied_set = set()
    for _ in range(M):
        a = int(next(it))
        b = int(next(it))
        occupied_set.add((a, b))
        
    moves = [(2, 1), (1, 2), (-1, 2), (-2, 1), (-2, -1), (-1, -2), (1, -2), (2, -1)]
    
    T_set = set()
    for (a, b) in occupied_set:
        for dx, dy in moves:
            x = a + dx
            y = b + dy
            if 1 <= x <= N and 1 <= y <= N:
                T_set.add((x, y))
                
    forbidden = occupied_set | T_set
    safe = N * N - len(forbidden)
    print(safe)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints. The first two values are N (grid size) and M (number of pieces).
2. **Occupied Squares**: The positions of existing pieces are stored in a set for quick access.
3. **Attacked Squares Calculation**: For each occupied square, all squares it can attack (using predefined knight moves) are computed. Only squares within the grid boundaries are considered, and duplicates are avoided using a set.
4. **Forbidden Squares**: The union of occupied and attacked squares gives all squares where a new piece cannot be placed (either occupied or attacked).
5. **Result Calculation**: The result is derived by subtracting the count of forbidden squares from the total grid squares (N²), providing the number of safe squares for placing a new piece. 

This approach efficiently narrows down the problem to manageable set operations, leveraging Python's set data structure for optimal performance.
