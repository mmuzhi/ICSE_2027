2. **Handling Special Cases**:

   - If only one part is non-empty (e.g., "**X", "X**", "*X*"), the solution is simply the length of that part.
3. **General Cases**: For patterns with two or three non-empty parts, use efficient string matching algorithms (like KMP) to find occurrences of each part in `s`. Then, determine the minimal substring that includes all required parts in the correct order, leveraging binary search for optimization.
4. **Efficiency**: The KMP algorithm is used for pattern matching to ensure linear time complexity relative to the string length, making the solution efficient for large inputs.

```python
            len_C = len(C)
        if D != "":
            occ['D'] = kmp_search(D, s)
            len_D = len(D)
        
        if A == "" and C == "" and D != "":
            return len_D
        
        elif A == "" and C != "" and D == "":
            return len_C
        
        elif A == "" and C != "" and D != "":
            min_len = float('inf')
            for start_C in occ['C']:
                idx_d = bisect.bisect_left(occ['D'], start_C + len_C)
                if idx_d >= len(occ['D']):
                    continue
                start_D = occ['D'][idx_d]
                length = start_D + len_D - start_C
                if length < min_len:
                    min_len = length
            return min_len if min_len != float('inf') else -1
        
        elif A != "" and C == "" and D == "":
            return len_A
        
        elif A != "" and C == "" and D != "":
            min_len = float('inf')
            for start_A in occ['A']:
                idx_d = bisect.bisect_left(occ['D'], start_A + len_A)
                if idx_d >= len(occ['D']):
                    continue
                start_D = occ['D'][idx_d]
                length = start_D + len_D - start_A
                if length < min_len:
                    min_len = length
            return min_len if min_len != float('inf') else -1
        
        elif A != "" and C != "" and D == "":
            min_len = float('inf')
            for start_A in occ['A']:
                idx_c = bisect.bisect_left(occ['C'], start_A + len_A)
                if idx_c >= len(occ['C']):
                    continue
                start_C = occ['C'][idx_c]
                length = start_C + len_C - start_A
                if length < min_len:
                    min_len = length
            return min_len if min_len != float('inf') else -1
        
        elif A != "" and C != "" and D != "":
            min_len = float('inf')
            for start_C in occ['C']:
                idx_d = bisect.bisect_left(occ['D'], start_C + len_C)
                if idx_d >= len(occ['D']):
                    continue
                start_D = occ['D'][idx_d]
                idx_a = bisect.bisect_right(occ['A'], start_C - len_A) - 1
                if idx_a < 0:
                    continue
                start_A = occ['A'][idx_a]
                length = start_D + len_D - start_A
                if length < min_len:
                    min_len = length
            return min_len if min_len != float('inf') else -1
```

### Explanation
1. **Pattern Decomposition**: The pattern `p` is split into three parts (A, C, D) using the '*' characters. These parts represent the fixed segments that must appear in the matching substring.
2. **Special Cases Handling**: If all parts are empty (pattern "**"), return 0. If only one part is non-empty, the solution is simply the length of that part.
3. **General Cases Handling**: For patterns with two or three non-empty parts, the solution involves:
   - Using the KMP algorithm to find all occurrences of each part in `s`.
   - For each occurrence of the second part (C), find the earliest occurrence of the third part (D) that follows it.
   - For the first part (A), find the latest occurrence that precedes the second part (C) to minimize the substring length.
4. **Efficiency**: The KMP algorithm ensures that pattern matching is done in linear time, and binary search is used to efficiently locate the required occurrences, making the solution scalable for large inputs. The overall approach ensures that the shortest valid substring is found or correctly determines that no such substring exists.
