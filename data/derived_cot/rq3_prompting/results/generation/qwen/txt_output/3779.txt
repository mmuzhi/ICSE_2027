=== ANSWER ===

To solve this problem, we need to maximize the total weight gained by eating all pizzas optimally. The key observation is that on odd-numbered days, we gain the weight of the heaviest pizza in the group of four pizzas eaten, while on even-numbered days, we gain the weight of the second heaviest pizza. 

### Approach
1. **Sorting the Pizzas**: First, we sort the list of pizza weights in descending order to easily access the largest and smallest pizzas.
2. **Odd Day Groups**: For the odd-numbered days, we form groups by selecting the largest remaining pizza and the three smallest remaining pizzas. The largest pizza in each such group contributes to the total weight.
3. **Even Day Groups**: For the even-numbered days, we form groups by selecting the two largest and two smallest remaining pizzas from the leftover pizzas. The second largest pizza in each such group contributes to the total weight.
4. **Combining Results**: The total weight is the sum of the maximum weights from all odd day groups and the second maximum weights from all even day groups.

### Solution Code
```python
from typing import List

class Solution:
    def maxWeight(self, pizzas: List[int]) -> int:
        pizzas.sort(reverse=True)
        n = len(pizzas)
        d = n // 4
        k = (d + 1) // 2
        total = 0
        left = 0
        right = n - 1
        
        for _ in range(k):
            total += pizzas[left]
            left += 1
            right -= 3
        
        m = d // 2
        R = pizzas[left:right+1]
        
        for _ in range(m):
            if not R:
                break
            total += R[1]
            R = R[2:-2]
        
        return total
```

### Explanation
1. **Sorting**: The pizzas are sorted in descending order to prioritize the largest pizzas for odd day groups and the smallest for even day groups.
2. **Odd Day Groups**: The first part of the code processes the odd day groups. For each group, it takes the largest remaining pizza (which will be the maximum weight for that group) and three smallest pizzas. The largest pizza's weight is added to the total.
3. **Even Day Groups**: The remaining pizzas are processed for even day groups. For each group, it takes the two largest and two smallest remaining pizzas. The second largest pizza's weight (which is the second element in the sorted list of remaining pizzas) is added to the total.
4. **Efficiency**: The algorithm efficiently processes the pizzas in linear time after sorting, making it suitable for large input sizes up to 200,000 pizzas. The sorting step dominates the time complexity with O(n log n), which is acceptable given the constraints.
