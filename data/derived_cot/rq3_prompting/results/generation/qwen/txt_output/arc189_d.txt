There are N slimes lined up in a row from left to right.
For i = 1, 2, \ldots, N, the i-th slime from the left has size A_i.

When a slime disappears due to absorption, the gap is immediately closed, and the slimes that were adjacent to the disappearing slime (if they exist) become adjacent (see the explanation in Sample Input 1).

N
A_1 A_2 \ldots A_N

Output

Print the answers B_K for each K = 1, 2, \ldots, N, separated by spaces, in the following format:
B_1 B_2 \ldots B_N

Constraints


- 2 \leq N \leq 5 \times 10^5
- 1 \leq A_i \leq 10^9
- All input values are integers.

Sample Output 1

- He absorbs the slime to his right. As a result, the absorbed slime disappears, and his size becomes 3 + 2 = 5. The state becomes (4, 13, 2, [5], 6).

To solve this problem, we need to determine the maximum size a slime can achieve by absorbing adjacent slimes that are strictly smaller than itself any number of times. The solution involves efficiently computing the maximum absorption for each slime in the array by leveraging stack-based boundary computation and prefix sums.

### Approach
1. **Boundary Computation**: For each slime, compute the nearest greater or equal element to its left and right using a stack. This helps in identifying the segments where the slime can absorb other slimes.
2. **Prefix Sum Array**: Precompute a prefix sum array to quickly calculate the sum of any segment in the array.
3. **Segment Sum Calculation**: For each slime, determine the sum of the segment between its left and right boundaries. This sum represents the maximum absorption possible for the slime.
4. **Result Calculation**: The final size of each slime is its original size plus the sum of the segment between its boundaries.

import sys

    for i in range(n-1, -1, -1):

        if stack:
