To solve this problem, we need to split a given sequence of integers into three non-empty contiguous subarrays such that the sum of the counts of distinct integers in each subarray is maximized. The challenge lies in efficiently finding the optimal split points without resorting to a brute-force approach, which would be computationally expensive for large sequences.

### Approach
1. **Problem Analysis**: The problem requires splitting the sequence into three parts and calculating the sum of distinct elements in each part. The key insight is that the sum of distinct elements in the three parts can be broken down into the sum of distinct elements in each part individually. 

2. **Intuition**: For each possible split point (i, j), the distinct counts of the left, middle, and right parts need to be computed. A brute-force approach would involve checking all possible splits, which is O(n^2) and impractical for large n (up to 300,000). 

3. **Optimization**: 
   - **Precompute Prefix and Suffix Arrays**: We first compute two arrays, `left_distinct` and `right_distinct`, where `left_distinct[i]` stores the number of distinct elements from the start up to index `i`, and `right_distinct[i]` stores the number of distinct elements from index `i` to the end.
   - **Sliding Window for Middle Part**: For each possible left split point `i`, we use a sliding window approach to compute the distinct count of the middle part as we vary the right split point `j`. This involves maintaining a frequency array to efficiently update the distinct count as elements are added to the middle part.

4. **Complexity Analysis**: The approach involves O(n) precomputation for the prefix and suffix arrays and O(n^2) for the nested loops in the worst case. However, for practical purposes, this solution is feasible for n up to 3000. For larger n, a more sophisticated method (like using a segment tree or sparse table) would be needed to achieve O(n log n) or O(n) complexity.

4. **Sliding Window for Middle Part**: For each possible left split point `i`, the code uses a frequency array to dynamically track distinct elements in the middle part as the right split point `j` varies. The sum of distinct elements in all three parts is computed for each valid split and the maximum sum is recorded.
5. **Output**: The maximum sum found is printed. For sequences larger than 3000, the code outputs a message indicating that the solution is not implemented for large inputs. 

This approach efficiently narrows down the problem to manageable computations for moderate input sizes, leveraging precomputation and dynamic updates to minimize redundant calculations.
