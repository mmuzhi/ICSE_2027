To solve this problem, we need to determine the number of empty squares in an N x N grid where a new piece can be placed without being captured by any existing pieces. The existing pieces are given, and a new piece is captured if it shares the same row, column, or either of the two diagonals (i+j or i-j) with any existing piece.

### Approach
1. **Problem Analysis**: The problem involves checking each empty square in the grid to see if it is not in the same row, column, or diagonal (both types) as any existing piece. Given the grid size can be as large as 10^9, a brute-force approach is infeasible. Instead, we use inclusion-exclusion principle to efficiently compute the forbidden squares (those that would be captured) and subtract this from the total grid squares to find the safe squares.

2. **Key Insights**:
   - **Forbidden Squares**: These are squares that lie in the same row, column, or either diagonal as any existing piece. The union of these sets gives the total forbidden squares.
   - **Inclusion-Exclusion Principle**: This helps in calculating the union of multiple sets (rows, columns, diagonals) by considering their intersections to avoid overcounting.

3. **Algorithm Selection**:
   - **Set Operations**: Use sets to store rows, columns, and diagonals (both i+j and i-j) for quick lookup.
   - **Binary Search**: For efficient counting of diagonal squares within specific ranges.
   - **Inclusion-Exclusion**: Compute the union size by adding individual set sizes, subtracting pairwise intersections, adding triple intersections, and subtracting the quadruple intersection.

4. **Complexity Analysis**:
   - **Time Complexity**: The solution involves iterating over sets of diagonals and rows/columns, which are limited to M (竕､1000) elements. The dominant operations are binary searches and nested loops over these sets, resulting in O(M^2) operations, which is feasible given M 竕､ 1000.
   - **Space Complexity**: The space required is O(M) to store the sets of rows, columns, and diagonals.

```python
        if start_i > end_i:
            return 0
        return end_i - start_i + 1

    C = sum(count1(d) for d in D1)
    D = sum(count2(d) for d in D2)
    
    A_inter_B = len(R_rows) * len(R_cols)
    
    A_inter_C = 0
    for i in R_rows:
        low = i + 1
        high = i + n
        l = bisect.bisect_left(D1_sorted, low)
        r = bisect.bisect_right(D1_sorted, high)
        A_inter_C += (r - l)
        
    A_inter_D = 0
    for i in R_rows:
        low = i - n
        high = i - 1
        l = bisect.bisect_left(D2_sorted, low)
        r = bisect.bisect_right(D2_sorted, high)
        A_inter_D += (r - l)
        
    B_inter_C = 0
    for j in R_cols:
        low = j + 1
        high = j + n
        l = bisect.bisect_left(D1_sorted, low)
        r = bisect.bisect_right(D1_sorted, high)
        B_inter_C += (r - l)
        
    B_inter_D = 0
    for j in R_cols:
        low = j - n
        high = j + n
        l = bisect.bisect_left(D2_sorted, low)
        r = bisect.bisect_right(D2_sorted, high)
        B_inter_D += (r - l)
        
    C_inter_D = 0
    for d1 in D1:
        for d2 in D2:
            if (d1 + d2) % 2 == 0 and (d1 - d2) % 2 == 0:
                i_val = (d1 + d2) // 2
                j_val = (d1 - d2) // 2
                if 1 <= i_val <= n and 1 <= j_val <= n:
                    C_inter_D += 1
                    
    A_inter_B_inter_C = 0
    for d in D1:
        start_i = max(1, d - n)
        end_i = min(n, d - 1)
        for i in R_rows:
            if i < start_i or i > end_i:
                continue
            j_val = d - i
            if j_val in R_cols:
                A_inter_B_inter_C += 1
                
    A_inter_B_inter_D = 0
    for d in D2:
        start_i = max(1, d + 1)
        end_i = min(n, d + n)
        for i in R_rows:
            if i < start_i or i > end_i:
                continue
            j_val = i - d
            if j_val in R_cols:
                A_inter_B_inter_D += 1
                
    A_inter_C_inter_D = 0
    for d1 in D1:
        for d2 in D2:
            if (d1 + d2) % 2 == 0 and (d1 - d2) % 2 == 0:
                i_val = (d1 + d2) // 2
                j_val = (d1 - d2) // 2
                if 1 <= i_val <= n and 1 <= j_val <= n and i_val in R_rows:
                    A_inter_C_inter_D += 1
                    
    B_inter_C_inter_D = 0
    for d1 in D1:
        for d2 in D2:
            if (d1 + d2) % 2 == 0 and (d1 - d2) % 2 == 0:
                i_val = (d1 + d2) // 2
                j_val = (d1 - d2) // 2
                if 1 <= i_val <= n and 1 <= j_val <= n and j_val in R_cols:
                    B_inter_C_inter_D += 1
                    
    A_inter_B_inter_C_inter_D = 0
    for d1 in D1:
        for d2 in D2:
            if (d1 + d2) % 2 == 0 and (d1 - d2) % 2 == 0:
                i_val = (d1 + d2) // 2
                j_val = (d1 - d2) // 2
                if 1 <= i_val <= n and 1 <= j_val <= n and i_val in R_rows and j_val in R_cols:
                    A_inter_B_inter_C_inter_D += 1
                    
    union_size = A + B + C + D - A_inter_B - A_inter_C - A_inter_D - B_inter_C - B_inter_D - C_inter_D + A_inter_B_inter_C + A_inter_B_inter_D + A_inter_C_inter_D + B_inter_C_inter_D - A_inter_B_inter_C_inter_D
    answer = n * n - union_size
    print(answer)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read and parsed into grid size `n` and the number of pieces `m`, followed by the coordinates of the pieces.
2. **Set Initialization**: Sets `R_rows` and `R_cols` store the rows and columns containing existing pieces. Sets `D1` and `D2` store the sums and differences of coordinates for the diagonals.
3. **Counting Forbidden Squares**: The sizes of the forbidden sets (rows, columns, diagonals) are computed using helper functions `count1` and `count2` which determine the number of squares on each diagonal.
4. **Inclusion-Exclusion Principle**: The union size of all forbidden squares is computed by adding individual set sizes, subtracting their pairwise intersections, adding triple intersections, and subtracting the quadruple intersection.
5. **Result Calculation**: The number of safe squares is obtained by subtracting the union size from the total grid squares (`n * n`).

This approach efficiently handles the large grid size by leveraging set operations and binary search, ensuring optimal performance even for the upper constraint limits.
