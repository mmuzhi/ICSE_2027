To solve this problem, we need to find the minimum sum of a subarray within a given array where the subarray's size is between `l` and `r` (inclusive) and the sum is positive. If no such subarray exists, we return -1. 

### Approach
1. **Prefix Sum Array**: We first compute a prefix sum array to efficiently calculate the sum of any subarray. The prefix sum array allows us to compute the sum of any subarray in constant time.
2. **Iterate Over Subarrays**: For each starting index `i` in the array, we determine the range of ending indices `j` such that the subarray from `i` to `j` has a length between `l` and `r`. 
3. **Check Positive Sum**: For each valid subarray, we check if its sum is positive. If it is, we compare it with the current minimum sum and update the minimum if the new sum is smaller.
4. **Early Termination**: If at any point we find a subarray with a sum of 1 (the smallest possible positive integer), we can immediately return 1 since no smaller positive sum can exist.
5. **Result Handling**: After checking all possible subarrays, if no valid subarray is found, we return -1; otherwise, we return the smallest positive sum found.

### Solution Code
```python
class Solution:
    def minimumSumSubarray(self, nums: List[int], l: int, r: int) -> int:
        n = len(nums)
        prefix = [0] * (n + 1)
        for i in range(1, n + 1):
            prefix[i] = prefix[i - 1] + nums[i - 1]
        
        min_sum = float('inf')
        for i in range(n):
            start_j = i + l - 1
            end_j = min(i + r - 1, n - 1)
            if start_j > end_j:
                continue
            for j in range(start_j, end_j + 1):
                s = prefix[j + 1] - prefix[i]
                if s > 0:
                    if s < min_sum:
                        min_sum = s
                    if min_sum == 1:
                        return 1
        return min_sum if min_sum != float('inf') else -1
```

### Explanation
1. **Prefix Sum Array**: The prefix sum array is constructed such that each element at index `i` contains the sum of all elements from the start of the array up to, but not including, the element at index `i` in the original array. This allows for efficient subarray sum calculations.
2. **Subarray Iteration**: For each starting index `i`, we determine the valid range of ending indices `j` such that the subarray length (from `i` to `j`) is between `l` and `r`. This is done by setting `start_j` to `i + l - 1` and `end_j` to the minimum of `i + r - 1` and the last valid index of the array.
3. **Sum Calculation and Comparison**: For each valid subarray, the sum is calculated using the prefix sum array. If the sum is positive, it is compared with the current minimum sum. If it is smaller, the minimum sum is updated. Additionally, if the minimum sum becomes 1, we immediately return 1 as it is the smallest possible positive sum.
4. **Result Handling**: After processing all subarrays, if no valid subarray is found (i.e., `min_sum` remains infinity), we return -1. Otherwise, the smallest positive sum found is returned.

This approach efficiently checks all possible subarrays within the given constraints, leveraging the prefix sum array for quick sum calculations, and optimizes by early termination if the smallest possible positive sum (1) is found. The complexity is O(n * (r - l + 1)), which is feasible given the problem constraints.
