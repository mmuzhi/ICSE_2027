To solve this problem, we need to hash a given string into a new string by processing it in chunks of a specified size. Each chunk is processed by summing the alphabet positions of its characters, taking the modulo 26 of that sum, and converting the result back to a lowercase letter. 

### Approach
1. **Problem Analysis**: The task involves breaking the input string into substrings of length `k`, processing each substring to compute a hash value, and then constructing the result string from these hash values. The hash value for each substring is derived by summing the positions of its characters in the alphabet (e.g., 'a' is 0, 'b' is 1, etc.), then taking the modulo 26 of this sum. The result is the character corresponding to this modulo value.
2. **Intuition**: The solution requires iterating over the string in steps of `k`, processing each substring individually. For each substring, we convert each character to its alphabet position, sum these positions, apply modulo 26, and map the result back to a character.
3. **Algorithm Selection**: The algorithm involves a simple loop to process each substring. For each substring, we compute the sum of the alphabet positions of its characters, then use modulo 26 to determine the resulting character.
4. **Complexity Analysis**: The algorithm processes each character exactly once, leading to a time complexity of O(n), where n is the length of the input string. This is efficient given the constraints (n ≤ 1000).

### Solution Code
```python
class Solution:
    def stringHash(self, s: str, k: int) -> str:
        n = len(s)
        result = []
        for i in range(0, n, k):
            total = 0
            for char in s[i:i+k]:
                total += ord(char) - ord('a')
            total %= 26
            result.append(chr(total + ord('a')))
        return ''.join(result)
```

### Explanation
1. **Initialization**: The solution starts by initializing an empty list `result` to store the hashed characters.
2. **Processing Substrings**: The outer loop iterates over the string in steps of `k`, processing each substring of length `k`.
3. **Summing Alphabet Positions**: For each character in the current substring, its alphabet position is found by subtracting the ASCII value of 'a' from its ASCII value. These positions are summed up.
4. **Modulo Operation**: The sum of the alphabet positions is taken modulo 26 to ensure it falls within the range of 0 to 25.
5. **Character Conversion**: The result of the modulo operation is converted back to a lowercase letter by adding the ASCII value of 'a' and converting to a character.
6. **Result Construction**: The processed characters are joined into a string and returned as the final result.

This approach efficiently processes each substring, computes the required hash values, and constructs the result string in linear time, making it optimal for the given problem constraints.
