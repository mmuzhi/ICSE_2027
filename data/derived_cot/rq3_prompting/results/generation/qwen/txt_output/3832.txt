=== ANSWER ===

To solve this problem, we need to determine the minimum cost required to reach each position in a line of people, starting from the end of the line. The key observation is that reaching a position can be done either by directly swapping with that position (paying its cost) or by first reaching a position further ahead and then swapping with the target position for free if it's behind you.

### Approach
1. **Forward Pass**: Traverse the cost array from left to right. For each position, the minimum cost to reach it is the lesser of its own cost or the minimum cost encountered so far from the left. This accounts for reaching the current position by swapping from any position to the left.
2. **Backward Pass**: Traverse the array from right to left. For each position, consider the cost of reaching it by first moving to a position further ahead (right) and then swapping back for free. The cost is the sum of the right position's minimum cost and the current position's cost. Update the current position's minimum cost if this combined cost is cheaper.

### Solution Code
```python
class Solution:
    def minCosts(self, cost: List[int]) -> List[int]:
        n = len(cost)
        if n == 0:
            return []
        dp = cost.copy()
        running_min = dp[0]
        for i in range(1, n):
            running_min = min(running_min, dp[i])
            dp[i] = min(dp[i], running_min)
        
        right_min = [float('inf')] * n
        if n > 1:
            right_min[n-1] = float('inf')
            for i in range(n-2, -1, -1):
                right_min[i] = min(dp[i+1], right_min[i+1])
        
        for i in range(n-1):
            candidate = cost[i] + right_min[i]
            if candidate < dp[i]:
                dp[i] = candidate
        
        return dp
```

### Explanation
1. **Forward Pass**: The algorithm starts by initializing a copy of the cost array as `dp`. It then iterates from the first to the last element, updating each element to be the minimum of its own cost or the minimum cost encountered so far from the left. This ensures that each position knows the cheapest way to reach it by coming from the left.
2. **Backward Pass**: After the forward pass, the algorithm computes an array `right_min` where each element at index `i` holds the minimum cost to reach any position to the right of `i`. Using this, it updates each position's cost by considering the cost of reaching it from the right (sum of the right position's minimum cost and the current position's cost). If this combined cost is cheaper, it updates the current position's cost.
3. **Result**: The final `dp` array contains the minimum cost to reach each position, combining both left and right traversal strategies for optimal results.

This approach efficiently computes the minimum costs by leveraging dynamic programming and careful traversal in both directions, ensuring optimal performance with a linear time complexity.
