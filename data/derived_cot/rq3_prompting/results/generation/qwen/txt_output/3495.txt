The distance of an obstacle at coordinate (x, y) from the origin is defined as |x| + |y|. The challenge is to handle up to 200,000 queries and return the k-th smallest distance after each query, or -1 if there are fewer than k obstacles.

### Approach
1. **Problem Analysis**: The problem requires maintaining a dynamic set of obstacles and efficiently querying the k-th smallest distance from the origin after each insertion. The key insight is that the distances can be large (up to 2e9) but the number of distinct distances from all queries is limited (up to 200,000), allowing us to use coordinate compression.

2. **Coordinate Compression**: We first collect all possible distances from all queries, sort them, and map each distance to a rank. This reduces the problem to working with a smaller range of ranks (1 to m, where m is the number of distinct distances).

3. **Fenwick Tree (Binary Indexed Tree)**: We use a Fenwick Tree to maintain the frequency of each compressed distance. This data structure supports efficient point updates and prefix sum queries, which are crucial for finding the k-th smallest distance.

4. **Binary Search via Fenwick Tree**: To find the k-th smallest distance, we perform a binary search on the Fenwick Tree to locate the smallest rank where the cumulative frequency (number of obstacles with distance ≤ current rank) is at least k. This leverages the properties of the Fenwick Tree for efficient range queries.

5. **Handling Queries**: For each query, we compute the distance, update the Fenwick Tree, and then determine the k-th smallest distance by querying the tree. If there are fewer than k obstacles, we return -1.
