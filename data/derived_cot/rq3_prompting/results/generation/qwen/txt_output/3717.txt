To solve this problem, we need to find the minimum number of operations required to transform an array such that there are at least `k` non-overlapping subarrays of exactly length `x` where all elements in each subarray are equal. The operations allowed are incrementing or decrementing any element by 1.

### Approach
1. **Problem Analysis**: The problem requires us to select `k` non-overlapping subarrays of length `x` in the array. For each selected subarray, we need to make all its elements equal with the least number of operations. The operations for each subarray are the sum of absolute differences between each element and the chosen value (which is optimally the median of the subarray).

2. **Key Insight**: The minimal number of operations for a subarray of length `x` is computed by sorting the subarray and summing the differences between paired elements (smallest and largest, second smallest and second largest, etc.).

3. **Algorithm Selection**:
   - **Sliding Window with Sorted List**: For each possible starting index of a subarray of length `x`, maintain a sorted list of elements in that subarray. The cost for the subarray is computed by summing the differences between the largest and smallest elements in each pair.
   - **Dynamic Programming (DP)**: Use a DP array where `dp[i][j]` represents the minimal cost to have selected `j` subarrays with the next subarray starting at index `i`. The DP is filled by either skipping the current index or starting a new subarray at the current index if possible.

4. **Complexity Analysis**: The algorithm involves:
   - **Preprocessing**: For each of the `O(n)` windows of size `x`, sorting the window takes `O(x log x)` time. In the worst case, this results in `O(n * x log x)` time.
   - **DP**: The DP processes each of the `O(n * k)` states, each involving a constant-time operation. Thus, the overall complexity is `O(n * x log x + n * k)`, which is feasible given the constraints.

### Solution Code
```python
import bisect
from typing import List

class Solution:
    def minOperations(self, nums: List[int], x: int, k: int) -> int:
        n = len(nums)
        if n < x * k:
            return -1
        
        cost_arr = [0] * (n - x + 1)
        if n - x + 1 > 0:
            sorted_list = sorted(nums[:x])
            left = 0
            right = x - 1
            while left < right:
                cost_arr[0] += sorted_list[right] - sorted_list[left]
                left += 1
                right -= 1
            
            for i in range(1, n - x + 1):
                pos_remove = bisect.bisect_left(sorted_list, nums[i-1])
                del sorted_list[pos_remove]
                pos_insert = bisect.bisect_left(sorted_list, nums[i+x-1])
                sorted_list.insert(pos_insert, nums[i+x-1])
                left = 0
                right = x - 1
                while left < right:
                    cost_arr[i] += sorted_list[right] - sorted_list[left]
                    left += 1
                    right -= 1
        
        dp = [[float('inf')] * (k+1) for _ in range(n+1)]
        for i in range(n, -1, -1):
            for j in range(k+1):
                if i < n:
                    dp[i][j] = min(dp[i][j], dp[i+1][j])
                if i <= n - x and j > 0:
                    dp[i][j] = min(dp[i][j], dp[i+x][j-1] + cost_arr[i])
        
        return min(dp[i][k] for i in range(n+1))
```

### Explanation
1. **Preprocessing**: The code first checks if the array can accommodate `k` non-overlapping subarrays of length `x`. If not, it returns -1. It then initializes an array `cost_arr` to store the minimal operations for each possible subarray of length `x`. Using a sorted list, it computes the cost for the first subarray and updates the list for subsequent subarrays by removing the outgoing element and inserting the incoming element.

2. **Dynamic Programming**: The DP array `dp` is filled from the end of the array to the beginning. For each index `i` and number of subarrays `j`, the algorithm either skips the current index or starts a new subarray at `i` (if `j > 0`), updating the DP value accordingly.

3. **Result Extraction**: The minimal value in the DP array for `k` subarrays is the answer, representing the least number of operations needed to achieve the desired configuration of the array. 

This approach efficiently combines sliding window processing with dynamic programming to solve the problem within feasible time complexity.
