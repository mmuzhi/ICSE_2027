To solve this problem, we need to compute the sum of all possible substrings of a given string of digits, where each substring is interpreted as a decimal integer. The challenge lies in efficiently calculating this sum without resorting to a brute-force approach, which would be computationally infeasible for large strings (up to 200,000 characters).

### Approach
1. **Problem Analysis**: The problem requires summing the values of all contiguous substrings of a given string. For example, for the string "379", the substrings are "3", "7", "9", "37", "79", and "379", and their sum is 3 + 7 + 9 + 37 + 79 + 379 = 514.

2. **Insight**: The key observation is that each digit in the string contributes to the total sum based on its position and the number of substrings it appears in. Specifically, a digit at position `i` (0-indexed) appears in `(i + 1)` substrings as the most significant digit, `i` substrings as the next significant digit, and so on. 

3. **Algorithm Selection**: We use a dynamic programming approach where `dp[i]` represents the sum of all substrings ending at position `i`. The recurrence relation for `dp[i]` is derived by considering that each substring ending at `i` can be formed by appending the current digit to all substrings ending at `i-1`, and adding the digit itself. The recurrence is:
   - `dp[i] = digit + 10 * dp[i-1] + digit * i`
   Here, `digit` is the integer value of the current character, and `i` is the number of substrings ending at `i-1`.

4. **Complexity Analysis**: The algorithm processes each character once, performing constant-time operations for each. The space complexity is linear, O(n), as we store only the current and previous `dp` values. The time complexity is O(n), making it efficient for large inputs.

### Solution Code
```python
n = int(input().strip())
s = input().strip()

if n == 0:
    print(0)
    exit(0)

dp = [0] * n
dp[0] = int(s[0])
total = dp[0]

for i in range(1, n):
    digit = int(s[i])
    dp[i] = digit + 10 * dp[i-1] + digit * i
    total += dp[i]

print(total)
```

### Explanation
1. **Initialization**: The code starts by reading the input values: the length of the string `n` and the string `s` itself. If the string is empty, it prints 0 and exits.

2. **Dynamic Programming Setup**: The `dp` array is initialized to store the sum of substrings ending at each position. The first element `dp[0]` is simply the first digit of the string.

3. **Iteration**: For each subsequent digit, the code calculates `dp[i]` using the recurrence relation. This involves:
   - Converting the current character to an integer.
   - Updating `dp[i]` by combining the current digit with the sum of substrings ending at the previous position (`dp[i-1]`), scaled by 10, and the product of the digit and its position index `i`.

4. **Summation**: The total sum is accumulated by adding each `dp[i]` to a running total.

5. **Output**: Finally, the total sum is printed.

This approach efficiently computes the desired sum by leveraging dynamic programming to avoid redundant calculations, making it suitable for large input sizes.
