To solve this problem, we need to compute the sum of Manhattan distances between every pair of pieces over all valid arrangements of `k` identical pieces on an `m x n` grid. The Manhattan distance between two cells `(x_i, y_i)` and `(x_j, y_j)` is defined as `|x_i - x_j| + |y_i - y_j|`. The solution involves leveraging combinatorial mathematics and modular arithmetic to efficiently compute the result without enumerating all possible arrangements.

### Approach
1. **Problem Analysis**: The problem requires summing the Manhattan distances for all pairs of pieces in every valid arrangement of `k` pieces on the grid. Given the constraints, a brute-force approach is infeasible. Instead, we use the linearity of combinations and symmetry to break the problem into manageable parts.

2. **Key Insight**: The total sum can be derived by considering each pair of cells in the grid and determining how many times each pair contributes to the Manhattan distance across all valid arrangements. The contribution of each pair is the Manhattan distance multiplied by the number of arrangements that include both cells, which is given by the combination `C(R-2, k-2)`, where `R = m * n` is the total number of cells.

3. **Mathematical Formulation**: The sum of Manhattan distances is the sum over all pairs of cells `(i, j)` of `(number of arrangements including both i and j) * (|x_i - x_j| + |y_i - y_j|)`. This simplifies to `C(R-2, k-2) * (Sum_x + Sum_y)`, where `Sum_x` and `Sum_y` are the sums of absolute differences in the x and y coordinates, respectively, for all pairs of cells in the grid.

4. **Efficient Calculation**: 
   - **Sum_x and Sum_y**: These are computed by considering the grid's structure. For the x-coordinates (rows), each row index `r` appears `n` times. The sum of absolute differences for all pairs is derived using the formula for the sum of absolute differences in a multiset, leveraging symmetry and combinatorial identities.
   - **Combinatorial Calculation**: The combination `C(R-2, k-2)` is computed using precomputed factorials and modular inverses for efficiency under large numbers.

5. **Modular Arithmetic**: Given the large possible values, all calculations are performed modulo `10^9 + 7` to avoid overflow and meet problem requirements.

### Solution Code
```python
MOD = 10**9 + 7

def main():
    import sys
    data = sys.stdin.read().split()
    m = int(data[0])
    n = int(data[1])
    k = int(data[2])
    R = m * n
    
    max_n = 100000
    fact = [1] * (max_n + 1)
    for i in range(1, max_n + 1):
        fact[i] = fact[i - 1] * i % MOD

    def nCr(n_val, r_val):
        if r_val < 0 or r_val > n_val:
            return 0
        return fact[n_val] * pow(fact[r_val], MOD - 2, MOD) % MOD * pow(fact[n_val - r_val], MOD - 2, MOD) % MOD

    Double_sum_x = 0
    for r in range(m):
        t = (m - 1 - r) * (m - r) // 2
        Double_sum_x += t

    Double_sum_y = 0
    for r in range(n):
        t = (n - 1 - r) * (n - r) // 2
        Double_sum_y += t

    total_x = n * n * Double_sum_x
    total_y = m * m * Double_sum_y
    total_val = (total_x + total_y) % MOD

    comb = nCr(R - 2, k - 2)
    result = comb * total_val % MOD
    print(result)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input values for `m`, `n`, and `k` are read from standard input.
2. **Factorial Precomputation**: Factorials up to `100,000` are precomputed modulo `10^9 + 7` to facilitate efficient combination calculations.
3. **Combination Function**: The `nCr` function computes combinations using precomputed factorials and modular inverses for efficiency.
4. **Double Sum Calculation**: The sums `Double_sum_x` and `Double_sum_y` are computed by iterating over each row and column, respectively, and summing the absolute differences in their indices. This leverages the grid's structure where each row or column index is repeated a fixed number of times.
5. **Total Manhattan Distance**: The total Manhattan distance is derived by combining the x and y components, scaled by the number of cells per row and column, and taken modulo `10^9 + 7`.
6. **Result Calculation**: The final result is the product of the combination `C(R-2, k-2)` and the total Manhattan distance, again modulo `10^9 + 7`.

This approach efficiently breaks down the problem using combinatorial mathematics and modular arithmetic, ensuring optimal performance even for large grid sizes.
