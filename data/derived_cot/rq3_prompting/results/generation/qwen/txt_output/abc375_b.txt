To solve this problem, we need to calculate the total cost for Takahashi's journey starting from the origin (0, 0), visiting N given points in order, and returning to the origin. The cost between two points is the Euclidean distance between them. 

### Approach
1. **Problem Analysis**: The problem involves computing the sum of Euclidean distances between consecutive points, including the initial move from the origin to the first point and the final move from the last point back to the origin. The constraints specify that the number of points can be as large as 200,000, so the solution must efficiently compute these distances without excessive computational overhead.

2. **Intuition**: The straightforward approach is to iterate through each point, compute the Euclidean distance from the previous point (or the origin for the first point), and accumulate these distances. The Euclidean distance between two points (a, b) and (c, d) is given by \(\sqrt{(a - c)^2 + (b - d)^2}\).

3. **Algorithm Selection**: The algorithm involves:
   - Reading the input values.
   - Calculating the distance from the origin to the first point.
   - Iterating through each subsequent point, computing the distance from the previous point to the current one.
   - Finally, computing the distance from the last point back to the origin.
   - Summing all these distances for the total cost.

4. **Complexity Analysis**: The algorithm processes each of the N points exactly once, performing a constant amount of work for each point (two subtractions, two multiplications, one addition, and one square root operation). Thus, the time complexity is O(N), which is optimal for the given constraints.

5. **Precision Consideration**: The problem requires an absolute or relative error of at most \(10^{-6}\). Using Python's `math.sqrt` ensures sufficient precision for the given constraints, as double-precision floating-point numbers (used by `math.sqrt`) provide about 15-16 decimal digits of precision, which is adequate for the problem's requirements.

### Solution Code
```python
import math

def main():
    n = int(input().strip())
    points = []
    for _ in range(n):
        data = input().split()
        x = int(data[0])
        y = int(data[1])
        points.append((x, y))
    
    total = 0.0
    if n >= 1:
        x0, y0 = points[0]
        total += math.sqrt(x0*x0 + y0*y0)
    
    for i in range(n - 1):
        x1, y1 = points[i]
        x2, y2 = points[i + 1]
        dx = x1 - x2
        dy = y1 - y2
        total += math.sqrt(dx*dx + dy*dy)
    
    if n >= 1:
        x_last, y_last = points[-1]
        total += math.sqrt(x_last*x_last + y_last*y_last)
    
    print("{:.20f}".format(total))

if __name__ == "__main__":
    main()
```

3. **Intermediate Segments**: For each subsequent point, the distance from the previous point is computed and added to the total. This involves subtracting the coordinates of consecutive points, squaring the differences, summing them, and taking the square root.

4. **Final Segment**: After processing all points, the distance from the last point back to the origin is computed and added to the total.

5. **Output**: The total cost is printed with a precision of 20 decimal places to meet the problem's accuracy requirements.

This approach efficiently computes the total journey cost by leveraging straightforward arithmetic operations and ensures the solution is both correct and optimal for large input sizes.
