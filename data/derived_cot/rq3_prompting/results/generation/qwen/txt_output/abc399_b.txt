=== ANSWER ===

To solve this problem, we need to determine the ranks of N participants in a contest based on their scores. The ranking procedure involves assigning the highest available rank to the participant(s) with the highest current score, then incrementing the rank by the number of participants just ranked before repeating the process until all participants are ranked.

### Approach
1. **Initialization**: Read the number of participants (N) and their scores. Initialize an array to store the ranks of each participant, initially set to 0. Also, maintain a set of indices representing participants whose ranks are yet to be determined.
2. **Rank Assignment Loop**: While there are undetermined participants:
   - **Find Maximum Score**: Determine the highest score among the remaining undetermined participants.
   - **Identify Participants**: Collect all participants with this highest score.
   - **Assign Rank**: Assign the current rank value to all these participants.
   - **Update Rank**: Increment the current rank by the number of participants just ranked.
   - **Remove Processed Participants**: Remove these participants from the set of undetermined participants.
3. **Output**: Print the ranks of all participants in the order of their input.

### Solution Code
```python
def main():
    n = int(input().strip())
    p = list(map(int, input().split()))
    
    rank = [0] * n
    undetermined = set(range(n))
    r = 1
    
    while undetermined:
        max_score = max(p[i] for i in undetermined)
        to_remove = [i for i in undetermined if p[i] == max_score]
        for idx in to_remove:
            rank[idx] = r
        undetermined -= set(to_remove)
        r += len(to_remove)
    
    for i in range(n):
        print(rank[i])

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The code starts by reading the number of participants (N) and their scores.
2. **Initialization**: The `rank` array is initialized to store the rank of each participant, and the `undetermined` set keeps track of participants who haven't been ranked yet. The variable `r` starts at 1, representing the initial rank.
3. **Loop Until All Ranked**: The loop continues as long as there are participants left to rank. In each iteration:
   - **Maximum Score**: The highest score among the undetermined participants is found.
   - **Group Identification**: All participants with this score are identified.
   - **Rank Assignment**: The current rank (`r`) is assigned to all identified participants.
   - **Update Rank**: The rank is incremented by the number of participants just ranked.
   - **Update Undetermined Set**: The identified participants are removed from the set of undetermined participants.
4. **Output**: After processing all participants, their ranks are printed in the order of their input.

This approach efficiently handles the ranking by always processing the highest-scoring participants first and dynamically updating the rank variable, ensuring correct and optimal rank assignment.
