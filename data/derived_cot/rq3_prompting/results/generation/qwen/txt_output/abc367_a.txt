=== ANSWER ===

To solve this problem, we need to determine whether Takahashi can shout his love for takoyaki every day at a specific time A, given that he goes to bed at time B and wakes up at time C. The solution involves checking if the time A falls within the intervals when Takahashi is awake based on his sleep schedule.

### Approach
1. **Problem Analysis**: The key observation is that Takahashi's awake periods depend on whether his sleep schedule wraps around midnight (i.e., if B > C) or not (i.e., if B ≤ C). 
2. **Case 1 (B ≤ C)**: In this scenario, Takahashi is awake from 0 to B and from C to 24 (end of the day). The time A must be in one of these intervals for him to be awake.
3. **Case 2 (B > C)**: Here, Takahashi's sleep period wraps around midnight, meaning he is awake from C to B. The time A must lie within this interval for him to be awake.
4. **Decision Making**: For each case, we check if the given time A falls within the respective awake intervals. If it does, the answer is "Yes"; otherwise, it's "No".

### Solution Code
```python
def main():
    A, B, C = map(int, input().split())
    if B <= C:
        if A < B or A >= C:
            print("Yes")
        else:
            print("No")
    else:
        if C <= A < B:
            print("Yes")
        else:
            print("No")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Input Handling**: The code reads three integers A, B, and C, representing the time Takahashi wants to shout, his bedtime, and his wake-up time, respectively.
2. **Case 1 (B ≤ C)**: The code checks if A is either before B (A < B) or after or equal to C (A ≥ C). If either condition is true, Takahashi is awake at time A, and the output is "Yes"; otherwise, it's "No".
3. **Case 2 (B > C)**: The code checks if A lies between C and B (inclusive of C and exclusive of B). If true, Takahashi is awake at time A, and the output is "Yes"; otherwise, it's "No".
This approach efficiently checks the two possible scenarios (sleep schedule wrapping or not) and determines Takahashi's availability at time A using straightforward conditional checks.
