To solve this problem, we need to determine the minimum number of strings (walks) required in an NG list such that every given two-letter product name appears as a contiguous substring in at least one of the strings, and no string contains any two-letter substring not present in the given product names. 

### Approach
1. **Graph Representation**: Treat each two-letter product name as a directed edge in a graph where nodes are the letters involved. For example, the product name "AB" becomes an edge from node 'A' to node 'B'.
2. **Degree Calculation**: For each node in the graph, calculate its out-degree (number of outgoing edges) and in-degree (number of incoming edges).
3. **Imbalance Calculation**: The imbalance for each node is the difference between its out-degree and in-degree. The total imbalance is the sum of these differences across all nodes.
4. **Minimum Walks Calculation**: The minimum number of walks required to cover all edges in the directed graph is given by half the total imbalance (both positive and negative imbalances). This is derived from the Chinese Postman Problem's solution for directed graphs, where the goal is to cover all edges with the minimum number of walks, allowing edge reuse.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    
    n = int(data[0])
    edges = []
    for i in range(1, 1 + n):
        edges.append(data[i].strip())
    
    graph = {}
    for edge in edges:
        u, v = edge[0], edge[1]
        if u not in graph:
            graph[u] = {'out_degree': 0, 'in_degree': 0}
        if v not in graph:
            graph[v] = {'out_degree': 0, 'in_degree': 0}
        graph[u]['out_degree'] += 1
        graph[v]['in_degree'] += 1
        
        if u == v:
            graph[u]['in_degree'] += 1
            
    nodes = list(graph.keys())
    a = 0
    b = 0
    for node in nodes:
        out_degree = graph[node]['out_degree']
        in_degree = graph[node]['in_degree']
        if out_degree > in_degree:
            a += (out_degree - in_degree)
        elif in_degree > out_degree:
            b += (in_degree - out_degree)
            
    total_imbalance = a + b
    if total_imbalance == 0:
        print(1)
    else:
        print(total_imbalance // 2)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Graph Representation**: The input product names are read and stored as directed edges in a graph. Each node's out-degree and in-degree are tracked.
2. **Degree Calculation**: For each node, the number of outgoing and incoming edges is counted. This helps in determining the balance of the graph.
3. **Imbalance Calculation**: The total imbalance is computed by summing the positive and negative differences between out-degree and in-degree for all nodes. This imbalance indicates how many times edges need to be traversed to balance the graph.
4. **Minimum Walks Calculation**: The solution leverages the Chinese Postman Problem's insight that the minimum number of walks needed to cover all edges is half the total imbalance. If the graph is already balanced (total imbalance zero), one walk suffices. Otherwise, the answer is the integer division of the total imbalance by 2.

This approach efficiently calculates the minimal number of strings required in the NG list by modeling the problem as a directed graph and applying graph theory principles to determine edge coverage.
