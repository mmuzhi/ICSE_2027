To solve this problem, we need to determine the maximum total number of moves Alice can achieve in a turn-based game where she and Bob alternately capture pawns with a knight on a 50x50 chessboard. The game involves selecting pawns in a way that maximizes the total moves for Alice while Bob aims to minimize it.

### Approach
1. **Problem Analysis**: The problem involves a knight moving on a chessboard to capture pawns. Each player, starting with Alice, selects a pawn to capture in the fewest possible moves. The total moves are the sum of the minimal moves required for each capture. The challenge is to model the game states and transitions efficiently, considering the optimal play from both players.

2. **Intuition**: The solution involves precomputing the minimal moves between all pairs of relevant points (start position and all pawn positions) using BFS. Then, using dynamic programming with memoization (minimax approach) to explore all possible game states, where each state is defined by the set of captured pawns (bitmask) and the current knight position. Alice aims to maximize the total moves, while Bob aims to minimize them.

3. **Algorithm Selection**: 
   - **BFS for Distance Matrix**: For each relevant point (start and all pawns), compute the minimal knight moves to every other point using BFS.
   - **Dynamic Programming with Memoization**: Define a DP function that, given a bitmask (representing captured pawns) and the current knight position, returns the optimal total moves from that state. The DP function checks all possible pawn captures, recursively solving subproblems for each resulting state.

4. **Complexity Analysis**: 
   - **BFS**: For each of the up to 16 points, BFS runs on a 50x50 grid, resulting in a time complexity of O(16 * 50^2) which is feasible.
   - **DP States**: The number of states is O(2^n * (n+1)), where n is the number of pawns (up to 15). Each state processes up to n transitions, leading to an overall complexity of O(2^n * n * (n+1)), which is manageable for n ≤ 15.

### Solution Code
```python
from collections import deque
from functools import lru_cache

class Solution:
    def maxMoves(self, kx: int, ky: int, positions: List[List[int]]) -> int:
        n = len(positions)
        if n == 0:
            return 0
        
        points = [(kx, ky)] + positions
        n_points = n + 1
        
        directions = [(2,1), (2,-1), (-2,1), (-2,-1), (1,2), (1,-2), (-1,2), (-1,-2)]
        dist_matrix = [[0] * n_points for _ in range(n_points)]
        points_set = set(points)
        
        for idx in range(n_points):
            start = points[idx]
            q = deque()
            q.append((start[0], start[1], 0))
            visited = set()
            visited.add((start[0], start[1]))
            dist_dict = {}
            while q and len(dist_dict) < n_points:
                x, y, moves = q.popleft()
                if (x, y) in points_set:
                    dist_dict[(x, y)] = moves
                    if len(dist_dict) == n_points:
                        break
                for dx, dy in directions:
                    nx, ny = x + dx, y + dy
                    if 0 <= nx < 50 and 0 <= ny < 50 and (nx, ny) not in visited:
                        visited.add((nx, ny))
                        q.append((nx, ny, moves + 1))
            
            for j in range(n_points):
                if points[j] in dist_dict:
                    dist_matrix[idx][j] = dist_dict[points[j]]
                else:
                    dist_matrix[idx][j] = 10**9
        
        @lru_cache(maxsize=None)
        def dp(mask, pos):
            if mask == (1 << n) - 1:
                return 0
            num_captured = bin(mask).count("1")
            if num_captured % 2 == 0:
                best = -1
                for i in range(n):
                    if not (mask & (1 << i)):
                        new_mask = mask | (1 << i)
                        moves_required = dist_matrix[pos][i+1]
                        next_moves = dp(new_mask, i+1)
                        total_moves = moves_required + next_moves
                        if best == -1 or total_moves > best:
                            best = total_moves
                return best
            else:
                best = 10**9
                for i in range(n):
                    if not (mask & (1 << i)):
                        new_mask = mask | (1 << i)
                        moves_required = dist_matrix[pos][i+1]
                        next_moves = dp(new_mask, i+1)
                        total_moves = moves_required + next_moves
                        if total_moves < best:
                            best = total_moves
                return best
        
        return dp(0, 0)
```

### Explanation
1. **BFS for Distance Matrix**: The BFS precomputes the minimal moves between all pairs of points (start and pawns) using a knight's movement pattern. This matrix helps in quickly determining the moves required to capture any pawn from any position during the game.
2. **Dynamic Programming with Memoization**: The DP function `dp(mask, pos)` computes the optimal total moves from the current state (defined by the bitmask of captured pawns and the knight's position). Alice maximizes the moves, while Bob minimizes them. The function explores all possible pawn captures, recursively solving subproblems for each resulting state.
3. **Base Case**: When all pawns are captured (bitmask is all 1s), the function returns 0, as no further moves are needed.
4. **Recursive Case**: For each uncaptured pawn, the function calculates the minimal moves to capture it, updates the state, and recursively computes the total moves. The result is derived by choosing the maximum (Alice) or minimum (Bob) value from these possibilities.
5. **Initial Call**: The game starts with no pawns captured (mask 0) and the knight at the start position (pos 0), initiating the DP computation to determine the optimal total moves.
