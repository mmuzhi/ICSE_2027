To solve this problem, we need to compute the sum of the remainders when the sum of all possible subarrays of a given sequence of integers is divided by a positive integer M. Given the constraints, a brute-force approach is infeasible, so we need an efficient method to handle large input sizes.

### Approach
1. **Prefix Sum Calculation**: Compute the prefix sums of the sequence to facilitate subarray sum calculations.
2. **Modulo Operation**: For each prefix sum, compute its modulo M to handle large numbers and simplify the remainder calculations.
3. **Prefix Sum of Modulo Values**: Calculate the prefix sums of the modulo values to efficiently compute the sum of modulo values for any subarray.
4. **Fenwick Tree (Binary Indexed Tree)**: Use a Fenwick Tree to keep track of the frequency of modulo values encountered so far. This allows efficient range queries to count how many previous modulo values are greater than the current modulo value, which is crucial for determining the remainder of subarray sums.
5. **Efficient Calculation**: For each position in the sequence, use the Fenwick Tree to quickly compute the number of relevant previous modulo values and update the total sum accordingly.

```python
import sys

class Fenw:
    def __init__(self, n):
        self.n = n
        self.tree = [0] * (n + 1)

    def update(self, index, delta):
        i = index + 1
        while i <= self.n:
            self.tree[i] += delta
            i += i & -i

    def query(self, index):
        if index < 0:
            return 0
        i = index + 1
        s = 0
        while i:
            s += self.tree[i]
            i -= i & -i
        return s

    def range_query(self, l, r):
        if l > r:
            return 0
        return self.query(r) - self.query(l - 1)

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    m = int(data[1])
    A = list(map(int, data[2:2 + n]))

    S = [0] * (n + 1)
    for i in range(1, n + 1):
        S[i] = S[i - 1] + A[i - 1]

    B = [0] * (n + 1)
    for i in range(1, n + 1):
        B[i] = S[i] % m

    T_arr = [0] * (n + 1)
    for i in range(1, n + 1):
        T_arr[i] = T_arr[i - 1] + B[i - 1]

    fenw_tree = Fenw(m)
    total_ans = 0

    for r in range(1, n + 1):
        x = B[r]
        if x + 1 <= m - 1:
            cnt2 = fenw_tree.range_query(x + 1, m - 1)
        else:
            cnt2 = 0

        term = B[r] * r - T_arr[r] + cnt2 * m
        total_ans += term

        fenw_tree.update(x, 1)

    print(total_ans)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Prefix Sum Calculation**: The array `S` stores the cumulative sums of the sequence up to each index, allowing efficient subarray sum calculations.
2. **Modulo Operation**: The array `B` stores each prefix sum modulo M, ensuring values are within manageable limits for further computations.
3. **Prefix Sum of Modulo Values**: The array `T_arr` stores cumulative sums of the modulo values, enabling quick access to the sum of any range of modulo values.
4. **Fenwick Tree**: This data structure efficiently tracks the frequency of modulo values encountered. For each position, it queries how many previous modulo values are greater than the current value, which helps in calculating the remainder of subarray sums without explicitly processing each subarray.
5. **Efficient Calculation**: For each element in the sequence, the algorithm calculates the contribution of all subarrays ending at that position by leveraging the Fenwick Tree to count relevant previous values and updates the total sum accordingly. This approach ensures optimal performance even for large input sizes.
