class Solution:
    def countKConstraintSubstrings(self, s: str, k: int) -> int:
        n = len(s)
        # Precompute prefix0: prefix0[i] is the count of '0' from 0 to i-1
        prefix0 = [0]*(n+1)
        for i in range(1, n+1):
            prefix0[i] = prefix0[i-1] + (1 if s[i-1]=='0' else 0)
        
        count = 0
        # Iterate over all starting indices
        for l in range(n):
            # For each starting index, iterate over ending indices
            for r in range(l, n):
                length = r - l + 1
                zeros = prefix0[r+1] - prefix0[l]
                ones = length - zeros
                if zeros <= k or ones <= k:
                    count += 1
        return count

To solve this problem, we need to count the number of substrings in a given binary string that satisfy the k-constraint. The k-constraint is defined as either the number of 0's in the substring being at most k or the number of 1's in the substring being at most k.

### Approach
1. **Problem Analysis**: The problem requires checking all possible substrings of the given binary string to determine if they meet the k-constraint. A substring is valid if it contains at most k 0's or at most k 1's. Given the constraints (string length up to 50), a brute-force approach is feasible where we check every possible substring.
2. **Intuition**: For each substring, we can efficiently compute the number of 0's and 1's using a prefix sum array. The prefix sum array helps in quickly calculating the count of 0's (or 1's) in any substring in constant time.
3. **Algorithm Selection**: We use a nested loop to iterate over all possible substrings. The outer loop fixes the starting index, and the inner loop varies the ending index. For each substring, we use the prefix sum array to compute the number of 0's. The number of 1's is derived from the substring length minus the number of 0's. The substring is valid if either count is within the allowed limit (≤ k).
4. **Complexity Analysis**: The algorithm involves O(n^2) operations due to the nested loops, where n is the length of the string. Each iteration of the inner loop performs O(1) operations using the prefix sum array. For n=50, this results in 1275 operations, which is efficient for the given constraints.

### Solution Code
```python
class Solution:
    def countKConstraintSubstrings(self, s: str, k: int) -> int:
        n = len(s)
        prefix0 = [0] * (n + 1)
        for i in range(1, n + 1):
            prefix0[i] = prefix0[i - 1] + (1 if s[i - 1] == '0' else 0)
        
        count = 0
        for l in range(n):
            for r in range(l, n):
                length = r - l + 1
                zeros = prefix0[r + 1] - prefix0[l]
                ones = length - zeros
                if zeros <= k or ones <= k:
                    count += 1
        return count
```

### Explanation
1. **Prefix Sum Array**: The `prefix0` array is constructed such that `prefix0[i]` contains the count of '0's from the start of the string up to (but not including) the i-th character. This allows O(1) range queries for the count of '0's in any substring.
2. **Nested Loop**: The outer loop iterates over each starting index `l` of the substring. The inner loop extends the substring from `l` to each possible ending index `r`.
3. **Substring Validation**: For each substring defined by `l` and `r`, the number of '0's is found using the prefix sum array. The number of '1's is the substring length minus the count of '0's. If either count is ≤ k, the substring is counted as valid.
4. **Result**: The total count of valid substrings is returned after processing all possible substrings.

This approach efficiently checks all substrings while leveraging the prefix sum array for quick count calculations, ensuring optimal performance for the given problem constraints.
