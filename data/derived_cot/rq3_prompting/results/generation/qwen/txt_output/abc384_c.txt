Takahashi decided to hold a programming contest.
The contest consists of five problems: A, B, C, D, E, with scores a, b, c, d, e, respectively.
There are 31 participants, and all of them solved at least one problem.
More specifically, for every non-empty subsequence (not necessarily contiguous) of the string ABCDE, there is a participant named after that subsequence who solved the problems corresponding to the letters in their name and did not solve the other problems.
For example, participant A solved only problem A, and participant BCE solved problems B, C, and E.
Print the names of the participants in order of their obtained scores, from the largest to the smallest.
The score obtained by a participant is the sum of the scores of the problems they solved.
If two participants obtained the same score, print the one whose name is lexicographically smaller first.
What does "lexicographically smaller" mean?
In short, "lexicographically smaller" refers to the order in which words would appear in a dictionary.
More precisely, for distinct strings S,T consisting of uppercase English letters, S is lexicographically smaller than T if either of the following conditions holds:

- The length |S| of S is less than the length of T, and the first |S| characters of T match S.
- There exists an integer 1\leq i\leq\min\{ |S|,|T|\} that satisfy both of the following two conditions:
- For every integer j with 1\leq j\lt i, the j-th character of S equals the j-th character of T.
- The i-th character of S is alphabetically smaller than the i-th character of T.


For example, if S= AB and T= ABC, the first condition holds, so S is lexicographically smaller than T.
If S= ABD and T= ACD, the second condition holds for i=2, so S is lexicographically smaller than T.

To solve this problem, we need to generate all possible non-empty subsequences of the string "ABCDE", calculate their corresponding scores based on given problem scores, and then sort these subsequences in descending order of their scores. If two subsequences have the same score, they should be sorted lexicographically in ascending order.

### Approach
1. **Problem Analysis**: The problem requires us to consider all non-empty subsequences of the string "ABCDE", where each subsequence represents a participant who solved the corresponding problems. The score for each participant is the sum of the scores of the problems they solved. The solution involves generating these subsequences, calculating their scores, and then sorting them appropriately.

2. **Intuition**: Each subsequence can be uniquely identified by a bitmask of 5 bits (since there are 5 problems). Each bit in the mask corresponds to a problem (A to E). For each bitmask from 1 to 31 (inclusive), we can determine which problems are included by checking the set bits. The score is computed by summing the scores of the included problems.

3. **Algorithm Selection**: We use bitmasking to generate all possible non-empty subsequences. For each bitmask, we construct the subsequence string and compute its score. We then sort the subsequences primarily by their scores in descending order and secondarily by their lexicographical order in ascending order.

4. **Complexity Analysis**: There are 31 possible non-empty subsequences (2^5 - 1). For each subsequence, we perform a constant amount of work (checking 5 bits and summing 5 numbers). Thus, the time complexity is O(31 * 5) = O(155), which is efficient for the given constraints.

### Solution Code
```python
def main():
    data = input().split()
    a, b, c, d, e = map(int, data)
    scores = [a, b, c, d, e]
    letters = ['A', 'B', 'C', 'D', 'E']
    participants = []
    
    for bitmask in range(1, 32):
        s = ""
        total = 0
        for i in range(5):
            if bitmask & (1 << i):
                s += letters[i]
                total += scores[i]
        participants.append((total, s))
    
    participants.sort(key=lambda x: (-x[0], x[1]))
    
    for score, name in participants:
        print(name)

if __name__ == "__main__":
    main()
```

1. **Reading Input**: The input consists of five integers representing the scores for problems A, B, C, D, and E.
2. **Generating Subsequences**: Using bitmasking, we generate all non-empty subsequences of "ABCDE". Each bitmask from 1 to 31 corresponds to a unique subsequence. For each bitmask, we check which bits are set to determine the included problems and compute the total score.
3. **Sorting Participants**: The participants are sorted primarily by their scores in descending order. For participants with the same score, their names are sorted lexicographically in ascending order.
4. **Output**: The sorted list of participant names is printed, each on a new line, from highest to lowest score, with lexicographical order breaking ties.

This approach efficiently handles the problem constraints and ensures the correct ordering of participants based on their scores and names.
