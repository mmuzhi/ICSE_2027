To solve this problem, we need to determine whether it's possible to transform a given integer sequence into a non-decreasing sequence by performing any number of operations where each operation involves selecting two indices \(i\) and \(j\) (with \(i < j\)) and incrementing \(A_i\) by 1 while decrementing \(A_j\) by 1. 

### Approach
1. **Problem Analysis**: The key observation is that the operations allow us to transfer values from right to left (i.e., from higher indices to lower indices). The total sum of the sequence remains invariant during these operations. For the sequence to be non-decreasing, each element must be less than or equal to the next element. 

2. **Insight**: The solution hinges on checking if the sequence can be rearranged into a non-decreasing form by only moving values from right to left. This is possible if and only if the sequence can be transformed such that each element is not greater than the sum of all elements to its right. 

3. **Algorithm Selection**: 
   - **Prefix Sum Check**: For each element in the sequence, ensure that the sum of the elements to its right is at least the difference between the current element and the target value in the non-decreasing sequence. However, a more efficient approach is to verify that the sequence can be adjusted by only right-to-left transfers to meet the non-decreasing condition.

4. **Complexity Analysis**: The solution involves a single pass through the sequence for each test case, making the time complexity \(O(N)\) per test case, where \(N\) is the length of the sequence. Given the constraints, this approach is efficient.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    t = int(data[0])
    index = 1
    results = []
    for _ in range(t):
        n = int(data[index]); index += 1
        A = list(map(int, data[index:index+n]))
        index += n
        
        # Check if the sequence can be made non-decreasing
        # The condition is that for each i, the sum of the remaining elements (from i+1 to end) 
        # must be at least the amount needed to make the sequence non-decreasing from the left.
        s = sum(A)
        prefix = 0
        possible = True
        for i in range(n):
            prefix += A[i]
            # The remaining sum is s - prefix
            # In a non-decreasing sequence, the element at i must be <= the next elements.
            # However, a simpler condition is to check if the current element is greater than the total of the remaining elements.
            if A[i] > s - prefix + i * A[i]:
                possible = False
                break
        results.append("Yes" if possible else "No")
    
    for res in results:
        print(res)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input is read in one go for efficiency, especially given the constraints.
2. **Processing Each Test Case**: For each test case, the sequence length and the sequence itself are read.
3. **Prefix Sum Check**: For each element in the sequence, we maintain a running sum (prefix).
