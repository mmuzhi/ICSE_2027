### Approach
1. **Problem Analysis**: The problem requires checking if the string can be fully reduced to an empty string by deleting any valid contiguous substring at any point. This is analogous to checking if the string is a balanced parentheses sequence but with multiple types of brackets.
2. **Intuition**: The solution involves using a stack to keep track of opening brackets. As we iterate through the string, whenever we encounter a closing bracket, we check if it matches the most recent unmatched opening bracket. If it does, we remove both; otherwise, the string is invalid.
3. **Algorithm Selection**: The stack-based approach is chosen because it efficiently handles nested and interleaved brackets by ensuring that each closing bracket correctly matches the most recent unmatched opening bracket.
4. **Complexity Analysis**: The algorithm processes each character in the string exactly once, leading to a time complexity of O(n), where n is the length of the string. The space complexity is O(n) in the worst case (when all characters are opening brackets), which is efficient for the given constraints.

def main():
    S = input().strip()
    stack = []
    match = {'(': ')', '[': ']', '<': '>'}
    for char in S:
        if char in '([{':
            stack.append(char)
        else:
            if not stack:
                print('No')

            top = stack.pop()
            if top != match[char]:

1. **Initialization**: The input string is read, and an empty stack is initialized to keep track of opening brackets.
2. **Iteration**: For each character in the string:

   - If the character is a closing bracket (')', ']', or '>'), the top of the stack is popped. If the popped opening bracket does not match the corresponding closing bracket (checked using a predefined dictionary), the string is immediately deemed invalid.
3. **Final Check**: After processing all characters, if the stack is empty, the string is valid; otherwise, it contains unmatched opening brackets and is invalid.
