 What is lexicographical order for sequences?
A sequence A = (A_1, \ldots, A_{|A|}) is lexicographically smaller than B = (B_1, \ldots, B_{|B|}) if either 1. or 2. below holds:

- |A|<|B| and (A_{1},\ldots,A_{|A|}) = (B_1,\ldots,B_{|A|}).
- There exists an integer 1\leq i\leq \min\{|A|,|B|\} such that both of the following are true:

- (A_{1},\ldots,A_{i-1}) = (B_1,\ldots,B_{i-1})
- A_i < B_i

Given the constraints (N ≤ 8 and each R_i ≤ 5), a brute-force approach is feasible because the maximum number of sequences to check is 5^8 = 390,625, which is manageable in Python.

2. **Intuition**: The solution involves generating all possible sequences using the Cartesian product of ranges [1, R_i] for each position i. For each generated sequence, we check if the sum of its elements is divisible by K. If it is, the sequence is printed in lexicographical order.

3. **Algorithm Selection**: The Cartesian product from the `itertools` library is used to efficiently generate all sequences in lexicographical order. This avoids the need for nested loops and simplifies the code. Each sequence is checked for the sum condition, and valid sequences are printed immediately.

4. **Complexity Analysis**: The time complexity is O(P * N), where P is the product of all R_i (up to 390,625) and N is the sequence length (up to 8). The space complexity is O(P * N) to store all sequences, but given the constraints, this is acceptable.

### Solution Code
```python
import sys
from itertools import product

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    k = int(data[1])
    R = list(map(int, data[2:2+n]))
    
    ranges = [range(1, r+1) for r in R]
    for seq in product(*ranges):
        total = sum(seq)
        if total % k == 0:
            print(" ".join(map(str, seq)))

if __name__ == "__main__":
    main()
```

3. **Checking Sum Condition**: For each generated sequence, the sum of its elements is computed. If this sum is divisible by K, the sequence is printed.
4. **Output**: Valid sequences are printed in lexicographical order, one per line, without any additional formatting or separators beyond the required spaces.
