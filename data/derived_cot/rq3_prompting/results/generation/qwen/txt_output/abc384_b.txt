AtCoder Regular Contest (ARC) is divided into two divisions.

- In ARC Div. 1, participants whose rating at the start of the contest is between 1600 and 2799, inclusive, are subject to rating updates.

=== ANSWER ===

To solve this problem, we need to determine Takahashi's rating after participating in N ARCs, considering the conditions under which his rating is updated in each contest. The key is to check the current rating before each contest to decide if an update is applicable, and then adjust the rating accordingly.

### Approach
1. **Initial Setup**: Start with the initial rating R.
2. **Iterate Through Each Contest**: For each contest, check the division (D_i) and the performance change (A_i).
3. **Update Condition Check**:
   - For Division 1 (D_i = 1), if the current rating is between 1600 and 2799 (inclusive), update the rating by adding A_i.
   - For Division 2 (D_i = 2), if the current rating is between 1200 and 2399 (inclusive), update the rating by adding A_i.
4. **Output the Final Rating**: After processing all contests, the final rating is the result.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().split()
    n = int(data[0])
    initial_rating = int(data[1])
    index = 2
    current_rating = initial_rating
    for _ in range(n):
        d = int(data[index])
        a = int(data[index + 1])
        index += 2
        if d == 1:
            if 1600 <= current_rating <= 2799:
                current_rating += a
        else:
            if 1200 <= current_rating <= 2399:
                current_rating += a
    print(current_rating)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first two values are the number of contests (N) and the initial rating (R).
2. **Processing Each Contest**: For each contest, the division (D_i) and performance change (A_i) are read. The current rating is checked against the division's rating update conditions.
3. **Rating Update**: If the current rating meets the conditions for the respective division, the performance change (A_i) is added to the current rating. Otherwise, the rating remains unchanged.
4. **Output**: After processing all contests, the final rating is printed.

This approach efficiently processes each contest in sequence, updating the rating only when necessary, and ensures the final rating reflects all applicable updates from the contests. The solution handles edge cases, such as ratings outside the update ranges, seamlessly.
