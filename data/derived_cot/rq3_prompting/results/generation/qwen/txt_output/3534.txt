To solve this problem, we need to count the number of pairs of indices (i, j) in an array where i < j and the elements at these indices are almost equal. Two integers are considered almost equal if one can be transformed into the other by performing at most one swap of any two digits in one of the integers.

### Approach
1. **Problem Analysis**: The key observation is that for two integers to be almost equal, one must be transformable into the other by swapping any two digits (or no swap if they are already equal). The solution involves checking each pair of integers in the array to see if this condition holds. Given the constraints (array length up to 100), a brute-force approach for each pair would be feasible but can be optimized by precomputing possible transformations for each unique number in the array.

2. **Intuition**: For each number in the array, we generate all possible numbers that can be formed by swapping any two digits (including the original number). This set of numbers represents all possible transformations of the original number with at most one swap. Then, for each number in the array, we check against all previously encountered numbers to see if either the current number can be transformed into a previously seen number or vice versa.

3. **Algorithm Selection**: 
   - **Precompute Transformations**: For each unique number in the array, compute all numbers that can be formed by swapping any two digits (including the original number). This is done by converting the number to a string, generating all possible digit swaps, and converting the resulting strings back to integers.
   - **Count Valid Pairs**: As we iterate through the array, for each number, we check all previously seen numbers. If either the current number can be transformed into a previously seen number or the previously seen number can be transformed into the current number, we increment our count by the frequency of the previously seen number.

4. **Complexity Analysis**: 
   - **Precomputation**: For each unique number (up to 100 numbers) with up to 7 digits, generating all digit swaps takes O(n^2) time per number, where n is the number of digits (at most 7). This results in a total of O(100 * 49) = 4900 operations, which is efficient.
   - **Main Loop**: For each of the 100 elements, we check against up to 100 previously seen numbers, each requiring two set lookups (O(1) each). This results in 100 * 100 * 2 = 20,000 operations, which is also efficient.

### Solution Code
```python
class Solution:
    def compute_T_set(self, x):
        s = str(x)
        T_set = set()
        T_set.add(x)
        n = len(s)
        for i in range(n):
            for j in range(i+1, n):
                lst = list(s)
                lst[i], lst[j] = lst[j], lst[i]
                new_s = ''.join(lst)
                new_num = int(new_s)
                T_set.add(new_num)
        return T_set

    def countPairs(self, nums: List[int]) -> int:
        distinct_nums = set(nums)
        T_dict = {}
        for num in distinct_nums:
            T_dict[num] = self.compute_T_set(num)
        
        seen = {}
        ans = 0
        for x in nums:
            for y, count_y in seen.items():
                if y in T_dict[x] or x in T_dict[y]:
                    ans += count_y
            seen[x] = seen.get(x, 0) + 1
        return ans
```

### Explanation
1. **compute_T_set Method**: This helper function takes an integer `x`, converts it to a string, and generates all possible integers that can be formed by swapping any two digits in the string representation of `x`. The original number is included in the set along with all transformed numbers.
2. **countPairs Method**: 
   - **Precomputation**: The method first computes the transformation set for each unique number in the input array using `compute_T_set`.
   - **Counting Pairs**: As it processes each number in the array, it checks all previously encountered numbers. For each previously seen number, it verifies if either the current number can be transformed into it or vice versa using the precomputed transformation sets. If the condition is met, it increments the answer by the count of the previously seen number.
   - **Tracking Seen Numbers**: The method maintains a dictionary `seen` to keep track of how many times each number has been encountered so far in the array.

This approach efficiently checks all possible pairs by leveraging precomputation and set operations, ensuring optimal performance even for the upper constraint limits.
