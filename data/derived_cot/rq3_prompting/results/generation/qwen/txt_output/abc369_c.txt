To solve this problem, we need to count the number of contiguous subsequences in an array that form an arithmetic progression. An arithmetic progression is a sequence where the difference between consecutive elements is constant. The solution involves efficiently identifying all such subsequences by leveraging dynamic programming and a precomputed array to track the next breaking point for each starting index.

### Approach
1. **Problem Analysis**: The task is to count all contiguous subsequences (arithmetic progressions) in an array. A subsequence is an arithmetic progression if the difference between consecutive elements is constant. The challenge is to do this efficiently for large arrays (up to 200,000 elements).

2. **Intuition**: For each starting index `l`, the longest contiguous arithmetic progression can be determined by checking how far the sequence can extend while maintaining a constant difference between consecutive elements. The common difference for the progression starting at `l` is `A[l+1] - A[l]` (if `l+1` exists). The breaking point is the first index where this difference changes.

3. **Algorithm Selection**: 
   - **Precompute Differences**: Calculate the difference between consecutive elements for each position in the array.
   - **Next Change Array**: For each position `i`, compute the next position where the difference changes. This helps in quickly determining the end of the longest arithmetic progression starting at any index.
   - **Count Progressions**: For each starting index `l`, use the precomputed next change array to determine the longest valid subsequence starting at `l` and sum the counts of all valid subsequences.

4. **Complexity Analysis**: The algorithm runs in O(N) time and space, where N is the number of elements in the array. This is efficient given the constraints, as it avoids the O(N^2) approach by leveraging precomputation and dynamic programming.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().split()
    if not data:
        print(0)
        return
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    if n == 0:
        print(0)
        return
    if n == 1:
        print(1)
        return
        
    diff = [0] * n
    for i in range(1, n):
        diff[i] = A[i] - A[i-1]
        
    next_change = [n] * n
    next_change[n-1] = n
    for i in range(n-2, -1, -1):
        if diff[i] != diff[i+1]:
            next_change[i] = i+1
        else:
            next_change[i] = next_change[i+1]
            
    ans = 0
    for l in range(n):
        if l == n-1:
            ans += 1
        else:
            next_break = next_change[l+1]
            if next_break < l+2:
                next_break = n
            ans += (next_break - l)
            
    print(ans)

if __name__ == "__main__":
    main()
```

5. **Counting Progressions**: For each starting index, uses the next change array to find the longest valid subsequence and sums the counts of all valid subsequences.
6. **Output**: Prints the total count of arithmetic progressions in the array.

This approach efficiently counts all contiguous arithmetic progressions by leveraging dynamic programming and precomputation, ensuring optimal performance for large input sizes.
