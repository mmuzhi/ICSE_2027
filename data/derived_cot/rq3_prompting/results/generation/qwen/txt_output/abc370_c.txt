 What is lexicographical order on arrays of strings?
A string S = S_1 S_2 \ldots S_N of length N is lexicographically smaller than a string T = T_1 T_2 \ldots T_N of length N if there exists an integer 1 \leq i \leq N such that both of the following are satisfied:

-  S_1 S_2 \ldots S_{i-1} = T_1 T_2 \ldots T_{i-1}
-  S_i comes earlier than T_i in alphabetical order.

An array of strings X = (X_1,X_2,\ldots,X_M) with M elements is lexicographically smaller than an array of strings Y = (Y_1,Y_2,\ldots,Y_M) with M elements if there exists an integer 1 \leq j \leq M such that both of the following are satisfied:

-  (X_1,X_2,\ldots,X_{j-1}) = (Y_1,Y_2,\ldots,Y_{j-1})

=== ANSWER ===

To solve this problem, we need to transform string S into string T with the minimum number of operations, where each operation involves changing one character in S and appending the resulting string to an array X. If there are multiple sequences of operations with the same minimal length, we must choose the lexicographically smallest sequence.

### Approach
1. **Problem Analysis**: The key observation is that each differing character between S and T must be changed exactly once to reach T. The minimal number of operations is the number of differing characters. The challenge is to determine the order of these changes such that the sequence of intermediate strings is lexicographically smallest.
2. **Intuition**: The lexicographical order of the sequence of strings is determined by comparing each string in order. To ensure the smallest sequence, at each step, we should choose the character change that results in the lexicographically smallest string.
3. **Algorithm Selection**: We use a greedy approach. For each operation, we consider changing each differing character to its target value in T and select the change that produces the smallest string. This process repeats until all differing characters are fixed.
4. **Complexity Analysis**: The algorithm involves up to 100 operations (each differing character), and for each operation, we compare up to 100 strings of length 100. This results in a time complexity of O(n^2 * m), where n is the string length and m is the number of differing characters (both up to 100), making it efficient for the given constraints.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data:
        return
    S = data[0].strip()
    T = data[1].strip()
    n = len(S)
    
    if S == T:
        print(0)
        return
        
    D = set()
    for i in range(n):
        if S[i] != T[i]:
            D.add(i)
            
    if not D:
        print(0)
        return
        
    current = S
    steps = []
    fixed_indices = set()
    
    for _ in range(len(D)):
        best_candidate = None
        best_index = None
        for i in D:
            if i in fixed_indices:
                continue
            candidate_list = list(current)
            candidate_list[i] = T[i]
            candidate_str = ''.join(candidate_list)
            if best_candidate is None or candidate_str < best_candidate:
                best_candidate = candidate_str
                best_index = i
                
        steps.append(best_candidate)
        current = best_candidate
        fixed_indices.add(best_index)
        D.discard(best_index)
        
    print(len(steps))
    for s in steps:
        print(s)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Reading Input**: The input consists of two strings, S and T, which are read from standard input.
2. **Initial Check**: If S and T are already identical, the output is 0 since no operations are needed.
3. **Identifying Differing Characters**: We collect all indices where characters in S and T differ into a set D.
4. **Greedy Selection**: For each operation, we generate all possible strings by changing each differing character to its target value in T. The lexicographically smallest string is chosen for the next step.
5. **Output**: The number of operations (length of steps) is printed first, followed by each intermediate string in the sequence.

This approach efficiently ensures the minimal number of operations and the lexicographically smallest sequence by leveraging a greedy selection at each step.
