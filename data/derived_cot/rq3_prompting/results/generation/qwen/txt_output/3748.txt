To solve this problem, we need to sort an n x n matrix such that the bottom-left triangle (including the main diagonal) is sorted in non-increasing order, and the top-right triangle (excluding the main diagonal) is sorted in non-decreasing order. The solution involves extracting the relevant diagonals from the original matrix, sorting them appropriately, and then reassigning the sorted values back to their original positions.

### Approach
1. **Problem Analysis**: The matrix is divided into two parts:
   - **Bottom-left Triangle (i >= j)**: This includes the main diagonal and all elements below it. Each diagonal in this triangle is sorted in descending order.
   - **Top-right Triangle (j > i)**: This includes all elements strictly above the main diagonal. Each diagonal in this triangle is sorted in ascending order.

2. **Intuition**: For each diagonal in the bottom-left and top-right triangles, we extract the elements, sort them according to the specified order, and then place them back into their original positions. The main diagonal is part of the bottom-left triangle and is sorted in descending order.

3. **Algorithm Selection**:
   - **Copy the Original Matrix**: To avoid modifying the input matrix during extraction, we create a copy of the original matrix.
   - **Extract Diagonals**: For each diagonal in the bottom-left and top-right triangles, collect the elements from the original matrix.
   - **Sort Diagonals**: Sort the bottom-left diagonals in descending order and the top-right diagonals in ascending order.
   - **Reassign Sorted Values**: Place the sorted values back into the result matrix in their original positions.

4. **Complexity Analysis**: The algorithm processes each element of the matrix exactly once during extraction and once during reassignment. Sorting each diagonal takes O(m log m) time, where m is the length of the diagonal. Since there are O(n) diagonals, the total time complexity is O(n² log n). The space complexity is O(n²) due to the storage of the original and result matrices.

### Solution Code
```python
from typing import List

class Solution:
    def sortMatrix(self, grid: List[List[int]]) -> List[List[int]]:
        n = len(grid)
        if n == 0:
            return grid
        
        original = [row[:] for row in grid]
        result = [row[:] for row in grid]
        
        for d in range(0, n):
            diagonal = []
            for i in range(d, n):
                j = i - d
                diagonal.append(original[i][j])
            diagonal_sorted = sorted(diagonal, reverse=True)
            for k in range(len(diagonal_sorted)):
                i_index = d + k
                j_index = k
                result[i_index][j_index] = diagonal_sorted[k]
        
        for d in range(1, n):
            diagonal = []
            for i in range(0, n - d):
                j = i + d
                diagonal.append(original[i][j])
            diagonal_sorted = sorted(diagonal)
            for k in range(len(diagonal_sorted)):
                i_index = k
                j_index = d + k
                result[i_index][j_index] = diagonal_sorted[k]
        
        return result
```

### Explanation
1. **Initialization**: The solution starts by checking if the matrix is empty. If not, it creates copies of the original matrix for extraction and result storage.
2. **Bottom-left Triangle Processing**: For each diagonal in the bottom-left triangle (determined by the difference i - j = d), the elements are extracted from the original matrix. These elements are sorted in descending order and then placed back into the result matrix at their original positions.
3. **Top-right Triangle Processing**: Similarly, for each diagonal in the top-right triangle (determined by j - i = d), the elements are extracted, sorted in ascending order, and then placed back into the result matrix.
4. **Result Return**: The modified result matrix, now with the bottom-left and top-right triangles sorted as required, is returned.

This approach efficiently processes the matrix by leveraging diagonal extraction and sorting, ensuring the solution meets the problem's requirements.
