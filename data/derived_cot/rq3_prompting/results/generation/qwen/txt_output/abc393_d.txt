To solve this problem, we need to find the minimum number of adjacent swaps required to make all the 1s in a binary string contiguous. The key observation here is that the optimal target segment for the contiguous 1s is determined by the median of certain transformed positions of the original 1s. 

### Approach
1. **Problem Analysis**: The problem involves transforming a binary string such that all 1s are grouped together with the least number of adjacent swaps. The solution leverages the fact that the minimal number of swaps is equivalent to the sum of the distances each 1 needs to move to reach its target position in the contiguous segment.
2. **Insight**: The target segment for the contiguous 1s can be determined by considering the median of the transformed positions of the 1s. The transformed positions are calculated by subtracting the index of each 1 from its original position. This transformation helps in identifying the optimal starting position for the contiguous segment.
3. **Algorithm Selection**: The algorithm involves:
   - Counting the number of 1s in the string.
   - Recording the positions of all 1s.
   - Transforming these positions by subtracting their indices.
   - Using the median of these transformed positions to determine the starting index of the target segment.
   - Calculating the total number of swaps required by summing the absolute differences between each original 1's position and its target position in the segment.
4. **Complexity Analysis**: The algorithm processes the string in linear time, making it efficient with a time complexity of O(N), where N is the length of the string. This is optimal for the given constraints.

### Solution Code
```python
def main():
    import sys
    data = sys.stdin.read().splitlines()
    if not data: 
        return
    N = int(data[0].strip())
    S = data[1].strip()
    
    k = S.count('1')
    if k == 0:
        print(0)
        return
        
    positions = [i for i, char in enumerate(S) if char == '1']
    
    q = [positions[i] - i for i in range(len(positions))]
    
    candidate = q[len(q) // 2]
    
    low_bound = 0
    high_bound = N - k
    candidate = max(low_bound, min(candidate, high_bound))
    
    total_swaps = 0
    for i in range(len(positions)):
        target = candidate + i
        total_swaps += abs(positions[i] - target)
        
    print(total_swaps)

if __name__ == '__main__':
    main()
```

5. **Finding Median**: The median of the transformed positions is used to determine the starting index of the target segment. The median is chosen because it minimizes the total distance the 1s need to move.
6. **Clamping the Median**: The median is clamped between 0 and the maximum valid starting index for the target segment (N - k) to ensure it lies within the bounds of the string.
7. **Calculating Swaps**: The total number of swaps is calculated by summing the absolute differences between each original 1's position and its target position in the segment.
8. **Output Result**: The total number of swaps is printed as the result.

This approach efficiently minimizes the number of adjacent swaps required to group all 1s in the string by leveraging median-based positioning and distance calculation.
