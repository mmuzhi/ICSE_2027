=== ANSWER ===

To solve this problem, we need to determine whether a given sequence of integers forms a geometric progression. A geometric progression is a sequence where each term after the first is found by multiplying the previous term by a constant, known as the common ratio. 

### Approach
1. **Problem Analysis**: The key observation here is that for a sequence to be a geometric progression, the ratio between consecutive terms must be consistent throughout the sequence. Specifically, the ratio of the second term to the first term should be the same as the ratio of the third term to the second term, and so on.
2. **Intuition**: The solution involves checking each consecutive pair of terms in the sequence to ensure that the ratio between them is the same as the ratio between the first two terms. This can be efficiently done using cross-multiplication to avoid floating-point operations and ensure precision.
3. **Algorithm Selection**: The algorithm iterates through each consecutive pair of terms in the sequence. For each pair, it checks if the product of the next term and the first term of the sequence equals the product of the current term and the second term of the sequence. If this condition fails for any pair, the sequence is not a geometric progression.
4. **Complexity Analysis**: The algorithm runs in O(N) time, where N is the number of terms in the sequence. This is because each term is checked exactly once. The space complexity is O(1) as no additional data structures are used beyond the input list.

### Solution Code
```python
def main():
    n = int(input().strip())
    A = list(map(int, input().split()))
    
    for i in range(n - 1):
        if A[i + 1] * A[0] != A[i] * A[1]:
            print("No")
            return
            
    print("Yes")

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The code starts by reading the number of terms, `n`, and the sequence of integers, `A`.
2. **Checking Geometric Progression**: The loop iterates through each consecutive pair of terms in the sequence. For each pair, it checks if the product of the next term and the first term (`A[i+1] * A[0]`) equals the product of the current term and the second term (`A[i] * A[1]`). If this condition fails for any pair, the sequence is not a geometric progression, and the program prints "No" and exits.
3. **Output Result**: If all consecutive pairs satisfy the condition, the program prints "Yes", confirming that the sequence is a geometric progression.

This approach efficiently verifies the geometric progression property using simple arithmetic checks and handles all edge cases within the given constraints.
