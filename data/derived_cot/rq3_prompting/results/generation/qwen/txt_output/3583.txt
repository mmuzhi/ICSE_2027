To solve this problem, we need to efficiently compute the sorted list of all possible GCD values of pairs in a given array and answer multiple queries about specific elements in this sorted list without explicitly storing the entire list due to its potentially enormous size.

### Approach
1. **Problem Analysis**: The problem requires calculating the GCD for all possible pairs of elements in an array, sorting these GCD values, and then answering queries about specific elements in the sorted list. Given the constraints (array size up to 10^5), a brute-force approach is infeasible due to high computational complexity (O(n^2)).

2. **Key Insight**: The GCD of any two numbers in the array must be a divisor of at least one number in the array. We can leverage number theory concepts, specifically the Möbius inversion formula, to count the number of pairs with GCD exactly equal to a given value.

3. **Algorithm Selection**:
   - **Frequency Array**: First, we create a frequency array to count occurrences of each number in the input array.
   - **Divisor Counting**: For each number `d` from 1 to a maximum value (50,000), count how many numbers in the array are divisible by `d`.
   - **Pair Counting**: For each `d`, compute the number of pairs where the GCD is divisible by `d` using combinations of the count of divisible numbers.
   - **Möbius Function**: Use a sieve method to precompute the Möbius function values, which helps in applying inclusion-exclusion principles to count pairs with GCD exactly `d`.
   - **Binary Search for Queries**: For each query, use binary search on a cumulative frequency array to find the smallest GCD value whose cumulative count exceeds the query index.

4. **Complexity Analysis**:
   - **Time Complexity**: The approach involves O(max_val log max_val) for sieve methods and O(max_val log max_val) for processing pairs, where max_val is 50,000. This is feasible within the problem constraints.
   - **Space Complexity**: The solution uses O(max_val) space for various arrays (frequency, divisor counts, Möbius function, etc.), which is manageable given the constraints.

### Solution Code
```python
import sys

class Solution:
    def gcdValues(self, nums: List[int], queries: List[int]) -> List[int]:
        max_val = 50000
        
        freq = [0] * (max_val + 1)
        for num in nums:
            if num <= max_val:
                freq[num] += 1
        
        cnt = [0] * (max_val + 1)
        for d in range(1, max_val + 1):
            total = 0
            for j in range(d, max_val + 1, d):
                total += freq[j]
            cnt[d] = total
        
        H = [0] * (max_val + 1)
        for d in range(1, max_val + 1):
            if cnt[d] < 2:
                H[d] = 0
            else:
                H[d] = cnt[d] * (cnt[d] - 1) // 2
        
        mu = [1] * (max_val + 1)
        is_prime = [True] * (max_val + 1)
        primes = []
        for i in range(2, max_val + 1):
            if is_prime[i]:
                primes.append(i)
                mu[i] = -1
            for p in primes:
                if i * p > max_val:
                    break
                is_prime[i * p] = False
                if i % p == 0:
                    mu[i * p] = 0
                    break
                else:
                    mu[i * p] = -mu[i]
        
        g = [0] * (max_val + 1)
        for d in range(1, max_val + 1):
            total_g = 0
            for k in range(1, max_val // d + 1):
                d_val = d * k
                if d_val > max_val:
                    break
                total_g += mu[k] * H[d_val]
            g[d] = total_g
        
        cum = [0] * (max_val + 1)
        for d in range(1, max_val + 1):
            cum[d] = cum[d - 1] + g[d]
        
        res = []
        for k in queries:
            low, high = 1, max_val
            while low < high:
                mid = (low + high) // 2
                if cum[mid] > k:
                    high = mid
                else:
                    low = mid + 1
            res.append(low)
        
        return res
```

### Explanation
1. **Frequency Array**: We start by counting how many times each number appears in the input array.
2. **Divisor Counting**: For each number `d` from 1 to 50,000, we count how many numbers in the array are divisible by `d`. This helps in determining how many pairs can potentially have a GCD divisible by `d`.
3. **Pair Counting**: Using the divisor counts, we compute the number of pairs where the GCD is divisible by each `d` using combinations (H array).
4. **Möbius Function**: This function helps in applying the inclusion-exclusion principle to count pairs with GCD exactly `d` by summing contributions from all multiples of `d`.
5. **Cumulative Frequency**: We build a cumulative array where each entry at index `d` holds the total number of pairs with GCD less than or equal to `d`.
6. **Query Processing**: For each query, we use binary search on the cumulative array to find the smallest GCD value whose cumulative count exceeds the query index, providing the answer efficiently without storing the entire sorted list. 

This approach efficiently handles the problem constraints by leveraging mathematical insights and binary search, avoiding the need for brute-force computation and storage.
