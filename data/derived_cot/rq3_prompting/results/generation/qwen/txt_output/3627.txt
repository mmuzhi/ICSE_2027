To solve this problem, we need to find the minimum time required to reach the bottom-right corner of an n x m grid, starting from the top-left corner. The movement between adjacent rooms takes exactly one second, but each room has a specific time constraint that dictates the earliest time we can start moving into it. 

### Approach
1. **Problem Analysis**: The problem involves navigating a grid where each cell has a constraint on the earliest time you can start moving into it. The key insight is that the time to enter a cell depends on both the current time (when you leave the previous cell) and the cell's move time. Specifically, you can only start moving into a cell if the current time is at least the cell's move time. The arrival time at the next cell is then the maximum of the current time and the next cell's move time, plus one second for the movement.

2. **Algorithm Selection**: We use Dijkstra's algorithm with a priority queue (min-heap) to explore the grid efficiently. The priority queue helps in always expanding the least time-consuming path first. For each cell, we consider its four adjacent cells and compute the earliest possible arrival time based on the current cell's arrival time and the adjacent cell's move time.

3. **Complexity Analysis**: The algorithm processes each cell in the grid, and for each cell, it checks up to four neighbors. The priority queue operations (insertions and extractions) take logarithmic time relative to the number of elements. Given the grid dimensions (up to 50x50), the total operations are manageable, making the solution efficient.

### Solution Code
```python
import heapq
from typing import List

class Solution:
    def minTimeToReach(self, moveTime: List[List[int]]) -> int:
        n = len(moveTime)
        m = len(moveTime[0])
        dp = [[float('inf')] * m for _ in range(n)]
        dp[0][0] = 0
        heap = [(0, 0, 0)]
        
        while heap:
            time, i, j = heapq.heappop(heap)
            if time != dp[i][j]:
                continue
            if i == n-1 and j == m-1:
                return time
            for di, dj in [(1,0), (-1,0), (0,1), (0,-1)]:
                ni, nj = i + di, j + dj
                if 0 <= ni < n and 0 <= nj < m:
                    next_arrival = max(time, moveTime[ni][nj]) + 1
                    if next_arrival < dp[ni][nj]:
                        dp[ni][nj] = next_arrival
                        heapq.heappush(heap, (next_arrival, ni, nj))
        return dp[n-1][m-1]
```

### Explanation
1. **Initialization**: We start by initializing a distance matrix `dp` with infinity, except for the starting cell (0, 0), which is set to 0. This matrix keeps track of the earliest arrival time for each cell.
2. **Priority Queue**: A min-heap is used to prioritize cells based on their arrival times. The heap starts with the initial cell (0, 0) and time 0.
3. **Processing Cells**: For each cell popped from the heap, we check if it is the target cell (n-1, m-1). If so, we return the current time. Otherwise, we explore its adjacent cells.
4. **Next Cell Calculation**: For each adjacent cell, the next arrival time is calculated as the maximum of the current cell's arrival time and the adjacent cell's move time, plus one second. If this time is better (smaller) than the previously recorded time for the adjacent cell, we update it and push it into the heap.
5. **Termination**: The loop continues until the target cell is reached, ensuring the shortest path is found due to the priority queue's properties. The solution efficiently handles the constraints and move time conditions to determine the minimal travel time.
