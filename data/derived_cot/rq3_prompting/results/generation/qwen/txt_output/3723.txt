To solve this problem, we need to identify elements in an array that are considered "good" based on their neighboring elements at specific distances. An element is good if it is strictly greater than the elements at positions `i - k` and `i + k` (if these positions exist). The solution involves iterating through each element in the array and checking the specified conditions.

### Approach
1. **Problem Analysis**: The task is to determine which elements in the array meet the criteria of being strictly greater than their neighbors at distance `k` on both sides (if those neighbors exist). The solution requires checking each element in the array against these conditions.
2. **Intuition**: For each element at index `i`, we check if the element at `i - k` (left neighbor) is less than the current element, and similarly for the element at `i + k` (right neighbor). If both neighbors exist, the current element must be greater than both; if only one neighbor exists, it must be greater than that neighbor.
3. **Algorithm Selection**: We iterate through each element in the array. For each element, we check the left and right neighbors (if they exist) using conditional checks. If both conditions are satisfied (either both neighbors exist and the element is greater than both, or only one neighbor exists and the element is greater than it), we include the element's value in the sum.
4. **Complexity Analysis**: The algorithm processes each element in the array exactly once, performing constant-time checks for each element. Thus, the time complexity is O(n), where n is the length of the array. This is efficient given the constraint that the array length is at most 100.

### Solution Code
```python
class Solution:
    def sumOfGoodNumbers(self, nums: List[int], k: int) -> int:
        n = len(nums)
        total = 0
        for i in range(n):
            left_ok = (i - k < 0) or (nums[i] > nums[i - k])
            right_ok = (i + k >= n) or (nums[i] > nums[i + k])
            if left_ok and right_ok:
                total += nums[i]
        return total
```

### Explanation
1. **Initialization**: We start by determining the length of the array (`n`) and initializing a variable `total` to accumulate the sum of good elements.
2. **Iteration**: For each index `i` in the array, we check two conditions:
   - **Left Neighbor Check**: If the left neighbor (at `i - k`) does not exist (`i - k < 0`), the condition is automatically satisfied. Otherwise, the current element must be strictly greater than the left neighbor.
   - **Right Neighbor Check**: Similarly, if the right neighbor (at `i + k`) does not exist (`i + k >= n`), the condition is satisfied. Otherwise, the current element must be strictly greater than the right neighbor.
3. **Summation**: If both conditions are met, the current element's value is added to `total`.
4. **Result**: After processing all elements, `total` contains the sum of all good elements, which is returned as the result.

This approach efficiently checks each element's neighbors and accumulates the sum of those meeting the specified criteria, ensuring optimal performance and correctness.
