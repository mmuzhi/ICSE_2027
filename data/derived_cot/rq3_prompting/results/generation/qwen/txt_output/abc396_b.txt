=== ANSWER ===

To solve this problem, we need to simulate a stack of cards where each card is initially labeled with the integer 0. The stack undergoes a series of operations, either adding a new card on top or removing the top card and outputting its value. The challenge is to correctly handle these operations while maintaining the stack's state and producing the required outputs for each removal operation.

### Approach
1. **Initialization**: Start with a stack containing 100 zeros, as specified in the problem.
2. **Processing Queries**: For each query:
   - **Type 1 (Add operation)**: Append the given integer to the top of the stack.
   - **Type 2 (Remove operation)**: Pop the top card from the stack and print its value. The problem guarantees that there is always at least one card in the stack during a removal operation.
3. **Output Handling**: The outputs for all removal operations are collected and printed in the order they occur.

The solution leverages Python's list data structure for efficient stack operations (append and pop) and handles each query sequentially, ensuring correctness by adhering to the problem constraints.

### Solution Code
```python
def main():
    Q = int(input().strip())
    stack = [0] * 100
    
    for _ in range(Q):
        data = input().split()
        if data[0] == '1':
            x = int(data[1])
            stack.append(x)
        else:
            print(stack.pop())

if __name__ == "__main__":
    main()
```

### Explanation
1. **Initialization**: The stack is initialized with 100 zeros, representing the initial state of the problem.
2. **Query Processing**:
   - For each query of type 1, the specified integer is appended to the stack, effectively placing it on top.
   - For each query of type 2, the top element of the stack is removed and printed. The stack's size is maintained by the problem's guarantee that removal operations only occur when the stack is non-empty.
3. **Output**: The results of all type 2 queries are printed immediately as they occur, ensuring the output matches the required format.

This approach efficiently handles the stack operations using Python's built-in list methods, which provide constant-time complexity for append and pop operations, making the solution both optimal and straightforward.
