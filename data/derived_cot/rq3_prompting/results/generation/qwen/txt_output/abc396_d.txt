Notes on XOR

For non-negative integers A and B, their XOR A \oplus B is defined as follows:


- In the binary representation of A \oplus B, the digit in the place corresponding to 2^k \,(k \ge 0) is 1 if and only if exactly one of the digits in the same place of A and B is 1; otherwise, it is 0.


For example, 3 \oplus 5 = 6 (in binary: 011 \oplus 101 = 110).

In general, the XOR of k integers p_1, \dots, p_k is defined as (\cdots ((p_1 \oplus p_2) \oplus p_3) \oplus \cdots \oplus p_k). It can be proved that it does not depend on the order of p_1, \dots, p_k.

To solve this problem, we need to find the minimum XOR value of the labels of edges on any simple path from vertex 1 to vertex N in a given undirected connected graph. The XOR operation is commutative and associative, meaning the order of operations does not affect the result. The challenge lies in efficiently exploring all possible simple paths without revisiting any vertex, ensuring we find the path with the minimum XOR value.

### Approach
1. **Graph Representation**: Represent the graph using an adjacency list where each node points to its neighbors along with the edge labels.
2. **Depth-First Search (DFS)**: Use an iterative DFS approach to explore all simple paths from vertex 1 (node 0 in 0-based indexing) to vertex N (node N-1). This approach avoids recursion depth issues and efficiently handles the state space.
3. **State Tracking**: For each node in the DFS stack, keep track of the current node, a bitmask representing visited nodes, and the cumulative XOR value of the edges traversed so far.
4. **Early Termination**: Whenever the DFS reaches the target node (vertex N), compare the current XOR value with the minimum found so far and update the answer if a smaller value is found.
5. **Complexity Consideration**: Given the constraint N ≤ 10, the number of simple paths is manageable (up to 40320 for a complete graph), making this approach feasible.

### Solution Code
```python
import sys

def main():
    data = sys.stdin.read().strip().split()
    if not data:
        return
    
    it = iter(data)
    n = int(next(it)); m = int(next(it))
    
    graph = [[] for _ in range(n)]
    for _ in range(m):
        u = int(next(it)) - 1
        v = int(next(it)) - 1
        w = int(next(it))
        graph[u].append((v, w))
        graph[v].append((u, w))
    
    if n == 1:
        print(0)
        return
        
    ans = float('inf')
    stack = [(0, 1, 0)]
    
    while stack:
        node, mask, current_xor = stack.pop()
        if node == n-1:
            if current_xor < ans:
                ans = current_xor
            continue
        for neighbor, weight in graph[node]:
            if mask & (1 << neighbor):
                continue
            new_mask = mask | (1 << neighbor)
            new_xor = current_xor ^ weight
            stack.append((neighbor, new_mask, new_xor))
    
    print(ans)

if __name__ == "__main__":
    main()
```

1. **Graph Representation**: The graph is built using an adjacency list where each entry stores connected nodes and the corresponding edge weights.
2. **DFS Initialization**: The DFS starts from node 0 (vertex 1) with an initial mask (bitmask representing visited nodes) set to 1 (only node 0 is visited) and a cumulative XOR value of 0.
3. **Iterative DFS**: The stack processes each node by popping the top element, checking if it's the target node (vertex N). If it is, the current XOR value is compared with the minimum found so far.
4. **Neighbor Exploration**: For each neighbor of the current node, if the neighbor hasn't been visited (checked via the bitmask), the algorithm pushes the neighbor onto the stack with an updated mask and XOR value.
5. **Result Calculation**: After processing all paths, the minimum XOR value encountered during the DFS is printed as the result.

This approach efficiently explores all possible simple paths while leveraging bitmasking to track visited nodes, ensuring correctness and optimality within the problem constraints.
