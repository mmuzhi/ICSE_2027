To solve this problem, we need to determine whether it's possible to move all balls from N boxes into a single specified box (X-th box) using a series of operations, and if possible, find the minimum number of operations required. The operations involve selecting a box, moving all its red and blue balls to two different boxes based on given permutations, and emptying the selected box.

### Approach
1. **Problem Analysis**: The problem involves redistributing balls between boxes using operations defined by permutations. Each operation moves all red and blue balls from one box to two other boxes. The goal is to consolidate all balls into the X-th box, leaving all other boxes empty. The key insight is that balls can only be moved within cycles defined by the permutations. 

2. **Key Insight**: For it to be possible to move all balls to the X-th box, every box containing initial balls (not the X-th box) must be part of a cycle that includes the X-th box in both the red and blue permutation graphs. 

3. **Algorithm Selection**:
   - **Cycle Detection**: For both red and blue permutations, detect all cycles in the permutation graphs.
   - **Condition Check**: Ensure that for every box with initial balls (not the X-th box), its cycle includes the X-th box.
   - **Minimum Operations Calculation**: The minimum number of operations required is the number of boxes (excluding the X-th box) that will ever contain balls during the operations, which is derived from the cycles containing initial balls.

4. **Complexity Analysis**: The algorithm involves cycle detection in two permutation graphs, each of size N. The cycle detection is done in O(N) time for each graph. The condition check and set operations are also linear in N, making the overall complexity O(N), which is efficient for the given constraints (N ≤ 2×10^5).

```python
            S_red.add(i)
        if B[i] > 0:
            S_blue.add(i)

    for i in S_red:
        cid = cycle_id_R[i]
        if X0 not in cycles_R[cid]:
            print(-1)
            return

    for i in S_blue:
        cid = cycle_id_B[i]
        if X0 not in cycles_B[cid]:
            print(-1)
            return

    T = set()
    for cid in range(len(cycles_R)):
        cycle_nodes = cycles_R[cid]
        if any(node in S_red for node in cycle_nodes):
            for node in cycle_nodes:
                if node != X0:
                    T.add(node)

    for cid in range(len(cycles_B)):
        cycle_nodes = cycles_B[cid]
        if any(node in S_blue for node in cycle_nodes):
            for node in cycle_nodes:
                if node != X0:
                    T.add(node)

    print(len(T))

if __name__ == '__main__':
    main()
```

### Explanation
1. **Cycle Detection**: The `find_cycles` function processes each permutation (for red and blue balls) to identify all cycles. Each node's cycle membership is recorded for later checks.
2. **Initial Ball Check**: Sets `S_red` and `S_blue` are populated with indices of boxes (excluding the X-th box) that initially contain red or blue balls.
3. **Condition Verification**: For each box in `S_red` or `S_blue`, it checks if its cycle includes the X-th box. If not, it's impossible to move all balls to the X-th box, and the function returns -1.
4. **Operation Count Calculation**: The set `T` is built by including all boxes (excluding the X-th box) from cycles that contain initial balls. The size of `T` gives the minimum number of operations needed, as each box in `T` must be emptied once during the operations. 

This approach efficiently checks feasibility and computes the minimal operations by leveraging cycle properties in permutation graphs, ensuring optimal performance for large inputs.
