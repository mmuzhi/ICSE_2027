To solve this problem, we need to determine if there exists a cycle containing vertex 1 in a directed graph and, if such a cycle exists, find the minimum number of edges in it. The approach involves using BFS (Breadth-First Search) to explore the graph from vertex 1 both forward and backward, then combining the results to find the shortest cycle.

### Approach
1. **Graph Representation**: Represent the graph using adjacency lists for both the original graph and its reverse. The reverse graph is constructed by reversing the direction of each edge in the original graph.
2. **Forward BFS**: Perform BFS starting from vertex 1 in the original graph to compute the shortest distances from vertex 1 to all other vertices.
3. **Backward BFS**: Perform BFS starting from vertex 1 in the reverse graph to compute the shortest distances from all vertices back to vertex 1. This is equivalent to finding the shortest path from each vertex to vertex 1 in the original graph.
4. **Cycle Detection**: For each vertex reachable from vertex 1 in both directions, the sum of the forward and backward distances gives the length of a cycle passing through vertex 1 and that vertex. The minimum such sum across all vertices is the answer. If no such cycle exists, return -1.

### Solution Code
```python
import sys
from collections import deque

def main():
    data = sys.stdin.read().split()
    if not data:
        print(-1)
        return
        
    it = iter(data)
    n = int(next(it))
    m = int(next(it))
    
    graph = [[] for _ in range(n+1)]
    rev_graph = [[] for _ in range(n+1)]
    
    for _ in range(m):
        a = int(next(it))
        b = int(next(it))
        graph[a].append(b)
        rev_graph[b].append(a)
        
    dist_forward = [-1] * (n+1)
    dist_backward = [-1] * (n+1)
    
    q = deque([1])
    dist_forward[1] = 0
    while q:
        cur = q.popleft()
        for neighbor in graph[cur]:
            if dist_forward[neighbor] == -1:
                dist_forward[neighbor] = dist_forward[cur] + 1
                q.append(neighbor)
                
    q = deque([1])
    dist_backward[1] = 0
    while q:
        cur = q.popleft()
        for neighbor in rev_graph[cur]:
            if dist_backward[neighbor] == -1:
                dist_backward[neighbor] = dist_backward[cur] + 1
                q.append(neighbor)
                
    ans = float('inf')
    for i in range(2, n+1):
        if dist_forward[i] != -1 and dist_backward[i] != -1:
            total = dist_forward[i] + dist_backward[i]
            if total < ans:
                ans = total
                
    print(ans if ans != float('inf') else -1)

if __name__ == '__main__':
    main()
```

### Explanation
1. **Graph Representation**: The graph and its reverse are built from the input data. Each edge in the original graph is reversed in the reverse graph to facilitate backward BFS.
2. **Forward BFS**: Starting from vertex 1, BFS traverses the original graph, recording the shortest path distances to all reachable vertices.
3. **Backward BFS**: Similarly, BFS on the reverse graph starts from vertex 1, recording the shortest path distances from all reachable vertices back to vertex 1.
4. **Cycle Detection**: For each vertex (from 2 to N), if it is reachable in both directions, the sum of its forward and backward distances gives a cycle length. The minimum such length is the answer. If no vertices meet the criteria, the answer is -1.

This approach efficiently checks for cycles containing vertex 1 by leveraging BFS to explore both directions from the vertex, ensuring optimal performance even for large graphs.
