1. **Problem Analysis**: For each type of garbage, we are given two parameters: a modulus `q_i` and a remainder `r_i`. The garbage is collected on days `x` where `x % q_i == r_i`. Given a query specifying a garbage type and a day `d_j`, we need to find the smallest day `x >= d_j` such that `x % q_i == r_i`.

2. **Key Insight**: The solution involves calculating the next occurrence of the collection day after `d_j`. If `d_j` itself satisfies the condition (`d_j % q_i == r_i`), then the answer is `d_j`. Otherwise, we compute the smallest non-negative `k` such that `(d_j + k) % q_i == r_i`. This `k` can be derived using the formula `(r_i - (d_j % q_i)) % q_i`, which efficiently handles both cases where the next occurrence is within the same modulus cycle or the next cycle.

3. **Algorithm Selection**: The algorithm processes each query independently. For each query, it checks if the current day meets the collection condition. If not, it calculates the next valid day using modular arithmetic, ensuring optimal performance even for large values (up to 10^9).

4. **Complexity Analysis**: The algorithm processes each query in constant time, making the overall complexity O(Q) for Q queries. Given the constraints (Q ≤ 100), this approach is efficient and feasible.

### Solution Code
```python
n = int(input().strip())
q_list = []
r_list = []
for _ in range(n):
    data = input().split()
    q_list.append(int(data[0]))
    r_list.append(int(data[1]))

q = int(input().strip())
for _ in range(q):
    data = input().split()
    t = int(data[0])
    d = int(data[1])
    q_val = q_list[t-1]
    r_val = r_list[t-1]
    m = d % q_val
    if m == r_val:
        print(d)
    else:
        k = (r_val - m) % q_val
        print(d + k)
```

### Explanation
1. **Reading Input**: The code starts by reading the number of garbage types `N`. For each type, it reads the modulus `q_i` and remainder `r_i` values.
2. **Processing Queries**: For each query, it reads the garbage type `t_j` and day `d_j`. The corresponding modulus `q_i` and remainder `r_i` are retrieved from the pre-stored lists.
3. **Checking Condition**: The code checks if `d_j` itself is a collection day by verifying if `d_j % q_i == r_i`. If true, it prints `d_j`.
4. **Calculating Next Day**: If `d_j` is not a collection day, it calculates the smallest non-negative `k` such that `(d_j + k) % q_i == r_i` using the formula `(r_i - (d_j % q_i)) % q_i`. The result is then `d_j + k`, which is printed.

This approach efficiently handles each query by leveraging modular arithmetic to determine the next collection day, ensuring correctness and optimal performance.
