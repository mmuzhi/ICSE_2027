=== ANSWER ===

To solve this problem, we need to determine the "power" of all subarrays of a given array `nums` of length `k`. The power of a subarray is defined as the maximum element in the subarray if all elements are consecutive and sorted in ascending order; otherwise, it is -1. 

### Approach
1. **Special Case Handling for k=1**: If the subarray length `k` is 1, each element in the array is trivially consecutive and sorted, so the power of each subarray is the element itself.
2. **Initialization**: For `k > 1`, initialize an array `results` to store the power of each subarray of length `k`.
3. **First Window Check**: Compute the number of consecutive adjacent pairs in the first subarray (from index 0 to `k-1`). If all `k-1` pairs are consecutive, the subarray's power is its last element; otherwise, it's -1.
4. **Sliding Window Technique**: Slide the window across the array, updating the count of consecutive pairs by:
   - Removing the consecutive condition of the leftmost pair of the current window.
   - Adding the consecutive condition of the new rightmost pair if it exists.
5. **Update Results**: For each window, if the count of consecutive pairs equals `k-1`, the subarray's power is its last element; otherwise, it's -1.

### Solution Code
```python
class Solution:
    def resultsArray(self, nums: List[int], k: int) -> List[int]:
        if k == 1:
            return nums
        
        n = len(nums)
        results = [-1] * (n - k + 1)
        
        consecutive_count = 0
        for j in range(k-1):
            if nums[j+1] == nums[j] + 1:
                consecutive_count += 1
                
        if consecutive_count == k-1:
            results[0] = nums[k-1]
        else:
            results[0] = -1
            
        for i in range(n - k):
            if nums[i+1] == nums[i] + 1:
                consecutive_count -= 1
                
            if i + k < n:
                if nums[i+k] == nums[i+k-1] + 1:
                    consecutive_count += 1
                    
            if consecutive_count == k-1:
                results[i+1] = nums[i+k]
            else:
                results[i+1] = -1
                
        return results
```

### Explanation
1. **Special Case Handling**: If `k` is 1, each element is a valid subarray by itself, so the result is the array itself.
2. **Initialization**: The `results` array is initialized with -1, indicating invalid subarrays initially.
3. **First Window Check**: The first subarray (from index 0 to `k-1`) is checked for consecutive pairs. If all pairs are consecutive, the result for this subarray is its last element.
4. **Sliding Window**: For each subsequent window, the leftmost pair's consecutive condition is removed from the count. If the new rightmost pair (if within bounds) is consecutive, it is added to the count. The result for the current window is set based on whether the count of consecutive pairs matches `k-1`.
5. **Efficiency**: The algorithm efficiently checks each subarray in linear time, making it suitable for the given constraints (array length up to 500). The sliding window technique ensures that each element is processed a constant number of times, maintaining optimal performance.
