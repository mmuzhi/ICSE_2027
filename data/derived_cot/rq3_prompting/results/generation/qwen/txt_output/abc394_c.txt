However, each "WA" is replaced exactly once, and the scanning is done in a single pass after each replacement. The worst-case time complexity is O(n^2), where n is the length of the string. This is because each replacement might require scanning up to n characters, and there could be up to n/2 replacements. Given the constraints, this approach might not be optimal for very large strings, but it is straightforward and handles the problem as specified.

### Solution Code
```python
s = input().strip()
s_list = list(s)
n = len(s_list)
i = 0
while i < n-1:
    if s_list[i] == 'W' and s_list[i+1] == 'A':
        s_list[i] = 'A'
        s_list[i+1] = 'C'
        if i >= 1 and s_list[i-1] == 'W':
            i -= 1
        else:
            i += 1
    else:
        i += 1
print(''.join(s_list))
```

### Explanation
1. **Initialization**: The input string is converted into a list of characters for easier manipulation. The length of the string and the initial pointer position (i) are also noted.
2. **Loop Through String**: The loop runs while the pointer `i` is within the bounds of the string. For each character at position `i`, it checks if the next character forms "WA".
3. **Replacement**: If "WA" is found, it is replaced with "AC". The pointer `i` is then adjusted: if a 'W' precedes the replacement, the pointer is moved left to check for any new "WA" formed by the replacement; otherwise, the pointer is moved right to continue scanning from the next position.
4. **Termination**: The loop exits when no more "WA" substrings are found, and the modified string is printed.

This approach ensures that all "WA" substrings are replaced in the correct order, handling the creation of new "WA" substrings after each replacement by re-scanning from the beginning of the string. While not the most efficient for very large strings, it correctly follows the problem's requirements.
