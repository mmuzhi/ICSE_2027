=== ANSWER ===

To solve this problem, we need to identify the person with the greatest integer among those who have unique integers in the given list. The solution involves checking each person's integer to see if it appears exactly once in the list. If multiple such integers exist, we select the largest one and return the corresponding person's label. If no such integer exists, we return -1.

### Approach
1. **Frequency Calculation**: First, we calculate the frequency of each integer in the list using a dictionary. This helps us determine which integers are unique (frequency 1).
2. **Identify Unique Integers**: We then iterate through the list to find all integers that appear exactly once.
3. **Find Maximum Unique Integer**: Among these unique integers, we identify the largest one.
4. **Determine Label**: The label of the person corresponding to this largest unique integer is then computed and returned. If no unique integers are found, we return -1.

This approach efficiently processes the input in linear time, making it suitable for large input sizes as specified in the problem constraints.

### Solution Code
```python
import sys
from collections import defaultdict

def main():
    data = sys.stdin.read().split()
    if not data:
        print(-1)
        return
    
    n = int(data[0])
    A = list(map(int, data[1:1+n]))
    
    freq = defaultdict(int)
    for num in A:
        freq[num] += 1
        
    max_val = -1
    candidate_index = -1
    
    for i, num in enumerate(A):
        if freq[num] == 1:
            if num > max_val:
                max_val = num
                candidate_index = i
                
    if candidate_index == -1:
        print(-1)
    else:
        print(candidate_index + 1)

if __name__ == "__main__":
    main()
```

### Explanation
1. **Reading Input**: The input is read all at once for efficiency, especially given the constraints. The first value is the number of people, `n`, followed by the list of integers `A`.
2. **Frequency Dictionary**: A dictionary `freq` is used to count occurrences of each integer in the list. This allows quick lookup to check if an integer is unique.
3. **Finding Maximum Unique Integer**: The code iterates through the list, checking each integer's frequency. If an integer is unique (frequency 1), it compares it with the current maximum unique integer. If it's larger, it updates the maximum and records the index.
4. **Output Result**: After processing all integers, if a unique integer was found, the corresponding label (index + 1) is printed. If no unique integers were found, -1 is printed.

This approach ensures optimal performance with a time complexity of O(N), where N is the number of people, by efficiently leveraging dictionary operations and a single pass through the list.
