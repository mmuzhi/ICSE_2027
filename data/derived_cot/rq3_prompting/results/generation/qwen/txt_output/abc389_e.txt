To solve this problem, we need to maximize the number of units purchased given a budget constraint where each product's cost is a quadratic function of the number of units bought. The challenge is to efficiently determine the optimal distribution of units across products to stay within the budget while maximizing the total units purchased.

### Approach
1. **Problem Analysis**: The problem involves purchasing units from multiple products where each product's cost is proportional to the square of the units bought. The goal is to maximize the total units bought without exceeding a given budget. The key observation is that products with lower cost factors (P_i) should be prioritized to minimize the total cost for a given number of units.

2. **Intuition**: The minimal cost for purchasing T units is achieved by always selecting the product with the smallest current marginal cost (which increases as more units are bought from that product). This greedy approach ensures that each unit is bought at the lowest possible incremental cost.

3. **Algorithm Selection**: We use a priority queue (min-heap) to efficiently manage the current marginal costs of all products. For each unit to be purchased, we select the product with the smallest marginal cost, update its marginal cost, and check if the total cost remains within the budget. This approach ensures that we always pick the most cost-effective product at each step.

4. **Complexity Analysis**: The algorithm involves binary search over possible values of T (total units) and for each T, a simulation using a priority queue. The binary search runs in O(log T) time, and each simulation step involves O(log N) operations for heap updates. Given the constraints (N up to 2×10^5 and T up to 10^6), the total complexity is manageable.

### Solution Code
```python
import heapq
import sys
import math

def main():
    data = sys.stdin.read().split()
    if not data:
        return
    n = int(data[0])
    M_val = int(data[1])
    P_list = list(map(int, data[2:2+n]))

    def feasible(T):
        if T == 0:
            return 0 <= M_val
        heap = []
        for i in range(n):
            heapq.heappush(heap, (P_list[i], i, 0))
        total_cost = 0
        count = 0
        while count < T:
            marginal_cost, idx, count_i = heapq.heappop(heap)
            total_cost += marginal_cost
            if total_cost > M_val:
                return False
            count += 1
            if count == T:
                break
            new_marginal = marginal_cost + 2 * P_list[idx]
            heapq.heappush(heap, (new_marginal, idx, count_i+1))
        return total_cost <= M_val

    upper_bound = min(10**6, int(math.isqrt(M_val)) + 1000000)
    low = 0
    high = upper_bound
    ans = 0
    while low <= high:
        mid = (low + high) // 2
        if feasible(mid):
            ans = mid
            low = mid + 1
        else:
            high = mid - 1

    print(ans)

if __name__ == '__main__':
    main()
```
