Since we want to maximize sum, we want to assign as high as possible, but must ensure distinctness. The standard approach: sort maximumHeight descending. For each tower in sorted order, we assign the min of its maximumHeight and the previously assigned height-1 (to ensure distinctness). After sorting descending, for each h in sorted list, assign the height = min(h, curr). Then update curr = assigned - 1. If assigned < 1, return -1. Sum all assigned.

Why this works? Greedy: we want to assign as high as possible, and the largest maximum heights should get the largest heights. Sorting descending ensures we consider the biggest constraints first. For each, we try to assign the maximum possible that is <= h and also distinct from previously assigned (which are larger). So we take the min of h and the next available integer (which is previous assigned - 1). This yields optimal sum.

=== ANSWER ===
```python
from typing import List

class Solution:
    def maximumTotalSum(self, maximumHeight: List[int]) -> int:
        # Sort descending so we process the largest constraints first
        maximumHeight.sort(reverse=True)
        total = 0
        next_available = None  # The largest height we can still assign

        for h in maximumHeight:
            if next_available is None:
                assign = h
            else:
                assign = min(h, next_available)

            if assign <= 0:
                return -1

            total += assign
            next_available = assign - 1

        return total
```
