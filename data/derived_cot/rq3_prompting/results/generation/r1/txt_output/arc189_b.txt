There are N pieces placed on a number line. Initially, all pieces are placed at distinct coordinates.
The initial coordinates of the pieces are X_1, X_2, \ldots, X_N.
Takahashi can repeat the following operation any number of times, possibly zero.

Under the constraints of this problem, it can be proved that all pieces always occupy distinct coordinates, no matter how one repeatedly performs the operation.

His goal is to minimize the sum of the coordinates of the N pieces.

- 0 \leq X_1 < X_2 < \cdots < X_N \leq 10^{12}
- All input values are integers.

1 5 7 10

Sample Output 1

21

- The coordinates of the 1st and 4th pieces in ascending order of coordinate are 1 and 10, so the coordinate of M in this operation is (1 + 10)/2 = 5.5.

- The 3rd piece from the left moves from coordinate 7 to 5.5 - (7 - 5.5) = 4.

```python
import sys

def solve() -> None:
    it = iter(sys.stdin.read().split())
    N = int(next(it))
    X = [int(next(it)) for _ in range(N)]

    # gaps between consecutive coordinates
    gaps = [X[i+1] - X[i] for i in range(N-1)]

    # separate by parity of the gap index (1-indexed)
    odd_gaps = []
    even_gaps = []
    for i, g in enumerate(gaps):
        # i = 0 -> gap index 1 (odd)
        if i % 2 == 0:
            odd_gaps.append(g)
        else:
            even_gaps.append(g)

    # sort each group ascending
    odd_gaps.sort()
    even_gaps.sort()

    # base sum: N * leftmost coordinate
    total = N * X[0]

    # assign odd gaps to odd positions (1,3,5,...)
    for i, g in enumerate(odd_gaps):
        pos = 2 * i + 1   # 1-indexed gap index
        weight = N - pos
        total += weight * g

    # assign even gaps to even positions (2,4,6,...)
    for i, g in enumerate(even_gaps):
        pos = 2 * i + 2   # 1-indexed gap index
        weight = N - pos
        total += weight * g

    print(total)

if __name__ == "__main__":
    solve()
```
