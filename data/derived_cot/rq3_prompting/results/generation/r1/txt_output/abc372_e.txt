We need to solve a dynamic connectivity problem with queries that add edges (only, no deletions) and ask for the k-th largest vertex in the connected component of a given vertex, where k <= 10. N, Q up to 2e5.

We need to answer queries online (process in order). Since edges are only added, we can use union-find (DSU). However, we need to retrieve the k-th largest vertex in the component quickly. Since k <= 10, we can maintain for each component a multiset (or sorted list) of the top 10 largest vertices. Specifically, for each root, we can keep a list (or a heap) of the up to 10 largest vertices in that component. When merging two components, we need to combine these lists: take the union of the two lists, sort descending, keep the top 10 (or less). This is O(10 log 10) per merge, essentially constant. So DSU with union by size/rank and maintaining the top-10 list will be efficient.

We also need to handle queries: for a vertex v, find its root, then if the size of the list (number of vertices in component) is less than k, output -1; else output the k-th largest, i.e., the element at index k-1 (0-index) in the descending list.

Edge cases: The component includes the vertex itself. Initially each vertex is alone, so its list contains just itself.

We need to store the list as a sorted list in descending order. Since we only need top 10, we can just keep a Python list of up to 10 elements. When merging, we can combine the two lists, sort descending, and slice to 10. Or we can merge like merging two sorted lists but it's simpler to just combine and sort. Since the lists are small (<=10), this is fine. However, we may have multiple merges; total merges O(N). Sorting a list of up to 20 elements is constant time.

Implementation steps:
- Read N, Q.
- Initialize DSU parent array of size N+1 (1-indexed).
- Initialize size array (optional for union by size).
- Initialize top list for each node: initially each node's list is [node].
- Process queries:
  - If type 1: u, v. Find roots ru, rv. If ru != rv, merge: union by size (attach smaller to larger). To merge, combine the top lists: new_list = sorted(top[ru] + top[rv], reverse=True)[:10]. Assign to the new root. Update parent.
  - If type 2: v, k. Find root r = find(v). If len(top[r]) < k: print(-1) else print(top[r][k-1]).

import sys

def solve() -> None:
    data = sys.stdin.buffer.read().split()
    it = iter(data)

    N = int(next(it))
    Q = int(next(it))

    parent = list(range(N + 1))
    size = [1] * (N + 1)

    # top[k] = list of the up to 10 largest vertices in the component whose root is k
    top = [[i] for i in range(N + 1)]

    def find(x: int) -> int:
        # path compression
        while parent[x] != x:
            parent[x] = parent[parent[x]]
            x = parent[x]
        return x

    out = []

    for _ in range(Q):
        typ = int(next(it))
        if typ == 1:                     # add edge
            u = int(next(it))
            v = int(next(it))
            ru = find(u)
            rv = find(v)
            if ru != rv:
                # union by size: attach smaller tree to larger one
                if size[ru] < size[rv]:
                    ru, rv = rv, ru
                parent[rv] = ru
                size[ru] += size[rv]

                # merge the two lists of top vertices
                merged = top[ru] + top[rv]
                merged.sort(reverse=True)
                top[ru] = merged[:10]    # keep only the 10 largest
                # top[rv] is no longer needed
        else:                            # type 2: query
            v = int(next(it))
            k = int(next(it))
            r = find(v)
            if len(top[r]) < k:
                out.append("-1")
            else:
                out.append(str(top[r][k - 1]))

    sys.stdout.write("\n".join(out))

if __name__ == "__main__":
    solve()
